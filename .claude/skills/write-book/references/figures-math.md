# Figures & math conventions

How book content should author figures and mathematics so the bundled reader
presents them well. Referenced by the chapter-writer template; the reader
features below shipped 2026-07-04.

## Figures

- A standalone image paragraph `![caption](figures/x.png)` is auto-wrapped by
  the reader into a centered, framed `<figure>` with the alt text as the
  caption, and click-to-zoom (lightbox). So: **always write a real caption in
  the alt text** — it is reader-visible, not metadata.
- Caption text must not contain `]` (pitfalls rule 1: it breaks inline-image
  embedding detection).
- Figures are generated by a Python script committed next to the book (see
  `build_figures.py` pattern), white background, and embedded via the reader
  config's `"inline_images": true`. In dark mode the reader shows them on a
  white plate automatically — do not generate dark-mode variants.
- Complex boxed/route diagrams: Python-generated + P5 rendered-screenshot
  check (pitfalls rule 2); hand-drawn ASCII only for simple boxes that
  `check_diagrams.py` can actually validate.

## Mathematics

Two supported registers — pick ONE per book at P1 and record it in the style
guide:

1. **Unicode math (default).** Inline math written with Unicode
   (×, ≥, ∫₀¹x²dx, ℵ_α, n(n+1)/2); multi-line derivations in fenced blocks.
   No reader config needed. Right for books with light-to-moderate math.
   Subscript convention: when a variable subscript has no clean Unicode
   glyph or reads ambiguously, write it with an underscore (ℵ_α, E_n) and
   use it consistently book-wide — never mix ₙ-style and _n-style for the
   same symbol.
2. **KaTeX (`"math": true` in the reader config).** For math-heavy books.
   Delimiters: `\( ... \)` inline, `$$ ... $$` display — **never single `$`**
   (prose dollar amounts stay literal by design). Fenced code and inline
   code are never rendered as math. Adds ~0.6 MB to the built reader
   (vendored KaTeX + fonts, offline-safe), so it is opt-in per book.
   Display blocks scroll horizontally on narrow screens.

Do not mix registers within a book: a KaTeX book writes ALL formulas in
LaTeX; a Unicode book never uses `\(` / `$$`.
