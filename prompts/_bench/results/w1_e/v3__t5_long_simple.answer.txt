```python
def chunk(lst, n):
    """
    Split lst into consecutive chunks of size n.

    The final chunk may be shorter than n if len(lst) is not evenly divisible.

    Args:
        lst: Any sequence (list, tuple, str, etc.).
        n:   Chunk size. Must be a positive integer.

    Returns:
        A list of chunks (same type as slices of lst).

    Raises:
        ValueError: If n <= 0.
    """
    if n <= 0:
        raise ValueError(f"Chunk size must be a positive integer, got {n!r}")
    return [lst[i : i + n] for i in range(0, len(lst), n)]
```

Failure modes worth noting:
- **`n <= 0`** — raises `ValueError` with the bad value in the message rather than silently returning an empty list or looping forever.
- **Empty `lst`** — `range(0, 0, n)` is empty, so `[]` is returned correctly without special-casing.
- **`n > len(lst)`** — returns `[lst]`, a single chunk containing the whole list; correct per spec.

What happens if a caller passes a lazy iterator instead of a sequence? `lst[i : i + n]` will raise `TypeError` — worth adding a guard or a note in the docstring if that's a real use case.
