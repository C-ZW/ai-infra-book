```python
def chunk(lst, n):
    """
    Split lst into consecutive chunks of size n.

    The final chunk may be shorter than n if len(lst) is not divisible by n.

    Raises ValueError if n <= 0.
    """
    if n <= 0:
        raise ValueError(f"chunk size must be a positive integer, got {n}")
    return [lst[i : i + n] for i in range(0, len(lst), n)]
```

Raises on `n <= 0` rather than returning something silently wrong — an empty list or infinite loop would be the failure modes there.

What's the most dangerous unhandled case in your usage: will `lst` ever be a lazy iterator instead of a sequence? If so, `range(0, len(lst), n)` breaks and you'd need an `iter`-based version instead.
