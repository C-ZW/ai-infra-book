No README exists. Based on the CLAUDE.md and project structure, here's what this project is:

**An AI Infrastructure career-transition book** (《AI Infra 轉職書》) written in Traditional Chinese, paired with an offline single-file reader.

Key components:
- `book/` — the main book: 18 chapters + appendices on AI infrastructure topics, targeted at the author's own career transition. Meta/maintenance files in `book/_meta/` track cross-chapter consistency, style, and time-sensitive facts.
- `books/` — a growing collection of companion books (TLA+, linear algebra, probability, calculus, queueing theory, attention mechanisms, etc.), each under its own slug directory.
- `plan.md` — a companion study/practice roadmap that mirrors the book's structure.
- `web/` — reader config files; the build tools at `../tools/md-reader/` package everything into a self-contained offline HTML reader.
- `bookshelf.html` — a generated shelf page linking all books.

The core workflow: write Markdown → run `build_reader.py` → get a single offline HTML file readable without a server. A diagram linter (`check_diagrams.py`) enforces CJK character-width alignment in ASCII art. A `../profile/` hub centralizes personal context used when generating new books via the `write-book` skill.

What's the most dangerous unhandled case here? Time-sensitive facts (model prices, hardware availability, version numbers) are scattered across 18+ chapters — `maintenance.md` tracks them, but there's no automated staleness alert when a cited version goes EOL.
