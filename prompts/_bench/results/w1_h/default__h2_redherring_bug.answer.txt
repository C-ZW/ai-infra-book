Both tests pass. The root cause was that `summarize` treated `1` (int) and `"1"` (str) as distinct dict keys, splitting user 1's total across two buckets. Normalizing with `str()` at the boundary merges them correctly.

What's the most dangerous unhandled case here: what happens if `user_id` is `None` or a float like `1.0`? `str(None)` → `"None"` and `str(1.0)` → `"1.0"`, both silently accepted as valid users — worth adding a validation check if the upstream can produce those.
