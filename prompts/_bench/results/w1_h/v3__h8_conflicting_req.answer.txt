There's a direct conflict here. The docstring is an explicit constraint against adding side effects — not a suggestion:

> **PURE FUNCTION — must stay free of side effects.** It is memoized by callers and invoked inside hot retry loops; any I/O or logging here would break those callers and add overhead on the hot path.

Adding logging inside `compute_checksum` would:
1. Break memoization — the cached result would skip the log, making call counts wrong.
2. Add I/O overhead on every hot-path retry.

Two cleaner options:

**A) Instrument at the call sites** — add `logger.debug(...)` wherever `compute_checksum(...)` is called. Call-level context (what data, which caller) is often more useful there anyway.

**B) Wrap it** — create a thin, non-memoized wrapper that logs and delegates:

```python
def compute_checksum_traced(data: bytes) -> str:
    result = compute_checksum(data)
    logger.debug("compute_checksum called", extra={"size": len(data), "digest": result[:8]})
    return result
```

Then route non-hot-path callers through the wrapper and leave hot-path/memoized callers on the bare function.
