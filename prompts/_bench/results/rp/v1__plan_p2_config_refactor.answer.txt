## Design decisions (state assumptions, flag the open ones)
- **Dataclass**: `@dataclass(frozen=True)` in `app/config.py`, fields `timeout:int=30`, `retries:int=3`, `verbose:bool=False`. Defaults equal to today's dict values, so `Settings()` reproduces current behavior exactly тЖТ "behavior identical" holds. `frozen=True` makes a passed-around instance safe to share (no accidental mutation re-creating a hidden global).
- **Passing style**: add a **required positional** `settings` param to each function (`connect(host, settings)`, `run(host, settings)`, `log(message, settings)`). Required, *not* defaulted тАФ a `settings=Settings()` default would evaluate once at def-time and become a de-facto shared global, defeating the task. Attribute access (`settings.timeout`) replaces subscript (`CONFIG["timeout"]`); types stay int/int/bool.
- **"Constructed once" site**: there is no application entry point in this repo, so the single construction lives at each top-level caller. Currently that's only the tests тАФ I'll construct one `Settings()` (a `pytest` fixture, or a module-level constant) and thread it into both calls.

## Ordered steps
1. **`app/config.py`** тАФ delete the `CONFIG` dict; add:

2. **`app/client.py`** тАФ drop `from app.config import CONFIG`; change `def connect(host):` тЖТ `def connect(host, settings):`; body uses `settings.timeout`, `settings.retries`.
3. **`app/runner.py`** тАФ drop `from app.config import CONFIG`; change `def run(host):` тЖТ `def run(host, settings):`; `conn = connect(host, settings)` (thread it through), `deadline = settings.timeout`. Keep `from app.client import connect`.

## Risks / failure modes
- **Forgotten thread-through**: the one cross-function call is `run`тЖТ`connect`. If `run` doesn't forward `settings`, `connect` raises `TypeError` (missing arg) тАФ caught immediately by step 4. This is the highest-value thing to get right.
- **API break for external consumers**: if anything *outside* this repo imports `app.config.CONFIG` or calls these functions with the old arity, removing the global / changing signatures is a breaking change. Repo-internally there are none, but I can't see external callers тАФ see clarifying questions.
- **Misuse тЖТ AttributeError not KeyError**: a wrong field name now raises `AttributeError` instead of the old `KeyError`; passing `settings=None` raises `AttributeError`. Tests don't exercise misuse, so behavior on the happy path is identical. I will *not* add input validation (`__post_init__` range/type checks) because that would change behavior beyond the task тАФ flagging it as optional hardening only.
- **Empty `app/__init__.py`**: confirmed 0 bytes, so no re-export of `CONFIG` to clean up.
