# ch04 — 指數分布：排隊世界的原子

> **本章解決什麼問題**：ch03 把機率工具磨利了，現在要安裝排隊理論的原子——指數分布。M/M/1 的兩個 M 都是「指數」的縮寫，整個 Part III 的可解性都建立在它身上。本章要把無記憶性從一條定義變成化簡推導的武器、用 hazard rate 回答「等越久是好消息還是壞消息」、並完整證明競賽引理——它是 ch07 組裝 CTMC 的引擎。最後誠實檢討：這顆原子在現實中哪裡成立、哪裡不成立。

## 從你已知的出發

你調過 HTTP client 的 timeout。凌晨兩點，一支平常 100 ms 的 query 已經跑了 2 秒，你盯著它想：「都跑這麼久了，應該快好了吧？」還是「跑這麼久了，八成是卡死了，砍掉重試比較快」？這兩種直覺指向完全相反的操作，而且你兩種都用過。

這個決策有精確的數學名字：hazard rate（風險率）。本章會給你判準——答案取決於服務時間分布的形狀，而指數分布正好是兩種直覺的分水嶺：在指數的世界裡，「已經等了多久」是徹底無用的資訊。這就是無記憶性（memorylessness），它聽起來像缺陷，卻是讓整個 Part III 的模型可以手算的原因。

同一個影子也出現在你做過的容量決策裡：SQS consumer pool 有兩個 worker 都在處理訊息，新訊息進來要等多久才輪得到？你大概心算過「每則平均 100 ms，它們做到一半，所以再等 50 ms 左右」。這個心算對不對、為什麼「碰巧」接近正確答案但推理全錯，本章的 worked example 會把它拆開重算。本章交付三樣武器：無記憶性（含它為什麼是連續世界的獨家）、hazard rate（timeout 直覺的分類學）、競賽引理（多台機器賽跑的完整代數）。

先排除一個命名地雷：你寫過的 exponential backoff，那個「指數」是重試間隔 2ⁿ 倍增的指數成長，是一個確定性排程；本章的「指數」是機率分布。兩者唯一的關係是常被放進同一句話裡。

## 指數分布的基本量

隨機變數 S ~ Exp(μ)（μ > 0 是 rate 參數）的定義（t ≥ 0）：

```text
密度    f(t)    = μe^(−μt)
CDF     F(t)    = 1 − e^(−μt)
tail    P(S>t)  = e^(−μt)        ← 全章主角，比 CDF 好用
```

用 ch03 的 tail 積分公式 E[X] = ∫₀^∞ P(X>t) dt（適用條件：X ≥ 0）：

```text
E[S]  = ∫₀^∞ e^(−μt) dt = 1/μ
E[S²] = ∫₀^∞ 2t·P(S>t) dt = 2∫₀^∞ t·e^(−μt) dt = 2/μ²
Var(S) = 2/μ² − (1/μ)² = 1/μ²
C²ₛ   = Var/E² = 1                ← 指數的身分證
```

C² = 1 就是 ch03 那把刻度尺的原點刻在這裡的原因：指數是「叢發程度」的基準分布。參數 μ 叫 rate 不是裝飾：等一下你會看到 μ·δ 正是「接下來 δ 這一小段時間內完成」的機率（hazard rate 一節兌現），所以「服務率 10 則/s」與「E[S] = 100 ms」是同一句話的兩種讀法。

指數從哪裡來？它是幾何分布的連續極限，這個推導值得走一遍，因為它解釋了無記憶性為什麼會遺傳。把時間切成寬 δ 的小格，每格獨立地以機率 p = μδ「完成」——這是 ch03 的幾何分布。存活超過 t（即前 t/δ 格都沒完成）的機率：

```text
P(X > t) = (1 − μδ)^(t/δ)  →  e^(−μt)   當 δ → 0
                                         ← 用 (1 − x/n)ⁿ → e^(−x)
```

幾何分布無記憶（已丟 n 次沒中，不改變後面的賠率），極限自然把這個性質帶進連續世界。「指數 = 每一瞬間擲一枚極小機率的硬幣」是本章最值得內化的心像，競賽引理的直覺全靠它。

取 μ = 10/s（推播 worker 的基準，E[S] = 100 ms）感受形狀：

- 中位數 = ln 2/μ ≈ 69.3 ms——**一半以上的工作比平均值快**，平均被右尾拉高。
- P(S > 100 ms) = e^(−1) ≈ 0.368：超過平均的機率約 37%。
- P99 = ln 100/μ ≈ 461 ms ≈ 4.6 × 平均。任何指數分布的 P99 都是平均的 ln 100 ≈ 4.6 倍——這是 ch08「M/M/1 的 P99 ≈ 4.6× 平均」金句的源頭，也是你日後檢查直方圖「像不像指數」的快篩之一。

## 無記憶性：定義、證明、唯一性

**定義**：非負隨機變數 X 無記憶，若對所有 s, t ≥ 0：

```text
P(X > s + t | X > s) = P(X > t)
```

白話：已經活過 s，再活 t 的機率跟剛出生時一樣。過去不留任何資訊。

**指數分布無記憶（完整推導）**：

```text
P(S > s+t | S > s) = P(S > s+t) / P(S > s)     ← 條件機率定義；{S>s+t} ⊂ {S>s}
                   = e^(−μ(s+t)) / e^(−μs)
                   = e^(−μt) = P(S > t)         ∎
```

兩行。但反方向才是重點：**連續世界裡只有指數分布無記憶**。

**唯一性（工程師的嚴謹：每步能口頭說出理由）**。令 G(t) = P(X > t)，G(0) = 1，G 單調不增、右連續。無記憶性等價於函數方程 G(s+t) = G(s)·G(t)。

先清理退化情形：若某個 t₀ > 0 使 G(t₀) = 0，則 G(t₀/2)² = G(t₀) = 0，反覆對半切，由右連續性逼出 G(0) = 0，矛盾——所以 G 處處為正。若 G(1) = 1，同樣的方程逼出 G(n) = 1 對所有 n，X 不是真正的（有限的）隨機變數。所以 G(1) = c ∈ (0, 1)。然後：

```text
G(n)   = G(1)ⁿ = cⁿ                    ← 對 n 歸納：G(n) = G(n−1)·G(1)
G(1/m) = c^(1/m)                       ← 因為 G(1) = G(1/m + … + 1/m) = G(1/m)^m
G(n/m) = c^(n/m)                       ← 前兩行合起來：所有正有理數被釘住
G(t)   = c^t（所有實數 t ≥ 0）          ← 無理數 t 用有理數上下夾，靠單調性收口
       = e^(−μt)，其中 μ = −ln c > 0    ∎
```

值得停一下的是最後一步：光有函數方程不夠——Cauchy 函數方程在不加任何正則條件時存在病態解。救你的是「G 是 tail 函數所以天生單調」這個免費的禮物。這就是本證明標「工程師的嚴謹」而非「直覺版」的原因：每一步都站得住，只是測度層級的細節我們用單調性一筆繳清。

離散世界的對應角色是幾何分布（ch03 見過）：「還要再丟幾次硬幣」不因已經丟過幾次而改變。幾何是離散世界唯一的無記憶分布，指數是它的連續極限。

**殘餘壽命**。無記憶性的直接改寫：已經進行了 t 的工作，剩餘時間 R = X − t（條件在 X > t 上）滿足 P(R > x | X > t) = P(X > x)——**剩餘時間的分布跟全新的一模一樣**。一台已經服務了 700 ms 的 worker，剩餘服務時間仍然是 Exp(μ)、期望仍然是完整的 1/μ。這對等下的 worked example 是關鍵。一般分布的殘餘壽命是另一個世界——你觀察到的工作會系統性偏長（inspection paradox），那是 ch12 的主角，這裡只立牌預告。

為什麼說無記憶性是**化簡推導的武器**，而不只是一條性質？因為它讓「系統現在長什麼樣」足以預測未來，不必拖著歷史。想描述一個佇列系統的狀態，照理你得記錄：裡面有幾個工作、每個正在服務的工作**已經做了多久**、下一個到達**已經醞釀了多久**。後兩項是連續量，狀態空間瞬間爆炸。但若服務與到達間隔都是指數的，「已經做了多久」對未來毫無資訊量——記了也白記——於是**一個整數（系統內人數）就是完整狀態**。ch06 的口號「狀態吃掉歷史」在佇列世界能成立，全靠指數分布把歷史變得不值得吃。這就是 M/M/1 能用一條生滅鏈解掉、而 M/G/1 需要另起爐灶（ch12）的根本原因。

回答開場的 timeout 問題（指數版）：如果服務時間真的是指數的，「已經等了 2 秒」不該改變你對剩餘時間的任何估計，timeout 也就沒有理由因此調整。但「如果」兩個字很重，往下看。

## Hazard rate：「等越久越快來」是誰的性質

把無記憶性放進更大的座標系。對有密度的非負隨機變數（且 P(X>t) > 0），定義 hazard rate：

```text
h(t) = f(t) / P(X > t)
```

讀法：h(t)·δ ≈ P(在 (t, t+δ] 內結束 | 已撐過 t)——「下一瞬間完成」的條件速率。它完全決定分布：對 ln P(X>t) 微分可得 P(X > t) = e^(−∫₀ᵗ h(u)du)。三種形狀，三種世界觀：

```text
 h(t)
  │*
  │ *          DFR：hyperexponential、Pareto（見 ch15）
  │  **             —— 撐越久，完成率越低，「等越久剩越久」
  │    ***
  │       *****
  │·····················  exponential：h(t) = μ，永遠一樣
  │        _____
  │    ___/
  │ __/
  │/           IFR：uniform、Erlang、固定工作量的 batch job
  └──────────────────────► t    —— 跑越久離終點越近，「等越久越快來」
```

- **IFR**（increasing failure rate）：確定性工作是極端例——一支固定要跑 10 分鐘的 batch job，跑到第 9 分鐘時 h 飆高，「快好了」的直覺**合法**。
- **指數**：h(t) ≡ μ，唯一的常數線。無記憶性的幾何化身。
- **DFR**（decreasing failure rate）：「跑越久代表剩越久」。混合分布天生往這邊倒，理由是貝氏的：撐得久這件事本身就是「你抽到的是慢工作」的證據（直覺版；混合指數恆為 DFR 的一般證明本書不證，見延伸閱讀）。

DFR 用數字感受一下。取 ch13 將沿用的兩點混合：90% 短工作 Exp(20)（平均 50 ms）、10% 長工作 Exp(1/0.55)（平均 550 ms），整體平均仍是 100 ms。h(0) = 0.9·20 + 0.1·1.82 ≈ 18.2/s；t → ∞ 時 h(t) → 1.82/s——掉了一個數量級。具體地：P(是長工作 | 已跑 1 秒) ≈ 1.000（短工作撐過 1 秒的機率是 e^(−20) ≈ 2×10⁻⁹，可忽略），所以 E[剩餘 | 已跑 1 秒] ≈ 550 ms——是全新工作平均值的 5.5 倍。**「跑很久的 query 砍掉重試」在 DFR 世界是定理級正確的操作**；真實服務時間常常比這更重尾（ch15）。

所以開場那兩種直覺都對，只是各屬於不同的分布形狀，而指數是中間那條分水嶺。判斷你在哪個世界，不需要擬合整條曲線，三個粗篩就夠：看直方圖估 C²（ch03 的方法；C² > 1 強烈暗示 DFR 型混合）、看 median/mean 比值（指數應 ≈ 0.693，重尾會把 mean 拉得遠高於 median）、看 P99/mean（指數應 ≈ 4.6）。更直接的辦法：從既有的 latency log 算「已經跑超過 t 的請求，最後總共跑了多久」的平均，對幾個 t 畫出來——這條「條件剩餘時間」曲線平的是指數、上升的是 DFR、下降的是 IFR。資料你本來就在收，只是從來沒這樣切過。

## 競賽引理：CTMC 的引擎

接下來這組引理是本章的主菜。設 X₁, …, X_k **獨立**、Xᵢ ~ Exp(λᵢ)，想像 k 個碼錶同時起跑，M = min(X₁, …, X_k) 是第一個響的時刻，I 是響的那顆的編號。記 Λ = λ₁ + … + λ_k。

**引理一（最小值）**：M ~ Exp(Λ)。完整推導：

```text
P(M > t) = P(X₁ > t, …, X_k > t)
         = Π P(Xᵢ > t)              ← 獨立性，唯一用到的地方
         = Π e^(−λᵢt) = e^(−Λt)     ∎
```

指數的 tail 是 e 的指數函數，相乘時 rate 直接相加——「k 個指數碼錶一起跑，等同一個 rate 加總的指數碼錶」。注意這是指數的專利：兩個確定性 100 ms 工作的 min 還是 100 ms，不是 50 ms；min 的封閉性不能帶去別的分布。

**引理二（勝者機率）與引理三（獨立性）**：一個積分同時證完。計算聯合機率 P(I = i, M > t)——第 i 顆在某時刻 s > t 響、且其他所有碼錶在 s 時都還沒響：

```text
P(I = i, M > t) = ∫_t^∞ λᵢe^(−λᵢs) · Π_{j≠i} e^(−λⱼs) ds   ← 對「i 在 s 響」做全機率積分
                = ∫_t^∞ λᵢ·e^(−Λs) ds
                = (λᵢ/Λ)·e^(−Λt)                            ∎
```

代 t = 0 得 **P(I = i) = λᵢ/Λ**：勝率正比於速率。而且對所有 t：

```text
P(I = i, M > t) = (λᵢ/Λ)·e^(−Λt) = P(I = i)·P(M > t)
```

聯合機率處處因式分解——**勝者是誰，與比賽花了多久，互相獨立**。

我認為這是全章最反直覺的一條，值得正面盯著它看。直覺說：比賽拖得久，是不是代表快的那顆「失常」、慢的那顆比較可能贏？不是。就算你已經等了 10 倍於平均的時間，下一瞬間誰先響的賠率**仍然是 λᵢ : λⱼ**。用「每瞬間擲硬幣」的心像重講一次（直覺版，與上面的積分證明互為表裡）：在每一小段 δ 裡，第 i 顆碼錶以機率 λᵢδ 響。已知「這一格有人響」，是第 i 顆的條件機率是 λᵢδ/Λδ = λᵢ/Λ——**δ 和「現在是第幾格」都消掉了**。勝負由一場每瞬間賠率恆定的抽籤決定，而抽籤的賠率跟已經抽了幾次無關；無記憶性保證沒有任何一顆碼錶會「越等越燙」。連續分布也保證 P(同時響) = 0，平手不需要處理。

具體數字養一下反射：λ₁ = 2、λ₂ = 1 的兩顆碼錶，min ~ Exp(3)（平均 1/3）、快錶勝率 2/3。已知這場比賽撐了超過 2 個平均長度才分勝負？勝率**還是 2/3**。已知快錶贏了？比賽時長的分布**還是 Exp(3)**。兩個條件化都拿不到任何東西——這就是引理三的全部內容。

（適用條件再說一次，因為它常被偷渡：**獨立**且**各自指數**。兩者缺一，三條引理全部作廢。）

為什麼說它是引擎：ch07 的 CTMC 整個結構就是這三條引理。系統停在一個狀態，每條出邊是一顆獨立的指數碼錶——引理一說停留時間是 Exp(出邊速率總和)，引理二說跳去哪裡的機率正比於速率，引理三說「跳去哪」與「停多久」可以拆開處理。沒有第三條，「狀態 + 速率」就不足以描述系統，整套生滅過程（ch07）與 M/M/1（ch08）的手算都不存在。

## Worked example：兩台都在忙的 worker pool

基準場景：推播 pool 有兩台 worker，服務時間各自獨立 ~ Exp(μ)、μ = 10/s（平均 100 ms）。你的 job 到達時兩台都在忙，而且你排第一。問：(a) 平均多久輪到你開始？(b) 你的總逗留時間 E[T]？

**先列直覺會怎麼算錯**（兩種都常見）：

- 錯法一：「每台平均做到一半，剩 50 ms，所以我等 50 ms。」結論碰巧接近，推理全錯——指數的殘餘不是「做到一半」，是完整重來的 Exp(10)、平均 100 ms。這個錯法在非指數世界會立刻露餡：若服務是確定性 100 ms、相位隨機，殘餘是 Uniform(0, 100 ms)，兩台的 min 平均是 100/3 ≈ 33.3 ms，不是 50。
- 錯法二：「殘餘還是 Exp(10)、平均 100 ms，所以我等 100 ms。」前半對，後半忘了你在跟**兩台**賽跑——你只需要其中一台空出來。

**正解，一步一步**：

```text
 t = 0: both workers busy, you are first in queue

 worker A ──[residual ~ Exp(10)]──┐
                                  ├──► W = min ~ Exp(20), E[W] = 1/20 s
 worker B ──[residual ~ Exp(10)]──┘

 then your own service: S ~ Exp(10), E[S] = 1/10 s
```

1. 無記憶性：兩台的**殘餘**服務時間各自仍是 Exp(10)，不管它們已各自做了多久。
2. 競賽引理一：你的等待 W = min(R_A, R_B) ~ Exp(10+10) = Exp(20)，**E[W] = 1/20 s = 50 ms**。
3. 引理二與三的順帶禮物：哪台先空出來是 50/50，而且就算你等了 400 ms 才輪到，仍然是 50/50。
4. 你自己的服務在**單一台** worker 上跑，S ~ Exp(10)、與 W 獨立（你的工作量跟前人何時做完無關）。注意你的服務速率是 10 不是 20——pool 加速的是等待，不是你自己被服務的速度（這就是 ch09 裡 E[T] 永遠 ≥ 1/μ 的地板）。

```text
E[T] = E[W] + E[S] = 1/20 + 1/10 = 0.15 s = 150 ms
Var(T) = 1/20² + 1/10² = 0.0025 + 0.01 = 0.0125 s²
C²_T  = 0.0125 / 0.15² = 0.556 < 1     ← 兩段獨立指數相加，比單一指數更「規律」
```

順手推廣：c 台 worker 全忙、你排第一，等待是 c 路殘餘 Exp(μ) 的競賽，W ~ Exp(cμ)——pool 越大，「下一個空位」來得越快，但你自己的服務永遠是 Exp(μ)。這個「等待縮短、服務不變」的結構就是 ch09 整章的骨架。

多走一步，把 T 的整個分布拿下（完整推導，全機率切「等待是否已超過 t」）：

```text
P(T > t) = P(W > t) + ∫₀^t 20e^(−20w)·e^(−10(t−w)) dw
         = e^(−20t) + 20e^(−10t)·(1 − e^(−10t))/10
         = 2e^(−10t) − e^(−20t)
```

自我複核：t = 0 時得 2 − 1 = 1 ✓；對 tail 積分 2/10 − 1/20 = 0.15 = E[T] ✓。數值感受：P(T > 300 ms) = 2e^(−3) − e^(−6) ≈ 0.0971；解 P(T > t) = 0.01 得 t₉₉ ≈ 530 ms ≈ 3.53 × 平均——C² < 1 的分布，tail 比指數的 4.6× 緊，與上面 C²_T = 0.556 互相印證。

## 指數假設的誠實檢討

M/M/1 的兩個 M 是兩個獨立的賭注，勝率很不一樣。

**到達間隔：常常賭得過**。大量互相獨立、個別稀疏的使用者匯流，疊加起來趨近 Poisson 過程，到達間隔自動變成指數——這不是巧合而是極限定理（Palm–Khintchine），ch05 專章處理，這裡只說結論：你的 ~10M 訊息/月推播流，在分鐘級時間窗內把到達間隔當指數，通常是體面的近似。什麼時候破：retry 迴圈（到達和系統狀態相關）、cron 觸發的批次上游、明顯的日夜尖峰（非齊次，ch05）。

**服務時間：常常賭輸**。真實服務時間直方圖很少長得像指數：cache hit/miss 造成雙峰（C²ₛ 遠大於 1 的混合）、固定大小 payload 接近確定性（C²ₛ ≈ 0）、慢 query 與大檔案造成重尾（ch15）。指數服務的真正角色是**分析上的方便**：無記憶性讓「系統內人數」單獨就構成 Markov 狀態（ch07/ch08 兌現）。賭輸的帳單在 ch12 結算：等待時間正比於 (1 + C²ₛ)/2，基準數字下把 C²ₛ = 10 的現實硬套指數模型，預測 0.4 s、實際 2.2 s——低估 5.5 倍。

那為什麼整個 Part III 還是要在指數假設上蓋房子？三個誠實的理由。第一，它是唯一能讓你**手算到底**的世界，而手算過一次完整模型的人，才有資格判斷近似什麼時候可信——M/M/1 是訓練場，不是終點。第二，許多 M/M 結論的**形狀**在一般分布下存活，只是係數變了：等待對 1/(1−ρ) 的爆炸形狀（ch08）原封不動地出現在 P-K（ch12）與 Kingman（ch14）裡，指數世界先讓你看清形狀。第三，有些結論意外地根本不依賴指數——哪些是、哪些不是，正是 ch12 之後的劇情主線。

決策版總結：到達端的指數假設先信後驗（驗法 ch05）；服務端的指數假設**永遠先量 C²ₛ 再說**（量法 ch03 已給）。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦與自我察覺 |
|---|---|---|
| 把無記憶性當通則 | 「等多久都一樣」只屬於指數；拿去推論一般等待，timeout/重試決策全錯 | 先看直方圖：C²ₛ ≈ 1、median/mean ≈ 0.693、P99/mean ≈ 4.6 三項快篩都過才像指數 |
| 「等很久了，應該快到了」 | 只在 IFR 世界合法；DFR/重尾世界正好相反（已跑 1 秒 → 預期再 550 ms 的例子） | 問自己服務時間是固定工作量（IFR）還是混合/重尾（DFR）；不確定就量 |
| 競賽引理偷渡到相關碼錶 | 兩台 worker 共用同一顆 DB、同 host 搶 CPU、GC 同步暫停——完成時間正相關，min 不再是 Exp(Σλ)，等待被低估 | 引理的適用條件唸三遍：獨立＋指數。共享瓶頸＝相關＝引理作廢 |
| min 的封閉性帶去非指數 | 「兩台平均 100 ms，所以 min 平均 50 ms」對確定性服務是錯的（min 仍 100 ms）、對 uniform 殘餘也是錯的（33.3 ms） | 1/Σλ 公式只在各路皆指數時成立；其他分布乖乖算 P(min>t) = ΠP(Xᵢ>t) |
| 服務時間硬套指數 | C²ₛ = 10 的現實 vs 指數模型：等待低估 5.5×（ch12 結帳） | 模型選型前先量 C²ₛ；C²ₛ 離 1 很遠就準備 M/G/1（ch12） |
| exponential backoff ≠ 指數分布 | 命名撞車導致「backoff 間隔是指數分布」的誤讀；backoff 是確定性倍增排程（加 jitter 才有隨機性） | 看到 exponential 先問：成長的指數，還是分布的指數？ |

## 紙上推演

**第 1 題 [10 分鐘 ★]**：闔上書，重講「無記憶性 + 連續 ⇒ 指數」。要求：寫出函數方程、說明退化情形（G 某處為 0、G(1) = 1）怎麼排除、整數→有理→實數三級跳各用了什麼、單調性在哪一步是救命的。

**第 2 題 [15 分鐘 ★★]**：hedged read：同一筆讀取同時發給三個 replica，回應時間獨立、速率分別 20、10、10（每秒）。(a) 第一個回應的期望時間？(b) 快 replica 勝出的機率？(c) 已知這次等了超過 100 ms 才有第一個回應，它來自快 replica 的機率變成多少？(d) 三個 replica 共用同一顆磁碟時，(a) 的答案會往哪個方向錯？

**第 3 題 [15 分鐘 ★★]**：異質雙 worker：μ₁ = 10、μ₂ = 5，兩台都在忙，你排第一。求 E[等待] 與 E[總逗留]。提示：你最後落在哪台 worker 上不是 50/50，而且這個條件化之所以合法，靠的是引理三。

**第 4 題 [10 分鐘 ★★]**：找破綻：「接駁車間隔是平均 10 分鐘的指數分布。我已經等了 9 分鐘，所以平均再等 1 分鐘就到。」指出論證錯在哪、什麼條件下它會變成對的、再多挖一層——「我隨機抵達車站」這件事本身還藏著什麼陷阱（一句話即可，ch12 兌現）。

### 推演解答

**第 1 題**：核心序列：G(s+t) = G(s)G(t)；若 G(t₀) = 0 則對半切配右連續性逼出 G(0) = 0 矛盾，故 G > 0；若 G(1) = 1 則 X 無限大不是真隨機變數，故 G(1) = c ∈ (0,1)；歸納得 G(n) = cⁿ，m 等分得 G(1/m) = c^(1/m)，組合得有理點 G(n/m) = c^(n/m)；無理點用有理數上下夾、單調性保證夾得攏，得 G(t) = c^t = e^(−μt)。常見錯路：跳過退化情形直接開算（被 ln 0 反咬）；把「實數延拓」當免費——沒有單調性時 Cauchy 方程有病態解，這一步才是整個證明的門檻。

**第 2 題**：(a) min ~ Exp(20+10+10) = Exp(40)，E = 1/40 s = 25 ms。(b) 20/40 = 0.5。(c) **還是 0.5**——引理三：勝者身分與比賽時長獨立，「等得久」不是任何 replica 的證據。直覺反對的原因是把「快 replica 沒先回」誤讀成資訊，但無記憶性下所有人每一瞬間重新出生。(d) 共用磁碟 → 回應時間正相關 → 「至少一個快」的好處縮水，真實 E[min] 比 25 ms **大**；極端完全相關時三路退化成一路。hedging 的收益是用獨立性買的。

**第 3 題**：殘餘各為 Exp(10)、Exp(5)；W = min ~ Exp(15)，E[W] = 1/15 ≈ 66.7 ms。落點機率：快台 10/15 = 2/3、慢台 1/3，且與 W 獨立（引理三，所以可以直接相乘相加不必對 W 條件化）：

```text
E[T] = 1/15 + (2/3)·(1/10) + (1/3)·(1/5)
     = 0.0667 + 0.0667 + 0.0667 = 0.200 s
```

常見錯路：用兩台平均服務時間 (100+200)/2 = 150 ms 當你的 E[S]——錯，你比較可能被快台接走，正確的混合是 (2/3)·100 + (1/3)·200 ≈ 133 ms。三項各等於 1/15 是這組數字的小巧合，不是定理。

**第 4 題**：破綻：指數間隔無記憶，已等 9 分鐘後剩餘等待仍是完整的 Exp(1/10 分鐘)，期望還是 10 分鐘，不是 1 分鐘。論證變對的條件：間隔是**確定性** 10 分鐘（IFR 極端），此時已等 9 分鐘 ⇒ 再等 1 分鐘，精確成立。再挖一層：「隨機抵達」的人傾向落在**偏長的間隔裡**——你撞上的那段間隔平均不是 10 分鐘而更長（inspection paradox，見 ch12）。
