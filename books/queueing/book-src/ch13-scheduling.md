# ch13 — 排程：誰先服務的代價與正義

> **本章解決什麼問題**：ch12 推完 P-K 之後留了一個缺口：整套結果蓋在 FCFS 上，而「誰先服務」是你手上最便宜的旋鈕——不加機器、不改業務邏輯，只換出工順序。本章先證明排程動不了什麼（工作量守恆、Kleinrock 守恆律），再算清楚它能動什麼：同一台伺服器、同一批工作，總體平均回應時間可以從 0.245 s 排到 1.17 s，差 4.8 倍。SRPT 的最優性、PS 的 insensitivity、「餓死長工作」的真相都在這裡結案；ch14 接著把到達端的變異數接進來，ch15 再讓重尾世界檢驗本章每個結論。

## 從你已知的出發

你做過的排程決策比你以為的多。SQS consumer 照到達順序逐條處理，是 FCFS；把報表類慢請求踢到獨立 worker pool，是用拓撲實作優先權（ch09 推演第 4 題埋過這顆種子）；K8s 給線上服務配高 PriorityClass、讓 batch pod 被驅逐讓位，是搶占；而你每天寫的 Node.js 服務，單執行緒上幾百個請求的 callback 交錯推進——那是哪種紀律？ch08 留過一句話：Node 的事件迴圈不是 FCFS。本章會給它正確的名字，也會誠實地說它哪裡不配這個名字。

ch12 結尾留了一道具體的題：排程基準——90% 短工作（50 ms）、10% 長工作（550 ms）、λ = 8/s、ρ = 0.8——FCFS 之下平均等待 0.65 s。不動硬體、只動出工順序，能壓到多少？這章把答案算到底，順便分清楚哪些帳是守恆的（你以為賺到的其實是轉嫁給別人），哪些是真的賺。

## 工作量守恆：排程改不了的那條線

定義 V(t)：時刻 t 系統內所有工作的剩餘服務需求總和，叫**未完成工作量（unfinished work／workload）**。它的動態簡單到殘忍：

```
 V(t)  未完成工作量：到達瞬間跳升該工作的 Sᵢ、非空時以斜率 −1 消化
 ▲
 │  |\  |\
 │  | \ | \
 │  |  \|  \
 │  |   |   \        |\
 │  |   |    \       | \
 └──┴───┴────────────┴────▶ t
    a₁  a₂    ↑空閒   a₃  ↑空閒
```

```text
到達時刻 aᵢ：V 跳升 Sᵢ      ← 跳的時刻與高度由到達過程與工作尺寸決定，與排程無關
V > 0 時：  dV/dt = −1     ← work-conserving：伺服器全速消化——消化「誰」不影響總量
V = 0 時：  dV/dt = 0
```

兩條規則裡都沒有「順序」這個字。所以在**每一條 sample path 上**，V(t) 對所有 work-conserving 紀律逐點相同——FCFS、LCFS、優先權、PS、SRPT，全部畫出同一條鋸齒。嚴謹度：**工程師的嚴謹**，每步理由都能口頭講出來；它甚至不需要 Poisson，任何到達過程都成立。但條件要抄下來，每一條都會在工程現實裡被打破：**單一伺服器全速工作（非空不偷懶）、搶占不損失已完成的工作（preempt-resume 而非 restart）、不丟棄工作、切換無成本**。

V 的平均值我們其實已經算過：FCFS 下，到達者的等待恰好等於到達瞬間的 V（伺服器會先消化完這座山才輪到你），PASTA（ch08）說「你看到的＝時間平均」，所以 E[V] = FCFS 的 E[T_Q] = W₀/(1−ρ)，其中 **W₀ ≔ λE[S²]/2** 就是 ch12 鋸齒面積論證的時間平均殘餘服務時間（這個值需要 M/G/1）。基準數字：W₀ = 8×0.0325/2 = 0.13 s，E[V] = 0.13/0.2 = 0.65 s。

這就是本章的第一定理，也是一切排程討論的地基：**排程不創造容量，V(t) 這座山一克都不會變小；排程能動的，只有這座山怎麼攤到每顆人頭上。**

## 不看長度的家族：平均全相同、tail 大不同

第一個攤法家族：**非搶占、且完全不看工作長度**的紀律——FCFS、非搶占 LCFS（最後到的先做）、RANDOM（隨機抽人）。現在把 ch12 那句「平均等待是這家族的紀律不變量」正面證完。

對任何非搶占紀律，V 可以拆成兩塊：服務中那位的殘餘，加上佇列中每位的**完整**尺寸（非搶占：排隊者還沒開始跑）：

```text
E[V] = W₀ + E[Σ 佇列中工作的尺寸]
     = W₀ + E[N_Q]·E[S]        ← 不看長度 ⇒ 佇列長度與尺寸獨立（工程師的嚴謹，同 ch12 那步）
     = W₀ + λE[T_Q]·E[S]       ← Little 用在佇列上（ch02）
     = W₀ + ρ·E[T_Q]

又 E[V] = W₀/(1−ρ)             ← 上一節：V 對全家族不變

⇒ E[T_Q] = W₀/(1−ρ)            ← 跟 FCFS 一字不差
```

多數人第一次聽都不信的就是 LCFS 那檔：**最後到的先服務，平均等待居然跟先到先服務一模一樣**。直覺修復法：LCFS 下有人被瘋狂插隊，但插隊者少等的每一秒，恰好就是被插者多等的那一秒——V 的面積總量固定；而「誰被抽中倒霉」與工作長度無關，所以攤到每顆人頭的期望值不動。注意這個論證用掉三個條件：非搶占（拆 V 那步）、不看長度（獨立性那步）、work-conserving（E[V] 不變那步）。缺一即破——下兩節就是專門去破它的。

平均相同，分布天差地遠。FCFS 的等待頂多是「眼前這座山」；LCFS 的倒霉鬼會被困在一整段忙碌期裡——ch12 算過，一支 2,100 ms 工作開頭的忙碌期平均 10.5 秒，LCFS 下早到的人就在裡面看著後到者一個個先走。精確可引的版本是：**在這個家族裡，FCFS 的等待時間變異數最小**（Kingman 1962，兩頁短文，完整證明見延伸閱讀）；LCFS 在家族裡是另一個極端。所以「換個出工順序救 P99」在這個家族裡是反向操作：平均不動、tail 只會更鬆。**想動平均，必須看長度；想動 tail 而不看長度，FCFS 已經是最優解。**

## Kleinrock 守恆律：等待的總帳與匯率

把工作分成 k 類：第 i 類到達率 λᵢ（總 Poisson 流按類分流，ch05 的 thinning 保證各類仍是獨立 Poisson）、尺寸 i.i.d.、ρᵢ = λᵢE[Sᵢ]。紀律可以看**類別**（包括「類別恰好等於長度」），但維持非搶占、類內不看長度、work-conserving。逐類重複上面的拆帳：

```text
E[V] = W₀ + Σᵢ E[N_Q,i]·E[Sᵢ]      ← 非搶占：排隊者帶完整尺寸；類內不看長度給獨立性
     = W₀ + Σᵢ λᵢE[T_Q,i]·E[Sᵢ]    ← Little 逐類用（ch02）
     = W₀ + Σᵢ ρᵢE[T_Q,i]

⇒ Σᵢ ρᵢ·E[T_Q,i] = ρ·W₀/(1−ρ)      ← Kleinrock 守恆律（M/G/1、非搶占、work-conserving、穩態）
```

這條恆等式是 Kleinrock 1965 年在 Naval Research Logistics Quarterly 上證的（涵蓋的紀律家族比這裡寫的還寬），它值得貼在牆上的原因是右邊**不含任何紀律資訊**：基準數字下永遠等於 0.8×0.13/0.2 = **0.52 s**。你怎麼排，這本總帳一毛不差。

關鍵在幣別。守恆的是 **ρ 加權**的等待，不是人頭加權：人頭平均 = Σ(λᵢ/λ)E[T_Q,i]，權重是 λᵢ；總帳權重是 ρᵢ = λᵢE[Sᵢ]，差一個 E[Sᵢ]。所以人頭平均完全可以動——**把等待搬到「每顆人頭攜帶較多工作量」的類別身上，總帳不變、人頭帳下降**。短工作優先就是這個套利：每單位 ρ 裡塞了更多顆人頭。守恆律同時也是審計工具：任何「大家都變快」的排程宣傳，拿 0.52 s 一驗便知有沒有人在帳外付錢。

## 看長度出工：非搶占優先權、SJF 與 VIP 的帳單

把套利做成公式。兩類，類 1 優先（非搶占：服務中的工作做完才換人），類內 FCFS。標記一位類 1 到達者，材料全是 ch12 那套——分解、PASTA、Little：

```text
E[T_Q,1] = W₀ + E[N_Q,1]·E[S₁]          ← 等：服務中者的殘餘（含類 2 的！非搶占趕不走）＋ 前面同類
         = W₀ + ρ₁E[T_Q,1]              ← PASTA ＋ Little
⇒ E[T_Q,1] = W₀/(1−ρ₁)

類 2 到達者多等兩塊：到達時已排隊的兩類，加上自己等待期間新來插隊的類 1：
E[T_Q,2] = W₀ + ρ₁E[T_Q,1] + ρ₂E[T_Q,2] + λ₁E[T_Q,2]·E[S₁]
⇒ E[T_Q,2](1−ρ₁−ρ₂) = W₀ + ρ₁·W₀/(1−ρ₁) = W₀/(1−ρ₁)
⇒ E[T_Q,2] = W₀/((1−ρ₁)(1−ρ₁−ρ₂))
```

一般 k 類（σᵢ ≔ ρ₁+…+ρᵢ）：**E[T_Q,i] = W₀/((1−σᵢ₋₁)(1−σᵢ))**。兩個分母各管一個機制：(1−σᵢ₋₁) 是「等待期間不斷插隊的高優先新客」，(1−σᵢ) 是「到達時已經排在前面的人」。適用條件：各類 Poisson、非搶占、類內不看長度、work-conserving、ρ < 1。SJF（Shortest Job First：到達時已知尺寸、每次挑最短的做）就是「類別＝尺寸」的極限：類切到連續時公式變成 E[T_Q(x)] = W₀/(1−ρ(x))²，ρ(x) 是尺寸 ≤ x 的工作貢獻的負載（敘述層級，推導同上但要極限手續）。

### Worked example：同一條管線的五張帳單

排程基準，全程不動：λ = 8/s、E[S] = 100 ms、ρ = 0.8。兩點分布下「短類優先」與 SJF 是同一個紀律（類內尺寸全同，按尺寸排＝按類排）：

```text
類 1（短）：λ₁ = 0.9×8 = 7.2/s、E[S₁] = 0.05 s、E[S₁²] = 0.0025 s²、ρ₁ = 0.36
類 2（長）：λ₂ = 0.8/s、       E[S₂] = 0.55 s、E[S₂²] = 0.3025 s²、ρ₂ = 0.44
W₀ = (λ₁E[S₁²] + λ₂E[S₂²])/2 = (0.018 + 0.242)/2 = 0.13 s   ← 與 λE[S²]/2 = 8×0.0325/2 同帳 ✓

FCFS：    E[T_Q] = W₀/(1−ρ) = 0.13/0.2 = 0.65 s（ch12 推演 2b ✓）
短優先：  E[T_Q,1] = 0.13/0.64 = 0.203 s
          E[T_Q,2] = 0.13/(0.64×0.2) = 1.02 s
長優先：  E[T_Q,2] = 0.13/0.56 = 0.232 s
          E[T_Q,1] = 0.13/(0.56×0.2) = 1.16 s
搶占短優先（公式見下節）：
          E[T_Q,1] = λ₁E[S₁²]/(2(1−ρ₁)) = 0.009/0.64 = 0.0141 s
          E[T_2]   = 0.55/0.64 + 1.02 = 1.88 s ⇒ E[T_Q,2] = 1.33 s
PS（見後）：E[T(x)] = x/(1−ρ) = 5x ⇒ E[T_Q,1] = 0.2 s、E[T_Q,2] = 2.2 s
```

| 紀律 | E[T_Q] 短 | E[T_Q] 長 | E[T_Q] 總體 | E[T] 總體 | Σρᵢ·E[T_Q,i] |
|---|---|---|---|---|---|
| FCFS | 0.65 s | 0.65 s | 0.65 s | 0.75 s | 0.52 ✓ |
| 短優先（非搶占）＝SJF | 0.203 s | 1.02 s | 0.284 s | 0.384 s | 0.36×0.203+0.44×1.02 = 0.52 ✓ |
| 長優先（非搶占） | 1.16 s | 0.232 s | 1.07 s | 1.17 s | 0.36×1.16+0.44×0.232 = 0.52 ✓ |
| 短優先（搶占） | 0.0141 s | 1.33 s | 0.145 s | 0.245 s | 0.588 ✗（不在守恆家族） |
| PS | 0.20 s | 2.20 s | 0.40 s | 0.50 s | 1.04 ✗（不在守恆家族） |

（總體 E[T_Q] = 0.9×短 + 0.1×長；E[T] = E[T_Q] + 0.1。）三張非搶占帳單的 ρ 加權全部對上 0.52——守恆律自我複核通過；兩張搶占帳單對不上，**這不是算錯，是它們不在守恆律的家族裡**（搶占把「排隊者帶完整尺寸」那步拆帳弄假了）。

讀這張表的三個重點。第一，ch12 的支票兌現：不動硬體，0.65 → 0.284 s（非搶占看長度），再到 0.145 s（搶占）——等待砍 78%。第二，**長工作沒有付出對等的慘痛**：短優先下長類等待 0.65 → 1.02 s，多付 56%，換來短類少付 69%、總體少付 56%——因為長類每顆人頭背 0.55 s 的工作量，匯率划算。第三，長優先那列是「VIP 通道」的帳單：把優先權給大客戶的批次匯出（重工作），總體平均幾乎翻倍（0.65 → 1.07 s），而守恆律提醒你這不是浪費、是轉嫁——0.52 一毛不差，全部由短請求用高匯率付掉。**VIP 通道在佇列論裡的精確代價：你讓 ρ 大的類插隊，就是讓 ρ 小的類付帳。** 順帶一句更一般的裁決：若第 i 類每秒等待的成本是 cᵢ，非搶占 M/G/1 下最小化平均等待成本的順序是按 cᵢμᵢ 遞減出工（**cμ rule**；μᵢ = 1/E[Sᵢ]）——SJF 是「人人成本相同」的特例，而「VIP = 成本高」只有在 cᵢ 真的高到蓋過 μᵢ 劣勢時才成立。

## 搶占與 SRPT：最優性的內容與代價

非搶占有一塊鐵地板：W₀。短工作再優先，撞上正在服務的 550 ms 工作也只能等——卡車已經上橋。搶占（preempt-resume）拆掉這塊地板。兩類搶占優先權的結果（**敘述層級**，推導見 Harchol-Balter；用「高優先類根本看不見低優先類」自洽檢核）：

```text
E[T_1] = E[S₁] + λ₁E[S₁²]/(2(1−ρ₁))      ← 就是純類 1 的 M/G/1 P-K：類 2 完全隱形
E[T_2] = E[S₂]/(1−ρ₁) + W₀/((1−ρ₁)(1−ρ))  ← 服務本身也被拉長（不斷被插斷）＋等待
```

條件：preempt-resume 不損失工作、切換零成本——兩條在現實都要打折。

把「看長度」與「搶占」推到極限就是 **SRPT（Shortest Remaining Processing Time）**：任何時刻都服務剩餘量最小的工作，新到的更短就立刻搶占。它的最優性是排隊理論最乾淨的定理之一：

**定理（敘述）**：單一伺服器、允許搶占、剩餘尺寸已知，則對**任意**到達時刻與尺寸序列（不需要 Poisson、不需要任何分布假設），SRPT 使每個時刻的系統內工作數 N(t) 逐 sample path 最小。

交換論證 sketch（**工程師的嚴謹**）：設某排程在某時刻服務剩餘量較大的 j，而較小的 k 也在系統內。把之後分給 j、k 的服務時段重新洗牌——先湊滿 k 的剩餘量、剩下給 j：兩者中先完成者的完成時刻不會變晚（k 較小），後完成者不變（總時段不變），其他工作毫無感覺。所以每個時刻的完成數不減、N(t) 不增；對每個違規時刻重複交換即得。完整證明是 Schrage 1968——體裁是 Letter to the Editor，三頁多一點（出處見 landscape／延伸閱讀）。N 最優之後，Little 接手：λ 是外生的，E[N] 最小 ⇒ E[T] = E[N]/λ 最小——**SRPT 是總體平均回應時間的最優紀律**。基準表的 0.245 s 因此是 SRPT 的上界（它不輸搶占短優先；精確值要積 Schrage–Miller 1966 的 per-size 公式，本書不算）。

接著處理那句反射式質疑：「SRPT 餓死長工作。」在 M/G/1、ρ < 1 的穩態下，這大半是迷思，而且有定理背書（以下全部「敘述＋條件」，證明見 Bansal & Harchol-Balter 2001，比較基準是 PS——下一節的「公平」標竿）：

- **ρ ≤ 1/2 時全員受益**：任何（連續、矩有限的）尺寸分布，每一種尺寸的 E[T(x)] 在 SRPT 下都 ≤ PS。
- **任何 ρ < 1**：只要尺寸 ≤ x 的工作貢獻的負載 ρ(x) ≤ 1/2，尺寸 x 就受益。重尾型負載（最大 1% 的工作背走過半負載，ch15 的世界）下這代表**至少 99% 的工作在任何負載下都偏好 SRPT**；指數分布下也有 81%。
- **最壞情況有上界**：任何尺寸、任何 ρ < 1，E[T(x)] 頂多是 PS 的 ρ/(2(1−ρ))+1 倍——ρ = 0.8 時 ≤ 3 倍、0.9 時 ≤ 5.5 倍。直覺：長工作反正要待很久，墊底的時段多半是系統本來就忙的時段；而且它一旦開跑，剩餘量遞減、優先權單調上升。
- **誠實的另一面**：尺寸有上界的分布、ρ → 1 時，**最大**那支工作在 SRPT 下確實比 PS 慘（同論文有反向定理）；過載 ρ > 1 時更直接——SRPT 讓 ρ(x) < 1 的工作仍有有限回應時間（PS 給所有人 ∞），代價是最大的工作真的永遠出不來。**穩態下餓死是迷思，過載下餓死是設計**：犧牲最大工作保全其他人，要明知故犯，不要不知不覺。

SRPT 的真正門檻是操作性的：要能搶占、要知道剩餘尺寸。尺寸未知時有不需尺寸的近親（FB/LAS：服務「已獲服務量最少」者），本書不展開，指路 Harchol-Balter。

## PS：理想化的 time-sharing 與 insensitivity

**PS（Processor Sharing）**：n 個工作在系統裡，每個以 1/n 的速率同時前進——把 round-robin 的時間片推到無限薄的理想化。它是搶占的、看不看長度都無所謂（人人均分）。M/G/1-PS 的核心結果（**敘述＋指路**：RR 取極限的原始分析是 Sakata–Noguchi–Oizumi 1971；教科書處理見 Harchol-Balter）：

```text
E[T(x)] = x/(1−ρ)          ← 尺寸 x 的工作，期望回應時間正比於自身尺寸
⇒ E[T] = E[S]/(1−ρ)        ← 對 x 取期望
```

第二行就是 **insensitivity（不敏感性）**：E[T] 只透過 E[S] 依賴服務分布——C²ₛ 從公式裡消失了。對照 P-K 的 (1+C²ₛ)/2 因子：FCFS 對變異數開帳單，PS 直接把帳單撕掉。基準管線不管 C²ₛ 是 0、1、2.25 還是 10，PS 的 E[T] 都是 0.1/0.2 = 0.5 s——ch12 那張差 11 倍的三連比表被壓成一個數，而且恰好是 ch08 的 M/M/1 數字（指數服務 C²ₛ = 1 時 FCFS 與 PS 平均同值，不是巧合，是 insensitivity 把所有分布都釘在這個值）。這個性質屬於一個小家族（PS、LCFS-PR、M/M/∞ 型的 IS——ch09 預告過 Erlang-B 的 insensitive 也是這家的），條件與邊界 ch18 用 BCMP 定理收攏。注意它只保**平均**：PS 的等待分布、tail 仍然依賴完整分布。

PS 與 FCFS 的平均對決有精確判據：兩者等待之比是 (1+C²ₛ)/2，所以 **C²ₛ > 1 時 PS 贏、< 1 時 FCFS 贏**、指數打平。基準的 C²ₛ = 2.25：PS 等待 0.4 s < FCFS 0.65 s。

PS 真正的招牌是公平的精確化。定義 **slowdown ≔ T/S**（回應時間除以自身服務時間——「我多揹了幾倍」）。PS 下 E[T(x)]/x = 1/(1−ρ) 對所有 x 相同：**人人期望 slowdown 都是 5**（ρ = 0.8）。對照基準表算 FCFS：短工作 0.70/0.05 = 14 倍、長工作 1.20/0.55 = 2.2 倍——**「先來後到」聽起來公平，在 slowdown 尺度上它是對短工作最殘忍的紀律**：50 ms 的活揹 14 倍，550 ms 的活只揹 2.2 倍。但 PS 也不是聖人：看絕對值，它把長類拖到 2.75 s——比 FCFS 的 1.2 s 兇得多（守恆家族外，ρ 加權帳單 1.04 是五張帳單裡最貴的）。PS 的公平是**比例**公平，買單的是長工作的絕對延遲。公平性理論還有更多刻度，本書停在 slowdown 這一格。

## 工程化身：event loop、OS 與被驅逐的 pod

**Node.js 單執行緒事件迴圈 ≈ PS 的工程化身**——這個 ≈ 值得拆開。像的地方：同時掛著幾百個 in-flight 請求，每個請求被 await 切成小段 CPU 片，事件迴圈輪流推進；拉遠看人人同時緩慢前進，正是 PS 的圖像。而且請求的大半「服務時間」是等下游 IO，那段不佔 CPU（性格上更像 ch09 的 M/M/∞，不佔這台單伺服器）——真正在被排程的是 CPU 片。不像的地方有三條。①**run-to-completion**：一個 callback 開始跑就沒有任何東西能打斷它，「搶占」只發生在你自己寫的 await 邊界——PS 假設時間片無限薄，Node 的時間片厚度＝你最長的同步片段。一段 200 ms 的同步 JSON.parse 期間全機請求凍結：那一刻你不是 PS，是「非搶占、不看長度」家族，W₀ = λE[S²]/2 原封不動回來咬你，而且 E[S²] 是**同步片段**的二階矩。②輪轉不均勻：microtask（promise continuation）佇列會抽乾才放手，timers 與 IO callback 分屬不同 phase 批次處理（官方文件明寫每個 phase「執行佇列中的 callback 直到耗盡」）——比 round-robin 偏心得多。③沒有份額保證：PS 給每人 1/n 是定義，event loop 給誰多少看誰的 callback 密。工程結論一句話：**把 await 切細＝把量子變薄＝向 PS 逼近；event loop lag 這個指標量的就是你離 PS 有多遠。** OS 的 time-sharing 排程器（Linux 的 CFS 家族）同理：加權 PS 的工程近似，量子粒度與切換成本是它離理想化的距離。

**K8s PriorityClass** 是優先權＋搶占，但搶占的實作是**驅逐**：被趕走的 pod 不從斷點續跑，已做的工作蒸發。這是 preempt-restart，不是 preempt-resume——直接違反 work conservation，本章所有公式共同的地基。被驅逐重跑＝系統憑空多出工作量，「總帳固定」死亡，所以高負載下優先權搶占可能讓總吞吐下降：不是 bug，是兩種搶占語意的數學差別。LLM serving 的 continuous batching 排程（誰進下一個 iteration）是本章工具的直接應用場——一句話帶過，見 ch20 與書架上的《從後端到 AI Infra》。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦 |
|---|---|---|
| 在不看長度家族裡換紀律救延遲 | FCFS→LCFS/RANDOM：平均不動（守恆）、變異數與 tail 只會更鬆（FCFS 已是家族內變異數最小，Kingman 1962） | 想動平均：看長度或搶占；動不了就去 ch14 降 C²；tail 想緊：留在 FCFS |
| 拿 P-K／M/M/1 套有隱藏優先權的系統 | worker 偷挑小訊息先做時，實際總體等待 0.284 s、P-K 卻說 0.65 s——高估 2.3 倍；長優先方向則嚴重低估 | 確認消費端真實出工順序；分類量 λᵢ、E[Sᵢ]、E[Sᵢ²]，套本章公式並用 0.52 對總帳 |
| 把搶占當免費 | preempt-resume 是模型假設；現實有 context switch、cache 失溫，K8s 驅逐更是 restart——工作量蒸發、守恆地基塌掉，吞吐不升反降 | 量「搶占稅」：λ 不變下 ρ 是否爬升；重跑率放上儀表板 |
| SRPT 的尺寸哪裡來 | 尺寸估錯時長工作被誤插到前面，優勢縮水甚至反噬；多數系統根本拿不到剩餘尺寸 | 拿不到尺寸就用不需尺寸的紀律（PS／FB 一族）；或粗分兩三級就能吃到大半好處（推演 2） |
| 「餓死長工作」當教條（或當迷思） | 兩個方向都錯：穩態 M/G/1 下重尾負載 99% 工作更好、最壞 ≤3×（ρ=0.8）；但 ρ→1 的最大工作與過載情境，餓死是真的 | 先看 ρ 與尺寸分布再表態；過載下用 SRPT＝明知犧牲最大者，配 admission control（ch10）兜底 |
| 在 Node 裡自認 PS | 一段厚同步片段讓全機瞬間退化成非搶占家族，P99 走樣而平均看似正常 | 監控 event loop lag；CPU 重活切片或丟 worker thread |

自我察覺的徵兆：**平均沒動、tail 動了**——家族內換了紀律（或 LCFS 式行為混進來，例如重試把舊請求推到隊尾）；**某類請求的延遲與全站 ρ 脫鉤**——系統裡藏著你不知道的優先權，按類拆開量 E[T_Q,i]、拿 ρ 加權總帳對 0.52 的同款公式，一驗便知。

## 紙上推演

**推演 1 [15 分鐘] ★★ — 守恆律驗證**
另一條管線：λ = 10/s，80% × 50 ms、20% × 200 ms。(a) 算 ρ、W₀、FCFS 等待；(b) 短優先非搶占的兩類等待；(c) 驗證兩種紀律的 Σρᵢ E[T_Q,i] 都等於 ρW₀/(1−ρ)；(d) 兩種紀律的人頭平均各多少？這組混合 C²ₛ < 1（算出來），SJF 還有肉嗎——這對「排程收益 vs C²ₛ」說了什麼？

**推演 2 [15 分鐘] ★★ — 優先權公式應用：C²ₛ = 10 的管線**
ch12 基準分布：50 ms（機率 40/41）、2,100 ms（機率 1/41）、λ = 8/s，FCFS 等待 2.2 s。給短工作非搶占優先權：算兩類等待、人頭平均、守恆驗證。然後解釋：短類等待仍是自身服務時間的 14 倍——瓶頸在公式哪一項？換搶占後短類等待多少？

**推演 3 [20 分鐘] ★★★ — 決策分析：長工作專用 pool vs SRPT（ch09 第 4 題結案）**
4 台 worker、排程基準混合流量放大四倍（λ = 32/s）。方案 A：拆 1 台做長工作專用、3 台短工作專用；方案 B：4 台共用佇列＋短優先（或 SRPT）。不要求算 M/M/c 優先權（本書不教多伺服器排程公式），要求：列出至少四個決策判準，每個判準說明它把你推向哪一案，最後給出「什麼情境選 A、什麼情境選 B」的裁決。

### 推演解答

**推演 1。** (a) E[S] = 0.8×50+0.2×200 = 80 ms，ρ = 10×0.08 = 0.8；E[S²] = 0.8×2,500+0.2×40,000 = 10,000 ms² = 0.01 s²；W₀ = 10×0.01/2 = 0.05 s；FCFS E[T_Q] = 0.05/0.2 = **0.25 s**。(b) ρ₁ = 8×0.05 = 0.4、ρ₂ = 2×0.2 = 0.4；E[T_Q,1] = 0.05/0.6 = **0.0833 s**、E[T_Q,2] = 0.05/(0.6×0.2) = **0.417 s**。(c) 目標 ρW₀/(1−ρ) = 0.8×0.05/0.2 = 0.2：FCFS 0.8×0.25 = 0.2 ✓；短優先 0.4×0.0833+0.4×0.417 = 0.0333+0.167 = 0.2 ✓。(d) FCFS 人頭 0.25 s；短優先 0.8×0.0833+0.2×0.417 = **0.15 s**，降 40%。C²ₛ = (10,000−6,400)/6,400 = 0.5625 < 1：這裡 PS 會輸 FCFS（PS 等待 = ρE[S]/(1−ρ) = 0.32 s > 0.25 s），SJF 卻照贏 40%。教訓：**C²ₛ > 1 是「PS 贏 FCFS」的門檻，不是「看長度有沒有用」的門檻**——只要長度可分級，優先權在任何 C²ₛ > 0 下都有肉；真正沒肉的只有 C²ₛ = 0 的同質工作（沒有「短」可優先）。

**推演 2。** λ₁ = 8×40/41 = 7.80/s、ρ₁ = 0.390；λ₂ = 8/41 = 0.195/s、ρ₂ = 0.410；W₀ = 8×0.11/2 = 0.44 s。E[T_Q,1] = 0.44/0.610 = **0.722 s**；E[T_Q,2] = 0.722/0.2 = **3.61 s**；人頭 = (40×0.722+3.61)/41 = **0.792 s**——2.2 → 0.792，省 64%。守恆：0.390×0.722+0.410×3.61 = 0.282+1.48 = 1.76 = 0.8×0.44/0.2 ✓。短類 0.722 s 的來源拆 W₀：λ₂E[S₂²]/2 = 0.195×4.41/2 = 0.430 s——98% 是「正在服務的 2,100 ms 工作的殘餘」，非搶占優先權對它無能為力（卡車已上橋）。搶占後短類只看同類殘餘：E[T_Q,1] = λ₁E[S₁²]/(2(1−ρ₁)) = 0.00976/0.610 = **0.0160 s**——再省 45 倍。C²ₛ 越極端，搶占相對於「只給優先權」的價值越大；ch15 把這件事推到重尾的極限。

**推演 3。** 判準框架：①**長度可知性**——B 需要到達時知道（或可靠估計）長短；拿不到就只剩 A：用 routing 代替排程是「窮人版 SJF」，分類錯誤率就是它的稅。②**搶占可行性**——工作能不能安全中斷（冪等？checkpoint？）；不能搶占時，A 的硬隔離買到搶占的部分效果：長工作再長也只堵自己 pool，不會把 W₀ 塞進短工作的帳（ch12 的殘餘機制）。③**池化稅 vs 變異數隔離**——ch09 三連比：拆 pool 讓「有人等且有機器閒」變可能，A 付池化稅；B 在穩態贏（守恆律＋優先權公式：總帳同、人頭帳更好），但兩類共享同一個 ρ，長類流量飆升會吃掉短類的 headroom。④**過載行為**——B 配 SRPT 在 ρ > 1 時餓死最長工作（本章定理的誠實面）；A 的長 pool 自己塞死、短 pool 無感，故障域清晰、好解釋好運維。裁決：穩態平均效能選 B；最壞情況隔離與運維清晰度選 A；常見折衷是共用＋優先權、外加每類 admission control（ch10 的 K）兜底。重點不是背答案，是每個取捨現在都有名字和公式。

## 自我檢核

口頭自答，講不清楚＝還沒懂：

1. work conservation 守恆的精確內容是什麼（哪個量、哪種意義、對哪些紀律）？哪三種工程現實會打破它？
2. 重講「FCFS／LCFS／RANDOM 平均等待全相同」：哪一步用了非搶占、哪一步用了不看長度、哪一步用了 V 不變？這個相等需要 Poisson 到達嗎？哪個「值」才需要？
3. Kleinrock 守恆律的幣別為什麼是 ρᵢ 不是 λᵢ？短優先為什麼能在總帳不變下壓低人頭平均？
4. 非搶占優先權公式的兩個分母因子各對應什麼機制？W₀ 為什麼包含低優先類的殘餘？
5. SRPT 最優性的精確主張：哪個量、哪種意義（逐 sample path 還是期望）、需要哪些操作條件？Little 在哪一步把 N 的最優翻成 E[T] 的最優？
6. 「SRPT 餓死長工作」何時是迷思（給負載與分布條件、最壞倍數）、何時是事實？
7. PS 的 insensitivity 說了什麼、沒說什麼？你的 Node 服務在什麼條件下有資格引用 E[T] = E[S]/(1−ρ)，什麼徵兆代表資格被取消了？
8. FCFS 在哪個尺度上「最公平」、在哪個尺度上「最殘忍」？用基準數字（14 倍 vs 2.2 倍）講一遍 slowdown 的觀點。
