## 機率的文法：條件、全機率與全期望

快速過一遍語言本身。一個機率模型有三樣東西：樣本空間 Ω（所有可能結果）、事件（結果的集合）、機率測度 P。P 只需要遵守三條公理：P(A) ≥ 0、P(Ω) = 1、互斥事件的機率可加。這背後有一整套測度論在撐腰，本書一句話跳過：工程上你永遠不會踩到那些病態案例（見延伸閱讀）。

**條件機率**：P(A|B) = P(A∩B)/P(B)，定義要求 P(B) > 0。讀法是「把世界縮小到 B 之內重新正規化」——條件化就是資訊更新：每多知道一件事，就把樣本空間裁一刀、剩下的機率重新正規化。定義移項就是鏈式法則 P(A∩B) = P(A|B)·P(B)，把聯合機率拆成「先發生、再發生」的敘事；它的多事件版本在 ch06 算 Markov chain 的路徑機率時會大量出現。兩個事件**獨立**的定義是 P(A∩B) = P(A)P(B)；注意獨立是一個要被論證的假設，不是預設值——本書每次用它都會明說。

**全機率定理**：若 B₁, …, Bₖ 互斥且窮盡（恰好一個會發生、各 P(Bᵢ) > 0），則

```text
P(A)  = Σᵢ P(A|Bᵢ)·P(Bᵢ)
E[X]  = Σᵢ E[X|Bᵢ]·P(Bᵢ)      ← 期望值版本，稱全期望定理
```

這是佇列推導的主力工具：把一個複雜的量，按「第一步發生了什麼」切開來分頭算。ch07 的 balance equations、ch12 的 P-K 推導，骨架都是它。

你其實已經在用它。推播 worker 的服務時間有兩種路徑：90% 是單純推送（50 ms），10% 要先重建裝置清單（550 ms）。你心算混合平均的那一刻——

```text
E[S] = E[S|短]·P(短) + E[S|長]·P(長)
     = 50·0.9 + 550·0.1 = 45 + 55 = 100 ms
```

——就是全期望定理。這組數字（排程基準，ch13 還會用它）本章稍後會再回來算它的 C²，結果會讓你重新看待這條管線。

## 隨機變數與 tail：P99 的數學身分

隨機變數（random variable）是把每個結果對應到一個數字的函數——服務時間 S、回應時間 T、系統內人數 N 都是。描述一個隨機變數的標準配備：

- **CDF**：F(t) = P(X ≤ t)。
- **tail**：F̄(t) = P(X > t) = 1 − F(t)。本書大量直接操作 tail，因為延遲問題天生是 tail 問題。
- 離散型用 **pmf** p(x) = P(X = x)；連續型用 **pdf** f(x)，機率是密度下的面積。

P99 的正式身分：使 P(T ≤ t) ≥ 0.99 的最小 t，也就是 tail 掉到 0.01 的位置——quantile 函數在 0.99 的取值。你的監控系統每分鐘都在對 empirical CDF 做這個查詢。

常用分布速查（每個值我都重算過；C² 這欄現在先存著，下面兩節就會知道它是什麼、為什麼重要）：

| 分布 | 模型化什麼 | E[X] | Var(X) | C² |
|---|---|---|---|---|
| Bernoulli(p) | 一次成敗 | p | p(1−p) | (1−p)/p |
| Binomial(n, p) | n 次獨立成敗中的成功數 | np | np(1−p) | (1−p)/(np) |
| Geometric(p)，取值 1, 2, … | 第一次成功發生在第幾次 | 1/p | (1−p)/p² | 1−p |
| Uniform(a, b) | 只知道範圍時的均勻猜測 | (a+b)/2 | (b−a)²/12 | U(0, b) 時為 1/3 |
| Exponential(μ) | 無記憶的持續時間 | 1/μ | 1/μ² | 1（深論見 ch04） |

geometric 是「重試直到成功」的分布，它是離散世界的無記憶分布——這句話 ch04 會精確化。重尾的 Pareto 故意不在表上：它會在 ch15 把這張表的世界觀掀掉（見 ch15）。

## 期望值的三件武器：LOTUS、indicator、線性

期望值：離散 E[X] = Σ x·p(x)，連續 E[X] = ∫ x·f(x) dx。定義要求加總／積分絕對收斂——先記下「期望值不是天賦人權」，ch15 會遇到連均值都不存在的分布。三件武器依序登場：

**武器一：LOTUS**。要算 E[g(X)]，不必先求出 g(X) 的分布：E[g(X)] = Σ g(x)·p(x)（連續版同理）。它的全名是 law of the unconscious statistician——因為人人都無意識地在用，沒注意到這其實是一條需要證明的定理。本書把它當工具用，證明屬於進階機率（見延伸閱讀）。最常用的特例：E[X²] = Σ x²·p(x)，等一下變異數全靠它。

**武器二：indicator**。事件 A 的指示變數 1{A}：A 發生取 1、否則取 0。完整推導只有一行：

```text
E[1{A}] = 1·P(A) + 0·P(Aᶜ) = P(A)      ← 機率被翻譯成期望值
```

威力在於「個數」類的隨機變數都能拆成 indicator 的和，然後交給武器三。

**武器三：期望值線性**。對任何隨機變數 X、Y 與常數 a、b：

```text
E[aX + bY] = a·E[X] + b·E[Y]
```

唯一的條件是各期望值有限。**不需要獨立——一個字都不需要**。這是全書被使用最多次、也最常被低估的事實。對照組：E[XY] = E[X]E[Y] 就**需要**獨立（精確說：不相關）才成立；線性沒有這種附帶條件。

組合技示範：8 個 job 各自獨立、均勻隨機指派到 4 個 shard。平均有幾個 shard 閒著？令 X = 閒置 shard 數 = Σᵢ 1{shard i 沒分到 job}。每個 shard 被單一 job 避開的機率是 3/4，八個 job 獨立，所以 P(shard i 閒置) = (3/4)⁸ ≈ 0.100，於是

```text
E[X] = 4 × (3/4)⁸ ≈ 0.400 個
```

注意這 4 個 indicator 彼此**高度相依**（若 3 個 shard 都空，job 全擠在剩下那個），但線性根本不在乎。不用枚舉任何聯合分布，兩行結束——這就是 indicator＋線性的標準殺法，ch18 算 fork-join 的 E[max] 還會再看到它。

## 幾何級數：全書的引擎

ch08 會推出 M/M/1 的平穩分布 πₙ = (1−ρ)ρⁿ——一個幾何形狀的分布。從那一刻起，全書的每一次「求和」幾乎都是幾何級數。這一節把它磨到閉著眼睛都會。

**完整推導**（部分和的老把戲，值得你抄一遍）：

```text
S_N     = 1 + ρ + ρ² + … + ρ^N
ρ·S_N   =     ρ + ρ² + … + ρ^N + ρ^(N+1)
相減：(1−ρ)·S_N = 1 − ρ^(N+1)
S_N = (1 − ρ^(N+1)) / (1 − ρ)            ← 對任何 ρ ≠ 1 都成立
```

當 |ρ| < 1，ρ^(N+1) → 0，所以 **Σ_{n≥0} ρⁿ = 1/(1−ρ)**。條件 |ρ| < 1 不是裝飾品：ρ ≥ 1 時級數發散，而 ch08 會告訴你，「平穩分布的正規化級數收斂」**就是**佇列穩定條件 ρ < 1 的數學身分——條件破掉時公式不會報錯，只會給你一個負數假裝是答案。

**微分技巧**（工程師的嚴謹：冪級數在收斂半徑內可逐項微分，這是實分析的定理，本書引用不證——見延伸閱讀）：

```text
Σ_{n≥0} ρⁿ = 1/(1−ρ)                    |ρ| < 1
兩邊對 ρ 微分：
Σ_{n≥1} n·ρ^(n−1) = 1/(1−ρ)²
兩邊乘 ρ：
Σ_{n≥1} n·ρⁿ = ρ/(1−ρ)²
```

同一個和也能走另一條路對帳（下一節「雙算」思想的預演）：把 n 讀成「n 個 1 的和」、交換求和順序，Σ_{n≥1} n·ρⁿ = Σ_{k≥1} (Σ_{n≥k} ρⁿ) = Σ_{k≥1} ρᵏ/(1−ρ) = ρ/(1−ρ)²。兩條路同答案——這種互相對帳的習慣，是手推公式時最便宜的防錯機制。

預告它的第一份工作：ch08 要算系統內平均人數 E[N] = Σ n·πₙ = Σ n·(1−ρ)ρⁿ = (1−ρ)·ρ/(1−ρ)² = ρ/(1−ρ)。代 ρ = 0.8 得 4——基準表那個「E[N] = 4」就是這三行。你現在已經會推 M/M/1 最重要的結果之一了，只是還不知道 πₙ 為什麼長那樣（ch07–08 的事）。

## 用 tail 算期望：E[X] = ∫ P(X > t) dt

期望值還有第二種算法，不走「值 × 機率」，改走「tail 面積」。它在佇列理論裡常常快得多，而且是 ch12 殘餘服務時間論證的核心。

**離散版（完整推導）**：X 取非負整數。關鍵恆等式：

```text
X = Σ_{n≥0} 1{X > n}        ← 自我檢查：X = 3 時右邊 = 1{3>0}+1{3>1}+1{3>2} = 3，其餘項全是 0
兩邊取期望，套線性與 E[1{A}] = P(A)：
E[X] = Σ_{n≥0} P(X > n)
```

（誠實標示：對「無窮多項」套線性需要收斂定理撐腰；因為每一項都非負，這裡的交換是合法的——完整論證見延伸閱讀，工程上你可以放心用。）

**連續版（工程師的嚴謹）**：X ≥ 0、有密度 f。

```text
∫₀^∞ P(X > t) dt = ∫₀^∞ ( ∫_t^∞ f(x) dx ) dt     ← tail 展開成積分
                 = ∫₀^∞ ( ∫₀^x dt ) f(x) dx       ← 交換順序：積分區域同樣是 {0 ≤ t < x}
                 = ∫₀^∞ x·f(x) dx = E[X]
```

交換積分順序的合法性由「被積函數非負」保證（Tonelli 定理，本書不展開）。圖像比代數好記：

```
 P(X > t)
 1 ┤*
   │ **
   │   **
   │     **          曲線下的整塊面積 ＝ E[X]
   │       ***
   │          ****
   │              *******
 0 └──────────────────────────────► t
```

**適用條件**：X ≥ 0。一般的 X 要把正負兩部分拆開分別處理，本書用到的場合（服務時間、等待、人數）全是非負的，條件天然滿足。同一招再推一格，就得到全書會反覆使用的二階矩版本：E[X²] = E[∫₀^X 2t dt] = ∫₀^∞ 2t·P(X > t) dt。

一行熱身：指數分布 tail 是 P(S > t) = e^(−μt)（深論見 ch04），所以 E[S] = ∫₀^∞ e^(−μt) dt = 1/μ。三秒鐘，不用分部積分。

### Worked example：geometric 期望的兩本帳

下游呼叫每次成功機率 p = 0.8，失敗立刻重試、各次獨立。X = 直到成功的嘗試次數 ~ Geometric(0.8)。用兩條完全不同的路算 E[X]，帳要對得起來。

```text
帳本一：tail 加總
P(X > n) = 0.2ⁿ                          ← 前 n 次全失敗
E[X] = Σ_{n≥0} 0.2ⁿ = 1/(1 − 0.2) = 1.25 次

帳本二：定義＋微分技巧
E[X] = Σ_{n≥1} n·(0.2)^(n−1)·0.8
     = 0.8 · Σ_{n≥1} n·(0.2)^(n−1)
     = 0.8 · 1/(1 − 0.2)²                ← 微分技巧的中間產物 Σ n·ρ^(n−1) = 1/(1−ρ)²
     = 0.8/0.64 = 1.25 次
```

兩本帳都是 1.25，對齊。順帶驗證了表裡的 E = 1/p。實戰心得：tail 路線通常更短、更不容易抄錯（tail 本身常有現成的乘法結構）；微分路線更通用，遇到 Σ n·ρⁿ 這種「值 × 機率」的形狀時是唯一選擇。兩條都要會，互相對帳是抓錯的最快方式。

## 變異數與 C²：全書主角登場

變異數量化「離平均多遠」：Var(X) = E[(X − E[X])²]。計算永遠用展開形：

```text
Var(X) = E[X² − 2X·E[X] + (E[X])²]
       = E[X²] − 2E[X]·E[X] + (E[X])²    ← 線性；E[X] 是常數
       = E[X²] − (E[X])²
```

因為平方非負，這條順便送你一個之後常用的事實：E[X²] ≥ (E[X])²，差距就是變異數。常用性質：Var(aX + b) = a²Var(X)（平移不影響散布，縮放平方倍）；標準差 σ = √Var 把單位拉回來。

但 Var 有個工程上的致命傷：它帶單位、帶尺度。「服務時間變異數 22,500 ms²」這句話你無法直接判讀——這算叢發嗎？跟另一條均值不同的管線怎麼比？解法是正規化：

> **C² = Var(X)/(E[X])²**——變異係數平方（squared coefficient of variation, SCV）。定義在 E[X] > 0 的非負隨機變數上；本書用 C²ₛ 記服務時間的、C²ₐ 記到達間隔的。

C² 無因次、且尺度不變：C²(aX) = a²Var(X)/(a·E[X])² = C²(X)。100 ms 等級的服務和 100 s 等級的批次作業可以放在同一把尺上比「形狀」。這把尺的刻度：

```
 C² = 0                  C² = 1                  C² > 1
───┼────────────────────────┼───────────────────────┼──────────►
 確定性:每件都一樣久      指數:隨機的基準刻度     叢發:少數巨件主宰
 (D/D/1 的世界,零驚奇)   (ch04 的原子)           (ch12 開帳單、ch15 失控)
```

把你的系統放上這把尺：cron 觸發的定時任務、固定大小的批次，間隔與耗時都落在 C² ≈ 0 附近；大量獨立使用者自發產生的請求，到達間隔天生接近指數（為什麼會這樣，ch05 會解釋），所以 C²ₐ ≈ 1 是個合理的預設猜測；而 cache hit 與 miss 的混合、偶發的 GC 暫停、月底那支報表 query——任何「平常很快、偶爾很慢」的形狀——都會把 C²ₛ 推過 1。讀法只有一句：C² < 1 比指數規律、C² > 1 比指數叢發，指數是那條「純隨機」的基準線。

為什麼全書的主角是它？預告帳單：ch12 的 P-K 公式說 M/G/1 的等待 E[T_Q] 正比於 (1 + C²ₛ)/2，ch14 的 Kingman 近似把到達也算進來、正比於 (C²ₐ + C²ₛ)/2——**變異性本身就是延遲的來源，而 C² 就是它的計價單位**。在推播基準（λ = 8、E[S] = 100 ms、ρ = 0.8）下，C²ₛ = 0、1、10 對應的等待是 0.2 s、0.4 s、2.2 s（見 ch12）。同一個平均、同一個使用率，帳單差 11 倍。

### Worked example：C² 三連發

三個服務時間分布，均值全部釘在 100 ms，逐一算 C²。

**之一：確定性。** S ≡ 100 ms。E[S²] = 10,000 ms²，Var = 10,000 − 100² = 0，**C² = 0**。零驚奇，ch01 你已看過 D/D/1 在 λ < μ 時完全不排隊。

**之二：指數（μ = 10）。** 用上一節的 2t 公式收割：

```text
E[S²] = ∫₀^∞ 2t·P(S > t) dt = ∫₀^∞ 2t·e^(−μt) dt = 2/μ²   ← ∫₀^∞ t·e^(−μt) dt = 1/μ²（分部積分一次）
Var(S) = 2/μ² − (1/μ)² = 1/μ²
C²     = (1/μ²) / (1/μ)² = 1
```

μ = 10 時 E[S] = 100 ms、σ = 100 ms。**指數分布的 σ 永遠等於均值、C² 永遠是 1**——這就是它被當成「隨機性的基準刻度」的原因（深論見 ch04）。

**之三：二點叢發。** 本章開頭那條管線：90% 短工作 50 ms、10% 長工作 550 ms，E[S] = 100 ms。

```text
E[S²] = 0.9·50² + 0.1·550²
      = 0.9·2,500 + 0.1·302,500
      = 2,250 + 30,250 = 32,500 ms²
Var   = 32,500 − 10,000 = 22,500 ms²    →  σ = 150 ms
C²    = 22,500 / 10,000 = 2.25
```

| 分布（均值都是 100 ms） | E[S²] | Var | σ | C² |
|---|---|---|---|---|
| 確定性 100 ms | 10,000 ms² | 0 | 0 | 0 |
| 指數 μ = 10 | 20,000 ms² | 10,000 ms² | 100 ms | 1 |
| 二點 90%×50 ms＋10%×550 ms | 32,500 ms² | 22,500 ms² | 150 ms | 2.25 |

只是 10% 的工作長了 11 倍，C² 就翻過指數、來到 2.25——而監控大盤上這條管線的「平均服務時間」三者一模一樣。

倒推一格更有感：要把 C² 推到 10（ch12 基準的叢發檔位）需要多誇張的分布？保持短工作 50 ms、均值 100 ms，設長工作 L 占比 q，解兩條帳 E[S] = 100、E[S²] = Var + E² = 110,000 ms²，得 q = 1/41 ≈ 0.0244、L = 2,100 ms。驗算：E[S] = (40·50 + 2,100)/41 = 4,100/41 = 100 ✓；E[S²] = (40·2,500 + 4,410,000)/41 = 110,000 ✓。也就是說：**2.4% 的請求各拖 2.1 秒，就足以把 C² 打到 10**。你的系統裡幾乎一定有這種請求——那支月底報表 query、那次 cold start。再往上，ch15 的 Pareto（α = 1.5）變異數乾脆是無限大、C² = ∞，這把尺直接爆表（見 ch15）。

## Markov 與 Chebyshev：平均值對 tail 承諾了什麼

回到值班夜的問題：只知道「平均 100 ms」，數學上對 P99 能說什麼？

**Markov 不等式（完整推導）**。適用條件：X ≥ 0、t > 0、E[X] 有限。

```text
t·1{X ≥ t} ≤ X                ← 逐情況檢查：X ≥ t 時左邊 = t ≤ X；X < t 時左邊 = 0 ≤ X
取期望（期望保序、線性）：
t·P(X ≥ t) ≤ E[X]
⇒  P(X ≥ t) ≤ E[X]/t
```

證明就是 indicator 武器的一次出手。注意三件事：X ≥ 0 是必要的（允許負值就能把均值做小而 tail 不變）；t ≤ E[X] 時上界 ≥ 1、毫無內容；它是**上界**，不是估計值。

套用：T ≥ 0、E[T] = 0.1 s。P(T ≥ 1 s) ≤ 0.1，P(T ≥ 10 s) ≤ 0.01。所以「平均 100 ms」對 P99 的全部承諾是：**不超過 10 秒**。更慘的是這個上界是達得到的——分布取 99% 機率 0 ms、1% 機率 10 s，均值恰是 100 ms，P99 恰是 10 s。平均值沒說謊，它只是承諾得就這麼少。

**Chebyshev 不等式（完整推導）**。多知道一個變異數，承諾立刻收緊。適用條件：Var(X) 有限（X 不必非負）、a > 0。

```text
P(|X − E[X]| ≥ a) = P( (X − E[X])² ≥ a² )      ← 同一個事件，兩邊平方
                  ≤ E[(X − E[X])²] / a²         ← 對非負的 (X − E[X])² 套 Markov
                  = Var(X)/a²
```

套用：若再知道 σ = 100 ms（指數等級的散布、Var = 0.01 s²），取 a = 1 s：P(|T − 0.1| ≥ 1) ≤ 0.01/1² = 0.01，所以 P(T ≥ 1.1 s) ≤ 0.01，P99 ≤ 1.1 s。若進一步知道完整分布是指數（μ = 10），直接解 tail：e^(−10t) = 0.01 給 t₉₉ = ln 100/10 ≈ 0.461 s（ln 100 ≈ 4.6052）——順帶看見 ch08 那句「指數世界 P99 ≈ 4.6× 平均」的出處。

| 你知道什麼 | 工具 | 對 P99 的承諾 |
|---|---|---|
| 只有 E[T] = 100 ms | Markov | ≤ 10 s（且存在分布真的這麼糟） |
| 加上 σ = 100 ms | Chebyshev | ≤ 1.1 s |
| 完整分布：Exponential(10) | 直接解 tail | = 0.461 s |

每多帶一階資訊，tail 的承諾就縮一個量級。這就是本書堅持把分布——至少把前兩階矩，也就是均值與 C²——隨身帶著的理由；也是 SLA 只寫平均值等於沒寫的數學證據。也別因為「鬆」就小看這兩條不等式：它們對分布**零假設**。當你手上只有監控吐出來的一兩個矩、分布形狀根本不可知時，distribution-free 的上界是唯一誠實的承諾——而 ch15 的重尾世界裡連這份誠實都要打折：變異數無限時，Chebyshev 直接失業（見 ch15）。Chebyshev 是雙邊不等式，單邊有更緊的版本（Cantelli），本書用不到、知道存在即可。至於中央極限定理：大量獨立量的和趨近常態分布——本書只在 ch19 計算模擬的信賴區間時需要它，在那之前一句話帶過（見 ch19）。

## 陷阱與防禦

本章工具的共同危險：用錯時不會報錯，只會安靜地給你一個長得像答案的錯數字。

| 陷阱 | 它怎麼騙你 | 防禦 |
|---|---|---|
| 把 Var(X+Y) = Var(X)+Var(Y) 當預設 | 可加性需要**不相關**；retry 與尖峰使流量正相關，實際變異比帳面大。極端例：Y = X 時 Var(2X) = 4Var(X)，帳面只記 2Var(X)，低估一半 | 相加變異數前先回答「這兩個量憑什麼獨立」；答不出來就別加。期望值線性才是無條件的 |
| E[g(X)] 當成 g(E[X]) | E[1/X] ≠ 1/E[X]：X 各半機率取 1 或 3，1/E[X] = 0.5、E[1/X] = 2/3 ≈ 0.667，差 33% | 一律過 LOTUS；凸函數時 Jensen 不等式告訴你偏哪一邊（E[X²] ≥ (E[X])² 同源） |
| P(A\|B) 與 P(B\|A) 互換 | 「警報響時 90% 真有事故」≠「有事故時 90% 會響」；事故的 base rate 低時兩者天差地遠 | 回到定義 P(A\|B) = P(A∩B)/P(B)，老老實實把聯合機率寫出來 |
| 幾何級數忘了 \|ρ\| < 1 | ρ ≥ 1 時級數發散，但 1/(1−ρ) 照樣吐出一個（負的）數字給你 | 每次求和先檢查收斂；ch08 會看到這條件正是佇列穩定條件 ρ < 1 |
| 把 Markov 上界當估計值 | 它是最壞情況保證，常鬆一個量級（指數真值 0.461 s vs 上界 10 s） | 讀成「保證不會更糟」；要「大概多少」就得帶更多矩或完整分布 |
| 用截斷的監控直方圖估 C² | 最上層 bucket 把長 tail 砍掉，Var 與 C² 被系統性低估——最叢發的部分剛好被藏起來 | 檢查最上層 bucket 的占比；只要不可忽略，就把估出的 C² 當**下界**用 |

## 紙上推演

**推演 1 [5 分鐘] ★ — 級數熱身（ch08 預習）**
求 Σ_{n≥0} n·(1−ρ)·ρⁿ 的封閉形式，並在 ρ = 0.8 求值。

**推演 2 [10 分鐘] ★★ — 條件期望分解（first-step 思維）**
下游呼叫每次固定耗時 100 ms、成功機率 0.8，失敗立刻重試、各次獨立。令 T 為直到成功的總耗時。(a) 用全期望定理對「第一次是否成功」分解，列方程解 E[T]；(b) 用「期望次數 × 單次耗時」對帳；(c) 改成失敗那次要等滿 1 s 的 timeout 才重試，重解 E[T]。

**推演 3 [10 分鐘] ★★ — indicator 求期望**
10 個 job 依序進站，處理時間 iid 且為連續分布（不會平手）。稱第 k 個 job 是「紀錄」，若它比前面所有 job 都長。求紀錄個數的期望值。提示：先用對稱性求 P(第 k 個是紀錄)。

**推演 4 [15 分鐘] ★★ — 從監控直方圖估 C²ₛ**
某服務的延遲直方圖（虛構）：

| bucket | 占比 |
|---|---|
| 0–20 ms | 30% |
| 20–100 ms | 50% |
| 100–500 ms | 15% |
| 500–2,000 ms | 5% |

用各 bucket 的中點當代表值，估 E[S]、E[S²]、C²ₛ。這個服務比指數更叢發嗎？再指出這種估法至少兩個系統性偏差與它們的方向。

### 推演解答

**推演 1。** 提出常數、套微分技巧：Σ n·(1−ρ)ρⁿ = (1−ρ)·ρ/(1−ρ)² = ρ/(1−ρ)。代 ρ = 0.8 得 0.8/0.2 = **4**。這正是 ch08 的 E[N]：當 πₙ = (1−ρ)ρⁿ 時，系統內平均 4 個人。常見錯路：糾結求和從 n = 0 還是 n = 1 開始——n = 0 那項是 0，兩者相同；以及忘記檢查 ρ < 1（推演靠你自覺，生產環境靠 ch10 的 backpressure）。

**推演 2。** (a) 第一次嘗試無論如何花 0.1 s；以 0.8 結束，以 0.2 進入「重新開始」——因為各次獨立，失敗後的未來是一份全新副本，期望仍是 E[T]：

```text
E[T] = 0.1 + 0.8·0 + 0.2·E[T]
0.8·E[T] = 0.1   ⇒   E[T] = 0.125 s
```

(b) 對帳：E[次數] = 1/0.8 = 1.25（本章 worked example），乘 0.1 s 得 0.125 s ✓。
(c) 成功那次花 0.1 s、失敗那次花 1 s 再重來：

```text
E[T] = 0.8·0.1 + 0.2·(1 + E[T]) = 0.28 + 0.2·E[T]
⇒ E[T] = 0.28/0.8 = 0.35 s
```

討論：timeout 把期望耗時從 125 ms 膨脹到 350 ms——「失敗的嘗試也占用時間」是 ch20 retry storm 機制的第一塊骨牌。(a) 的「未來是全新副本」用了嘗試間獨立；如果下游故障有持續性（失敗預示更多失敗），這個方程就立不出來。這種「對第一步條件化」的招式，ch07 的 balance equations 與 ch12 的 busy period 是它的進階版。

**推演 3。** 前 k 個 job 的處理時間 iid 連續，最大值落在哪個位置是均勻的（對稱性、無平手），所以 P(第 k 個是紀錄) = 1/k。令紀錄數 R = Σₖ 1{第 k 個是紀錄}，線性給

```text
E[R] = Σ_{k=1..10} 1/k = H₁₀ ≈ 2.93 個
```

indicator 之間明顯相依（第 3 個是紀錄會改變第 5 個是紀錄的條件機率），但線性照樣輾過去。常見錯路：想直接算 R 的分布——可以做但完全多餘。注意 1/k 論證依賴 iid：若處理時間有趨勢（每小時逐漸變慢），對稱性就破了。調和數 H_n 在 ch18 會以 E[max] = H_n/μ 的身分回鍋。

**推演 4。** 中點代表值：10、60、300、1,250 ms。

```text
E[S]  = 0.3·10 + 0.5·60 + 0.15·300 + 0.05·1,250
      = 3 + 30 + 45 + 62.5 = 140.5 ms
E[S²] = 0.3·100 + 0.5·3,600 + 0.15·90,000 + 0.05·1,562,500
      = 30 + 1,800 + 13,500 + 78,125 = 93,455 ms²
Var   = 93,455 − 140.5² = 93,455 − 19,740.25 ≈ 73,715 ms²
C²ₛ   = 73,715 / 19,740.25 ≈ 3.73
```

C²ₛ ≈ 3.73 > 1：比指數更叢發——拿去餵 ch12 的 P-K，等待會是同均值指數服務的 (1+3.73)/2 ÷ 1 ≈ 2.4 倍。系統性偏差：①中點假設抹掉 bucket 內部的散布，E[S²] 被低估 → C² 偏低；②最上層 bucket 在 2 s 截斷，更長的 tail 整段消失 → Var 再被低估。兩個偏差同向，所以 3.73 應該讀成**下界**：真實系統至少這麼叢發。（次要誤差：寬 bucket 內若分布偏向低端，中點會高估該格貢獻——方向不定，但通常被前兩項壓過。）
