# ch11 — 可逆性與 Burke 定理：網路的鑰匙

> **本章解決什麼問題**：ch08–ch10 把單站佇列拆解得夠徹底了，但你的系統從來不是一站——API 後面接 worker，worker 後面接下一個 worker。要把單站理論升級成網路理論（ch16–18），得先回答一個問題：**佇列的輸出長什麼樣？**本章用一個出乎意料的工具——把時間倒著放——回答它：M/M 佇列的穩態輸出仍是 Poisson（Burke 定理），而且系統對自己吐過什麼毫無記憶。這把鑰匙立刻打開串聯佇列的乘積形式；它打不開的那扇門（feedback），正是 ch16 驚奇的伏筆。

## 從你已知的出發

你做容量規劃時有一個默契的習慣：一段一段算。API 服務一筆帳、SQS 後面的 consumer 一筆帳、下游的結算 worker 再一筆帳，每段各自套用「λ 進來、μ 服務」的單站模型。這個習慣藏著一個從沒被檢查過的假設：**每一段的輸入都長得像 Poisson**。第一段沒問題——大量獨立使用者的匯流，ch05 的 Palm–Khintchine 直覺給了執照。但第二段呢？它的輸入不是使用者，是**上一段的輸出**：一條被佇列搓揉過、被服務時間整形過的機器流。沒有任何「大量獨立來源」的論證罩著它。如果上游輸出不是 Poisson，你對下游套的每一條 M/M/1 公式都是在沙地上蓋房子。

第二條線索來自監控直覺。值班時你看著 completions/sec 面板，過去一分鐘 push worker 吞吐暴增，你心想：「剛吐出一串 job，佇列現在應該比較空了。」這個推論聽起來無懈可擊——東西出去了，裡面當然變少。本章會證明一個讓人坐立難安的定理：對穩態的 M/M 佇列，**這個推論的資訊量恰好是零**。離開歷史對「現在裡面有多少」一個字都不透露。

這兩條線索由同一個定理收束：Burke 定理。而證明它的工具——時間反轉——本身就值回票價：把監控錄影倒著放，問「我分不分得出來」，這個看起來像哲學謎語的問題，會變成你解平穩分布的實用技巧。

## 時間反轉：把監控錄影倒著放

從 ch07 的 CTMC 出發：狀態空間上的過程 X(t)，轉移速率 q(i→j)，假設 ergodic、平穩分布 π 存在，而且**系統已經跑在穩態**（X(0) ~ π）。現在把一段長度 τ 的錄影倒著放，定義反轉過程 X′(t) = X(τ−t)。

第一個問題：倒放還是 Markov 嗎？是。Markov 性質有一個時間對稱的講法：**給定現在，過去與未來條件獨立**（ch06 的「狀態吃掉歷史」換句話說）。這句話裡「過去」與「未來」的地位完全對等——你把時間軸翻面，它一字不改（工程師的嚴謹：這個對稱講法與 ch06 定義的等價性，完整版見 Kelly 第 1 章）。

第二個問題：倒放的速率是多少？算給你看（完整推導）：

```text
P(X(t−h) = j | X(t) = i)
  = P(X(t−h) = j, X(t) = i) / P(X(t) = i)        ← 條件機率定義
  = πⱼ·(q(j→i)·h + o(h)) / πᵢ                    ← 穩態：P(X(t−h)=j) = πⱼ；小區間轉移（ch07）
⇒ 反轉鏈的速率  q′(i→j) = πⱼ·q(j→i) / πᵢ
```

注意穩態假設在哪裡出力：P(X(t−h) = j) = πⱼ 對所有 t 成立，反轉速率才不隨時間變。如果系統還在暖機（transient），倒放的「速率」會隨時刻漂移，整套機器報廢——這個細節在陷阱與防禦會回來咬人。

兩個立即的健康檢查。第一，反轉鏈從 i 流出的總速率沒變：

```text
Σ_{j≠i} q′(i→j) = (1/πᵢ)·Σ_{j≠i} πⱼ·q(j→i)       ← 代定義
                = (流入 i 的總機率流)/πᵢ
                = (流出 i 的總機率流)/πᵢ           ← global balance（ch07）
                = Σ_{j≠i} q(i→j)
```

所以倒放影片裡每個狀態的停留時間分布跟正放一模一樣。第二，π 也是反轉鏈的平穩分布：由定義 πᵢ·q′(i→j) = πⱼ·q(j→i)，對 i 加總就得到反轉鏈在 j 的 global balance。倒放的世界跟正放共享同一套長期比例——這很合理：把影片倒著放，各狀態的螢幕時間佔比當然不變。

## 可逆性與 detailed balance

有了反轉鏈，可逆性的定義就一句話：

> **定義**：穩態過程 X 是**可逆的（reversible）**，若對任何 τ 與任何時點組 t₁, …, tₖ，(X(t₁), …, X(tₖ)) 與 (X(τ−t₁), …, X(τ−tₖ)) 同分布。

白話：你拿到一段穩態系統的監控錄影，但沒人告訴你它是正放還是倒放——可逆，就是**統計上無法分辨**。等價地說：反轉鏈的速率跟原鏈一模一樣，q′ = q。把這個條件跟上面的速率公式擺在一起，就得到全章的樞紐定理：

> **定理（可逆 ⇔ detailed balance）**：穩態 CTMC 可逆，若且唯若存在機率分布 π 使得對所有狀態對 i ≠ j：πᵢ·q(i→j) = πⱼ·q(j→i)。此時 π 就是平穩分布。

這組方程叫 **detailed balance**：不只每個狀態收支平衡（global balance），而是**每一條邊上**、兩個方向的機率流逐對相等。證明兩個方向都短（工程師的嚴謹：在有限維分布的層級；測度層級的版本見 Kelly 第 1 章）：

```text
（⇒）可逆 ⇒ q′(i→j) = q(i→j)，代入 q′ 的公式：
     πⱼ·q(j→i)/πᵢ = q(i→j)  ⇒  πᵢ·q(i→j) = πⱼ·q(j→i)    ← detailed balance 逐字

（⇐）先證 π 平穩：對固定 i，把 detailed balance 對所有 j ≠ i 加總：
     Σ_{j≠i} πᵢ·q(i→j) = Σ_{j≠i} πⱼ·q(j→i)
     └── 流出 i 的總流 ──┘   └── 流入 i 的總流 ──┘          ← 正是 global balance
     再代回反轉速率：q′(i→j) = πⱼ·q(j→i)/πᵢ = q(i→j)
     速率相同、起始分布同為 π ⇒ 兩條鏈同法則 ⇒ 可逆。∎
```

注意這裡白送了一份大禮：**只要你找得到滿足 detailed balance 的 π，它自動就是平穩分布**——因為 detailed balance 逐條加總就是 global balance。這份大禮的實用價值在於 detailed balance 方程便宜得多：每條只有兩項、牽涉兩個未知數；global balance 每條方程牽涉一個狀態的所有鄰居。於是有了這套「先猜再驗」的標準操作：

1. 在轉移圖上挑一個參考狀態，先設它的 π 值為 1（正規化最後再說）。
2. 沿著邊用 πⱼ = πᵢ·q(i→j)/q(j→i) 一路傳播，把每個狀態表成參考狀態的倍數。
3. 圖上有迴圈時，繞一圈回到起點必須得到同一個值。對不上＝這條鏈不可逆，detailed balance 無解，乖乖回去解 global balance（π 仍然存在，只是沒這條捷徑）。
4. 對得上：正規化，然後把解代回**每一條** detailed balance 複核。成功的同時你免費得到兩件事：π 是平穩分布、這條鏈可逆。

第 3 步其實是一條獨立的判準在操作上的長相——**Kolmogorov 迴圈判準**：穩態鏈可逆，若且唯若沿任何迴圈，順著走的速率乘積等於逆著走的速率乘積（敘述層級，不需先知道 π；證明見 Kelly 第 1 章）。它把可逆性變成純粹看速率的幾何性質：**可逆 ⇔ 轉移圖上沒有淨環流**。一條只朝單方向轉的圈（紙上推演第 2 題）有環流，倒放立刻露餡；而沒有迴圈的圖——比如一條線——想有環流也沒地方放。這句話直接預告了下一節。

## 生滅過程皆可逆

生滅過程（ch07）的轉移圖就是一條線：

```text
        λ₀         λ₁         λ₂
     ─────►     ─────►     ─────►
 (0)        (1)        (2)        (3) ⋯
     ◄─────     ◄─────     ◄─────
        μ₁         μ₂         μ₃
```

> **定理**：任何 ergodic 的生滅過程，跑在穩態時都是可逆的。

證明（完整推導，建立在 ch07 已證的結果上）：

```text
生滅鏈的轉移只發生在相鄰狀態之間：q(n→n+1) = λₙ、q(n→n−1) = μₙ，其餘速率為 0。
ch07 用相鄰 cut 推出的 local balance：
    πₙ·λₙ = πₙ₊₁·μₙ₊₁        （對所有 n）
逐對檢查 detailed balance：
    相鄰對 (n, n+1)：πₙ·q(n→n+1) = πₙ₊₁·q(n+1→n)   ← 就是上面那條 local balance
    非相鄰對 (i, j)：兩邊速率皆 0，0 = 0             ← 自動成立
所有狀態對都過 ⇒ detailed balance 成立 ⇒ 可逆。∎
```

用 Kolmogorov 判準看更快：一條線沒有迴圈，環流無處可藏。所以這不是巧合，是一維結構的必然——ch07 說「生滅鏈的 local balance 與 global balance 一致」，本章把同一件事換了個名字：**生滅鏈天生滿足 detailed balance**。

立刻入袋的系（corollary）：穩態的 M/M/1、M/M/c、M/M/∞、M/M/1/K 全部可逆。停下來體會一下這句話有多怪：一個 M/M/1 佇列的佇列長度錄影，倒著放，統計上依然是一個合法的 M/M/1 佇列長度錄影。你的直覺大概在抗議——「正放是到達把它堆高、服務把它消掉，倒放哪有這回事」。抗議無效，但這份抗議值錢：下一節整段都在兌現它。

## Burke 定理：穩態輸出是 Poisson，而且系統失憶

> **Burke 定理**（Burke 1956；適用於 M/M/1、M/M/c、M/M/∞，λ < cμ，系統跑在穩態）：
> 1. 離開過程是速率 λ 的 Poisson 過程。
> 2. 任一時刻 t 的系統狀態 N(t)，與 t 之前的全部離開歷史獨立。

Burke 1956 的原始證明走輸出間隔的直接分析；隔年 Reich 1957 給出第二個證明，也就是今天通行的時間反轉路線——許多人（包括我）認為它是排隊理論裡最漂亮的論證之一。以下就走這條路（工程師的嚴謹：每一步能口頭說出理由；完整證明見 Burke 1956 與 Harchol-Balter 第 16 章）。

先把 M/M/1 的樣本路徑畫出來。N(t) 是一條樓梯：↑跳＝到達、↓跳＝離開：

```text
N(t)
 2 │        ┌────┐     ┌──┐
 1 │   ┌────┘    └─────┘  └────┐
 0 │───┘                       └─────
   └──────────────────────────────────► t
       a    a    d     a  d    d

a = 到達（↑跳）   d = 離開（↓跳）
```

把這段錄影倒著放。時間軸左右翻轉之後，每個跳躍的方向也翻轉，於是有這本對照字典：

| 正放影片裡的事件 | 倒放影片裡看到的 |
|---|---|
| 到達（↑跳）發生在時刻 t | 離開（↓跳）發生在時刻 τ−t |
| 離開（↓跳）發生在時刻 t | **到達（↑跳）發生在時刻 τ−t** |

證明只有四步：

1. **穩態 M/M/1 可逆**（上一節）：倒放的錄影在統計上就是一個 M/M/1——到達率 λ、服務率 μ，一模一樣。
2. **字典翻譯**：正放影片的離開時刻們，恰好是倒放影片的到達時刻們（時間座標翻轉而已）。
3. **倒放影片的到達是 Poisson(λ)**：因為倒放影片是一個如假包換的 M/M/1，而 M/M/1 的到達過程依定義是 Poisson(λ)。
4. **Poisson 倒放仍是 Poisson**：Poisson 過程由「平穩獨立增量＋區間計數 Poisson 分布」刻畫（ch05），每一項都不在乎時間箭頭朝哪邊——區間裡有幾個點，正著數倒著數一樣多。

所以：正放影片的離開過程是 Poisson(λ)。第 1 句證完。

第 2 句靠同一本字典再翻一次。在任何 M/M/1 裡（倒放那個也是），**未來的到達與現在的狀態獨立**：N′(s) 由「s 之前的到達」與「s 之前的服務進度」拼出來，而 Poisson 的獨立增量保證 s 之後的到達跟這一切無關——到達流不偷看系統（這正是 ch08 PASTA 背後那個 lack of anticipation 假設）。現在翻譯：倒放影片裡「s 之後的到達」＝正放影片裡「t = τ−s 之前的離開」；倒放影片裡「s 時刻的狀態」＝正放影片裡「t 時刻的狀態」（N′(s) = N(τ−s) = N(t)）。於是：**N(t) 與 t 之前的離開歷史獨立**。∎

整個證明裡 μ 只出現在「λ < μ 所以穩態存在」這一處。這值得瞪著看一會兒：輸出的速率是 λ 不奇怪——穩態下進多少就得出多少，這是守恆（ch02）；奇怪的是**形狀**：間隔獨立、同為 Exp(λ)，服務率 μ 在輸出的統計面貌裡完全隱形。μ = 10.1 的勉強系統和 μ = 1000 的閒散系統，穩態輸出流在統計上無法區分。

### 反直覺正面對決：剛吐出一串 job，不代表現在比較空

第 2 句才是真正讓人睡不著的。回到值班場景：過去一分鐘離開了一大串 job——佇列現在比較空嗎？定理說：**你的條件機率一動也不動**，P(N(t) = n | 任何離開歷史) = πₙ。

直覺為什麼錯？因為有兩股推論在拔河，而你只聽見其中一股：

- 「離開很多 ⇒ 東西出去了 ⇒ 裡面應該變少」——往**空**的方向推。
- 「離開很多 ⇒ 伺服器這段時間幾乎沒停 ⇒ 裡面本來就堆得高」——往**滿**的方向推。

Burke 定理說這兩股在 M/M 穩態下**恰好抵銷**，精確到一個 bit 都不剩。「恰好」不是修辭，可以動手算。先要一塊墊腳石（工程師的嚴謹）：M/M/1 裡「離開瞬間留下 n 個」的機率就是 πₙ——因為 N 一次只動 ±1，在 n 與 n+1 之間畫一條線，上穿與下穿的次數最多差 1，長期頻率相等；於是「離開後一瞬留下 n」的分布等於「到達前一瞬看到 n」的分布，而後者由 PASTA（ch08）等於 πₙ。現在用本章基準的站 1（λ = 6、μ = 10、ρ = 0.6）算「下一次離開要等多久」：

```text
剛有一個 job 離開。留下的狀態 ~ π：
  忙（留下 ≥ 1 個）：機率 1 − π₀ = ρ = 0.6   → 下個間隔 ~ Exp(10)
  空（留下 0 個）  ：機率 0.4               → 先等到達 Exp(6)，再服務 Exp(10)

間隔密度：
  Exp(6)+Exp(10) 的卷積 = (6·10/(10−6))·(e^(−6t) − e^(−10t)) = 15·(e^(−6t) − e^(−10t))
  f(t) = 0.6·10·e^(−10t) + 0.4·15·(e^(−6t) − e^(−10t))
       = 6·e^(−10t) + 6·e^(−6t) − 6·e^(−10t)
       = 6·e^(−6t)                                ← 恰好是 Exp(λ=6)，μ 的痕跡全數抵銷
```

「忙的時候快（Exp(10)）」與「空的時候慢（先等到達）」兩種情境，以 ρ 為權重混出來的不多不少正是 Exp(λ)。誠實標示：這個計算只驗證了**邊際**間隔是指數——要構成 Poisson 過程還需要間隔彼此獨立，那一半便宜的算術買不到，正是時間反轉論證的價值所在。

最後把定理的不對稱性說破，免得你把它記成比實際更神的樣子。N(t) 與**過去的**離開獨立，但與**未來的**離開當然相關——現在堆了 20 個，接下來的離開肯定比較密。奇怪嗎？對照輸入端就不奇怪了：未來的到達與 N(t) 獨立（到達不偷看系統），但過去的到達正是堆出 N(t) 的材料。時間反轉把輸入端的不對稱原封不動搬到輸出端，方向對調。Poisson 過程本身對稱，**「誰決定誰」的因果結構不對稱**——定理漂亮就漂亮在它把這件事翻譯得分毫不差。

## 串聯佇列：乘積形式初登場

鑰匙到手，開第一扇門。本章基準的串聯系統（tandem queue）：

```text
 Poisson(λ=6) ──► [ 站1：M/M/1，μ₁=10 ] ──► [ 站2：M/M/1，μ₂=8 ] ──► 離開系統

                 中間那條內部流：站1 的輸出＝站2 的輸入（Burke：仍是 Poisson(6)）
```

工程對應：站 1 是 API 處理、站 2 是推播 worker，事件流一站接一站。穩定性先檢查：λ = 6 同時小於 μ₁ = 10 與 μ₂ = 8，兩站都撐得住。

Burke 第 1 句說：站 1 的穩態輸出是 Poisson(6)，而它正是站 2 的輸入；站 2 的服務時間 Exp(8) 與這條輸入流獨立——所以**站 2 自己就是一個正牌 M/M/1(λ=6, μ=8)**，ch08 的全套公式直接適用。這已經把「對每段各自容量規劃」的合法性買下一半。另一半——兩站的狀態彼此什麼關係——靠第 2 句（工程師的嚴謹）：N₂(t) 由「站 2 在 t 之前的到達流」與「站 2 自己的服務時間」拼出來；前者就是站 1 在 t 之前的離開流，由 Burke 第 2 句與 N₁(t) 獨立；後者跟整個站 1 無關。兩塊材料都與 N₁(t) 獨立，成品 N₂(t) 也就與 N₁(t) 獨立。聯合分布於是**逐點乘開**：

```text
站1：ρ₁ = 6/10 = 0.6     π₁(n₁) = 0.4·(0.6)^n₁        （ch08 的 (1−ρ)ρⁿ）
站2：ρ₂ = 6/8  = 0.75    π₂(n₂) = 0.25·(0.75)^n₂

π(n₁, n₂) = π₁(n₁)·π₂(n₂) = 0.1·(0.6)^n₁·(0.75)^n₂     ← 例：兩站全空 π(0,0) = 0.1

E[N₁] = ρ₁/(1−ρ₁) = 0.6/0.4   = 1.5
E[N₂] = ρ₂/(1−ρ₂) = 0.75/0.25 = 3
E[N]  = E[N₁] + E[N₂] = 4.5                     ← 期望值線性（ch03），這步不需要獨立性
E[T]  = E[N]/λ = 4.5/6 = 0.75 s                 ← Little's Law 套整條管線

複核（另一條路，逐站算回應時間再相加）：
E[T₁] = 1/(μ₁−λ) = 1/4 = 0.25 s                 ← ch08：M/M/1 的 T ~ Exp(μ−λ)
E[T₂] = 1/(μ₂−λ) = 1/2 = 0.50 s
E[T]  = 0.25 + 0.50 = 0.75 s ✓
```

讀這張帳單的方式：μ₂ = 8 才是瓶頸——它離 λ 只剩 2/s 的餘裕，一站就貢獻了 4.5 個在席數裡的 3 個、0.75 秒裡的 0.5 秒。想改善整條管線先動誰，數字已經把話說完（紙上推演第 4 題會逼你算另一個選項）。

再多收一筆紅利（敘述＋計算；獨立性本身本書不證）。Reich 在同一篇 1957 論文裡證明：同一個 job 在兩站的逗留時間 T₁、T₂ **互相獨立**。配上 ch08 的 T ~ Exp(μ−λ)，總逗留 T = T₁ + T₂ 是獨立的 Exp(4) 與 Exp(2) 之和，尾巴可以閉式寫出：

```text
P(T > t) = (4·e^(−2t) − 2·e^(−4t))/(4−2) = 2·e^(−2t) − e^(−4t)
t₉₉：令 2·e^(−2t) ≈ 0.01（另一項到時只剩 ~2.5×10⁻⁵，可棄）
     t₉₉ ≈ ln(200)/2 = (ln 2 + ln 100)/2 = (0.6931 + 4.6052)/2 ≈ 2.65 s
```

P99 ≈ 2.65 s ≈ 3.5 倍平均（0.75 s）——比單站 M/M/1 的 4.6 倍（ch08）緊一些：兩段獨立的延遲相加，極端值互相稀釋了一點。

最後一個誠實聲明，它是 ch16 的預告片：這裡的獨立是**同一時刻快照**的獨立——固定 t，N₁(t) 與 N₂(t) 獨立。**過程**可不獨立：站 1 此刻的長龍預告著站 2 待會兒的長龍（burst 會順流而下），N₁(t) 與 N₂(t+s) 相關得很。乘積形式是穩態快照的性質，不是時間軌跡的性質——這句話在 ch16 會以更驚人的形式回來。

## Burke 的極限：feedback 殺死內部流的 Poisson

串聯能贏，是因為流只往前。現在加一條現實中無所不在的邊：失敗重試。完成的 job 以機率 p 回到佇列尾端重排（Bernoulli feedback，這裡取瞬時回流的理想化）：

```text
               ┌───────────── p ────────────┐
               ▼                            │
 Poisson(λ) ──► + ──► [ server μ ] ─────────┴──► 真正離開（機率 1−p）

 「+」＝合流點：外部新工作與回鍋工作在這裡匯入同一條佇列
```

問：伺服器實際看到的輸入流（外部流＋回饋流的合流）還是 Poisson 嗎？**不是。**這不是近似誤差，是定性死亡——Burke 本人 1976 年算出這條合流的邊際到達間隔是兩個指數的混合、不是指數，Poisson 就此出局（Burke 1976, IEEE Trans. Comm. 24: 575–576）。

直覺上死因很清楚：回饋流**偷看系統**。回鍋的到達緊跟在服務完成之後發生，跟佇列自己的狀態強烈相關——整套 Poisson 機器賴以運轉的 lack of anticipation（ch08）被正面擊穿。體感版：取 λ 很小、μ 很大、p 接近 1——外部工作難得來一個，一來就在伺服器裡連環回鍋，輸入端看到的是「長久死寂，然後一陣機關槍」。間隔高度叢發（C²ₐ > 1）、前後相關，跟 Poisson 的均勻無記憶相去十萬八千里。可逆性論證在哪一步斷掉？回饋讓「到達」不再是外生的 Poisson 流，字典第 3 步「倒放影片是如假包換的 M/M/1」直接報廢。

有趣的是有一條流活了下來。瞬時回流下 N(t) 仍是生滅過程——外部到達使 n→n+1（速率 λ），服務完成以機率 1−p 使 n→n−1（速率 μ(1−p)），回鍋不改變 n。生滅 ⇒ 可逆 ⇒ 對這條鏈重跑四步論證：它的↓跳——**真正離開系統的流**——仍是 Poisson(λ)（工程師的嚴謹；穩定性條件相應變成 λ < μ(1−p)）。所以精確的講法是：**feedback 殺死內部流的 Poisson 性，但放過對外輸出**。這個格局在 ch16 的 Jackson 網路裡原樣放大：網路內部的流普遍不是 Poisson，對外離開的流仍是。

於是 ch16 的驚奇已經可以預告了：基準範例裡 node2 以 p = 0.2 把失敗送回 node1，內部流統統不是 Poisson——但 Jackson 定理會說，穩態快照的乘積形式**仍然成立**，每站「彷彿」獨立 M/M/1。Poisson 死了，乘積形式卻活著；化解這個悖論的概念工具（什麼條件下輸入輸出都是 Poisson 且與狀態獨立）叫 **quasi-reversibility**，本書不展開，指路 Kelly《Reversibility and Stochastic Networks》。

工程翻譯一句：你的管線只要有 retry、有 DLQ 重投、有任何「失敗繞回上游」的邊，內部每段流量就不再有 Poisson 執照——對單站套 M/M/1 之前，先看一眼輸入是從哪裡來的。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦與察覺 |
|---|---|---|
| 把 Burke 用在暫態 | 定理只保**穩態**。清積壓時站 1 以滿速 μ₁ = 10/s 往下游灌，而站 2 只吃得下 8/s——用「下游輸入 = λ = 6」做的容量帳當場作廢，backlog 沿管線下移 | 排空事件（部署重啟、尖峰剛過、佇列 drain）期間，下游容量帳要用上游的 **μ** 不是 λ。徵兆：上游 backlog 下降的同時下游延遲飆升 |
| 把 Burke 用在 M/G/1 | 定理需要指數服務。一般服務時間下輸出不是 Poisson：M/D/1 輸出比 Poisson 平滑（下游比 M/M 預測的好過），重尾服務輸出叢發（下游慘過預測）——方向感，敘述層級 | 服務時間 C²ₛ 偏離 1 越遠越別信。下游要量化時用 C² 參數傳播的分解近似（見 ch18） |
| 把第 2 句讀成「狀態與輸出整體獨立」 | 獨立的是 N(t) 與**過去的**離開；與未來的離開當然相關（堆 20 個時接下來離開必然較密） | 記住不對稱性的來源：輸入端「未來到達 ⊥ 現在」經反轉變成輸出端「過去離開 ⊥ 現在」。方向弄反，推論全錯 |
| 對「剛吐出一串」做反應 | 穩態下離開歷史對現在狀態的資訊量為零；據此調度（如把流量切過去）等於擲硬幣 | 要知道現在多滿，去量 N(t) 本身（佇列深度指標），不要從吞吐反推 |
| 有 feedback 還按 Poisson 算每段 | retry／DLQ 重投讓內部流叢發（C²ₐ > 1）且與佇列狀態相關，M/M/1 公式系統性低估等待（ch14 的 Kingman 會把 C²ₐ 的帳算給你看） | 畫出流量圖，凡是有迴邊的，內部流一律當非 Poisson 處理；流量帳先解（ch16 traffic equations），等待帳再說 |
| 把快照獨立當過程獨立 | 乘積形式只管固定時刻的聯合分布；burst 沿管線傳播，跨站延遲在時間上高度相關——「兩站同時壞」的機率不是 π 相乘能回答的 | 涉及時間相關的問題（同一 burst 連環打擊、跨站 SLA 的同時違約）改用模擬（ch19） |

## 紙上推演

**1. [10 分鐘 ★] 猜驗法熱身。**兩台 worker 共用一條佇列（M/M/2）：λ = 6/s，每台 μ = 4/s。(a) 先說明為什麼可以直接用 detailed balance 解 π 而不必碰 global balance；(b) 用猜驗法解出 πₙ 與 π₀，並算 P(全空) 與 P(必須等待)。

**2. [15 分鐘 ★★] 單向圓環：可逆性的反例。**三狀態 A→B→C→A，速率皆為 1，反方向速率皆為 0。(a) 求平穩分布並驗證 global balance；(b) 驗證 detailed balance 失敗、Kolmogorov 迴圈判準失敗；(c) 寫出反轉鏈的速率，說明「倒放影片」長什麼樣；(d) 用猜驗法硬做會在哪一步翻車？

**3. [15 分鐘 ★★★] 口頭重講 Burke。**閉卷，把時間反轉證明完整講一遍，依序回答：(a) 為什麼穩態 M/M/1 可逆？(b) 倒放之後，原本的離開變成什麼？(c) 為什麼倒放影片的到達是 Poisson(λ)？(d) 第 2 句（狀態與離開歷史獨立）是從倒放世界的哪一條性質翻譯來的？(e) 「穩態」這個假設在證明的哪幾步出力？拿掉會壞在哪？

**4. [15 分鐘 ★★] 直覺誤用＋升級決策。**(a) 值班同事說：「push worker 過去一分鐘吞吐暴增，佇列現在一定比較空，可以把灰度流量切過來了。」指出這句話在穩態下錯在哪；再指出在**非**穩態（清積壓）下它錯得更兇的原因。(b) 串聯基準（λ=6、μ₁=10、μ₂=8）只有一筆升級預算：把 μ₁ 提到 12，或把 μ₂ 提到 10。各算新的 E[T]，說出該選哪個、贏多少。

### 推演解答

**1.** (a) M/M/2 是生滅過程（μₙ = min(n,2)·μ，見 ch09），本章已證生滅過程的平穩解滿足 detailed balance——所以 detailed balance 解出來的就是 π，合法。先檢查穩定：λ = 6 < 2·4 = 8，ρ = 0.75。(b) 沿著線傳播：

```text
0↔1：π₁·4 = π₀·6  ⇒ π₁ = 1.5·π₀
n↔n+1（n≥1）：πₙ₊₁·8 = πₙ·6 ⇒ πₙ₊₁ = 0.75·πₙ
⇒ πₙ = 1.5·(0.75)^(n−1)·π₀（n ≥ 1）
正規化：π₀·[1 + 1.5·(1/(1−0.75))] = π₀·(1 + 6) = 7·π₀ = 1 ⇒ π₀ = 1/7 ≈ 0.143
P(必須等待) = P(N ≥ 2) = 1.5·π₀·(0.75 + 0.75² + ⋯) = 1.5·3·π₀ = 4.5/7 ≈ 0.643
```

複核：用 ch09 的 Erlang-C 結構驗 π₀——a = λ/μ = 1.5，[1 + 1.5 + (1.5²/2)/(1−0.75)] = 1 + 1.5 + 4.5 = 7 ✓。常見錯路：n ≥ 1 之後死亡速率忘了從 4 換成 8。

**2.** (a) 對稱性猜 π = (1/3, 1/3, 1/3)；驗 global balance：每個狀態流入 = 1/3·1 = 流出 ✓。(b) detailed balance 在邊 A–B 上要求 π_A·1 = π_B·0，即 1/3 = 0，失敗。Kolmogorov：順走 A→B→C→A 乘積 = 1，逆走乘積 = 0，不等，失敗。(c) q′(i→j) = πⱼ·q(j→i)/πᵢ = q(j→i)：反轉鏈是同樣的圓環**反方向轉**（A→C→B→A，速率 1）。倒放影片看得出旋轉方向反了——可逆性的失敗就是「轉移圖上有淨環流」這件事的精確化。(d) 猜驗法第 2 步就翻車：沿 A→B 傳播要解 π_B = π_A·q(A→B)/q(B→A) = π_A·(1/0)，除以零——方程無解，這正是「不可逆」在操作上的訊號。注意教訓：π 好端端存在（global balance 解得出來），死的只是捷徑。

**3.** 參考答案骨架：(a) M/M/1 是生滅過程，穩態生滅過程滿足 detailed balance（ch07 的相鄰 cut 方程逐字就是），故可逆。(b) 時間軸翻轉使↓跳變↑跳：原離開時刻成為倒放影片的到達時刻。(c) 可逆 ⇒ 倒放影片與正放同法則，是一個 M/M/1，其到達依定義為 Poisson(λ)；再加一句「Poisson 倒放仍是 Poisson」（平穩獨立增量不認時間方向）。(d) 從「M/M/1 的未來到達與現在狀態獨立」（lack of anticipation）翻譯：倒放的未來到達＝正放的過去離開、倒放的現在狀態＝正放的現在狀態。(e) 穩態出力兩次：反轉速率公式推導裡 P(X(t−h)=j) = πⱼ 要求每一時刻的邊際都是 π（否則 q′ 隨時間變，倒放不是時齊 CTMC）；可逆性的定義本身是穩態過程的性質——從空系統起跑的 M/M/1 倒放後是一條「先知道結局」的怪過程，整個論證不成立。這也是為什麼清積壓期間的輸出不是 Poisson(λ)。

**4.** (a) 穩態下：Burke 第 2 句說 N(t) 與過去離開獨立——「吞吐暴增」對佇列現況的資訊量是零，這個決策等於沒看資料。非穩態下更糟：吞吐能持續壓在 μ ≈ 滿速，恰恰代表伺服器沒停過、backlog 還在——同一個觀察的正確讀法跟同事的結論**方向相反**。要嘛量佇列深度，要嘛別動。(b) 算帳：

```text
基準：E[T] = 1/(10−6) + 1/(8−6) = 0.25 + 0.50 = 0.75 s
升級 μ₁→12：E[T] = 1/(12−6) + 1/(8−6) = 0.167 + 0.50 ≈ 0.667 s   （省 0.083 s）
升級 μ₂→10：E[T] = 1/(10−6) + 1/(10−6) = 0.25 + 0.25 = 0.50 s    （省 0.25 s）
```

選 μ₂：同樣的預算，瓶頸站的收益是非瓶頸站的 3 倍。這是 ch02 bottleneck 分析的機率版重演——錢永遠先花在離 λ 最近的那個 μ 上。
