# 寫作風格指南（內部文件，非書籍內容）

本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名（工作名）：《等待的數學：排隊理論》。

## 這本書是什麼、不是什麼

- **是**：一本**數學嚴謹為主**的興趣書。讀者要能親手推導每個主要公式（從 πₙ=(1−ρ)ρⁿ 到 Pollaczek–Khinchine、從 Little's Law 到 Jackson 定理）、口頭重講每個關鍵證明、並對「公式的適用條件」有肌肉記憶。目標是把讀者既有的「queue 理論直覺」升級成**可推導、可質疑、知道邊界**的硬功夫。
- **不是**：
  - 不是容量規劃手冊——工程應用是橋接錨點與點綴（每章「從你已知的出發」＋ch20 案例章），不是主軸。
  - 不是模擬工具教學——Python 小實驗自寫短程式，不教 simpy/ciw 的 API（它們的存在與定位講一句即可）。
  - 不是研究所隨機過程全課——matrix-analytic、fluid/diffusion limits、MDP 控制只在 ch21 地圖上標位置，不展開。
  - 不服務 AI Infra 轉職敘事——純興趣定位；LLM serving 只在 ch20 當點綴，深度指向書架上的《從後端到 AI Infra》。
- 驗收標準（全書主線，ch21 收官）：①為什麼 ρ→1 時一切爆炸、爆炸的形狀是什麼；②隨機性（變異數）本身就是延遲的來源；③平均值與 tail 是兩回事；④每個經典公式「條件不滿足時會怎麼騙你」；⑤網路層級的湧現行為（product form 的驚奇與極限）；⑥模擬什麼時候比公式誠實、怎麼讓模擬誠實。

## 數學記號規範（全書一致，附錄 A 是速查表；閱讀器不渲染 LaTeX，一律純文字）

- **內文行內**用 Unicode：λ μ ρ π σ² Σ ∫ ∞ √ ≈ ≠ ≤ ≥ → ⇒ ∀ ∃ ∈，prime 用 ′。
- **核心記號表**（全書唯一基準，不得混用教科書的 L/W 系統）：
  - λ 到達率、μ 單一伺服器服務率、c 伺服器數、ρ = λ/(cμ) 使用率、a = λ/μ offered load
  - S 服務時間隨機變數（E[S] = 1/μ）、T 回應時間（含服務）、T_Q 等待時間（不含服務）
  - N 系統內人數、N_Q 佇列中人數；E[N] = λE[T]（Little's Law 的標準形）
  - πₙ = P(N = n) 平穩機率；C²ₐ、C²ₛ 為到達間隔／服務時間的變異係數平方（SCV）
  - 與教科書 L/W 記號的對照**只在 ch02 與附錄 A 各給一次**，內文不用 L/W。
- 下標規則：單一小寫字母下標用 Unicode（πₙ、π₀、C²ₛ、C²ₐ）；大寫或多字母下標用底線（E[T_Q]、E[N_Q]、λ_eff）。
- Kendall 記號 A/S/c/K 在 ch01 先見面（地圖語言）、ch08 正式定義。
- **多行推導**放在 ```text 圍欄區塊裡，等號對齊、每步右側可加「← 為什麼」短註。禁止把長推導硬塞進行內。
- 數字基準：e ≈ 2.71828、ln 2 ≈ 0.6931、ln 100 ≈ 4.6052（全書統一，按需要截短，不得互相矛盾）。機率與時間值預設保留 3 位有效數字（如 0.596、74.6 ms）。
- 圖：ASCII diagram（狀態轉移圖、佇列示意、hockey stick 曲線、全書地圖），必須通過 `check_diagrams.py` 的 CJK=2 欄寬檢查。曲線用 `*` 描點、座標軸用 `─ │ └`；不追求精確比例，追求形狀直覺。

## 每章骨架（必須遵守，標題名稱照抄——一致性掃描會 grep 這些標題）

1. 章首：`# chNN — 章名：副標`，接著 blockquote：
   `> **本章解決什麼問題**：…（這章在全書地圖的哪裡、為什麼存在、跟前後章的關係，3-5 行）`
   每個 Part 的第一章（ch01、ch03、ch07、ch12、ch16、ch19）在 blockquote 後放全書 ASCII 路線圖（基準圖在 outline.md，照抄並把「◄ 你在這裡」標到本 Part），讀者偏好架構圖先行。
2. `## 從你已知的出發` — 橋接段：把本章核心概念錨定在讀者的工程實戰上，要具體（SQS consumer 數量怎麼決定、k6 壓測的 P99 曲線、3,200 條 WebSocket 連線的容量帳、RDS CPU 80% 警戒線），不要泛泛而談。
3. 核心內容（多個 `##` section）。**推導優先於結論**：先讓讀者看到「為什麼非得是這個形狀」，再給公式；歷史脈絡服務於數學動機（Erlang 在電話交換機房遇到什麼問題），不是編年史。
4. `## 陷阱與防禦` — 本書招牌段落（故障視角的數學版）：這個公式／模型**的適用條件是什麼**、條件不滿足時**它怎麼騙你**（給出具體會錯多少的例子）、**怎�麼自我察覺**（什麼徵兆代表模型不對）。用表格或條列。
5. `## 紙上推演` — 每章 2~4 題，標注預估時間（如 **[15 分鐘]**）與難度（★~★★★）。題型：手推公式、手算數值表、判斷模型適用性、找出錯誤論證的破綻、口頭重講一個證明。題目後緊接 `### 推演解答` 給完整解答與討論（為什麼這樣想、常見錯路）。
   指定章節（ch05、ch08、ch12、ch15、ch17、ch19、ch20——見 outline）額外加 `### Python 小實驗`：本機可跑、標準函式庫優先（`random`、`statistics`、`heapq`；numpy/matplotlib 可選用、要標明）、每段程式 ≤ 40 行、**必須固定亂數種子**（`random.seed(42)`）、預期輸出寫成「理論值 X，模擬值應落在 X±誤差」的形式而不是假裝模擬會輸出精確值。程式碼與註解一律英文。
6. `## 自我檢核` — 5~8 個問題，**口頭自答**：講得出來才算過關，講不清楚＝還沒懂。優先出「為什麼」與「什麼條件下不成立」，不是「是什麼」。
7. `## 延伸閱讀` — 真實連結（以 `_meta/landscape-2026-06.md` 驗證過的為優先），每條一句話說明為什麼值得讀、讀哪一段。

## 深度標準

- **每章至少一個 worked example**：帶具體數字、一步一步算的完整過程（基準範例「推播管線」的數字見 outline 跨章基準表），不准跳步。
- **嚴謹度的誠實標示**：每個論證明確標示自己是哪一級——「完整推導」「工程師的嚴謹：每步能口頭說出理由」「直覺版（本書不證，見延伸閱讀）」。禁止「顯然」「易證」；禁止把直覺論證偽裝成證明。該不證的（PASTA 完整證明、Jackson 定理一般形、SRPT 最適性完整證明、heavy-traffic 極限定理）明寫不證並指路。
- **每個公式出場必附適用條件**：什麼假設下成立（Poisson 到達？指數服務？穩態？FCFS？）、哪個假設最常在現實中破掉。這是本書與「背公式」的根本差別。
- **數值自我複核**：所有數值（推導結果、表格、Erlang 公式值）寫章時必須自己重算一遍，不得抄自記憶或外部來源。跨章基準數值以 outline 基準表為準；表中標「初算參考」的值必須獨立重算，算出不同要停下來報告而不是沿用。
- 每個概念給「它在解什麼問題」：無記憶性解什麼、PASTA 解什麼、可逆性解什麼。禁止無動機的定義。
- 決策框架：什麼時候 M/M/1 夠用、什麼時候要 G/G/1 近似、什麼時候只能模擬——給判準，不給教條。
- 寧深勿廣，不灌水。每章 5,000~9,000 中文字，依主題深度調整。
