## 從你已知的出發

你的系統畫出來就是一張呼叫圖：API 收事件、推播 worker 發送、失敗的訊息繞回去重走一次。ch11 給了這張圖一半的理論執照：只要流一路向前（tandem），Burke 定理保證上游輸出仍是 Poisson，每段各自套 M/M/1 合法。但 ch11 結尾也吊銷了另一半執照：只要有一條迴邊——retry、DLQ 重投、任何「失敗繞回上游」——內部流就不再是 Poisson，而且不是近似誤差，是定性死亡。

於是你卡在一個尷尬的位置。一邊是工程現實：迴邊無所不在，你的 SQS 結算 pipeline、webhook 重送、推播失敗重投，全都長這樣。另一邊是理論潔癖：沒有 Poisson 輸入，M/M/1 的每條公式都失去前提。直覺的結論是「per-station 模型死了，乖乖去模擬」。本章的主角說：不。穩態的聯合分布仍然逐站相乘——這個「仍然」精確到什麼程度、靠什麼成立、騙不騙人，是本章要一刀一刀切開的。

第二條線索來自監控。每個 microservice 各有 dashboard，各自的 RPS、各自的使用率，看起來都健康。但「各自的 RPS」是量出來的結果，不是設計時就知道的輸入：一條 retry 迴邊會把外部 6/s 放大成站內 7.5/s，放大係數藏在路由結構裡，不在任何單一服務的儀表板上。ch02 你用 visit ratio Vᵢ 與 service demand Dᵢ 做過 bottleneck 分析，當時 Vᵢ 是從量測讀出來的數字；本章補上它的出生證明——traffic equations 會從 routing matrix 把 Vᵢ 生出來。「每個服務各自看起來健康、整體卻在懸崖邊」這句值班室老話，讀完本章會變成一個你能親手算出臨界點的定理。

## 開放網路：把呼叫圖變成數學

**開放 Jackson 網路（open Jackson network）**：k 個服務站，外加一個「外面的世界」。工作從外面進來、在站與站之間流動、最終離開。完整的假設清單如下——每一條都要唸出聲，因為它們正是 ch18 要逐一打破的清單：

1. **外部到達是 Poisson**：站 i 收到速率 rᵢ 的外部 Poisson 流，各站的外部流互相獨立；總外部到達率 λ₀ = Σᵢ rᵢ。
2. **指數服務**：站 i 是單一伺服器，服務時間 i.i.d. ~ Exp(μᵢ)，與一切獨立（Jackson 1963 推廣到狀態相依速率，涵蓋多伺服器站，紙上推演第 3 題會用到）。
3. **機率式路由（probabilistic routing）**：在站 i 完成的工作，以機率 Pᵢⱼ 前往站 j、以機率 1 − Σⱼ Pᵢⱼ 離開系統——擲骰子決定，跟工作身分、佇列狀態、過往歷史全部無關。k×k 矩陣 P = (Pᵢⱼ) 叫 **routing matrix**，每列總和 ≤ 1（次隨機）。
4. **FCFS、無限緩衝**：每站先到先服務，永不丟件。
5. **開放性**：從任何站出發，沿著路由走，以機率 1 最終離開系統——工作不會永遠困在裡面打轉。

對照你的呼叫圖：條件 3 排除了「路由看佇列長度挑短的」（load balancer 的 join-shortest-queue）；條件 4 排除了 bounded queue 與 load shedding（見 ch10）；條件 3 也排除了 fork-join——一個請求複製成多份、要等全部回來（ch18 的主角）。Jackson 世界是個理想國，但它是第一個能整網求解的理想國。

本章貫穿範例（跨章基準）：外部 Poisson(λ₀ = 6/s) 進入 node1（API，μ₁ = 10/s），處理完送 node2（推播 worker，μ₂ = 8/s）；node2 完成後以 p = 0.2 失敗、送回 node1 重試，以 0.8 真正離開。routing matrix：P₁₂ = 1、P₂₁ = 0.2，其餘為 0；外部流 r₁ = 6、r₂ = 0。

```
                   ┌──────────── p = 0.2（失敗重試送回 node1） ────────────┐
                   ▼                                                       │
 Poisson(λ₀=6) ──► + ──► [ node1：API，μ₁=10 ] ──► [ node2：推播，μ₂=8 ] ──┴──► 離開（1−p = 0.8）
                         λ₁ = 7.5/s                λ₂ = 7.5/s
```

「+」是合流點：外部新工作與 retry 回流匯入同一條佇列。圖上的 λ₁ = λ₂ = 7.5 從哪來？下一節算給你看。

## Traffic equations：先把流量帳算平

問每一站：穩態下你每秒吞吐多少？設站 i 的總到達率（＝總離開率，穩態守恆）為 λᵢ，數帳：

```text
λᵢ = rᵢ + Σⱼ λⱼ·Pⱼᵢ        （i = 1, …, k）
      ↑        ↑
   外部直達   各站 j 每秒完成 λⱼ 件、其中比例 Pⱼᵢ 被送來站 i
```

這組線性方程叫 **traffic equations（流量方程）**。先把它的身分講清楚：它只是流量守恆——不需要 Poisson、不需要指數、不需要 FCFS，跟 ch02 的 operational laws 同一國籍。事實上 λᵢ/λ₀ 就是 ch02 的 visit ratio Vᵢ（每個工作平均造訪站 i 幾次），traffic equations 是 Forced Flow Law 的出生證明。機率假設要到下一節的定理才進場。

**解的存在與唯一**（工程師的嚴謹：每步能口頭說出理由）。把路由看成一條 Markov chain：狀態是 {站 1, …, 站 k, 離開}，「離開」是吸收態。條件 5（開放性）說：從每站出發都以機率 1 被吸收。吸收鏈的標準結果（ch06 的 transient 狀態）：每個工作對站 i 的期望造訪次數有限。逐輪數流量：第一輪是直達的 r、第二輪是走了一步路由的 rᵀP、第三輪 rᵀP²……加總是一個收斂的級數（吸收性保證它收斂），其極限就是 traffic equations 唯一的解；用矩陣寫是 λ = (I − Pᵀ)⁻¹ r，逆矩陣存在正因為那個級數收斂。完整的線性代數版本本書不展開——但對本章基準你不需要矩陣，一條幾何級數就夠：

```text
λ₂ = λ₁                    ← node1 完成的全部送 node2
λ₁ = 6 + 0.2·λ₂            ← 外部直達 + node2 的回流
⇒ λ₁ = 6 + 0.2·λ₁  ⇒  λ₁ = 6/(1−0.2) = 7.5 = λ₂

逐輪讀法（ch03 的幾何級數）：
λ₁ = 6 + 6·0.2 + 6·0.2² + ⋯ = 6·(1 + 0.2 + 0.04 + ⋯) = 6/0.8 = 7.5
     首次     第一次重試   第二次重試
```

每個工作平均造訪每站 V = 1/(1−p) = 1.25 次——retry 是個流量放大器，放大倍率是幾何級數和。於是 service demand：D₁ = V/μ₁ = 0.125 s、D₂ = V/μ₂ = 0.15625 s，ch02 的 bottleneck bound 立刻給出整網吞吐上限 λ₀ < 1/D_max = 6.4/s。外部流量 6/s 看起來離 μ₂ = 8 很遠，實際上已經吃掉整網容量的 93.75%——這就是迴邊把放大係數藏起來的樣子。

## Jackson 定理：敘述與「彷彿獨立」的精確含義

> **Jackson 定理**（Jackson 1957；工程版敘述）：開放 Jackson 網路（上述五條件）中，設 traffic equations 的解為 λ₁, …, λ_k。若 ρᵢ = λᵢ/μᵢ < 1 對每一站成立，則穩態存在，且聯合平穩分布為
>
> π(n₁, n₂, …, n_k) = Πᵢ πᵢ(nᵢ)，其中 πᵢ(nᵢ) = (1−ρᵢ)·ρᵢ^nᵢ。

這個形狀叫**乘積形式（product form）**。常見的口訣是「每站彷彿獨立的 M/M/1（參數 λᵢ）」——這句話極容易被讀過頭，先把它精確說了什麼列出來：

1. **邊際分布**：站 i 的佇列長度分布恰是 M/M/1(λᵢ, μᵢ) 的幾何分布 (1−ρᵢ)ρᵢⁿ——即使餵它的流根本不是 Poisson。
2. **快照獨立**：穩態下任取一個固定時刻 t，N₁(t), …, N_k(t) 互相獨立。例：基準裡 P(兩站同時有工作) = ρ₁·ρ₂ = 0.75 × 0.9375 ≈ 0.703，真的可以直接相乘。
3. **計算配方**：任何「穩態快照」量——E[Nᵢ]、E[N]、P(N₁ ≥ a, N₂ ≥ b)——都可以假裝每站是獨立 M/M/1 來算，答案精確、不是近似。

再把它**沒**說的三件事列出來，每一件都有人栽過：

1. **沒有說站 i 的輸入是 Poisson(λᵢ)**。有 feedback 時它明確不是（下下節正面對決）。λᵢ 的身分只是平均流量（吞吐），不是某個 Poisson 過程的速率。
2. **沒有說各站作為「過程」獨立**。N₁(t) 與 N₂(t+s) 可以高度相關——burst 會順流而下。獨立的是同一時刻的快照。
3. **沒有說逗留時間分布也照 M/M/1**。每次過站的**平均**逗留 E[Tᵢ] = 1/(μᵢ−λᵢ) 是精確的（對站 i 用 Little：E[Nᵢ]/λᵢ，不依賴分布假設）；但**分布**層級的聲明——每次過站 ~ Exp(μᵢ−λᵢ)、整條路徑是獨立指數和——在有 feedback 或可超車的網路一般不成立（tandem 那種 overtake-free 特例見 ch11；一般網路的逗留時間分布是出名的難題，本書不展開）。

「彷彿」二字的正確讀法：它是一張**計算執照**，不是機制描述。把每站單獨抽出來、假裝餵它一條 Poisson(λᵢ)、算出 M/M/1 平穩分布、再乘起來——這套流程的每一步單獨看都不合法（輸入不是 Poisson！），但最終的聯合分布正確。數學偶爾會發這種「過程錯、答案對」的紅利，收下之前你得先驗證它——下一節動手。

## 工程師的驗證：兩站 feedback 網路的 global balance

對基準拓撲（外部流進 node1、node1→node2、node2 以 p 回流）驗證乘積形式。狀態是 (n₁, n₂)，CTMC 的全部轉移（ch07 的競賽引理組裝；回流取瞬時，與 ch11 同一個理想化）：

| 事件 | 狀態變化 | 速率 |
|---|---|---|
| 外部到達 node1 | (n₁, n₂) → (n₁+1, n₂) | λ₀ |
| node1 完成、送 node2（n₁ ≥ 1） | (n₁, n₂) → (n₁−1, n₂+1) | μ₁ |
| node2 完成、離開（n₂ ≥ 1） | (n₁, n₂) → (n₁, n₂−1) | μ₂(1−p) |
| node2 完成、回流 node1（n₂ ≥ 1） | (n₁, n₂) → (n₁+1, n₂−1) | μ₂·p |

先試 ch11 的捷徑：detailed balance。看邊 (n₁, n₂) → (n₁−1, n₂+1)（速率 μ₁），反向的轉移是 node2 回流 (n₁−1, n₂+1) → (n₁, n₂)（速率 μ₂p）。detailed balance 要求 π(n₁,n₂)·μ₁ = π(n₁−1,n₂+1)·μ₂p；代入幾何猜測，右邊是 π(n₁,n₂)·(ρ₂/ρ₁)·μ₂p = π(n₁,n₂)·p·μ₁——要求 μ₁ = p·μ₁，p < 1 時不可能。**這條鏈不可逆**：工作沿路徑流動本身就是轉移圖上的淨環流（p = 0 的 tandem 也一樣不可逆——那條邊的反向速率乾脆是 0）。ch11 的猜驗捷徑與 Kolmogorov 判準在網路裡全數陣亡，乘積形式如果成立，不是可逆性的禮物，只能靠 global balance 硬驗。

那就硬驗（完整推導——對此拓撲；猜 π(n₁,n₂) = C·ρ₁^n₁·ρ₂^n₂）。內部狀態 n₁ ≥ 1、n₂ ≥ 1：

```text
流出：π(n₁,n₂)·(λ₀ + μ₁ + μ₂)                    ← 三種事件都可能發生

流入（哪些狀態一步跳得進來）：
  π(n₁−1, n₂)·λ₀                                 ← 外部到達
  π(n₁+1, n₂−1)·μ₁                               ← node1 完成、送 node2
  π(n₁, n₂+1)·μ₂(1−p)                            ← node2 完成、離開
  π(n₁−1, n₂+1)·μ₂·p                             ← node2 完成、回流

兩邊同除 π(n₁,n₂)。π 是幾何形，鄰居的比值只剩 ρ 的冪次：
  λ₀ + μ₁ + μ₂  ?=  λ₀/ρ₁ + μ₁·(ρ₁/ρ₂) + μ₂(1−p)·ρ₂ + μ₂p·(ρ₂/ρ₁)
```

現在代入 traffic equations 的解——λ₁ = λ₂ = Λ = λ₀/(1−p)，ρ₁ = Λ/μ₁、ρ₂ = Λ/μ₂：

```text
  λ₀/ρ₁        = λ₀·μ₁/Λ          = (1−p)·μ₁     ← λ₀ = (1−p)·Λ
  μ₁·(ρ₁/ρ₂)   = μ₁·(Λ/μ₁)·(μ₂/Λ) = μ₂
  μ₂(1−p)·ρ₂   = (1−p)·Λ          = λ₀
  μ₂p·(ρ₂/ρ₁)  = μ₂·p·(Λ/μ₂)·(μ₁/Λ) = p·μ₁

右邊加總 = (1−p)μ₁ + μ₂ + λ₀ + p·μ₁ = λ₀ + μ₁ + μ₂  ✓
```

值得瞪著看的是配對方式：μ₁ 的流出由**兩個不同來源**補回——外部到達那項補 (1−p)μ₁、回流那項補 pμ₁；而 μ₂ 的流出由 node1 完成那項整筆補上。帳能軋平，恰恰因為 ρᵢ 取的是 traffic equations 的解——這族方程（內部加邊界）同時成立這件事，正是把流量守恆逼出來的力學。角落狀態 (0,0) 也順手驗掉：流出 π(0,0)·λ₀，流入只有 π(0,1)·μ₂(1−p) = π(0,0)·ρ₂·μ₂(1−p) = π(0,0)·λ₀ ✓。兩條邊界線（n₁ = 0 與 n₂ = 0）留作紙上推演第 2 題——轉移清單在邊界會缺項，自己列一次才會記住。

一般形（k 站、任意次隨機 routing、各站狀態相依速率）同一招但記號繁重，本書明寫不證：Jackson 1957 原文只有四頁，可直接讀；教科書有一條更聰明的路——站層級的 local balance（「站 i 因服務完成流出的機率流 = 因到達流入的機率流」），見 Harchol-Balter 第 17 章。另一個「為什麼」藏在 ch11 的工具裡：把 Jackson 網路時間反轉，得到的是**另一個 Jackson 網路**（路由反向、流量不變）——現代證明就從這個觀察出發（敘述層級，Kelly 第 3 章）。

## 驚奇正面對決：Poisson 已死，乘積形式還活著

現在把 ch11 賒的帳結清。基準網路裡，node1 的實際輸入是「外部 Poisson(6) ＋ node2 的回流」的合流。ch11 已證：這條合流**不是 Poisson**——回流緊跟在 node2 的服務完成之後，跟系統狀態強相關，lack of anticipation 被正面擊穿，邊際間隔是兩個指數的混合（Burke 1976）。一般的判準也有了：

> **定理（Melamed 1979，敘述層級）**：開放 Jackson 網路穩態下，一條邊上的流是 Poisson 過程，若且唯若沒有任何工作可能兩度經過這條邊（這條邊不在任何迴路上）。

對照基準：node1→node2 這條邊與回流邊都在迴路「node1→node2→node1」上，所以**兩條內部流都不是 Poisson**；唯一的例外是離開邊（0.8 那條）——它不在任何迴路上，仍是如假包換的 Poisson(6)，與 ch11 結尾「feedback 殺死內部流、放過對外輸出」完全一致。

於是悖論擺上桌：站的輸入不是 Poisson，憑什麼站的邊際分布還是 M/M/1 的幾何分布、聯合分布還能相乘？注意定理從頭到尾**沒有宣稱**輸入是 Poisson——上一節的驗證裡，「輸入像什麼過程」根本沒出場，出場的只有轉移速率與流量守恆。global balance 是一張**快照**的收支表：它約束的是「穩態下系統停在每個狀態的機率」，不是「系統怎麼在狀態間遊走的軌跡」。Poisson 性是軌跡的性質，它死了；乘積形式是快照的性質，它活著。一句話收束，請把它帶去 ch17 與 ch18：

**獨立性是穩態快照的性質，不是過程的性質。**

具體感受一下這句話的兩面。快照面：穩態下任一時刻，P(N₁ ≥ a, N₂ ≥ b) = P(N₁ ≥ a)·P(N₂ ≥ b)，精確相乘，連 0.703 那種「兩站同時忙」的數字都合法。過程面：node1 此刻排了 20 個，這批工作半秒後會湧進 node2——N₁(t) 與 N₂(t + 0.5) 相關得很，任何「尖峰沿管線傳播」的問題都不在乘積形式的管轄範圍（要嘛上 ch19 的模擬，要嘛閉嘴）。同一時刻恰好不相關，是 global balance 軋出來的精密抵銷——跟 ch11「離開歷史對現在狀態資訊量為零」是同一族的驚奇。我不會假裝這在直覺上理所當然；它不是，它是驗算出來的事實。化解這層驚奇的一般理論（什麼條件下站的輸入輸出與狀態能這樣解耦）叫 quasi-reversibility，本書不展開，指路 Kelly。

## 基準範例全解：retry 的真實價格

把全章工具串成一次完整的容量分析。系統：λ₀ = 6/s → node1（μ₁ = 10）→ node2（μ₂ = 8），node2 以 p = 0.2 回流 node1。

**第一步，traffic equations**：λ₁ = λ₂ = 6/(1−0.2) = 7.5/s（前面已解）。

**第二步，穩定性檢查**：ρ₁ = 7.5/10 = 0.75 < 1 ✓、ρ₂ = 7.5/8 = 0.9375 < 1 ✓。兩站都活著——但 node2 已經站在 hockey stick 的陡段（ch01/ch08 的使用率階梯：0.9375 已過 0.9 那級）。

**第三步，乘積形式與全部指標**：

```text
π(n₁,n₂) = (0.25·0.75^n₁)·(0.0625·0.9375^n₂)
  例：π(0,0) = 0.25·0.0625 ≈ 0.0156      ← 兩站全空的時間只有 1.56%

E[N₁] = ρ₁/(1−ρ₁) = 0.75/0.25   = 3
E[N₂] = ρ₂/(1−ρ₂) = 0.9375/0.0625 = 15
E[N]  = 3 + 15 = 18

E[T]  = E[N]/λ₀ = 18/6 = 3.0 s            ← Little 套整個網路
```

Little 這一步的邊界紀律值得放大：λ 用的是**外部**到達率 λ₀ = 6，不是 Σλᵢ = 15。「系統」這個邊界上，工作以 6/s 進、6/s 出；retry 在邊界內打轉，不跨邊界。複核走另一條路：每次過站的平均逗留（對單站用 Little，精確）E[T₁] = 1/(10−7.5) = 0.4 s、E[T₂] = 1/(8−7.5) = 2 s，每個工作平均各訪 V = 1.25 次，E[T] = 1.25·(0.4 + 2) = 3.0 s ✓。

**第四步，讀帳單**。對照 ch11 的無 retry 串聯（同樣 λ₀ = 6、同樣兩台機器）：E[T] = 0.75 s。加一條 20% 的回邊，延遲變成 **4 倍**。把帳拆開：如果 retry 只是「多走一輪」（每站逗留不變、訪次 ×1.25），帳單應該是 1.25 × 0.75 ≈ 0.94 s；剩下的 2 秒多全是因為回流把 ρ₂ 從 0.75 推到 0.9375——把瓶頸推上了曲線陡段。**retry 的主要價格不是重做的工，是被推高的使用率**。瓶頸識別同一口氣完成：D₂ = 0.15625 s > D₁ = 0.125 s，node2 是瓶頸，整網容量 λ₀ < 1/D₂ = 6.4/s，現在的 6/s 是容量的 93.75%。

**第五步，把 p 推上去看懸崖**。穩定性要求 λ₀/(1−p) < μ₂ = 8，即 p < 1 − 6/8 = **0.25**。臨界值離現在只有 5 個百分點。逐點算（E[N₁] = 6/(10(1−p)−6)、E[N₂] = 6/(8(1−p)−6)）：

| p | λᵢ = 6/(1−p) | ρ₂ | E[N₁] | E[N₂] | E[T] |
|---|---|---|---|---|---|
| 0 | 6.00 | 0.75 | 1.5 | 3 | 0.75 s |
| 0.1 | 6.67 | 0.833 | 2.0 | 5 | 1.17 s |
| 0.2 | 7.50 | 0.938 | 3.0 | 15 | 3.00 s |
| 0.22 | 7.69 | 0.962 | 3.33 | 25 | 4.72 s |
| 0.24 | 7.89 | 0.987 | 3.75 | 75 | 13.1 s |
| 0.25 | 8.00 | 1 | — | 發散 | ∞（臨界） |
| 0.3 | （8.57） | 1.07 | — | — | 不穩定 |

p = 0.3 重解一次給你看死法：λ₁ = 6 + 0.3λ₁ ⇒ λ₁ = 6/0.7 ≈ 8.57，ρ₂ ≈ 8.57/8 ≈ 1.07 > 1——traffic equations 還有解，但解違反穩定條件，乘積形式作廢，E[T] 不存在。**retry 多 10 個百分點，系統從穩態到爆炸。**爆炸長什麼樣（工程師的嚴謹）：node2 飽和後吞吐封頂在 μ₂ = 8，實際回流是 0.3 × 8 = 2.4/s，node1 實際輸入 8.4/s（它自己還活著，8.4 < 10），但 node2 每秒進 8.4、出 8——積壓以 0.4 件/s ≈ 每小時 1,440 件的速度線性成長，沒有任何穩態值可言。注意這裡 traffic equations 的不動點已經不再描述實際流量（它假設每站吞吐＝到達率，飽和站做不到）——不穩定時公式不是「不準」，是換了物理。

最後一句誠實聲明：本章的 p 是外生常數。現實裡 p 會隨負載漂移——延遲上升、timeout 增多、retry 變密、負載更高——那個會把暫時尖峰鎖死成永久壅塞的自我增強動力學（retry storm、metastable failure），是 ch20 的主題；本章給你的是靜態地圖上的懸崖位置：p* = 0.25。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦與察覺 |
|---|---|---|
| Little 用錯邊界 | 拿 E[N]/Σλᵢ = 18/15 = 1.2 s 當回應時間——這是「每次過站」的平均逗留，不是工作的端到端時間 3.0 s，差 2.5 倍 | 先畫邊界再套 Little：跨邊界的速率才是 λ。端到端用 λ₀，單站用 λᵢ，混用必錯（ch02 的邊界一致性） |
| 各站自查 ρ 就宣告穩定 | 用外部速率算：「6 < 8，node2 健康」——漏掉回流放大，真實 λ₂ = 7.5；p 再漂 5 個百分點就翻車 | 永遠先解 traffic equations 再算 ρᵢ。監控徵兆：站內 RPS 顯著高於外部請求率，差距就是迴邊流量 |
| 把「彷彿 M/M/1」升級成分布聲明 | 拿 T ~ Exp(μᵢ−λᵢ) 算單站 P99、再把各站 P99 疊成網路 P99——feedback 下逗留分布不是指數、各次過站也不獨立 | 乘積形式只擔保快照量與（經 Little 的）平均值。tail 層級的問題：tandem 特例見 ch11，一般網路交給模擬（ch19） |
| 條件破了照套 | join-shortest-queue（路由看狀態）、有限 buffer／blocking、fork-join、priority、非指數服務——任何一條都讓乘積形式失效，誤差方向還不一定 | 對照五條件逐一打勾。打不了勾的，ch18 給你邊界地圖與近似選項；C²ₛ ≠ 1 的單站帳先回 ch12/ch14 |
| 把 p 當常數 | 量測得到「retry 率 20%」就寫死進模型——尖峰時 timeout 變多，p 自己會往懸崖走 | 把 p* 當預算上限管理：算出臨界值（基準：0.25），告警閾值設在遠低於它的位置；動態見 ch20 |
| 快照獨立當過程獨立 | 用 π 相乘回答「node1 的尖峰半秒後會不會打爆 node2」——時間相關的問題，乘積形式管不著 | 凡問題裡有「之後」「傳播」「連環」字眼，改用模擬（ch19）。同一時刻的聯合機率才可相乘 |

## 紙上推演

**1. [15 分鐘 ★★] 另一組 routing。**三站：gateway A（μ_A = 10/s）、worker B（μ_B = 4/s）、audit C（μ_C = 5/s）。外部 Poisson(5/s) 全部進 A；A 完成後一半去 B、一半去 C；B 完成後以 0.4 退回 A 重做（NACK）、0.6 離開；C 完成後直接離開。(a) 解 traffic equations；(b) 算各站 ρᵢ、E[N]、端到端 E[T]；(c) 外部流量漸增，誰先爆、整網容量上限是多少？

**2. [20 分鐘 ★★★] 把驗證補完。**本章只驗了內部狀態與 (0,0)。對基準拓撲補驗兩條邊界線：(a) 狀態 (0, n₂)、n₂ ≥ 1；(b) 狀態 (n₁, 0)、n₁ ≥ 1。各自先列「流出哪些、誰能流入」（小心：邊界上轉移清單會缺項），再代幾何猜測與 traffic equations 收帳。

**3. [20 分鐘 ★★] 加一台 vs 砍 retry。**基準系統只有一筆預算，兩個方案：(A) node2 加一台同款機器（兩台 μ = 8 共用佇列，成為 M/M/2 站——Jackson 1963 的狀態相依推廣保證該站邊際是 M/M/2 的平穩分布，可直接用 ch09 工具）；(B) 工程修復把 retry 率從 0.2 砍到 0.1。各算新的 E[T] 與整網容量上限，給出裁決與裁決的但書。

### 推演解答

**1.** (a) λ_A = 5 + 0.4·λ_B、λ_B = 0.5·λ_A、λ_C = 0.5·λ_A ⇒ λ_A = 5 + 0.2·λ_A ⇒ **λ_A = 6.25、λ_B = λ_C = 3.125**。(b) ρ_A = 0.625、ρ_B = 0.78125、ρ_C = 0.625；E[N_A] = E[N_C] = 0.625/0.375 ≈ 1.67、E[N_B] = 0.78125/0.21875 ≈ 3.57；E[N] ≈ 6.90，E[T] = 6.90/5 ≈ **1.38 s**（注意除以外部的 5，不是 Σλᵢ = 12.5）。(c) 一般化：λ_A = 1.25·λ₀、λ_B = λ_C = 0.625·λ₀。三條約束：1.25λ₀ < 10 ⇒ λ₀ < 8；0.625λ₀ < 4 ⇒ λ₀ < 6.4；0.625λ₀ < 5 ⇒ λ₀ < 8。**B 先爆，容量 λ₀ < 6.4/s**——與 ch02 的 X ≤ 1/D_max 對帳：D_B = 0.625/4 = 0.15625 s，1/D_B = 6.4 ✓。常見錯路：忘了 NACK 迴圈也放大 C 的流量（C 吃的是 A 的輸出之半，A 被回流撐大）。

**2.** (a) 狀態 (0, n₂)：node1 空，流出只有外部到達與 node2 服務，速率 λ₀ + μ₂（注意 μ₂ 含離開與回流兩個去向，但都算流出）。流入：從 (1, n₂−1) 經 node1 完成（速率 μ₁）、從 (0, n₂+1) 經 node2 離開（速率 μ₂(1−p)）；回流與外部到達都需要 n₁ = −1 的來源，不存在。收帳（同除 π(0,n₂)）：

```text
λ₀ + μ₂ ?= μ₁·(ρ₁/ρ₂) + μ₂(1−p)·ρ₂ = μ₂ + λ₀  ✓   ← 用本文同一組恆等式
```

(b) 狀態 (n₁, 0)：node2 空，流出 λ₀ + μ₁。流入：從 (n₁−1, 0) 經外部到達（λ₀）、從 (n₁, 1) 經 node2 離開（μ₂(1−p)）、從 (n₁−1, 1) 經回流（μ₂p）；node1 完成需要來源 (n₁+1, −1)，不存在。收帳：

```text
λ₀ + μ₁ ?= λ₀/ρ₁ + μ₂(1−p)·ρ₂ + μ₂p·(ρ₂/ρ₁)
         = (1−p)μ₁ + λ₀ + p·μ₁ = μ₁ + λ₀  ✓
```

常見錯路有二：忘記回流轉移**同時動兩個座標**（n₁+1、n₂−1），把它誤列成只動 n₂；以及在 (0, n₂) 的流出裡漏掉回流那份 μ₂p（node1 空不影響 node2 把工作送回來）。

**3.** 方案 A：node2 成為 M/M/2 站，λ₂ = 7.5、每台 μ = 8，a = λ/μ = 0.9375、ρ = 0.469。Erlang-B 遞迴（ch09）：B(1) = 0.9375/1.9375 ≈ 0.484、B(2) = 0.9375·0.484/(2 + 0.9375·0.484) ≈ 0.185；換 Erlang-C：C = 2·0.185/(2 − 0.9375·0.815) ≈ 0.299；E[N_Q] = C·ρ/(1−ρ) ≈ 0.264，E[N₂] = 0.264 + 0.9375 ≈ 1.20。E[N] = 3 + 1.20 = 4.20 ⇒ **E[T] ≈ 0.700 s**；瓶頸搬家到 node1（ρ₁ = 0.75 > 0.469），容量上限變成 λ₀/(1−p) < 10 ⇒ λ₀ < 8。方案 B：p = 0.1 ⇒ λᵢ = 6.67，E[N] = 2 + 5 = 7 ⇒ **E[T] = 7/6 ≈ 1.17 s**；容量 λ₀ < 8 × 0.9 = 7.2。裁決：純看 E[T] 與容量，A 全贏（0.700 s vs 1.17 s；8 vs 7.2）。但書：A 花的是機器錢、B 花的是工程時間，匯率表是上面的數字；更重要的是 B 直接攻擊**會漂移的那個參數**——懸崖位置 p* 從「距離 0.05」變成「距離 0.15」，對 ch20 的 retry storm 情境，這份餘裕可能比平均延遲值錢。兩個方案不互斥，數學的工作是讓你知道各買到什麼。
