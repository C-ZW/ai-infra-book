# ch06 — Markov chain：狀態吃掉歷史

> **本章解決什麼問題**：ch04 給了你無記憶性、ch05 給了你到達流，但「系統」還沒登場。本章建立全書的引擎：把系統寫成「狀態＋轉移」，並回答三個問題——什麼樣的模型只看現在就夠（Markov 性質）、長期各狀態佔多少時間（平穩分布）、憑什麼一條軌跡的時間平均能代表整個分布（遍歷定理）。ch07 把這顆引擎接上連續時間與佇列，ch08 用它生出第一個完整模型，ch19 的模擬方法學靠本章發的執照開工。

## 從你已知的出發

你寫過或設定過 circuit breaker：closed 正常放行、連續失敗到門檻就跳到 open 全部擋掉、冷卻時間到了進 half-open 放探測請求，成功回 closed、失敗退回 open。三個狀態、幾條箭頭，這張圖你閉著眼睛都畫得出來——在 TLA+ 那本書裡你手畫過一堆同款的狀態轉移圖。

但有一個問題那張圖答不了：**長期下來，這個 breaker 有多少比例的時間處於斷路？**冷卻時間從 30 秒調成 60 秒，這個比例會怎麼動？狀態圖只告訴你「哪裡可以去」，不告訴你「各地各待多久」。要回答，箭頭上得標機率，然後解一組方程。這就是 Markov chain：你已經會畫的圖，加上你大學看過的線性方程組，合出一個能回答「長期比例」的機器。

另一條線索接 ch04：無記憶性是「時間軸上的失憶」——等了多久不影響還要等多久；Markov 性質是「軌跡上的失憶」——怎麼走到這個狀態，不影響接下來往哪走。同一個哲學，從一個隨機變數升級到一整條過程。而你的監控系統其實天天在做本章的事：面板上「服務處於 degraded 的時間佔比」就是在估某個狀態的長期機率，只是你從來沒問過這個估計什麼時候合法。本章結束時，這個問題有名字（遍歷定理）、有條件（不可約）、也有會翻車的情境（時間齊次假設破掉）。

## Markov 性質：狀態把歷史吃掉

一個離散時間隨機過程 X₀, X₁, X₂, … 取值於狀態空間 S（本章 S 有限）。它是 Markov chain，若對所有 n 與所有狀態：

```text
P(Xₙ₊₁ = j | Xₙ = i, Xₙ₋₁ = iₙ₋₁, …, X₀ = i₀) = P(Xₙ₊₁ = j | Xₙ = i)
```

白話：給定現在，過去對未來沒有額外資訊。本書再加一條**時間齊次**假設：這個機率不隨 n 改變，記作 P(i→j)。

我喜歡的讀法是「狀態吃掉歷史」：狀態必須**裝下未來需要知道的一切**。這句話把 Markov 性質從「世界的性質」翻轉成「你的設計責任」——Markov 性質不是某個系統天生有或沒有的，而是**你選的狀態變數夠不夠肥**。

拿 breaker 當例子。如果跳閘規則是「連續 5 次失敗才開路」，那麼光記 {closed, open, half-open} 三個狀態**不是** Markov chain：處在 closed 時，下一步會不會跳閘取決於「目前已連續失敗幾次」——這是歷史，三狀態裝不下它。修法不是放棄，而是**擴狀態**：把 closed 拆成 closed₀、closed₁、…、closed₄（下標＝目前連續失敗數），歷史就被吃進狀態裡，Markov 性質恢復。建模的真功夫全在這一步：選一組「最小但裝得下未來」的狀態。佇列理論裡最重要的一次選擇是「狀態＝系統內人數」——它什麼時候夠、什麼時候不夠，是 ch08 與 ch12 的主旋律。

## 轉移矩陣與 Chapman–Kolmogorov

把所有 P(i→j) 排成矩陣 P，第 i 列是「從 i 出發、下一步去哪」的分布，所以每列加總為 1。如果 μₙ 是時刻 n 的分布（橫向量），那 μₙ₊₁ = μₙP——機率向量擺左邊，因為矩陣的「列」才是條件分布。

n 步轉移機率記 P^(n)(i→j) = P(Xₙ = j | X₀ = i)。它滿足 **Chapman–Kolmogorov 方程**（完整推導）：

```text
P^(m+n)(i→j)
  = Σₖ P(Xₘ = k | X₀ = i) · P(Xₘ₊ₙ = j | Xₘ = k, X₀ = i)   ← 全機率定理：以第 m 步的位置分案
  = Σₖ P(Xₘ = k | X₀ = i) · P(Xₘ₊ₙ = j | Xₘ = k)            ← Markov 性質：給定 Xₘ，X₀ 是廢話
  = Σₖ P^(m)(i→k) · P^(n)(k→j)                              ← 時間齊次：只看步數差
```

這正是矩陣乘法的定義，所以 P^(n) 就是矩陣冪 Pⁿ。先說清楚：這個事實在本章只當**記帳工具**——我們不會去算任何矩陣的冪，更不會碰特徵值。C-K 的實戰價值是那句口訣：「算多步轉移，就對中繼點分案」，這招在 ch07 推 CTMC、ch11 玩時間反轉時會一再出現。

實務上 P 從哪來？兩條路。一條是**從機制寫下來**：跳閘門檻、探測成功率這些你設定或量測得到的量，直接化成轉移機率——下一節的 worked example 走這條。另一條是**從 log 數出來**：把軌跡切成一步步，P(i→j) 用「觀測到 i→j 的次數 ÷ 離開 i 的總次數」估。第二條路有個安靜的坑：樣本少的狀態（一個月才進兩次的 open）估出來的那一列噪音極大，而 π 偏偏對稀有狀態的轉移機率很敏感——這是工程裡「模型看起來科學、數字其實是擲筊」的常見來源。

## 狀態分類：chain 的病理學

不是每個 chain 都有「唯一且可信的長期比例」。病有三種，每種都有工程長相。

先要兩個詞：j 從 i **可達**，若存在 n 使 P^(n)(i→j) > 0；i、j **互通**，若彼此可達。互通把狀態空間切成幾個等價類。整條 chain 只有一個類，叫**不可約（irreducible）**——任兩狀態互相走得到。

| 病態 | 一句話定義 | 工程長相 | 它弄壞什麼 |
|---|---|---|---|
| 可約（reducible） | 存在回不去的狀態或吸收類 | 部署狀態機裡 bricked 的終態；毒訊息進了 DLQ 不再回流；洩漏掉的 worker 只能重啟 | 平穩分布不唯一，長期行為依起點而異；「算出一個 π」可能只是其中一個 |
| 週期（periodic） | 回到自己只可能在某個 d > 1 的倍數步 | 兩個服務嚴格輪替互呼叫；ping-pong 同步協定 | 分布 P(Xₙ = j) 永遠振盪不收斂（但時間平均仍然收斂，見下文） |
| 暫態（transient） | 離開後以機率 1 不再回來 | 有限空間：暖機／初始化狀態，過了就不回頭；無限空間：過載佇列的長度一路向右漂 | 長期佔比為 0；無限狀態時可能連平穩分布都不存在 |

正式定義：狀態 i 的週期是 {n : P^(n)(i→i) > 0} 的最大公因數 d；d = 1 叫**非週期（aperiodic）**。有 self-loop（P(i→i) > 0）的狀態自動非週期，這是工程模型裡週期病罕見的原因——但「罕見」不等於「不會」，紙上推演第 4 題就是一個。狀態 i 是**常返（recurrent）**，若從 i 出發以機率 1 回到 i；否則**暫態（transient）**。有限不可約 chain 的所有狀態都常返——跑不掉、又到處去得了，遲早回來（這句是工程師的嚴謹：有限狀態下總有狀態被造訪無限多次，不可約性再把這個性質傳染給全部狀態）。

無限狀態空間多一層陰影：常返還分「平均回返時間有限」（正常返）與「會回來、但平均要等無限久」（零常返）。直覺版、本書不證：佇列在 ρ = 1 的臨界點上得的正是零常返這種病——必定會再排空，但平均排空時間無限（見 ch08）。本章接下來一律假設**有限**狀態空間，無限的留給 ch07 用 ergodicity 條件處理。

## 平穩分布：解 π = πP 只需要代入消去

分布 π 是**平穩分布（stationary distribution）**，若

```text
π = πP，   Σⱼ πⱼ = 1，   πⱼ ≥ 0
```

逐分量讀：π_j = Σᵢ πᵢ·P(i→j)，每個狀態「流進來的機率」恰好補滿「此刻在這裡的機率」。π 有三種讀法，條件一個比一個強，後面兩節逐一兌現：

1. **不動點**：X₀ ~ π 則永遠 Xₙ ~ π——系統「開機就在穩態」。這是定義，無條件。
2. **極限分布**：不管從哪起步，P(Xₙ = j) → π_j。需要不可約＋非週期。
3. **時間比例**：一條軌跡待在 j 的時間佔比 → π_j。需要不可約（不需非週期）。

先安撫一件事，因為它擋住太多人：**解 π = πP 不需要任何矩陣代數，更不需要特徵值。**它就是 |S| 條一次方程（其中必有一條是多餘的——每列加總為 1 保證了這點）加上正規化條件 Σπ = 1。操作三步：挑好解的方程做代入消去、把每個 πⱼ 表成某一個變數的倍數、最後用 Σπ = 1 定刻度。你國高中就會了，生鏽的只是膽量。

### Worked example：circuit breaker 的長期斷路比例

把開場那個 breaker 變成具體模型。取 tick = 1 秒，三個狀態 C（closed）、O（open）、H（half-open）：

- C：下游每秒有 5% 的機率讓失敗計數衝過門檻而跳閘（先把計數器粗化成一個機率，這筆帳「陷阱與防禦」第 1 條會回頭算）。
- O：冷卻計時器幾何化——每秒 50% 機率到期進 H，平均冷卻 2 秒（真實系統是固定時長，這是近似，同樣記在陷阱第 1 條）。
- H：送一個探測請求，60% 成功回 C，40% 失敗退回 O；一個 tick 內見分曉。

```text
              0.05
   ┌─────┐ ─────────► ┌─────┐
   │  C  │            │  O  │ ◄────┐
   └─────┘            └─────┘      │
      ▲                  │         │
      │ 0.6              │ 0.5     │ 0.4
      │                  ▼         │
      │               ┌─────┐      │
      └────────────── │  H  │ ─────┘
                      └─────┘

   (self-loop: C→C 0.95, O→O 0.5)
```

| 從＼到 | C | O | H |
|---|---|---|---|
| C | 0.95 | 0.05 | 0 |
| O | 0 | 0.50 | 0.50 |
| H | 0.60 | 0.40 | 0 |

寫平穩方程（逐欄讀「流入」）：

```text
π_C = 0.95·π_C + 0.60·π_H        … (1)
π_O = 0.05·π_C + 0.50·π_O + 0.40·π_H   … (2)
π_H = 0.50·π_O                   … (3)
```

代入消去，全程不准跳步：

```text
(1) ⇒ 0.05·π_C = 0.60·π_H  ⇒  π_C = 12·π_H
(3) ⇒ π_O = 2·π_H
正規化：π_C + π_O + π_H = 12·π_H + 2·π_H + π_H = 15·π_H = 1
⇒ π_H = 1/15 ≈ 0.0667，π_O = 2/15 ≈ 0.133，π_C = 12/15 = 0.800
```

(2) 是被丟掉的「多餘」方程，正好拿來當第一道複核。**自我複核**——把 π 代回三條方程：

```text
(1)：0.95·(12/15) + 0.60·(1/15) = (11.4 + 0.6)/15 = 12/15 = π_C ✓
(2)：0.05·(12/15) + 0.50·(2/15) + 0.40·(1/15) = (0.6 + 1 + 0.4)/15 = 2/15 = π_O ✓
(3)：0.50·(2/15) = 1/15 = π_H ✓
```

**解讀**：長期而言這個 breaker 有 π_O ≈ 13.3% 的時間處於斷路、6.67% 在試探、80.0% 正常放行。順手再榨一個數字：穩態下跳閘的頻率是 π_C·0.05 = 0.0400 次/秒——平均每 25 秒跳閘一次。另一個有體感的讀法：時間佔比 = 訪問頻率 × 每次平均停留。幾何停留的平均是 1/(1−P(留在原地))：C 每次待 20 tick、O 待 2 tick、H 待 1 tick——C 佔比高是因為待得久，不是因為去得頻繁。

**調一個參數**：把冷卻平均從 2 秒拉到 5 秒（O→O 改 0.8、O→H 改 0.2），其他不動。方程 (1) 沒變所以 π_C = 12·π_H 照舊；新 (3) 是 π_H = 0.2·π_O，即 π_O = 5·π_H；正規化 18·π_H = 1：

```text
π = (12/18, 5/18, 1/18) ≈ (0.667, 0.278, 0.0556)
複核 (2)：0.05·(12/18) + 0.80·(5/18) + 0.40·(1/18) = (0.6 + 4 + 0.4)/18 = 5/18 ✓
```

斷路比例從 13.3% 跳到 27.8%（×2.08），正常放行從 80.0% 掉到 66.7%。機制很清楚：每次進 O 待 2.5 倍久，而進 O 的頻率只略降（流入 O 的速率從 0.0667/tick 變 0.0556/tick；乘上平均停留 2 與 5，恰好回得出 0.133 與 0.278——又一道複核）。這是好是壞，數學不替你決定：如果下游真的病著（探測只有六成成功），多待在 O 正是 breaker 的本職——少放請求過去煩它；如果失敗只是瞬時抖動，這 27.8% 就是你白白吃掉的可用性。模型只負責把代價算清楚。

## 存在、唯一、收斂：什麼時候可以信 π

剛才那組數字值多少信任？答案是這條定理（**工程師的嚴謹：精確敘述＋直覺，完整證明指路延伸閱讀**）：

> **定理**：有限、不可約的 Markov chain 存在唯一平穩分布 π，且每個 πⱼ > 0。若再加上非週期，則對任何起點 i、任何狀態 j，P(Xₙ = j | X₀ = i) → πⱼ，而且收斂速度是幾何的（誤差以某個 rⁿ、r < 1 衰減）。

三個條件各買一樣東西：**有限**買存在（機率跑不掉）、**不可約**買唯一（只有一個迴路系統，沒有「看你出生在哪」的歧義）、**非週期**買收斂（分布不再振盪）。breaker chain 三項全過：互通（C→O→H→C 一圈）、C 有 self-loop 所以非週期，於是那個 13.3% 既唯一、又是任何起點的極限。

收斂為什麼成立？**耦合（coupling）直覺**（直覺版，本書不證）：開兩條獨立的 chain，一條從你指定的起點出發，一條直接從 π 出發（所以它每一步的分布都恰好是 π）。兩條各走各的，直到第一次落在同一個狀態——稱那一刻為 τ。從 τ 起讓兩條黏在一起走同一條路；黏合不改變任何一條的邊際分布，因為每一步依然照 P 轉移。於是 Xₙ 的分布與 π 的差距至多是 P(τ > n)：兩條還沒相遇，才有機會不同。剩下的工作是證明有限＋不可約＋非週期保證 P(τ > n) → 0，而非週期性正是在這一步出力——想像週期 2 的 ping-pong，兩條 chain 若起步異相，一條在偶數位、一條在奇數位，永遠擦肩而過。被我掃進地毯下的是「兩條獨立 chain 構成的乘積 chain 仍不可約」這個非平凡引理，完整版見 Levin–Peres–Wilmer 第 4–5 章。

順帶一句：線性代數路線（Perron–Frobenius 定理）也能到同一個地方，但本書一次都不會用到它。

## Balance equations：把 π 讀成機率流量守恆

π = πP 還有第三種身分，是全書接下來最常用的那種。從第 j 條方程出發（完整推導）：

```text
π_j = Σᵢ πᵢ·P(i→j)                                ← π = πP 的第 j 條
π_j · Σₖ P(j→k) = Σᵢ πᵢ·P(i→j)                    ← 左邊乘 Σₖ P(j→k) = 1（每列加總為 1）
π_j · Σ_{k≠j} P(j→k) = Σ_{i≠j} πᵢ·P(i→j)          ← 兩邊同減 π_j·P(j→j)
└── 流出 j 的機率流 ──┘  └── 流入 j 的機率流 ──┘
```

把 πᵢ·P(i→j) 想成穩態下「單位時間從 i 流向 j 的機率流量」，這條式子說：**穩態 = 每個狀態的流入等於流出**。這不是比喻，是 π = πP 的逐字翻譯。

守恆不只逐狀態成立，**對任何切割（cut）都成立**：取狀態集合 A，把每個 j ∈ A 的收支平衡加總，A 內部互流的項在兩邊同時出現、互相抵銷，剩下跨界的：

```text
Σ_{j∈A} Σ_{k∉A} π_j·P(j→k)  =  Σ_{i∉A} Σ_{j∈A} πᵢ·P(i→j)
└──── A 流出去的總流量 ────┘   └──── 流進 A 的總流量 ────┘
```

拿 breaker 驗收。切 {C} 對 {O, H}：跨界只有 C→O（流出）與 H→C（流入）：

```text
            π_C·0.05 = 0.0400
          ─────────────────────►
   ┌───────┐                     ┌──────────┐
   │  {C}  │                     │  {O, H}  │
   └───────┘                     └──────────┘
          ◄─────────────────────
            π_H·0.60 = 0.0400   ✓
```

一條 cut 直接給出 π_C = 12·π_H——跟代入消去得到的第一條等式一模一樣，但這次它有物理意義：跳閘的頻率必須等於修復的頻率，否則機率會在某一側堆積，就不是穩態。這個讀法在 ch07 會變成主力武器：生滅過程的狀態排成一條線，在 n 與 n+1 之間下一刀，每刀恰好給一條方程，πₙ 的通解就這樣一路切出來（見 ch07）。

## Ergodic 定理：時間平均＝期望

還欠一個兌現：監控面板上「breaker 處於 open 的時間佔比」是**一條軌跡的時間平均**，憑什麼等於 π_O 這個**跨平行宇宙的機率**？

> **遍歷定理（ergodic theorem）**（工程師的嚴謹：敘述）：有限、不可約的 Markov chain，對任何函數 f 與任何起點，(1/N)·Σₙ₌₁..N f(Xₙ) → Σⱼ πⱼ·f(j) 以機率 1 成立。取 f 為「在狀態 j」的 indicator，就得到：待在 j 的時間佔比 → πⱼ。

注意條件裡**沒有非週期**：ping-pong chain 的分布永遠振盪，但時間平均乖乖收斂到 (1/2, 1/2)。「分布收斂」與「時間平均收斂」是兩件事，後者便宜得多。

直覺版（本書不證，循環論證的嚴格版見 Harchol-Balter 第 9 章）：每次回到狀態 j，Markov 性質讓 chain 在機率意義上「重新開始」——歷史被吃掉了，從 j 再出發的未來與上一次從 j 出發的未來同分布且獨立。整條軌跡於是被切成一段段獨立同分布的「循環」，對循環用大數法則，時間平均就釘在期望上。

這條定理是兩張執照。**監控的執照**：你看一週的 open 佔比來估 π_O，合法——前提是這一週裡 P 沒變（陷阱第 5 條）。**模擬的執照**：跑一條夠長的軌跡、取時間平均，就能估任何 Σπⱼf(j)，不必跑一萬條平行軌跡（見 ch19）——但「夠長是多長」「誤差條怎麼畫」是 ch19 整章的主題，這裡先別樂觀。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦與察覺 |
|---|---|---|
| 畫得出狀態圖 ≠ Markov chain | 轉移由計數器或固定計時器驅動（連續 5 次失敗、冷卻固定 30 秒）時，停留時間不是幾何分布。幾何近似平均對了、分布形狀錯了：固定 30 秒冷卻絕不會出現 1 秒就回來的樣本，幾何模型卻說有 ≈3% 機率（tick=1s、平均 30）——依賴停留分布的結論（如逾時內恢復的機率）會整段作廢 | 把計數器與計時器折進狀態：closed₀…closed₄、open 剩 k tick。狀態空間變大，但 Markov 性質是擴狀態換來的，不是狀態圖送的 |
| 可約 chain 硬解 π | 有吸收類時平穩分布不唯一，你解出的「π」只是其中一個；模擬結果依起點漂移 | 先檢查互通性。發現吸收態，先問「這是 bug 還是 feature」：是終態就改問吸收機率與吸收時間（不同的方程組），不是就修模型 |
| 週期 chain 等分布收斂 | P(Xₙ = j) 永遠振盪；照固定相位取樣的指標會 alias 出鬼影 | 時間平均仍收斂到 π，照用；要分布收斂就加一點 self-loop（lazy 化），π 不變（推演第 4 題） |
| 拿 π 回答 transient 問題 | π 是長期比例，對「多久第一次斷路」「事故後多久回穩」一個字都沒說——那是首達時間（hitting time）與 mixing time 的領域 | 認清問題型態：長期比例找 π；首達時間另解一組線性方程（延伸閱讀 Norris、LPW），別硬掰 |
| 時間齊次是個假設 | 真實流量有日夜週期，P(i→j) 在一天內變來變去；用全天資料估一個 P 解出的 π 是「不存在的平均日」 | 分時段估 P、分段平穩；尖峰另建模。徵兆：不同時窗估出的 P 差很多（ch05 非齊次 Poisson 的同款病） |

## 紙上推演

**1. [15 分鐘 ★★] 服務健康鏈。**某服務每分鐘觀測一次，狀態 {H 健康, D 降級, R 重啟中}，轉移矩陣：H→H 0.9、H→D 0.1；D→H 0.3、D→D 0.5、D→R 0.2；R→H 1.0。求平穩分布，並把解代回全部方程複核。長期有多少比例時間不健康？

**2. [15 分鐘 ★★] 用 cut 解一條一維鏈。**worker 積壓量分四級 {0, 1, 2, 3}。每 tick：升一級機率 0.3（第 3 級不升）、降一級機率 0.5（第 0 級不降）、否則原地不動。用「相鄰兩級之間下一刀」的 cut 方程解 π，不要用代入消去硬碰整組方程。解完說一句：你看到了什麼結構？

**3. [10 分鐘 ★★] 誰沒有唯一平穩分布？**三條 chain：(a) 本章的 breaker chain；(b) 三狀態 {S, A, F}，S→A 0.7、S→F 0.3、A→A 1、F→F 1；(c) 兩狀態 ping-pong，A→B 機率 1、B→A 機率 1。哪一條沒有唯一平穩分布？為什麼？另外兩條各有什麼毛病或沒毛病？

**4. [10 分鐘 ★★★] 週期性陷阱。**對 (c) 的 ping-pong：從 A 出發，寫出 P(Xₙ = A) 的數列；它收斂嗎？時間平均收斂嗎？這跟收斂定理矛盾嗎？最後：把兩個狀態各加 0.1 的 self-loop（其餘機率等比例縮成 0.9），驗證 π 不變，並說明為什麼「lazy 化」永遠不動 π。

### 推演解答

**1.** 平穩方程（逐欄流入）：

```text
π_H = 0.9·π_H + 0.3·π_D + 1.0·π_R   … (1)
π_D = 0.1·π_H + 0.5·π_D             … (2)
π_R = 0.2·π_D                       … (3)
(2) ⇒ 0.5·π_D = 0.1·π_H ⇒ π_D = 0.2·π_H
(3) ⇒ π_R = 0.2·(0.2·π_H) = 0.04·π_H
正規化：π_H·(1 + 0.2 + 0.04) = 1 ⇒ π_H = 25/31 ≈ 0.806
π_D = 5/31 ≈ 0.161，π_R = 1/31 ≈ 0.0323
```

複核 (1)：0.9·(25/31) + 0.3·(5/31) + 1/31 = (22.5 + 1.5 + 1)/31 = 25/31 ✓；(2)：0.1·25/31 + 0.5·5/31 = 5/31 ✓；(3)：0.2·5/31 = 1/31 ✓。不健康比例 π_D + π_R = 6/31 ≈ 19.4%。常見錯路：把轉移矩陣「逐列」當方程抄（那是 P 的定義，不是 πP 的分量）；平穩方程要逐**欄**收集流入。

**2.** 在第 i 級與第 i+1 級之間下刀，跨界只有「升」與「降」兩種流：

```text
0.3·πᵢ = 0.5·πᵢ₊₁   ⇒   πᵢ₊₁ = 0.6·πᵢ（i = 0, 1, 2）
正規化：π₀·(1 + 0.6 + 0.36 + 0.216) = 2.176·π₀ = 1
⇒ π = (125/272, 75/272, 45/272, 27/272) ≈ (0.460, 0.276, 0.165, 0.0993)
```

抽查第 1 級的完整收支當複核：流出 = π₁·(0.3+0.5) = 0.8·(75/272) = 60/272；流入 = π₀·0.3 + π₂·0.5 = 37.5/272 + 22.5/272 = 60/272 ✓。看到的結構：一維鏈上每刀一條方程、解成**幾何遞推** πᵢ₊₁ = (升率/降率)·πᵢ——這就是 ch07 生滅過程通解的全部骨架，ch08 的 πₙ = (1−ρ)ρⁿ 是它的直系後代。

**3.** 沒有唯一平穩分布的是 **(b)**：A 與 F 是兩個吸收類，chain 可約。任何 (0, p, 1−p)（0 ≤ p ≤ 1）都滿足 π = πP——平穩分布有無限多個，「長期行為」取決於被哪個吸收類吃掉，正確的問題是吸收機率（此例從 S 出發 0.7 進 A）而不是 π。(a) 不可約＋非週期，唯一 π 且分布收斂，滿分。(c) 是陷阱選項：它**有**唯一平穩分布 π = (1/2, 1/2)——不可約就夠保唯一——它的毛病是週期 2 導致分布不收斂，那是第 4 題的事。把「沒有唯一 π」和「分布不收斂」混為一談，是本章最常見的概念事故。

**4.** 從 A 出發：P(Xₙ = A) = 1, 0, 1, 0, …（n 偶為 1、奇為 0），不收斂。時間平均：前 N 步在 A 的比例是 ⌈N/2⌉/N → 1/2 = π_A，收斂。與定理不矛盾：分布收斂那條的前提含**非週期**，ping-pong 的週期是 2，前提不成立、定理沒有承諾。lazy 化驗證：新矩陣 A→A 0.1、A→B 0.9（B 對稱），檢查 π = (1/2, 1/2)：0.5·0.1 + 0.5·0.9 = 0.5 ✓。一般論證：lazy 化把 P 換成 (I + P)/2 一類的混合，而 π 同時滿足 π = πI 與 π = πP，所以也滿足任何兩者的加權平均——self-loop 破壞週期、卻動不了 π。代價是收斂變慢一倍量級：用模擬的眼光看，這是用時間換對齊。
