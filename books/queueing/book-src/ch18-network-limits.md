# ch18 — 網路的邊界：product form 之外

> **本章解決什麼問題**：ch16 給了你 Jackson 的驚奇、ch17 給了封閉網路的 MVA，但兩者都站在同一排假設上。本章回答三個問題：乘積形式的國界究竟劃在哪裡（BCMP 定理——順便把 ch09 的 Erlang-B insensitive 與 ch13 的 PS insensitivity 兩張支票一次兌現）；你每天寫的系統有哪些特徵會打破它（priority、blocking、batching、retry 相關性，以及本章主角 fork-join——`Promise.all` 的數學身分）；打破之後還剩什麼（分解近似的思路，然後是模擬——把棒子交給 ch19）。這是 Part V 的收尾：精確解的地圖到此為止，框架從這裡開始。

## 從你已知的出發

打開你任何一個 service 的程式庫，搜尋 `Promise.all`。我猜命中數是兩位數起跳：拿一個 user id 同時打三個內部 API 再組裝回應；scatter-gather 打 8 個 shard 收齊再排序；結算 pipeline 對一批訂單並行發查。這個模式好寫、好讀、天天上線——而它在排隊理論裡的名字叫 **fork-join**，是公認最難精確分析的網路結構之一。每路服務時間是隨機的，`Promise.all` 等的是**最慢的那一路**；你每寫一次 fan-out，就簽了一張「以 max 計價」的帳單。這章會把這張帳單算清楚。

先把帳算在更大的框架裡。ch16 的 Jackson 定理掛著一排條件：外部到達 Poisson、每站指數服務、FCFS、機率式路由（獨立擲骰）、無限緩衝、單一 class。拿這張清單對照你真實的系統：health check 有插隊通道（priority，違反）；nginx 設了 `max_conns`、SQS 佇列有 visibility timeout（有限容量與 blocking，違反）；到處是 `Promise.all`（同步，違反）；retry 在 timeout 時觸發——恰恰是系統慢的時候（路由與壅塞相關，違反）。幾乎每一格都打叉。那麼 Part V 推到現在的精確解，對你的系統還剩多少效力？

誠實的答案分三層，正好是本章的三段路：第一層，精確解的國界其實比 Jackson 大——BCMP 定理把它推到多 class 與一般服務分布，而且邊界的形狀會讓你意外；第二層，你那些打叉的特徵確實在國界之外，每一個都有明確的「為什麼」；第三層，出了國界不是只剩兩手一攤——分解近似給方向，模擬給真相（見 ch19）。

## BCMP 定理：product form 的精確國界

1975 年，Baskett、Chandy、Muntz、Palacios 四人在 *Journal of the ACM* 發表了一條把 Jackson 推到極限的定理，後人取四人姓氏首字母叫它 **BCMP 定理**。它管開放、封閉、混合網路，允許多個工作類別（class）各走各的路由，並回答一個精確的問題：**哪些「服務站類型」的組合仍然保有乘積形式**。答案是恰好四種：

| 類型 | discipline | 服務分布 | 多 class 待遇 |
|---|---|---|---|
| 1 | FCFS | **必須指數**，且同站所有 class 共用同一個 μ | 不允許各 class 不同服務率（μ 可隨佇列長度變化） |
| 2 | PS（processor sharing） | 一般分布※ | 每個 class 可以有自己的分布 |
| 3 | IS（無限伺服器，delay station） | 一般分布※ | 每個 class 可以有自己的分布 |
| 4 | LCFS-PR（後到先服務、搶占可續） | 一般分布※ | 每個 class 可以有自己的分布 |

※技術條件是「Laplace 轉換為有理函數」（Cox 分布）；這族分布可以任意逼近一般分布，所以工程上讀成「幾乎任意」即可——稠密性論證本書不展開。

只要網路裡每一站都屬於這四型、路由是機率式的、外部到達是（可隨網路總人數調變的）Poisson，平穩分布就是乘積形式：π(n₁,…,n_k) = (1/G)·Π fᵢ(nᵢ)，每站一個只依賴自己參數的因子，封閉與混合網路差在正規化常數 G（ch17 的老朋友，計算痛點也一樣，MVA 思路同樣適用）。嚴謹度標示：**本章只做敘述層級**，定理的證明（驗 global balance 的推廣版，沿 ch16 的路線但帳目多很多）指向原始論文與延伸閱讀。

這張表最值得盯著看的是它的不對稱：**FCFS 那格被鎖死在指數，另外三格卻幾乎免費**。直覺從 ch12 來——FCFS 之下，服務中那支工作「已經做了多久」是貨真價實的狀態（一般分布的殘餘壽命依賴年齡），整個網路的未來都掛在這個連續量上；而 PS、IS、LCFS-PR 三種 discipline 有一種對稱性：誰先來誰後到不影響「每支在場工作此刻分到多少服務速率」的結構，年齡資訊在平穩分布層面被抹平了。這個直覺的嚴格化是 Kelly 的 symmetric queues 理論（本書不證，見延伸閱讀）。

## insensitivity：三張舊支票一次兌現

上面那個「幾乎免費」有個正式名字：**不敏感性（insensitivity）**。BCMP 類型 2、3、4 的站，**佇列長度的平穩分布只透過各 class 的 E[S] 依賴服務分布**——你把直方圖換成任何同均值的形狀，πₙ 一個數字都不動，經 Little's Law（ch02），平均回應時間也不動。

這收攏了三顆前面埋的種子：

- **ch09 的 Erlang-B**：M/G/c/c 的損失機率只看 a = λE[S]，分布形狀無關——損失系統「沒位子就走」，本質上是個有限的 IS 家族成員。當時的口訣「loss 看均值就夠，delay 要看變異數」，後半句現在有了精確版：**delay 要不要看變異數，取決於 discipline**。
- **ch09 的 M/G/∞**：佇列長度 Poisson(a)，對服務分布不敏感——它就是 BCMP 類型 3 的單站版。
- **ch13 的 PS**：M/G/1-PS 的 E[T] = E[S]/(1−ρ)，跟 M/M/1 一模一樣、跟 C²ₛ 無關（條件：Poisson 到達、PS、ρ<1）。當時是敘述加指路，現在你知道它住在哪個定理裡。

對照組是 ch12 的 P-K 公式：M/G/1-**FCFS** 的等待正比於 (1+C²ₛ)/2，對分布敏感得很。同一條到達流、同一個 E[S]，C²ₛ = 10 之下 FCFS 等 2.2 s、PS 平均只等 M/M/1 的份——discipline 的選擇決定你要不要付變異數的帳單（代價在 tail 與 slowdown，ch13 講過）。

兩個誠實的邊界，先立牌，陷阱表還會再敲一次：

1. insensitivity 保的是 **N 的平穩分布**（以及經 Little 的平均 T）。**回應時間 T 的分布不在保固內**——M/G/1-PS 的 P99 仍然依賴服務分布的形狀。「PS 之下重尾無害」是錯讀，重尾傷的就是 tail（ch15）。
2. 它屬於 PS／IS／LCFS-PR，**不屬於 FCFS**。你的 SQS worker pool 是 FCFS：拿 E[S] 單獨做容量規劃，在 C²ₛ = 10 的工作組合上等待低估 5.5 倍（ch12 的帳）。Node.js 的 event loop 近似 PS（ch13 討論過哪裡像哪裡不像），所以「平均延遲用 E[S] 估」在那裡反而站得住。

## 打破 product form 的工程現實

乘積形式的證明引擎是 balance 結構：每站的機率流可以**一站一站獨立結帳**。任何把兩站狀態鎖在一起的機制，都會把帳本黏住。逐一過你系統裡的嫌疑犯：

**Priority（跨 class 插隊）**。BCMP 類型 1 要求 FCFS 對 class 一視同仁，類型 2–4 的 discipline 也都是 class 對稱的。一旦某站讓 health check 或 VIP 流量插隊，該站的狀態就不能再用「總人數」摘要，乘積形式整體失效。單站的 priority 公式 ch13 給過；**網路級**的 priority 一般沒有封閉解，這是「公式之外」最常見的入口之一。

**有限 buffer 與 blocking**。ch10 的牆放進網路裡會傳染：下游滿了，上游那支完成的工作走不了，要嘛佔著伺服器（blocking after service）、要嘛整站停擺——兩站的狀態徹底耦合，環狀拓撲還可能死鎖。ch10 結尾那句「blocking 是 ch18 的戰場」就是這裡，而戰報很短：一般沒有乘積形式，文獻靠近似與模擬。值得記的例外：**純損失網路**（電話那種「沒電路就拒接、絕不排隊」的網路）反而保有乘積形式——Erlang 的世界又一次展現它的好脾氣（指路 Kelly）。

**Batching**。批次到達或批次服務違反「一次動一個」的轉移結構。用 ch14 的語言：批次把 C²ₐ 炸大、還製造同批工作之間的相關性——兩階矩近似都喊吃力，乘積形式更不用說。等湊批與立即發的權衡是 ch20 的節目。

**Retry 相關性**。ch16 把 retry 建成 Bernoulli 路由：每支工作獨立擲骰、以機率 p 回頭——這保住了 Jackson 結構，也算出 p 從 0.2 到 0.3 就讓 ρ₂ 衝破 1 的警世故事。但真實的 retry 不擲骰：**timeout 在系統慢的時候觸發**，回頭的流量與壅塞正相關，越塞越退、越退越塞。這不只打破乘積形式，連「狀態空間裡的 Markov 路由」都不是。它通往的世界（雙穩態、metastable failure）留給 ch20，這裡只立界碑：**Jackson 的 retry 模型是樂觀下界**。

**同步（fork-join）**。最後一個，也是最重要的一個——它值得獨立一節。

## fork-join：最常見卻最難

```
                +--> [queue 1] --> (server 1) --+
                |                               |
 jobs --> fork -+--> [queue 2] --> (server 2) --+--> join --> done
                |                               |
                +--> [queue 3] --> (server 3) --+
```

一支工作抵達 fork 點，被複製成 n 份同時送進 n 條分支，各自排隊、各自服務；join 點等**全部** n 份完成才放行。`Promise.all`、scatter-gather、MapReduce 的一個 wave、RAID 的條帶讀寫——同一個結構。

它為什麼難？join 等的是 n 條分支逗留時間的 **max**，而這 n 個量彼此**相關**：兄弟工作同一瞬間出生，上游一個 burst 會同時灌進所有分支，每條佇列的忙閒同漲同跌。狀態空間是 n 條佇列的聯合狀態，拆不開——乘積形式的「一站一站結帳」在 join 的同步約束前直接破產。難到什麼程度：**指數服務下，平均回應時間的精確公式只有 n = 2 有**（Flatto & Hahn 1984，用了複變函數的 uniformization 技巧）；n ≥ 3 至今只有 bounds 與近似（Nelson & Tantawi 1988 的 scaling approximation 是工程上最常用的一支，作者報告 n ≤ 32 時對模擬誤差在 5% 內；bounds 的源頭是 Baccelli–Makowski 一系），綜覽見延伸閱讀的 Thomasian survey。

但「精確解難」不代表「沒法估」。先把問題剝到最乾淨的一層，親手算出形狀。

### 動手算：E[max] = H_n/μ（無排隊的理想化）

**先聲明這是什麼、不是什麼**：以下假設 n 條分支**即到即服務、完全不排隊**，每路服務時間獨立同分布 Exp(μ)。也就是說我們算的是 **n 個獨立指數服務時間的 max 的期望**，不是真實佇列系統裡 fork-join 的逗留時間（那要加上每路的等待、還有相關性——下一小節誠實處理）。理想化的價值：它給出扇出代價的**下限與形狀**，而且算得動。

工具是 ch04 的競賽引理加 ch03 的期望值線性。把 max 切成 n 段「存活競賽」：

```text
段 1：n 路都在跑。最先完成者 ~ Exp(nμ)（競賽引理：min 的速率相加）
      → 這段期望 1/(nμ)
段 2：剩 n−1 路。無記憶性（ch04）：存活者全部「重新開始」，
      下一個完成 ~ Exp((n−1)μ) → 期望 1/((n−1)μ)
  ⋮
段 n：剩最後 1 路 ~ Exp(μ) → 期望 1/μ

E[max] = (1/μ)·(1/n + 1/(n−1) + ⋯ + 1) = H_n/μ
```

H_n = 1 + 1/2 + ⋯ + 1/n 是調和數——ch03 紙上推演裡「紀錄個數」那題的老朋友，這裡以 E[max] 的身分回鍋。嚴謹度標示：**工程師的嚴謹**——每段的分布靠競賽引理（ch04 證過 min ~ Exp(Σλ)、且勝者身分與時長獨立）與無記憶性，段與段相加靠期望值線性；各段甚至互相獨立（無記憶性把歷史抹掉），這個獨立性等下算變異數會用到。**條件**：n 路獨立、同為指數、無排隊——三個都會在現實中破，先記著。

n = 2 驗算，換 ch03 的 tail 積分路線對帳：E[max] = ∫₀^∞ P(max > t) dt = ∫₀^∞ (2e^(−μt) − e^(−2μt)) dt = 2/μ − 1/(2μ) = 1.5/μ = H₂/μ ✓。兩條路線同一個答案，這個推導值得你抄一遍。

代入推播管線的基準 E[S] = 100 ms（μ = 10/s），逐項重算 H_n（自我複核：逐項相加，n=10 那行是 1 + 0.5 + 0.3333 + 0.25 + 0.2 + 0.1667 + 0.1429 + 0.125 + 0.1111 + 0.1 = 2.929）：

| 扇出 n | H_n | E[max]（μ=10） | 相對單路 |
|---|---|---|---|
| 1 | 1.000 | 100 ms | 1.00× |
| 2 | 1.500 | 150 ms | 1.50× |
| 4 | 2.083 | 208 ms | 2.08× |
| 10 | 2.929 | 293 ms | 2.93× |
| 100 | ≈5.187 | ≈519 ms | 5.19× |

n=100 用 H_n ≈ ln n + γ（γ ≈ 0.5772，Euler–Mascheroni 常數；修正項 1/(2n) 已含）：4.6052 + 0.5772 + 0.005 ≈ 5.187。**帳單以 ln n 成長**：扇出翻 10 倍，平均代價多一截 ln 10/μ ≈ 230 ms，不是翻 10 倍——對數很仁慈，但它只仁慈在均值。

順手算個有用的小帳：第 n 段（等最後一名）的期望是整整 1/μ = 100 ms，**全帳單裡最貴的一項是等最後一個**。這直接指向緩解手段（見本節末）。另外，因為各段獨立，Var(max) = (1/μ²)·Σ1/k²；n=10 時 Σ1/k² ≈ 1.550，C² = 1.550/2.929² ≈ 0.18——max 是個「集中化」的操作，分布比單路的指數（C²=1）窄得多。這解釋了等下 tail 算出來的一個現象。

### 尾部的帳單：P99 劣化估算

max 的 CDF 是乘出來的：P(max ≤ t) = (1 − e^(−μt))ⁿ（條件：n 路獨立）。解 P99：

```text
(1 − e^(−μt))ⁿ = 0.99  ⇒  e^(−μt) = 1 − 0.99^(1/n)
n=10：0.99^(1/10) = e^(ln 0.99/10) = e^(−0.001005) ≈ 0.998995
      e^(−μt) ≈ 0.001005  ⇒  μt = ln(995.5) ≈ 6.903
      t₉₉ ≈ 0.690 s（μ = 10）
對照 n=1：t₉₉ = ln 100/μ = 4.6052/10 ≈ 0.461 s
```

好用的近似來自 union bound：P(max > t) ≤ n·e^(−μt)（尾部漸近準確），令右邊 = 0.01 得 **t₉₉ ≈ ln(100n)/μ**——n=10 給 ln 1000/10 = 0.691 s，跟精確值差不到 1 ms。讀法：**扇出把 P99 從 ln 100 推到 ln(100n)，加法地長一截 ln n**。

三個彼此等價、但工程衝擊力不同的讀法（n = 10）：

1. **絕對值**：P99 從 461 ms 變 690 ms，劣化 1.50 倍——比平均的 2.93 倍溫和（因為 max 把分布壓窄了，C² ≈ 0.18；P99/mean 從 4.6 掉到 0.690/0.293 ≈ 2.36，ch08 的「4.6× 規則」在這裡失效是**往下**失效）。
2. **配額制**：要 join 的 P99 守在 t，每一路得保證 P(S ≤ t) ≥ 0.99^(1/10) ≈ 0.9990——**每路要交出自己的 P99.9 才能換到整體的 P99**。一般地 0.99^(1/n) ≈ 1 − 0.01/n：扇出 n 路，每路的尾部預算被砍成 1/n。
3. **稀有事件變日常**：單路落入自己最慢的 1% 是稀有事件；10 路裡至少一路中招的機率是 1 − 0.99¹⁰ ≈ 9.6%——**每 10 個請求就有一個在等某路的「百年一遇」**。這正是 The Tail at Scale 那篇文章的核心觀察（見 ch20），你現在有它的手算版。

自我複核：0.99¹⁰ = e^(10·ln 0.99) = e^(−0.1005) ≈ 0.9044，1 − 0.9044 = 0.0956 ✓；6.903/4.6052 ≈ 1.499 ✓。

### 真實系統裡的 fork-join：理想化哪裡會破

上面三個條件逐一面對現實：

**「無排隊」破**。真實分支是一條佇列，逗留 = 等待 + 服務。最便宜的修補是**獨立近似**：每路當成獨立的 M/M/1，其逗留時間 ~ Exp(μ−λ)（ch08 的精確結果），把上面所有公式裡的 μ 換成 μ−λ。基準數字（每路 λ=8、μ=10、ρ=0.8）：單路平均 0.5 s、P99 2.30 s（ch08 基準）；扇出 10 路後 join 平均 ≈ H₁₀/(μ−λ) = 2.929/2 ≈ 1.46 s、P99 ≈ ln 1000/2 ≈ 3.45 s。倍率不變（公式只是換尺度），但絕對值提醒你：**高使用率與扇出相乘，1/(μ−λ) 與 H_n 兩個放大器疊在一起**。

**「獨立」破**。兄弟同時出生、共享同一條到達流，各分支的等待**正相關**。方向可以說清楚（工程師的嚴謹）：對正相關（associated）的隨機變數，P(全部 ≤ t) ≥ Π P(各自 ≤ t)，所以真實的 max 隨機地**小於等於**獨立版的 max——獨立近似是偏保守（高估）的方向，Baccelli–Makowski 的 bounds 正是用這類相關性結構夾出來的。量化一眼：n=2 的精確解（Flatto–Hahn 的結果，Nelson–Tantawi 整理成這個形）是 E[T₂] = ((12−ρ)/8)·1/(μ−λ)，係數從 ρ→0 的 1.5（正是 H₂）滑到 ρ→1 的 1.375；ρ=0.8 時係數 1.4，獨立近似的 1.5 高估約 7%。**但注意條件**：這個「保守」是此模型（共享到達、服務獨立）的性質；若分支共享下游資源（同一顆 DB、同一條網路），相關性可以反咬，方向不再有保證。

**「指數」破**。服務若是重尾（ch15），max 的行為由最重的那路主宰，H_n 的對數仁慈消失，P99 劣化遠比上表兇——這是「scatter-gather 的 P99 由最慢分片決定」在重尾世界的加強版。量化只能靠模擬（ch19）。

**緩解手段**的方向現在都讀得出來：①能不扇就不扇（n 進對數也進帳單）；②**只等前 k 名**：quorum 讀寫（k-of-n）把期望砍成 (H_n − H_(n−k))/μ——n=10 只等前 9 名，期望從 293 ms 掉到 (2.929−1)/10 ≈ 193 ms，**放掉最後一名就省下最貴的 100 ms**；③對落後者補射或綁定取消——hedged / tied requests，它們的設計與帳單見 ch20。

## 分解近似：把網路拆回一站一站的 G/G/1

fork-join 之外，多數「條件不滿足」的開放網路有一條務實的路，源頭是 Whitt 1983 年在 Bell Labs 做的 **QNA（Queueing Network Analyzer）**。思路一句話：**放棄聯合分布，假裝每站是一條獨立的 G/G/1（或 G/G/m），只在站與站之間傳播兩個數字——流量 λ 與變異性 C²ₐ**，然後每站套 ch14 的 Kingman/VUT 公式。一階矩（流量）的傳播就是 ch16 的 traffic equations，精確；二階矩（C²）的傳播靠三條規則：

```text
① 合流（superposition）：λ = Σλⱼ；C²ₐ ≈ 各支流 C² 的加權混合
   （最樸素的權重是流量比 λⱼ/λ；QNA 用更講究的混合——近似）
② 分流（Bernoulli split, 機率 p）：λ′ = pλ；C²′ = p·C² + (1 − p)
   （條件：renewal 輸入＋獨立擲骰，此式精確；分流讓 C² 朝 1 靠）
③ 過站（departure）：C²_d ≈ ρ²·C²ₛ + (1 − ρ²)·C²ₐ
   （近似。ρ→1 伺服器不停手，輸出像服務過程；ρ→0 工作穿堂而過，
    輸出像到達過程——ch11「M/D/1 輸出比 Poisson 平滑」的量化版）
```

嚴謹度標示：②在條件下精確，①③是近似，**QNA 的公式推導本書不展開**（指路延伸閱讀）。對全部是指數／Poisson 的網路，三條規則都退化成 C² = 1，分解近似退回 Jackson——它是 Jackson 的誠實推廣，而不是另一套世界觀。

這也終於給 ch01 經驗法則審判的第 6 條（「上下游各自做容量規劃就好」）正式判決：**它不是定理，是一個有名字的近似**。合法性分三級：tandem 無 feedback 且全指數時精確（ch11 Burke）；一般開放網路是 QNA 式近似，誤差來源明確；fork-join 與 blocking 之下連近似的形都不對，要換工具。

誤差從哪來，自我察覺什麼：C² 是一個數字，**抓不到時間相關性**（ch14 立過的牌——burst 的「何時來」比「多分散」更傷）；分解假設站與站獨立，**深的 feedback 迴圈**讓同一支工作的多次過站相關，QNA 對此有專門修正但效果有限；**重尾**讓兩階矩本身失去代表性（ch15）。徵兆：網路裡有強 feedback、有共享資源、或量到的 C² 大得反常——這時候抽一站用模擬對帳，對不上就整套換模擬。

## 公式 → 近似 → 模擬：決策框架

把本章與 Part V 收成一張決策圖：

```text
 全部站都落在 BCMP 四型？路由是機率式？緩衝無限？無同步？
   │
   ├─ 是 ──▶ 公式：Jackson（ch16）／BCMP／MVA（ch17）
   │         便宜、精確、可手算——能用就用，別客氣
   │
   ├─ 只差「溫和」假設（FCFS 非指數、到達非 Poisson），
   │  且只要平均值與方向？
   │         ──▶ 分解近似（QNA 思路）：traffic equations
   │             ＋ C² 傳播 ＋ 每站 Kingman（ch14）
   │             帶著誤差意識用：feedback 深、重尾、強相關時降級
   │
   └─ 有 fork-join／blocking／priority 交互／retry 相關性，
      或你要的是 tail 與 transient，不是平均？
              ──▶ 模擬（ch19）——但模擬會用四種方式騙你，
                  方法學是下一章的全部內容
```

判準不是教條，是成本與問題形狀的匹配：公式以秒計、近似以小時計（要量 C²、要驗證）、模擬以天計（寫對、跑夠、算對信賴區間）。**順序永遠由便宜往貴走**，而且貴的工具要拿便宜工具當 ground truth 校準——ch19 開場就是這件事。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦 |
|---|---|---|
| 把 insensitivity 當萬靈丹 | 它屬於 PS／IS／LCFS-PR，不屬於 FCFS。FCFS worker pool 拿 E[S] 規劃，C²ₛ=10 時等待低估 5.5 倍（ch12） | 先問 discipline 再問分布；口訣升級版：**delay 看不看變異數，取決於 discipline** |
| insensitivity 推到 tail | 保固只到 N 的分布與平均 T；M/G/1-PS 的 P99 照吃服務分布的形狀 | tail 問題回 ch13/ch15 的工具，或模擬 |
| 用單路延遲給 fan-out 訂 SLA | 平均差 H_n 倍（n=10 是 2.93×）；P99 上每路要交 P99.9 才換到整體 P99；10 路裡 9.6% 的請求在等某路的「最慢 1%」 | 寫下扇出帳單表；扇出數進架構評審；考慮 quorum／hedging（ch20） |
| 把 E[T] ≥ ΣDᵢ 用在有 fork-join 的系統 | ch02 那條 bound 假設工作同一時刻只在一站受服務；並行讓 E[T] 可以小於 ΣDᵢ——bound 直接不成立 | 有 fork-join 時，下界改用「最長分支」：E[T] ≥ max 路徑的 ΣDᵢ |
| 拿 ch16 的 Bernoulli retry 模型做正式容量 | 真實 retry 與壅塞正相關，Jackson 解是樂觀下界；它連 p=0.2→0.3 就爆炸都告訴你了，真實系統更早 | 把 Jackson 解當 smoke test；retry 動態看 ch20 的雙穩態模型 |
| 有限 buffer／blocking 照套乘積形式 | blocking 把上下游狀態鎖死，各站獨立結帳的前提消失；算出來的 π 連定性都可能錯 | 純損失網路是例外（指路 Kelly）；其餘靠近似文獻或模擬 |
| 分解近似當精確解報告 | C² 抓不到時間相關與重尾；深 feedback 下站間獨立假設失真，誤差無上界保證 | 報告標注「近似」與假設清單；抽站模擬對帳；重尾（ch15）直接跳模擬 |

## 紙上推演

**1. [10 分鐘 ★★] 國界巡邏。**判斷下列四個網路有沒有乘積形式的平穩分布，各用一句話說出理由：(a) 三站開放網路，Poisson 到達，全部 FCFS＋指數服務，Bernoulli 路由，緩衝無限；(b) 同 (a)，但第二站改成 PS、服務時間 lognormal（C²ₛ = 4）；(c) 同 (a)，但第二站是 FCFS 且收兩個 class，平均服務時間分別是 50 ms 與 550 ms；(d) 同 (a)，但第二站到第三站之間緩衝上限 5，滿了就堵住第二站。

**2. [15 分鐘 ★★] 扇出帳單。**scatter-gather 打 4 個 shard，理想化：每路獨立 Exp(μ)、μ = 10/s、不排隊。(a) 算 E[max] 與相對單路的倍率；(b) 算 join 的 P99（精確或 union bound 皆可），並回答「每路要交出第幾百分位才能守住整體 P99」；(c) 改成只等前 3 名（3-of-4 quorum），E 變多少？省下的是哪一段？(d) 現在誠實化：每路其實是 ρ = 0.8 的 M/M/1（λ=8、μ=10）。用獨立近似估 join 的平均，並說明這個估計偏哪個方向、為什麼。

**3. [10 分鐘 ★★] 選工具。**一個系統：API 層（Node.js，近似 PS）→ 同步扇出 8 個 shard（各自 FCFS、服務時間直方圖右偏 C²ₛ ≈ 6）→ 聚合後寫入 DB pool（連線數上限 20，滿了排隊，等待逾時觸發 client retry）。對「API 平均延遲」「聚合層 P99」「retry 對容量的影響」三個問題，各選「公式／近似／模擬」並說理由。

### 推演解答

**1.** (a) **有**——這就是 Jackson 網路（ch16），條件逐項滿足。(b) **有**——PS 是 BCMP 類型 2，服務分布一般（lognormal 的 Laplace 轉換可由 Cox 分布逼近，工程上直接讀「任意」）；而且該站 insensitive，平均指標只用 E[S] 就能算。(c) **沒有**——FCFS 站要保乘積形式必須指數**且所有 class 同一個 μ**（BCMP 類型 1 的硬條件）；兩個 class 平均差 11 倍，直接出局。注意細節：若兩 class 是同參數指數，反而可以。(d) **沒有**——有限緩衝的 blocking 把二、三站狀態耦合，乘積形式的獨立結帳失效。常見錯路：(c) 很多人以為「多 class 就交給 BCMP」，忘了 BCMP 對 FCFS 站根本沒放寬 class 條件。

**2.** (a) H₄ = 1 + 0.5 + 0.3333 + 0.25 = 2.0833，E[max] = H₄/μ ≈ **208 ms**，單路 100 ms 的 **2.08 倍**。(b) 精確：0.99^(1/4) = e^(−0.0100503/4) ≈ 0.99749，e^(−μt) ≈ 0.00251，μt = ln 398.5 ≈ 5.988，t₉₉ ≈ **0.599 s**；union bound 給 ln 400/10 ≈ 0.599 s，一致。單路 P99 是 0.461 s，劣化 1.30 倍。每路要交出 0.99^(1/4) ≈ 0.9975，即**約 P99.75**（用 1 − 0.01/n = 0.9975 心算同樣到位）。(c) E[前 3 名到齊] = (H₄ − H₁)/μ = 1.0833/10 ≈ **108 ms**——省下 100 ms，正是「等最後一名」那段 Exp(μ) 的全額，帳單最貴的一項。(d) 每路 M/M/1 的逗留 ~ Exp(μ−λ) = Exp(2)（ch08），獨立近似：E ≈ H₄/(μ−λ) = 2.0833/2 ≈ **1.04 s**。方向：**高估**——四路共享同一條到達流，等待正相關，正相關變數的 max 隨機地小於獨立版（P(全部≤t) ≥ ΠP(各自≤t)）；n=2 的精確解顯示 ρ=0.8 時獨立近似約高估 7%，n=4 沒有精確解，但方向相同。常見錯路：(d) 忘了把 μ 換成 μ−λ，算出 208 ms——那是空系統的數字，差 5 倍。

**3.** 參考答案（你的理由比選項重要）：**API 平均延遲 → 公式**。PS 站 insensitive，E[T] = E[S]/(1−ρ) 只要量 E[S] 與 ρ，C²ₛ 再醜都不影響平均——這正是 insensitivity 的現金價值。**聚合層 P99 → 模擬**。fork-join（n=8）疊上右偏服務（C²ₛ≈6）：精確解不存在、E[max] 公式假設指數、tail 是 insensitivity 與兩階矩近似都不保的量——三個工具同時失效，只剩模擬，而且要按 ch19 的方法學跑。粗估可先用獨立近似＋union bound 給個方向，但報告要標注假設。**retry 對容量 → 先近似後模擬**。先用 ch16 的 Bernoulli feedback 解 traffic equations 找「哪個 p 會把哪站推破 ρ=1」——這是樂觀下界但便宜；真實的相關 retry 與雙穩態行為要靠 ch20 的模型與模擬。整題的框架就是本章的決策圖：由便宜往貴走，每一步知道自己丟了什麼假設。

## 自我檢核

口頭回答，講不清楚就回去重讀對應小節：

1. BCMP 四種站類型是哪四種？哪一型對服務分布最嚴格、嚴格在哪兩點？為什麼偏偏是 FCFS（用 ch12 的狀態論證講）？
2. insensitivity 精確地說是「哪個量」對「什麼」不敏感？兩個常見的過度推論（FCFS、tail）各錯在哪？
3. Erlang-B insensitive、Erlang-C 敏感——這個 ch09 的不對稱，放進本章框架後怎麼解釋？
4. blocking 為什麼殺死乘積形式？哪一類網路是著名例外？
5. E[max] = H_n/μ 的競賽切段推導重講一遍：每段的分布靠哪條引理？段與段相加靠什麼？哪三個條件在現實中會破？
6. 「扇出 n 路，每路要交出 P99.9 才換到整體 P99」是怎麼算出來的？n=10 時「至少一路落入自己最慢 1%」的機率是多少？
7. E[max of 服務時間] 與真實 fork-join 逗留時間差在哪三件事？獨立近似偏哪個方向、依據是什麼、什麼時候連方向都不保？
8. QNA 的三條傳播規則是什麼？哪條精確（條件呢）、哪兩條是近似？什麼徵兆出現時你該放棄分解近似改用模擬？
