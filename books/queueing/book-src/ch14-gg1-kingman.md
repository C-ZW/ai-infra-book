# ch14 — G/G/1 與 Kingman：工程師的萬用近似

> **本章解決什麼問題**：ch12 放掉了指數服務、ch13 放掉了 FCFS 的出工順序，本章放掉最後一根柱子——Poisson 到達。代價立刻攤牌：G/G/1 沒有封閉解，這是全書第一次正面承認「公式到頭了」。但你會拿到兩樣替代品：**Lindley recursion**，一條精確到可以直接餵 production trace 的遞迴（它就是 ch19 模擬的核心迴圈）；以及 **Kingman 近似**——讀成 VUT 公式之後，它是全書的工程總綱：等待 ≈ 變異性 × 使用率 × 時間。ch10 借走的整形帳單在這裡還本付息，ch15 會炸掉它的重尾邊界，ch18 拿它當網路分解的磚塊。

## 從你已知的出發

做容量規劃時你真正擁有的是什麼？不是「到達是 Poisson、服務是指數」的證書，而是監控系統裡的四個數字：QPS、平均服務時間、到達間隔的抖動、服務時間的抖動。Part III 的公式全都要求前兩個假設，而你的系統兩個都不滿足：推播管線的到達流是使用者行為、整點 cron 批次、下游 retry 的匯流——ch05 的 Palm–Khintchine 直覺只擔保「大量獨立稀疏來源」那部分像 Poisson，cron 與 retry 是少數、肥大、彼此相關的來源，它們把到達間隔的變異推離指數。本章基準就用這個設定：量測得 **C²ₐ = 2**——比 Poisson 叢發，但還沒病態。

手上還有兩張舊借據。ch10 的整形段落借用了一條「ch14 的公式」算出 0.4 s → 0.2 s；ch12 推演 3 留了話：「兩根槓桿的正式比價是 ch14 的 worked example」。本章全部結清。最後一個錨點是你做過的壓測：k6 固定速率打出來的延遲總是比 production 好看，差距還不小。這不是工具的 bug——固定間隔把 C²ₐ 做成了 0，你壓測的是另一個系統。本章會算出你低估了幾倍。

## G/G/1：公式到此為止

記號對齊：G/G/1 ＝ 到達間隔 Aₙ 是 i.i.d. 任意分布（第一個 G，General；E[A] = 1/λ）、服務時間 Sₙ 是 i.i.d. 任意分布、與到達獨立（第二個 G）、單一伺服器、FCFS、無限緩衝、ρ = λE[S] < 1。

為什麼沒有封閉解？要描述某時刻的系統，你得記三件事：佇列長度、服務中工作已進行多久、距上一次到達多久。兩個連續時鐘，而且兩個都不再無記憶——ch12 殺掉了服務端的時鐘，現在到達端的也死了。Part III 的生滅引擎雙缸熄火；PASTA 也陣亡（到達不是 Poisson）。倖存者名單很短但很硬：Little's Law（ch02 說它與分布、排程、結構全部無關——現在你知道那個「無關」多值錢）、Utilization Law（ρ = λE[S]）、work conservation（ch13）。

還有更深一層的「沒有」。G/G/1 的 E[T_Q] 不是由兩個分布的少數幾階矩決定的——**整個分布形狀都進場**：兩組前兩階矩完全相同、形狀不同的分布，會給出不同的等待。所以「沒有封閉解」不是還沒找到，而是答案本質上是一個吃整條分布的泛函。Lindley 1952 把問題化成一條積分方程；一般解法要動用 Wiener–Hopf 分解，只在特殊結構（M/G/1、GI/M/1 等）落回公式——存在聲明，本書不展開。

本章策略雙軌：一條**精確的遞迴**（不給公式、給演算法），一條**漸近的近似**（給公式、附誤差說明書）。

## Lindley recursion：沒有公式，但有一條精確的迴圈

設第 n 位乘客在 tₙ 到達、等了 Wₙ、服務 Sₙ；Aₙ₊₁ 是第 n 與 n+1 位之間的到達間隔。完整推導，只有兩個 case：

```text
第 n 位：tₙ 到達、等 Wₙ、服務 Sₙ ⇒ 離開時刻 Dₙ = tₙ + Wₙ + Sₙ
第 n+1 位：tₙ₊₁ = tₙ + Aₙ₊₁ 到達

若 tₙ₊₁ < Dₙ：前一位還沒清完 ⇒ Wₙ₊₁ = Dₙ − tₙ₊₁ = Wₙ + Sₙ − Aₙ₊₁
若 tₙ₊₁ ≥ Dₙ：到達時伺服器已空 ⇒ Wₙ₊₁ = 0

合併：Wₙ₊₁ = max(0, Wₙ + Sₙ − Aₙ₊₁)      ← Lindley recursion（Lindley 1952）
```

用掉的假設只有四個：單一伺服器、FCFS、work-conserving（有工作就不偷懶）、緩衝無限。清單上**沒有任何分布假設**——這是一條逐樣本路徑成立的恆等式，對任何序列 {Sₙ}、{Aₙ} 都對，包括從你 production log 撈出來的那條。手追一遍（單位 ms）：

| n | 到達 tₙ | Sₙ | Wₙ 的計算 | Wₙ |
|---|---|---|---|---|
| 1 | 0 | 100 | 空系統 | 0 |
| 2 | 80 | 120 | max(0, 0+100−80) | 20 |
| 3 | 230 | 80 | max(0, 20+120−150) = max(0, −10) | 0 |
| 4 | 290 | 200 | max(0, 0+80−60) | 20 |
| 5 | 390 | 60 | max(0, 20+200−100) | 120 |

看 job 3：Wₙ + Sₙ − Aₙ₊₁ = −10 ms，被 max(0,·) 沒收。那 10 ms 是伺服器的閒置——存不起來、之後也補不回來。這個不對稱就是 ch01「λ < μ 還是會排隊」的代數形態：**盈餘（提早完工）被歸零，赤字（遲到）全額遞延**。等待是一具棘輪，只有撞到 0 才重置；佇列系統所有的非線性都藏在這個 max 裡。

它的工程價值被嚴重低估：把 access log 的到達時刻與服務時間餵進去，三行程式重放歷史；然後改劇本問反事實問題——服務快 10% 等待掉多少？到達整形之後尖峰還在嗎？不需要任何模型假設，**trace 本身就是模型**。ch19 會把這條遞迴正式任命為 G/G/1 模擬的核心迴圈，並補上方法學（warmup、信賴區間——那裡才是讓模擬說真話的主場）。多伺服器的對應遞迴要追每台機器的剩餘工作向量，本書不展開——ch19 直接用事件驅動模擬。

理論面的收尾：ρ < 1 時 Wₙ 的分布收斂到唯一的平穩分布（Lindley 1952，本書不證）；求這個平穩分布＝解 Lindley 積分方程，於是繞回上一節的死路。精確路線到此為止，接下來換近似。

## ρ → 1：等待為什麼變成指數

定義 Uₙ = Sₙ − Aₙ₊₁，遞迴變成 Wₙ₊₁ = max(0, Wₙ + Uₙ)：**等待是一個被反射在 0 的隨機漫步**，步伐 Uₙ 的均值與變異數是

```text
E[U] = E[S] − E[A] = −(1−ρ)/λ < 0        ← 負漂移：長期往 0 走
Var(U) = σ²ₛ + σ²ₐ                        ← S 與 A 獨立，變異數相加
```

```
 Wₙ（第 n 個 job 的等待；反射在 0 的隨機漫步）
 ▲
 │                 *
 │              * * *
 │     *       *     *
 │    * *   * *       *           * *
 │   *   * *           *         *   *
 │  *     *             *   *   *     *
 └─*─────────────────────*─*─*─────────▶ n
   撞到 0 ＝ 伺服器閒置、漫步重新出發；
   ρ→1 時漂移趨近 0，兩次觸底之間的「長征」越來越長
```

直覺版論證（本書不證，嚴格版見延伸閱讀）：ρ 接近 1 時漂移 −(1−ρ)/λ 微弱，漫步很少回到 0——等待在「長征」裡累積，每段長征是**幾百幾千步 Uₙ 的總和**。很多 i.i.d. 隨機變數的和，中央極限定理出手：和的行為只由 E[U] 與 Var(U) 決定，分布的第三階矩以上、形狀細節，全部被加總洗掉。這就是 heavy traffic 的核心訊息——**飽和邊緣，世界只剩前兩階矩**。負漂移漫步反射在 0 的平穩分布是指數形（把它想成 ch08 的 M/M/1 在連續極限的影子），均值為 Var(U)/(2|E[U]|)：

```text
E[T_Q] ≈ Var(U)/(2|E[U]|) = λ(σ²ₐ + σ²ₛ)/(2(1−ρ))     ← ρ→1 的極限形
```

嚴格版的歸屬（依 landscape 查證）：Kingman 1961 的三頁短文 "The Single Server Queue in Heavy Traffic"（PCPS）首次給出，1962 年 JRSS-B 的長文 "On Queues in Heavy Traffic" 一般化——常被混引成一篇，其實是兩篇。順帶的禮物：既然 ρ→1 時等待趨向指數分布，ch08 的「P99 ≈ 4.6 × 平均」經驗式在 G/G/1 的飽和區近似還魂——條件是 ρ 夠高、尾巴夠輕（ch15 會沒收後者）。

低 ρ 時這套論證從第一句就斷：漫步常常觸底，等待由一兩步內的事件決定（你撞上誰的殘餘），沒有「長征」讓 CLT 洗掉形狀——記住這個斷點，誤差帳要用。

## Kingman 上界：一頁紙的推導

heavy traffic 給的是極限，不是保證。同樣的式子其實對**所有** ρ < 1 都是上界——這是 Kingman 1962 年在 Biometrika 的第三篇論文（"Some inequalities for the queue GI/G/1"；1961–62 他連發三槍）。推導值得你抄一遍，全程只有一個不等號。

把遞迴寫成無截斷的形式：Wₙ₊₁ = Wₙ + Uₙ + Yₙ，其中 Yₙ = max(0, −(Wₙ+Uₙ)) 是這一步被沒收的閒置。關鍵恆等式：Yₙ > 0 時 Wₙ + Uₙ = −Yₙ，所以 (Wₙ+Uₙ)·Yₙ = −Yₙ²（Yₙ = 0 時兩邊都是 0）。平方展開：

```text
Wₙ₊₁² = (Wₙ+Uₙ)² + 2(Wₙ+Uₙ)Yₙ + Yₙ² = (Wₙ+Uₙ)² − Yₙ²

穩態下取期望，E[Wₙ₊₁²] = E[Wₙ²] = E[W²]：
E[W²] = E[W²] + 2E[W]E[U] + E[U²] − E[Y²]   ← Wₙ 與 Uₙ 獨立（W 只由更早的 U 決定）
⇒ E[W] = (E[U²] − E[Y²]) / (2|E[U]|)         ← E[U] < 0，移項

一次方的期望平衡：0 = E[U] + E[Y] ⇒ E[Y] = |E[U]|（恆等式，不是不等式）
全程唯一的不等號：E[Y²] ≥ (E[Y])² = E[U]²    ← 變異數非負，丟掉的是 Var(Y)
⇒ E[W] ≤ (E[U²] − E[U]²)/(2|E[U]|) = Var(U)/(2|E[U]|) = λ(σ²ₐ + σ²ₛ)/(2(1−ρ))
```

嚴謹度：**工程師的嚴謹**——每步能口頭交代；「穩態二階矩存在且相等」當作前提（需要 ρ < 1 與兩個分布的二階矩有限），不另證。鬆緊的物理：丟掉的 Var(Y) 是**閒置塊的變異數**。ρ 低時閒置常見、大小不一，丟得多——M/M/1、ρ = 0.8 代入：上界 8×(1/64 + 0.01)/0.4 ≈ 0.513 s，精確值 0.4 s，偏高 28%。ρ → 1 時閒置幾乎絕跡、Var(Y) → 0，上界貼住真值——與上一節的極限同形，不是巧合。

基準數字（C²ₐ = 2、C²ₛ = 1）：σ²ₐ = C²ₐ/λ² = 2/64 = 0.03125 s²、σ²ₛ = C²ₛE[S]² = 0.01 s²，上界 = 8 × 0.04125/0.4 = **0.825 s**。這是模型假設內「保證不超過」的數字；接下來把它磨成更準的估計。

## VUT：把上界校準成全書總綱

從上界到通行的 Kingman 近似只有一步：**把到達端的變異數打 ρ² 折**，σ²ₐ → ρ²σ²ₐ。誠實聲明：這一步不是推導，是**校準**——折扣的大小恰好讓 C²ₐ = 1 時公式落在 P-K 上（M/G/1 精確），而 ρ → 1 時折扣消失、與 heavy-traffic 極限相容。一條在「已知精確解的特例」與「已知極限」之間內插的公式，這就是 Kingman 近似的真實身分：

```text
E[T_Q] ≈ λ(ρ²σ²ₐ + σ²ₛ)/(2(1−ρ))
       = λE[S]²(C²ₐ + C²ₛ)/(2(1−ρ))      ← ρ²σ²ₐ = ρ²C²ₐ/λ² = C²ₐE[S]²；σ²ₛ = C²ₛE[S]²
       = ρ/(1−ρ) · (C²ₐ + C²ₛ)/2 · E[S]  ← λE[S] = ρ
```

換個順序讀，就是它通行世界的名字——**VUT 公式**（製造業文獻的命名，Hopp & Spearman 的《Factory Physics》把它叫 the VUT equation，當經文掛在工廠牆上）：

```text
E[T_Q] ≈ (C²ₐ + C²ₛ)/2 · ρ/(1−ρ) · E[S]
          ──────┬──────   ───┬───   ─┬──
           V 變異性       U 使用率   T 時間
```

**等待 ＝ 變異性 × 使用率 × 時間尺度**。三個因子各自獨立成章：V 是 ch03 埋的主角 C² 的最終形態——到達與服務的變異**各付各的帳、等權平分**；U 是 ch08 的 hockey stick，原樣存活；T 定尺度。全書的工程訊息濃縮成一行：想縮短等待，三根槓桿——降變異、降使用率、縮服務時間——別無其他。

適用條件先講在前面（這章本身就是在講「近似的誠實」）：①到達間隔 i.i.d.（renewal）——C²ₐ 只描述邊際分布，間隔序列的**時間相關性不在模型裡**；②服務 i.i.d.、與到達獨立、不看長度出工（FCFS；平均值同 ch12 的紀律家族）；③ρ < 1、穩態；④二階矩有限（C² 要存在，重尾見 ch15）；⑤除下面兩個特例外，**它沒有精確的時候**。

兩個健全性檢查，也是你敢把它帶上線的理由：

- **C²ₐ = C²ₛ = 1**（M/M/1）：E[T_Q] ≈ 4 × 1 × 0.1 = **0.4 s**——ch08 基準的精確值，分毫不差（M/M/1 是 M/G/1 的特例，下一條的特例）。
- **C²ₐ = 1、C²ₛ 任意**（M/G/1）：化回 ρ/(1−ρ) · E[S] · (1+C²ₛ)/2——**正是 P-K，精確**（ch12）。ch12 的 0.2／0.4／2.2 s 三連比全部被 VUT 收編。

被兩個精確結果與一個極限定理三點釘住的內插——這比「經驗公式」高一個信任等級，但仍然是近似。

### Worked example：基準 0.6 s 全算

```text
量測：λ = 8/s、E[S] = 100 ms（μ = 10/s）⇒ ρ = 0.8
      C²ₐ = 2（cron 批次＋retry 匯流，比 Poisson 叢發）
      C²ₛ = 1（服務抖動與指數同級）

V = (C²ₐ + C²ₛ)/2 = (2+1)/2 = 1.5
U = ρ/(1−ρ) = 0.8/0.2 = 4
T = E[S] = 0.1 s

E[T_Q] ≈ 1.5 × 4 × 0.1 = 0.6 s            E[T] = 0.6 + 0.1 = 0.7 s
E[N_Q] = λE[T_Q] = 4.8                     ← Little（ch02）
E[N] = E[N_Q] + ρ = 5.6                    ← E[N_服務中] = ρ（ch02）
上界對照：E[T_Q] ≤ 0.825 s（上一節）
```

同 λ 同 μ 的 M/M/1 是 0.4 s：到達端的叢發（C²ₐ 從 1 到 2）讓等待多 50%，而 QPS 與平均服務時間的儀表板上**什麼都看不到**。想要 tail：借 heavy-traffic 的指數形狀，t₉₉ ≈ ln 100 × 0.6 ≈ 2.76 s——直覺版估計，ρ = 0.8 算「夠飽和」的邊緣，當量級用、別簽進 SLA。

### Worked example：同樣的錢——C² 砍半 vs ρ 降 0.05

ch12 推演 3 的承諾。基準 0.6 s，兩種花錢方式：

| 選項 | 動作 | 新參數 | E[T_Q] | 改善 |
|---|---|---|---|---|
| V 槓桿 | 上游 cron 拆成 pacing：C²ₐ 2→0.5 | V = 0.75 | 4×0.75×0.1 = 0.3 s | **−50%** |
| U 槓桿（加容量） | μ 加 6.7%：10→10.7 | ρ = 0.75、T = 93.8 ms | 3×1.5×0.0938 = 0.422 s | −30% |
| U 槓桿（切流量） | λ 8→7.5 | U = 3 | 3×1.5×0.1 = 0.45 s | −25% |

ρ = 0.8 這一格，V 槓桿明顯贏。但兩根槓桿的**收益結構**不同，這才是重點：

- V 砍半在**任何** ρ 都恰好砍半等待——乘法結構，收益固定。
- U 槓桿的收益是 hockey stick 的局部斜率：ρ 0.8→0.75 只值 25%；ρ 0.95→0.90 是 19→9，值 **−52.6%**。交叉點約在 ρ ≈ 0.94：懸崖邊先降 ρ，沒有別的話好說。
- V 槓桿有**地板**：C² ≥ 0，而且服務若維持指數，V ≥ (0+1)/2 = 0.5——砍半的次數有限。U 槓桿沒有地板，但每一步都是容量帳單。

工程裁決：正常營運區（ρ ≤ 0.9）的變異性整治常常接近免費午餐——整形、pacing、去抖動、拆批次都是軟體活（ch10 的 leaky bucket 正是 V 槓桿的化身、token bucket 是 U 槓桿的守門員）；飽和區（ρ > 0.94）一切讓位給降 ρ。ch20 的容量規劃 walkthrough 會把這張天平用在真實量測上。

## 準確度的誠實帳

一條近似的價值，取決於你知不知道它什麼時候錯、往哪邊錯。三個薄弱帶：

**低 ρ**：heavy-traffic 的「長征」邏輯失效，等待由單次撞擊決定，分布形狀重新 matter——相對誤差變大（絕對值倒是小，U 因子壓著）。方向：**C²ₐ < 1 時偏高**。數字：ρ = 0.8、等距到達＋指數服務（C²ₐ = 0、C²ₛ = 1），Kingman 給 0.2 s，精確值（GI/M/1 理論可解，本書不推導）≈ 0.169 s——偏高 18%，ρ 越低偏越多。所以 ch10 按 Kingman 記的整形帳單 0.4 → 0.2 s，精確帳其實更漂亮（≈ 0.17 s）：近似在這個方向上是保守的。要更準：Krämer–Langenbach-Belz（1976）的修正因子就是為 C²ₐ < 1 區設計的（指路，本書不推）。

**重尾**：Pareto α ≤ 2 時變異數無限，C² 根本不存在，公式無定義；更陰的是有限樣本估出的 C²ₛ 永遠是個「看起來合理」的有限數，而且隨樣本變大一路上漂——你以為在用兩階矩近似，其實在用雜訊。方向：**嚴重低估**。徵兆與後果 ch15 整章處理。

**相關到達**：C²ₐ 量的是到達間隔的**邊際**變異，抓不到間隔序列的自相關——on/off 叢發、整點齊發、retry 連環，都能在 C²ₐ 不大的偽裝下讓等待爆掉，Kingman **低估、可差到數量級**。要把相關性建進模型，關鍵字是 MMPP（Markov-modulated Poisson process）與 Whitt 的 QNA（1983）；要誠實，直接 Lindley 重放 trace——本章推演 1(b) 會讓你親手看到：同樣的邊際分布、不同的順序、四倍的等待。

最後一層誠實：就算一切 i.i.d.、輕尾、ρ 中等，兩組前兩階矩相同、形狀不同的分布也會給出不同等待——VUT 對它們一視同仁，誤差幾十 % 是常態、量級通常可信。正確定位：**羅盤與比較器**（兩個設計哪個好、先拉哪根槓桿、數量級多少），不是**精算器**。要承諾數字，去 ch19。

## G/G/c：Allen–Cunneen 式擴展

多伺服器版本沒有新理論，只有同一招校準的再利用：拿 M/M/c 的精確等待（ch09 的 Erlang-C）當底，乘上同一個變異性因子——通行名稱掛在 Allen 與 Cunneen 名下（教科書與工具圈慣稱；原始出處難考，未驗證到第一手文獻）：

```text
E[T_Q] ≈ ErlangC(c, a)/(cμ − λ) · (C²ₐ + C²ₛ)/2
         ────────┬──────────────   ──────┬──────
          M/M/c 的精確等待（ch09）     同一個 V 因子
```

健全性檢查照舊：c = 1 時 ErlangC = ρ，底數變 ρE[S]/(1−ρ)，還原 Kingman；C²ₐ = C²ₛ = 1 還原 M/M/c 精確值。基準小算：ch09 的 c = 4、λ = 32/s、μ = 10/s，E[T_Q]^M/M/4 ≈ 74.6 ms；接上本章的 C²ₐ = 2、C²ₛ = 1，× 1.5 ≈ **112 ms**，E[T] ≈ 212 ms。

地位要誠實標：它比 c = 1 的 Kingman 更沒有理論地位——連 heavy-traffic 那層保證都更弱（多機極限是另一套理論，ch09 提過的 Halfin–Whitt 一帶而過、本書不展開），c = 1 的三個薄弱帶原樣繼承且更糟。用法不變：羅盤，不是精算器。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦 |
|---|---|---|
| 拿 VUT 簽 SLA | 兩個特例之外它永遠有誤差，幾十 % 是常態；分布形狀與相關性都不在模型裡 | 用它比較設計、選槓桿、估量級；要承諾數字用模擬（ch19）或上界 |
| 固定速率壓測餵假 C²ₐ | 固定間隔 ⇒ C²ₐ ≈ 0：壓測量到 0.2 s 的系統，真流量（C²ₐ = 2）下是 0.6 s——低估 3 倍 | 壓測重放真實 trace；至少按量測到的 C²ₐ 生成到達流 |
| C²ₐ 漏掉時間相關 | cron／retry／on-off 流可以 C²ₐ ≈ 1 但等待爆掉，低估可達數量級 | 按 1 秒與 1 分鐘兩種桶分別算到達數的變異對均值比：Poisson 兩者皆 ≈ 1；隨桶變大而上升 ⇒ 相關性在場，改 trace 重放或 MMPP |
| 重尾下的 C² 是雜訊 | α ≤ 2 時 C² 無定義；樣本估計值隨樣本量一路漂，永遠偏低 | 畫 running C² 曲線，不收斂就停用兩矩近似（ch15） |
| 低 ρ 區硬讀近似值 | 相對誤差大；C²ₐ < 1 時偏高約 20% 起跳 | 低 ρ 時等待本來就小，別摳近似的小數點；要準用 KLB 修正或模擬 |
| ρ ≥ 1 或非穩態 | U 因子變負或發散，公式無意義；尖峰動態它也不管 | 先驗 ρ < 1、量測窗近穩態；過載見 ch10、動態見 ch20 |

自我察覺最值錢的一條是第三行：**C²ₐ 對統計視窗的依賴**。i.i.d. 的世界裡，到達數的變異對均值比不隨桶的大小變；它隨桶變大而上升，就是時間相關性的指紋——這時 VUT 給的數字是樂觀的下限，別當預測用。

## 紙上推演

**推演 1 [15 分鐘] ★ — Lindley 手追 6 個 job**
等距到達、間隔 100 ms（固定速率壓測的世界）：Aₙ₊₁ ≡ 100。
(a) S = [150, 150, 50, 50, 50, 50] ms：手追 W₁…W₆，算總等待。
(b) 同一袋服務時間換個順序：S = [50, 50, 50, 50, 150, 150]，重追。兩袋的 E[S] 與 C²ₛ 完全相同，總等待差多少？這對「C² 抓不到什麼」說了什麼？

**推演 2 [15 分鐘] ★★ — Kingman 應用兩題**
(a) 影音轉檔服務量測：λ = 5/s、E[S] = 150 ms、C²ₐ = 4（上傳高度叢發）、C²ₛ = 3（檔案大小混雜）。估 E[T_Q]、E[T]、E[N_Q]；三根槓桿（C²ₐ→1、C²ₛ→1、λ 砍到 ρ = 0.7）各值多少，先拉哪根？
(b) 三個場景判斷 VUT 可不可用、不可用為什麼：①ρ = 0.3、C²ₐ = 1.2、C²ₛ = 0.8 的內部 RPC；②服務時間 Pareto α = 1.8 的物件儲存；③量測 C²ₐ = 1.1、但到達全擠在整點前後 2 秒的報表服務。

**推演 3 [10 分鐘] ★★ — 固定間隔壓測低估多少**
production 量測：C²ₐ = 2、C²ₛ = 1、ρ = 0.8、E[S] = 100 ms。壓測平台以完全固定的間隔發請求（C²ₐ = 0），同 λ 同 μ。用 VUT 算兩個世界的 E[T_Q] 與低估倍數；再答：壓測改用 Poisson 發生器（C²ₐ = 1），低估剩多少？

**推演 4 [15 分鐘] ★★★ — 上界推導重講**
闔上書，重推 E[T_Q] ≤ λ(σ²ₐ + σ²ₛ)/(2(1−ρ))。要求：說出 Yₙ 的物理意義；指出全程唯一的不等號、丟掉的量；解釋為什麼 ρ → 1 時上界變緊、ρ 低時變鬆。

### 推演解答

**推演 1。** (a) 逐步：W₁ = 0；W₂ = max(0, 0+150−100) = 50；W₃ = max(0, 50+150−100) = 100；W₄ = max(0, 100+50−100) = 50；W₅ = max(0, 50+50−100) = 0；W₆ = max(0, 0+50−100) = 0。總等待 **200 ms**。兩支長工作開路，賒的帳由後面三位分期償還。
(b) W = [0, 0, 0, 0, 0, 50]，總等待 **50 ms**——同一袋工作、同樣的到達，順序一換差 4 倍。長工作在前，賒帳給所有後繼者；長工作在後，只賒給最後一位，而前面短工作墊出的空檔被 max(0,·) 沒收也無妨——本來就沒人在等。教訓：E[S] 與 C²ₛ 是**邊際**統計量，對順序（序列相關）全盲；Kingman 對 (a)(b) 給出同一個答案。這就是「相關性誤差」的最小可重現案例，也是壓測要整段重放 trace、不要打亂順序重抽樣的理由。

**推演 2。** (a) ρ = 5×0.15 = 0.75 ⇒ U = 3；V = (4+3)/2 = 3.5；T = 0.15 s。E[T_Q] ≈ 3×3.5×0.15 = **1.58 s**、E[T] ≈ 1.73 s、E[N_Q] = 5×1.575 = 7.88。槓桿：C²ₐ→1 ⇒ V = 2 ⇒ 0.9 s（−43%）；C²ₛ→1 ⇒ V = 2.5 ⇒ 1.13 s（−29%）；λ→4.67（ρ = 0.7）⇒ U = 2.33 ⇒ 1.23 s（−22%）。先拉 C²ₐ：最大的單一數字，而且上傳叢發常有軟體解（pacing、前置佇列）；砍流量最貴又最沒效。附註：這些百分比是 VUT 內部的比較——近似誤差對三個選項大致同方向，所以**排序**比**數值**可信。
(b) ①可用但讀對身分：U = 0.43、V = 1.0，E[T_Q] ≈ 0.43·E[S]——絕對值小、相對誤差可能幾十 %，當量級估計。②不可用：α = 1.8 ⇒ Var(S) = ∞ ⇒ C²ₛ 不存在，樣本算出的值是會漂的雜訊；換 ch15 的工具。③形式上可用、實際上會騙人：C²ₐ = 1.1 看似無辜，但「整點前後 2 秒」是極端時間相關，邊際分布與真實機制脫鉤——Kingman 嚴重低估；老實做法是按整點叢發用 Lindley 手算或模擬。

**推演 3。** 真實：0.6 s（worked example）。固定間隔：V = (0+1)/2 = 0.5 ⇒ 4×0.5×0.1 = **0.2 s**，低估 **3 倍**。Poisson 發生器：V = 1 ⇒ 0.4 s，仍低估 1.5 倍——把到達「修好」成 Poisson 只還了一半的債，剩下的藏在 C²ₐ = 2 與 1 的差距裡。補一刀：固定間隔世界的精確值其實比 0.2 還低（≈ 0.17 s，GI/M/1 理論，本書不推導），對照 production 的真實差距比 3 倍更難看。寫進壓測 checklist：**到達流的形狀是測試設計的一部分，不是工具的預設值**。

**推演 4。** 主鏈：Yₙ = max(0, −(Wₙ+Uₙ)) 是這一步被沒收的閒置；恆等式 (Wₙ+Uₙ)Yₙ = −Yₙ² 讓平方展開塌掉；穩態二階矩相等消去 E[W²]；一次方平衡 E[Y] = |E[U]| 是恆等式；唯一的不等號是 E[Y²] ≥ (E[Y])²，丟掉 Var(Y)——閒置塊的變異數。ρ → 1：閒置幾乎絕跡、Var(Y) → 0，上界貼住真值（與 heavy-traffic 極限同形）；ρ 低：閒置常見且大小參差，Var(Y) 肥大——M/M/1、ρ = 0.8 的對照：上界 0.513 s vs 精確 0.4 s（+28%）。常見錯路：把 E[Y] = |E[U]| 那步當成不等號——它是期望平衡的恆等式；整個推導的不等號只有一個。
