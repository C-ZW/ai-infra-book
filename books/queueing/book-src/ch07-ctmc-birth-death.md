## 從你已知的出發

打開你的監控面板，看那條 active connections 曲線：3,200 條 WebSocket 連線不是一次到位的，它是一條每次只動 ±1 的鋸齒——有人連上（+1）、有人斷線（−1）。autoscaling 也一樣：instance 數量 scale-up 一台、scale-down 一台，永遠在一條整數軸上左右移動。這類「狀態排成一直線、每步只能去隔壁」的系統，本章給它名字：生滅過程（birth-death process）。生＝+1、滅＝−1，如此而已。

但先得處理一個更基本的尷尬。ch06 的 Markov chain 按 tick 走，而你的系統沒有 tick——訊息不會「每 100 ms 整點檢查一次要不要到達」，它想來就來。ch06 那種「升一級機率 0.3」的講法，放到連續時間會變成「多小的時間窗算一步？」這種沒有好答案的問題。出路你其實已經拿在手上：ch04 的指數碼錶。把狀態機的每條箭頭從「機率」換成「速率」——每條出邊掛一顆獨立的指數碼錶，先響的那顆決定跳去哪。ch04 結尾說競賽引理是「CTMC 的引擎」，本章就是交車的日子：rate diagram（速率圖）不過是你畫了十年的狀態轉移圖，把箭頭上的標籤從機率換成每秒幾次。

值得先把投資報酬講清楚：接下來五章的所有模型——M/M/1（ch08）、M/M/c 與 Erlang 公式（ch09）、有限容量的 M/M/1/K（ch10）——**全部是同一張生滅圖換參數**。M/M/1 是 λₙ、μₙ 全為常數；M/M/c 是 μₙ = min(n, c)·μ；M/M/1/K 是把鏈鋸短。本章推一次通解，後面五章每章只需要代入自己的 λₙ、μₙ 然後算級數。學一次、用五章，這是全書槓桿最高的一筆推導。

本章的路線：先組裝 CTMC 並把 ch06 的 balance equations 升級到連續時間；再切到生滅過程，證明「相鄰 cut 流量守恆」在一維結構下與 global balance 等價；用它推出全體生滅過程的通解；最後回答存在性——正規化級數收斂與否，就是你的系統有沒有穩態的分界線。

## CTMC：狀態機掛上指數碼錶

**定義（建構式）**：一個連續時間 Markov chain 由狀態空間 S 與一組速率 q(i→j) ≥ 0（i ≠ j）組成。過程進入狀態 i 時，每條出邊 j 各啟動一顆**獨立**的指數碼錶 ~ Exp(q(i→j))，先響者勝：過程跳到勝者指向的狀態，一切重新開始。

ch04 的三條競賽引理立刻把這個定義拆成可計算的零件。記 qᵢ = Σⱼ q(i→j) 為狀態 i 的總出速率：

```text
停留時間：min 引理 ⇒ 在 i 停留 ~ Exp(qᵢ)，平均 1/qᵢ
跳去哪裡：勝者引理 ⇒ P(跳到 j) = q(i→j)/qᵢ
獨立性  ：跳去哪裡與停了多久互相獨立
        ⇒ CTMC = 一條「只記路徑」的離散鏈（jump chain）＋每站各自獨立的指數停留
```

第三條是整個結構成立的支點：沒有它，「停得久」會洩漏「接下來去哪」的資訊，狀態就不再吃掉歷史。而停留時間非指數不可，不是品味問題——Markov 性質要求「已經停了多久」不影響未來，這正是無記憶性，而 ch04 證過指數是連續分布中唯一的無記憶分布。反過來說，任何時間齊次、「給定現在、過去無資訊」的連續時間跳躍過程，停留時間必為指數、跳轉必由速率決定（工程師的嚴謹：敘述；完整證明見延伸閱讀 Norris 第 2 章）。

速率的單位是「每單位時間幾次」，不是機率。q(i→j) = 3/s 的白話：待在 i 的期間，往 j 的這顆碼錶平均每 1/3 秒響一次；在一小段 δ 內它響的機率是 q(i→j)·δ + o(δ)（ch05 的 o(h) 語言）。速率可以大於 1，機率不行——這個單位差是陷阱表第 1 條的主角。

拿一個具體的小機器練讀圖。一台推播 worker 有三個狀態：idle（等工作）、busy（處理中）、crashed（掛了等重啟）。到達速率 8/s、服務速率 10/s、處理中可能當掉（速率 0.02/s，數字灌水過好讓 π 看得見）、重啟速率 0.5/s：

```text
        ┌─────────── 10 (done) ───────────┐
        ▼                                 │
     (idle) ────── 8 (arrival) ──────► (busy)
        ▲                                 │
        │                                 │ 0.02 (crash)
        │                                 ▼
        └──── 0.5 (restart) ───────── (crashed)
```

競賽讀法：busy 有兩條出邊，停留 ~ Exp(10 + 0.02)、平均約 99.8 ms；每次離開時 P(完成) = 10/10.02、P(當掉) = 0.02/10.02 ≈ 0.002；而且「這單做得特別久」完全不增加當掉的嫌疑（引理三）。注意「出邊速率可以相加」的前提就是競賽引理的前提：各邊獨立、各自指數——這是建模假設，不是天上掉下來的。

順手把記帳工具備好：把速率排成**生成矩陣（generator/Q matrix）**——非對角元 Q[i][j] = q(i→j)、對角元 Q[i][i] = −qᵢ，每列加總為 0（對照 DTMC 的 P 每列加總為 1）。它在本書只有一個用途：下一節的 balance equations 用矩陣寫就是 πQ = 0。瞬時分布的演化由 Kolmogorov 微分方程 dP(t)/dt = P(t)Q 描述，本書不解它（穩態邊界一節會交代為什麼敢不解）。

## Global balance：流量守恆升級到連續時間

CTMC 的平穩分布 π 跟 ch06 同一個靈魂：πⱼ 是長期下來待在狀態 j 的**時間佔比**。它滿足的方程，用 ch06 練好的「機率流量守恆」直接寫——把 πᵢ·q(i→j) 讀成穩態下從 i 流向 j 的機率流量：

```text
global balance：對每個狀態 j，
    πⱼ·qⱼ = Σ_{i≠j} πᵢ·q(i→j)        （流出 j 的總流量 = 流入 j 的總流量）
加上正規化 Σⱼ πⱼ = 1
```

為什麼是這組方程（工程師的嚴謹：每步能口頭說出理由）：取一段長度 T 的觀察窗。從 j 跳到 k 的次數 ≈（窗內待在 j 的總時間）× q(j→k) ≈ T·πⱼ·q(j→k)——前一個 ≈ 是遍歷定理把時間平均換成 πⱼ（ch06 發的執照），後一個是「待在 j 的每一秒，往 k 的碼錶以速率 q(j→k) 響」。而任何一條軌跡上，「進入 j 的次數」與「離開 j 的次數」最多差 1——每次離開前必先進入。兩邊除以 T、讓 T → ∞，差 1 被攤平，就得到流入＝流出。比 ch06 還乾淨：連續時間沒有 self-loop、沒有 P(j→j) 要扣。

ch06 的 cut 守恆原封不動地升級：把任何狀態集合 A 的逐狀態收支加總，內部互流對銷，得「流出 A 的總流量 = 流入 A 的總流量」。這把刀下一節要連砍無限刀。

連續時間還白送一份禮：**沒有週期性問題**。DTMC 的 ping-pong 鏈分布永遠振盪（ch06 陷阱），但 CTMC 的停留時間是連續隨機變數，不可能鎖相——不可約且正常返的 CTMC，從任何起點，時刻 t 的分布都收斂到 π（敘述；證明見 Norris 第 3 章）。

解 global balance 的手感跟 ch06 一樣是國中代數。拿 worker 三狀態機驗收（完整計算）：

```text
crashed 收支：0.02·π_B = 0.5·π_C          ⇒ π_C = 0.04·π_B
idle 收支   ：8·π_I = 10·π_B + 0.5·π_C    ⇒ 8·π_I = 10.02·π_B
                                          ⇒ π_B = 0.798·π_I
正規化      ：π_I·(1 + 0.798 + 0.0319) = 1
⇒ π_I = 0.546、π_B = 0.436、π_C = 0.0174
```

長期約 1.74% 的時間掛著等重啟。複核 busy 的收支（解的時候沒用過這條，最佳的免費檢查）：流入 = 8·0.546 = 4.37，流出 = 10.02·0.436 = 4.37 ✓。

一個馬上會咬人的區別先押在這裡：π 是**時間**佔比，不是**訪問次數**佔比。jump chain 只記「跳過哪些站」，它的平穩分布跟 π 差一個停留時間的加權（1/qᵢ）。模擬 CTMC 時數「到過幾次」估 π，會把停留短的狀態灌水——陷阱表第 2 條有數字。

## 生滅過程：一維世界的 local balance

**定義**：狀態空間 {0, 1, 2, …}，只允許 ±1 的轉移——狀態 n 的出生速率 λₙ（n → n+1）、死亡速率 μₙ（n → n−1，n ≥ 1）。這是本章主視覺，之後五章的所有模型都是往這張圖填不同的 λₙ、μₙ：

```text
        λ₀          λ₁          λ₂          λ₃
     ──────►     ──────►     ──────►     ──────►
  (0)        (1)         (2)         (3)         …
     ◄──────     ◄──────     ◄──────     ◄──────
        μ₁          μ₂          μ₃          μ₄
```

佇列翻譯：n＝系統內人數，出生＝到達、死亡＝完成。CTMC 建構在每個狀態 n ≥ 1 給兩顆碼錶賽跑：Exp(λₙ) 的到達 vs Exp(μₙ) 的完成；停留 ~ Exp(λₙ + μₙ)，下一步往右的機率 λₙ/(λₙ+μₙ)。本章假設所有 λₙ > 0、μₙ > 0（不可約：哪裡都到得了）。速率掛下標不是排場，是建模的全部自由度：到達會看臉色（rate limiting 把高積壓時的 λₙ 壓低）、服務會隨人數變（多開 worker 讓 μₙ 變大、資源競爭讓它變小），全都收在這兩串數字裡。

在 n 與 n+1 之間下一刀。這一刀只切過**兩條邊**——λₙ 往右、μₙ₊₁ 往左，cut 守恆直接給：

```text
local balance：πₙ·λₙ = πₙ₊₁·μₙ₊₁        （每對相鄰狀態，右流 = 左流）
```

這叫 local balance（逐邊平衡），比 global balance（逐狀態平衡）強得多——它說的不只是每個狀態收支平衡，而是**每一條邊上的兩個方向各自抵銷**。一般的 chain 沒有這種好事，但一維結構免費奉送。兩個方向都值得親手走一遍。

**local ⇒ global**：狀態 n 的 global balance 恰好是左右兩條 cut 方程相加——(λₙ+μₙ)πₙ = λₙ₋₁πₙ₋₁ + μₙ₊₁πₙ₊₁ 的右邊第一項配 cut(n−1, n)、第二項配 cut(n, n+1)。一行完成。

**global ⇒ local**（完整推導，這個方向才是稀有的）：記 Fₙ ≡ πₙλₙ − πₙ₊₁μₙ₊₁ 為 cut(n, n+1) 的淨流量（往右減往左）：

```text
狀態 0 的 global balance：π₀λ₀ = π₁μ₁            即 F₀ = 0   ← 0 只有一個鄰居
狀態 n ≥ 1 的 global balance：
    (λₙ + μₙ)·πₙ = λₙ₋₁·πₙ₋₁ + μₙ₊₁·πₙ₊₁
移項分組：
    πₙλₙ − πₙ₊₁μₙ₊₁ = πₙ₋₁λₙ₋₁ − πₙμₙ           即 Fₙ = Fₙ₋₁
歸納：Fₙ = Fₙ₋₁ = … = F₀ = 0                     ∎
```

直覺只有一句：cut(n, n+1) 是狀態 ≤ n 與狀態 ≥ n+1 之間**唯一的橋**，淨流量要嘛沿著鏈一路恆定、要嘛是零；而最左端是牆（狀態 0 沒有左鄰），把恆定值釘死在零。一維結構就是這樣把 global 壓成 local 的。

對照組看清楚這份禮有多特殊。三個狀態排成單向圈：0 → 1 → 2 → 0，速率全為 1：

```text
 (0) ──1──► (1) ──1──► (2)
  ▲                     │
  └─────────1───────────┘
```

對稱性給 π = (1/3, 1/3, 1/3)，global balance 逐狀態驗證成立（每站流入 1/3、流出 1/3）。但邊 0→1 上：右流 π₀·1 = 1/3，回流 π₁·q(1→0) = 0——local balance 死透了。機率在圈上**循環**，淨流量不為零；穩態不等於沒有流動，只等於水位不變。什麼樣的 chain 才享有逐邊平衡，是 ch11 可逆性的主題（見 ch11），這裡只需要記住：在生滅過程裡它是定理，在別處它是要證明的奢侈品。

## 通解：πₙ = π₀·Π(λᵢ/μᵢ₊₁)

local balance 是一階遞推，整條鏈像拉鍊一樣解開（完整推導）：

```text
cut(n, n+1)：πₙ₊₁ = (λₙ/μₙ₊₁)·πₙ
逐層展開：
π₁ = (λ₀/μ₁)·π₀
π₂ = (λ₁/μ₂)·π₁ = (λ₁λ₀)/(μ₂μ₁)·π₀
…
πₙ = π₀·Π_{i=0}^{n−1} (λᵢ/μᵢ₊₁)        ← 通解：n 條 cut，一刀一個因子
正規化：
1 = Σₙ πₙ = π₀·S，  S ≡ Σ_{n=0}^{∞} Π_{i=0}^{n−1} (λᵢ/μᵢ₊₁)   （n=0 項＝空乘積＝1）
⇒ π₀ = 1/S
```

讀法：πₙ 是一串「上行速率除以下一級下行速率」的連乘——每爬一級，付一個 λᵢ/μᵢ₊₁ 的係數。係數 < 1 的階梯越爬越難，係數 > 1 的階梯越爬越順。ch06 紙上推演第 2 題那條「幾何遞推」就是這個結構的離散預演；ch08 的 M/M/1 是它最出名的特例（全部係數相同），這裡不展開（見 ch08）。

**適用條件**，一個都不能少：①生滅結構（只有 ±1 轉移）——少了它沒有「唯一的橋」，local balance 不成立；②所有速率為正（不可約）；③穩態存在——這就是下一節，因為 S 可能是 ∞。

## Ergodicity：級數收斂，穩態才存在

> **定理（生滅過程的 ergodicity；工程師的嚴謹：敘述＋直覺，完整證明指路延伸閱讀）**：設生滅過程不可約（λₙ > 0、μₙ₊₁ > 0 對所有 n ≥ 0）且非爆炸（見下）。若 S < ∞，則平穩分布存在、唯一、由通解給出，且從任何起點，時刻 t 的分布與時間平均都收斂到 π（正常返／ergodic）。若 S = ∞，平穩分布**不存在**。

S = ∞ 的物理意義值得正面盯著看。它不是「系統很滿」——是**任何有限狀態的長期時間佔比都是零**。想把通解正規化，每個 πₙ 都會被除成 0：機率一路向右逃逸，對任何固定的 n，P(N(t) = n) → 0。佇列要嘛一去不回頭地長大（transient，λ 系統性壓過 μ 時的行為），要嘛在臨界點上反覆回到 0 卻平均要等無限久（ch06 預告過的零常返，λ 與 μ 勢均力敵的病）。區分這兩種病需要第二條級數（Karlin–McGregor 判準，本書不展開，見延伸閱讀）；對工程師，第一條就夠用——**S 收斂與否，就是你的系統有沒有穩態的審判**，ch01 賒的「穩定性邊界」這裡兌現一半，另一半（M/M/1 的 ρ < 1）下一章開庭。

最快的體感：速率全為常數 λ、μ 時，S = Σ(λ/μ)ⁿ 是幾何級數（ch03 的老朋友），λ < μ 收斂、λ ≥ μ 發散——「服務追不上到達就沒有穩態」第一次從經驗變成定理。但通解的威力在於 λₙ、μₙ 可以隨 n 變，而級數判準照樣裁決，worked example 與紙上推演會給出三種命運迥異的設計。

兩筆技術誠實帳。**爆炸（explosion）**：若 λₙ 隨 n 長得太兇（例如 λₙ = 2ⁿ，純出生時總耗時期望 Σ1/λₙ = 2 < ∞），過程可以在有限時間內跳無限多次、衝出所有狀態——「非爆炸」是定理的前提之一。判準與證明見 Norris 第 2 章；本書出現的模型 λₙ 不是有界就是遞減，安全。**transient 行為**：本章與本書的所有答案都是 t → ∞ 的穩態；「尖峰過後多久排空」「多久第一次衝到容量上限」是 Kolmogorov 微分方程、首達時間與 fluid 近似的領域（uniformization 是把 CTMC 化回離散鏈做數值解的標準工具，一句話帶過）——本書主攻穩態，動態世界的地圖在 ch21（見 ch21）。這個取捨對工程不算太委屈：SLA、容量規劃、staffing 問的都是長期分布，穩態正面回答；但事故復原時間、尖峰的暫態高峰是 transient 問題，本書的公式對它們**保持沉默**，硬套會把「平衡後的世界」誤當「過渡中的世界」——知道工具的邊界在哪，跟會用工具一樣重要。

## Worked example：admission control 的生滅模型

把通解開上路。場景：單一推播 worker，μ = 10/s（E[S] = 100 ms）、尖峰 λ = 8/s（基準數字）。上游 API 做機率式 load shedding：系統內已有 n 件工作時，新到達只以機率 1/(n+1) 被收下，其餘直接拒絕。被收下的到達流是 Poisson 流的 thinning（ch05），所以這是生滅過程：

```text
λₙ = λ/(n+1) = 8/(n+1)、μₙ = μ = 10：

        8           4          8/3           2
     ──────►     ──────►     ──────►     ──────►
  (0)        (1)         (2)         (3)         …
     ◄──────     ◄──────     ◄──────     ◄──────
       10          10          10          10
```

套通解，記 a = λ/μ = 0.8（offered load，記號表的老面孔）：

```text
Π_{i=0}^{n−1} λᵢ/μᵢ₊₁ = Π_{i=0}^{n−1} λ/((i+1)·μ) = aⁿ/n!     ← 分母收集出 n!
S = Σ_{n≥0} aⁿ/n! = e^a                                       ← 指數級數，永遠收斂
π₀ = e^(−a)，  πₙ = e^(−a)·aⁿ/n!                               ← Poisson(a) 分布！
```

n! 把幾何級數馴化成指數級數——admission control 把佇列長度的分布從幾何尾（ch08）壓成階乘衰減的 Poisson 尾。代 a = 0.8（e^(−0.8) ≈ 0.449，自算複核：e^0.8 ≈ 2.2255）：

| n | πₙ | 累積 |
|---|---|---|
| 0 | 0.449 | 0.449 |
| 1 | 0.359 | 0.809 |
| 2 | 0.144 | 0.953 |
| 3 | 0.0383 | 0.991 |
| 4 | 0.00767 | 0.999 |
| 5 | 0.00123 | 1.000 |

P(N ≥ 3) = 1 − 0.953 = 0.0474；同樣 λ、μ 但不擋人的 M/M/1 給 πₙ = 0.2·0.8ⁿ（ch08 推導），P(N ≥ 3) = 0.8³ = 0.512——差 10.8 倍。平均更乾脆：E[N] = Σ n·e^(−a)aⁿ/n! = a·Σ e^(−a)aⁿ⁻¹/(n−1)! = a = 0.8，對照 M/M/1 的 E[N] = 4（ch03 已算過）。

但帳要記兩本。算**吞吐**——而且故意用兩種算法，互相當審計：

```text
進帳（被接受的到達率）：
λ_eff = Σₙ πₙ·λₙ = Σₙ e^(−a)·(aⁿ/n!)·λ/(n+1)
      = (λ/a)·e^(−a)·Σₙ a^(n+1)/(n+1)!        ← (aⁿ/n!)/(n+1) = a^(n+1)/(a·(n+1)!)
      = μ·e^(−a)·(e^a − 1) = μ·(1 − e^(−a))    ← λ/a = μ；級數差一個 n=0 項
出帳（完成離開率）：
      Σ_{n≥1} πₙ·μ = μ·(1 − π₀) = μ·(1 − e^(−a))   ✓ 兩本帳相等
```

兩條路必須相等不是巧合——穩態下進出系統的流量守恆，正是 cut(0, 1) 以上整段的總收支。數值：λ_eff = 10·(1 − 0.449) = 5.51/s，而上游開價 8/s——**31.2% 的工作被擋在門外**。被接受的工作倒是過得很爽：Little's Law（ch02，對被接受的流用 λ_eff）給 E[T] = E[N]/λ_eff = 0.8/5.51 = 0.145 s，對照 M/M/1 的 0.5 s。

**自我複核：把解代回 balance equations**（寫完平穩分布必做的儀式）。先數值抽查：

```text
cut(0, 1)：λ₀π₀ = 8·0.449 = 3.59；μ₁π₁ = 10·0.359 = 3.59          ✓
狀態 1 的 global balance：
  流出 = (λ₁ + μ)·π₁ = (4 + 10)·0.359 = 5.03
  流入 = λ₀π₀ + μπ₂ = 3.59 + 1.44 = 5.03                          ✓
```

再代數總驗（n ≥ 1 全體狀態一次過，用 λ = μa）：

```text
流出：(λ/(n+1) + μ)·aⁿ/n!
流入：(λ/n)·aⁿ⁻¹/(n−1)! + μ·aⁿ⁺¹/(n+1)!
逐項配對：
  (λ/n)·aⁿ⁻¹/(n−1)! = λ·aⁿ⁻¹/n! = μ·aⁿ/n!        ← 配中流出第二項（cut(n−1, n)）
  μ·aⁿ⁺¹/(n+1)! = (μa)·aⁿ/((n+1)·n!) = (λ/(n+1))·aⁿ/n!  ← 配中流出第一項（cut(n, n+1)）✓
```

配對不是隨便配的：兩組恰好就是左右兩條 local balance——global 拆成 local 的那個結構，在代數裡原樣現形。

最後一個值得正面寫出來的事實：S = e^a **對任何 a 都收斂**。把 λ 拉到 20/s（a = 2，徹底過載），系統照樣 ergodic：E[N] = 2、λ_eff = μ(1 − e^(−2)) = 8.65/s、丟掉 56.8%。admission control 買到的是**無條件的穩定性**，帳單從延遲那欄整頁搬到 goodput 那欄——這就是 ch10「丟棄／等待」三角的開場白（見 ch10）。誠實聲明：1/(n+1) 這個接受機率（文獻通稱 discouraged arrivals）狠得不像話——n = 1 就擋掉一半；真實系統多半用硬閾值（佇列滿了才拒收），那是 M/M/1/K 的世界，ch10 處理。本例的價值在於它把「λₙ 隨 n 遞減 ⇒ 分布尾巴被壓扁、穩定性無條件化」這條因果鏈用閉式解攤開給你看。

## 陷阱與防禦

| 陷阱 | 它怎麼騙你 | 防禦與察覺 |
|---|---|---|
| 把機率標上 rate diagram | 「跳轉機率 0.5」放上速率圖，單位整個錯——速率可以是 3/s、可以大於 1，乘上 πₙ 才是流量；機率乘 πₙ 是無意義的數 | 每個標籤過單位檢查：1/時間才能上圖。要機率？P(下一步往右) = λₙ/(λₙ+μₙ)，從速率算出來 |
| 數訪問次數估 π | jump chain 的訪問佔比 ≠ 時間佔比。兩狀態 A⇄B、q(A→B)=1、q(B→A)=100：跳轉嚴格輪流、訪問各 50%，但時間 π_A = (1/1)/(1/1+1/100) ≈ 0.990——差 49 個百分點 | 模擬與 log 分析一律按停留時間加權；口訣「π 是碼錶量出來的，不是計數器」 |
| 非指數停留硬當 CTMC | 固定 30 s 的重啟倒數、固定 cooldown，停留時間不是指數；速率圖給的 π 平均對、依賴停留分布的結論全錯（ch06 陷阱 1 的連續版） | 擴狀態（把倒數拆成多段）或用相位逼近（phase-type，ch21 地圖一句話）；先問每條邊「你真的是一顆指數碼錶嗎」 |
| 通解寫完不驗級數 | πₙ = π₀·Π(…) 形式上永遠寫得出來；S = ∞ 時硬正規化得到的「π」是垃圾——穩態根本不存在，模擬會看到佇列單調竄高、與「解」毫無關係 | 寫完通解先判 S 收斂再談其他；徵兆：模擬越跑平均越大、不收斂（ch19 的 warmup 救不了不存在的穩態） |
| 把 local balance 帶出一維 | 「穩態＝每條邊平衡」在一般 chain 是錯的：單向三圈 π 均勻、global 全過，但每條邊淨流 1/3——機率在循環 | local balance 是結構的贈品：一維（本章）或可逆（ch11）才有；用之前先指出你的「唯一的橋」在哪 |

## 紙上推演

**1. [15 分鐘 ★★] 用 cut 解一條有限生滅鏈。**worker 的積壓分四級 {0, 1, 2, 3}（3 = 飽和）。上行速率 λ₀ = λ₁ = λ₂ = 6/s；下行速率 μ₁ = 4、μ₂ = 8、μ₃ = 8/s（積壓變高時你開了 turbo）。(a) 用相鄰 cut 解出 π。(b) 長期有多少比例的時間處於飽和？(c) 用一條你解題時沒用過的方程複核。

**2. [15 分鐘 ★★] 三個生滅過程，判 ergodicity。**對每一個寫出 S 的通項、判收斂、並用一句話說出物理意義：(a) λₙ = 12、μₙ = 10（固定超載）；(b) λₙ = 8、μₙ = 10n（每件工作一到就有自己的 worker）；(c) λₙ = 2、μₙ = 10/n（工作越積、服務越慢——thrashing／GC 壓力的卡通版）。

**3. [10 分鐘 ★★] 口頭重講：local balance 為什麼在生滅過程成立。**不看書，從 global balance 出發講到 Fₙ = 0，要講出「唯一的橋」與「左端是牆」各扮演什麼角色，並用單向三圈說明少了一維結構會發生什麼。

### 推演解答

**1.** (a) 三條 cut：6π₀ = 4π₁ ⇒ π₁ = 1.5π₀；6π₁ = 8π₂ ⇒ π₂ = 0.75π₁ = 1.125π₀；6π₂ = 8π₃ ⇒ π₃ = 0.84375π₀。正規化：π₀(1 + 1.5 + 1.125 + 0.84375) = 1 ⇒ π₀ = 32/143 ≈ 0.224，π₁ = 48/143 ≈ 0.336，π₂ = 36/143 ≈ 0.252，π₃ = 27/143 ≈ 0.189。(b) π₃ ≈ 18.9%。(c) 抽狀態 2 的 global balance（解題只用過 cut）：流出 = (6+8)·0.252 = 3.52；流入 = 6·0.336 + 8·0.189 = 2.01 + 1.51 = 3.52 ✓。常見錯路：把四條 global balance 全列出來硬解四元聯立——能解，但你會錯過「一維結構把它壓成三條一階遞推」這個本章的整個重點。

**2.** (a) 通項 (12/10)ⁿ = 1.2ⁿ，幾何發散 ⇒ 非 ergodic。物理：到達系統性壓過服務，佇列以平均 λ−μ = 2/s 的斜率跑向無限（transient）。(b) 通項 Π 8/(10(i+1)) = 0.8ⁿ/n!，S = e^0.8 < ∞ ⇒ ergodic——而且把 8 換成任何 λ 都收斂：服務容量跟著佇列線性長，永遠追得上（這是 ch09 的 M/M/∞，先偷看一眼）。(c) 通項 Π 2(i+1)/10 = n!/5ⁿ，階乘輾壓幾何 ⇒ 發散 ⇒ 非 ergodic，**儘管 λ = 2 遠小於 μ₁ = 10**。物理：係數 λ/μₙ₊₁ = (n+1)/5 在 n = 4 跨過 1——積壓一旦衝過 4，每多一件事服務就更慢、漂移轉向外側，回不來了。服務隨擁塞劣化的系統沒有「夠小的 λ」這回事，這是 ch20 retry storm 與 metastable failure 的第一張底片。

**3.** 骨架：①狀態 0 只有一個鄰居，global balance 本身就是 cut(0,1)——左端是牆，釘下 F₀ = 0；②狀態 n 的 global balance 移項後恰是 Fₙ = Fₙ₋₁——淨流量沿鏈恆定；③歸納得全部 Fₙ = 0。「唯一的橋」：cut(n, n+1) 是左右兩半之間僅有的通道，所以「狀態集合 {0..n} 的總收支」直接落在這兩條邊上。單向三圈裡每條 cut 都被兩條以上的邊跨越（去程與回程走不同的路），守恆只約束「總和」、約束不到單邊——淨流量繞圈跑，local balance 無從成立。講不出③的人通常卡在把 F₀ = 0 當成假設——它不是，它是狀態 0 的 global balance 本身。
