# 等待的數學

> 排隊理論——寫給一位資深後端工程師的數學之旅。

## 這本書是什麼

你寫了六年的高併發系統，對「queue 滿了會怎樣」「使用率別超過 80%」「P99 比平均慘」有一肚子直覺。這些直覺大多是對的——但你說不出**為什麼**。這本書把那批直覺一條條變成可推導、可質疑、知道邊界的數學：從 πₙ = (1−ρ)ρⁿ 到 Pollaczek–Khinchine、從 Little's Law 到 Jackson 定理，每個公式你都親手推一遍，並且永遠先問「它在什麼條件下會騙我」。

三個設定，開讀前先知道：

- **數學嚴謹為主**。這不是容量規劃手冊。工程應用是橋接的錨點與點綴（每章開場＋ch20 收網），主菜是推導本身：你要能口頭重講每個關鍵證明，講不清楚就還沒懂。
- **紙上推演＋少量程式**。多數動手是紙和筆：手推公式、手算數值表、手追 Lindley recursion。七章另附 `Python 小實驗`（純標準函式庫、固定種子），讓你親眼看到理論值與模擬值對得上——以及重尾下對不上。
- **你的系統是貫穿範例**。一條 SQS 風格的推播管線（λ=8、μ=10、ρ=0.8）從 ch01 的體感開始，一路被 M/M/1、Erlang、P-K、Kingman、Jackson 反覆解剖，到 ch20 變成一次完整的容量規劃 walkthrough。

## 怎麼讀

每章固定骨架：開場地圖 → **從你已知的出發**（橋接你的後端經驗）→ 核心推導 → **陷阱與防禦**（這個公式在什麼條件下騙你）→ **紙上推演**（2~4 題，附完整解答與預估時間）→ **自我檢核**（口頭自答）→ 延伸閱讀。

推演題請真的拿紙筆做完再看解答；自我檢核用說的，不是用想的。考核過關制：上一章的檢核講不順，先別開下一章。每個公式記得連它的適用條件一起記——附錄 B 的「公式總表」把全書公式與其適用條件並列，沒有條件的公式表是危險品。

## 路線

- **Part I（ch01–02）**：為什麼 ρ→1 一切爆炸；不用機率就能算的鐵律（Little's Law、bottleneck bounds）。
- **Part II（ch03–06）**：機率工具箱、指數分布、Poisson 過程、Markov chain——從「大學看過」磨到能用。
- **Part III（ch07–11）**：生滅過程引擎；M/M/1 全套；M/M/c 與 Erlang 公式；有限容量；可逆性與 Burke 定理。
- **Part IV（ch12–15）**：放掉指數假設——P-K 與變異數的帳單、排程的代價、Kingman 萬用近似、重尾如何毀掉平均值。
- **Part V（ch16–18）**：佇列網路——Jackson 的乘積形式驚奇、封閉網路與 MVA、product form 的邊界與 fork-join。
- **Part VI（ch19–21）**：模擬方法學（讓蒙地卡羅說真話）；把理論帶回機房（retry storm、autoscaling、batching）；全書地圖與下一步。

六條驗收主線（ch21 收官，讀完要能逐條口頭自答）：①為什麼 ρ→1 爆炸、爆炸的形狀；②隨機性（變異數）本身就是延遲來源；③平均值與 tail 是兩回事；④每個公式條件不滿足時怎麼騙你；⑤網路層級的湧現行為；⑥模擬什麼時候比公式誠實。

事實性內容（歷史歸屬、工具版本、業界案例）的查證基準時點是 **2026-06**；維護方式見 `_meta/maintenance.md`。
