# ch05 — Poisson 過程：到達的標準模型

> **本章解決什麼問題**：ch04 給了你指數分布這顆原子；本章把原子串成鏈，建立描述「到達」的標準隨機過程。你會拿到三個等價定義（一個拿來模擬、一個拿來算數、一個拿來建模與驗證）、兩個保結構的運算（疊加與分流）、一個漂亮的結構定理（條件均勻性），以及「為什麼真實世界的到達流這麼常像 Poisson」的理由（Palm–Khintchine）。ch06–07 的 Markov 引擎與 ch08 起的所有 M/M/* 模型，第一個字母 M 指的就是本章——這一章決定你之後信不信得過那個假設。

## 從你已知的出發

你的 WebSocket 通知系統一個月送 ~10M 則訊息。換算一下：以 30 天計，一個月有 2,592,000 秒，平均到達率約 3.86 則/s；尖峰時段則是基準表的 λ = 8 則/s。監控儀表板上的 QPS 是一條每分鐘平均線，但容量問題全部發生在「平均線底下的隨機抖動」裡：

- 尖峰平均 8 則/s，**某一秒湧進 12 則以上**的機率多大？直覺常猜「很罕見」，本章 worked example 會算出 ≈ 0.112——大約每 9 秒就有一秒。
- 你把 30% 的訊息路由到 iOS 推播 shard。那條子流長什麼樣？要重新壓測量測嗎？本章的答案：不用，它就是 Poisson(2.4)，而且跟 Android 子流**獨立**——這句話的條件與證明都在本章。
- 三個上游服務把事件匯進同一條 pipeline。匯流之後的到達流比較平滑、還是比較抖？

按機率路由到 shard，在數學裡叫**分流（thinning）**；多個上游匯流，叫**疊加（superposition）**。你每天在做的兩個工程操作，本章把它們變成定理——並且講清楚定理什麼時候不保佑你（least-loaded 路由、retry 流、批次上游）。

開工前先校準一個常見的視覺直覺：「平均 3.86 則/s」不是「每 0.26 秒準時來一則」。等距是確定性流（D 流）的長相；Poisson 流的長相是**群聚與空檔交錯**——短間隔比你以為的多（指數密度在 0 附近最高，ch04），偶爾又冒出長到讓你懷疑系統掛掉的空檔。盯著 access log 覺得「到達怎麼這麼不均勻」，多半不是異常，是指數間隔的正常臉。

## 計數過程：先把「到達流」變成數學物件

要談「到達」，先固定詞彙。一個**計數過程（counting process）**{N(t), t ≥ 0} 滿足：N(0) = 0；N(t) 取非負整數；t 增加時 N(t) 不減；對 s < t，N(t) − N(s) 是區間 (s, t] 內的到達數。

同一條到達流有三種等價的記法，之後會不停切換視角：

- **計數視角**：N(t)＝到時刻 t 為止累計到達數；
- **時刻視角**：第 i 個到達發生在時刻 τᵢ（τ₁ < τ₂ < …）；
- **間隔視角**：相鄰到達的間隔 Gᵢ = τᵢ − τᵢ₋₁（取 τ₀ = 0）。

```
N(t)
 4 │                              ******
 3 │                    **********
 2 │          **********
 1 │    ******
 0 │****
   └────┴─────┴─────────┴─────────┴────► t
        t1    t2        t3        t4
```

三者攜帶的是同一份資料，橋梁是這條對偶關係：**{N(t) ≥ n} 與 {τₙ ≤ t} 是同一個事件**——「到 t 為止至少來了 n 個」就是「第 n 個在 t 之前到」。等一下證等價性會直接用它。

## Poisson 過程的三個等價定義

Poisson 過程是同一個東西的三張面孔。先把三張臉各自看清楚，再談它們為什麼是同一張。

### 定義一：指數間隔（拿來模擬）

> 間隔 G₁, G₂, … 獨立同分布（i.i.d.）、皆為 Exp(λ)，則 N(t) 是速率 λ 的 Poisson 過程。

這是**建構式**的定義：抽一個 Exp(λ) 等多久、到達、再抽一個……Python 小實驗就是照這個定義疊出整條流的。立即推論：E[Gᵢ] = 1/λ、C² = 1（ch03）；每段間隔無記憶（ch04）——盯著流看了 3 秒沒等到，下一個到達的剩餘等待仍是 Exp(λ)。

### 定義二：獨立平穩增量＋Poisson 計數（拿來算數）

> N(0) = 0，且：
> ① **獨立增量**——不相交區間裡的到達數互相獨立；
> ② **平穩增量**——N(t+s) − N(s) 的分布只看區間長度 t，不看位置 s；
> ③ N(t) 服從 Poisson(λt)：P(N(t) = n) = e^(−λt)·(λt)ⁿ/n!。

要算「某段時間來幾個」的機率，用這張臉。兩個馬上要用的事實：**E[N(t)] = λt 且 Var(N(t)) = λt**——均值等於變異數是 Poisson 的指紋（Python 小實驗第一題就驗這個）；以及任何長度 t 的窗口都同分布（平穩性）。

**Worked example：λ = 8 時，某一秒 ≥ 12 則的機率**（完整手算，不跳步）。N(1) ~ Poisson(8)，要算 P(N(1) ≥ 12) = 1 − P(N(1) ≤ 11) = 1 − e⁻⁸·Σₖ₌₀¹¹ 8ᵏ/k!。逐項算 8ᵏ/k!（每項可由前一項乘 8/k 得到，手算友善）：

| k | 8ᵏ/k! | 累計 |
|---|---|---|
| 0 | 1.00 | 1.00 |
| 1 | 8.00 | 9.00 |
| 2 | 32.00 | 41.00 |
| 3 | 85.33 | 126.33 |
| 4 | 170.67 | 297.00 |
| 5 | 273.07 | 570.07 |
| 6 | 364.09 | 934.16 |
| 7 | 416.10 | 1350.26 |
| 8 | 416.10 | 1766.36 |
| 9 | 369.87 | 2136.23 |
| 10 | 295.89 | 2432.12 |
| 11 | 215.20 | 2647.32 |

e⁻⁸ ≈ 0.000335463，所以 P(N(1) ≤ 11) ≈ 0.000335463 × 2647.32 ≈ 0.888，**P(N(1) ≥ 12) ≈ 0.112**。平均 8 則/s 的流，每 9 秒左右就有一秒湧進 12 則以上——不是罕見事件。順手多算一格：P(N(1) ≥ 11) = 1 − e⁻⁸ × 2432.12 ≈ 0.184。也就是說，若單一 worker 一秒最多消化 10 則（μ = 10，基準表），尖峰時段約 18% 的秒數裡**當秒到達就超過當秒產能**。λ < μ 卻照樣積壓（ch01 的觀察），現在有了第一個定量註腳；積壓的完整代價 ch08 算。

### 定義三：微小區間刻畫（拿來建模與驗證）

> N(0) = 0、增量獨立且平穩，且當 h → 0：
> P(在 (t, t+h] 恰有 1 個到達) = λh + o(h)；
> P(在 (t, t+h] 有 ≥ 2 個到達) = o(h)。

o(h) 的精確意思：f(h) 屬於 o(h) ⇔ f(h)/h → 0（h → 0）——「比 h 高一階的小量」。這張臉最抽象，卻是**工程上最誠實**的一張，因為它把物理假設攤開給你檢查：

1. 每一瞬間的到達傾向是常數 λ（沒有時段差——之後非齊次版會放掉這條）；
2. 不會兩個同時到（**orderliness**——批次到達直接違反這條）；
3. 各區間互不相干（retry 迴圈直接違反這條）。

當你問「我的系統到達是 Poisson 嗎」，問的其實是這三條成不成立。

### 等價性：證一個方向，其餘指路

三個定義互相等價。等價的實用意義是**換臉自由**：推導時挑最好算的臉（定義二）、寫模擬挑可建構的臉（定義一）、審自己系統挑假設攤得最開的臉（定義三）——而你有權在論證的任何一步切換，因為它們是同一個過程。本章完整證最有內容的一段（定義三 ⇒ 定義二的 Poisson 計數），其餘標明路線與出處：

| 方向 | 本章處理 |
|---|---|
| 定義三 ⇒ 定義二 | **完整推導**（ODE 路線，下方） |
| 定義二 ⇒ 定義一 | 第一個間隔完整推；後續間隔的「重啟」論證指路 |
| 定義一 ⇒ 定義二 | 指路（Erlang 卷積路線，見延伸閱讀 Harchol-Balter Ch.12） |

**定義三 ⇒ N(t) ~ Poisson(λt)**（完整推導；用到的工具只有一階線性 ODE 與積分因子）。記 Pₙ(t) = P(N(t) = n)。先注意由定義三，P(某長度 h 區間 0 個到達) = 1 − λh + o(h)（把「恰 1 個」與「≥ 2 個」扣掉）。

```text
n = 0：
P₀(t+h) = P₀(t) · (1 − λh + o(h))          ← (0,t] 與 (t,t+h] 增量獨立；平穩性給後者機率
(P₀(t+h) − P₀(t)) / h = −λP₀(t) + o(h)/h
令 h → 0：  P₀′(t) = −λP₀(t)，P₀(0) = 1
⇒ P₀(t) = e^(−λt)

n ≥ 1：拆 (0,t+h] = (0,t] ∪ (t,t+h]，後段放 0、1、≥2 個到達：
Pₙ(t+h) = Pₙ(t)(1 − λh) + Pₙ₋₁(t)·λh + o(h)   ← ≥2 個的所有情形機率合計 o(h)
⇒ Pₙ′(t) = −λPₙ(t) + λPₙ₋₁(t)
兩邊乘積分因子 e^(λt)：
(e^(λt)·Pₙ(t))′ = λ·e^(λt)·Pₙ₋₁(t)
歸納假設 Pₙ₋₁(t) = e^(−λt)(λt)^(n−1)/(n−1)!：
(e^(λt)·Pₙ(t))′ = λ·(λt)^(n−1)/(n−1)!
積分（Pₙ(0) = 0 定常數為 0）：
e^(λt)·Pₙ(t) = (λt)ⁿ/n!
⇒ Pₙ(t) = e^(−λt)·(λt)ⁿ/n!        ∎
```

誠實標示：把 o(h) 從機率等式帶進極限、以及對 n 做歸納的起點 n = 1，逐步都能口頭說出理由，但「o(h) 對 t 一致」這類分析細節我們接受了——要測度論等級的版本見延伸閱讀的 Gallager。

**定義二 ⇒ 定義一（第一個間隔）**：用對偶關係，P(G₁ > t) = P(N(t) = 0) = e^(−λt)，所以 G₁ ~ Exp(λ)。一行搞定——但只搞定了第一個間隔。要說「G₂ 也是 Exp(λ) 且與 G₁ 獨立」，需要「過程在隨機時刻 τ₁ 重新開始」：獨立增量只保證在**固定**時刻切開兩邊獨立，τ₁ 是隨機的，把固定換成隨機需要一段 strong Markov 風格的論證。這是三角形裡最容易被含糊帶過的一步，本書不證，見 Gallager 講義 §2.2。

## 疊加與分流：到達流的代數

Poisson 過程在兩個最常見的流操作下封閉——這是它成為「標準模型」的第二個理由（第一個理由在 Palm–Khintchine 那節）。

```
superposition (merge two independent streams)

  stream A, rate lamA ──►──┐
                           ├──►  merged stream, rate lamA+lamB
  stream B, rate lamB ──►──┘

thinning (independent coin flip per arrival, P(keep) = p)

                           ┌──►  kept stream,    rate p*lam
  stream, rate lam ──►─────┤
                           └──►  dropped stream, rate (1-p)*lam
```

### 疊加（superposition）

> **定理**：N₁、N₂ 是**互相獨立**的 Poisson 過程，速率 λ₁、λ₂，則 N₁ + N₂ 是速率 λ₁ + λ₂ 的 Poisson 過程。前提就一個字：獨立——它也是工程上最會破的那個。

**證明 sketch（工程師的嚴謹）**：對合流驗定義三。增量獨立：各自獨立增量＋兩流互相獨立。恰一個到達：P = λ₁h(1−λ₂h) + λ₂h(1−λ₁h) + o(h) = (λ₁+λ₂)h + o(h)。至少兩個：≤ P(流1 ≥ 2) + P(流2 ≥ 2) + P(兩流各 ≥ 1) = o(h) + o(h) + λ₁λ₂h² = o(h)。三條全過。∎

間隔視角交叉驗證：合流的下一個到達 = min(兩流各自的下一個到達)；兩邊剩餘等待都是指數（無記憶），min(Exp(λ₁), Exp(λ₂)) ~ Exp(λ₁+λ₂)——正是 ch04 的競賽引理。而且競賽引理第二條免費送你：**這個到達來自流 i 的機率是 λᵢ/Σλⱼ**。k 條流歸納即得一般版。

工程讀法：k 條獨立 Poisson 上游匯進同一條 pipeline，下游只需要**一個數字** Σλᵢ 就能完整建模——形狀不變、速率相加，這是「下游容量規劃可以無視上游拓撲」的數學理由。注意這份方便是 Poisson 限定的：兩條各自每 30 秒整點觸發的 cron 流（D 流）疊加，相位對齊時是雙倍批次、相位錯開時是交錯等距，合流長相完全取決於相位——沒有「形狀不變」可言。

### 分流（thinning）

> **定理**：速率 λ 的 Poisson 過程，每個到達**獨立擲幣**（機率 p 進 A 流、1−p 進 B 流；擲幣 i.i.d. 且與過程本身獨立），則 A 是 Poisson(pλ)、B 是 Poisson((1−p)λ)，**而且 A 與 B 互相獨立**。

前半句符合直覺，最後一句不符合：A + B 明明等於總流，兩個子流怎麼可能獨立？先看計算（固定 t 的完整計算）：

```text
P(N_A(t) = j, N_B(t) = k)
= P(N(t) = j+k) · C(j+k, j) · p^j (1−p)^k      ← 給定總數，逐筆獨立擲幣
= e^(−λt) (λt)^(j+k)/(j+k)! · (j+k)!/(j!·k!) · p^j (1−p)^k
= [e^(−pλt) (pλt)^j / j!] · [e^(−(1−p)λt) ((1−p)λt)^k / k!]
                                                ← e^(−λt) 拆成兩個指數因子
```

聯合機率精確分解成「Poisson(pλt) 的 pmf × Poisson((1−p)λt) 的 pmf」——邊際各是 Poisson、且在每個固定 t 互相獨立。把「固定 t」升級成「整條過程獨立」要對多個區間重複同一招，本書不展開（見 Harchol-Balter Ch.12）。直覺修復：知道「iOS 這秒收很多」當然暗示總量大，但 Poisson 總量的不確定性恰好被獨立擲幣**乾淨拆成兩份互不相干的不確定性**——這是 Poisson 專屬的性質，一般到達流分流之後子流仍然相關。

**Worked example：p = 0.3 路由到 iOS shard**。主流 λ = 8 則/s，每則訊息按 message_id 雜湊獨立決定、30% 進 iOS 推播：

- iOS 子流：Poisson(0.3 × 8) = **Poisson(2.4)**，平均間隔 1/2.4 ≈ 0.417 s；
- Android 子流：Poisson(5.6)；兩條子流獨立——iOS shard 的容量規劃**不需要**看 Android 的瞬時負載；
- 某一秒 iOS 一則都沒有的機率：e^(−2.4) ≈ 0.0907。

條件檢查（這段比結論重要）：「獨立擲幣、與過程獨立」對 message_id 雜湊近似成立。換成 **round-robin** 就不成立——輪流分配下，子流間隔是兩段 Exp(8) 之和，即 Erlang-2，C² = 0.5：流變得**更平滑**（有時這正是你要的），但它不是 Poisson，拿 M/M/1 公式算那個 shard 會**高估**等待。換成 **least-loaded／JSQ 路由**更糟——路由決策看佇列狀態，「與過程獨立」整個垮掉，子流既不是 Poisson、彼此也不獨立，本節兩個定理一個都用不上。

## 條件均勻性：知道總數之後，Poisson 沒有個性

> **定理（敘述）**：給定 N(t) = n，n 個到達時刻 (τ₁, …, τₙ) 的條件分布，等同於「n 個 i.i.d. Uniform(0, t) 隨機變數排序後」的分布（順序統計量），聯合密度在 0 < u₁ < … < uₙ < t 上恆為 n!/tⁿ。

n = 1 的情形完整推（0 ≤ s ≤ t）：

```text
P(τ₁ ≤ s | N(t) = 1)
= P(N(s) = 1, N(t) − N(s) = 0) / P(N(t) = 1)     ← 獨立增量拆分子
= [λs·e^(−λs)] · [e^(−λ(t−s))] / [λt·e^(−λt)]
= s / t                                           ← λ 全部約掉
```

s/t 正是 Uniform(0, t) 的 CDF。一般 n 的版本是同一招加排列計數，本書不重複（見 Gallager 講義 §2.5）。注意推導最後那行的奇觀：**λ 消失了**。λ 決定「來幾個」；一旦告訴你來了幾個，「落在哪裡」就是純然均勻、毫無偏好——Poisson 是給定速率下「最沒有個性」的到達模式：不群聚、不規律、對任何時刻一視同仁。

三個用途：①條件計算瞬間變簡單——尖峰某分鐘來了 30 則，前 10 秒的則數就是 Binomial(30, 10/60)，期望 5 則，跟 λ 無關（紙上推演第 2 題）；②第二種模擬配方——先抽 N(t) ~ Poisson(λt)，再均勻撒點排序，跟定義一的逐間隔法等價；③事故重建的快速估計——已知尖峰那一小時共約 28,800 則（8/s × 3600）、事故窗口是其中的 4 分鐘，則窗口內期望 28,800 × 4/60 = 1,920 則，不需要重撈 log 也不需要 λ：條件均勻性把「按時間比例分配」升格成定理；④它是後面許多經典證明的暗器（「Poisson 到達看到什麼」是 ch08 PASTA 的主題，此處不展開）。

## 為什麼世界這麼常是 Poisson：Palm–Khintchine

Erlang 在 1909 年用哥本哈根的電話資料實證呼叫到達服從 Poisson 分布（見 landscape 歸屬）。為什麼大自然要配合？因為 Poisson 不是被假設出來的，是被**聚合製造**出來的。

> **Palm–Khintchine 定理（直覺版，本書不證，見延伸閱讀）**：把 m 條**互相獨立**的（renewal）到達流疊加，若總速率收斂到有限的 λ、且每條流都「一致稀疏」（任何一條在固定窗口內到達的機率隨 m → ∞ 一致地變小），則疊加流趨近速率 λ 的 Poisson 過程。

把它讀成「點過程世界的中央極限定理」：CLT 說大量獨立小貢獻之**和**趨向常態；Palm–Khintchine 說大量獨立稀疏到達流之**疊加**趨向 Poisson。關鍵在於**個別來源不必是 Poisson**——每個使用者的發訊行為可以怪到天上去（睡覺、開會、突然狂發），只要來源夠多、互相獨立、沒有誰獨大。

計數版的迷你推導當錨點（工程師的嚴謹；這只說明單一窗口的計數收斂，完整定理是過程層級的收斂）：固定窗長 t，m 個獨立使用者各自在窗內發訊機率約 λt/m，總數 K ~ Binomial(m, λt/m)：

```text
P(K = k) = C(m,k) · (λt/m)^k · (1 − λt/m)^(m−k)
C(m,k)/m^k = m(m−1)…(m−k+1) / (m^k · k!) → 1/k!     ← k 固定、m → ∞
(1 − λt/m)^(m−k) → e^(−λt)
⇒ P(K = k) → e^(−λt) (λt)^k / k!
```

這就是「稀有事件定律」：大量、獨立、個別稀少 ⇒ Poisson 計數。你的 ~10M 訊息/月來自數十萬個彼此不認識的使用者，每人每天幾則——三個條件全中，所以尖峰窗內的到達流近似 Poisson(8) 是有定理護航的，不是偷懶。

**什麼時候破**（三條前提各對應一種工程現實）：

1. **行為相關（獨立性破）**：同一個外部刺激同步喚醒大量來源——球賽進球、行銷推播送達的那一刻、跨日結算。來源不再獨立，到達在事件時刻附近劇烈群聚。徵兆：計數的 Var/Mean（dispersion index）遠大於 1。
2. **retry 迴圈（獨立性破＋速率看狀態）**：失敗製造重試、重試加重失敗——到達率變成系統狀態的函數。它連「固定速率的點過程」都不是，任何 λ = 常數的模型先天失格（feedback 的代價 ch16 算、retry storm ch20 算）。
3. **批次上游（稀疏性破）**：某個上游一口氣倒 500 則進來。「沒有誰獨大」直接不成立；粗時間尺度的計數可能還像 Poisson，間隔分布卻重度叢發（C²ₐ ≫ 1）。

順手兌現 ch04 留的問題：為什麼「到達近似 Poisson」常見、「服務時間近似指數」少見？因為到達是大量獨立來源聚合的產物，有極限定理護航；服務時間是單一工作的內在性質，沒有任何定理保證它像指數——實務上它常常重尾（ch12、ch15 處理）。

## 非齊次 Poisson：λ(t) 與尖峰時段

平穩增量說「速率永遠一樣」，日夜節律直接打臉。修法是保留定義三的骨架、讓速率隨時間走：**非齊次 Poisson 過程**以速率函數 λ(t) 定義——P(在 (t, t+h] 恰一個到達) = λ(t)h + o(h)、≥ 2 個仍是 o(h)、**獨立增量保留、平穩增量丟掉**。同樣的 ODE 路線可推出：

> N(t) − N(s) ~ Poisson(∫ₛᵗ λ(u) du)。記 Λ(t) = ∫₀ᵗ λ(u) du（mean function），則 E[N(t)] = Λ(t)。

小算盤：給推播管線一天的輪廓——夜間 8 小時 λ = 1/s、白天 14 小時 λ = 4/s、尖峰 2 小時 λ = 8/s。日總量 = 3600 × (8×1 + 14×4 + 2×8) = 3600 × 80 = 288,000 則/日，月約 8.6M 則——與 ~10M/月的量測同量級，輪廓合理。全日平均速率 λ̄ = 288,000/86,400 ≈ 3.33/s。現在看清楚兩件事：

- 「尖峰 λ = 8」這句基準的精確身分：**在尖峰窗內把 λ(t) 視為局部常數**（locally stationary），齊次理論整套適用於那個窗口。本書後面所有 λ = 8 的計算都站在這個立足點上。
- 用 λ̄ ≈ 3.33 做容量規劃是用錯模型：μ = 10 的 worker 按平均看 ρ ≈ 0.33 一派輕鬆，尖峰實況是 ρ = 0.8——之後你會知道這兩個 ρ 的延遲差距是好幾倍（ch08 的使用率階梯）。

兩個一句話的延伸：①**時間換刻**——把時鐘換成 Λ(t) 的「操作時間」，非齊次流變回速率 1 的齊次流，檢驗與模擬都靠這招；②模擬 λ(t) 流的標準做法是先生成一條 λ_max 的齊次流、再按 λ(t)/λ_max 擲幣保留——thinning 的再就業。條件均勻性也有非齊次版：給定總數，到達時刻不再均勻，而是按密度正比於 λ(s) 撒點。

## 陷阱與防禦

Poisson 是**可檢驗的假設**，不是預設真理。每條假設、它在工程上怎麼破、騙你多少、怎麼自我察覺：

| 假設 | 工程上怎麼破 | 它怎麼騙你（會錯多少） | 自我察覺 |
|---|---|---|---|
| 增量獨立 | retry 迴圈；事件同步喚醒用戶 | 真實 C²ₐ = 2 的流硬套 Poisson（C²ₐ = 1）：等待低估 1/3——Kingman 修正因子 (C²ₐ+C²ₛ)/2 從 1 變 1.5（基準：0.4 s vs 0.6 s，見 ch14） | 多窗長算計數的 Var/Mean；把到達對齊外部事件時間軸看群聚 |
| 平穩增量 | 日夜節律、行銷活動 | 用全日平均 λ̄ ≈ 3.33 規劃尖峰 λ = 8 的系統：ρ 估 0.33、實況 0.8，延遲差好幾倍 | 分時段估 λ；尖峰窗內用局部常數、跨時段用 λ(t) |
| orderliness（不同時到達） | 批次上游、gateway 攢一批一起 flush | 粗尺度計數像 Poisson，批內排隊把等待吃掉；M/M/1 嚴重低估 | 看間隔直方圖：有沒有一坨貼著 0 的間隔；算 C²ₐ |
| thinning 的獨立擲幣 | round-robin、least-loaded/JSQ 路由 | round-robin 子流是 Erlang-2（C² = 0.5），按 Poisson 算會高估等待；JSQ 子流與佇列狀態相關，公式全失格 | 檢查路由規則：看不看狀態？逐筆獨立嗎？ |
| 「mean ≈ var 所以是 Poisson」 | ——（這是檢驗方法自己的陷阱） | 必要不充分：間隔相關的流也可能在某個窗長碰巧 mean ≈ var | 多個窗長重測 dispersion；加看間隔分布與間隔自相關 |

防禦的最小流程：①畫間隔直方圖、算 C²ₐ（Poisson 應接近 1，P(G > t) 在半對數紙上近似直線）；②至少三個窗長（例如 100 ms、1 s、10 s）各算一次計數的 Var/Mean（Poisson 在每個尺度都應接近 1；叢發流常只在某個尺度露餡）；③讀架構圖——你的流上游有沒有 retry、批次、看狀態的路由。三關都過，才把 M 寫進模型的第一個字母。

## 紙上推演

**第 1 題 [15 分鐘 ★★] 定義互推**：從定義二出發，證明第一個間隔 G₁ ~ Exp(λ)。接著說明：為什麼「G₂ 也是 Exp(λ) 且與 G₁ 獨立」**不是**同一行論證的免費贈品？缺的那塊論證直覺上長什麼樣？

**第 2 題 [10 分鐘 ★★] 條件均勻性應用**：尖峰時段某一分鐘恰好到了 30 則。(a) 前 10 秒落入則數的分布與期望？(b) 30 則全部擠在後 30 秒的機率？(c) λ 有出現在你的答案裡嗎？為什麼？

**第 3 題 [10 分鐘 ★] 三條真實到達流的審判**：A：大量獨立使用者手動發訊息；B：每 30 秒整點觸發的 cron 健康檢查；C：失敗後固定延遲 1 秒重試的 retry 流。哪些近似 Poisson？逐條用三定義或 Palm–Khintchine 的前提說理由。

**第 4 題 [10 分鐘 ★] 疊加與競賽**：iOS 子流 2.4/s 與 Android 子流 5.6/s 匯流。(a) 合流是什麼過程？前提是什麼？(b) 已知此刻來了一則，它是 iOS 的機率？(c) 連續 3 則都是 Android 的機率？

### 推演解答

**解 1**：對偶關係給 P(G₁ > t) = P(N(t) = 0) = e^(−λt)，即 Exp(λ)。G₂ 不是贈品，因為它要求過程「在 τ₁ 重新開始」：獨立增量保證的是**固定**時刻切開兩邊獨立，而 τ₁ 是隨機時刻。直覺修補：τ₁ 之前的歷史決定了 τ₁，而獨立增量讓 τ₁ 之後的增量與那段歷史無關——把這個直覺做嚴謹需要 strong Markov 風格的論證（Gallager §2.2）。常見錯路是一句「無記憶性所以成立」帶過：無記憶性是**單一隨機變數**的性質，「在隨機時刻重啟」是**過程層級**的性質，兩者差一個量級的論證。

**解 2**：(a) 條件均勻性 ⇒ 每則獨立均勻落在 60 秒內，落在前 10 秒的機率 10/60 = 1/6，故則數 ~ Binomial(30, 1/6)，期望 30/6 = 5 則。(b) 每則落在後 30 秒機率 1/2，全部擠進去 = (1/2)³⁰ ≈ 9.31 × 10⁻¹⁰。(c) λ 不在答案裡——這正是條件均勻性的內容：λ 只決定「來幾個」，總數給定後它就出局了。

**解 3**：**A**：近似 Poisson——Palm–Khintchine 三前提（多、獨立、無人獨大）大致成立；例外是行銷推播或重大事件讓行為同步的時刻。**B**：不是——間隔是確定的 30 秒（C²ₐ = 0），且在整點附近的小區間到達機率接近 1 而非 λh，直接違反定義三。值得多想一層：Palm–Khintchine **不要求**個別來源是 Poisson，所以大量「相位隨機」的獨立 cron 疊加起來仍趨近 Poisson；現實的破口是大家都對齊整點——相位相關 = 不獨立 = 前提破。**C**：不是——到達率是系統狀態的函數（失敗越多重試越多），增量不獨立，連固定速率的點過程都不是；它是 ch16/ch20 的 feedback 題材。

**解 4**：(a) 兩條子流獨立（thinning 定理的結論，前提是當初按獨立擲幣拆流），疊加定理 ⇒ 合流是 Poisson(2.4 + 5.6) = Poisson(8)——拆開再合回去，毫髮無傷。(b) 競賽引理：2.4/8 = 0.3。(c) 各到達的來源標籤 i.i.d. ⇒ 0.7³ = 0.343。

### Python 小實驗

只用標準函式庫。實驗一照**定義一**疊出 Poisson 過程，驗證**定義二**的指紋（均值 ≈ 變異數 ≈ λ）——兩個定義在程式裡互相對質。

```python
import random
import statistics

random.seed(42)

LAM = 8.0
N_RUNS = 2000

def count_in_unit_interval(lam):
    """Build the process from exponential gaps; count arrivals in (0, 1]."""
    t = 0.0
    n = 0
    while True:
        t += random.expovariate(lam)
        if t > 1.0:
            return n
        n += 1

counts = [count_in_unit_interval(LAM) for _ in range(N_RUNS)]
m = statistics.mean(counts)
v = statistics.variance(counts)
print(f"mean of N(1)     = {m:.3f}")
print(f"variance of N(1) = {v:.3f}")
print(f"variance / mean  = {v / m:.3f}")
```

理論值：E[N(1)] = Var(N(1)) = 8、Var/Mean = 1。2,000 次重複下均值的標準誤約 √(8/2000) ≈ 0.063，模擬均值應落在 8 ± 0.2；樣本變異數波動較大，應落在 8 ± 0.8；Var/Mean 應落在 1 ± 0.1。

實驗二驗證疊加定理：兩條獨立流（3/s 與 5/s）合併後，間隔應是 Exp(8)——平均 0.125 s、C² = 1。

```python
import random
import statistics

random.seed(42)

def arrival_times(lam, horizon):
    """Arrival epochs of a Poisson process on [0, horizon]."""
    times, t = [], 0.0
    while True:
        t += random.expovariate(lam)
        if t > horizon:
            return times
        times.append(t)

HORIZON = 2000.0
merged = sorted(arrival_times(3.0, HORIZON) + arrival_times(5.0, HORIZON))
gaps = [b - a for a, b in zip(merged, merged[1:])]
m = statistics.mean(gaps)
c2 = statistics.variance(gaps) / m**2
print(f"mean gap = {m:.4f}")
print(f"C^2      = {c2:.3f}")
```

理論值：平均間隔 1/8 = 0.125 s、間隔 C² = 1。約 16,000 個間隔下，平均間隔應落在 0.125 ± 0.003、C² 應落在 1 ± 0.1。

想拷打它：把實驗二其中一條流換成**固定間隔**的 cron 流（`t += 1.0/lam`），再看合流的 C² 掉到哪裡——Poisson 的指紋立刻消失；這正是陷阱表「疊加的前提是每條都隨機且獨立」的體感版。
