# 附錄 B — 公式總表

這是全書的招牌附錄，但請先讀這段警告再讀表格。**一條沒有適用條件的公式是危險品**——它長得跟正確答案一模一樣，差別只在你不知道它什麼時候在騙你。本書每個公式出場時都附了「什麼假設下成立、哪個假設最常在現實中破掉」，這張表把那些條件收進「精確適用條件」欄，那欄才是本表的靈魂；「公式」欄只是它的標籤。

讀法建議：先看「精確適用條件」確認你的系統符不符合，再抄「公式」。表只收**書中推導過或正式敘述過**的公式（記號依 style-guide：E[N]/E[T] 系統，不用教科書的 L/W；對照表見附錄 A）。同一條公式跨章引用時以首次出現章為準，後續章的兌現在條件欄註明。三位有效數字、數字常數（e ≈ 2.71828、ln 2 ≈ 0.6931、ln 100 ≈ 4.6052）全書統一。

凡是標**近似**的，請當羅盤不當計算機；凡是標**精確**的，也只在條件欄列的假設全成立時精確——條件破一個，精確就退化成「不知道錯多少」。

## Part I — 不用機率的鐵律（ch01–02）

這批律只靠「數數字」就成立，不需要任何分布假設——這是它們的威力，也是它們的天花板：它們綁住平均值，但對 tail 一無所知。

| 模型／定律 | 公式 | 精確適用條件（哪些假設、哪個最常破） | 首次出現章 |
|---|---|---|---|
| Little's Law | E[N] = λE[T] | 只需兩件事：①**穩定、長期平均存在**（λ 與逗留時間平均收斂、系統不無限堆積）②**邊界一致**（三個量數同一群 job、同一條虛線框）。與分布、相關性、排程、內部結構**全部無關**。**最常破**：非穩態窗口（事故中 backlog 單調上漲，A/C 偏離 1，算出的「平均延遲」沒有指涉物）——算之前先看 A/C。它是守恆律不是效能模型，不告訴你值是多少、tail 多肥。 | ch02 |
| Little 三邊界恆等式 | E[N_Q] = λE[T_Q]；E[N_服務中] = λE[S] = ρ；E[N] = E[N_Q] + ρ、E[T] = E[T_Q] + E[S] | 同 Little。第三式是 Utilization Law 被 Little 重新發明：使用率＝「平均有多少 job 正在被服務」。c 台時平均忙碌台數 = λE[S]，除以 c 是每台使用率。**最常破**：把 ρ 當成「服務位的 E[T]」之類的時間量——它是比例。 | ch02 |
| Utilization Law | ρ = X·E[S]（單站 ρᵢ = X·Dᵢ） | 操作分析框架；無分布假設。Dᵢ = Vᵢ·E[Sᵢ] 是每個 job 的固有服務需求（量測性質 Dᵢ = Bᵢ/C）。**最常破**：把當下浮動的使用率當成 Dᵢ——Dᵢ 不隨流量變、才進得了容量試算表。 | ch02 |
| Response Time Law（封閉系統） | E[T] = N/X − Z | **封閉迴圈**：N 個使用者固定繞圈、think time Z、迴圈內永遠剛好 N 人。它就是 Little 換邊界（完整推導）。**最常破**的不是公式而是誤用情境：固定 VU 的壓測裡 N 被釘死，X 與 E[T] 鎖在蹺蹺板兩端——**這種系統天生不會過載**，跟線上開放流量是兩種動物（見 ch17）。 | ch02 |
| Forced Flow Law | Xᵢ = Vᵢ·X，其中 Vᵢ = Cᵢ/C | 操作分析；visit ratio Vᵢ 是每個 job 平均訪問裝置 i 的次數。各裝置流量被整體吞吐「強制」鎖定。無分布假設。 | ch02 |
| Bottleneck bounds（封閉系統） | X ≤ 1/D_max；X ≤ N/(ΣDᵢ + Z)；拐點 N* = (ΣDᵢ + Z)/D_max；E[T] ≥ ΣDᵢ | **工程師的嚴謹**。第一條需 Dᵢ 為常數（不隨負載變）；第二條另需 **job 同一時刻只在一個地方接受服務**（沒有 fork-join 並行——有的話 E[T] ≥ ΣDᵢ 不再成立，見 ch18）。**最常破**：把漸近線當預測——真實曲線在拐點附近是圓角、離 bound 最遠（精確曲線要 MVA，見 ch17）。 | ch02 |

## Part II — 機率工具（ch03–06）

這批不是「佇列公式」而是後面全書的引擎與量尺，列出書中當主力工具反覆使用的幾條。

| 工具 | 公式 | 精確適用條件（哪些假設、哪個最常破） | 首次出現章 |
|---|---|---|---|
| 變異係數平方（SCV） | C² = Var(X)/E[X]² | 任意非負隨機變數（E[X] > 0）。刻度：C²=0 確定性、C²=1 指數、C²>1 叢發。**最常破**：只量平均不量 C²——它是後面 P-K／Kingman 的主角，平均值看不見它。 | ch03 |
| 幾何級數與微分技巧 | Σₙ ρⁿ = 1/(1−ρ)、Σₙ n·ρⁿ = ρ/(1−ρ)² | **需要 \|ρ\| < 1**（級數收斂）；推 πₙ 正規化與 E[N] 的引擎。**最常破**：ρ ≥ 1 時硬套——級數發散，對應佇列不穩態。 | ch03 |
| Tail 積分求期望 | E[X] = ∫₀^∞ P(X > t) dt | 非負隨機變數。全書求期望的通用後門。 | ch03 |
| Markov／Chebyshev 不等式 | P(X ≥ a) ≤ E[X]/a；P(\|X−μ\| ≥ kσ) ≤ 1/k² | Markov 需 X ≥ 0；Chebyshev 需變異數有限。**用途**：平均值對 P99 到底承諾了多少（答案：很少）。**最常破**：重尾下變異數無限，Chebyshev 直接失效（見 ch15）。 | ch03 |
| 指數分布無記憶性 | P(S > t+s \| S > s) = P(S > t)；連續分布中唯一 | 化簡推導的武器；殘餘壽命＝原分布。**最常破**：服務時間常常不是指數（叢發或重尾），用無記憶性化簡會錯（見 ch12/15）。到達間隔較常像（ch05 解釋為什麼）。 | ch04 |
| 競賽引理 | min(X₁,…,Xₖ) ~ Exp(Σλᵢ)；P(Xᵢ 最先) = λᵢ/Σλⱼ；勝者身分與競賽時長獨立 | 各 Xᵢ ~ Exp(λᵢ) 且互相獨立。CTMC 的引擎、fork-join E[max] 的零件。**最常破**：非指數或相關時三條全垮。 | ch04 |
| Poisson 疊加／分流 | 疊加：Σ 獨立 Poisson(λᵢ) ~ Poisson(Σλᵢ)；分流：機率 p 抽出的子流 ~ Poisson(pλ)，且與補流獨立 | 各流獨立 Poisson；分流是獨立硬幣。**最常破**：上游有重試迴圈、批次、行為相關時匯流不再 Poisson（Palm–Khintchine 的邊界，見 ch16 feedback）。 | ch05 |
| 平穩分布 | π = πP（離散）；解法＝代入消去解線性方程組＋正規化 | 有限、不可約、非週期 chain：存在、唯一、收斂（工程師嚴謹）。**最常破**：週期性（振盪不收斂但 π 仍存在）或可約（多個平穩分布）。 | ch06 |

## Part III — 生滅與經典模型（ch07–11）

| 模型／定律 | 公式 | 精確適用條件（哪些假設、哪個最常破） | 首次出現章 |
|---|---|---|---|
| 生滅過程通解 | πₙ = π₀·Π_{i=0}^{n−1} (λᵢ/μᵢ₊₁)；π₀ = 1/S，S ≡ Σ_{n=0}^{∞} Π_{i=0}^{n−1} (λᵢ/μᵢ₊₁) | ①生滅結構（只有 ±1 轉移）②所有速率為正（不可約、非爆炸）③**穩態存在需 S < ∞**。**最常破**：通解形式上永遠寫得出來，但 S = ∞ 時硬正規化得到的「π」是垃圾——穩態根本不存在，模擬會看到佇列單調竄高、與「解」毫無關係。寫完通解**必驗級數收斂**。 | ch07 |
| M/M/1 平穩分布 | πₙ = (1−ρ)ρⁿ，ρ = λ/μ | 假設清單：①到達 Poisson(λ) ②服務 i.i.d. Exp(μ)、與到達獨立 ③c=1 ④K=∞（永不丟棄）⑤N=∞（開放）⑥FCFS ⑦**ρ < 1**。**最常破**：服務 Exp（C²ₛ=1）——服務固定（M/D/1）時真實等待是本表的一半、服務重尾時低估到不知幾倍（見 ch12/15）。服務時間沒有極限定理護航，**永遠要量 C²ₛ**。 | ch08 |
| M/M/1 平均量 | E[N] = ρ/(1−ρ)；E[T] = 1/(μ−λ) = E[S]/(1−ρ)；E[T_Q] = ρ/(μ−λ)；E[N_Q] = ρ²/(1−ρ) | 同 M/M/1 全套假設。E[T]/E[T_Q] 由 Little 從 E[N]/E[N_Q] 換來。**最常破**：同上，C²ₛ ≠ 1。 | ch08 |
| M/M/1 回應時間分布 | T ~ Exp(μ−λ)：P(T > t) = e^(−(μ−λ)t) | 額外需 **FCFS**（換成 PS／LCFS 此分布變形——平均 E[T] 仍對，但分布與 tail 不對）。**最常破**：拿這個分布算非 FCFS 系統的 P99。 | ch08 |
| M/M/1 的 P99 | t₉₉ = ln 100/(μ−λ)；一般 t_p = ln(1/(1−p))/(μ−λ)；「P99 ≈ 4.6× 平均」 | 需回應時間分布成立（M/M/1 + FCFS）。「4.6×」＝ ln 100 ≈ 4.6052。**最常破**：服務非指數時整條規則死掉——重尾下 P99 與平均脫鉤、倍數離譜（見 ch15，且 4.6 與 ch15 的每九倍率 4.64 是撞名，意義完全不同）。 | ch08 |
| 使用率階梯 | E[T] = E[S]/(1−ρ)：ρ=0.5→0.2 s、0.8→0.5 s、0.9→1.0 s、0.95→2.0 s、0.99→10 s | M/M/1（E[S] = 100 ms 基準）。hockey stick 的數值兌現。**最常破**：有牆的系統（有限 buffer）——延遲會飽和而非爆炸（見 ch10）。 | ch08 |
| PASTA | 到達者抵達瞬間（不含自己）看到狀態 n 的長期比例 = 時間平均 πₙ | 需 ①**到達是 Poisson** ②**lack of anticipation**（系統現在不洩漏到達流的未來）。**最常破**：到達不是 Poisson——等距／整形後的近等距流、有限母體（在場人數越多到達越少）、balking 放行後的流，到達者看到的都 ≠ 時間平均。用前先問：到達 Poisson 嗎？到達率會不會看狀態？ | ch08 |
| M/M/c 平穩分布 | n ≤ c：πₙ = π₀·aⁿ/n!；n ≥ c：πₙ = π₀·(aᶜ/c!)·ρⁿ⁻ᶜ（a = λ/μ、ρ = λ/(cμ) = a/c） | 假設同 M/M/1 但 **c 台共用單一佇列**、μₙ = min(n,c)·μ、ρ < 1。**最常破**：「有 c 台機器」≠ M/M/c——hash/sticky 分流其實是 c 個獨立 M/M/1，同硬體 E[T] 差約 2.9 倍。先看路由：拉式（consumer 主動領）≈ 共用佇列；推式 hash ≈ 獨立佇列。 | ch09 |
| Erlang-C（延遲機率） | C(c, a) = [aᶜ/c!·1/(1−ρ)] / [Σ_{n=0}^{c−1} aⁿ/n! + aᶜ/c!·1/(1−ρ)] | M/M/c（共用佇列、ρ < 1）。**最常破**：對服務分布**敏感**——等待大致按 (1+C²ₛ)/2 縮放，C²ₛ = 10 時低估約 5.5 倍。口訣：**loss 看均值就夠，delay 要看變異數**（與下面 Erlang-B 最容易記反）。 | ch09 |
| M/M/c 平均等待 | E[T_Q] = C(c, a)·1/(cμ−λ) | 同 Erlang-C。讀法：等的機率 × 等的人平均等多久。基準 c=4、a=3.2：C ≈ 0.596、E[T_Q] ≈ 74.6 ms、E[T] ≈ 175 ms。 | ch09 |
| Erlang-B（損失機率）遞迴 | I(0)=1、I(c)=1+(c/a)·I(c−1)，B(c,a)=1/I(c)；等價 B(c,a) = (aᶜ/c!)/Σ_{k=0}^{c} aᵏ/k! | M/M/c/c **損失系統**（沒位子就走、不排隊）。對任意 a 都有定義（容量有限）。**關鍵性質**：對服務分布 **insensitive**——只依賴 a = λE[S]，與 C²ₛ 無關（ch18 的 BCMP insensitivity 兌現）。**最常破**：跟 Erlang-C 記反（delay 敏感、loss 不敏感）。 | ch09 |
| M/M/∞ | πₙ = e⁻ᵃ·aⁿ/n!，即 N ~ Poisson(a) | 無限伺服器、無排隊（T_Q ≡ 0）。對服務分布 insensitive。是 √a staffing 的理論基礎（Var(N) = a）。 | ch09 |
| 平方根 staffing | c ≈ a + β·√a（β 約在 1 上下） | QED regime：c 與 λ 一起放大、保持 (1−ρ)√c → β 時 P(等待) 收斂到只依賴 β 的極限（β=1 約 0.22）。直覺推導，嚴格版 Halfin–Whitt 指路。**最常破**：非 Poisson 到達或非指數服務、批次到達、重尾——√a 緩衝的論證失效。 | ch09 |
| M/M/1/K 平穩分布 | πₙ = (1−ρ)·ρⁿ/(1−ρ^{K+1})，n = 0…K（ρ ≠ 1）；ρ = 1 時 πₙ = 1/(K+1) | 有限容量 K（牆）。**ρ 可以 > 1**——這是與 M/M/1 的關鍵差別（牆吸收了不穩定）。**最常破**：拿無限 buffer 公式算有牆的系統，或只報漂亮的 E[T] 不報 π_K。監控出現 429/503/drop 計數那刻，無限 buffer 公式出局。 | ch10 |
| Blocking 機率與 goodput | P(到達被擋) = π_K（精確）；λ_eff = λ·(1−π_K) | 需 Poisson 到達（用 PASTA：到達看到滿就被擋）。**最常破**：Little's Law 用 λ 不用 λ_eff——λ=12 過載時用 λ 算 E[T] 低估 19%，越過載騙得越凶。進 Little 的 λ 是「真的進了系統」的流量。 | ch10 |
| 過載 blocking 地板 | K→∞ 時 π_K → 1 − 1/ρ（ρ > 1） | 流量守恆的必然：λ=12、μ=10 時至少 1−1/1.2 ≈ 16.7% 必被丟。**最常破**：以為「buffer 開大一點」能救過載——K→∞ 不消滅損失，只把它換成延遲（bufferbloat）。 | ch10 |
| 串聯（tandem）乘積形式 | π(n₁, n₂) = π₁(n₁)·π₂(n₂) = (1−ρ₁)ρ₁^{n₁}·(1−ρ₂)ρ₂^{n₂}，ρᵢ = λ/μᵢ | 各站 M/M/1（或 M/M/c/∞）、指數服務、**無 feedback**、穩態、各站 ρᵢ < 1。靠 Burke 定理（下行）。**最常破**：有 feedback 時內部流不再 Poisson，各站當獨立 M/M/1 算會系統性低估（見 ch16）。 | ch11 |
| Burke 定理 | ①離開過程是速率 λ 的 Poisson 過程 ②任一時刻 t 的狀態 N(t) 與 t 之前的全部離開歷史獨立 | M/M/1、M/M/c、M/M/∞，λ < cμ，**跑在穩態**。**最常破**：用在暫態——清積壓時上游以滿速 μ 往下灌，「下游輸入 = λ」的容量帳當場作廢（排空期間下游容量帳要用上游的 **μ** 不是 λ）。次常破：用在 M/G/1（需指數服務，否則輸出非 Poisson）；把第 2 句讀成「與未來輸出也獨立」（只與過去獨立）。 | ch11 |

## Part IV — 超越指數（ch12–15）

放掉指數假設，變異數的帳單登場。這批公式的條件欄特別重要——它們正是「指數世界的公式什麼時候騙你」的答案。

| 模型／定律 | 公式 | 精確適用條件（哪些假設、哪個最常破） | 首次出現章 |
|---|---|---|---|
| 殘餘服務時間（inspection paradox） | E[殘餘 \| 服務中] = E[S²]/(2E[S]) = E[S]·(1+C²ₛ)/2；時間平均殘餘 W₀ = λE[S²]/2 | 服務時間 i.i.d.。「你總是撞上長工作」的數學——被長度加權。**最常破**：誤以為撞上的工作平均長度＝E[S]（其實被 (1+C²ₛ)/2 放大；重尾下放大到極端，見 ch15）。 | ch12 |
| Pollaczek–Khinchine 平均等待 | E[T_Q] = λE[S²]/(2(1−ρ)) = ρ/(1−ρ)·E[S]·(1+C²ₛ)/2 | M/G/1：①Poisson 到達（PASTA 那步）②服務 i.i.d.、與到達獨立、**不看長度出工**（FCFS 或非搶占同類家族）③ρ < 1、穩態 ④**E[S²] < ∞**。基準三連比 C²ₛ ∈ {0,1,10} → 0.2/0.4/2.2 s；M/D/1 等待是 M/M/1 一半。**最常破**：只量平均服務時間就建模——C²ₛ = 10 的現實當成指數（C²ₛ=1）會低估 5.5 倍。次常破：E[S²] = ∞（重尾，見 ch15）時公式回答「∞」，不是壞掉是真警報。到達非 Poisson 時 P-K 沒有 C²ₐ 旋鈕、會錯（改用 ch14 Kingman）。 | ch12 |
| Kleinrock 守恆律 | Σᵢ ρᵢ·E[T_Q,i] = ρ·W₀/(1−ρ)，W₀ = λE[S²]/2 | M/G/1、**非搶占、work-conserving、穩態**。右邊**不含任何紀律資訊**（基準下永遠 = 0.52 s）。守恆的是 **ρ 加權**等待，不是人頭加權——所以人頭平均可以動（短工作優先就是這個套利）。**最常破**：以為「換紀律能讓大家都變快」——總帳一毛不差，有人在帳外付錢。搶占會打破 work-conserving 家族的前提。 | ch13 |
| 非搶占優先權等待 | E[T_Q,i] = W₀/((1−σᵢ₋₁)(1−σᵢ))，σᵢ ≔ ρ₁+…+ρᵢ（兩類：E[T_Q,1]=W₀/(1−ρ₁)、E[T_Q,2]=W₀/((1−ρ₁)(1−ρ₁−ρ₂))） | M/G/1、各類 Poisson、**非搶占**、類內不看長度、work-conserving、ρ < 1。SJF 是「類別＝尺寸」連續極限 E[T_Q(x)] = W₀/(1−ρ(x))²（敘述層級）。**最常破**：系統有隱藏優先權（worker 偷挑短訊息）卻拿 P-K 套——實際 E[T_Q] 與 P-K 可差 2 倍以上。 | ch13 |
| PS 回應時間與 insensitivity | E[T(x)] = x/(1−ρ)；E[T] = E[S]/(1−ρ) | M/G/1-PS（processor sharing）、work-conserving、ρ < 1。**insensitivity**：E[T] 只依賴 E[S]，C²ₛ 完全 drop out。**最常破**：①只保 N 的分布與**平均** T，不保 T 的**分布／tail**——M/G/1-PS 的 P99 仍依賴 C²ₛ（見 ch15）②insensitivity 屬於 PS/IS/LCFS-PR，**不屬於 FCFS**——你的 worker pool 是 FCFS，C²ₛ=10 時照樣低估 5.5 倍。 | ch13 |
| cμ rule | 非搶占下按 cᵢμᵢ（成本率 × 服務率）遞減順序服務，最小化平均成本 | M/G/1、非搶占（敘述層級）。SJF 是 cᵢ 一致時的特例。 | ch13 |
| Lindley recursion | W_{n+1} = max(0, W_n + S_n − A_{n+1}) | **精確、無分布假設**（對任意樣本序列成立）。只需：單伺服器、FCFS、work-conserving、buffer 無限。它就是 ch19 模擬 G/G/1 的核心迴圈。**最常破**：幾乎不破（這是它珍貴的原因）——但它給的是逐一 job 的遞迴，不直接給穩態平均，要靠模擬統計。 | ch14 |
| Kingman 近似（VUT 公式） | E[T_Q] ≈ ρ/(1−ρ)·(C²ₐ+C²ₛ)/2·E[S]；上界版 E[T_Q] ≤ λ(σ²ₐ+σ²ₛ)/(2(1−ρ)) | G/G/1 的**近似**（V × U × T：變異 × 使用率 × 時間尺度），ρ→1 時漸準。兩個**精確特例**：C²ₐ=C²ₛ=1（M/M/1）、C²ₐ=1 任意 C²ₛ（即 P-K）。需二階矩有限。**最常破**：①固定速率壓測餵假 C²ₐ≈0——量到 0.2 s，真實生產（C²ₐ=2）是 0.6 s、低估 3 倍 ②相關到達——C²ₐ 抓不到時間相關（cron 批次、retry 迴圈），低估可達一個數量級 ③低 ρ（< 0.5）高估約 20% ④重尾（C² 無定義，見 ch15）。 | ch14 |
| Allen–Cunneen（G/G/c） | E[T_Q] ≈ [ErlangC(c,a)/(cμ−λ)]·(C²ₐ+C²ₛ)/2 | G/G/c 近似——比 c=1 版更弱，c > 1 無 heavy-traffic 保證。當羅盤不當計算機。繼承 Kingman 三個弱區。 | ch14 |
| Pareto 矩存在性 | P(S > t) = (k/t)^α（t ≥ k）；E[S^m] = αk^m/(α−m) 當 m < α，否則 = ∞ | Pareto 尾。**臨界值**：α ≤ 1 → E[S] = ∞（任何 λ > 0 都不穩定）；1 < α ≤ 2 → E[S] 有限但 **Var = ∞ → C²ₛ 無定義 → P-K = ∞**；α > 2 → 前兩階矩有限。**最常破**：用有限樣本估 C²ₛ 然後套 ch12/14——估計值永遠偏低且隨視窗變長一路爬，你以為 C²ₛ=12 其實無上界。 | ch15 |
| 重尾下的失效 | M/G/1 FCFS：E[T_Q] = ∞（1<α≤2，ρ<1）；PS：E[T] = E[S]/(1−ρ) 不受影響；P(撞上 > t) = (k/t)^{α−1} | α ≤ 2。系統仍**穩定**（ρ < 1、佇列會清空）但 E[T_Q] 的矩發散——「穩定」≠「平均有限」。救贖：PS/SRPT 在重尾下壓倒性勝出（只需 E[S] < ∞）。**最常破**：拿輕尾「P99 ≈ 4.6× 平均」（ch08）外推重尾分位數——每多一個 9 再乘約 4.64 倍（Pareto α=1.5），錯得離譜。 | ch15 |

## 決策速查：什麼時候用哪個

這節把各章的決策框架壓成一張流程。核心問句永遠是「我量過 C²ₐ 和 C²ₛ 了嗎」——沒量過，下面任何選擇都是猜。

**先問三件事**：到達是不是 Poisson（C²ₐ ≈ 1）？服務是不是指數（C²ₛ ≈ 1）？系統是開放還封閉？

```text
            ┌─ 只要平均量的關係、不要分布？
            │     → Little's Law / 操作律（ch02）。零分布假設，但只給平均、不給 tail。
            │
            ├─ 開放、單站、到達 Poisson？
            │     ├─ 服務也指數（C²ₛ≈1）────────→ M/M/1（ch08）。精確、好算。
            │     │      多伺服器共用佇列 ──────→ M/M/c / Erlang-C（ch09）。
            │     │      有牆（有限 buffer）─────→ M/M/1/K（ch10）。要報 loss。
            │     ├─ 服務非指數、C²ₛ 已量 ──────→ P-K（ch12）。精確（需 E[S²]<∞）。
            │     └─ 連到達都非 Poisson（C²ₐ≠1）→ Kingman / VUT（ch14）。近似，ρ→1 較準。
            │
            ├─ 多站網路、機率路由、無 fork-join/priority/blocking？
            │     ├─ 開放 ─────→ Jackson（ch16）：各站當 M/M/1，乘積形式。
            │     └─ 封閉 ─────→ MVA（ch17）：固定母體，遞迴手算。
            │
            └─ 以下任一成立 → 公式到頭了，模擬（ch19）：
                  • 重尾（C² 無定義或矩發散，ch15）
                  • fork-join / 同步 / 相關 retry（ch18）
                  • 時間相關到達（C²ₐ 抓不到，ch14）
                  • 要 tail 分布而非平均（Little/Kingman 只給平均）
```

**口袋判準**：

- **M/M/1 夠用**，當且僅當：到達近 Poisson、服務近指數（量過 C²ₛ ≈ 1）、單一共用佇列、ρ 不貼近 1、你只要平均與「指數型」P99。多數「先估個數量級」的場合它夠了。
- **需要 P-K**：服務明顯非指數（你量到 C²ₛ 遠離 1），但到達仍是 Poisson。P-K 在這個條件下是**精確**的，不是近似——別退而求其次用 Kingman。前提：E[S²] 有限（不是重尾）。
- **需要 Kingman/VUT**：到達也非 Poisson（叢發或被整形）。這是工程現場最常見的情況，也是 back-of-envelope 神器。記住它是**近似**：低 ρ 高估、重尾與相關到達嚴重低估。它的價值在那個 V × U × T 的世界觀——告訴你「砍 C² 還是降 ρ」哪個槓桿划算。
- **只能模擬**：重尾（矩發散，公式回答 ∞ 或 C² 無定義）、fork-join／同步（Promise.all 的 tail）、時間相關到達（cron 批次、retry 迴圈——C²ₐ 這個單一數字抓不到「什麼時候叢發」）、或你要的是完整 tail 分布而不是平均。模擬會騙人（ch19 的四大陷阱：warmup 污染、輸出自相關、種子紀律、稀有事件），所以模擬前先用 M/M/1 ground truth 校準你的程式、輸出永遠帶區間。

**如果只帶走三個公式**（ch21）：Little（E[N] = λE[T]，萬用驗算器）、VUT（Kingman/P-K 一家，變異數的帳單）、Erlang-C（多伺服器 staffing）。其餘都是這三條在特定條件下的特例或推廣——而每一條的「特定條件」，就在上面那張表的第三欄。
