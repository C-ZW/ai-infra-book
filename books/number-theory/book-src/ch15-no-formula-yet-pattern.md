# ch15 — 沒有公式，卻有規律

> **本章解決什麼問題**：到這裡，Part IV 已經把「質數分布的細節之謎」攤開了——孿生質數合不上的縫（ch12）、黎曼零點藏著的誤差（ch13）、驗證再多也不是證明（ch14）。本章收這個 Part 的張力，回答一個被問了上千次、也被誤解了上千次的問題：**有沒有一條公式，代進去就吐出質數？** 答案是「沒有一條好用的」，但本章真正的工作不是宣布壞消息，而是拆穿一個誘人的錯誤推論——很多人聽到「沒有質數公式」，下一句就接「所以質數是隨機的、沒規律」。這一步錯得離譜。沒有生成公式，跟沒有規律，是兩件完全不同的事。這是脊椎張力「亂與律共存」的最後一次、也是最精煉的一次辨析。

## 從你已知的出發

你寫過 hash function。給它一個 key，它吐出一串看起來像亂碼的雜湊值——`a3f9c1...`。這串值有個關鍵性質：**它沒有「反推公式」。** 你不能拿著 `a3f9c1` 倒推回原本的 key，hash 是單向的，這正是它能拿來當指紋、當密碼存儲的原因（呼應 ch17 的陷門直覺）。

但你會說 hash 的輸出「沒有規律」嗎？絕對不會。你心裡很清楚：

- hash 輸出**有極強的統計規律**——好的 hash 把輸入均勻地撒滿整個輸出空間，每個 bucket 命中的機率幾乎一樣，這就是它能拿來做分桶（`x % n`）的根據。
- 它的**雪崩效應**是規律——改動輸入一個 bit，輸出有一半的 bit 會翻轉。
- 它**確定**——同一個 key 永遠吐同一個值，零隨機性。

所以 hash 是「沒有反推公式，卻有滿滿的統計規律，而且完全確定」的東西。你天天依賴這三件事同時成立。沒有人會因為「hash 沒有反推公式」就說「hash 是隨機的」——那會讓一致性雜湊、布隆過濾器、bucket 分桶全都失去意義。

質數就是數論版的這件事。**沒有一條「代進去吐出第 n 個質數」的好公式（無生成公式），但質數服從 PNT 這條鐵律、有 Ulam 螺旋上肉眼可見的直線（有規律），而且每個整數是不是質數是個百分百確定的事實（零隨機性）。** 三件事同時為真，跟你對 hash 的直覺一模一樣。

這章要做的，就是把「沒有公式」和「沒有規律」這兩件被混為一談的事，徹底劈開。

## 「質數公式」到底在問什麼

先把問題問清楚，否則答案會滑來滑去。「有沒有質數公式」這句話，其實藏著好幾個不同的問題，難度天差地遠：

```text
「質數公式」可能指的四種東西（按難度遞增排列）：

  (A) 一條公式 f(n)，對每個 n 都吐出「某個質數」（不要求是第 n 個）
       → 存在，但都是廢的（見下文 Mills 常數）

  (B) 一條公式 f(n)，吐出「第 n 個質數」pₙ（2,3,5,7,11,...）
       → 沒有已知的「好」公式；有存在但不可計算的怪胎

  (C) 一條簡單多項式 f(n)，對所有 n 都吐出質數
       → 不可能（可證明沒有，見下文）

  (D) 能精準預測「下一個質數在哪」的封閉公式
       → 沒有，這是質數最硬的謎之一
```

大眾嘴上說的「質數沒有公式」，混著 (B)(C)(D) 在講。而最容易被誤解的，是把「沒有 (B)(C)(D) 這種好公式」直接讀成「質數毫無規律」。本章會逐個拆。

關鍵的精確意義：**「公式」這個詞偷偷預設了「好用」。** 一條你算不動、或算了等於沒算的公式，存在跟不存在沒差別。下面你會看到，數學家其實「有」幾條質數公式——但它們全是廢的，廢的方式還各有各的精彩。

## 美麗的陷阱：歐拉的 n²−n+41

讓我們從一個真的會吐出質數、而且吐得異常漂亮的多項式講起。它是本章的 worked example，也是「公式」這條路最誘人的一次失敗。

考慮這個二次式：

```text
   f(n) = n² − n + 41
```

代進去算算看。我每一步自己手算複核（n²−n 再加 41）：

```text
   n  │ n²    n²−n   f(n)=n²−n+41 │ 質/合
   ───┼───────────────────────────┼──────
   0  │  0     0      41          │ 質
   1  │  1     0      41          │ 質   ← n=0,1 都給 41，不意外（n²−n 都是 0）
   2  │  4     2      43          │ 質
   3  │  9     6      47          │ 質
   4  │ 16    12      53          │ 質
   5  │ 25    20      61          │ 質
   6  │ 36    30      71          │ 質
   7  │ 49    42      83          │ 質
   8  │ 64    56      97          │ 質
   9  │ 81    72     113          │ 質
   10 │100    90     131          │ 質
   ⋮             （一路連續質數）
   40 │1600  1560    1601         │ 質   ← 撐到 n=40，連續 41 個質數
   ───┼───────────────────────────┼──────
   41 │1681  1640    1681         │ 合 ← 1681 = 41 × 41，破了！
```

停下來感受一下這個東西有多漂亮：**n 從 0 一路代到 40，整整 41 個連續輸入，每一個都吐出質數。** 41, 41, 43, 47, 53, 61, 71, 83, 97, 113, 131, …, 1601。你如果在 1700 年代手算到這裡，幾乎不可能不相信自己找到了「質數公式」。

然後 n=41 給你一巴掌。f(41) = 41²−41+41 = 41²= 1681 = 41×41，合數。而且它破得毫無懸念——你甚至一眼就能看出來為什麼必破：

```text
   f(41) = 41² − 41 + 41 = 41²        ← 41 顯然整除它
   f(82) = 82² − 82 + 41 = 41(2·82−2+1)/... 
           實際上：82²−82+41 = 41×(164−2+1)=41×163 = 6683  ← 41 又整除
```

（複核 f(82)：82²=6724，6724−82=6642，6642+41=6683；6683÷41=163，41×163=6683 ✓，確為合數。）

道理一旦看穿就很簡單：**任何只有整數係數的非常數多項式，都不可能對所有 n 都吐出質數。** 這不是「目前還沒找到」，是可以證明的「不可能」——這就是上面分類裡的 (C)。下一節把它證給你看，這個證明短到工程師能一步步讀完。

關於歸屬，我得誠實標一下（這是 landscape 沒細分、我查證後要 hedge 的地方）：這條 `n²−n+41` 通常歸給**勒讓德（Legendre）1798 年**，它給出 n=0..40 共 41 個連續質數；而**歐拉（Euler）1772 年**原本寫的是形式略不同的 `n²+n+41`，給出 n=0..39 共 40 個連續質數。兩者是同一個「幸運數 41」的近親（把 n 換成 −n 或平移一格就互換），本書沿用 `n²−n+41` 這個形式、並依慣例稱「歐拉那一族質數豐產二次式」。重點不在誰先寫下哪個正負號，而在這個 41 為什麼這麼特別——它是極少數能撐這麼長的「歐拉幸運數」之一（2026-06 查證：這類幸運數只有 2, 3, 5, 11, 17, 41 六個，與虛二次域類數 1 有深刻關聯，那是代數數論、本書不開）。

## 為什麼沒有多項式能永遠吐質數（這個證明能一步步讀）

上一節說「任何整係數非常數多項式都會在某處吐出合數」。這是本章少數要完整寫透的證明之一（嚴謹度：工程師的嚴謹級，每步能口頭說出理由），因為它短、它漂亮、而且它正面回答了 (C)。

假設你有一個整係數多項式 f(n)（次數 ≥ 1，不是常數）。我要證：它一定有某個輸入吐出合數。

```text
   設 f(n) = c_k·n^k + ... + c_1·n + c_0，整數係數，次數 k≥1。

   第 1 步：找一個 f 給出質數的點。
     如果 f 從來不給質數，那它輸出全是合數（或 ±1/0），早就不是「質數公式」了，
     直接結案。所以假設存在某個 a，使 f(a) = p 是質數。

   第 2 步：看 f(a + t·p) 對任意整數 t 的值。
     把 n = a + t·p 代進多項式。展開後，每一項
        c_j·(a + t·p)^j
     用二項式展開，除了「全取 a 的那一項 c_j·a^j」之外，
     其餘每一項都至少帶一個因子 p（因為都含 t·p）。
     所以：
        f(a + t·p) = f(a) + (一堆都被 p 整除的東西)
                   = p     + (p 的倍數)
                   → 必被 p 整除！

   第 3 步：收網。
     f(a + t·p) 對每個整數 t 都被 p 整除。
     而多項式的值會隨 t 變大而趨於無窮（次數≥1），
     所以只要 t 夠大，f(a + t·p) 是個「被 p 整除、又比 p 大」的數
     → 它有因數 p、又不等於 p → 合數。            ∎
```

一句話總結這個證明的靈魂：**多項式對「模 p」是有週期性的**——f(a)≡0 (mod p) 就逼著 f(a+p)、f(a+2p)、… 全都 ≡0 (mod p)。質數討厭被別的數整除，多項式卻週期性地讓某個 p 反覆整除它的輸出。這兩種性格根本合不來。所以 (C) 是死路，不是「還沒找到」，是「找不到」。

n²−n+41 的失敗，現在有了精確的解釋：它在 n=41 撞上的，正是這個「f(a)=p ⇒ p | f(a+p)」的鐵律——f(0)=41，於是 41 必整除 f(41)、f(82)、f(123)…。它能撐 41 步已經是奇蹟，但鐵律從不缺席，只是遲到。

## 存在、但廢掉的公式：Mills 常數

「沒有好用的多項式公式」之後，你可能想：那有沒有**別種**公式？有的——而且有人真的寫出了「f(n) 對每個 n 都是質數」的公式（上面分類的 (A)）。問題是它廢得很有教育意義。

1947 年米爾斯（Mills）證明了：存在一個實數常數 A，使得

```text
   ⌊ A^(3ⁿ) ⌋   對每個正整數 n 都是質數
   （⌊·⌋ 是向下取整；A ≈ 1.3063778838...，這個近似值要假設黎曼假設才算得出）
```

代進去：⌊A³⌋、⌊A⁹⌋、⌊A²⁷⌋… 全是質數。看起來這不就是質數公式嗎？

陷阱在 A 這個常數本身。**你要先知道 A 的精確值，才能用這條公式算出質數。可是要知道 A 的值，唯一的辦法是——先去找到那些質數，再反推 A。** 這是一個完美的循環論證式的廢：公式的「答案」被偷偷塞進了那個常數裡。它沒有「生成」任何質數，它只是把一串你已經得知的質數，編碼進一個小數點後無限長的數字裡。

```text
   Mills 公式的廢在哪：
     表面：A^(3ⁿ) 取整 → 質數，一條漂亮的封閉公式
     真相：A 的值 = 「把 Mills 質數序列反推出來的那個常數」
           你得先有質數，才能算出 A；
           A 不是獨立已知的，它是質數序列的「壓縮存檔」

   類比：我宣稱「f(n) = 你要的第 n 個質數」是一條質數公式。
        對啊，它確實對每個 n 給出第 n 個質數——
        但它什麼也沒告訴你，因為「你要的第 n 個質數」就是答案本身。
        Mills 常數是這個廢的、戴了微積分手套的高級版本。
```

而且 2026-06 查證：Mills 常數的可計算性還**依賴黎曼假設**——目前已算到約 6850 位小數，但全部建立在「假設 RH 為真」之上；不假設 RH，A 的值在今天的方法下完全算不出來。這把 ch13、ch14 都串起來了：連這條「存在的質數公式」，都被黎曼零點和「未證的假設」綁住。

教訓很乾淨：**「公式存在」和「公式有用」是兩回事。** 質數確實有公式（Mills、Wilson 定理改寫的、各種編碼把戲），但它們無一例外地把難處藏進某個你算不出或必須先知道答案才能填的地方。難沒有消失，只是被搬家了——這跟你做效能優化時「把延遲從 A 環節搬到 B 環節，但總成本沒降」是同一種既視感。

值得多停一下這個「難只是搬家」的型態，因為它在數論裡反覆出現、而且工程師特別容易看穿它。威爾遜定理（Wilson's theorem）是另一個漂亮的例子：它說「p 是質數，若且唯若 (p−1)! ≡ −1 (mod p)」。這是一條乾淨的「質數判定公式」——而且是雙向的，不像費馬小定理那種會被偽質數騙過的單向測試（ch10），威爾遜定理是充要條件，沒有漏洞。聽起來完美，對吧？但它一樣是廢的，廢在 (p−1)! 這個階乘：要判斷 p 是不是質數，你得算 (p−1) 個數連乘、再取模——這比直接拿 p 去試除還慢得多。它把「判定質數」這件難事，翻譯成了「算一個巨大的階乘」這件**更難**的事。它在邏輯上無懈可擊、在計算上一文不值。Mills 把難搬進一個算不出的常數、威爾遜把難搬進一個算不動的階乘——形狀不同，但本質同一：**質數的難是守恆的，公式只能搬運它、不能消滅它。** 你會在效能調校、在分散式系統的 CAP 取捨裡反覆遇到同型的事——某個成本看起來被消掉了，仔細一看只是換了個地方冒出來。質數公式給你的，是這個守恆律在純數學裡最乾淨的標本。

## 梅森數不是質數公式

順帶處理一個常見的誤會：很多人以為梅森數 `2^p − 1` 是某種「質數公式」。把這個說清楚也順便給 ch16 鋪路。

```text
   梅森數：Mₚ = 2^p − 1（p 是質數）

   p=2  → 2²−1 = 3      質
   p=3  → 2³−1 = 7      質
   p=5  → 2⁵−1 = 31     質
   p=7  → 2⁷−1 = 127    質
   p=11 → 2¹¹−1 = 2047  合！ = 23 × 89   ← 公式在這裡就破了
```

（複核 p=11：2¹¹=2048，2048−1=2047；2047÷23=89，23×89=2047 ✓ 合數。這是「指數質、結果未必質」的最小反例，1536 年 Regius 就指出了。）

所以 `2^p−1` 不是質數生成公式——它生出的數**多數是合數**，質數（梅森質數）反而是稀有的例外，稀有到全世界用分散式計算搜了幾十年才找到 52 個（ch16 詳談）。`2^p−1` 給的不是「保證質數」，而是「一個值得拿去檢驗的候選人」——它的價值在於梅森數有特別快的質數性檢驗（Lucas–Lehmer），不在於它是公式。把「梅森數」當「質數公式」，就像把「履歷投遞」當「錄取通知」。

## 把刀磨利：沒有公式 ≠ 沒有規律

現在到本章的核心、也是脊椎張力的最後一次辨析。前面我們確認了：

- 沒有簡單多項式對所有 n 吐質數（可證，(C)）；
- 有的公式（Mills）存在卻廢（(A)）；
- 沒有已知的好用公式算第 n 個質數或預測下一個質數（(B)(D)）。

把這些加起來，很多人的結論是：「所以質數是隨機散落的、毫無規律。」**這一步是全書最該被攔下來的錯誤推論。** 它把「無生成公式」偷換成「無規律」，而這兩者根本不是一回事。

回到「從你已知的出發」的 hash 直覺，把它整理成一張對照表——這是 T8 的正確版：

```text
   「無公式」 ≠ 「無規律」：質數同時是這三件事

   ┌──────────────────┬──────────────────────────────────────────────┐
   │ 無「生成公式」   │ 沒有「代入 n → 吐出第 n 個質數」的好公式     │
   │                  │ 沒有多項式對所有 n 吐質數（已證不可能）      │
   │                  │ 沒有封閉式精準預測「下一個質數在哪」         │
   ├──────────────────┼──────────────────────────────────────────────┤
   │ 卻有「規律」     │ PNT：π(x) ~ x/ln x（整體計數的鐵律，ch07）   │
   │                  │ 平均間隙 ~ ln x（ch12）                      │
   │                  │ Ulam 螺旋上的對角直線（肉眼可見，ch18）      │
   │                  │ 孿生質數密度的 H–L 啟發式（ch12）            │
   ├──────────────────┼──────────────────────────────────────────────┤
   │ 而且「確定」     │ 每個整數是不是質數，是百分百確定的事實       │
   │                  │ 零隨機性——「像隨機」只是我們算不出的表象     │
   └──────────────────┴──────────────────────────────────────────────┘
```

三件事同時為真。質數**沒有**讓你一步算出個別位置的公式（亂），但它的**整體行為**服從 PNT 這條光滑到能畫成曲線的統計律（律），同時每個質數的身分又是**鐵板釘釘的確定**（不是擲骰子)。這正是脊椎張力「亂與律共存」最精煉的版本：亂在「個別位置無公式」，律在「整體分布有 PNT」，而貫穿兩者的是「確定但測不準」。

我認為這是整本書最該被工程師記住的一句話：**「算不出個別」和「沒有規律」是兩件不同的事。** 你早就接受了亂流（湍流）每個分子的軌跡算不出、卻服從統計力學的規律；你早就接受了單一請求的延遲測不準、總流量卻能容量規劃（這呼應《馴服隨機》「個體不可測、總體可測」那條線）。質數要你接受的，是同一型的事——只是它更極端，因為質數連「隨機」都不是，它是**完全確定卻看起來像隨機**。確定性沒有買到可預測性，這件事本身就值得震撼。

再把這個分界推到底，因為它容易被輕輕滑過去。「確定」這個詞在這裡是強意義的：給你任何一個整數，比方說 1000003，它是不是質數，這個問題**沒有任何不確定性**——它要嘛是、要嘛不是，答案在這個整數被寫下來的那一刻就已經是死的，不依賴你怎麼問、用什麼工具、有沒有運氣。這跟擲骰子根本不同層次：骰子的結果在落定前真的是開放的，質數的身分卻從來不開放，只是**藏著**。我們缺的不是「結果還沒發生」，而是「快速看穿它的方法」。所以當有人說「質數看起來很隨機」，精確的講法應該是「質數的局部分布**算不出捷徑**，所以看起來像隨機」——但「像隨機」和「是隨機」之間，隔著整個確定論。這個區別在工程上你其實天天用：一個雜湊值在 key 固定後就完全確定了，沒有任何隨機性，但它「看起來像亂數」，好到你可以拿它當偽隨機數來分桶、來抽樣。質數是同一回事的純數學版——它是宇宙裡最確定的東西之一，卻把自己的規律藏在一個我們至今打不開的尺度上。亂與律共存的最終真相，不是「一半亂一半律」，而是「同一批完全確定的數，在個別尺度看像亂、在整體尺度看是律」——尺度，才是那把分開亂與律的刀。

## 直覺的陷阱

| 常見誤解 | 哪一步出錯 | 正確版（怎麼自我察覺） |
|---|---|---|
| 「質數沒有公式，所以是隨機的、沒規律」 | 把「無生成公式」直接等同「無規律」 | 兩者不同：無公式說的是「算不出個別」，但 PNT、Ulam 直線都是規律（T8）。徵兆：你能不能舉出一條質數的規律？舉得出（PNT）就知道「沒規律」是錯的 |
| 「n²−n+41 是質數公式」 | 看它連對 41 個就以為對所有 n 都對 | 它在 n=41 必破（1681=41²），且**任何**整係數多項式都會在某處吐合數（可證）。徵兆：你有沒有問「它在哪裡破、為什麼一定破」 |
| 「Mills 常數是真正的質數公式」 | 沒看出 A 必須先從質數反推 | 公式存在但廢——A 是質數序列的壓縮存檔，你得先有質數才知道 A；難只是被搬進常數裡（且依賴 RH） |
| 「2^p−1 是梅森質數公式」 | 把候選人當保證 | 多數梅森數是合數（2¹¹−1=2047=23×89 即破）；它給候選、不給保證 |
| 「既然 PNT 給了規律，應該能反推出質數公式」 | 把「統計規律」誤當「個別預測」 | PNT 給的是平均密度（整體），不給第 n 個質數的位置（個別）；統計律不等於生成公式（同 ch07 T5「漸近非相等」的精神） |

最深的那個陷阱是第一個，它是整個 Part IV 的總陷阱：把「測不準個別」滑進「無規律」。每次你想說「質數是隨機的」，先問自己——你說的是「我算不出下一個」（真），還是「它沒有任何規律」（假）？這兩句話差了一整本書。

## 紙上推演

### 推演題 1（★，[10 分鐘]）

不查上面的表，自己手算 `n²−n+41` 在 n=0, 1, 2, 3, 4 的值，確認都是質數；再算 n=41 的值，確認它是合數並寫出它的分解。最後用一句話回答：為什麼 n=41 一定會破？（提示：f(0) 等於多少，這個數和 41 有什麼關係。)

### 推演題 2（★★，[15 分鐘]）

向一個剛聽說「質數沒有公式」就斷言「所以質數是隨機的」的工程師朋友，反駁他。要求：(a) 承認「沒有好用的生成公式」這部分他說對了；(b) 舉出**至少兩條**質數確實服從的規律；(c) 用一個他熟悉的工程類比（hash、亂流、延遲分布任選），說清楚「算不出個別」和「沒有規律」為什麼是兩件事。

### 推演題 3（★★，[15 分鐘]）

有人給你看 Mills 公式 `⌊A^(3ⁿ)⌋ 對每個 n 都是質數`，宣稱「質數公式找到了，數論的謎解開了」。請指出這個宣稱的破綻：這條公式為什麼「存在卻沒用」？它把「難」搬到了哪裡？（不需要任何複變分析，只需要想清楚「要用這條公式，你得先知道什麼」。)

### 推演解答

**第 1 題。** 手算（n²−n 再加 41，每步複核）：

```text
   n=0：0−0+41 = 41         （質數）
   n=1：1−1+41 = 41         （質數；n=0,1 同給 41，因 n²−n 都是 0）
   n=2：4−2+41 = 43         （質數）
   n=3：9−3+41 = 47         （質數）
   n=4：16−4+41 = 53        （質數）

   n=41：41²−41+41 = 1681
        1681 = 41 × 41      （合數）
```

為什麼 n=41 一定破？因為 f(0)=41，而上一節證過的鐵律說「f(a)=p ⇒ p 整除 f(a+p)」。這裡 a=0、p=41，所以 41 必整除 f(0+41)=f(41)。果然 f(41)=1681=41²被 41 整除，是合數。常見錯路：以為「連對 40 多個就快證明它永遠對」——恰恰相反，這個多項式能撐到 41 已是奇蹟，但模 41 的週期性保證它逃不過 n=41 這一關。

**第 2 題。** 一個合格的反駁大致是：「你說『沒有一條代進去就吐出第 n 個質數的好公式』——這部分你對，連『沒有任何整係數多項式能對所有 n 吐質數』都是能證明的。但你下一句『所以質數是隨機的、沒規律』就跳錯了。質數有規律，而且是鐵一般的規律：第一，質數定理 π(x) ~ x/ln x——小於 x 的質數個數服從一條光滑曲線，誤差小到能畫出來；第二，質數的平均間隙約是 ln x，越往上越稀但稀得有章法。這就像 hash：你不能從雜湊值反推 key（沒有反推公式），但 hash 輸出均勻撒滿空間、雪崩效應、完全確定——它規律得很，你天天靠這些規律做分桶。質數也一樣：算不出『這一個』在哪，不代表『整體』沒規律。『測不準個別』和『沒有規律』是兩件事。」常見錯路：只反駁「有公式」而忘了對方真正的錯是「無公式⇒無規律」這個推論——要攻的是那個箭頭，不是前提。

**第 3 題。** 破綻在常數 A。這條公式要算出質數，你**必須先知道 A 的精確值**；但 A 沒有獨立的來源——求 A 的唯一辦法，是先把那串 Mills 質數找出來，再反推回去逼近 A。所以這是循環：你得先有質數，才有 A，才能用公式「算出」質數。它沒有生成任何東西，只是把一串已知的質數壓縮編碼進一個小數點後無限長的常數裡。「難」被搬到了「求 A」這一步，而求 A 的難，正好等於原本「找質數」的難——一點都沒少（而且 2026-06 連算 A 都要假設黎曼假設）。常見錯路：被「封閉公式」的外表唬住，忘了問「要用它，我得先知道什麼」——一旦問了，常數裡藏的循環就現形。類比：宣稱「f(n)=第 n 個質數」是公式——對，但廢，因為答案就是它自己。

## 自我檢核

口頭自答，講不清楚就是還沒懂：

1. 「質數公式」這句話可以拆成哪幾種不同難度的問題？哪一種是「可證明不存在」的？
2. n²−n+41 為什麼這麼誘人？它在 n 等於多少時破？為什麼那一步**一定**會破（用 f(0) 解釋）？
3. 「任何整係數非常數多項式都會在某處吐出合數」——這個證明的核心一句話是什麼？（提示：f(a)=p 之後，p 對哪些輸入也整除？）
4. Mills 常數提供了一條「對每個 n 都吐質數」的公式，為什麼它沒用？它把難搬到哪裡了？跟 RH 有什麼關係？
5. 為什麼 2^p−1 不是質數生成公式？舉最小的反例。
6. 「沒有生成公式」和「沒有規律」差在哪？各舉一個質數「無公式」的面向、和一個「有規律」的面向。
7. 把這章扣回 hash 的直覺：hash 哪三個性質，正好對應質數的「無公式／有規律／確定」？
8. 用一句話講清「亂與律共存」在本章的最終形式——亂在哪個尺度、律在哪個尺度、貫穿兩者的「確定」又是什麼意思？
