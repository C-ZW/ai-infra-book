# ch07 — 質數定理 PNT：亂中的鐵律

> **本章解決什麼問題**：上一章（ch06）我們量出了一條觀察——在 x 附近，質數平均每隔 ~ln x 才冒一個，而那欄「實測間隔 ÷ ln x」的比值悄悄爬向 1 卻還沒到（停在 0.92）。本章把這條觀察釘成定理：質數定理（prime number theorem, PNT），π(x) ~ x/ln x。這是全書第一個高潮，也是中央張力的數論巔峰——**個別質數的位置沒有公式、測不準，整體計數卻服從一條光滑到能寫成方程式的鐵律**。亂與律，在同一批數上同時為真。本章是 Part II 的收尾：我們從「質數有多稀」（ch04–06）走到「稀到一條精確曲線」，下一個 Part 換到模算術的新世界。

## 從你已知的出發

你心裡有一條信念，每天上線時都在用它：**單一請求不可預測，整體流量可規劃**。

你說不出下一筆 HTTP 請求精確在第幾毫秒到、來自哪個 IP、會打哪個 endpoint——那是噪音，賭不得。但你敢拍著容量規劃文件跟老闆說「這個服務尖峰大約 8000 QPS，P99 延遲在 120ms 上下，雙十一給我多開三台就穩」。為什麼一個你「測不準個體」的系統，你卻能對它的「總體」下這麼篤定的判斷？因為**大數一平均，噪音就被磨平，露出底下那條平滑的曲線**。你規劃的從來不是某一筆請求，是那條曲線。

質數定理（PNT）就是這件事的數論版，而且是它最純粹、最極端的版本。

把每個整數想成一筆「請求」，「是質數」想成某種事件。個別來看，第 1000001 個整數是不是質數？沒有公式能告訴你，你只能去試除——這是純粹的噪音，連 PRNG 那種「確定的假亂」都不如，質數的「下一個在哪」連個 seed 都沒有（ch15 會把這層挑明）。可是當你不問個體、改問「到 x 為止總共冒了幾個」——也就是上一章的 π(x)——噪音就被磨平了，露出一條光滑曲線：**π(x) ~ x/ln x**。

這就是 PNT 的全部震撼，我先把它擺在這裡，這一章剩下的都是在把這句話講透、講到你能轉述給另一個工程師：

> 質數確定到沒有一絲隨機（n 是不是質數是鐵板釘釘的事實，沒有任何骰子），它的「下一個在哪」卻沒有任何公式可循；可是它的**累積計數**，乖乖貼著一條兩百年才被人類證出來的光滑曲線。測不準的個體 ＋ 測得準的總體，是同一批數的兩個真相。

如果你讀過《馴服隨機》，這句話會有強烈的既視感——那本書反覆講「個體不可測、總體可測」（大數法則的精神）。差別在於：機率那本書裡個體**真的**帶隨機（骰子真的在滾）；質數這裡**個體一點都不隨機**，它確定得不能再確定，卻照樣服從一條像統計律的曲線。我認為這是質數比骰子更詭異的地方，也是這一章我最想讓你帶走的那一刀。

先把上一章交棒過來的東西擺好（你剛看過，這裡只是擦亮）：π(x) 是「不超過 x 的質數個數」的階梯函數（ch06）；「平均間隔 ~ ln x」等價於「密度 ~ 1/ln x」；那欄比值在 100 萬還停在 0.92、沒到 1。本章要回答的就是：那條曲線到底是什麼？比值為什麼「會到 1 但還沒到」？這個「還沒到」為什麼非寫成 `~`、不能寫成 `=`？

## PNT 在說什麼：一句話，與一個關鍵的波浪號

質數定理的陳述短到可以印在一件 T 恤上：

```text
        π(x)
PNT：  ────────  →  1      （當 x → ∞）
       x / ln x

等價寫法： π(x) ~ x / ln x

讀法：「π(x) 漸近於 x/ln x」
      ~ 這個波浪號念「漸近等於」（asymptotically equal），
        它說的是「兩者的比值趨近 1」，
        絕對不是「兩者相等」。
```

整章的重量幾乎全壓在那個 `~` 上，所以我要在你還沒讀任何別的東西之前，先把它釘死。**`~` 不是 `=`。** 它不保證 π(x) 等於 x/ln x，甚至不保證兩者的差距變小——它只保證兩者的**比值**趨近 1。這是個微妙但要命的區別，我用上一章那欄你已經算過的數字直接給你看證據：

```text
回放 ch06 worked example 步驟 4 的最後一欄（x/π(x) ÷ ln x）：

  x          比值
  ──────────────────
  100        0.869
  1000       0.862
  10000      0.884
  100000     0.906
  1000000    0.922   ← 正悄悄爬向 1，但在 100 萬還停在 0.92

把它倒過來看，就是 π(x) 相對 x/ln x 的比值：
  π(x) ÷ (x/ln x) 在 100 萬時 ≈ 1/0.922 ≈ 1.085
  也就是：實際質數個數，比 x/ln x 預測的多了約 8.5%。
```

看清楚了嗎？在 100 萬這個尺度，x/ln x **系統性地低估** π(x)，低了約 8.5%。PNT 不否認這件事——PNT 只說，當你把 x 推向無窮，這個比值會慢慢、慢慢地爬到 1。它不承諾在任何**有限**的 x 上兩者相等，事實上在所有你能手算的 x 上它們都不相等（在夠大的 x，例如 x ≥ 17 之後，x/ln x 系統性偏低）。這就是「漸近」的精確意義，也是本章最容易被誤讀、最該被守住的一條（landscape T5，下面〈直覺的陷阱〉會專門再夯一次）。

為什麼叫它「鐵律」？因為儘管 π(x) 這條階梯是由「每個質數在哪」這種沒有公式、純噪音的事實一格一格拼起來的，它的**整體走勢**卻被 x/ln x 這條光滑曲線鎖死——鎖到比值必然趨近 1，沒有任何餘地。你預測不了任何一階尖角，卻能斷言整條階梯的漸近形狀。這就是「亂中的律」。

## 一個更好的逼近：對數積分 Li(x)

x/ln x 抓住了 PNT 的精神，但它有個你已經看到的毛病——一路偏低。有沒有更貼的逼近？有，而且高斯少年時就猜中了它：**對數積分（logarithmic integral）Li(x)**。

先別被「積分」嚇到。它的動機其實非常工程師、非常直白。我們在 ch06 說「在 x 附近，密度約 1/ln x」。x/ln x 等於是粗暴地假設「整段 0 到 x，密度都用 x 那一點的 1/ln x」——但這顯然不對！低端（小數那帶）質數密得多，密度應該更高。x/ln x 用了**終點**那個偏低的密度去乘整段，當然會低估。

那正確的做法是什麼？工程師的本能：**別用單點密度乘區間，把區間切碎、每一小段用該段的局部密度、再加起來**。這就是積分：

```text
與其用終點的密度 1/ln x 乘整段 x（會低估，因為低端更密），
不如沿路把每個 t 處的局部密度 1/ln t 累加起來：

  Li(x) ＝ ∫₂ˣ  dt / ln t        ← 從 2 積到 x，沿路用「當地」密度

直覺：
  - 在 t 小的地方，ln t 小 → 1/ln t 大 → 那一帶質數密
  - 在 t 大的地方，ln t 大 → 1/ln t 小 → 那一帶質數稀
  - 把沿路每一小段的「當地密度」加總，得到的估計
    自然比「全程用終點那個偏低密度」更準。
```

下界從 2 起（而不是 0），是為了避開 t=1 處 ln t=0 的爆點（1/ln 1 = 1/0，發散；嚴格定義要繞過 t=1 取主值，那是《馴服無限》的細節，這裡略過，用 ∫₂ˣ 這個工程版就夠）。

關鍵直覺只有一句：**Li(x) 是「沿路用當地密度」的累加，x/ln x 是「全程用終點密度」的粗估**。前者考慮了「低端更密」這件事，所以更準。事實上 PNT 也可以、而且更精確地寫成 **π(x) ~ Li(x)**——兩個逼近漸近上等價（比值都 →1），但 Li(x) 在有限 x 貼得近得多。等一下的 worked example 你會親眼看到 Li(x) 把誤差砍掉一兩個數量級。

這裡埋一顆種子留給第二個高潮（ch13）：x/ln x 偏低、Li(x) 很貼但也不是完美——π(x) 和 Li(x) 之間還有一個**誤差項**在抖動。那個誤差項裡藏著什麼、它的大小被什麼控制？答案是黎曼 ζ 函數的零點，那就是「質數的音樂」（ch13）。本章只到「Li(x) 是更好的逼近」為止，誤差的精細結構留給黎曼那一瞥。

## worked example：三條逼近並排，看 ~ 的真身

把 PNT 的兩個逼近（x/ln x、Li(x)）跟真實的 π(x) 並排算一遍，一步不跳。基準值 π(100)=25、π(10⁴)=1229、π(10⁶)=78498 是全書跨章基準（ch06／landscape）。ln 用本書基準 ln10≈2.3026，每乘一個 10 就加一個 ln10。Li(x) 的值取自標準對數積分表（不是本書能手算的，我標明出處與量級，重點看它跟 π 多貼）。

```text
步驟 1：算 ln x（每多一個 10，ln 加 2.3026）
  ln 100     = 2 × 2.3026 = 4.6052
  ln 10000   = 4 × 2.3026 = 9.2103     ← 4×2.3026=9.2104，取 9.2103 ✓
  ln 1000000 = 6 × 2.3026 = 13.8155    ← 6×2.3026=13.8156，取 13.8155 ✓

步驟 2：算 x / ln x，逐個長除法複核
  100      / 4.6052   = 21.71
        ← 4.6052×21 = 96.71，餘 3.29/4.6052 ≈ 0.71 → 21.71 ✓
  10000    / 9.2103   = 1085.7
        ← 9.2103×1085 = 9993.2，餘 6.8/9.2103 ≈ 0.74 → 1085.7 ✓
  1000000  / 13.8155  = 72382
        ← 13.8155×72382 = 1000013（差 13，捨入誤差），→ 72382 ✓

步驟 3：把三欄並排，加上 Li(x)（取自標準對數積分表）
  x          x/ln x      π(x)（真值）    Li(x)（更佳逼近）
  ──────────────────────────────────────────────────────────
  100        21.7        25              ~29
  10000      1086        1229            ~1246
  1000000    72382       78498           ~78627

步驟 4：算兩個逼近各差多少、差多少比例
  ── x/ln x vs π(x)（看它一路低估）──
  x=100:    25 − 21.7   = +3.3      → π 比 x/ln x 多 13%
  x=10⁴:    1229 − 1086 = +143      → π 比 x/ln x 多 13%
  x=10⁶:    78498 − 72382 = +6116   → π 比 x/ln x 多 8.5%
    觀察：誤差絕對值越來越大（3→143→6116），
          但相對比例在縮（13%→13%→8.5%）——比值往 1 爬。
          ★ 這就是「漸近」：比例收斂，絕對差距不收斂。

  ── Li(x) vs π(x)（看它貼得多近）──
  x=100:    29 − 25     = 4         → Li 高估 4
  x=10⁴:    1246 − 1229 = 17        → Li 高估 17
  x=10⁶:    78627 − 78498 = 129     → Li 高估 129
    觀察：Li(x) 在 100 萬只差 129（占 π 的 0.16%），
          而 x/ln x 差了 6116（占 7.8%）。
          Li 把相對誤差砍掉約 50 倍。
```

把步驟 4 兩塊對讀，PNT 的整個面貌就清楚了。**第一塊**是 `~` 的真身：x/ln x 的絕對誤差（3→143→6116）一路在漲，不是「越來越準」——但**相對**誤差（13%→8.5%）在縮，比值往 1 爬。所以 `~` 講的死活是「比值收斂」，不是「差距收斂」，更不是「相等」。任何時候你想把 `~` 寫成 `=`，回頭看這個 6116——在 100 萬，x/ln x 漏算了六千多個質數，這不是相等。

**第二塊**是 Li(x) 為什麼配得上「更好的逼近」：它在 100 萬只差 129，相對誤差 0.16%，比 x/ln x 準了約 50 倍。原因正是上一節那句話——它「沿路用當地密度累加」，照顧了「低端更密」，所以不像 x/ln x 那樣系統性偏低。

值得圈一個微妙的轉向：x/ln x 一路**低估**（π 比它大），Li(x) 卻一路**高估**（π 比它小）。π(x) 夾在這兩個逼近之間。Li 高估這件事看起來鐵板一塊（你算得到的 x 全部 Li > π），但它其實不是永遠成立——這正是 ch14「驗證不是證明」要講的故事（Skewes 數：Li−π 的符號會在某個天文數字後翻轉，雖然在所有可手算範圍內 Li 都更大）。我把這顆種子先埋在這個 129 旁邊。

## 兩百年的路：從少年高斯到 1896 的兩封捷報

PNT 不是一個人在一個下午想出來的。它是一條**先被猜、過了一百年才被證**的定理，而這段歷史本身就是「亂與律」的縮影——人類花了一百年才搞清楚，那條曲線為什麼非長成 x/ln x 不可。我照 landscape 的事實基準走（2026-06 查證），把年表攤平給你：

```text
PNT 的兩百年（年份／歸屬以 landscape 為準，2026-06）

  18 世紀末   高斯（Gauss，約 1792–93，十五歲上下）
              與勒讓德（Legendre）各自從質數表「猜」出 π(x) 與
              x/ln x 同階；高斯更以 Li(x) 形式記下（更準的版本）。
                 ↓ 只是猜想，沒有證明，整整一個世紀
  1850 年代   柴比雪夫（Chebyshev）證出 π(x) 被夾在
              c₁·x/ln x 與 c₂·x/ln x 之間（c₁,c₂ 是具體常數，
              都接近 1）——確立了「階」是對的，但比值是否
              恰好 →1 還沒證下來。離 PNT 一步之遙。
                 ↓
  1859       黎曼（Riemann）一篇 6–9 頁論文，把 ζ 函數延拓到
              複平面、寫出 π(x) 的顯式公式——架好了通往 PNT
              （與後面 ch13 那座高潮）的橋，但他本人沒證 PNT。
                 ↓
  1896       阿達瑪（Hadamard，31 歲）與德拉瓦萊普桑
              （de la Vallée Poussin，30 歲）在同一年、各自獨立
              證出 PNT。關鍵的最後一步：證明 ζ(s) 在直線
              Re(s)=1 上沒有零點，再由此推出比值 →1。
```

我想停在 1896 那一格多講一句，因為它有種數學史特有的戲劇性：**兩個三十出頭的年輕人，在同一年、互不知情、各自走通了同一條一百年沒人走通的路**。這不是巧合——黎曼 1859 的那座橋（ζ 與質數的連結）擺在那裡三十多年，工具終於成熟到「臨門一腳」可以被踢進去，於是兩個人幾乎同時踢進。他們的證明都靠複變分析（complex analysis）——要證的核心是「ζ(s) 在 Re(s)=1 上無零點」，這是個關於複數函數的命題。這條路為什麼能從「ζ 的零點」走到「質數的計數」，是解析數論的深水區，本書不游進去（指向《馴服無限》的 ζ、解析延拓、頻譜工具）。

這裡我要誠實標示嚴謹度：**本書不證 PNT。** 證明要動用複變分析，遠超「工程師能一步步看懂」的範圍——這跟前面歐幾里得質數無窮（ch03）、FTA 核心引理（ch02）那種「短到能完整寫透」的證明是兩個量級。對 PNT，我只給你陳述、給你逼近、給你年表、給你「為什麼 Li 更準」的直覺；嚴格證明指向延伸閱讀與《馴服無限》。把直覺論證偽裝成證明是我絕不會做的事——你現在懂的是 PNT 在說什麼、為什麼震撼，不是 PNT 怎麼被證出來，這個界線我要劃清楚。

順帶一個值得記住的後話：1896 那兩個證明都繞了複變分析。很多人覺得「一個關於整數的定理憑什麼非要用複數來證」很不對勁——這份不對勁催生了半個世紀後（1948–49）Selberg 與 Erdős 的「初等證明」（不用複變分析）。但那是另一個故事了，這裡只提一句，讓你知道 PNT 至今仍有人在用新方法重證（指向延伸閱讀的 Quanta 那篇）。

## 驚嘆點：個別測不準，平均卻精準到可寫成方程式

讓我把這一章的震撼直接擺到桌面，因為這是本書的核心商品，我要它能被你轉述：

```text
同一批數，兩個都對的真相：

  亂（局部 / 個體）           律（整體 / 平均）
  ──────────────────────     ──────────────────────
  下一個質數在哪？無公式      π(x) ~ x/ln x（甚至 ~ Li(x)）
  第 n 個質數精確值？算不出   平均密度 ~ 1/ln x，精準可測
  間隔忽大忽小（孿生↔沙漠）   累積計數貼著光滑曲線
  個體確定但不可預測          總體服從兩百年才證出的鐵律
```

我認為這是整本書前半最美的一頁，原因是這個對比的「強度」：**質數不是『有點亂、大致有律』，它是兩個極端同時拉滿**——個別位置亂到連個生成公式都沒有（連 PRNG 的 seed 都沒有），整體計數律到能寫成一行方程式並被嚴格證明。換成你熟的語言：這像是一個系統，每一筆請求到達時刻都是真正的、不可預測的噪音（不是偽隨機，是連 seed 都沒有的那種），可是它的累積 QPS 曲線卻光滑、可寫成公式、而且這條公式有人花了兩百年才證明它必然如此。如果真有這種系統，你會覺得它詭異到不可思議——質數就是這個系統，而且它是純數學的、確定的、一點隨機都沒有的。

這一刀也是全套六本書共享的母題的數論版。混沌那本講「決定卻不可預測」（方程式完全確定，長期軌跡卻測不準）；機率那本講「個體不可測、總體可測」（骰子個別賭不得、大數平均穩如鐵）。質數這裡是兩者的混血又比兩者更極端：**完全確定（比混沌還死板，沒有微分方程，就是一條整數線）、個體不可預測（比混沌更徹底，連公式都沒有）、總體卻服從鐵律（比機率更詭異，因為這裡根本沒有隨機源）**。同一個震撼結構，質數是它最純的標本。

## 直覺的陷阱

PNT 是全書「最常被講錯一句話」的定理。守住下面這幾條（對齊 landscape T5、T3）。

| 錯誤直覺 | 它在哪一步帶你下溝 | 正確版（怎麼自我察覺） |
|---|---|---|
| 「π(x) ~ x/ln x 就是 π(x) = x/ln x」 | 把「漸近」當「相等」，把波浪號讀成等號 | `~` 只保證**比值** →1，不保證相等、甚至不保證差距變小。worked example 裡 x=10⁶ 時 x/ln x 漏算了 6116 個質數——這不是相等。徵兆：你想把 `~` 寫成 `=`，或想用 x/ln x「算出」π(10⁶) 的精確值——停，它給的是漸近、不是等號。 |
| 「PNT 能算出第 n 個質數」 | 把「平均密度的定理」誤推成「質數位置的公式」 | PNT 量的是**有多少**（計數的漸近），不是**在哪裡**（個別位置）。它連「第一百萬個質數」都只能估到量級（ch06 推演題 3 估 1.38×10⁷，真值 1.55×10⁷），給不出精確值。徵兆：你以為 PNT 是個「質數生成器」——它不是，質數沒有生成公式（ch15）。 |
| 「PNT 是個近似，所以不夠嚴謹／是經驗公式」 | 把「漸近陳述」誤當「湊出來的配適曲線」 | PNT 是被**嚴格證明**的定理（1896，Hadamard／de la Vallée Poussin），不是經驗配適。「漸近」是一個有精確定義的數學陳述（比值極限 =1），它的「不等於」恰恰是它**嚴謹**的一部分，不是它的破綻。徵兆：你把它跟工程裡「大概湊一個公式 fit 一下數據」混為一談——層級完全不同。 |
| 「PNT 已證，所以質數分布已經被完全搞懂了」 | 把「平均被馴服」誤當「細節也被馴服」 | PNT 馴服的是**平均行為**（計數的主項）。π(x) 與 Li(x) 之間的**誤差**有多大、孿生質數能多近、間隙能多遠——這些**細節全是未解的謎**（Part IV）。徵兆：你以為證了 PNT 就等於懂了質數——平均被收服了，魔鬼還住在誤差項裡（ch13 的黎曼假設正是在問這個誤差）。 |

最該被點名的具名陷阱是第一條：**把 `~` 讀成 `=`**。它之所以致命，是因為它表面上「差不多」——x/ln x 確實跟 π(x) 同量級、確實越來越「像」，所以人很容易心一鬆就把波浪號抹成等號。但這一抹，你就同時丟掉了兩件本書最重要的東西：一是 PNT「漸近不是相等」的精確意義（它正是嚴謹所在），二是那個「π 與逼近之間的誤差」——而那個誤差，恰恰是第二個高潮（ch13 黎曼假設）的全部主題。守住 `~`，你才接得住後面的戲。自我察覺的徵兆很簡單：**只要你在紙上寫下 π(x) = x/ln x 這個等號，你就錯了**；它永遠是 `~`。

## 紙上推演

全程紙筆，不要去跑任何程式。ln 用本書基準（ln10≈2.3026、ln100≈4.6052、ln(10⁶)≈13.8155）。

### 推演題 1（★★，[15 分鐘]）

你的同事在 code review 裡寫了一行註解：「質數定理告訴我們 π(x) = x/ln x，所以 π(10⁶) = 1000000/13.8155 ≈ 72382 個質數。」用本章學到的東西，指出這句話**兩個**問題（一個是符號用錯，一個是數值結論被它帶歪），並算出「被漏掉」的誤差到底有多大。

### 推演題 2（★★，[15 分鐘]）

PNT 有一個常被引用的推論：**第 n 個質數 pₙ ~ n·ln n**。請從 PNT（π(x) ~ x/ln x）出發，自己把這個推論推一次（ch06 推演題 3 你已經摸過門把，這裡寫完整），然後用它估「第 1 億個質數（n=10⁸）大約多大」。

### 推演題 3（★★★，[20 分鐘]）

用你自己的話，向一個只懂後端、沒碰過數論的工程師，講清楚「為什麼 PNT 說的 `~` 不能寫成 `=`」。要求：（a）不准用「漸近」這個詞（他不懂），只能用比值、極限、或你們系統裡的類比；（b）要能解釋「為什麼絕對誤差越來越大，卻還說『越來越準』」這個聽起來矛盾的事。把你的講法寫成三到五句話。

### 推演解答

**推演題 1。**

```text
問題一（符號）：把 ~ 寫成 = 是錯的。
  PNT 是 π(x) ~ x/ln x（漸近，比值→1），不是 π(x) = x/ln x（相等）。
  在有限的 x，兩者並不相等。

問題二（數值結論被帶歪）：72382 不是 π(10⁶)，是 x/ln x 的值；
  真實的 π(10⁶) = 78498（全書基準）。
  註解把「x/ln x 的估計值」當成了「質數的精確個數」。

被漏掉的誤差：
  π(10⁶) − x/ln x = 78498 − 72382 = 6116
  占真值比例 = 6116 / 78498 ≈ 0.0779 ≈ 7.8%

  也就是說，這行註解用 x/ln x 當等號，
  一口氣漏算了 6116 個質數（近 8%）。
  正確的講法：π(10⁶) ≈ x/ln x，且 x/ln x 系統性低估約 8%；
            想更準該用 Li(10⁶) ≈ 78627（只差 129）。
```

常見錯路：只抓到「符號錯」就收手，沒去算那個 6116——但本章的整個重點就是「`~` 與 `=` 的差，是一個**看得見、算得出**的數（6116 個質數），不是吹毛求疵的符號潔癖」。能把這個 6116 算出來並說出它占 7.8%，你才真的懂了波浪號為什麼非寫不可。

**推演題 2。**

```text
設第 n 個質數是 pₙ。「到 pₙ 為止恰好有 n 個質數」，即：
  π(pₙ) = n

把 PNT 的逼近 π(x) ≈ x/ln x 代入 x = pₙ：
  n ≈ pₙ / ln(pₙ)
  →  pₙ ≈ n · ln(pₙ)            …（★）

(★) 右邊還有 pₙ，要把它換掉。技巧：pₙ 比 n 大，但只大個
「對數倍」（因為質數稀疏的代價就是一個 ln），取對數後差距更小，
所以 ln(pₙ) ≈ ln(n · ln n) = ln n + ln(ln n) ≈ ln n
（ln(ln n) 相對 ln n 小很多，一階近似丟掉）。代回 (★)：

  pₙ ≈ n · ln n              ← 第 n 個質數的漸近估計

估第 1 億個質數（n = 10⁸）：
  ln(10⁸) = 8 × 2.3026 = 18.421
  p_{10⁸} ≈ 10⁸ × 18.421 = 1.842 × 10⁹ ≈ 18.4 億

  （查核：實際第 1 億個質數是 2,038,074,743 ≈ 2.04×10⁹。
   估計 1.84×10⁹ 偏低約 10%——偏低的原因是推導時把 ln(pₙ) 近似為
   ln n，但 pₙ > n，所以 ln(pₙ) > ln n，這一步引入了低估誤差：
   真實 pₙ ≈ n·ln(pₙ) > n·ln n。
   量級漂亮命中，10% 的偏差正是「漸近、非相等」的代價。）
```

常見錯路一：以為「第 n 個質數 ≈ n」（把質數當成跟整數一樣密）——錯了整整一個 ln n 倍，這個 ln n 倍正是「質數稀疏」的全部代價。常見錯路二：在 (★) 那步卡住，不敢把 ln(pₙ) 近似成 ln n——其實這正是漸近分析的家常手法，「對數的對數可以忽略」。能自己走完 π(pₙ)=n → pₙ≈n·ln n，你就把 PNT「反過來用」了一次。

**推演題 3。**

這題沒有標準答案，下面是一個能過關的範例講法（你的講法只要守住「比值 vs 差距」「比值趨近 1」「不准說相等」就算過）：

```text
範例講法（給後端工程師）：

「你看 metric 的時候，從來不看『絕對誤差』，看的是『誤差占比』
 對吧？質數定理就是這個意思。x/ln x 是個估計值，它估的質數個數
 跟真值的【差距】其實越拉越大——到一百萬時差了六千多個——但這六千
 多個占真值的【比例】卻越來越小（從 13% 掉到 8%，還在掉）。

 它說的『越來越準』，準的是【比例】不是【差距】：你把 x 推到無窮，
 估計值除以真值會無限逼近 1，可是它倆永遠不會真的相等，就像你的
 P99 估計可以無限逼近真實 P99、卻永遠差那麼一點點。

 所以不能寫等號——寫等號等於宣稱『差距是零』，但差距明明在變大；
 能寫的只有『比值 →1』，那個波浪號就是這個意思。」
```

常見錯路：講成「x/ln x 約等於 π(x)，誤差很小可忽略」——這反而錯了，因為**絕對誤差不小、還在變大**（6116 並不小）。要過關，必須講清楚「準的是比例、不是差距」這個反直覺的點。能把「絕對誤差變大卻說越來越準」這個表面矛盾講順，你就真的拿捏住「漸近」了——而你連「漸近」這個詞都沒用到。

## 自我檢核

口頭自答，講得出來才算過。優先回答「為什麼」。

1. PNT 的陳述是什麼？那個 `~` 為什麼**不能**寫成 `=`？用一個具體數字（worked example 裡的）說明「不相等」到底差多少。
2. 「絕對誤差越來越大，卻說 x/ln x 越來越準」——這句話為什麼不矛盾？準的是什麼、不是什麼？
3. 為什麼 Li(x) 比 x/ln x 準？用「沿路用當地密度累加 vs 全程用終點密度」這組對比講一遍。
4. x/ln x 一路**低估** π(x)、Li(x) 一路**高估** π(x)——π(x) 夾在中間。你能說出 x/ln x 為什麼偏低嗎？（提示：它用了哪一點的密度去乘整段？）
5. PNT 是「猜想」還是「被證明的定理」？被誰、在哪一年證的？關鍵的最後一步（關於 ζ 函數）是什麼？本書為什麼**不**展開這個證明？
6. 本章的驚嘆點：質數「個別測不準、整體測得準」——這跟你做容量規劃時「單筆請求不可測、整體流量可規劃」的信念，是同一回事嗎？哪裡相同、哪裡質數更詭異？（提示：質數有沒有隨機源？）
7. 「PNT 已證」是不是等於「質數分布已被完全搞懂」？哪一部分被馴服了、哪一部分還是謎？（這題接 Part IV）

- **Wolfram MathWorld, "Prime Number Theorem"**（https://mathworld.wolfram.com/PrimeNumberTheorem.html）：PNT 的精確陳述、x/ln x 與 Li(x) 的並列、歷史與各種等價形式；想看本章 worked example 之外更多基準值與逼近比較，從這裡入手。
- **Tom Apostol, "A Centennial History of the Prime Number Theorem"**（https://calteches.library.caltech.edu/3832/1/Apostol.pdf）：從高斯／勒讓德的猜想，到柴比雪夫、黎曼、1896 年 Hadamard 與 de la Vallée Poussin 的兩封捷報，一篇可讀性高的百年史；想把本章那張年表讀厚，讀這篇。
- **Quanta Magazine, "Mathematicians Will Never Stop Proving the Prime Number Theorem"**（https://www.quantamagazine.org/mathematicians-will-never-stop-proving-the-prime-number-theorem-20200722/）：為什麼一個 1896 就證完的定理，至今仍不斷被人用新方法重證（複變分析、初等、Tauberian…）；想感受 PNT 在數學家心中的份量，讀這篇科普。
- **《馴服無限》（calculus）談對數、漸近與積分的那部分**：ln x 為什麼漲得這麼慢、「漸近 ~」的極限定義、Li(x)=∫₂ˣ dt/ln t 這個積分在說什麼、ζ 函數的解析延拓——本章把這些當結果借用，想看它們怎麼長出來、PNT 的證明為什麼非走複變分析不可，回那本書。
- **《馴服隨機》（probability）談「個體不可測、總體可測」（大數法則精神）的那條線**：本章的核心震撼是它的數論版，但質數更詭異——那本書裡個體**真的**有隨機源（骰子在滾），質數這裡個體**完全確定**卻照樣服從統計律。兩相對讀，最能咀嚼出質數「無隨機源卻有統計律」這一刀的怪。
- 接著讀本書 **ch08**：本章收掉了 Part II（質數有多稀、稀成一條什麼曲線）。下一章換場景——把無窮的整數線摺進一個有限的時鐘（同餘／模算術），那是你天天在 `x % n`、環形緩衝裡寫的世界，數論版即將登場。
