# ch17 — RSA 一瞥：把分解變成鎖

> **本章解決什麼問題**：前面整個 Part III 把模算術的工具備齊了——同餘（ch08）、輾轉相除與貝祖（ch09）、費馬小定理（ch10）、歐拉定理與 φ（ch11）。這一章把這些工具一次扣起來，看它們合起來做成了人類用質數造出的最有用的東西：RSA 公鑰密碼。核心直覺只有一句——**兩個大質數相乘很容易，把乘積分解回去極難**——這個「容易去、難回來」的不對稱，加上歐拉定理給的一道後門，就變成全世界的鎖。本章給陷門（trapdoor）的直覺、用全書統一的玩具金鑰一步步手算加解密、講清楚「為什麼私鑰能解、別人不能」，並守住一件常被講錯的事：RSA 的安全**不是被數學證明過的**，是一個還沒被攻破的經驗難度。

## 從你已知的出發

你用過 TLS、用過 SSH key。你心裡對「公鑰／私鑰」有一個操作層的圖像：有一把公鑰可以到處貼、貼在 GitHub 上、印在名片上都無所謂；還有一把私鑰你藏在 `~/.ssh/` 裡、設成 `chmod 600`、丟了就完蛋。任何人拿公鑰可以加密一段訊息給你，但只有你拿私鑰解得開。你天天在用這個性質，但你大概從沒推過底下那數學——它到底憑什麼「人人能鎖、只有一人能開」。

你對「單向」這件事其實也有肌肉記憶。你算過雜湊：丟一個字串進 SHA-256，秒出一串 32 bytes；可是要從那串 32 bytes 倒推回原字串，沒有捷徑，只能暴力猜。雜湊是**單向函數（one-way function）**——去容易、回來難。RSA 跟雜湊共享這個「去容易回來難」的骨架，但它多了一個雜湊沒有的東西：**一道後門**。雜湊是真的回不去（設計上就不該回得去）；RSA 是「沒鑰匙回不去，有鑰匙一秒回」。這種「對全世界單向、對持鑰人雙向」的函數，密碼學叫它**陷門函數（trapdoor function）**——一個你能正著算、但只有握著祕密的人能反著算的函數。陷門這個概念是 Diffie 與 Hellman 1976 年提出的（他們指出「應該存在這種函數」但沒給出一個），RSA 是第一個真正做出來、能用的陷門函數實例。

而那道後門是什麼？這就是這一章最漂亮的地方：**後門就是 n=pq 的兩個質因數 p 和 q**。把 p、q 相乘得到 n，你拿乘法表就能做（容易去）；但給你 n，要找回 p、q——那是質因數分解，當 n 夠大時，這是人類目前不會快速解的問題（難回來）。知道 p、q 的人能算出一個解密指數 d，不知道的人算不出。**質數的不可分性（ch01、ch02 講的「乘法原子」）在這裡第一次變成一把鎖的核心**：因為一個合數只有唯一一種拆成質數的方式（算術基本定理），而「找出那種方式」恰好是難的，這個難就成了你 HTTPS 連線背後的那把鎖。本章要做的，就是把這條從「質數很難分解」到「全世界的鎖」的路，用一個你手算得出來的玩具一步步走完。

先把術語擦亮：陷門函數（trapdoor function）＝正向容易、逆向需要祕密才容易的函數；公鑰（public key）＝公開、用來加密的那把（RSA 裡是一對數 (e, n)）；私鑰（private key）＝保密、用來解密的那把（是 d，配合 n）。本章 e 一律指 RSA 公鑰指數（不是自然常數，本書自然對數場合一律用 ln，見附錄 A 記號歧義專節）；d 指私鑰指數；p、q 在這章是 n 的兩個質因數。

## 那個不對稱：相乘 vs 分解

整個 RSA 站在一個極度不對稱的事實上。先把這個不對稱看清楚，後面所有東西都好懂了。

```text
方向一（容易）：給你 p 和 q，算 n = p·q
  p = 61,  q = 53   →   n = 3233       ← 一次乘法，瞬間
  就算 p、q 各有幾百位數，相乘也只是大數乘法，電腦一眨眼就好。

方向二（難）：給你 n，找回 p 和 q
  n = 3233   →   ?·? = 3233            ← 小數還能試除找出 61·53
  n = 一個 617 位數（2048-bit）→ 目前無已知的快速古典演算法
```

你對「乘法很快、分解很慢」可能覺得理所當然，但值得停一下確認這個不對稱有多懸殊，因為 RSA 的整條命脈都吊在它上面。**乘法的成本隨數字位數平緩地長**（兩個 d 位數相乘，課本演算法 O(d²)，更聰明的演算法更快）；**而把一個大合數分解成質因數，目前最好的古典演算法（數體篩，number field sieve）成本是次指數級的**——隨位數增加時爆炸得遠比多項式快。這個成本上的鴻溝，就是 RSA 鎖的全部力氣來源。

這不是紙上談兵。一個具體的、有日期的座標：RSA 公司曾公開懸賞分解一串特定的合數（RSA Factoring Challenge）。其中 **RSA-250——一個 250 位十進位（829 位元）的合數——在 2020-02 才被分解出來**，動用了大約 2700 個 CPU 核心年（core-years）的算力、跑數體篩演算法（2026-06 為止這仍是公開挑戰中被古典電腦分解的最大者）。換算一下那個重量：2700 核心年，意思是 2700 顆 CPU 連跑一整年，或一萬顆跑三個多月。而這還只是 829 位元——今天實務上 RSA 金鑰至少 2048 位元，差距是天文數字。**你每天的 HTTPS 連線，安全感就建立在「沒有人想花這個代價、也沒有已知的捷徑能省下這個代價」上**。記住「沒有已知的捷徑」這幾個字，本章末尾的陷阱段會回來叩它。

那「難分解」怎麼變成「能加密又能解密」？這就需要 Part III 的工具登場了。

## RSA 的五個步驟：金鑰怎麼生、訊息怎麼鎖

RSA 全部就五步。我先把骨架列出來，再逐步用玩具金鑰填數字、講每一步在幹嘛、為什麼非這樣不可。

```text
【金鑰生成】（只有你做、用到祕密 p、q）
  1. 選兩個質數 p、q，算 n = p·q              ← n 公開；p、q 保密（後門）
  2. 算 φ(n) = (p−1)(q−1)                       ← 用 ch11 的 φ(pq)=(p−1)(q−1)
  3. 選一個 e，要求 gcd(e, φ(n)) = 1            ← e 公開（加密指數）
  4. 算 d，使 e·d ≡ 1 (mod φ(n))               ← d 保密（解密指數）；ch09 貝祖求逆元

  公鑰 = (e, n)   ← 貼出去
  私鑰 =  d       ← 鎖起來（n 公開、d 保密）

【加密】（任何人拿公鑰做）
  5a. 訊息 m（一個 < n 的數）→ 密文 c = m^e mod n

【解密】（只有你拿私鑰做）
  5b. 密文 c → 還原 m = c^d mod n
```

五步裡每一步都該問「為什麼」，我們一條一條過。但先把一件最容易記錯的事釘死，因為它是這套東西最常被誤背的地方：

**第 3 步的合法條件是 `gcd(e, φ(n)) = 1`，不是 `gcd(e, n) = 1`。** e 要跟 **φ(n)** 互質，不是跟 n 互質。理由在第 4 步——你要能算出 d 使 `e·d ≡ 1 (mod φ(n))`，也就是 e 在模 φ(n) 底下要有乘法逆元（這個 d 就是 e 的逆元）；而 ch09 講過，一個數在 mod M 底下有逆元，**充要條件就是它跟 M 互質**。這裡的 M 是 φ(n)，所以條件看的是 e 跟 φ(n) 的 gcd。把它記成「跟 n 互質」是個常見的滑手，會在某些 e 上給你錯誤的判斷。

## 一步步填數字：全書統一的玩具金鑰

現在用全書統一的玩具金鑰把五步走一遍。這組數字（p=3, q=11）小到你能手算每一步、大到能完整展示機制。**注意它只是教學玩具——真實 RSA 的 p、q 是上百位數的質數；這裡的 n=33 小到你一眼就能分解，所以毫無安全性可言，它的價值純粹是讓你看清流程。**

```text
【步驟 1】選 p=3, q=11，算 n = 3·11 = 33
         n = 33 公開；p=3、q=11 是後門（真實情境下保密）

【步驟 2】φ(n) = (p−1)(q−1) = (3−1)(11−1) = 2·10 = 20
         （這正是 ch11 的 φ(pq)=(p−1)(q−1)；也可寫 φ(33)=φ(3)·φ(11)=2·10=20）

【步驟 3】選 e 使 gcd(e, φ) = gcd(e, 20) = 1
         取 e = 3：gcd(3, 20) = 1 ✓（3 與 20 無公因數）→ e=3 合法

【步驟 4】算 d 使 e·d ≡ 1 (mod 20)，即 3·d ≡ 1 (mod 20)
         試 d=7：3·7 = 21 = 20 + 1 ≡ 1 (mod 20) ✓  → d = 7

  公鑰 = (e, n) = (3, 33)
  私鑰 =  d     = 7
```

關於步驟 3 有個小地方值得點破，免得你被一個數值巧合絆住：**這裡 e=3，恰好跟質因數 p=3 數字相同——這純粹是巧合，不影響合法性**。合法性看的是 `gcd(e, φ)=gcd(3, 20)=1`（成立），條件根本沒去看 e 跟 p 或 n 的關係。你不會因為 e 跟某個質因數撞了數字就出問題；判斷標準從頭到尾只有「e 跟 φ(n) 互質」這一條。（這組 p=3, q=11, e=3 是全書統一基準，動它會牽連 ch11 與附錄 A，所以我們就帶著這個小巧合走，但記得它只是巧合。）

步驟 4 我這裡是用「試」的，因為數字小。真實情境 φ 是幾百位數、不能用試的，要用 ch09 的**反向輾轉相除（擴展歐幾里得演算法）**從 `gcd(e, φ)=1` 反推出貝祖係數，那個係數就是 d。這就是為什麼 ch09 的貝祖等式是 RSA 的引擎——它保證 d 一定存在（因為 gcd=1），而且給出構造 d 的方法。**金鑰生成的全部祕密就濃縮在這裡：算 d 需要 φ(n)，而算 φ(n)=(p−1)(q−1) 需要 p 和 q。沒有 p、q 就沒有 φ，沒有 φ 就沒有 d。** 後門的位置，精確地說，就在這條依賴鏈上。

## 加密與解密：完整手算一遍

金鑰備好，來鎖一則訊息。訊息得先編碼成一個小於 n 的數 m（真實 RSA 把位元組串塞進大數，玩具裡我們直接拿小數字當訊息）。取 m=2。

```text
【加密】c = m^e mod n = 2^3 mod 33
         2^3 = 8，  8 < 33，所以 c = 8
         （任何人拿公鑰 (3, 33) 都能做這步：把 2 鎖成 8）

【解密】m = c^d mod n = 8^7 mod 33
         要算 8^7 mod 33——不要真的去算 8^7 = 2097152 再除！
         用逐步取模（每一步都先 mod 33 把數字壓小）：

   8^1 ≡ 8                       (mod 33)
   8^2 = 64 = 33 + 31 ≡ 31       (mod 33)      ← 31 ≡ −2，下面用 −2 更省
   8^3 = 8^2 · 8 ≡ 31·8 = 248
         248 = 7·33 + 17 = 231 + 17 ≡ 17       (mod 33)
   8^6 = (8^3)^2 ≡ 17^2 = 289
         289 = 8·33 + 25 = 264 + 25 ≡ 25       (mod 33)
   8^7 = 8^6 · 8 ≡ 25·8 = 200
         200 = 6·33 + 2 = 198 + 2 ≡ 2          (mod 33)

         → m = 2   ✓  解回原訊息！
```

自我複核（用 31≡−2 那條捷徑重算一次，交叉驗證）：8²≡−2，所以 8^6=(8²)³≡(−2)³=−8≡25 (mod 33)（−8+33=25，對上了）；8^7≡25·8=200≡2。兩條路殊途同歸，放心。

停下來看這件事剛剛發生了什麼：訊息 2 被公鑰鎖成了 8，然後私鑰把 8 又解回了 2。**加密用 e=3、解密用 d=7，兩個指數合起來（m^e 再 ^d = m^(ed)）把訊息原封不動送回來。** 這不是運氣，是歐拉定理逼出來的——下一節講為什麼。

但先享受一下這個畫面的反直覺：公鑰 (3, 33) 是公開的，任何人都能把 2 加密成 8。可是反過來，**從密文 8 還原訊息，你需要 d=7；而要算出 d=7，你需要 φ=20；要算 φ，你需要知道 33=3·11**。攻擊者手上只有公開的 33 和 3。對 33 這種小數，他一眼分解出 3·11，這玩具當然一秒就破。但把 33 換成一個 617 位的數，「一眼分解」就變成 2700 核心年起跳的工程——**鎖就是這樣鎖上的：不是把門焊死，是把開門需要的那把鑰匙，藏在一個沒人算得動的分解問題後面**。

## 為什麼解得回去：歐拉定理的後門

加密把 m 變成 m^e，解密把它變成 (m^e)^d = m^(e·d)。要讓訊息原封返回，我們需要：

```text
        m^(e·d) ≡ m   (mod n)        ← 對所有訊息 m 都要成立
```

這個 `≡ m` 不是湊巧，是被金鑰生成時「e·d ≡ 1 (mod φ(n))」這個條件保證的。把這條展開：e·d ≡ 1 (mod φ(n)) 意思是 e·d = 1 + k·φ(n) 對某個整數 k 成立。代進去：

```text
   m^(e·d) = m^(1 + k·φ(n))
           = m · m^(k·φ(n))
           = m · ( m^φ(n) )^k                    ← 把指數 k·φ(n) 拆成 (φ(n))·k

   而歐拉定理（ch11）說：若 gcd(m, n)=1，則 m^φ(n) ≡ 1 (mod n)

   所以   m^(e·d) ≡ m · (1)^k = m · 1 = m        (mod n)   ✓
```

**這就是 RSA 能解密的全部數學。** 解密不是某種反運算、不是「把加密倒過來跑」，而是再做一次模冪、靠歐拉定理讓多出來的那一大坨 m^(k·φ(n)) 整個塌縮成 1，於是只剩下原本的 m。歐拉定理那條「a^φ(n)≡1」的鐵律（ch11 用脊椎分解 60 養出來的那條），在這裡是讓鎖能被正確鑰匙打開的物理定律。

驗一下玩具：n=33，φ=20，e·d=3·7=21=1+1·20，所以 k=1。`m^21 = m·(m^20)^1 ≡ m·1 = m`——這正是上一節 8^7 算出 2 的內幕，e·d=21 而 m^20≡1，所以 m^21≡m。

有一個你該注意到的細節：歐拉定理要求 `gcd(m, n)=1`，但訊息 m 萬一剛好跟 n 有公因數（比如 m=3、跟 n=33 共用因數 3）怎麼辦？這裡有個漂亮的補救：對 RSA 的 n=pq（兩個相異質數的積），可以證明 `m^(ed)≡m (mod n)` 對**所有** m 都成立、連不互質的 m 也成立（要動用中國剩餘定理分別在 mod p 和 mod q 上論證，本書不展開——把它記成「RSA 的解密對全部訊息都對，不只互質的那些」即可，嚴格證明指向延伸閱讀）。玩具裡你可以自己驗 m=3：c=3³=27 mod 33，再 27^7 mod 33 會乖乖回到 3。

## 直覺的陷阱

RSA 是科普裡最容易被講半對的東西。下面幾條，每一條我都見過聰明人講錯。

```text
陷阱清單（左：常見錯版／右：正確版）

─────────────────────────────────────────────────────────────
① e 的合法條件
   錯：e 要跟 n 互質（gcd(e, n)=1）
   對：e 要跟 φ(n) 互質（gcd(e, φ(n))=1）——因為 d 是 e 在
       mod φ(n) 下的逆元，逆元存在的條件看的是 φ(n) 不是 n
─────────────────────────────────────────────────────────────
② RSA 的安全性是「被證明」的嗎？           ← 本章最重要的一條
   錯：數學證明了大數分解不可能 / 永遠很難，所以 RSA 安全
   對：分解「目前」沒有已知的快速古典演算法，RSA 安全建立在
       這個【經驗難度】上，不是已被證明的難。P 對 NP 仍未解；
       沒人證明過分解一定慢。它只是「至今沒人找到捷徑」
─────────────────────────────────────────────────────────────
③ 量子電腦現在能破 RSA 了嗎？             ←（2026-06，會更新）
   錯：量子電腦已經能破 RSA / 馬上就破了
   對：Shor 演算法【理論上】能在量子電腦上多項式時間分解大數，
       但 2026-06 為止【還沒有】能分解實用規模（2048-bit）RSA 的
       量子電腦存在。2026-03 有團隊把「破 RSA 需要多少量子位元」
       的資源估計往下修（估到較少的物理位元），但那是【估計門檻】、
       不是【已經做到】——這條會隨硬體進展更新
─────────────────────────────────────────────────────────────
④ 公鑰能加密，所以公鑰也能解密？
   錯：公鑰 (e,n) 公開，拿它就能解
   對：公鑰只能正向 m→c；解密要 d，d 藏在 φ(n) 後面、
       φ(n) 藏在 p、q 後面。公開的只有 n、e，反推 d 要先分解 n
─────────────────────────────────────────────────────────────
⑤ 「容易乘、難分解」是 RSA 唯一在用的事？
   對一半：難分解是【必要】的（分解了就破），但 RSA 真正
   依賴的是更強的假設——「不分解 n 也算不出 e 次方根」（RSA 問題）。
   分解難是它的下界保證；本書只一瞥，記「至少要難分解」即可
─────────────────────────────────────────────────────────────
```

第②條值得單獨講，因為它和本書的脊椎張力扣在一起。RSA 的安全是一種**經驗難度**——「至今沒人找到快速分解法」——而不是一條定理。這和 ch14 的主旋律是同一件事的兩面：**「驗證了很久都沒出事」不等於「被證明永遠不會出事」。** 哥德巴赫猜想驗證到 4×10^18 沒反例，但沒被證明；RSA 被攻了快五十年沒被快速分解攻破，但也沒被證明不可破。差別在於，數學家對哥德巴赫的態度是「很可能對、等一個證明」，而對 RSA，全世界的密碼系統其實是**賭**「分解很難」這件事至今為真、且短期內不會被推翻——這個賭注夠大，大到值得我們同時去發展不依賴分解難度的後量子密碼（post-quantum cryptography）。怎麼自我察覺你掉進這個陷阱？徵兆是你會說出「RSA 在數學上是安全的」——只要你用了「數學上證明安全」這種措辭，你就把一個經驗賭注誤當成了定理。正確的講法永遠帶著時效與條件：「以目前已知的演算法、在古典電腦上，分解夠大的 n 算力不可行」。

## 紙上推演

### 推演題

**題 1（★★，[20 分鐘]）** 用全書玩具金鑰 p=3, q=11, n=33, φ=20, e=3, d=7，把訊息 **m=5** 完整加密再解密一遍。加密算 c=5^e mod 33，解密算 c^d mod 33，每一步逐步取模、不要算出大數，最後確認解回 5。算完後，用「負數同餘」捷徑（把 26 寫成 −7）重算一次解密，交叉驗證。

**題 2（★★，[15 分鐘]）** 解釋為什麼「知道 n 卻不知道 p、q」就算不出 d。請沿著金鑰生成的依賴鏈，講清楚 d、φ、p、q 之間是誰需要誰；並說明攻擊者手上有公開的 (e, n)，唯一缺的那一塊是什麼、為什麼補不上。

**題 3（★★，[15 分鐘]）** 判斷對錯並說明理由：「RSA 之所以安全，是因為數學家證明了大數分解一定很慢。」如果這句話錯，正確版該怎麼講？順帶說一句 Shor 演算法在這個圖像裡的位置（2026-06）。

### 推演解答

**題 1 解答。**

```text
加密：c = 5^3 mod 33
   5^3 = 125
   125 = 3·33 + 26 = 99 + 26 ≡ 26   (mod 33)
   → c = 26

解密：m = 26^7 mod 33（逐步取模）
   26^1 ≡ 26                        (mod 33)
   26^2 = 676 = 20·33 + 16 = 660+16 ≡ 16   (mod 33)
   26^3 = 26^2·26 ≡ 16·26 = 416
          416 = 12·33 + 20 = 396 + 20 ≡ 20  (mod 33)
   26^6 = (26^3)^2 ≡ 20^2 = 400
          400 = 12·33 + 4 = 396 + 4 ≡ 4      (mod 33)
   26^7 = 26^6·26 ≡ 4·26 = 104
          104 = 3·33 + 5 = 99 + 5 ≡ 5         (mod 33)
   → m = 5   ✓ 解回原訊息

負數同餘捷徑（交叉驗證）：26 ≡ −7 (mod 33)
   (−7)^2 = 49 ≡ 16            (49−33=16，對上 26^2)
   (−7)^3 = (−7)·16 = −112 ≡ −112 + 4·33 = −112+132 = 20  （對上 26^3）
   (−7)^6 = ((−7)^3)^2 ≡ 20^2 = 400 ≡ 4                    （對上 26^6）
   (−7)^7 = (−7)^6·(−7) ≡ 4·(−7) = −28 ≡ −28 + 33 = 5     （對上）
   兩條路都得 5。✓
```

常見錯路：(1) 真的去算 26^7=8031810176 再除——會算到手軟還容易算錯，逐步取模的全部意義就是每步把數字壓回 0..32。(2) 加密時忘了 125 還要 mod 33、直接把 c 當成 125——但 c 必須 < n，這步漏了後面全錯。(3) 用負數捷徑時符號掉了，(−7)^7 是負的（奇次方），別寫成正 28。

**題 2 解答。** 依賴鏈（從你想要的 d 往回追）：

```text
   想算 d  ──需要──▶  φ(n)  ──需要──▶  p 和 q
   （e·d≡1 mod φ）     （φ=(p−1)(q−1)）   （n=pq 的兩個質因數）

   攻擊者有：n（公開）、e（公開）
   攻擊者缺：p、q  →  缺 φ  →  缺 d
```

要算 d，得先有 φ(n)；要算 φ(n)=(p−1)(q−1)，得先知道 p 和 q。攻擊者手上只有公開的 n 和 e。他唯一缺的那塊，就是 **n 的質因數分解**——把 n 拆成 p·q。對玩具 n=33 他一眼拆出 3·11，於是整條鏈瞬間補完、d 算得出來，玩具因此毫無安全性。但把 n 換成幾百位的數，「把 n 拆成 p·q」就是那個 2700 核心年起跳、目前無已知捷徑的分解問題——這塊補不上，鏈就斷在這，d 永遠算不出。**注意關鍵：難的不是「知道 φ 算 d」（那步用 ch09 貝祖很快），也不是「知道 p、q 算 φ」（一個乘法），難的是整條鏈最上游的「從 n 找回 p、q」。** 後門的鎖眼就在那。

（嚴格說，攻擊者不分解 n、直接想辦法算 e 次方根，理論上也許有別的路——這叫 RSA 問題，是否真的等價於分解仍是開放問題；但「能分解就一定能破」是確定的，所以「難分解」是安全的必要條件，本書記到這層即可。）

**題 3 解答。** 這句話**錯**。沒有人證明過大數分解一定很慢——這正是 landscape 守住的 T9 與本章陷阱②。正確版：RSA 的安全建立在**經驗難度**上——「以 2026-06 已知的古典演算法，分解夠大的 n 在可行算力內做不到」——這是一個至今未被攻破、但**未被證明永不可破**的假設。事實上「分解到底有多難」連個證明下界都沒有；P 對 NP 還沒解，沒人能排除哪天冒出一個快速古典分解法。

Shor 演算法的位置：它是一個**量子**演算法，理論上能在量子電腦上以多項式時間分解大數——也就是說，如果有一台夠大的量子電腦，RSA 在它面前理論上會倒。但 2026-06 為止，還沒有能分解實用規模（2048-bit）RSA 的量子電腦存在；2026-03 有研究把「需要多少量子位元才破得了」的資源估計往下修，但那是把門檻估得更低、不是已經跨過門檻。所以準確的一句話是：「RSA 對古典電腦是經驗上安全的、對足夠大的量子電腦理論上不安全，而那種量子電腦 2026-06 尚不存在——這條敘述會隨硬體進展更新。」常見錯路：把「Shor 理論上能破」說成「量子電腦已經破了 RSA」，把一個能力證明（演算法存在）誤當成一個既成事實（機器已造出）。

## 延伸閱讀

- **Rivest, Shamir, Adleman, "A Method for Obtaining Digital Signatures and Public-Key Cryptosystems"（1977 投稿、1978 刊於 *Communications of the ACM*）**：RSA 的原始論文。短、可讀、就是本章玩具的大人版——看他們怎麼把「分解難」變成可用的密碼系統。注意發表年份史料上有 1977／1978 兩說（工作與公開描述在 1977、正式刊出 1978），本書依基準寫「1977 公開描述」。
- **Clifford Cocks, "A Note on Non-Secret Encryption"（GCHQ 內部報告 1973，1997 解密公開）**：比 RSA 早四年、被英國情報機關鎖在抽屜裡二十四年的等價系統。讀「公鑰密碼其實被發明過兩次、第一次沒人能說」這段史，看 The Aperiodical 對 GCHQ 解密文件的整理（aperiodical.com 2016-03）。
- **Diffie & Hellman, "New Directions in Cryptography"（1976）**：提出「陷門函數應該存在」的那篇——RSA 是它預言的東西第一個被造出來的實例。想知道「公鑰密碼」這個想法本身從哪來，讀這篇的前幾頁。
- **RSA Factoring Challenge / Integer Factoring Records（Paul Zimmermann 維護的分解紀錄頁，members.loria.fr/PZimmermann/records/factor.html）**：想對「分解到底多難」有具體重量感，看 RSA-250（829-bit，2020-02，約 2700 核心年）這類紀錄——古典電腦分解的公開最大者（2026-06）。
- **歐拉定理保證解密可逆**：本章「為什麼解得回去」整段的數學引擎在 ch11——若你對 m^(ed)≡m 那步還不踏實，回去把 a^φ(n)≡1 重看一遍，那是 RSA 站著的地基。
- 關於「驗證 ≠ 證明」、為什麼「沒被攻破」不等於「安全被證明」，與本章陷阱②同源——見本書談「驗證不是證明」那章（ch14）。
