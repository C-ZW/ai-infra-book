# ch13 — 黎曼假設一瞥：質數的音樂

> **本章解決什麼問題**：ch07 的 PNT 給了質數的平均行為——π(x) 大約是 x/ln x、更準是 Li(x)。但「大約」底下藏著一個誤差：Li(x) 不等於 π(x)，兩者有差。這個誤差不是亂跳的雜訊，而是有結構的——它是一串波疊起來的。黎曼 1859 年那篇只有幾頁的論文，給了一條公式，把這串波的每一個頻率，連到一個叫 ζ 函數的東西的「零點」。黎曼假設說：這些零點全都排在一條直線上。本章不證它、不碰複變分析（那是《馴服無限》的地盤），只講一件事——為什麼「零點排在一條線上」這句話，等價於「質數分布盡可能規律、誤差盡可能小」。這是本書的第二個高潮，也是脊椎 B「質數長在哪裡」最深的一層：我們連誤差的形狀都看見了，卻還沒能證明它真的那麼小。

Part IV 從 ch12 的孿生質數開始，盯著質數分布的細節。ch12 看的是「相鄰質數間的間隙」這個離散的細節；這一章換一個角度，看「π(x) 整體偏離 Li(x) 多少」這個連續的誤差。兩者其實是同一個謎的兩面——都是 PNT 那條平均曲線管不到的東西。而這一章要講的，是人類對「誤差」這一面的理解，竟然深到能寫出它的精確形狀，只差證明它真的成立。

我先把話說在前頭：這是全書我寫得最忐忑的一章。黎曼假設（Riemann hypothesis, RH）的完整內容要動用複變函數、解析延拓、ζ 函數的零點——那些本書一律不展開（指向《馴服無限》）。但 RH 的**精神**——它到底在斷言什麼、為什麼數論學家認為它是「整個數學裡最重要的未解問題」——是可以用工程師的語言講清楚的，而且講清楚之後你會發現它美得不可思議。我會誠實地把「直覺」和「我故意略過的嚴格機制」分開標示。這一章的目標不是讓你會算 ζ，而是讓你能對另一個工程師講清楚：**質數的『亂』，可能只是一串我們還沒證明會乖乖排好的波。**

## 從你已知的出發

你做過訊號處理，至少在概念上碰過傅立葉（Fourier）。一段看起來毫無規律的波形——音訊、振動、感測器讀數——你把它丟進 FFT，它就拆成一堆正弦波：每個頻率一個振幅。原本「亂」的時域訊號，在頻域裡變成幾根清楚的峰。**「亂」往往只是「一堆規律的波疊在一起」的另一種長相。** 你沒看出規律，是因為你在錯的座標系看它。

這正是黎曼對質數做的事，而且這個類比準到驚人。

把鏡頭拉回 ch07。PNT 說 π(x) ~ x/ln x，更準的逼近是 Li(x)（對數積分，定義與積分細節見 ch07）。我們在 ch07 也誠實標過：Li(x) 不等於 π(x)，它系統性地稍微高估了一點點。那個「一點點」的差——Li(x) − π(x)——就是質數分布的「誤差訊號」。

```text
你熟的傅立葉直覺               黎曼對質數做的事
----------------------------   ----------------------------------
一段「亂」的時域訊號         →  π(x)：質數計數，個別位置看似亂
拆成一堆正弦波（FFT）       →  把 π(x) 拆成「主項 + 一堆波」
每個頻率一個振幅             →  每個 ζ 零點貢獻一個波（一個頻率）
主成分（DC / 低頻）         →  主項 Li(x)（平滑、好預測）
殘差（高頻細節）            →  零點帶來的振盪（修正主項到真正的 π(x)）
「亂只是疊在一起的規律」     →  質數的「亂」＝一串波疊起來的結果
```

黎曼 1859 年發現：**π(x) 可以寫成一個平滑的主項（本質上是 Li(x)），減去一串波，而每一個波，對應 ζ 函數的一個「非顯然零點」。** 每個零點就是一個頻率。把所有零點的波疊起來，主項上的小起伏就被修正成真正的、一階一階跳的 π(x)。

這就是「質數的音樂」（the music of the primes）這個說法的來源——數學家 Marcus du Sautoy 2003 年用它當書名而廣為流傳（查證值，2026-06）。質數的分布，是一段樂曲；ζ 的每個零點，是一個音符（一個頻率、一個諧波）；黎曼假設，則是斷言這些音符全都調在同一個音高上。

別擔心你還不知道 ζ 是什麼、零點是什麼——這一章會用最少的機制把這幅圖像搭起來。你只要先接受一件事：**質數的「亂」，可能只是傅立葉意義下「一堆波疊在一起」，而我們連那些波的頻率藏在哪都知道了。**

## 先看誤差：Li(x) 到底差 π(x) 多少

要懂黎曼在修正什麼，得先把「誤差」這個東西看清楚。回到 ch07 的對照表，把實際 π(x) 和逼近 Li(x) 並排：

```text
x          π(x)（實際）   Li(x)（逼近，四捨五入）   誤差 Li(x)−π(x)
--------   ------------   ----------------------   ---------------
10²        25             30                        +5
10³        168            178                        +10
10⁴        1229           1246                       +17
10⁶        78498          78628                       +130
10⁸        5761455        5762209                      +754
```

（這些 Li(x) 值是查證過的標準對照值；π(x) 是本書跨章基準。本章只需看「誤差」那一欄的趨勢，不需你手算 Li 的積分。）

幾件事值得停下來看：

- **誤差一直是正的**——Li(x) 在這個範圍內始終略高於 π(x)。這是 ch07 講過的「系統性高估」。（這個「一直為正」其實有個著名的陷阱，留到 ch14 講 Skewes 數時引爆——劇透：它**不會**永遠為正。）
- **誤差在長大**，但長得**很慢**：x 從 10² 漲到 10⁸（漲了一百萬倍），誤差才從 5 爬到 754。相對於 π(x) 本身（從 25 漲到 576 萬），這個誤差小得可笑。
- 關鍵問題來了：**這個誤差到底有多大？它的成長有沒有上界？** 這正是黎曼假設要回答的核心問題。

用工程師的話：PNT 給了你主項（Li(x)），現在你要問的是「殘差有多大、會不會失控」。如果你做容量規劃，你不會只關心平均負載，你會死死盯著「最壞情況下，實際值偏離預測值多少」。RH 就是在問質數版的這個問題——而且它給的答案漂亮到不真實。

## 黎曼的顯式公式：把誤差拆成波

1859 年，黎曼發表了他唯一的一篇數論論文〈論小於給定數值之質數個數〉（*Über die Anzahl der Primzahlen unter einer gegebenen Größe*，查證值），約六到九頁（landscape 基準）。在這篇論文裡，他做了一件當時沒人想得到的事：他寫出了一條**精確的公式**，把 π(x) 表達成一串東西的和——這就是後人稱的**顯式公式（explicit formula）**。

我不會寫出它的完整數學形式（那要 ζ 函數和複數，超出本書邊界）。但它的**骨架**可以用白話說清，而骨架就是全部的直覺所在：

```text
顯式公式的骨架（直覺版，故意略去複變分析的精確形式）：

   π(x) ≈ Li(x)  −  Σ（每個 ζ 零點 ρ 貢獻的一個振盪項）  −  小修正

          └─主項─┘    └──────────一串波──────────┘

   · 主項 Li(x)：平滑、單調、好預測——就是 PNT 給你的那條曲線。
   · 那一串波：每個 ζ 的「非顯然零點」ρ 貢獻一個波。
              這些波疊起來，把平滑的主項「啃」成真正一階一階跳的 π(x)。
   · 每個波是一個振盪（像正弦波），有自己的頻率和振幅。
```

這就是傅立葉類比的精確落點：**Li(x) 是 DC / 低頻主成分，那一串零點的波是高頻細節，兩者相加（嚴格說是相減）才還原出真正的 π(x)。**

現在你需要知道「非顯然零點」是什麼——只要知道到能往下走的程度就好。ζ 函數（Riemann zeta function）是一個吃複數、吐複數的函數。它在某些點上「等於零」。其中一批零點落在負偶數（−2, −4, −6, …），叫**顯然零點（trivial zeros）**，它們無聊、好懂、跟質數的關係不大。另一批零點落在一個叫「臨界帶」的區域裡，叫**非顯然零點（non-trivial zeros）**——這批才是有趣的，它們就是顯式公式裡那一串波的來源。每一個非顯然零點 ρ 是一個複數，它的位置決定了對應那個波的**頻率**（波跳多快）和**衰減**（波多快變小）。

我必須在這裡誠實標一條線：**「ζ 零點如何精確地對應一個振盪項」，需要複變分析，本書不展開（指向《馴服無限》談解析延拓與頻譜那條線）。** 我給你的是這幅圖像的形狀，不是它的嚴格機制。但形狀已經足夠讓下一步的驚嘆成立。

## 黎曼假設：所有零點排在同一條線上

現在主角登場。每個非顯然零點 ρ 是一個複數，可以寫成 ρ = σ + it 的形式——σ 是它的「實部」、t 是「虛部」。把所有非顯然零點畫在複平面上，它們散在臨界帶（0 < σ < 1 這條垂直帶子）裡的某些位置。

黎曼在 1859 那篇論文裡，**順帶**提了一句猜想（landscape 守住：他主要在算 π(x) 的顯式公式，RH 是順帶一提的）：

```text
黎曼假設（Riemann Hypothesis, RH）：
   ζ 函數的所有「非顯然零點」，實部 σ 都恰好等於 1/2。

   也就是：所有非顯然零點，都排在複平面上「實部 = 1/2」這條
   垂直線上——這條線叫「臨界線（critical line）」。

   狀態（2026-06）：仍未證明。它是七個千禧年大獎問題
   （Clay Millennium Prize Problems）之一，懸賞 100 萬美元，
   源於 Riemann 1859，亦列於 Hilbert 1900 年的 23 個問題。
```

光看這句話，你完全看不出它跟質數有什麼關係——它講的是某個複變函數的零點長在哪。這正是 RH 最迷人也最讓人抓狂的地方：**一句關於 ζ 零點位置的純技術陳述，竟然死死地控制著質數的分布。** 橋接這兩者的，就是上一節那條顯式公式。

要看出關聯，回到「每個零點貢獻一個波」這件事。一個零點 ρ = σ + it 貢獻的那個波，**振幅大約正比於 x^σ**——也就是說，零點的實部 σ 越大，那個波的振幅就**越大**，對主項的「啃咬」就越兇、誤差就越大。

```text
零點實部 σ 如何控制誤差大小（直覺版）：

   某個零點 ρ = σ + it 貢獻的波，振幅 ~ x^σ

   · 若某個 σ = 0.9（靠近臨界帶右邊）→ 波振幅 ~ x^0.9，很大 → 誤差大
   · 若某個 σ = 0.5（臨界線上）       → 波振幅 ~ x^0.5 = √x，較小 → 誤差小
   · 所有 σ 都 = 0.5（RH 為真）        → 所有波都被壓到 √x 這個最小量級
                                          → 誤差被壓到「盡可能小」
```

這就是 RH 的真正意義，翻成質數的語言：

> **黎曼假設等價於：π(x) 與 Li(x) 的誤差，盡可能地小（大約是 √x 乘上一個 ln 因子的量級）。**

這個 √x ln x 的誤差界，是查證過的：RH 等價於 |π(x) − Li(x)| 被 √x·ln x 量級的東西控制住（更精確的形式要 ψ(x)、ϑ(x) 等切比雪夫函數，本書不展開，查證值 2026-06）。而且這是**已知可能的最小誤差**——你不可能要求質數的計數誤差比這更小了。

所以把整句話收攏成一個工程師能轉述的版本：**質數看起來「亂」，是因為主項 Li(x) 上疊了一串波。黎曼假設說，這些波的振幅全都一樣小（被壓在 √x 這個地板上）——換句話說，質數分布的『亂』，是『盡可能規律』的那種亂。沒有哪一個諧波特別大聲、特別搶戲；所有頻率都調在同一個音量上。** 這就是為什麼有人說 RH 為真的話，質數會是「盡可能均勻地隨機」——它的不規律恰好小到不能再小。

## 為什麼這是「音樂」：所有音符同一個音量

「質數的音樂」這個隱喻，現在可以講到底了。

把顯式公式裡每個零點貢獻的波，想成一個音符。一個音符有兩個屬性：**音高（頻率，由零點的虛部 t 決定，t 越大頻率越高）** 和 **音量（振幅，由實部 σ 決定，σ 越大越響）**。

```text
質數的音樂（隱喻，對齊顯式公式）：

   零點 ρ = σ + it ──┬── 虛部 t → 這個音符的「音高」（頻率）
                     └── 實部 σ → 這個音符的「音量」（振幅 ~ x^σ）

   把所有零點的音符疊起來演奏 → 主項 Li(x) 上的起伏 → 真正的 π(x)

   黎曼假設說「所有 σ = 1/2」，翻成音樂：
      → 所有音符的音量「完全相同」（都被壓在 √x 這個地板上）
      → 沒有任何一個諧波蓋過其他、沒有刺耳的尖峰
      → 質數的樂曲是「完美平衡」的——這就是「盡可能規律」
```

如果 RH 是**假的**——也就是說，有某個零點的實部 σ > 1/2——那會發生什麼？那個零點貢獻的波，振幅就是 x^σ > √x，比其他音符**響**。在質數的樂曲裡，會冒出一個特別大聲的諧波，把誤差頂上去，質數分布就會出現一個比 RH 預期「更不規律」的偏離。所以 RH 不只是個技術猜想，它是在說：**質數的樂曲沒有走音的音符；每個頻率都恰到好處地小聲。**

這是我認為本書最美的兩頁之一（另一頁是 ch03 歐幾里得那五行）。歐幾里得的美是「兩千年前一個五行論證證出質數無窮」那種樸素的、徒手的美。黎曼這個美是另一種——**它把「質數為什麼看起來亂」這個問題，翻譯成「一段樂曲的所有音符是不是同一個音量」這個問題，然後說：我猜是。** 一百六十多年了，沒人能證明這段樂曲真的這麼和諧，但也沒人找到半個走音的音符。

## 數值驗證：找了十兆個音符都沒走音（但還是不算證明）

人類沒能證明 RH，但用電腦狂算了它。演算法很直接（概念上）：一個一個把 ζ 的非顯然零點找出來，檢查它的實部是不是 1/2。

```text
RH 的數值驗證（2026-06，這是進展型事實，會更新）：
   · 已計算的非顯然零點，數量達到「兆」的量級
     （約十兆個量級的零點被檢查，全部落在臨界線上，查證值）。
   · 較嚴謹的成果：Platt 與 Trudgian 2021 嚴格驗證了臨界帶
     某個高度以下的全部零點（達兆量級）都在臨界線上、且是單零點。
   · 截至 2026-06：找到的每一個非顯然零點，實部都是 1/2，
     沒有任何一個例外。
```

十兆個零點，沒有一個走音。如果這是工程，你大概已經把這條規則寫進程式碼、加個註解「實務上永遠成立」然後去喝咖啡了。但這是數學，而**這正是 RH 最殘酷的教訓，也是它直通 ch14 的地方**：

**驗證了十兆個零點，不能證明第十兆零一個零點不會走音。**

數學的真，是「對所有的」——所有非顯然零點，無一例外。十兆個零點相對於「無窮多個零點」，就像你測了十兆個輸入相對於「所有可能的輸入」：在無窮面前，十兆等於零。只要有一個零點的實部不是 1/2，RH 就垮了，質數的樂曲就有一個走音的音符。而「沒人找到反例」跟「不存在反例」之間，隔著的就是「證明」這道線。

這個「驗證 ≠ 證明」的主題，是 ch14 的全部內容，那裡會用 Skewes 數給你一個「前面全對、後面才崩」的真實案例（劇透：本章那欄「誤差一直是正的」會在那裡被推翻）。所以這一章你只要先把這顆種子種下：**RH 的十兆個零點，是「驗證的鐵證」，不是「證明的鐵證」——這兩件事在數學裡天差地別。**

## 直覺的陷阱

黎曼假設是科普重災區，幾乎每一句通俗轉述都踩過下面某個坑。守住正確版。

| 常見誤解 | 哪一步出錯 | 正確版（怎麼自我察覺） |
|---|---|---|
| 「黎曼假設給了算第 n 個質數的公式」 | 把「顯式公式控制誤差」誤讀成「能精確算出每個質數位置」 | 顯式公式是把 π(x) 拆成主項＋波，描述的是**整體計數**的誤差結構，不是「代入 n 吐出第 n 個質數」。徵兆：你以為 RH 一旦證出來就能預測下一個質數——不能，它管的是誤差的**界**，不是個別位置 |
| 「已經算了十兆個零點都在線上，等於證明了」 | 把數值驗證當證明 | 十兆相對於無窮是零；只要有一個零點走音 RH 就垮（接 ch14，T7）。徵兆：你說得出「數學的真是對**所有**、不是對**已測的**」嗎 |
| 「黎曼假設說質數是完全隨機的」 | 把「盡可能規律」誤讀成「完全隨機 = 無規律」 | RH 恰恰相反：它說誤差**盡可能小**、質數**盡可能規律**。每個諧波同一音量＝沒有失控的偏離。徵兆：你把「亂」和「無規律」當同義詞了——質數的亂是「規律到極致」的亂（T8 的精神） |
| 「網路上有人證出 RH 了」 | 把未經同行審查的宣稱當成果 | RH 仍未解（2026-06）。網路上常有「AI 證出 RH」「新路徑」之類宣稱，**一律當未證**——須同行審查與社群共識，2026-06 無此共識。徵兆：消息來源是不是 Clay 研究所或頂級期刊的正式確認 |
| 「ζ 零點跟質數有什麼關係？這只是巧合」 | 沒看出顯式公式是橋 | 顯式公式**精確地**把每個零點連到質數計數誤差的一個波；零點實部直接控制誤差大小。這不是巧合，是 1859 年就寫下的恆等式。徵兆：你能說出「零點 → 一個波 → 修正 Li(x) 到 π(x)」這條鏈嗎 |

最深的那個陷阱是第三個：把「亂」當「無規律」。這是整本書反覆要拆的那把鎖。質數**看起來**亂（個別位置沒有生成公式），但 RH 斷言這個亂是「盡可能規律」的——誤差被壓到理論下限。**「亂」在這裡的意思是「像最均勻的隨機那樣不可預測」，而不是「沒有任何結構」。** 一段所有音符同音量的樂曲，聽起來可能像白噪音，但它的頻譜其實完美平衡——這正是質數的亂與律共存的最精緻版本（本書反覆叩問的那句話）。

## 紙上推演

### 推演題 1（★★，[15 分鐘]）

不查書，用自己的話向一個沒讀過這章的工程師講清楚：「為什麼黎曼假設是『質數盡可能規律』的陳述？」要求說到三件事——(a) π(x) 可以拆成「主項 + 一串波」；(b) 每個波的振幅由對應零點的實部控制（實部越大、波越響、誤差越大）；(c) 「所有零點實部都是 1/2」為什麼就讓誤差被壓到最小。最後用傅立葉的話收一句。

### 推演題 2（★★★，[20 分鐘]）

接 ch14 的引線。已經有人用電腦驗證了約十兆個 ζ 非顯然零點，全部落在臨界線上、無一例外。解釋：為什麼這**還不算**證明了黎曼假設？把「數學的真」和「已驗證」的差別講清楚，並舉一個你工作中「測了大量 case 都過、但仍不能保證程式對所有輸入都對」的例子來類比。

### 推演題 3（★★，[15 分鐘]）

承上節誤差表：x=10² 時誤差 +5、x=10⁸ 時誤差 +754。誤差在長大。一個粗糙的問題：如果 RH 為真、誤差量級大約是 √x（先忽略 ln 因子），那麼從 x=10² 到 x=10⁸，誤差「應該」大約放大幾倍？把這個粗估和表上實際的放大倍數（5 → 754）對照，討論：吻合嗎？不完全吻合代表什麼？

### 推演解答

**第 1 題。** 一個合格的版本大致是：

「質數計數函數 π(x)——也就是『不超過 x 的質數有幾個』——可以寫成一個平滑的主項，加上一串波。主項就是 PNT 給的 Li(x)，它光滑、好預測。那一串波，每一個對應 ζ 函數的一個『零點』，把平滑的主項修正成真正一階一階跳的 π(x)。**(a)** 所以 π(x) = 主項 + 一串波。**(b)** 每個零點是個複數，它的『實部』決定對應那個波有多大聲——實部越大，波的振幅越大（大約是 x 的實部次方），對主項的擾動就越兇、質數分布的誤差就越大。**(c)** 黎曼假設說所有零點的實部都恰好是 1/2，這就讓每個波的振幅都被壓到 √x 這個量級——這是可能的最小值。所有波都一樣小、沒有哪個特別響，誤差就被壓到理論下限。**收句（傅立葉）：** 質數的『亂』其實是一串波疊出來的，黎曼假設說這些波的振幅全都一樣小——所以質數的亂，是『盡可能規律』的那種亂。」

常見錯路：說成「RH 證出來就能算下一個質數」——錯，顯式公式管的是**整體計數的誤差**，不是個別質數位置（陷阱一）。另一個錯路：把「實部 1/2」說成某種隨機性的來源——它恰恰是讓誤差**最小**、最規律的條件。

**第 2 題。** 數學的真是「對所有的」：黎曼假設斷言**所有**非顯然零點（無窮多個）的實部都是 1/2，一個例外都不能有。電腦驗證了約十兆個——但十兆相對於無窮多個，比例是零。只要第十兆零一個零點的實部不是 1/2，RH 就被推翻。「沒找到反例」和「不存在反例」之間隔著「證明」這道線：證明是演繹的、覆蓋所有情形；驗證是抽樣的、只覆蓋你測過的那些。

工程類比（你的版本可以不同，要抓住「抽樣 ≠ 全稱」）：你寫了一個處理使用者輸入的函數，跑了一千萬個測試 case 全綠——這不能保證它對**所有**輸入都對，因為輸入空間是無窮的（或至少天文數字大），你測的永遠是其中一個有限子集。真正能保證「對所有輸入都對」的，是形式驗證或數學證明，不是測試覆蓋率。RH 的處境一模一樣：十兆個零點 = 十兆個綠燈，但輸入空間是「全部非顯然零點」，無窮大。常見錯路：覺得「十兆已經夠多了，實務上等於證明」——這在工程裡或許能接受（你會寫個註解去喝咖啡），但在數學裡，「對所有的」沒有「實務上」這個折扣。這正是 ch14 的全部主題。

**第 3 題。** 先做粗估。若誤差量級 ~ √x，那麼：

```text
   x 從 10² 到 10⁸，x 放大了 10⁸ / 10² = 10⁶ 倍。
   √x 放大的倍數 = √(10⁶) = 10³ = 1000 倍。

   → 若誤差純粹 ~ √x，誤差「應該」放大約 1000 倍。

   表上實際：x=10² 誤差 +5，x=10⁸ 誤差 +754。
   實際放大倍數 = 754 / 5 ≈ 150.8 倍。
```

（複核：√(10⁶)=10³=1000，無誤；754÷5＝150.8，無誤。）

吻合嗎？**不完全吻合**——粗估說放大約 1000 倍，實際只放大約 151 倍。差了大約 6.6 倍。這代表什麼？

幾個誠實的點。第一，誤差表上的數字是**這一個方向**（Li−π）在**這幾個特定 x**上的取值，不是誤差的「量級上界」——顯式公式裡那一串波會互相抵消、震盪，某些 x 上誤差恰好較小，所以拿兩個點的比值去推「量級」本來就很粗。第二，我故意忽略了 ln 因子（真正的界是 √x·ln x 量級），而且 √x 只是 RH 為真時的**上界量級**、不是「誤差就等於 √x」——多數時候誤差遠小於上界。所以「不吻合」完全不代表 RH 有問題，只代表「拿兩個資料點除一除去估漸近量級」是個很糙的手法（和 ch07、ch12 拿小 x 估漸近式同一個毛病：~ 是漸近、不是相等，T5）。常見錯路：看到 151 ≠ 1000 就說「所以誤差不是 √x 量級、RH 錯了」——這是把「量級上界」誤當「精確等式」，也是把兩個資料點的比值誤當漸近行為。這題真正的教訓是：**√x 是誤差的天花板量級，不是它的值；用少數幾個點反推漸近，永遠只能看個大概。**
