# 把系統寫成定理

> TLA+ 與形式化方法——寫給一位資深後端工程師的紙上之旅。

## 這本書是什麼

你寫了六年的高併發系統：佇列、冪等 consumer、分散式交易、WebSocket 叢集。你靠測試、code review 與直覺把它們撐在線上——而你也清楚，有些 bug 是這三樣東西天生抓不到的。這本書教另一條路：把系統寫成數學物件，把「它不會出錯」變成可以證明的定理。

三個設定，開讀前先知道：

- **純紙上推演**。全書不裝任何工具、不寫程式。TLC、TLAPS 這些工具只學原理與讀法，動手的部分是紙和筆：手畫狀態空間、手追 trace、手寫歸納證明。
- **深入到證明層**。不止於「會讀 spec」：歸納不變量、refinement、機器證明的讀法都是正餐，不是甜點。
- **你的系統是貫穿範例**。一個 SQS 風格的結算 pipeline 從 ch02 的狀態機開始，一路走到 ch15 把「at-least-once＋dedup＝exactly-once」變成定理。

## 怎麼讀

每章固定骨架：開場地圖 → **從你已知的出發**（橋接你的後端經驗）→ 核心機制 → **陷阱與防禦**（這個概念怎麼給你假安全感）→ **紙上推演**（2~4 題，附完整解答與預估時間）→ **自我檢核**（口頭自答：講不清楚＝還沒懂）→ 延伸閱讀。

推演題請真的拿紙筆做完再看解答；自我檢核用說的，不是用想的。考核過關制：上一章的檢核講不順，先別開下一章。

## 路線

- **Part I（ch01–02）**：為什麼測試不夠；一切都是狀態機。
- **Part II（ch03–05）**：邏輯、集合、歸納法——數學地基，從「大學看過」磨到能用。
- **Part III（ch06–09）**：TLA+ 的動作邏輯與時序邏輯；讀懂完整 spec；TLC 的原理與極限。
- **Part IV（ch10–13）**：四個經典協議精讀手推——Peterson、2PC、Paxos、Raft。
- **Part V（ch14–16）**：證明層——歸納不變量、refinement、TLAPS 與定理證明全景。
- **Part VI（ch17–18）**：業界誰在用、值不值；TLA+ 之外的方法地圖與下一步。
