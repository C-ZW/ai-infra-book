# ch06 — 對角化：造出說「我不可證」的句子 G

> **本章解決什麼問題**：到上一章為止，我們已經有了兩個零件——哥德爾編碼把句子和證明都變成了數字（ch05），而 ch04 已經告訴我們：只要造得出一個算術句子 G，讓它（透過編碼）說「我在 F 內不可證」，第一不完備定理的骨架就站起來了。但「造得出」這三個字，到現在還是個賒帳。自然語言可以隨手寫「這句話不可證」，靠的是「這句話」這個指針——可是 F 內部沒有「這句話」這種詞。本章補上最後一塊、也是最精巧的一塊：**對角化引理（diagonal lemma）**，它保證對任何性質 ψ，F 內部都造得出一個句子 D 在說「我有性質 ψ」，完全不靠自然語言的指針。把 ψ 取成「不可證」，脊椎句 G 就正式成形。下一章（ch07）才引爆它；本章只負責**造出來**。

## 從你已知的出發

你寫過 quine，所以你其實已經懂對角化了——只是還沒人告訴你它叫這個名字。

回想 quine 的核心難題：一個程式怎麼印出自己的完整原始碼？天真的做法馬上撞牆。你想讓程式裡有一個字串 `s`，內容就是「整個程式的原始碼」，然後 `print(s)`。但「整個程式的原始碼」**包含 `s` 這一行本身**，而 `s` 又要裝下包含自己的那整段文字——你掉進無窮回退，字串永遠裝不完自己。

quine 的解法你也記得：**別讓字串裝下自己，讓字串裝下「自己以外的部分」，再用一個機械動作把自己補回去。** 程式分兩半：一段資料 `s`（它是程式碼的文字複本，但**不含**「印出 `s`」這個動作本身），加一段邏輯（拿 `s` 當原料，先原樣印一遍當資料，再把它當程式碼印一遍）。關鍵的一招是：**那段資料被用了兩次**——一次當「被引用的文字」，一次當「要執行的程式」。同一串東西，先被當成 data 擺著，再被當成 code 跑起來，於是程式拿到了自己，而沒有無窮回退。

```text
quine 的兩用招（self-application）

   一段資料 s（程式碼的文字，但不含「印 s」這動作）
        │
        ├──► 當「資料」：原樣印出來        ┐
        │                                  ├─► 拼起來＝完整原始碼
        └──► 當「程式」：照著 s 跑          ┘
                                          ↑
                              同一串 s 被用了兩次：先 data、後 code
```

對角化引理是這一招的數學版，而且是**同一招**，不是類比。它要解決的難題一模一樣：一個算術句子，怎麼談論「自己的編碼」這個數字？天真的做法同樣撞牆——你想把 G 的編碼 ⌜G⌝ 寫進 G 裡，但 ⌜G⌝ 是個依賴 G 整體長相的數字，G 還沒寫完你算不出 ⌜G⌝，算不出 ⌜G⌝ 你就寫不完 G。無窮回退，跟 quine 一模一樣。

解法也一模一樣：別讓句子裝下自己的編碼，讓句子裝下一個**模板的編碼**（模板是「自己以外的部分」），再用一個機械的算術動作——把模板的編碼代回模板自己——把「自己」補回去。那個「把模板代進自己」的動作，就是 quine 裡「同一串 s 被用兩次」的算術化身。對角化引理的全部內容，就是證明這個動作在算術裡做得出來、而且做得乾淨。

所以本章你不是在學一個外星概念。你是在看你早就會的那個 quine 把戲，被翻譯成 F 能算的算術。

## 難題的精確形狀：F 沒有「這句話」

先把難題釘死，因為它決定了對角化為什麼**非得這麼繞**。

ch04 我們寫脊椎句時用的是白話：「G 在本系統 F 內無法被證明。」這個「G」是一個指針，指回句子自己。在自然語言裡，這種指針是免費的——你只要說「這句話」「本句」，聽者自動知道你指誰。但 F 是一台搬符號的機器（ch01），它的詞彙表裡只有邏輯符號、變數、`0`、`S`（後繼）、`+`、`×`、`=` 這些。它**沒有**一個叫「這句話」的符號，也造不出來。它能談的只有數字和數字之間的算術關係。

那 F 怎麼可能談到「自己」？答案的前半在 ch05 已經給了：透過編碼，F 能談論**任何一個具體的數字**，而句子都被編成了數字，所以 F 能談論「編號為 n 的那個句子」——只要 n 是一個寫得出來的具體數字。問題只剩後半：

```text
我們要的句子 G，必須滿足：
   G 在說的，是「編號為 ⌜G⌝ 的句子不可證」
   而 ⌜G⌝ 又必須恰好是 G 自己的編碼

   ┌─────────────────────────────────────────────────┐
   │  G 要提到一個數字 n（＝某句子的編碼）              │
   │  並斷言「n 號句子不可證」                         │
   │  但我們要 n 剛好等於 ⌜G⌝（G 自己的編碼）          │
   │  雞生蛋：n 依賴 G 的長相，G 又依賴 n              │
   └─────────────────────────────────────────────────┘
```

這就是那個無窮回退。你不能先寫好 G、再算它的編碼、再把編碼填進 G——因為填進去之後 G 就變了，編碼也跟著變，你填的數字立刻過期。對角化引理是專門拆掉這個雞生蛋的工具。它的妙處在於：它**不去解那個循環方程**，而是構造一個句子，讓循環自動閉合。

## 對角化引理：陳述與它在說什麼

先給陳述，再花力氣解釋它每個字在做什麼。本書統一寫法（與 outline 跨章基準一致）：

```text
對角化引理（diagonal lemma / 不動點引理 fixed-point lemma）

  設 F 是夠強表達基本算術的系統。對任意一元公式 ψ(x)，
  都存在一個句子 D，使得：

        F ⊢ D ↔ ψ(⌜D⌝)

  讀作：F 證明「D，若且唯若 ψ 對 D 自己的編碼成立」。
  白話：D（透過編碼）正好在說「我有性質 ψ」。
```

慢慢拆。

- **ψ(x)** 是一個「性質」，帶一個空位 x。你可以把它讀成「x 有某性質」。例如 ψ(x) 可以是「x 是偶數的編碼」「x 是一條公理的編碼」「x 不可證」。它是輸入，你想造哪種自我指涉句、就餵哪個 ψ 進去。

- **D** 是引理保證存在的那個句子，是輸出。

- **⌜D⌝** 是 D 自己的編碼（ch05 那個質數冪次積大數）。注意它出現在 ψ 的括號裡——ψ 談的不是「D」這個句子，而是「⌜D⌝」這個**數字**。F 不認識句子，只認識數字，所以 ψ 收到的是 D 的編碼。

- **`D ↔ ψ(⌜D⌝)`** 是說：D 這句話，和「ψ 對 ⌜D⌝ 成立」這句話，真假完全綁在一起。所以斷言 D，就等於斷言「ψ 對我自己的編碼成立」——D 在說「我有性質 ψ」。

- 最關鍵：**`F ⊢`**（F 證明這個等價）。對角化引理不只說「存在這麼一個 D」，它說 F **內部就能證明** D 和 ψ(⌜D⌝) 等價。這個「F ⊢」是後面 ch07 引爆時的彈藥——因為 G 的論證要在 F 內部跑，而這個等價必須是 F 自己認的。

把它讀成一句工程白話：**給我任何一個一元謂詞 ψ，我都能交給你一個句子 D，它（解碼之後）正好在斷言自己有性質 ψ；而且這個自我指涉不是我嘴上說的，是 F 用算術算出來、自己也承認的。**

這就是為什麼它是引擎。ch04 的脊椎句 G 想說「我不可證」，靠的是白話的「我」；對角化引理把「我」這個指針徹底消滅，換成一段純算術的代換運算，造出來的 D 在 F 眼裡只是一個關於某個大數字的算術斷言，毫無自我意識——但我們在外面用編碼這把鑰匙一解，發現它談的數字正好是它自己。**自我指涉被算術地建構出來，而不是被語言硬塞進去**——這正是 ch04 末尾哥德爾那條註腳（footnote 15）說的「沒有惡性循環」的精確兌現。

## 它怎麼造出來：對角線那一招

現在拆開「怎麼造」。這是本章的 worked example 之前的暖身，我會給直覺與骨架，嚴格的形式化證明指向延伸閱讀（誠實標示：以下是**工程師的嚴謹**——每步能口頭講出理由，但不是逐符號的形式證明）。

整個構造的心臟是一個算術函式，叫它 **diag**（對角化函式）。它做的事，就是 quine 裡「把資料當程式用」的算術版：

```text
diag 函式在做什麼（純機械、可在算術內表達）

  輸入：一個數字 n（它是某個一元公式 B(x) 的編碼，n = ⌜B(x)⌝）
  步驟：
    1. 解碼 n，還原出公式 B(x)            ← 質因數分解，靠 FTA（ch05）
    2. 把 n 這個數字本身，代入 B 的空位 x  ← 得到封閉句 B(n̄)
       （n̄ 是「數字 n 的算術表示」，即 SSS…S0 那串後繼符號）
    3. 把 B(n̄) 重新編碼，得到一個新數字     ← 再做一次質數冪次積
  輸出：⌜B(n̄)⌝
```

注意第 2 步：它把**一個公式的編碼 n**，代回**那個公式自己的空位**。這就是「對角線」一詞的來歷——你有一張二維表，橫軸是「公式」、縱軸是「代進去的數字」，diag 走的是對角線上「公式 × 它自己的編碼」那一格。也是 quine 裡「同一串 s 用兩次」的算術化身：n 先當「被解碼的公式」用一次，再當「代進去的數字」用一次。

diag 全程只是解碼、代換、再編碼——三個純機械步驟，沒有一步需要「理解意義」。所以（這是關鍵收穫）**diag 是一個遞迴函式，能在 F 內部用一個算術公式表達出來**。F 算得出 diag。

有了 diag，對角化引理的構造就是一個漂亮的閉環。給定你要的性質 ψ(x)，分三步：

```text
構造 D（給定目標性質 ψ(x)）

  第 1 步：造一個「模板」公式 θ(x)，內容是
            θ(x) ：「ψ( diag(x) )」
          意思是「對 x 號公式做對角化、得到的那個句子，有性質 ψ」
          θ 有一個空位 x。它是 G 以外的部分（quine 裡那個不含自己的 s）

  第 2 步：算出模板自己的編碼  q = ⌜θ(x)⌝
          （q 是個具體數字，因為 θ 已經寫死了）

  第 3 步：把 q 代回 θ 自己的空位，得到封閉句
            D ：θ(q̄) ＝「ψ( diag(q) )」
```

現在見證閉環。D 就是 θ(q̄)，而 q 是 θ 的編碼。我們對 θ 餵進它自己的編碼 q——這正是 diag 的定義動作！所以：

```text
  diag(q) = diag(⌜θ(x)⌝)
          = ⌜把 q 代入 θ 得到的句子⌝
          = ⌜θ(q̄)⌝
          = ⌜D⌝                       ← diag(q) 算出來剛好是 D 自己的編碼

  而 D 在說的是「ψ( diag(q) )」
            也就是「ψ( ⌜D⌝ )」          ← D 在說「我有性質 ψ」！

  於是  F ⊢ D ↔ ψ(⌜D⌝)               ← 對角化引理得證（骨架）
```

讀者，這裡值得你停十分鐘，因為這是全書最精巧的一頁，而你的直覺正要在一個地方打結：「為什麼 diag(q) 會剛好等於 ⌜D⌝？這不還是雞生蛋嗎？」

不是。看清楚閉環為什麼合法：**D 從來不需要『先算出自己的編碼再填進去』。** D 裡填的數字是 q——是**模板 θ 的編碼**，不是 D 的編碼。q 在第 2 步就能算出來，因為那時候 θ 已經寫死了、D 還不存在，沒有循環。「D 談的是自己的編碼」這件事，不是我們填進去的，是**事後** diag 這個算術函式幫我們補上的——diag(q) 在算術上**碰巧**等於 ⌜D⌝，因為 D 的定義（把 q 代進 θ）和 diag 的定義（把編碼代進它解碼出的公式）是同一個動作。

這正是哥德爾註腳說的：句子一開始只是一個關於某數字（diag(q)）的算術斷言，毫無自我意識；只有事後我們算出 diag(q) = ⌜D⌝，才認出它指涉自己。自我指涉是 diag 這個機械函式**算出來**的副產品，不是語言塞進去的。quine 不會無窮回退，是因為它印的是 s（自己以外的部分）再機械補全；D 不會無窮回退，是因為它填的是 q（模板的編碼）再由 diag 機械補全。**同一招。**

## Worked example：用 quine 思路走一遍「模板代入自己」

抽象的閉環看一遍可能還不踏實。我們用 quine 的思路、把它落到一個具體（但仍是紙上推演、不寫可執行碼）的走查，看「模板代入自己」如何產生「指涉自己」的句子。為了看得清楚，先用一個無害的 ψ：

```text
取 ψ(x) ：「x 是偶數」    （一個你一眼能判真假的性質，先不用「不可證」）
目標：造一個句子 D，讓 D 說「我的編碼是偶數」
```

照三步走：

```text
第 1 步｜模板 θ(x) ＝「diag(x) 是偶數」
        （口語：把 x 號公式對角化後得到的句子，其編碼是偶數）
        θ 有一個空位 x，是個寫得死死的公式。

第 2 步｜算 θ 自己的編碼 q = ⌜θ(x)⌝
        （這是一個具體的大數字。怎麼算？照 ch05：把 θ 的符號串
          逐個查碼，做 2^(a₀+1)·3^(a₁+1)·5^(a₂+1)·… 那個質數冪次積。
          數字很大但有限、可機械算出。這裡不必把它真的乘出來，
          只要知道「q 是一個確定的、寫得出來的數字」。）

第 3 步｜D ＝ θ(q̄) ＝「diag(q) 是偶數」
        （把第 2 步那個具體數字 q，代回 θ 的空位 x。）

驗證閉環：
   diag(q) = diag(⌜θ(x)⌝)
           = ⌜把 q 代入 θ 的結果⌝
           = ⌜θ(q̄)⌝ = ⌜D⌝
   所以 D 說的「diag(q) 是偶數」＝「⌜D⌝ 是偶數」
   ＝「我的編碼是偶數」                ← D 在斷言自己的編碼是偶數！
```

停下來體會這句 D。它從頭到尾只是一個關於數字的算術斷言：「diag(q) 是偶數」，其中 q 是某個具體大數字、diag 是某個算術函式。一台不懂自我指涉的機器讀它，只會看到「某個算出來的數字是不是偶數」——純算術，毫無玄機。但我們在外面算一下 diag(q)，發現它等於 ⌜D⌝，於是這句平凡的算術斷言**其實在談它自己的編碼**。

這句 D 是真是假？取決於 ⌜D⌝ 這個數字到底是奇是偶——一個確定的算術事實（雖然手算它要把整串編碼乘出來，工程上麻煩、原則上明確）。重點不是它的真假，重點是：**我們真的造出了一個算術句子，它在談自己的編碼，全程沒用到「這句話」。** 對角化把「自我指涉」這件聽起來很玄的事，降級成了「解碼—代換—編碼」三個機械步驟。

現在把 ψ 換掉，就得到脊椎句。這是全章的目的地：

```text
取 ψ(x) ＝ ¬Prov_F(x)    （x 不可證；Prov_F 是 ch05 造的可證性謂詞）

對角化引理交給我們一個句子 G，使
        F ⊢ G ↔ ¬Prov_F(⌜G⌝)

白話：G ↔「⌜G⌝ 不可證」 ＝ G 說「我不可證」
這就是脊椎句 G 的形式版正式成形（與 ch04 白話版對上）。
```

```text
脊椎句 G ：「G 在本系統 F 內無法被證明。」
形式版：  F ⊢ G ↔ ¬Prov_F(⌜G⌝)
          （由對角化引理對 ψ = ¬Prov_F 取不動點而得）
```

注意：到這裡我們**只造出了 G，還沒說它真、也沒說它不可證**。「F ⊢ G ↔ ¬Prov_F(⌜G⌝)」只是說「G 和『G 不可證』這件事綁在一起、且 F 認這個綁定」。它真不真、F 證不證得到，是 ch07 的工作。本章的全部成就，是把 ch04 賒的那筆帳——「造得出這個句子」——付清了。脊椎句現在是一個合法的、F 內部的算術句子，不再是白話的比喻。

## 為什麼叫「對角化」：康托爾的影子

「對角化」「對角線」這名字不是隨便取的，它來自康托爾（Georg Cantor）證明實數不可數的那條對角線。值得點一下，因為它讓你看到同一個動作在數學裡反覆出現（深入指向《馴服無限》談可數與不可數那章；這裡只擦亮、不重推）。

康托爾的對角論證大意：假設你能把所有實數（取 0 到 1 之間、用無限小數表示）排成一張可數的清單，第 1 個、第 2 個、第 3 個……然後他造一個新數，它的第 n 位**故意跟清單上第 n 個數的第 n 位不同**——走的正是那張表的對角線，再把每一格改掉。這個新數和清單上每一個都至少差一位，所以它不在清單裡，矛盾。對角線 ＋ 「跟對角線上的自己作對」是它的招式。

```text
康托爾對角線               對角化引理
─────────────             ─────────────
一張表：列＝清單上第 n 個   一張表：列＝第 n 個公式
       行＝小數第 n 位            行＝代進去的第 n 個數
走對角線：取第 n 個的第 n 位 走對角線：把第 n 個公式餵它自己的編碼（diag）
動作：把對角線上每格「改掉」 動作：對對角線那格「套上性質 ψ」
產物：一個不在清單裡的數     產物：一個說「我有性質 ψ」的句子
共同點：用「自己餵自己」(對角線) 製造一個逃出原系統掌握的東西
```

兩者的血緣是「**把一個東西餵給它自己**」——康托爾餵的是「第 n 個的第 n 位」，哥德爾餵的是「第 n 個公式的編碼 n」（diag）。停機問題（ch09）會第三次用這招，造一個「對自己作對」的程式。所以你可以記一句話：**對角化是『自我指涉』這台引擎在數學裡的標準點火方式**——康托爾用它造出逃出清單的數，哥德爾用它造出逃出證明的句子，圖靈用它造出判不出停不停的程式。同一根火柴，點了三個世紀最深的三把火。

## 一點歷史：這是誰的引理

歸屬要寫清楚，因為它常被簡化成一人之功（照 landscape 第 9 條與 web 查證，2026-06）。

哥德爾在他 1931 年的不完備論文裡，**用了對角化引理的一個特例**——他為了造 G 而做的「代換＋自我指涉」構造，本質就是 diag，但他是針對「不可證」這個特定性質手工打造的，沒有把它抽象成「對任意 ψ 都成立」的一般引理。

把它一般化成「對任意一元公式 ψ(x) 都存在不動點 D」這個我們今天看到的形式，**首次明確發表的是卡爾納普（Rudolf Carnap），在他 1934 年的《語言的邏輯語法》（Logische Syntax der Sprache）裡**。哥德爾自己 1934 年的講稿裡引用了卡爾納普這個一般化（並在註腳致謝）。所以標準的歸屬寫法是：**哥德爾 1931 用其特例、卡爾納普 1934 一般化**，有些文獻因此主張稱它為「哥德爾—卡爾納普不動點定理」。

這裡誠實 hedge 一句（landscape 第 9 條要求）：「對角化引理」這個名字是後世所加；而卡爾納普 1934 一般化的精確程度，學界有細節討論——有論者指出「這個一般化在今天看來顯然，但在 1934 年一點都不顯然」。教科書傳統把一般化歸給卡爾納普，本書採用這個傳統寫法，但不把它當成毫無爭議的鐵案。一句話：構造的靈魂是哥德爾的，把它磨成一把通用工具的，是卡爾納普。

## 直覺的陷阱

對角化是這本書裡最容易讓人「以為懂了其實沒懂」的一章，因為它的閉環太巧，巧到容易被讀成魔術或詭辯。守住下面幾條。

| 陷阱 | 錯誤直覺 | 它在哪一步帶你下溝 | 正確版 |
|---|---|---|---|
| 「G 把自己的編碼填進自己，這是循環定義／非法」 | 既然 G 談 ⌜G⌝，那一定是先有 G 才有 ⌜G⌝，循環 | 沒看出 D 填的是**模板 θ 的編碼 q**，不是 D 的編碼；q 在 D 存在之前就算得出，沒有循環 | 「D 談自己的編碼」是 diag 這個算術函式**事後**算出來的（diag(q)=⌜D⌝），不是填進去的。閉環合法 |
| 「對角化引理就是說謊者，只是換了包裝」 | 都靠自我指涉，所以一樣可疑 | 把「造出自我指涉句」當成「造出悖論」 | 對角化只**造句子**，不保證句子矛盾。餵 ψ=「偶數」造出無害句、餵 ψ=¬Prov 造出 G、餵會自爆的真假謂詞才出事——出不出事看 ψ，不看對角化 |
| 「對角化需要某種無限或自我意識的魔法」 | 一個句子能談自己，背後一定有玄學 | 把 diag 想成神祕，沒看出它只是解碼—代換—編碼三個機械步 | diag 是純遞迴函式，F 內部可表達。整個構造零魔法，全是 ch05 那套編碼在搬數字 |
| 「對角化引理本身就證明了不完備」 | 既然它造出了 G，定理是不是就成立了 | 把「造出 G」當成「引爆 G」 | 本章只給 `F ⊢ G ↔ ¬Prov_F(⌜G⌝)`，沒說 G 真、沒說 G 不可證。引爆要等 ch07，還要用到一致性與三條件 |
| 「對角化引理只對『不可證』有效」 | 因為我們只拿它造 G | 把工具誤當成它的一個用途 | 引理對**任意** ψ 都成立。換 ψ 能造塔斯基不可定義性（ψ＝真理謂詞→出事，ch12）、勒布定理、停機歸謬（ch09）——它是通用不動點機器 |

最該自我察覺的一條：**當你覺得「這是循環論證」「這是文字遊戲」時，停下來找那個 q。** 問自己：「D 裡填的到底是哪個數字？是 D 的編碼還是模板的編碼？」只要看清楚填的是模板 θ 的編碼 q（D 出生前就算得出），而「指涉自己」是 diag 事後補的，循環的錯覺就會消失。對角化不是把矛盾藏起來，它是把「自我指涉」這件事拆成了不含循環的機械步驟——這恰恰是它了不起的地方。

## 紙上推演

> 全程紙筆，不需要跑任何程式。所有「編碼／代換」用文字描述或小數字示意，不必把巨大的質數冪次積真的乘出來。

### 推演題 1 — 換一個 ψ，說出對角化會造出什麼句子 ★★ **[12 分鐘]**

把目標性質 ψ 依序換成下面三個，分別用一句白話說出「對角化引理會交給你一個怎樣的自我指涉句 D」，並判斷它大概是真是假（或無法立刻判斷）：

(a) ψ(x) ＝「x 是一條公理的編碼」
(b) ψ(x) ＝「x 的編碼長度小於 100 個符號」（「我很短」）
(c) ψ(x) ＝ Prov_F(x)（「x 可證」，注意這次**沒有**否定號）

### 推演解答

```text
(a) ψ＝「是公理」 → D 說「我是一條公理」
    真假：看 F 的公理表裡有沒有 D 這個編碼。D 是對角化造出來的、
    通常長相複雜，幾乎不會剛好是某條公理，所以多半「我是公理」為假。
    但這不矛盾——一個假的自我指涉句完全合法（它只是說錯了自己）。

(b) ψ＝「我很短」 → D 說「我的編碼長度小於 100 個符號」
    真假：對角化造出的 D 通常頗長（含 diag、含具體大數字 q），
    多半超過 100 符號，所以「我很短」為假。同樣：假句合法，不出事。

(c) ψ＝Prov_F → D 說「我可證」    ← 這個有意思
    這是「勒布句」的雛形。它不像 G（說「我不可證」），而是說「我可證」。
    它真不真？勒布定理（Löb，ch08 會一句帶過）給出驚人答案：
    這種「我可證」的句子，F 真的證得出來、而且它真。先擱著，
    這裡只要看出：同一台對角化機器，換個 ψ 就造出完全不同命運的句子。
```

共同教訓：對角化是中性的造句機器，**句子的命運（真／假／可證／自爆）由 ψ 決定，不由對角化決定**。把「對角化＝製造悖論」這個錯覺在這題裡徹底丟掉。

### 推演題 2 — 為什麼對角化不需要自然語言的「這句話」 ★★★ **[20 分鐘]**

向一個沒讀過本章的工程師解釋：自然語言寫「這句話不可證」靠的是「這句話」這個指針，但 F 裡沒有這個詞——對角化引理是怎麼在**沒有指針**的情況下，造出一個指涉自己的句子的？要求講清楚「模板 θ」「編碼 q」「diag」三者各扮演什麼角色，並指出「自我指涉」是在哪一步、由誰補上的。

### 推演解答

骨架答案（你的版本只要涵蓋這幾個關節即可）：

```text
1. 難題：F 沒有「這句話」這種指針，只能談具體數字。要 G 談自己，
   卡在雞生蛋——⌜G⌝ 依賴 G 的長相，G 又要含 ⌜G⌝。

2. 對角化的繞法（quine 的招）：不直接填 G 的編碼，改填「模板的編碼」。
   - 模板 θ(x)：「diag(x) 有性質 ψ」。它是「G 以外的部分」，寫得死、
     有一個空位 x。對應 quine 裡那段不含自己的資料 s。
   - 編碼 q＝⌜θ⌝：θ 寫死後就能算出來的具體數字。此時 G 還不存在，
     所以算 q 沒有循環。
   - diag：一個算術函式，把「公式的編碼」代回「公式自己」再重新編碼。
     對應 quine 裡「同一串資料先當 data、再當 code」那一招。

3. 造 D＝θ(q̄)：把 q 代回 θ。D 說的是「diag(q) 有性質 ψ」，
   表面上只是個關於數字 diag(q) 的平凡算術斷言。

4. 自我指涉是『事後』補上的：我們在外面算 diag(q)，發現
   diag(q)＝⌜θ(q̄)⌝＝⌜D⌝。於是「diag(q) 有性質 ψ」其實＝
   「⌜D⌝ 有性質 ψ」＝「D 自己有性質 ψ」。
   —— 指涉自己這件事，是 diag 這個機械函式算出來的，
      不是「這句話」這個指針指出來的。
```

一句話總結（這是你要能脫口而出的）：**對角化用「模板的編碼」加上「把編碼代回模板的算術函式 diag」，機械地製造出『談自己』的效果，於是完全不需要自然語言的指針。**「我」這個字被一段算術代換取代了。

常見錯路：(1) 說「D 把自己的編碼填進去」——錯，填的是模板 θ 的編碼 q；(2) 說「diag 算出 G」——diag 不算 G，它算的是編碼之間的對應，是這個對應**碰巧**讓 D 指向自己；(3) 把「自我指涉」說成構造的前提——它是構造的**結論**，事後才浮現。

### 推演題 3 — 對角化 vs 康托爾對角線：對齊那一格 ★★ **[15 分鐘]**

康托爾用對角線造出一個「不在清單裡」的實數；哥德爾用對角化造出一個「逃出證明」的句子。把兩者的「表格、對角線、對自己作對的動作、最終產物」四個要素一一對齊，並用一句話說出它們共同的招式是什麼。

### 推演解答

```text
要素        康托爾                          哥德爾（對角化引理）
────────   ──────────────────────────     ──────────────────────────
表格       列＝清單第 n 個實數              列＝第 n 個一元公式
           行＝小數第 n 位                  行＝代進空位的第 n 個數
對角線      取第 n 個數的第 n 位             把第 n 個公式餵它自己編碼 n（diag）
對自己作對  把對角線那位「改成不一樣的」      對對角線那格「套上性質 ψ」
產物       一個不在清單裡的新實數            一個說「我有性質 ψ」的句子 D
────────   ──────────────────────────     ──────────────────────────
```

共同招式：**把一個東西餵給它自己（走對角線），再對「自己餵自己」的結果動手腳，造出一個原系統罩不住的東西。** 康托爾罩不住的是「清單」（新數逃出可數清單），哥德爾罩不住的是「證明」（G 逃出可證範圍）。停機問題（ch09）會第三次用同一招。記住這個血緣，你會發現本書後半的「驚雷」其實都是這一根火柴點的。

## 自我檢核

口頭自答，講得出來才算過關：

1. F 內部沒有「這句話」這個詞，為什麼這對「造一個說『我不可證』的句子」是個真正的難題，而不是雞蛋裡挑骨頭？
2. 對角化引理的陳述 `F ⊢ D ↔ ψ(⌜D⌝)` 裡，每個部件（ψ、D、⌜D⌝、那個 ↔、最前面的 `F ⊢`）各在說什麼？為什麼「F ⊢」這三個字是後面引爆 G 的彈藥？
3. diag 函式到底做哪三件事？為什麼它能在 F 內部用算術表達出來（提示：它沒有一步需要「理解意義」）？
4. 構造 D 時，我們填進模板的是哪個數字——D 自己的編碼，還是模板 θ 的編碼 q？為什麼填後者就沒有循環？
5. 「D 指涉自己」這件事是構造的**前提**還是**結論**？是誰、在哪一步把它補上的？
6. 為什麼對角化引理不是「換了包裝的說謊者」？同一台對角化機器，餵什麼 ψ 會造出無害句、餵什麼會造出 G、餵什麼會自爆？
7. 本章造出了 G，為什麼還不能說「所以 F 不完備」？還缺哪些東西（提示：真、不可證、三條件、一致性）要等 ch07？
8. 康托爾對角線、哥德爾對角化、（預告）圖靈停機，三者共同的「招式」用一句話怎麼說？
