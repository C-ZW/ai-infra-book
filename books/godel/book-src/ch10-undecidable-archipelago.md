# ch10 — 不可判定的群島：決定問題與可計算性的版圖

> **本章解決什麼問題**：上一章（ch09）讓你看了停機問題——圖靈用「會不會停」重演了哥德爾的對角化好戲，造出一個沒有任何程式能判定的問句。本章把鏡頭從那一個句子拉開，看清它坐落的整片版圖：希爾伯特那台「能判定任意命題真假」的夢幻機器（決定問題，Entscheidungsproblem），在 1936 年被邱奇與圖靈各自獨立否決；而「可機械計算」這個直覺，被圖靈機、λ-演算、遞迴函數三種長相完全不同的模型同時抓住，匯成 Church–Turing 論題。停機不是孤島，它屬於一整群不可判定問題組成的群島——一階邏輯有效性、Post 對應、希爾伯特第十問題都在島上。讀完你會看到：希爾伯特計畫四根柱子裡的「可判定」這一根，是在這裡正式倒下的（ch02 立的標靶，第二根結算）。本章是廣度章，不重走 ch09 的停機歸謬，只點名引用它的結論。

## 從你已知的出發

你心裡早就有一條紅線：**「寫一個工具，判定任意程式有沒有 bug」這種需求，一聽就知道做不到。**

不是因為工程師不夠努力。是你隱約知道，有些「判定任意 X 是否具有性質 P」的請求，原則上沒有通用解。最乾淨的例子就是 ch09 的停機問題：沒有一個程式 `willHalt(P, w)` 能對任意程式 `P` 與輸入 `w` 正確回答「`P` 跑 `w` 會不會停」。你接過「幫我偵測所有死迴圈」的需求，你知道頂多做到啟發式——抓得到一些明顯的、漏掉一堆刁鑽的，永遠不可能零漏判。

而且這條紅線比你以為的還長。考慮這些需求，每一個你大概都本能地皺過眉：

- 「寫個分析器，判定任意函式會不會 throw。」
- 「寫個工具，判定兩段程式碼是不是等價（對所有輸入給一樣的輸出）。」
- 「寫個 linter，標出所有死碼（永遠執行不到的分支）。」
- 「寫個檢查器，判定任意函式是不是純函數（無副作用）。」

這些不是「目前還沒人做出來」，是**原則上不存在通用演算法**。而且它們不是各自獨立的壞運氣——它們全是同一個極限的不同投影。1953 年萊斯（Henry Rice）把這件事一次講死：**程式的任何「非平凡的語意性質」都不可判定**（Rice's theorem，2026-06 查證）。「語意性質」指的是關於程式**行為**的性質（會不會停、會不會 throw、算不算純函數），相對於「語法性質」（程式碼裡有沒有 `if`、有幾行）；「非平凡」指這性質不是對所有程式都成立、也不是對所有程式都不成立。一句話：**只要你問的是「這程式實際上會做什麼」，而那問題的答案不是恆真也不是恆假，就沒有通用判定器。** 你對「完美靜態分析器不存在」的工程直覺，其實就是萊斯定理的庶民版。

本章要做的，是把這條你早就摸到的紅線，接回哥德爾的脊椎，並告訴你它有多長、屬於多大一片版圖、又拆掉了希爾伯特的哪根柱子。我們先回到 1928 年，希爾伯特畫下那台夢幻機器的規格。

## 決定問題：希爾伯特要的那台判定機

回憶 ch02。希爾伯特計畫有四根柱子：**完備**（每個真命題都可證）、**一致**（不證矛盾）、**有限性一致性自證**（用不會被質疑的手段證明自己不會爆）、以及第四根——**可判定（decidable）**。

第四根的精確名字叫**決定問題（Entscheidungsproblem，本書保留德文原詞並附中譯「決定問題」）**。希爾伯特與阿克曼（Wilhelm Ackermann）1928 年在《理論邏輯綱要》（*Grundzüge der theoretischen Logik*）裡把它寫成一個乾淨的請求（2026-06 查證，照 landscape）：

```text
決定問題（Entscheidungsproblem，Hilbert–Ackermann 1928）：

  有沒有一個一般的機械程序（一台機器、一套演算法），
  能對「一階謂詞邏輯」裡任意給定的一個式子 φ，
  在有限步內正確回答：「φ 是不是（普遍）有效的？」

  （「有效 valid」＝在所有可能的解釋／模型下都為真——
    純邏輯為真，不靠任何特定領域的事實。）
```

對工程師，這台機器是什麼？是**數學版的「綠燈全自動 CI」**。希爾伯特要的是：你把任何一個邏輯命題丟進去，機器轉一轉，吐出 `valid` 或 `not valid`，一定停、一定對。有了它，數學就「自動化」了——任何猜想，輸進去，等答案。沒有靈感、沒有大海撈針、沒有卡關，只有一台不知疲倦的判定機在跑。

希爾伯特本人對這台機器極度樂觀。他在 1930 年柯尼斯堡那句著名的「Wir müssen wissen, wir werden wissen（我們必須知道，我們必將知道）」（ch02），喊的就是這個信念：數學沒有「原則上不可知」的問題，差別只在我們還沒算到。決定問題若有解，這信念就有了鋼鐵地基。

這裡要先把一個容易撞混的東西分開，因為它正是 ch03 講過的完備性定理在這裡的回音：

```text
別撞混這兩件事（同一個一階邏輯，兩個不同的問句）：

  (1) 一階邏輯「完備」嗎？           ← 哥德爾 1929 完備性定理：是。
      （每個語意有效的式子，都可證。「規則夠用」。）

  (2) 一階邏輯有效性「可判定」嗎？   ← 決定問題：否。（本章。）
      （有沒有機器能對任意式子判定它有不有效。）

  兩者不矛盾：規則夠用（有效⇒一定證得出），
  但「證得出」這件事本身沒有判定上界——
  你可以機械地把所有證明一條條列出來（半可判定），
  卻沒有保證何時該放棄找的停止信號。
```

第二個問句的答案是「否」。而且它是在哥德爾拆掉前三根柱子（ch07–08）後，由另外兩個人、用另外兩種武器，在同一年砍下的。

## 1936：兩個人、兩種武器、同一個否決

決定問題的否定答案，1936 年由邱奇（Alonzo Church）與圖靈（Alan Turing）**各自獨立**給出。這件事的戲劇性在於：兩人用了長相完全不同的工具，得到同一個結論——這本身就是 Church–Turing 論題的第一個證據。

**邱奇在前（約早七個月）。** 邱奇用他發明的 **λ-演算（lambda calculus）**——一套只有「函數抽象與套用」兩個動作的極簡計算模型——證明了「判定兩個 λ-表達式是否等價」沒有通用解，並據此把決定問題否決掉。他的關鍵論文〈An Unsolvable Problem of Elementary Number Theory〉1935-04-19 向美國數學學會宣讀、1936-04 刊出（2026-06 查證），另有一篇短文〈A Note on the Entscheidungsproblem〉1936-04-15 投稿、刊於《Journal of Symbolic Logic》創刊號（1936-03，照 landscape）。

**圖靈在後（同年）。** 圖靈用他發明的**圖靈機**，先證停機問題不可判定（ch09），再把決定問題歸約到停機問題：若有判定機能解決一階有效性，就能拿它解停機——但停機已知無解，矛盾。所以決定問題無解（Turing 1936，論文〈On Computable Numbers, with an Application to the Entscheidungsproblem〉；本書用「1936」並一句註明：所在卷標年 1937、含 1937 勘誤，照 landscape T11）。

兩件事值得停下來看。

第一，**圖靈寫論文時並不知道邱奇已經做出來了。** 等他快收尾，刊著邱奇論文的那期期刊寄到了劍橋，圖靈才趕忙補了一個附錄，草草證明「圖靈機可計算」與「λ-可定義」其實是同一回事（2026-06 查證）。兩條獨立的路，撞在同一個終點。

第二，順帶澄清一個常被張冠李戴的細節：「Turing machine（圖靈機）」這個名字不是圖靈自己取的，是邱奇 1937 年在一篇書評裡首創的（JSL 2(1), 1937, pp. 42–43，2026-06 查證）。

把這一刀接回本書的脊椎：希爾伯特立了四根柱子，哥德爾的兩條定理砍倒了前三根（完備←ch07、有限性一致性自證←ch08、外加「能自證一致」這個夢←ch08），而**第四根「可判定」，是邱奇與圖靈在 1936 年砍倒的。** ch02 立的那個標靶，到這裡四根全倒。

## Church–Turing 論題：可計算的三張臉

邱奇與圖靈撞在同一個終點，不是巧合。1930 年代，好幾個人從完全不同的動機出發，各自定義「什麼叫一個函數是可機械計算的」，結果這些定義**全部等價**：

```text
「可機械計算」的三張臉（1930 年代各自提出，後被證兩兩等價）：

  圖靈機（Turing 1936）        一條無限長磁帶 + 一個讀寫頭 + 一張狀態轉移表
                              ≈ 你熟的：帶無限記憶體的有限狀態機

  λ-演算（Church 1930s）       只有「定義函數」與「套用函數」兩個動作
                              ≈ 你熟的：純函數、無副作用、一切皆函數

  一般遞迴函數（Gödel–Herbrand–Kleene）  從幾個基本函數出發，
                              用組合、原始遞迴、最小化堆出來的函數
                              ≈ 你熟的：遞迴定義的函式

         全部等價 ⇒ 它們框住的是「同一類」函數
```

這個「同一類」的斷言，就是 **Church–Turing 論題（Church–Turing thesis）**：

> **凡是直覺上「能被機械程序計算」的函數，恰好就是圖靈機能算的（＝λ-可定義的＝一般遞迴的）。**

對工程師，這個直覺你早就有：你知道「不同語言、不同 runtime 算得出的東西其實是同一類」。一個問題若 Python 解得了，C、Haskell、Lisp、一台你手刻的暫存器機，原則上也都解得了（只差快慢與優雅）；解不了的，換語言也救不了。「圖靈完備（Turing-complete）」這個你天天掛在嘴上的詞，講的正是這件事——一個語言只要強到能模擬圖靈機，它能算的就是「可計算」的全部，不多不少。

但這裡有一個**極重要、極容易被科普寫錯**的點，我要單獨拉出來講，因為 landscape 把它列為低信心陷阱項：

```text
Church–Turing 是「論題（thesis）」，不是「定理（theorem）」。

  「圖靈機 = λ = 遞迴」這三者等價——這部分是「定理」，
   被嚴格證明了（三個形式模型互相等價）。

  但「這三者 = 我們直覺中『能機械計算』的全部」——這部分是「論題」，
   不是定理。因為「直覺上能機械計算」是個非形式的概念，
   你沒辦法對一個沒有精確定義的東西做數學證明。
   它只能被一再印證（八十多年沒人找到反例），無法被證明。
```

為什麼這個區別重要？因為它本身就是個小小的哥德爾式謙卑：我們對「可計算」的信心，不是來自一條被證明的定理，而是來自「八十年來，每一個被認真提出的計算模型，最後都被證明不超過圖靈機」。這是經驗的歸納，不是邏輯的必然。你寫程式時對「圖靈完備就是天花板」的信任，建立在同樣的沙地上——只是這片沙地夯得非常非常實。

## 不可判定的群島

現在把視野徹底拉開。停機問題不是一座孤島。一旦圖靈證了它不可判定，一個標準技巧立刻量產出一整群不可判定問題：**歸約（reduction）**。

歸約的工程直覺你熟得很：**「如果能解 B，就能解 A；而 A 已知無解；所以 B 也無解。」** 你 debug 時天天這樣推——「如果這個快取沒壞，那這個 bug 就不可能發生；但 bug 發生了，所以快取壞了」。把「快取壞了」換成「停機問題有解」，把「bug 發生了」換成「矛盾」，就是不可判定性證明的骨架（ch09 的停機歸謬就是把「停機可判定」歸到一個自我指涉的矛盾）。

於是，只要把停機問題「藏」進另一個問題 B 裡——證明「能解 B ⇒ 能解停機」——B 就被釘成不可判定。這台量產機跑了八十年，造出一整片群島：

```text
不可判定的群島（點名給直覺，本書不展開各自證明）：

  ┌─ 停機問題 ──────────── 任意 (程式 P, 輸入 w)，P(w) 會停嗎？（ch09，群島的源頭）
  │
  ├─ 一階邏輯有效性 ─────── 任意一階式子 φ，普遍有效嗎？（決定問題，Church/Turing 1936）
  │
  ├─ 程式等價 ──────────── 兩段程式對所有輸入給一樣的輸出嗎？（Rice 定理涵蓋）
  │
  ├─ 任意非平凡語意性質 ── 會不會 throw？是不是純函數？（Rice 1953，一網打盡）
  │
  ├─ Post 對應問題（PCP） ── 一組「上下兩行字串」的骨牌，能不能排出上下相同的序列？
  │                          （Emil Post 1946；常被當成證別的問題不可判定的跳板）
  │
  └─ 希爾伯特第十問題 ───── 任意整係數多項式方程（丟番圖方程），有沒有整數解？
                            （Matiyasevich 1970 完成的 MRDP 定理：無通用演算法）
```

我特別想讓你看清兩座島，因為它們把「不可判定」從「程式的怪癖」拉到了「純數學的腹地」：

**Post 對應問題（PCP，Post 1946，2026-06 查證）。** 想像一副骨牌，每張骨牌上下各印一個字串，比如上 `ab` 下 `a`、上 `b` 下 `bb`。問題是：能不能挑一串骨牌（可重複），讓「上面接出來的字串」等於「下面接出來的字串」？這問題看起來像個拼字遊戲，毫無「自我指涉」的味道——但它不可判定。它之所以重要，是因為它比停機問題「形狀更簡單」，常被當成跳板：要證某個新問題不可判定，把 PCP 歸約進去，往往比直接歸約停機問題省力。

**希爾伯特第十問題（MRDP 定理，Matiyasevich 1970 完成，集 Davis、Putnam、Robinson、Matiyasevich 四人二十一年之功，2026-06 查證）。** 希爾伯特 1900 年的第十問題問：有沒有通用演算法，判定任意整係數多項式方程（丟番圖方程，如 `x² + y² = z²` 這類，只求整數解）有沒有整數解？答案是**沒有**。這座島離程式碼最遠、離「純數學」最近——它說的是「連『這條方程有沒有整數解』這麼樸素的數論問題，都沒有萬用判定法」。不可判定性不是計算機科學的家務事，它住在數論的正中央。

這就是「群島」這個詞的意思：不可判定不是一個孤立的技術現象，是一整片相連的大陸架，從程式行為一路延伸到丟番圖方程。而它們全都同源——都可以追溯到那一招自我指涉的對角化。

## 三層地貌：可判定 / 半可判定 / 完全不可判定

把群島放進更大的地圖，要分三層。這個分層是本章給你的核心版圖直覺，務必握緊：

```text
可計算性的三層地貌（按「機器能幫你到什麼程度」分）：

  可判定（decidable）
      有機器，對任意輸入，「是」與「否」都在有限步內正確回答、保證停。
      例：命題邏輯有效性（真值表硬算）、Presburger 算術（只有加法，ch13）。
          ── 兩面都答得了，這是最舒服的一層。

  半可判定（semi-decidable，＝遞迴可列舉 r.e.）
      只有「是」這一面保證在有限步答得了；「否」那面可能永遠等不到。
      例：一階邏輯有效性、停機問題、「φ 在 F 中可證嗎」。
          ── 你能寫個程式把所有「是」的情形一條條列出來（會停的程式跑了就知道、
             可證的句子把所有證明列舉遲早撞到），但對一個「否」的輸入，
             你的程式會永遠跑下去，你永遠不敢斷定「它真的不會停 / 真的不可證」。

  完全不可判定（neither r.e. nor co-r.e.）
      連半邊都搆不到，「是」和「否」兩面都列舉不出來。
      例：判定圖靈機是否全停（對所有輸入都會停，即 Tot 問題），比停機更深一層。
          ── 這層離「機器能幫忙」最遠。
```

對工程師，最該內化的是**半可判定**這一層，因為它解釋了你天天碰到的不對稱：

- **測試／執行**是半可判定的「是」那一面：程式真的停了、bug 真的觸發了，你跑一下就確認得了。但「這程式對所有輸入都不會停 / 永遠不出 bug」，你跑不出來——你只能跑有限多個輸入，永遠剩下「萬一還有沒測到的」。
- **證明搜尋**也是半可判定的「是」那一面（ch01 講過「驗易、找難」的不對稱，這裡是它的精確版）：你可以寫一台機器，把系統 F 的所有合法證明一條條列舉出來（公理就那些、規則就那些，機械地排列組合），於是「φ 可證」這件事，只要 φ 真可證，機器遲早會列舉到它的證明。但「φ 不可證」呢？機器列到天荒地老都沒撞到，你也不敢喊停——因為你不知道是「真的沒有」還是「下一條就是」。

把這個半可判定的不對稱，連回脊椎句 G：

```text
脊椎句 G ：「G 在本系統 F 內無法被證明。」（ch04/06 那句）

  「列舉 F 的所有證明」是半可判定的——可證的遲早列到，不可證的永遠等不到。
  而 G 恰恰落在「永遠等不到」那一邊：F ⊬ G（ch07）。
  你那台列舉證明的機器，會永遠跑下去，永遠列不到 G 的證明——
  因為根本沒有。但 G 在 ℕ 中是真的（ch12 細談）。
  ── 停機問題的「會停的列得到、不停的等不到」，
     和不完備的「可證的列得到、G 等不到」，是同一個半可判定的不對稱。
     這就是 ch09 說的「孿生」在版圖層級的樣子。
```

可判定 / 半可判定的這道縫，就是哥德爾與圖靈的極限在可計算性地圖上的等高線。希爾伯特要的是「全部可判定」——整張地圖都是最舒服的第一層。實際的地貌是：絕大多數有趣的問題，都落在第二層甚至第三層。

## Worked example：希爾伯特四柱結算表

本章是 Part IV 收束「拆希爾伯特標靶」的關鍵節點。我把 ch02 立的四根柱子拿出來，逐根結算——哪根、哪一章、被誰、用什麼砍倒。這張表不是可計算性教科書的判定速查表，是**把全書前半的標靶結帳**的對帳工具，每一格都連回脊椎：

```text
希爾伯特計畫四柱 vs 誰拆了它（全書標靶結算）：

  柱子                希爾伯特要的            倒在哪一章 / 被誰 / 用什麼武器
  ──────────────────────────────────────────────────────────────────────
  ① 完備              每個真命題都可證        ch07 第一不完備定理（哥德爾 1931）
     （complete）      （沒有真而不可證的）     ── 造出 G：真（在 ℕ）但 F ⊬ G。
                                              真理溢出證明。

  ② 一致自證          系統能用自己的有限手段    ch08 第二不完備定理（哥德爾 1931）
     （+有限性）       證明自己不會矛盾          ── F ⊬ Con(F)。連保命符都摸不到。
                                              （根岑 ch15 用外部更強原則補救，
                                                但「自證」的夢仍死。）

  ③ 可判定            有機器判定任意命題真假    ch09／ch10 決定問題否決
     （decidable）                            （Church 1936 λ／Turing 1936 機器）
                                              ── 連一階有效性都無通用判定法。
                                                 ◄ 本章砍的就是這根。

  ④（含於①②）有限性    一致性證明只用            ch08（自證不可能）＋ ch15
                       不可質疑的有限手段        （根岑用到 ε₀ 超限歸納，
                                              超出有限手段——夢的有限性版本也死）
  ──────────────────────────────────────────────────────────────────────
  結算：四根全倒。前三根哥德爾砍（1931），第三根可判定那一刀
        由邱奇與圖靈補上（1936）。同一個自我指涉的對角化，
        在「可證性」面前是不完備，在「可計算性」面前是不可判定。
```

逐格複核一遍，確認沒有跳步、沒有掉條件：

- **第①根**靠 ch07：在三條件（一致、可遞迴公理化、夠強表達算術）下，G 真而不可證。注意這裡守住條件——不是「所有系統都不完備」，是「夠強又一致的系統」。✓
- **第②根**靠 ch08：`F ⊬ Con(F)`，其中 `Con(F) ≡ ¬Prov_F(⌜0=1⌝)`。前提同樣是那三條件（外加 Prov 滿足 HBL 三條件 D1–D3）。✓
- **第③根**靠本章：決定問題無解，Church 1936 在前（λ-演算）、Turing 1936 在後（機器＋歸約停機）。砍的是「一階邏輯有效性可判定」。✓
- **第④根**是有限性要求，它纏在①②裡——希爾伯特要的「一致性證明只用有限手段」，被第二定理斷了「自證」這條路，而 ch15 根岑雖證了 PA 一致，用的卻是到 ε₀ 的超限歸納（超出有限手段）。所以「有限性自證」這個夢，兩頭都不通。✓

對照一句話收尾：哥德爾在「真 vs 可證」的軸上拆穿了希爾伯特，圖靈與邱奇在「能算 vs 不能算」的軸上拆穿了他。**兩條軸，同一招對角化，同一個自我指涉的極限。** 這就是 ch09 標題「孿生子」的全貌——本章把這對孿生子放回它們共同統治的整片版圖。

## 直覺的陷阱

本章的版圖最容易被三種混淆弄亂。先抓具名的：

| 陷阱 | 錯誤直覺帶你下溝的那一步 | 正確版（怎麼自我察覺） |
|---|---|---|
| **「不可判定＝無法決定／見仁見智」** | 看到「undecidable」就讀成日常的「下不了決定」「看你怎麼想」「沒有標準答案」，於是把它當成相對主義或主觀性。 | 「不可判定（undecidable）」是個精確技術義：**不存在一個通用機械程序，對任意輸入都正確回答。** 它跟「答案不存在」「答案主觀」毫無關係——每個具體的停機實例，答案都是客觀確定的「停」或「不停」，只是沒有**統一**的演算法一次解決全部。察覺法：問自己「我是不是把『沒有萬用判定機』偷換成了『這件事沒有客觀答案』？」前者是對的，後者是越界。（這正是 ch14 濫用解剖會放大的術語漂移。） |
| **「Church–Turing 論題已經被證明了」** | 把「圖靈機＝λ＝遞迴」（這是定理）的證成，順手安到「這些＝直覺可計算的全部」（這是論題）頭上，於是以為「可計算就是圖靈機能算的」是被證明的鐵律。 | 三個形式模型互相等價——是定理。但「它們窮盡了直覺的可計算」——是論題，因為「直覺可計算」沒有形式定義，無法被數學證明，只能被一再印證、沒找到反例。察覺法：問「我這個斷言裡，有沒有一端是非形式的概念？」只要有，那就是論題不是定理。 |
| **「半可判定＝大概可以判定／判定得差不多」** | 把「半（semi）」聽成「打了個折的可判定」「九成可判定」，以為只是效率問題。 | 半可判定是**結構性的單邊**：「是」那面保證有限步答得了，「否」那面**可能永遠等不到**——不是慢，是原則上沒有停止信號。停機問題半可判定，意味著你能確認「會停」的程式，但對一個實際永不停機的程式，你的偵測器會跟著它一起永遠跑下去。察覺法：問「『否』那一面有沒有保證會在有限步停？」沒有，就是真正的半可判定，不是效率問題。 |

還有一個更隱蔽、值得獨立點名的：**把「沒有通用判定機」誤讀成「每個個案都解不了」**。完全相反。希爾伯特第十問題沒有通用演算法，但無數個別的丟番圖方程，數學家照樣解得漂漂亮亮；停機問題不可判定，但你寫的絕大多數函式，你一眼就看得出會不會停。「不可判定」否定的是**一台搞定全部的萬用機器**，不是**每一個個案**。把「無萬用解」滑成「全都無解」，是這片版圖最常見的悲觀誤讀——它比實際情況絕望得多。

## 紙上推演

以下純紙筆，不需要任何程式。

### 題目一（★，10 分鐘）：把「可計算」三模型對應到你熟的東西

Church–Turing 論題說圖靈機、λ-演算、一般遞迴函數框住同一類函數。請把這三個模型各對應到一個你工作中熟悉的東西，並用一句話說明為什麼這個對應抓得住該模型的精神。然後回答：為什麼「它們三個等價」是定理，而「它們＝可計算的全部」只是論題？

### 題目二（★★，15 分鐘）：歸約的方向別搞反

歸約證明的骨架是「能解 B ⇒ 能解 A；A 無解；故 B 無解」。現在有人要證「程式等價問題（判定兩段程式對所有輸入是否同輸出）不可判定」，他說：「我把程式等價問題歸約到停機問題——也就是說，如果我能解停機，我就能解程式等價，所以程式等價不可判定。」

請指出他的論證**方向錯在哪**，並寫出正確方向應該怎麼歸約（不必給細節，只要說清楚「拿哪個已知無解的問題、藏進哪個待證問題」）。

### 題目三（★★，15 分鐘）：哪一層？

把下面五個問題各歸到「可判定 / 半可判定 / 完全不可判定」哪一層，並說明判準（特別是：「否」那一面有沒有保證有限步停）：

（a）命題邏輯（不含量詞）的某個式子是不是恆真（tautology）？
（b）給定圖靈機 P 與輸入 w，P(w) 會停嗎？
（c）給定一階邏輯式子 φ，φ 普遍有效嗎？
（d）給定系統 F 與句子 φ，F ⊢ φ 嗎（φ 在 F 中可證嗎）？
（e）給定系統 F 與句子 φ，φ 是不是 F 的獨立句（F ⊬ φ 且 F ⊬ ¬φ）？

### 推演解答

**題目一。** 一組合理對應（你的措辭可不同，抓住精神即可）：

```text
  圖靈機      ≈ 帶無限磁帶的有限狀態機。
              精神：把「計算」剝到最原始——讀一格、寫一格、移動、換狀態。
              它對應你心裡「最底層的機器在幹嘛」那個圖像。

  λ-演算      ≈ 純函數（無副作用、一切皆函數、靠套用驅動）。
              精神：計算＝把函數套到引數上、化簡。你寫 Haskell / 用 map-reduce
              的那套「資料流過一連串純函數」的味道。

  一般遞迴函數 ≈ 遞迴定義的函式（從基本函數＋遞迴＋最小化堆起來）。
              精神：計算＝用更小的同類結果定義自己。你寫遞迴的直覺。
```

為什麼「三者等價」是定理而「三者＝可計算的全部」是論題：因為「三者等價」兩端都是**有精確定義的形式系統**，可以嚴格證明一個能模擬另一個（互相編碼）；而「可計算的全部」裡的「可計算」是個**非形式、靠直覺把握**的概念，沒有數學定義，你無法對一個沒被定義的東西做證明。**常見錯路**：說「論題沒被證明是因為太難」——不是難，是**對象不是形式物件**，原則上不可證，只能被經驗一再印證。

**題目二。** 他的方向**反了**。他說的「把程式等價歸約到停機」（能解停機 ⇒ 能解程式等價）證出來的是「程式等價**不會比停機更難**」——這推不出程式等價無解（停機無解，不代表所有不比它難的問題都無解，命題邏輯就比它容易還可判定）。

正確方向是**反過來**：把**已知無解的停機問題**，歸約**進**待證的程式等價問題——證「能解程式等價 ⇒ 能解停機」。具體說：拿任意停機實例 (P, w)，造兩段程式——程式甲忽略輸入永遠回傳 0；程式乙先模擬 P 跑 w，若 P(w) 停了才回傳 0（否則一直跑）。則「甲乙等價（對所有輸入同輸出）」⟺「P(w) 會停」。於是若有程式等價判定機，拿它判甲乙是否等價，就解了停機——但停機無解，矛盾。故程式等價不可判定。**口訣**：歸約要把**已知無解的**藏進**待證的**，方向是「無解者 ⇒ 待證者」，別搞反。

**題目三。**

- （a）**可判定**。命題邏輯沒有量詞，變數有限，真值表窮舉所有指派、有限行，「是」與「否」都在有限步停。✓
- （b）**半可判定**。會停的——跑它，停了就確認「是」；永不停的——你跟著跑到天荒地老，「否」那面沒有停止信號。（ch09。）
- （c）**半可判定**。一階有效性：有效的式子可證（完備性定理），列舉所有證明遲早撞到——「是」那面有限步答得了；無效的式子，你列舉證明永遠列不到它的證明，但也不敢斷言「真的無效」（沒有停止信號）。本章主角。
- （d）**半可判定**。和 (c) 同構：列舉 F 的所有證明，φ 真可證就遲早列到（「是」答得了）；φ 不可證則永遠等不到（「否」搆不著）。脊椎句 G 就落在「否」那邊。
- （e）**共半可判定（co-r.e.）**。「否」那一面（＝非獨立，即 F ⊢ φ 或 F ⊢ ¬φ）是兩個 r.e. 集的聯集，仍是 r.e.——列舉 F 的所有證明，只要撞到 φ 或 ¬φ 的某一個，就在有限步確認「非獨立」。因此獨立性問題本身是其補集，屬於 co-r.e.（Π₁），比半可判定深一格、但還不到「兩面都搆不著」。**常見錯路**：誤以為「兩個 co-r.e. 條件的合取更難」——但「(F ⊬ φ) ∧ (F ⊬ ¬φ)」的補集是「(F ⊢ φ) ∨ (F ⊢ ¬φ)」，r.e. 對聯集封閉，所以補集仍 r.e.，獨立性問題仍是 co-r.e.，不會跳到完全不可判定層。

## 自我檢核

口頭自答，講得出來才算過關，優先答得出「為什麼」：

1. 決定問題（Entscheidungsproblem）問的精確是什麼？它對應希爾伯特四柱的哪一根？為什麼說有了它「數學就自動化了」？
2. 決定問題是被誰、在哪一年、用什麼武器否決的？邱奇與圖靈的工具有什麼不同，這個「兩條獨立的路撞同一終點」為什麼本身就是 Church–Turing 論題的證據？
3. 為什麼 Church–Turing 是「論題」不是「定理」？哪一部分是定理、哪一部分是論題，分界在哪？（提示：找出斷言裡那個非形式的概念。）
4. 「半可判定」精確是什麼意思？拿你熟的「測試只能證明 bug 存在、不能證明 bug 不存在」講一遍這個單邊不對稱。
5. 「不可判定」為什麼**不**等於「無法決定／見仁見智／沒有客觀答案」？一個不可判定的問題，它的每個個案有沒有客觀答案？
6. 「沒有通用判定機」和「每個個案都解不了」差在哪？為什麼把前者讀成後者是悲觀的誤讀？
7. 希爾伯特四根柱子各死在哪一章、被誰砍的？為什麼說不完備（可證性的極限）和不可判定（可計算性的極限）是「同一招對角化的兩張臉」？
8. 脊椎句 G 落在「可判定 / 半可判定 / 完全不可判定」哪一層的哪一邊？這跟「列舉所有證明的機器永遠列不到 G」有什麼關係？

- **Wikipedia,「Hilbert's tenth problem」** https://en.wikipedia.org/wiki/Hilbert%27s_tenth_problem ——MRDP 定理（Matiyasevich 1970）把不可判定性釘進數論腹地；讀它如何說明「連丟番圖方程有沒有整數解都無通用判定法」。
- **《把系統寫成定理》TLA+（書架既有）** ——若你讀過，它的 model checker 在有限狀態空間裡「保證有限步給答案」正是停在「可判定」那一層；一旦狀態空間無界，你就掉進本章的半可判定地貌。可一句呼應、不依賴。
