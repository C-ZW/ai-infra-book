# ch03 — 真，不等於可證：兩條從沒接在一起的線

> **本章解決什麼問題**：這是本書最重要的一次概念清理。「真（true）」和「可證（provable）」在你腦裡可能是同一件事的兩個說法——一個命題若是真的，總該證得出來吧？這章把它們拆成兩條獨立的線：真是語意的（在標準模型 ℕ 裡成立，記作 ⊨），可證是語法的（系統內有一串合規則的有限推導，記作 ⊢）。Part I 的 ch01 立起「形式系統＝符號遊戲」、ch02 立起希爾伯特要這台機器算盡一切；本章說明這個夢的縫隙憑什麼存在——而且它**不是**邏輯規則不夠力造成的。脊椎句 G 還沒出場，但等它在 Part III 引爆時，引爆的正是這條縫：真理溢出證明。

## 從你已知的出發

你每天都在跟「真 ≠ 可證」這條縫打交道，只是換了名字。

你寫了一個函式，跑了一整套測試，全綠。CI 給你一盞綠燈。請問：這代表這個函式「正確」了嗎？

你心裡很清楚答案是「不」。測試通過只證明了一件事——**在你寫出來的那些輸入上**，函式行為符合你寫出來的那些斷言。它沒有、也不可能窮盡所有可能輸入。一個函式可以通過一萬個測試，然後在第一萬零一個你沒想到的邊界值上爆掉。Dijkstra 那句被引到爛的話講的就是這件事（這是轉述的常見版本，1969 年左右散見於他的講稿與筆記）：測試只能證明 bug 存在，不能證明 bug 不存在。

把這件事的結構抽出來，你會發現它有兩條線：

```text
線 A（你能機械驗證的）：測試套件跑完全綠
        ↑ 這是「可機械檢查」的——跑一遍就有答案，不需要理解程式在幹嘛
線 B（你真正想要的）：函式對所有合法輸入都行為正確
        ↑ 這是「事實上為真」——涵蓋你寫不完、跑不完的全部情形

線 A 成立  ⇏  線 B 成立     （綠燈不保證無 bug）
線 B 成立  ⇏  線 A 成立     （正確的函式也可能沒被任何測試覆蓋到）
```

兩條線**相關**——你寫測試就是希望 A 能逼近 B——但它們**不是同一條線**。你不會把「測試全綠」跟「程式正確」劃等號；那是初級工程師才會犯的錯。

型別檢查也一樣。型別檢查器接受你的程式（⊢ 的工程版：可機械驗證的「合法」），跟你的程式真的沒 bug（⊨ 的工程版：在所有執行路徑上行為對）是兩件事。型別系統再強，也有它判不出的真命題——一段 TypeScript 過了 `tsc`，照樣可以在 runtime `undefined is not a function`。

這一章要做的，就是把這個你在工程裡早已內化的直覺，搬進形式系統的世界，並且講清楚一件反直覺的事：**這條縫不是因為「規則不夠多」造成的**。你的第一反應大概是「那就多加幾條規則、多寫幾個測試、把型別系統做強一點，縫就補起來了」。本章後半會告訴你，希爾伯特當年也是這麼想的，而哥德爾在 1929 年（注意，是不完備定理的**前兩年**）親手證明了一件事，正好說明這條縫的源頭根本不在規則——在別的地方。

## 兩條線：⊢（可證，語法）與 ⊨（真，語意）

先把記號釘死，這兩個符號全書都會用，分清楚它們是讀懂後面每一章的前提。

```text
F ⊢ φ    讀作「F 證明 φ」
         ── 純語法。意思是：在系統 F 裡，存在一串有限長的符號串序列，
            從公理出發、每一步都合某條推理規則、最後一行正好是 φ。
         ── 這是 ch01 講的那台搬符號的機器產出的東西。
            「F ⊢ φ 成立嗎」原則上可機械檢查（給你一串宣稱的證明，逐步驗就行）。

ℕ ⊨ φ    讀作「φ 在 ℕ 中為真」（或寫 ℕ ⊨ φ）
         ── 純語意。意思是：把 φ 裡的符號照「標準意思」解釋
            （0 就是零、S 就是加一、+ 就是加法、× 就是乘法、∀ 跑遍全部自然數），
            這樣解釋出來的那句斷言，事實上在自然數上成立。
         ── 這裡沒有「推導」「步驟」「規則」這些字。它問的是事實，不是程序。
```

關鍵在於：⊢ 在問「系統內走得到嗎」，⊨ 在問「事實上對嗎」。一個是關於**證明這個東西**（一串符號），一個是關於**世界這個東西**（自然數實際長怎樣）。它們連討論的對象都不同類——前者是語法物件，後者是語意事實。

舉一個故意極端的例子幫你把兩條線徹底分開。想像一個爛系統 F₀，它的公理裡有一條是「1 = 2」（先不管它有多荒謬，它語法上是個合法系統）。那麼：

- F₀ ⊢「1 = 2」：**成立**。因為「1 = 2」是它的公理，一行就證完了。
- ℕ ⊨「1 = 2」：**不成立**。在真正的自然數裡，一不等於二，這是事實。

於是 F₀ ⊢「1 = 2」但 ℕ ⊭「1 = 2」。**可證但不真**。這證明了「可證」絕不蘊涵「真」——除非你額外要求系統是「健全的」（見下節）。反過來那一半——「真但不可證」——才是哥德爾的炸彈，但它得等 G 造出來才看得到，本章先只把承諾放在這裡。

> 這裡值得你停十分鐘，因為你的直覺正要騙你。你大概覺得「正常的系統當然只證真的東西啊，誰會去設計一個證得出 1=2 的系統」。對，正常系統確實如此——那叫**健全（sound）**。但「系統健全」是一個**額外的假設**，不是免費的。把它拆出來看清楚，是理解後面一切的關鍵。

## 健全（sound）：把兩條線「半綁」起來的那根繩子

「可證 ⇏ 真」可以靠一個性質擋掉，這個性質叫**健全（soundness）**：

```text
F 是健全的（sound）  ≡  凡 F 證得出來的，都在 ℕ 中為真。
                        用記號：F ⊢ φ  ⇒  ℕ ⊨ φ。
```

一個健全的系統，它的公理在 ℕ 中都真、它的推理規則都「保真」（真的東西推出來還是真的），於是整條證明鏈從頭真到尾，結論不會假。你日常預設用的算術系統（皮亞諾算術 PA）就是健全的——它證得出來的算術命題，在自然數裡都對。

注意這根繩子只綁住**一個方向**：

```text
        若 F 健全：
                可證  ────────►  真      （F ⊢ φ  ⇒  ℕ ⊨ φ）   ← 繩子綁住這個方向
                可證  ◄───╳────  真      （ℕ ⊨ φ  ⇒  F ⊢ φ ？）← 這個方向沒被綁住！
```

「可證 ⊆ 真」（健全保證的）讓你安心——系統不會騙你，它說證出來的就真的對。但「真 ⊆ 可證」是另一回事，那叫**完備（complete）**，是完全不同的性質，而它正是哥德爾要拆的那一根。一個健全的系統，完全可以漏掉某些真句子（證不出來）。那塊「真但證不出」的區域，就是不完備的家。

我特別要你記住「健全只綁一個方向」這件事，因為日常語言裡「真」和「可證」混用慣了，你會不自覺以為它們互相蘊涵。把它畫成兩個圈會清楚很多：

```text
       所有句子的宇宙
  ┌─────────────────────────────────────────┐
  │                                          │
  │     ┌───────────────────────────┐        │
  │     │  在 ℕ 中為真的句子（⊨）    │        │
  │     │                           │        │
  │     │    ┌─────────────────┐    │        │
  │     │    │ F 證得出的句子   │    │        │
  │     │    │   （⊢）          │    │        │
  │     │    │                 │    │        │
  │     │    └─────────────────┘    │        │
  │     │         ↑                 │        │
  │     │   若 F 健全：可證圈 ⊆ 真圈 │        │
  │     │                           │        │
  │     │  ←── 這圈內、可證圈外 ──→  │        │
  │     │      ＝「真但不可證」      │        │
  │     │      ＝不完備的家          │        │
  │     │      （ch07 的 G 住這裡）  │        │
  │     └───────────────────────────┘        │
  │                                          │
  │   （若 F 不健全，可證圈會凸出真圈外，     │
  │     凸出的那塊＝「可證但不真」的假定理）   │
  └─────────────────────────────────────────┘
```

健全把可證圈壓進真圈裡。不完備問的是：可證圈會不會**填滿**真圈？哥德爾的答案——只要 F 夠強又一致——是**不會**，永遠有真圈內、可證圈外的那一塊。本章不證它（ch07 才引爆），這裡只把舞台搭好：真圈跟可證圈是兩個不同的圈，它們**有沒有**重合是個需要證明的問題，不是定義上的必然。

## 縫隙從哪來？——一個你絕對猜錯的源頭：完備性定理（1929）

現在來到本章最反直覺、也最容易被科普寫錯的一頁。

你的工程直覺會說：「真句子證不出來，那一定是推理規則不夠力嘛——少了某條規則，所以有些真的東西推不到。再補規則不就好了？」

這個直覺**錯得很乾淨**，而且推翻它的人就是哥德爾自己，比不完備定理還早兩年。

1929 年，哥德爾的博士論文證明了**完備性定理（completeness theorem）**。注意這個名字——「完備性」，跟「不完備」只差一個字，是科普混淆的重災區（landscape T4），務必分清楚。完備性定理講的是**一階邏輯（first-order logic）**：

```text
哥德爾完備性定理（1929）：
   在一階邏輯裡，凡是「語意有效」的式子，都「語法可證」。
   ── 語意有效（valid）＝在這套公理的「所有」模型裡都成立。
   ── 語法可證＝從邏輯公理＋推理規則推得出來。

   一句話：   在所有模型都真  ⇒  證得出來。
   記號版：   T ⊨ φ  ⇒  T ⊢ φ      （⊨ 這裡指「T 的每個模型都讓 φ 真」）
```

把它跟你「規則不夠力」的直覺對齊：完備性定理正好說**一階邏輯的推理規則一點都不缺**。任何在「所有模型」裡都真的東西，這套規則都推得到，一個不漏。邏輯機器是滿格的。

那縫隙到底哪來的？既然規則不缺，「真但不可證」怎麼還會發生？

關鍵在「所有模型」這四個字。完備性定理保證的是「在**所有**模型都真 ⇒ 可證」。但「G 真」要的是「在**標準模型 ℕ** 這**一個**模型裡真」。這是兩個不同的「真」：

```text
        「在所有模型都真」      vs       「在 ℕ 這一個模型裡真」
              ↑                                  ↑
        完備性定理綁住的         範圍大很多      不完備講的
        （這個 ⇒ 可證）                         （這個 ⇏ 可證）

   有沒有可能：某句 G 在 ℕ 裡真，
   但在「另一個也滿足 F 公理的模型」裡假？
   ── 有。這種模型叫「非標準模型（non-standard model）」。
   ── 一旦存在這種模型，G 就「不是在所有模型都真」，
      於是完備性定理的前提不成立、它不保證 G 可證。
   ── G 在 ℕ 真、卻在非標準模型假  ⇒  G 不可證。縫隙就在這裡裂開。
```

讓我把這句話講到你能轉述給同事的程度。**縫隙的真正源頭，不是邏輯規則不夠，而是 F 的公理「抓不住」標準模型 ℕ。** F 的公理就像一份介面規格（spec），ℕ 是你心裡想的那個標準實作。問題是：同一份 spec 可以有**好幾個不同的實作**（模型），而 F 的公理這份 spec 漏掉了某些東西，導致存在一個「非標準實作」，它把所有公理都滿足了，卻在某個你以為理所當然的性質上跟 ℕ 不一樣。F 證得出的東西必須在**它的所有模型**（包括那個古怪的非標準實作）裡都成立；而 G 只在 ℕ 裡成立、在非標準模型裡不成立——所以 F 證不到 G。不是 F 的推理太弱，是 F 的公理框得太鬆，框進了它沒打算框的模型。

（非標準模型本身是個迷人的東西——裡面有「比所有 0,1,2,3,… 都大」的「無窮大自然數」，卻照樣滿足歸納法等全部一階公理。它牽涉到可數/不可數與無窮的細節，本書 ch12 會回來給直覺，更深的無窮探討見《馴服無限》談可數與不可數那部分。這裡你只需要抓住一件事：spec 沒夾死實作，就會有意外的實作。）

這是我認為整個 Part I 最值得停下來的一頁。希爾伯特的夢之所以會死，不是因為「我們的邏輯太笨」，而是因為**任何一份你寫得出來（可遞迴公理化）的算術 spec，都框不死自然數這個你心裡的標準實作**。後面整本書都在把這句話的機制一層層攤開。

## 塔斯基的不對稱：「可證」說得出口，「真」說不出口

縫隙還有第二個、更深的源頭，跟上面那個是同一件事的另一面。它解釋了**為什麼**哥德爾能造得出 G，而說謊者悖論（ch04）卻造不出害死系統的東西。

1933 年，塔斯基（Alfred Tarski）證明了**不可定義性定理（undefinability theorem）**：

```text
塔斯基不可定義性定理（1933；哥德爾 1930 證不完備時已私下知道，
寫在 1931 給馮諾伊曼的信裡）：

   「算術真理」這個概念，無法在算術內部定義。
   ── 沒有任何一個算術公式 True(x) 能正確表達
      「編號 x 的那個句子在 ℕ 中為真」。
```

對照哥德爾的發現，會看到一個關鍵的**不對稱**：

```text
        概念                能不能在算術系統內部表達？
   ────────────────────────────────────────────────────
   「x 是可證的句子」        ✅ 可以。可證性謂詞 Prov_F(x) 寫得出來
   （Prov_F，語法性質）         ——「存在一串編號為某數的證明，結尾是 x」
                              這是 ch05/ch06 要建的東西，純語法、可機械列舉。

   「x 是真的句子」          ❌ 不行。塔斯基說 True(x) 不可定義。
   （True，語意性質）           ——「真」是系統碰不到的、在系統外面的概念。
```

為什麼這個不對稱要命？因為如果「真」也能在系統內部寫成一個謂詞 True(x)，你就能在算術裡造一個句子說「我不為真」（用 ch06 的對角化技巧把「我」指回自己），這句話會像說謊者悖論一樣**自爆**——真則假、假則真，系統直接矛盾。塔斯基定理保證了這個災難不會發生：「真」根本進不來，所以你造不出「我不真」這種自爆句。

但「可證」進得來。於是哥德爾改用「可證」——造一句「我不**可證**」（這就是脊椎句 G）。因為「不可證」是系統內表達得出的語法謂詞，這句話造得出來、是合法的算術句子；而且它不自爆，只是優雅地溢出證明（ch04 會把「為什麼換成不可證就不自爆」走一遍，ch06 把句子造出來，ch07 引爆）。

```text
脊椎句 G ：「G 在本系統 F 內無法被證明。」
（形式版會在 ch06 寫成 F ⊢ G ↔ ¬Prov_F(⌜G⌝)，現在只記住白話版）
```

換句話說：**「真」和「可證」的差別，不只是哲學上的兩條線，它還是一個系統「碰得到 vs 碰不到」的硬差別。** 可證是系統摸得到自己的手；真是系統摸不到的、它賴以生存卻無法言說的東西。這個不對稱，就是真理能溢出證明的物理基礎。整本書接下來都是在這條不對稱上工作。

## Worked example：三個命題，把「真」和「可證」並排算一遍

下面用三個具體命題，把「在 ℕ 中真不真」和「在某弱系統 F 內證不證得出」並排攤開，讓你親眼看到兩欄可以不一致。挑的系統 F 故意取一個**很弱**的算術系統（想像成只有 Robinson 算術 Q 那種等級：會算具體的加減乘，但**沒有歸納法**），這樣「證不出來」的例子才舉得乾淨。

```text
命題                      在 ℕ 中真嗎（⊨）      在弱系統 F 中可證嗎（⊢）
─────────────────────────────────────────────────────────────────────
(1) 2 + 3 = 5            真                    可證
                         （自然數的事實）       （F 會算具體加法，逐步算得出）

(2) ¬(2 + 3 = 6)         真                    可證
    即 2 + 3 ≠ 6         （事實）               （F 算出 2+3=5，再導 5≠6）

(3) ∀x (0 + x = x)       真                    不可證！
    「0 加任何數還是自己」 （對每個具體 x 都成立） （弱 F 沒有歸納法，無法
                                                一口氣涵蓋「所有 x」）
─────────────────────────────────────────────────────────────────────
```

逐欄複核命題 (3)，這是整張表的重點：

```text
「在 ℕ 中真嗎」這一欄：
   ∀x (0+x=x) 在自然數裡當然真——0+0=0、0+1=1、0+2=2、…，
   每一個具體的 x 代進去都對，無一例外。所以 ℕ ⊨ ∀x(0+x=x)。✅

「在弱系統 F 中可證嗎」這一欄：
   F 沒有歸納法。它能逐一驗每個「具體數字」：
        0+0=0 可證、0+1=1 可證、0+2=2 可證、…
   但「逐一可證」≠「∀ 版可證」。要從「每個具體 x 都對」
   跳到「∀x 都對」這一個總句，需要歸納法——而弱 F 沒有。
   於是這個 ∀ 句 F 證不出來。F ⊬ ∀x(0+x=x)。❌
   （這不是 F「太笨」，是這個弱系統的表達/推理力就到這。
     換成有歸納法的 PA，(3) 就可證了——但 PA 又會有它自己證不出的真句 G，
     那是 ch07 的事。沒有系統逃得掉，只是縫隙挪了位置。）

自我複核：(3) 這一格出現了「真但不可證」——
   ℕ ⊨ ∀x(0+x=x) 但 F ⊬ ∀x(0+x=x)。兩欄不一致。✅ 正是本章要的演示。
```

把命題 (3) 看清楚，你就摸到不完備的雛形了：一個對「每個具體個案」都成立的事實，系統卻證不出那個「一網打盡」的總句。哥德爾 1931 的 G 是這種現象的**招牌升級版**——連加上歸納法的強系統 PA 都還是逃不掉，而且那個證不出的真句不再是 (3) 這種無辜的算術常識，而是一句**精心構造、指涉自己**的話。命題 (3) 是熱身；ch07 的 G 是正餐。但兩者骨架一樣：**真的那一欄，總有可證的那一欄追不上的東西。**

## 直覺的陷阱

哥德爾是科學史上被誤引最嚴重的結果之一，而幾乎所有誤引都從本章這個「真 vs 可證」沒分清楚開始。先在這裡埋下解毒劑。

| 陷阱（常見錯誤直覺） | 錯在哪一步 | 正確版 |
|---|---|---|
| **「真就是可證、可證就是真，本來就同一件事」** | 把語意（⊨，事實上對）和語法（⊢，系統內推得到）當同義詞。日常語言混用慣了，這是最底層的錯。 | 兩條獨立的線。健全只保證「可證 ⇒ 真」這**單向**；反向（真 ⇒ 可證）叫完備，是另一回事、且哥德爾證了它對夠強系統不成立。 |
| **「真句子證不出，一定是規則不夠，補規則就好」** | 把縫隙怪到推理規則頭上。 | 完備性定理（1929）證明一階邏輯規則**不缺**：所有模型都真的都可證。縫隙來自**公理抓不住 ℕ**（有非標準模型），不是規則弱。補規則補不到點上。 |
| **「完備性定理和不完備定理互相矛盾，哥德爾自打嘴巴」** | 把兩個「完備」當同一個。（landscape T4） | 不矛盾。**完備性（1929）**＝一階邏輯「語意有效⇒可證」，講邏輯規則夠用；**不完備（1931）**＝夠強算術理論「有真而不可證」，講公理框不住標準模型。同一人、相鄰兩年、不同層次。 |
| **「哥德爾證明了有些數學命題對錯無法知道」** | 把「F 證不出」聽成「沒人知道對錯」。（landscape T8，ch12 命脈） | G 在 ℕ 中有**確定真值（真）**，我們在 F 之外明明白白知道它真。「不可證」≠「真值不明」。本章先點，ch12 釘死。 |
| **「哥德爾證明真理不存在/一切相對」** | 把「系統內可證性有極限」滑成「真理沒有了」。（landscape T1） | 恰恰相反——定理講的是某系統內的可證性，G 反而是個有確定真值的真句。「真」不但存在，還**溢出**了「可證」。把不完備讀成相對主義是把技術詞當日常詞偷渡（ch14 大解剖）。 |

自我察覺的方法：每次你想說「哥德爾證明了……真理/數學/系統怎樣怎樣」，先停下來問自己一句——**「我這句話裡的『真』和『可證』，分得清是哪一個嗎？」** 如果分不清、或者你發現自己把兩個詞當成可互換，那你八成正要滑進上面某一格。整本書最便宜的防身術，就是隨時把這兩個詞拆開來看。

## 紙上推演

### 推演題 1 — 用「測試 vs 正確」向同事解釋 ⊢ 和 ⊨ ★ **[10 分鐘]**

你的同事說：「我這個服務的測試 100% 全綠，所以它是對的。」請用最多五句話，借「測試 vs 正確」這組工程概念，向他解釋 ⊢（可證）和 ⊨（真）的差別，並指出他的推論漏在哪。不要用任何邏輯符號。

### 推演解答

一個合格的回答應該包含這幾個要點：

> 「測試全綠」對應 ⊢——它是你**能機械驗證**的東西：跑一遍就知道，不需要理解程式在幹嘛。「服務真的對」對應 ⊨——它是你**真正想要**的：對所有合法輸入都行為正確，涵蓋你寫不完、跑不完的全部情形。這兩件事相關但不相等。測試全綠頂多保證「你寫出來的那些案例上對」，它逼近正確性，但永遠有沒覆蓋到的輸入。所以「測試全綠 ⇒ 服務正確」這個推論的漏洞，就是把「可機械驗證的（⊢）」直接當成「事實上為真的（⊨）」——這正是哥德爾全書要拆的那條縫。

常見錯路：(a) 把焦點放在「測試覆蓋率不夠高」上——這方向沒錯但不夠深，就算覆蓋率 100%（每行都跑到），也不等於每個**輸入**都試過，⊢ 和 ⊨ 的差別跟覆蓋率多高無關，是「可驗證的有限東西」對「為真的無限東西」的結構差別。(b) 把它講成「測試會有 bug」——不是測試本身有 bug，是測試這個**方法**在原理上只能證偽、不能證真。

### 推演題 2 — 完備性 vs 不完備，哪個講語意、哪個講語法，為什麼不矛盾 ★★ **[20 分鐘]**

哥德爾 1929 證了「完備性定理」、1931 證了「不完備定理」。一個外行聽到會說：「他先證完備、又證不完備，這不是自相矛盾嗎？」請回答：(a) 這兩個定理各自講的是「語意完備」還是「語法的什麼」？(b) 它們各自的對象（適用於什麼）是什麼？(c) 用一句話說明為什麼不矛盾。

### 推演解答

```text
(a) 各講什麼：
    完備性定理（1929）：講「語意有效 ⇒ 語法可證」。
        ── 它是一座橋：把「在所有模型都真（⊨，語意）」
           接到「推得出來（⊢，語法）」。
        ── 它說的是一階邏輯的「推理規則夠不夠用」——結論：夠。
    不完備定理（1931）：講「有真而不可證」。
        ── 它是一道縫：在「ℕ 中真（⊨ 單一模型）」和「F 中可證（⊢）」之間。
        ── 它說的是某個算術理論的「公理抓不抓得住標準模型」——結論：抓不住。

(b) 各自的對象：
    完備性 → 一階邏輯本身（任何一階理論的邏輯推理層面）。
    不完備 → 特定的、夠強表達算術的形式理論（如 PA），
            且要滿足三條件（一致、可遞迴公理化、夠強含 Q）。
    ── 注意對象不同層次：一個是「邏輯規則」，一個是「算術理論」。

(c) 為什麼不矛盾（一句話）：
    完備性說的「可證」是「在所有模型都真的東西可證」；
    不完備說的「不可證」是「只在標準模型 ℕ 真、在非標準模型假的東西不可證」——
    後者的前提（所有模型都真）根本不成立，所以前者管不到它，兩者各說各的、不衝突。
```

常見錯路：(a) 把「完備性定理」的「完備」跟「系統完備（每個句子可證或其否定可證）」搞混——它們是**兩個不同的「完備」**。完備性定理講的是「有效⇒可證」（邏輯規則完整），不是「每個算術句都被決定」。一階邏輯**有**完備性定理（規則完整），但 PA **不**完備（有 G 兩邊都證不出）——同一個詞、相反的結論，不矛盾，因為「完備」指的根本是兩件事。(b) 以為「相鄰兩年的兩個相反結果」一定有一個錯——兩個都對，因為它們的對象（一階邏輯 vs 算術理論）和「真」的範圍（所有模型 vs 單一標準模型）都不同。

### 推演題 3 — 健全只綁一個方向，舉出反向會出事的例子 ★★ **[15 分鐘]**

本章說健全（sound）只保證「可證 ⇒ 真」這單向，反向（真 ⇒ 可證）不保證。請：(a) 用本章 Worked example 裡的命題，舉出一個「真但（在弱 F 中）不可證」的具體例子，說明它正是反向失效的見證；(b) 解釋為什麼「健全」這個性質擋不住這種例子。

### 推演解答

```text
(a) 例子：命題 (3) ∀x (0+x=x)。
    ── 它在 ℕ 中真（每個具體 x 代入都成立）。
    ── 它在弱系統 F（無歸納法）中不可證（無法從「逐個具體成立」
       跳到「∀ 總句」）。
    ── 所以這是一個「ℕ ⊨ φ 但 F ⊬ φ」——真但不可證。
       它正是「真 ⇒ 可證」（完備性方向）失效的見證：φ 真了，
       可證卻沒跟上。

(b) 為什麼健全擋不住：
    健全只說「F ⊢ φ ⇒ ℕ ⊨ φ」（凡證出的都真）。
    它對「ℕ ⊨ φ 但 F ⊬ φ」這種情形完全沒有發言權——
    健全管的是「證出來的會不會是假的」（不會），
    它根本不管「真的東西有沒有被證出來」（可能沒有）。
    一個健全的系統可以乖乖只證真句，同時漏掉一大堆真句。
    把健全（可證⊆真）誤當成完備（真⊆可證），就是這題的陷阱。
```

常見錯路：把命題 (3) 在弱 F 證不出，誤解成「(3) 在 ℕ 裡也有問題/不一定真」。不對——(3) 在 ℕ 中是鐵打的真，問題純粹出在弱 F 的證明力跟不上，不在 (3) 的真值。真值（語意）和可證（語法）是兩條線，(3) 正是用來示範它們可以分家。

## 自我檢核

口頭自答，講得出來才算過關：

1. ⊢ 和 ⊨ 各問的是什麼問題？（提示：一個問「系統內走得到嗎」、一個問「事實上對嗎」；對象一個是語法物件、一個是語意事實。）為什麼說它們「連討論的對象都不同類」？
2. 「健全（sound）」綁住了哪個方向？沒綁住哪個方向？為什麼說一個健全的系統「可以乖乖只證真句、卻同時漏掉一堆真句」？
3. 為什麼「真句子證不出來」**不是**「推理規則不夠力」造成的？完備性定理（1929）在這裡扮演什麼角色——它證明了規則的什麼？
4. 「縫隙來自公理抓不住標準模型 ℕ」這句話是什麼意思？非標準模型在這裡為什麼是關鍵？（提示：spec vs 多重實作。）
5. 完備性定理（1929）和不完備定理（1931）為什麼不矛盾？它們各自講語意還是語法、對象各是什麼？（這題講不清楚就回去重讀「縫隙從哪來」那節。）
6. 塔斯基的不對稱是什麼——哪個概念系統內表達得出、哪個表達不出？為什麼哥德爾選「可證」而不是「真」來造 G？（提示：如果「真」表達得出會發生什麼災難？）
7. 「哥德爾證明有些命題對錯無法知道」這句話錯在哪？G 的真值到底明不明？（這題是 ch12 的命脈，先在這裡建立直覺。）

- **Stanford Encyclopedia of Philosophy, "Gödel's Incompleteness Theorems"**（plato.stanford.edu/entries/goedel-incompleteness/）：權威且免費。本章相關的請讀它對「真 vs 可證」「標準模型」「完備性定理 vs 不完備定理」的釐清段落——它把這幾個最常被混淆的概念分得很乾淨。
- **Stanford Encyclopedia of Philosophy, "Gödel's completeness theorem" / Wikipedia 同名條目**（en.wikipedia.org/wiki/Gödel's_completeness_theorem）：讀「完備性定理講語意有效⇒可證、與不完備定理不矛盾」那段，正好解掉本章「兩個完備」的混淆（landscape T4）。
- **Wikipedia, "Tarski's undefinability theorem"**（en.wikipedia.org/wiki/Tarski's_undefinability_theorem）：讀「真不可定義、可證可表達」這個不對稱——本章把它當引子，這條不對稱是後面 G 為什麼造得出來的根。
- **Torkel Franzén,《Gödel's Theorem: An Incomplete Guide to Its Use and Abuse》（2005）**：第 1–2 章把「真 vs 可證」「不完備不是不一致」講得最清楚、也最克制。本章的「直覺的陷阱」清單，正餐版在這本書（ch14 會大量回到它）。
- **《如何不騙自己》談科學的真與已驗證那部分**（姊妹書，章序未定案）：把本章「形式系統的真 vs 可證」對照到「科學的真 vs 已驗證」——數學證明和科學證據是兩種不同的「確定」，這個對照能讓你更立體地理解「為真」和「能擔保為真」的差距。
- **《馴服無限》談可數與不可數那部分**（書架既有）：本章一筆帶過的非標準模型、無窮大自然數，背後的可數/不可數機制在那本書裡。ch12 會回來，想先打底可以翻它。
