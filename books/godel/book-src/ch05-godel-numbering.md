# ch05 — 把句子變成數字：哥德爾編碼

> **本章解決什麼問題**：上一章哥德爾把「假」換成「不可證」，造出脊椎句 G 的雛形——「G 在本系統 F 內無法被證明」。但這裡藏著一個還沒兌現的承諾：憑什麼一個只會做算術的形式系統 F，能談論「自己的某個句子可不可證」？句子是符號串、可證性是「存不存在一串合規則的推導」，這些都是**語法**的東西，F 的詞彙裡只有 0、+、×、=、∀ 這些算術符號，根本沒有「句子」「證明」這種詞。本章就是補上這座橋：哥德爾編碼，把每個符號、每條公式、每串證明都唯一地變成一個自然數。一旦「句子」變成「數字」，「這個句子可不可證」就變成「這個數字有沒有某個算術性質」——而算術性質，正是 F 會談的東西。這一章不引爆任何定理，它只是把舞台的地板鋪好；但沒有這塊地板，ch06 的對角化和 ch07 的引爆都站不起來。本章也是全書的招牌跨書連結：編碼能唯一解碼，靠的是《乘法的原子》裡的算術基本定理。

## 從你已知的出發

你每天都在做哥德爾編碼，只是沒這麼叫它。

你寫一個 API，前端傳來一個物件，你 `JSON.stringify` 把它壓成一個字串、塞進佇列；另一端 `JSON.parse` 還原。你存一個檔案，作業系統把這串位元組編號、寫進磁區，之後靠 inode 還原。你做一次 git commit，整棵檔案樹被 SHA 雜湊成一個 40 字元的 ID。這些全是同一件事：**把一個結構壓成一個可還原的識別碼**。

哥德爾在 1931 年做的，就是這件事的數學版——只是他的識別碼不是字串、不是位元組陣列，而是一個**自然數**。為什麼非得是自然數？因為他要編碼的對象（邏輯句子、數學證明）所活在的那個形式系統 F，唯一會處理的東西就是自然數。F 不認得字串，但它認得 1729 這個數、認得「1729 能不能被 7 整除」這種問題。所以如果你想讓 F「談論」一個句子，你得先把那個句子變成一個 F 看得懂的數。

這裡有一個你熟到不能再熟的直覺，是整章的引擎：**把程式碼當資料（code as data）**。

```text
你的世界                          哥德爾的世界
─────────────────────            ─────────────────────
原始碼字串 "function f(){...}"     邏輯句子  "∀x (x = x)"
   │ 它是「程式」也是「資料」       │ 它是「句子」也是「一串符號」
   ▼                                ▼
AST（抽象語法樹，可被另一段          編成一個自然數（可被算術
程式遍歷、檢查、改寫）              謂詞檢查：「它是合法公式嗎」）
   │                                │
   ▼                                ▼
linter 問：「這段碼合法嗎」          F 問：「這個數是合法公式的編碼嗎」
＝對 AST 跑一個謂詞                  ＝對這個數跑一個算術謂詞
```

你早就接受了「一段程式可以被另一段程式當資料來檢查」——這是 linter、編譯器、formatter 的全部日常。哥德爾要你再走一步：**一個形式系統，也可以把自己的句子當資料來檢查**——只要先把句子序列化成數字。序列化的工具就是本章主角，哥德爾編碼。而它能無碰撞、可完美還原，靠的是一個你在數論裡見過的老定理。

我認為這一章是全書最容易被低估的一章。它沒有戲劇性的引爆，讀起來像「技術細節」。但所有戲劇性的引爆都因為它才可能發生：**它是把「系統談論自己」從一句哲學口號變成一個算術事實的那一步**。

## 為什麼非編碼不可：F 的詞彙裡沒有「句子」

先把問題講到位，不然編碼會像一個無動機的技巧。

回到 ch01 的設定：一個形式系統 F（你可以想成皮亞諾算術 PA）是一台搬符號的機器。它的詞彙是 `0`、後繼函數 `s`、`+`、`×`、`=`、`<`、邏輯連接詞 `¬ ∧ ∨ →`、量詞 `∀ ∃`、變數 `x y z…`、括號。它的公理講的是這些符號之間的算術關係（`∀x (x + 0 = x)` 之類）。它的推理規則告訴你怎麼從已有的句子搬出新句子。

現在我們想讓 F 表達脊椎句 G：「G 在 F 內不可證」。把它攤開，G 想說的是：

```text
G 想說的事 ：「不存在一串符號，它是『G 這個句子』在 F 裡的合法證明。」
```

問題立刻浮現。這句話裡的「一串符號」「是……的合法證明」「G 這個句子」——**沒有一個是算術概念**。F 的詞彙裡找不到「符號串」這個名詞，找不到「合法證明」這個謂詞。你沒辦法在 PA 裡寫出 `∀(符號串 p) ¬(p 是 G 的證明)`，因為 PA 不會講「符號串」。

這就是哥德爾要跨的鴻溝。鴻溝的兩岸是：

```text
語法岸（meta，關於 F 的）          算術岸（object，F 內部的）
──────────────────────            ──────────────────────
符號                              ？
公式（符號的合法排列）             ？
證明（公式的合法序列）             ？
「p 是 φ 的證明」這個關係           ？
「φ 可證」這個性質                  ？
```

哥德爾的橋，是在右邊每一格填上一個自然數或一個算術謂詞，讓左岸的每個語法概念在右岸都有一個算術替身。填法只有一個核心動作：**給每個符號一個號碼，再把號碼序列壓成一個數**。壓的方法，下一節登場。

一旦橋搭好，右岸長這樣：

```text
語法岸                            算術岸（編碼後）
──────────────────────            ──────────────────────
符號                          →   一個小自然數（符號碼）
公式                          →   一個自然數（符號碼序列的編碼）
證明                          →   一個自然數（公式編碼序列的再編碼）
「p 是 φ 的證明」              →   一個算術謂詞 Prf(p, f)（兩個數的關係）
「φ 可證」                     →   一個算術謂詞 Prov_F(f) ≡ ∃p Prf(p, f)
```

最後一行是這整章的目的地。**「φ 可證」變成了「存在一個數 p，使得 p 和 f 滿足某個算術關係」**——一個 F 內部寫得出來的句子。橋搭好之後，「F 談論自己」不再是修辭，是算術。

## 編碼的三層：符號 → 公式 → 證明

哥德爾編碼是分層的。每一層都用同一個壓縮動作，套在前一層的產物上。

### 第一層：給每個符號一個碼

挑一張表，把 F 用到的每個符號配一個固定的正整數。表怎麼配無所謂，只要**一對一、固定**。本章用一張迷你玩具表（真正的 PA 符號更多，但機制一模一樣）：

```text
符號    符號碼
─────   ─────
0        1
s        2      （後繼，s0 = 1, ss0 = 2 …）
=        3
(        4
)        5
+        6
∀        7
x        8
¬        9
```

於是「符號」這個語法物件，在算術岸的替身就是它的符號碼。`0` 是 1，`=` 是 3。注意這一步還很弱：它只把**單個符號**變成數。一個句子是一串符號，我們需要把**一串**號碼也壓成一個數。

### 第二層：把符號序列壓成一個數（核心動作）

這是哥德爾的關鍵手法，也是本書要你記牢的編碼約定。給定一串符號碼 (a₀, a₁, …, aₙ)，把它編成：

```text
                a₀+1     a₁+1     a₂+1            aₙ+1
   ⌜序列⌝  =  2      · 3      · 5      · … · pₙ

   其中 p₀=2, p₁=3, p₂=5, p₃=7, … 是由小到大的質數，
   第 k 個位置用第 k 個質數，指數放「第 k 個符號碼 + 1」。
```

兩個要點，各有理由：

- **用質數的冪次積**：每個位置綁一個不同的質數，指數記那個位置的符號碼。為什麼這樣就能還原？因為質因數分解是唯一的——這正是算術基本定理（這裡借的是《乘法的原子》談算術基本定理唯一分解那章）。下一節專講這條命脈。
- **指數一律「＋1」**：因為符號碼可能是 0（雖然我們的玩具表從 1 起，但一般情況符號碼可以從 0 編），而 `p⁰ = 1` 會讓那個質數「消失」在乘積裡，解碼時無從分辨「這個位置的碼是 0」還是「這個位置根本不存在」。把指數一律加 1，保證每個用到的位置的指數都 ≥ 1，那個質數一定現身。本書全書統一這個「＋1」約定，這裡講明白為什麼。

一個句子（合法的符號排列）就是一串符號，套這個動作，得到它的哥德爾數 ⌜φ⌝。

### 第三層：把證明（句子序列）再壓一次

一個證明在 ch01 的意義下，是一串公式 (φ₁, φ₂, …, φₘ)：從公理出發，每一步合規則，最後一行是要證的句子。每個 φᵢ 已經在第二層被編成一個數 ⌜φᵢ⌝。於是「一個證明」就是「一串數」(⌜φ₁⌝, …, ⌜φₘ⌝)——再套**同一個壓縮動作**，把這串數編成一個更大的數。

```text
   ⌜證明⌝  =  2^(⌜φ₁⌝+1) · 3^(⌜φ₂⌝+1) · 5^(⌜φ₃⌝+1) · … · p^(⌜φₘ⌝+1)
```

這個數會大到天文等級（質數的指數本身就是一串句子的編碼，動輒幾十位數）。但**大小無所謂**——它仍然是一個確定的、唯一的自然數。哥德爾把一整份數學證明，序列化成了一個（極其龐大的）整數。這跟你把整棵 git 工作樹雜湊成一個 commit 物件是同一種精神，只是哥德爾要的是**可逆**（能完整還原），而 SHA 是刻意不可逆的單向雜湊。可逆這件事，又回到算術基本定理。

三層下來，核心收穫只有一句：

```text
   語法物件（符號／公式／證明）  ←——可逆——→  自然數
                                  （靠 FTA 唯一分解）
```

語法世界整個被搬進了算術世界。而 F 會算術。

## 解碼為什麼唯一：算術基本定理是引擎

編碼有用，前提是它**可還原**：給我一個哥德爾數，我得能毫無歧義地讀回原本那串符號。否則「F 談論句子 φ」就會變成「F 談論一坨可能對應好幾個句子的數」，整個計畫垮掉。

可還原性完全押在一件事上：

```text
算術基本定理（FTA）：每個大於 1 的自然數，都可以唯一地
寫成質數的乘積（不計順序）。1729 = 7 · 13 · 19，沒有別種寫法。
（見《乘法的原子》談算術基本定理唯一分解那章。）
```

把它套到我們的編碼上：一個哥德爾數 N，它的質因數分解是唯一的，所以「2 的指數是多少、3 的指數是多少、5 的指數是多少……」這些**答案是被 N 鎖死的**，沒有第二種讀法。指數一律減回 1，就拿回每個位置的符號碼；查表，拿回符號。整串符號被一字不差地還原。

我認為這是全章最該停下來品味的一頁：**哥德爾的編碼之所以無碰撞、可完美還原，不是靠任何巧妙的工程設計，而是直接借用了一條兩千年的數論事實**。質因數分解唯一，本來只是個關於整數的漂亮性質；哥德爾把它變成「系統能安全談論自己句子」的承重牆。數論那本書在講 FTA 時就埋了伏筆指向這裡——這是兩本書之間最名正言順的一次握手。

反過來想最有感：**如果質因數分解不唯一會怎樣？** 假設存在某個數 N，既能分解成 2²·3⁵、又能分解成 2³·3³（這在真實算術裡不可能，但我們做思想實驗）。那麼解碼 N 時，你會讀到兩串不同的符號碼 (1,4) 和 (2,2)，對應兩個不同的句子。N 同時是兩句話的編碼。於是「N 可證嗎」這個問題沒有確定的指涉對象——可證性謂詞 Prov_F(N) 要對哪個句子負責？編碼退化成一個會碰撞的雜湊，整座橋斷掉。FTA 保證這永遠不會發生：**一個數，至多編碼一個序列**。

（小心一個常被忽略的角落：不是每個自然數都是某個合法句子的編碼。`6 = 2·3` 對應序列 (0,0)，可能根本不是合法公式；像 100 = 2²·5² 缺了質數 3 這格，連「一個沒有缺口的序列」都不是。所以哥德爾還得另外造一個算術謂詞「x 是合法公式的編碼」來篩。重點是：**這個篩子本身也是算術的**——「x 的質因數分解有沒有缺口、每個指數減 1 後查表是不是合法符號、整串排列符不符合公式文法」全都是對 x 做的、可機械檢查的算術判斷。語法合法性，變成了算術合法性。）

## Worked example：手算一次編碼，再手算一次解碼

光說不算，這一節我們把一個極短的句子親手編一遍、再把一個給定的數親手解一遍。全程手算、逐步複核，不跳步。沿用上面的玩具符號表。

### 編碼：把「0 = 0」變成一個數

句子是三個符號：`0`、`=`、`0`。查表拿符號碼：

```text
符號     0    =    0
符號碼   1    3    1        ← 這是序列 (a₀,a₁,a₂) = (1, 3, 1)
位置質數 2    3    5        ← 第 0,1,2 個質數
指數     a+1: 2,  4,  2     ← 一律「＋1」
```

套公式，逐項算，每算一步停下來複核：

```text
   ⌜0 = 0⌝ = 2^(1+1) · 3^(3+1) · 5^(1+1)
           = 2^2     · 3^4     · 5^2

   第 1 項：2^2 = 4              ← 複核：2·2 = 4   ✓
   第 2 項：3^4 = 81             ← 複核：3·3=9, 9·9=81   ✓
   第 3 項：5^2 = 25             ← 複核：5·5 = 25   ✓

   乘起來： 4 · 81 = 324         ← 複核：4·80=320, +4 = 324   ✓
           324 · 25 = 8100      ← 複核：324·25 = 324·100/4 = 32400/4 = 8100   ✓

   ⌜0 = 0⌝ = 8100
```

所以句子「0 = 0」的哥德爾數是 8100。它就是一個普通的自然數——F 可以對它做加減乘除、問它能不能被 7 整除、問它是不是某個更大證明數的因子。**「0 = 0」這個句子，現在以 8100 的身份活在 F 的算術世界裡。**

（順帶確認「＋1」的必要：如果不加 1，符號碼 1,3,1 的指數就是 1,3,1，沒問題；但若某個符號碼是 0，比如句子裡有個碼為 0 的符號，它的指數就是 0、那個質數消失，解碼時會把「碼 0」誤讀成「沒有這個位置」。加 1 一勞永逸地堵住這個洞。）

### 解碼：把 18000 還原成符號串

現在反過來。給你一個數 N = 18000，宣稱它是某個句子的編碼。還原它，靠的就是 FTA——把它質因數分解，讀指數。

```text
   18000 拆質因數（一路除）：
   18000 / 2 = 9000          ← 除掉一個 2
    9000 / 2 = 4500          ← 第二個 2
    4500 / 2 = 2250          ← 第三個 2
    2250 / 2 = 1125          ← 第四個 2；1125 是奇數，2 除盡
                              → 2 的指數 = 4
    1125 / 3 = 375           ← 第一個 3
     375 / 3 = 125           ← 第二個 3；125 不是 3 的倍數
                              → 3 的指數 = 2
     125 / 5 = 25            ← 第一個 5
      25 / 5 = 5             ← 第二個 5
       5 / 5 = 1             ← 第三個 5；到 1，結束
                              → 5 的指數 = 3

   複核乘回去：2^4 · 3^2 · 5^3 = 16 · 9 · 125
            = 144 · 125 = 18000   ✓   （144·125 = 144·1000/8 = 18000）

   18000 = 2^4 · 3^2 · 5^3
```

質數用到 2、3、5，連續沒缺口，good——這是一個合法序列的編碼。讀指數、各減 1，拿回符號碼：

```text
   位置質數   2    3    5
   指數       4    2    3
   減 1       3    1    2     ← 符號碼序列 (3, 1, 2)

   查玩具表：碼 3 = '='，碼 1 = '0'，碼 2 = 's'

   還原符號串： =  0  s        即「= 0 s」
```

FTA 保證這是 18000 **唯一**的讀法：質因數分解只有這一種，指數被鎖死，符號串被鎖死。換一個人來解、用任何方法解，都只會得到「= 0 s」這一個答案。這就是「可完美還原」的全部底氣——不是工程上的巧妙，是數論上的必然。

（這個還原出的「= 0 s」在我們的玩具文法裡未必是合法公式——沒關係。解碼動作只負責「把數變回符號串」；「這串符號合不合語法」是另一個算術謂詞的工作，見上一節談的篩子。）

## 從編碼到可證性謂詞：Prov_F 的雛形

編碼三層搭完，可以收割本章真正想要的東西了。

回想語法岸最頂上那兩格：「p 是 φ 的證明」和「φ 可證」。編碼之後：

```text
   「p 是 φ 的證明」
      ＝ 「p（一串公式的編碼）的最後一行正好是 φ，
          且每一行要嘛是公理、要嘛由前面幾行依規則推出」
      ＝ 一個只對兩個數 p、f 做質因數分解、查指數、比對的算術判斷
      ＝ 一個算術謂詞  Prf(p, f)            ← 注意：這是可機械檢查的
```

關鍵在「**可機械檢查**」。給定具體的兩個數 p 和 f，要判斷 Prf(p, f) 真不真，你只要：把 p 分解、讀出它編碼的那串公式、檢查最後一行的編碼是不是 f、檢查每一行的合法性——全是對數字做的有限步算術操作。沒有任何一步需要「理解」或「靈感」。這正是 ch01「可證性是純語法、可機械檢查」那句話，在編碼之後的算術版身分證。（讀過書架上 TLA+ 那本《把系統寫成定理》的話，這種「拿一個宣稱的東西、逐步機械驗證對錯」的味道你熟。）

接著，把「可證」定義出來：

```text
   Prov_F(f)  ≡  ∃p  Prf(p, f)
              ＝ 「存在一個數 p，使得 p 是 f 的證明」
              ＝ 「f 編碼的那個句子，在 F 裡證得出來」
```

讀一下這個 `∃p`。它說的是「存在某個證明」——**但沒說你找得到**。這正是可證性的真面目，也呼應你早有的工程直覺：你能寫一台機器，把所有合法證明一條條按長度列出來（半可判定、遞迴可列舉的工程版——想像一個無窮迴圈，第 1 步生成所有長度 1 的符號串、檢查哪些是合法證明，第 2 步長度 2，依此類推）。如果 φ 真的可證，這台機器**遲早**會吐出它的證明，你會等到。但如果 φ 不可證，機器會永遠跑下去，你永遠不知道是「還沒到」還是「根本不會到」。`∃p` 的存在量詞，把這個「遲早會找到、但找不到時無法判定」的不對稱性精確地裝了起來。

Prov_F 還滿足一些更細的性質（它是 Σ₁ 的、滿足 HBL 三條件），但那些要到 ch08 才用得上，這裡先擱著。本章只需要你帶走一句話：

```text
   「φ 在 F 中可證」 這個原本純語法的概念，
   現在是一個 F 內部寫得出來的算術句子 Prov_F(⌜φ⌝)。
```

有了 Prov_F，下一章就能對它取否定（¬Prov_F），再用對角化造出一個說「我不可證」的句子——脊椎句 G：`F ⊢ G ↔ ¬Prov_F(⌜G⌝)`。本章的地板鋪到這裡，ch06 在上面蓋房子。

## 直覺的陷阱

哥德爾編碼是技術細節密度最高的一章，誤解多半不是「想歪了哲學」，而是「以為某個工程直覺可以直接套」。挑三個具名陷阱。

| 陷阱 | 錯誤直覺把你帶溝裡的那一步 | 正確版 / 怎麼自我察覺 |
|---|---|---|
| **陷阱一：「編碼是個巧妙的技巧，換一套也行」** | 把「質數冪次積」當成哥德爾隨手挑的一種序列化，覺得「用別的也無妨」，於是沒意識到唯一可解碼是**被 FTA 撐住的**，不是被某個約定撐住的。 | 換序列化方式可以（配對函數、進位制都行），但**任何**版本都必須保證「一個數至多對應一個序列」這條無碰撞性，而保證它的數學事實本質上就是 FTA 那一類的唯一分解。可解碼性不是設計選擇，是承重牆。自我察覺：問自己「如果這個數可以被讀成兩串不同符號，會怎樣？」——答不上來就是還沒抓到 FTA 的角色。 |
| **陷阱二：「編碼把 F 變得『更強』、能談論的東西變多了」** | 以為編碼像給 F 裝了個新外掛、擴充了它的表達力，所以「F 能談論自己」是新增的能力。 | 編碼**沒有給 F 加任何公理或符號**。F 還是只會算術、詞彙一個沒變。改變的是**我們的看法**：我們發現「某些算術謂詞」恰好可以被**解讀成**關於 F 句子的語法陳述。F 不知道自己在談論自己，它只是在算數字；是哥德爾的編碼讓那些數字「在我們眼中」有了語法意義。能談論自己的，是「夠強的算術」本來就有的潛能，編碼只是把它顯影出來。自我察覺：問「F 多了哪條公理？」——答案是「一條都沒有」。 |
| **陷阱三：「哥德爾數很大，所以這只是理論上成立、實際算不出來」** | 把「證明的哥德爾數是天文數字」誤當成「這個構造有實際缺陷／不嚴謹／要靠近似」。 | 大小完全不影響嚴謹性。每個哥德爾數都是一個**確定的、有限的**自然數，所有相關謂詞（是不是合法公式、是不是證明）都是**可判定**地對它計算的——只是計算量大。哥德爾要的是「這些概念是算術的、原則上可機械檢查的」，不是「跑得快」。把「數大」聽成「不可靠」是混淆了「實作效率」和「原則可行性」。自我察覺：你不會因為 SHA-256 的輸出是個大數就說雜湊「實際上算不出來」——同理。 |

最值得防的是陷阱二。它隱蔽，因為「F 現在能談論自己了」這句話本身沒錯，錯在把「能」理解成「被加裝了新能力」。哥德爾的震撼恰恰在於：**一個系統只要強到能做基本的乘法（從而能談質因數分解、能編碼），它就『已經』能談論自己，不需要任何額外授權**。能力是內建的，不是外掛的——這也是為什麼 ch13 那些「逃得掉」的系統（如只有加法、沒有乘法的 Presburger 算術）逃得掉：它們弱到編不了碼，於是談不了自己。編碼要乘法，乘法是門檻。

## 紙上推演

全程紙筆手算，不要打開直譯器。沿用本章玩具符號表（0→1, s→2, =→3, (→4, )→5, +→6, ∀→7, x→8, ¬→9）。

### 推演題 1 — 手編一個哥德爾數 ★ **[10 分鐘]**

把符號串「s 0」（後繼作用在 0 上，也就是數字 1）編成哥德爾數。逐步寫出質數冪次積、算到最後的整數，每步自我複核。

### 推演解答

`s 0` 是兩個符號，查表符號碼 (s, 0) = (2, 1)。

```text
   位置質數   2     3
   符號碼     2     1
   指數 +1    3     2

   ⌜s 0⌝ = 2^3 · 3^2 = 8 · 9 = 72        ← 複核：8·9 = 72  ✓
```

所以 ⌜s 0⌝ = 72。常見錯路：忘了「＋1」，寫成 2^2·3^1 = 12——那是漏掉約定。另一個錯路：把兩個符號用**同一個**質數的兩個指數塞（2^2 之類），那就沒法分辨位置，解碼即潰。記住：**一個位置配一個質數**，這才是「序列」資訊被保住的地方。

### 推演題 2 — 手解一個哥德爾數 ★★ **[12 分鐘]**

給定 N = 8100，把它解碼回符號串（這正是 Worked example 編出來的數，但請當作不知道、純靠質因數分解還原）。寫出分解過程、讀指數、查表得符號。

### 推演解答

一路除質數：

```text
   8100 / 2 = 4050
   4050 / 2 = 2025      → 2025 是奇數，2 除盡，2 的指數 = 2
   2025 / 3 = 675
    675 / 3 = 225
    225 / 3 = 75
     75 / 3 = 25        → 25 不是 3 的倍數，3 的指數 = 4
     25 / 5 = 5
      5 / 5 = 1         → 到 1，5 的指數 = 2

   複核：2^2 · 3^4 · 5^2 = 4 · 81 · 25 = 324 · 25 = 8100   ✓

   8100 = 2^2 · 3^4 · 5^2
   指數 (2,4,2)，各減 1 → 符號碼 (1,3,1)
   查表：1='0', 3='=', 1='0'  →  「0 = 0」
```

還原得「0 = 0」，與當初編進去的一致——FTA 保證不會解出第二種答案。常見錯路：指數讀完忘了減 1，得到符號碼 (2,4,2)，碼 4 是 `(`、碼 2 是 `s`，解出莫名其妙的「( s (」——這就是漏掉「＋1」約定的反向後果，編碼和解碼的「＋1／−1」必須成對。

### 推演題 3 — 為什麼非要質因數分解唯一 ★★★ **[18 分鐘]**

不查書，用自己的話回答：哥德爾編碼這整套機制，是靠算術基本定理（質因數分解唯一）的哪一個性質撐住的？做個思想實驗——假設質因數分解**不**唯一（某個數能拆成兩組不同的質數冪次），編碼會在哪一步、出什麼事？把「壞掉的後果」一路推到可證性謂詞 Prov_F。

### 推演解答

撐住整套機制的性質是**唯一性**（存在性其實不太重要，重要的是「至多一種分解」）。

思想實驗：假設有個數 N 同時等於 2²·3⁵ 又等於 2³·3³（真實算術裡不可能，純假設）。解碼 N 時，「2 的指數是 2 還是 3？3 的指數是 5 還是 3？」沒有確定答案——你會讀出兩串不同的符號碼，對應兩個不同的句子 φ 和 ψ。**N 同時是 φ 和 ψ 的編碼**。

把壞掉的後果往上推：

```text
   分解不唯一
     → 一個哥德爾數對應多個句子（編碼變成會碰撞的雜湊）
     → 「⌜φ⌝」這個記號失去確定指涉（⌜φ⌝ 可能也是 ⌜ψ⌝）
     → Prf(p, f) 裡的 f 到底指哪個句子，無法確定
     → Prov_F(f) ≡ ∃p Prf(p, f)：「f 可證」要對 φ 還是 ψ 負責？
     → 可證性謂詞失去意義，G「我不可證」裡的「我」指涉不清
     → 整座「系統談論自己」的橋斷掉
```

一句話：**唯一分解保證「一個數至多編碼一個句子」，這條無碰撞性是「⌜φ⌝ 有確定意義」的全部地基**。所以哥德爾不是「用了 FTA 來解碼」這麼輕描淡寫——他是把整座橋的承重，押在了質因數分解唯一這條兩千年的數論事實上（見《乘法的原子》談算術基本定理唯一分解那章）。常見錯路：答「靠質數有無限多個」——無限多質數保證**長句子也編得了**（位置不會用完），是必要的，但它保證的是「編得進去」，不是「讀得回來」；讀得回來唯一靠唯一分解，兩者別混。

## 自我檢核

口頭自答，講得出來才算過關，講不清楚就是還沒懂。優先答「為什麼」。

1. 為什麼哥德爾**非得**把句子編成數字，不能直接讓 F 談論符號串？（提示：F 的詞彙裡有什麼、沒有什麼。）
2. 編碼的三層分別把什麼變成數？為什麼第二層（符號序列→數）這個「壓縮動作」可以原封不動地套到第三層（公式序列→數）？
3. 質數冪次積的「指數＋1」約定在防什麼？拿掉它，什麼情況會解碼失敗？
4. 為什麼可完美還原（解碼唯一）押在算術基本定理上、而不是押在某個聰明的編碼設計上？如果分解不唯一，可證性謂詞會出什麼事？
5. 「無限多質數」和「質因數分解唯一」在編碼裡各保證了什麼？為什麼這兩件事不能混為一談？
6. 編碼之後，F 真的「變強」了嗎、多了哪條公理？（小心陷阱二。）「F 能談論自己」這句話，正確的意思是什麼？
7. 為什麼說 Prf(p, f)「可機械檢查」？把「給定 p、f，判斷 p 是不是 f 的證明」這件事拆成幾個對數字做的算術動作，講出來。
8. Prov_F(f) ≡ ∃p Prf(p, f) 裡的那個 `∃p`，為什麼對應到你工程經驗裡「半可判定／遞迴可列舉」的不對稱（找得到時遲早找到、找不到時等不到答案）？

## 延伸閱讀

- **Stanford Encyclopedia of Philosophy，「Gödel's Incompleteness Theorems」補充節「Gödel Numbering」** https://plato.stanford.edu/entries/goedel-incompleteness/sup1.html ——權威且免費，本章編碼機制的標準參考。讀它對「符號→公式→證明」三層編碼與「語法謂詞變算術謂詞」的精確處理；它用的是經典「不＋1」版（如 ⟨3,1,2⟩ → 2³·3¹·5² = 600），對照本書「＋1」版能看清約定差異只是為了避 0 冪、機制完全相同。
- **Ernest Nagel & James Newman《Gödel's Proof》** ——薄薄一本經典科普，第 7 章把哥德爾編碼講得極清楚、例子手算友善。讀它對「映射（mapping）把元數學陳述變成算術陳述」的鋪陳，跟本章「兩岸搭橋」同調，且更慢更細。
- **算術基本定理（Wikipedia）** https://en.wikipedia.org/wiki/Fundamental_theorem_of_arithmetic ——一頁看清「質因數分解唯一」的精確陳述與證明骨架，這是本章解碼唯一性的數論命脈。讀「Uniqueness」一節即可。
- **Douglas Hofstadter《Gödel, Escher, Bach》** ——第 9 章前後談哥德爾編碼與「同構（isomorphism）」，把「兩個層次（符號層 vs 數字層）之間的對應」講成全書核心隱喻；讀它對「為什麼一個層次能映射到另一個層次」的直覺，補本章工程化敘述之外的美感面（定理精確邊界仍以 SEP／Franzén 為準）。
- 跨書：質因數分解唯一性＝算術基本定理，見《乘法的原子》談算術基本定理唯一分解那章——那本在講 FTA 時已埋下指向本書的伏筆，本章正是來兌現這次握手；質數本身的無窮性與分布也在那裡，本章用到時一句帶過、不重推。
