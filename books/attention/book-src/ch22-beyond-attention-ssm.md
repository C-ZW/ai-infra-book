## 從你已知的出發

注意力最貴的東西，其實不是「算」，是「記」。

回想 ch14 的痛：生成第 n 個 token 時，self-attention 要讓這個 query 去看**全部** n 個歷史 token 的 key 與 value。每多生成一個 token，要回看的歷史就長一格——這是 O(n) 的回看，配上整段序列就是 O(n²)。更要命的是 ch16 提過的：為了不每步重算，工程上把過去算過的 K/V 全部快取起來（KV cache），於是**記憶體佔用隨序列長度線性成長**，context 拉到 100k、1M，KV cache 就把顯存吃光（這條的部署深度見《從後端到 AI Infra》ch05）。

你做後端的本能，碰到「狀態隨資料量無上限成長」會立刻警覺。你處理過這種題：一個服務要對一條無限長的事件流做統計，你**不會**把每一筆事件都存著、每來一筆就把整條歷史重掃一遍——那是 O(n²)、記憶體爆炸的寫法。你會維護一個**固定大小的 running aggregate**：一個 running sum、一個 running count、一個指數移動平均（EMA）、一個滑動視窗的摘要。新事件來了，你只更新這個固定大小的狀態，舊事件就丟掉。每步 O(1)、記憶體 O(1)、整條流走完 O(n)。

```text
兩種「記住過去」的工程姿態（你都寫過）：

(A) 全量回看（= self-attention）          (B) 固定狀態遞推（= SSM）
    每來一筆，回看全部歷史              每來一筆，只更新一個固定大小的狀態
    記憶體隨 n 成長（存全部）            記憶體 O(1)（狀態大小固定）
    成本 O(n²)                          成本 O(n)
    能精準撈回任一筆舊資料              舊資料被壓進狀態，細節會被稀釋
    └─ 你避之唯恐不及的全表掃描          └─ 你愛用的 running aggregate / EMA
```

這就是本章的全部張力，而且它正好是你的兩個老習慣的對撞。注意力選了 (A)：不壓縮、全留著、每次精準回看——代價是 O(n²) 與爆炸的 KV cache。SSM 選了 (B)：把整段歷史壓進一個**固定大小的狀態向量**，每步只更新它——代價是，被壓進去的東西，**細節會漏**。

你早就在 (B) 上吃過虧：用 EMA 做監控的人都知道，一個很久以前的尖峰，幾百步之後在 EMA 裡幾乎沒了痕跡——固定狀態的「記憶」是會衰退的。本章要講的，就是當「記住整段語言」這件事用 (B) 來做時，會省下什麼、又會漏掉什麼，以及 2026 年的業界怎麼在 (A) 與 (B) 之間做你最熟的那個取捨：**常用路徑走便宜的快取、少數需要精確的走全查詢**。

## 為什麼會有人想挑戰注意力：把 O(n²) 與 KV 的代價算給你看

先把動機釘死，免得 SSM 聽起來像「為了不一樣而不一樣」。挑戰注意力的人，盯的就是 ch14 那兩筆帳。

**第一筆帳：訓練/長序列的 O(n²) 計算。** 序列長 n，注意力矩陣 n×n（ch07）。n 從 1k 到 100k，格數從一百萬到一百億（ch14 算過）。這是平方成長。

**第二筆帳：推論時的 KV cache 記憶體。** 自迴歸生成每一步都要回看全部歷史的 K/V，於是把它們快取起來，記憶體**隨 n 線性成長、且永不釋放**（生成愈長佔愈多）。在長 context、多並發的服務場景，這往往比算力先撞牆（部署細節指向《從後端到 AI Infra》ch05，本章只取「它是個痛」這個事實）。

ch15 與 ch16 已經是針對這兩筆帳的回應，但都在「注意力框架內」動手腳：

- **稀疏/線性注意力（ch15）**：少算一些對、或換掉 softmax 用結合律——省了，但是**近似**，賠上精確回憶。
- **FlashAttention（ch16）**：精確、不近似，但它省的是**搬運**，計算量還是 O(n²)、KV cache 還是要存。

SSM 這條線的賭注更激進：**乾脆不要 n×n 矩陣這個東西。** 不在「注意力框架內」優化，而是換一個從一開始就 O(n)、狀態 O(1) 的序列建模器。它的祖宗不是 NLP，是控制論裡的**狀態空間方程**——一個你大學如果修過訊號與系統會眼熟的東西。

## SSM 是什麼：一個固定大小狀態的遞推

把控制論的包裝拆掉，連續時間的線性狀態空間模型只是兩條方程。離散化（給定步長後變成「一步一步算」）之後，長這樣：

```text
狀態更新：   h_t = A · h_{t-1} + B · x_t      ← 用「上一刻的狀態」和「這一刻的輸入」算「新狀態」
輸出：       y_t = C · h_t                     ← 從狀態讀出這一刻的輸出

x_t：第 t 步的輸入（這個 token 的向量）
h_t：第 t 步的「狀態」——一個固定大小的向量（把過去壓進來的記憶）
A、B、C：矩陣參數（學來的）。A 控制「舊狀態怎麼衰退/旋轉」、B 控制「新輸入怎麼寫進狀態」、C 控制「怎麼從狀態讀出」
```

關鍵在 h_t 是**固定大小**。不管你已經看過 5 個還是 5 萬個 token，h_t 永遠是同一個維度的向量。新 token 進來，就用 A 把舊狀態「轉一下、衰退一點」，再用 B 把新輸入加進去。整段序列走完，每步 O(1)、總共 O(n)，**沒有任何 n×n 矩陣**。

這個遞推，你看著眼熟——它就是 ch15 結尾埋的那個影子：線性注意力寫成自迴歸形式時，那個「邊生成邊累加的固定大小狀態」。Katharopoulos 的「Transformers are RNNs」（2020）其實已經把線性注意力變成一個 RNN 遞推了。SSM 是同一族的、結構更講究的成員。

那它「記得住長程依賴嗎」？這正是 SSM 早期最難的點。如果 A 隨便設，狀態要嘛爆炸要嘛迅速衰退（你的 EMA 直覺：衰減太快，幾步前的東西就沒了）。**S4**（Gu, Goel & Ré 2021，arXiv 2111.00396，ICLR 2022；2026-06 查證）的突破就在這裡：它用一個叫 **HiPPO** 的數學框架去構造 A、B 矩陣，讓那個固定狀態能**最優地記住一段輸入歷史的多項式投影**（用正交多項式基底去逼近過去的訊號）——簡單說，是一套「怎麼設計衰退方式，才能在固定大小的狀態裡盡量不丟長程資訊」的理論配方。S4 靠它在 Long Range Arena 這個專測長程依賴的 benchmark 上拿下 SOTA，首次解開長度 16k 的 Path-X 任務（2026-06 查證）。

> HiPPO 的完整數學（正交多項式、為什麼那組特定的 A 矩陣最優）是一整篇論文的事，本書只取結論：**它是「怎麼在固定狀態裡記住最多長程歷史」的理論配方**。深入見延伸閱讀。

S4 漂亮，但有個硬傷：A、B、C 是**固定**的（不隨輸入變），這叫線性時不變（LTI）。LTI 讓 S4 可以用 FFT 卷積飛快訓練，但也讓它「笨」——它對每個 token 一視同仁地寫進狀態，**沒辦法根據內容決定「這個 token 重要、多記一點；那個 token 是廢話、跳過」**。而語言恰恰需要這種選擇性：該記住的是名詞、實體，不是每個 "the" 和 "of"。

## Mamba：讓狀態學會「選擇性遺忘」

**Mamba**（Gu & Dao 2023，arXiv 2312.00752，COLM 2024；2026-06 查證）補的就是這個洞。它的核心改動叫**選擇性（selective）SSM**：讓 B、C（還有控制離散化步長的參數）**變成輸入 x_t 的函數**——B(x_t)、C(x_t)。

這一改，意義是什麼？把它翻回你的 running aggregate 直覺：

```text
S4（LTI）的狀態更新：   h_t = A·h_{t-1} + B·x_t
    B 固定 → 每個 token 用「同一個力道」寫進狀態（不分輕重）

Mamba（選擇性）的更新： h_t = A·h_{t-1} + B(x_t)·x_t
    B 隨輸入變 → 模型可以「看這個 token 是什麼」再決定寫多用力
    遇到關鍵實體（animal）→ 用力寫進狀態、之後別讓它太快衰退
    遇到填充詞（the、didn't）→ 寫得很淡，甚至幾乎跳過
```

這就是你維護 running aggregate 時其實一直在做的事：不是每筆事件都同等更新統計量，重要的事件你給它更大權重、雜訊你壓低。Mamba 把這個「依內容決定寫多少」變成可學習的、可微分的。

代價是：B、C 一旦隨輸入變，S4 那套 FFT 卷積的快速訓練就不能用了（卷積要求 LTI）。Mamba 的解法是一個**硬體感知的平行掃描（hardware-aware parallel scan）**：遞推看起來是循序的（h_t 要等 h_{t-1}），但 SSM 的遞推有結合律，可以用 prefix-scan（你寫過的 parallel prefix sum 的同一個招）平行化；再配上 kernel fusion 和 recomputation 把資料留在快的記憶體（這幾招的味道你在 ch16 的 FlashAttention 見過——都是 IO-aware）。結果：Mamba 序列長度**線性時間**、號稱約 5× Transformer 的吞吐量（2026-06 查證）。

**Mamba 動了三步骨架的哪一步？這是本章你必須想清楚的一句。** 答案出乎意料：**它把整個三步骨架換掉了。** 注意力的三步——打分（query 對每個 key 算相似度）、正規化（softmax）、加權混合（對所有 value 加權）——每一步都預設「我手上有全部 key/value、我要對它們做點什麼」。SSM 沒有「全部 key/value」這個東西，它只有一個被不斷覆寫的固定狀態。沒有打分（沒有 query-key 內積）、沒有 softmax、沒有「對 n 個 value 加權」。**它不是三步骨架的一個變體（像 ch15 那些），它是另一台機器。** 這就是為什麼這一章叫「注意力之外」——前面所有變體都還住在三步骨架裡，SSM 第一次走出了門。

下一節會看到，這扇門外其實有一條暗道通回來——但要先讓脊椎那個 it 在這裡撞牆。

## 脊椎在這裡斷：it 找不到 animal 的連線

從 ch01 到 ch21，脊椎那個 it 每一章都越懂越深：ch04 它對三個 key 打出原始分數 [4,2,0]、ch05 softmax 成權重、ch06 縮放後成全書正典的 [0.665, 0.245, 0.090]、ch07 我們畫出 3×3 注意力矩陣看到「it 那列在 animal 欄最濃」、ch21 還把它畫成一個 QK/OV 電路。每一次，核心問題都是同一個：**it 這個位置，該看 [animal, street, it] 裡的哪一個？** 而答案永遠是一張可以指出來的圖——it→animal 那條最粗的連線。

到了 SSM，**這條線畫不出來了。**

```text
注意力裡的 it（ch07，能畫出來）：       SSM 裡的 it（畫不出來）：

       key→  animal street  it          token 依序流過，壓進一個固定狀態 h
query↓                                    ┌────────────────────────────┐
animal       ▓▓    ░     ░               animal → h₁ = A·h₀ + B(animal)·x_animal
street       ░     ▓     ░                            （把 animal 寫進狀態）
it           ███   ▒     ░               street → h₂ = A·h₁ + B(street)·x_street
             ▲                                        （狀態被更新、animal 被衰退一點）
        it 對 animal 的權重 0.665          it     → h₃ = A·h₂ + B(it)·x_it
        ——可以指著這格說「就這條」                    y_it = C·h₃
                                          ▲
                                   it 的輸出只能從 h₃ 讀出。
                                   h₃ 裡「還剩多少 animal」？
                                   沒有一個「it 看 animal」的權重可指。
```

SSM 處理脊椎句子時，token 是**一個一個流過去**的：animal 先進來、寫進狀態 h₁；street 進來、把狀態更新成 h₂（此時 animal 的成分被 A 衰退了一點、又被 street 蓋進來一些）；輪到 it，狀態變成 h₃，it 的輸出 y_it = C·h₃。

問題來了：**y_it 裡「來自 animal 的貢獻」有多少？沒有一個乾淨的數字可以回答。** animal 的資訊早在兩步前就被壓進狀態、被後續的 street 和 it 一路覆寫稀釋。注意力裡那個「it 對 animal 的權重 = 0.665」是一個可以指著熱圖說「就這格」的東西；SSM 裡，animal 的影響糊在 h₃ 這個固定向量的每一個分量裡，**沒有 N×N 矩陣、沒有一條 it→animal 的連線可畫**。

用 ch20 的話講：注意力至少給了你一張「看起來像解釋」的熱圖（不管它是不是真解釋）；SSM 連那張熱圖都沒有。問題的形狀整個變了——

```text
注意力問的問題：  「it 該看 [animal, street, it] 裡的哪一個？」
                  → 答案是一個權重分布 [0.665, 0.245, 0.090]，可指可畫

SSM 問的問題：    「流到 it 這一步時，那個固定狀態 h₃ 裡，
                   還留住了多少 animal 的資訊？」
                  → 答案藏在狀態的衰退裡，是一個資訊漏斗的問題，不可指不可畫
```

這是脊椎在全書**唯一一次刻意失效**。我沒有要修好它——這個失效本身就是本章最好的教具。它精準照出 SSM 與注意力的根本差別：**注意力把整段歷史攤平存著、要用時精準回看（所以能指出 it→animal）；SSM 把整段歷史壓進固定狀態、用衰退的方式記住（所以只能問「漏斗留住了多少」）。** 脊椎的斷裂，就是「固定狀態 vs 全量回看」這個取捨在一個三 token 句子上的縮影。

讓我用一個更小的數字把「資訊漏斗」說死。簡化到一個純量狀態的 SSM，A=0.9（每步狀態衰退到九成）、B=1、C=1。假設只有 animal 那一步有訊號（x=1），之後都是 0（看後面還剩多少 animal）：

```text
t=0  animal 進來    h = 0.9·0 + 1·1 = 1.0000   ← animal 剛寫進去，滿格
t=1  （下一個 token）h = 0.9·1.0     = 0.9000   ← 衰退到 90%
t=2                 h = 0.9·0.9     = 0.8100
t=3                 h = 0.9·0.81    = 0.7290
t=4                 h = 0.9·0.729   = 0.6561

「animal 還剩多少」= A^k，隨距離指數衰退：
  隔  1 步：0.9^1  = 0.90   （還記得很清楚）
  隔  5 步：0.9^5  = 0.59
  隔 10 步：0.9^10 = 0.35
  隔 20 步：0.9^20 = 0.12
  隔 50 步：0.9^50 = 0.0052 （幾乎沒了——你的 EMA 老朋友）
```

這就是固定狀態的宿命：愈遠的 token，在狀態裡留下的痕跡愈淡（A^k 指數衰退）。Mamba 的選擇性讓模型可以**對重要 token 設更接近 1 的有效衰減、把它記得更久**，但「固定大小狀態裝不下無限長歷史的全部細節」這個物理上限，選擇性只能緩解、不能取消。**這就是 SSM 用 O(1) 狀態換掉「精確回憶」的代價——它記得住「大意」，但要它幾千步之後精準複述開頭定義的那個變數名，它的漏斗早把細節漏光了。** 這正是 ch15 線性注意力「在精確回憶上吃虧」的同一個病，因為——下一節揭曉——它們本來就是同一條線。

## 對偶之美：SSM 其實是線性注意力

走出三步骨架的門之後，有一條暗道把 SSM 接回來，而且接得異常漂亮。

**Mamba-2 / SSD**（Dao & Gu 2024，arXiv 2405.21060，「Transformers are SSMs」，ICML 2024；2026-06 查證）證明了一件事：**SSM 與（因果的）線性注意力，是同一個數學物件的兩種看法。** 這個等價框架叫**結構化狀態空間對偶（Structured State Space Duality, SSD）**。

直覺版的橋怎麼搭？回想 ch15 那個關鍵動作——線性注意力的自迴歸形式可以寫成「邊掃邊累加一個固定大小的狀態」：

```text
線性注意力（自迴歸/因果形式，ch15 結尾）：
    狀態 S_t = S_{t-1} + φ(k_t) v_tᵀ      ← 每來一個 token，把它的 key/value 外積累加進狀態
    輸出 y_t = φ(q_t)ᵀ S_t                ← query 去查這個累積狀態

SSM（這一章）：
    狀態 h_t = A·h_{t-1} + B(x_t)·x_t      ← 每來一個 token，更新固定大小狀態
    輸出 y_t = C·h_t

兩者並排看——結構一模一樣：「遞推一個固定大小的狀態，再從它讀出輸出」。
差別只在 SSM 多了一個 A（讓舊狀態衰退/旋轉），而線性注意力的 A 退化成「不衰退」(=1)。
```

SSD 把這個直覺做成嚴格的等價：**SSM 的整段計算可以寫成一個特殊的結構化矩陣（半可分矩陣，semiseparable matrix）的乘法，而那正好對應到一個帶因果遮罩的線性注意力。** 反過來，線性注意力就是 A 取特定形式的 SSM。Mamba-2 進一步把 SSM 的 A 限制成「純量乘以單位矩陣」（所有對角元相同），讓計算能寫成半可分矩陣的塊狀分解，借用成熟的矩陣乘法硬體，訓練比 Mamba 快 2–8×（這主要是改訓練效率，不是改推論行為；2026-06 查證）。

這個對偶為什麼美？因為它把全書的兩條線收束成一個物件：

```text
              一個物件，兩種讀法

   讀成「全量回看」              讀成「固定狀態遞推」
   = (因果)線性注意力             = SSM
   ──────────────────           ──────────────────
   打分→正規化→加權混合           h_t = A·h_{t-1} + B·x_t
   （三步骨架，ch15）              y_t = C·h_t（本章）
        │                              │
        └──────── SSD 等價 ────────────┘
        （半可分矩陣的兩種分解；Dao & Gu 2024）

   注意：這條對偶接的是「線性」注意力（ch15），不是 softmax 注意力。
   softmax 那個 exp 拆不開（ch15 的死結），不在這個對偶裡——這一點是關鍵。
```

注意最後那句警語，它是本章最容易被熱情沖昏頭講錯的地方：**SSD 對偶接的是線性注意力，不是 softmax 注意力。** 軟性的 softmax 注意力——那個能把權重做得極尖、扛精確回憶的、全書正典脊椎用的那個——**不在這個對偶裡**，因為 softmax 的 exp 拆不開（ch15 的死結，正是 O(n²) 的鎖也是它精確回憶能力的來源）。所以「SSM = 注意力」這句話只在「線性注意力」這個前提下成立，而線性注意力正是 ch15 說會在精確回憶上吃虧的那個。對偶之美不是「SSM 取代了 softmax」，而是「SSM 和那個會漏精確回憶的線性注意力，原來是一家人」。脊椎那個 it 找不到 animal，跟線性注意力糊掉精確檢索，是同一個病在同一條線上的兩個症狀。

## 2026-06 的現況：誰贏了？（誠實標時點）

這一節全部標 `（2026-06）`，並嚴格區分「已驗證事實」與「趨勢/推測」——這是 landscape 對 frontier 章的鐵律。

**已驗證事實（2026-06）：**

- **softmax 注意力仍主導生產級 LLM。** 第一線的旗艦模型（GPT-5 系列、Claude、Gemini 等）核心仍是 softmax 注意力。沒有一個純 SSM 模型在通用能力上稱霸市場（landscape §7，兩來源覆核 upheld）。
- **混合架構（hybrid）已是出貨中的真實路線，不是論文玩具。** 代表作：
  - **Jamba**（AI21，2024-03）：首個生產級**混合 SSM-Transformer**，約 **1/8 層**是注意力、其餘是 Mamba 層；52B 總參數 / 12B 活躍（MoE）、256K context、Apache 2.0（2026-06 查證）。
  - **Nemotron-H**（NVIDIA，2025）：把約 **92% 的注意力層換成 Mamba-2 塊**，吞吐量比同規模 Transformer（如 Llama-3.1、Qwen-2.5）快達 3×，準確度持平或更好；後續 Nemotron 3 系列把這套混合做進 MoE（2026-06 查證）。
  - **MLA**（DeepSeek，見 ch16）：另一條路——**保留 softmax**，只把 KV 壓成低秩潛在向量省記憶體。
- **SSM 自己也在進化。** **Mamba-3**（Together.ai，arXiv 2603.15569，ICLR 2026；2026-06 查證）引入複數值狀態更新（用 RoPE 那招把旋轉位置資訊直接編進遞推，呼應 ch10）與 MIMO 變體，主打**推論**效率，長序列在 H100 上快達 7×，下游平均小幅領先 Transformer baseline。

**趨勢/共識（非終局，可能被推翻）：**

- 新共識傾向「**1-in-8 到 1-in-10 的混合**」：保留少數幾層完整 softmax 注意力扛**精確回憶**（in-context learning、大海撈針、精準複製），其餘大多數層用便宜的 Mamba/線性層省 O(n²) 與 KV。這正是 ch15 那筆「近似會在精確回憶上吃虧」學費的工程解法——**不是二選一，是分工**。
- 純 SSM vs 混合 vs 改進的 softmax，**長期勝者未定論**。SSM 證明了「注意力不是序列建模的唯一解」，但還沒證明「不需要注意力」。

把它翻成你最熟的取捨，一句話收尾：

```text
業界 2026-06 的混合架構 = 你做系統時的老智慧：
  「大多數請求走便宜的快取/摘要路徑（SSM/線性層，O(n) 狀態），
    少數真的需要精確的查詢，才走貴的全表掃描（softmax 注意力，O(n²) 精確回憶）。」
  你不會讓所有請求都全表掃描，也不會全砍掉精確查詢——你混著用。
```

**那「attention is all you need」2026 還成立嗎？** 我的誠實答案（會隨時間變，標 2026-06）：作為「序列建模唯一機制」的字面宣稱，已經**不成立**了——SSM 證明了固定狀態遞推也能打。但作為「混合架構裡那個不可或缺、扛精確回憶的核心零件」，softmax 注意力**still very much needed**。這句口號從「全部」退守成「不可或缺的一部分」——而退守得很有尊嚴，因為退守的理由（精確回憶）正是它從 ch04 一路證明過來的看家本領。

## 故障模式與直覺陷阱

SSM/Mamba 是 2026 最容易被「行銷化」的題目（X is all you need 的句型在這個領域是傳統藝能），下面是真正會把你帶溝裡的地方。

### 招牌陷阱：「Mamba 取代了 Transformer」

這是 2024 以來最常見的誤讀，幾乎每篇科技新聞標題都犯。事實（2026-06）：**沒有。** 純 SSM 沒有稱霸生產級通用 LLM，旗艦仍是 softmax 注意力，而真正出貨的「後注意力」路線是**混合**（保留少數注意力層），不是「拔掉注意力」。會掉進這個陷阱，通常是把「Mamba 在某些 benchmark/吞吐量上贏」誤讀成「全面取代」——而它輸的那些 benchmark（精確回憶類）恰恰是被新聞標題略過的。

### 陷阱清單

| 陷阱 | 錯誤直覺 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| 「SSM 取代了注意力」 | 「Mamba 比 Transformer 快又好，所以注意力過時了」 | 忽略純 SSM 在精確回憶上輸 softmax；忽略「出貨的是混合、不是替代」 | 問：你說的「贏」是哪個 benchmark？是吞吐量還是大海撈針？後者 SSM 多半輸 |
| 把對偶當「SSM = softmax 注意力」 | 「SSD 證明 Transformer 是 SSM，所以兩者等價」 | 對偶接的是**線性**注意力，softmax 的 exp 拆不開、**不在**對偶裡（ch15 死結） | 看清等價的前提：是 linear attention，不是你天天用的 softmax attention |
| 以為固定狀態「不丟資訊」 | 「Mamba 的選擇性能記住該記的，所以沒漏」 | 固定大小狀態有物理上限（A^k 指數衰退）；選擇性只緩解、不取消漏斗 | 在「複製一長串、回憶開頭定義的變數」這類任務上掉分，先懷疑狀態漏接 |

### 自我察覺的徵兆

- 你開始講「注意力死了/被取代了」——停。問自己：你說的取代，有哪個生產級旗艦模型是純 SSM？（2026-06：沒有。）
- 你想對一個 Mamba 模型畫「它在看哪裡」的熱圖——停。SSM 沒有那張圖，你要問的是「狀態裡還留著什麼」，不是「它看哪裡」。
- 你把「SSD 對偶」當成「SSM 就是 Transformer」——停。回去看前提：對偶接的是線性注意力，softmax 不在裡面。
- 你看到一個吞吐量數字就下「SSM 更好」的結論——停。問是哪個任務；長程精確回憶上，便宜往往是從那裡扣的。

### 推演解答

**推演 1 解答。**

```text
維度                     softmax 注意力             SSM（Mamba）
─────────────────────────────────────────────────────────────────
每步計算（生成第 n 步）    O(n)（回看全部歷史）       O(1)（只更新固定狀態）
記憶體隨 n 成長            O(n)（KV cache 線性漲）     O(1)（狀態大小固定）
精準撈回任一舊 token       能（全留著、精準回看）       難（壓進狀態、細節衰退漏掉）
脊椎 it→animal 連線        能畫（權重 0.665，ch07）    畫不出（無 N×N 矩陣）
```

一句話總結：**SSM 用「固定大小狀態」換掉「O(n²) 與爆炸的 KV」，代價是「精確回憶」——它記得住大意，記不牢細節。** 注意力是反過來的取捨：付 O(n²) 與大 KV，換來精準回看任一個 token 的能力。

常見錯路：以為「O(1) 狀態」是純賺。不是——那個固定大小是有上限的容器，序列愈長愈像資訊漏斗（推演 2 的 A^k 證明）。

**推演 2 解答。**

狀態（A=0.9, B=1, C=1，只有 t=0 有 x=1）：

```text
t=0  h = 0.9·0 + 1·1 = 1.0000
t=1  h = 0.9·1.0     = 0.9000
t=2  h = 0.9·0.9     = 0.8100
t=3  h = 0.9·0.81    = 0.7290
t=4  h = 0.9·0.729   = 0.6561

animal 還剩多少 = A^k：
  k=1: 0.90    k=5: 0.59    k=10: 0.35    k=20: 0.12    k=50: 0.0052
```

為什麼注意力能指出 0.665 而 SSM 不能：注意力**把 animal 的 value 整個留著**，it 來查時對它算一個明確的權重（0.665）並加權混合——這個權重是一個可指、可畫的數字。SSM **不存 animal 的 value**，只把它「寫進」一個會被後續 token 一路覆寫、又被 A 指數衰退的固定狀態；輪到 it 時，狀態裡「來自 animal 的成分」糊在每個分量裡、沒有一個乾淨的數字。

跟精確回憶的關係：精確回憶 = 「從遠處精準撈回某一個特定 token」。注意力做得到（全留著、精準回看）；SSM 的 A^k 衰退（k=50 只剩 0.0052）說明遠處 token 的痕跡幾乎沒了——這正是 SSM 在精確回憶上吃虧的物理根源。Mamba 的選擇性能對重要 token 設更接近 1 的衰減記更久，但固定容器的上限取消不掉。

**推演 3 解答。**

```text
線性注意力（自迴歸）：  S_t = S_{t-1} + φ(k_t) v_tᵀ      y_t = φ(q_t)ᵀ S_t
SSM：                   h_t = A·h_{t-1} + B·x_t          y_t = C·h_t

結構相同：兩者都是「遞推一個固定大小狀態 + 從它讀出輸出」。
差別：SSM 多了 A（讓舊狀態衰退/旋轉）；線性注意力的 A 退化成 1（不衰退、純累加）。
```

SSD 對偶接的是**線性注意力**，不是 softmax。為什麼 softmax 不在裡面：softmax 的 exp(q·k) 拆不開（ch15 的死結），沒辦法寫成「先把 key 那邊聚合成固定狀態」的遞推形式——而那個遞推形式正是 SSM/線性注意力的共同骨架。softmax 那個拆不開的 exp，同時是它 O(n²) 的鎖、和它精確回憶能力的來源。

這對「SSM 能不能取代 softmax」的意思：對偶證明的是「SSM 和**線性**注意力等價」，而線性注意力恰恰是 ch15 說會在精確回憶上吃虧的那個。所以對偶**不**等於「SSM 取代了 softmax」——它等於「SSM 和那個會漏回憶的線性注意力是一家人」。softmax 的看家本領（精確回憶）不在這個對偶裡，也正因如此，2026 的混合架構才要保留它。

常見錯路：看到論文標題「Transformers are SSMs」就以為「所有 Transformer = SSM」。標題是漂亮的簡化；嚴格的等價只在線性注意力上成立。

**推演 4 解答（口述要點）。**

- **時點**：2026-06，這題的答案會隨時間變，講的時候要說清楚。
- **已驗證事實**：softmax 注意力仍是生產級旗艦（GPT-5/Claude/Gemini）的核心；混合架構（Jamba 約 1/8 層注意力、Nemotron-H 約 92% 層換成 Mamba）已經在出貨；不存在純 SSM 稱霸通用 LLM。
- **趨勢/推測**：新共識傾向「1-in-8 到 1-in-10」的混合比例；純 SSM vs 混合 vs 改進 softmax 的長期勝者未定論。
- **類比**：混合 = 大多數層走便宜的 SSM/線性（O(n) 狀態，扛得住「記大意」），少數幾層留 softmax 全查詢（O(n²)，扛「精準撈回那一個」）——就像你系統裡常用路徑走快取、少數需要強一致的請求才走全查詢。
- **一句收尾**：「attention is all you need」作為「唯一機制」已不成立，但作為「混合裡不可或缺、扛精確回憶的零件」依然 needed。

4. 脊椎那個 it 為什麼在 SSM 裡找不到 animal 的連線？這個失效照出了 SSM 與注意力的什麼根本差別？用 A^k 指數衰退把「資訊漏斗」講清楚。

## 延伸閱讀

- **Gu, Goel & Ré 2021「Efficiently Modeling Long Sequences with Structured State Spaces」（S4）**，arXiv 2111.00396（ICLR 2022）。SSM 這條線的奠基作，HiPPO 如何讓固定狀態記住長程歷史。想懂「為什麼固定狀態能記長程」讀它的動機與 Long Range Arena 結果，HiPPO 數學細節可先略過。
- **Gu & Dao 2023「Mamba: Linear-Time Sequence Modeling with Selective State Spaces」**，arXiv 2312.00752（COLM 2024）。讀「選擇性」那一節——B/C 變成輸入的函數，是 Mamba 對 S4 的關鍵一改；對照本章「重要 token 多記」的直覺。
- **Dao & Gu 2024「Transformers are SSMs: ... Structured State Space Duality」（Mamba-2/SSD）**，arXiv 2405.21060（ICML 2024）。本章「對偶之美」的源頭。讀它如何用半可分矩陣把 SSM 與線性注意力接起來；特別注意它接的是線性注意力、不是 softmax。
- **Together.ai 2026「Mamba-3: Improved Sequence Modeling using State Space Principles」**，arXiv 2603.15569（ICLR 2026）。最新一代 SSM，複數值狀態（呼應 ch10 的旋轉位置）與 MIMO、主打推論效率。frontier 資料，讀時記得標 2026 時點。
- **AI21 部落格「Attention was never enough: Tracing the rise of hybrid LLMs」**（ai21.com）。混合架構的脈絡與動機綜述（Jamba 出身），把「為什麼是混合不是替代」講得最白；搭配本章 2026 現況節讀。
- **NVIDIA「Nemotron-H: A Family of Accurate and Efficient Hybrid Mamba-Transformer Models」**，arXiv 2504.03624（2025）。生產級混合的代表，讀「換掉 92% 注意力層」的設計取捨與吞吐量數字。
