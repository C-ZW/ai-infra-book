## 從你已知的出發

回到 ch01 的你。

那時我給你一句英文：*"the animal didn't cross the street because **it** was too tired."*，問你 it 指 animal 還是 street。你一眼知道是 animal——累的是動物不是街——但你說不出「機器憑什麼也知道」。當時你手上只有直覺：it 比較「像」animal。你給了三個直覺分數，混出「it 偏 animal」，然後我說：這就是三步骨架的預告，真正的計算從 ch04 開始。

現在你走完了。同一個 it，你不再只有直覺。你能：給它一個 query 向量、給三個 token 各一個 key、算內積得到原始分數 [4, 2, 0]、除以 √d 防止 softmax 飽和、softmax 成一組和為 1 的權重 [0.665, 0.245, 0.090]、用這組權重去混合三個 value、得到輸出 (0.755, 0.335)——一個偏向 animal 的向量。你不只算得出來，你還知道每一步在解決什麼問題、最常被誤解成什麼、它的理論身世是什麼、2026 年它是否還是唯一解。

這正是這本書一開始承諾的事：**那些「該看哪裡」的查找規則，你以前手寫死在 routing table 和 `WHERE` 子句裡，注意力讓模型自己學出來——而且這件事是算出來的，不是規定的。** 你天天在系統裡做「該看哪裡」的決策，差別只有一個：你的規則是你親手鋪的水道，注意力的規則是從資料裡長出來的、軟性的、按相似度加權的。這個差別，撐起了 2017 年以降整個大型語言模型的時代。

這一章，我要你把這件事收成一條能口頭講完的故事線。先看地圖。

## 三步骨架是唯一主角：從 Bahdanau 到 SSM 都是它的精煉

如果這本書只能留一句話，是這句：**注意力的所有變體，本質都是「打分（score）→ 正規化（normalize）→ 加權混合（mix）」這三步的精煉。** 每一個被點名的東西，都只是動了其中一步——沒有一個例外。我把全書點過名的變體，按它動了哪一步排成一張表。這張表是你「不被名詞淹沒」的最後一塊浮木：

```text

─────────────────────────────────────────────────────────────────────────

QKV 投影             把「打分」的輸入拆成 query/key 兩個不同投影           ch04
softmax              「正規化」這一步本身（soft-argmax，輸出機率分布）     ch05
÷√d 縮放             縮「打分」的尺度，防 softmax 飽和                     ch06
self-attention       讓 Q/K/V 同源（誰打分、被誰打分），不改三步本身       ch07

```

讀這張表，你會看到一個漂亮的事實：**三步骨架的彈性，幾乎全來自「打分」這一步。** query/key 投影、位置編碼、稀疏連結、cross-attention、KV 共享，動的都是「誰跟誰打分、用什麼打分、打分前蓋什麼戳」。「正規化」這一步動得少（softmax → 核函數是少數例外），「加權混合」這一步幾乎沒人動過——value 進場、按權重平均，從 Bahdanau 到 FlashAttention 都是這麼做。

唯一真正跳出這張表的，是表格最後一行打了 ⚠️ 的 **SSM/Mamba**。它不是動三步骨架的某一步——它整個換掉了這個範式：不再對所有位置打分、正規化、混合，而是把過去壓進一個固定大小的狀態裡遞推（ch22）。這是全書唯一一次脊椎「斷掉」——SSM 沒有 N×N 注意力矩陣，畫不出 it→animal 的那條連線。而正是這個斷裂，照出了注意力用 O(n²) 成本換來的東西：**精確回憶。** SSM 用固定狀態換掉了它，省了成本，代價是「漏斗有沒有留住 animal」變成一件不保證的事。這是 ch22 留給你的、也是 2026 年整個領域還在權衡的取捨。

## 七條驗收目標：口試與模範答案要點

ch01 開宗明義給了七條驗收目標。現在它們是七道口試題。規矩照 style-guide：**口頭自答，講得出來才算過。** 每題我先給題目，再給「模範答案要點清單」——你自答完，對照清單看自己漏了哪一點。優先答「為什麼」而不是「是什麼」。

**第 1 題：注意力到底在算什麼？（一句話）**

要點清單：
- 一句話：**對所有位置做一次「依相關性加權的平均」。**
- 拆成三步：打分（query 對每個 key 算相似度，標準作法是內積）→ 正規化（softmax 把一排分數變成和為 1 的權重）→ 加權混合（用權重對所有 value 做加權平均）。
- 反直覺點（必講）：它**不是**「找最相關的一個」，是**混合全部**，相關的權重大、不相關的權重小但不為零。脊椎裡 it 對 street 也有 0.245 的權重，不是 0。

**第 2 題：為什麼要 Q/K/V 三個角色、為什麼除以 √d、為什麼用 softmax？**

要點清單：
- **Q/K/V**：軟性字典查找。query＝「我在找的」、key＝「我是的」、value＝「我能提供的內容」。query 和 key 用**不同**的學來投影，因為「我找的」≠「我是的」是兩件事（非對稱）；若 W_Q=W_K，每個 token 最像自己，退化（ch04）。
- **÷√d**：變異數論證（Vaswani 2017 §3.2.1）。q、k 各分量獨立、期望值 0 變異 1 時，內積 q·k 的變異數＝d_k；d_k 大則分數分布變寬，把 softmax 推進飽和區（一個權重→1、其餘→0、梯度趨近 0、學不動）。除以 √d_k 把變異數拉回 ~1。原始 Transformer d_k=64→√d_k=8（ch06）。
- **softmax**：它是 argmax 的可微分「軟」版（soft-argmax）。硬 argmax 只挑最大值、不可微、只看一個；softmax 按比例看全部、處處可微（能訓練）、輸出是合法機率分布（非負、和為 1）（ch05）。

**第 3 題：multi-head 在分什麼工？位置編碼補了什麼洞？**

要點清單：
- **multi-head**：把 d_model 切成 h 顆頭（原始 h=8、d_k=d_model/h=64），每顆頭在**不同子空間**做完整三步骨架，輸出 concat 再過 W_O。為什麼要多頭：一顆頭的 softmax 只能聚焦**一種**關係（一個權重分布），多頭讓模型同時抓句法、指代、位置等多種關係。脊椎：head A 抓「誰是主詞」、head B 抓「it→animal 指代」（ch08）。
- **位置編碼補的洞**：self-attention 本質是**集合**運算，**置換不變**——重排 token 只重排輸出、權重不變（脊椎：「animal street it」和「it street animal」對模型一樣）。集合沒有順序，所以順序資訊必須**外加**。正弦編碼加在 embedding 上；RoPE 用旋轉，內積只剩相對位置（接《圓的影子》）（ch09–10）。

**第 4 題：為什麼是 O(n²)？業界怎麼想「替這張沒有索引的表建索引」？**

要點清單：
- **O(n²) 的來源**：self-attention 要算 n×n 的注意力矩陣（n 個 query × n 個 key），每格是一對 token 的相似度。這是「一張建不了索引的表」——hash/B-tree 索引靠可排序/可雜湊的鍵把查找降到 O(log n)/O(1)，但「相似度」不是這種鍵，無法預先建索引、每對都要算（ch14，接《從後端到 AI Infra》ch03）。
- **兩條路**：① **稀疏**（改打分的連結結構，只算部分 token 對，如 Longformer/BigBird）；② **線性**（改正規化，把 softmax 換成核函數、用矩陣乘法結合律把 (QKᵀ)V 重排成 Q(KᵀV)，O(n²)→O(n)）。兩者都是**近似**，常損精確回憶。③ 另有 **FlashAttention**：不近似，用 online softmax＋分塊、不實體化 n×n 矩陣，數學上完全等於標準注意力（精確）（ch15–16）。

**第 5 題：注意力的理論身世是什麼？（核迴歸／Hopfield／集合運算）**

要點清單：
- **核迴歸**（Tsai 2019）：attention 是 Nadaraya-Watson 核平滑——以相似度為核的加權平均，輸出＝條件期望的估計（接《馴服隨機》加權平均＝期望）。
- **聯想記憶**（現代 Hopfield，Ramsauer 2020）：連續狀態的 Hopfield 更新規則**等價於** Transformer attention；K/V＝存的記憶、query＝線索，attention＝軟性內容定址檢索（回扣 ch04 字典、Graves NTM）。
- **集合運算**（Set Transformer，Lee 2019）：attention 是集合上的置換等變運算（回收 ch09 為何要加位置）；與 GNN／訊息傳遞相通；純堆 attention 會**秩坍縮**（Dong 2021，雙指數坍縮到秩 1），殘差與 FFN 是解藥（回收 ch13）。
- **嚴謹度標示（必講）**：這些是深刻的**等價／特例視角**（attention 寫得出核迴歸、Hopfield 的形式），不是巧合也不是鬆散類比——但要說清哪裡嚴格、哪裡只是直覺（ch17–19）。

**第 6 題：注意力權重能不能當「解釋」？機制可解釋性看到了什麼？**

要點清單：
- **論戰未定論（必講「未定論」）**：Jain & Wallace 2019「Attention is not Explanation」vs Wiegreffe & Pinter 2019「Attention is not not Explanation」。分歧本質是「解釋」的**定義**之爭——**忠實性（faithfulness）**：模型真的據此決策 vs **合理性（plausibility）**：看起來合理。不是事實對錯，至今（2026-06）無共識（ch20）。
- **頭確實有結構**：某些頭對應句法關係（Clark 2019）；少數重要頭扛大部分工作（Voita 2019：WMT 英俄上剪掉 48 頭中的 38 頭只損 0.15 BLEU）——但「有些頭有意義」不等於「整體可解釋」。
- **機制可解釋性**（不同層次的解釋）：Anthropic 把一顆頭拆成 **QK 電路**（看哪裡）＋**OV 電路**（搬什麼）（Elhage 2021）；**induction heads**（Olsson 2022）執行 `[A][B]…[A]→[B]` 的補全，其出現時機與 in-context learning 能力突增同時，是 ICL 的機制源（ch21）。

**第 7 題：2026 年「attention is all you need」還成立嗎？**

要點清單（**誠實標時點 2026-06、區分事實與推測**）：
- **已驗證事實**：softmax 注意力**仍主導**生產級 LLM（GPT-5 系列、Claude、Gemini 等）。SSM 這條線（S4 2021、Mamba 2023、Mamba-2/SSD 2024）證明了 SSM 是因果線性注意力的特例（對偶之美，接 ch15），純 SSM 在算法類／精確回憶任務上仍輸 Transformer。
- **趨勢（非終局）**：**新共識傾向「混合」**——以小比例（如 1-in-8 ~ 1-in-10）注意力層保留檢索能力、其餘用 Mamba/線性層（Jamba、Qwen、Nemotron 等已出貨）。
- **誠實結論**：**不存在純 SSM 稱霸**；純 SSM vs 混合 vs 改進 softmax 的長期勝者**未定論**。所以「attention is all you need」這句 2026 年的精確版本是：「**attention 仍是不可或缺的那一塊，但不再是全部——你需要它來精確回憶，但不需要每一層都是它。**」

順帶一個冷笑話收尾這題：那句「Attention Is All You Need」的論文標題後來變成了一個迷因。有人去 arXiv 數，標題帶「all you need」的論文從 2009 到 2025 有七百多篇（2026-06 查證）；最早一篇其實在高能物理、比這篇早六年，而第一篇**故意**致敬 Vaswani 的，是一篇叫「CNN Is All You Need」的機器翻譯論文——用 CNN 來回敬「attention is all you need」，這個領域的幽默感大概就是這樣。所以下次有人跟你說「X is all you need」，你可以微笑：通常 X 都不是全部。

## 故障模式與直覺陷阱

收官章的故障視角，是把全書六個招牌陷阱列成一張「總帳」。這六個是讀者最容易帶著走、卻錯得最隱蔽的直覺。每一個我給「錯誤直覺」「會在哪一步把你帶溝裡」「正確版」。

| 錯誤直覺 | 會在哪裡害你 | 正確版（出自哪章） |
|---|---|---|
| 注意力是「找最相關的那一個」 | 你以為 attention 是 argmax，於是不懂為什麼 street 也有權重、不懂溫度在調什麼 | 它是**混合全部**（加權平均），soft 不是 hard。脊椎 it 對 street 有 0.245 的權重，不是 0（ch05） |
| softmax 永遠是好的、不會壞 | 你忘了它會**飽和**——分數一大就退化成接近 one-hot、梯度趨近 0、模型學不動 | 這正是為什麼要除以 √d（ch06）；也是為什麼後面要有殘差（ch13） |
| 位置是模型內建的、它「知道」順序 | 你會忽略置換不變性，搞不懂為什麼換個句子順序模型可能崩、為什麼要 RoPE | attention 對輸入**置換不變**，順序資訊**必須外加**（ch09）。集合沒有順序（ch19） |
| 注意力熱圖就是模型的「解釋」 | 你把相關性 dashboard 當因果根因，過度解讀熱圖、用它「證明」模型為何決策 | **未定論**（ch20）。忠實 vs 合理是兩件事；熱圖看起來合理 ≠ 它忠實反映了決策過程 |
| 高效注意力（稀疏/線性）是免費午餐 | 你以為換上線性注意力就 O(n) 又不損效果，結果精確回憶任務上吃癟 | 稀疏/線性是**近似**，省成本但常損 token 級精確回憶（ch15）。FlashAttention 才是精確（ch16） |
| 純 SSM/Mamba 已經取代了 attention | 你跟人說「Transformer 過時了」，結果生產級 LLM 全還在用 softmax | （2026-06）softmax 仍主導，**混合**是新共識，**純 SSM 未稱霸**，長期勝者未定論（ch22） |

這張表沒有新內容——它就是全書六個 `## 故障模式與直覺陷阱` 段落的濃縮。但我要你把它當成出師檢查表：**這六個你要是還會踩，代表對應那一章還沒讀透。** 它們的共同病根只有一個——**把「軟」當成「硬」、把「近似/趨勢」當成「確定」。** 注意力的美和它的陷阱，都來自那個 soft：softmax 是軟的 argmax、查找是軟的命中、解釋是軟的（未定論的）、未來是軟的（未定論的）。記住這一點，這六個陷阱你基本不會踩。

- **Vaswani et al. 2017「Attention Is All You Need」**（arXiv 1706.03762）：回到原點，重讀 §3.2（縮放點積與多頭）。讀完全書再讀它，你會發現這篇論文比你第一次看時清楚得多——因為現在每個符號你都有白話直覺在旁邊。這是全書脊椎機制（除了脊椎那個 it 的例句，那出自 Google 2017 部落格、非本論文）的唯一權威來源。
- **Jay Alammar「The Illustrated Transformer」**（jalammar.github.io/illustrated-transformer）：把這篇圖解當成「視覺版的本書」對照看——它普及了脊椎那個 it 的例子。讀完全書再看它的圖，你能指出每張圖對應本書哪一章、哪一步。
- **本書附錄 A（記號速查）／附錄 B（術語表）／附錄 C（延伸閱讀總地圖）**：收官後的常備工具。附錄 A 用來做「口述驗收」（每個記號你唸不唸得出來、講不講得出白話意義）；附錄 C 把各章延伸閱讀按「往理論／往實作／往前沿」三條路線整理好，給「如果只看三樣」的建議。
- **書架上的鄰書**：往運算深度走《矩陣是動詞》（內積/投影/SVD/特徵向量）；往訓練的數學走《馴服無限》（梯度）；往機率語義走《馴服隨機》（softmax 是分布、加權平均＝期望）；往 RoPE 的底層走《圓的影子》（旋轉/複數/e^{iθ}）；往「與工作相關」的部署走《從後端到 AI Infra》（KV cache、FlashAttention 系統面、推論物理）。這本書把直覺給足了，運算與系統的深度，在它們那裡等你。
