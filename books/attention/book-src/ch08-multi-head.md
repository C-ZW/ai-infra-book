# ch08 — 多頭注意力：為什麼要很多顆頭

> **本章解決什麼問題**：到 ch07 為止，我們有了完整的一次自注意力——每個 token 對全序列打分（ch04）、softmax 正規化（ch05）、除 √d 校正尺度（ch06）、整張 n×n 矩陣一次算完（ch07）。一切都對，但有個藏在 softmax 裡的限制沒人說破：一次注意力只能產出**一組**權重，也就是只能聚焦**一種**關係。it 想同時知道「誰是主詞」和「我這個代名詞指誰」這兩件不同的事，一組權重忙不過來。多頭注意力（multi-head attention）的全部用意就是這個——**並排跑很多次注意力，每一次在不同的子空間裡看一種關係，再把結果接起來**。本章只動三步骨架的一件事：把「打分→正規化→混合」這整套**複製成 h 份平行跑**，最後用一個投影把它們縫合。看懂它，你就明白為什麼真實 transformer 不是一顆大注意力、而是 8 顆、16 顆、64 顆小注意力並聯。

## 從你已知的出發

你查一筆資料時，常常不是只查一個維度。要找「某個使用者最近的高額訂單」，你不會只靠一個索引——你可能同時走 `user_id` 索引找出他的訂單、走 `created_at` 索引篩最近、走 `amount` 索引篩高額，三個索引各看資料的一個面向，最後把結果**交集／合併**起來。一個索引只擅長回答一種問題；要回答一個複合問題，你並排用好幾個專門化的索引。

多頭注意力是同一個套路。一個 query 想知道的往往不是單一關係。回到脊椎——序章那個句子：

> *"the animal didn't cross the street because **it** was too tired."*

代名詞 **it** 這個位置，到底想從別的詞身上得到什麼？至少兩件不同的事：

- **句法上**：這句話的主詞是誰？（animal 是 didn't cross 的主詞，street 只是 cross 的受詞）——這影響 it 該怎麼當代名詞用。
- **指代上**：it 指的是哪個先行詞？（語意上「累的」是動物不是街，所以 it→animal）——這是序章那個懸念本身。

這是**兩種不同的相關性**。「誰是主詞」靠的是句法位置的線索，「我指誰」靠的是語意合理性的線索。它們未必看同一批維度、未必給出同一組權重。一顆頭、一個 softmax，只能算出**一組**「該看哪裡」的權重——它要嘛偏句法、要嘛偏指代，沒辦法同時是兩者。就像你不能用同一個索引同時回答「最近的」和「最高額的」——除非你並排用兩個。

多頭注意力就是「並排用很多個」。它把表示空間切成幾塊（**子空間 subspace**），每一塊配一顆獨立的注意力頭，每顆頭在自己那塊空間裡跑完整的三步骨架、看一種關係，最後把各頭的結果**concat（接起來）**、再過一個投影縫合。在三步骨架裡，多頭不新增第四步——它是把整套三步**橫向複製 h 份**：

```text
                ┌─[頭 1: 打分→softmax→混合]─┐
 同一批         ├─[頭 2: 打分→softmax→混合]─┤
 token  ──────► ├─[頭 3: 打分→softmax→混合]─┤──► concat ──► W_O ──► 輸出
 的 Q/K/V       │           ⋮               │
                └─[頭 h: 打分→softmax→混合]─┘
                  每顆頭各有自己的 W_Q/W_K/W_V，在不同子空間看一種關係
```

口訣不變：相關性決定權重、權重決定混合。多頭只是說——**一種相關性不夠，我要同時算好幾種**。

## 一顆頭為什麼不夠：softmax 的單焦點限制

先把問題說精確。為什麼「一顆頭只能看一種關係」不是一句口號，而是 softmax 的數學性質？

回想 ch05–ch06：一顆頭對 it 這個 query，算出對 [animal, street, it] 三個 key 的分數，softmax 成一組權重——脊椎的正典是 **[0.665, 0.245, 0.090]**（縮放後）。這組權重是**一個機率分布**：非負、和為 1。它代表「我這次注意，把多少比例的目光分給每個位置」。

關鍵在於：**它只有一組**。一個分布，一個峰。當這組權重把 0.665 押在 animal 上，它就已經做出了選擇——它是一個「以語意/某種綜合相關性挑出 animal」的分布。如果同一個 it 還想要另一組權重（比方說一組純看句法、把目光押在「動詞 cross 的主詞位置」上的權重），這顆頭給不出來。它的輸出是**用這一組權重**對 value 做加權平均，混出來就是一個向量。一旦混完，「我是看句法看到 animal、還是看語意看到 animal」的區別就被平均掉了——你只拿到一個結果向量，看不出它是循哪條線索來的。

這正是原始論文那句話的意思。Vaswani 等人 2017 §3.2.2 寫得很直白（2026-06 覆核原文）：

> *"Multi-head attention allows the model to jointly attend to information from different representation subspaces at different positions. With a single attention head, averaging inhibits this."*

「**With a single attention head, averaging inhibits this**」——單一顆頭，那個加權**平均**的動作本身，會把多種子空間的資訊壓成一個，互相抵銷、糊在一起。注意這裡的 *averaging* 指的就是三步骨架第三步「加權混合」：一次混合只產出一個向量，多種關係被攤平成一個。

> **這裡了不起／反直覺在哪**：限制不在「頭不夠大」，而在 **softmax 只能輸出一個分布、混合只能產出一個向量**。你給一顆頭再多維度，它還是只能算一組權重、混出一個結果。要同時表達多種注意模式，唯一的辦法是**多算幾次**——不是把頭做大，是把頭做多。這就是為什麼解法叫 multi-**head**（多頭）而不是 bigger-head（大頭）。

工程類比：你有一個 `GROUP BY` 查詢，一次只能 group by 一個維度組合、得到一張匯總表。想同時要「按使用者匯總」和「按地區匯總」兩張表，你不會把一個 `GROUP BY` 寫得更複雜——你跑兩個查詢。多頭就是跑 h 個查詢，每個 group by 不同的東西。

## 機制：切子空間、平行跑、concat、過 W_O

把機制拆成四個動作，每個都對應你已經懂的東西。設模型的表示維度是 d_model，要切成 h 顆頭。

**動作一：每顆頭把 Q/K/V 投影到一個較小的子空間。** ch04 講過，Q/K/V 是 token 向量經學來的投影矩陣 W_Q/W_K/W_V 得到的。多頭只是讓**每顆頭各有一組自己的 W_Q^i/W_K^i/W_V^i**，把 d_model 維的輸入投影到較小的 **d_k = d_model / h** 維。原始 Transformer：d_model=512、h=8、所以 d_k = d_v = 512/8 = **64**（landscape §2）。每顆頭在自己這 64 維的子空間裡工作。

**動作二：每顆頭在自己的子空間裡跑完整的三步骨架。** 第 i 顆頭拿自己投影出的 Q^i、K^i、V^i，做一次標準的縮放點積注意力——打分（Q^i K^iᵀ）、除 √d_k、softmax、混合 V^i。注意縮放的分母是 **√d_k = √64 = 8**（不是 √d_model）——因為每顆頭實際工作的維度是 d_k，ch06 的變異數論證（內積變異數 = 維度）算的是這顆頭的子空間維度。每顆頭吐出一個 d_v = 64 維的輸出向量。

```text
head_i = Attention(Q W_Q^i,  K W_K^i,  V W_V^i)
       = softmax( (Q W_Q^i)(K W_K^i)ᵀ / √d_k ) · (V W_V^i)
         └────────────── 就是 ch04–ch06 那一整套，只是在第 i 個子空間裡 ──────────────┘
```

**動作三：把 h 顆頭的輸出 concat（接起來）。** h 顆頭各吐 d_v 維，接成一個 h × d_v = 8 × 64 = 512 維的長向量。concat 純粹是「把 8 段接成一段」，沒有任何計算，就像你把 8 個查詢結果拼進同一筆記錄的 8 個欄位。

**動作四：過一個輸出投影 W_O 縫合。** concat 出來的長向量再乘一個學來的矩陣 W_O，投回 d_model = 512 維。W_O 的作用是**讓各頭的資訊互相溝通、混成最終表示**——concat 只是把各頭並排放著，W_O 才決定「綜合這幾顆頭，最後寫進這個 token 的表示是什麼」。完整公式（Vaswani 2017 §3.2.2，2026-06 覆核）：

```text
MultiHead(Q, K, V) = Concat(head_1, …, head_h) · W_O
  其中 head_i = Attention(Q W_Q^i, K W_K^i, V W_V^i)

維度（原始 Transformer）：
  W_Q^i, W_K^i ∈ ℝ^(512 × 64)      ← 把 512 維投到 64 維子空間
  W_V^i        ∈ ℝ^(512 × 64)
  W_O          ∈ ℝ^(8·64 × 512) = ℝ^(512 × 512)   ← 把 concat 的 512 維縫回 512 維
```

在三步骨架的座標上：多頭把整套「打分→正規化→混合」複製成 h 份平行跑（動作一、二），再用 concat＋W_O 把 h 個結果縫成一個（動作三、四）。它**沒有**動三步骨架的任何一步的內部——每顆頭裡的打分、softmax、混合都和單頭一模一樣。它動的是**外層的並聯結構**。

### 一個漂亮的免費午餐：多頭幾乎不花額外成本

這是多頭設計裡最容易被忽略、卻最值得停下來算一次的事。直覺上你會以為「跑 8 次注意力＝8 倍成本、8 倍參數」。**錯，幾乎一樣。** 因為每顆頭的維度被砍到 d_k = d_model/h，剛好把「頭數變多」抵銷掉。

算參數量（先把 bias 項略過不計，抓主幹）。設維度 d。

```text
單頭、用滿 d_model 維：
  W_Q, W_K, W_V 各是 d × d，加上 W_O 是 d × d
  → 4 d²

多頭、h 顆頭、每顆 d_k = d/h：
  每顆頭 W_Q^i/W_K^i/W_V^i 各是 d × (d/h)，三個共 3·d·(d/h)
  h 顆頭：h × 3·d·(d/h) = 3 d²
  W_O 是 (h·d_v) × d = (h·d/h) × d = d × d = d²
  → 3d² + d² = 4 d²
```

**4d²，兩邊一模一樣。** 用原始數字代入（d=512、h=8、d_k=64）：兩種算法都是 4 × 512² = 1,048,576 個參數（我已用 Python 核對相等）。多頭不是「多花 8 倍買 8 顆頭」，而是「把同一筆參數預算，從一顆笨重的大頭，重新切成 8 顆各看一個面向的專門小頭」。同樣的道理，計算量（FLOPs）也大致不變：8 顆頭各在 64 維上跑、合起來等於一顆頭在 512 維上跑的量級。

> **這裡了不起在哪**：多頭幾乎是免費的——你用同樣的參數和算力，把「一種注意」換成「八種注意」。代價只有 concat 與 W_O 那一點點，而換來的是表達力的質變。這是深度學習裡少見的「結構上的好設計直接買到能力」的例子，不是靠堆更多參數硬換的。

## 紙上推演前的 worked example：脊椎的兩顆頭

把脊椎的 4 維拆成 **2 顆頭 × 2 維**，親手看兩顆頭怎麼各看一種關係。這是把 ch06 那一顆單頭，改成兩顆並聯。

**先聲明這是構造出來的玩具。** 真實模型的 W_Q/W_K/W_V 是訓練學出來的，我們沒有；這裡我**直接給每顆頭已經投影好的 2 維 q/k/v**，目的是讓你看清「兩顆頭給出兩組不同權重、各偏一種關係」的機制，不是宣稱真實模型恰好長這樣。維度 d_k = 2，所以每顆頭縮放的分母是 **√d_k = √2 ≈ 1.414**（ch06 的論證套在 2 維子空間上）。

脊椎 query 是 q_it=(1,1,1,1)（ch04 基準）。把它的前兩維給頭 A、後兩維給頭 B：q_it^A=(1,1)、q_it^B=(1,1)。三個 key 也各自有頭 A／頭 B 兩個 2 維分量（這就是「兩顆頭看不同子空間」的具體含義）：

```text
            頭 A 的 key（句法子空間）   頭 B 的 key（指代子空間）
  animal         (2,  1)                    (1,  1)
  street         (0,  1)                    (0, −1)
  it             (1, −1)                    (1,  0)
```

### 頭 A：抓「誰是主詞」

頭 A 的 query q^A=(1,1)，對三個 key 算內積（打分），再除 √d_k=√2：

```text
打分（內積 q^A · k^A）：
  animal: (1,1)·(2, 1) = 1·2 + 1·1 = 3
  street: (1,1)·(0, 1) = 1·0 + 1·1 = 1
  it    : (1,1)·(1,−1) = 1·1 + 1·(−1) = 0
                                       頭 A 原始分數 [3, 1, 0]

÷ √d_k = ÷√2 ≈ ÷1.414：
  [3/1.414,  1/1.414,  0] ≈ [2.121,  0.707,  0]
```

softmax（一步不跳；e 的次方取到三位）：

```text
e^2.121 ≈ 8.342
e^0.707 ≈ 2.028
e^0     = 1.000
                和 = 8.342 + 2.028 + 1.000 = 11.370

  animal: 8.342 / 11.370 = 0.734
  street: 2.028 / 11.370 = 0.178
  it    : 1.000 / 11.370 = 0.088
                                 頭 A 權重 [0.734, 0.178, 0.088]（核：和=1.000 ✓）
```

頭 A 把 **0.734** 的目光押在 animal——很果斷地認定「主詞是 animal」。

### 頭 B：抓「it 指代誰」

頭 B 用**另一個子空間**的 key（上表右欄），同樣的 query 分量 q^B=(1,1)，但因為 key 不同，打出的分數完全不同：

```text
打分（內積 q^B · k^B）：
  animal: (1,1)·(1, 1) = 1 + 1  =  2
  street: (1,1)·(0,−1) = 0 + (−1) = −1
  it    : (1,1)·(1, 0) = 1 + 0  =  1
                                       頭 B 原始分數 [2, −1, 1]

÷ √2 ≈ ÷1.414：
  [2/1.414,  −1/1.414,  1/1.414] ≈ [1.414,  −0.707,  0.707]
```

softmax：

```text
e^1.414  ≈ 4.113
e^−0.707 ≈ 0.493
e^0.707  ≈ 2.028
                和 = 4.113 + 0.493 + 2.028 = 6.634

  animal: 4.113 / 6.634 = 0.620
  street: 0.493 / 6.634 = 0.074
  it    : 2.028 / 6.634 = 0.306
                                 頭 B 權重 [0.620, 0.074, 0.306]（核：和=1.000 ✓）
```

頭 B 也偏 animal（0.620），但**權重的形狀完全不同**：它幾乎完全不看 street（0.074，因為 street 在指代子空間裡和 it 不相容，分數是負的），卻給了 it 自己不小的份（0.306）。對照頭 A 的 [0.734, 0.178, 0.088]——同樣是「偏 animal」，兩顆頭循著截然不同的線索得到結論，這正是「不同子空間看不同關係」。

> 誠實標示：兩顆頭最後都偏向 animal，是我刻意這樣構造的（脊椎的主線就是 it→animal）。真實模型裡，很多頭根本不指向 animal——有的看相鄰位置、有的看標點、有的看罕見詞（這是 ch20/ch21 的內容，Voita 2019、Clark 2019 查證過）。重點不是「每顆頭都看 animal」，而是「**每顆頭算出一組獨立的權重、看一種關係**」。

### concat 與 W_O：把兩顆頭縫起來

每顆頭用自己的權重對自己的 value 做加權混合。給兩顆頭都用脊椎 value 的 2 維版 v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)（為了和全書 value 一致；真實模型每頭有自己的 W_V）：

```text
頭 A 輸出 = 0.734·(1,0) + 0.178·(0,1) + 0.088·(1,1)
         = (0.734+0.088,  0.178+0.088) = (0.822, 0.266)

頭 B 輸出 = 0.620·(1,0) + 0.074·(0,1) + 0.306·(1,1)
         = (0.620+0.306,  0.074+0.306) = (0.926, 0.380)
```

concat（接起來，零計算）：

```text
Concat(頭A, 頭B) = (0.822, 0.266,  0.926, 0.380)    ← 4 維
                    └─頭A的2維─┘  └─頭B的2維─┘
```

最後過 W_O（在脊椎玩具裡 d_model=4，concat 也是 4 維；為了看清結構，我們先取 W_O = 單位矩陣 I，即「不混、直接通過」）：

```text
MultiHead 輸出 = Concat(頭A, 頭B) · W_O = (0.822, 0.266, 0.926, 0.380)
                                          （W_O=I 時原樣通過）
```

對照 ch06 的單頭正典輸出 **(0.755, 0.335)**——那是一個 2 維向量、混的是「一種綜合相關性」。多頭給出的是一個 **4 維**向量，前兩維帶著「頭 A 怎麼看主詞」、後兩維帶著「頭 B 怎麼看指代」。同一個 it，現在它的表示同時編碼了兩條線索，而不是被攤平成一個。真實模型的 W_O 不會是單位矩陣——它會把這 4 維重新線性組合，讓兩顆頭的資訊交流、產出最終要寫進 it 這個位置的表示。但 concat 已經保住了關鍵：兩種關係沒被互相平均掉。

如果你想看演算法骨架，多頭就是「對每顆頭跑一次 ch06 那五行、再 concat」：

```python
import numpy as np

def multi_head(Q, K, V, WQ, WK, WV, WO, h):
    # WQ/WK/WV: lists of per-head projection matrices; WO: output projection
    heads = []
    for i in range(h):
        Qi, Ki, Vi = Q @ WQ[i], K @ WK[i], V @ WV[i]   # project into head i's subspace
        heads.append(scaled_dot_product_attention(Qi, Ki, Vi))  # ch06's five lines, per head
    return np.concatenate(heads, axis=-1) @ WO          # concat heads, then mix with W_O
```

每顆頭裡的 `scaled_dot_product_attention` 就是 ch06 那個函式，原封不動。多頭不過是「迴圈跑 h 次、concat、過 W_O」。程式和上面手算對得上：餵進脊椎兩顆頭的投影，concat 出的就是 (0.822, 0.266, 0.926, 0.380)。

## 為什麼是「不同子空間」而不是「同一空間跑很多次」

有個自然的疑問：如果只是想要多組權重，何必把維度切小、各給不同投影？直接在**同一個 d_model 維空間**裡跑 h 次注意力、每次用不同的 W_Q/W_K/W_V，不也能得到 h 組權重嗎？

能，但那樣會貴 h 倍（每顆頭都用滿 d_model 維，參數和算力都是 h 倍），而且實證上沒必要。多頭的設計精髓正是「**切子空間**」這個取捨：

- **把 d_model 切成 h 塊**，每顆頭只在自己那塊 d_k = d_model/h 維裡工作。這逼著每顆頭**專門化**——它沒有足夠維度去同時表達所有關係，只能挑一個面向學好。約束反而是好事：它讓不同頭自然分工，而不是 h 顆頭學成一模一樣。
- **總成本守恆**（前面算過的 4d²）。切子空間讓「頭變多」和「每頭變小」精確抵銷，所以你用一份預算買到多種注意。

換句話說，多頭不是「把注意力做 h 遍」，而是「把一個 d_model 維的注意力，**重新切分**成 h 個低維注意力的並聯」。維度沒有變多，只是被重新分配給了不同的專門功能。這和你做容量規劃時「與其開一台超大機器跑所有服務，不如切成幾台各跑專門服務」是同一種智慧——切分帶來專門化，總資源不變。

至於每顆頭實際學到了什麼關係（句法頭、指代頭、位置頭、罕見詞頭…），那是訓練的結果、也是可解釋性的研究對象。Voita 等人 2019（「Analyzing Multi-Head Self-Attention」，arXiv 1905.09418，landscape §6 查證）發現：訓練好的模型裡，少數「重要的頭」扛起大部分工作、且常有可辨認的語言學角色（位置、句法依存、罕見詞），其餘大量頭可以剪掉——他們在英俄 WMT 上把 48 顆 encoder 頭剪掉 38 顆，BLEU 只掉 0.15。這既證明了「頭確實會專門化分工」，也預告了一個張力：頭很多，但不是每顆都同等重要（ch20 會深入「頭是否可剪、是否可解釋」，ch21 會把單顆頭拆成 QK/OV 電路）。本章只需要你抓住機制：**多頭＝多種專門化的注意並聯**。

## 故障模式與直覺陷阱

| 直覺陷阱 / 故障 | 為什麼錯／在哪一步把你帶溝裡 | 怎麼自我察覺、正確版是什麼 |
|---|---|---|
| 「多頭＝把一顆大注意力跑 h 遍，所以貴 h 倍」 | 算錯成本。每顆頭的維度被砍到 d_k = d_model/h，總參數仍是 4d²、算力也大致不變。多頭是**重新切分**一份預算，不是加倍買。 | 自己算一次參數：單頭 4d²、多頭 3d²(投影)+d²(W_O)=4d²。相等。多頭幾乎免費。 |
| 「每顆頭都看同一個地方（都指向 animal）」 | 把脊椎玩具當通則。我刻意讓兩顆頭都偏 animal 是為了配合主線；真實模型多數頭看別處（位置、標點、罕見詞，Voita 2019）。 | 記住重點是「每顆頭算**一組獨立權重、看一種關係**」，不是「都看同一處」。看真實模型的注意力熱圖會發現各頭形狀差很多。 |
| 「頭越多一定越好」 | 邊際遞減且有冗餘。Voita 2019：48 頭剪到 10 頭只損 0.15 BLEU——大量頭是冗餘的。頭數是個要權衡的超參數，不是越多越強。 | 別把「h 大」當銀彈。h 是設計選擇（原始 h=8）；過多頭只是浪費、不會等比例變強。 |
| 「concat 會把各頭的資訊混在一起」 | concat **不混**，它只是把 h 段並排接成一段（零計算）。真正讓各頭資訊交流的是後面的 **W_O**。 | 分清楚：concat＝拼接（保留各頭獨立），W_O＝縫合（讓各頭溝通）。少了 W_O，各頭就只是並排堆著。 |
| 「縮放要除 √d_model」 | 在多頭裡縮放的分母是 **√d_k**（每顆頭的子空間維度），不是 √d_model。ch06 的變異數論證算的是這顆頭實際工作的維度。 | 原始 Transformer：d_k=64 → √d_k=8（不是 √512≈22.6）。每顆頭在自己的 d_k 維裡縮放。 |
| 「多頭解決了一切，所以一層只要多頭就夠」 | 多頭給的是「多種注意」，但純堆注意力（即使多頭）會**秩坍縮**（Dong 2021，ch13/ch19）——輸出退化。還需要殘差與 FFN。 | 多頭是一層裡的一個零件，不是全部。ch13 會把多頭裝進完整的一層 block（配殘差、LayerNorm、FFN）。 |

**和後面章節的鉤子**：多頭讓「同一個 it 同時被多種關係看」，但每顆頭看到的權重形狀差很大，這直接引出兩個後續問題——(1) 這些頭各自在看什麼？某些頭恰好對應句法/指代等可辨認角色（ch20 的可解釋性論戰：頭有角色，但這算不算「解釋」？）；(2) 怎麼把一顆頭的職責拆得更乾淨？Anthropic 把每顆頭拆成 **QK 電路**（決定看哪裡）與 **OV 電路**（決定搬什麼），脊椎的「it→animal 指代頭」會在 ch21 被畫成一個具體電路。本章先把「多頭在分什麼工」這層直覺打穩。

## 紙上推演

### 推演題

**第 1 題 [12 分鐘]（★）** 親手把脊椎兩顆頭算完整。給頭 A 的 key（animal=(2,1)、street=(0,1)、it=(1,−1)）、頭 B 的 key（animal=(1,1)、street=(0,−1)、it=(1,0)），兩顆頭的 query 都是 (1,1)，d_k=2、√d_k=√2≈1.414：(a) 算頭 A 的原始分數、除 √2、softmax 得權重；(b) 對頭 B 做同樣三步；(c) 用 value v_animal=(1,0)、v_street=(0,1)、v_it=(1,1) 算兩顆頭各自的輸出向量；(d) concat 成最終 4 維向量（取 W_O=I）。一句話說：兩顆頭都偏 animal，但它們的**權重形狀**哪裡不同、各自在說什麼。

**第 2 題 [10 分鐘]（★★）** 用 softmax 的性質解釋「為什麼一顆頭的 softmax 限制了它只能聚焦一種模式」。提示：(a) 一顆頭的輸出是用「一組」權重對 value 做的「一次」加權平均；(b) 這組權重是一個機率分布（和為 1）；(c) 引用 Vaswani 2017「averaging inhibits this」這句話，說清楚「averaging」指三步骨架的哪一步、它抹掉了什麼。

**第 3 題 [12 分鐘]（★★）** 手算把 d=4 拆成 2 顆頭後的維度與參數量，並驗證「多頭幾乎不花額外成本」。(a) 每顆頭的 d_k 是多少？(b) 每顆頭的 W_Q^i 是幾乘幾的矩陣、多少參數？三個投影（W_Q/W_K/W_V）一顆頭共多少？兩顆頭共多少？(c) W_O 是幾乘幾、多少參數？(d) 多頭總參數量是多少？對照「單頭用滿 d=4」的 4d² 是多少？相等嗎？

**第 4 題 [8 分鐘]（★★）** 設計題。針對脊椎句子 *"the animal didn't cross the street because it was too tired"*，舉出**三種**你會希望模型擁有的「頭」，各說明它該看哪種關係、對 it 這個位置該把權重押在哪裡。提示：想想句法（主詞/受詞）、指代（先行詞）、位置（相鄰詞）、語意角色（誰累了）這些不同面向。

### 推演解答

**第 1 題。**
(a) 頭 A 原始分數：(1,1)·(2,1)=3、(1,1)·(0,1)=1、(1,1)·(1,−1)=0 → **[3, 1, 0]**；÷√2 → [2.121, 0.707, 0]；softmax：e^2.121≈8.342、e^0.707≈2.028、e^0=1.000，和=11.370 → **[0.734, 0.178, 0.088]**（和=1.000 ✓）。
(b) 頭 B 原始分數：(1,1)·(1,1)=2、(1,1)·(0,−1)=−1、(1,1)·(1,0)=1 → **[2, −1, 1]**；÷√2 → [1.414, −0.707, 0.707]；softmax：e^1.414≈4.113、e^−0.707≈0.493、e^0.707≈2.028，和=6.634 → **[0.620, 0.074, 0.306]**（和=1.000 ✓）。
(c) 頭 A 輸出 = 0.734·(1,0)+0.178·(0,1)+0.088·(1,1) = **(0.822, 0.266)**；頭 B 輸出 = 0.620·(1,0)+0.074·(0,1)+0.306·(1,1) = **(0.926, 0.380)**。
(d) concat = **(0.822, 0.266, 0.926, 0.380)**（W_O=I 原樣通過）。
形狀差異：兩顆頭都把最大權重給 animal（0.734 vs 0.620），但頭 A 把次多的份給了 street（0.178），幾乎不理 it（0.088）——像在「比較三個名詞誰是主詞」；頭 B 幾乎不看 street（0.074，因為在它的子空間裡 street 與 it 不相容、分數為負），反而給了 it 自己 0.306——像在「以 it 自身為基準找最相容的先行詞」。同一個偏向、兩條不同線索，這就是「不同子空間看不同關係」。常見錯路：以為兩顆頭只是把同一組分數算兩遍——不是，key 不同（不同子空間），分數從根本就不同。

**第 2 題。**
(a) 一顆頭跑完三步骨架，輸出是「**用這一組** softmax 權重對 value 做的**一次**加權平均」，結果是**一個**向量。
(b) 這組權重 softmax 出來是一個機率分布（非負、和為 1），它已經把「目光怎麼分配」鎖死成一種分配方式——一個峰、一種焦點。想要另一種焦點（另一組權重），這顆頭給不出第二組。
(c) Vaswani 2017 的「averaging」指的就是三步骨架第三步「**加權混合**」。一次混合把多種潛在的關係（句法的、指代的…）攤平成一個輸出向量，彼此抵銷、無法區分——「averaging inhibits this」。要保住多種關係，唯一辦法是先**分頭算出多組權重、多個混合結果**，最後才 concat 縫合——這正是多頭。

**第 3 題。**（d=4、h=2）
(a) d_k = d_model/h = 4/2 = **2**。
(b) W_Q^i 是 d_model × d_k = 4 × 2 = **8** 個參數；三個投影（W_Q/W_K/W_V）一顆頭共 3×8 = 24；兩顆頭共 2×24 = **48**。
(c) W_O 是 (h·d_v) × d_model = (2×2) × 4 = 4 × 4 = **16** 個參數。
(d) 多頭總計 48 + 16 = **64**。單頭用滿 d=4 是 4d² = 4×16 = **64**。**相等** ✓。多頭把同一筆 64 個參數，從「一顆 4 維大頭」重切成「兩顆 2 維小頭＋一個縫合矩陣」，能力變強、成本不變。

**第 4 題（範例答案，合理即可）。**
- **句法/主詞頭**：看「誰是動詞 cross/tired 的主詞」。對 it 該把權重押在 animal（主詞）多於 street（cross 的受詞）。
- **指代/共指頭**：看「代名詞 it 的先行詞是哪個名詞」。靠語意合理性（「累的」配「動物」），權重押 animal。
- **相鄰/位置頭**：看「我左邊緊鄰的詞是什麼」。對 it 可能押在 because 或 tired 上——這種頭不解指代，但提供局部上下文（ch09 會講位置，ch20/Voita 提到確有「位置頭」）。
（其他合理答案：標點頭、罕見詞頭、語意角色頭等。關鍵是每顆頭看**一種**關係。）

## 自我檢核

口頭自答，講得出來才算過：

1. 用一句話說多頭在三步骨架的哪個位置、動了什麼（提示：把整套「打分→正規化→混合」橫向複製 h 份平行跑，再 concat＋W_O 縫合；不動任何一步的內部）。
2. 為什麼「一顆頭只能看一種關係」？把它連到「softmax 只輸出一個分布、混合只產出一個向量、averaging inhibits this」。
3. 多頭的四個動作各是什麼（切子空間投影 → 各頭跑三步骨架 → concat → 過 W_O），其中哪一步是零計算的拼接、哪一步才讓各頭資訊交流？
4. 為什麼說「多頭幾乎不花額外成本」？把參數量算給另一個工程師聽：單頭 4d² vs 多頭 3d²+d²。
5. 多頭裡縮放除的是 √d_k 還是 √d_model？為什麼？（每顆頭實際工作維度是 d_k，ch06 變異數論證算的是子空間維度。）
6. 為什麼要「切子空間」而不是「在同一個 d_model 維空間裡跑 h 次」？（切子空間逼出專門化、且總成本守恆。）
7. Voita 2019「48 頭剪 38 只損 0.15 BLEU」說明了關於多頭的什麼兩件事？（頭會專門化分工；但頭有冗餘、不是每顆同等重要——頭數是要權衡的超參數。）
8. concat 和 W_O 的分工是什麼？少了 W_O 會怎樣？（concat 並排保留各頭獨立、W_O 縫合讓各頭溝通；少 W_O 各頭就只是堆著、無法綜合。）
