# ch21 — 機制可解釋性：QK/OV 電路與 induction heads

> **本章解決什麼問題**：ch20 把「attention 權重是不是解釋」這場嘴仗打到「未定論」就收手——熱圖告訴你頭**看哪裡**，但不告訴你模型**為什麼**這樣算。本章換一條路：別問權重，去拆**電路**。Anthropic 2021 把一顆注意力頭拆成兩個各自能讀懂的零件——**QK 電路**（決定看哪裡）與 **OV 電路**（決定把什麼搬過去）；2022 年他們用這副工具找到了 **induction head**，一個學會「上次這個 pattern 後面接什麼、就照抄」的小程式，而它的出現時機正好對上模型「會 in-context learning」的那一刻。本章不動三步骨架的任何一步——QK 是它的「打分＋正規化」、OV 是它的「加權混合」，我們只是把這兩步**從一顆頭裡拆開、分別讀懂在算什麼**。最後把脊椎那條 it→animal 畫成一個具體的 QK/OV 電路收尾。

## 從你已知的出發

你逆向工程過別人的程式。沒有文件，只有一個黑盒函式，你得搞清楚它在幹嘛。你不會盯著它的 output 猜——那是 ch20 那種「看熱圖猜模型在想什麼」的層次，靠不住。你會做的是把它**拆成模組**：這段在算位址、那段在搬資料，這個分支決定走哪條路、那個迴圈在累加。每一塊單獨拿出來，你都能說出它在做什麼；拼回去，整個函式就不再是黑盒。

機制可解釋性（mechanistic interpretability）就是把這套逆向工程的紀律，用到訓練出來的神經網路上。它和 ch20 的差別，正是「看 log 猜根因」與「拆原始碼讀邏輯」的差別：

```text
ch20 注意力權重當解釋          ch21 機制可解釋性
──────────────────────         ─────────────────────────
看「頭把權重放哪」              拆「頭的權重矩陣在算什麼函數」
相關性層次（看哪裡）           機制層次（為什麼這樣看、搬了什麼）
熱圖：一張 dashboard           電路：一段可讀的反組譯
未定論：這算不算解釋           可驗證：剪掉它、in-context learning 就垮
```

CPU 你熟。一顆 CPU 取一個資料，做的是兩件可以分開講的事：**定址（addressing）**——算出「要去記憶體的哪一格拿」；**取數（fetch/move）**——把那一格的內容搬進暫存器。定址邏輯和搬運邏輯是兩套電路，你可以分開讀懂、分開驗證。

Anthropic 2021 的關鍵洞見就是：**一顆注意力頭，骨子裡也是「定址」加「搬運」這兩套可以拆開的電路。** 定址那套叫 QK 電路（決定看序列的哪一格），搬運那套叫 OV 電路（決定從那一格搬什麼內容過來）。本章要把這兩套拆給你看，然後給你看一個用它們組裝出來的、真實存在於模型裡的小程式：induction head。

> 一句性質標示（照 landscape §6）：本章主要依據 **Anthropic 的 Transformer Circuits Thread**——Elhage et al. 2021「A Mathematical Framework for Transformer Circuits」（2021-12-22）與 Olsson et al. 2022「In-context Learning and Induction Heads」（arXiv 2209.11895）。這條線**不是傳統同儕評審期刊**，但在機制可解釋性領域影響極大（2026-06 查證）。引用時我會標清楚哪些是它們的實證發現、哪些只是我給你的直覺講法。

## 把一顆頭拆成兩半：QK 決定看哪裡，OV 決定搬什麼

回到 ch04。一顆注意力頭裡，有三個學來的投影矩陣 W_Q、W_K、W_V，加上多頭那一章（ch08）出場的輸出投影 W_O。一顆頭算一次注意力，動到的就是這四個矩陣。Elhage 2021 的拆法是：**這四個矩陣不是各自獨立的四件事，它們兩兩配對，組成兩個有獨立意義的「電路」。**

第一個配對決定**注意力模式**——也就是 softmax 出來的那排權重「把注意力放在哪幾格」。算這排權重時，query 經過 W_Q、key 經過 W_K，然後算內積。兩個投影夾著一次內積，可以併成一個矩陣：

```text
分數(i,j) = (xᵢ W_Q) · (xⱼ W_K)ᵀ = xᵢ (W_Q W_Kᵀ) xⱼᵀ
                                        └────┬────┘
                                      QK 電路 W_QK = W_Q W_Kᵀ
```

這個 **W_QK = W_Q W_Kᵀ** 就是 **QK 電路**（QK circuit）。它是一個直接吃「位置 i 的 token」和「位置 j 的 token」、吐出「i 該不該看 j」分數的東西——**它就是『定址邏輯』**。注意 W_Q 和 W_K 從來不會單獨出現去影響別人，它們永遠以這個乘積的形式一起作用在分數上。所以與其分別研究 W_Q、W_K（各自沒有獨立意義），不如直接研究它們的乘積 W_QK——這才是「決定看哪裡」的那個函數。

第二個配對決定**內容搬運**——一旦 QK 電路決定了「看位置 j」，要從 j 搬什麼過來、寫進 i 的表示裡？算這個時，被看的 token 經過 W_V 變成 value，加權混合後再經過 W_O 投影回去寫進殘差流（residual stream，ch13 的那條主幹）。兩個投影夾著中間，也可以併成一個矩陣：

```text
從 j 搬給 i 的內容 = (xⱼ W_V) W_O = xⱼ (W_V W_O)
                                       └────┬────┘
                                     OV 電路 W_OV = W_V W_O
```

這個 **W_OV = W_V W_O** 就是 **OV 電路**（OV circuit）。它吃「被看到的那個 token」，吐出「要寫進當前位置的內容更新」——**它就是『搬運/取數邏輯』**。它完全不管「該看誰」（那是 QK 的事），它只管「看到了，那搬什麼過去」。

把兩半擺在一起對到三步骨架，你會發現拆得乾乾淨淨：

```text
            一顆注意力頭
            ┌────────────────────────────────────────────
   QK 電路   │  打分 + 正規化（決定權重 α）「該看哪裡」
   W_Q W_Kᵀ │      → 三步骨架的第 1、2 步（score → softmax）
            │
            ▼   權重 α 決定混合比例
            │
   OV 電路   │  加權混合（用 α 混 value、寫回殘差）「搬什麼」
   W_V W_O   │      → 三步骨架的第 3 步（weighted mix）
            └────────────────────────────────────────────
```

這是本章第一個值得停下來的點。**QK 電路就是三步骨架的「打分→正規化」，OV 電路就是「加權混合」。** ch20 那場嘴仗之所以僵在那裡，有一部分原因是大家盯著「權重 α」吵——但 α 只是 QK 電路的**輸出**，它本身不是機制。真正的機制藏在兩個矩陣裡：QK 是「依什麼規則決定看哪」，OV 是「看到了搬什麼」。把它們分開，你問的問題從「這個權重對不對」升級成「這顆頭實作了什麼**演算法**」。

> 嚴謹度標示。我這裡寫 `W_QK = W_Q W_Kᵀ`、`W_OV = W_V W_O`，是把 Elhage 2021 的核心拆解講成「夠用的直覺」。論文的完整版用的是更乾淨的張量記號（每顆頭對殘差流是一個低秩的線性／雙線性運算，QK 是雙線性、OV 是線性），還處理了多頭、層之間的合成、以及把 embedding/unembedding 也納進來的端到端電路。本章只取「一顆頭＝QK 電路＋OV 電路、兩者可分開讀懂」這個主結論；完整的電路代數與張量分解（virtual heads、Q/K/V-composition）指向延伸閱讀，不在這裡展開。乘積矩陣 W_QK、W_OV 通常是**低秩**的（因為 d_k < d_model，ch08），低秩的直覺可回扣《矩陣是動詞》ch19–20。

## 為什麼非拆不可：一個跨層的小程式藏在「合成」裡

你可能會問：拆成兩半，圖好看，但有什麼實際用處？用處是它讓你能看懂**跨層的合成**——而 attention 真正有趣的計算，幾乎都不在單一顆頭裡，而在「兩顆頭接力」裡。

Elhage 2021 先從最簡單的模型熱身：**只有一層、只有 attention（沒有 FFN）的 transformer**。他們證明這種模型基本上等於一堆「skip-trigram」的集合——形如「A … B → C」的規則：看到 B，回頭看到前面有個 A，就調高下一個字是 C 的機率（2026-06 查證，照 framework 論文）。一層頭能做的就這麼多：QK 電路挑出「該往回看哪個 token」，OV 電路根據看到的 token 調整輸出機率。**單層頭只能看『當前 token 本身』來決定往回看誰**——它沒有辦法「根據前一個 token 是什麼」來定址，因為它手上只有當前位置的內容。

這個限制很要命。語言裡最有用的模式是「**重複**」：如果序列前面出現過 `... Harry Potter ...`，現在又出現了 `Harry`，那下一個字極可能又是 `Potter`。要做這件事，模型在 `Harry` 這個位置得問的是：「上次 `Harry` 出現時，**它後面**接的是什麼？」——這需要「根據前一個 token 來往回找」，單層頭做不到。

於是 induction head 登場，而它的關鍵正是**兩層頭的合成**——這就是為什麼非把頭拆成 QK/OV、再研究電路如何接力不可。

## induction head：學出來的「照抄上次的下一個」

**induction head**（Olsson et al. 2022，照 landscape §6）是一顆注意力頭，它在序列裡執行這個補全：

```text
[A][B] ...... [A] → [B]
 │   │          │     │
 看過 A 後面是 B   又看到 A   就預測：下一個還是 B
```

白話講：**「我上次見到這個 token 時，它後面接什麼？把那個照搬過來當預測。」** 你寫過模式比對、寫過自動補全規則；induction head 就是這條規則，但**不是你手寫的，是模型自己從資料裡學出來的**。它不是查一張寫死的表，它是一顆會泛化的頭——`[A][B]…[A]→[B]` 對任意 token A、B 都成立，甚至 A、B 是模型訓練時沒見過的新詞、隨機字串都行（這點等下很關鍵）。

它怎麼用兩層頭做出來？這是本章的機制核心，慢慢拆（2026-06 查證，照 Olsson 2022 對標準 induction 電路的描述）：

**第一層：previous-token head（前一 token 頭）。** 這顆頭很笨但很有用——它的 QK 電路學到的規則是「永遠看我**左邊那一格**」（位置 i 看 i−1）；它的 OV 電路把**前一個 token 的身分**搬到當前位置。效果是：每個位置現在除了自己，還在殘差流裡攜帶了一份「我前面那個 token 是誰」的資訊。

**第二層：induction head 本身。** 現在到了第二個 `A`（當前位置）。這顆頭的 QK 電路要做 **prefix matching（前綴比對）**：拿「當前 token A」去跟序列裡每個位置比，但比的不是那個位置的 token 本身，而是**那個位置攜帶的『我前一個 token 是誰』**——而那份資訊，正是第一層 previous-token head 幫忙搬好的。所以：

```text
當前位置 = 第二個 A
QK 電路問：序列裡哪個位置「它的前一個 token」== A？
答：第一個 B 的位置（因為 B 的前一個 token 就是第一個 A）
→ 注意力指向第一個 B
```

注意力鎖定第一個 B 之後，**OV 電路做 copying（複製）**：把第一個 B 的身分搬到當前位置、寫進殘差流，效果是調高「下一個 token = B」的機率。

把兩層串起來看，這就是一個漂亮的小程式：

```text
位置:      1    2    3    4    5    6
token:     A    B    x    y    A    ?
                                    │
 第一層 previous-token head 已讓每格帶上「前一格是誰」：
   位置2(B) 帶著「我前面是 A」；位置6 帶著「我前面是 A」…

 第二層 induction head 在位置6（當前是 A，要預測 ?）：
   QK 電路：找「前一格 == 當前 token A」的位置 → 命中位置2(B)
            └─ prefix matching：比的是「前綴」，不是 token 本身
   OV 電路：把位置2 的 token（B）複製過來 → 預測 ? = B
            └─ copying：抬高 logit(B)
```

這就是為什麼一定要先把頭拆成 QK/OV、再研究跨層合成：**induction 不是任何單一電路做的事，它是「第一層的 OV 把前一 token 搬好」與「第二層的 QK 拿它來比對」這兩個電路接力的產物。** 盯著權重熱圖你永遠看不出這個結構（你只會看到「第二層那顆頭把權重放在某個早先位置」），但拆成電路、追蹤殘差流裡搬了什麼，這個跨層的小程式就現形了。Elhage 2021 把這種「一層的輸出當另一層的輸入」叫 **composition（合成）**，induction head 用到的正是 **K-composition**——前一層的輸出影響了後一層 QK 電路裡 key 那一側（2026-06 查證，照 framework 論文；機制細節指向延伸閱讀）。

## 為什麼這顆頭重要：它是 in-context learning 的機制源頭

到此為止 induction head 只是一個「會照抄重複 pattern」的巧妙電路。它之所以是這一章、甚至這整個領域的招牌，是 Olsson 2022 的第二個發現——它和**大模型最神奇的能力之一**綁在一起。

**in-context learning（脈絡內學習，ICL）**：你不重新訓練模型，只在 prompt 裡給它幾個例子，它就「現學現賣」。給它 `海→sea, 山→mountain, 河→river, 雲→?`，它接 `cloud`。模型的權重一個字節都沒變，它純粹靠**讀當前這段脈絡**就學會了「中翻英」這個臨時規則。這是 GPT-3 以降「few-shot prompting」之所以能成立的根。

Olsson 2022 的核心實證（照 landscape §6，2026-06 查證）：**訓練過程中存在一個「phase change（相變）」**——在訓練早期某個點，loss 曲線會出現一個小小的「鼓包（bump）」，而就在這個窗口裡，兩件事**同時**發生：

1. 模型的 in-context learning 能力**突然大幅變強**（給它脈絡、它開始會現學）。
2. 模型裡**形成了 induction heads**。

```text
訓練步數 ──────────────────────────────────►
                    ╱╲  ← loss 曲線在這裡出現小鼓包（phase change）
  loss  ───────────╱  ╲___________________
                   │
                   ├── 此窗口內：induction heads 形成
                   └── 同時：in-context learning 能力突增
```

「同時發生」只是相關，不是因果。Olsson 2022 為了把這條因果坐實，給了六條互補證據（2026-06 查證，這裡點名幾條代表性的）：相變的**共現**（兩件事卡在同一個訓練窗口）、架構**共擾動**（動架構讓 induction head 早一點/晚一點形成，in-context learning 變強的時點也跟著移）、以及**直接消融（ablation）**——在測試時把 induction heads 敲掉，in-context learning 能力大幅下降。多條獨立的線都指向同一個結論：**induction heads 是 in-context learning 這個能力的機制來源**（至少是最主要的來源之一）。

這是本章最值得你停下來的一頁，因為它示範了機制可解釋性的全部價值。**一個你以為很玄的高層能力（模型怎麼會「現學」），被追溯到一個你能在紙上畫出來、能手動敲掉、能驗證因果的具體電路。** ch20 那場「權重算不算解釋」的嘴仗之所以僵住，是因為它停在「相關性」層次——熱圖看起來合理，但無法證明模型**真的據此計算**。induction head 這條線往前推了一大步：它不只說「這顆頭看起來在做 pattern matching」，它說「敲掉這顆頭，這個能力就消失」——這是**忠實性（faithfulness）**的證據，不只是合理性（plausibility）。注意我說「推了一大步」不是「結案」——這是對**特定能力**（ICL 中可由複製式 pattern 解釋的部分）找到了機制，不是宣稱整個模型都被解釋了（這個邊界等下在故障模式裡會再框一次）。

> 誠實標時點與性質（2026-06）。induction head 與 in-context learning 的關聯是 Olsson 2022 的實證發現，主要在小到中型、attention-only 與小型完整模型上做的；它**解釋了 ICL 的一大塊**，但「ICL 全部都是 induction head 幹的」是過度延伸——後續研究指出更大模型有更複雜、更抽象的 in-context 機制（fuzzy/泛化的 induction、其他電路）。本章寫「機制源頭之一、奠基性發現」，不寫「ICL 就等於 induction head」。

## 把脊椎畫成一個 QK/OV 電路

現在回到 ch01 那個 it，用本章的工具把它畫成電路。脊椎句子 *"the animal didn't cross the street because **it** was too tired"*，簡化序列 [animal, street, it]（這個代名詞消解的例子**出自 Google AI 2017-08 部落格、非原始論文**，經 Illustrated Transformer 普及；2026-06 查證）。

我們關心的那條連結是 **it → animal**：it 這個位置該去看 animal，並把 animal 的內容搬過來。用 QK/OV 的語言講，這條連結拆成兩半：

- **QK 電路**（定址）：在 it 的位置上，問「我這個代名詞該指向誰」，把注意力放到 animal。
- **OV 電路**（搬運）：注意力鎖定 animal 後，把 animal 攜帶的內容（它是個會累的生物）搬進 it 的表示裡。

先把 QK 那一半用脊椎數字跑一次——這正是 ch04–06 算過無數遍的那組，現在我們明確說「這就是 QK 電路在算的東西」。q_it 對三個 key 打分（ch04 的內積，原樣搬來，每個我都重算一遍確認）：

```text
QK 電路的打分（內積）：
q_it · k_animal = (1)(2)+(1)(1)+(1)(1)+(1)(0) = 4
q_it · k_street = (1)(0)+(1)(1)+(1)(1)+(1)(0) = 2
q_it · k_it     = (1)(1)+(1)(-1)+(1)(0)+(1)(0) = 0
raw scores = [animal 4, street 2, it 0]
```

縮放（除以 √d=2，ch06）再 softmax（這步全書算了無數次，再算一次：e²=7.389、e¹=2.71828、e⁰=1，和=11.107）：

```text
/√d=2:  [2, 1, 0]
softmax([2,1,0]) = [7.389/11.107, 2.718/11.107, 1/11.107]
                 = [animal 0.665, street 0.245, it 0.090]   ← 全書正典脊椎權重
```

**這 [0.665, 0.245, 0.090] 就是 QK 電路的輸出**——「該看哪裡」的那排權重，animal 最濃。OV 電路接手，用這排權重去混 value（v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)，這是 OV 電路要搬的「內容」），混出寫回 it 位置的更新：

```text
OV 電路的搬運（加權混合 value）：
輸出 = 0.665·(1,0) + 0.245·(0,1) + 0.090·(1,1)
     = (0.665+0.090, 0.245+0.090)
     = (0.755, 0.335)   ← 偏向 v_animal，全書正典輸出
```

畫成電路圖，兩半各司其職：

```text
   it 位置的 query (q_it)
            │
            ▼
   ┌─── QK 電路（定址：該看哪裡）
   │   q_it 對每個 key 打分 → 縮放 → softmax
   │      animal: 4 → 2 → 0.665  ◄── 最濃，「指代對象在這」
   │      street: 2 → 1 → 0.245
   │      it    : 0 → 0 → 0.090
   └─► 權重 α = [0.665, 0.245, 0.090]
            │
            ▼
   ┌─── OV 電路（搬運：搬什麼）
   │   用 α 加權混合 value，寫回 it 位置
   │      0.665·v_animal + 0.245·v_street + 0.090·v_it
   │      = (0.755, 0.335)  ◄── 主要把 animal 的內容搬進 it
   └─► 寫回殘差流：it 現在「沾上了 animal 的內容」
```

讀這張圖：**QK 電路負責讓 it『指向』animal（定址），OV 電路負責把 animal 的內容『搬進』it（取數）。** 這正是開頭 CPU 那個比喻——定址邏輯算出「去哪一格」，取數邏輯把那一格搬回暫存器，兩套電路分開、各自可讀。ch01 的懸念「it 該看 animal」，在 ch05 是「softmax 權重最大」、在 ch17 是「核迴歸的加權平均」、在 ch18 是「線索被 animal 那個記憶吸引子吸過去」，在本章是「QK 電路把注意力定址到 animal、OV 電路把 animal 的內容搬過來」。**同一個 (0.755, 0.335)，第五種讀法。**

> 老實話：脊椎只是個示意。我得跟你講清楚，這張 it→animal 的電路圖是**示意性的**——脊椎的向量是我們為了能手算硬編的，不是從真實模型逆向出來的權重。代名詞消解（coreference）在真實模型裡確實有頭參與（Clark 2019 發現某些頭對應共指關係，ch20 提過），但「it→animal」這條具體連結是不是某顆頭乾淨地做的，那要做真正的電路分析才能下結論。本章用脊椎畫電路，是為了讓你**看懂 QK/OV 這副框架怎麼套到一條連結上**，不是宣稱真實模型裡就長這樣。把它當「框架的教學範例」，別當「實證的電路發現」。

## 故障模式與直覺陷阱

機制可解釋性是 ch20 那場嘴仗之後最讓人鬆一口氣的東西——終於有「拆得開、驗得了」的解釋了。但正因為它讓人鬆口氣，最容易讓人滑進「過度樂觀」的坑。

| 陷阱 | 錯誤直覺 | 哪裡翻車 | 怎麼自我察覺 |
|---|---|---|---|
| **把 QK/OV 拆解當成「整個模型已被解釋」** | 「既然能拆電路，那 LLM 已經是白盒了」 | QK/OV 是**單顧一顆頭**的乾淨拆法；真實大模型有成千上萬顆頭、層層合成、還有 FFN 與 superposition（一個方向擠進多個特徵），絕大多數計算**還沒**被逆向出來 | 問：「我能畫出這個模型完成這個任務的**完整**電路嗎？」幾乎永遠不能。被乾淨理解的多半是玩具模型或單一電路 |
| **把脊椎那張 it→animal 電路當成真實發現** | 「書上畫了電路，所以模型真的有顆 it→animal 頭」 | 脊椎向量是手編的教學範例，不是逆向出來的真實權重；真實 coreference 機制要做實際電路分析才知道 | 問：「這些數字是從訓練好的模型量出來的，還是為了好算硬編的？」本書的是後者，是框架示範 |
| **以為 induction head 只會死板照抄字面** | 「它就是個字串比對、`[A][B]…[A]→[B]` 的硬規則」 | 真實 induction heads 會做**fuzzy/抽象**的 pattern 補全（Olsson 2022 明說），不只字面重複；它的泛化能力正是它和 in-context learning 掛鉤的原因 | 問：「它對訓練沒見過的新 token、隨機字串還成立嗎？」成立——這證明它學的是『複製上次的下一個』這個運算，不是某張查找表 |
| **把「相關」當「因果」（沒做消融就下結論）** | 「這顆頭看起來在做 X，所以模型靠它做 X」 | 這正是 ch20 的老毛病在機制層復發；induction head 之所以站得住，是因為 Olsson **真的做了 ablation**（敲掉它、能力就掉），不是光看它「看起來像」 | 問：「有人**介入**過嗎（剪掉/改掉這顆頭看能力變化），還是只觀察了它的行為？」沒介入就只是合理性，不是忠實性 |
| **把 Transformer Circuits 當已蓋棺定論的同儕評審結論** | 「Anthropic 說的，就是定論」 | Transformer Circuits Thread **不是傳統同儕評審期刊**（landscape §6）；影響力大，但仍是一條活躍演進、會被後續工作修正的研究線 | 問：「這是被獨立複現、寫進教科書的定論，還是某個研究團隊的奠基性提案？」induction head 已被廣泛複現，但整套電路框架仍在發展 |

第一個陷阱是這章最該防的，它和 ch20 那場嘴仗其實是同一條斷層的兩側。ch20 教你「別把熱圖當解釋」；本章給了你比熱圖強得多的工具（拆電路、做消融），很容易讓你矯枉過正、以為「現在模型已經透明了」。沒有。**機制可解釋性是一門很年輕、進展很快、但離『完整理解一個前沿大模型』還非常遠的學科（2026-06）。** 我們手上有的是：幾個被乾淨逆向的小電路（induction head 是最漂亮的一個）、一套拆頭的框架（QK/OV）、和一些工具（消融、後來的 sparse autoencoder/superposition 那條線，本章不展開、見延伸閱讀）。這比 ch20 的「看熱圖瞎猜」強太多了——但它是「我們讀懂了幾段反組譯」，不是「我們有完整原始碼」。誠實地拿著這個工具，它很有力；把它吹成「LLM 已是白盒」，你就從 ch20 的坑跳進另一個坑。

## 紙上推演

### 推演題 1 — 把一顆頭拆成 QK 與 OV 兩個職責 **[12 分鐘]** ★

不查書，向另一個工程師解釋：一顆注意力頭裡的 W_Q、W_K、W_V、W_O 四個矩陣，怎麼配對成 QK 電路與 OV 電路？各自吃什麼、吐什麼、對應三步骨架的哪一步？用 CPU 的「定址 vs 取數」打一個比方，並各舉一個它在做什麼的例子。最後回答：為什麼 Elhage 2021 主張「研究 W_Q W_Kᵀ 的乘積，比分別研究 W_Q 和 W_K 有意義」？

### 推演題 2 — 手追一遍 induction head 的補全 **[18 分鐘]** ★★

給序列 `[貓][吃][魚] ... [貓] → ?`。

(a) 用 `[A][B]…[A]→[B]` 的模板，口頭預測 induction head 會補什麼，並說出理由（它在「照抄」什麼）。
(b) 把這個補全拆成**兩層頭**：第一層 previous-token head 做了什麼（每個位置被加上了什麼資訊）？第二層 induction head 的 QK 電路在比對什麼（注意：比的是 token 本身還是「前一個 token」）？它的 OV 電路搬了什麼？
(c) 為什麼這件事**單層頭做不到**、非得兩層合成不可？（提示：單層頭在當前位置手上只有什麼資訊？）

### 推演題 3 — induction head 與 in-context learning 的關係 **[12 分鐘]** ★★

向另一個工程師解釋 Olsson 2022 的核心發現。至少要點到：(a) 什麼是 in-context learning（給一個你熟的 few-shot prompt 例子）；(b) 「phase change」是什麼、訓練時哪兩件事在這個窗口同時發生；(c) 「同時發生」只是相關，Olsson 用什麼證據把它推向因果（至少講出「消融」這一條，並說明為什麼消融比「看它行為像」更有說服力）；(d) 為什麼這個發現是機制可解釋性比 ch20「看熱圖」強的地方（提到忠實性 vs 合理性）。

### 推演解答

**題 1 解答（要點）。**

```text
配對          = 哪兩個矩陣        吃什麼 → 吐什麼              三步骨架   CPU 比方
──────────────────────────────────────────────────────────────────────────────
QK 電路 W_QK  = W_Q W_Kᵀ        (token i, token j) → 分數    打分+正規化  定址：去哪一格
              （決定看哪裡）      → softmax 成權重 α                     （算地址）
OV 電路 W_OV  = W_V W_O          被看到的 token → 寫回的內容   加權混合     取數：搬內容
              （決定搬什麼）                                            （搬進暫存器）
```

例子：QK 電路可能學到「指向最近的主詞」「指向前一格」（previous-token head）這類定址規則；OV 電路可能學到「把這個 token 的身分複製過去」「把詞性資訊寫進去」這類搬運規則。為什麼研究乘積有意義：因為 W_Q 和 W_K **從不單獨影響分數**，它們永遠以 `xᵢ (W_Q W_Kᵀ) xⱼᵀ` 這個夾心的形式一起作用——分別看 W_Q、W_K 各自的數值沒有獨立的可解釋意義（它們之間有任意的可逆變換自由度），但乘積 W_QK 是「(i,j) → 該不該看」這個確定的函數，這才是「定址邏輯」本身。常見錯路：把 QK 電路說成「算 query 和 key」就停——重點是它們**併成一個函數**、那個函數就是注意力模式的成因。

**題 2 解答。**

(a) 補 `魚`。理由：序列前面出現過 `[貓][吃][魚]`，現在又出現 `[貓]`——induction head 照抄「上次 `貓` 後面接的是什麼」，上次接 `吃`……等等，要小心：`[A][B]…[A]→[B]` 補的是「上次 A 的**正後方**那一個」。上次 `貓`（A）的正後方是 `吃`（B），所以嚴格按模板補 `吃`。（這題故意有個坑：很多人會憑語意補「魚」，但純粹的 induction 機制補的是**位置上的下一個**＝`吃`。真實大模型會混入語意，但本題考的是純 induction 電路的機械行為——它照抄的是「上次這個 token 的正後方」。）

(b) 兩層拆解：
```text
第一層 previous-token head：
  讓每個位置帶上「我前一格是誰」。
  位置「吃」帶著「前一格＝貓」；末尾的「貓」自己被往後看的位置記住。
第二層 induction head：
  當前在末尾的「貓」，要預測 ?。
  QK 電路：找「前一格 == 當前 token（貓）」的位置 → 命中「吃」的位置
           （因為「吃」的前一格正是第一個「貓」）—— 比的是『前綴』，不是 token 本身
  OV 電路：把「吃」複製到當前位置 → 抬高 logit(吃)
```
(c) 單層頭做不到，因為在當前位置（第二個「貓」），單層頭手上只有**當前 token 自己的內容**（「貓」）。要做 induction，它得問「哪個位置的**前一格**是貓」——這需要「每個位置先被標上它前一格是誰」，而那份資訊得靠**前一層**（previous-token head）先搬好。沒有第一層鋪路，第二層的 QK 電路無從比對「前綴」。所以 induction 是兩層的 K-composition，不是任何單層頭能獨力完成。常見錯路：在 (b) 把第二層 QK 說成「找另一個『貓』」——不對，它找的是「前一格是貓的那個位置」，這正是 previous-token head 存在的理由。

**題 3 解答（要點）。** (a) in-context learning：不改權重，只在 prompt 裡給範例，模型就現學一個臨時規則。例：`2→4, 3→6, 5→?` 模型接 `10`（現學「乘 2」）；或 few-shot 翻譯、分類。(b) phase change：訓練早期 loss 曲線出現一個小鼓包的窗口，在這個窗口裡 in-context learning 能力突增、且 induction heads 同時形成。(c) 相關→因果：Olsson 給了多條證據，最有力的是**消融（ablation）**——測試時把 induction heads 敲掉，in-context learning 能力大幅下降；還有「共擾動」——動架構讓 induction head 早/晚形成，能力變強的時點跟著移。消融比「看行為像」強，因為它是**介入**（intervention）：你主動拿掉一個零件、看結果變化，這能建立因果；光觀察「這顆頭看起來在做 pattern matching」只是相關，可能那顆頭根本不是真正幹活的那個。(d) 這正是比 ch20 強的地方：ch20 停在「熱圖看起來合理（plausibility/合理性）」，無法證明模型真的據此計算；induction head 這條線做到了「敲掉它、能力就消失（faithfulness/忠實性）」——這是模型**真的依賴**這個機制的證據，不只是看起來合理。常見錯路：把結論說成「ICL 就是 induction head」——過度延伸，正確版是「induction head 是 ICL 的（主要）機制源頭之一」。

## 延伸閱讀

- **Elhage et al. 2021「A Mathematical Framework for Transformer Circuits」**（transformer-circuits.pub/2021/framework，2021-12-22）——本章 QK/OV 拆解的源頭。先讀「Summary of Results」與「Zero-Layer / One-Layer Transformers」兩節，看一層 attention-only 模型如何等於 skip-trigram 的集合、QK 與 OV 如何各司其職；再讀「Two-Layer」那節看 induction head 怎麼靠 composition 組出來。完整張量代數可略過，抓「一顆頭＝QK 電路＋OV 電路、跨層靠合成」這個主結論（2026-06 查證，性質：非傳統同儕評審的研究 thread）。
- **Olsson et al. 2022「In-context Learning and Induction Heads」**（arXiv 2209.11895，transformer-circuits.pub/2022/in-context-learning-and-induction-heads）——induction head 與 in-context learning 那條因果線的源頭。讀「What is an Induction Head?」與六條證據裡的「phase change co-occurrence」「ablation」兩條即可抓主結論；本章 phase change 那張圖與消融論證都來自這裡。
- **Anthropic「Transformer Circuits Thread」首頁**（transformer-circuits.pub）——這整條研究線的入口，按時間排了從 framework、induction heads 到後來 superposition / monosemanticity / sparse autoencoder 的文章；想往「本章沒展開的下一步」走（superposition＝一個方向擠多個特徵、SAE＝把特徵拆乾淨）就從這裡接（2026-06 查證）。
- **Anthropic「In-context Learning and Induction Heads」官方互動版**——論文的網頁版，phase change、prefix matching、copying 都有互動圖；比讀 PDF 直觀，和本章「兩層合成」那節對照著看最省力。
- **ch20（注意力是解釋嗎）與 ch08（多頭）**——本章是 ch20 的續集（從「看哪裡的相關性」升級到「拆電路的機制」）也是 ch08 的續集（多頭裡「不同頭分不同工」在這裡被拆成具體電路）。卡在「權重算不算解釋」就回 ch20；忘了頭為什麼要分工就回 ch08。
