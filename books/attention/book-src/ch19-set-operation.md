# ch19 — 注意力即集合運算：置換等變與訊息傳遞

> **本章解決什麼問題**：這是「理論身世」三章的最後一塊。ch17 說注意力是核迴歸、ch18 說它是聯想記憶；本章換上第三副、也是最根本的鏡片——注意力是一個**集合上的運算**。ch09 你已經親手算出「裸 self-attention 看不見順序」，當時我答應給你嚴格版，現在兌現：這個性質叫**置換等變（permutation equivariance）**，它不是工程小瑕疵，是 attention 的深層身分證——它把輸入當「一袋無序的東西」處理，所以順序非外加不可（回收 ch09）。看清這件事，三個收穫接踵而來：注意力其實是一輪在**全連接圖上的訊息傳遞**（接上你做過的 gossip／聚合協定，也接上 GNN）；它的**表達力**有多強（通用近似、甚至圖靈完備，但要看清前提）；以及為什麼純堆 attention 會**秩坍縮**（回收 ch13「為何需要殘差與 FFN」）。本章不新增三步骨架的任何一步——它退後一步，問「這三步作用在一個**集合**上會有什麼結構性後果」，順帶回答「為什麼是 softmax，換掉它行不行」。

## 從你已知的出發

你天天在處理「這東西有沒有順序」這個區別，ch09 我們用 `Array` vs `Set` 把它點過一次。這一章把那個直覺往兩個方向推。

第一個方向你也熟：**對稱性要求**。你寫過去重、聚合（`SUM`、`COUNT`、`GROUP BY`）、寫過比較兩個 JSON 物件是否「邏輯相等」——這些函數有一條鐵律：**對無序的東西，結果不能依賴你恰好用什麼順序餵它**。`sum([3,1,2])` 必須等於 `sum([1,3,2])`；判斷兩個 set 相等不能因為你 iterate 的順序不同就翻臉。如果你的聚合函數對輸入順序敏感，那是 bug。這個「對無序輸入，輸出不准偷看順序」的要求，數學上就叫**置換不變/置換等變**——而 self-attention 天生就滿足它（ch09 算過，本章證明）。

第二個方向，是你做過的**分散式聚合協定**。想一個 gossip 或 service mesh 的場景：每個節點手上有一份本地狀態，每一輪它去問鄰居「你現在是什麼狀態」，把鄰居們的回報**按某種權重聚合**進自己的狀態，更新一次；下一輪再來。你關心三件事：每個節點問誰（連結結構）、每個鄰居的回報佔多少權重（聚合權重）、聚合完怎麼更新自己。

```text
  一輪 gossip / 訊息傳遞                  一層 self-attention
  ─────────────────────────             ──────────────────────────
  每個節點問它的鄰居                       每個 token 問序列裡所有 token
  鄰居回報各佔多少權重（你寫死/算出）       權重 = softmax(q·k)（算出來的）
  把鄰居的回報加權聚合進自己               加權混合所有 value
  更新本地狀態，進下一輪                    過殘差/FFN，進下一層
```

把這兩條線拼起來，你就摸到本章的論點了：**self-attention 是「在一張全連接圖上、用學出來的權重做一輪訊息傳遞」的運算，而它作用的對象本質是一個集合（所以對順序無感）。** 「全連接」是因為 ch07 講過每個 token 對所有 token 都算一次（沒有預設的鄰接結構，人人都是人人的鄰居）；「學出來的權重」是因為邊權重 = softmax(q·k)，不是你寫死的（對照 gossip 裡你手寫的權重）。這一章就是把這兩個比喻釘成嚴格的數學，然後看它帶來什麼後果。

## 嚴格版：self-attention 置換等變

ch09 我用脊椎手算示範了「重排 [animal, street, it]，it 給 animal 的權重一個數字都不變」。那是個別例子。現在給一般證明——別怕，它短得驚人，而且每一步你都能口頭說出理由（工程師的嚴謹）。

先把記號立好。把整個序列疊成一個矩陣 **X**：每一**列**是一個 token 的向量，n 個 token 就是 n 列。self-attention 對 X 做的事，攤開就是三步骨架的矩陣版（ch07 的形式）：

```text
  Q = X·W_Q,  K = X·W_K,  V = X·W_V        ← 三個學來的投影（ch04）
  A = softmax( Q·Kᵀ / √d )                  ← n×n 注意力矩陣，逐列 softmax
  SelfAttn(X) = A · V                        ← 加權混合
```

「重排序列」這個動作，用一個**置換矩陣 P** 表示。P 是一個 n×n 的方陣，每列每行恰好一個 1、其餘是 0——它的作用就是把矩陣的列重新洗牌。P·X 就是「把 X 的列按 P 規定的順序重排」。我們要證的等變性，寫成一條式子是：

```text
        SelfAttn(P·X) = P · SelfAttn(X)
        ─────┬───────   ────┬─────────
        先重排輸入再算       先算再重排輸出
```

白話：**先把 token 洗牌再跑 attention，等於先跑 attention 再把輸出按同樣方式洗牌。** 洗牌和 attention 可以交換次序——順序的調換被原封不動地搬過去了，沒有任何資訊因重排而改變。這就是「等變（equivariant）」：排列作用在輸入上 = 排列作用在輸出上。

證明走一遍，盯著 P 怎麼穿過每一步：

```text
  輸入重排:   X → P·X
  投影:       Q' = (P·X)·W_Q = P·(X·W_Q) = P·Q     ← P 與右乘 W 無關，穿過去
              同理 K' = P·K,  V' = P·V
  打分:       Q'·K'ᵀ = (P·Q)·(P·K)ᵀ = P·(Q·Kᵀ)·Pᵀ  ← 分數矩陣被「列換＋行換」
  正規化:     softmax 逐列做。第 i 列換到第 π(i) 列、列內元素也跟著行換 Pᵀ 重排，
              但 softmax 對「同一列內元素的順序」不敏感（ch05：重排分數→重排權重，
              集合不變），所以 A' = P·A·Pᵀ
  混合:       A'·V' = (P·A·Pᵀ)·(P·V) = P·A·(Pᵀ·P)·V = P·A·V = P·SelfAttn(X)
                                              └─┬─┘
                                       Pᵀ·P = I（置換矩陣正交，洗回來）
```

關鍵的一步是 `Pᵀ·P = I`：置換矩陣是正交矩陣，「先按 P 洗、再按 Pᵀ 洗回去」剛好抵銷。中間那個 `A' = P·A·Pᵀ` 的 Pᵀ，在混合時遇到 V' 自帶的 P，被它抵掉了——這就是為什麼分數矩陣被「列換又行換」攪了一通，最後輸出卻乾乾淨淨只剩一個 P。

> **嚴謹度標示**：這是**嚴格證明**，不是直覺類比。唯一用到的事實是 (a) P 與右乘投影矩陣可交換、(b) softmax 逐列且對列內重排是等變的、(c) Pᵀ·P=I。三條你都能口頭驗。前提：這裡的 self-attention 是 ch07 的**裸版**（無位置編碼）。一旦加上位置編碼（ch09），輸入就不再只是 X，而是 X+PE，PE 那一塊不隨 token 洗牌——等變性**被刻意打破**，這正是我們加位置的目的。

把上面的 `SelfAttn(PX)=P·SelfAttn(X)` 對到 ch09 的脊椎實驗：我用 Python 對過（這正是 ch09 那句「位元級相同」的嚴格版）——拿一組隨機投影、把 3 個 token 用 P 洗牌，`SelfAttn(P·X)` 和 `P·SelfAttn(X)` 兩個矩陣**逐元素完全相等**。順序的資訊，attention 從頭到尾沒碰過。

### 不變 vs 等變：差一個「你還在不在乎輸出順序」

ch09 提過一個更狠的特例：**置換不變（permutation invariant）**。兩者差一線，值得釘清楚，因為 Set Transformer 那篇論文（下一節）的標題用的是「不變」，而 self-attention 的主體是「等變」。

```text
  置換等變   SelfAttn(P·X) = P·SelfAttn(X)    輸出跟著洗牌，但內容不變
             （self-attention 的 body：n 進 n 出，逐位置對應）

  置換不變   f(P·X) = f(X)                     輸出根本不動，連洗牌都看不出
             （需要再做一次「聚合」把 n 個輸出收成 1 個：sum/mean/pooling）
```

直覺：**等變**是「我還給你 n 個輸出，跟你的 n 個輸入一一對應，你重排我就跟著重排」；**不變**是「我把 n 個輸出聚合成一個總結（比如全部相加、或一個固定 query 去混合它們），這個總結你怎麼洗牌都一樣」。脊椎裡這兩件事都出現過：整層 self-attention 的「三個輸出向量」是**等變**的（重排輸入就重排這三個輸出）；但單看 **it 那一個 query** 的輸出 (0.755, 0.335)，它在任何順序下都不變——因為它是「it 對所有 token 的混合」這個聚合，這是**不變**的展現。`sum` 是不變、`map` 是等變，大概就是這個分野。

## Set Transformer：把「attention 是集合運算」做成正式框架

「self-attention 是集合運算」不是我事後安的帽子。2019 年有一篇論文就是衝著「用 attention 處理**集合輸入**」去設計的：

**Lee et al. 2019「Set Transformer: A Framework for Attention-based Permutation-Invariant Neural Networks」（arXiv 1810.00825，ICML 2019，照 landscape §5）。** 它要解的問題是「輸入是一個集合、輸出是這個集合的某個函數」——比如給一堆點，估計它們的聚類中心；給一堆數，算它們的某個統計量。這類問題的鐵律就是上一節那條：**輸出不准依賴元素的順序**。Set Transformer 的貢獻是把 attention 拆成兩種積木（2026-06 查證）：

- **SAB（Set Attention Block）= self-attention，是置換等變的**——n 個元素進、n 個元素出，互相 attend，重排輸入就重排輸出。這正是我們上一節證的東西，他們把它當成「讓集合元素彼此交互」的零件。
- **PMA（Pooling by Multi-head Attention）= 置換不變的池化**——用一組學來的「seed 向量」當 query，去 attend 那 n 個元素，把它們**聚合**成固定數量的輸出。這一步把「等變」收成「不變」（呼應上一節 `sum`/聚合那條線）。

論文還順手解決了 O(n²)：他們的 **ISAB（Induced Set Attention Block）** 引入 r 個「inducing points（誘導點）」（r ≪ n），讓元素不直接兩兩 attend、而是透過這 r 個中介，把成本從 O(n²) 壓到 O(nr)（2026-06 查證）。如果你覺得這個「用少數中介代替全連接」的想法眼熟——對，它和 ch15 稀疏/低秩注意力是同一個家族的想法，只是這裡的動機是「集合」而非「序列」。

Set Transformer 對本書的意義不在它的具體架構，而在它的**證詞**：當一群人從「我要處理集合」這個完全不同的動機出發，他們自然而然地選了 attention，並且必須小心地維護置換對稱性。**attention 是處理集合的對的工具——這不是 NLP 的巧合，是它的數學本性。** ch18 我們看到聯想記憶的人也獨立撞上了 attention；這裡是集合學習的人第三次撞上它。同一台機器，第三個發明動機。

## 注意力即訊息傳遞：在全連接圖上學出邊權重

現在把開頭那個 gossip 比喻釘成數學。這是本章我覺得最值得你內化的一頁，因為它一口氣把 attention 接到一整個領域（圖神經網路，GNN）。

把序列裡的 n 個 token 看成一張圖的 **n 個節點**。self-attention 既然「每個 token 對所有 token 都算一次」（ch07），那這張圖就是**全連接**的——每個節點和每個節點之間都有一條邊（包含自己到自己）。一層 self-attention，就是在這張圖上跑**一輪訊息傳遞（message passing）**：

```text
  訊息傳遞（GNN 的通用骨架）          self-attention 是它的特例
  ──────────────────────────         ────────────────────────────
  1. 每個節點 i 對鄰居 j 算邊權重      α_ij = softmax_j( q_i · k_j / √d )
       （GNN 通常靠固定的圖結構）          ← 全連接 + 權重是「學＋算」出來的
  2. 每個鄰居送一則「訊息」             訊息 = v_j（value 投影）
  3. 節點 i 按邊權重聚合所有訊息        h_i' = Σ_j α_ij · v_j
  4. 更新 i 的狀態（進下一輪/層）       過殘差 + FFN（ch13），進下一層
```

逐行就是三步骨架：**打分 = 算邊權重、正規化 = softmax 讓一個節點分給鄰居的權重和為 1、加權混合 = 聚合鄰居訊息**。差別只在「邊權重哪來」：傳統 GNN 的圖結構是給定的（誰是誰的鄰居寫死在資料裡，比如社交網路的好友關係），邊權重常常固定或簡單；self-attention 是在**全連接**圖上，讓**每條邊的權重由兩端節點的內容算出來**（q·k）、而且 Q/K 投影是學的。

這個對應不只是比喻——它有一條歷史證據鏈。**Graph Attention Networks（GAT，Veličković et al. 2017，arXiv 1710.10903，ICLR 2018；2026-06 查證）** 就是反過來把 self-attention 搬進 GNN：讓圖上每個節點用 attention 機制決定「聚合鄰居時，每個鄰居佔多少權重」，而不是像更早的 GCN 那樣用固定的、正規化過的鄰接矩陣權重（GAT 論文明說 GCN 是 GAT 的一個特例，把 attention 函數換成固定的、譜上正規化的鄰接矩陣；2026-06 查證）。NLP 的人從「翻譯」走到 attention、圖學習的人從「節點分類」走到 attention，兩邊在 2017–2018 年又一次會合。**self-attention = 在全連接圖上、邊權重由內容算出來的訊息傳遞**——這句話兩個領域都認。

### Worked example：脊椎 it 是 3 節點圖上的一次訊息聚合

把脊椎搬成一張 3 節點圖。節點 = {animal, street, it}，全連接。每個節點攜帶一則「訊息」（就是它的 value）：

```text
  3 節點全連接圖（每個節點到每個節點都有邊，含自環）

        animal ●────────● street
          │ ╲          ╱ │
          │   ╲      ╱   │
          │     ╲  ╱     │
          │      ╲╱      │
          │      ╱╲      │
          │    ╱    ╲    │
          └──●───────────┘
             it

  節點訊息（value）：  animal: (1,0)   street: (0,1)   it: (1,1)
```

現在讓 **it 這個節點做一輪聚合**。它對三條出邊（到 animal、street、自己）算邊權重——這步就是 ch04 的打分，原樣搬來：

```text
  邊 it→animal:  q_it·k_animal = 4
  邊 it→street:  q_it·k_street = 2
  邊 it→it:      q_it·k_it     = 0
  raw 邊分數 = [animal 4, street 2, it 0]
```

除以 √d=2 得 [2,1,0]，過 softmax 成「正規化的邊權重」（這組數全書算了無數次，再算一次確認）：

```text
  e² = 7.389,  e¹ = 2.718,  e⁰ = 1,  和 = 11.107
  邊權重 α = [ 7.389/11.107, 2.718/11.107, 1/11.107 ]
          = [ animal 0.665, street 0.245, it 0.090 ]   ← 全書正典脊椎權重
```

it 節點按這組邊權重聚合三個鄰居送來的訊息（value）：

```text
  h_it' = 0.665·(1,0) + 0.245·(0,1) + 0.090·(1,1)
        = (0.665+0.090, 0.245+0.090)
        = (0.755, 0.335)   ← 偏向 animal 的訊息，正是全書正典輸出
```

**(0.755, 0.335)。** 又是它。ch05 它是「softmax 權重最大」、ch17 它是「核迴歸的加權平均」、ch18 它是「線索被 animal 吸引子吸過去」，**這裡它是「it 節點從鄰居 animal 那裡聚合來最多的訊息」。** 第五種說法，同一個數字。圖的鏡片照亮的那個面是：**注意力是節點向鄰居「要資訊」再聚合——它最重的那條邊，指向它最該聽誰的話。** it 最該聽 animal 的，所以它把自己更新成偏向 animal 攜帶的訊息。

（多說一句把它和「整層」接起來：上面只算了 it 一個節點。一整層 self-attention 是**三個節點同時各做一次這樣的聚合**——animal 也對所有節點算邊權重聚合一次、street 也一次。三個節點的輸出合起來，就是 ch07 那個 3×3 注意力矩陣乘上 V。本章的等變性說的就是：這三個輸出和三個輸入一一對應，你重排輸入它們就跟著重排。）

## 表達力一瞥：這台集合機器能算多強的東西

既然 attention 是一種這麼受對稱性約束的運算（連順序都看不見），一個自然的疑問是：**它的表達力會不會因此很弱？** 答案出人意料地是「不會，反而強得驚人」——但每個「強」的宣稱都帶前提，本書的規矩是把前提寫清楚、不把類比偽裝成定理。

**通用近似（universal approximation）。** 在足夠的頭數、維度與深度下，Transformer 可以任意精度逼近一大類「序列到序列」的連續函數（這是「通用近似器」這個說法的內容）。直覺上的關鍵其實正是本章主題的反面：**裸 attention 是置換等變的、表達力受對稱性所限**，所以要逼近一般（非置換等變的）序列函數，你**必須**靠位置編碼把對稱性打破、再靠 FFN 提供逐位置的非線性。換句話說，attention 負責「混合資訊」、位置編碼負責「分辨位置」、FFN 負責「逐點變換」，三者合起來才通用——**單靠 attention 不夠**（這條伏筆下一節的秩坍縮會收）。嚴謹度標示：通用近似定理的精確敘述與證明本書不展開（直覺帶過，指向延伸閱讀）；你只要抓住「為什麼需要這三件事湊齊」這個結構直覺。

**圖靈完備（Turing complete）。** 更猛的一個結果：**Pérez et al.「Attention is Turing-Complete」（JMLR 2021，照 landscape；2026-06 查證 JMLR v22）** 證明 Transformer 可以模擬任意圖靈機——也就是說，原則上它能算「任何可計算的東西」。但這個宣稱的前提**極不現實**，必須講清楚，否則就是把定理當口號：

```text
  「Attention is Turing-Complete」的前提（Pérez et al. 2021，2026-06 查證）
  ────────────────────────────────────────────────────────────────────
  (1) 任意精度（arbitrary precision）  ← 最致命的前提
        實數權重與中間數值要能存無限位元；一旦限制成有限精度，
        Transformer 退化成有界的有限狀態機，圖靈完備性立刻失效
  (2) 特定的位置編碼（含 i、1/i、1/i² 這類分量）
  (3) hard attention（硬注意力，而非可微的 softmax）
  (4) 先把輸入完整讀一遍、且允許足夠多步的自迴歸解碼（等於給它無界的「工作帶」）
```

最該記住的是 **(1)**：圖靈機的威力來自它有一條**無限長的紙帶**；一個真實的 Transformer 跑在有限位元的硬體上、context 也有限，它其實是一台（雖然巨大的）**有限狀態機**，嚴格說**不**圖靈完備。所以這個定理的正確讀法不是「LLM 什麼都能算」，而是「attention＋自迴歸這套架構在計算上沒有原理性的天花板——天花板是精度與長度這些工程資源，不是機制本身」。這是個漂亮、但需要小心引用的結果（landscape 提醒：明寫前提）。

> **這裡反直覺在哪**：你大概以為「看不見順序」是個弱點——一個連 dog bites man 和 man bites dog 都分不出的機制（ch09），能強到哪去？但表達力的結論反過來說：attention 的力量恰恰來自它「先把所有東西平等地當集合對待」這個極簡的對稱起點，**威力是靠後面外加的零件（位置、FFN、深度）一層層解放出來的**。極簡的集合運算 + 補上的零件 = 通用甚至（理想化下）圖靈完備。這跟全書的主旋律一致：三步骨架本身極小，撐起整個時代靠的是怎麼把它組裝起來。

## 為什麼純堆 attention 會坍縮：秩坍縮（回收 ch13）

上一節說「單靠 attention 不夠、要靠 FFN 與深度」。ch13 也賣過一個關子：純堆 attention 會**秩坍縮**，殘差與 FFN 是解藥。現在用「集合運算」這副鏡片，正好把這個現象看穿——它其實是訊息傳遞的一個已知毛病。

**Dong et al. 2021「Attention is Not All You Need: Pure Attention Loses Rank Doubly Exponentially with Depth」（arXiv 2103.03404，ICML 2021 oral，照 landscape §5；2026-06 查證）** 的結果：把**純** self-attention（拿掉殘差、拿掉 MLP）一層層往上堆，輸出會**雙指數速度**地坍縮到**秩 1**——也就是所有 token 的輸出向量收斂成幾乎一模一樣的同一個向量。論文把這個傾向叫「token uniformity（token 趨同）」（2026-06 查證）。

為什麼？用訊息傳遞的直覺一秒就懂：**每一層 attention 都是「每個節點把鄰居的向量加權平均進自己」——平均是一個平滑操作。** 你對一鍋資料反覆做加權平均，它會越來越平、差異越來越小，最後糊成一團。一次平均把差異抹掉一點，兩次抹掉更多，層層複合下來，所有 token 就被「平均」到同一個點。圖神經網路那邊早就給這個毛病起了名字——**over-smoothing（過度平滑）**：GNN 堆太多層，節點特徵會收斂成同一個值、失去區分度（2026-06 查證，self-attention 也被分析為一種低通濾波器，層數越多越只剩低頻、抹掉高頻細節）。**秩坍縮和 over-smoothing 是同一件事的兩個名字**——一個從矩陣的秩看，一個從圖上節點趨同看，都是「反覆加權平均把東西抹平」的後果。這又是「attention＝訊息傳遞」這個對應的紅利：GNN 踩過的坑，attention 一模一樣會踩。

解藥也就是 ch13 那兩樣，現在你看得懂它們為什麼有效：

- **殘差連接（residual）**：`輸出 = 輸入 + Attention(輸入)`。那個「+ 輸入」把每個 token 自己原本的向量直通保留下來——平均抹不掉它，因為每一層都把原料重新加回去。Dong 的論文明確指出**殘差是阻止坍縮的關鍵**（2026-06 查證）。
- **FFN/MLP**：逐位置的非線性。它對每個 token **獨立**做一次非線性變換，不跨 token 混合——這恰好把「混合（attention）抹平」之後又重新拉開各 token 的差異。論文同樣點名 MLP 阻止退化（2026-06 查證）。

所以「為什麼一層 transformer 裡 attention 不能單獨用、非配殘差和 FFN 不可」（ch13 的問題），在本章有了根上的答案：**attention 是一個平滑（混合）算子，反覆平滑會坍縮；殘差保住個性、FFN 拉開差異，正是用來抵抗這個平滑傾向的。** 三者是配套的，少一個這台機器就糊掉。

## 為什麼是 softmax：競爭、合法分布，以及換掉它行不行

最後收一個全書一直欠著的問題：三步骨架的第二步為什麼非得是 **softmax**？ch05 給過「它是 soft-argmax、輸出是合法機率分布」；放在「集合運算/訊息傳遞」的脈絡裡，還能再補一個關鍵性質——**競爭（competition）**。

softmax 的分母是「對所有 key 的 exp 分數求和」。這意味著各個位置的權重**互相搶**同一塊「和為 1 的預算」：一個位置的分數高、它的權重就漲，**別的位置的權重必然被擠低**。對「全域檢索」這件事，這個競爭性恰恰是想要的——當 it 在全序列裡找「我指誰」，你要的是它**選出**最相關的（animal 漲、其餘讓位），而不是對每個位置各自獨立地打個分、彼此不相干。softmax 把「分配注意力」變成一場零和的競賽，這正是「該看哪裡」需要的語義。

```text
  softmax 的三個性質（在集合/檢索脈絡下為何各有用）
  ─────────────────────────────────────────────────────
  合法分布   非負、和為 1        加權混合 = 一個良定義的加權平均（接 ch17 期望）
  soft-argmax 放大差距、可微      軟選擇、能訓練（ch05）；硬 argmax 不可微
  競爭       分母耦合所有位置      全域檢索：贏家漲、其餘必須讓位（本節重點）
```

那能不能換掉它？能，而且這是個活躍的研究方向，但要誠實標時點。**Apple 2024「Theory, Analysis, and Best Practices for Sigmoid Self-Attention」（arXiv 2409.04431，ICLR 2025，照 landscape；2026-06 查證）** 把 softmax 換成**逐位置獨立的 sigmoid**（每個位置自己過一個 sigmoid，分母不再耦合所有位置——**放棄了競爭性**）。結果：sigmoid attention 的 Transformer **仍是通用近似器**，且他們的 FlashSigmoid 核在 H100 上比 FlashAttention-2 推論快約 17%（2026-06 查證）。其他替代品（ReLU/Softplus 注意力等）也有人試。

但這裡是取捨的核心：放棄競爭性換來了什麼、損失了什麼？

```text
  正規化方式    競爭性   權重和    取捨
  ───────────────────────────────────────────────────────────────
  softmax       有       =1        全域檢索強（贏家通吃式選擇）；但分母耦合，
                                   長序列下所有位置要一起算（接 ch16 online softmax）
  sigmoid       無       不固定    逐位置獨立、好平行、可更省；但失去「搶預算」的
              （Apple 2024）        競爭，全域「選出一個最相關的」這件事變弱
  linear/核     改寫     ≈1        ch15：換掉 softmax 用核技巧換 O(n)，是近似、
              （ch15）              token 級精確回憶會吃虧
```

⚠️（2026-06）替代品這塊演變很快，但實證的共識是：**softmax 在「全域競爭/精確檢索」上仍然領先，多數混合模型即使引入別的層也保留 softmax 注意力**（landscape §5、§7，標時點）。所以本書的立場：softmax 不是唯一可能、也不是天上掉下來的神諭，它是「合法分布 + soft-argmax + 競爭」這三條性質的一個特別好用的組合；換掉它的代價，通常是丟掉其中某一條（最常丟的是競爭性）。哪個更好，2026-06 未定論，這條線一路通往 ch22 的架構之爭。

## 故障模式與直覺陷阱

| 陷阱 | 錯誤直覺 | 哪裡翻車 | 怎麼自我察覺 |
|---|---|---|---|
| **把「等變」當「不變」** | 以為 self-attention 的輸出「重排輸入也不會變」 | 整層 self-attention 是**等變**（n 進 n 出、輸出跟著洗牌）；只有再加一次聚合（pooling／單一 query 的混合）才**不變**。把兩者混為一談，你會誤判模型對順序的敏感度 | 問：「我看的是 n 個輸出向量，還是把它們聚合成 1 個？」前者等變、後者才可能不變 |
| **以為「圖靈完備」=「LLM 無所不能」** | 把 Pérez 的定理當成「Transformer 能算任何東西、沒有上限」 | 定理前提是**任意精度＋無界長度**；真實硬體有限精度，Transformer 其實是有界有限狀態機。把理想化結論套到現實會嚴重高估 | 問：「這個定理假設了無限的什麼？」（無限精度、無界紙帶）。前提一收回，威力就回到工程資源限制裡 |
| **以為注意力只看「最相關的一個」** | 把「全域檢索」想成「argmax 選出一個鄰居」 | 它是**加權混合所有鄰居**（訊息傳遞聚合全部鄰居），softmax 只是讓最相關的權重大。脊椎 (0.755,0.335) 是混合不是單選 | 問：「輸出是某個鄰居的 value 本身嗎？」通常不是，是加權平均（回收 ch05、ch18 的同款陷阱） |
| **以為堆越多層 attention 越強** | 「深就是好，多堆幾層 self-attention」 | 純堆 attention 會**秩坍縮/over-smoothing**——層層加權平均把所有 token 抹成同一個向量、越深越糊 | 問：「這些層之間有殘差和 FFN 嗎？」沒有的話，深度反而害你（Dong 2021） |
| **以為 softmax 是注意力的定義、不可替換** | 把「attention」和「softmax」綁死 | softmax 是三步骨架第二步的一個**選擇**（好用但非唯一）；sigmoid/linear/ReLU 都能放進去，各有取捨。綁死會看不懂 ch15 線性注意力和 ch22 的替代方案 | 問：「換成 sigmoid 會失去什麼？」（競爭性）。能答出這個，才算懂 softmax 在做什麼 |
| **把「集合運算」當成「attention 不懂語言結構」** | 「它只是個無序聚合，談不上理解句法」 | 裸 attention 確實是集合運算，但配上位置編碼、深度、FFN，它能逼近極強的序列函數（通用近似）。集合是**起點**不是天花板 | 問：「拿掉位置/FFN/殘差後它還剩什麼？」剩一台會坍縮的對稱平均機——這才是它的素顏，成品遠不止於此 |

**最值得單獨拎出來的失效模式：秩坍縮。** 它最陰險的地方是「看起來該越深越好，實際越深越糊」。如果你（在腦中）把一個純 attention 模型一直加深，它不會越來越聰明，而會越來越像「所有 token 都在說同一句話」——因為每一層都在做加權平均、平均會把差異抹平，雙指數地抹。這正是為什麼真實 Transformer **每一個** attention 子層都緊跟著殘差和 FFN，從來不裸用。當你哪天聽到「我們堆了很深的純注意力塔」這種說法，心裡該亮紅燈：問它殘差和 FFN 在哪。

## 紙上推演

### 推演題 1 — 證明 self-attention 置換等變 **[18 分鐘]** ★★

不看正文，自己把 `SelfAttn(P·X) = P·SelfAttn(X)` 推一遍。X 每列一個 token、P 是置換矩陣。要點到：(a) 為什麼 Q'=P·Q（P 怎麼穿過右乘的 W_Q）；(b) 分數矩陣為什麼變成 P·(QKᵀ)·Pᵀ；(c) softmax 逐列那一步為什麼還是等變的；(d) 最後 `Pᵀ·P=I` 在哪裡、抵掉了什麼。寫完用一句話回答：加上位置編碼後，這個等變性還成立嗎？為什麼這正是我們想要的？

### 推演題 2 — 脊椎 it 作為一次圖上訊息聚合 **[15 分鐘]** ★

把脊椎 [animal, street, it] 看成 3 節點全連接圖，節點訊息（value）為 v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)。讓 **it 節點**做一輪訊息傳遞：(a) 用 q_it=(1,1,1,1) 與三個 k 算三條出邊的 raw 分數；(b) 除以 √d=2、手算 softmax 得邊權重（寫出 e 的次方、求和、相除）；(c) 按邊權重聚合三則訊息，得 it 更新後的向量。最後用「訊息傳遞」的語言講一句：it 的新向量為什麼偏向 animal？並指出這跟 ch05/ch17/ch18 算出的是不是同一個輸出。

### 推演題 3 — 把秩坍縮接到訊息傳遞，並說殘差為何救得了 **[12 分鐘]** ★★

向另一個工程師解釋：(a) 為什麼純堆 self-attention 會「秩坍縮」（用「反覆加權平均把東西抹平」的直覺，連到 GNN 的 over-smoothing）；(b) 殘差連接 `輸出 = 輸入 + Attention(輸入)` 為什麼能阻止坍縮；(c) FFN 又補了什麼。最後回答：這跟 ch13「為什麼一層 transformer 裡 attention 不能單獨用」是不是同一件事？

### 推演題 4 — softmax 換成 sigmoid，得失什麼 **[10 分鐘]** ★★

列出 softmax 在「集合檢索」脈絡下的三個性質（合法分布、soft-argmax、競爭）。若把第二步的 softmax 換成「逐位置獨立的 sigmoid」（Apple 2024），哪一個性質被放棄了？這對「it 要在全序列裡選出指代對象」這件事是好是壞？再對照 ch15 的線性注意力——它換掉 softmax 的方式和 sigmoid 一樣嗎，各自的代價是什麼？

### 推演解答

**題 1 解答（要點）。**

```text
  X → P·X（重排 token）
  (a) Q' = (PX)W_Q = P(X W_Q) = P·Q   ← P 左乘、W_Q 右乘，互不干涉，P 直接穿過去
                                          同理 K'=P·K, V'=P·V
  (b) Q'K'ᵀ = (PQ)(PK)ᵀ = P(QKᵀ)Pᵀ    ← 分數矩陣被「列換 P + 行換 Pᵀ」
  (c) softmax 逐列做。列被 P 重排、列內元素被 Pᵀ 重排；softmax 對「列內元素順序」
      等變（ch05：重排分數→重排權重、集合不變），故 A' = P·A·Pᵀ
  (d) A'V' = (P A Pᵀ)(P V) = P A (Pᵀ P) V = P A V = P·SelfAttn(X)
                                  └──┘ PᵀP=I（置換矩陣正交，行換被洗回來）
```

加位置編碼後**不再成立**：輸入變成 X+PE，PE 不隨 token 洗牌（它釘在「第幾個位置」上），所以 `SelfAttn(P(X+PE)) ≠ P·SelfAttn(X+PE)`。**這正是我們要的**——ch09 講過，裸 attention 看不見順序是個洞；打破等變性就是補洞，讓模型能分辨 "dog bites man" 和 "man bites dog"。常見錯路：以為位置編碼「修好了 attention 的 bug」——它沒改 attention，它是故意往輸入裡摻一塊不對稱的料，把對稱性破壞掉。

**題 2 解答。**

```text
  (a) raw 邊分數：q_it·k_animal=4, q_it·k_street=2, q_it·k_it=0 → [4,2,0]
  (b) /√d=2 → [2,1,0]
      e²=7.389, e¹=2.718, e⁰=1, 和=11.107
      邊權重 α = [7.389/11.107, 2.718/11.107, 1/11.107] = [0.665, 0.245, 0.090]
  (c) h_it' = 0.665·(1,0)+0.245·(0,1)+0.090·(1,1) = (0.755, 0.335)
```

訊息傳遞的講法：it 這個節點對三條出邊算了權重，**指向 animal 的那條最重（0.665）**，所以它聚合進來的訊息主要是 animal 攜帶的 (1,0)，更新後的向量 (0.755, 0.335) 因此偏向 animal。**這和 ch05（softmax 權重最大）、ch17（核迴歸加權平均）、ch18（吸引子檢索）算出的是同一個 (0.755, 0.335)**——同一台機器的第五種說法。常見錯路：把 (0.755,0.335) 當成 v_animal=(1,0) 本身——它偏向但不等於，因為還聚合了 street 和 it 的訊息。

**題 3 解答（要點）。** (a) 每層 attention = 每個節點把鄰居 value 加權平均進自己；**平均是平滑操作，反覆平滑把差異越抹越小**，層層複合下所有 token 趨同、輸出矩陣坍縮到秩 1（Dong 2021 證了是雙指數速度）。這和 GNN 的 over-smoothing（堆太多層、節點特徵糊成一團）是同一個現象。(b) 殘差 `輸出 = 輸入 + Attention(輸入)` 把每個 token 原本的向量**直通保留**——平均抹不掉它，因為每層都把原料重新加回來，個性得以保住。(c) FFN 對每個 token **獨立**做非線性，不跨 token 混合，正好把被 attention 抹平的差異**重新拉開**。(d) **是同一件事**：ch13 問「為什麼 attention 不能單獨用」，本章的根上答案是「attention 是平滑算子、會坍縮，殘差＋FFN 是抵抗平滑的配套」。常見錯路：只說「殘差幫助梯度流動」——那是訓練視角的另一個好處，但本章要的是「殘差保住 token 個性、阻止秩坍縮」這個表達力視角。

**題 4 解答（要點）。** softmax 三性質：合法分布（非負、和為 1 → 加權混合是良定義的加權平均）、soft-argmax（放大差距、可微 → 能訓練）、**競爭**（分母耦合所有位置 → 一個漲、別的必須讓位）。換成逐位置獨立的 sigmoid：**放棄的是競爭性**（每個位置自己過 sigmoid、分母不再耦合，權重也不保證和為 1）。對「it 在全序列選出指代對象」這件**全域檢索**——競爭性其實是想要的（要 animal 漲、其餘讓位），所以丟掉競爭對「選出一個最相關的」偏不利；但換來逐位置獨立、好平行、可更省（FlashSigmoid 比 FA2 快約 17%，2026-06）。對照 ch15 線性注意力：它也換掉 softmax，但方式不同——用核函數 φ(q)φ(k)ᵀ 配結合律把 O(n²) 壓成 O(n)，代價是**近似**、token 級精確回憶會吃虧。兩者都動三步骨架的**第二步（正規化）**，但 sigmoid 是「換一個逐位置函數」、linear 是「換成可重排的核乘積」，取捨各異。常見錯路：以為「不用 softmax 就不是 attention 了」——三步骨架的身分在「打分→正規化→混合」，第二步用什麼正規化是可換的零件。

## 延伸閱讀

- **Lee et al. 2019「Set Transformer」**（arXiv 1810.00825，ICML 2019）——把「attention 處理集合」做成正式框架的源頭。讀它對 SAB（置換等變的 self-attention）與 PMA（置換不變的池化）的定義，看「等變零件 + 不變池化」如何拼出一個處理集合輸入的網路；ISAB（誘導點降到 O(nr)）那段可對照 ch15。
- **Veličković et al. 2017「Graph Attention Networks」**（arXiv 1710.10903，ICLR 2018）——反方向的證據：把 self-attention 搬進圖神經網路，讓節點用 attention 算「聚合鄰居時每個鄰居的權重」。讀引言它怎麼把 attention 定位成 GNN 的訊息傳遞權重，「attention = 學出來的邊權重」這句話就坐實了。
- **Dong et al. 2021「Attention is Not All You Need: Pure Attention Loses Rank Doubly Exponentially with Depth」**（arXiv 2103.03404，ICML 2021 oral）——秩坍縮的原始證明。先看 Figure 1（純 attention 雙指數坍縮到秩 1）與摘要的主結論，再看它怎麼證殘差與 MLP 阻止坍縮；這是 ch13「為何需殘差/FFN」的數學依據。標題故意反 Vaswani 那句，是這個領域的標題梗之一。
- **Pérez et al.「Attention is Turing-Complete」**（JMLR 2021，v22/20-302）——表達力上限的結果。重點不是讀完證明，而是抓住「在任意精度＋無界長度的理想化下 Transformer 可模擬圖靈機」這個結論與它的前提；對照本章「有限精度下其實是有限狀態機」的誠實版讀法。
- **Apple 2024「Theory, Analysis, and Best Practices for Sigmoid Self-Attention」**（arXiv 2409.04431，ICLR 2025）——softmax 不是唯一解的代表作。看它怎麼證 sigmoid attention 仍是通用近似器、FlashSigmoid 為何能快；對照本章「sigmoid 放棄了競爭性」這個取捨。⚠️ 替代品領域演變快（2026-06），讀時標時點。
- **ch09（位置編碼）、ch13（一整層到一整個模型）、ch15（稀疏與線性注意力）、ch22（注意力之外）**——本章的四個接點：ch09 的「洞」在這裡拿到嚴格證明；ch13 的「為何需殘差/FFN」在這裡拿到根上答案；ch15 的線性注意力是「換掉 softmax」的另一條路；ch22 的架構之爭從「softmax 還是不是必需」一路延伸。
- **ch17（核迴歸）、ch18（聯想記憶）**——「理論身世」三章的前兩塊。三章合起來才是 attention 完整的理論身世：核迴歸看「輸出是什麼（條件期望）」、聯想記憶看「為什麼能容錯（吸引子、β）」、本章看「它的對稱性與結構後果（集合、訊息傳遞、坍縮）」。
- **《矩陣是動詞》（置換矩陣與正交、SVD/低秩）**——本章用到 `Pᵀ·P=I`（置換矩陣是正交矩陣）；秩坍縮裡的「秩」與 Set Transformer 的低秩誘導點，深一層的線代直覺在那本（其 ch19–20 的 SVD/低秩）。本章只用結論。
