# ch03 — 第一次注意：Bahdanau 的對齊（加性與乘性）

> **本章解決什麼問題**：ch02 把固定向量瓶頸的痛攤開了——整句被壓成一個向量，長句的資訊在解碼前就被稀釋。本章講第一個真正的解藥：2014 年 Bahdanau 的「對齊模型」，讓 decoder 在生成每個目標詞時**回頭動態加權**源句的每一個位置，而不是只吃一個被壓扁的摘要。更重要的是，這個機制第一次完整地展示了本書的三步骨架（打分→正規化→加權混合）。它還不叫 self-attention（那是 ch07），Q/K/V 也還沒登場（ch04），分數也還沒除以 √d（ch06）；但骨架已經整副現身。讀完你會看出：後面整本書都是在精煉這同一件事。

## 從你已知的出發

你做過資料映射（data mapping）。來源系統送來一筆訂單，欄位叫 `cust_id`、`ship_to`、`line_items`；目標系統要的是 `customerRef`、`deliveryAddress`、`items`。最笨的做法是寫一張**固定的對應表**：來源欄位 A 永遠對到目標欄位 B，寫死在程式裡。這在 schema 穩定時很好。但如果來源格式會變、或一個目標欄位要**綜合好幾個來源欄位**（`deliveryAddress` = `ship_to` + `region` + `postal`）、而且哪幾個來源欄位重要會**隨情境變**，固定對應表就開始崩。

ch02 的 seq2seq 就是這種「寫死的對應表」的極端版：它甚至沒有對應表，它把**整個來源**壓成一個固定向量，然後叫 decoder 從那一個向量裡把所有目標詞變出來。翻譯第 1 個詞、第 10 個詞、第 30 個詞，吃的都是同一個被壓扁的向量。長句一來，需要的細節早就在壓縮時被擠掉了。

Bahdanau 的點子，用你的話講就是：**別寫死對應表，讓系統在生成每個目標詞時，自己去查「來源的哪幾個位置現在最相關」，然後按相關度把它們混合起來**。這正是一次動態的、軟性的查找——和 ch04 要講的「軟性字典查找」是同一個血脈，只是此刻它還叫「對齊（alignment）」。對齊這個詞很傳神：翻譯本來就有「目標詞 ↔ 源詞」的對應關係（英文 "cat" 對齊法文 "chat"），Bahdanau 讓模型**自己算出**這個對齊，而不是人去標。

把它擺進三步骨架：

```text
            ┌──────────┐   ┌────────────┐   ┌──────────────┐
 decoder ─► │ 1. 打分  │─► │ 2. 正規化  │─► │ 3. 加權混合  │ ──► context
 狀態 s     │ 對齊分數 │   │  softmax   │   │  加權源向量  │     向量 c
 源向量 h ───────────────────────────────────────┘（值在此進場）
```

本章三步全到齊，但**主角是「打分」那一步**：Bahdanau 用一個小神經網路打分（加性），Luong 隔年用內積打分（乘性）——後者就是 ch04 起整本書的主幹。我們要看清這兩種打分方式怎麼算、差在哪、為什麼最後是乘性贏了。

## 先看清舊世界的形狀：固定向量到底卡在哪

複習 ch02 一句話：seq2seq 的 encoder 把源句 `x₁…x_T` 讀完，吐出一個固定維度的 context 向量 c（通常就是最後一個 hidden state），decoder 拿著 c 一個一個生出目標詞。

問題的形狀是這樣的（用 ASCII 把資訊流畫出來）：

```text
源句長度 5：
  x1 x2 x3 x4 x5  ──壓縮──►  [ c ]  ──►  y1 y2 y3 ...
                              ↑單一固定向量，所有目標詞共用

源句長度 40：
  x1 x2 ... x40   ──壓縮──►  [ c ]  ──►  y1 y2 ... y40
                              ↑同樣大小，塞 40 個詞的資訊
```

c 的維度不會因為句子變長就變大。把 40 個詞壓進跟 5 個詞同樣大的向量裡，必然有東西被擠掉——而且 decoder 生第 40 個目標詞時，c 裡和句首相關的資訊早被後面的詞覆蓋過了。這就是 ch02 說的「把一本書摘要成一句話再翻譯」。

Bahdanau 等人 2014 的論文標題把藥方寫在標題裡了：**Neural Machine Translation by Jointly Learning to Align and Translate**（arXiv 1409.0473，提交 2014-09，ICLR 2015）——「一邊學對齊、一邊學翻譯」。關鍵動作只有一個：**不要只把最後一個 hidden state 傳給 decoder，把每一個位置的 hidden state 都留著**。

```text
Bahdanau：每個源位置的 hidden state 都保留
  x1 → h1
  x2 → h2          decoder 生 y_i 時，
  x3 → h3   ────►  針對 (h1,h2,h3,h4,h5) 算一組權重 α，
  x4 → h4          按 α 把它們加權混合成「這一步專用的」c_i
  x5 → h5
```

每生一個目標詞，c 不再是同一個——它是**這一步動態算出來的** c_i。生「cat」時權重壓在源句的 "chat" 上，生「black」時權重移到 "noir" 上。固定向量瓶頸被打破，因為根本不再有「一個固定向量」這回事。

> 這裡值得停十秒。Bahdanau 的洞見不是「弄個更聰明的壓縮」，而是**不要壓縮**。把所有源位置都留著，把「該看哪裡」這件事從「壓縮時一次決定」改成「生成時每步動態決定」。整本書的注意力都繼承了這個姿態：不丟資訊，到要用的時候再依相關性取。

那些被保留的 hidden state，Bahdanau 論文叫它們 **annotation（標註向量）**——每個源位置一個 h_j。而且它們來自一個**雙向 RNN**：正向讀一遍得到 →h_j、反向讀一遍得到 ←h_j，把兩個接起來 h_j = [→h_j ; ←h_j]，所以每個 annotation 同時帶著「它左邊」和「它右邊」的上下文（2026-06，已對照 Bahdanau 原文確認）。這個「每個位置都看得到兩邊」的雙向性，會在 ch07 的 self-attention 與 ch11 的遮罩裡變成核心對照——先記著。

## 三步骨架第一次完整現身

Bahdanau 對齊的數學，剝到骨頭就是三步。decoder 此刻有一個內部狀態 s（直覺上是「我下一個想生什麼詞」的當前需求），源句有一排 annotation h₁…h_T。

**第一步，打分（score）。** 對每個源位置 j，算一個「對齊能量（alignment energy）」e_j，衡量「decoder 現在的需求 s，跟源位置 j 有多相關」：

```text
e_j = a(s, h_j)          ← a 是一個「打分函數」，這是本章的主戰場
```

Bahdanau 用的 a 是一個**小神經網路**（一個單隱藏層的前饋網路），這叫**加性注意力（additive attention）**，公式長這樣（2026-06，已對照原文與多份解讀確認形式）：

```text
a(s, h_j) = vᵀ · tanh(W₁ h_j + W₂ s)
            └─┬─┘  └──────┬────────┘
          一個向量      把 s 和 h_j 各自線性變換後相加，
          把結果壓       過 tanh 非線性，得到一個隱藏向量
          成一個純量
```

注意這裡 W₁、W₂、v 都是**學來的**參數（ch04 會把「權重矩陣是學來的」這件事正式講清楚；這裡先接受：訓練讓這個小網路學會「什麼樣的 s 該對什麼樣的 h_j 打高分」）。為什麼叫「加性」？因為它把 W₁h_j 和 W₂s **相加**在 tanh 裡面。

**第二步，正規化（normalize）。** 一排對齊能量 e₁…e_T 還是任意大小的實數，沒法直接當權重。用 softmax 把它們變成一組**和為 1 的非負權重**：

```text
α_j = softmax_j(e_j) = exp(e_j) / Σ_k exp(e_k)
```

這組 α 就是「注意力權重」——它是一個機率分布（呼應《馴服隨機》：和為 1、非負）。softmax 的細節留 ch05 大講，這裡先用它「把分數變成百分比配額」。

**第三步，加權混合（mix）。** 用 α 對所有 annotation 做加權平均，得到這一步專用的 context 向量：

```text
c = Σ_j α_j · h_j          ← 值（這裡 value 就是 annotation h_j 自己）在此進場
```

decoder 拿著這個 c（連同自己的狀態）去生下一個目標詞。生下一個詞時，s 變了，整個三步重算一次，c 也變了。

口訣回扣：**相關性決定權重（打分→softmax），權重決定混合**。Bahdanau 2014 就是這句話的第一個可運作版本。在這個雛形裡，「value」和「key」還沒分家——拿來打分的是 h_j、拿來混合的也是 h_j（同一個 annotation 身兼二職）。ch04 才會把這兩個角色拆開成 K 和 V，並再分出 Q。**先記住這個「尚未分家」的狀態，它是理解 ch04 為什麼要拆三個投影的起點。**

## 加性 vs 乘性：兩種打分方式，同一個位置

Bahdanau 用小神經網路打分（加性）。隔一年，2015，Luong、Pham、Manning 在 **Effective Approaches to Attention-based Neural Machine Translation**（EMNLP 2015）裡問了一個更省的問題：**幹嘛用一個神經網路打分？直接用內積不就好了？**——這就是**乘性注意力（multiplicative attention）**。Luong 給了三種打分函數（2026-06，已對照 Stanford 原文 PDF 與多份解讀確認）：

```text
打分函數 score(s, h_j)：

  dot      :  sᵀ h_j                ← 直接內積。最省，沒有額外參數
  general  :  sᵀ W_a h_j            ← 內積前先過一個學來的矩陣 W_a
  concat   :  vᵀ tanh(W_a [s ; h_j])← 把 s、h_j 接起來再過 MLP（≈ Bahdanau 的加性）
```

`dot` 形式——`sᵀh_j`——就是**內積即相似度**（深入見《矩陣是動詞》ch15）。兩個向量越「同方向」內積越大，分數越高。這正是 ch04 起整本書打分步的標準作法。`general` 是它的小升級：先用 W_a 把 h_j 旋轉伸縮一下再算內積，多一點彈性（你會在 ch04 看到，這個 W_a 的角色後來被 Q/K 的不同投影吸收了）。`concat` 基本上就是 Bahdanau 的加性形式換個名字。

Luong 還區分了 **global vs local attention**：global 每一步都看**所有**源位置（就是上面講的）；local 只看當前對齊點附近的一個小窗口（先猜一個對齊位置，只在它周圍打分），是對「源句很長時 global 太貴」的早期回應。**local attention 是後來稀疏注意力（ch15）的精神先祖**——只看一個窗口而非全部——這條線先埋著。

那加性和乘性到底差在哪、為什麼最後乘性贏？答案藏在 Vaswani 2017 原論文的一個腳註裡（2026-06，已對照原文確認），它一句話講透：

> 加性與乘性在**理論複雜度上相近**；但乘性（dot-product）**在實務上快得多、也省記憶體**，因為它能用**高度優化過的矩陣乘法**實作。

把它翻成你的工程直覺：

| | 加性（additive，Bahdanau） | 乘性（multiplicative，Luong dot） |
|---|---|---|
| 打分怎麼算 | 過一個小 MLP：`vᵀtanh(W₁h+W₂s)` | 直接內積：`sᵀh` |
| 額外參數 | 有（W₁、W₂、v） | dot 形式：沒有 |
| 硬體友善度 | 差——逐對算 MLP，難批次化 | 好——整批變成一次大矩陣乘法 QKᵀ（ch07） |
| 小 d 時表現 | 與乘性相當 | 與加性相當 |
| 大 d 時（未縮放） | 較穩 | 內積值容易變很大，把 softmax 壓死（ch06 的 √d 要救的就是這個） |

關鍵在「硬體友善度」那一列。乘性的好處不是它更聰明，而是**它能把「每個目標位置對每個源位置打分」整批塞進一次矩陣乘法**——而矩陣乘法是 GPU 算得最快的東西（深入見《矩陣是動詞》ch06）。加性的 MLP 打分難以這樣批次化。當 Transformer 要對**所有位置兩兩**打分（ch07 的 n×n）時，這個差別決定生死。所以 Vaswani 選了乘性——代價是大 d 時內積會爆，於是補上 √d 縮放（ch06）。**這是一條清晰的因果鏈：要批次化 → 選內積 → 內積會爆 → 除以 √d。**

> 本章的驚嘆點：Bahdanau 用一個學來的小神經網路打分，看起來更「聰明」；但贏的是更笨、更省的內積。原因不是數學上更對，而是**內積剛好是硬體最會算的東西**。整個 Transformer 時代某種程度上是「選了一個對 GPU 友善的打分函數」的後果——這是工程約束反過來塑造了架構，值得記住。

## 「誰最早發明 attention」：一段該誠實處理的歸屬

技術史很愛問「誰最早」，而「誰最早發明 attention」這題的答案不乾淨，本書照保守版寫、不選邊（2026-06，已對照史料整理）：

- **Bahdanau, Cho & Bengio 2014** 是現代共識裡「在 NMT 脈絡正式確立 attention mechanism 這個詞與機制」的人。本章講的就是它。要說「注意力是哪篇論文確立的」，指它沒問題。
- 但**更早的可微分注意力雛形**確實存在。**Alex Graves 2013**（Generating Sequences With RNNs，arXiv 1308.0850）在手寫合成裡用了高斯核混合的可微分「位址」機制，可視為注意力的雛形——只是他沒把它命名成一個獨立的 mechanism。隔年 Graves 等人的 **Neural Turing Machines（2014，arXiv 1410.5401）** 用可微分注意力讀寫一塊外部記憶，那是「軟性內容定址記憶體」的直接前身——這條線會在 ch18（注意力即聯想記憶）回扣。
- **Jürgen Schmidhuber** 則主張他 1991–1992 的「Fast Weight Programmers」更早就有相關想法。這個歸屬爭議在學界反覆出現；查檢其早期論文，並未明確使用「attention mechanism」這個術語。

所以本書的寫法是：**機制的「確立」歸 Bahdanau 2014，並誠實提一句 Graves 更早的可微分雛形與 Schmidhuber 的爭議，不替任何一方下判決。** 這不是和稀泥——技術史本來就常是「想法在多處同時萌芽、被某一篇論文清楚命名後才定型」。對你寫程式的人來說這很熟：一個 pattern 常常很多人各自摸到過，但要等到有人把它寫成一個有名字的抽象（像 "circuit breaker"、"saga"），它才算「被發明」。Bahdanau 之於 attention，就是那個「把它命名清楚」的時刻。

## Worked example：迷你翻譯對齊（手算）

把三步骨架用具體數字跑一遍。場景：源句法文 **"le chat noir"**（3 個詞，黑貓），decoder 此刻要生英文的 **"cat"**。

**設定（本章專屬小例子，非脊椎那組 4 維數字——脊椎從 ch04 開始）。** encoder 給三個 annotation（2 維，方便手算），decoder 當前狀態 s 代表「我現在想找『貓』這個概念」：

```text
源 annotation：
  h₁ = (1, 1)   ← "le"   （冠詞）
  h₂ = (2, 0)   ← "chat" （貓，目標詞該對齊這裡）
  h₃ = (0, 2)   ← "noir" （黑）

decoder 狀態：
  s  = (2, 0)   ← 「我在找跟『貓』有關的源位置」
```

**第一步，打分（用乘性 dot：score = sᵀh_j）。** 逐個算內積：

```text
e₁ = s·h₁ = (2)(1) + (0)(1) = 2
e₂ = s·h₂ = (2)(2) + (0)(0) = 4   ← 最高，s 與 "chat" 最同方向
e₃ = s·h₃ = (2)(0) + (0)(2) = 0
```

對齊能量 = [2, 4, 0]。"chat" 拿到最高分，符合直覺（找貓就該對齊貓）。

**第二步，正規化（softmax）。** 算 e 的次方、求和、相除（e≈2.71828）：

```text
exp(2) = 7.389
exp(4) = 54.598
exp(0) = 1.000
                 ── 和 = 7.389 + 54.598 + 1.000 = 62.987

α₁ = 7.389 / 62.987 = 0.117
α₂ = 54.598 / 62.987 = 0.867   ← 絕大部分權重壓在 "chat"
α₃ = 1.000 / 62.987 = 0.016
```

注意力權重 α = [0.117, 0.867, 0.016]，和為 1。（如果這組數字看起來眼熟——對，它和 ch05 用未縮放脊椎分數 [4,2,0] 算出的 [0.867, 0.117, 0.016] 是同一組 e 值、只是位置不同。softmax 的機械是一模一樣的，先在這裡見一面。）

**第三步，加權混合。** 用 α 對三個 annotation 加權平均，得到這一步的 context 向量：

```text
c = 0.117·(1,1) + 0.867·(2,0) + 0.016·(0,2)

  第一維： 0.117·1 + 0.867·2 + 0.016·0 = 0.117 + 1.734 + 0     = 1.851
  第二維： 0.117·1 + 0.867·0 + 0.016·2 = 0.117 + 0     + 0.032 = 0.149

  c ≈ (1.851, 0.149)
```

context 向量 c ≈ (1.851, 0.149)，幾乎就是 h₂ = (2,0) 的方向——也就是 "chat"。decoder 拿著這個 c 去生 "cat"，它收到的源資訊**主要來自 "chat" 這個位置**，不是被三個詞攪成一團的固定向量。瓶頸破了。

對齊熱圖（生 "cat" 這一步，三個源位置的權重濃淡）：

```text
              le    chat   noir
  生 "cat" → ░░    ████    ·       （0.117 / 0.867 / 0.016）
```

「cat 該看 chat」這件事，是**算**出來的，不是規定的——這正是全書脊椎 it 的同款故事，只是脊椎那題問的是同一句話裡的代名詞（ch04 起）。

**加性版本長什麼樣（為什麼這裡只算一格）。** 同樣這組向量，如果用 Bahdanau 的加性打分 `vᵀtanh(W₁h_j + W₂s)`，得先有訓練好的 W₁、W₂、v 才有意義——它們是**學來的**，不能隨手填數字假裝算出對的對齊。為了讓你看見 tanh 那一步是怎麼動的，我們只追一格、且用最簡的 W₁=W₂=單位矩陣、v=(1,1)（這只是示範機械，不是真的學好的權重，所以別拿它的結果當「正確對齊」）：

```text
對 h₂="chat"：
  W₁h₂ + W₂s = (2,0) + (2,0) = (4, 0)
  tanh(4, 0)  = (0.999, 0.000)        ← tanh 把大值壓到接近 ±1（飽和）
  vᵀ·(...)    = 1·0.999 + 1·0.000 = 0.999
```

看到關鍵差別了嗎：加性的 tanh **天生會飽和**——輸入再大，輸出頂多到 ±1，三個源位置的分數因此被壓在差不多的範圍，softmax 後較「平」。乘性的內積**沒有上限**，分數差距可以拉得很開（我們上面 dot 算出 [2,4,0]，softmax 後 "chat" 直接吃掉 0.867）。這個「內積無上限」既是乘性的力量（能形成尖銳的對齊），也是它的風險（大 d 時太尖、把 softmax 壓進飽和區）——ch06 的 √d 就是來馴服這個風險的。

## 故障模式與直覺陷阱

| 陷阱 | 錯誤直覺 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| 「Bahdanau 注意力 = self-attention」 | 以為這就是 Transformer 那套 | 把「誰看誰」搞錯——這裡是 **decoder 看 encoder（跨序列）**，不是「序列看自己」 | 問自己：打分時 query 和 key 是同一個序列嗎？這裡 s 來自目標端、h 來自源端，**不同序列**，所以這是 cross 形態的雛形（ch12），不是 self（ch07） |
| 「注意力選出最相關的那一個源詞」 | 以為 α 是個 argmax、挑一個贏家 | 第三步——以為 c = h₂，其實 c = **所有** h 的加權混合（c ≈ (1.851, 0.149) 裡有 h₁、h₃ 的零頭） | 看 α 是不是 one-hot。它不是；它是分布，永遠混合全部，只是相關的權重大（ch05 把這個反直覺講到底） |
| 「對齊權重 = 模型的可解釋翻譯對應」 | 把 α 熱圖當「模型為什麼這樣翻」的證明 | 把相關性當因果——熱圖好看不代表它忠實反映決策 | 記住這是 ch20 全書故障視角的高潮（「attention 是解釋嗎」至今未定論）。對齊圖**可能合理但未必忠實** |
| 「加性比較笨、乘性比較聰明（或反之）」 | 以為其中一個數學上更對 | 選打分函數時用錯理由 | 它們在小 d 表現相當；乘性贏在**硬體友善**（能批次成矩陣乘法），不是更聰明。代價是大 d 要補 √d（ch06） |
| 「加性的 tanh 沒副作用」 | 忽略 tanh 飽和 | 以為加性分數能無限拉開 | tanh 上限 ±1，分數差距被天花板壓住——這也是為什麼加性在大 d 時反而比未縮放乘性穩 |

最值得單獨點名的失效是**第一個**：把 Bahdanau 當成 self-attention。它們的三步骨架一模一樣，但「誰當 query、誰當 key/value」完全不同。Bahdanau 是**目標端的 decoder 狀態 s 當查詢者，去看源端的 annotation**——兩個序列、一個查一個。self-attention（ch07）是**序列看自己**（每個位置同時是查詢者和被查者）。本章是 cross 形態的祖先，self 形態還要等到 ch07。把這個「誰看誰」記在心上，後面三種注意力（ch12）就不會亂。

## 紙上推演

### 推演題 1：加性與乘性各算一次，比較 **[20 分鐘] ★★**

用同一組向量，把乘性（dot）的對齊權重完整算出來，並把加性的單格 tanh 追一遍。

- decoder 狀態 s = (1, 2)
- annotation：h₁ = (2, 0)、h₂ = (0, 2)、h₃ = (1, 1)

(a) 用乘性 `score = s·h_j` 算三個對齊能量。
(b) 對它們做 softmax（寫出 exp 值、和、三個權重）。
(c) 算 context 向量 c = Σ α_j h_j。
(d) 用加性（W₁=W₂=I、v=(1,1)）追 h₂ 那一格的 tanh 分數，說一句它和乘性那格的差別在哪。

### 推演題 2：對齊為何打破瓶頸 **[10 分鐘] ★**

口頭（或寫 150 字內）回答：seq2seq 把整句壓成一個固定向量 c；Bahdanau 改成每步算一個 c_i。用「生第 30 個目標詞」這個具體情境，講清楚 Bahdanau 拿到的資訊為什麼比 seq2seq 多、多在哪。不准只說「因為有注意力」——要講到「保留所有 annotation」「動態加權」這兩個機制。

### 推演題 3：誰看誰 **[10 分鐘] ★★**

把 Bahdanau 對齊和「序列看自己（self-attention，ch07 預告）」用一張表比較：誰當查詢者（query 的來源）、誰當被查者（key/value 的來源）、是不是同一個序列。然後用一句話說：為什麼說 Bahdanau 是「cross 形態的雛形」而不是 self。

---

### 推演解答

**題 1（a）打分。** 內積：

```text
e₁ = s·h₁ = (1)(2)+(2)(0) = 2
e₂ = s·h₂ = (1)(0)+(2)(2) = 4
e₃ = s·h₃ = (1)(1)+(2)(1) = 3
```

對齊能量 = [2, 4, 3]。

**（b）softmax。**

```text
exp(2) = 7.389
exp(4) = 54.598
exp(3) = 20.086
                 ── 和 = 7.389 + 54.598 + 20.086 = 82.073

α₁ = 7.389  / 82.073 = 0.090
α₂ = 54.598 / 82.073 = 0.665
α₃ = 20.086 / 82.073 = 0.245
```

權重 [0.090, 0.665, 0.245]，和為 1。（眼熟嗎？這又是脊椎正典那組 e 值——softmax 的機械永遠一樣，只是分數排列不同。）

**（c）context。**

```text
c = 0.090·(2,0) + 0.665·(0,2) + 0.245·(1,1)
  第一維：0.090·2 + 0.665·0 + 0.245·1 = 0.180 + 0 + 0.245 = 0.425
  第二維：0.090·0 + 0.665·2 + 0.245·1 = 0 + 1.330 + 0.245 = 1.575
  c ≈ (0.425, 1.575)
```

c ≈ (0.425, 1.575)，主要朝 h₂ = (0,2) 方向，符合「h₂ 拿最高權重 0.665」。注意 c 不等於 h₂——它混了 h₃（0.245）和一點 h₁（0.090），這就是「混合全部、不是挑一個」。

**（d）加性單格。** 對 h₂：W₁h₂ + W₂s = (0,2) + (1,2) = (1, 4)；tanh(1, 4) = (0.762, 0.999)；vᵀ·(...) = 0.762 + 0.999 = 1.761。差別：乘性那格 e₂ = 4（無上限，可以很大）；加性過 tanh 後每個分量被壓在 ±1 內，加總頂多到 +2，分數天生「擠在一起」，softmax 後較平。乘性能拉出更尖的對齊，但大 d 時這個「尖」會失控（ch06）。

**題 2。** 要點：seq2seq 生第 30 個詞時，手上只有那一個固定向量 c——而 c 是把整句壓出來的，第 1 個源詞的資訊早被後 39 個詞覆寫得差不多了。Bahdanau 把**每個源位置的 annotation h_j 全部留著沒壓**，生第 30 個目標詞時，decoder 用當下的狀態 s 對這 T 個 annotation **重新打一次分**、算出專屬這一步的 c₃₀。所以它能在這一步把權重精準壓到「跟第 30 個目標詞相關的那幾個源詞」上，不受其他詞稀釋。兩個機制缺一不可：**保留所有 annotation**（資訊沒被壓掉）＋**每步動態加權**（按當下需求取用）。

**題 3。**

```text
              查詢者(query)來源   被查者(key/value)來源   同一序列？
  Bahdanau    decoder 狀態 s      encoder annotation h    否（目標 vs 源）
  self-attn   序列自己的位置      序列自己的位置          是
```

Bahdanau 是「目標端回頭查源端」，兩個不同的序列，一邊查一邊。這正是 ch12 講的 **cross-attention**（交叉注意力）的雛形——Q 來自一端、K/V 來自另一端。self-attention（ch07）則是序列看自己，每個位置既是查詢者也是被查者，Q/K/V 同源。所以 Bahdanau 是 cross 的祖先，不是 self。一句話：**Bahdanau 教會了模型「回頭看別人」，Transformer 的 self-attention 再進一步教它「也回頭看自己」。**

## 自我檢核

口頭自答，講得出來才算過：

1. 用三步骨架（打分→正規化→加權混合）把 Bahdanau 對齊講一遍，每一步對應論文裡的哪個量（e_j、α_j、c）？

3. 加性與乘性打分各怎麼算？為什麼最後是乘性（內積）成為 Transformer 的主幹——關鍵理由是「更聰明」還是別的？
4. 乘性的內積沒有上限，這既是力量也是風險。風險是什麼？哪一章用什麼數字來馴服它？
5. 為什麼說 Bahdanau 是 cross-attention 的雛形而不是 self-attention？「誰當 query、誰當 key/value」在這裡是怎麼分的？
6. 在 Bahdanau 雛形裡，「key」和「value」還沒分家——同一個 annotation h_j 既拿來打分又拿來混合。ch04 為什麼要把它們拆開？（先給你能想到的直覺，ch04 會驗證。）
