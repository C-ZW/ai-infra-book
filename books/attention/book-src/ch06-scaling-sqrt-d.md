# ch06 — 為什麼除以 √d：一個防止注意力崩掉的小數字

> **本章解決什麼問題**：ch05 用未縮放的分數 [4,2,0] 把三步骨架跑完一次，得到很尖的權重 [0.867, 0.117, 0.016]。但縮放點積注意力的公式裡明明寫著 `softmax(QKᵀ / √d_k)`——那個 `/√d_k` 是哪來的、為什麼非要不可？這一章只談這一個動作：在「打分」與「正規化」之間插進一個除法。它動的不是 query/key/value，也不是 softmax 本身，而是**送進 softmax 的分數的尺度**。看懂它，你就知道為什麼一個看起來像補丁的小常數，是讓深層 transformer 學得動的關鍵；也會在本章把全書的脊椎權重從 ch05 的「尖版」校正成 ch07 起一律沿用的「正典版」。

## 從你已知的出發

你一定看過這種場景：一個排序鍵、一個分數、一個 hash，原本好好的，資料量一變、維度一加，數值就「炸」了。整數溢位、浮點數失去精度、某個權重吃掉全部流量。除以 √d 解決的就是同一類問題——**尺度（scale）失控**。

最貼切的橋接是你做加權路由（weighted routing）或加權隨機抽樣時遇過的事。假設你有三個後端服務，你給它們算「健康分數」，再用 softmax（或任何指數型權重）把分數轉成流量配比。分數本身只要相對關係對就好：A=4、B=2、C=0，和 A=400、B=200、C=0，相對排序一模一樣。但你把它們各自丟進 `exp` 再正規化，結果天差地別：

```text
softmax([4, 2, 0])    = [0.867, 0.117, 0.016]   ← A 拿八成多，還留了給 B、C 一點
softmax([400, 200, 0]) = [1.000, 0.000, 0.000]  ← A 拿掉「全部」，B、C 形同不存在
```

第二種情況下，你的 router 失去了調節能力：哪怕 B 的分數從 200 漲到 250、甚至追到 399，流量配比幾乎一動也不動，永遠是 A 獨吞。你會說：「分數的**絕對大小**不該決定我把多少權重給次優選項，但它確實決定了。」這正是 softmax 的脾氣——它對輸入的**尺度極度敏感**。分數整體放大，分布就被推向 one-hot；分數整體縮小，分布就被推向均勻。

注意力裡的內積分數，正好有一個讓尺度不受控膨脹的源頭：**維度 d**。維度愈高，q·k 這個內積的典型大小就愈大（待會我們會算給你看，是 √d 倍）。所以 Vaswani 那篇論文在算完 QKᵀ、丟進 softmax 之前，插了一個除法把尺度拉回來。這就是「縮放點積注意力（scaled dot-product attention）」裡那個「縮放」的全部含義——不是縮放 query、不是縮放 value，是縮放**分數**，好讓 softmax 在一個「還聽得進話」的尺度上工作。

在三步骨架裡，這一步坐在哪？

```text
            ┌──────────┐   ┌────────────┐   ┌──────────────┐
 query ──►  │ 1. 打分  │─► │ 2. 正規化  │─► │ 3. 加權混合  │ ──► 輸出
 keys  ──►  │  score   │   │  softmax   │   │ weighted sum │
 values ─────────────────────────────────────────┘
                  │
                  └─►【÷ √d】插在打分之後、正規化之前
```

`/√d` 不是獨立的第四步，它是「打分」與「正規化」之間的**接合面校正**：把第一步算出的原始分數，校正到第二步（softmax）能正常運作的尺度。本章從頭到尾只動這一個地方。

## 先把問題演一遍：尺度怎麼把 softmax 壓死

ch05 已經讓你看過 softmax 對尺度的敏感。這裡把它推到極端，讓你親眼看到「壓死」長什麼樣。

回到脊椎——序章那個句子 *"the animal didn't cross the street because **it** was too tired."* 簡化成 [animal, street, it]。ch04 算出 it 這個位置的 query 對三個 key 的原始內積分數是：

```text
q_it · k_animal = 4
q_it · k_street = 2
q_it · k_it     = 0
                          原始分數 [4, 2, 0]
```

ch05 對這組分數做 softmax，得到 [0.867, 0.117, 0.016]——已經很尖，animal 拿八成多。現在做一個思想實驗：假設我們的維度不是 4 而是更大，內積分數會等比例膨脹（先接受這個結論，下一節證明）。把分數放大成原來的兩倍 [8, 4, 0]、四倍 [16, 8, 0]，看 softmax 怎麼變：

```text
softmax([4,  2, 0]) = [0.867,    0.117,    0.016  ]   ← 尖，但 street/it 還在
softmax([8,  4, 0]) = [0.982,    0.018,    0.0003 ]   ← 更尖，street 幾乎沒了
softmax([16, 8, 0]) = [0.999665, 0.000335, 0.0000]   ← 等於 one-hot，只剩 animal
```

（你可以自己核：softmax([16,8,0]) 先減最大值算 e⁰=1、e⁻⁸≈0.000335、e⁻¹⁶≈0.0000001，和≈1.000335，相除即得。）

到了 [16, 8, 0] 這一步，分布幾乎就是 one-hot：animal=0.9997，其餘趨近 0。表面上看「它非常果斷地選了 animal」似乎是好事——但這對一個**還在學習**的模型是災難。原因要分兩面講，而且第二面才是真正致命的。

### 第一面：飽和（saturation）讓 softmax 失去調節能力

當一個權重 → 1、其餘 → 0，softmax 進入**飽和區（saturation region）**。在飽和區裡，分數稍微變一點，輸出幾乎不動——就像你那個 router 在分數 [400,200,0] 時，B 從 200 漲到 399 流量也不變。模型表達「我有 70% 把握看 animal、30% 還想參考 street」這種**柔和、混合**的能力被抹平了，只剩「全有或全無」。注意力的精髓（ch05 的招牌反直覺）是**混合全部、相關的權重大**；飽和把「混合」退化成「硬挑一個」，等於把 attention 變回了一個不可微的 argmax。

### 第二面：飽和 = 梯度消失（這才是致命的）

這一面需要一點訓練的鷹架，但只到「夠用」為止，反向傳播的完整數學指向《馴服無限》（微積分那本講梯度）。

模型怎麼「學」？簡化說：它看自己的輸出錯多少，然後**沿著梯度（gradient，輸出對參數的敏感度）往回調**權重矩陣 W_Q、W_K。梯度大，代表「這個參數稍微動一下，輸出就明顯變好或變壞」，模型就知道該往哪調、調多少。梯度趨近 0，代表「這個參數怎麼動，輸出都幾乎不變」——模型收不到任何訊號，**學不動了**。這就是「梯度消失（vanishing gradient）」。

softmax 在飽和區的梯度有多小？softmax 對它輸入分數的敏感度，對角項正比於 `p_i·(1−p_i)`、交叉項正比於 `−p_i·p_j`（這是 softmax 的 Jacobian，《馴服無限》會推；這裡只用結論）。關鍵是：

```text
當 p_i → 1：p_i·(1−p_i) → 1·0 = 0
當 p_i → 0：p_i·(1−p_i) → 0·1 = 0
```

不管權重衝到 1 還是被壓到 0，這個敏感度都 → 0。也就是說，**一旦 softmax 飽和成 one-hot，它對所有分數的梯度都趨近 0**。訊號傳不回去，W_Q、W_K 收不到「該怎麼調投影」的指引——這一層注意力就學不動了。`p_i(1−p_i)` 的最大值出現在 p=0.5（值 0.25），也就是分布最「猶豫」的時候梯度最大、學得最快；愈接近 one-hot，學得愈慢，直到停擺。

> **這裡了不起／反直覺在哪**：你會直覺以為「分數差距愈大、模型愈果斷愈好」。錯。對一個還沒學好的模型，**過早的果斷是學習的死亡**。它在還沒搞清楚 it 到底該看哪裡之前，就被尺度逼成 one-hot、梯度歸零、永遠卡在當下這個（可能是錯的）選擇。除以 √d 的全部目的，就是**別讓模型在學會之前就先把話說死**。

把這兩面合起來：高維度 → 內積分數膨脹 → softmax 飽和 → 既失去混合能力、又梯度消失。`/√d` 是來掐住第一環「分數膨脹」的。

## 為什麼分數會隨維度膨脹：變異數論證

前面一直說「維度愈高、內積分數愈大」。現在嚴格一點把它算出來。這是 Vaswani 2017 §3.2.1 那個著名腳註的內容（landscape §2 已錄原文），我們自己重推一遍，不抄記憶。

**設定（這是論證的前提，要明說）**：假設 query 向量 q 和 key 向量 k 的每一個分量，都是**獨立**的隨機變數，期望值 0、變異數 1。這是個理想化假設（真實的 q、k 是學出來的、未必獨立同分布），但它抓住了核心直覺：**內積是 d 個乘積項的和，項數愈多，和的波動愈大**。

內積是 `q·k = Σᵢ qᵢkᵢ`（i 從 1 到 d）。我們要算這個和的變異數。

**第一步，算單一項 qᵢkᵢ 的期望值與變異數。**
- 期望值：因為 qᵢ、kᵢ 獨立且各自期望值 0，`E[qᵢkᵢ] = E[qᵢ]·E[kᵢ] = 0·0 = 0`。
- 變異數：期望值是 0，所以 `Var(qᵢkᵢ) = E[(qᵢkᵢ)²] = E[qᵢ²]·E[kᵢ²]`（再次用獨立）。而 `E[qᵢ²] = Var(qᵢ) + (E[qᵢ])² = 1 + 0 = 1`，kᵢ 同理 = 1。所以 `Var(qᵢkᵢ) = 1·1 = 1`。

**第二步，算 d 項之和的變異數。**獨立隨機變數相加，變異數可以直接相加（這是《馴服隨機》講過的變異數可加性）：

```text
Var(q·k) = Var(Σᵢ qᵢkᵢ) = Σᵢ Var(qᵢkᵢ) = d × 1 = d
                                              ↑
                                    d 個各自變異數 1 的項相加
```

結論：**內積 q·k 的期望值是 0，變異數是 d。** 變異數是 d，代表它的標準差（典型波動幅度）是 **√d**。換句話說，內積分數的典型大小，隨維度以 **√d** 的速率成長。

```text
d = 4   → 標準差 √4  = 2     （脊椎用的維度）
d = 64  → 標準差 √64 = 8     （原始 Transformer 每顆頭的 d_k）
d = 128 → 標準差 √128≈ 11.3
```

我用隨機模擬核對過這件事：d=4 時，大量 q·k 樣本的變異數約 3.99；d=64 時約 63.9——和理論值 d 吻合。

現在問題清楚了：你用 d=64 的維度，內積分數的典型量級就在 ±8 上下，分數之間動輒差好幾，softmax 一吃就飽和（回想上一節 [16,8,0] 已經是 one-hot 了）。**維度愈高，這個問題愈嚴重**。而我們又偏偏想要高維度（表達力更強）。矛盾在這。

### 解法與「為什麼是 √d 不是 d」

解法極簡：把分數除以它自己的標準差，讓變異數回到 1。標準差是 √d，所以——

```text
Var(q·k / √d) = Var(q·k) / (√d)² = d / d = 1
                                   ↑
                       除以常數 c，變異數除以 c²
```

除以 √d，變異數從 d 拉回 1，不管維度多大都一樣。這就是公式 `softmax(QKᵀ / √d_k)` 裡 `/√d_k` 的全部來歷。

**為什麼是 √d 而不是 d？** 這是最常被問、也最容易想錯的一點。直覺陷阱是：「變異數是 d，那除以 d 不就把變異數變成 1/d、更穩了嗎？」會錯，是因為你要校正的是**標準差（分數的典型大小、尺度）**，不是變異數本身。

- 變異數的單位是「分數的平方」，標準差才和分數同一個單位（同一個尺度）。要把一個量「縮回單位尺度」，要除以它的**標準差**，也就是 √d。
- 除以 d（過度縮放）會把標準差打成 `√d / d = 1/√d`，變異數變成 1/d——分數被壓得太小、太擠在一起，softmax 反而趨近**均勻分布**，注意力變得「什麼都看一點、什麼都看不清」，一樣學不好（只是壞在另一端）。
- 除以 √d（剛好）讓標準差 = 1、變異數 = 1，落在 softmax 既不飽和、也不糊成一團的甜蜜點。

一句話記住：**√d 是分數的標準差，除以標準差就是把分數的尺度歸一回 1。** 不是隨便挑的常數，是被變異數論證唯一指定的那一個。

> 嚴謹度標示：這是**直覺版＋一階論證**。「q、k 各分量獨立、期望值 0、變異數 1」是理想化假設，真實模型未必嚴格成立（LayerNorm 會幫忙把尺度拉近，見 ch13）。但這個論證抓住了正確的**標度律（scaling law）**——分數尺度 ∝ √d——而這正是 √d 縮放有效的原因。嚴格的隨機分析指向延伸閱讀。

## 脊椎校正：從 ch05 的「尖版」到 ch07 的「正典版」

現在把 √d 用回脊椎，把全書要用一輩子的那組權重定下來。脊椎維度 **d = 4**，所以 **√d = √4 = 2**。

ch04 的原始分數是 [4, 2, 0]。除以 √d = 2：

```text
原始分數          [4,   2,   0  ]
÷ √d = ÷2     →   [2,   1,   0  ]    ← 縮放後的分數
```

對縮放後的 [2, 1, 0] 做 softmax，一步都不跳：

```text
e² = 7.389
e¹ = 2.718
e⁰ = 1.000
              和 = 7.389 + 2.718 + 1.000 = 11.107

softmax 權重：
  animal: 7.389 / 11.107 = 0.665
  street: 2.718 / 11.107 = 0.245
  it    : 1.000 / 11.107 = 0.090
                                    縮放後權重 [0.665, 0.245, 0.090]
```

（核對：0.665 + 0.245 + 0.090 = 1.000 ✓，是個合法機率分布。）

再走第三步「加權混合」，用 ch04 的 value：v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)：

```text
輸出 = 0.665·(1,0) + 0.245·(0,1) + 0.090·(1,1)
     = (0.665, 0)  + (0, 0.245)  + (0.090, 0.090)
     = (0.665+0.090,  0.245+0.090)
     = (0.755, 0.335)
                                    縮放後輸出 ≈ (0.755, 0.335)
```

**本章正式宣告**：縮放後的權重 **[0.665, 0.245, 0.090]** 與輸出 **(0.755, 0.335)** 是全書的**正典脊椎數字**。從 ch07 起的每一章（自注意力矩陣、多頭、遮罩、核迴歸、Hopfield、QK/OV 電路……）一律用這一組，不再用 ch05 那個未縮放的尖版。ch05 的 [0.867, 0.117, 0.016] 從此只當「對照組」，提醒你縮放在做什麼。

並排看「尖 → 軟」的對照：

```text
            animal   street   it       這一列的形狀
未縮放 [4,2,0]   0.867    0.117    0.016    尖（animal 八成多，幾乎獨吞）
縮放   [2,1,0]   0.665    0.245    0.090    軟（animal 主導，但 street 還有兩成多發言權）
```

熱圖看得更直接（█ 最濃、░ 最淡，濃度對應權重大小）：

```text
未縮放：  it ► animal ████████▓  street █░         it ░
縮放：    it ► animal ███████    street ██▒        it █░
                      ↑偏向 animal 都成立，但縮放版給了 street/it 該有的發言權
```

兩個版本都「偏向 animal」——這是好事，代表縮放沒有破壞 it→animal 的正確傾向。縮放改變的不是**誰最大**，而是**最大者贏多少**：它把過度自信的 0.867 拉回到還留有餘地的 0.665，讓 street 的 0.245 和 it 的 0.090 重新有了發言權。對訓練中的模型，這個「餘地」就是梯度能流動、還能繼續學的空間。

注意：在脊椎的小尺度（d=4，分數量級 ±2）裡，未縮放其實也沒到 one-hot 那麼慘——0.867 還不算飽和死。√d 縮放的威力要在 d=64 那種真實尺度才看得驚心。下一節把脊椎放大到那個尺度演給你看。

## 把脊椎放大到 d=64：看不除會死得多難看

脊椎 d=4 太小，看不出 √d 是救命還是錦上添花。原始 Transformer 每顆頭的維度是 **d_k = d_model/h = 512/8 = 64**（landscape §2），對應 **√d_k = 8**。在這個尺度下，內積分數的典型量級在 ±8 上下，分數兩兩相差好幾是常態。

做個對齊脊椎的放大實驗：假設在 d=64 的世界裡，it 對三個 key 的原始內積分數，量級被 √d_k = 8 倍放大，大約落在 **[16, 8, 0]**（保持和脊椎一樣的相對形狀 4:2:0，只是整體乘了 8 倍——這正是變異數論證預測的）。

```text
不除 √d_k：softmax([16, 8, 0]) = [0.999665, 0.000335, 0.0000]
                                  └────────── 幾乎就是 one-hot ──────────┘
            → softmax 飽和、梯度 ≈ 0、模型學不動

除以 √d_k = 8：[16,8,0] / 8 = [2, 1, 0]
            softmax([2, 1, 0]) = [0.665, 0.245, 0.090]
            → 回到脊椎正典，軟、可微、學得動
```

看到沒——**同一組相對分數**，不除以 √d_k，softmax 把它壓成 one-hot（梯度近 0、學不動）；除以 √d_k = 8，剛好回到脊椎那組健康的 [0.665, 0.245, 0.090]。`/√d_k` 就是把右邊那個「分數量級被維度膨脹了 8 倍」的問題，精確地除回去。維度愈高，膨脹愈狠，這個除法愈不可或缺。這也是為什麼縮放在 d=4 的玩具脊椎裡只是「讓分布軟一點」，在 d=64 的真實模型裡卻是「能不能訓練」的生死線。

如果你想看演算法骨架，scaled dot-product attention 的核心就五行，那個 `/ np.sqrt(d_k)` 就是本章的全部主角：

```python
import numpy as np

def scaled_dot_product_attention(Q, K, V):
    d_k = K.shape[-1]                      # the dimension we just argued about
    scores = Q @ K.T / np.sqrt(d_k)        # score, then SCALE: this one division is ch06
    weights = softmax(scores, axis=-1)     # normalize (ch05)
    return weights @ V                      # weighted mix (ch04/ch05)
```

用脊椎餵進去（Q 是 q_it、K 是三個 key、d_k=4）：`scores = [4,2,0]/2 = [2,1,0]`，softmax 出 `[0.665, 0.245, 0.090]`，乘 V 得 `(0.755, 0.335)`——和我們上面手算的正典數字一模一樣。程式和手算對得上，這就是本章唯一新增的那行 `/ np.sqrt(d_k)`。

## 故障模式與直覺陷阱

| 直覺陷阱 / 故障 | 為什麼錯／在哪一步把你帶溝裡 | 怎麼自我察覺、正確版是什麼 |
|---|---|---|
| 「除以 √d 是為了數值溢位（overflow）」 | 部分對但抓錯重點。溢位是用「減去最大值」那招防的（見 ch16 online softmax）。`/√d` 的目的是**統計尺度**——把分數變異數拉回 1，防 softmax 飽和、梯度消失，不是防 `exp` 算爆。 | 問自己：減最大值已經能防溢位了，為什麼還要除 √d？因為兩者治不同的病：減最大值治「`exp` 數值爆」，除 √d 治「softmax 太尖、學不動」。 |
| 「應該除以 d，因為變異數是 d」 | 在第二步（正規化）之前把分數壓過頭。你要校正的是**標準差（尺度）**＝√d，不是變異數。除以 d 會讓變異數變 1/d，分數擠成一團、softmax 趨近均勻、注意力糊掉。 | 記「除以標準差才是歸一尺度」。變異數 d → 標準差 √d → 除 √d。除 d 是過度縮放。 |
| 「分數差距愈大、模型愈果斷愈好」 | 對**訓練中**的模型是毒。過早的 one-hot＝梯度消失＝學習停擺（p_i(1−p_i)→0）。模型還沒學會就把話說死，永遠卡在當下選擇。 | 看權重是不是「太早變 one-hot」。健康的注意力在訓練早期應該是軟的、留餘地的；果斷是學會之後的結果，不是手段。 |
| 「縮放會改變 it 到底看哪裡（誰最大）」 | 不會。縮放是對所有分數除同一個正常數，**不改變排序**——animal 仍是最大。縮放只改「最大者贏多少」（0.867→0.665），不改「誰最大」。 | 並排看脊椎兩版：兩版 animal 都最大。縮放動的是分布的**尖銳度**，不是 argmax。 |
| 把 `/√d` 當成獨立的演算法步驟 | 它不是第四步，是「打分」和「正規化」之間的尺度校正，融在公式 `softmax(QKᵀ/√d)` 裡。當成獨立步驟容易忘了它純粹服務 softmax。 | 回扣三步骨架：√d 不增刪步驟，它只是讓第一步的產物以正確尺度進入第二步。 |
| 「飽和只在 d 很大時才發生，小模型不用管」 | 危險的「應該沒事」假設。即使 d 不大，只要 q、k 的尺度（embedding 沒正規化好、或某些維度數值偏大）失控，分數一樣會膨脹進飽和區。√d 是**對維度**的系統性校正，不是唯一的尺度防線。 | LayerNorm（ch13）也在管尺度。√d 治「維度造成的膨脹」，LayerNorm 治「逐層數值漂移」，兩者互補，缺一仍可能飽和。 |

**和後面章節的鉤子**：softmax 飽和導致的「深層難訓練」不是只靠 √d 就解決的。把很多層注意力疊起來，還會遇到更根本的退化——**秩坍縮（rank collapse）**：純注意力堆疊，輸出會雙指數地塌成秩 1（Dong 2021，ch19 詳述）。√d 處理單層 softmax 的尺度，**殘差連接與 FFN**（ch13）才是阻止跨層坍縮的解藥。本章先把單層這關過了。

## 紙上推演

### 推演題

**第 1 題 [10 分鐘]（★）** 親手把脊椎縮放算完整。給原始分數 [4, 2, 0]、d=4：(a) 算 √d，寫出縮放後分數；(b) 一步不跳算 softmax（e²、e¹、e⁰、求和、相除）得三個權重；(c) 用 v_animal=(1,0)、v_street=(0,1)、v_it=(1,1) 算加權混合輸出；(d) 和 ch05 未縮放的 [0.867,0.117,0.016] 並排，一句話說縮放改變了什麼、沒改變什麼。

**第 2 題 [15 分鐘]（★★）** 變異數論證的核心。(a) 用「q、k 各分量獨立、期望值 0、變異數 1」的假設，說明為什麼單一項 qᵢkᵢ 的期望值是 0、變異數是 1；(b) 為什麼 d 項之和的變異數是 d（用了什麼性質）；(c) 據此說明「為什麼除以 √d 而不是除以 d」——若除以 d 會發生什麼？

**第 3 題 [12 分鐘]（★★）** 飽和與梯度消失。手算 softmax([16, 8, 0])（提示：先減最大值 16 再算，e⁰=1、e⁻⁸≈0.000335、e⁻¹⁶≈0），看它多接近 one-hot；然後用「softmax 的敏感度正比於 p_i(1−p_i)」說明：當 animal 權重 ≈ 0.9997 時，模型對分數的梯度大約是多少量級？這對「模型還能不能繼續學 it 該看哪裡」意味著什麼？

**第 4 題 [8 分鐘]（★★）** 把它接回你的後端直覺。你有一個用 softmax 把「服務健康分數」轉成流量配比的 router。分數 [4,2,0] 給出 [0.867,0.117,0.016]。現在因為換了打分函數，所有分數整體乘了 10 變成 [40,20,0]。(a) 不用精算，定性說新配比會長怎樣；(b) 這對應注意力裡的什麼故障？(c) 你會怎麼「縮放」修它，讓配比回到能調節的狀態？

### 推演解答

**第 1 題。**
(a) √d = √4 = **2**；縮放後分數 = [4,2,0] / 2 = **[2, 1, 0]**。
(b) e² = 7.389、e¹ = 2.718、e⁰ = 1.000；和 = 7.389+2.718+1.000 = **11.107**；權重 = [7.389/11.107, 2.718/11.107, 1.000/11.107] = **[0.665, 0.245, 0.090]**（驗：和 = 1.000 ✓）。
(c) 輸出 = 0.665·(1,0)+0.245·(0,1)+0.090·(1,1) = (0.665+0.090, 0.245+0.090) = **(0.755, 0.335)**。
(d) 縮放**沒改變**誰最大（兩版都是 animal 主導、it→animal 的正確傾向保留）；縮放**改變了**最大者贏多少——從 0.867 拉回 0.665，把被 ch05 尖版壓掉的 street（0.117→0.245）、it（0.016→0.090）的發言權還回來。對訓練中的模型，這個「沒把話說死」的餘地就是梯度能流、能繼續學的空間。常見錯路：以為縮放會讓 it 改看 street——不會，縮放是對所有分數除同一正常數，不動排序。

**第 2 題。**
(a) 期望值：qᵢ、kᵢ 獨立且各期望值 0，所以 E[qᵢkᵢ]=E[qᵢ]E[kᵢ]=0·0=0。變異數：因期望值是 0，Var(qᵢkᵢ)=E[(qᵢkᵢ)²]=E[qᵢ²]E[kᵢ²]（獨立可拆）；而 E[qᵢ²]=Var(qᵢ)+E[qᵢ]²=1+0=1，kᵢ 同；故 Var=1·1=1。
(b) 用了**獨立隨機變數的變異數可加性**（《馴服隨機》）：d 個彼此獨立、各變異數 1 的項相加，總變異數 = d×1 = d。
(c) 變異數是 d ⇒ 標準差（典型尺度）是 √d。要把分數的**尺度**歸一回 1，得除以它的**標準差** √d（變異數 d/(√d)²=d/d=1）。若除以 d，標準差變成 √d/d=1/√d、變異數變 1/d——分數被壓得太小、太擠，softmax 趨近均勻分布，注意力「什麼都看一點、什麼都看不清」，一樣學不好（壞在另一端）。核心：要校正的是標準差不是變異數，所以是 √d 不是 d。

**第 3 題。** softmax([16,8,0])：減最大值 16 → [0,−8,−16]；e⁰=1、e⁻⁸≈0.000335、e⁻¹⁶≈0.0000001；和≈1.000335；權重 ≈ **[0.99966, 0.00033, 0.0000]**——幾乎就是 one-hot [1,0,0]。當 animal 權重 p≈0.9997，softmax 的敏感度 ∝ p(1−p) ≈ 0.9997×0.0003 ≈ **0.0003**，是 p=0.5 時最大值 0.25 的約 1/800——梯度小到幾乎沒有。意味著：模型對「該怎麼調 W_Q、W_K 才能看得更準」幾乎收不到訊號，**這一層卡死、學不動了**。它在還沒驗證「看 animal 是對的」之前，就被尺度逼成 one-hot——這正是 √d 要防的事。

**第 4 題。**
(a) [40,20,0] 經 softmax 會壓成幾乎 **[1, 0, 0]**——A 獨吞流量，B、C 形同關閉，router 失去調節能力（B 從 20 漲到 39 都不會改變配比）。
(b) 對應注意力的 **softmax 飽和**：分數尺度過大把 softmax 推進 one-hot，失去「混合」能力（且在模型裡同時造成梯度消失）。
(c) 把分數除以一個能把尺度拉回正常的常數（這裡除以 10 就回到 [4,2,0]、配比回到能調節的 [0.867,0.117,0.016]）。在注意力裡，這個「該除多少」由維度決定，正規答案就是除以 √d——讓分數變異數歸一回 1。

## 自我檢核

口頭自答，講得出來才算過：

1. 用一句話說 `/√d` 在三步骨架的哪個位置、動了什麼、**沒**動什麼（提示：打分與正規化之間的尺度校正，不動 Q/K/V、不動 softmax 本身、不改排序）。
2. 為什麼內積 q·k 的變異數是 d？把「期望值 0 變異數 1 的獨立分量」這個前提，一路推到「變異數可加性 ⇒ 和的變異數 = d」。
3. 為什麼是除以 √d 不是除以 d？（要校正標準差不是變異數；除 d 會把分布壓成均勻、壞在另一端。）
4. softmax 飽和為什麼讓模型「學不動」？把「飽和 → p(1−p)→0 → 梯度→0 → 收不到學習訊號」這條鏈講順。
5. 為什麼 √d 在 d=4 的脊椎裡只是「讓分布軟一點」，在 d=64 的真實模型裡卻是「生死線」？（分數尺度 ∝ √d，d=64 時膨脹 8 倍，不除直接飽和成 one-hot。）
6. 「除以 √d 是為了防數值溢位」哪裡對、哪裡錯？（治不同的病：溢位用減最大值防；√d 防的是統計尺度造成的飽和。）
7. 為什麼縮放不會改變「it 看哪裡（誰最大）」、只改變「最大者贏多少」？
8. 為什麼說「過早的果斷對訓練中的模型是死亡」？這和 ch05 的「attention 是混合全部、不是挑一個」是同一件事的哪兩面？

## 延伸閱讀

- **Vaswani et al. 2017「Attention Is All You Need」§3.2.1 與該節腳註**（arXiv 1706.03762，https://arxiv.org/abs/1706.03762 ）：本章的原始出處。直接讀那段腳註——它就用三句話講完「q·k 變異數為 d_k、大 d_k 把 softmax 推進梯度極小區、故除 √d_k」。值得對照你自己的推導看原文多精簡。
- **《馴服隨機》（機率統計）變異數可加性與 softmax 作為機率分布的章節**：本章變異數論證的地基（獨立和的變異數相加）就在那本；softmax 輸出是合法機率分布也在那本講過。本章只是把這兩個結論用在注意力上。
- **《馴服無限》（微積分）梯度與鏈式法則的章節**：「梯度消失」的完整數學（softmax 的 Jacobian、p_i(1−p_i) 怎麼來、反向傳播怎麼把它乘進去）指向那本。本章只用「梯度小＝學不動」這個結論，不展開反向傳播。
- **Dong et al. 2021「Attention is Not All You Need: Pure Attention Loses Rank Doubly Exponentially with Depth」**（arXiv 2103.03404，https://arxiv.org/abs/2103.03404 ）：本章末尾的鉤子。√d 治單層 softmax 的尺度，但純注意力疊深仍會秩坍縮——讀它了解為什麼還需要殘差與 FFN（ch13 接、ch19 詳述）。
- **何時回來重讀**：當你在 ch16 看到 FlashAttention 的 online softmax「邊掃邊減 running max」時，回頭對照本章故障表第一列——你會發現「減最大值防溢位」和「除 √d 防飽和」是兩件被同一個 softmax 牽連、卻治不同病的事。
