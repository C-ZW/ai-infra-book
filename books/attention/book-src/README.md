# 注意力機制的原理：直覺、機制與理論

> 一本寫給工程師、純為了搞懂原理而讀的注意力（attention）書。不教訓練、不教部署、不刷論文——是想讓你徹底看懂：一個極簡的點子（**打分 → 正規化 → 加權混合**）怎麼撐起 2017 年以降整個大型語言模型的時代，而且你能用自己的話對另一個工程師講清楚。

## 這本書在做什麼

注意力最迷人的地方，是它小到幾乎不像個機制：每個位置丟出一個 query，對所有 key 算相似度分數，softmax 成一排和為 1 的權重，再用這排權重去**加權混合**所有 value。就這三步。這本書從一個代名詞的難題出發——

> *"the animal didn't cross the street because **it** was too tired."*

**it** 指 animal 還是 street？人一眼知道是 animal。這本書要證明：讓 it「**該看哪裡**」是**算**出來的，而那套算法，就是上面那三步——我們叫它**三步骨架**。從 Bahdanau 到 RoPE 到 FlashAttention 到 Mamba，全是這三步的精煉。

讀完，你應該能用自己的話對另一個工程師講清楚這七件事：

1. 注意力到底在算什麼（一句話：對所有位置做一次「依相關性加權的平均」）。
2. 為什麼要 Query/Key/Value 三個角色、為什麼除以 √d、為什麼用 softmax。
3. multi-head 在分什麼工、位置編碼補了「集合沒有順序」這個洞。
4. 為什麼是 O(n²)、業界怎麼想「替這張沒有索引的表建索引」。
5. 注意力的理論身世（核迴歸／Hopfield 聯想記憶／集合上的置換等變運算）。
6. 注意力權重能不能當「解釋」（招牌故障視角），機制可解釋性看到了什麼。
7. 2026 年「attention is all you need」還成立嗎（SSM/Mamba 與混合架構之爭）。

## 怎麼讀

- **架構圖先行**：每個 Part 的第一章都有全書地圖，先看清自己在哪，再鑽細節。
- **抓住三步骨架**：全書每介紹一個變體（√d、multi-head、RoPE、稀疏/線性、FlashAttention、SSM…），都會明確點出它**動了三步的哪一步、為什麼**。這是你不被名詞淹沒的浮木。
- **跟著同一個 it 走**：全書有一條脊椎——同一個代名詞 it，每深入一層就重新算它一次（內積打分 → softmax → √d 縮放 → 自注意力矩陣 → 多頭 → 位置 → 遮罩 → QK/OV 電路）。看著它越懂越深，是最划算的讀法。
- **動手做以紙筆為主**：每章末的「紙上推演」用紙筆算、用嘴巴講；**講得出來才算懂**。少數地方附 5–15 行可讀的 NumPy/虛擬碼讓演算法骨架一目了然（你讀程式如母語）——但不是 lab，不必安裝、不必跑大模型。
- **「故障模式與直覺陷阱」是招牌**：每章都有一節拆解「這概念最常被誤解成什麼、它在真實模型裡怎麼壞」（attention 不是取最相關的一個、softmax 會飽和、位置要外加、熱圖≠解釋…）。

## 書架上的鄰居

本書自足、不跨書也讀得懂，但會在需要時指向書架上的鄰居：

- **《矩陣是動詞：線性代數作為空間變換》**——內積即相似度、投影、矩陣乘法、SVD/低秩，attention 的線代運算深度在那裡。
- **《從後端到 AI Infra：LLM 推論基礎設施工程》**——KV cache、PagedAttention、FlashAttention 的**系統/部署**深度在那裡（本書只講演算法想法，刻意與工作脫鉤）。
- **《圓的影子：三角函數作為旋轉與週期》**——旋轉、複數、e^{iθ}，RoPE 那一章直接接它。
- **《馴服隨機》**（softmax 即機率分布／加權平均即期望）、**《馴服無限》**（梯度與積分）——需要時一句帶過。

那麼，從第一章開始：一個代名詞該看哪裡，以及機器憑什麼算得出來。
