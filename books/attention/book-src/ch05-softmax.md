# ch05 — softmax：把分數變成「該看哪裡」的權重

> **本章解決什麼問題**：上一章我們把 it 對三個 token 的相關性打成了三個原始分數 [4, 2, 0]。但分數不是答案——它們可正可負、可大可小，加起來也不是 1，不能直接拿來「分配注意力」。本章要做的就是三步骨架的**第二步：正規化**——用 softmax 把任意一排分數變成一個和為 1 的權重分布，然後完成**第三步：加權混合**，第一次把三步骨架從頭到尾跑完一次，算出 it 這個位置的輸出向量。讀完你會發現一件反直覺的事：注意力**從來不是「找出最相關的那一個」**，而是「按相關度混合全部」。為了把焦點留給 softmax 本身，本章刻意用**未縮放**的分數（除以 √d 的縮放留到 ch06）。

## 從你已知的出發

你做過負載均衡的流量分配。後端有三台機器，你手上有三個「健康分數」：A 機器 4 分、B 機器 2 分、C 機器 0 分。你不能直接把 `[4, 2, 0]` 當成流量配比——它加起來是 6 不是 100%，而且如果某次健康檢查回傳負分，配比就出現負數，整個分配邏輯崩掉。你需要一個函數，把任意一排分數**正規化成一個百分比配比**：全部非負、加起來剛好 100%、而且分數高的拿到的比例要明顯更大。

softmax 就是這個函數，而且它比你手寫的「除以總和」更聰明。如果你只是把 `[4, 2, 0]` 除以總和 6，得到 `[0.67, 0.33, 0]`——C 機器拿到剛好 0%，A 只比 B 多一倍。但 softmax 會先用 `exp` 把每個分數放大，再正規化，讓**分數差距被指數級拉開**：高分的更高、低分的趨近 0、但沒有任何一個真正歸零（除非分數是 −∞，那是 ch11 遮罩在做的事）。

還有一個你天天在動的旋鈕：LLM API 的 `temperature`。你調 `temperature=0` 想要穩定輸出、調 `temperature=1.2` 想要它「有創意一點」——你動的就是 softmax 裡的同一個溫度參數 T。本章會把這個旋鈕的內部機制拆開給你看（2026-06 多數 LLM API 的 `temperature` 預設範圍 0.0–2.0，正是除在 softmax 之前的那個 T）。

在三步骨架上，本章站在中間這一步：

```text
            ┌──────────┐   ┌────────────┐   ┌──────────────┐
 query ──►  │ 1. 打分  │─► │ 2. 正規化  │─► │ 3. 加權混合  │ ──► 輸出
 keys  ──►  │  score   │   │  softmax   │   │ weighted sum │
 values ─────────────────────────────────────────┘（值在此進場）
```

第一步 ch04 已經做完，產出 [4, 2, 0]。本章把第二步（softmax）做透，順手把第三步（加權混合）跑完——因為兩步合在一起，才看得出「權重決定混合」這句口訣的完整意思。

## softmax 在做什麼：把分數變成一個機率分布

softmax 接收一排實數分數 z = (z₁, z₂, …, zₙ)，輸出一排權重 α = (α₁, α₂, …, αₙ)：

```text
                  exp(zᵢ)
        αᵢ  =  ─────────────         ← 先取 exp，再除以全部 exp 的總和
                Σⱼ exp(zⱼ)
```

兩步就講完了：

1. **取指數**：每個分數 zᵢ 過一次 `exp`。`exp` 永遠是正的，所以負分數也會變成正權重（這就解決了你負載均衡裡「健康分數可能是負的」那個坑）。`exp` 還是單調遞增的，所以**分數的大小順序被完整保留**——本來 A 最高，softmax 後 A 還是最高。
2. **除以總和**：把每個 `exp(zᵢ)` 除以「所有 `exp` 加起來」。這一步保證全部加起來剛好是 1。

輸出的這排數字 αᵢ 有三個鐵打的性質：

- **每個都 ≥ 0**（因為 `exp` 永遠為正，分母也為正）；
- **加起來剛好 = 1**（因為分子全部加起來就等於分母）；
- **保序且放大差距**（分數高的權重大；而且因為先過了 `exp`，差距被指數級拉開）。

前兩個性質合起來，意思就是：**softmax 的輸出是一個機率分布**（見《馴服隨機》：一排非負、和為 1 的數就是一個分布）。這不是比喻——它真的是一個合法的機率分布。在注意力裡，這排數字 αᵢ 叫**注意力權重（attention weight）**，意思是「it 這個位置，把它有限的『注意力預算』分配多少比例去看第 i 個 token」。預算總共 100%，全部分完，不多不少。

> 工程師的嚴謹：上面三個性質你都能口頭說出理由（exp 為正 → 非負；分子和=分母 → 和為 1；exp 單調 → 保序）。這不是「顯然」，是兩行就能驗的事實。

### 為什麼非得是 exp：和「除以總和」差在哪

你可能會抗議：要把一排數正規化成和為 1，我直接「每個除以總和」不就好了，何必繞一圈取 exp？這個問題問得好，而且答案有三層，每一層都是 exp 不可被取代的理由：

```text
   方法              [4, 2, 0] 正規化後          問題
   ──────────────────────────────────────────────────────────────
   直接除以總和       [0.67, 0.33, 0.00]         遇負分數會出負權重；
   (z / Σz)                                       0 分被歸到剛好 0
   softmax           [0.867, 0.117, 0.016]       全程非負、差距被拉開
   (exp(z) / Σexp(z))                             0 分仍留 1.6% 不歸零
```

1. **負分數**。內積分數可正可負（k_it 那一維有個 −1，內積完全可能是負的）。直接除以總和，分母可能是負的、權重可能是負的——「負的注意力比例」沒有意義，整個機率語義崩掉。exp 把任何實數壓成正數，從根上免疫。
2. **指數級拉開差距**。直接除以總和，4 比 2 的權重就是 2 比 1（線性）；softmax 後是 e⁴ 比 e²＝e² ≈ 7.4 比 1（指數）。注意力需要「夠果斷」——相關的明顯勝出、不相關的明顯讓位——線性的拉開力道不夠，softmax 的指數放大才造得出「聚焦」的效果。
3. **平移不變、處處可微、梯度漂亮**。softmax 對「全部分數同加一個常數」結果不變（後面溢位修法就靠這個），且它的微分形式特別乾淨（αᵢ(δᵢⱼ − αⱼ)），讓梯度下降好走。這些是「除以總和」給不了的數學性質，也是它能撐起整個訓練流程的底氣（深入見《馴服無限》）。

所以 exp 不是裝飾，它同時解決了「負分數」「不夠果斷」「能不能訓練」三件事。這也是為什麼從 Bahdanau 2014 到 2026 的 frontier 模型，正規化這一步幾乎清一色是 softmax——替代品（ch19 會談 sigmoid/ReLU 注意力）動的正是這一步，但都得先回答「你拿什麼補上 exp 的這三個好處」。

## 它是 soft 的 argmax：為什麼不直接挑最大的就好

這是本章你最該停下來想十分鐘的地方。

你可能會問：既然 it 對 animal 的分數（4）最高，那為什麼不直接挑 animal、忽略其他兩個？這個「挑最大值」的操作有個名字叫 **argmax**——回傳「分數最大的那一個是第幾個」。argmax 看起來才是「該看哪裡」的正解，softmax 反而像是在和稀泥。

argmax 有兩個致命傷，正好是 softmax 要救的：

- **它不可微分**。argmax 的輸出是「第 1 個」這種離散的選擇，分數動一點點，輸出要嘛不變、要嘛突然跳到別的選項。一個輸出會「跳變」的函數，沒有平滑的斜率，**梯度下降抓不到該往哪走**——而模型就是靠梯度下降學出那些投影矩陣的（ML 鷹架：模型靠「往坡下走」調整權重，需要每一步都有斜率可循，見《馴服無限》梯度）。一個學不動的零件，再正確也沒用。
- **它只看一個**。argmax 把 100% 的注意力全押在 animal 上，street 和 it 自己的資訊**完全丟棄**。但語言裡的線索常常是分散的：判斷 it 指代誰，主詞 animal 最重要沒錯，可是 street 是不是也提供了一點「另一個候選」的對照資訊？硬 argmax 一刀切，把這些都扔了。

softmax 是 argmax 的「軟化版本」——所以它有時也被直接叫 **soft-argmax**。它保留了 argmax 的精神（分數最高的拿最多權重），但補上兩個缺的東西：**處處可微**（能訓練）、**按比例看全部**（不丟資訊）。

這個名字不是後人附會的。「softmax」一詞是 John Bridle 在 1989 年提出的（他稱之為「normalised exponential」，正規化指數），他當時的動機原話就是：它「保留輸入值的排序，是『贏者全拿』操作的一個可微分推廣」（2026-06 查證）。換句話說，softmax 從出生那天起，它的工作描述就是「**可微分的 winner-take-all**」。

> **這裡反直覺在哪**（請你能用自己的話轉述）：很多人以為「注意力就是模型找出最相關的那個詞」。**錯**。注意力是按相關度**混合全部**——只是相關的那個權重特別大而已。it 的輸出不是「animal 的向量」，而是「86.7% 的 animal ＋ 11.7% 的 street ＋ 1.6% 的 it」混出來的東西。「找一個」是 argmax；注意力做的是 soft-argmax。把這兩件事分清楚，後面整本書都會輕鬆很多。

## 跑完一次三步骨架：脊椎 it 的完整計算

現在把序章那個 it（*"the animal didn't cross the street because **it** was too tired"*，簡化序列 [animal, street, it]）的計算從頭跑到尾。這是全書三步骨架第一次完整落地。

第一步打分，ch04 已經算出原始內積分數（這裡用**未縮放**的版本，把焦點留給 softmax）：

```text
       score(it, animal) = q_it · k_animal = 4
       score(it, street) = q_it · k_street = 2
       score(it, it)     = q_it · k_it     = 0
```

第二步正規化，對 [4, 2, 0] 做 softmax。不准跳步，逐項算：

```text
  取 exp（e ≈ 2.71828）：
       exp(4) = 54.598          ← e⁴
       exp(2) =  7.389          ← e²
       exp(0) =  1.000          ← e⁰，任何數的 0 次方都是 1

  求總和：
       Σ = 54.598 + 7.389 + 1.000 = 62.987

  各自除以總和：
       α_animal = 54.598 / 62.987 = 0.867
       α_street =  7.389 / 62.987 = 0.117
       α_it     =  1.000 / 62.987 = 0.016

  驗算：0.867 + 0.117 + 0.016 = 1.000  ✓（和為 1，是個合法分布）
```

讀出這排權重的意思：it 這個位置，把 **86.7%** 的注意力放到 animal、**11.7%** 給 street、只留 **1.6%** 給它自己。注意原始分數只是 4 比 2 比 0（差距不算懸殊），但過了 `exp` 之後，animal 的權重變成 street 的七倍多——這就是「指數放大差距」的威力。

第三步加權混合。每個 token 帶著自己的 value 向量（ch04 基準）：v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)。用上面的權重對這三個 value 做加權平均：

```text
  輸出 = 0.867·(1,0) + 0.117·(0,1) + 0.016·(1,1)

  第一維：0.867·1 + 0.117·0 + 0.016·1 = 0.867 + 0      + 0.016 = 0.883
  第二維：0.867·0 + 0.117·1 + 0.016·1 = 0      + 0.117 + 0.016 = 0.133

  輸出 ≈ (0.883, 0.133)
```

it 位置的輸出向量是 **(0.883, 0.133)**。看它的方向：第一維 0.883 遠大於第二維 0.133——也就是說，這個輸出**強烈偏向 v_animal=(1,0) 的方向**。

這就是整本書要證明的那件事，第一次被算出來：模型沒有任何一條「it 指 animal」的硬規則，但只要那些投影矩陣學得好，三步骨架就會讓 it 的輸出自動偏向 animal。**「該看哪裡」是算出來的，不是規定的。** 序章只給了直覺，這裡是數字。

> 一個常見誤讀先擋掉：輸出 (0.883, 0.133) **不等於** animal 的 value (1,0)，它是混合的結果，只是偏 animal。如果你以為「注意力選中了 animal、輸出就是 animal 的向量」，你就把 soft-argmax 又讀成 argmax 了。

如果你想用程式核對這組數字，五行 numpy 就夠（程式跑出來的就是上面手算的那組）：

```python
import numpy as np

scores = np.array([4.0, 2.0, 0.0])          # raw scores from ch04 (unscaled)
weights = np.exp(scores) / np.exp(scores).sum()
print(weights)                                # [0.867 0.117 0.016]

V = np.array([[1, 0], [0, 1], [1, 1]])        # v_animal, v_street, v_it
print(weights @ V)                            # [0.883 0.133]
```

## 溫度：同一組分數，為什麼可以又尖又平

softmax 還有一個旋鈕，藏在分數裡：**溫度（temperature）T**。做法是在取 exp 之前，先把每個分數除以 T：

```text
                  exp(zᵢ / T)
        αᵢ  =  ───────────────
                Σⱼ exp(zⱼ / T)
```

T=1 就是原本的 softmax（什麼都沒動）。改變 T 不改變分數的排序（除以正數保序），但會改變權重分布的**銳利度**。直覺：

- **T 小（< 1）→ 分布變「尖」**。除以一個小於 1 的數，等於把分數放大，差距被拉得更開，最大的那個權重衝向 1，其餘趨近 0。**T → 0 時，softmax 趨近硬 argmax**（變成 one-hot，只剩贏家）。
- **T 大（> 1）→ 分布變「平」**。除以一個大於 1 的數，等於把分數壓扁，差距縮小，各權重趨於平均。**T → ∞ 時，softmax 趨近均勻分布**（每個都 1/n，誰都不偏）。

拿脊椎的 [4, 2, 0] 親手驗證這兩個極端。

**T = 0.5（變尖）**：分數除以 0.5 等於乘 2 → [8, 4, 0]。

```text
  exp(8) = 2980.96,  exp(4) = 54.598,  exp(0) = 1.000
  Σ = 2980.96 + 54.598 + 1.000 = 3036.56
  α = [2980.96/3036.56, 54.598/3036.56, 1.000/3036.56]
    ≈ [0.982, 0.018, 0.0003]                ← 幾乎全押 animal，逼近 one-hot
```

**T = 2（變平）**：分數除以 2 → [2, 1, 0]。

```text
  exp(2) = 7.389,  exp(1) = 2.718,  exp(0) = 1.000
  Σ = 7.389 + 2.718 + 1.000 = 11.107
  α = [7.389/11.107, 2.718/11.107, 1.000/11.107]
    ≈ [0.665, 0.245, 0.090]                 ← 明顯比 T=1 平緩，animal 只剩六成多
```

並排看這三組權重，同一組原始分數 [4,2,0]，溫度不同就「又尖又平」：

```text
   溫度      α_animal   α_street   α_it      形狀
   ─────────────────────────────────────────────────────
   T = 0.5    0.982      0.018      0.0003    很尖（逼近 argmax）
   T = 1      0.867      0.117      0.016     尖（本章脊椎）
   T = 2      0.665      0.245      0.090     平（更平均）
```

這裡有個工程上的取捨值得記一筆：溫度太低（太尖）雖然「決斷」，但**梯度的變異數會變大、訓練變得不穩**（接近 one-hot 時，幾乎所有梯度都堆在贏家那一格，一點點分數擾動就讓贏家換人，梯度跳動劇烈）；溫度太高（太平）梯度平滑好訓練，但分布太均勻、近似誤差大、學不出「該看哪裡」（2026-06 查證）。注意力要的是中間那個甜蜜帶——這正是 ch06「除以 √d」想把分布從過尖拉回適中的深層動機（不只是怕飽和，也是怕梯度抖）。

兩個值得記住的彩蛋：

第一，溫度的高低與**熵**直接掛鉤——分布越平，熵越高（越不確定）；分布越尖，熵越低（越篤定）。T → ∞ 時熵到最大（均勻），T → 0 時熵趨近 0（one-hot 沒有不確定性）。這個「溫度」之名來自統計物理的 Boltzmann 分布，softmax 的結構與它一模一樣，T 扮演的角色就是物理溫度（2026-06 查證；深入見《馴服隨機》熵與分布）。

第二個彩蛋直接是給 ch06 的鋪墊，請你記住這個巧合：**T=2 算出來的 [0.665, 0.245, 0.090]，正是把原始分數 [4,2,0] 除以 2 之後做 softmax 的結果**。而下一章要講的「除以 √d」，在脊椎裡 √d 剛好就是 2。也就是說——除以 √d 在數學上的作用，**就等於把溫度調到 √d**。ch06 不是要引入一個新機制，它只是告訴你：這個「降溫」不是隨便調的旋鈕，而是有一個變異數的理由，逼著你非除這一下不可。本章先把這個 [0.665, 0.245, 0.090] 算出來放著，ch06 起它就成了全書正典的脊椎權重。

> 回到你熟的旋鈕：LLM API 的 `temperature` 就是這個 T。你設 `temperature=0`（或趨近 0），模型每次都挑機率最高的 token、輸出穩定可重現（逼近 argmax）；你設 `temperature=1.5`，分布被壓平、低機率的 token 也有機會被抽到、輸出更「發散」（2026-06 多數 API T 範圍 0–2，T=1 是模型訓練出的原始分布）。注意：這個 T 是**生成時抽樣**用的旋鈕，和模型內部每一層 attention 的 √d 縮放是兩回事——別把兩者搞混（ch06 會再強調一次）。

## 故障模式與直覺陷阱

| 陷阱 | 錯誤直覺 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| **把 attention 當「選一個」** | 「注意力就是找出最相關的那個 token」 | 你會以為輸出 = 最高分 token 的 value，忽略混合；遇到「兩個候選都重要」的句子就解釋不通 | 問自己：我說的是 argmax 還是 softmax？輸出有沒有把全部 value 都混進去？ |
| **以為 softmax 輸出是「信心」** | 「α_animal=0.867 代表模型 86.7% 確定 it 指 animal」 | 把注意力權重當成模型的「決策信心」或「解釋」——這是 ch20 整章在拆的謬誤；權重是「分配給誰的混合比例」，不是「對外的判斷機率」 | 提醒自己：softmax 在這裡只是把分數正規化成混合配比，語義上的「機率」要很小心（ch20）。 |
| **softmax 飽和**（saturation） | 「分數越大越好，反正會正規化」 | 分數一大（如 [40,20,0]），softmax 直接逼近 one-hot，最大權重→1、其餘→0；**梯度趨近 0，模型學不動**——這正是 ch06 要除 √d 的核心痛點 | 看到原始分數量級很大（幾十、上百）就警覺：softmax 可能已經飽和、退化成硬 argmax 了。 |
| **數值溢位**（overflow） | 「照公式 exp(zᵢ)/Σexp(zⱼ) 算就好」 | 分數稍大（如 z=1000），`exp(1000)` 在浮點數裡直接變 `inf`，分子分母都是 inf，結果是 `NaN`，整批計算汙染 | 實作時永遠先減去該排的最大值再取 exp（見下方推演說明）；手算量級大的例子也該這樣做。 |
| **以為溫度改了排序** | 「調高溫度會讓別的 token 變最相關」 | 溫度只改銳利度、不改排序（除以正數保序）；最高分的永遠最高，只是差距被拉開或壓平 | 記住：T 動的是「尖/平」，不是「誰第一」。 |

最該記住的是 **softmax 飽和**——它是 ch06 整章存在的理由。softmax 在「分數差距適中」時最健康（既能放大差距、又每個都還有一點梯度）；一旦分數差距太大，它就退化成沒有梯度的硬 argmax。本章脊椎用的是適中的 [4,2,0]，所以看起來沒事；ch06 會給你看當 d_k=64、分數量級膨脹到八倍時，softmax 怎麼被壓死。

## 紙上推演

### 推演一：手算 softmax 並讀出它有多尖 **[15 分鐘] ★**

對分數 [4, 2, 0] 手算 softmax（不准看上文，自己從 exp 算起），得到三個權重。然後回答：原始分數 animal 只比 street 高一倍（4 vs 2），但 softmax 後 α_animal 是 α_street 的幾倍？這個「幾倍」說明了 softmax 的什麼性質？

### 推演二：反駁「注意力就是找最相關的詞」 **[10 分鐘] ★★**

你的同事說：「attention 不就是模型挑出句子裡最相關的那個詞嗎？」用脊椎 it 的這次計算寫一段 100 字內的反駁，要點出：(a) 輸出向量到底是什麼（具體數字）；(b) 為什麼「挑一個」（argmax）和「混合全部」（softmax）是兩件事；(c) 如果真的「只挑一個」，會丟掉什麼、且為什麼不能訓練。

### 推演三：溫度與「降溫」的預測 **[20 分鐘] ★★**

(a) 對 [4,2,0] 分別用 T=0.5 與 T=2 重算 softmax（手算，逐步驟）。(b) 觀察：把整組分數**整體縮小**（如除以 2），權重會變尖還是變平？用你算的結果回答，並用一句話解釋為什麼。(c) 進階：為什麼直接用公式 `exp(zᵢ)/Σexp(zⱼ)` 算 [1000, 998, 996] 會在電腦上爆掉？該怎麼改寫才安全（提示：softmax 對「每個分數同減一個常數」是不變的）？

### 推演解答

**推演一**：α = [0.867, 0.117, 0.016]（算法見本章「跑完一次三步骨架」一節）。α_animal / α_street = 0.867 / 0.117 ≈ **7.4 倍**。但原始分數只差一倍（4 vs 2）。這個放大來自 `exp`：權重比是 exp(4)/exp(2) = e⁴/e² = e² ≈ 7.39，也就是說 softmax 把「分數的差」變成「權重的比的指數」。這就是為什麼說 softmax「放大差距」——它對分數差距的反應是指數級的，這也預告了「分數一大就會飽和」的故障（差距稍大，比值就爆炸成接近 one-hot）。

**推演二**（範例答案）：「不是『挑一個』，是『按比例混合全部』。it 的輸出是 0.883·(1,0)+0.133 方向的混合 ≈ (0.883, 0.133)，裡面同時有 animal、street、it 三者的成分，只是 animal 占 86.7%。『挑一個』是 argmax，它不可微分、模型沒法用梯度下降學；而且它會把 street 和 it 的資訊整個丟掉。softmax 是 argmax 的可微軟化版（soft-argmax），既能訓練、又保留全部資訊。所以注意力的精神是『相關的權重大』，不是『只看最相關的一個』。」

**推演三**：(a) T=0.5 → [0.982, 0.018, 0.0003]（變尖，逼近 one-hot）；T=2 → [0.665, 0.245, 0.090]（變平）。(b) 整體縮小分數（除以一個 >1 的數）等於**升高溫度**，所以權重會**變平**（更平均）。原因：縮小分數讓 token 之間的差距變小，exp 拉開差距的力道也跟著變小。這正是 ch06「除以 √d」要做的事——讓尖銳的 [4,2,0]（→ 0.867）變平成 [2,1,0]（→ 0.665）。(c) 直接算 `exp(1000)` 會溢位成 `inf`，分子分母都是 inf → `NaN`。修法：利用 softmax 的平移不變性——對每個分數先減去該排的最大值 m，`softmax(z) = softmax(z − m)`（分子分母同乘 e⁻ᵐ，約掉，結果完全不變），這樣最大的 exp 變成 `exp(0)=1`、其餘都 ≤ 1，不會溢位。這個「先減最大值」的技巧，正是 ch16 FlashAttention 的 online softmax 要追蹤「running max」的原因——同一個數值穩定的招數，分塊版而已。
