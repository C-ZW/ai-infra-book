# ch11 — 遮罩：只能看過去

> **本章解決什麼問題**：ch07 給了你一張對稱的自注意力矩陣——每個 token 都能看遍全句，animal 看得到 it、it 也看得到 animal。但 GPT 這類「一個字一個字生成」的模型有個鐵律：寫到第 i 個字時，它**還不知道**第 i+1 個字是什麼。讓它在訓練時偷看未來，等於讓考生看著答案練習，上場就廢。這一章只解決一件事：怎麼讓注意力**只能看過去**。答案是一個叫「因果遮罩（causal mask）」的小動作——在 softmax 之前，把所有「看向未來」的分數設成 −∞。它不動 query/key/value，也不動 softmax 公式，它動的是**送進 softmax 的分數**：把非法位置的分數壓到 −∞，softmax 後自動變成 0。本章也順手講另一種遮罩（padding mask），並讓你看清 ch07 那張對稱矩陣的對稱性，在這裡被**單向**打破了。

## 從你已知的出發

你做過 event sourcing 嗎？沒做過也沒關係，你一定維護過 append-only 的 log——交易紀錄、稽核 log、Kafka topic、git commit 歷史。這類系統有一條你從不質疑的鐵律：

> **重播到第 t 個事件時，你只能讀 ≤ t 的事件，讀不到 t 之後的。** 未來還沒發生，log 裡根本沒有那幾行。

你重算某個帳戶在 3 月 1 日的餘額，只能把 3 月 1 日（含）之前的交易加總。要是你的程式不小心把 3 月 5 日的退款也算進去了，那不叫 bug，那叫**時光旅行**——你用了一筆「在那個時間點根本不存在」的資訊。任何依時間順序重建狀態的系統，都得保證這件事不發生。

自迴歸（autoregressive）語言模型生成文字，就是一次 append-only 的重播。它一個 token 一個 token 往後寫，寫第 i 個 token 時，手上只有第 1 到 i 個 token（第 i 個是它剛寫完、正準備據以預測下一個的）。第 i+1、i+2…還沒生出來，它**不該**、也**不能**看。

問題來了：ch07 的自注意力是**雙向**的。在那張 3×3 矩陣裡，每個 token 都對全句所有 token 算了相似度——包括它右邊的。如果什麼都不做，模型在預測「下一個字」時，注意力會大大方方地去看它正要預測的那個字（以及更後面的），這就是時光旅行。**因果遮罩就是那條 append-only 鐵律在注意力裡的實作**：把所有「看向右邊（未來）」的注意力連結，在它變成權重之前先掐斷。

你大概也猜到掐斷的手法了——就像你在 ACL 裡把無權限的列濾掉、在 query 裡加一條 `WHERE event_time <= :t` 把未來的行擋在外面。注意力裡的「擋」更乾脆：把非法連結的分數設成 −∞，讓 softmax 給它分配**剛好 0** 的權重。本章就講這一個動作，以及它為什麼非得是 −∞、不能是 0。

## 在三步骨架的哪裡：又是「打分」和「正規化」之間

先定位。遮罩這件事坐在三步骨架的哪一格？

```text
            ┌──────────┐   ┌────────────┐   ┌──────────────┐
 query ──►  │ 1. 打分  │─► │ 2. 正規化  │─► │ 3. 加權混合  │ ──► 輸出
 keys  ──►  │  score   │   │  softmax   │   │ weighted sum │
 values ─────────────────────────────────────────┘
                  │
                  └─►【遮罩：把非法位置的分數設 −∞】插在打分之後、正規化之前
```

和 ch06 的 `/√d` 一樣，遮罩是「打分」與「正規化」之間的一道**加工**——它不是獨立的第四步。打分（ch04）算出一整排相似度分數，遮罩在這些分數進 softmax（ch05）之前，把不該被看到的位置覆寫成 −∞。然後 softmax 照常跑，加權混合（value 進場）照常跑。query/key/value 三個角色一個沒變，softmax 公式一個字沒改，變的只有**送進 softmax 的那排數字**。

值得停一下記住這個型態：本書到目前為止，凡是想調整注意力的「該看哪裡」，動的幾乎都是同一個地方——**送進 softmax 的分數**。ch06 縮放它（除 √d）、本章遮蓋它（設 −∞）。softmax 是個極度聽話的閥門，你想讓它怎麼分配權重，不用去改它，改它的**輸入**就好。這個觀念到 ch15（線性注意力換掉 softmax）和 ch16（online softmax 分塊算）都還會回來。

## 因果遮罩：把未來設成 −∞

來看具體怎麼做。延用 ch05/ch06 的記號：先有縮放後的分數，再 softmax。遮罩插在中間。

標準的寫法是把遮罩當成一個**加性**矩陣 M 加進分數裡（landscape §2 的原文也是這個形式）：

```text
Attention(Q, K, V) = softmax( QKᵀ / √d  +  M ) · V
                                          ↑
                              M 在 softmax 之前加進分數
```

M 是一個和注意力分數同形狀的矩陣，它只有兩種值：

```text
M[i][j] = 0      若 j ≤ i   （位置 j 在 i 的過去或當下，合法 → 不動分數）
M[i][j] = −∞     若 j > i   （位置 j 在 i 的未來，非法 → 把分數打到 −∞）
```

讀法：第 i 列代表「站在第 i 個 token 的 query 視角」，第 j 欄代表「它對第 j 個 token 的 key 打的分」。`j > i` 就是「i 想看它右邊的 j」——未來，非法。把那一格的分數加上 −∞（等於覆寫成 −∞），其餘合法格子加 0（不動）。

這個 M 長得像一個下三角全 0、上三角全 −∞ 的矩陣。以三個位置 [1, 2, 3] 為例：

```text
        j=1    j=2    j=3
i=1  [   0    −∞     −∞  ]   ← 第 1 個 token：只能看自己
i=2  [   0     0     −∞  ]   ← 第 2 個 token：能看 1、2
i=3  [   0     0      0  ]   ← 第 3 個 token：能看 1、2、3（全部過去＋當下）
        └── 下三角＋對角線 = 0（合法），嚴格上三角 = −∞（未來，遮掉）
```

注意對角線（i=j，自己看自己）是 0，**合法**——「當下」這個 token 自己當然看得到自己。被遮的只有**嚴格上三角**（j > i，真正的未來）。

然後 softmax 一吃到 −∞，那一格就變成 0：

```text
e^(−∞) = 0
```

於是第 i 列做 softmax 時，未來那些格子的指數值是 0，對正規化的分母毫無貢獻，分到的權重剛好是 0。注意力被乾淨地限制在「自己與左邊」。

> **這裡了不起／反直覺在哪**：自注意力的本體——Q/K/V、內積、softmax、加權混合——從頭到尾**一個字都沒改**。要把一個「什麼都能看」的雙向模型，變成一個「只能看過去」的生成模型，你需要的全部改動，就是在分數矩陣上**加一個下三角為 0、上三角為 −∞ 的常數矩陣**。同一套注意力，蓋上不同的遮罩，就是 BERT（雙向）和 GPT（單向）的根本分野（這個分野留到本章最後一節說透）。一個 if-else 等級的小動作，劃出了「理解」與「生成」兩種模型家族。

### 為什麼設 −∞ 而不是設 0：這是本章最該停的地方

幾乎每個人第一次都會問：「要把一個連結『關掉』，直接把它的權重設成 0 不就好了嗎？為什麼要繞一圈設 −∞？」

這個問題問得好，而且答錯的後果很具體。關鍵在於**你想設成 0 的是誰、在哪一步**。

要記住順序：遮罩動的是 **softmax 之前的分數**，不是 softmax 之後的權重。如果你把「分數」設成 0，會發生什麼？分數 0 餵進 softmax，它的指數是 `e⁰ = 1`——**這不是 0，是 1**。也就是說，你以為把那個位置關掉了，實際上 softmax 還是給了它一個 `1 / Σ` 的、不小的權重。它照樣參與了加權混合，未來照樣被偷看了（2026-06 查證，這正是新手最常踩的坑）。

```text
想關掉未來位置，但設「分數 = 0」：
   e⁰ = 1  →  softmax 後權重 = 1/Σ ≠ 0   ✗ 沒關掉！它還在投票
想關掉未來位置，設「分數 = −∞」：
   e^(−∞) = 0  →  softmax 後權重 = 0/Σ = 0  ✓ 乾淨歸零
```

softmax 是個**指數**閥門，它的「0 點」不在輸入 0，而在輸入 −∞。`exp` 把 −∞ 映成 0，把 0 映成 1。所以「我想讓這個權重變成 0」對應的輸入，是 −∞，不是 0。

換個你熟的角度想：softmax 之後的權重是一個**機率分布**（和為 1，ch05 講過）。你不能直接把分布裡某一項硬塞成 0——那會破壞「和為 1」、整個分布失去意義。正確的做法是在**進入 softmax 之前**就把那一項排除掉，讓 softmax 在「剩下的合法項」上重新正規化。設 −∞ 就是「在正規化之前排除」：被設 −∞ 的項對分母 Σ 沒有貢獻，剩下的合法項自動瓜分掉全部的 1。權重的「和為 1」性質完好無損，只是參與分配的成員少了未來那幾個。

```text
設 −∞ 的效果，等於「縮小參與 softmax 的集合，再重新正規化」：
   原本 3 個位置爭奪權重  →  遮掉 1 個未來位置  →  剩 2 個位置瓜分全部的 1
```

這也是為什麼遮罩**加在 softmax 前**這件事是對的、不可調換順序的。加在後面（先 softmax 再把某些權重清 0）你還得手動重新正規化，否則和不為 1；加在前面，softmax 的正規化天然就幫你算對了。一句話：**遮罩屬於「正規化之前」，因為它要借 softmax 的正規化來把權重分對。**

## Worked example：給脊椎的 3×3 矩陣套上因果遮罩

回到脊椎——序章那個句子 *"the animal didn't cross the street because **it** was too tired."*，簡化序列 **[animal, street, it]**。注意這個順序：animal 在最左、it 在最右，正好對應它們在句子裡出現的先後（it 是後面才冒出來的代名詞）。這個順序待會是重點。

ch07 給了你完整的 3×3 自注意力矩陣（每個 token 當 query，對三個 key 打分、softmax）。本章把那張矩陣請回來，蓋上因果遮罩，看它怎麼變。

先看**縮放後的分數矩陣**（已經除過 √d=2，沿用 ch06 的尺度）。其中 **it 那一列是全書正典 [2, 1, 0]**（ch04 打分、ch06 縮放定下的脊椎數字）；animal、street 兩列的分數**在這裡只當示意**（它們的 query 投影本書基準表沒鎖死，真正的全矩陣在 ch07，這裡只為了演示遮罩的形狀，數字選得能看出對稱性即可）：

```text
縮放後分數矩陣（列 = query 視角，欄 = 對哪個 key 打分）
              ┌─ animal ─┬─ street ─┬──  it  ──┐
  q=animal    │    3     │    1     │    0     │   ← 示意
  q=street    │    1     │    3     │    0     │   ← 示意
  q=it        │    2     │    1     │    0     │   ← 正典脊椎 [2,1,0]
              └──────────┴──────────┴──────────┘
```

### 第一步：先看雙向（無遮罩）長什麼樣——回收 ch07

對每一列各自做 softmax（這就是 ch07 的雙向自注意力，每個 token 看遍全句）：

```text
雙向（無遮罩），逐列 softmax：
  q=animal: softmax([3,1,0]) = [0.844, 0.114, 0.042]   ← animal 看得到右邊的 it（0.042）
  q=street: softmax([1,3,0]) = [0.114, 0.844, 0.042]   ← street 也看得到右邊的 it
  q=it    : softmax([2,1,0]) = [0.665, 0.245, 0.090]   ← it 看左邊的 animal/street（正典）
```

（核對 it 列：e²=7.389、e¹=2.718、e⁰=1，和=11.107，得 [0.665, 0.245, 0.090]——和 ch06 宣告的正典脊椎權重一字不差 ✓。animal 列：e³=20.086、e¹=2.718、e⁰=1，和=23.804，得 [0.844, 0.114, 0.042] ✓。）

關鍵觀察：**雙向矩陣是「滿的」**——每一列三個欄位都有非零權重。特別看 animal 那一列：它對 it 分了 0.042 的權重。animal 在句子最左邊，it 在最右邊，**animal 看到了它右邊的、未來的 it**。在一個理解任務（BERT）裡這完全沒問題；但在一個生成任務裡，這就是時光旅行。

### 第二步：蓋上因果遮罩——對稱被打破

現在套上下三角 0、上三角 −∞ 的遮罩。逐列看「合法的分數」與遮罩後重算的 softmax：

```text
因果遮罩後（第 i 列只保留欄 j ≤ i，其餘設 −∞，再 softmax）：

  q=animal（第 1 列，只能看自己）
     遮罩後分數 [  3 ,  −∞ ,  −∞ ]
     softmax    [ 1.000, 0.000, 0.000 ]   ← 只剩自己，權重被迫全給自己

  q=street（第 2 列，能看 animal、street）
     遮罩後分數 [  1 ,   3 ,  −∞ ]
     softmax    [ 0.119, 0.881, 0.000 ]   ← it 那欄歸零，animal/street 重新瓜分

  q=it（第 3 列，能看 animal、street、it——全部）
     遮罩後分數 [  2 ,   1 ,   0 ]
     softmax    [ 0.665, 0.245, 0.090 ]   ← 和雙向時一模一樣（it 在最右，本來就沒未來可看）
```

（核對 street 列：合法分數 [1, 3]，e¹=2.718、e³=20.086，和=22.804，得 [0.119, 0.881] ✓。）

把雙向與因果並排成熱圖（█ 最濃、░ 最淡，█ 對應權重大；空白＝被遮掉、權重 0）：

```text
雙向（ch07，對稱）              因果遮罩（ch11，單向下三角）
        an  st  it                     an  st  it
 animal ███  ░   ░             animal ███
 street  ░  ███  ░             street  ░  ███
 it     ██▓  █   ░             it     ██▓  █   ░
        └ 每列都滿 ┘                   └ 嚴格上三角全空（未來看不到）┘
```

兩張圖並排，本章的轉折一目了然：

1. **因果矩陣是個下三角**。所有嚴格上三角的格子（未來連結）權重都是 0，被乾淨地擋掉了。
2. **animal 那一列塌成 [1, 0, 0]**。這是最戲劇性的變化。在雙向時 animal 還能參考 street 和 it；蓋上遮罩後，animal 是第一個 token，左邊什麼都沒有，只能看自己——權重被迫 100% 給自己。**句子的第一個 token 永遠只能盯著自己看**（這個現象在真實模型裡和「attention sink」有關，故障段會講）。
3. **it 那一列完全沒變**，還是正典的 [0.665, 0.245, 0.090]。為什麼？因為脊椎序列 [animal, street, it] 裡，**it 排在最後**——它本來就沒有任何「未來」可看，遮罩對它毫無作用。it 合法地看著它左邊的 animal（0.665）和 street（0.245），ch01 那個「it 該看 animal」的懸念在因果世界裡照樣成立，而且是**完全合法**地成立。

第 2 點和 ch07 的對照是本章的脊椎時刻，值得明寫：**ch07 的雙向矩陣是對稱的——animal 看得到 it、it 也看得到 animal。因果遮罩把這個對稱單向地打破了：it 仍看得到 animal（animal 在它過去），但 animal 再也看不到 it（it 在 animal 的未來）。** 注意力的流向，從「兩兩互看」變成了「只能往回看」。一張對稱的關係圖，被遮罩裁成了一張只有「指向過去」的箭頭的有向圖。

### 可選：causal mask 的幾行 numpy

如果你想看演算法骨架而非概念，因果遮罩的核心就這幾行。這段跑出來的 it 列，正是上面手算的正典 [0.665, 0.245, 0.090]：

```python
import numpy as np

def causal_mask(n):
    # upper triangle (j > i) is the future -> True means "mask it out"
    return np.triu(np.ones((n, n), dtype=bool), k=1)

scores = np.array([[3., 1., 0.],     # q=animal  (illustrative rows)
                   [1., 3., 0.],     # q=street
                   [2., 1., 0.]])    # q=it      (canonical spine [2,1,0])

scores[causal_mask(len(scores))] = -np.inf   # set the future to -inf BEFORE softmax
weights = softmax(scores, axis=-1)            # each row renormalizes over legal positions
# weights[2] -> [0.665, 0.245, 0.090]   (matches the hand-computed it row)
```

`np.triu(..., k=1)` 取的是**嚴格上三角**（不含對角線），剛好就是「未來」那些格子。把它們設 −np.inf，softmax 自動把它們的權重算成 0。整個因果遮罩，本質就是這一行 `scores[mask] = -inf`。

## 另一種遮罩：padding mask

因果遮罩管「時間」（不准看未來）；還有一種遮罩管「真假」——**padding mask（填充遮罩）**。它的存在純粹是工程上的便利，但你會在每個真實系統裡看到它，值得認得。

問題的來源你很熟：模型一次處理一**批（batch）**句子才有效率，但同一批裡的句子長度不一。「貓在睡覺」是 4 個 token、「那隻黑貓昨天整個下午都在沙發上睡覺」是 15 個。矩陣運算要求每一列等長，於是短句要被**補齊（padding）**到最長句的長度——尾巴塞幾個沒有意義的特殊 token（`[PAD]`），把 4 撐到 15。

```text
原句：     [ 貓 ,  在 ,  睡 ,  覺 ]
補齊後：   [ 貓 ,  在 ,  睡 ,  覺 , PAD, PAD, ... , PAD ]
                                  └────── 11 個沒有意義的填充 ──────┘
```

這些 `[PAD]` 是湊數的，沒有任何語意。如果讓真實的 token 去 attend 它們，模型就會把一堆垃圾混進加權平均裡。所以——**padding mask 把所有 `[PAD]` 欄位的分數也設成 −∞**，讓任何 token 對 `[PAD]` 的注意力權重歸 0（2026-06 查證）。手法和因果遮罩一模一樣（softmax 前設 −∞），只是遮的對象不同：

```text
因果遮罩：遮「未來位置」（j > i）        ← 管時間順序，保自迴歸
padding 遮罩：遮「填充位置（PAD）」      ← 管批次對齊，濾掉湊數的空 token
```

兩種遮罩可以**疊加**：生成模型在一個 batch 裡，既要不看未來（causal），又要不看 PAD（padding）。實作上就是兩個 −∞ 遮罩矩陣相加，一起加進分數再 softmax。它們本質是同一招的兩個用途——**用 −∞ 把「不該看的位置」在 softmax 之前排除掉**。

padding mask 是「與部署相關」那一側的東西（怎麼 batching、怎麼 pad、序列怎麼打包進記憶體），深度指向《從後端到 AI Infra》ch06（batching）。本書認得它、知道它和因果遮罩同源即可，不展開工程細節。

## 雙向 vs 單向：BERT、GPT 與遮罩的選擇

把本章收進全書的大圖。同一套自注意力，蓋不蓋因果遮罩、蓋哪種，決定了模型「看世界的方式」，也決定了它擅長什麼。這是 ch13 三種模型家族的伏筆，這裡先把「遮罩這條軸」講清楚。

| | 因果遮罩？ | 每個 token 能看到 | 擅長 | 代表 |
|---|---|---|---|---|
| **編碼器（encoder）** | **不蓋** | 全句（左右兩邊都看） | 理解、分類、抽取 | BERT |
| **解碼器（decoder）** | **蓋** | 只看自己與左邊（過去） | 生成（一個字一個字往後寫） | GPT |

關鍵對照（2026-06 查證）：

- **BERT 是雙向的（bidirectional）**：它**不蓋**因果遮罩，每個 token 都能 attend 全句的左邊和右邊。為什麼可以？因為 BERT 做的是**理解**任務——你給它一整句完整的話，要它判斷情感、抽取實體、做克漏字填空。整句話一開始就攤在桌上，沒有「未來尚未發生」的問題，當然要兩邊都看（左右都是線索）。這就是 ch07 那張對稱矩陣的自然棲息地。
- **GPT 是單向的（unidirectional / causal）**：它**蓋**因果遮罩，每個 token 只能看自己與左邊。為什麼必須？因為 GPT 做的是**生成**任務——它要學會「給定前面的字，預測下一個字」。訓練時，如果讓它在預測第 i+1 個字時偷看到第 i+1 個字（答案），它會學到一個作弊的捷徑（直接抄答案），上線生成時沒有未來可抄，立刻原形畢露。因果遮罩強迫它在訓練時也只能依賴過去，學到的才是真本事。

兩者**自注意力的機制完全相同**——同樣的 Q/K/V、同樣的內積、同樣的 softmax。**差別只在那個遮罩矩陣蓋不蓋。** 這是本章想讓你帶走的最大震撼之一：把一個世界級的「理解模型」改造成一個世界級的「生成模型」，在注意力這一層，差的不是架構，是**一個下三角的常數矩陣**。

> 為什麼今天的主流 LLM（GPT、Claude、Gemini…）幾乎都是 decoder-only 的單向因果模型？因為「預測下一個 token」這個訓練目標既通用又能無限自監督（任何文字都是現成的訓練資料：拿前 i 個字猜第 i+1 個），而它要成立，就**必然**要因果遮罩來防作弊。三種模型家族的完整譜系（含 encoder-decoder 的 T5）留 ch13；本章你只要記住：**遮罩是「生成」這件事的入場券。**

順帶說一個 ch12 的鉤子：剛剛講的都是 token 看**同一個序列**裡的其他 token（自注意力）。下一章會出現「目標序列的 token 去看**另一個**源序列」的交叉注意力（cross-attention）——那時遮罩的故事又會變一個樣子（cross-attention 通常**不蓋**因果遮罩，因為源序列是已經完整給好的）。先記著這個懸念。

## 故障模式與直覺陷阱

| 直覺陷阱 / 故障 | 為什麼錯／在哪一步把你帶溝裡 | 怎麼自我察覺、正確版是什麼 |
|---|---|---|
| **「要關掉一個連結，把它的權重（或分數）設成 0 就好」** | 致命的順序錯誤。遮罩動的是 **softmax 前的分數**；分數設 0 → `e⁰=1` → softmax 後權重不是 0、未來照樣被看到。 | 默念 `e⁰=1`、`e^(−∞)=0`。想讓「權重」變 0，要把「分數」設 −∞，不是設 0。 |
| **把遮罩加在 softmax 之後** | 先 softmax 再把某些權重清 0，會破壞「和為 1」，你得手動重新正規化，多一步又容易錯。 | 記住：遮罩屬於「正規化之前」。它要借 softmax 的正規化自動把剩下的合法權重分對。 |
| **以為「設 −∞」是直接寫 `float('-inf')` 就沒事** | 危險的「應該沒事」假設。真實實作常用一個**很大的負數**（如 float16 的 −65504）而非字面 −∞，因為 −∞ 在某些運算下會產生 `NaN`（如 `−∞ + (−∞)`、或一整列全被遮時 softmax 變 `0/0`）。 | 看到一整列權重變 `NaN`，先懷疑「是不是有一列被全遮了」（沒有任何合法位置可分），或「−∞ 參與了減法/加法」。實務用大負數＋保證每列至少一個合法位置。 |
| **以為因果遮罩改變了 it 對 animal 的權重** | 不會。it 在脊椎序列裡排最後，它本來就沒有未來可看，遮罩對它**毫無作用**——it 列在雙向和因果下都是 [0.665, 0.245, 0.090]。 | 遮罩只動「有未來可看」的位置。最後一個 token 的列永遠不受因果遮罩影響。 |
| **混淆 padding mask 與 causal mask** | 兩者都用 −∞，但管的事不同：causal 管時間（不看未來）、padding 管真假（不看湊數的 PAD）。把 padding 當成 causal，會在「雙向 encoder 也需要 padding mask」時想不通。 | 問自己：「我在擋的是『未來』還是『湊數的空 token』？」BERT 不蓋 causal 但**照樣要** padding mask。 |
| **「第一個 token 只看自己，是 bug」** | 不是 bug，是因果遮罩的數學必然——第一個 token 左邊沒東西，softmax 只能把權重全給自己（[1,0,0]）。 | 看到第一個 token 列是 [1,0,...] 屬正常。但要警覺它的近親：**attention sink**。 |

**具名失效模式：attention sink（注意力沉沒）。** 在真實的因果模型裡，有一個被反覆觀察到的現象：**序列開頭的 token（尤其第一個）會吸走大量的注意力權重**，即使它語意上根本不重要（可能只是個 `[BOS]` 開頭符號）。直覺上的成因和本章直接相關：因果遮罩下，靠前的 token 是**每一個後續 token 都看得到**的「公共過去」——它在每一列的合法集合裡都出現，於是 softmax 反覆把權重往它身上倒；而第一個 token 自己又只能看自己（[1,0,0]）。模型似乎學會了「當我這一步沒什麼特別想看的，就把多餘的注意力預算倒給開頭那個 token」，像一個洩壓閥。這個現象在長 context 與串流生成（streaming）裡有實際的工程後果（為什麼裁掉開頭 token 會讓模型崩掉），但那屬於部署/KV cache 的範疇，深度指向《從後端到 AI Infra》ch05。本章你只要建立直覺：**第一個 token 在因果遮罩下的特殊地位（人人看得到、自己只能看自己），是 attention sink 的結構性源頭。**

## 紙上推演

### 推演題

**第 1 題 [12 分鐘]（★）** 親手把因果遮罩套到脊椎矩陣上。給縮放後的 3×3 分數矩陣（列＝query：animal / street / it；欄＝key：animal / street / it）：

```text
  q=animal  [ 3, 1, 0 ]
  q=street  [ 1, 3, 0 ]
  q=it      [ 2, 1, 0 ]
```

(a) 寫出因果遮罩矩陣 M（哪些格子設 −∞）。(b) 對 **animal 列**和 **it 列**各算一次遮罩後的 softmax（一步不跳：寫出合法分數、各自的 e 次方、求和、相除）。(c) 比較 animal 列遮罩前 [0.844, 0.114, 0.042] 與遮罩後的差別，一句話說因果遮罩對 animal「拿掉了什麼」。

**第 2 題 [10 分鐘]（★★）** 「設 −∞ 而不是設 0」。(a) 若把 it 列要遮的位置（這題假設 it 不是最後一個、它右邊還有一個 token x，分數本來是 5）的分數設成 **0** 而非 −∞，手算它經 softmax 後會分到多少權重（用 it 對 animal/street/x = [2, 1, 0_設成的0]，注意這裡「0」是被你錯設的遮罩值）？(b) 為什麼這代表「未來沒被擋住」？(c) 改成設 −∞ 後權重變多少？用 `e⁰` 和 `e^(−∞)` 的值解釋這兩種做法的根本差別。

**第 3 題 [8 分鐘]（★★）** 把因果遮罩對到 append-only log。(a) 你維護一個交易 log，要重算帳戶在第 t 筆交易後的餘額——這對應因果遮罩的哪條規則？(b) 如果你的程式不小心讀到了第 t+1 筆（退款），對應到注意力裡是什麼故障？(c) 用「self-attention 機制不變、只多一個下三角遮罩」這句話，向另一個工程師解釋 BERT（理解）和 GPT（生成）的差別。

### 推演解答

**第 1 題。**
(a) 因果遮罩 M（j > i 設 −∞，其餘 0）：
```text
        an   st   it
animal [ 0  −∞   −∞ ]
street [ 0   0   −∞ ]
it     [ 0   0    0 ]
```
(b) **animal 列**：合法分數只剩 [3]（自己），softmax 單一元素 = **[1.000, 0.000, 0.000]**——只剩自己，權重全給自己。
**it 列**：合法分數 [2, 1, 0]（it 在最後，全部合法，遮罩沒動它）；e²=7.389、e¹=2.718、e⁰=1，和=11.107；權重 = [7.389/11.107, 2.718/11.107, 1.000/11.107] = **[0.665, 0.245, 0.090]**（和正典脊椎一致 ✓）。
(c) 遮罩**拿掉了 animal 對未來（street、it）的所有視野**：雙向時 animal 還能看 street（0.114）和 it（0.042），遮罩後這兩欄歸 0，權重被迫 100% 回到自己身上。animal 是第一個 token，左邊無物，只能盯自己。常見錯路：以為遮罩會「重新分配」未來的權重到別的未來位置——不會，它把所有未來都歸 0，只在**合法（過去＋當下）**的位置裡重新正規化。

**第 2 題。**
(a) 錯誤地把要遮的位置 x 設成分數 0，於是 it 的分數是 [2, 1, 0]（animal, street, x），其中第三個 0 是「本該被遮卻被你設成 0 的未來」。softmax([2,1,0]) = [0.665, 0.245, **0.090**]——**x（未來）分到了 0.090 的權重**。
(b) 0.090 不是 0。未來的 token x 實實在在地參與了加權混合，貢獻了 9% 的輸出——模型偷看了未來。因為 `e⁰ = 1`，設成 0 的分數在 softmax 裡和「分數本來就是 0 的合法位置」沒有任何區別，根本沒被當成「該排除」。
(c) 改設 −∞：分數變 [2, 1, −∞]，`e^(−∞) = 0`，x 對分母毫無貢獻；合法的 [2, 1] 重新正規化：e²=7.389、e¹=2.718，和=10.107，得 [0.731, 0.269, **0.000**]——x 權重剛好 0，乾淨擋掉。根本差別：`e⁰=1`（設 0 → 還在投票）vs `e^(−∞)=0`（設 −∞ → 退出投票）。softmax 的「關閉鍵」在 −∞，不在 0。

**第 3 題。**
(a) 對應「第 i 個 token 只能 attend j ≤ i」——重算第 t 筆後的餘額只能讀 ≤ t 的交易，等同因果遮罩的下三角規則（只看過去與當下）。
(b) 讀到 t+1（未來退款）＝注意力 attend 到了 j > i 的未來位置＝**因果遮罩失效（時光旅行）**。在生成模型裡這等於訓練時偷看答案，學到作弊捷徑、上線崩掉。
(c) 一個說得通的版本：「BERT 和 GPT 的 self-attention 機制一模一樣——同樣 Q/K/V、內積、softmax。唯一的差別是 GPT 在分數矩陣上多蓋一個下三角為 0、上三角為 −∞ 的因果遮罩，逼每個 token 只能看自己和左邊；BERT 不蓋，每個 token 兩邊都看。所以 BERT 適合『理解』（整句一開始就攤開、左右都是線索），GPT 適合『生成』（一個字一個字往後寫，不准偷看還沒寫的）。把理解模型改成生成模型，在注意力這層差的就是一個遮罩矩陣。」——點到「機制相同、只差遮罩」「雙向 vs 單向」「理解 vs 生成」就算抓到核心。

## 自我檢核

口頭自答，講得出來才算過：

1. 因果遮罩在三步骨架的哪個位置、動了什麼、**沒**動什麼？（提示：打分與正規化之間，覆寫送進 softmax 的分數；不動 Q/K/V、不動 softmax 公式。）
2. 為什麼遮罩要設 **−∞** 而不是設 **0**？用 `e⁰=1` 和 `e^(−∞)=0` 講一遍，並說清楚「你想設成 0 的其實是 softmax 之後的權重，但你能動的只有 softmax 之前的分數」。
3. 為什麼遮罩要加在 softmax **之前**而不是之後？（借 softmax 的正規化把剩下的合法權重自動分對、保住和為 1。）
4. 脊椎序列 [animal, street, it] 套上因果遮罩後，哪一列**完全沒變**、為什麼？哪一列塌成 [1, 0, 0]、為什麼？

6. causal mask 和 padding mask 都用 −∞，差別是什麼？為什麼**雙向的 BERT 不蓋 causal mask，卻照樣需要 padding mask**？
7. BERT（雙向）和 GPT（單向）的 self-attention 機制差在哪？（機制完全相同，只差蓋不蓋因果遮罩。）為什麼生成任務**必須**蓋？
