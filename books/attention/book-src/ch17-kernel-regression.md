## 從你已知的出發

你做過這件事，只是沒給它起學名。

你維護一個服務，想預測「一個新請求的延遲大概是多少」。你手上有一堆歷史請求，每筆記了它的特徵（payload 大小、來源地區、時段）和實際延遲。新請求進來，你不會去硬套一條公式，你會做更務實的事：**找出歷史裡跟它最像的那幾筆，把它們的延遲做個加權平均——越像的權重越大**。payload 大小幾乎一樣、地區也對得上的歷史請求，你給它高權重；八竿子打不著的，給低權重甚至忽略。最後混出一個估計值。

這套「拿相似的鄰居加權平均來估計」的做法，你可能用 KNN（k-nearest neighbors）的加權版做過，或者在做時間序列平滑、做推薦時的協同過濾時碰過它的變形。它有一個統計學上的正式名字：**核迴歸（kernel regression）**，而最經典的那個版本叫 **Nadaraya-Watson 估計量**（Nadaraya 1964、Watson 1964 各自獨立提出，2026-06 查證）。它的精神一句話講完：

> 一個查詢點的預測值，是所有樣本值的加權平均；權重由「查詢點和每個樣本有多像」決定，越像權重越大，且所有權重加起來為 1。

把這句話和你已經滾瓜爛熟的三步骨架並排——你會發現它們是**同一句話**。query＝查詢點，key＝樣本的位置（用來算「有多像」），value＝樣本的值，softmax＝「越像權重越大、加起來為 1」的那個機制，加權混合＝「加權平均」。注意力不是發明了一個新運算，它是把這個 1964 年就有的核迴歸，做成了**可學習、可微分、可平行**的版本。

```text
        Nadaraya-Watson 核迴歸（1964）          注意力（2017）
        ────────────────────────────           ──────────────────
        查詢點 x                          ←→    query q
        樣本位置 xᵢ（拿來算相似度）       ←→    key kᵢ
        樣本值 yᵢ（要被平均的東西）        ←→    value vᵢ
        核 K(x, xᵢ)＝相似度               ←→    打分 q·kᵢ（內積即相似度）
        正規化權重 ΣK=1                    ←→    softmax（正規化成分布）
        加權平均 Σ wᵢyᵢ                    ←→    加權混合 Σ αᵢvᵢ
```

本章在三步骨架上**哪一步都沒動**。它不是引入一個新零件，而是給你一副新眼鏡：戴上它，你看見的不再是「query/key/value 三個投影」，而是「一個查詢點對一堆帶標籤的樣本做局部加權迴歸」。同一個機制，兩種讀法。下面我們先把核迴歸講清楚，再把它和注意力一個零件一個零件對上，最後用脊椎那個 it 親手驗證這個對應是真的、不是比喻。

## Nadaraya-Watson：用鄰居加權平均來預測

先把核迴歸這個東西本身講清楚，假裝還沒聽過注意力。

你有一堆訓練樣本 (x₁, y₁), (x₂, y₂), …, (xₙ, yₙ)——每個 xᵢ 是輸入（如請求特徵），每個 yᵢ 是對應的輸出（如延遲）。現在來了一個新的查詢點 x，你想預測它的輸出 f(x)。

最笨的辦法：找出離 x 最近的那一個樣本，直接回傳它的 y（這是 1-NN，最近鄰）。問題是它太武斷——只看一個鄰居，一個雜訊點就能把預測帶歪，而且預測曲線會是一格一格的階梯，不平滑。

Nadaraya-Watson 的辦法溫柔得多：**不挑一個，全都用，但按「離得多近」加權**。離 x 近的樣本，它的 y 在平均裡占比重；離得遠的，占比小。怎麼把「離多近」變成「占多少權重」？用一個**核函數（kernel）** K——它吃進「x 和 xᵢ 的距離」，吐出一個「相似度分數」（距離越近分數越高）。最常用的是高斯核（Gaussian kernel）：

```text
        K(x, xᵢ) = exp( −(x − xᵢ)² / 2 )      ← 距離平方越大，值越小（指數衰減）
                                                  距離為 0 時 K=exp(0)=1（最大）
```

有了核，預測就是「核加權的平均」。先算每個樣本的核值當原始權重，再正規化成「加起來為 1」，最後加權平均所有 yᵢ：

```text
                  Σᵢ K(x, xᵢ) · yᵢ                      K(x, xᵢ)
        f(x)  =  ──────────────────   =   Σᵢ  ─────────────────  · yᵢ
                  Σⱼ K(x, xⱼ)                  Σⱼ K(x, xⱼ)
                                              └──────┬───────┘
                                              這一坨就是樣本 i 的權重 wᵢ
```

中間那一坨 `K(x,xᵢ) / Σⱼ K(x,xⱼ)` 是樣本 i 分到的權重 wᵢ：非負（核值非負）、加起來為 1（分子全加起來就是分母）。看到沒——**這跟 softmax 的兩個鐵打性質（非負、和為 1）一模一樣**。事實上，如果你把高斯核代進去，那個正規化分數**就是一個 softmax**：

```text
                   exp(−(x−xᵢ)²/2)
        wᵢ  =  ─────────────────────   =   softmax( −(x−xᵢ)²/2 )ᵢ
                Σⱼ exp(−(x−xⱼ)²/2)
```

（這是 Dive into Deep Learning「Attention Pooling」那章用的同一條式子，2026-06 查證。）核迴歸的「正規化高斯核」和注意力的「softmax」不是像，是**字面上同一個函數**。差別只在打分的方式：核迴歸用「負的距離平方」當分數，注意力用「內積」當分數——但兩者都是「相似度的某種度量」，正規化的機器是同一台。

> **這裡了不起／反直覺在哪**（請你能用自己的話轉述）：softmax 不是某個深度學習的人為了 attention 發明的怪招。它在統計裡早就以「正規化高斯核」的身分存在了幾十年——它就是「把一排相似度分數變成一個加起來為 1 的權重分布」的標準做法。當你說「注意力用 softmax 把分數變權重」，你其實在說「注意力做的是核迴歸的正規化」。同一件事，兩個領域各取了一個名字。

### 一個 1 維手算：把房價的鄰居加權平均

別停在公式上，親手算一遍最有感。場景：你想用「房子的大小」預測「房價」，手上有三筆歷史成交（為了能心算，數字都湊整）：

```text
        樣本     大小 xᵢ    成交價 yᵢ
        ──────────────────────────────
        A         2.0        4.0
        B         3.0        6.0
        C         6.0        2.0       ← C 是個遠處的離群點（大房子反而便宜）
```

現在來了一間大小 **x = 3.0** 的房子，預測它的價格。用高斯核（頻寬 h=1，即 `K=exp(−(x−xᵢ)²/2)`）。

第一步，打分——算每個樣本對查詢點的核值（相似度）：

```text
        到 A：x−xᵢ = 3.0−2.0 =  1.0   → K = exp(−1.0²/2) = exp(−0.5) = 0.6065
        到 B：x−xᵢ = 3.0−3.0 =  0.0   → K = exp( 0      ) = exp(0)    = 1.0000   ← 完全重合，相似度最高
        到 C：x−xᵢ = 3.0−6.0 = −3.0   → K = exp(−3.0²/2) = exp(−4.5) = 0.0111   ← 太遠，幾乎沒貢獻
```

第二步，正規化——把三個核值變成加起來為 1 的權重：

```text
        Σ K = 0.6065 + 1.0000 + 0.0111 = 1.6176
        w_A = 0.6065 / 1.6176 = 0.375
        w_B = 1.0000 / 1.6176 = 0.618
        w_C = 0.0111 / 1.6176 = 0.007
        （驗算：0.375 + 0.618 + 0.007 = 1.000 ✓）
```

第三步，加權混合——用權重對三個 yᵢ 做加權平均：

```text
        f(3.0) = 0.375·4.0 + 0.618·6.0 + 0.007·2.0
               = 1.500    + 3.708    + 0.014
               ≈ 5.22
```

預測房價約 **5.22**。讀一下這個結果為什麼合理：查詢點 3.0 和 B（也是 3.0）完全重合，所以 B 拿了 61.8% 的權重，預測值被它的 6.0 強烈拉動；A 稍遠拿 37.5%；C 那個離群點離得太遠，只拿了 0.7%，幾乎被無視——這正是我們要的，不讓遠處的雜訊污染預測。

現在把這三步的名字換掉，再讀一次：第一步**打分**（算相似度核）、第二步**正規化**（變成和為 1 的權重）、第三步**加權混合**（對 value 加權平均）。**這就是三步骨架。** 你剛剛在算房價，但你用的是和算 it 該看哪裡一模一樣的機器。

## 把注意力一個零件一個零件對上核迴歸

1 維房價的例子讓你看到「形狀」對得上。現在嚴格一點，把注意力的每個零件**逐項**對到核迴歸，看對應是不是真的處處成立。

```text
   核迴歸的角色                    注意力的角色            說明
   ──────────────────────────────────────────────────────────────────
   查詢點 x                        query q               「我要預測誰」
   樣本位置 xᵢ                      key kᵢ                「拿來量相似度的座標」
   樣本值 yᵢ                        value vᵢ              「真正要被平均的內容」
   核 K(x, xᵢ)＝相似度              打分 score(q, kᵢ)      「x 和 xᵢ 有多像」
   正規化的核權重 wᵢ                注意力權重 αᵢ          「非負、和為 1 的混合配比」
   加權平均 Σ wᵢ yᵢ                 加權混合 Σ αᵢ vᵢ       輸出（一個對 value 的局部平均）
```

最關鍵、也最該停下來看的，是**核函數**那一格。核迴歸通常用「距離」當相似度（離得近＝像），注意力用「內積」當相似度（內積大＝像）。這兩者其實是一家人：如果 q 和 k 的長度固定，內積 `q·k = ‖q‖‖k‖cos θ` 就只看夾角，而夾角小（方向像）等價於歐氏距離近——`‖q−k‖² = ‖q‖² + ‖k‖² − 2q·k`，長度固定時「距離小」和「內積大」是同一件事（內積即相似度，深入見《矩陣是動詞》ch15）。所以注意力選的核是**指數內積核（exponential dot-product kernel）**：

```text
        κ(q, k) = exp( q·k / √d )          ← 注意力用的核
```

注意這個 `q·k/√d`——它正是 ch06 那個縮放後的分數！把它放進「正規化的核權重」公式，你得到的就是 `softmax(QKᵀ/√d)`，一個字都不差。也就是說：

> 縮放點積注意力 `Attention(Q,K,V) = softmax(QKᵀ/√d)·V`，**就是**以「指數內積核」為核的 Nadaraya-Watson 核迴歸。query 是查詢點、key 是樣本座標、value 是樣本值，softmax 把核值正規化成和為 1 的權重，乘上 V 就是核加權平均。

這是 Tsai et al. 2019「Transformer Dissection: A Unified Understanding of Transformer's Attention via the Lens of Kernel」（arXiv 1908.11775，EMNLP-IJCNLP 2019，2026-06 查證）系統性建立的視角——他們把注意力寫成一個 **kernel smoother（核平滑器）**，未正規化的注意力＝一個核、正規化後的注意力＝核平滑因子。

> 嚴謹度標示：這是一個**深刻的等價視角，不是巧合也不是鬆散的比喻**——你真的可以把縮放點積注意力的公式，一字不漏地寫成 Nadaraya-Watson 核迴歸的公式（上面對應表每一格都成立）。要誠實標清楚的地方有兩處：① 這個「核」是**非對稱**的（κ(q,k) ≠ κ(k,q)，因為 q 經過 W_Q、k 經過 W_K，是不同投影——這正是 ch04「我找的≠我是的」的回聲），也**不保證**是傳統核方法要求的半正定核；Tsai 的論文明確說他們**放寬**了核必須對稱半正定的要求（2026-06 查證）。② 注意力比經典核迴歸多了「投影矩陣 W_Q/W_K/W_V 是學出來的」——經典 Nadaraya-Watson 的核是**固定**的（你手選高斯核、手調頻寬），注意力的核是**學出來的**（模型自己學該用什麼相似度、頻寬也能參數化）。把這兩點記清楚，這個對應就既漂亮又站得住腳。

## 加權混合就是條件期望：連回《馴服隨機》

核迴歸的視角還順手解開了一個你可能一直覺得「就是個加權平均嘛」的東西——加權混合那一步，到底在估計什麼？

答案是**條件期望**。在《馴服隨機》裡你看過：一個隨機變數的期望，就是「各個取值乘上它的機率，加起來」——E[Y] = Σ P(yᵢ)·yᵢ。而注意力的輸出是 Σ αᵢ·vᵢ，其中 αᵢ 是一個合法機率分布（非負、和為 1，ch05 證過）。把這兩個式子並排：

```text
        期望         E[Y]       = Σ  P(yᵢ) · yᵢ        ← 機率當權重
        注意力輸出   Attention  = Σ  αᵢ   · vᵢ        ← 注意力權重當「機率」
```

它們是同一個結構。所以注意力的輸出，可以讀成「**以注意力權重為機率分布，對 value 取的期望值**」。更貼切地說，因為這個分布是「給定 query q」之後才算出來的，它估計的是一個**條件期望** E[value | query]——「在我知道查詢點 q 是這個的條件下，value 的期望值是多少」。

這也是統計學裡 Nadaraya-Watson 的本來面目：它就是**條件期望 E[Y | X=x] 的一個非參數估計量**。迴歸要估的本來就是 E[Y|X]（給定輸入，輸出的期望），核迴歸用「鄰居加權平均」去逼近它，注意力用「學出來的相似度加權平均」去逼近它。

> **這裡值得你停十分鐘**：「加權混合」這三個字，在 ch04–05 你只把它當成「按權重把 value 平均起來」的機械步驟。現在它有了統計意義——它在**估計一個條件期望**。it 那個位置的輸出 (0.755, 0.335)，不只是「三個 value 的加權平均」，它是「**給定 it 這個 query，value 空間的條件期望估計**」。三步骨架的第三步，從一個算術操作，升格成一個統計估計量。同一個數字，意義變深了。

回到脊椎驗證這件事（脊椎正典數字，ch06 起一律用這組）：縮放後權重 [0.665, 0.245, 0.090] 是一個機率分布；value 是 v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)。輸出：

```text
        E[value | q=it] = 0.665·(1,0) + 0.245·(0,1) + 0.090·(1,1)
                        = (0.665+0.090, 0.245+0.090)
                        = (0.755, 0.335)
```

這個 (0.755, 0.335) 就是「以 [0.665, 0.245, 0.090] 為機率、對 value 取的期望」。it 該看哪裡的那個輸出，本質是一個條件期望的估計——是核迴歸對「value | it」這個迴歸函數的局部估計值。

## 不同的核：為什麼偏偏是 exp

核迴歸的視角給你一個自然的問題：核函數有一大家子（高斯核、線性核、RBF、多項式核…），注意力為什麼偏偏選「exp 內積」這個核？這個問題很值得問，因為它把 ch05「為什麼非得是 exp」的回答提升到了核方法的層次。

```text
   核的選擇            權重長相                    特性與代價
   ─────────────────────────────────────────────────────────────────────
   均勻核（box）       窗內全 1、窗外全 0           硬切窗，不可微，邊界跳變
   線性核 q·k          直接拿內積當權重             可能為負（負權重沒有混合語義）；
                                                    不放大差距、不夠果斷
   RBF / 高斯核        exp(−‖q−k‖²/2h²)            看距離；經典核迴歸的預設
   exp 內積核（softmax）exp(q·k/√d) 再正規化         看方向相似度；非負、放大差距、
                                                    處處可微、平移不變——注意力的選擇
```

注意力選 exp 內積核，理由其實 ch05 全講過了，只是現在用核的語言重述一遍會更清楚它「贏在哪」：

- **非負**。exp 把任何實數（內積可正可負）壓成正數——權重才有「混合配比」的意義。線性核 q·k 直接當權重會出現負值，混合語義崩掉。
- **指數放大差距**。exp 讓相似度的差被指數級拉開（ch05 算過：分數差 1 換成權重比約 e≈2.72 倍），注意力才「夠果斷」——相關的明顯勝出。均勻核（一刀切窗）和線性核都做不到這種柔和又果斷的聚焦。
- **處處可微、平移不變**。模型靠梯度下降學那些投影矩陣，需要核處處有斜率（均勻核在窗邊界不可微，學不動）；平移不變（分數同加常數結果不變）讓數值穩定的把戲（減最大值，ch16 online softmax 的核心）成立。

換句話說，注意力沒有亂選核——它選了一個**同時滿足「非負、果斷、可學」的核**，而 exp 內積核是這幾個條件下最自然的那一個。這也預告了一件事：ch15 的**線性注意力**和 ch19 末尾的 **sigmoid/ReLU 注意力**，動的正是「換掉這個核」——它們把 exp 內積核換成別的核（線性核、sigmoid 核），想省掉 softmax 的成本，代價就是失去 exp 帶來的某些好處（ch15 會看到線性注意力在精確回憶上吃虧）。核的視角讓「換 softmax」這件事有了統一的說法：**換注意力的正規化，本質就是換核**。

## 故障模式與直覺陷阱

| 陷阱 | 錯誤直覺 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| **把「等價視角」當成「就是同一個東西、毫無區別」** | 「注意力＝核迴歸，那核方法的所有定理都能直接套」 | 經典核方法要求核**對稱、半正定**；注意力的指數內積核**非對稱**（q、k 不同投影）、不保證半正定。直接套 Mercer 定理、RKHS 那套會出錯 | 提醒自己：這是「寫得出同一個公式」的等價，不是「滿足同一組數學前提」。Tsai 自己也明說放寬了對稱半正定要求 |
| **以為核迴歸的核是固定的，所以注意力也是** | 「注意力就是個固定的高斯平滑器」 | 經典 Nadaraya-Watson 的核是**手選固定**的；注意力的核**經 W_Q/W_K 學出來**、相似度的定義是訓練出來的——這正是它比經典核迴歸強的地方 | 問自己：W_Q、W_K 是常數還是學來的參數？是學來的，所以「核」是學出來的 |
| **把「加權混合＝期望」當成「輸出是某個 value」** | 「條件期望 E[value\|q] 應該等於最像的那個 value」 | 期望是**混合**出來的，幾乎永遠不等於任何一個單獨的 value。輸出 (0.755,0.335) 不等於 v_animal=(1,0)——它是期望，落在 value 們的「中間」 | 回扣 ch05：注意力是 soft-argmax，期望是混合全部、不是挑一個 |
| **以為核迴歸視角能解釋「attention 為什麼有效」** | 「既然是核迴歸，那它的泛化保證就解釋了 LLM 為什麼強」 | 核迴歸是一個**描述性**的視角（描述注意力在算什麼），不是一個**解釋力**的理論（它沒解釋為什麼學出來的核剛好抓對了語言結構）——別把「看清結構」當成「解釋了能力」 | 區分「這個機制在算什麼」（核迴歸答得很好）和「它為什麼學得這麼好」（這是開放問題，核迴歸沒回答） |
| **過度套 KNN 直覺，以為注意力只看「k 個最近鄰」** | 「注意力就是加權版 KNN，只看附近幾個」 | Nadaraya-Watson 和注意力都**用全部樣本**（每個都有非零權重），不是只取 top-k；「只看 k 個」是 KNN 的硬截斷，注意力沒有（除非你刻意做 sparse，ch15） | 看權重：softmax 給每個 token 的權重都 >0（除非被 −∞ 遮罩）。沒有「只看最近 k 個」這回事 |

最值得記住的是第一個——**「等價」不等於「全等」**。核迴歸視角是一面好鏡子，照出「注意力在算什麼」照得清清楚楚；但它是一個**描述**，不是一張可以把核方法所有定理直接搬過來的通行證。注意力的核是非對稱、學出來的、不保證半正定——這些「不一樣」恰恰是它能成事的地方。把鏡子當成望遠鏡（以為它能解釋能力的來源），是這個視角最容易被過度解讀的方向。

## 紙上推演

### 推演一：1 維核迴歸手算 **[15 分鐘] ★**

訓練樣本三筆：(x₁=2.0, y₁=4.0)、(x₂=3.0, y₂=6.0)、(x₃=6.0, y₃=2.0)。查詢點 **x=3.0**，用高斯核 `K=exp(−(x−xᵢ)²/2)`。(a) 算三個核值（不准看上文，自己從距離算起）；(b) 正規化成三個權重，驗算加起來是 1；(c) 算加權平均預測值 f(3.0)；(d) 用一句話說明為什麼 C 樣本（離群點）幾乎沒影響預測。

### 推演二：把注意力翻譯成核迴歸 **[12 分鐘] ★★**

把縮放點積注意力 `softmax(QKᵀ/√d)·V` 的每個零件，逐項對到 Nadaraya-Watson 的角色（query↔?、key↔?、value↔?、softmax↔?、加權混合↔?）。然後回答：(a) 注意力用的「核」是什麼（寫出 κ(q,k)）？(b) 它和經典高斯核最大的兩個不同是什麼（提示：對稱性、固定 vs 學習）？(c) 為什麼說這是「等價視角」而不只是「比喻」？

### 推演三：加權混合＝條件期望 **[15 分鐘] ★★**

用脊椎正典數字：權重 [0.665, 0.245, 0.090]、value v_animal=(1,0)、v_street=(0,1)、v_it=(1,1)。(a) 把輸出 (0.755, 0.335) 重新寫成「以 [0.665,0.245,0.090] 為機率分布、對 value 取的期望 E[value|q=it]」的形式，並算出來核對；(b) 解釋為什麼這個輸出**不等於**任何一個單獨的 value，這和 ch05 的哪個招牌反直覺是同一件事；(c) 進階：為什麼說注意力在估計的是「條件」期望而不只是期望？「條件」在哪裡？

### 推演解答

**推演一**。
(a) 距離：3.0−2.0=1.0、3.0−3.0=0.0、3.0−6.0=−3.0。核值：exp(−1.0²/2)=exp(−0.5)=**0.6065**、exp(0)=**1.0000**、exp(−3.0²/2)=exp(−4.5)=**0.0111**。
(b) Σ=0.6065+1.0000+0.0111=1.6176；權重 = [0.6065/1.6176, 1.0/1.6176, 0.0111/1.6176] = **[0.375, 0.618, 0.007]**（驗：和=1.000 ✓）。
(c) f(3.0)=0.375·4.0 + 0.618·6.0 + 0.007·2.0 = 1.500+3.708+0.014 ≈ **5.22**。
(d) C 的座標 6.0 離查詢點 3.0 有 3 個單位，高斯核 exp(−4.5)≈0.011 已經很小，正規化後只剩 0.7% 權重——它的離群值 2.0 幾乎進不了平均。這正是核迴歸（與注意力）的好處：遠的（不相似的）自動被壓到接近忽略，不用手寫規則去排除離群點。常見錯路：以為要先把 C 剔除——不用，核函數自動幫你把它的權重壓到趨近 0。

**推演二**。對應：query↔查詢點 x、key↔樣本位置 xᵢ、value↔樣本值 yᵢ、softmax↔正規化的核權重（和為 1）、加權混合↔核加權平均 Σwᵢyᵢ。
(a) 核 κ(q,k) = **exp(q·k/√d)**（指數內積核，正規化前）。
(b) 兩個不同：① **非對稱**——κ(q,k)≠κ(k,q)，因為 q 經 W_Q、k 經 W_K 是不同投影（經典高斯核 exp(−‖x−xᵢ)²) 對稱）；② **學出來而非固定**——W_Q/W_K 是訓練學的參數，所以「用什麼相似度」是學出來的（經典核迴歸的核是手選固定的）。
(c) 因為你能把注意力的公式**一字不漏地寫成** Nadaraya-Watson 的公式（每個零件都對得上、不是鬆散類比）。比喻是「像」，等價是「寫得出同一條式子」。但要誠實標：這個核非對稱、不保證半正定，不能直接套經典核方法的所有定理。

**推演三**。
(a) E[value|q=it] = 0.665·(1,0)+0.245·(0,1)+0.090·(1,1) = (0.665+0.090, 0.245+0.090) = **(0.755, 0.335)**——和輸出一致 ✓。權重 [0.665,0.245,0.090] 是合法機率分布（非負、和為 1），所以「加權混合」就是「對 value 取期望」。
(b) 因為期望是**混合**出來的：(0.755,0.335) 不等於 (1,0)、(0,1)、(1,1) 任何一個，它落在三者「中間偏 animal」。這和 ch05「注意力是 soft-argmax、混合全部不是挑一個」是同一件事——期望本來就不會等於任一個樣本值（除非分布是 one-hot）。常見錯路：以為「最像的 value 是 animal，輸出就該是 animal 的 (1,0)」——那是 argmax 不是期望。
(c) 「條件」在於這個機率分布 [0.665,0.245,0.090] 是**給定 query=it 之後**才算出來的——換一個 query（比如讓 animal 當 query），權重分布就不同，期望也不同。所以它估計的是 E[value | query=這個特定 query]，是隨 query 變的條件期望，不是一個固定的無條件期望。

- **Dive into Deep Learning,「Attention Pooling: Nadaraya-Watson Kernel Regression」**（https://classic.d2l.ai/chapter_attention-mechanisms/nadaraya-watson.html ）——把「注意力＝核迴歸」講得最清楚、最動手的一篇教材，本章 1 維房價例子的高斯核 `f(x)=Σ softmax(−½(x−xᵢ)²)yᵢ` 就是它的式子；想看「非參數→參數化（頻寬可學）」怎麼自然過渡到注意力，讀這篇。

- **《矩陣是動詞》ch15「內積即相似度」**——本章「指數內積核 vs 高斯距離核其實一家」靠的是「長度固定時，內積大⇔距離近」這個事實，那本把內積作為相似度度量講得很細；想確認「方向相似度」和「距離相似度」怎麼換算，回那章。
