本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名：《注意力機制的原理》（副標：直覺、機制與理論）。

## 這本書是什麼、不是什麼

- **是**：一本「原理與欣賞」的書。目標是讓讀者徹底看懂「注意力（attention）」這個機制到底在算什麼、為什麼這個極簡的點子（打分→正規化→加權混合）能撐起 2017 年以降整個大型語言模型的時代——讀完有「原來就這麼一件事，竟能長成這樣」的震撼，**而且能用自己的話對另一個工程師把它講清楚**：Q/K/V 是什麼、為什麼除以 √d、multi-head 在分什麼工、位置編碼補了什麼洞、為什麼是 O(n²)、attention 權重能不能當解釋、2026 年它是否還是唯一解。
- **不是**：
  - 不是訓練教學或框架課——不教 PyTorch/JAX、不寫 `nn.Module` 樣板、不教怎麼訓練一個 transformer、不調參。
  - 不是論文綜述——不堆名詞、不羅列 SOTA 排行榜；每個被點名的變體都要講清楚它**改了三步骨架的哪一步、為什麼**，否則不收。
  - 不是部署/服務工程書——**KV cache 怎麼管、FlashAttention 怎麼吃滿 SRAM、PagedAttention、prefill/decode 分離、量化、batching 全部不展開**，指向書架上《從後端到 AI Infra》（那是「與工作相關」的書；本書刻意只談原理，與工作無關，見「跨書邊界」）。
  - 不是線性代數課——內積、矩陣乘法、投影、SVD 的**深入**指向《矩陣是動詞》；本書只用「夠用的直覺」把它們當工具，需要時喚醒。
- **驗收標準（全書七條主線，收官章口試）**：
  ① 注意力到底在算什麼（一句話：對所有位置做一次「依相關性加權的平均」——三步骨架）；
  ② 為什麼要 Q/K/V 三個角色、為什麼除以 √d、為什麼 softmax（soft-argmax）；
  ③ multi-head 在分什麼工、位置編碼補了「集合沒有順序」這個洞；
  ④ 為什麼是 O(n²)、業界怎麼想「替這張沒有索引的表建索引」（sparse/linear/FlashAttention 的想法層）；
  ⑤ attention 的理論身世（核迴歸／Hopfield／集合上的置換等變運算）——它不是憑空冒出來的；
  ⑥ attention 權重能不能當「解釋」（招牌故障視角）、機制可解釋性看到了什麼（induction heads、QK/OV 電路）；
  ⑦ 2026 年「attention is all you need」還成立嗎（SSM/Mamba/混合架構之爭，誠實標時點）。

## 全書脊椎：同一個句子裡的「it」（對應機率那本的「同一枚硬幣」、線代那本的「同一個矩陣」）

全書貫穿**同一個極小範例**，每深入一層就重新算它一次。脊椎句子（沿用原始 Transformer 論文的招牌視覺化例子）：

> *"the animal didn't cross the street because **it** was too tired."*

核心懸念：代名詞 **it** 指的是 animal 還是 street？人一眼知道是 animal（累的是動物不是街）。本書要證明：注意力機制就是讓 **it** 這個位置「**該看哪裡**」——把最大的權重放到 animal 上——而這件事是**算**出來的，不是規定的。

為了能手算，脊椎用三個 token 的簡化序列 **[animal, street, it]**，每個 token 配一個小向量（先 2 維、深入後擴到 4 維），Q/K/V 投影是小矩陣。**全書所有章節共用同一組脊椎數字**（精確向量與矩陣見 `outline.md` 的「脊椎基準表」，不得各章另創）。各章用到時**必須明寫「這是序章那個 it」**，並在收官章總對帳。脊椎的多層讀法：

- 序章 / 動機：it 該看 animal 還是 street？先只給直覺，不算。
- scaled dot-product 章：手算 q_it·k_animal、q_it·k_street、q_it·k_it 三個分數 → 除以 √d → softmax 得權重（如 [0.7, 0.2, 0.1]）→ 加權混合三個 value → 輸出偏向 animal。**三步骨架第一次完整跑完**。
- softmax/√d 章：同一組分數，改溫度與是否除 √d，看權重從「平」到「尖」如何變，理解 √d 在防什麼。
- multi-head 章：head A 抓句法（誰是主詞）、head B 抓指代（it→animal），concat 起來——同一個 it，兩顆頭看不同子空間。
- 位置編碼章：拿掉位置，"animal street it" 與 "it street animal" 給出**完全一樣**的注意力（置換不變）→ 這就是非加位置不可的洞。
- masking 章：生成時 it 只能看它左邊的 token（animal、street 在前，合法）；對比雙向 encoder 可以兩邊都看。
- cross-attention 章：把句子翻成中文，目標端的 query 去看英文來源端的 key/value（同一個 it，跨序列查找）。
- 可解釋性章：it→animal 這條連結被畫成一個具體的 QK/OV 電路與「指代頭」。
- 收官章：「同一個 it，現在你看懂它八層」總對帳表。

## 全書統一骨架：三步骨架（每章都回扣）

注意力的**所有**變體，本質都是這三步的精煉。全書用同一組詞，不得換名：

```text
            ┌──────────┐   ┌────────────┐   ┌──────────────┐
 query ──►  │ 1. 打分  │─► │ 2. 正規化  │─► │ 3. 加權混合  │ ──► 輸出
 keys  ──►  │  score   │   │  softmax   │   │  weighted sum │
 values ─────────────────────────────────────────┘（值在此進場）
```

1. **打分（score）**：query 與每個 key 算「相關性分數」，標準作法是內積 q·k（內積即相似度——見《矩陣是動詞》ch15）。
2. **正規化（normalize）**：把一排分數用 softmax 變成「和為 1 的權重」——一個機率分布（見《馴服隨機》：這是一個分布）。
3. **加權混合（mix）**：用權重對所有 value 做加權平均，得到輸出。

口訣：**相關性決定權重，權重決定混合**。書中每介紹一個變體（√d 縮放、multi-head、RoPE、linear/sparse、FlashAttention、cross-attention、SSM 對照…），都要明確點出它**動了哪一步、為什麼**。這是讀者「不被名詞淹沒」的浮木。
