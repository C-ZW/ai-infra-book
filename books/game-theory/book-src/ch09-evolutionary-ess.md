# ch09 — 演化賽局與 ESS：均衡不需要會算的玩家

> **本章解決什麼問題**：到 ch08 為止，「均衡」一直暗藏一個前提——有個會算報酬、會推理對手、會挑最佳反應的理性玩家坐在那裡。本章把這個前提整個抽掉，問一個瘋狂的問題：**如果沒有人在算，均衡還會出現嗎？** 答案是會，而且這正是 Part III「合作的考古學」最後一塊拼圖：演化穩定策略（ESS）告訴你，天擇可以代替理性去執行「最佳化」，讓族群自己滾到一個沒有突變能入侵的穩定點。它接在 ch07（重複賽局）與 ch08（一報還一報與無名氏定理）之後，把那兩章「合作要靠未來陰影撐住」的故事換成「合作要靠繁殖差異篩出來」的版本；同時它是脊椎七層的第六層——把 ch01 那張囚徒困境矩陣放進一整個族群，看背叛者怎麼把合作者吃乾抹淨。

## 從你已知的出發

你跑過漸進放量（canary rollout）。新版本上線，先給 1% 流量，盯著它的錯誤率、P99 延遲、轉換率。表現好，比例往上調到 5%、20%、100%；表現差，回滾，比例歸零。整個過程裡，**沒有任何一個版本「決定」要勝出**——是你的放量規則根據指標，自動把表現好的版本比例往上推。版本本身不會算、不會推理、不會挑最佳反應；它只是「碰巧」在這個流量、這個負載下跑得好，於是被你的機制保留、擴大。

把「你盯著指標手動調比例」換成「指標差就自動縮、好就自動放」，你就得到了一個沒有設計者在場的選擇過程。這就是本章的核心轉換：**「最佳化」不一定要由一個會算的腦袋來做，它可以由一個「表現好就被放大、表現差就被淘汰」的迴圈來做。** 天擇就是這個迴圈，跑在生物族群上、跑了幾億年。

你其實對這個直覺非常熟，只是沒這樣命名：

- **A/B test**：兩個版本同場競爭，轉換率高的那個「比例上升」，最後吃掉全部流量。沒有版本在策略性思考，是環境（使用者點擊）在篩。
- **自動彈性伸縮（autoscaling）**：負載高就多開 pod、低就回收。沒有 pod 在「決定」要不要存在，是負載這個外部壓力在決定族群大小。
- **GC 與快取淘汰**：LRU 不「思考」哪個 entry 該死，它只執行一條「最久沒用的先淘汰」的規則。被淘汰的 entry 沒做錯什麼，它只是在當前的存取模式下「適存度低」。

這些系統的共通點是：**有個明確的選擇壓力（指標、負載、存取頻率），表現好的東西比例上升、表現差的下降，而被選的東西本身完全不需要理解這場競爭。** 演化賽局就是把賽局理論搬進這個框架。本章要回答兩個問題：（一）當「玩家」變成「一整個族群裡某種行為的比例」、「報酬」變成「繁殖成功率」，納許均衡這個概念會變成什麼？（二）那個新概念——演化穩定策略——憑什麼比納許均衡更挑剔，挑剔在哪？

先預告本章最反直覺的一刀，因為你的直覺正要騙你：**你會忍不住覺得「動物會打架但很少打死對方，是因為牠們『懂得』節制」。錯。沒有任何動物懂任何東西。那個節制是一個比例，是天擇在「愛打架的基因」和「愛退讓的基因」之間滾出來的平衡點——一個均衡，而場上沒有任何一個玩家在算報酬。**

## 四個翻譯：把賽局搬進族群

賽局理論原本的語言是「理性玩家挑策略最大化報酬」。演化生物學要用它，得先把每個詞翻譯過去。這張對照表是本章的地基，值得你停下來逐行對：

```text
古典賽局理論              演化賽局理論
─────────────────────    ─────────────────────────────────
玩家（player）       →   族群裡的一個個體（不會算、不會選）
策略（strategy）     →   表現型／行為型（基因決定，天生帶著）
報酬（payoff）       →   適存度（fitness）＝繁殖成功的多寡
「挑最佳反應」       →   天擇：適存度高的型，下一代比例上升
納許均衡             →   演化穩定策略（ESS）：擋得住突變入侵的比例
混合策略 (p, 1−p)    →   兩種詮釋：① 族群裡 p 比例是型 A、(1−p) 是型 B
                         ②每個個體有 p 機率表現成 A（多型 vs 個體混合）
```

最關鍵、也最容易被忽略的是中間那條：**「挑最佳反應」被「天擇」取代了。** 古典賽局裡，是玩家自己在腦中比較「我選 A 還是 B 報酬高」然後選高的那個。演化賽局裡，沒有人在比較——是「碰巧帶著高適存度行為型的個體，留下更多後代，於是這個行為型在下一代的族群裡佔比更高」。**比較與最佳化沒有消失，它從「個體腦中的計算」搬到了「世代之間的統計」。** 這是整章的樞紐，後面所有驚嘆都從這裡長出來。

順帶釐清一個你大概會卡的點：演化賽局裡的「報酬」不是金錢、不是效用、不是快樂，是**適存度**——平均而言留下多少能存活到繁殖的後代。一隻鳥多搶到一份食物，意義是「多撐過一個冬天、多下一窩蛋」，不是「多賺一塊錢」。所以這裡的「玩家在乎報酬」要重新理解成「報酬高的型，被天擇自動放大」——在乎的不是玩家，是這個篩選迴圈。

（混合策略的「族群比例」這層，本質上是把一個機率分布鋪在整個族群上；要深究機率分布本身的運算，見《馴服隨機》談機率分布與期望值那幾章，本書只用結果。）

## ESS：擋得住入侵的比例

納許均衡問的是：「哪個策略組合下，沒有人想**單方面**偏離？」演化賽局問一個表面像、骨子裡更嚴的問題：「哪種行為比例，普遍存在之後，**擋得住一小撮突變入侵**？」這就是演化穩定策略（evolutionarily stable strategy, ESS），由梅納德・史密斯（John Maynard Smith）與普萊斯（George Price）在 1973 年那篇《動物衝突的邏輯》（Nature 246:15–18）提出。

精確意義先講白話：**一個策略 S 是 ESS，意思是「整個族群都在玩 S 的時候，任何初始稀少的突變策略 T 都無法靠天擇壯大起來」。** S 是個演化的終點——滾到這裡就賴著不走了，因為任何想入侵的新花樣都會被篩掉。

把這個白話拆成可檢查的條件（這是「工程師的嚴謹」：每一步你都能口頭說出理由）。設 E(X, Y) 表示「玩 X 的個體，碰上玩 Y 的個體時拿到的適存度」。一個策略 S 要當 ESS，得滿足以下兩條之一成立（梅納德・史密斯的標準條件，見延伸閱讀）：

```text
ESS 的兩個條件（S 是現任、T 是任何想入侵的突變）：

  條件一（嚴格較優）：  E(S, S) > E(T, S)
     ← S 對 S 的表現，嚴格好過 T 對 S。突變一冒頭就吃虧，當場被篩掉。

  條件二（打平就比後手）：若 E(S, S) = E(T, S)，
                          則需 E(S, T) > E(T, T)
     ← 突變對現任打平、暫時混進來了；但只要現任「對突變」比
       「突變對突變」更強，等突變變多一點點，現任就反殺回去。
```

注意條件一長得跟「嚴格納許均衡」一模一樣（S 是對 S 自己的嚴格最佳反應）。**ESS 比納許均衡多的，是條件二那層保險。** 納許均衡只要求「沒人想單方面偏離」，它管不住「一票人同時偏離成同一個新策略、而且這個新策略碰到原策略時打平」這種情況——打平的突變在納許的框架下不會被擋，但在演化裡，只要它變多一點點、開始彼此相遇，現任能不能反殺它，就決定了族群會不會被入侵。**ESS = 嚴格納許均衡，或者「弱納許均衡 + 對入侵者的反殺優勢」。** 這層額外的挑剔，正是「演化動態的穩定」比「靜態的無人想偏離」多出來的東西。

這裡得誠實標一句嚴謹度：上面是 ESS 的**定義條件**，不是「ESS 等價於某個動態穩定點」的證明。ESS 與 replicator dynamics（下一節）的漸近穩定點之間的精確關係（ESS ⇒ 漸近穩定，反之不一定）是有定理的，本書給直覺、不展開，指向延伸閱讀。

## 鷹鴿賽局：為什麼動物不把對手往死裡打

ESS 的招牌例子是鷹鴿賽局（Hawk–Dove），梅納德・史密斯用它回答一個老生物學謎題：同種動物爭資源時，明明都有爪有牙，為什麼大多是虛張聲勢、很少打到一方重傷？「為了物種好」是錯的答案（天擇不為物種著想，只為基因的繁殖差異著想）。正確答案是一個 ESS。

設定：兩個個體爭一份價值 V 的資源。每個個體天生是「鷹」（Hawk，遇敵就拚到底）或「鴿」（Dove，擺姿態，對方真打就退）。打架受傷的成本是 C。報酬（適存度增益）矩陣如下，**這是 landscape 二的鷹鴿基準，本書取 V=2、C=4（C>V，自然界常態：受傷比資源貴）**：

```text
鷹鴿賽局（行＝對手；報酬＝我方期望適存度增益）。V=2、C=4：
                對手：鷹(H)          對手：鴿(D)
    我方：鷹(H)   (V−C)/2 = −1        V = 2
    我方：鴿(D)   0                   V/2 = 1

逐格白話：
  鷹遇鷹：拚到底，一半機率拿 V、一半機率受傷付 C，
          平均 (V−C)/2 = (2−4)/2 = −1（兩敗俱傷，期望是淨虧）
  鷹遇鴿：鴿讓步，鷹白拿整份資源 V = 2
  鴿遇鷹：鴿退，啥都沒拿、也沒受傷，得 0
  鴿遇鴿：擺姿態後平分，各 V/2 = 1
```

先看純策略：有沒有「全鷹」或「全鴿」的 ESS？

- **全鴿能被入侵嗎？** 能。一個鷹突變混進全鴿族群，它每場都碰到鴿、每場白拿 V=2，遠高於鴿遇鴿的 1。鷹的適存度爆高，比例暴漲。**全鴿不是 ESS**——太好欺負，鷹一定來。
- **全鷹能被入侵嗎？** 能。全鷹族群裡每場都是鷹遇鷹，平均 −1（互相砍到虧）。一個鴿突變混進來，它遇鷹就退、拿 0——0 雖然不多，但**比 −1 好**。鴿的相對適存度更高，比例上升。**全鷹也不是 ESS**——太愛拚，鴿子撿便宜。

兩個純策略都會被入侵。所以 ESS 一定是個**混合**：族群裡鷹和鴿以某個比例共存，共存到誰也入侵不了誰。這個比例是多少？

## Worked example：手算鷹鴿 ESS 比例

求那個「誰也入侵不了誰」的比例 p*（族群中鷹的比例）。用的工具和 ch05 解混合策略一模一樣——**無差異原理**：在穩定比例下，當鷹和當鴿的期望適存度必須相等。直覺：如果當鷹明顯划算，鷹比例就會繼續漲、不穩定；要穩定，兩種型的「日子過得一樣好」，誰也沒有比例上漲的壓力。

設族群中鷹佔比例 p、鴿佔 (1−p)。一個個體隨機碰到對手，有 p 機率碰到鷹、(1−p) 機率碰到鴿。

```text
一隻「鷹」在這個族群裡的期望適存度：
  E(鷹) = p · (鷹遇鷹) + (1−p) · (鷹遇鴿)
        = p · (−1)      + (1−p) · 2
        = −p + 2 − 2p
        = 2 − 3p                          … (1)

一隻「鴿」在這個族群裡的期望適存度：
  E(鴿) = p · (鴿遇鷹) + (1−p) · (鴿遇鴿)
        = p · 0         + (1−p) · 1
        = 1 − p                           … (2)

無差異：令 E(鷹) = E(鴿)
  2 − 3p = 1 − p
  2 − 1  = 3p − p          ← 把 p 移到一邊、常數移到另一邊
  1      = 2p
  p*     = 1/2                            ← ESS 鷹比例
```

得到 **p\* = 1/2**：穩定族群是一半鷹、一半鴿。對照公式 p* = V/C = 2/4 = 1/2，吻合。（這個公式只在 **C>V** 時成立——受傷比資源貴，本書設定 V=2、C=4 正是這種常態。一旦 V≥C，打架穩賺、鷹變成嚴格優勢策略，唯一 ESS 是全鷹 p\*=1；硬套 V/C 會算出 ≥1 的無效比例。）

**第二視角交叉驗算（最佳反應 / 偏離回拉）**，這是混合策略最容易算錯的地方，務必雙視角各驗一次：假設族群暫時偏到鷹比例 p = 0.6（鷹偏多）。代回 (1)(2)：

```text
p = 0.6 時：
  E(鷹) = 2 − 3(0.6) = 2 − 1.8 = 0.2
  E(鴿) = 1 − 0.6 = 0.4
  → E(鴿) > E(鷹)：鷹太多時當鴿划算 → 鴿比例上升 → 鷹比例被拉回 ↓

p = 0.4 時（鷹偏少）：
  E(鷹) = 2 − 3(0.4) = 2 − 1.2 = 0.8
  E(鴿) = 1 − 0.4 = 0.6
  → E(鷹) > E(鴿)：鷹太少時當鷹划算 → 鷹比例上升 → 拉回 ↑
```

兩邊都把比例往 0.5 拉——p\* = 1/2 是個**會自我修復**的穩定點，這正是 ESS 抗入侵的意思：任何偏離（包括突變帶來的小偏離）都會被天擇推回去。驗算通過。

這裡有一個 ch05 沒給你的新詮釋，停十秒想清楚：p* = 1/2 可以是「族群裡一半個體是純鷹、一半是純鴿」（**族群多型**，polymorphism），也可以是「每個個體都有一半機率表現成鷹、一半成鴿」（**個體混合**）。賽局理論的數學分不出這兩者——它只給比例。生物學上兩種都存在。這就是「混合策略 = 族群比例」這層詮釋的全部威力：ch05 那個抽象的 (1/2, 1/2)，在這裡有了血肉。

## replicator dynamics：往坡上爬的不是腦，是繁殖

上面我們直接「猜」了穩定比例再驗算。但族群是怎麼**滾**到那裡的？這就是 replicator dynamics（複製者動態）要講的事。完整方程本書不推（指向延伸閱讀，也指向《矩陣是動詞》談特徵值與穩定性那層），但它的直覺一句話就能抓住，而且你已經在 A/B test 裡用過：

> **適存度高於族群平均的型，比例上升；低於平均的，比例下降；上升或下降的「成長率」（相對速度），正比於它高出（或低於）平均多少。**（精確點說，比例變化的絕對速度還要乘上該型當前的佔比——佔比趨近 0 的型，就算相對成長率很高，絕對增量也很小。完整方程見延伸閱讀。）

這就是達爾文式選擇寫成微分方程。注意它的精妙：一個型不需要「絕對好」，只需要**比當前族群平均好**。這跟你調漸進放量的邏輯完全同構——新版本不需要完美，只要比現在線上的平均表現好，它的流量比例就會被往上推。

把這個直覺套回鷹鴿。在 p=0.6（鷹偏多）時，我們算過鴿的適存度 0.4 高於鷹的 0.2，也高於族群平均（介於兩者之間）；於是鴿比例上升、鷹比例下降，往 0.5 走。在 p=0.4 時反過來。**replicator dynamics 讓族群像球滾進碗底一樣，自己滾到 ESS。** 而碗底的位置（p\*=1/2），完全由報酬矩陣決定——沒有任何一隻鳥知道這個比例、沒有任何一隻鳥在「維持」它。

這一節是「演化動態軌跡」這種「視覺本身就是論點」的內容，將來會配一張 replicator 相圖（鷹鴿族群比例沿時間滾向 p\*=1/2 的軌跡）。在圖補上前，上面的數值表已經完整呈現了「偏離被拉回」的動態。

## 把脊椎放進族群：背叛者入侵合作者

現在做本章對脊椎的那一刀——**這是 ch01 那張囚徒困境矩陣**，但這次不是兩個會算的玩家對坐，而是一整個族群裡「合作者」和「背叛者」兩種行為型在繁殖競爭。脊椎被重看到第六層。

```text
脊椎囚徒困境（ch01 那張矩陣，數字＝獲益點數＝這裡的適存度增益）：
                     對方：合作(C)      對方：背叛(D)
    我方：合作(C)      (3, 3)            (0, 5)
    我方：背叛(D)      (5, 0)            (1, 1)
    T,R,P,S = 5,3,1,0；背叛＝嚴格優勢策略
```

問：合作（C）是 ESS 嗎？用條件一檢查。一個全合作族群（大家相遇都拿 R=3），冒出一個背叛突變 T：

```text
背叛突變在全合作族群裡：每場都碰到合作者，拿 T = 5
現任合作者碰到合作者：拿 R = 3
  → E(D, C) = 5  >  E(C, C) = 3
```

背叛突變的適存度（5）狠狠輾過現任合作者（3）。**E(D,C) > E(C,C)，連 ESS 條件一都過不了——合作不是 ESS，背叛者一冒頭就以更高的繁殖率擴張，把合作者吃掉。** 反過來，背叛（D）是 ESS 嗎？全背叛族群大家拿 P=1，一個合作突變進來碰到背叛者拿 S=0 < 1，立刻被篩掉。**背叛是 ESS。** 族群會滾向全背叛——那個 (1,1) 的雙輸結局。

停在這裡感受一下這一刀的重量。在 ch01，(背叛,背叛) 之所以發生，故事是「兩個聰明人各自算報酬、發現背叛是優勢策略」。**現在這個故事裡沒有聰明人、沒有算報酬、沒有人理解什麼是優勢策略——背叛照樣勝出。** 個體理性 vs 集體理性的裂縫（全書中央張力）在這裡換了一張臉：**不需要任何「個體理性」，光靠天擇的繁殖篩選，就足以把族群推進那個對所有成員都更糟的 (1,1)。** 集體最優 (3,3) 依然非均衡，依然救不回來——而這次連「自私」都不能怪，因為沒有任何主體在自私，只有比例在動。這是我認為整章最冷的一頁：**困境不需要壞人，甚至不需要人。**

那合作在演化裡就沒救了嗎？有，但要靠額外結構——和 ch07/ch08 救合作的工具同源，只是換成演化語言：

- **重複（未來陰影的演化版）**：如果個體會反覆相遇，「以牙還牙」這種行為型（ch08 的 tit-for-tat）可以抵抗背叛入侵——背叛者第一次占便宜，之後被報復，長期適存度未必高。Axelrod 1984《合作的演化》正是把錦標賽結果重讀成演化模擬。
- **親緣選擇（kin selection）**：對親屬合作 = 幫自己的基因傳下去。漢彌爾頓法則（Hamilton's rule，rb > c：親緣度 × 受益 > 代價時利他演化得出來）一句帶過，數學指向延伸閱讀。
- **聲譽與空間結構**：合作者群聚（只跟鄰居互動）時，合作者彼此相遇拿 R、被背叛者占便宜的機會變少，合作就有立足之地。

把這些對回 ch08：**ch08 用「δ 夠大、未來陰影夠長」讓會算的玩家選擇合作；ch09 用「重複/親緣/結構改變了繁殖篩選」讓不會算的族群演化出合作。** 同一件事——救合作——一個靠理性、一個靠選擇壓力。這正是本章存在的理由：它證明合作（和困境）都不是「理性」的專利。

## 直覺的陷阱

演化賽局是擬人化的重災區——它的詞彙（「策略」「最佳化」「想要」）全都借自會思考的玩家，一不小心就把「思考」也借進去了。守住下面這幾條正確版：

| 錯誤直覺 | 哪一步把你帶溝裡 | 正確版（自我察覺的徵兆） |
|---|---|---|
| **ESS 需要動物「理性地選混合策略」** | 看到「策略」「最佳化」「均衡」就腦補一個會算的主體 | ESS 裡沒有任何個體在算報酬（landscape T9）。「策略」是基因決定的行為型、「最佳化」由天擇代勞、「均衡」是擋得住突變的比例。**徵兆：你開始說「鳥決定當鷹」——鳥沒決定任何事，是帶鷹基因的鳥碰巧多留了後代。** |
| **ESS 就是納許均衡換個名字** | 看到「條件一長得像納許」就以為完全等價 | ESS 比納許嚴格：它多了條件二（對打平的入侵者要能反殺）。**每個 ESS 都是納許均衡，但有些納許均衡不是 ESS**（弱均衡可能被中性突變慢慢侵蝕）。徵兆：你說不出「條件二在防什麼」。 |
| **動物不把對手打死是「為了物種好」** | 把天擇想成有全局目標的設計師 | 天擇沒有「為物種好」這個目標，只有基因的繁殖差異。鷹鴿節制是 p\*=1/2 這個 ESS 的副產品，不是誰的善意。徵兆：你開始用「為了……好」解釋演化結果。 |
| **演化一定滾向「最好」的結果** | 把「適存度高的比例上升」誤讀成「族群會找到集體最優」 | 演化只滾向 ESS，而 ESS 可以爛透——脊椎困境演化出全背叛 (1,1)，集體最優 (3,3) 沒救。**選擇壓力放大的是「相對適存度高」，不是「集體好」**（中央張力的演化版）。徵兆：你預期演化會通往合作/和諧。 |
| **混合 ESS 表示「每隻動物都半鷹半鴿」** | 把族群層級的比例硬塞回個體 | p\*=1/2 可以是「一半個體純鷹、一半純鴿」（族群多型），也可以是「每個體 50% 機率當鷹」（個體混合）——數學分不出，兩者都存在。徵兆：你斷言只能是其中一種。 |

最值得反覆自我察覺的是第一條。整套演化賽局的詞彙都在偷渡「意圖」。你每次讀到「策略想要最大化適存度」，要在心裡翻譯成「帶這個行為型的個體碰巧留下較多後代」。能穩定做這個翻譯，你就真的懂了「均衡不需要會算的玩家」這個驚嘆點——而且能講給另一個工程師聽：**這就像 LRU 快取，它不『想要』留住熱門資料，它只是執行淘汰規則，結果熱門資料『碰巧』留下來了。沒有意圖，只有篩選。**

## 紙上推演

### 推演題一（★★，**[20 分鐘]**）換個成本，重算鷹鴿 ESS

把鷹鴿賽局的成本從 C=4 改成 **C=3**（價值仍 V=2，仍 C>V）。重畫報酬矩陣，用無差異法手算新的 ESS 鷹比例 p\*，並用「偏離回拉」交叉驗算一次。對照公式 p\*=V/C 檢查。

### 推演題二（★★，**[20 分鐘]**）把脊椎困境放進族群

用 ch01 脊椎矩陣（T,R,P,S=5,3,1,0），把報酬讀成適存度。一個 99% 合作者、1% 背叛者的族群，這一代過後，背叛者的比例會上升還是下降？用「背叛者的平均適存度 vs 合作者的平均適存度」論證，並說清為什麼這代表「合作不是 ESS」。

### 推演題三（★★★，**[15 分鐘]**）口頭題：均衡不需要會算的玩家

不准用「策略」「最佳化」這兩個詞，向一個只懂後端、沒學過賽局的同事解釋：鷹鴿賽局的 ESS（一半鷹一半鴿）是怎麼在「沒有任何動物在算」的情況下出現並維持的。要求他聽完能複述「為什麼這是個均衡卻沒有玩家」。

### 推演解答

**題一解答。** V=2、C=3，矩陣：

```text
                對手：鷹(H)          對手：鴿(D)
    我方：鷹(H)   (V−C)/2 = −0.5      V = 2
    我方：鴿(D)   0                   V/2 = 1

設鷹比例 p：
  E(鷹) = p·(−0.5) + (1−p)·2 = −0.5p + 2 − 2p = 2 − 2.5p
  E(鴿) = p·0 + (1−p)·1 = 1 − p
無差異：2 − 2.5p = 1 − p
        2 − 1 = 2.5p − p
        1 = 1.5p
        p* = 2/3
對照公式 p* = V/C = 2/3 ✓

交叉驗算（取 p = 0.8 > 2/3，鷹偏多）：
  E(鷹) = 2 − 2.5(0.8) = 2 − 2 = 0
  E(鴿) = 1 − 0.8 = 0.2  → 鴿較優 → 鷹比例下降，往 2/3 拉回 ✓
```

**ESS 鷹比例 = 2/3。** 常見錯路：(一) 把 (V−C)/2 算成 (2−3)/2 = −1（少除以 2，或把 V−C 看成 −2）——正確是 −0.5。(二) 用錯無差異對象，去令「鷹遇鷹 = 鴿遇鴿」而不是「E(鷹)=E(鴿)」——無差異是針對**兩種型在整個族群裡的期望適存度**，不是某兩格。(三) 忘了交叉驗算，把不穩定的解當 ESS。注意成本越低（C 從 4 降到 3），鷹比例越高（1/2 → 2/3）——打架越不痛，愛拚的型越吃香，直覺吻合。

**題二解答。** 族群 99% C、1% D。一個個體碰到的對手有 0.99 機率是 C、0.01 是 D。

```text
合作者的期望適存度：
  E(C) = 0.99·(C遇C=R=3) + 0.01·(C遇D=S=0)
       = 0.99·3 + 0.01·0 = 2.97

背叛者的期望適存度：
  E(D) = 0.99·(D遇C=T=5) + 0.01·(D遇D=P=1)
       = 0.99·5 + 0.01·1 = 4.95 + 0.01 = 4.96
```

E(D) = 4.96 遠高於 E(C) = 2.97，也高於族群平均（≈2.99）。**背叛者比例會上升**——它的相對適存度高，下一代佔比擴大。這正是「合作不是 ESS」的演化版本：在全（近全）合作族群裡，背叛突變 E(D,C)=5 > E(C,C)=3，連 ESS 條件一都過不了，背叛者必然入侵。常見錯路：(一) 以為 1% 太少、影響不到——比例多寡不改變「哪種型適存度高」這個方向，方向決定誰擴張。(二) 把「集體最優是 (3,3)」當成「演化會往那裡走」——演化放大的是相對適存度高的型（背叛），不是集體好的結局，這就是中央張力的演化臉。

**題三解答（示範講法）。** 「想像一個系統裡有兩種自動化規則在跑：一種叫『硬碰硬』，搶資源時絕不讓步；一種叫『軟退讓』，碰到搶就讓。每種規則控制的節點，跑得好（搶到資源、沒被搞壞）就會被複製出更多同款節點，跑得差就慢慢被回收——就像表現好的服務版本流量被放大、差的被回滾，沒有節點在『決定』什麼，是放量機制在篩。現在關鍵來了：如果全場都是『硬碰硬』，它們互相搶會兩敗俱傷、平均很虧，這時混進來幾個『軟退讓』反而活得好（至少不受傷），於是軟退讓被放大；可是一旦軟退讓變多、場上好欺負，『硬碰硬』又開始白吃白拿、被放大回來。兩股力量一推一拉，最後卡在一半一半——多一點硬就有利於軟、多一點軟就有利於硬，誰也擠不掉誰。這個『一半一半』就是均衡：它穩定（任何偏離都被拉回），但場上沒有任何節點知道這個比例、沒有任何節點在維持它，是篩選迴圈自己滾到這裡的。」要點：用篩選機制（放量/回收）替掉「思考」，講出「兩種型互為對方多寡的剋星」的拉鋸，並點明「穩定但無玩家」。
