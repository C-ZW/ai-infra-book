# ch16 — 公共財與共有財的悲劇：多人囚徒困境

> **本章解決什麼問題**：前面十五章，脊椎那張囚徒困境矩陣一直是「兩個人」的故事。但你系統裡那些「各自最佳化卻整體崩潰」的災難——retry storm、共用連線池被搶光、cache stampede——從來不是兩個玩家，是幾百個 client、幾十個服務同時在做同一件理性的蠢事。本章把脊椎困境從 2 人放大到 n 人：共有財悲劇（tragedy of the commons）與公共財賽局（public goods game）。你會看到一件比兩人困境更冷的事——**人數越多，困境不是被稀釋，而是被放大**：每個人對結果的影響越小，搭便車（free-riding）越誘人，集體崩潰越難擋。本章是 Part VI 的核心個案，把全書中央張力推到規模上的極限；它也呼應 ch13（機制設計能改報酬矩陣救困境）與 ch07/ch08（重複與聲譽能救合作）。最後我們看 Ostrom 怎麼用真實世界的證據，反駁「共有財必然崩潰」這個太悲觀的結論。

## 從你已知的出發

你一定處理過「共用資源被搶光」這類事故。最乾淨的版本是**共用資料庫連線池**：一個 pool 開了 100 條連線，前面掛著十幾個服務。平常相安無事。某天流量一尖峰，每個服務都覺得「我多開幾條連線、把我的 P99 壓下來」是理性的——對它自己的 SLA 來說，這確實是最佳選擇。於是十幾個服務同時這麼想、同時這麼做，連線池在幾秒內被抽乾，後面所有人的請求一起排隊、一起 timeout。**每個服務都做了對自己最划算的事，結果是所有人一起雪崩。**

這你眼熟——它就是脊椎囚徒困境（ch01 那張 5/3/1/0 的矩陣）。但有一件事，兩人版的困境沒教你：**這裡不是兩個玩家，是十幾個。** 而人數一多，事情的味道變了。

在兩人困境裡，你背叛的後果至少有一半落在你自己頭上（對方也會背叛、你掉進 (1,1)）。但在連線池這種多人場景裡，你多抽十條連線的壞處——池子更快耗盡——被攤平到所有十幾個服務身上，你自己只承擔其中一小份；而多抽十條連線的好處（你的 P99 變好）卻**全部歸你**。好處集中、壞處分散。這個不對稱，正是多人困境比兩人困境更惡毒的地方，也是本章要講透的東西。

你還會在別的地方撞到同一個結構：

- **cache stampede**：熱 key 過期的瞬間，每個 client 都覺得「我自己向後端重算一次」是理性的，結果幾千個 client 同時打到後端、把它壓垮。沒有人有動機「替大家先扛著、讓別人搭便車用我算好的結果」。
- **CI runner 排隊**：共用的 runner 池，每個人都想「我這個 job 重要、先插隊跑」。runner 是有限的共有財，各自搶占的結果是大家一起等更久。
- **on-call 與開源維護**：值班輪到的苦差事、開源專案沒人想接的 issue——這是反過來的「公共財」：維護是大家都受益的公共品，但**沒人想出力**，因為別人出力你也能白用。free-riding（搭便車）在這裡有個更熟悉的名字：「反正會有別人修」。

這一章就是要把這些你在生產環境裡反覆撞到的牆，接到一個精確的名字上——**多人囚徒困境**——並回答兩個問題：為什麼人越多越難合作？以及，既然如此，現實世界（和你的團隊）到底是怎麼把有些共有財救回來、沒有每次都崩的？

## 共有財悲劇：每個牧人都對，草原一起死

先請出這一章的歷史主角。1968 年 12 月，生態學家 Garrett Hardin（哈丁）在《Science》期刊發表了一篇後來被引爆數萬次的文章，叫〈The Tragedy of the Commons〉（共有財的悲劇）。他用一個牧場的寓言講清楚了一件事，而那件事，本質上就是把脊椎困境放大到很多人。

```text
共有財悲劇：Hardin 1968 的牧場寓言

  一片公有草原（the commons），向所有牧人開放。
  每個牧人都可以在上面多放一頭牛。

  你（一個理性的牧人）盤算「我要不要多養一頭牛」：
      好處：多一頭牛的全部收益 → 100% 歸你。            （+1 單位）
      壞處：草原多承受一頭牛的啃食、加速退化
            → 但這個壞處由「所有牧人」共同承擔，
              你只攤到其中一小份（1/n）。              （−1/n 單位）

  只要 n ≥ 2，+1 > 1/n，多養一頭牛永遠對你划算（人越多，1/n 越小、越划算）。
  → 每個理性牧人的結論都一樣：多養。
  → 每個人都多養 → 草原被啃到崩潰 → 所有人的牛都餓死。
```

請你盯著那個不對稱看十秒，因為它是整章的引擎：**多養一頭牛的好處全歸你，壞處攤給大家。** 在 Hardin 的話裡，每個牧人「被鎖進一套讓他不斷增加牛群的系統——在一個有限的世界裡」。每一步都是局部理性的，每個牧人都沒做錯任何計算；但這些局部理性加起來，是把所有人賴以為生的草原一起毀掉。

這跟脊椎困境是同一條裂縫，只是換了規模。ch01 那張矩陣裡，「個體理性 ⇒ 集體次優」發生在兩個人之間；共有財悲劇裡，它發生在一大群人之間，而且——這是關鍵——**人越多，裂縫越寬**。兩人困境裡背叛的代價你還承擔一半；n 人共有財裡，你過度使用的代價只攤到 1/n，n 越大、這份代價越接近零，過度使用就越是壓倒性的優勢策略。

我要在這裡先 hedge 一件事，免得這個寓言被你當成鐵律。Hardin 的框架後來被嚴厲批評過，批評不是說「悲劇不存在」，而是說他**搞錯了「commons」這個詞**。他寓言裡那片「向所有人開放、無人管理」的草原，學術上叫 **open access（開放取用）**——真正無主、無規則的資源；而歷史上的 commons（共有地）其實往往**有**一套社群規則（誰能放幾頭牛、誰來監督）。Hardin 拿一個無規則的極端，去暗示所有共有資源都注定崩潰，再推出「只有私有化或政府強管才能救」的結論——這一步是過度推論。本章後半要請出的 Ostrom，畢生的研究就是在打這個結論。但**作為一個賽局結構**，Hardin 的寓言是精準的：在無規則、無重複、無懲罰的純粹版本裡，過度使用確實是優勢策略，崩潰確實是均衡。我們先把這個純粹版本算清楚，再談怎麼救。

## 公共財賽局：把悲劇縮進一張可以手算的表

牧場寓言很有畫面，但「一頭牛攤多少草」不好量化。賽局理論把同一個結構提煉成一個乾淨、可以手算的實驗室版本——**公共財賽局（public goods game）**。它是把脊椎囚徒困境推廣到 n 人最標準的方式，我們花點力氣把它算到底。

```text
公共財賽局（n 人版）的規則

  n 個玩家，每人手上有一筆私有的錢（這裡每人 1 元）。
  
  每人同時、私下決定：把這 1 元「投入公共池」還是「自己留著」。
  
  公共池裡的總額會被「乘上一個倍數 m」（1 < m < n），
  然後「平分給所有 n 個人」——不管你有沒有投。
  
  關鍵設計：m > 1（投進去的錢會放大，所以全投對集體有利）
            m < n（你投的 1 元，自己只分回 m/n < 1，所以投錢個人虧）
  
  「自己留著」= 搭便車（free-riding）：享受別人投入的放大紅利，自己不出。
```

那個 `1 < m < n` 的雙重條件，是整個困境的數學核心，值得拆開講。`m > 1` 保證這是個**值得做的合作**：每投進去 1 元，社會總共多出 m 元，全投是集體最優。`m < n` 保證**個人不想投**：你投的 1 元被平分後，自己只拿回 m/n，而 m/n < 1，所以對你個人是淨虧。兩個條件一夾，就生出了困境：**對集體最好的事（全投），對每個個人都是賠本生意。** 這正是脊椎困境「(合作,合作) 集體最優、卻非均衡」在 n 人版的化身。

### worked example：3 人公共財賽局，逐一論證搭便車是優勢策略

把 n=3、m=2 代進去，這是本章的核心計算，我一步步算、不跳步。每人有 1 元，投進去的總額乘 2、三人平分。先確認這組數字滿足困境條件：`1 < 2 < 3` ✓——倍數 2 大於 1（全投有賺）、小於人數 3（個人投錢虧本）。

我用「最佳反應」的視角：**固定別人的選擇，問我自己投或不投哪個划算。** 賽局理論找優勢策略的標準動作，就是檢查「不管別人怎麼做，我這一步該怎麼選」。

```text
3 人公共財賽局：m=2，每人 1 元，池子×2 後三人平分
記號：我最後拿到的錢 = (自己留下的) + (從公共池分回的 1/3)

先算「我投 1 元」對我自己的淨影響：
  我那 1 元進池 → 乘 2 變 2 元 → 三人平分，我分回 2/3 元。
  我花了 1 元、分回 2/3 元 → 我個人淨虧 1/3 元。
  ← 注意：另外 2/3 + 2/3 = 4/3 元跑去便宜了另外兩個人！

所以不論「另外兩人」投了幾個，我多投那 1 元，對「我自己」永遠是 −1/3。
這就足以判定：不投（搭便車）是嚴格優勢策略。但我們把四種情況攤開驗：

  情況（用「另外兩人投了幾人」分類）：
  
  ┌──────────────┬────────────────┬────────────────┬──────────────┐
  │ 另外兩人     │ 我「不投」拿到 │ 我「投」拿到   │ 投 vs 不投   │
  │ 投入人數     │ (留1 + 分回)   │ (留0 + 分回)   │              │
  ├──────────────┼────────────────┼────────────────┼──────────────┤
  │ 0 人投       │ 1 + 0   = 1    │ 0 + 2/3 = 2/3  │ 不投多賺 1/3 │
  │ 1 人投       │ 1 + 2/3 = 5/3  │ 0 + 4/3 = 4/3  │ 不投多賺 1/3 │
  │ 2 人投       │ 1 + 4/3 = 7/3  │ 0 + 6/3 = 2    │ 不投多賺 1/3 │
  └──────────────┴────────────────┴────────────────┴──────────────┘

  每一列，「不投」都比「投」多賺正好 1/3 元。
  → 不管另外兩人怎麼選，不投永遠較好 → 不投是嚴格優勢策略。
```

三個人各自做完上面這張表，得到的結論一模一樣：不投。於是唯一的純策略納許均衡是**三人全都搭便車、公共池是空的、每人最後拿回 1 元**。這是 (背叛, 背叛, 背叛)——脊椎困境 (D,D) 的三人版。

現在看集體最優，也就是大家都不該錯過的那個結局：

```text
對照「全投」vs「全不投」的集體結果：

  全投（三人都投 1 元）：
     池子 = 3 元 → 乘 2 = 6 元 → 三人各分 2 元。
     每人最終：0（留）+ 2（分回）= 2 元。  ← 每人 2 元！

  全不投（納許均衡）：
     池子 = 0 → 每人最終：1（留）+ 0 = 1 元。  ← 每人 1 元

  全投每人 2 元 > 全不投每人 1 元。
  集體最優（全投）對每個人都嚴格更好，卻不是均衡。
```

把這兩塊拼起來，那個驚嘆點就無法迴避了：**全投讓每個人都拿 2 元，全不投讓每個人都只拿 1 元；明明全投對每一個人都更好，但每一個人的優勢策略卻都是不投。** 三個完全理性、各自最佳化的玩家，會穩穩地走進那個對三人都較差的結局——而且沒有人算錯。這跟 ch01 兩個囚犯掉進 (1,1) 是同一件事，只是現在有三個囚犯，而且你馬上會看到，囚犯越多越糟。

### 自我複核：用「社會總額」交叉驗算一次

我用第二個角度檢查上面沒算錯——算每種結局的**社會總財富**（三人加總），看它是否隨投入人數單調上升。

```text
複核（社會總額 = 三人最終拿到的錢相加）：

  投入人數 k → 池子 = k 元 → ×2 = 2k 元 → 三人分回共 2k 元
              留下的人共留 (3−k) 元
  社會總額 = (3−k) + 2k = 3 + k

  k=0（全不投）：總額 = 3   （每人 1）
  k=1：          總額 = 4
  k=2：          總額 = 5
  k=3（全投）：  總額 = 6   （每人 2）

  社會總額隨投入人數 k 單調上升（每多一人投，社會多賺 m−1 = 1 元）✓
  → 集體最優確實是「全投」（總額 6 最大）。
  
  但對「投錢的那個人自己」：他投 1 元、分回 2/3，個人虧 1/3——
  社會多賺的 1 元，加上我自己投的這 1 元，共 2 元被三人分掉；
  我只拿回 2/3，另外兩人共白拿了 4/3（每人 2/3）。
  個人帳與集體帳方向相反，這就是困境。 ✓
```

兩個視角對上了：最佳反應視角說「每個人不投多賺 1/3」，社會總額視角說「每多一人投、集體多賺 1 元」。同一個 1 元的去向——你投進去社會多 1 元（m−1=1），但這 1 元被三人分掉、你只拿回 2/3、淨虧 1/3——同時解釋了「為什麼集體想要你投」和「為什麼你自己不想投」。困境的本質，就是這兩本帳對不起來。

## 人越多越糟：困境如何隨 n 惡化

現在來回答本章開頭那個比兩人困境更冷的問題：**人數變多，困境是被稀釋還是被放大？** 直覺上你可能會想「人多力量大、總有人會合作吧」，但賽局結構說的恰恰相反。

關鍵在兩個隨 n 變動的量。固定那個放大倍數背後的「社會效率」不變（每投 1 元，社會多賺一個固定比例），但讓人數 n 變大，看搭便車的誘惑怎麼變。用一個稍微一般化的設定：每人投 1 元、池子乘 m、平分給 n 人，且維持 `m < n`（困境條件）。

```text
搭便車誘惑隨 n 變化（每人投 1 元、池子×m、n 人平分）：

  你投 1 元，自己分回 m/n。
  你個人的淨損失（搭便車相對於投入的好處）= 1 − m/n。

  把放大效率固定成「每人投入讓社會的每人份額增加一個比例」，
  看 n 變大時你個人的損失 1 − m/n：

  ── 數值感受（取 m 固定 = 2）──
     n=2： 1 − 2/2 = 0      ← m=n，剛好不算困境（投不投平手）
     n=3： 1 − 2/3 = 1/3    （本章 worked example）
     n=10：1 − 2/10 = 4/5
     n=100：1 − 2/100 = 0.98

  n 越大 → m/n 越小 → 你投錢分回的越少 → 搭便車的個人好處越大。
  你過度使用/不貢獻的「壞處」被攤到 n 個人身上，自己只擔 1/n →
  n→∞ 時你幾乎完全不承擔自己造成的損害。困境隨 n 單調惡化。
```

這就是 Hardin 牧場寓言裡「壞處攤給大家」那個不對稱的數學形狀：**你行為的代價被 1/n 稀釋，但好處全歸你。** n 越大，這個稀釋越徹底，搭便車越是壓倒性的選擇。兩人困境（n=2）其實是這條光譜上最溫和的一端——你還承擔自己背叛代價的一半；到了 n=100 的共用連線池，你多抽連線的代價只攤到你頭上 1/100，剩下 99/100 全潑給別人。

這帶出一個對你工程直覺很有用的推論，也是「直覺的陷阱」會再碰的一點：**為什麼大型開源專案、大公司的共用資源、全球性的環境問題特別難治理？** 不是因為參與的人比較壞，而是因為 n 特別大——每個人對結果的邊際影響小到趨近於零，搭便車的理性誘惑大到趨近於壓倒性。小團隊的共用資源（n=5 的小組共用測試環境）往往能靠自覺維持，同樣的資源放大到 n=500 的全公司就治理不動了——這個差別，光看 n 就能預測一半。

順帶把這條接回另一條全書線索：這也解釋了**為什麼「重複與聲譽」（ch07/ch08）在大群體裡效力會減弱**。兩人重複賽局裡，tit-for-tat 能直接懲罰背叛你的那一個人；但在 n=100 的共有財裡，你連「是誰在過度使用」都未必看得出來，更別說精準報復——監督與懲罰的成本隨 n 暴漲，這正是 Ostrom 的治理方案要解決的第一個問題。

## 救援：Ostrom 與「悲劇並非必然」

如果這一章到此為止，它會是一篇犬儒宣言：人越多越自私、共有財注定崩潰、只能靠私有化或老大哥強管。Hardin 1968 的結論差不多就是這樣。但這個結論是**錯的**——不是數學錯，是它漏看了現實世界一直在做的事。把這個漏洞補起來、並因此拿到 2009 年諾貝爾經濟學獎（嚴格說是瑞典央行紀念諾貝爾經濟學獎，見 ch13 的註明）的，是 Elinor Ostrom（歐斯壯）——**經濟學獎史上第一位女性得主**。

Ostrom 沒有坐在黑板前推翻 Hardin。她做的是去**數**——花了數十年，跑遍全世界蒐集真實的共有資源案例：瑞士阿爾卑斯山的牧場、日本的共有林、菲律賓的灌溉系統、緬因州的龍蝦漁場。她發現一件 Hardin 的寓言完全沒預料到的事：**有大量社群，靠自己長出的規則，幾百年來成功管理共有資源，既沒有私有化、也沒有政府強管，就是沒崩潰。** 她把這些成功案例的共通點，提煉成八條「設計原則」（design principles），寫進 1990 年的《Governing the Commons》（治理共有財）。

我不打算把八條原則全背一遍——對你來說，把它們讀成「賽局結構上各自在修什麼」更有用。它們其實是把前面幾章的救火工具，落地成可操作的社群制度：

```text
Ostrom 八原則 → 對應的賽局救火機制（挑幾條給工程直覺）

  ① 清楚的邊界（誰是這個共有財的成員、外人排除）
       → 把無限大的 n 圈成有限、已知的 n。
          「open access（誰都能用）」變成「有名單的 commons」，
          搭便車者跑不掉、可被辨識。（直接砍掉 n→∞ 的惡化）

  ④ 監督（monitoring，有人在看誰用了多少）
       → 解決大 n 的「看不出是誰在過度使用」問題。
          沒有監督，懲罰無從談起。

  ⑤ 分級懲罰（graduated sanctions，初犯輕罰、累犯加重）
       → 這就是 tit-for-tat 的精神（ch08）：可激怒但寬容。
          不是一犯就逐出（太狠、嚇跑人），而是逐步加碼。

  ③ 集體選擇（規則由使用者自己參與制定）
       → 規則是內生的，所以大家有遵守的動機（不是外人強加）。

  ⑥–⑧ 衝突解決機制、上級認可、巢狀治理
       → 把大群體拆成「巢狀」的小群體（小 n 容易合作），
          再層層疊起來——用結構規避大 n 的詛咒。
```

看出這在做什麼了嗎？Ostrom 的每一條原則，本質上都是在**把純粹版公共財賽局的某個「困境前提」拆掉**。純粹版假設：n 無限大且匿名（① 邊界把它變有限、可辨識）、單回合無未來（④⑤ 監督加懲罰把它變成有未來陰影的重複賽局，ch07）、無法溝通與協調（③ 集體選擇引入溝通）、無法分群（⑥–⑧ 巢狀化把大 n 拆成多個小 n）。**換句話說，現實世界救共有財的辦法，正是賽局理論早就指出的那幾把救火工具——重複、聲譽、可信的懲罰、機制設計——只是 Ostrom 證明了它們不必由中央政府從上而下安裝，社群能自己長出來。**

這跟你的團隊實踐是直接對得上的。你的 CI runner、共用測試環境、on-call 輪值——這些共有財之所以沒每天崩，靠的往往不是某個老大下的死命令，而是團隊自己長出的規範：「跑大 job 前先在頻道喊一聲」（① 邊界＋③ 集體規則）、「誰把環境搞壞了大家看得到」（④ 監督）、「連續佔用的人會被白眼、再犯會被請去喝咖啡」（⑤ 分級懲罰）。一個健康的工程團隊治理共有財的方式，幾乎是 Ostrom 八原則的逐條翻版——而它之所以在你的五人小組有效、放大到五百人公司就失靈，回頭看就是 n 變大、邊界與監督跟不上。

我要誠實標一下嚴謹度：Ostrom 的八原則是**從成功案例歸納出的經驗規律**（直覺版），不是一條能保證「照做必不崩潰」的定理。她自己也強調，沒有萬靈藥，制度要適應當地脈絡。但她做到了一件決定性的事：**把「共有財必然崩潰」從一條鐵律，降級成一個只在特定前提下成立的特例。** 悲劇是可能的、不是必然的——這跟 folk theorem 的口吻一模一樣（ch08）：合作可以是均衡，但要靠對的條件撐著。

## 中央張力：規模上的爆炸

把這一章放回全書脊椎來收一下。前面碰中央張力時，「個體理性 vs 集體理性」的裂縫一直在兩個人之間（脊椎困境）或一小群人之間（協調賽局 ch10、信任賽局 ch12）。本章做的，是把同一條裂縫**沿著 n 軸拉到極限**，然後給它一個量化的形狀。

```text
中央張力的規模爆炸：同一條裂縫，沿 n 軸放大

  兩人困境（ch01）        n 人共有財（ch16）
    個體理性 ──┐            個體理性 ──┐
               ↓ 分裂                  ↓ 分裂得更開
    集體理性 ──┘            集體理性 ──┘
  
  你背叛的代價            你過度使用的代價
  自己承擔 1/2            自己只承擔 1/n（n→∞ 趨近 0）
  
  好處集中、壞處分散：n 越大，壞處被攤得越薄，
  搭便車越是壓倒性的優勢策略 → 裂縫隨 n 單調變寬。
```

這是這一章最值得你帶走的一句話：**個體理性與集體理性的裂縫，不是固定寬度的——它隨著「多少人共用、每個人的影響多小」而張開。** 兩人困境是這條裂縫最窄的一端；全球氣候、全公司共用配額、全網路的壅塞控制，是它最寬的一端。理解了這個，你就懂了為什麼有些合作問題（小組共用測試機）幾乎不用治理就自己解決，有些（跨團隊搶共用資源）怎麼喊都喊不動——差的不是人品，是 n。

而救援的方向，全書到這裡已經湊齊了：要嘛把 n 變小、變有界（Ostrom 的邊界、巢狀化）；要嘛裝上未來的陰影（重複、聲譽，ch07/ch08）；要嘛直接改報酬矩陣（機制設計加懲罰，ch13）；要嘛指望人把集體理性內化進效用（公平偏好，ch12）。四把救火工具，本章用一個它們全都派得上用場的場景，把它們收攏成一張圖。

## 直覺的陷阱

n 人困境特別容易養出幾個似是而非的直覺，挑最常見的拆給你看。

| # | 錯誤直覺 | 它在哪一步把你帶溝裡 | 正確版 |
|---|---|---|---|
| 1 | 「人多力量大，總會有人合作把共有財撐住」 | 把「人多」想成合作的助力。其實 n 越大，每個人的影響越小、搭便車的個人誘惑越大——困境隨 n **惡化**不是緩解 | 你過度使用的代價攤到 1/n，n 越大攤得越薄。大群體比小群體**更**難自發合作，光看 n 就能預測一半 |
| 2 | 「共有財必然崩潰，只能靠私有化或政府強管」（Hardin 的結論） | 把一個「無規則、無重複、匿名」的極端純粹版（open access），當成所有共有資源的命運 | Ostrom 用幾百年的真實案例反駁：社群能自己長出規則（邊界、監督、分級懲罰），不靠私有化或中央強管也能避免悲劇。悲劇可能、非必然 |
| 3 | 「搭便車的人就是自私、缺德」 | 把結構問題歸因到人品。在純粹版裡，不投是**嚴格優勢策略**——理性、不缺德的人也會搭便車，因為投錢個人就是虧 | 搭便車是報酬結構逼出來的理性反應，不是道德缺陷。罵人沒用，要改的是結構（邊界/監督/懲罰/重複）——這正是 Ostrom 的路 |
| 4 | 「這是共有財悲劇」可以套在任何資源耗盡上 | 把「共有財悲劇」當成萬用標籤亂貼。它有**精確結構**：好處集中個人、壞處分散全體、`1<m<n` | 先驗證結構再貼標籤：是不是真的「我用的好處全歸我、代價攤給大家」？有沒有重複/監督/懲罰已經在運作（那它可能根本不在純粹版裡）？ |
| 5 | 「公共財賽局和共有財悲劇是兩回事」 | 被「貢獻 vs 過度取用」的表面差異騙了 | 兩者是同一枚硬幣：公共財是「該貢獻卻搭便車」（不投），共有財是「該節制卻過度取用」（多放牛）。數學同構——好處集中、壞處分散、優勢策略導致集體崩潰 |

**自我察覺的徵兆**：如果你發現自己在用「大家自覺一點就好了」來「解決」一個共有財問題，那通常是個訊號——你在跟報酬結構對抗，而結構會贏。一個搭便車是優勢策略的局，光靠喊話和訴諸自覺撐不住（n 越大越撐不住）。停下來問：**我能不能把邊界圈出來（誰在用）、把使用變得可見（監督）、給重複互動裝上未來的陰影（再犯有後果）？** 只要你發現自己想靠「人應該更無私」來解困境，多半是漏看了「結構才是優勢策略的來源」這件事。

## 紙上推演

### 推演題一（★★，20 分鐘）：把 2 人困境推廣到 n 人，看困境如何隨 n 惡化

脊椎囚徒困境是兩人的。本題請你把它推廣到 n 人公共財賽局，並量化「人越多越糟」。

設定：n 個玩家，每人有 1 元，可選擇投入公共池或留著；池子裡的總額乘 m=2 後平分給全部 n 人（維持 `m < n`，即 n ≥ 3）。

(a) 寫出「我投 1 元」對我自己的淨損益（用 n 表示），證明只要 n > 2，不投就是嚴格優勢策略。
(b) 算 n=3、n=10、n=100 三種情況下，「搭便車相對於投入」對我個人多賺多少，說明困境如何隨 n 惡化。
(c) 一句話說明：n=2 時這個賽局發生什麼特殊情況？

### 推演解答

```text
(a) 我投 1 元的淨損益（m=2、n 人平分）：
       我那 1 元 → ×2 = 2 元 → n 人平分，我分回 2/n。
       我花 1 元、分回 2/n → 個人淨損益 = 2/n − 1。
       n > 2 時，2/n < 1 → 2/n − 1 < 0 → 投錢個人淨虧。
       
    這個淨虧「不論別人投幾個」都成立（別人的投入分回的部分，
    我投或不投都照拿，不影響我這 1 元的邊際帳）。
    → 不投嚴格優於投，與別人選擇無關 → 不投是嚴格優勢策略。 ✓

(b) 搭便車相對投入，我個人多賺 = 1 − 2/n：
       n=3：  1 − 2/3  = 1/3   ≈ 0.33
       n=10： 1 − 2/10 = 4/5   = 0.80
       n=100：1 − 2/100        = 0.98
    
    n 越大 → 分回的 2/n 越小 → 投錢虧得越多 → 搭便車誘惑越大。
    白話：你過度使用的代價攤到 n 個人頭上、自己只擔 1/n，
    n 越大越接近「自己完全不承擔」→ 困境單調惡化。 ✓

(c) n=2 時：2/n = 2/2 = 1 → 投錢淨損益 = 1−1 = 0。
    投與不投打平、m=n 不再滿足「m<n」的困境條件 →
    退化成「投不投無所謂」的邊界情形（嚴格說已不是困境）。
    但注意：退化的是「m=2 這個選擇」，不是「兩人公共財」本身——
    只要把倍數壓到 m<2（例如 m=1.5，仍滿足 1<m<n=2），
    n=2 就重新是真困境（投 1 元分回 0.75、個人淨虧 0.25）。
    換句話說，本題固定 m=2 才讓 n=2 剛好踩在邊界上；
    n≥3 時 m=2 一定是真困境，n=2 則要 m<2 才是。 ✓
```

**解讀**：這題把「人越多越糟」從一句口號變成一條公式 `1 − m/n`：搭便車的個人好處隨 n 上升、趨近 1（趨近「投錢純粹是送錢給別人」）。這就是 Hardin 牧場「壞處攤給大家」的數學形狀。

**常見錯路**：① 把「我分回的」算成 2（池子全額）而非 2/n（平分後我那份）——忘了平分。② (c) 算出 n=2 打平就以為「兩人不會有困境」——錯，是這個**對稱公共財參數化**在 n=2 退化；真正的兩人囚徒困境（5/3/1/0）用的是不同報酬、背叛仍嚴格優勢。兩者別混為一談。③ 以為「不投虧 2/n−1」要看別人投幾個——不會，別人投入分回的部分你照拿、與你這 1 元的邊際決定無關。

### 推演題二（★★，20 分鐘）：你系統裡的共有財悲劇，與一個 Ostrom 式治理

不准用記憶背定義，要用本章的結構自己組裝。

(a) 從你的工程經驗裡挑一個**真正符合共有財結構**的場景（提示：好處集中在用的人身上、代價分散給所有人、且 `1<m<n` 式的「對集體值得、對個人不划算」）。明確指出：誰是玩家、什麼是共有財、什麼是「過度使用/搭便車」、好處與代價分別落在誰頭上。
(b) 套用 Ostrom 至少**三條**原則，設計一個能緩解它的治理方案。每條原則要對應到它拆掉了純粹版的哪個「困境前提」。
(c) 說明為什麼「叫大家自覺一點」在這個場景裡撐不住（用 n 與優勢策略講）。

### 推演解答

這題沒有唯一答案，給一個合格範本（以共用資料庫連線池為例）對照：

```text
(a) 結構辨識：共用 DB 連線池（pool 100 條，前面 12 個服務）
      玩家：12 個服務（每個各自最佳化自己的 SLA）。
      共有財：100 條連線（有限、共用）。
      搭便車/過度使用：尖峰時多抽連線壓自己的 P99。
      好處集中：我多抽連線 → 我的 P99 變好（全歸我）。
      代價分散：池子更快被抽乾 → 所有 12 個服務一起排隊
                （代價攤給大家，我只擔 1/12）。
      → 好處集中、代價分散、個體理性 ⇒ 集體雪崩。是共有財悲劇。 ✓

(b) Ostrom 式治理（三條原則，各拆一個困境前提）：
      ① 清楚邊界 → 給每個服務分配連線配額（max connections per
         service）。把「open access 隨便抽」變成「有名單、有上限」。
         拆掉的前提：匿名、無界的過度使用。
      ④ 監督 → 連線池用量 dashboard，誰抽了多少全隊可見。
         拆掉的前提：「看不出是誰在過度使用」。
      ⑤ 分級懲罰 → 超配額先告警、持續超標自動限流(throttle)、
         再犯則熔斷該服務的低優先請求。初犯輕、累犯重。
         拆掉的前提：背叛無後果（裝上未來的陰影，呼應 ch07/ch08）。

(c) 為什麼「叫大家自覺」撐不住：
      尖峰時，對「每一個服務自己」而言，多抽連線壓 P99 永遠是
      最佳反應（好處全歸自己、代價只擔 1/12）——不投/不節制是
      優勢策略。n=12 已經夠大到讓「自覺」對抗不了優勢策略；
      光靠喊話，第一個破戒的服務就會把規範帶崩（它 P99 變好、
      沒受懲罰，其他服務理性跟進）。要改的是結構（配額+監督+
      限流），不是訴諸無私。 ✓
```

**解讀**：這題的重點是「先驗證結構、再開治理藥方」。注意 (b) 的每條原則都不是憑感覺加的——它精準對應純粹版的一個困境前提（無界、不可見、無後果），把那個前提拆掉，困境就被削弱一分。

**常見錯路**：① 挑了一個其實**不符合**共有財結構的場景（比如「大家都不寫測試」——這比較像公共財貢獻問題，其實也算，但若舉「兩個服務搶一個鎖」就只是兩人賽局、n 太小不算規模困境）。② 治理方案全靠「教育大家、提高意識」——這正是陷阱 #3、#5，與優勢策略對抗。③ 套 Ostrom 原則卻說不出每條拆掉哪個前提，變成貼標籤。

### 推演題三（★★，15 分鐘）：口頭題——為什麼「每個人都理性使用」反而耗盡共享資源

不准用紙筆。對一個會說「資源是大家的，理性的人應該會省著用吧」的同事，用 90 秒講清楚三件事：①為什麼在純粹版共有財裡，「過度使用」反而是每個理性個人的優勢策略；②這跟「人自私缺德」有沒有關係；③人越多，這件事為什麼越嚴重。

### 推演解答

合格的口頭回答長這樣（自己講完對照）：

> 「關鍵是好處和代價落在不同人頭上。你多用一單位共享資源——多抽一條連線、多放一頭牛——好處**全歸你自己**，但代價（資源更快耗盡）是**全體一起攤**，你只擔其中 1/n。所以對你個人算這筆帳，多用永遠划算，不管別人怎麼做——這就是優勢策略，理性的人都會這麼選。注意這跟自私缺德沒關係：一個完全理性、不缺德的人，做這道算術也會得出『多用』，因為節制是賠本生意——他省下的資源主要便宜了別人。所以罵人沒用，要改的是結構。而人越多越糟：n 越大，你造成的代價被攤得越薄、自己擔的越接近零，多用的誘惑就越壓倒性——這就是為什麼小組共用測試機靠自覺還撐得住，全公司共用配額怎麼喊都喊不動。差的不是人品，是 n。」

**給分點**：有沒有講出「好處集中、代價分散 1/n」這個機制（①的核心）；有沒有明確把它跟「自私缺德」切開、指出理性不缺德的人也會這麼做（②）；有沒有講出 n 越大裂縫越寬、並舉出規模差異的例子（③）。只會說「大家自私所以資源耗盡」的，沒及格——那把結構問題誤讀成人品問題，正是本章要打掉的陷阱。

## 自我檢核

口頭自答，講得出來才算過關。優先答「為什麼」。

1. 共有財悲劇和脊椎囚徒困境（ch01）是同一條中央張力嗎？它把「個體理性 ⇒ 集體次優」從幾人放大到幾人？放大時哪個量在變？
2. 在 3 人公共財賽局（m=2、每人 1 元）裡，為什麼「不投」是嚴格優勢策略？用「我投 1 元、自己只分回 2/3」這筆邊際帳講給自己聽。
3. 為什麼全投（每人 2 元）對每個人都比全不投（每人 1 元）好，卻不是納許均衡？這跟 (合作,合作) 在 ch01 的處境一樣嗎？
4. 「人越多，共有財困境越嚴重」——這個惡化的數學形狀是什麼（用 1 − m/n 講）？為什麼大群體比小群體更難自發合作？
5. Hardin 1968 的結論（共有財必然崩潰、只能私有化或政府強管）漏看了什麼？Ostrom 用什麼證據反駁它？她憑這拿了哪一年的什麼獎、有何特別之處？
6. Ostrom 八原則裡的「邊界」「監督」「分級懲罰」，各自拆掉了純粹版公共財賽局的哪一個困境前提？這跟前面章節（重複、聲譽、機制設計）的救火工具是同一套東西嗎？
7. 為什麼說「搭便車的人自私缺德」這個歸因是錯的、而且有害？正確的歸因會讓你採取什麼不同的行動？
8. 公共財賽局（該貢獻卻不投）和共有財悲劇（該節制卻過度取用）為什麼說是「同一枚硬幣」？它們共用哪個報酬結構？

## 延伸閱讀

- **Garrett Hardin (1968), "The Tragedy of the Commons"**（*Science*, 162: 1243–1248）——點燃整個領域的那篇原文。讀它的牧場寓言段落感受結構之精準，但讀的時候帶著本章的 hedge：他的「commons」其實是無規則的 open access，他「只能私有化或強管」的結論是過度推論。網路上有公開 PDF。
- **Elinor Ostrom (1990), *Governing the Commons: The Evolution of Institutions for Collective Action***——反駁 Hardin 悲觀論的經典。不必整本啃，讀她歸納的「八條設計原則」那章，把每一條讀成「它在拆公共財賽局的哪個困境前提」。這本讓她拿了 2009 年諾貝爾經濟學獎（經濟學獎首位女性得主）。
- **John Ledyard (1995), "Public Goods: A Survey of Experimental Research"**（收於 *Handbook of Experimental Economics*）——公共財賽局實驗研究的標準回顧。讀它五條核心發現（一次性比理論預測更多貢獻、重複後搭便車增加、溝通促進合作、門檻促進合作、MPCR 越高合作越多）——這些正是 Ostrom 八原則在實驗室裡的數量化版。
- **Mancur Olson (1965), *The Logic of Collective Action***——「為什麼大團體比小團體更難為共同利益行動」的奠基論述，跟本章「困境隨 n 惡化」直接呼應。想從政治經濟學角度看同一件事，讀它的前兩章。
- 跨書：n 人共有財裡「過度使用的代價如何被機率性地分攤、社會總額怎麼算期望」的機率機制，見《馴服隨機》ch08 期望值那章；公共財貢獻決策中「一個人面對不確定該貢獻多少」的單人理性面，見《在不確定中下注》談期望效用那本。本章只負責把困境放大到 n 人、並指出救援方向，不深入這兩塊。
