## 從你已知的出發

你一定踩過**腦裂（split-brain）**。

一個五節點的叢集，原本有一個 leader 在發號施令、其他四個是 follower。某天網路抖動，把叢集切成兩半：三個節點在這邊、兩個節點在那邊。兩邊各自看不到對方，於是各自開了一輪選舉，各自選出一個 leader，各自開始接受寫入。等網路恢復，你面對的是兩份互相矛盾的資料、兩個都覺得自己是合法 leader 的節點，以及一份要花你整個週末去對帳的損毀紀錄。

現在我問你一個問題：**這場災難裡，「誰當 leader」這件事重要嗎？**

答案是——一點都不重要。節點 A 當 leader、節點 B 當 leader、節點 C 當 leader，對系統的正確性來說完全等價，三個都能把活幹得一樣好。叢集根本不在乎是誰坐上那個位子。它**唯一**在乎的，是所有節點**對「是同一個」這件事達成共識**。腦裂不是因為選錯了人，是因為**沒選成同一個人**。

這就是協調賽局的全部精神，而且你天天在跟它打交道：

- **leader election**：選誰當 leader 不重要，選不一致就腦裂。Raft、ZooKeeper 之類的協定，本質上整套機器都是在解一個協調賽局——而且為了避免「兩個都剛好賭自己當 leader」的對稱僵局，它們刻意引入隨機化選舉超時（randomized election timeout），讓某個節點先醒來、打破對稱。這跟 ch05 的 backoff+jitter 是同一招：用隨機化把對稱的協調僵局敲碎。
- **分散式共識**：Paxos／Raft 要全體 follower 收斂到同一個值。哪個值被選上，多數情況下系統不在乎；它在乎的是大家收斂到「同一個」。
- **標準與協定的採用**：為什麼整個業界用同一套 HTTP 狀態碼、同一個 timestamp 格式、REST 還是 gRPC 的某個慣例？很多時候不是因為那個選項「最優」，是因為它**顯著**——大家都猜「別人也會猜大家都用它」，於是它就成了大家都用的那個。QWERTY 鍵盤一點都不最優，但你現在打字用的就是它。
- **schema 版本協調**：上下游服務要同時切到 v2，誰先切誰後切不重要，**切的時間點不一致**才會在中間那段時間炸出一堆反序列化失敗。

注意這些例子有一個共同的、跟囚徒困境**截然不同**的味道。囚徒困境裡，雙方的利益是**對立**的（我背叛你能多撈）。協調賽局裡，雙方的利益是**一致**的——大家都想協調成功，沒有人想腦裂。但「利益一致」居然**不保證**結果好。這就是本章要逼出來的那一刀：**個體理性可以不夠用，不是因為大家想互相傷害，而是因為大家想合作卻賭錯了方向。**

## 純協調賽局：兩個均衡，一樣好，但你得選對同一個

先看最乾淨的版本。兩個服務 A 和 B 要約好用同一個序列化格式，選 JSON 或選 Protobuf。對它們來說，**選哪個都行，只要選一樣**。選一樣，雙方都能正常通訊，各拿報酬 1；選不一樣，誰都解析不了對方，各拿 0。

```text
純協調賽局（格內 = (我方報酬, 對方報酬)；越大越好）：

                   對方：JSON       對方：Protobuf
    我方：JSON       (1, 1)  ← 對上了  (0, 0)  ← 雞同鴨講
    我方：Protobuf   (0, 0)  ← 雞同鴨講 (1, 1)  ← 對上了
```

用 ch02 那套**最佳反應分析**掃一遍：

- 如果對方出 JSON，我出 JSON 拿 1、出 Protobuf 拿 0 → 我的最佳反應是 JSON。
- 如果對方出 Protobuf，我出 JSON 拿 0、出 Protobuf 拿 1 → 我的最佳反應是 Protobuf。

兩條對角線 (JSON, JSON) 和 (Protobuf, Protobuf) 都是雙方互為最佳反應——**兩個純策略納許均衡**。而且它們**一樣好**，都給 (1,1)。沒有任何一個在報酬上比另一個優。

這就是協調賽局跟囚徒困境的結構差異，值得你停下來盯一眼：

- 囚徒困境（ch01 那張 5/3/1/0）：唯一純策略 NE 是 (背叛,背叛)，而且它**雙輸**——均衡存在但很糟。
- 純協調賽局：有**兩個**純策略 NE，而且它們都**很好**——問題不在均衡糟，問題在均衡**太多、沒有一個天生勝出**。

純報酬矩陣到此就**沉默**了。它能告訴你「(JSON, JSON) 和 (Protobuf, Protobuf) 都是均衡」，但它**打死不肯告訴你該選哪一個**。這不是我們算得不夠仔細，是矩陣裡的資訊本來就不足以決定。對稱的賽局有對稱的均衡，數學沒有偏好。

那玩家真的坐進這個賽局，會怎樣？兩種失敗：

1. **賭錯方向**：A 賭大家會用 JSON、B 賭大家會用 Protobuf，結果落到 (JSON, Protobuf)，雙方各拿 0。**兩個都理性、都想協調，卻一起落空。** 這就是腦裂的數學形狀。
2. **不敢動**：A 想「萬一我選 JSON 但 B 選了 Protobuf 怎麼辦」，於是僵在那不敢先承諾；B 也這麼想。兩邊都等對方先動，誰也不動。

你看出來了嗎——**這跟脊椎那張囚徒困境是同源的**。困境是「每個人都最佳化、結果集體更糟」；協調失敗是「每個人都最佳化、結果集體錯位」。兩者都是同一條裂縫的不同臉孔：**個體理性不保證集體達成它本來想要的東西**。差別只在，困境裡集體理性要的是「別一起跳坑」，協調裡集體理性要的是「往同一個方向跳」。困境裡個體的誘惑是背叛（私利對立）；協調裡沒有對立的私利，純粹是「我猜不到你會猜什麼」這個資訊缺口，就足以讓兩個一心想合作的玩家撞牆。

> **這裡反直覺在哪，請你能轉述**：你以為「大家利益一致」就萬事大吉——畢竟沒人想害誰。但協調賽局告訴你，**利益一致是不夠的**。只要存在多個一樣好的均衡，而玩家又不能事先溝通，理性本身就選不出來。合作失敗的原因不是「有人壞」，而是「沒有共同的猜測焦點」。腦裂不是因為哪個節點居心不良，是因為它們缺一個「大家都知道大家都會選它」的點。

## 那個混合均衡：協調賽局裡最該被警告的東西

純協調賽局除了兩個純策略 NE，還有第三個均衡——一個**混合策略納許均衡**，而它正是「協調的恐怖之處」最赤裸的數學表達。

用 ch05 的**無差異原則**來找。設對方出 JSON 的機率是 q。我要混合，前提是我對 JSON 和 Protobuf 無差異：

```text
我出 JSON   的期望報酬 = 1·q + 0·(1−q) = q
我出 Protobuf 的期望報酬 = 0·q + 1·(1−q) = 1−q

令兩者相等（無差異）：   q = 1 − q   →   q = 1/2          ← 對方各 1/2

對稱地，我也是各 1/2。所以混合 NE = 雙方都「丟硬幣決定」。
```

現在算這個混合均衡的**期望報酬**，這才是重點。我各 1/2、對方各 1/2，四格等機率各 1/4：

```text
落點機率與報酬：
   (JSON, JSON)         1/4 → 我拿 1
   (JSON, Protobuf)     1/4 → 我拿 0
   (Protobuf, JSON)     1/4 → 我拿 0
   (Protobuf, Protobuf) 1/4 → 我拿 1

我的期望報酬 = 1/4·1 + 1/4·0 + 1/4·0 + 1/4·1 = 1/2
```

停在這個數字上。在兩個純策略均衡裡，你穩穩拿 **1**。在這個混合均衡裡，你只拿 **1/2**——因為有一半的時間你們在雞同鴨講。**這是一個雙方都在隨機亂猜的均衡，沒有人能單方面改善（只要對方在亂猜，你出什麼都一樣是 1/2），所以它是個如假包換的納許均衡——而且它是最差的那個。**

這個混合均衡就是腦裂的穩態描述：兩個節點都不知道對方會選誰，於是各自「賭」，一半時間賭一致、一半時間賭岔開。系統卡在一個「大家都在猜、誰也猜不準」的均衡裡，而它的期望表現比任何一個「大家都選同一個」都差。

這給了你一個冷酷的洞見：**協調賽局裡，真正的敵人不是某個壞均衡，而是『沒有人帶頭收斂』這件事本身。** 只要還停在「各自猜」的狀態，理性玩家就可能永遠停在那個 1/2 的爛均衡裡，因為沒有任何一方有單方面跳出去的誘因——你先跳到 JSON，對方還在亂猜，你照樣只有一半機率對上。要逃出去，需要某種矩陣**外**的東西把大家的猜測**對齊**。那個東西，就是謝林點。

## 性別戰：協調，但摻了一點私心

純協調賽局有點太乾淨了——兩個均衡完全等價，現實裡很少這麼好心。更常見的是：大家**都想協調成功**（沒人要腦裂），但**對「協調到哪個」各有偏好**。協調與衝突摻在一起。這就是賽局理論裡有個老掉牙名字的經典——**性別戰（battle of the sexes）**。

故事的傳統版本：一對伴侶想一起出門（分開過夜誰都不開心），但一個偏好看歌劇、一個偏好看球賽。一起去歌劇，兩人都還行、但偏好歌劇的那個更爽；一起去球賽，反過來；各去各的，兩人都最慘。我把它換成你熟的場景：兩個團隊都同意「我們得統一用同一套監控堆疊」（分裂成兩套誰都受不了），但 A 團隊已經熟 Prometheus、B 團隊已經押注在另一套上。**統一比分裂好，但統一到自己熟的那套更好。**

```text
性別戰（格內 = (我方報酬, 對方報酬)；越大越好）：
我方偏好 Opera（歌劇）、對方偏好 Football（球賽）。

                   對方：Opera      對方：Football
    我方：Opera      (2, 1)  ← 都來我場 (0, 0)  ← 各去各的
    我方：Football   (0, 0)  ← 各去各的 (1, 2)  ← 都來他場
```

讀一下這張表的味道，它跟純協調差在哪：對角線（協調成功）仍是兩個均衡，但這次它們**不等價**了——(Opera, Opera) 我拿 2 你拿 1，(Football, Football) 我拿 1 你拿 2。**我偏好前者、你偏好後者。** 協調是共同利益，但「協調到哪」是利益衝突。這比純協調更貼近真實談判：沒人想談崩，但每個人都想談到對自己更有利的那個版本。

這張矩陣有**三**個納許均衡，下一節的 worked example 會把它們一個一個算出來。先記住它教你的事：當協調摻進了私心，玩家不只要解決「怎麼選同一個」，還要面對「憑什麼是你那個、不是我那個」的拉鋸——而誰都不肯先讓步時，那個雙方亂猜的混合均衡（最差的結果）就在門口等著。

## Worked example：把性別戰的三個納許均衡全找出來

我要把這張矩陣的均衡**全部**手算出來，包括那個最容易被漏掉、也最該被警告的混合均衡。每一步都不跳，最後用兩個視角交叉複核。

```text
性別戰矩陣（重抄一遍，方便對照）：
                   對方：Opera      對方：Football
    我方：Opera      (2, 1)           (0, 0)
    我方：Football   (0, 0)           (1, 2)
```

**第一步：用最佳反應找純策略 NE。** 逐欄看「對方固定時，我該怎麼回」，再逐列看「我固定時，對方該怎麼回」。

```text
我的最佳反應（對方固定）：
  對方 Opera   → 我 Opera 拿 2、Football 拿 0 → 我選 Opera ✓
  對方 Football→ 我 Opera 拿 0、Football 拿 1 → 我選 Football ✓

對方的最佳反應（我固定）：對方是行玩家、報酬是每格第二個數
  我 Opera     → 對方 Opera 拿 1、Football 拿 0 → 對方選 Opera ✓
  我 Football  → 對方 Opera 拿 0、Football 拿 2 → 對方選 Football ✓

雙方互為最佳反應的格子（兩個 ✓ 重疊）：
  (Opera, Opera)        ← 純策略 NE 1
  (Football, Football)  ← 純策略 NE 2
```

兩個純策略 NE。在 (Opera, Opera) 我拿 2、你拿 1；在 (Football, Football) 我拿 1、你拿 2。**對稱地不公平——每個人都偏好「協調到自己場」的那個均衡。**

**第二步：用無差異原則找混合 NE。** 設**對方**出 Opera 的機率為 q（出 Football 為 1−q）。我願意混合的唯一條件，是我對 Opera 和 Football **無差異**（否則我會純出報酬高的那個，根本不混）：

```text
我出 Opera    的期望報酬 = 2·q + 0·(1−q) = 2q
我出 Football 的期望報酬 = 0·q + 1·(1−q) = 1 − q

令兩者相等：   2q = 1 − q
              3q = 1
              q  = 1/3        ← 對方出 Opera 的機率 = 1/3、Football = 2/3
```

注意這個反直覺點（ch05 已經敲過一次，這裡再敲）：**我的混合機率，是由「讓對方無差異」決定的；對方的混合機率，是由「讓我無差異」決定的。** 所以要解我的機率，得從對方的報酬下手。設**我**出 Opera 的機率為 p，讓**對方**對他的兩個選項無差異（對方是行玩家，看每格第二個數）：

```text
對方出 Opera    的期望報酬 = 1·p + 0·(1−p) = p
對方出 Football 的期望報酬 = 0·p + 2·(1−p) = 2(1−p)

令兩者相等：   p = 2(1−p)
              p = 2 − 2p
              3p = 2
              p  = 2/3        ← 我出 Opera 的機率 = 2/3、Football = 1/3
```

**第三個 NE 是：我（偏好歌劇）以 2/3 機率出 Opera、對方（偏好球賽）以 2/3 機率出 Football。** 各自更常選自己偏好的那個，但都留了 1/3 的機率讓步。

**第三步：算這個混合均衡的期望報酬，看它有多爛。** 我出 Opera 機率 p=2/3、對方出 Opera 機率 q=1/3。四格的機率：

```text
落點機率（我的 Opera 機率 2/3、對方的 Opera 機率 1/3）：
  (Opera,    Opera)    = 2/3 · 1/3 = 2/9 → 我拿 2
  (Opera,    Football) = 2/3 · 2/3 = 4/9 → 我拿 0
  (Football, Opera)    = 1/3 · 1/3 = 1/9 → 我拿 0
  (Football, Football) = 1/3 · 2/3 = 2/9 → 我拿 1

機率自我複核：2/9 + 4/9 + 1/9 + 2/9 = 9/9 = 1 ✓

我的期望報酬 = 2/9·2 + 4/9·0 + 1/9·0 + 2/9·1
            = 4/9 + 0 + 0 + 2/9
            = 6/9 = 2/3
```

**用第二步的無差異視角交叉複核**：既然我出 Opera 和出 Football 在均衡裡無差異，我的期望報酬應該等於「我出 Opera 的期望」=2q=2·(1/3)=2/3。✓ 兩個視角對上了。

把三個均衡並排，這張表就是本章的縮影：

```text
性別戰的三個納許均衡（我的期望報酬）：
  (Opera, Opera)       純策略 → 我拿 2   （我的最愛）
  (Football, Football) 純策略 → 我拿 1   （你的最愛、我能接受）
  混合 (2/3, 1/3) 對 (1/3, 2/3) → 我拿 2/3 ← 最差！比任何協調成功都糟
```

**這就是整章最該被你帶走的數字。** 那個「公平」的混合均衡——雙方都不肯把方向盤完全交給對方、各自賭一把——給出的期望報酬 **2/3**，比兩個純策略均衡的 **2** 和 **1** 都低。換句話說：當雙方僵在「誰都不先讓」的對稱賭局裡，他們會落到一個**比任一方讓步都更糟**的結果。在分散式系統裡，這就是「兩個節點都不肯放棄競選 leader、反覆瓜分選票、誰也選不上」的活脫脫寫照。協調賽局最深的教訓不是「該選哪個」，是**「拖著不收斂，比收斂到你不喜歡的那個還糟」**。

## 謝林點：賽局理論解不掉、人類卻天天在解的那一步

矩陣到此沉默了。它告訴你有多個一樣好（或差不多好）的均衡，但選不出來。可現實裡，人類**經常**協調成功——而且是在**完全不能溝通**的情況下。怎麼辦到的？

1960 年，Thomas Schelling（謝林）在《The Strategy of Conflict》（衝突的策略）裡做了一個現在傳遍每本賽局教科書的實驗。他問一群人一個問題（這裡照他原書的精神重述）：

> 你明天必須在紐約市跟一個陌生人碰面，但你們**事先無法以任何方式聯絡**——沒約時間、沒約地點。你會去哪裡、幾點？

純報酬矩陣的角度看，這是個無解的賽局：紐約有無數個地點、二十四小時無數個時刻，每一個「雙方都到同一處」都是一個一樣好的納許均衡，多到天文數字。理論說玩家應該迷失在這片均衡的汪洋裡。

但謝林發現，他那個「不科學的樣本」（他自己這麼形容）裡的人，答案驚人地集中：壓倒性多數選了**中午 12 點**，而多數人選了**中央車站（Grand Central）的大鐘／詢問處**。這些人從沒商量過，卻不約而同往同一個點收斂。

關鍵在於：**中央車站的大鐘並不會給你更高的報酬。** 在酒吧碰面、在圖書館碰面、在帝國大廈碰面，只要兩人都去了，報酬完全一樣。報酬矩陣裡，這些點全部等價。中央車站之所以勝出，純粹是因為它**顯著（salient）**——它在文化、傳統、脈絡裡是「紐約的碰面點」，於是它成了一個大家都猜「別人也會猜這裡」的點。中午也一樣：它不是報酬最高的時刻，它只是**自然**——一天的正中央、一個不偏不倚的整點。

謝林把這種解叫做**焦點／謝林點（focal point／Schelling point）**：協調賽局有多個均衡時，人們會收斂到某個因「顯著、自然、文化共識」而從一堆等價選項裡脫穎而出的解。它運作的機制是一層套一層的猜測——**「我猜你會猜大家都選它，所以我也選它，於是它真的成了大家選的那個。」** 顯著性把分散的猜測**對齊**到同一個點，這正是上一節那個 1/2、2/3 爛混合均衡所缺的東西。

這裡有一句話我要請你務必聽清楚，因為它是本章最容易被講反的地方——**謝林點不是賽局理論「算」出來的，它是賽局理論「算不出來」的證據。**

謝林本人從沒宣稱理論能預測人會在大鐘下碰面。恰恰相反：他指出**人們確實會**收斂到焦點，**然後攤開手承認——這件事純報酬矩陣解釋不了**。焦點來自心理顯著性、文化共識、共同脈絡，這些東西**不在矩陣裡**。賽局理論可以告訴你「有 N 個均衡」，但「人會選哪一個」要靠矩陣外的資訊。這是賽局理論一個誠實的、它自己都承認的**局限**，不是它的勝利。把謝林點吹成「賽局理論能預測人類協調行為」，正好把這個故事的寓意講反了。

謝林與 Robert Aumann（奧曼）因為在賽局理論對衝突與合作的貢獻，共同獲得 2005 年諾貝爾經濟學獎（嚴格說是瑞典央行紀念諾貝爾經濟學獎，由瑞典央行 1968 年設立，不在諾貝爾本人 1895 年遺囑的五個獎裡——本書沿用俗稱，這裡註明一次）。

回到你的系統。為什麼整個業界協調到同一套協定慣例、同一個 timestamp 格式、同一個 HTTP 狀態碼語意？大多時候靠的不是「誰最優」的論證，而是顯著性：某個慣例先被一個有影響力的專案／RFC 採用、被反覆引用、進了所有人的「預設猜測」，於是它成了那個「大家都猜別人也會用」的焦點。QWERTY、RFC 3339 的時間格式、語意化版本號（SemVer）——它們的統治力，相當程度來自焦點效應而非最優性。你設計協定時若想讓它被廣泛採用，與其證明它最優，不如讓它**顯著**：對齊既有慣例、給一個無歧義的「明顯該這麼做」的預設。Leader election 協定則反過來——它沒有現成的文化焦點可借，所以乾脆用隨機化超時**人工製造**一個不對稱，讓某個節點先醒來、把自己變成那個焦點。

## 直覺的陷阱

協調賽局是「利益一致卻仍可能失敗」最反直覺的領域，而謝林點是全書最容易被講反方向的概念。下面三個陷阱按「最常掉進去」排序。

| 陷阱 | 錯誤直覺 | 它在哪一步把你帶溝裡 | 正確版 |
|---|---|---|---|
| **謝林點是「算」出來的** | 「賽局理論能預測人會在大鐘下碰面，真厲害」 | 把「Schelling 觀察到人會收斂」誤讀成「理論推導出收斂點」。一旦你以為焦點能從報酬算出來，你就會去矩陣裡找它——而它根本不在矩陣裡 | 焦點來自**心理顯著性／文化／脈絡**，是矩陣**外**的東西。謝林點是賽局理論的**局限**（純報酬選不出均衡），不是它的勝利。理論只說「有多個均衡」，「選哪個」要靠矩陣外資訊 |
| **「利益一致就不會失敗」** | 「協調賽局裡大家都想成功、沒人想害誰，所以一定協調得成」 | 忽略了「多個一樣好的均衡＋不能溝通＝可能各自賭錯」。你以為失敗一定源於有人壞，於是看到協調失敗就去抓「內鬼」 | 利益一致**不夠**。只要均衡不唯一、又不能事先對齊猜測，理性玩家就可能一起賭錯（腦裂）。失敗源於**缺共同焦點**，不是有人居心不良 |
| **把混合均衡當「公平的好結果」** | 「雙方各退一步、各賭一半，聽起來最公平最理性」 | 只看到「對稱、公平」，沒去算它的期望報酬。對稱不等於好 | 性別戰的混合均衡期望報酬 **2/3**，比兩個純均衡（2 和 1）**都差**。它是「誰都不肯先讓」的僵局，是最該逃離的均衡，不是最該追求的妥協 |
| **把協調賽局當成囚徒困境** | 「都是兩個玩家、都會失敗，應該差不多」 | 用困境的直覺（防著對方占你便宜）去處理協調問題，於是猛加懲罰、猛防背叛——但協調賽局根本沒有「背叛能多撈」這回事 | 困境是**利益對立**（背叛佔優、唯一均衡雙輸）；協調是**利益一致**（多個均衡都好、難在選同一個）。困境要破壞背叛誘因，協調要**對齊猜測**——藥方完全不同 |

**怎麼自我察覺你掉進去了**：如果你在協調問題（leader election、標準採用、schema 切換）上猛加「懲罰機制」防著誰背叛，你多半是把它誤當囚徒困境了——協調賽局沒有背叛者，只有賭錯方向的人，藥方是製造一個顯著的焦點（或人工的不對稱），不是加大棒。如果你聽到「賽局理論預測人會碰面在大鐘下」覺得很順耳，停一下：理論**沒有**預測，是 Schelling 觀察到、然後承認理論解釋不了。

## 紙上推演

全程紙筆手算。先自己做，再看解答。

### 推演題 1：數出純協調賽局的所有純策略納許均衡（★★，[15 分鐘]）

三個微服務 A、B、C 要協調到**同一個** API 版本，可選 v1／v2／v3。為簡化，先看**兩個**服務的版本，報酬規則：兩個選同一版 → 各拿 1；選不同版 → 各拿 0。寫出 3×3 報酬矩陣，找出所有純策略納許均衡，數一數有幾個。然後回答：報酬矩陣能不能告訴你該選哪一版？

### 推演題 2：判斷一個「協調 vs 困境」的情境（★★，[15 分鐘]）

把下面這個情境翻成 2×2 賽局，判斷它是「協調賽局」還是「囚徒困境」，並說出你的系統該用哪一類藥方（製造焦點／對齊猜測，還是破壞背叛誘因）：

> 兩個團隊共用一條 CI 管線。每個團隊可以選「把大型測試拆成小批次跑」或「整包一次塞進去跑」。如果兩邊都拆小批次，管線順暢，各拿 3；如果兩邊都整包塞，互相排擠資源、都慢，各拿 1；如果一邊拆、一邊整包塞，整包那邊插隊先跑爽到（拿 5），拆的那邊被餓著（拿 0）。

### 推演題 3：解一個非對稱協調賽局的混合均衡（★★★，[25 分鐘]）

把性別戰的報酬改一下：我（偏好 Opera）協調到 Opera 拿 **3**、協調到 Football 拿 1；對方（偏好 Football）協調到 Football 拿 3、協調到 Opera 拿 1；各去各的雙方拿 0。

```text
                   對方：Opera      對方：Football
    我方：Opera      (3, 1)           (0, 0)
    我方：Football   (0, 0)           (1, 3)
```

找出全部三個納許均衡（兩個純、一個混合），並算出混合均衡裡我的期望報酬。它仍然比兩個純均衡都差嗎？

### 推演解答

**題 1 解答。**

```text
三服務先簡化為兩服務的純協調矩陣（格內 = (我, 對方)）：
                對方 v1     對方 v2     對方 v3
    我方 v1    (1, 1)      (0, 0)      (0, 0)
    我方 v2    (0, 0)      (1, 1)      (0, 0)
    我方 v3    (0, 0)      (0, 0)      (1, 1)
```

最佳反應：對方出 v1，我出 v1 拿 1、其餘拿 0 → 我跟 v1；對方出 v2 我跟 v2；v3 同理。所以**對角線三格 (v1,v1)、(v2,v2)、(v3,v3) 全是純策略納許均衡**——共 **3 個**，而且三個一樣好（都給 (1,1)）。

報酬矩陣**不能**告訴你選哪一版——三個均衡完全對稱，數學沒有偏好。要選出來得靠謝林點：例如「v3 是最新版」「文件預設範例用 v2」「老大在群組裡說過用 v1」這類**矩陣外**的顯著性。

**常見錯路**：以為「v3 最新所以是均衡裡最好的」——錯，在這個純協調報酬下三個均衡報酬相同，「最新」是矩陣外的顯著性、不是報酬優勢。把「我覺得 v3 比較好」偷渡進這個矩陣，就是把焦點誤當成報酬。（推廣到三服務同時協調，純策略 NE 一樣是「三方全選同一版」那幾個，共 3 個，道理不變。）

**題 2 解答。**

```text
                對方：拆小批次   對方：整包塞
    我方：拆小批次   (3, 3)        (0, 5)
    我方：整包塞     (5, 0)        (1, 1)
```

報酬結構：T=5（我整包、對方拆，我插隊）> R=3（都拆）> P=1（都整包）> S=0（我拆、對方整包，我被餓）。這是 **T>R>P>S＝5>3>1>0——它就是脊椎那張囚徒困境**（ch01 那張矩陣）！「整包塞」是嚴格優勢策略（5>3 且 1>0），唯一 NE 是 (整包,整包)=(1,1)，但 (拆,拆)=(3,3) 對雙方更好。

所以這是**囚徒困境，不是協調賽局**。藥方是**破壞背叛（整包塞）的誘因**——例如重複賽局＋聲譽（ch07）、或機制設計加個「整包塞要排到隊尾」的外部懲罰（ch13），而不是「製造焦點」。

**常見錯路**：看到「兩團隊要協調 CI 用法」就反射認定是協調賽局。判別關鍵在報酬結構：**有沒有「我偏離、對方守規矩，我反而多撈」的格子（T>R）？** 有，就是困境（利益對立）；沒有、偏離只會讓自己更差，才是協調（利益一致）。題目裡「整包那邊插隊拿 5」就是 T>R 的鐵證——這是困境披了協調的皮。

**題 3 解答。**

純策略 NE：最佳反應掃描跟性別戰完全同構——對角線 (Opera,Opera) 與 (Football,Football) 互為最佳反應，**兩個純策略 NE**。在 (Opera,Opera) 我拿 3、對方拿 1；(Football,Football) 我拿 1、對方拿 3。

混合 NE，用無差異原則。設對方出 Opera 機率 q，讓**我**無差異：

```text
我出 Opera    = 3q + 0·(1−q) = 3q
我出 Football = 0·q + 1·(1−q) = 1 − q
令相等：  3q = 1 − q  →  4q = 1  →  q = 1/4   ← 對方出 Opera 1/4、Football 3/4
```

設我出 Opera 機率 p，讓**對方**無差異（對方看第二個數，協調到 Opera 拿 1、到 Football 拿 3）：

```text
對方出 Opera    = 1·p + 0·(1−p) = p
對方出 Football = 0·p + 3·(1−p) = 3(1−p)
令相等：  p = 3(1−p)  →  p = 3 − 3p  →  4p = 3  →  p = 3/4   ← 我出 Opera 3/4
```

我的期望報酬（用無差異捷徑最快：我出 Opera 的期望 = 3q = 3·1/4 = 3/4）：

```text
交叉複核（四格展開，我出 Opera 機率 3/4、對方出 Opera 機率 1/4）：
  (Opera,    Opera)    = 3/4·1/4 = 3/16 → 我拿 3
  (Opera,    Football) = 3/4·3/4 = 9/16 → 我拿 0
  (Football, Opera)    = 1/4·1/4 = 1/16 → 我拿 0
  (Football, Football) = 1/4·3/4 = 3/16 → 我拿 1
  機率和 = 3/16+9/16+1/16+3/16 = 16/16 = 1 ✓
  期望 = 3/16·3 + 3/16·1 = 9/16 + 3/16 = 12/16 = 3/4 ✓ 與捷徑一致
```

**混合均衡我拿 3/4，仍然比兩個純均衡（3 和 1）都差。** 結論穩固：協調賽局裡，「對稱地各賭一半」幾乎總是最爛的那個均衡——不論偏好強度怎麼調，僵著不收斂都比收斂到任一方都糟。

**常見錯路**：把 p 算成「讓我自己無差異」的那條方程的解。記住 ch05 的鐵律——**我的機率 p 由「讓對方無差異」決定**，不是讓我自己無差異。算反了會得到 p=1/4，數字剛好對稱地錯，很難一眼看出，必須用四格展開交叉複核才抓得到。

## 自我檢核

口頭自答，講得出來才算過。

1. 協調賽局跟囚徒困境，結構上最根本的差別是什麼？（提示：利益對立 vs 利益一致；唯一壞均衡 vs 多個好均衡。）為什麼這個差別決定了藥方完全不同？
2. 純報酬矩陣為什麼**選不出**多個均衡裡的哪一個？這跟「我們算得不夠仔細」是不是同一回事？
3. 謝林點是不是賽局理論「算」出來的？如果不是，那它從哪來、為什麼說它是賽局理論的**局限**而不是勝利？（這題答錯就是 T8 陷阱。）
4. 性別戰那個混合均衡，期望報酬比兩個純均衡都**低**——這為什麼反直覺？「對稱／公平」為什麼不等於「好」？
5. 為什麼「大家利益一致、都想協調成功」**不保證**協調成功？舉一個你系統裡的具體例子（leader election 腦裂／schema 切換時機不一致）。
6. leader election 協定用「隨機化選舉超時」打破對稱——這跟 ch05 的 backoff+jitter、跟「製造一個謝林點」分別是什麼關係？
7. 協調失敗跟脊椎那張囚徒困境的「個體理性 ⇒ 集體更糟」，怎麼說它們是**同一條裂縫**的兩張臉？（提示：一個是集體更糟、一個是集體錯位，但都是個體理性不保證集體達成所願。）

## 延伸閱讀

- **Thomas Schelling,《The Strategy of Conflict》(1960)** — 謝林點的原始出處，紐約碰面實驗就在裡頭。讀「focal point」那幾頁就夠，看他如何一邊展示人類會協調、一邊坦承理論解釋不了。賽局理論最誠實的一本書之一。
- **史丹佛哲學百科：Prisoner's Dilemma 條目**（plato.stanford.edu/entries/prisoner-dilemma/）— 雖以困境為主，但把「協調 vs 困境」的結構差異講得很清楚，可對照本章第一節。（landscape 已驗證連結。）
- **維基百科：Focal point (game theory)**（en.wikipedia.org/wiki/Focal_point_(game_theory)）— 謝林點的快速綜述與紐約實驗的數字（中央車站、中午）來源，含後續行為實驗的整理。查證焦點實驗細節時讀這條。（landscape 已驗證連結。）
- **NobelPrize.org，2005 年經濟學獎頁面** — Schelling 與 Aumann 的得獎理由「透過賽局理論分析增進對衝突與合作的理解」，附背景介紹。想確認獎項歸屬與年份讀官方頁。
- 跨書：協調賽局裡「每個玩家在不確定下該怎麼賭哪個均衡」的**單人理性**那一面，深入處見《在不確定中下注》談不確定下決策那章——本章只用「對手也在猜」的互動面，單人怎麼賭留給它。混合均衡期望報酬的機率運算，機制見《馴服隨機》ch08 期望值。
