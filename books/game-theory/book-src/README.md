# 當理性互相碰撞

### 賽局理論的直覺與驚嘆

> 當每個人都完全理性、各自最佳化自己的選擇，結果卻可能讓所有人一起變糟——而「理性」本身在多人互動裡，會分裂成「個體理性」與「集體理性」兩種互相打架的東西。
>
> 這本書想讓你親身感受這一刀——並且能用自己的話，對另一個工程師講清楚：**囚徒困境的 (背叛, 背叛) 為什麼是穩定的均衡卻是雙輸，而合作要靠什麼條件才救得回來。**

---

這是一本寫給資深後端工程師的賽局理論「欣賞」書。純興趣、和工作無關、純紙上推演——它不教你證明納許存在定理、不要你跑任何程式、不發 lab 作業。它只想帶你看懂一件反直覺的事：**「大家都聰明」不保證「大家都好」，合作不是預設值，而是一個需要被解釋、甚至被設計出來的奇蹟。**

你已經在工作裡碰過賽局的影子，只是沒這樣叫它：retry storm 裡每個 client 重試都是個體理性、整體把服務打垮卻是集體災難（這就是工程版的囚徒困境）；leader election 選誰當 leader 不重要、選不一致才腦裂（協調賽局）；circuit breaker 被打爆就斷、恢復了再試、不記仇（某種以牙還牙）；backoff+jitter 用隨機去同步打破壞均衡（混合策略）；你設計過的 API 配額、計費方案、排程優先級，全都是「設計報酬讓使用者的理性行為導向你要的系統行為」（機制設計）。這本書把這些直覺接上一個更大的故事。

## 這本書怎麼讀

- **純紙上推演**：每章的「紙上推演」是手填報酬矩陣、找均衡、解混合策略、口頭自答的思辨題，不是要你開終端機。所有數值示範一律手算、攤在你眼前的 ` ```text ` 表裡。
- **直覺與精確意義優先**：每個概念先回答「它在問什麼問題」（納許均衡在問「哪個策略組合下沒人想單方面變」、混合策略在問「怎麼讓對手猜不到又不吃虧」），再給形狀。定理給直覺、給條件、給結論，嚴格證明指向延伸閱讀。
- **講得出來才算過**：每章末的「自我檢核」是口頭題——對齊「能不能把這個概念講給另一個工程師聽」的深度。優先問「為什麼」而不是「是什麼」。講得出來，才算真的懂了。
- **故障視角是賣點**：賽局理論正是「聰明人集體犯傻」最反直覺的領域。每章的「直覺的陷阱」段，專門守住那些被講爛了的庸俗版（「賽局理論就是你死我活的零和」、「以牙還牙是最強策略」、「囚徒困境證明合作不可能」），還你正確版。

## 一條脊椎貫穿全書

整本書反覆回到**同一張 2×2 囚徒困境報酬矩陣**（數字＝獲益點數、越大越好）：

```text
                  對方：合作(C)    對方：背叛(D)
   我方：合作(C)     (3, 3)          (0, 5)
   我方：背叛(D)     (5, 0)          (1, 1)
   T>R>P>S＝5>3>1>0；2R>T+S＝6>5
```

每深入一個 Part，你就重新認識這張矩陣一次：ch01 它讓困境誕生；ch02 它示範均衡為何不等於最優；ch04 它對照出零和只是特例；ch07 重複玩它、未來的陰影救回合作；ch09 把它丟進演化族群；ch13 反過來改它的報酬把困境救出來；ch18 一次對帳這七層意義。中央那條反覆叩問的張力始終不變：**每個人都最佳化自己，結果集體更糟——個體理性與集體理性，如何在同一張矩陣裡分裂。**

## 書架上的鄰居

這本書只用自己的欣賞聲音講「為何反直覺、合作為何是奇蹟」；需要更深的運算時，它會把你指向書架上相鄰的書，而不是重講一遍：

- 一個人在不確定下該怎麼選、期望效用、效用不只是金額 →《在不確定中下注》（本書只在「對手會反應你的選擇」時才接手）
- 期望值怎麼算、混合策略的期望報酬運算、族群比例作為機率分布 →《馴服隨機》
- 策略情境下非理性的心理機制、公平偏好從哪來 →《思考的陷阱》
- 混合策略幾何、演化動態的特徵值與穩態 →《矩陣是動詞》

## 給誰讀

唯一讀者：一位 6 年以上的資深後端工程師。你不需要任何賽局理論背景——你需要的直覺，retry storm、背壓、leader election、限流、spot 競價、backoff+jitter 早就教會你了。本書把這些工程經驗當成錨點，從「對手也會算」起步，一路帶你看到「合作可以被設計出來」。

準備好了，就從 **ch01 — 囚徒困境登場：每個人都對，結果一起錯** 開始。你的直覺正準備在接下來每一章騙你一次——這正是值得你停下來、慢慢讀的地方。
