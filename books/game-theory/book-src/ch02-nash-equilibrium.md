# ch02 — 納許均衡：穩定不等於最好

> **本章解決什麼問題**：ch01 用「優勢策略」把你帶進囚徒困境——背叛不管對方怎麼選都比較好，於是兩個理性玩家一起掉進 (1,1)。但優勢策略只是個方便的特例：多數賽局裡沒有哪個選項「不管對方怎麼做都最好」，你需要一把更通用的尺。這把尺就是納許均衡（Nash equilibrium）。本章把它講透——它定義的是「穩定」，不是「最優」，而這兩件事在多人互動裡會徹底分家。這是全書最核心的一章；ch03 之後的所有工具（最佳反應消去、混合策略、重複賽局、機制設計）都站在這個定義上。

## 從你已知的出發

你維運過卡在壞狀態裡出不來的系統。

想像一個微服務叢集在重試風暴（retry storm）裡。下游被打爆、回應變慢，每個上游 client 都在重試——因為對**單一 client** 而言，重試是理性的：別人都在搶連線池，我不重試就只有我拿不到回應，我先餓死。於是沒有任何一個 client 願意第一個停手。整個系統卡在一個誰都不滿意、卻誰都不敢單方面離開的狀態。

請你停在「誰都不敢單方面離開」這句話上十秒鐘。這不是「大家都笨」，恰恰相反——這是每個節點都正確地算過自己的最佳選擇之後的結果。「我單方面停止重試」這個動作，對我自己是嚴格變差的（別人照搶，我直接出局）。所以這個爛狀態是**穩定**的：沒有人能靠**自己一個人**改變行為而獲益。

這就是納許均衡的全部直覺。納許均衡不是「最好的狀態」，而是「沒有人能單方面改善的狀態」。重試風暴裡的全體重試是一個納許均衡；它糟透了，但它穩定。要跳出去，你得讓**大家同時**改變（協調退避、加 jitter 去同步、circuit breaker 強制斷開）——而「需要大家同時動、靠任何人自己動沒用」正是均衡的指紋。

你其實天天在跟納許均衡打交道，只是沒給它名字：

- **leader election 沒收斂時的對峙**：每個節點都在等別人先承認自己是 follower，沒人肯先退，腦裂（split-brain）僵住。
- **所有人都在輪詢一個過載的 API**：單方面降低自己的輪詢頻率＝自己的資料更新最慢，沒人想當那個人。
- **快取全部同時失效後的 cache stampede**：每個請求都去打 DB 是「對我這條請求最快」的選擇，整體把 DB 壓垮。

這一章要做的，是把這個你已經有的工程直覺，磨成一個能用在任何賽局上的精確工具——並且讓你看清楚一件最反直覺的事：**穩定的均衡，可以是對所有人都糟的結局，而這一點也不矛盾。**

## 優勢策略太奢侈了，我們需要一把更通用的尺

先說清楚 ch01 給你的工具有什麼限制。

ch01 靠的是**優勢策略（dominant strategy）**：背叛這個選項，不管對方選 C 還是 D，我的報酬都比合作高（5>3 而且 1>0）。當你有優勢策略，決策簡單到不像賽局——你根本不用管對方在想什麼，閉著眼睛選優勢策略就對了。囚徒困境之所以乾淨，正因為它雙方都有優勢策略，一步解完。

問題是：**絕大多數有趣的賽局裡，沒有人有優勢策略。** 你的最佳選擇會隨對方的選擇而變。

舉個你熟的例子。兩個服務要協調用同一個序列化格式，選 JSON 或選 Protobuf 都行，只要**選一樣的**就能通訊；選不一樣就互不相容、雙輸。這裡「我該選 JSON 還是 Protobuf」完全取決於對方選什麼——對方選 JSON 我就該選 JSON，對方選 Protobuf 我就該選 Protobuf。沒有任何一個選項是「不管對方怎樣都最好」的。優勢策略這把尺在這裡量不出東西。

所以我們需要一個**不依賴優勢策略也能用**的均衡概念。它要能回答這個更一般的問題：

> 在一個策略組合裡，如果每個玩家都「假設別人不動」，他自己還想不想動？如果所有人都不想動——這個組合就站得住。

這就是納許均衡。1950 年，二十一歲的約翰・納許（John Nash）在普林斯頓的博士論文裡把這個想法寫成定義（指導教授正是 ch01 替囚徒困境命名的塔克（Albert Tucker），這個巧合很賽局理論）。同年他以一篇短文在《PNAS》公告了存在性結果，完整版 1951 年刊在《Annals of Mathematics》。這一個定義後來幾乎重寫了經濟學，也讓他在 1994 年與哈薩尼（John Harsanyi）、澤爾騰（Reinhard Selten）共獲諾貝爾經濟學獎——「非合作賽局均衡理論的開創性分析」。

（一個誠實的小註：「諾貝爾經濟學獎」嚴格說是「瑞典央行紀念諾貝爾經濟學獎」，由瑞典央行 1968 年設立、1969 起頒，不在諾貝爾 1895 遺囑的五個獎之列。全書沿用俗稱，這裡註明一次。）

## 納許均衡：沒有人想單方面偏離

先給精確意義，再給形狀。

**納許均衡在問的問題是**：哪個策略組合，在「其他人都維持不動」的前提下，沒有任何一個玩家能靠改變自己的策略而拿到更高報酬？

把它拆成一句工程師能複誦的話：

> **一個策略組合是納許均衡，若且唯若——每個玩家的策略，都是「對其他玩家當前策略」的最佳反應。**

「最佳反應（best response）」是這裡唯一的零件：固定住所有對手的選擇，我能拿到最高報酬的那個選擇，就是我對這個局面的最佳反應。納許均衡就是「所有人的選擇互相都是對方的最佳反應」——一個自我一致（self-consistent）的不動點，誰都沒有單方面改進的空間。

注意這個定義裡藏著的兩個關鍵限制，它們是後面所有「陷阱」的根源：

1. **「單方面」是核心。** 均衡只擋得住「一個人自己偷偷改」的偏離。它完全不保證「大家一起改」不會更好——事實上囚徒困境裡，大家一起從 (D,D) 改成 (C,C) 就是雙贏，但這需要兩個人**同時**動，不在「單方面」的射程內。均衡對「協同偏離」一無所知。
2. **均衡只談穩定，不談好壞。** 定義裡沒有任何一個字提到「報酬要高」「對大家要好」「要 Pareto 最優」。它只說「沒人想自己動」。一個爛到家的結局，只要沒人能靠自己脫身，它就是均衡。

第二點是本章——也是整本書——最該被你帶走的一句話：**納許均衡是穩定（stable），不是最優（optimal）。** 把這兩個詞當成同義詞，是賽局理論最常見的誤解，沒有之一。

## Worked example：在脊椎矩陣上找出所有純策略納許均衡

來把定義套到 ch01 那張矩陣上，用「最佳反應分析」一格一格地查。這是全書最值得你親手算一遍的推導。

這是 ch01 那張矩陣（全書脊椎，T,R,P,S=5,3,1,0，數字＝獲益點數、越大越好；報酬寫成 (我方, 對方)，列＝我方策略、行＝對方策略）：

```text
本章脊椎：囚徒困境基準矩陣（這是 ch01 那張矩陣）
                     對方：合作(C)      對方：背叛(D)
    我方：合作(C)      (3, 3)            (0, 5)
    我方：背叛(D)      (5, 0)            (1, 1)

    T>R>P>S ＝ 5>3>1>0
```

**找納許均衡的機械程序：** 對每一個玩家、固定對方的選擇，圈出他的最佳反應；最後「兩個玩家的最佳反應同時被滿足」的那些格，就是純策略納許均衡。

**第一步——我方（列玩家）的最佳反應，固定對方的行：**

```text
對方選 C（看左欄，比較我方兩列的「我方報酬」＝每格第一個數）：
    我方選 C → 我得 3
    我方選 D → 我得 5      ← 5 > 3，我的最佳反應是 D ✔

對方選 D（看右欄，比較我方兩列的「我方報酬」）：
    我方選 C → 我得 0
    我方選 D → 我得 1      ← 1 > 0，我的最佳反應是 D ✔
```

我方的最佳反應**恆為 D**（這正是 ch01 說的「D 是嚴格優勢策略」——優勢策略只是「最佳反應永遠是同一個」的特例）。

**第二步——對方（行玩家）的最佳反應，固定我方的列。** 這張矩陣對稱，所以對方的演算法鏡像對稱（比較每格第二個數＝對方報酬）：

```text
我方選 C（看上列，比較對方兩行的「對方報酬」）：
    對方選 C → 對方得 3
    對方選 D → 對方得 5    ← 5 > 3，對方的最佳反應是 D ✔

我方選 D（看下列，比較對方兩行的「對方報酬」）：
    對方選 C → 對方得 0
    對方選 D → 對方得 1    ← 1 > 0，對方的最佳反應是 D ✔
```

對方的最佳反應也**恆為 D**。

**第三步——找交集。** 把兩邊的最佳反應標到矩陣上（`*` 標我方的最佳反應、`#` 標對方的最佳反應）：

```text
標記最佳反應（* ＝我方最佳反應，# ＝對方最佳反應）：
                     對方：C            對方：D
    我方：C          (3, 3)            (0, 5#)
    我方：D          (5*, 0)           (1*, 1#)   ← 兩個記號同時出現！

  唯一同時被 * 和 # 命中的格 ＝ (D, D) ＝ (1, 1)
  ⇒ 唯一的純策略納許均衡是 (背叛, 背叛)
```

只有右下角同時帶 `*` 和 `#`——也就是說，在 (D,D) 這個組合裡，我方在玩最佳反應、對方也在玩最佳反應。**(D,D)=(1,1) 是唯一的純策略納許均衡。**

**第四步——反過來驗算，確認其他三格都不是。** 這一步是工程師的嚴謹：每個「不是均衡」都要能指出「誰會想偏離、偏到哪、賺多少」。

```text
逐格做「單方面偏離」測試（誰能靠自己改一手而得益？）：

 (C,C)=(3,3)：我方單方面改成 D → 我的報酬 3 → 5（變好！）
              ⇒ 我想偏離，(C,C) 不是均衡。
              （它是雙方最好的結局，卻擋不住背叛的誘惑——這就是困境）

 (C,D)=(0,5)：我方單方面改成 D → 我的報酬 0 → 1（變好！）
              ⇒ 我想偏離，不是均衡。

 (D,C)=(5,0)：對方單方面改成 D → 對方報酬 0 → 1（變好！）
              ⇒ 對方想偏離，不是均衡。

 (D,D)=(1,1)：我方單方面改成 C → 我的報酬 1 → 0（變差）
              對方單方面改成 C → 對方報酬 1 → 0（變差）
              ⇒ 沒有人想偏離。穩定。✔ 這是納許均衡。
```

兩種視角——「最佳反應交集」（第一到三步）與「單方面偏離測試」（第四步）——指向同一格 (D,D)，互相印證。這個交叉驗算的習慣請你保留：找均衡時用最佳反應快速圈，驗均衡時用偏離測試逐格確認。

**停下來看清楚這裡發生了什麼。** 唯一穩定的結局 (1,1)，對雙方都比 (3,3) 差。納許均衡這把尺老老實實地告訴你：這個雙輸的結局，沒有人能靠自己脫身。它是**穩定的雙輸**。均衡 ≠ 最優，這張矩陣就是鐵證。這也正是 ch01 中央張力的 ch02 版本——個體理性（各自玩最佳反應）把集體鎖死在次優均衡裡，而「均衡」這個詞本身完全沒有「對大家好」的意思。

## 優勢策略均衡一定是納許均衡，反之不然

上面那個 worked example 順手揭露了優勢策略和納許均衡的關係，值得單獨點明，因為它釐清「為什麼我們需要納許均衡這個更大的概念」。

**優勢策略均衡是納許均衡的特例。** 如果每個玩家都有優勢策略（最佳反應**恆為同一個**、與對方無關），那這組優勢策略當然互為最佳反應——它根本連「看對方」都不用看就已經是最佳反應了。所以「優勢策略均衡」自動是「納許均衡」。囚徒困境的 (D,D) 正是這種雙重身分。

**但反過來不成立。** 納許均衡只要求「對方當前選擇下我最佳」，不要求「對方任何選擇下我都最佳」。回到那個 JSON / Protobuf 協調的例子：

```text
協調賽局（選同一個才通；選一樣得 1，選不一樣得 0）：
                     對方：JSON         對方：Protobuf
    我方：JSON       (1, 1)            (0, 0)
    我方：Protobuf   (0, 0)            (1, 1)

最佳反應分析：
  對方選 JSON     → 我方最佳反應 JSON（1>0）
  對方選 Protobuf → 我方最佳反應 Protobuf（1>0）   ← 我的最佳反應「會變」！

⇒ 我沒有優勢策略（最佳反應依對方而定）
⇒ 但 (JSON,JSON) 和 (Protobuf,Protobuf) 兩個都是純策略納許均衡
   （在任一個裡，沒人想單方面換成不相容的格式）
```

這個賽局沒有任何玩家有優勢策略，卻有**兩個**純策略納許均衡。優勢策略那把尺在這裡完全失效，納許均衡卻照樣量得出來。這就是為什麼納許均衡是更通用的工具——它涵蓋了「最佳選擇隨對方而變」的所有情況，而那是現實裡的常態。

順帶看見了一個新東西：**均衡可以不只一個。** 這張協調矩陣有兩個。當有多個均衡時，「沒人想單方面偏離」就管不到「大家會收斂到哪一個」——選 JSON 還是 Protobuf 的兩個均衡同樣穩定，賽局理論本身選不出來。這個「多重均衡的選擇問題」我們留到 Part IV 的協調賽局與謝林點（見 ch10）展開，這裡只先讓你看見它存在。

## 納許存在定理：每個有限賽局裡都至少有一個均衡

你可能已經在問一個焦慮工程師該問的問題：**會不會有些賽局根本沒有納許均衡？** 如果這個工具有時候「查無此物」，那它就沒那麼通用了。

剪刀石頭布就是個警訊。你出剪刀，對方最佳反應是石頭；對方出石頭，你最佳反應是布；你出布，對方最佳反應是剪刀……繞一圈回來，**沒有任何一個純策略組合是穩定的**——不管停在哪一格，總有一方想偏離。剪刀石頭布沒有**純策略**納許均衡。

納許 1950/1951 的存在定理就是來回答這個焦慮的：

> **納許存在定理（直覺版，嚴格證明本書不展開）**：每一個**有限賽局**（有限個玩家、每人有限個策略），只要允許**混合策略**（在自己的選項上放一個機率分布、隨機地選），就**至少存在一個**納許均衡。

關鍵在「混合策略」這四個字。剪刀石頭布沒有純策略均衡，但有一個混合策略均衡：各以 1/3 機率出剪刀、石頭、布。當對手是真隨機 1/3 時，你出任何一手的期望報酬都一樣（無從利用），你也沒有偏離的理由——這就是均衡。**存在性是靠『允許隨機化』買來的。**

證明本身用的是 Kakutani 不動點定理（角谷不動點定理，1941 年從 Brouwer 不動點推廣而來）：把「每個人對別人策略的最佳反應」看成一個從策略空間到自身的**對應（correspondence）**——之所以是「對應」而非「函數」，是因為對同一個對手局面，最佳反應可能不只一個（多個策略並列最佳），這正是必須用 Kakutani（容許多值對應）而非 Brouwer（只處理單值函數）的關鍵。這個對應在混合策略構成的緊緻凸集上滿足 Kakutani 的條件，於是必有不動點——而不動點恰恰就是「每個人都在對別人玩最佳反應」的那個點，也就是納許均衡。**這只是直覺，不是證明**；嚴格論證（為什麼最佳反應對應有閉圖、值域凸非空）指向延伸閱讀，本書不展開。

對你有用的，是把這個定理的「保證」和「不保證」分清楚——這是工程師讀定理該有的紀律：

```text
納許存在定理保證什麼、不保證什麼：
  ✔ 保證：有限賽局（含混合策略）至少有一個納許均衡——「查得到」
  ✘ 不保證：均衡唯一（協調賽局有兩個以上、見 ch10）
  ✘ 不保證：均衡是純策略（剪刀石頭布只有混合均衡、見 ch05）
  ✘ 不保證：均衡對大家好（囚徒困境的均衡是雙輸）
  ✘ 不保證：玩家找得到 / 收斂到它（這是計算與動態問題，定理不管）
```

混合策略均衡怎麼算、那個「讓對手無差異」的反直覺技巧，是 ch05 的主角，這裡只用它的結論：**存在性靠混合策略撐住，所以「沒有均衡」這件事在有限賽局裡不會發生。** 你手上這把尺，永遠量得到東西。

## 直覺的陷阱

納許均衡是整本書最容易「自以為懂了其實沒懂」的概念，因為「均衡」這個詞在日常語感裡偷渡了太多它其實不保證的東西。下面四個陷阱，每一個都會在某一步把你帶溝裡。

| 陷阱 | 錯誤直覺 | 在哪一步出錯 | 正確版（怎麼自我察覺） |
|---|---|---|---|
| **均衡＝最優** | 「均衡是賽局的最佳解、是大家會滿意的結果」 | 把「沒人想單方面動」偷換成「對大家最好」 | 均衡只保證穩定、不保證好壞。囚徒困境 (1,1) 是均衡卻雙輸。**徵兆**：你說「均衡」時心裡浮現的是「皆大歡喜」——停，那是 Pareto 最優，不是納許均衡 |
| **均衡＝沒人能變更好** | 「在均衡裡，誰都無法得到更高報酬」 | 漏掉「單方面」三個字 | 只保證「**一個人自己**改不會更好」。大家**一起**改可以更好（PD 從 (D,D) 一起變 (C,C) 雙贏），均衡管不到協同偏離。**徵兆**：你忘了問「誰跟誰一起動」 |
| **均衡唯一** | 「算出一個納許均衡就解完了」 | 假設均衡像方程式的根一樣唯一 | 賽局可以有多個純策略均衡（協調賽局）、甚至連續多個。**徵兆**：你只圈了一格就收工，沒檢查其他格 |
| **沒有均衡** | 「剪刀石頭布沒有穩定解，所以賽局理論在這裡無解」 | 只在純策略裡找 | 純策略可能無均衡，但允許混合策略後存在定理保證至少有一個。**徵兆**：你找不到純策略均衡就下結論「無解」，忘了隨機化 |

最致命、也最該反覆自問的是第一個：**穩定不等於最好。** 一個焦慮的工程師其實天生對它有免疫力——你看過太多「穩定的爛狀態」：卡在重試風暴裡的叢集、誰都不肯先退的 leader election、技術債堆到沒人敢動的服務。這些都穩定（沒人能靠自己脫身），也都糟糕。把「納許均衡」翻譯成「系統卡住的穩態」，而不是「系統的最佳解」，你就抓住它了。

## 紙上推演

全程純紙筆。報酬矩陣手畫、最佳反應手圈、偏離測試逐格寫下「誰賺多少」。每題先自己做完再看解答。

### 第 1 題（★★，**[15 分鐘]**）找出協調賽局的所有純策略納許均衡

兩個服務要決定送訊息走哪條 queue，可選 Kafka 或 RabbitMQ。只要兩邊走同一條就能通訊（各得 2），走不同條就互不相通（各得 0）。但兩邊都偏好 Kafka——若都走 Kafka，各得 3 而非 2。報酬矩陣如下（(我方, 對方)）：

```text
                     對方：Kafka        對方：RabbitMQ
    我方：Kafka       (3, 3)            (0, 0)
    我方：RabbitMQ    (0, 0)            (2, 2)
```

請用最佳反應分析找出**所有**純策略納許均衡，並對每一個非均衡格指出「誰想偏離、偏到哪、報酬怎麼變」。

### 第 2 題（★★，**[15 分鐘]**）證明「優勢策略均衡一定是納許均衡，反之不然」

不靠記憶，用納許均衡的定義（每個玩家的策略是對其他玩家當前策略的最佳反應）論證：

(a) 若每個玩家都有嚴格優勢策略，這組優勢策略構成的組合必然是納許均衡。
(b) 舉一個有納許均衡、但沒有任何玩家有優勢策略的賽局，說明反向不成立。

### 第 3 題（★，**[10 分鐘]**）口頭題：均衡為什麼不等於最好的結果

向一個沒學過賽局理論的工程師同事，用囚徒困境的脊椎矩陣（這是 ch01 那張）口頭解釋：為什麼 (背叛,背叛)=(1,1) 明明對雙方都比 (合作,合作)=(3,3) 差，卻是「均衡」、而 (3,3) 不是。你的解釋裡必須出現「單方面」這個詞，並說清楚它為什麼是關鍵。

---

### 推演解答

**第 1 題解答。**

最佳反應分析（這張矩陣對稱，算一邊即可鏡像）：

```text
我方的最佳反應（固定對方的行，比較每格第一個數）：
  對方選 Kafka     → 我方 Kafka 得 3、RabbitMQ 得 0 → 最佳反應 Kafka
  對方選 RabbitMQ  → 我方 Kafka 得 0、RabbitMQ 得 2 → 最佳反應 RabbitMQ
（對方對稱，同理）

標記交集：
                     對方：Kafka        對方：RabbitMQ
    我方：Kafka      (3*, 3#)          (0, 0)
    我方：RabbitMQ   (0, 0)            (2*, 2#)

⇒ (Kafka, Kafka)=(3,3) 和 (RabbitMQ, RabbitMQ)=(2,2) 都是純策略納許均衡。
```

兩個都是均衡。對非均衡格做偏離測試：

- **(Kafka, RabbitMQ)=(0,0)**：我方單方面改成 RabbitMQ → 0→2（變好），想偏離 → 不是均衡。
- **(RabbitMQ, Kafka)=(0,0)**：對方單方面改成 RabbitMQ → 0→2（變好），想偏離 → 不是均衡。

**關鍵觀察**：這個賽局有**兩個**純策略納許均衡，而且其中一個 (2,2) 對雙方都比另一個 (3,3) 差——但 (2,2) 照樣是穩定均衡！在 (RabbitMQ,RabbitMQ) 裡，任何一方單方面改走 Kafka 都會從 2 掉到 0（對方還在 RabbitMQ、不相容），所以誰也不敢先動。這是「協調失敗」的縮影：明明大家都偏好 Kafka，卻可能集體卡在較差的 RabbitMQ 均衡裡，因為沒人能單獨脫身。**怎麼讓大家收斂到比較好的那個均衡**，純報酬矩陣答不出來，要靠謝林點（見 ch10）。

**常見錯路**：(1) 只圈到 (3,3) 就收工，漏掉 (2,2)——納許均衡不保證唯一，逐格都要查。(2) 以為「(2,2) 比 (3,3) 差所以不是均衡」——這正是「均衡＝最優」的陷阱；好壞與穩定無關。

**第 2 題解答。**

**(a)** 設玩家 i 有嚴格優勢策略 sᵢ*，意思是：**不管其他玩家選什麼**，sᵢ* 給 i 的報酬都嚴格高於 i 的其他任何策略。特別地，當其他玩家正在玩他們各自的優勢策略時，sᵢ* 仍然是 i 的最佳反應（因為它對「任何」對手組合都是最佳的，自然也對「這一個」組合最佳）。這對每個玩家都成立，所以「每個人的優勢策略」這個組合裡，**每個人的策略都是對其他人當前策略的最佳反應**——這恰好就是納許均衡的定義。故優勢策略均衡必為納許均衡。（囚徒困境的 (D,D) 即此例。）

**(b)** 反向不成立。取第 1 題的 Kafka/RabbitMQ 協調賽局：沒有任何玩家有優勢策略（我的最佳反應隨對方而變——對方走 Kafka 我該走 Kafka，對方走 RabbitMQ 我該走 RabbitMQ），卻有兩個純策略納許均衡。所以「是納許均衡」推不出「有優勢策略」。

**常見錯路**：把 (a) 寫成「優勢策略報酬最高所以是均衡」——「報酬最高」不是均衡的定義，「互為最佳反應」才是；要扣著定義走。另外注意 (a) 用的是**嚴格**優勢策略；弱優勢策略的版本有微妙差異（弱優勢均衡仍是納許均衡，但「嚴格高於」要放寬成「不低於」），這留待 ch03 細談。

**第 3 題解答（口頭版範本）。**

「你看這張表。(合作,合作) 兩個人各拿 3，比 (背叛,背叛) 各拿 1 好得多。那為什麼穩定的是雙輸的那個？

關鍵在『單方面』這三個字。納許均衡問的不是『哪個結果對大家最好』，而是『站在某個結果上，**有沒有人能靠自己一個人改變選擇而賺到更多**』。

從 (合作,合作) 出發：只要我**自己偷偷**改成背叛，對方還傻傻合作，我就從 3 跳到 5——我有動機背叛。所以 (合作,合作) 站不住，它不是均衡。

從 (背叛,背叛) 出發：我**自己**改成合作？對方還在背叛，我就從 1 掉到 0，更慘。對方同理。誰自己動誰吃虧，所以沒人動。它穩定，它是均衡——儘管它對我們倆都糟。

所以『均衡』不是『最好』，是『沒人能自己脫身』。我們倆**一起**改成合作明明雙贏，但那需要我們**同時**動、還得互相信任對方不趁機背刺——而均衡這個概念只管『一個人自己動』，管不到『兩個人一起動』。這就是為什麼合作在這裡這麼難。」

**自我檢核**：你的解釋有沒有出現「單方面 / 自己一個人」？有沒有說清「一起動可以更好、但均衡不管協同偏離」？如果你只說了「因為背叛比較好」而沒點出「單方面」，你其實只重複了 ch01 的優勢策略，沒講到 ch02 的均衡。

## 自我檢核

口頭自答，講得出來才算過關。優先回答「為什麼」。

1. 用一句話說出納許均衡的定義，且這句話裡必須包含「最佳反應」或「單方面」——為什麼這個詞是不可省的？
2. 為什麼 (背叛,背叛) 是均衡而 (合作,合作) 不是？把「單方面偏離」的報酬變化具體說出來（從哪個數到哪個數）。
3. 「納許均衡是穩定不是最優」——舉一個你維運過的系統壞穩態（重試風暴、卡死的 leader election、技術債），說明它為什麼是個納許均衡。
4. 優勢策略均衡和納許均衡是什麼關係？為什麼「優勢策略均衡 ⇒ 納許均衡」成立、反過來不成立？
5. 剪刀石頭布沒有純策略納許均衡——這跟「納許存在定理保證至少有一個均衡」矛盾嗎？定理靠什麼東西化解這個張力？
6. 納許存在定理「不保證」哪四件事？（唯一性、純策略、對大家好、玩家找得到——你能各舉一例嗎？）
7. 協調賽局有兩個納許均衡時，賽局理論能告訴你大家會收斂到哪一個嗎？為什麼不能？
8. 把「穩定不等於最好」這句話，翻成你會對另一個工程師講的一句工程白話。

## 延伸閱讀

- **Stanford Encyclopedia of Philosophy, "Prisoner's Dilemma"**（https://plato.stanford.edu/entries/prisoner-dilemma/）——讀「Nash equilibrium」與「Pareto」對照那幾段，把「穩定 ≠ 最優」講得比本章更哲學、也更嚴謹；囚徒困境來歷（Flood–Dresher、Tucker）也在這裡核實。
- **NobelPrize.org, Economic Sciences 1994 press release**（https://www.nobelprize.org/prizes/economic-sciences/1994/press-release/）——Nash/Harsanyi/Selten 的官方頒獎詞，一手確認「非合作賽局均衡理論」的措辭與三人分工（Nash＝均衡、Selten＝子賽局完美、Harsanyi＝不完全資訊），別憑記憶轉述。
- **MIT OCW 6.254 Game Theory, Lecture 5 "Existence of a Nash equilibrium"**（https://ocw.mit.edu/courses/6-254-game-theory-with-engineering-applications-spring-2010/）——若你想看本章略過的存在性證明：最佳反應對應、Kakutani 不動點如何套上去，一步步寫給工程背景的人，正是「直覺版」之外的那層嚴謹。
- **John Nash, "Non-Cooperative Games", Annals of Mathematics 54 (1951): 286–295**——原始論文本身。納許均衡的定義其實只佔幾頁、出奇地短；讀它感受一下「一個改寫一門學科的定義可以多麼樸素」。
- 單人理性那一面——「一個玩家在不確定下到底該怎麼算最佳選擇」——是姊妹書《在不確定中下注：決策與理性》的地盤（談期望效用、效用不是金額那部分）；本章的「最佳反應」預設了玩家會替自己最佳化，那個最佳化的內核在那本書裡。
