# ch12 — 人類實際怎麼玩：公平、信任與報酬矩陣外的東西

> **本章解決什麼問題**：前面十一章我們都在一個假設底下推演——玩家是「完全理性、只關心自己報酬最大化」的算計機器。這個假設讓數學很漂亮，但它對真人成立嗎？本章把賽局理論的預測拖到實驗室裡，跟真人的行為對照。你會看到一件值得停下來想的事：在最後通牒賽局（ultimatum game）裡，真人會**拒絕白拿的錢**，只因為對方分得太多。這不是因為人笨、也不是因為人不理性——而是因為**報酬矩陣裡的數字，從來就不只有金錢**。本章是 Part IV 的收尾，也是全書中央張力一個罕見的反向案例：當「公平」被寫進個體的效用函數，個體理性與集體理性在這裡不再分裂，反而部分彌合。規範分析（一個人面對不確定該怎麼選的數學）指向《在不確定中下注》，公平偏好的心理機制指向《思考的陷阱》；本章只負責鬆動一個假設——「報酬＝金錢」。

## 從你已知的出發

你設計過計費方案。也許是 API 的分級配額，也許是一個 SaaS 的定價階梯，也許只是內部一個資源排程器的優先順序規則。你在設計的時候，腦子裡跑的模型多半是這樣的：**使用者是理性的報酬最大化機器**——只要這個方案對他的總成本/總效益是正的，他就會用；只要某個 tier 的價格低於他願意付的上限，他就會升級。你的試算表裡，每個格子都是錢。

然後你上線了，發現真人不是這樣玩的。

我看過這種事不只一次。某個方案在純算術上對使用者明明是划算的——換算下來他每個月還省了錢——但他就是不用，甚至跑去用一個貴一點的競品。你做用戶訪談，得到的回答不是「你太貴」，而是某種版本的：「**我覺得這個收費方式不公平。**」也許是因為你對超量部分收的 overage 費率高得像懲罰；也許是因為老用戶發現新用戶拿到更好的價格；也許是因為你某次悄悄調整了免費額度，被當成背叛。在你的試算表裡，這些都不該影響決策——他的錢包數字明明是正的。但他寧可付出真金白銀的代價，也要拒絕一個「他覺得不公平」的方案。

如果你只把使用者建模成「錢包數字最大化」，你會**系統性地預測錯**他們的行為。而且錯的方向很固定：你會高估他們對「對我有利但不公平」的方案的接受度，低估「覺得被佔便宜」這件事的破壞力。這不是邊角案例——這是行為賽局理論（behavioral game theory）過去四十年最穩固的發現之一：**真人會為了公平，付出可量化的代價。**

這一章就是要把這個你在生產環境裡撞過的牆，接到一套精確的語言上。我們會看到，這件事**不需要**用「人是非理性的」來解釋。更準確、也更有用的講法是：人的報酬矩陣裡，除了金錢，還有別的東西——而一旦你把那個東西算進去，他們的行為又變回理性的了。順帶一提，這跟你天天面對的另一件事——**信任**——是同一回事：為什麼有的 client 你願意給更高的配額、有的你寸步不讓？那不是錢算出來的，是一段互動歷史算出來的。本章後半會把信任也拆開。

## 最後通牒賽局：白拿的錢，為什麼有人不要

先把這一章的主角請上台。**最後通牒賽局（ultimatum game）**由 Werner Güth、Rolf Schmittberger、Bernd Schwarze 三人在 **1982 年**首次做成實驗（照 landscape），規則簡單到一句話講完：

```text
最後通牒賽局（ultimatum game）的規則

  有一筆錢，比方說 100 元，要在兩個陌生人之間分。
  
  玩家 A（提議者 proposer）：提出一個分法，「我拿 x、你拿 100−x」。
  玩家 B（回應者 responder）：看到這個提議後，二選一——
      接受(accept) → 就照這個分法拿錢，賽局結束。
      拒絕(reject) → 兩個人都拿 0，賽局結束。
  
  只玩一次。雙方互不相識、之後也不會再見（無重複、無聲譽）。
```

這是一個序貫賽局（見 ch06），所以我們可以用賽局樹來看，也可以用逆向歸納來「解」它。先解出「完全理性自利」的標準預測——這一步很重要，因為它就是等下要被真人打臉的那個預測。

### 標準理論怎麼說：給最小正數，必接受

用 ch06 的逆向歸納，從樹的底部往上倒推。先看回應者 B 的決策節點：

```text
逆向歸納：標準理論（雙方都只關心自己拿到的錢，越多越好）

  情境：A 提議「A 拿 99、B 拿 1」。輪到 B 決定。
  
  B 的兩個選項，比較他自己拿到的錢：
      接受 → B 拿 1 元
      拒絕 → B 拿 0 元
  
  1 > 0，所以理性的 B 會「接受」。← B 對「任何正數」都該接受

  既然 A 預見「B 會接受任何正數」，A 往回推自己的最佳提議：
      給 B 越少、A 自己留越多。
      給 B 「1 元」（最小正數）→ A 拿 99，B 接受。
  
  標準理論的均衡：A 提議 (99, 1)，B 接受。
  ← 這是這個賽局唯一的「子賽局完美均衡」（見 ch06）
```

讓我把這個結論講白：**在「兩個玩家都只關心自己的錢」的假設下，理性的回應者應該接受任何正的提議**——因為拿 1 元也比拿 0 元好。理性的提議者預見到這點，於是會把對方壓到只給 1 元（或最小貨幣單位），自己獨吞剩下的。這就是冷冰冰的子賽局完美均衡：(99, 1)，B 接受。

這個預測有個很乾淨的邏輯：拒絕是**自我傷害**。B 拒絕的那一刻，他不是在懲罰 A——他是在同時懲罰 A 和**他自己**，因為兩人都歸零。一個只關心自己錢包的 B，沒有任何理由這麼做。

### 真人怎麼玩：近半的提議，與「我不爽就同歸於盡」

現在把這個賽局發給真人玩。Güth 等人 1982 年做的、以及之後成百上千次的複現，得到的結果穩定到可以當教科書常數（驗證值照 landscape）：

```text
最後通牒賽局：標準預測 vs 真人行為

                    標準理論預測          真人實際行為（典型）
                    ----------------      --------------------
  提議者給對方       最小正數（≈1%）       常落在 40%~50%
  回應者拒絕門檻     從不拒絕（任何正）     低於 20%~30% 常被拒

  兩個預測都錯，而且錯得很大、很穩定：
    1. 提議者並沒有把對方壓到 1 元，而是主動給了近一半。
    2. 回應者真的會拒絕，寧可兩人都拿 0，也不接受「太小」的一份。
```

第二行是這一章的震撼點，請你停十秒。一個人面前放著「白拿 20 元」和「一毛不拿」兩個選項，而他選了「一毛不拿」。從錢包的角度，這是純粹的損失——他主動燒掉了 20 元，只為了讓對方也拿不到那 80 元。標準理論說沒有理性的人會這麼做。但真人會，而且穩定地會，跨幾百個實驗、不同金額、不同國家都會。

這裡有一個常見的逃生口，我要先把它堵掉：「也許是因為 20 元太少，他懶得在意？」實驗者試過把金額放大——在某些研究裡放大到相當於受試者好幾個月的薪水——拒絕「不公平的大餅」的行為依然存在。錢變多了，「不公平」的刺痛也跟著等比放大。這不是「金額小到無所謂」能解釋的。

那要怎麼解釋？最廉價、也最容易讓你停止思考的答案是：「**人就是不理性嘛。**」我認為這是這一章最該被打掉的一句話——它聽起來像個解釋，其實什麼都沒解釋。它沒告訴你拒絕門檻為什麼穩定落在 20%~30%、沒告訴你提議者為什麼**預見**到這件事而主動給近半、更沒告訴你你該怎麼設計你的計費方案。把「不理性」當答案，等於放棄建模。

## 把公平算進報酬：理性 ≠ 自利

更好的解釋——也是本書唯一要你接受的那一步——是這樣的：**回應者拒絕低提議，是完全理性的；只是他最大化的東西，不只有錢。**

回想 ch01 那張脊椎矩陣，我們花了很大力氣強調「報酬（payoff）越大越好」。但我們從來沒說報酬**只能**是錢。報酬是**效用（utility）**——一個玩家對某個結局的總體偏好程度（效用為什麼不等於金額、效用怎麼來，是《在不確定中下注》談單人理性那本的地盤，本書只借結論）。如果一個人對「我拿 20、對方拿 80」這個結局的真實感受，除了「+20 元的爽」之外，還包含「被當盤子佔便宜的不爽」，那他的效用就不是 +20，而是 +20 再扣掉那份不爽。當不爽夠大，淨效用變成負的，**拒絕（拿 0）反而是效用較高的選項**。

讓我們把這個直覺寫成一個 worked example，補一個具體的「公平項」進回應者的效用函數，然後重新解這個賽局。這是本章的核心計算，我會一步步算、不跳步。

### worked example：把「不公平的不爽」加進效用，重解均衡

我們用一個刻意簡化的公平模型（這是行為經濟學 Fehr–Schmidt 1999「不公平趨避 inequity aversion」模型的玩具版；嚴格模型指向延伸閱讀）。設總額 100 元，A 提議「A 拿 a、B 拿 b」（a + b = 100）。回應者 B 的效用不只是 b，而是：

```text
回應者 B 的效用模型（加進「劣勢方的不爽」）

  U_B(接受) = b − β · max(a − b, 0)
              └─錢─┘   └────── 不公平懲罰項 ──────┘

    b         = B 實際拿到的錢
    a − b     = 「對方比我多拿」的差距（只在 B 吃虧時為正）
    β         = B 對「自己吃虧的不公平」有多在意（β ≥ 0，這裡取 β = 0.5）
    max(·,0)  = 只有當 B 拿得比 A 少時才有這份不爽；B 拿得多時不罰自己

  U_B(拒絕) = 0    ← 兩人都拿 0，沒有錢、也沒有不公平差距（0 − 0）

  決策規則：U_B(接受) ≥ U_B(拒絕) = 0 時，B 才接受。
```

現在用這個模型，逐一檢查幾個提議，看 B 會不會接受。取 β = 0.5：

```text
逐一代入：B 在 β=0.5 下，接受哪些提議？（總額 100）

  提議 (A拿, B拿) = (99, 1)：
     差距 a−b = 99 − 1 = 98
     U_B(接受) = 1 − 0.5×98 = 1 − 49 = −48
     −48 < 0 → 拒絕。  ← 標準理論說「該接受」，但加了公平項後 B 拒絕

  提議 (80, 20)：
     差距 = 80 − 20 = 60
     U_B(接受) = 20 − 0.5×60 = 20 − 30 = −10
     −10 < 0 → 拒絕。  ← 白拿 20 元，他還是拒絕

  提議 (70, 30)：
     差距 = 70 − 30 = 40
     U_B(接受) = 30 − 0.5×40 = 30 − 20 = +10
     +10 > 0 → 接受。

  找「臨界提議」：U_B(接受) = 0 的那一點。
     設 B 拿 b、A 拿 100−b，差距 = (100−b) − b = 100 − 2b
     令  b − 0.5×(100 − 2b) = 0
         b − 50 + b = 0
         2b = 50
         b = 25
     → B 的拒絕門檻 ≈ 25%。給 B 少於 25 元，他拒絕；25 元以上，接受。
```

看那個結果：**25%**。我們隨手取了 β = 0.5，就湊出了一個落在真人實測「20%~30% 拒絕門檻」正中央的數字。這不是巧合在於數字本身（β 是我挑的），而是巧合在於**這個結構天生會產出一個「正的拒絕門檻」**——只要回應者的效用裡有一個正的不公平懲罰項，他就會理性地拒絕足夠低的提議。換句話說，「拒絕白拿的錢」不是理性的反例，而是**在一個更完整的報酬定義下，理性的必然結論**。

現在把提議者也補完，整個均衡就閉合了。A 如果**知道** B 的拒絕門檻是 25%，他用逆向歸納（ch06）會怎麼出？

```text
提議者 A 的逆向歸納（已知 B 在 b < 25 時會拒絕）

  A 想最大化自己拿到的 a = 100 − b，但前提是 B 得接受（否則 A 拿 0）。
  
    給 b = 1   → B 拒絕 → A 拿 0      ✗ 貪心反而歸零
    給 b = 25  → B 剛好接受 → A 拿 75  ✓ 在「會被接受」的前提下 A 拿最多
    給 b = 40  → B 接受 → A 拿 60      ✗ 給多了、A 少賺

  純自利的 A 的最佳提議：給 b = 25（壓在門檻上），自己拿 75。
```

於是模型預測 A 給 25%、B 接受。真人實測 A 常給 40%~50%，比這還慷慨一些——那是因為真人提議者往往**也**有公平偏好（不只是怕被拒，自己也覺得太貪心不舒服），或者對 B 的門檻沒把握、寧可給多一點買保險。但關鍵的定性結論已經到手：**一旦把公平寫進報酬，「給近半、拒絕低提議」從『不理性的異常』變成『理性的均衡』。** 我們沒有放棄理性，我們只是修對了報酬矩陣。

### 自我複核：兩個視角交叉驗算

讓我用兩個角度檢查上面的門檻沒算錯。

```text
複核一（直接代門檻值 b=25）：
   差距 = (100−25) − 25 = 75 − 25 = 50
   U_B(接受) = 25 − 0.5×50 = 25 − 25 = 0  ✓ 恰好臨界，與「拒絕門檻=25%」一致

複核二（極端情形 sanity check）：
   若 β = 0（完全不在意公平）：門檻 b 滿足 b − 0 = 0 → b = 0
   → 退化回標準理論：接受任何正提議。 ✓ β=0 時模型必須還原成「純自利」，它做到了

複核三（單調性 sanity check）：
   β 越大（越在意公平）→ 100−2b 的係數越大 → 解出的門檻 b 越高。
   β=0.5 → 25%；若 β=1 → b − 1×(100−2b)=0 → 3b=100 → b≈33%。
   → 越在意公平、拒絕門檻越高，方向合理。 ✓
```

三個複核都對：臨界點吻合、β=0 退化回標準理論、β 越大門檻越高。這個模型是「直覺版」——它把一個複雜的人類情緒壓成一條線性懲罰，真實的公平感絕不只這麼簡單（Fehr–Schmidt 原模型還有「自己佔便宜也不舒服」的另一項，嚴格版指向延伸閱讀）。但它足以說清楚那個驚嘆點：**拒絕白拿的錢，是理性的——前提是你承認報酬不只有錢。**

## 信任賽局：把「相信對方會還」量化

公平不是報酬矩陣外唯一的東西。另一個你天天在系統裡處理、卻很少叫它名字的東西，是**信任（trust）**。賽局理論有一個專門的實驗來量它——**信任賽局（trust game）**，也叫投資賽局（investment game），由 Berg、Dickhaut、McCabe 三人在 **1995 年**設計。

```text
信任賽局 / 投資賽局（Berg–Dickhaut–McCabe 1995）

  玩家 A（投資者）與玩家 B 一開始「各」拿到一筆本金，比方說各 10 元。
  
  第一步：A 決定把自己本金的多少（0~10 元）「投資」給 B。
          這筆錢在路上會「乘三倍」——A 投 x，B 收到 3x（B 本來那 10 元不動）。
  
  第二步：B 收到 3x 後，從手上（本金 10 + 收到 3x）決定「回送」多少給 A（0 ~ 都行）。
          B 回送多少完全由 B 自己決定，沒有任何強制。
  
  只玩一次、互不相識、無重複、無聲譽、無合約。
```

這個賽局的精妙在於它把「信任」拆成了兩半，而且兩半都可以用錢量出來：

- **A 投出去的錢，就是 A 對 B 的信任量。** 投越多，代表 A 越相信 B 會回報。投 0 代表完全不信。
- **B 回送的錢，就是 B 對信任的回報（reciprocity，互惠）。** B 大可以全部吞掉。

先解標準理論的預測，又是一次逆向歸納：

```text
逆向歸納：標準理論（雙方純自利）

  最後一步先看 B：B 手上有 3x，回送任何一塊錢都是純損失。
     純自利的 B → 回送 0。  ← B 把 3x 全吞了

  A 預見「B 會吞光」，往回推：
     A 投出的任何 x，回收都是 0，等於白白損失 x。
     純自利的 A → 投 0。  ← 一毛都不投

  標準理論均衡：A 投 0、B 沒機會回送。雙方各留 10 元本金，合作完全沒發生。
```

注意這個結局有多慘：A 投 0、B 收 0，兩人各留著一開始的 10 元，**社會總財富完全沒有增長**（總和維持 20 元）。可是這個賽局藏著一個巨大的合作紅利——那筆「乘三倍」。如果 A 敢投全部 10 元，B 收到 30 元（加上本來的 10，手上 40），只要 B 願意回送 15 元，**兩個人都比一開始更有錢**（A 從 10 變 15、B 從 10 變 25，總和從 20 漲到 40）。標準理論的悲劇在於：這個對雙方都好的結局，**靠純自利達不到**，因為它要求 B 在最後一步「明明可以吞光、卻選擇回報」。

這結構你應該覺得眼熟——它就是脊椎困境（ch01）換了件衣服。**個體理性（B 吞光、A 不投）導致集體次優（合作紅利蒸發）**，正是全書中央張力。而真人怎麼玩？Berg 等人的原始實驗裡，A 平均投出超過一半的本金，B 也確實回送了相當的金額——三十二組裡有三十組違背了純自利預測。**真人會信任，也會回報信任**，即使在一次性、匿名、無任何強制的情境下。

這跟你的工程直覺是接得上的。你給一個有良好歷史的 client 更高的配額，不是因為合約逼你，而是因為你「信任」它不會濫用——你在投資一段關係。而當它真的守規矩，你更願意放寬。當它某次突然爆量、像在佔便宜，你的信任會瞬間縮回去。這整套動態，就是信任賽局在重複版本裡的樣子（重複如何把合作救回來，是 ch07/ch08 的主場；本章只標出「信任」這個量在一次性賽局裡就已經違背純自利預測了）。

## 中央張力的反向案例：當集體理性被寫進個人效用

到這裡，值得把這一章放回全書的脊椎來看，因為它在中央張力上做了一件前面所有章都沒做的事。

全書到目前為止，「個體理性 vs 集體理性」一直是一道**裂縫**：每個人最佳化自己（個體理性），結果集體更糟。囚徒困境裡背叛佔優、共享資源被各自耗盡、信任賽局裡 B 吞光——個體的算盤和集體的好處朝相反方向拉。我們找到的救火工具（重複、聲譽、機制設計）都是**從外部**把集體理性塞回來：用未來懲罰、用名聲、用改寫的規則，逼個體的算盤指向集體好的方向。

公平偏好是一個**不一樣**的故事，我認為這是它在全書裡最值得玩味的位置。在最後通牒賽局和信任賽局裡，集體理性不是被外力塞回來的——它被**內化進了個體的效用函數**。真人之所以拒絕「不公平但正報酬」，正是因為他把「公平、互惠」這些原本屬於集體層面的價值，算進了自己的個人偏好。一旦「報酬」不只有錢、集體的好被寫進個體最佳化的目標，**個體理性與集體理性就在這裡部分彌合，而不是分裂**。B 拒絕低提議是個體理性的（最大化他含公平項的效用），同時也維護了一種集體規範（不公平的提議不該得逞）。兩者在這裡指向同一邊。

```text
中央張力：脊椎困境 vs 本章的反向案例

  脊椎困境（ch01）：       公平偏好（ch12）：
    個體理性 ──┐             個體理性 ──┐
               ↓ 分裂                   ↓ 彌合
    集體理性 ──┘             集體理性 ──┘
    （各自最佳化 → 集體更糟）  （集體價值被寫進個體效用 → 兩者同向）

  差別在「集體的好」放在哪裡：
    困境裡它在報酬矩陣「之外」，要靠外力（重複/聲譽/機制）拉回來。
    公平裡它被搬進報酬矩陣「之內」，個體最佳化時就自動顧到它。
```

這給了「合作如何可能」一個全書其他地方沒給的答案。前面的答案都是條件式的——合作要靠未來夠長（ch07）、要靠 δ 夠大（ch08）、要靠設計對的機制（ch13）。公平偏好提供的答案是：**如果合作的價值能被內化進個體的偏好，合作就不必只靠外力**。一個真心在意公平與互惠的群體，不需要每筆交易都靠合約和懲罰撐著。這既是好消息（人類確實演化出了這種內化，ch09 的演化視角能談為什麼），也是一個誠實的提醒——它**不是普世常數**，下一節就講。

## 一個誠實的但書：公平不是普世常數

我必須在這裡 hedge 一個容易被過度推論的點，否則這一章會變成廉價的「人性本善」宣言。

「真人會為公平付代價」是穩固的；但「拒絕門檻是 20%~30%、提議者給近半」**不是寫死的人類常數**。跨文化實驗顯示，這些數字會隨社會結構大幅變動。最有名的是 Henrich 等人在多個小規模社會做的研究：祕魯亞馬遜的 Machiguenga（馬奇根加）族受試者，平均只提議給對方約 26%，而且回應者幾乎接受任何低於 20% 的提議——他們事後解釋是「我幹嘛拒絕白來的錢」。同一個賽局，在 UCLA 學生身上提議者平均給 48%、沒人提議低於 20%。**同樣的報酬矩陣，不同文化玩出不同的均衡。**

這告訴我們兩件事，都很重要：

1. 那個「公平項」係數 β——一個人對不公平有多在意——**是文化與社會結構塑造的，不是物理常數**。一個高度仰賴匿名市場交易與互惠合作的社會，會養出較高的 β；一個交易多在親屬間、不太跟陌生人做一次性買賣的社會，β 較低。報酬矩陣外的東西，本身是被環境寫進去的。
2. 對你的工程實踐而言：**你不能假設你的使用者的「公平感」跟你的一樣。** 你覺得天經地義的計費規則，在另一個市場、另一個用戶群裡可能被讀成羞辱。公平是真的、但它的校準值因人因地而異——這正是為什麼這一章叫「人類實際怎麼玩」，而不是「人類普世這樣玩」。

到此這一章的定位就清楚了：它不是要你把「公平」當成新的鐵律去算，而是要你**鬆動一個假設**——「報酬等於金錢」。真人的報酬矩陣裡裝著公平、互惠、信任、被尊重；這些東西可以被量化（用一個效用項）、可以被預測（在同一文化內相當穩定）、也會被你設計的規則觸發或激怒。賽局理論沒有因此失效——恰恰相反，它告訴你**該往報酬矩陣裡多放幾欄**。

## 直覺的陷阱

這一章專治一個流傳極廣的誤解，外加幾個它的近親。

| # | 錯誤直覺 | 它在哪一步把你帶溝裡 | 正確版 |
|---|---|---|---|
| 1 | 「人會拒絕白拿的錢 ⇒ 人是不理性的」 | 把「理性」偷偷等同於「只關心錢」。一旦回應者效用裡有公平項，拒絕低提議就是**最大化效用**的理性選擇——不理性的是你的報酬定義，不是那個人 | 理性 ≠ 自利。拒絕「不公平但正報酬」是**理性地最大化一個含公平項的效用**。錯的是「報酬＝金錢」這個假設，不是人 |
| 2 | 「公平偏好證明賽局理論不適用於真人」 | 把「標準理論的某個假設錯了」誤讀成「整套工具錯了」。其實只要把效用修對，納許均衡、逆向歸納這些工具照樣運轉 | 賽局理論沒被推翻，被修的是**輸入**（報酬要含公平/互惠），不是**方法**。修對報酬後行為又變回均衡 |
| 3 | 「人性本善，所以合作是預設」 | 把「真人有公平偏好」過度推成「合作不需要條件」。但公平偏好的強度有文化變異、也擋不住大群體的搭便車 | 公平偏好讓合作**更容易**，不是**保證**。它是「報酬矩陣外的一欄」，不是廢除困境的萬靈丹（共有財悲劇見 ch16 仍會發生） |
| 4 | 「拒絕門檻 20–30% 是人類常數」 | 把幾個西方大學樣本當全人類。Machiguenga 的 26% 提議、近乎全接受，直接反例 | 公平**偏好的存在**穩固，**校準值（β、門檻）有文化變異**（Henrich 跨文化研究）。別把實驗室數字當物理常數（跨文化變異，2026-06 仍是研究中的活問題） |
| 5 | 「提議者給近半是出於善良」 | 把策略性的計算當成道德。提議者給多，很大一部分是**怕被拒**（逆向歸納預見 B 的門檻）的理性反應，不必然是利他 | 給近半混合了兩個動機：自己的公平偏好 + 對「B 會拒絕低提議」的理性預期。後者是純策略，不需要善良 |

自我察覺的徵兆：如果你發現自己在用「他就是不理性」來解釋某個使用者行為，那通常是個訊號——你的報酬模型漏了一欄。停下來問：**這個人除了錢，還在最大化什麼？**（被尊重？公平？掌控感？不被當盤子？）把那一欄補進去，他的行為十之八九會變回理性的。「不理性」是建模偷懶的藉口，不是答案。

## 紙上推演

### 推演題一（★★，20 分鐘）：把「拒絕」翻譯成「公平值多少錢」

某人在最後通牒賽局（總額 100 元）中，**拒絕了「對方拿 90、他拿 10」的提議**，但**接受了「對方拿 75、他拿 25」的提議**。用本章的效用模型 U_B(接受) = b − β·max(a−b, 0)，反推這個人的公平係數 β 落在什麼範圍。換句話說：他心裡「被佔便宜的不爽」每一塊錢差距值多少？

提示：拒絕 (90,10) 代表在 b=10 時 U_B(接受) < 0；接受 (75,25) 代表在 b=25 時 U_B(接受) ≥ 0。兩個不等式夾出 β。

### 推演解答

```text
列兩個不等式（差距 a−b = (100−b) − b = 100 − 2b）：

  拒絕 (90,10)：b=10，差距 = 100 − 20 = 80
     U_B = 10 − β·80 < 0
     → β > 10/80 = 0.125

  接受 (75,25)：b=25，差距 = 100 − 50 = 50
     U_B = 25 − β·50 ≥ 0
     → β ≤ 25/50 = 0.5

  夾出：0.125 < β ≤ 0.5
```

**解讀**：這個人的公平係數在 0.125 到 0.5 之間。白話講——「對方每比我多拿 1 元，我的痛苦折算成 0.125 到 0.5 元之間」。他不是無限在意公平（β 不到 1，否則連 (75,25) 都會拒），也不是完全不在意（β > 0.125，否則 10 元他就收了）。**這就是把一個道德情緒（被佔便宜的憤怒）翻譯成一個可量化的報酬欄位**——這正是本章要你帶走的能力。

常見錯路：① 忘記差距是 a−b 而誤用 a（對方拿的絕對值），算出的門檻會錯。② 把「接受」的不等式寫成嚴格 < 0，邊界搞反——接受是 U_B ≥ 0。③ 只用一個提議就想定出 β 的單一值；單筆交易只能給出一邊的界，要**兩筆**（一拒一接）才能夾出區間。

### 推演題二（★★，15 分鐘）：信任賽局裡，純自利為什麼讓合作蒸發

信任賽局：A 與 B 各有本金 10 元，A 投資額乘三倍給 B，B 自行決定回送多少。假設雙方都純自利（只關心自己最後拿到的錢、無重複、無聲譽）。

(a) 用逆向歸納證明：純自利下唯一的結局是「A 投 0」。
(b) 指出這個結局蒸發掉了多少「合作紅利」——也就是，若 A 投全部、B 回送一半，雙方各拿多少，比起 (a) 的結局好多少。

### 推演解答

```text
(a) 逆向歸納（從最後一步倒推）：
    最後一步是 B：B 手上有「本金 10 + 收到 3x」，回送任何金額都是純損失（他自己少拿）。
       純自利 B → 回送 0。  對任何 x 都成立。
    A 預見「不管投多少，回收都是 0」：
       A 投 x → 自己剩 (10 − x)，回收 0 → 最終 (10 − x)。
       要最大化 (10 − x) → 投 x = 0。
    唯一均衡：A 投 0、B 無動作。雙方各留本金（A:10, B:10），合作零發生。

(b) 合作紅利的蒸發：
    若 A 投全部 10 → B 收到 3×10 = 30（加上本金 10，手上 40）。
       B 回送 15 → A 最終拿 0 + 15 = 15、B 最終拿 40 − 15 = 25。
       合計 40。
    對比 (a) 的純自利結局：A 拿 10、B 拿 10，合計 20。
    蒸發掉的合作紅利 = 40 − 20 = 20 元（那筆「乘三倍」放大出來、卻沒人敢碰的錢）。
    A 個人：15 vs 10，多賺 5；B 個人：25 vs 10，多賺 15；總體：40 vs 20，多出 20。
```

**解讀**：這就是脊椎困境換衣服——個體理性（B 吞光所以 A 不投）導致集體次優（20 元合作紅利消失）。真人之所以投得出去、也回送，靠的正是「報酬矩陣外的東西」：信任與互惠被算進了效用。

常見錯路：① 以為 B「應該」回送以維持名聲——本題明設**無重複無聲譽**，純自利 B 在最後一步沒有任何理由回送，名聲動機要等 ch07/ch08 重複版才登場。② 把合作紅利算成 40（合作結局的財富總額）而非「相對純自利結局（20）多出的 20」；紅利是**淨增量**，不是總額。

### 推演題三（★★★，20 分鐘）：口頭題——「人會拒絕白拿的錢」為什麼不代表人不理性

不准用紙筆。對著一個只聽過「賽局理論假設人都自私理性」的工程師同事，用 90 秒講清楚三件事：①最後通牒賽局裡標準理論預測什麼、真人做了什麼；②為什麼「拒絕白拿的錢」可以是**理性**的；③這跟「人是不理性的」這個廉價答案差在哪、為什麼後者其實沒解釋任何東西。

### 推演解答

一個合格的口頭回答應該長這樣（自己講完對照）：

> 「標準理論說：給你 1 元也比 0 元好，所以你該接受任何正提議，提議者於是把你壓到 1 元。但真人面對『對方拿 90、你拿 10』，會直接拒絕，寧可兩人都歸零。看起來像是『人不理性、放著錢不拿』——但這個解釋是假的。真正發生的是：他最大化的不只是錢包，還包括『不被當盤子佔便宜』這份感受。一旦你把那份不爽算進他的報酬，拒絕 10 元就是**理性**的——他用 10 元換掉了那份羞辱，淨值對他是划算的。所以理性沒被推翻，被推翻的是『報酬等於金錢』這個假設。而『人不理性』之所以是廉價答案，是因為它什麼都預測不了——它說不出拒絕門檻為什麼穩定落在 20–30%、提議者為什麼**預見**到而主動給近半。把公平算進報酬，這些全都能算出來、能設計。」

**判分點**：有沒有把「理性」和「自利」明確拆開（②的核心）；有沒有指出錯的是報酬定義而非方法；有沒有講出「不理性」是建模偷懶（③，不能只說它錯、要說它為什麼沒解釋力——預測不出門檻數字）。漏掉第③點是最常見的不及格——很多人能講前兩點，卻把「人不理性」當成一個合法的競爭解釋，沒看穿它根本不是解釋。

## 自我檢核

口頭自答，講得出來才算過關。優先答「為什麼」。

1. 最後通牒賽局的標準理論預測是什麼？為什麼逆向歸納會推出「提議者只給最小正數」？
2. 真人拒絕「不公平但正報酬」的提議——為什麼這**不**等於「人是不理性的」？「理性」和「自利」差在哪？
3. 在本章的效用模型 U_B = b − β·max(a−b,0) 裡，β 代表什麼？把 β 調到 0，模型會退化成什麼？這個退化為什麼是模型「正確」的證據？
4. 信任賽局裡，純自利的標準理論為什麼預測「A 一毛都不投」？這個結局蒸發掉了什麼？它和脊椎囚徒困境的中央張力是同一回事嗎，怎麼說？
5. 為什麼說公平偏好是全書中央張力的「反向案例」？「集體理性被寫進個體效用」跟前面章節（重複、機制設計）把集體理性塞回來的方式，差在哪？
6. Machiguenga 的跨文化結果告訴我們公平偏好的哪個部分是穩固的、哪個部分有變異？為什麼這對你設計跨市場的計費方案是個警告？
7. 為什麼提議者「給近半」不能簡單歸因於善良？裡面混了哪個純策略性的動機？
8. 如果你發現自己在用「使用者就是不理性」解釋某個行為，本章建議你改問什麼問題？

## 延伸閱讀

- **Güth, Schmittberger & Schwarze (1982), "An experimental analysis of ultimatum bargaining"**（*Journal of Economic Behavior & Organization*）——最後通牒賽局的原始論文。想看「近半提議、低提議被拒」這個結果第一次被記錄下來的樣子，讀它的結果表。
- **Berg, Dickhaut & McCabe (1995), "Trust, Reciprocity, and Social History"**（*Games and Economic Behavior*）——信任/投資賽局的原始設計。讀它怎麼用「乘三倍 + 自由回送」把抽象的「信任」變成可以拿尺量的錢。
- **Henrich et al. (2001/2005), 跨文化最後通牒賽局研究**（如 *Behavioral and Brain Sciences* 的 "Economic man in cross-cultural perspective"）——十五個小規模社會的對照。讀它你會徹底戒掉「實驗室數字是人類常數」的習慣；Machiguenga 那一段尤其打臉。
- **Fehr & Schmidt (1999), "A Theory of Fairness, Competition, and Cooperation"**（*Quarterly Journal of Economics*）——本章玩具效用模型的嚴格版（不公平趨避）。想看「把公平寫進效用函數」怎麼被正式做成可預測多種賽局的模型，讀這篇；它是行為經濟學被引用最多的文獻之一。
- **Colin Camerer, *Behavioral Game Theory*（2003）**——把「真人怎麼玩賽局」整本講透的標準參考書。當你想知道公平、信任、有限理性怎麼系統性地修正標準預測，這是入口。
- 跨書：一個人面對不確定該怎麼選的**規範基準**（效用為什麼不等於金額、期望效用怎麼來），見《在不確定中下注》談單人理性與期望效用那本；公平偏好背後的**心理機制**（為什麼憤怒、為什麼互惠），見《思考的陷阱》談認知與情緒驅動推理那本。本章只負責把「報酬＝金錢」這個假設鬆開，不深入這兩塊。
