# ch06 — 賽局樹：序貫、承諾與可信威脅

> **本章解決什麼問題**：前五章的賽局都是「同時出手」——雙方蒙著眼睛各自下注，用矩陣表示就夠了。但現實裡大多數互動有先後：誰先動、誰看著誰反應。一旦有了時間軸，矩陣就不夠用了，我們得換一種武器——賽局樹（game tree）。本章要回答三個被「同時賽局」藏起來的問題：先動是優勢還是劣勢？一個威脅在什麼條件下是「可信」的、什麼時候只是空話？以及一個最反直覺的招式——**主動綁住自己的手，反而讓你更強**。這是 Part II 的收尾（把純策略不夠用的故事補完），也是後面 Part III「重複賽局」的地基（重複就是序貫的特例）。

## 從你已知的出發

你寫過的系統裡，「順序」幾乎決定一切。

想想 TLS 握手。它不是雙方同時把一堆參數丟出來碰運氣，而是嚴格的序貫：client 先送 `ClientHello`（我支援這些 cipher），server 看到之後才回 `ServerHello`（那我們就用這個）。每一步都建立在「我看到了你上一步做什麼」之上。如果你硬把它畫成一張同時出手的報酬矩陣，你會丟掉這個賽局**最重要的資訊**——誰看著誰反應。兩階段提交（2PC）也是：coordinator 先問「能 commit 嗎」，每個 participant 投票，coordinator 再根據投票結果決定廣播 commit 還是 abort。順序錯了，整個協定就垮了。

再想一個更有「賽局味」的例子：circuit breaker。你的服務 A 對下游 B 設了熔斷器，邏輯是「B 你要是再給我吐 500，我就斷開連線，半開狀態探測通過才放你回來」。這句話本質上是一個**威脅**：「你再失敗，我就懲罰你（斷你）。」問題來了——這個威脅**可信**嗎？如果斷開 B 會讓 A 自己也直接掛掉（B 是 A 的關鍵依賴、沒有降級路徑），那這個威脅其實是空的：真到了那一步，A 為了自保根本不敢斷。B（如果它會算）就會看穿這點，繼續擺爛。反過來，如果 A 有 fallback、斷開 B 對自己幾乎無痛，那這個威脅就是**真的**——B 最好乖一點。

「同樣一句威脅，可不可信取決於『真執行起來，威脅方自己會不會更慘』」——這就是本章的核心，賽局理論管它叫**可信威脅（credible threat）vs 空洞威脅（empty threat）**。納許均衡分不出這兩者，但我們接下來要造的工具——子賽局完美均衡——可以。

還有一個你天天在用、卻可能沒意識到它是「賽局招式」的東西：**deprecation 公告**。你要逼大家從舊 API 遷到新 API，最有效的一招不是好言相勸，而是**公開宣布「舊 API 在 X 月 Y 日關閉」並且把日期釘死、寫進文件、發 email**。為什麼公開、為什麼釘死日期？因為這是在**燒掉自己的退路**——一旦公開承諾，你就很難再偷偷延期（延期的信譽成本太高了），於是大家知道「他這次是來真的」，遷移才會真的發生。你把「我可能會心軟延期」這個選項**主動砍掉**，反而讓你更強。這正是本章最反直覺的一頁：**承諾的價值來自放棄選擇。**

## 從矩陣到樹：當賽局有了時間軸

前面幾章的報酬矩陣有個隱藏假設：雙方**同時**選策略，誰都不知道對方選了什麼。剪刀石頭布是這樣，囚徒困境的標準版也是這樣（兩名囚犯被分開偵訊、同時決定招不招）。這類賽局叫**同時賽局（simultaneous game）**，用矩陣（賽局理論術語叫「標準式 normal form」，也譯策略式）表示剛剛好。

但很多賽局是**序貫的（sequential）**：一方先動，另一方看到之後再動。這時候矩陣會丟掉關鍵資訊，我們要改用**展開式（extensive form）**，也就是**賽局樹**。

賽局樹的三個零件，你看一眼就懂：

```text
賽局樹的解剖（讀法：從上往下、從根到葉）

              [ 玩家1 ]            ← 決策節點（換誰出手）
              /        \
         選 A /          \ 選 B    ← 分支（一個可選的行動）
            /              \
       [ 玩家2 ]         [ 玩家2 ]  ← 又一層決策節點
        /    \            /    \
       L      R          L      R
      / \    / \        / \    / \
   (報酬葉) ...                      ← 葉節點：(玩家1報酬, 玩家2報酬)

  節點 = 「輪到誰決定」
  分支 = 「他可以做什麼」
  葉   = 「走到這條路徑，雙方各拿多少」
```

矩陣的每一格，現在變成樹的一片葉子（一條從根到底的完整路徑）。差別在於：樹**明確畫出了順序**——玩家 2 在自己的節點上，是**已經看到玩家 1 選了什麼**才動的。這個「看得到歷史」正是同時賽局沒有、序貫賽局才有的東西，也是「可信威脅」這整套概念能存在的前提。

> 小提醒：如果序貫賽局裡後手**看不到**前手做了什麼（資訊隱藏），我們會用「資訊集（information set）」把後手分不清的節點圈起來——那其實就退化回同時賽局了。本章先處理**完全資訊（perfect information）**的序貫賽局：每一步，輪到的人都看得見之前發生的全部。資訊不對稱的序貫賽局（訊號賽局）留到 ch11。

## 逆向歸納：從結局倒著想

序貫賽局怎麼解？答案是一個你其實已經會的思路，只是賽局理論給它取了名字：**逆向歸納（backward induction）**。

直覺一句話：**要知道現在該怎麼選，先想清楚『如果走到最後，理性的人會怎麼收尾』，然後一路倒推回來。** 你下棋算殘局、你做技術決策時問「假設三個月後這個方案真的上線了，那時候我會後悔嗎」，用的都是這個。

機制是這樣（這是「工程師的嚴謹」級——每一步你都能口頭說出理由）：

1. 找到樹**最底層**的決策節點（最後一個出手的人）。
2. 在那個節點，輪到的玩家會選讓**他自己**報酬最大的分支。把不會被選的分支砍掉，這個節點就「塌縮」成一個確定的報酬。
3. 往上一層。現在上一層的玩家做選擇時，**已經知道**下面的人會怎麼收尾（因為你剛算完了），所以他也能挑對自己最好的。
4. 一路倒推到根。沿途留下的那條路徑，就是這個賽局的解。

逆向歸納解出來的東西，有個正式名字叫**子賽局完美均衡（subgame-perfect equilibrium, SPE）**，這個精煉概念由 Reinhard Selten（澤爾騰）在 1965 年提出，他也因此與 Nash、Harsanyi 共獲 1994 年諾貝爾經濟學獎（「諾貝爾經濟學獎」嚴格說是瑞典央行紀念諾貝爾的經濟學獎，1969 起頒，並非諾貝爾遺囑裡的五個獎之一——但俗稱沿用）。SPE 的精神是：**一個均衡不只要在整局看起來穩，還要在『每一個子賽局』裡都站得住腳**——包括那些「實際上不會走到」的分支。為什麼這個要求這麼重要？因為它正是揪出「空洞威脅」的照妖鏡。我們用一個 worked example 把它算清楚。

## Worked example：市場進入賽局——一個算出來的「空話」

場景（這是 Selten 連鎖店悖論的單回合精簡版；連鎖店悖論出自 Selten 1978，而子賽局完美這個工具則是他 1965 年那篇就奠下的）：一家**新進者（Entrant）**考慮要不要進入某個市場，那裡已經有一個**在位者（Incumbent）**。賽局兩步走：

- 第一步：新進者選 **進入（In）** 或 **不進入（Out）**。
- 第二步：**如果**新進者進入了，在位者選 **打價格戰（Fight）** 或 **容忍共享市場（Accommodate）**。

報酬設定（數字＝獲益點數，越大越好；(新進者, 在位者)）：

```text
市場進入賽局（賽局樹）

                    [ 新進者 ]
                   /          \
            In    /            \   Out
                 /              \
          [ 在位者 ]          (0, 10)   ← 新進者不進場，在位者獨佔大餅
            /      \
     Fight /        \ Accommodate
          /          \
      (-1, 1)       (2, 5)
   兩敗俱傷       共享市場
   打價格戰       在位者讓一點、
   雙方都流血      但比打仗好

  報酬讀法：(新進者報酬, 在位者報酬)
```

先用逆向歸納解，再看一個很多人會掉進去的陷阱。

**第一步：解最底層（在位者的節點）。**

只有在「新進者已經進場」時，在位者才需要動。比較他的兩個選項：

```text
在位者面對「已經進場的新進者」，他自己的報酬：
    Fight（打）       → 在位者得 1
    Accommodate（容忍）→ 在位者得 5

    5 > 1  ⇒ 理性的在位者選 Accommodate（容忍）
```

關鍵：**真到了那一步，打價格戰對在位者自己也是虧的**（1 < 5——打仗他也流血）。所以「我會打到底」這句話，**執行起來在位者自己更慘**。把 Fight 這條分支砍掉，在位者的節點塌縮成報酬 **(2, 5)**。

**第二步：往上一層（新進者的節點）。**

新進者現在知道：「如果我進場，理性的在位者會容忍，我拿 2。如果我不進場，我拿 0。」

```text
新進者的比較（已知在位者會容忍）：
    In  → 走到 (2, 5)，新進者得 2
    Out → (0, 10)，    新進者得 0

    2 > 0  ⇒ 新進者選 In（進入）
```

**子賽局完美均衡＝（新進者：進入；在位者：容忍），結局 (2, 5)。**

**自我複核（交叉驗算）。** 倒著走一遍：在位者節點 5>1 選容忍 ✓；新進者預期拿 2，2>0 選進入 ✓；路徑 In→Accommodate→(2,5)，沒有任何一方在自己出手的那一步能改善（新進者改 Out 從 2 掉到 0；在位者改 Fight 從 5 掉到 1）。穩。

**現在看陷阱在哪。** 在位者其實**很想**威脅新進者：「你敢進來，我就打到你死（Fight），我們一起變 (-1, 1)，你別進來！」如果新進者**信了**這個威脅，他會算「進場會被打到 -1，不如 Out 拿 0」，於是乖乖不進場，結局 (0, 10)——在位者獨佔大餅。

這個威脅**有用嗎**？逆向歸納說：**沒用，因為它不可信。** 真等新進者進來了，在位者站在自己的節點上，面對「打 → 我得 1」vs「容忍 → 我得 5」，他**不會**真的打——打仗傷的是他自己。新進者只要會算，就知道這個威脅是**空話（empty threat）**，照樣進場。

這就是子賽局完美均衡比納許均衡強的地方。事實上，「在位者威脅打、新進者因此不進場」**這個策略組合在納許均衡的意義下是成立的**——給定新進者選 Out，在位者「我會打」的宣言沒被執行（因為沒人進場），所以新進者沒有單方面改善的動機（進場會被「打」），形式上沒人想偏離。**納許均衡分不出這個威脅是真是假**，因為它只檢查「均衡路徑上」沒人想偏離，不檢查那些「沒走到的分支」上玩家會不會說話不算話。子賽局完美均衡多要求一件事：**在每一個子賽局裡（包括「新進者已進場」這個實際上不會走到的子賽局），玩家的計畫都得是當下最佳反應**——一檢查，在位者的「打」立刻露餡（他到那會選容忍）。所以 (Out, 威脅打) 是納許均衡、卻**不是**子賽局完美均衡，被剔除了。

**這就是「可信威脅 vs 空洞威脅」的精確定義**：一個威脅可信，若且唯若「真要執行時，威脅方確實會願意執行它」——也就是執行威脅在那個子賽局裡是威脅方的最佳反應。納許均衡放過空話，子賽局完美均衡抓住它。

## 承諾：主動砍掉自己的選項，反而更強

上面的故事裡，在位者輸在哪？輸在他**有選擇**。正因為真到了那一步他**可以**選容忍，他「我會打」的威脅才是空的。

那如果在位者能**讓自己沒得選**呢？

假設在位者進場前先做一件事：簽一份對外公開的合約——「只要有人進入我的市場，我**自動**啟動價格戰，違約賠 100」。現在重畫賽局：在位者的「容忍」選項報酬被那 100 的違約金壓垮了，他真的只剩「打」這條路。新進者一看，「進場 → 一定被打 → -1 < 0」，於是**不進場**，結局回到 (0, 10)。

在位者透過**綁住自己的手**，把原本的空洞威脅變成了可信威脅，結果獨佔市場。**他放棄了一個選項（容忍），反而過得更好。**

這就是**承諾（commitment）**的弔詭，也是 Thomas Schelling（謝林）在《The Strategy of Conflict》（衝突的策略，1960）裡反覆敲打的洞見：**在策略互動裡，「選項更少」可以是「力量更大」。** 他最有名的比喻是膽小鬼賽局（chicken）——兩台車對撞，誰先轉向誰輸。Schelling 說，最強的玩法是**當著對手的面，把方向盤拆下來丟出窗外**：現在你**無法**轉向了，對手看到你已經沒有退路，只好他先轉。你用「放棄轉向的能力」買到了勝利。

歷史與生活裡到處是這招：將軍**燒掉自己身後的橋**讓士兵無法撤退（背水一戰）；國家把核反擊設計成**自動觸發**而非「總統再決定一次」（讓報復不依賴臨場的勇氣）。共同點都是：**把「我可能會軟」這個選項實體地砍掉，讓威脅變得不可逆，於是可信。**

回到你的系統。前面說的 deprecation 公告就是工程版的「丟掉方向盤」：

```text
威脅可信度，取決於「執行起來威脅方自己慘不慘」

  空洞威脅                         可信威脅（靠承諾鞏固）
  ─────────────                    ──────────────────────
  「舊 API 之後會關，               「舊 API 在 6/30 關閉」——
   再看看吧」                        公開、釘死日期、寫進文件
   ↓                                ↓
  延期對你幾乎無痛                  延期的信譽成本被你自己拉到很高
   ↓                                （改口＝公開打臉、文件全錯）
  大家賭你會延 → 沒人遷移            ↓
                                   大家知道你軟不了 → 真的去遷移

  circuit breaker 也一樣：
  「我會斷你」可信 ⇔ 斷開對 A 自己幾乎無痛（有 fallback／降級路徑）
  如果斷開 B 會讓 A 一起掛 → 威脅是空的，B 看穿就繼續擺爛
```

設計可信威脅，工程上就是**讓「執行懲罰」對你自己足夠便宜**（給自己留 fallback、把承諾公開到難反悔）。這跟你在做容量規劃、SLA 協商時的直覺是同一回事：嘴上說的籌碼不值錢，能被驗證、不可逆的承諾才算數。

## 把脊椎張力搬上時間軸：蜈蚣賽局

到這裡你可能覺得，序貫賽局好像跟全書脊椎（那張囚徒困境矩陣）沒什麼關係。其實關係深得很——逆向歸納能造出**個體理性導致集體次優**的最純粹版本，叫**蜈蚣賽局（centipede game）**（Rosenthal，1981）。

先回憶**ch01 那張矩陣**：T,R,P,S = 5,3,1,0，背叛是嚴格優勢策略，雙方各自最佳化 → (背叛,背叛)=(1,1)，明明 (合作,合作)=(3,3) 對雙方都更好。中央張力：**每個人都最佳化自己，結果集體更糟。** 在同時賽局裡，這個裂縫一次爆發。蜈蚣賽局把同一道裂縫**攤平在時間軸上**，讓你看著它一格一格地裂開。

規則：桌上有一筆會長大的錢。兩個玩家輪流決定「**拿（Take）**」還是「**讓（Pass）**」。只要有人選拿，賽局立刻結束、按當下分錢；只要兩人都讓，桌上的錢就翻倍、輪到下一個人。**錢越往後越多，但每一步輪到的人，「現在就拿」永遠比「讓給對方、然後被對方拿走」多一點點。**

這個賽局有個招牌弔詭，數學細節指向延伸閱讀，這裡給一組**倒推方向絕對乾淨、你能逐步驗算**的報酬（四步版，桌上的餅越往後越大；報酬 (玩家1, 玩家2)）：

```text
蜈蚣賽局（四步版；每個節點輪到的人選 Take 結束 或 Pass 交棒）

  玩家1      玩家2      玩家1      玩家2
   ●─Pass──→ ●─Pass──→ ●─Pass──→ ●─Pass──→ (6, 9)  ← 都讓到底
   │         │         │         │
  Take      Take      Take      Take
   │         │         │         │
   ▼         ▼         ▼         ▼
 (4, 1)    (3, 6)    (8, 5)    (7, 10)

  設計性質（每一步，輪到的人「現在 Take」嚴格優於「Pass 後被對方 Take」）：
    玩家1 第1步 Take 得 4 > 若 Pass、玩家2 第2步 Take 時自己只得 3
    玩家2 第2步 Take 得 6 > 若 Pass、玩家1 第3步 Take 時自己只得 5
    玩家1 第3步 Take 得 8 > 若 Pass、玩家2 第4步 Take 時自己只得 7
  桌上的餅越往後越大（每停一步的總和：第1步 5、第2步 9、第3步 13、
  都讓到底 15、第4步 17）——越晚停，餅越大。關鍵的悲劇在於：逆向歸納
  會讓玩家1 在「餅最小」的第1步就 Take（總和 5、結局 (4,1)），而只要
  願意往後讓，(6,9) 對雙方都嚴格更好（6>4、9>1）。
```

逆向歸納從**最後一個節點**倒著解，注意這組數字讓每一步的結論都無懈可擊：

```text
逆向歸納（從右往左，每步都驗一次）：
  第4步（玩家2）： Take 得 10  vs  Pass 到底得 9   → 10>9，玩家2 選 Take。
  第3步（玩家1）： Take 得 8   vs  Pass（玩家2 接著 Take，自己拿 7） → 8>7，選 Take。
  第2步（玩家2）： Take 得 6   vs  Pass（玩家1 接著 Take，自己拿 5） → 6>5，選 Take。
  第1步（玩家1）： Take 得 4   vs  Pass（玩家2 接著 Take，自己拿 3） → 4>3，選 Take。

  結果：玩家1 在第一個節點就 Take，賽局秒結束、結局 (4,1)。
        明明一路 Pass 到底 (6,9) 對兩人都好得多。
```

**看出來了嗎？這跟 ch01 那張矩陣是同一道裂縫。** 囚徒困境裡，兩個理性玩家各自選優勢策略 → (1,1)，集體最優 (3,3) 達不到。蜈蚣賽局裡，每一步的理性選擇（現在 Take 比 Pass 後被對方 Take 好）像骨牌一樣連鎖，把雙方早早趕下車——逆向歸納讓玩家1 在「餅最小」的第一步就 Take (4,1)，而只要願意往後讓，雙方都能爬到對彼此都更好的局面（如 (6,9) 對 (4,1) 是雙贏）。**個體理性，一步一步、無懈可擊地，把集體推向次優。** 差別只是：囚徒困境一槍斃命，蜈蚣賽局凌遲——你能眼睜睜看著逆向歸納的每一刀。

有意思的是（這是個誠實的 hedge）：真人玩蜈蚣賽局，**幾乎都不會第一步就拿**——McKelvey 與 Palfrey 1992 年發表於 Econometrica 的實驗顯示，人類會合作著往後讓好幾步，行為「嚴重偏離」子賽局完美均衡的預測。這跟人類玩囚徒困境其實會合作（ch12 細談）是同一種「真人沒那麼冷血」的現象。逆向歸納是**理性的鐵律**，但它預測的是「完全理性、且共同知道彼此完全理性」的玩家——這個假設本身就值得懷疑（這也是蜈蚣賽局被當成「逆向歸納悖論」的原因）。

而本書一路在講的**救火工具**，這裡又冒頭了：承諾。如果玩家 1 能在開局前**可信地承諾**「我會一路 Pass 到至少第三步」，並且讓對方相信，雙方就有機會爬到更高的報酬。**承諾、重複（ch07）、機制設計（ch13）——這些都是『個體主動綁住自己、去逼出集體較好結果』的同一類工具，本章把它搬到了時間軸上。**

## 直覺的陷阱

序貫賽局最容易把人帶溝裡的，是「威脅」與「先動」這兩件事。挑兩個具名陷阱講透。

| 陷阱 | 錯誤直覺 | 它在哪一步騙你 | 怎麼自我察覺（正確版） |
|---|---|---|---|
| **空洞威脅當真** | 「對方放話說會報復，所以我得退讓」 | 你只看到威脅被**說出口**，沒檢查「真要執行時，威脅方自己會不會更慘」 | 倒著想：站在威脅方的最後一個節點上，執行威脅 vs 不執行，哪個對他自己報酬高？如果不執行更高，這威脅就是空話——納許均衡看不穿、逆向歸納能 |
| **先動一定有利（先動者優勢迷思）** | 「序貫賽局裡先出手的人佔便宜」 | 你把「先動 = 主導」當成普遍法則 | 看賽局結構：先動有利**只在**「先承諾能改變後手的最佳反應」時成立。很多賽局是**後動有利**（後手看到前手底牌再出招，如討價還價的還價方、剪刀石頭布若有人先亮手）。先動是優勢還是劣勢，取決於「資訊」與「承諾」哪個更值錢——沒有通則 |

第三個更隱蔽、值得單獨點出的：**「逆向歸納＝賽局理論的鐵律，所以人就會這樣玩」。** 不對。逆向歸納告訴你「完全理性、且彼此共同知道對方完全理性」的玩家會怎麼玩；它是一個**規範性**的解（理性應該如何），不是**描述性**的預測（人實際如何）。蜈蚣賽局就是最大的反例——理論說秒結束，真人合作好幾步。把逆向歸納的結論直接當成「人類行為預測」，是濫用（ch12 會把這條線劃清楚）。徵兆：當你發現「理論說 A，但所有真人都做 B」，別急著說人不理性，先問「是不是『完全理性的共同知識』這個假設本來就不成立」。

## 紙上推演

### 第 1 題：解一棵小賽局樹（★★，20 分鐘）

兩家公司，**挑戰者**先決定要不要發動價格戰搶市占，**龍頭**再回應。報酬 (挑戰者, 龍頭)：

```text
                  [ 挑戰者 ]
                 /          \
        發動戰  /            \  不發動
               /              \
         [ 龍頭 ]            (0, 6)
          /     \
   反擊  /       \  退讓
        /         \
    (-2, 0)      (3, 2)
```

用逆向歸納求子賽局完美均衡。挑戰者會發動價格戰嗎？龍頭「我會反擊到底」的威脅可信嗎？

### 推演解答

**底層（龍頭）**：龍頭只在「挑戰者已發動」時動。反擊得 0，退讓得 2，**2 > 0 → 龍頭選退讓**，節點塌縮成 (3, 2)。

**上層（挑戰者）**：已知龍頭會退讓，發動 → 得 3；不發動 → 得 0。**3 > 0 → 挑戰者發動。**

**子賽局完美均衡 =（挑戰者：發動；龍頭：退讓），結局 (3, 2)。**

「龍頭會反擊到底」**不可信**——真打起來龍頭自己從 2 掉到 0，他不會做。挑戰者看穿這點，照樣發動。**常見錯路**：看到龍頭嘴上能威脅反擊（把挑戰者壓到 -2），就以為挑戰者會怕而不發動。這正是「空洞威脅當真」——你沒檢查龍頭執行威脅時自己更慘。注意：(不發動, 反擊) 其實是個納許均衡（挑戰者不發動，龍頭的「反擊」沒被執行也就沒露餡），但它**不是**子賽局完美均衡，被逆向歸納剔除。

### 第 2 題：一個「公開承諾反而讓你更強」的工程／生活例子（★★，15 分鐘）

舉一個你工作或生活裡「主動砍掉自己的選項、結果反而過得更好」的例子，並用本章語言解釋為什麼。

### 推演解答

工程例（deprecation 死線）：你要逼內部團隊從舊的計費 SDK 遷到新版。**做法**：在 RFC 與全公司 email 裡釘死「舊 SDK 在 9/30 停止維護、10/1 起呼叫直接回 410」，並把這日期寫進 CI 的硬性檢查。**為什麼更強**：你**主動放棄了「心軟延期」這個選項**——延期的代價（公開打臉、文件全錯、CI 規則要回滾）被你自己拉到很高。於是各團隊不再賭「他應該會延吧」，而是真的排期去遷。承諾的力量來自**不可逆**：能被輕鬆收回的承諾不值錢，公開釘死、難反悔的才算數。

對照空洞版：如果你只在某次會議口頭說「之後會關喔，再看看」，延期對你幾乎無痛，大家正確地預判你會拖，於是沒人動——**威脅不可信，因為執行起來你不痛、收回也不痛。**

生活例（同構）：想戒掉睡前滑手機，把充電器放到客廳、臥室不留任何螢幕——你**實體砍掉了「躺床上隨手拿手機」的選項**，比單純下決心有效得多。這跟 Schelling 把方向盤丟出窗外是同一招：**用放棄能力換取結果。**

**常見錯路**：把「承諾」理解成「下更大的決心／喊更響的口號」。不是。承諾的關鍵不是意志力，是**讓反悔在客觀上變昂貴或不可能**——綁手的是結構，不是決心。

### 第 3 題（口頭題）：可信威脅 vs 空洞威脅，為什麼納許均衡分不出、子賽局完美能（★★★）

口頭回答：這兩種威脅差在哪？為什麼納許均衡會放過空洞威脅，而子賽局完美均衡能抓住它？

### 推演解答

**可信威脅**＝真要執行時，威脅方確實願意執行（執行它是威脅方在那個子賽局的最佳反應）；**空洞威脅**＝威脅方真到那一步會發現執行對自己更慘、於是不會做，所以只是嚇人的空話。

**為什麼納許均衡分不出**：納許均衡只檢查「在均衡**路徑**上，沒有玩家想單方面偏離」。空洞威脅的可怕之處在於——它讓對手退讓，於是那個「會執行懲罰」的分支**根本不會被走到**。沒走到，就沒機會驗證威脅方說話算不算話，於是形式上「沒人想偏離」，納許均衡判它合格。

**為什麼子賽局完美能抓住**：子賽局完美均衡多要求一件事——**在每一個子賽局裡（包括那些「不會走到」的分支），每個玩家的計畫都必須是當下的最佳反應**。它強迫你去檢查「萬一真走到那個分支」，威脅方會不會說話不算話。一檢查，空洞威脅立刻露餡（威脅方在那會選不執行），於是被剔除。一句話：**納許均衡只管均衡路徑上的事，子賽局完美連『沒走到的劇本』都要求合理——空話藏在沒走到的劇本裡，所以只有後者抓得到它。**

## 自我檢核

口頭自答，講得出來才算過關（優先答「為什麼」）：

1. 為什麼有了「先後順序」之後，報酬矩陣就不夠用、得換成賽局樹？樹比矩陣多畫出了什麼資訊？
2. 逆向歸納為什麼要「從最後一步倒著解」？如果從第一步正著解，會漏掉什麼？
3. 用市場進入賽局講：在位者「我會打價格戰」這句威脅為什麼不可信？「不可信」精確指的是哪件事？
4. 子賽局完美均衡比納許均衡「多要求」了什麼？為什麼正是這個「多要求」讓它能剔除空洞威脅？
5. 承諾為什麼能「越少選項越強」？把 Schelling 丟方向盤、燒橋、deprecation 死線串成同一個道理。
6. 蜈蚣賽局怎麼把 ch01 那張囚徒困境矩陣的「個體理性 ⇒ 集體次優」搬到時間軸上？它和囚徒困境是同一道裂縫，差別在哪？
7. 「先動一定有利」哪裡錯了？舉一個後動有利的例子。
8. 為什麼「逆向歸納說人會 X，但真人都做 Y」不能直接推出「人不理性」？（提示：逆向歸納預設了什麼假設？）

## 延伸閱讀

- **Stanford Encyclopedia of Philosophy, "Game Theory"（賽局理論條目）** — https://plato.stanford.edu/entries/game-theory/ ——展開式賽局、子賽局完美、逆向歸納與其爭議寫得清楚嚴謹；想看「逆向歸納悖論」的哲學討論看「backward induction」一節。
- **Thomas Schelling,《The Strategy of Conflict》（衝突的策略，1960）** ——「承諾＝放棄選項換力量」「可信威脅」這套思想的源頭，方向盤、燒橋的比喻都出自這裡；談判與嚇阻章節即使今天讀也犀利。
- **Reinhard Selten 連鎖店悖論（chain store paradox）** — https://www.econlib.org/library/Enc/bios/Selten.html ——本章市場進入賽局的原型，Selten 用它示範逆向歸納（與它在「重複」情境下的弔詭，預告 ch07）；Econlib 的 Selten 小傳是好入口。
- **蜈蚣賽局（centipede game）** — https://en.wikipedia.org/wiki/Centipede_game_(game_theory) ——Rosenthal 1981 提出、McKelvey & Palfrey 1992 首次實驗；想看「理論說秒結束、真人卻合作」的落差，這條最直接。
- 跨書：序貫賽局裡「一個玩家在不確定下該怎麼選」的單人理性基準，承襲自《在不確定中下注》談期望效用與單人理性那章；承諾與聲譽在**重複**情境下如何被養出來，接到本書 ch07。
