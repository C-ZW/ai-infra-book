## 從你已知的出發

你寫過一個「偏好聚合」的函式，只是你大概沒這樣叫它。

想想你做過的任何一次**團隊技術選用投票**：要不要從 REST 換成 gRPC？資料庫選 PostgreSQL、MySQL 還是 DynamoDB？六個工程師各有一張心裡的排序，你要把這六張排序壓成一個團隊決定。你拿了張白板，每人投一票給最愛的那個，數一數，最高票出線。三十秒搞定。

或者想想你 backlog 的 **feature 優先順序排序**。產品、後端、前端、SRE 各有一份「該先做什麼」的順序，你的工作是把這四份順序合併成一份大家照著做的 roadmap。你寫過一個小腳本，把每個人對每個 feature 打的分加起來、由高到低排，輸出一份「客觀的」優先順序列表。

這兩件事，數學上是同一個東西：**一個把多份個體排序（rankings）吃進去、吐出一份集體排序（或集體選擇）的函式。** 社會選擇理論（social choice theory）管它叫**社會福利函式（social welfare function）**——名字很學術，但它的型別簽章你一眼就懂：

```text
aggregate(preferences: Ranking[]): Ranking
```

而你身為一個寫過無數聚合函式的後端工程師，對「聚合」這件事是有強烈直覺的：`sum`、`avg`、`max`、`majority` 都很乾淨、很可靠、沒什麼好擔心的。把一堆數字聚成一個數字，能出什麼大錯？

這一章就是要告訴你：**這個 `aggregate` 函式，是你寫過最危險的聚合函式之一**，危險到 Kenneth Arrow（阿羅）在 1951 年用一條定理證明了——**它的「正確版本」根本不存在**。不是你還沒找到好的演算法，是任何演算法都會在某個你列得出來的合理性質上爆掉。這跟你熟的那些聚合不一樣：`sum` 不會背叛你，但 `aggregate(rankings)` 會，而且是**數學上保證**會。

你可能已經在分散式系統裡撞過它的近親了：

- **Raft／Paxos 的多數決**：節點對「下一個 log entry 是什麼」投票，靠 quorum 收斂。但這裡其實偷偷迴避了 Arrow 的麻煩——共識協定要協調的是「**選同一個值**」（協調賽局，見 ch10），值本身好壞同質、選項之間沒有「每個節點各有一套完整偏好排序」的衝突。Arrow 處理的是更難的問題：節點們對選項**有實質且互相矛盾的偏好排序**時，怎麼聚合。
- **策略性投票**：你一定看過有人在團隊投票時**不投他真正最愛的那個**，而是「投一個比較可能贏、又比我最討厭的好」的妥協選項——怕自己那一票浪費掉。這就是策略性投票（strategic voting），它跟上一章拍賣的誠實出價形成尖銳對比：第二價格拍賣裡說真話是優勢策略，投票裡說真話卻常常害你自己。
- **「客觀」排序的循環**：你那個把每人分數加總的腳本，有沒有出現過詭異的結果——換一個無關的爛 feature 進候選清單，前兩名的排序居然顛倒了？那不是你腳本有 bug，那是 Arrow 定理裡「無關選項獨立性」被違反的活樣本。

中央張力在這裡換了一張臉。囚徒困境裡，個體理性導致集體**更糟**（見 ch01 那張矩陣）。投票裡，個體理性（策略性投票、誠實偏好的衝突）導致集體偏好**根本沒有一個融貫的形狀**——集體甚至湊不出一個「理性」的排序。個體都理性，集體卻可能連遞移性（transitivity）都保不住。這是裂縫的一個新版本：不是集體結果差，是集體**意志**本身不存在。

## 投票作為集體選擇機制：先把問題說清楚

機制設計（見 ch13）的視角是：給定一個目標結局，設計規則讓理性玩家自動導向它。拍賣的目標是「把東西配給最珍視它的人」，維克里機制漂亮地辦到了。投票的目標看起來更卑微——**只是把大家的偏好公平地聚合成一個集體決定**——卻是個出乎意料地難纏的機制。

先把舞台擺好。有 n 個投票者（voter），每人對 m 個選項（alternative）有一份**完整的偏好排序**：A 比 B 好、B 比 C 好之類。一個**投票規則（voting rule）**就是一個函式，把這 n 份個體排序吃進去，吐出一份集體排序（或至少一個集體贏家）。

我們對這個函式有期待。我們希望它「公平」「合理」。問題是——當你試著把「公平合理」拆成幾條精確的數學要求，再去找一個同時滿足全部的函式時，你會發現自己在找一個不存在的東西。

但在 Arrow 那記重拳之前，先看一個更早、更具體的警訊。它本身就夠讓人不安了。

## Condorcet 悖論：多數決會自己咬自己的尾巴

最自然的投票直覺是**多數決（majority rule）**：A 和 B 兩兩對決，誰得到的票多誰勝。這套規則乾淨、民主、無可指摘。法國數學家 Marquis de Condorcet（孔多塞）在 **1785 年**發現了它一個要命的毛病。

設想三個投票者，對三個選項 A、B、C，各有一份**完全正常、完全理性、完全遞移**的偏好排序：

```text
Condorcet 悖論：三人三選項（≻ 讀作「優於」）

  投票者 1：  A ≻ B ≻ C      （最愛 A、其次 B、最討厭 C）
  投票者 2：  B ≻ C ≻ A      （最愛 B、其次 C、最討厭 A）
  投票者 3：  C ≻ A ≻ B      （最愛 C、其次 A、最討厭 B）

  每個人的排序都遞移、都沒毛病：1 喜歡 A>B>C、不會冒出 C>A。
```

注意這三份排序很美——它們是把 A、B、C 三個選項**循環旋轉**一格得到的（1 是 ABC、2 是 BCA、3 是 CAB）。每個選項當第一名一次、第二名一次、第三名一次，公平到不能再公平。現在我們用多數決，把三個選項**兩兩對決**：

```text
兩兩多數決（每對問「幾個人覺得左邊優於右邊」）：

  A vs B：誰覺得 A≻B？  投票者 1（A≻B≻C ✓）、投票者 3（C≻A≻B ✓）→ 2 票
          誰覺得 B≻A？  投票者 2（B≻C≻A ✓）            → 1 票
          ⇒ A 以 2:1 擊敗 B            ← 社會偏好 A ≻ B

  B vs C：誰覺得 B≻C？  投票者 1（A≻B≻C ✓）、投票者 2（B≻C≻A ✓）→ 2 票
          誰覺得 C≻B？  投票者 3（C≻A≻B ✓）            → 1 票
          ⇒ B 以 2:1 擊敗 C            ← 社會偏好 B ≻ C

  C vs A：誰覺得 C≻A？  投票者 2（B≻C≻A ✓）、投票者 3（C≻A≻B ✓）→ 2 票
          誰覺得 A≻C？  投票者 1（A≻B≻C ✓）            → 1 票
          ⇒ C 以 2:1 擊敗 A            ← 社會偏好 C ≻ A
```

把這三條社會偏好排成一行，你就看見那個讓孔多塞失眠的東西：

```text
  社會偏好：  A ≻ B   且   B ≻ C   且   C ≻ A
                └──────────────────────────┘
                        A ≻ B ≻ C ≻ A ???

  多數決說：社會偏好 A 勝過 B、B 勝過 C——那社會應該偏好 A 勝過 C 吧？
  不。多數決同時說社會偏好 C 勝過 A。一個環。沒有第一名。
```

**這就是 Condorcet 悖論：每個人的偏好都遞移，多數決聚合出來的集體偏好卻是循環的——A 贏 B、B 贏 C、C 卻贏 A。** 沒有哪個選項是「孔多塞贏家」（能打贏所有對手的選項）。你問「社會最想要哪個」，答案是一個首尾相咬的環。

讓我用工程師的方式講為什麼這該嚇到你。你心裡那個 `aggregate` 函式，預設輸出是一個**全序（total order）**——一份可以拿來排序、可以取最大值的列表。`sum` 永遠給你一個良序的總和，`max` 永遠選得出贏家。但偏好聚合不保證：**三個遞移的輸入，聚合出一個不遞移的輸出。** 這在你寫過的任何數值聚合裡都不會發生——把三個遞移的全序加起來，怎麼會冒出一個環？因為「多數決」不是在加數字，它是在對**每一對**選項分別開一場投票，而這些獨立的對決之間**沒有任何東西保證它們彼此融貫**。

它的工程版你見過：你那個「把每人分數加總再排序」的腳本之所以看起來沒事，只是因為**加總（Borda 式計數）剛好繞開了循環**——它強行給每個選項一個總分，總分當然能排序。但代價是它違反了 Arrow 後面要講的另一條性質（無關選項獨立性），所以才會出現「加一個爛 feature 進清單、前兩名顛倒」的鬼打牆。**你沒有逃掉這個問題，你只是把它從『循環』這個症狀換成了『排序不穩定』那個症狀。** Arrow 定理接下來會告訴你：你換到哪個症狀都行，但你逃不掉**全部**症狀。

中央張力在這裡：三個投票者各自的偏好都完美理性、完美遞移（個體理性無懈可擊），聚合出的集體偏好卻連「理性」最起碼的要求——遞移性——都達不到。**個體的理性沒有加總成集體的理性。** 跟囚徒困境同源，但更深一層：困境裡集體結果差，這裡集體連一個融貫的「想要什麼」都形成不了。

## 策略性投票：為什麼誠實在投票裡不是優勢策略

Condorcet 悖論講的是規則本身的毛病。現在加進玩家的算計——這才是賽局理論真正接手的地方。

回想 ch14 拍賣的好消息：第二價格（維克里）拍賣裡，**照實出你的真實估值是（弱）優勢策略**——不管別人怎麼出，照實出永遠不會更差。誠實被設計成了理性。投票呢？我們來看誠實在投票裡會不會也是優勢策略。

用最常見的**單純多數決（plurality，每人投一票給最愛的，最高票勝）**。三個選項 A、B、C，七個投票者，偏好如下：

```text
單純多數決下的策略性投票（7 名投票者）：

  3 人：  A ≻ B ≻ C      （死忠 A 派）
  2 人：  C ≻ B ≻ A      （死忠 C 派）
  2 人：  B ≻ C ≻ A      （B 派，但第二喜歡 C、最討厭 A）

  如果人人誠實投最愛的那一個：
     A 得 3 票、C 得 2 票、B 得 2 票  →  A 勝。
```

現在站在最後那 2 個 **B 派**的角度想。他們最愛 B，但 B 只拿 2 票、贏不了。誠實投 B 的結果是——他們最討厭的 A 上位（A 拿 3 票勝出）。對 B 派來說這是最糟結局。

於是他們動腦筋：「B 反正贏不了，我這兩票投 B 等於浪費。但 C 是我的第二選擇、而且我最討厭 A。如果我們兩個改投 C，C 就有 2+2=4 票，超過 A 的 3 票，C 勝。」

```text
  B 派改投策略票（投第二喜歡的 C，而非最愛的 B）：
     A 得 3 票、C 得 2+2=4 票、B 得 0 票  →  C 勝。

  對 B 派而言：誠實投 B → A 勝（最糟）
              策略投 C → C 勝（次佳，遠勝最糟）
  ⇒ 說謊（不投最愛）讓 B 派得到更好的結果。誠實在這裡是劣勢策略。
```

**這就是策略性投票，也是投票和拍賣最尖銳的分水嶺。** 在維克里拍賣裡，偏離真實估值只會害你自己；在多數決裡，**偏離你的真實偏好常常救你自己**。誠實在拍賣是優勢策略，在投票卻是劣勢策略。同樣是機制設計，一個能把誠實設計成理性，一個不能——這個落差正是本章「好消息／壞消息」標題的來歷。

而這不是多數決特有的瑕疵。**Gibbard–Satterthwaite 定理**（Allan Gibbard 1973、Mark Satterthwaite 1975 各自獨立證明）把它一般化成一句殘酷的話：

> 只要選項有三個以上，任何「非獨裁、且每個選項都可能勝出」的確定性投票規則，**都可以被某個投票者透過謊報偏好操弄**——也就是說，沒有任何（非平凡的）投票規則能做到「對所有人、所有情境，誠實都是優勢策略」。

把它跟維克里拍賣並排，落差刺眼：拍賣能設計出 strategy-proof（誠實防作弊）的機制，投票在三選項以上**證明做不到**。你想設計一個「沒人有動機謊報偏好」的投票系統？Gibbard–Satterthwaite 說：除非你讓某一個人獨裁、或者把選項砍到剩兩個，否則辦不到。

（順帶一提，三個選項是門檻不是巧合。**只有兩個選項時，單純多數決其實非常乖**——它誠實防作弊、對稱、滿足一堆好性質。麻煩全是從第三個選項開始的。所以「要不要 merge 這個 PR」的二選一投票很安全，「三個架構方案選一個」就開始長出策略空間。）

## Arrow 不可能定理：壞消息的數學形狀

Condorcet 是「多數決」這條特定規則的毛病，Gibbard–Satterthwaite 是「誠實不可能是優勢策略」。Arrow 1951 的不可能定理（出自他的博士論文《Social Choice and Individual Values》）比這兩者都更釜底抽薪：它**不針對任何特定規則**，而是先寫下四條「任何公平的聚合規則都該滿足」的性質，然後證明——**沒有任何規則能同時滿足這四條**（在選項三個以上時）。

先把四條件講清楚。我盡量用你能對映到 code review 的語言講，因為每一條單獨看都像一條你會在 PR 裡要求的、毫無爭議的不變量（invariant）。

```text
Arrow 的四個條件（社會福利函式吃 n 份排序、吐 1 份集體排序）：

 ① 無限制定義域（Unrestricted Domain）
    任何「每人各自遞移」的偏好組合，函式都得能處理、給出輸出。
    ── 工程版：不能對輸入挑食。任何合法的偏好排序組合都是合法 input，
       不准說「這種排序我不收」。

 ② 帕累托（Pareto / 一致同意）
    如果『每一個人』都偏好 A 勝過 B，那集體排序也必須 A 勝過 B。
    ── 工程版：全體一致的偏好不准被推翻。沒人能否決全民共識。

 ③ 無關選項獨立性（Independence of Irrelevant Alternatives, IIA）
    集體在 A 與 B 之間怎麼排，只能取決於每個人對 A、B 這「一對」的相對偏好；
    跟大家怎麼看 C（或任何第三選項）無關。
    ── 工程版：A 與 B 的相對名次，不准因為清單裡多了/少了一個無關的 C 而翻轉。

 ④ 非獨裁（Non-Dictatorship）
    不存在某一個投票者，使得「集體排序永遠等於這個人的排序」、其他人形同空氣。
    ── 工程版：沒有 root 帳號。不准有人說了算、其他票全是裝飾。
```

每一條你都會同意。「不挑食」「尊重全民共識」「無關選項不該影響」「別搞獨裁」——這四句拿到任何一場關於「怎樣的投票才公平」的辯論裡，都是底線中的底線，沒有一條過分。

**Arrow 定理（1951）：當選項有三個以上時，沒有任何社會福利函式能同時滿足這四個條件。**

換句話說，任何你設計的投票規則，只要它能吃任意偏好（①）、尊重全民共識（②）、不被無關選項干擾（③），那它**一定是獨裁**（違反④）——集體排序永遠等於某個固定那個人的排序。反過來，只要你堅持非獨裁，就**必然**得在前三條裡犧牲掉至少一條。

這不是「目前找不到」，是 Arrow 用一個構造性的證明關死了所有門：**這樣的函式在邏輯上不存在。** 嚴謹度誠實標示：本書給的是直覺與條件，完整證明（透過「決定性集合」一路收縮到單一獨裁者的論證）不展開，指向延伸閱讀的 SEP 條目。我能誠實給你的是它**為什麼**該震撼你的工程直覺——

你寫 `aggregate` 函式時，腦中那份「規格」其實就是這四條的某種模糊版本：別挑輸入、尊重共識、別被雜訊干擾、別讓某個人說了算。你會自然假設「這規格雖然嚴格，但總有個實作能滿足吧，頂多難寫一點」。Arrow 說：**這份規格不是難實作，是無法實作。** 它跟「寫一個永不出錯的去重複函式」不一樣——去重複只是工程難題；偏好聚合是邏輯上的**矛盾規格**，就像要你寫一個「對所有輸入都停機、且能判定任意程式是否停機」的函式。不存在不是因為你不夠強，是因為這個型別簽章本身自相矛盾。

回到 Condorcet 那個悖論，現在你看得更深了：多數決犧牲的是哪一條？它**保住**了②帕累托、④非獨裁、①無限制定義域——代價是輸出可能根本不是個全序（連遞移都保不住，更別說滿足③）。而你那個「分數加總」腳本保住了「輸出是全序」，犧牲的是③ IIA——所以才會「加一個爛 feature 進清單、前兩名顛倒」。**每個規則都在 Arrow 的四條裡選一條來破，沒有例外。你只能選擇你願意承受哪一個症狀。**

中央張力在這裡走到全書的一個極點。前面的困境是「個體理性 ⇒ 集體更糟」，至少集體還有個明確的「更糟結果」可指。Arrow 說的是：當每個個體的偏好都完美理性，把它們聚合成一個「同樣理性、又公平」的集體意志這件事，**本身就不存在**。不是聚合得不好，是「公平地聚合」這個目標在數學上是空集合。個體理性與集體理性的裂縫，在這裡裂成了一道不可能跨越的鴻溝。

## 好消息與壞消息：為什麼拍賣行、投票不行

把這一章的標題收攏。你現在手上有兩個機制設計的案例，結論恰好相反：

```text
                  拍賣（ch14，好消息）        投票（ch15，壞消息）
  目標            把東西配給最珍視者          把偏好公平聚合成集體決定
  誠實是…         （弱）優勢策略 ✓            常是劣勢策略 ✗（策略性投票）
  誠實能防作弊？   能（維克里第二價格）        三選項以上不能（Gibbard–Satterthwaite）
  完美機制存在？   存在                        不存在（Arrow 四條件不可兼得）
```

差在哪？一個工程師會問的、最該問的問題。我給一個直覺版的答案（嚴格的機制設計理論在延伸閱讀）：

**拍賣聚合的是「一維、可比較、可用錢結算」的訊息——每個人的估值是一個數字，機制只要把東西給數字最大的人、再用第二價格切斷他謊報的誘因就行。** 維克里的巧妙在於「你付的價跟你的出價脫鉤」（你付第二高價、不是你自己出的價），於是謊報出價動不了你要付的錢、只會害你得標機會，照實出最划算。訊息是一維的、可貨幣化的，所以有辦法設計出「誠實最划算」的結算規則。

**投票聚合的是「多維、不可共度、無法用單一尺度結算」的偏好排序。** 「我覺得 PostgreSQL 比 MySQL 好多少」沒有一個跟錢一樣的共同單位可以結算（這正是為什麼投票用排序、不用估值——你信得過大家誠實報出「我願意為 PostgreSQL 多付三萬」嗎？）。Arrow 的 IIA 條件本質上就是在要求「A 對 B 的集體判斷不准偷渡第三方資訊」，而一旦你只能用序數（ordinal）排序、不能用基數（cardinal）強度，可用的資訊就窮到撐不起一個融貫的聚合——Condorcet 的環就是這個貧乏的症狀。

所以「好消息／壞消息」不是運氣，是結構：**當偏好能被壓進一個可結算的共同尺度（錢），機制設計能漂亮收工；當偏好是多維、不可共度的排序，Arrow 證明完美聚合不存在。** 這對你設計系統有一個非常實際的教訓：每次你想「投票表決」一件事，先問——**這些偏好能不能換算成一個共同的、可信的數字尺度？** 能（譬如真的有預算、有可量化的 SLA 影響），就用拍賣式／計分式機制，把它變成可結算的問題；不能（譬如純口味、純架構審美），就老實承認你在玩一個 Arrow 保證沒有完美解的賽局，於是**把力氣花在『選一個大家能接受的不完美規則並講清楚它犧牲了哪一條』，而不是幻想能找到完美規則**。後者是在找一個不存在的函式。

Arrow 因為這條定理（以及一般均衡理論的貢獻）在 **1972 年**獲頒諾貝爾經濟學獎（與 John Hicks 共得；嚴格說是瑞典央行紀念諾貝爾經濟學獎，由瑞典央行 1968 年設立，不在諾貝爾本人 1895 年遺囑的五個獎裡——本書沿用俗稱，全書已註明此區別）。他證明定理時年僅三十出頭，這條定理至今是社會選擇理論的奠基石。

## 直覺的陷阱

投票是「人人都有民主直覺、卻幾乎人人把 Arrow 講錯」的重災區。下面三個陷阱按「最常掉進去」排序。

| 陷阱 | 錯誤直覺 | 哪一步把你帶溝裡 | 正確版 |
|---|---|---|---|
| **「Arrow 證明民主沒用／投票是騙局」** | 既然沒有完美投票規則，那投票本身就是假的、毫無意義。 | 把「沒有滿足**全部**理想性質的規則」誤讀成「沒有**任何**有用的規則」。 | Arrow 證明的是「不可能同時滿足那四條」，**不是**「所有規則都一樣爛」。不同規則犧牲不同條件，在不同情境各有優劣；它是「沒有完美解、要懂取捨」的定理，不是「投票無用」的虛無主義宣言。把它當犬儒藉口是最常見的誤用。 |
| **「無關選項獨立性聽起來無關緊要」** | IIA 是四條件裡最技術、最不重要的一條，可以隨便犧牲。 | 沒意識到 IIA 正是擋住「加一個爛選項就翻轉前兩名」這種荒謬的那條；犧牲它＝你的排序會被無關候選人操弄（spoiler effect，攪局者效應）。 | IIA 是四條裡最常被**實際違反**、後果也最有感的一條。現實選舉裡「第三方候選人分票害得本來會贏的人輸掉」就是 IIA 被破的代價。它一點都不無關緊要——它是「無關選項不該干擾」這個你天天靠它的不變量。 |
| **「Condorcet 循環是罕見的病態案例，現實不會碰到」** | 那種首尾相咬的偏好太刻意了，真實投票哪會這麼巧。 | 把「需要特殊構造」誤當成「機率趨近於零」。其實只要選項夠多、偏好夠分歧，循環出現的機率不低，而且**你看不到它**——多數投票規則會直接吐一個贏家、把底下的循環藏起來。 | 循環在三方以上的真實偏好分布裡並不罕見（選項越多、越可能）。更危險的是它**沉默**：你的聚合函式照樣回一個「贏家」，但那個贏家可能在兩兩對決裡輸給另一個選項。你不知道，不代表它沒發生——這正是該讓你睡不著的那種失敗模式。 |

自我察覺的徵兆：如果你發現自己在用 Arrow 定理**證明某個結論**（「所以民主是假的」「所以投票都該廢掉」），你十之八九講錯了——Arrow 是個**不可能**定理，它關掉的是「完美聚合」這個選項，它不替任何具體政治結論背書。它該讓你更謙卑（承認取捨無可避免），不是更犬儒。

## 紙上推演

全程紙筆，不要碰鍵盤。每題先自己算，再對解答；解答裡的「常見錯路」是重點。

### 第 1 題（★★，[20 分鐘]）構造一個「策略性投票害死誠實」的情境

**題目**：設計一組投票者偏好，使得在**單純多數決**（每人一票投最愛的，最高票勝）下，某一群投票者**誠實投票會得到他們最糟的結果，而謊報（投第二喜歡的）能得到較好的結果**。寫出：(a) 各群的偏好排序與人數；(b) 全體誠實時誰勝；(c) 哪一群該如何策略投票、改投後誰勝；(d) 用一句話說明這跟維克里拍賣的對比。

**這題在練**：把「誠實在投票裡不是優勢策略」從一句話變成你親手擺出來的反例。

### 第 2 題（★★，[15 分鐘]）用自己的話陳述 Arrow 四條件

**題目**：不准翻回前面。憑記憶寫出 Arrow 不可能定理的四個條件，每條配一句「它在要求什麼」的白話，再配一個你工作裡的對映（code review 的不變量／API 設計原則皆可）。寫完再回去對照，看你漏了哪條、把哪條講弱了。

**這題在練**：你能不能把四條件講給另一個工程師聽——這是 ch15 的口試門檻。特別注意你是不是把③ IIA 講漏或講弱了（最常被遺忘的一條）。

### 第 3 題（★★，[20 分鐘]）找出「分數加總」腳本違反了哪一條

**題目**：用 **Borda 計分法**（m 個選項，第一名給 m−1 分、第二名 m−2 分……最後一名 0 分，加總後最高分勝）。考慮以下兩種情境，三個投票者：

```text
情境甲（候選只有 A、B）：
  投票者 1：A ≻ B
  投票者 2：A ≻ B
  投票者 3：B ≻ A

情境乙（同樣三人，但清單多塞一個爛選項 C，沒人真的想要 C）：
  投票者 1：A ≻ B ≻ C
  投票者 2：A ≻ C ≻ B      ← 注意這人覺得 C 比 B 好
  投票者 3：B ≻ C ≻ A
```

算出兩種情境下 Borda 計分的 A、B 名次。**A 和 B 的相對排名有沒有因為加了無關的 C 而改變？** 若有，這違反了 Arrow 的哪一個條件？

**這題在練**：親手看見 IIA 被違反——加一個沒人要的選項，居然動了真正候選人之間的勝負。

### 第 4 題（★★★，[20 分鐘]）口頭題：拍賣行、投票不行，差在哪

**題目**：向一個只懂第二價格拍賣、不懂投票的工程師解釋：為什麼維克里拍賣能把「誠實」設計成優勢策略，投票卻撞上 Arrow 不可能與策略性投票。不准只說「投票比較難」，要說出**結構性的差別**（提示：聚合的訊息是什麼形狀、能不能用一個共同尺度結算）。三分鐘版。

**這題在練**：本章「好消息／壞消息」的核心——機制設計什麼時候行、什麼時候撞牆。

### 推演解答

**第 1 題解答**：一組乾淨的反例（與正文同構，但你該能獨立擺出來）：

```text
  3 人：  A ≻ B ≻ C
  2 人：  C ≻ B ≻ A
  2 人：  B ≻ C ≻ A      ← 這群是關鍵

(b) 全體誠實：A 得 3、C 得 2、B 得 2 → A 勝。
(c) 最後那 2 個 B 派，最討厭 A。誠實投 B → B 輸、A 勝（B 派的最糟）。
    他們改投第二喜歡的 C：C 得 2+2=4 > A 的 3 → C 勝。
    對 B 派：C 勝（次佳）遠勝 A 勝（最糟）。⇒ 謊報得利，誠實是劣勢。
(d) 維克里拍賣：偏離真實估值只會害自己（誠實＝優勢策略）。
    多數決投票：偏離真實偏好常救自己（誠實＝劣勢策略）。同是機制設計、結論相反。
```

**常見錯路**：(1) 把人數湊成「誠實時 B 派最愛的就會贏」——那就沒有說謊的動機了，反例失效；關鍵是讓 B 派的最愛**註定贏不了**、且他們有個「比最糟好」的妥協選項可投。(2) 忘了檢查改投後**真的會翻盤**（C 的票數要真的超過 A），只憑感覺說「他們應該策略投票」。(3) 把它講成「B 派背叛了民主」——不，他們完全理性，正是**規則**逼出了不誠實，這才是 Gibbard–Satterthwaite 的點。

**第 2 題解答**：四條件（對照正文檢查你的版本）：

```text
 ① 無限制定義域：任何「人人各自遞移」的偏好組合都得能處理，不准挑輸入。
 ② 帕累托：全體一致偏好 A≻B，集體就必須 A≻B。（尊重全民共識）
 ③ 無關選項獨立性 IIA：集體對 A、B 的排序只看大家對這一對的偏好，
                       與大家怎麼看第三選項 C 無關。（無關選項不准干擾）
 ④ 非獨裁：不存在一個人，集體排序永遠等於他、其他票形同空氣。
 結論：選項 ≥3 時，①②③④ 不可兼得。要 ①②③ 就必是獨裁；要非獨裁就得犧牲前三之一。
```

**常見錯路**：(1) 把 ③ IIA 漏掉或講成「選項要互相獨立」——它不是講選項間的統計獨立，是講「**集體對某一對的判斷不准被第三者影響**」。(2) 把②帕累托講成「多數決」——帕累托只要求**全體一致**（unanimity）時尊重，不是多數時；這弱很多。(3) 把④非獨裁誤記成「人人平等／一人一票」——Arrow 沒要求對稱，只要求「不存在一個說了算的獨裁者」，門檻比平等低，所以更難迴避。

**第 3 題解答**：算 Borda 分（兩選項時第一名 1 分、第二名 0 分；三選項時 2/1/0）。

```text
情境甲（兩選項，1分/0分）：
  A：投票者1給1 + 投票者2給1 + 投票者3給0 = 2 分
  B：投票者1給0 + 投票者2給0 + 投票者3給1 = 1 分
  ⇒ A(2) ≻ B(1)。社會排序 A 在 B 前面。

情境乙（三選項，2分/1分/0分）：
  A：1給A=2、2給A=2、3給A=0          → 2+2+0 = 4 分
  B：1給B=1、2給B=0、3給B=2          → 1+0+2 = 3 分
  C：1給C=0、2給C=1、3給C=1          → 0+1+1 = 2 分
  ⇒ A(4) ≻ B(3) ≻ C(2)。
```

這個情境裡 A 仍排在 B 前面，**相對名次沒翻**——但這只是因為我選的數字不夠刁。換一組三人偏好就能讓「加進 C」真的動到 A、B 的相對結果：

```text
情境甲'（兩選項，1分/0分）：
  投票者 1：A ≻ B
  投票者 2：A ≻ B
  投票者 3：B ≻ A
  ⇒ A=2、B=1 → A ≻ B（A 嚴格贏 B）

情境乙'（同樣三人，塞進沒人真想要的 C）：
  投票者 1：A ≻ B ≻ C
  投票者 2：A ≻ B ≻ C
  投票者 3：B ≻ C ≻ A      ← 只有這人把 C 排在 A 前面
  A：2+2+0 = 4 分
  B：1+1+2 = 4 分
  C：0+0+1 = 1 分
  ⇒ A=4、B=4 → A、B 變成平手（A 的領先被抹平了）
```

加一個沒人要的 C，就把「A 嚴格贏 B」變成了「A、B 平手」——**A、B 的相對結果確實被一個無關選項改動了**。誠實標一個邊界：在**三名**投票者下，Borda 法最多只能把 A、B 從嚴格勝負攪成平手（或反過來），要把「A ≻ B」**嚴格翻轉成 B ≻ A** 得要更多投票者（最少五人就辦得到）——但這不影響結論，**動到了就是動到了**：IIA 要求「A 對 B 的集體排序只看大家對這一對的偏好」，而 Borda 法的得分依賴每個選項跟『所有其他選項』的相對位置，所以塞進無關的 C 會改變大家給 A、B 的分數差，足以攪動 A、B 的相對結果。這正是違反 ③ 無關選項獨立性（IIA）。Borda 法買到了「輸出永遠是全序、永遠選得出贏家」（繞開 Condorcet 循環），代價就是賣掉了 IIA。

**常見錯路**：(1) 算成「A、B 名次永遠不變」就下結論「Borda 沒問題」——你只是沒挑到會動的數字；要證明 Borda 違反 IIA，得找到**一組**「加了 C 之後 A、B 相對結果就改變」的偏好（存在性，一個反例就夠），不是檢查「我手上這組沒動」。注意這個改變不必是嚴格的勝負對調——把「A 嚴格贏 B」攪成「A、B 平手」一樣是 IIA 被破（無關的 C 動到了 A 對 B 的集體判斷）；三名投票者下只攪得出平手，要嚴格對調需要更多人。(2) 把違反的條件答成「帕累托」——帕累托在這裡沒被破（沒有全體一致的偏好被推翻）；被破的是 IIA。(3) 忘了 Borda 是序數計分、卻偷偷塞進「強度」直覺（「我超級喜歡 A」給更多分）——標準 Borda 只看名次、不看強度。

**第 4 題解答（口頭，講出來才算過）**：結構性差別有三層，缺一不可：

```text
  1. 訊息維度：拍賣的偏好是『一維的估值』（一個數字）；
     投票的偏好是『多維的排序』（A>B>C 整串），無法壓成一個數字。
  2. 共同尺度：拍賣有錢這個可結算的共同單位，「給最高者、收第二價」
     就能切斷謊報誘因；投票（序數排序）沒有共同尺度可結算。
  3. 後果：有共同尺度 → 設計得出誠實=優勢策略（維克里）；
     沒有 → Arrow 證明連『公平聚合』都不存在、Gibbard–Satterthwaite
     證明誠實不可能是優勢策略。
  一句話：能換算成可信的錢，機制設計就能漂亮收工；只能比序、不能結算，
          就撞上 Arrow 的天花板。
```

**常見錯路**：只說「投票比較複雜／選項比較多」——那是現象不是原因；兩選項投票其實很乖（多數決誠實又防作弊），麻煩是從「多維排序＋無共同尺度」來的，不是從「複雜」來的。

## 延伸閱讀

- **Stanford Encyclopedia of Philosophy，「Arrow's Theorem」**（https://plato.stanford.edu/entries/arrows-theorem/）——四條件的精確陳述與完整證明思路（透過「決定性集合」收縮到獨裁者），本章不展開的那部分在這裡。哲學系寫的，但對工程師意外好讀，先讀條件陳述那一節。
- **Stanford Encyclopedia of Philosophy，「Voting Methods」**——把單純多數決、Borda、Condorcet 方法、即時排序複選（IRV）等規則攤開比較，每個犧牲哪條 Arrow 條件講得很清楚；想知道「那我到底該用哪個投票規則」時讀這篇。
- **維基百科「Condorcet paradox」**（https://en.wikipedia.org/wiki/Condorcet_paradox）——孔多塞 1785 原始悖論、循環多數的更多例子與出現機率的討論；本章的環只是最小例。
- **Kenneth Arrow,《Social Choice and Individual Values》(1951)**——不可能定理的原典，薄薄一本。不必啃完證明，讀他的「導論」就值回票價：你會看到一個經濟學家如何把「公平」這個含糊的詞，逼成四條能被證偽的數學條件——這個「把直覺逼成精確規格」的動作本身，就是這本書教你的東西。
- **跨書**：投票者「謊報偏好對自己是否划算」這種單人在不確定下的理性計算，深入處（期望效用、效用不是金額）指向《在不確定中下注：決策與理性》談單人理性那部分；本章只用到「對手會反應你的選擇」這一面。把多份排序看成資料、聚合的穩定性涉及向量與序的結構時，可順帶翻《矩陣是動詞：線性代數》談序與結構那部分（點到即止，本章不展開）。
