# 決定卻測不準

### 混沌理論的直覺與驚嘆

---

這本書要說服你一件事，而這件事和你六年來的工程直覺正面衝突：

**一個系統可以「下一刻完全被方程式決定、沒有半點隨機」，同時「長期完全無法預測」。**

你會本能地反駁：如果完全被決定，那只要算得夠仔細，不就能預測嗎？算力不夠、模型不準，那是工程問題，不是原理問題吧？

不是。這正是二十世紀數學最反直覺的發現之一。有一整類系統——天氣、三個互相拉扯的天體、一條最簡單的人口遞迴式——它們**沒有一丁點隨機**，每一步都由前一步唯一決定；但只要你對起點的掌握差了千分之一、甚至差了小數點後第二十位，這個誤差會被系統自己**指數放大**，快到再強的電腦、再精密的儀器都追不上。預測的牆不在工程，在原理。

拉普拉斯（Laplace）曾經想像一個惡魔：知道宇宙此刻每顆粒子的位置與速度，加上無限的算力，過去與未來都攤在牠眼前。這本書，就是看著這隻惡魔怎麼死的。**殺死牠的不是隨機，是決定論自己。**

## 這本書是寫給誰的

寫給一位資深後端工程師（也就是你）。純興趣，不為轉職、不服務任何敘事——只為了那個「原來如此」的瞬間，以及讀完之後能用自己的話，對另一個工程師講清楚震撼在哪。

你早就見過混沌，只是沒這樣稱呼它：retry storm 怎麼從一點抖動滾成雪崩、autoscaler 怎麼在兩個副本數之間來回震盪停不下來、為什麼追一個分散式 bug 需要 bit-identical 的重放、為什麼 PRNG 明明完全確定卻「看起來」那麼隨機。這些都是混沌的工程化身。本書會一路用這些你熟的東西當橋，把你載到天氣、星空與碎形那邊去。

## 這本書怎麼讀

- **純紙上推演**。沒有任何要你去跑的程式或 lab。每章的「紙上推演」是手算幾步、在圖上口頭推軌跡、或思辨題——講得出來才算懂，講不清楚就是還沒懂。
- **直覺與圖像優先，硬數學從簡**。必要的式子會講透它的直覺，不堆推導。真正的重活交給插圖：分岔圖、Lorenz 蝴蝶、Mandelbrot 集這些招牌視覺，書裡都有預先算好的圖。
- **一條遞迴式貫穿全書**。你只需要記住一行式子：

  > xₙ₊₁ = r · xₙ · (1 − xₙ)

  它簡單到像一道國中數學，卻在轉動唯一的旋鈕 r 時，依序上演「穩定 → 分裂 → 倍週期 → 混沌 → 混沌裡的秩序孤島」的完整劇情。全書七個地方會回來看它一次，每次都看懂多一層；最後一章把七層攤在同一張表上對帳。

## 全書地圖

- **Part I — 鐘錶宇宙的裂縫**：決定論的夢，與它的第一道裂縫（拉普拉斯、龐加萊的三體問題、勞侖次的蝴蝶、把「決定／隨機／可預測」三個被混為一談的概念拆開）。
- **Part II — 一條遞迴式裡的宇宙**：脊椎登場。不動點、穩定、倍週期分岔、費根堡普適性、混沌登場與秩序孤島。
- **Part III — 混沌的肖像**：混沌長什麼樣子。相空間、奇異吸子、碎形、碎維度。
- **Part IV — 為什麼測不準**：不可預測的機制與極限。Lyapunov 指數、可預測地平線（天氣為什麼只能測兩週）、拉伸與摺疊這台製造機。
- **Part V — 與混沌共處**：混沌不等於雜訊（怎麼分辨）、駕馭與利用混沌、總收束（同一條遞迴式的七層意義，以及惡魔究竟死在哪一刀）。

附錄有記號速查、詞彙表與閱讀地圖。書架上若有《馴服隨機》《馴服無限》《矩陣是動詞》，本書會在邊界處指過去——但不依賴它們，獨立讀完沒有問題。
