# ch06 — 不動點與穩定：旋鈕還小的時候

> **本章解決什麼問題**：ch05 把脊椎遞迴式 xₙ₊₁ = r·xₙ·(1−xₙ) 端上桌，你也已經在 cobweb 圖上看著它收斂到一個值。但「為什麼會收斂、收斂到哪、什麼時候不再收斂」這三件事，ch05 只給了直覺。這一章把它變成可以手算的判據：不動點（fixed point）是 x*=f(x*) 的解，而它穩不穩定，全看曲線在交點處的**斜率有多陡**——|f′(x*)|<1 就吸引、就彈回。這是整本書穩定性分析的基石。守住它，ch07 旋鈕一過 3、不動點失穩、世界一分為二，你才會覺得是意料之中而不是魔術。

## 從你已知的出發

你調過控制迴圈（control loop）。也許是 autoscaler 的擴縮策略，也許是某個自製的限流器，也許只是把上一秒的延遲餵回去決定下一秒要不要降載。你一定見過這兩種結局：

- **增益（gain）調得保守**：系統有點偏差，下一輪修一點、再下一輪修一點，誤差越來越小，最後安安靜靜停在目標值。
- **增益調得太兇**：系統一偏差就過度反應（overshoot），修過頭、反向再修過頭，於是在目標值兩側來回甩，幅度甚至越甩越大——這就是震盪（oscillation），嚴重時就是 flapping，副本數在 N 和 2N 之間跳個不停。

工程上你會說「加點阻尼（damping）」「把增益降下來」。你憑經驗知道**修正力道太強會失穩**。本章要做的，就是把這條經驗變成一個可以代數計算、一眼判定的條件。

關鍵的橋是這句話：**每一輪，誤差被乘上一個固定的倍數。** 如果這個倍數的絕對值小於 1，誤差每輪縮小，系統收斂；如果大於 1，誤差每輪放大，系統發散。對一個離散回授迴圈 xₙ₊₁=f(xₙ) 來說，這個「倍數」就是函數在不動點附近的斜率 f′(x*)。

具體一點。假設系統的平衡點是 x*，這一輪你離平衡點有個小偏差 εₙ，也就是 xₙ = x* + εₙ。下一輪：

```text
  xₙ₊₁ = f(xₙ) = f(x* + εₙ)
        ≈ f(x*) + f′(x*)·εₙ      ← 平衡點附近把曲線當直線（一階近似）
        = x*     + f′(x*)·εₙ      ← 因為 x* 是不動點，f(x*)=x*

  所以  εₙ₊₁ = xₙ₊₁ − x* ≈ f′(x*)·εₙ
```

讀最後一行：**下一輪的偏差，等於這一輪的偏差乘上 f′(x*)。** 這跟你心裡那個「增益太高就震盪」是同一件事，只是現在它有名字、有數字。再迭代下去：εₙ ≈ f′(x*)ⁿ·ε₀。乘的倍數絕對值小於 1，εₙ→0（收斂）；大於 1，εₙ 指數放大（發散）。臨界就在等於 1。

這就是全章的核心，剩下的都是把它套到脊椎遞迴式上、把數字算出來。

## 不動點：系統「停得下來」的地方

先定義。**不動點（fixed point）** 是迭代下去不再變的點：你餵 x* 進去，吐出來還是 x*。

```text
        x* = f(x*)        ← 不動點的定義：迭代不再改變的值
```

工程直覺很乾淨：不動點就是回授迴圈的**穩態候選**。autoscaler 真正停在某個副本數，是因為「當前負載算出的目標副本數」剛好等於「當前副本數」——餵進去等於吐出來。是不是真的會停在那裡（穩不穩定）是下一節的事；這一節只先把「停得下來的地方」找出來。

把脊椎遞迴式 f(x) = r·x·(1−x) 代進定義，解 x* = r·x*·(1−x*)：

```text
  x* = r·x*·(1 − x*)
  x* − r·x*·(1 − x*) = 0
  x*·[1 − r·(1 − x*)] = 0          ← 把 x* 提出來
```

兩個因子，給兩個解：

```text
  解一：x* = 0                      ← 「滅絕點」：沒有人口，下一代還是沒有人口
  解二：1 − r·(1 − x*) = 0
        r·(1 − x*) = 1
        1 − x* = 1/r
        x* = 1 − 1/r                ← 「共存點」：人口穩定在某個正比例
```

第一個 x*=0 永遠存在，不管 r 多少——空的系統永遠是空的。第二個 x* = 1 − 1/r 才有意思，它隨旋鈕 r 移動：

```text
  r = 1.5  →  x* = 1 − 1/1.5  = 1 − 0.6667 = 0.3333
  r = 2.0  →  x* = 1 − 1/2.0  = 1 − 0.5    = 0.5
  r = 2.5  →  x* = 1 − 1/2.5  = 1 − 0.4    = 0.6
  r = 3.0  →  x* = 1 − 1/3.0  = 1 − 0.3333 = 0.6667
```

注意一個邊界情況：當 r<1 時，1 − 1/r 是**負數**，掉出了 [0,1] 這個合法區間。人口比例不可能是負的——所以 r<1 時「共存點」根本不存在，系統只剩 x*=0 這個滅絕點可去。這跟生態直覺一致：成長率 r 小於 1，連自我替補都做不到，人口必然歸零。後面算穩定性時會看到，r<1 時 x*=0 確實是穩定的。

找到不動點只是第一步。x*=0.6 是「系統可能停在 0.6」，但 ch05 你親眼看到 r=2.5 的軌跡是**真的**收斂到 0.6 的。憑什麼？換個 r，它就不收斂了。差別在哪？答案在斜率。

## 穩定判據：看交點處的斜率有多陡

回到從你已知的出發那段推出來的式子：εₙ₊₁ ≈ f′(x*)·εₙ。把它講成一句判據：

```text
  |f′(x*)| < 1   →   穩定（attracting，吸引）：偏差每輪縮小，軌跡彈回不動點
  |f′(x*)| > 1   →   不穩定（repelling，排斥）：偏差每輪放大，軌跡彈開
  |f′(x*)| = 1   →   臨界（critical）：一階看不出來，得看更高階——這是分岔要發生的地方
```

為什麼是**絕對值**？因為斜率可以是負的。f′(x*) 是負的時候，εₙ 每輪會變號——偏差從正跳到負、再跳回正——這就是「在不動點兩側來回」的振盪式逼近。只要絕對值小於 1，每次甩過去的幅度都比上次小，最後還是收斂，只是用螺旋／來回的方式收斂，不是直直地滑進去。這正對應你調控制迴圈時「有點 overshoot 但會穩下來」的那種手感。

這裡有個漂亮的幾何畫面，也是這章值得你停十分鐘的地方。把 cobweb 圖攤開：橫軸 xₙ，縱軸 xₙ₊₁，畫上曲線 y=f(x) 和對角線 y=x，兩者的交點就是不動點（因為交點上 f(x)=x，正是不動點定義）。迭代的軌跡在「曲線」和「對角線」之間階梯狀彈跳。**曲線在交點處的斜率，決定階梯是越走越小（收進去）還是越走越大（甩出去）。**

```text
  情況 A：交點處斜率「緩」（|f′(x*)|<1）→ 階梯收縮、彈回 → 穩定

      xₙ₊₁
        |          / y = x（對角線，斜率永遠是 1）
        |        /
        |      /·——.            曲線在交點附近比對角線平緩
        |    /·  ↓  |  ___---‾‾  （斜率絕對值 < 1）
        |  / ·__|___|‾            軌跡像階梯一格一格縮進交點
        |/__·___|___|_____ xₙ     每一步離 x* 更近 → 彈回
              x*
        看點：曲線比對角線「躺」，所以反彈幅度一次比一次小

  情況 B：交點處斜率「陡」（|f′(x*)|>1）→ 階梯張開、彈開 → 不穩定

      xₙ₊₁
        |        |      / y = x
        |        |    /
        |        |  /            曲線在交點附近比對角線陡
        |    ___-|-/             （斜率絕對值 > 1）
        |  -‾   /|               軌跡像階梯一格一格張開
        |‾____/__|_______ xₙ     每一步離 x* 更遠 → 彈開
              x*
        看點：曲線比對角線「立」，所以反彈幅度一次比一次大
```

「斜率緩 vs 斜率陡」的分界，正是斜率絕對值等於對角線斜率（恆為 1）的那一刻。曲線比對角線躺，誤差被壓縮；曲線比對角線立，誤差被放大。這就是 |f′(x*)|<1 的純幾何意義——不需要任何代數，光看兩條線誰比較陡，就知道不動點吸不吸引。

## 把判據套到脊椎遞迴式：f′(x*)=2−r

現在把脊椎遞迴式的斜率算出來。f(x) = r·x·(1−x)，先展開再微分，一步都不跳：

```text
  f(x)  = r·x·(1 − x)
        = r·x − r·x²            ← 乘開
  f′(x) = r − 2·r·x             ← 逐項微分：r·x 的導數是 r、r·x² 的導數是 2r·x
        = r·(1 − 2x)            ← 提出 r
```

（如果「導數＝瞬時斜率」這件事生鏽了，這就是「x 變一點點，f 變多少倍」的那個倍率，擦亮即可，見《馴服無限》的導數章。）

把非零不動點 x* = 1 − 1/r 代進 f′(x) = r·(1 − 2x)：

```text
  f′(x*) = r·(1 − 2·x*)
         = r·(1 − 2·(1 − 1/r))         ← 代入 x* = 1 − 1/r
         = r·(1 − 2 + 2/r)             ← 把括號裡展開：2·(1 − 1/r) = 2 − 2/r
         = r·(−1 + 2/r)                ← 1 − 2 = −1
         = −r + 2                      ← r·(−1) = −r，r·(2/r) = 2
         = 2 − r                       ← 整理
```

漂亮得有點過分。整條曲線、整個 r，最後不動點處的斜率就是乾乾淨淨的：

```text
        f′(x*) = 2 − r
```

套上 |f′(x*)|<1 的穩定判據：

```text
  |2 − r| < 1
  −1 < 2 − r < 1                ← 拆開絕對值
  −1 − 2 < −r < 1 − 2           ← 三邊各減 2
  −3 < −r < −1
   3 > r > 1                    ← 三邊乘 −1，不等號要反向（負數乘法）
```

也就是：

```text
        1 < r < 3   ⟺   非零不動點 x* = 1 − 1/r 穩定
```

把幾個臨界點挑出來看，每個都對得上前面的直覺：

```text
  r → 1⁺ ：f′(x*) = 2 − 1 = +1   ← |f′|=1，臨界下界。其實 r<1 時 x* 出界，由 x*=0 接管
  r = 2  ：f′(x*) = 2 − 2 =  0   ← 斜率全平！「超穩定（superstable）」，收斂最快
  r = 2.5：f′(x*) = 2 − 2.5 = −0.5  ← 負斜率，來回振盪式收斂（|·|<1，穩）
  r = 3  ：f′(x*) = 2 − 3 = −1   ← |f′|=1，臨界上界，穩定就此終結 → ch07 接手
```

r=2 那個 f′(x*)=0 特別有意思。斜率是 0，意思是不動點附近曲線是平的——一階近似下偏差被乘以 0，等於一步就被吃掉大半，收斂快到離譜。這叫**超穩定點（superstable point）**，是穩定區裡「阻尼最足」的那個 r。從 r=2 往兩邊走，斜率絕對值都變大，收斂變慢，直到 r=1 和 r=3 兩端斜率絕對值頂到 1，穩定崩潰。

注意斜率的**符號**怎麼隨 r 變：r<2 時 f′(x*)=2−r>0（正斜率，軌跡單調滑進不動點）；r>2 時 f′(x*)=2−r<0（負斜率，軌跡來回振盪著進不動點）。所以 r=2.5 的軌跡會「上一輪偏高、下一輪偏低」地夾逼 0.6，而不是直直滑進去。這個符號細節等一下 worked example 會親眼驗證——它正是 ch05 那張 r=2.5 迭代表為什麼忽高忽低的原因。

至於 x*=0 這個滅絕點，順手也算一下：f′(0) = r·(1 − 0) = r。所以 |f′(0)|=r，r<1 時 x*=0 穩定（人口滅絕、且穩穩地滅絕），r>1 時 x*=0 失穩（一點點人口就會被放大、離開 0）。兩個不動點剛好在 r=1 交班：r<1 滅絕點吸引、共存點出界；r>1 滅絕點排斥、共存點接手吸引——直到 r=3 共存點也撐不住。

## Worked example：r=2.5 穩、r=3.2 失穩，手算到底

光有判據不夠，得親手把兩個 r 算穿，而且要跟 ch05 的迭代表對得上。

**情況一：r=2.5。** 算不動點與斜率：

```text
  x*     = 1 − 1/2.5 = 1 − 0.4 = 0.6
  f′(x*) = 2 − 2.5   = −0.5
  |f′(x*)| = 0.5 < 1            ← 判定：穩定，且是負斜率（振盪式逼近）
```

判據說穩定。現在手動迭代驗證（這是 ch05 那條遞迴式，x₀=0.1，每步保留 4 位小數、複算一次）：

```text
  n   xₙ        計算 xₙ₊₁ = 2.5·xₙ·(1 − xₙ)                  離 x*=0.6 的偏差
  0   0.1000    2.5 × 0.1000 × 0.9000 = 0.2250              −0.5000
  1   0.2250    2.5 × 0.2250 × 0.7750 = 0.4359              −0.3750  ← 偏差還是負，在縮
  2   0.4359    2.5 × 0.4359 × 0.5641 = 0.6148              −0.1641
  3   0.6148    2.5 × 0.6148 × 0.3852 = 0.5921              +0.0148  ← 變號了！跨過 x*
  4   0.5921    2.5 × 0.5921 × 0.4079 = 0.6038              −0.0079  ← 又變回負，幅度更小
  5   0.6038    2.5 × 0.6038 × 0.3962 = 0.5981              +0.0038
  6   0.5981    2.5 × 0.5981 × 0.4019 = 0.6010              −0.0010
        … 偏差絕對值一路縮小、左右交替 → 收斂到 0.6
```

兩件事一次對上：(1) 它**收斂到 0.6**，和不動點 x*=1−1/2.5=0.6 完全一致；(2) 從第 3 步起偏差開始**左右變號**（−0.1641 → +0.0148 → −0.0079 → …），正是 f′(x*)=−0.5 那個負斜率預言的振盪式逼近。而且每次的偏差大致是上一次的一半左右（−0.1641 之後是 +0.0148……比例沒那麼精準，因為遠離不動點時一階近似不準，但越靠近 x*、比例越接近 |−0.5|=0.5）。這就是 ch05 那張收斂表背後的機制：不是它「碰巧」收斂，是 |f′(x*)|=0.5<1 逼它收斂。

**情況二：r=3.2。** 同樣先算：

```text
  x*     = 1 − 1/3.2 = 1 − 0.3125 = 0.6875
  f′(x*) = 2 − 3.2   = −1.2
  |f′(x*)| = 1.2 > 1            ← 判定：不穩定（排斥）
```

判據說不穩定。意思是：x*=0.6875 仍然是個不動點（你餵 0.6875 進去，吐出來還是 0.6875，不信代代看：3.2×0.6875×0.3125=0.6875，對），但它是**排斥**的——只要起點不是精準的 0.6875，哪怕差一丁點，偏差就會被每輪乘上 −1.2 而放大，軌跡被推開。手動迭代看會發生什麼（x₀=0.5）：

```text
  n   xₙ        計算 xₙ₊₁ = 3.2·xₙ·(1 − xₙ)
  0   0.5000    3.2 × 0.5000 × 0.5000 = 0.8000
  1   0.8000    3.2 × 0.8000 × 0.2000 = 0.5120
  2   0.5120    3.2 × 0.5120 × 0.4880 = 0.7995
  3   0.7995    3.2 × 0.7995 × 0.2005 = 0.5129
  4   0.5129    3.2 × 0.5129 × 0.4871 = 0.7994
  5   0.7994    3.2 × 0.7994 × 0.2006 = 0.5130
        … 它不收斂到 0.6875，而是穩穩地在 0.5130 和 0.7995 兩個值間來回跳
```

軌跡沒有停在不動點，而是停進了一個**在兩個值之間交替的循環**——0.5130、0.7995、0.5130、0.7995……這正是 ch07 要講的 2-cycle（週期 2 軌道）。本章只負責講清楚一件事：**為什麼 x*=0.6875 留不住軌跡**——因為 |f′(x*)|=1.2>1，它從吸引變成排斥了。r 從 2.5 升到 3.2，跨過了 r=3 那道 |f′|=1 的牆，不動點失穩，世界準備一分為二。至於分裂後跳到哪兩個值、為什麼是「加倍」而不是「加一」，留給 ch07。

對帳一下兩個情況：

```text
  r      x* = 1−1/r    f′(x*) = 2−r    |f′(x*)|    判定        軌跡實況
  2.5    0.6           −0.5            0.5 < 1     穩定        收斂到 0.6（振盪式）
  3.2    0.6875        −1.2            1.2 > 1     不穩定      跳成 2-cycle，不動點留不住
```

同一條遞迴式、同一個公式 f′(x*)=2−r，只因為旋鈕從 2.5 轉到 3.2，一個收斂、一個失穩。穩定性不是系統的固有屬性，是**旋鈕的函數**——這是本章我最想讓你帶走的一句話。

## 一句話的多維呼應：Jacobian 特徵值

脊椎遞迴式只有一維（一個 x），所以「斜率」就是一個數 f′(x*)，判據就是 |f′(x*)|<1。真實系統常是多維的——Lorenz 系統三維、你的服務有好幾個耦合的狀態量。多維時「斜率」變成一個矩陣（Jacobian，雅可比矩陣），它記錄「每個方向的偏差會怎麼影響每個方向的下一輪偏差」。

判據的形狀完全一樣，只是「斜率」升級成「矩陣的特徵值」：**不動點穩定 ⟺ Jacobian 在該點的所有特徵值的模（絕對值）都小於 1。** 特徵值就是「矩陣往某個方向拉伸幾倍」的那個倍率（見《矩陣是動詞》談特徵值的章）；模小於 1 表示那個方向的偏差被壓縮，所有方向都被壓縮，整體就彈回不動點。一維是這件事的特例——一維的「特徵值」就是 f′(x*) 本身。這裡只給直覺、不推導；要看線性化分析怎麼做，去《矩陣是動詞》。

（小心一個常見混淆：這是**離散**映射的判據，看的是「模 < 1」。連續時間的微分方程系統判據不同——看特徵值的**實部是否為負**。本書脊椎是離散迭代，所以一律用「模 < 1」。)

## 為什麼 |f′| 就是「每輪的誤差倍率」

值得把 |f′(x*)|<1 背後那一行代數攤開來看，因為它是後面好幾章的種子。假設這一輪你離不動點還有一小段偏差，記成 εₙ = xₙ − x*。下一輪的偏差是多少？

```text
  εₙ₊₁ = xₙ₊₁ − x*
       = f(xₙ) − f(x*)              ← 因為 x* 是不動點，x* = f(x*)
       ≈ f′(x*)·(xₙ − x*)          ← 一階近似：f 在 x* 附近約等於它的切線
       = f′(x*)·εₙ                  ← 換回 εₙ
```

讀這行：**偏差每過一輪，大致就乘上一個固定的數 f′(x*)。** 這個數就是「誤差倍率」。乘 0.5（r=2.5）→ 偏差越乘越小、收斂；乘 −1.2（r=3.2）→ 偏差越乘越大、彈開（負號只是讓它每輪換邊，不影響「變大」）。穩定不穩定，整件事就濃縮成「這個倍率的絕對值，比 1 大還小」。

這也解釋了為什麼前面 r=2.5 的迭代表，偏差比例會「越靠近 x* 越接近 0.5」——遠離不動點時切線近似不準，越靠近、f 越像它的切線，每輪乘 0.5 就越準。

這個「誤差倍率」其實你在控制系統裡天天打交道，只是換了名字。一個比例回授迴圈的迴路增益（loop gain）就是這個倍率：增益的絕對值小於 1，擾動被一輪輪吃掉，系統穩穩坐回設定點；增益絕對值大於 1，一點點偏差被越放越大，系統開始來回過衝、震盪發散。把積分增益（integral gain）調太兇的 PID 會嗡嗡振盪停不下來，原因和這裡 r 過 3 後不動點留不住，是同一件事——你把局部誤差倍率推過了 1。所以「調 PID」和「轉 r 這個旋鈕」，在數學上是同一個動作：挪動那個決定誤差被放大還是被壓縮的倍率。

把這個畫面記牢，因為它會長大。現在我們看的是「**一個**不動點附近、**一輪**的倍率」。等 ch14 講 Lyapunov 指數時，問的是同一件事的全域版本：軌跡不待在某個不動點、而是一路亂跑，那就把「每一步當下的 ln|f′(xₙ)|」沿著整條軌跡長期平均起來——平均後是正、是負，就決定了整個系統是發散（混沌）還是收斂。ch06 這個 |f′(x*)|，是那把尺的局部、靜止版；ch14 的 λ，是它的全域、運動版。同一把尺。

## 直覺的陷阱

| 誤解 | 為什麼錯／會在哪一步把你帶溝裡 | 正確版 |
|---|---|---|
| 「不動點＝系統一定會停在那裡」 | 不動點只是「停得下來的候選位置」，不保證軌跡會去。r=3.2 的 x*=0.6875 是貨真價實的不動點，但軌跡永遠到不了——除非你精準地從它出發。把「存在不動點」當成「會收斂」，會讓你在 r>3 時困惑為什麼系統不穩。 | 不動點分**吸引（穩定）**與**排斥（不穩定）**兩種。要判定哪種，得看斜率 |f′(x*)|，不是看它存不存在。 |
| 「斜率是正的才穩定」 | 只看斜率正負，會把 r=2.5（f′=−0.5，負但穩）誤判成不穩定。負斜率只是說「振盪式逼近」，不是發散。 | 判據是斜率的**絕對值** |f′(x*)|<1。符號只決定逼近方式（單調 vs 振盪），絕對值才決定收不收斂。 |
| 「r 越大，系統越穩（成長率高嘛）」 | 直覺上「成長快」像是好事，但 r 太大反而**失穩**。r 從 2 往上，斜率 2−r 的絕對值越來越大，r 過 3 就崩。增益太高就震盪，這跟你的控制迴圈經驗完全一致，別被「成長率」這個詞騙了。 | 穩定區是有上限的 1<r<3。修正力道（這裡是 r）太強會 overshoot 到失穩，太弱（r<1）則連存活都做不到。 |
| 「r=3 是平滑過渡，跨過去也沒什麼」 | r=3 時 |f′(x*)|=1，一階近似剛好失效——這正是**分岔（bifurcation）**發生的臨界點，系統的定性行為在這裡質變（從收斂變週期），不是量變。把它當普通一點，會錯過 ch07 整個倍週期級聯的起跑線。 | r=3 是臨界點。|f′|=1 時穩定性判據本身失效（一階看不出來），系統正要從「停在一點」翻成「在兩點間跳」。 |
| 「測不到收斂，一定是迭代次數不夠」 | 在 r>3 區，再迭代一萬次也不會收斂到不動點——因為它根本是排斥的。把「不收斂」一律歸因於「跑得不夠久」，會讓你白等。 | 先用 |f′(x*)| 判定。|f′|>1 時，不收斂是**原理上的**，不是耐心問題。 |

## 紙上推演

### 推演題 1 ★ **[10 分鐘]**

給 r=2.8。(a) 算非零不動點 x* 和斜率 f′(x*)；(b) 判定穩定還是不穩定，並說出它會以「單調」還是「振盪」的方式逼近不動點；(c) 不用完整迭代，只用斜率，估計「離不動點的偏差大約幾輪縮一半」。

#### 推演解答

(a) 直接代公式：

```text
  x*     = 1 − 1/2.8 = 1 − 0.3571 = 0.6429
  f′(x*) = 2 − 2.8   = −0.8
```

(b) |f′(x*)| = 0.8 < 1 → **穩定**。斜率是**負的**（−0.8），所以偏差每輪變號 → **振盪式**逼近：軌跡會在 0.6429 兩側來回跳著收進去，不是直直滑進去。

(c) 每輪偏差乘上 |−0.8|=0.8。要縮到一半，問 0.8ⁿ ≈ 0.5。手算：0.8²=0.64、0.8³=0.512、0.8⁴=0.4096。所以**大約 3 輪偏差縮一半**（0.8³=0.512 最接近 0.5）。注意這比 r=2.5 慢（r=2.5 時每輪乘 0.5，一輪就縮一半）——因為 r=2.8 更靠近失穩邊界 r=3，斜率絕對值更大、阻尼更弱、收斂更慢。常見錯路：忘了取絕對值，看到 −0.8 就說「不穩定」；或忘了負斜率代表振盪、誤以為單調收斂。

### 推演題 2 ★★ **[15 分鐘]**

口頭（寫下要點）解釋：**為什麼 r=3 是臨界點？** 不要只說「因為 |f′|=1」，要講清楚「|f′|=1 這件事在 cobweb 圖上長什麼樣、為什麼一階近似在這裡失效、以及這個臨界為什麼預告了一場質變而不只是收斂變慢」。

#### 推演解答

要點：

1. **代數面**：r=3 時 f′(x*)=2−3=−1，所以 |f′(x*)|=1，剛好踩在穩定判據的邊界上。
2. **cobweb 幾何面**：|f′|=1 表示曲線在交點處和對角線一樣陡（只是斜率是 −1，方向相反）。階梯既不收縮也不張開——偏差每輪乘 −1，幅度**不變、只變號**。軌跡會繞著不動點打轉，既不掉進去也不甩出來。
3. **一階近似失效**：穩定判據 εₙ₊₁≈f′(x*)·εₙ 是把曲線當直線的一階近似。當 |f′(x*)|=1，一階項剛好「平手」，決定命運的是被我們忽略掉的**更高階**項（曲線的彎曲）。所以光看斜率（一階）已經判不出穩不穩定——這就是「臨界」的精確意思：一階資訊用罄。
4. **為什麼是質變不是量變**：在 r<3，軌跡收斂到**一個**值；r 一過 3，不動點失穩，軌跡改去**兩個**值之間跳（2-cycle）。系統的長期行為從「週期 1」變「週期 2」——這是定性的改變（分岔），不是「收斂慢一點」這種量的改變。r=3 是這場分裂的起跑線（細節 ch07）。

常見錯路：把 r=3 講成「系統開始發散／爆掉」。不是。它不發散（x 永遠在 [0,1]，被 (1−x) 這個擁擠項壓著），它是從「停一點」翻成「跳兩點」。失穩 ≠ 爆炸。

### 推演題 3 ★★ **[15 分鐘]**

有人主張：「既然 r=3.2 時 x*=0.6875 是不動點，那只要我把初始值**精準設成** 0.6875，系統就會永遠待在 0.6875，這樣不就『穩定』了嗎？」這個論證哪裡有破綻？它和「穩定」這個詞的真正意思差在哪？順便：這跟你在分散式系統裡追 bug 要 bit-identical 重放有什麼血緣？

#### 推演解答

破綻在於混淆了「不動點」和「**穩定的**不動點」。論證前半段是對的：在數學的理想世界裡，精準的 0.6875 餵進去確實永遠吐出 0.6875（3.2×0.6875×0.3125=0.6875）。但這不是「穩定」。

**穩定（stability）講的是對擾動的抵抗力**：把起點放在 0.6875 **附近**（不是精準地在上面），軌跡會不會自己回到 0.6875？|f′(x*)|=1.2>1 的答案是「不會，反而被推開」。所以 0.6875 是個**排斥**不動點——像把一支鉛筆筆尖朝下立在桌上：理論上有個平衡位置，但任何一絲擾動都讓它倒下。穩定的不動點才像碗底的彈珠：推一下會自己滾回去。

血緣（這題的工程橋）：在真實的浮點與有限精度世界裡，「精準的 0.6875」根本存不下來——0.6875 剛好是 2 的冪的和（0.5+0.1875=0.5+3/16），這個還能精確表示，但絕大多數不動點（像 0.6429）是無理或無限循環的，浮點必然有捨入誤差 εₙ。對**排斥**不動點，這個捨入誤差會被每輪乘上 1.2 放大，幾十輪後就把你推離不動點、推進那個 2-cycle。這正是 SDIC（敏感依賴，ch03）的近親，也是為什麼分散式追 bug 要 bit-identical 重放：不穩定的系統裡，最後一位小數的差別會被放大成完全不同的軌跡。「理論上精準就停得住」在工程上等於「停不住」——因為精準是你給不起的。

常見錯路：以為「是不動點」就蘊含「會穩定停留」。本章整章就在拆這個——不動點是必要條件、不是充分條件，充分條件是 |f′(x*)|<1。

## 自我檢核

口頭自答，講得出來才算過關：

1. 不動點的定義是什麼？為什麼說它是回授迴圈的「穩態候選」而不是「穩態」？
2. 穩定判據 |f′(x*)|<1 是怎麼從「偏差每輪乘 f′(x*)」推出來的？為什麼取絕對值？
3. 在 cobweb 圖上，「斜率緩」和「斜率陡」分別對應什麼軌跡行為？分界線為什麼是斜率絕對值等於 1（而不是 0）？
4. 把 f′(x)=r(1−2x) 代入 x*=1−1/r 得到 f′(x*)=2−r——這幾步你能不看書重推一遍嗎？
5. 為什麼穩定區是 1<r<3 這個**有上限**的區間？r 太大為什麼反而失穩（用「增益太高就震盪」的工程直覺講）？
6. r=2 為什麼叫超穩定點？f′(x*)=0 在 cobweb 圖上意味著什麼？
7. r=2.5 的軌跡為什麼是「忽高忽低」地收斂而不是直直滑進去？這跟 f′(x*) 的什麼性質有關？
8. 「x* 是不動點」和「x* 是穩定不動點」差在哪？用 r=3.2 的 x*=0.6875 舉例說明這個差別會怎麼坑你。
