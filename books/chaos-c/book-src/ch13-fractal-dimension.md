# ch13 — 碎維度：介於線與面之間

> **本章解決什麼問題**：上一章你親手造了 Koch 曲線（一段變四段、長度漲到無窮）和 Cantor 集（一段變兩段、長度縮到零），我反覆說它們「比線多、比面少」「卡在維度中間」，但沒給你那個維度數字。這章補上：一個碎形「有多碎」，可以被壓成一個數——而且這個數可以不是整數。我們先用線段、正方形、立方體這些你早就知道維度的東西，把「維度＝份數對放大率取對數」這件事建立起來；再把同一條公式套到碎形上，算出 Koch ≈ 1.26、Cantor ≈ 0.63、Sierpiński ≈ 1.585。最後把這個直覺接回 ch11 那隻勞侖次蝴蝶——它的吸子維度約 2.06，「比面多一點」終於有了精確意思。本章是 Part III 的收尾：混沌的肖像，到這裡有了一個能量化的形容詞。

## 從你已知的出發

你每天都在讀一種「維度」，只是沒這樣叫它。

想你最熟的那件事：**量一個東西的成本怎麼隨「規模」縮放**。你壓測一個服務，把負載從 1×加到 2×，看 P99 怎麼變。如果負載翻倍、P99 也大致翻倍，你會說「這段是線性的」；如果負載翻倍、P99 漲到四倍，你皺眉說「這裡有 O(n²) 的東西」。你判斷的方式，其實是**在問一個指數**：當「輸入」乘上某個倍率，「輸出」乘上的倍率是它的幾次方？線性是 1 次方，平方是 2 次方。這個「幾次方」就是縮放律（scaling law）的核心，而它正是維度的本體。

把它講得更精確一點。你想知道一段程式的成本隨輸入怎麼長，最乾淨的辦法是**畫 log-log 圖**：橫軸放 log(輸入大小)、縱軸放 log(花的時間)。如果點落在一條直線上，那條**直線的斜率就是那個指數**——斜率 1 是線性、斜率 2 是平方、斜率 1.5 是某種介於兩者之間的縮放。你看冪律（power law）分布也是這樣讀的：把資料畫成 log-log，落在直線上、斜率就是冪次（見《馴服隨機》ch12 談冪律）。log-log 圖上的斜率＝縮放指數，這個直覺你已經有了。

本章的整件事，就是把這個你已經會的直覺，套到一個新問題上：**一個形狀，當你把量它的尺度縮小，「量到的份數」怎麼放大？** 答案的指數，就是這個形狀的維度。

```text
    你已經會的：  log(時間)  對  log(輸入)   畫直線，斜率 = 成本的縮放指數
    本章要做的：  log(份數)  對  log(1/尺度) 畫直線，斜率 = 形狀的維度

                              ▲ 同一種「log-log 斜率 = 縮放指數」直覺
```

差別只在：你以前讀的是「時間隨輸入怎麼長」，現在讀的是「測得的份數隨解析度怎麼長」。維度不是什麼玄學，它就是**一條 log-log 直線的斜率**——你看 P99 隨負載縮放、看檔案壓縮率隨資料量縮放時用的同一把尺。理解這一點，「維度可以是 1.26」就不再嚇人：就像你完全能接受「某段程式的縮放指數是 1.5，介於線性和平方之間」一樣。

下面先用整數例把這把尺校準，再去量碎形。

## 維度就是「份數對放大率取對數」

先別碰碎形。我們從三個你閉著眼睛都知道維度的東西出發：線段、正方形、立方體。目標是找出一條公式，**用它去算這三個東西，要剛好得到 1、2、3**——如果公式在已知情況下不對，套到碎形上就沒意義。

玩法是這樣：把一個形狀切成**自己的縮小複製品**，記兩件事——切成幾份（N）、每份相對原來縮小到多少（縮小倍率 s，例如縮小到 1/3 就 s = 1/3）。然後看 N 和 s 之間的關係。

```text
  線段：對半分。
      ●━━━━━━━━●  →  ●━━━●●━━━●
      切成 N = 2 份，每份長度是原來的 s = 1/2     （放大率 1/s = 2）

  正方形：邊長對半 → 變成 4 個小方塊。
      ┌───────┐      ┌───┬───┐
      │       │  →   │ □ │ □ │
      │       │      ├───┼───┤
      └───────┘      │ □ │ □ │
                     └───┴───┘
      切成 N = 4 份，每份邊長是原來的 s = 1/2     （放大率 1/s = 2）

  立方體：邊長對半 → 變成 8 個小立方體。
      切成 N = 8 份，每份邊長是原來的 s = 1/2     （放大率 1/s = 2）
```

現在盯著這三行的數字看，規律自己會跳出來：放大率（1/s）都是 2，但份數 N 是 2、4、8 ——也就是 2¹、2²、2³。**指數剛好是我們認得的維度 1、2、3。**

這不是巧合，是維度的定義在運作。把「線性尺寸放大 k 倍，內容物變 k 的幾次方」這個你早有的直覺寫死：邊長放大 2 倍，線段內容變 2 倍（2¹）、面積變 4 倍（2²）、體積變 8 倍（2³）。那個指數就是維度。倒過來解這個指數——「N 是放大率的 D 次方」即 N = (1/s)^D ——兩邊取對數就得到通式：

```text
            ln N
    D = ─────────         （N 份，每份縮小為原來的 s 倍；放大率 r = 1/s）
          ln(1/s)

    也寫成 D = ln N / ln r，r = 1/s 是放大倍率。對數的底可任選
    （ln、log₁₀、log₂ 都行），因為分子分母同時換底、比值不變。
```

拿它複核三個已知例（放大率 1/s = 2，所以分母都是 ln2）：

```text
  形狀      份數 N   縮小 s   放大率 1/s   D = ln N / ln(1/s)        = ?
  ───────   ──────   ──────   ─────────   ─────────────────────   ────
  線段        2       1/2        2         ln2 / ln2                 = 1
  正方形      4       1/2        2         ln4 / ln2 = 2ln2/ln2      = 2
  立方體      8       1/2        2         ln8 / ln2 = 3ln2/ln2      = 3
```

公式在已知情況下完全正確。線段 1 維、正方形 2 維、立方體 3 維，一個不差。**這就是我們的尺，現在它校準好了。**

停一下，把這把尺的精神講成一句能轉述的話：**維度就是「你把形狀切成 N 份縮小複製品時，份數 N 對放大率 1/s 取的對數」**——也就是「放大率每翻倍，份數翻幾倍」的那個次方。對線是 1 次方、對面是 2 次方、對體是 3 次方。到目前為止，因為我們選的形狀「切幾份」和「縮多少」剛好對得上整數次方，D 是漂亮的整數。

下一節的全部驚嘆，就藏在這個「剛好對得上」被打破的時候。

## 把同一條尺套到碎形上：Koch 與 Sierpiński

碎形和線段、正方形的唯一差別，是它的「份數 N」和「縮小倍率 s」**不再對得上整數次方**。但公式 D = ln N / ln(1/s) 一個字都不用改——它不在乎結果是不是整數。

### Koch 曲線：N = 4，s = 1/3

回想 ch12 的 Koch 規則：把每一段中間三分之一挖掉、換成等邊三角形的兩邊，**一段變四段、每段是原來的 1/3**。用上一節的語言重講一次：整條 Koch 曲線，是由 **4 個** 縮小到 **1/3** 的 Koch 曲線複製品拼成的。所以：

```text
    N = 4         （切成 4 個自己的縮小複製品）
    s = 1/3       （每份縮小到原來的 1/3）
    放大率 1/s = 3

           ln N     ln 4
    D = ─────────  = ─────
         ln(1/s)     ln 3
```

逐步把 ln 值代進去手算（保留 4 位小數，這些值全書統一）：

```text
    ln 4 ≈ 1.3863          （= 2 × ln2 = 2 × 0.6931 = 1.3862，取 1.3863）
    ln 3 ≈ 1.0986

           1.3863
    D  =  ────────  ≈  1.2619
           1.0986

    複核除法：1.0986 × 1.26 = 1.3842，差 0.0021；
              1.0986 × 1.262 = 1.3864，幾乎正中 → D ≈ 1.2619 ✓
```

**Koch 曲線的維度 ≈ 1.2619。** 這是本書第一個非整數維度，值得你停十分鐘消化。

為什麼這個數介於 1 和 2 之間？因為 Koch 曲線**比一條線多、比一塊面少**，而且這個「多／少」現在是可量的：

- 它顯然比 1 維的線「多」——一條普通線段你切成縮 1/3 的份，只會得到 3 份（N = 3 → D = ln3/ln3 = 1，就是普通線）。Koch 卻得到 **4** 份。多出來的那一份，就是它每一級都長出來的尖角、它那無窮無盡的曲折。份數比線多 ⇒ 維度比 1 大。
- 它又比 2 維的面「少」——一塊面你切成縮 1/3 的份，會得到 9 份（3×3 的小方格，N = 9 → D = ln9/ln3 = 2，就是普通面）。Koch 只有 4 份，遠不到 9。它再怎麼皺，也沒皺到填滿一塊面積。份數比面少 ⇒ 維度比 2 小。

```text
    切成「縮 1/3」的份，份數 N 決定維度：

     N = 3  →  D = 1     （普通的線，沒長出新東西）
     N = 4  →  D ≈ 1.26  （Koch：比線多了一份曲折，但遠不到面）
     N = 9  →  D = 2     （普通的面，3×3 填滿）
              ▲
         Koch 的 4 卡在 3（線）和 9（面）之間 ⇒ 維度卡在 1 和 2 之間
```

這就是「碎維度」三個字的全部意思：Koch 比線多、比面少，所以它的維度是 1 和 2 之間的一個分數 1.26。一句能轉述的話——**維度 1.26 量的是「它比一條線多塞了多少曲折進去」，1.26 比 1 多出來的那 0.26，就是它的『碎』有多碎。**

### Sierpiński 三角：N = 3，s = 1/2

再算一個招牌值，這次份數和縮小倍率都換。Sierpiński 三角（Sierpiński triangle）的造法：拿一個實心等邊三角形，連中點、把中間倒過來的那個小三角挖掉，剩下角落 **3 個** 小三角，每個邊長是原來的 **1/2**；對這 3 個再各自挖中間，無窮做下去。

```text
  第 0 代          第 1 代              第 2 代
   ╱╲             ╱╲                  ╱╲
  ╱  ╲           ╱  ╲                ╱__╲
 ╱    ╲    →    ╱____╲      →       ╱╲  ╱╲
╱      ╲       ╱╲    ╱╲            ╱__╲╱__╲
‾‾‾‾‾‾‾‾      ╱__╲  ╱__╲          ╱╲  ╱╲  ╱╲   （中間白洞越挖越多）
              （留 3 個角落，      ╱__╲╱__╲╱__╲
               中間挖空）
```

整個 Sierpiński 三角，是由 **3 個** 縮小到 **1/2** 的自己拼成的（左下、右下、頂上各一個）。所以：

```text
    N = 3         （3 個角落的縮小複製品）
    s = 1/2       （每份邊長縮到一半）
    放大率 1/s = 2

           ln N     ln 3      1.0986
    D = ─────────  = ─────  =  ────────  ≈  1.585
         ln(1/s)     ln 2      0.6931

    複核除法：0.6931 × 1.585 = 1.0985 ≈ 1.0986 ✓
```

**Sierpiński 三角的維度 ≈ 1.585。** 它也介於 1 和 2 之間，而且比 Koch 更靠近 2——這很合直覺：Sierpiński 是從一塊實心的面挖洞挖出來的（出身是 2 維），挖到處處是洞但還沒挖光；Koch 是從一條線長出來的（出身是 1 維），長到處處是曲折但還沒填滿面。一個從面退下來、一個從線爬上去，最後都卡在中間，但 Sierpiński 卡得更高（1.585 > 1.26），因為它身上「面」的成分還比較多。

把 Koch 和 Sierpiński 算法並排，你會看到通式怎麼一視同仁地對待它們：

```text
  碎形            N    s      D = ln N / ln(1/s)              ≈
  ────────────   ──   ────   ────────────────────────────   ──────
  Cantor 集        2   1/3    ln2/ln3 = 0.6931/1.0986          0.6309
  Koch 曲線        4   1/3    ln4/ln3 = 1.3863/1.0986          1.2619
  Sierpiński 三角  3   1/2    ln3/ln2 = 1.0986/0.6931          1.585

  對照（整數，公式照樣對）：
  普通線段         2   1/2    ln2/ln2                          1
  普通面（方格）   9   1/3    ln9/ln3 = 2.1972/1.0986          2
```

Cantor 集那一行我順手算給你看：ln2/ln3 = 0.6931/1.0986 ≈ **0.6309**，這是個**小於 1** 的維度。它合理嗎？非常合理——Cantor 集（ch12）是長度縮到 0、只剩一堆塵的東西，它比一條 1 維的線「少」（份數 N = 2 比普通線的 3 還少），所以維度掉到 1 以下。它又比「一個點」（0 維）多——一個點你怎麼切都還是一個點（N = 1 → D = ln1/ln3 = 0）。Cantor 的 N = 2 卡在 1（點）和 3（線）之間，所以維度卡在 0 和 1 之間。**同一條公式，把「比點多、比線少」和「比線多、比面少」用完全一樣的方式量出來。**

## 盒計數維度：給沒有漂亮自相似規則的形狀量維度

上面那條 D = ln N / ln(1/s) 漂亮，但有個前提：你得**知道形狀的自相似規則**——切成幾份、縮多少。Koch、Cantor、Sierpiński 都是我們親手用規則造的，所以 N 和 s 直接讀得出來。可是真實世界的碎形（海岸線、一片雲、ch11 那隻勞侖次蝴蝶）**沒有人發給你造它的規則**。你手上只有一個畫出來的形狀。怎麼量它的維度？

辦法叫**盒計數維度**（box-counting dimension，又稱 Minkowski–Bouligand 維度），思路土得可愛：**拿格子去蓋它，數要幾個盒子才蓋得住，然後把格子縮小、再數一次。** 看「格子縮小時，需要的盒數怎麼放大」——又回到那個指數。

定義是這樣（D 的下標 box 表示盒計數）：

```text
                  ln N(ε)
    D_box = lim  ─────────         ε = 格子邊長；N(ε) = 蓋住整個形狀所需的盒數
            ε→0   ln(1/ε)
```

它跟自相似通式 D = ln N / ln(1/s) 長得幾乎一樣——把「縮小倍率 s」換成「格子邊長 ε」、把「切成幾份 N」換成「要幾個盒子蓋住 N(ε)」，骨架完全相同。差別只在盒計數**不需要知道規則**，純粹靠「拿格子蓋、數數」，所以對任何畫得出來的形狀都能用。

用一張示意把「格子縮一半、盒數怎麼漲」畫出來。假設我們要蓋住一段 Koch 那種曲折的曲線（用 ⌒ 代表它的鋸齒走向）：

```text
  大格子（邊長 ε）：曲線穿過 4 個格子
  ┌──┬──┬──┬──┐
  │  │░░│░░│  │        N(ε) = 4 個盒子
  ├──┼──┼──┼──┤        （░ = 曲線穿過、要算進去的盒子）
  │░░│  │  │░░│
  └──┴──┴──┴──┘

  小格子（邊長 ε/2，格子數變 4 倍細）：曲線穿過 9 個格子
  ┌─┬─┬─┬─┬─┬─┬─┬─┐
  │ │ │░│░│ │ │ │ │     N(ε/2) = 9 個盒子
  ├─┼─┼─┼─┼─┼─┼─┼─┤
  │ │░│ │ │░│ │ │ │     格子邊長縮一半（1/ε 變 2 倍），
  ├─┼─┼─┼─┼─┼─┼─┼─┤     但盒數沒只變 2 倍（那是線）、
  │░│ │ │ │ │░│ │ │     也沒變 4 倍（那是面），
  ├─┼─┼─┼─┼─┼─┼─┼─┤     而是變約 2.4 倍 → 維度介於 1 和 2
  │ │ │ │ │ │ │░│░│
  └─┴─┴─┴─┴─┴─┴─┴─┘
```

關鍵就在最後那句：**格子邊長縮一半（解析度 1/ε 變 2 倍），盒數變了幾倍？**

- 如果你蓋的是一條普通**直線**，格子縮一半、盒數大約變 **2 倍**（2¹）。
- 如果你蓋的是一整塊**面**，格子縮一半、盒數變 **4 倍**（2²，因為面是二維的，長寬都多一倍格子）。
- 如果你蓋的是 **Koch 那種曲線**，盒數變的倍數**卡在 2 和 4 之間**——大約 2^1.26 ≈ 2.4 倍。那個指數 1.26，就是它的維度。

這就是盒計數的全部精神，而且它把本章開頭那個工程直覺收尾收得乾淨：**把每次量到的 (格子邊長 ε, 盒數 N(ε)) 畫成 log-log 圖，點會落在一條直線上，那條直線的斜率就是維度。** 一模一樣是你讀「P99 隨負載怎麼縮放」「成本隨輸入怎麼長」用的那把尺——log-log 斜率 ＝ 縮放指數 ＝ 維度。

```text
    log N(ε)
      │              ●          直線斜率 = D
      │          ●               （盒數隨解析度的縮放指數）
      │      ●
      │  ●
      └──────────────────  log(1/ε)
         （格子越小 → 右邊；盒數越多 → 上面）
```

盒計數的好處正是它的土：它不問你形狀怎麼來的、有沒有漂亮的自相似規則，只要你能拿格子蓋、能數盒子，就量得出維度。這也是為什麼它（和它的近親們）是實務上量「真實世界碎形」維度的主力——下一節那隻蝴蝶就是靠這類方法量出 2.06 的。

（一個誠實的小註：盒計數維度和自相似維度在「規則漂亮」的碎形上會給出同一個數，但對某些病態的形狀，不同的維度定義可能給出不同的值。最「正統」、數學家最認的那個維度叫 **Hausdorff 維度**，它不靠方格、而是用「能不能用任意小的覆蓋件把形狀蓋住、那些覆蓋件的尺寸的 s 次方總和會不會爆掉」來定義——存在一個臨界的 s，讓這個總和在 s 偏小時是無窮、s 偏大時是零，那個臨界值就是 Hausdorff 維度。這個測度版的精確定義本書不展開，給你方向就好，細節見〈延伸閱讀〉。對本章的所有碎形，這三種維度給的數字一致，你不必區分。）

## 回到那隻蝴蝶：維度 2.06 是什麼意思

現在把這一整套尺，套回 Part III 的主角——ch11 那隻勞侖次（Lorenz）蝴蝶吸子。

ch11 我說過它「比面多一點」，當時那是一句憑感覺的話。現在它有數字了：**勞侖次吸子的碎維度約 2.06**（以 landscape 為準；常引的 Kaplan–Yorke 維度 ≈ 2.062，關聯維度 ≈ 2.055，概括成「≈ 2.06」）。這個吸子活在 3 維的相空間（x, y, z 三個座標）裡，但它本身的維度**不是 3、也不是 2，是 2.06**——卡在 2 和 3 之間，剛好擦過 2 一點點。

「2.06」這個數，把 ch11 那三件看似打架的事一次說圓了：

```text
  吸子的三個性質              維度數字怎麼解釋
  ────────────────────────  ──────────────────────────────────────────
  軌跡被困在有限的盒子裡        維度沒到 3 → 它沒有填滿三維空間，
  （有界）                     而是被壓在一張薄薄的、近乎二維的曲面上

  軌跡永不重複、無窮多層        維度超過 2（多了 0.06）→ 那張「曲面」不是
  地疊在一起（碎形）           乾淨的二維面，它皺、它分層、它每放大一級就
                              冒出新的薄層 → 比一張面「多塞了一點東西」

  軌跡永不自交                 維度 < 3 留下的那點空間，剛好讓無窮多層
  （三維裡不交叉）             薄片彼此錯開、永遠不撞在一起
```

把「2.06」拆開讀最有味道：**整數部分的 2 說「它基本上是一張面」，小數部分的 0.06 說「但這張面皺到、分層到、無窮疊到，比一張乾淨的面多出了 0.06 個維度的『料』」。** 它不是 2（不是乾淨的面），也不是 3（沒填滿空間），它是一張**皺得測量起來『比面多一點』的面**。這就是「奇異吸子是碎形」這句話的精確版——奇異（strange），指的正是它有個非整數的維度。

而這個 2.06 怎麼量出來的？正是上一節的盒計數那一類方法（吸子沒有漂亮的自相似規則可以直接讀 N 和 s，只能拿格子去蓋、數盒子、看 log-log 斜率，或用與之相近的 Lyapunov 維度公式估）。所以你看：從整數例校準的那把尺，到 Koch 的 1.26，到一隻氣象學家在 1963 年算出來的混沌吸子的 2.06——**是同一條 D = ln N / ln(1/s) 的精神，同一種 log-log 斜率直覺，從頭貫穿到尾。**

維度可以不是整數。一旦你接受這件事——而你現在應該接受了，因為你親手算了 Koch 的 1.26、看著它卡在線和面中間——混沌的肖像就完整了：它有形狀（相空間裡的吸子，ch10–11），這形狀是碎形（自相似＋無窮細節，ch12），而這碎形的「碎」可以被壓成一個介於整數之間的數（碎維度，本章）。Part III 到此收尾。

## 直覺的陷阱

碎維度是「聽起來懂、算起來錯、講起來歪」的重災區。三個誤解我見過最多：

```text
  誤解                          會在哪一步出錯                正確版（以 landscape／手算查證）
  ──────────────────────────  ──────────────────────────  ──────────────────────────────
  「維度一定是整數，1.26 是      把 1.26 當成「測量誤差」或    維度本來就可以是分數。D = lnN/ln(1/s)
   近似或四捨五入沒做完」          「還沒收斂到 2」，想著          對碎形算出的就是個無理數（ln4/ln3）。
                                「再細一點會變成整數」          它不是 1 或 2 的近似，它就是 1.26

  「Koch 是 N=3 份」              算成「挖掉中間 1/3，剩兩段     一段被『替換成四段』：左 1/3、右 1/3、
   （把份數數錯）                 加補進去…」隨手寫 N=3 或       加三角形兩邊各 1/3，共 N=4。N 數錯，
                                3 份 → 算出 D=1（普通線！）     維度整個垮掉，這是最致命的計算錯

  「維度大 = 圖看起來複雜／       看到一張很炫的圖就猜它          維度量的是「縮放時份數的增速」，
   面積大」                       維度高，或把維度當成面積        是 log-log 斜率，跟「看起來多複雜」
                                                              「面積多大」是兩回事。Sierpiński 越
                                                              挖洞面積越小，維度卻是固定的 1.585
```

第一個誤解最深，因為它違反你從小到大的算術本能——維度怎麼可能不是整數？這裡值得你停下來，因為你的直覺正要騙你。**自我察覺法**：每當你想說「這個 1.26 應該要收斂成整數」，回去看校準那把尺的三行——線段 N=2 給 D=1、面 N=4 給 D=2，整數是因為「份數剛好對得上放大率的整數次方」這個**巧合**才出現的。Koch 的份數 4 對放大率 3，4 不是 3 的任何整數次方（3¹=3、3²=9），所以指數**必然**落在 1 和 2 之間的某個無理數。整數維度才是特例，分數維度是常態——只是你以前只量過線、面、體這些剛好整數的東西。

第二個誤解是純計算事故，但它最常發生、後果最嚴重。Koch 的 N 是 **4** 不是 3。如果你不小心數成 3，算出 D = ln3/ln3 = 1，就把一個碎形誤判成一條普通線，整章的震撼歸零。守法很簡單：**N 是「替換後的份數」，不是「剩下的份數」。** Koch 把中間那段「換成」三角形的兩邊，不是「留下」——換完數總共幾段：左、右、加三角形的左斜、右斜，四段。每次代公式前，先把這一級的形狀在心裡畫出來、數清楚有幾個縮小複製品，再下筆。

第三個誤解把維度和「複雜度」「面積」混為一談。維度量的是一個非常特定的東西：**縮放時份數的增速**（log-log 斜率），不是「看起來多花俏」，也不是「佔多大面積」。一個鐵證：Sierpiński 三角你挖洞挖得越多代、它的**面積越趨近 0**（每代留 3/4 的面積，(3/4)ⁿ → 0），可是它的**維度從頭到尾固定是 1.585**，一點都不隨「面積縮小」而變。面積和維度是兩個正交的量——維度問的是「碎得多厲害（怎麼縮放）」，不是「佔多大、長多複雜」。把這兩件事分開，你才不會看到一張很滿的圖就喊「維度高」。

## 紙上推演

### 推演題 1 ★ **[10 分鐘]**

不准看上面的表，自己從頭把通式 D = ln N / ln(1/s) 套到三個招牌碎形上，算出數值（ln2≈0.6931、ln3≈1.0986、ln4≈1.3863 給你，其餘自己組）：(a) Koch 曲線（N=4, s=1/3）、(b) Cantor 集（N=2, s=1/3）、(c) Sierpiński 三角（N=3, s=1/2）。算完回答：為什麼 Cantor 的維度小於 1，而 Koch 和 Sierpiński 的大於 1？

#### 推演解答

逐條代入、手算除法複核：

```text
  (a) Koch：       D = ln4 / ln3 = 1.3863 / 1.0986 ≈ 1.2619
                   複核：1.0986 × 1.262 = 1.3864 ✓（正中 1.3863）

  (b) Cantor：     D = ln2 / ln3 = 0.6931 / 1.0986 ≈ 0.6309
                   複核：1.0986 × 0.631 = 0.6932 ✓（正中 0.6931）

  (c) Sierpiński： D = ln3 / ln2 = 1.0986 / 0.6931 ≈ 1.585
                   複核：0.6931 × 1.585 = 1.0985 ≈ 1.0986 ✓
```

為什麼 Cantor 掉到 1 以下、另外兩個在 1 以上？關鍵在「把同一條形狀切成縮小複製品時，份數 N 跟『維持原維度所需的份數』比，是多還是少」：

```text
  以縮小倍率 s 來看，「維持某維度」需要的份數：
    縮 1/3 的線需要 N=3 才湊回一條線；縮 1/2 的線需要 N=2
    縮 1/3 的面需要 N=9 才湊回一塊面；縮 1/2 的面需要 N=4

  Cantor：    s=1/3，N=2 < 3（線的門檻）  → 比線少 → D < 1（=0.63）
  Koch：      s=1/3，N=4 > 3 但 < 9       → 比線多比面少 → 1 < D < 2（=1.26）
  Sierpiński：s=1/2，N=3 > 2 但 < 4       → 比線多比面少 → 1 < D < 2（=1.585）
```

**常見錯路**：(1) 把 Koch 數成 N=3（致命，見〈直覺的陷阱〉），算出 D=1。(2) 把對數的底搞混而慌張——其實 ln、log₁₀、log₂ 算出來一樣，因為分子分母同時換底比值不變；你只要全程用同一個底就好。(3) 把 Sierpiński 的 s 寫成 1/3——它是邊長對半（s=1/2），不是 1/3。

### 推演題 2 ★★ **[12 分鐘]**

設計一個新碎形：把每一段線分成 **5 個** 縮小複製品、每份縮小為原來的 **1/3**。(a) 算它的維度。(b) 它落在 1 和 2 之間的哪裡，比 Koch 更靠近線還是更靠近面？為什麼？(c) 如果改成「分成 9 份、每份縮 1/3」，維度會是多少？這個結果說明了什麼？

#### 推演解答

(a) 直接代通式（N=5, s=1/3, 放大率 1/s=3）：

```text
    D = ln5 / ln3 = 1.6094 / 1.0986 ≈ 1.4650

    （ln5 = ln(10/2) = ln10 − ln2 ≈ 2.3026 − 0.6931 = 1.6095，取 1.6094）
    複核：1.0986 × 1.465 = 1.6094 ✓
```

(b) D ≈ 1.465，比 Koch 的 1.2619 **更大、更靠近面**。直覺：同樣是「每份縮 1/3」，這個新碎形每級長出 5 份、Koch 只長 4 份——份數增速更快，意味著它每縮一級就塞進更多東西，更「貪心」地往填滿面的方向走，所以維度更高。份數 N 越大（在固定 s 下），維度越大。

(c) N=9、s=1/3：

```text
    D = ln9 / ln3 = 2.1972 / 1.0986 = 2   （因為 9 = 3²，ln9 = 2ln3）
```

維度**剛好等於 2**——這正是一塊普通的面（把一個正方形切成 3×3＝9 個縮 1/3 的小方格）。這說明：**碎維度的公式對整數情況完全相容**，當份數剛好對得上放大率的整數次方（9 = 3²），它就退化成你熟悉的整數維度。碎形不是另一套規則，它跟線、面、體用的是同一條公式——只是份數沒對上整數次方而已。

**常見錯路**：把 ln5 記成「5 沒有好算的對數就跳過」——用 ln5 = ln10 − ln2 拆開就好。或誤以為「5 份比 4 份多一份，維度也只多一點點」——維度是對數關係，差別要看 ln5 vs ln4，1.4650 比 1.2619 多了 0.2，不算小。

### 推演題 3 ★★ **[12 分鐘]**

ch11 說勞侖次吸子「比面多一點」，本章給了維度 ≈ 2.06。請用自己的話，把「2.06」這個數的整數部分和小數部分分別解釋成吸子的什麼幾何性質，並說清楚它為什麼**不是** 2、也**不是** 3。

#### 推演解答

把 2.06 拆成「2」和「0.06」兩部分分別讀：

```text
    整數部分 2  →  吸子基本上是一張「面」——軌跡近乎被壓在一張
                  二維的曲面（流形）上跑，不是在三維空間裡到處亂飄

    小數部分 0.06 → 但這張面不乾淨：它皺、它分無窮多層、每放大一級
                  就冒出更薄的新層。這些無窮疊起來的薄層，讓它「比一張
                  乾淨的面多塞了 0.06 個維度的料」→ 這就是它的碎形性
```

為什麼不是 2？因為如果它是乾淨的 2 維面，軌跡疊上去遲早會重複或自交（一張面上的曲線塞久了會撞到自己）。但混沌軌跡永不重複（ch11），它靠的正是那無窮多層薄片——這些薄片讓維度超過 2，多出來的 0.06 就是「比面多的那點碎」。

為什麼不是 3？因為如果維度是 3，它就填滿了整塊三維空間——但軌跡其實被吸進一個很薄的集合裡（吸子是「被吸到的有限區域」），遠沒佔滿空間。維度卡在 2 和 3 之間的那點「沒填滿」的餘裕，剛好給無窮多層薄片彼此錯開、永不相撞的空間（呼應 ch11：三維裡軌跡看似交叉、其實在不同高度錯開）。

**常見錯路**：(1) 把 2.06 聽成「吸子的面積是 2.06」或「它在 2.06 維空間裡」——維度不是面積，相空間還是好端端的 3 維，是吸子**本身**的維度是 2.06。(2) 以為「2.06 ≈ 2，那不就是個面、幹嘛大驚小怪」——關鍵全在那 0.06：差一點點不到 2 就是乾淨的面（會自交、會重複），剛好超過 2 才容得下無窮不重複的薄層。0.06 不是誤差，是混沌的容身之處。

## 自我檢核

口頭自答，講得出來才算過關：

1. 通式 D = ln N / ln(1/s) 裡的 N 和 s 各是什麼？用線段對半（N=2, s=1/2）驗證它給出 D=1。（要點：N＝切成幾個縮小複製品、s＝每份縮小倍率；ln2/ln2=1。）
2. 為什麼正方形切成 4 小方、立方體切成 8 小立方，會分別給出維度 2 和 3？這跟「邊長放大 2 倍、面積變 4 倍、體積變 8 倍」是同一件事嗎？（要點：N=2^D，指數就是維度；是同一件事。）
3. 從頭算 Koch = ln4/ln3，並解釋為什麼它介於 1 和 2 之間。（要點：≈1.2619；N=4 比「線的 3」多、比「面的 9」少。）
4. Cantor 集的維度小於 1（≈0.63），這合理嗎？它跟「點」和「線」的關係是什麼？（要點：合理；N=2 比線的 3 少 → 比線少比點多 → 0<D<1。）
5. 盒計數維度在量什麼？它和自相似通式 D=lnN/ln(1/s) 哪裡一樣、哪裡不一樣？（要點：量「格子縮小時盒數的增速」即 log-log 斜率；骨架同，但不需要知道自相似規則，對任何畫得出的形狀都能用。）
6. 「log-log 圖的斜率就是維度」這句話，跟你平常讀「P99 隨負載怎麼縮放」是不是同一種直覺？講清楚。（要點：是；都是「輸出隨輸入的縮放指數＝log-log 斜率」。）
7. 勞侖次吸子維度 ≈ 2.06，「2」和「0.06」分別代表什麼？為什麼它不是 2、不是 3？（要點：2＝基本是張面、0.06＝皺到分無窮層比面多；不是 2 才容得下不重複、不是 3 才沒填滿空間。）
8. 「維度大就是圖看起來複雜／面積大」哪裡錯？拿 Sierpiński 當反例說。（要點：維度是縮放指數不是複雜度或面積；Sierpiński 面積→0 但維度固定 1.585。）

## 延伸閱讀

- **Mandelbrot, "How Long Is the Coast of Britain? Statistical Self-Similarity and Fractional Dimension," *Science* 卷 156（1967）**——「分數維度」（fractional dimension）這個詞與概念的原始出處（ch12 已引）。本章是它的數字版：讀它如何從「量尺越短、長度越長」的 log-log 直線斜率，正式定義出一個分數維度。（2026-06，卷號 156，常被誤植 165。）
- **Falconer, *Fractal Geometry: Mathematical Foundations and Applications*（碎形幾何標準教科書）**——想看本章誠實標示「不展開」的 Hausdorff 維度測度定義（s 維 Hausdorff 測度如何在臨界 s 從無窮跳到零），以及盒計數維度與 Hausdorff 維度何時相等、何時不等，這本是經典入門。讀第 2–3 章（維度的定義與基本性質）即可，不必啃全書。（2026-06）
- **plus.maths.org, "How to compute the dimension of a fractal"**——用 Koch、Sierpiński 一步步示範 D = ln N / ln(1/s) 與盒計數的科普文章，圖文清楚。想找「比本章更多範例的練手」可讀它，把 N 與 s 多代幾個碎形熟悉公式。（2026-06，線上免費。）
- **Sprott, "Lyapunov Exponent and Dimension of the Lorenz Attractor"（sprott.physics.wisc.edu）**——勞侖次吸子各種維度數字（Kaplan–Yorke ≈ 2.062、關聯維度 ≈ 2.055、容量維度 ≈ 2.00）的權威出處與算法。想知道「2.06」是怎麼從 Lyapunov 指數估出來、為什麼 Kaplan–Yorke 比關聯維度可信，讀它。（2026-06；本章吸子維度即以此與 landscape 為準。）
- **本書 ch14〈Lyapunov 指數〉**——本章把「混沌有形狀、形狀是碎形、碎形有分數維」收尾；下一章換個角度量混沌——不量空間的維度，量**時間**裡相鄰軌跡分離有多快（Lyapunov 指數 λ）。你會發現估勞侖次吸子維度 2.06 用的 Kaplan–Yorke 公式，原料正是那組 Lyapunov 指數——兩章是同一批數字的一體兩面。
