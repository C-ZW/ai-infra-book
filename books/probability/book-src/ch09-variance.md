# ch09 — 變異數與標準差：不確定的大小

> **本章解決什麼問題**：ch08 給了你期望值——隨機變數的「中心在哪」。但只有中心不夠。平均延遲 50ms 的兩個系統，一個穩如老狗、一個三不五時飆到 2 秒，期望值看不出差別。這一章要量的是另一個維度：**散開的程度**——不確定到底有多大。我們會定義變異數（variance）與標準差（standard deviation），講清楚一個會困擾很多人的設計選擇（為什麼用平方而不是絕對值），然後抵達本章真正的寶藏：**獨立的東西相加，變異數可以直接相加**——這條看似平凡的性質，會擠出一個叫 **√n 法則**的東西，而那正是 Part IV 兩大定理（ch13 大數法則、ch14 中央極限）整座大廈的地基。你現在量的 P99−P50、jitter、延遲抖動，講的全是這一章的語言。

## 從你已知的出發

你看 dashboard 從來不只看一個數字。

如果有人跑來跟你說「我們服務的平均延遲是 50 毫秒」，你的第一反應不是點頭，是反問：「P99 呢？」因為你太清楚平均會騙人。兩個服務可以平均延遲都是 50ms，但一個的延遲幾乎每次都貼著 50ms 上下抖一兩毫秒，另一個平常 30ms、可是每隔一陣子就有一發 1500ms 的尾巴把平均拉回 50。對使用者來說，這兩個系統天差地別。平均值——也就是 ch08 的期望值——只告訴你重心在哪，它對「會抖多兇」這件事**完全沉默**。

所以你早就學會了第二個本能：量散開的程度。你用 P99 減 P50 看尾巴拉多長，你給 retry 加 jitter 讓重試時間「散開」避免同步風暴，你盯著延遲的抖動（你可能直接叫它 jitter 或 variance）判斷系統穩不穩。你做容量規劃時，你怕的不是平均負載，是**尖峰**——是那個分布的右尾。

這一章就是把你這個本能變成精確的數學。它在問一個跟「重心在哪」垂直的問題：**這個隨機數字，平常離它的重心有多遠？** 答案有兩個名字——變異數與標準差——而它們之間的關係，加上一條關於「相加」的性質，會直接導出整本書最划算的一個事實：**為什麼把很多次量測平均起來能降噪、而且降噪的速度恰好是 √n。** 你做 A/B test 時隱約知道「樣本要夠多才看得出差異」，你做負載測試時知道「多跑幾輪平均比較準」——這些全是 √n 法則。讀完這章，你會知道那個 √ 是從哪裡長出來的。

我先預告一個結論，因為它太反直覺、值得你帶著疑問讀下去：**把兩個各自會抖 ±3 的誤差加起來，結果不是 ±6，而是 ±4.24 左右。** 誤差會「抵銷一部分」，而且抵銷的數學恰好是這一章的主角。

## 變異數：到平均的平方距離，取期望

先把問題講清楚。我們有一個隨機變數 X，已經（用 ch08 的方法）算出它的期望值 μ = E[X]。現在我們想量「X 平常離 μ 多遠」。

最自然的念頭是：算「偏差」X − μ，看它平均有多大。但這裡馬上撞牆。X 有時比 μ 大（偏差為正）、有時比 μ 小（偏差為負），而 μ 是重心，所以正偏差和負偏差會**剛好抵銷**——E[X − μ] 恆等於 0，對任何隨機變數都一樣。

```text
E[X − μ] = E[X] − μ = μ − μ = 0      ← 永遠是 0，量不出任何東西
           ──────────
           偏差的期望恆為 0，因為 μ 本來就是平衡點（ch08 的重心直覺）
```

這不是巧合，這正是「μ 是重心」的定義。重心的左右力矩相等，所以直接平均偏差，永遠得到 0。要量散開，你得先想辦法**讓正負偏差不要抵銷**。

有兩條路把負號弄掉：取絕對值 |X − μ|，或取平方 (X − μ)²。統計選了平方。於是：

```text
變異數 Var(X) = E[(X − μ)²]
                ──────────────
唸法：「X 到它平均的距離，平方之後，再取期望（平均）」
白話：平常偏離中心多遠，但是用『平方距離』來量
```

每一個偏差先平方（負號消失、而且離得越遠懲罰越重），再按機率加權平均。這個數越大，代表 X 平常離重心越遠，越「散」、越不確定。

一個馬上有用的等價算法，叫**計算公式**，推一次你以後都用它：

```text
Var(X) = E[(X−μ)²]
       = E[X² − 2μX + μ²]              ← 把平方展開
       = E[X²] − 2μ·E[X] + μ²         ← 期望的線性性（ch08），μ 是常數可提出
       = E[X²] − 2μ·μ + μ²            ← E[X] 就是 μ
       = E[X²] − μ²
         ────────────
口訣：「平方的期望，減去，期望的平方」（E[X²] − (E[X])²）
```

`Var(X) = E[X²] − μ²`。這條比定義好算太多——你只要會算 E[X²]（把每個值平方後加權平均）和 μ，相減就好，不用先算一堆偏差。等下算硬幣和骰子都用它。

注意這個式子順帶證明了一件事：E[X²] ≥ μ²，也就是 E[X²] ≥ (E[X])²。因為變異數是平方的期望，**永遠 ≥ 0**（平方不可能是負的，加權平均也不會變負）。變異數等於 0，只發生在一種情況：X 根本不隨機，每次都吐同一個值（一個常數的散開程度當然是 0）。

### 脊椎硬幣：Var = 1/4

來算這枚貫穿全書的硬幣（**這就是 ch01 那枚公正硬幣**）。單擲一次，用指示變數記錄：正面 X=1、反面 X=0，各 1/2。這是最簡單的隨機變數，叫**伯努利（Bernoulli）**。ch08 算過 E[X] = 1/2。現在算變異數，用計算公式：

```text
先算 E[X²]：X 只取 0 或 1，而 0²=0、1²=1，所以 X² 跟 X 是同一個東西
E[X²] = 0²·(1/2) + 1²·(1/2) = 1/2

Var(X) = E[X²] − μ² = 1/2 − (1/2)² = 1/2 − 1/4 = 1/4
                                                  ─────
標準差 SD(X) = √(1/4) = 1/2
```

**單擲硬幣：Var = 1/4，SD = 1/2。** 用定義法交叉驗算一次（不准只信一種算法）：偏差只有兩種，X=1 時偏差 1−1/2 = 1/2、X=0 時偏差 0−1/2 = −1/2，平方都是 1/4：

```text
Var(X) = E[(X−μ)²] = (1/2)²·(1/2) + (−1/2)²·(1/2)
       = (1/4)·(1/2) + (1/4)·(1/2) = 1/8 + 1/8 = 1/4      ✓ 兩法一致
```

順帶記一個結論：伯努利（成功機率 p）的變異數是 **p(1−p)**。代 p=1/2 得 1/2·1/2 = 1/4，這也是 p(1−p) 在 p=1/2 取最大的位置——**公正硬幣是最不確定的硬幣**，偏向任一面（p 接近 0 或 1）變異數都更小，這很合直覺：一枚幾乎總是出正面的硬幣，結果很好猜，散開程度小。

## 為什麼是平方，不是絕對值

這裡值得你停五分鐘，因為這是一個真正的設計選擇，而且這個選擇曾經被兩位大人物吵過架。

絕對值 |X − μ| 其實更直覺。它叫**平均絕對離差（mean absolute deviation, MAD）**，意思白到不行：「平常偏離中心幾單位」，跟資料同單位、不用開根號、小學生都懂。平方距離反而怪——偏離 3 單位，貢獻的是 9；這個「9」是什麼鬼東西，延遲毫秒的平方嗎？單位都變了。

那為什麼統計幾乎全用平方？三個理由，由弱到強：

**理由一：數學乖巧（可微分）。** 絕對值函數 |x| 在 x=0 那個尖點不可微分，平方函數 x² 處處平滑。當你要「找一個值讓總偏差最小」（這是統計、最佳化、機器學習天天在做的事——最小平方法、損失函數），平方給你乾淨的、唯一的解析解；絕對值給你一個有尖角、可能不唯一的解。你做梯度下降時偏好平方損失（MSE）不是偶然，是因為它**處處有漂亮的梯度**。

**理由二，也是本章的真正理由：平方距離可以相加，絕對值不行。** 這個我們下一節專門講——它是 √n 法則的引擎。先記住一句：**獨立的東西相加，變異數（平方距離）直接相加；但「絕對偏差」加起來沒有這麼乾淨的法則。** 這是平方碾壓絕對值的決定性優勢，整章後半都靠它。

**理由三：對常態分布是「最有效率」的估計。** 這條把我們帶進一段精彩的學術吵架。

### 一場吵了七年的架（Eddington vs Fisher）

1914 年，天文學家亞瑟·愛丁頓（Arthur Eddington）在他的書《Stellar Movements》裡寫道：算一串觀測的誤差時，用「絕對殘差的平均」（MAD）比用「平方殘差的平均」（也就是變異數那一路）**又實際又理論上更好**（2026-06 查證）。愛丁頓是實務派，他覺得絕對值單純、抗離群值，何必平方。

年輕的羅納德·費雪（Ronald Fisher）不同意。1920 年他用一個他剛發明的武器——統計量的**效率（efficiency）**——正面回擊（2026-06 查證）。費雪證明：**如果資料服從常態分布，要達到同樣的精度，用 MAD 估母體散開程度需要的樣本量比用標準差多約 14%。** 換句話說，同樣一批資料，標準差「榨出的資訊」更多、估計更緊。對常態世界，平方贏，而且贏得有數學憑據。費雪這一仗奠定了標準差成為統計預設的地位。

但這故事有個你該知道的尾巴，因為它正是「直覺的陷阱」的好料：1960 年，圖基（Tukey）回頭確認了愛丁頓當年的直覺其實也對——**只要分布稍微偏離常態（多一點點重尾），MAD 反而比標準差穩**（2026-06 查證）。而真實世界的分布，幾乎沒有真正常態的。所以這場架的正確結論不是「平方永遠對」，是：**平方對常態最有效率、數學最乖巧，但它對離群值（大偏差）的懲罰是平方級的，碰到重尾世界它會被極端值帶著走。** 你用平均延遲的「標準差」描述一個重尾的延遲分布時，那幾發 1500ms 的尾巴會被平方放大、主宰整個變異數——這正是你該警覺的時候。記住這個尾巴，ch12 講長尾、ch13 講大數法則的前提時還會回來。

本書接下來跟著主流用平方（變異數、標準差），但你現在知道這是個有取捨的選擇，不是天上掉下來的真理。

## 標準差：把單位扳回來

變異數有個惱人的毛病：單位是平方的。延遲以毫秒計，變異數的單位是「毫秒²」——一個沒有物理意義的東西。Var = 1/4 的那枚硬幣，這個 1/4 是「正面數的平方」，你沒辦法拿它跟「正面數」本身比大小。

解法簡單到近乎廢話：**開根號開回來。**

```text
標準差 SD(X) = √Var(X) = √(E[(X−μ)²])
              ─────────
跟資料同單位的『散開尺度』。Var 量的是「平方距離」，SD 把它扳回「距離」
```

標準差才是你真正能拿來「讀」的那個數。它跟資料同單位（毫秒就是毫秒），它的意思是「X 平常離重心大約幾單位」。硬幣的 SD = 1/2，意思是單擲的正面數平常離重心 1/2（0 或 1，離 0.5 都是 0.5，這裡剛好精確）。你在 SRE 場景講的「幾個標準差之外」「3-sigma 告警」，用的就是這把尺——σ 是希臘字母 sigma，全書用它代表標準差，σ² 代表變異數。

```text
記號約定（全書一致）：
σ  = SD(X) = 標準差    ← 跟資料同單位，是你拿來讀的那個
σ² = Var(X) = 變異數   ← 平方單位，數學上好操作的那個
μ  = E[X]  = 期望值（中心，ch08）
```

一句話定位三者的分工：**μ 說中心在哪，σ² 是數學上好算好加的那個量，σ 是你拿來跟人講的那個量。** 平常溝通用標準差（同單位、可解讀），背後算術用變異數（可加、可微）。它們只差一個平方/開根號，但這個 √ 的非線性，馬上要製造本章最大的反直覺。

## 變異數相加：本章的引擎

現在進入這一章真正值錢的地方。如果你只帶走一個結論，帶這個。

問題：兩個隨機變數 X 和 Y，你把它們加起來得到 X+Y。X+Y 的變異數是多少？

期望值那邊（ch08）答案漂亮得不像話：E[X+Y] = E[X] + E[Y]，**而且不需要任何條件**，獨不獨立都成立（期望線性性）。那變異數呢？會不會也這麼好？

答案是：**只要 X 和 Y 獨立，變異數就直接相加。**

```text
若 X、Y 獨立：  Var(X + Y) = Var(X) + Var(Y)
               ─────────────────────────────
獨立的東西相加，「平方距離」直接相加——這就是平方擊敗絕對值的決定性理由
```

這條叫**比奈梅公式（Bienaymé's formula）**——對，又是那位比奈梅，他 1853 年就寫下了這條變異數相加律，比車比雪夫 1867 年的不等式還早（2026-06 查證；landscape 提過這對「兩個年份」的歸屬，這裡是同一位先生的另一項功勞）。它其實只要「不相關」就成立（比獨立弱一點的條件），但獨立一定不相關，所以你先記「獨立 → 變異數相加」這個夠用的版本。完整的、連相關都算進去的版本長這樣，留給 ch10：

```text
一般情形（含相關項，ch10 補完）：
Var(X + Y) = Var(X) + Var(Y) + 2·Cov(X, Y)
                                ────────────
Cov 是共變異數（covariance）。X、Y 獨立時 Cov = 0，那一項消失，回到上面的乾淨版
```

為什麼獨立能讓平方距離乾淨相加，這裡給**直覺版**（嚴格證明把上面展開、用獨立讓交叉項 E[(X−μ_X)(Y−μ_Y)] = 0，本書不逐步推、指向延伸閱讀）：把偏差想成向量。X 的偏差和 Y 的偏差如果獨立，它們「方向無關」，就像兩條互相垂直的向量。垂直向量相加，長度遵守**畢氏定理**——斜邊的平方等於兩股的平方和。變異數是「偏差長度的平方」，所以它直接相加；而標準差是「長度」本身，長度不會直接相加（直角三角形的斜邊不等於兩股之和）。**這就是為什麼變異數加、標準差不加。** 那個 2Cov 項，就是兩向量不垂直時的餘弦補正（餘弦定理）。你心裡記這張圖：

```text
                          σ(X+Y) = √(σ_X² + σ_Y²)   ← 斜邊（標準差）
                     ┌────────/
                     │      /
              σ_Y    │    /                  變異數（平方）：σ_X² + σ_Y²  ← 直接相加
            （股）   │  /                     標準差（長度）：要開根號，不能直接加
                     │/  ) 直角＝獨立
                     └──────────
                        σ_X（股）
```

### 為什麼 ±3 加 ±3 不是 ±6

回到開頭那個反直覺。兩個獨立的誤差源，各自標準差 σ = 3（口語的「±3」）。加起來的標準差是多少？

直覺喊 6。錯。正確做法：**變異數先加，再開根號回標準差。**

```text
Var(X) = 3² = 9，  Var(Y) = 3² = 9
Var(X+Y) = 9 + 9 = 18                  ← 變異數相加（獨立）
SD(X+Y) = √18 = √(9·2) = 3√2 ≈ 4.243   ← 開根號扳回標準差
                                          ──────
不是 6，是 4.24。誤差「抵銷」了一部分
```

**±3 加 ±3 是 ±4.24，不是 ±6。** 為什麼小於 6？因為兩個獨立誤差不會總是「同方向一起變大」——有時 X 偏正、Y 偏負，互相抵掉一些。它們**部分抵銷**，所以合起來的抖動比單純相加小。標準差會直接相加（變成 6）只發生在一種極端情況：兩者完全正相關（Cov 最大，永遠同方向同幅度一起動）——那它們根本不是獨立，是同一回事複製兩份。

這個「不是 6 是 4.24」就是 √2 第一次冒頭。把它推廣到 n 個，√ 就會變成 √n，整本書後半的主角登場。

## √n 法則：平均為什麼比個體穩 √n 倍

把上面那條相加律套在「n 個一樣的東西相加」上，會擠出一個漂亮到讓人起雞皮疙瘩的結論。

設你有 n 個**獨立同分布（iid）**的隨機變數 X₁, X₂, …, Xₙ，每個的期望都是 μ、變異數都是 σ²（標準差 σ）。「獨立同分布」就是工程裡的「同一個隨機過程重複跑 n 次、彼此不影響」——重複量測同一個延遲、重複擲同一枚硬幣、重複抽同一個母體。

**先看「和」** Sₙ = X₁ + … + Xₙ：

```text
Var(Sₙ) = Var(X₁) + … + Var(Xₙ) = n·σ²      ← 變異數相加，n 個 σ²
SD(Sₙ)  = √(n·σ²) = σ·√n                      ← 開根號，√n 跑出來了
          ────────────────────
和的標準差 = σ·√n：加越多項，總抖動以 √n 的速度長大（不是 n 倍）
```

注意這個微妙之處：你加了 n 項，「總和」的期望是 n·μ（線性長大，n 倍），但「總和」的標準差只長 √n 倍。**和的中心跑得比和的抖動快。** 擲 100 次硬幣，正面數的期望是 50（中心在 50），標準差只有 √100/2 = 5——抖動只有中心的十分之一。擲越多，這個比例越懸殊，這是 ch13 大數法則的種子。

**再看「平均」** X̄ = Sₙ / n。平均就是把和除以 n，而「除以常數 n」對變異數的影響是「除以 n²」（因為變異數是平方尺度，常數要連平方一起出來）：

```text
縮放規則：Var(aX) = a²·Var(X)        ← 常數 a 提出時要平方（平方尺度）

Var(X̄) = Var(Sₙ / n) = (1/n²)·Var(Sₙ) = (1/n²)·(n·σ²) = σ²/n
SD(X̄)  = √(σ²/n) = σ/√n
         ──────────
平均的標準差 = σ/√n：這個數有個專名，叫『標準誤（standard error）』
```

**平均的標準誤 = σ/√n。** 這是本章的皇冠。它在說：你把 n 次量測平均起來，這個平均值的抖動是**單次抖動的 1/√n**。量 4 次取平均，誤差砍一半（√4=2）；量 100 次取平均，誤差變 1/10；量 10000 次，1/100。

**「平均比個體穩 √n 倍」**——這一句話就是降噪的全部數學。你做負載測試多跑幾輪取平均比較準、你做 A/B test 要累積夠多樣本才看得出小差異、你把雜訊量測平均掉，全都是在用 σ/√n。為什麼平均能降噪？因為各次量測的隨機誤差**部分抵銷**了（跟 ±3+±3=±4.24 同一個抵銷），而抵銷的速度恰好是 √n。

這也立刻告訴你一個殘酷的工程現實，叫**報酬遞減**：

```text
誤差要砍一半（÷2），樣本要 ×4（因為 √4 = 2）
誤差要砍到 1/10，樣本要 ×100
誤差要砍到 1/100，樣本要 ×10000
            ────────────────────────
√n 是個慢慢長的函數：你想讓估計更準，代價是平方級暴漲的樣本
```

這就是為什麼 A/B test 要偵測「越小的提升」就需要「越多得離譜的流量」——你想把可偵測的差異減半，樣本得翻四倍。這不是你的測試框架爛，是 √n 的鐵律。民調為什麼總是 ±3%、為什麼砸更多錢也很難壓到 ±1%（要九倍受訪者），同一個道理，ch16、ch17 會把這條算到底。

## Worked example：脊椎硬幣的 √n 法則，「和變寬、平均變窄」

把上面全部套回脊椎硬幣，做一次完整的數字對照。這是 ch13「同一組數據兩種讀法」最重要那一頁的彩排，請你跟著算。

**設定**：擲一枚公正硬幣 n 次，X = 正面數。每一擲是伯努利，Var = 1/4、SD = 1/2（前面算過）。n 擲是 n 個獨立伯努利相加，所以：

```text
正面數 X 的期望：E[X] = n · (1/2)  = n/2
正面數 X 的變異：Var(X) = n · (1/4) = n/4         ← 變異數相加
正面數 X 的標準差：SD(X) = √(n/4)   = √n / 2
                                       ──────
（這組數字 ch11 二項分布、ch14 中央極限會一字不差地再用一次）
```

**「正面比例」p̂ = X/n** 的標準差（把上面的 X 除以 n，變異數除以 n²）：

```text
Var(p̂) = Var(X)/n² = (n/4)/n² = 1/(4n)
SD(p̂)  = √(1/(4n)) = 1/(2√n)
         ───────────
比例的標準差 = 1/(2√n)
```

現在把 n=100 和 n=10000 並排，看「和（正面數）」和「平均（比例）」兩種讀法朝相反方向跑：

```text
            正面數 X（和的讀法）              正面比例 p̂（平均的讀法）
  n        期望     標準差 SD(X)=√n/2          期望     標準差 SD(p̂)=1/(2√n)
─────────────────────────────────────────────────────────────────────────
  100       50       √100/2 = 5               0.5       1/(2·10)  = 0.05
 10000     5000      √10000/2 = 50            0.5       1/(2·100) = 0.005
─────────────────────────────────────────────────────────────────────────
變化      ×100倍     ×10倍（√100=10）          不變      ÷10倍（變窄）
          中心       抖動長大，但比中心慢                 抖動縮小
```

盯著這張表看十秒，因為它同時藏了 ch13 全書最反直覺的對帳：

- **和（正面數）的抖動「變寬」**：n 從 100 到 10000（×100），正面數的標準差從 5 漲到 50（只 ×10，因為 √100=10）。**絕對的偏離越擲越大**——擲一萬次，正面數典型會偏離 5000 約 ±50；如果換算成「正面減反面」的差，那個差的標準差是 2×50=100，所以「正面減反面」典型在 ±100 上下。擲越多次，正面和反面的**絕對差距越拉越開**（∝√n），不是越來越接近。
- **平均（比例）的抖動「變窄」**：同樣 n 從 100 到 10000，比例的標準差從 0.05 縮到 0.005（÷10）。**比例越擲越貼近 0.5**——擲一萬次，正面比例典型只偏離 0.5 約 ±0.005，幾乎釘死在一半。

**同一組擲幣，同一個 √n，往兩個相反方向講故事。** 看「正面減反面」，你會說「天哪差距越來越大，混沌」；看「正面比例」，你會說「穩得不得了，收斂到 0.5」。兩個都對，因為一個除了 n、一個沒有。賭桌上喊「開了五次紅、該開黑了」的人，腦子裡用的是「差距會自己縮回去」的錯誤模型——但 √n 法則告訴你**絕對差距不會縮、只會漲**；縮的是「比例」，而比例縮是因為被後面大量的擲幣**稀釋**，不是因為老天爺派反面來「補」。這就是賭徒謬誤的數學核心，ch13 整章在拆它。本章你只要先把這張「和變寬、平均變窄」的表刻進腦子。

一句話收尾這個 worked example：**√n 是同一枚硬幣的兩張臉。** 它讓和發散、讓平均收斂，差別只在你有沒有除以 n。

## 直覺的陷阱

| 你以為 | 實際上 | 怎麼自我察覺 |
|---|---|---|
| 兩個各 ±3 的誤差加起來是 ±6 | 是 **±4.24**（√(9+9)=√18）。**變異數相加，標準差不加**——獨立誤差會部分抵銷 | 你把標準差直接相加時——停。先問「我加的是變異數還是標準差」。標準差過了 √ 這道非線性的牆，不能直接加 |
| 把樣本數從 100 加到 400，誤差大概也少很多 | 誤差只**砍一半**（√400/√100 = 2）。誤差要砍一半要四倍樣本，砍到 1/10 要一百倍 | 你期待「樣本翻倍、誤差減半」——錯，那要翻四倍。是 √n 不是 n，報酬遞減得很兇 |
| 擲越多次，正面和反面的「數量」會越來越接近（老天爺會平衡） | 正面減反面的**絕對差距越擲越大**（∝√n）。收斂的是**比例**不是**差距**（賭徒謬誤的根） | 你說「開太多紅了該開黑」——這假設了「差距會自我修正」。√n 法則說差距只漲不縮，縮的是被稀釋的比例（ch13 引爆） |
| 平均沒變，系統就沒變糟 | 平均不動、**變異數可以爆掉**。P99 飆高但平均紋風不動，正是「中心沒變、散開變大」 | 你只盯平均延遲說「沒事」——尾巴（變異/P99）可能已經爛了。期望和變異是兩個垂直的維度，要分開看 |
| 變異數的大小可以直接拿來「讀」這個量散多開 | 變異數是**平方單位**（毫秒²），不能直接讀。要讀就看**標準差**（開根號扳回同單位） | 你拿一個變異數數字說「散開 25」——25 什麼？毫秒²。先開根號變 5ms 才是人話 |
| 用標準差/變異數描述任何分布都很穩當 | 對**重尾**分布，平方會被極端值主宰（一發 1500ms 尾巴貢獻 1500² 進變異數）。費雪當年也只證了「對常態」最有效率 | 你算延遲的標準差時它大得離譜又不穩——可能不是 bug，是重尾。平方對離群值的懲罰是平方級的（ch12 長尾會回來） |

招牌謬誤點名：**標準差相加。** 這是本章最常見、連工程師都會犯的翻車——把兩個 ±3 加成 ±6，把「誤差合成」當加法。記住那道牆：**變異數活在平方世界（可加），標準差活在線性世界（要過 √ 才能回去）。** 任何時候你想把兩個「±幾」加起來，先各自平方、相加、再開根號。這一個動作，會擋掉你估「誤差預算」「合成不確定度」時九成的錯。

## 紙上推演

> 全程紙筆，不寫程式。每題先自己推，再看解答。變異數題最容易在「平方該不該乘進去」和「加的是變異數還是標準差」這兩處手滑——每個數值至少用兩種方式對一次。

### 推演題 1：一顆公正骰子的變異數 **[10 分鐘]** ★★

一顆公正六面骰，X = 點數（1 到 6，各 1/6）。

(a) 用計算公式 Var(X) = E[X²] − μ² 算變異數，再開根號得標準差。
(b) ch08 算過 μ = E[X] = 3.5；驗證 E[X²] 要用「每個值平方後加權平均」，不是「μ 的平方」。把這兩者的差別講清楚（為什麼 E[X²] ≠ (E[X])²，差的那一塊就是變異數）。

#### 推演解答

(a) 先算 E[X²]：把 1 到 6 各自平方，加權平均（每個權重 1/6）：

```text
E[X²] = (1² + 2² + 3² + 4² + 5² + 6²) / 6
      = (1 + 4 + 9 + 16 + 25 + 36) / 6
      = 91 / 6 ≈ 15.1667

μ = 3.5，所以 μ² = 3.5² = 12.25 = 49/4

Var(X) = E[X²] − μ² = 91/6 − 49/4
       = 182/12 − 147/12      ← 通分到 12
       = 35/12 ≈ 2.9167

SD(X) = √(35/12) ≈ √2.9167 ≈ 1.708
        ─────────
骰子：Var ≈ 2.917，SD ≈ 1.71（標準差比硬幣的 0.5 大很多——骰子比硬幣「散」）
```

交叉驗算（定義法，挑兩個偏差驗一下趨勢）：偏差是 X−3.5，取值 −2.5, −1.5, −0.5, 0.5, 1.5, 2.5，平方是 6.25, 2.25, 0.25, 0.25, 2.25, 6.25，加權平均：

```text
(6.25 + 2.25 + 0.25 + 0.25 + 2.25 + 6.25) / 6 = 17.5 / 6 ≈ 2.9167   ✓ 一致
```

(b) E[X²] ≈ 15.17 是「先平方每個值、再平均」；(E[X])² = 12.25 是「先平均、再平方」。**兩者順序不同，差出來的 2.92 正是變異數。** 直覺上 E[X²] 一定 ≥ (E[X])²，因為平方是凸函數，「先平方再平均」會被大值的平方拉高（這叫 Jensen 不等式，本書不展開、知道方向就好）。差的那一塊 E[X²] − μ² 永遠 ≥ 0，這正是「變異數不會是負的」的另一個看法。把這兩者搞混（用 μ² 當 E[X²]）會直接把變異數算成 0，是這題最常見的錯路。

### 推演題 2：為什麼 100 增到 400 只把誤差減半 **[10 分鐘]** ★★

你在做負載測試，每輪量一次平均延遲，重複 n 輪後取總平均當估計。單輪量測的標準差是 σ。

(a) 寫出「n 輪平均」的標準誤公式，說明它跟 σ、n 的關係。
(b) 把 n 從 100 加到 400，誤差變成原來的幾分之幾？要把誤差砍到原來的 1/10，n 要多大？
(c) 用一句話向同事解釋「為什麼多跑三倍的輪數（100→400）只換到誤差減半」。

#### 推演解答

(a) n 輪 iid 量測，每輪標準差 σ。平均的標準誤：

```text
SE = σ / √n      ← 這就是 √n 法則的平均版
```

(b) 誤差跟 √n 成反比，所以比較兩個 n 只要比 √n：

```text
n: 100 → 400 時，誤差比 = √100 / √400 = 10 / 20 = 1/2     ← 砍一半

要誤差變 1/10：需要 √n 變 10 倍 → n 變 100 倍
原本 n，新的 n' = 100·n。若從 100 起跳，要到 10000 輪
            ────────────
報酬遞減：√n 慢慢長，精度的代價是平方級暴漲的樣本
```

(c) 「因為降噪靠的是 √n 不是 n——誤差是 σ/√n，n 翻成四倍，√n 才翻兩倍，所以四倍的輪數只換到誤差減半。想再砍一半，得再翻四倍。」交叉驗算：100→400 是 ×4，√4=2，誤差 ÷2，相符。這也是 A/B test 要偵測越小的差異就要越多流量的同一條鐵律。

### 推演題 3：用「變異數相加、標準差不加」反駁 ±6 **[15 分鐘]** ★★★

兩個獨立的延遲來源（例如兩個串接的微服務），各自延遲的標準差都是 3ms。有人說「串起來總延遲的抖動就是 3+3 = ±6ms」。

(a) 算出總延遲（兩者相加）的正確標準差。
(b) 指出「3+3=6」錯在哪一步——它偷偷假設了什麼？
(c) 在什麼極端情況下，總抖動真的會逼近 ±6？那個情況跟「獨立」差在哪？

#### 推演解答

(a) 變異數先加，再開根號：

```text
Var(各) = 3² = 9
Var(總) = 9 + 9 = 18         ← 獨立，變異數相加（比奈梅）
SD(總)  = √18 = 3√2 ≈ 4.24ms
          ─────────
正確答案 ±4.24ms，不是 ±6ms
```

(b) 「3+3=6」是把**標準差**直接相加。但標準差不能加——能加的是變異數（平方距離）。標準差過了 √ 這道非線性的牆，加法在那邊不成立。把 SD 相加，等於偷偷假設「兩個誤差永遠同方向、同幅度一起動」，也就是**完全正相關**——那根本不是獨立。獨立的兩個誤差會**部分抵銷**（有時一正一負互消），所以合起來比 6 小。

(c) 只有當兩者**完全正相關**（相關係數 ρ=1，永遠手牽手同向同幅）時，總抖動才真的是 6：

```text
完全正相關：Var(總) = Var(X)+Var(Y)+2Cov(X,Y)，Cov 取最大 = σ_X·σ_Y = 9
           Var(總) = 9 + 9 + 2·9 = 36，SD = √36 = 6
                                          ─────
這時標準差才真的相加（3+3=6），但這代表兩者根本是同一個東西複製兩份
```

跟獨立差在那個 2Cov 項：獨立時 Cov=0（那一項消失，得 √18≈4.24）；完全正相關時 Cov 拉到最大（得 6）。真實系統多半落在中間。重點是：**預設獨立、把變異數相加，是合理起點；但把標準差相加（當成完全正相關）幾乎一定高估抖動**——除非你的兩個延遲源真的是被同一個瓶頸鎖死的（那它們本來就不該當兩個獨立來源算）。這條 2Cov 的完整故事在 ch10。

### 推演題 4：和變寬、平均變窄 **[20 分鐘]** ★★★

脊椎硬幣，擲 n 次，X = 正面數。

(a) 寫出 X 的期望、變異數、標準差（用 n 表示），並算 n=400 的三個值。
(b) 寫出比例 p̂ = X/n 的標準差，算 n=400 的值。
(c) 把 n=400 跟 n=100 比：正面數的標準差變大幾倍？比例的標準差變小幾倍？用這個對照講清楚「為什麼同一組擲幣，和在發散、平均在收斂」。

#### 推演解答

(a) n 個獨立伯努利相加：

```text
E[X]   = n/2
Var(X) = n/4
SD(X)  = √n / 2

n=400：E[X] = 200，Var(X) = 100，SD(X) = √400/2 = 20/2 = 10
```

(b) 比例 = X 除以 n，變異數除以 n²：

```text
Var(p̂) = (n/4)/n² = 1/(4n)
SD(p̂)  = 1/(2√n)

n=400：SD(p̂) = 1/(2·√400) = 1/(2·20) = 1/40 = 0.025
```

(c) 跟 n=100 對照（n=100 時 SD(X)=5、SD(p̂)=0.05，前面 worked example 算過）：

```text
              n=100      n=400      變化
─────────────────────────────────────────────
正面數 SD(X)    5          10        ×2 倍（變大，√4=2）
比例   SD(p̂)   0.05       0.025      ÷2 倍（變小）
─────────────────────────────────────────────
n 從 100 到 400（×4），√n 變 2 倍：
  和的抖動 ∝ √n         → ×2，發散（絕對偏離越擲越大）
  平均的抖動 ∝ 1/√n     → ÷2，收斂（比例越擲越貼 0.5）
```

正面數的標準差變大 2 倍（√4=2），比例的標準差變小 2 倍。**同一個 √n，因為一個乘 n、一個除 n，往相反方向跑。** 講法：「正面數」是 n 個東西的**和**，項數越多總抖動越大（√n 漲），所以絕對差距發散；「比例」是那個和**除以 n**，分母 n 壓過了分子的 √n（淨效果 1/√n），所以收斂。賭徒看「正面減反面」覺得混沌（那是和、在發散），統計學家看「正面比例」覺得秩序（那是平均、在收斂）——兩人盯著同一桌牌。這正是 ch13 大數法則要正式引爆的「同一數據兩種讀法」，你已經先親手算過一遍了。

## 自我檢核

口頭自答，講得清楚才算過。優先答「為什麼」。

1. 為什麼直接平均「偏差 X−μ」量不出散開程度？這個「永遠是 0」的事實，跟「μ 是重心」是同一件事嗎？
2. 變異數為什麼用平方而不是絕對值？至少講出兩個理由（可加性、可微分），並說出費雪當年贏了愛丁頓、但圖基後來替愛丁頓平反了什麼——這個尾巴跟「重尾分布」有什麼關係？
3. 變異數和標準差的分工是什麼？為什麼日常溝通用標準差、背後算術用變異數？「單位」在這裡扮演什麼角色？
4. 「變異數相加但標準差不加」——為什麼？用畢氏定理／垂直向量那張圖講給人聽，並解釋「±3 加 ±3 為什麼是 ±4.24 不是 ±6」。
5. √n 法則的兩個版本（和的 SD = σ√n、平均的標準誤 = σ/√n）怎麼從「變異數相加」推出來？為什麼「除以 n」會變成「變異數除以 n²」？
6. 「把樣本從 100 加到 400 只把誤差減半」——這個報酬遞減的數學是什麼？它怎麼解釋「A/B test 要偵測越小的差異就要越多流量」？
7. 脊椎硬幣擲 n 次：為什麼「正面減反面」的差距越擲越大（發散），但「正面比例」越擲越貼近 0.5（收斂）？同一組數據，這兩個讀法為什麼不矛盾？（這題講得清楚，你就提前拿到 ch13 的鑰匙了。）
8. 「平均延遲沒變，所以系統沒變糟」——這句話漏看了哪個維度？用「期望和變異是兩個垂直方向」回答，並連到你看 P99 的工程本能。

## 延伸閱讀

- **Eddington vs Fisher 的 MAD vs SD 之爭**（2026-06 查證）。想看「為什麼是平方」這個設計選擇被真人吵過架，搜尋 Eddington 1914《Stellar Movements》的 MAD 主張、Fisher 1920 用 efficiency 反擊（常態下 MAD 需多約 14% 樣本）、以及 Tukey 1960 替 Eddington 平反（稍偏離常態 MAD 反而穩）。Gorard 2005 在《British Journal of Educational Studies》有一篇「重訪這場 90 年老辯論」的回顧，是把這故事講完整的好入口——讀它你會明白標準差是個有取捨的預設，不是真理。
- **比奈梅（Bienaymé）1853 的變異數相加律**。本章引擎的歷史出處。比奈梅比車比雪夫早寫下「獨立變數和的變異數等於變異數之和」這條，後人正名為 Bienaymé's formula（landscape 提過這對「兩個年份」的歸屬之爭，這裡是同一位先生的功勞）。想看它如何直接餵給弱大數法則，留到 ch13 的車比雪夫不等式，那裡會看到「只靠變異數」就能把收斂壓出來。
- **任何扎實機率教材的「變異數與獨立和」一節**（如 Blitzstein & Hwang《Introduction to Probability》，網路有免費 PDF 與哈佛 Stat 110 課程影片）。如果你想把「Var(X+Y)=Var(X)+Var(Y)+2Cov」的展開、交叉項為何在獨立時歸零，一步步看清楚（本章只給了畢氏定理的直覺版），看它的推導；但記住本書立場——你要的是「為什麼平方距離會相加」的直覺，不是背公式。
- **本書內部銜接**：那個被本章丟掉的 2Cov(X,Y) 項是 ch10 的主角，共變異與相關（correlation）會把「不獨立時變異數怎麼加」補完，並回答「不相關是不是就等於獨立」。本章埋的 √n 法則是 Part IV 的地基——ch13 大數法則用它解釋「比例為何收斂」，ch14 中央極限用「平均 ~N(μ, σ²/n)」把這個 √n 標準誤接到鐘形上。而「平方對重尾不穩」這個尾巴，ch12 講長尾、ch13 講大數法則前提時會回來算帳。
