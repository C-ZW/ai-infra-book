## 從你已知的出發

你天天在跟「沿時間累積的隨機」打交道，只是沒把它叫做隨機漫步。

最貼的例子是 **exponential backoff + jitter**。一個請求失敗了，你不會立刻重試（會打出重試風暴），你退避一段時間再來；失敗就把退避時間加倍，而且加上一個隨機抖動避免所有 client 同步。把「這一輪退避時間」想成一條沿著重試次數走的軌跡——它在跳、有隨機成分、會累積。你早就直覺地知道：這條軌跡**沒有記憶**，第五次重試的退避不會因為前四次失敗就「比較容易成功」。系統不欠你一個成功。記住這句話，本章後半的賭徒謬誤就是它的數學版。

再看**佇列長度**。一個 work queue，每秒有任務進來、有 worker 消化。進出速率大致打平的時候，佇列長度不是穩穩停在某個值——它在一條看不見的中線附近**隨機漂移**，這秒進得多漲一格、下秒消化得快跌一格。你盯著 dashboard 上那條鋸齒線，心裡其實在問一個隨機漫步的問題：「它會不會剛好一路漲、漲到爆掉？」會的，只是機率多大、要等多久——這正是這一章要算的東西。

還有一個更貼錢包的：**攤平加碼**。某個投資跌了，有人說「再買一點拉低均價，反彈就回本了」;跌更多，「再加碼，這樣反彈一點點就解套」。這套操作有個古老的賭桌名字叫「翻本加倍法」，它在數學上有一個極乾淨的結論：**在資金有限的前提下，它必敗。** 不是「風險高」，是結構性的必敗。本章的賭徒破產會把這件事證給你看。

這三件事——退避軌跡、佇列漂移、攤平加碼——共享同一個骨架：**一個隨機量沿時間一步一步累加，你關心的不是「下一步」，而是「這條軌跡長期會跑去哪、會不會撞到某條線」。** 這就是隨機漫步。我們先把脊椎硬幣升級成它。

## 把硬幣變成一條軌跡：一維隨機漫步

**對，就是 ch01 那枚公正硬幣。** 我們已經認識它七次了——古典 1/2、期望 1/2、變異數 1/4、二項、比例收斂、常態近似、檢定公正性。現在做第八次升級，而且是最戲劇性的一次：**把每一擲變成走一步。**

規則極簡單：

```text
擲到正面 → 往右走一步,位置 +1
擲到反面 → 往左走一步,位置 −1
```

從原點 0 出發，擲一次硬幣決定往哪走一步，再擲再走。設第 i 擲的步伐是 Sᵢ:正面 Sᵢ=+1、反面 Sᵢ=−1，各 1/2。那麼 n 步之後你的**位置**就是：

```text
Xₙ = S₁ + S₂ + … + Sₙ      ← 把前 n 步全部加起來
```

注意這個「加起來」。ch09 到 ch15 我們一直在加 iid 隨機變數——加總的變異數、加總的鐘形。隨機漫步就是**同一個加總，只是這次我們不丟掉中間過程，而是把每一個部分和 Xₙ 都記下來、沿時間畫出來**。它不再是一個數字，是一條軌跡。

畫個樣子給你看。擲出「正反正正反反正反」這八步，每步在數線上 ±1，把位置一格一格描出來（● 是你當下的位置，愈靠右愈往未來）:

```text
步數     0    1    2    3    4    5    6    7    8
擲出          正   反   正   正   反   反   正   反
位置     0   +1    0   +1   +2   +1    0   +1    0

 +2                          ●
 +1           ●        ●          ●          ●
  0    ●            ●                   ●          ●
 −1
```

每一步只往上或往下一格，把這些 ● 連起來就是一條上上下下的鋸齒。換一組硬幣結果，你得到完全不同的另一條鋸齒。隨機漫步研究的不是某一條，是**所有可能軌跡的統計行為**。

先把幾件能立刻算出來、且都是回收前面章節的事釘死：

**第一，任何一步的期望位移是 0。** E[Sᵢ] = (+1)(1/2) + (−1)(1/2) = 0。所以 n 步之後位置的期望 E[Xₙ] = 0。**漫步「平均而言」哪都不去**，永遠以原點為中心——它沒有漂移（drift）。這是「公正」的數學意思。

**第二，它散開的速度是 √n。** 每一步的變異數 Var(Sᵢ) = E[Sᵢ²] − 0² = 1(因為 (+1)²=(−1)²=1)。步與步獨立，變異數相加（ch09 的招牌），所以 Var(Xₙ) = n，標準差 SD(Xₙ) = √n。

停在這裡看一眼，因為這個 √n 是全書的老朋友又來了。ch09 講過「和的標準差長 √n、平均的標準差縮 √n」;ch13 講過硬幣「正面減反面」的典型量級 ~√n。**「正面減反面」正是這條漫步的位置**——正面 +1、反面 −1，加起來不就是正面數減反面數嗎？所以漫步位置 Xₙ 的典型大小就是 ±√n，跟 ch13 那張表的右半邊是同一件事。你那時學的「比例收斂、差距發散」，現在有了一張臉：**差距發散，就是這條鋸齒以 √n 的速度離開原點。**

**第三，n 步後的位置分布就是二項分布換了座標。** 如果 n 步裡有 k 步往右（正面），位置就是 Xₙ = k − (n−k) = 2k − n。而 k ~ Binomial（n, 1/2）。所以位置分布完全由二項決定——這也代表（ch14）當 n 大時，Xₙ 的分布趨近常態 N（0， n）。漫步走得越久，它「現在在哪」的分布越像一個越攤越平的鐘形，中心釘在 0、寬度長 √n。

我把前幾步的位置分布列出來，你會看到二項那個熟悉的形狀：

```text
n=1:  位置 −1 或 +1,各 1/2
      −1 ▓▓▓▓▓▓
      +1 ▓▓▓▓▓▓

n=2:  位置 −2、0、+2
      −2 ▓▓▓          (反反,1/4)
       0 ▓▓▓▓▓▓       (正反或反正,2/4)
      +2 ▓▓▓          (正正,1/4)

n=3:  位置 −3、−1、+1、+3
      −3 ▓            (1/8)
      −1 ▓▓▓          (3/8)
      +1 ▓▓▓          (3/8)
      +3 ▓            (1/8)

n=4:  位置 −4、−2、0、+2、+4
      −4 ▓            (1/16)
      −2 ▓▓▓▓         (4/16)
       0 ▓▓▓▓▓▓       (6/16)
      +2 ▓▓▓▓         (4/16)
      +4 ▓            (1/16)
```

注意兩件事。其一，**位置的奇偶性跟步數綁死**：走偶數步只可能停在偶數位置，走奇數步只可能停在奇數位置（你不可能走 4 步停在位置 1——每步 ±1,4 步的和必為偶數）。這個小觀察待會算「回到原點」時很關鍵。其二，**中間最高、兩邊最矮**，就是二項那個鐘形雛形——n=4 時中央位置 0 的機率 6/16，比兩翼都高。漫步最可能「大致待在中間」，但它的觸角（尾巴）一路伸向 ±n。

到此為止隨機漫步看起來只是「二項分布加了時間軸」，沒什麼新東西。真正的反直覺在下一節——當你問一個**只有沿時間展開才問得出**的問題：它會回家嗎？

## 會回家，但別等它：常返性與無限的回返時間

問題很單純：從原點出發的對稱漫步，**會不會再回到原點？**

直覺上你可能猜「會吧，它在 0 附近晃」，或者猜「不一定，它可能一路往右就不回頭了」。兩個直覺都不夠精確。正確答案漂亮到我認為是本章最值得停下來的一頁：

> **一維對稱漫步以機率 1 回到原點。** 不是「很可能」，是**幾乎必然**（almost surely）——只要你讓它走下去，它一定會踩回 0，而且一定會回來無限多次。

這個性質叫**常返性（recurrence）**。它是 George Pólya（波利亞）1921 年證明的隨機漫步常返定理的一維情形（Pólya 1921）。

先讓你對「機率 1」這件事有點感覺。漫步可能一路往右走 10 步、100 步不回頭——這條軌跡存在，機率不是 0。但它**遲早**會被拉回來。為什麼？因為漫步沒有漂移，它的中心永遠是 0，而它以 √n 的速度左右擺盪；走得越遠，「下一段把它拉回去」的累積機會也越多，多到把「永遠不回來」這件事的機率擠成了 0。嚴格證明本書不展開(它要算 Σ P(第 2n 步剛好在原點) 這個級數發不發散——一維/二維發散⟹常返，三維收斂⟹不常返)，但直覺是：**在一維這條線上，漫步沒有足夠的「逃跑空間」，它被困在自己的中線附近，跑不掉。**

好，既然保證會回來，下一個自然的問題是：**要等多久？** 這裡就是那個讓人下巴掉下來的反轉：

> 回到原點的**期望時間是無限大**。

讀兩遍。**它保證回來（機率 1），但你期望要等的時間是 ∞。** 這在數學上叫**零常返（null recurrent）**：回得來，可是回返時間的期望發散。

這兩句話放在一起像是矛盾的——怎麼會「一定回來」又「平均等無限久」？它沒有矛盾，它在告訴你一件關於這條軌跡的深刻事實：**大部分時候漫步回得很快（走兩步就踩回來的機率就有 1/2），但偶爾它會盪出去盪得極遠、極久才回來，而這些「極久」的尾巴重到把平均拉到無限。** 這跟 ch12 講長尾時「期望可能發散」是同一種病：絕大多數樣本乖乖的，但尾巴太肥，平均被少數巨值綁架。

翻成工程白話。想像你的佇列長度是一條對稱漫步（進出剛好打平），你問「它多久會回到初始長度？」——答案是：**它一定會回去（別恐慌，它不會永遠漲上去），但你不能用『平均多久回去』來做容量規劃，因為那個平均是無限的。** 換句話說：它會自己康復，可是你訂不出 SLA。「會回家，但別等它」的工程版本，就是「它會自癒，但別把自癒時間寫進保證」。

這一節我要你帶走的驚嘆，用一句話講給另一個工程師聽：**「保證發生」和「平均要等多久」是兩件完全獨立的事——一件事可以百分之百會發生，而它的等待時間期望卻是無限大。** 你的直覺把這兩件事綁在一起（「既然一定會，那平均應該等得到吧」），這個綁定是錯的，零常返就是反例。

### 維度的魔法：醉漢回得了家，醉鳥回不了

Pólya 的定理還有更狂的一半，我用一句話帶過（細節不展開，屬全書「隨機過程不深入」的範圍）:

**常返性跟維度有關。一維、二維對稱漫步常返（機率 1 回原點）;三維以上暫態（transient），回不去了。**

二維是在棋盤上隨機走（東南西北各 1/4），也常返——醉漢在無限大的平面城市裡亂走，**一定**走得回家。但三維——在空中飛、上下左右前後都能動——漫步以正機率**永遠回不到出發點**。

這件事有個流傳很廣的講法，出自數學家角谷靜夫（Shizuo Kakutani）在 UCLA 演講時的玩笑（2026-06，查證為角谷的口頭妙語）:

> **「醉漢終究會找到回家的路，但醉鳥可能永遠迷失。」**
> (A drunk man will eventually find his way home, but a drunk bird may get lost forever.)

醉漢在地面走（二維）——常返，回得了家。醉鳥在天上飛（三維）——暫態，回不了家。直覺上：維度越高，「外面的空間」長得越快，漫步有太多方向可以逃，迷路一去不回。我第一次聽到時不太相信：憑什麼多一個維度，「一定回來」就變成「可能永遠回不來」？但它是真的，這個從二維到三維的相變正是 Pólya 1921 那篇論文的核心。你只要記得這個畫面：**醉漢回得了家、醉鳥回不了。**

## 賭徒破產：為什麼莊家總是贏

現在把漫步關進一個有牆的房間，它立刻變成關於錢、關於必敗的故事。

上一節的漫步在無限長的線上自由擺盪。但真實賭局不是這樣——你有**本金**（不能跌破 0，跌破就破產出局），通常也有個**目標**（贏到某個數就收手）。把這兩條線當成兩面**吸收牆**：漫步一旦撞到任何一面就停住、遊戲結束。這就是**賭徒破產問題（gambler's ruin）**。

它的歷史比你想的老：這是 Pascal（帕斯卡）和 Fermat（費馬）1654 年通信時就碰過的問題，惠更斯（Huygens）1657 年史上第一本機率書《論賭博中的推理》的書末第五題就是它（照 landscape）。賭徒破產跟全書開場的分賭注問題是同一批人、同一個年代搞出來的——機率論從娘胎裡就在算「會不會輸光」。

設定基準配置：

```text
你的本金     i 元
你的目標     N 元(贏到 N 就收手走人)
每一局       公正賭局,贏 +1、輸 −1,各 1/2
吸收牆       位置 0(破產出局)、位置 N(達標收手)
```

這就是一條從位置 i 出發、左牆 0、右牆 N 的對稱漫步。問題：**從 i 出發，先撞到 0（破產）的機率是多少？**

答案乾淨到不可思議：

```text
                       N − i
P(從 i 出發會破產)  =  ───────
                         N

                         i
P(從 i 出發會達標)  =  ───────
                         N
```

### 為什麼是這條線性公式：三十秒說清

我不要你死背，這個結果有一個漂亮到值得講的理由。設 r(i) = 從本金 i 出發的破產機率。

**邊界條件先釘死：** r(0) = 1（已經在 0，百分百破產）;r(N) = 0（已達標，百分百不破產）。

**遞迴關係：** 從 i 出發，擲一次硬幣，1/2 機率走到 i+1、1/2 走到 i−1，之後再從新位置出發。所以：

```text
r(i) = ½·r(i+1) + ½·r(i−1)      ← 破產機率是左右兩邊的平均
```

把它移項：r(i) − r(i−1) = r(i+1) − r(i)。這在說什麼？**相鄰兩格的破產機率差，處處相等**——也就是 r(i) 是一條**直線**（等差）。一條從 r(0)=1 直線降到 r(N)=0 的直線，就是：

```text
        N − i
r(i) = ───────      ← 從 1 線性降到 0
          N
```

驗算邊界：i=0 → N/N = 1 ✓;i=N → 0/N = 0 ✓。漂亮。「破產機率」這種看起來該很複雜的東西，在公正賭局下竟然是一條直線——而那條直線的斜率，就是「你離破產牆有多近」。

### Worked example:手算幾個配置

**配置一：本金 100，想翻倍到 200，公正賭局。** i=100、N=200:

```text
                N − i     200 − 100     100
P(破產)     =  ─────── =  ───────── =  ───── =  0.5
                  N          200         200
```

一半一半。想把 100 翻成 200，你有 50% 機率反而輸光。聽起來「還算公平」對吧？畢竟是公正賭局。先記住這個 0.5，等一下加進莊家優勢你就笑不出來了。

**配置二：本金 1，想贏到 100（以小搏大），公正賭局。** i=1、N=100:

```text
                N − i     100 − 1      99
P(破產)     =  ─────── =  ───────  =  ───── =  0.99
                  N         100        100
```

99% 破產。拿 1 塊去拚 100 塊，即使每局完全公平，你也幾乎必輸光。這就是「以小搏大」的數學真相：**公平不代表你撐得住**，本金太薄、目標太遠，公正賭局照樣把你吃乾抹淨。注意對稱的另一面：達標機率 i/N = 1/100 = 1%——你確實有 1% 機率把 1 塊變 100 塊，而且**期望是公平的**（1% × 100 = 1 元 = 本金，沒賺沒賠）。公平藏在「小機率大報酬」裡，不在「我撐得住」裡。

**配置三：本金 50、目標 100，公正賭局。** i=50、N=100:P(破產) = 50/100 = 0.5。從正中間出發，破產與達標各半，符合對稱直覺。

把這幾個排在一起，你看出規律：**公正賭局下，破產機率就是「你離破產牆的相對距離」**——本金占目標的比例越低，破產機率越高，而且是直線關係。沒有玄機，就是那條線。

### 加進 1% 莊家優勢：直線塌成懸崖

以上都是「公正賭局」（每局 50/50）。但真實賭場、真實的有手續費/點差的市場，**沒有公正這回事**——莊家總有一點點優勢。我們看一個極小的優勢會造成多大的災難。

設每局你贏的機率 p 不再是 0.5，而是 0.495（輸的機率 q=0.505）——只差 1%，莊家優勢小到你幾乎感覺不到。這時漫步有了向左的**漂移**（平均每局期望 −0.01），公式不再是線性，而是：

```text
                  (q/p)^N − (q/p)^i
P(從 i 破產)  =  ───────────────────       ← q/p = 0.505/0.495 ≈ 1.0202
                     (q/p)^N − 1
```

不用被公式嚇到，重點是那個 **(q/p) 的次方**——它是指數，而指數會碾壓一切。我帶你算配置一（i=100、N=200）看它變成什麼：

```text
q/p = 0.505 / 0.495 = 1.02020…

(q/p)^100  ≈ 1.0202^100  ≈ 7.32      ← 用 100·ln(1.0202) ≈ 100·0.02000 = 2.000,e^2.000 ≈ 7.39,取約 7.3
(q/p)^200  ≈ 1.0202^200  ≈ 53.6      ← (7.3)² ≈ 53

                53.6 − 7.32      46.3
P(破產)  ≈  ─────────────── =  ─────  ≈  0.881
                 53.6 − 1        52.6
```

(這幾個指數是手算近似，用 ln(1.0202)≈0.0200 換成 e 的次方再估；精確值約 0.88，我們要的是量級不是小數點後三位。)

看清楚發生了什麼。公正賭局時，本金 100 翻倍到 200 的破產機率是 **0.5**。只給莊家 **1%** 的優勢，破產機率就從 0.5 跳到約 **0.88**。**一個你幾乎感覺不到的劣勢，把你的破產機率從一半推到將近九成。**

而且這還只是想翻倍。如果你貪心一點、目標訂遠一點，或者你坐在賭桌前夠久（等於 N 變大、局數變多），那個 (q/p)^N 的指數會繼續爆炸，破產機率**指數逼近 1**。這就是為什麼「久賭必輸」不是道德訓誡，是數學定理：**只要莊家有任何正的優勢，你坐得夠久，破產機率趨近 100%。** 莊家不需要每局都贏，它只需要那 1%，加上時間。時間在指數裡，你在指數的對立面。

我把這個對照做成一張表，因為它是這一節的全部震撼：

```text
本金 100、目標 200          每局勝率 p      破產機率
─────────────────────────────────────────────────
公正賭局                    0.500           0.500
莊家優勢 1%                 0.495           ≈ 0.88
莊家優勢 2%                 0.490           ≈ 0.98
─────────────────────────────────────────────────
（劣勢從 0% 到 2%,破產機率從一半衝到幾乎必然）
```

優勢是線性增加的（0%、1%、2%），破產機率是非線性暴衝的——這就是指數在公式裡的指紋。**微小而持續的劣勢，在足夠的時間下，等於確定的毀滅。** 把這句話刻起來，下一節的「翻本法」會撞死在它上面。

## 鞅一瞥：翻本加倍法為什麼必敗

現在來處理那個古老的、每隔幾年就騙一批人的「必勝法」——翻本加倍下注法（martingale betting system，18 世紀法國賭徒搞出來的，martingale 這字本來指馬具上一條防馬抬頭的韁繩;2026-06 查證）。

策略長這樣，簡單到誘人：

```text
押 1 元賭硬幣。
輸了 → 下一把押 2 元。再輸 → 押 4 元。再輸 → 押 8 元 …… 每輸一次就加倍。
只要贏一把,把之前所有的輸全部賺回來,還淨賺 1 元。然後從頭再來。
```

數學上驗一下：假設你連輸 k 把才贏。前 k 把總共輸了 1+2+4+…+2^(k−1) = 2^k − 1 元；第 k+1 把押 2^k 元贏了，進帳 2^k 元。淨賺 2^k − (2^k − 1) = **+1 元**。**不管你連輸幾把，只要最後贏一把，就淨賺 1 元。** 而硬幣總會出一次正面（對吧？），所以這策略「保證」每一輪都賺 1 元。

聽起來無懈可擊。它錯在哪？

### 公平賭局的期望不變：鞅的核心

先講一個概念，叫**鞅（martingale）**——這是隨機過程裡描述「公平賭局」的數學物件，本書只取它一句話的直覺（完整理論不展開，屬全域不深入範圍）:

> **鞅 = 一個過程，在任何時刻，你對「下一刻的期望值」就等於「現在的值」。** 翻成賭桌語言：**公平賭局裡，不管你用什麼下注策略，你財富的期望值永遠不變。**

這是個驚人地強的定理（叫選擇抽樣定理 / optional stopping，本書不證）：**沒有任何「下注時機」或「下注金額」的策略，能把一個公平賭局的期望報酬變成正的。** 加倍、減半、看手氣、看星座——對期望值統統無效。期望就是 0，你動不了它。翻本法當然也在「任何策略」裡，所以它的期望報酬也是 0。

那為什麼上面算出來「每輪穩賺 1 元」？哪裡騙了你？

### 騙局藏在「資金無限」這個假設裡

翻本法的「保證賺」偷渡了一個前提：**你有無限的資金、賭場也沒有下注上限，讓你可以一路加倍下去直到贏。** 在這個幻想世界裡，它確實穩賺——因為公平賭局裡，鞅理論允許「無限資金 + 無限時間」的策略避開虧損（代價是你需要的資金本身是無限的）。

但你的資金是**有限**的。看看加倍長多快：

```text
連輸把數    這一把要押    累積已輸
─────────────────────────────────
  1          1            0
  2          2            1
  3          4            3
  4          8            7
  …          …            …
 10        512          511
 11       1024         1023
 …
 20    524288       524287       ← 連輸 20 把,這一把要押 52 萬
```

連輸 10 把，你下一把要押 1024 元（原始注的 1024 倍）;連輸 20 把要押 52 萬。**只要你撞到資金上限（或賭場的下注上限），沒辦法再加倍，那一連串的輸就無法翻本——你一次賠掉前面所有的累積虧損。** 而連輸 10 把、20 把的機率雖小，卻**不是零**（連輸 10 把機率 1/1024），你只要在桌前夠久，遲早撞上一次。

把這個結構看穿：**翻本法用「很高機率賺一點點」換「很低機率賠掉一切」。** 它把你的損益曲線改造成「絕大多數時候小賺、偶爾一次毀滅性大賠」——而那一次大賠，期望上剛好抵銷掉前面所有的小賺（因為鞅理論說期望不變）。它沒有創造任何優勢，只是把風險重新打包成一個騙過你直覺的形狀：你看到的是一長串綠色的小勝，看不到那條埋在尾巴裡、終將到來的紅色懸崖。

這跟上一節的賭徒破產是同一件事的兩面。賭徒破產說「本金有限 + 持續下注 = 終將觸底」;翻本法是「故意把注越下越大去逼近那面破產牆」。它不但沒躲開破產，還跑去撞牆。**有限資金 + 公平（甚至不利）賭局 + 無限時間 = 破產機率趨近 1**，翻本法改變不了這個結局，它只改變了破產來臨的「形狀」——讓它來得更突然、更大。

### 工程橋接：攤平加碼是穿西裝的翻本法

回到開頭那個攤平加碼。「跌了就加碼拉低均價，反彈就解套」——把它跟翻本法並排：

```text
翻本法              攤平加碼
────────────────────────────────────
輸了 → 加倍下注      跌了 → 加碼買進
賭「總會贏一把」     賭「總會反彈一次」
小機率連輸 → 爆倉    小機率一路跌 → 套死/爆倉
資金有限 → 必敗      資金有限 → 同一個必敗結構
```

**結構完全一樣。** 攤平加碼也是「用很高機率小賺，換很低機率賠掉一切」;它也假設你有無限資金可以一路加碼到反彈；它也在資金有限的真實世界裡，把你的損益打包成「一長串小勝 + 尾巴裡一個毀滅性大賠」。差別只在賭桌換成了 K 線圖，「總會出正面」換成了「總會反彈」。如果你看懂了翻本法為什麼必敗，你就看懂了為什麼「攤平加碼」對一個沒有正期望優勢的標的（或一個你不確定有沒有優勢的標的）是同一個陷阱。**鞅理論不在乎你下注的是硬幣還是股票，它只在乎期望值——而沒有優勢的賭局，期望值動不了。**

（說句公道話：攤平加碼在「標的本身有正期望、你資金充足且能扛到底」時不必然失敗——那時你不是在賭公平/不利賭局。問題在於用它的人通常賭不起、也不確定有沒有正期望。鞅理論揭穿的是「它本身是必勝法」這個宣稱，不是「任何加碼都是錯的」。）

## 接下去：漫步是布朗運動的離散前身

最後埋一個給 ch22 的引線。我們這一章的漫步是**離散**的：一步走一格、一拍走一步。但你盯著 dashboard 上的延遲曲線、或盯著股價，它們看起來是**連續**的鋸齒——時間是流動的，位移是任意小的抖動。

如果你把隨機漫步的步長縮到無限小、時間間隔也縮到無限小取極限，會得到**布朗運動（Brownian motion）**——花粉在水裡的抖動、股價的飄移、網路延遲放大到極致的毛刺，全是同一個數學物件。而這一章你已經摸到它的兩根指紋了：位置分布是常態（ch14 的 CLT），散開速度是 √（時間）（這一章的 √n）。ch22 會告訴你，要讓這個極限有意義，步長 Δx 跟時距 Δt 必須滿足 Δx ~ √Δt——**那個 √ 又是這個 √**。隨機漫步是布朗運動穿著像素風的前身;ch22 把像素抹平，你會看到一條處處連續、卻處處是毛刺、處處不可微的怪物曲線。先記著這條線，我們在三章後接上。

## 直覺的陷阱

隨機漫步與賭徒破產是賭徒謬誤的老巢，也是「保證」與「期望」最容易被綁錯的地方。逐條拆。

| 陷阱 | 錯誤直覺 | 它在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| **賭徒謬誤** | 「連輸這麼多把，該贏了吧」「最近故障多，接下來該平靜了」 | 偷渡了「漫步有記憶、會自我修正」。它沒有。每一步都是獨立的全新硬幣，過去的輸不增加未來贏的機率 | 你發現自己在用「該」「輪到」「平衡一下」這種詞描述獨立事件的下一步——警報響了 |
| **把「保證」當「快」** | 「對稱漫步保證回原點，那它應該很快會回來」 | 把「機率 1 發生」綁上了「期望時間有限」。零常返：回得來，但期望等無限久 | 你預設「一定會發生的事，平均等得到」——這個綁定本身就是錯的，零常返是反例 |
| **公正 = 撐得住** | 「公正賭局 50/50，我本金少一點也沒差吧」 | 忽略了破產機率 = (N−i)/N。本金薄、目標遠，公正賭局照樣 99% 輸光 | 你只看「每局公平」，沒算「本金占目標的比例」。後者才決定你會不會被吃掉 |
| **小劣勢 = 小風險** | 「莊家才優勢 1%，影響不大」 | 優勢進了 (q/p)^N 的指數。1% 劣勢 + 時間 = 破產機率從 0.5 衝到 0.88、再趨近 1 | 你把「每局的劣勢」線性外推到「總體的劣勢」。指數不是線性，久賭把微小劣勢放大成確定毀滅 |
| **翻本法/攤平必勝** | 「每輪穩賺 1 元」「跌了加碼總會解套」 | 偷渡「資金無限」。有限資金下，小機率的連敗會一次賠掉全部累積收益 | 你算得出「保證賺」卻算不出「萬一連輸到爆倉怎麼辦」——你忘了把破產那條尾巴算進期望 |

最值得單獨拎出來的是**賭徒謬誤 vs 賭徒破產的混淆**（landscape 特別點名，兩者別搞混）:

- **賭徒謬誤（gambler's fallacy）**是個**認知錯誤**：以為獨立事件會自我修正（「連紅該開黑了」）。它是錯的，因為硬幣沒有記憶。
- **賭徒破產（gambler's ruin）**是個**正經的數學問題**：本金有限、持續下注，算你破產的機率。它是對的，而且結論很硬：久賭必輸。

諷刺的是，信賭徒謬誤的人，常常正是被賭徒破產收割的人——他「相信該贏了」所以加碼死撐（翻本法），結果撞上「本金有限必破產」的鐵牆。**一個謬誤的直覺，把人送進一個真實的破產定理。**

## 紙上推演

### 推演題 1 [10 分鐘] ★

用基準公式 P(破產) = (N−i)/N 算下面三個公正賭局的破產機率，並各用一句話解讀：

(a) 本金 100，想翻到 200。
(b) 本金 10，想翻到 1000。
(c) 本金 990，想再贏到 1000（只想小賺 10）。

算完後回答：為什麼 (c) 的破產機率這麼低，但這個策略其實藏著什麼風險？

#### 推演解答

直接套公式：

```text
(a) i=100, N=200:  P(破產) = (200−100)/200 = 100/200 = 0.50
(b) i=10,  N=1000: P(破產) = (1000−10)/1000 = 990/1000 = 0.99
(c) i=990, N=1000: P(破產) = (1000−990)/1000 = 10/1000 = 0.01
```

解讀：

- **(a) 50%**：翻倍是「離兩面牆一樣遠」的對稱情形（從中點出發），破產與達標各半。公正不代表安全，翻倍有一半機率輸光。
- **(b) 99%**：以小搏大，本金只占目標的 1%，離破產牆極近、離達標牆極遠。即使每局公平，99% 會先撞破產牆。「拿小錢拚大錢」在數學上幾乎注定失敗——這不是手氣問題，是結構。
- **(c) 1%**：你只想小賺（從 990 拚到 1000），離達標牆很近、離破產牆很遠，所以很可能成功。

**(c) 藏的風險**：破產機率只有 1%，聽起來很安全——但代價是，那 1% 的事件會讓你**賠掉全部 990 元**，只為了賺 10 元。期望算一下：0.99 機率賺 10 元、0.01 機率賠 990 元 → 期望 = 0.99×10 − 0.01×990 = 9.9 − 9.9 = **0**（公正賭局，期望本來就是 0）。這正是翻本法的結構：**高機率小賺、低機率巨賠，期望剛好抵銷。** 「幾乎總是贏一點點」騙過你的直覺，讓你忽略那條罕見但毀滅性的尾巴。看到「破產機率很低但一旦發生就賠光」的損益形狀，警鈴就該響。

### 推演題 2 [15 分鐘] ★★

口頭（或寫下）向一個正在「攤平加碼」的朋友解釋賭徒破產，要讓他聽懂為什麼這策略有結構性問題。你的解釋必須包含：（1） 翻本法的「保證賺」是怎麼算出來的;(2) 它偷渡了什麼假設;(3) 攤平加碼為什麼是同一回事。

#### 推演解答

一個能讓非數學朋友聽懂的講法骨架：

**(1) 保證賺怎麼來的**：「翻本法是這樣——輸了就把下注加倍，只要哪一把贏了，就把前面所有的輸全賺回來還倒賺一塊。你連輸 k 把總共輸 2^k−1 塊，第 k+1 把押 2^k 塊贏回 2^k 塊，淨賺 1 塊。數學上，只要你最後贏一把，這一輪就賺 1 塊，看起來穩賺。」

**(2) 偷渡了什麼**：「但這個『保證』假設你有**無限**的錢可以一路加倍下去。連輸 10 把，下一把要押 1024 塊；連輸 20 把要押 50 多萬。只要你哪次撞到錢不夠（或賭場下注上限），那一連串的輸就翻不回來，你一次賠掉前面累積的全部。而連輸這麼多把的機率雖小，卻不是零——你賭夠久，遲早撞上一次。它的真面目是：**用很高機率賺一點點，換很低機率賠光全部。** 算期望，這兩邊剛好抵銷，它沒給你任何優勢，只是把風險藏進尾巴。」

**(3) 攤平加碼同理**：「你跌了加碼、再跌再加碼，賭的是『總會反彈』，就跟翻本法賭『總會贏一把』一模一樣。資金有限的話，只要它一路跌不反彈，你會一路加碼到套死或爆倉——同一個結構。除非這標的本身有正期望、而且你扛得到反彈，否則你不是在投資，是在用加碼逼近自己的破產牆。」

常見錯路：朋友會說「可是反彈的機率很高啊」。對，**機率高正是陷阱的一部分**——它讓你忽略那個低機率但毀滅性的結局。要把焦點從「機率」拉到「期望」和「萬一連敗的後果」。

### 推演題 3 [10 分鐘] ★★

解釋這個看似矛盾的陳述，讓它不再矛盾：「一維對稱漫步**保證**回到原點（機率 1），但**期望**回返時間是無限大。」

提示：想想 ch12 講長尾時「期望可能發散」的直覺。

#### 推演解答

關鍵是把「機率 1 會發生」和「期望要等多久」當成**兩件獨立的事**——直覺把它們綁在一起，鬆綁就不矛盾了。

**為什麼保證回來**：漫步沒有漂移，中心永遠是 0，而且它以 √n 的幅度左右擺盪。走得越久，「被拉回原點」的累積機會越多，多到「永遠不回來」的機率被擠成 0。（嚴格版要看一個級數發不發散，本書不展開，但直覺是：一維這條線上沒有足夠的逃跑空間。）

**為什麼期望卻是無限**：回返時間的**分布有極肥的尾巴**。絕大多數時候漫步回得很快（走兩步就踩回原點的機率就有 1/2），但偶爾它會盪出去很遠、要走極久才回來，而這些「極久」的情形雖然機率小，卻小得不夠快——它們的「時間 × 機率」貢獻加起來發散。期望是「時間的加權平均」，被這條肥尾綁架，拉到無限。

**鬆綁的那句話**：一件事「會不會發生」由機率決定（這裡是 1），「平均要等多久」由回返時間的分布決定（這裡是 ∞）——這是兩個獨立的問題。一件事可以百分之百發生，而它的等待時間期望卻是無限。這跟 ch12 的長尾是同一種病：樣本幾乎都乖，但尾巴太肥，平均被少數巨值綁架。

常見錯路：覺得「一定會發生」就推出「平均等得到」。這個推論沒有根據——零常返就是現成的反例，專門打這個直覺。

### 推演題 4 [12 分鐘] ★★

一條對稱漫步從原點出發。

(a) 走 4 步後，回到原點（位置 0）的機率是多少？手算。
(b) 走 3 步後，回到原點的機率是多少？（小心）
(c) 用 (a)(b) 的觀察，說一句關於「漫步什麼時候可能在原點」的通則。

#### 推演解答

**(a) 走 4 步回原點**：要回到 0,4 步裡必須剛好 2 步往右、2 步往左（右-左=0）。這是 4 步裡選 2 步往右，C(4,2) = 6 種走法；總走法 2⁴ = 16 種，每種等機率。

```text
P(X₄ = 0) = C(4,2) / 2⁴ = 6/16 = 3/8 = 0.375
```

（對照前面 n=4 的位置分布表，位置 0 的機率正是 6/16 ✓。）

**(b) 走 3 步回原點**：要回到 0，需要往右步數 = 往左步數，但 3 是奇數，不可能拆成兩個相等的整數。所以：

```text
P(X₃ = 0) = 0      ← 奇數步絕對回不了原點
```

**(c) 通則**：**漫步只可能在偶數步回到原點；奇數步時必定不在原點。** 因為每步 ±1，n 步的位置 Xₙ 與 n 同奇偶（n 步的和的奇偶性等於 n 的奇偶性）。要回到 0（偶數），步數 n 必須是偶數。更一般地：漫步走 n 步後，位置只能是與 n 同奇偶的那些值。

延伸體會：這就是為什麼算「漫步回原點」時，只需看第 2、4、6…（偶數）步；這也是常返性嚴格證明裡那個級數 Σ P(X₂ₙ=0) 只對偶數項求和的原因。常見錯路：忘了奇偶限制，在 (b) 硬套組合公式算出個非零機率——奇數步回原點是**結構上不可能**，不是機率小。
