# ch08 — 期望值：長期平均，不是最可能值

> **本章解決什麼問題**：ch07 教你把隨機結果變成數字（隨機變數），也給了它的完整行為剖面（分布）。但一整張分布表攤在你面前，你常常只想要一個數來概括它——「這玩意兒平均來說多大？」這就是期望值。它聽起來像國中算術，但藏了三個會反咬你的東西：期望值常常不是最可能發生的值、甚至根本不是可能發生的值；它有一條叫「線性性」的性質，威力大到不像真的、而且連獨立都不需要；還有一個三百年前的賭局，期望值算出來是無限大，卻沒有一個正常人願意多付。這章是 Part III 的第二站，把 ch07 的隨機變數壓成一個數，下一章（ch09）再問「這個數靠不靠譜、它會抖多大」。

## 從你已知的出發

你每天都在算期望值，只是你叫它「平均」。

你看 dashboard 上那條 average latency 的線，你心裡想的是「這支 API 一般來說多慢」。你做容量規劃，估「平均每秒進來幾個請求」，乘上「平均每個請求吃多少 CPU」，得到一個總負載的估計。你算雲端帳單，「平均每天跑幾次這個 batch job」乘上「每次大概多少錢」。這些全都是期望值，而且你做得很順，順到你從來沒停下來想過它到底在算什麼。

期望值在量化的東西很單純：**如果這件隨機的事重複很多很多次，每次的結果平均下來會是多少。** 注意這句話的重點在「很多很多次」和「平均」。它不是預測下一次會發生什麼——下一次仍然是隨機的、不可測的。它是在描述一個長期的、總體的趨勢。這跟你對 P99 的直覺是同一種思維：你從來不假裝知道下一個請求多慢，你只敢對「一大堆請求的整體行為」說話。期望值就是這種總體語言裡最基本的一個詞。

但這章我真正想讓你帶走的不是這個——這個你已經會了。我想讓你看到的是兩件你大概沒意識到的事。

第一件，**平均會騙你**，而且騙得很有系統。你報「平均延遲 50ms」的時候，如果背後是一條長尾分布（大多數請求很快、少數請求慢到爆），那個 50ms 可能根本沒有任何一個請求真的長那樣——它被少數幾個巨值硬生生拉高，既不是典型值、也不是你會遇到的值。這就是為什麼資深的人看延遲只信百分位數、不信平均。這個坑這章先挖開給你看，ch12 講長尾時會掉進去看個夠。

第二件更猛。期望值有一條性質叫**線性性（linearity of expectation）**：兩個隨機量相加，它們的期望值就是各自期望值相加，`E[X+Y]=E[X]+E[Y]`。聽起來廢話對吧？但這條性質有個地方完全不廢話——**它不需要 X 和 Y 獨立**。不管這兩個量怎麼糾纏、怎麼互相影響、相關係數是多少，它都成立。這件事划算到不像話，我會用它變一個魔術：n 封信隨機塞進 n 個信封，平均有幾封塞對人？答案是 1，跟 n 多大完全無關，而且你不用算那個會讓人想撞牆的「錯排數」。這是本書最便宜又最強的工具，這章的重頭戲。

先從那個「平均不是最可能值」的小陷阱熱身，因為它最快讓你的直覺鬆動。

## E[X] 是重心，不是最可能值

先把定義擺出來，它很短：

```text
E[X] = Σ x · P(X = x)
        └┬┘  └───┬───┘
       每個可能  那個值
       的取值    發生的機率
```

白話：把隨機變數每一個可能的取值，乘上它發生的機率，全部加起來。就這樣。每個值按照「它有多常出現」來加權，所以期望值是一個**機率加權平均**。

最好的物理圖像是**重心**。想像一根沒有重量的尺，在每個可能取值的位置上放一塊積木，積木的重量就是那個值的機率（所有積木加起來剛好 1 公斤，因為機率總和是 1）。那麼期望值就是這根尺的平衡點——你手指要放在哪裡，整根尺才不會往任何一邊倒。機率大的地方積木重，會把平衡點往自己這邊拉；機率小的地方拉力弱。

這個重心圖像會立刻幫你戳破一個超常見的誤會：**期望值不是最可能發生的那個值。** 最可能的值（眾數，機率最高的那個取值）跟重心是兩回事。重心被「所有」積木的位置和重量共同決定，包括那些很遠、很輕、但因為夠遠所以還是把平衡點拉過去一點的積木。

來個經典到有點冷的例子。台灣的家庭平均生幾個小孩？假設某年的數字是 1.8 個。這個 1.8 是一個誠實的期望值，但它荒謬之處一眼就看得出來：**沒有任何一個家庭真的生 1.8 個小孩。** 你生 0 個、1 個、2 個、3 個，就是不可能生 1.8 個。1.8 既不是最可能的值（最可能的可能是 2 個），更不是一個「可能的值」——它是個不存在於現實的小數。但它完全正確、完全有用：你拿它乘上家庭總數，就能估出全國的新生兒總數。期望值是總體的語言，它根本沒打算描述任何一個個體。

把這件事釘死，因為它是這章最容易被略過、又最容易在直覺上咬你的點：

```text
最可能值（眾數）：機率最高的那個取值   ← 「哪個結果最常單獨出現」
期望值（平均）：  所有取值的機率加權重心 ← 「重複很多次後平均落在哪」
這兩個常常不相等，期望值甚至可能根本不是一個可能出現的值。
```

### 脊椎硬幣：E[單擲]=1/2

來認領全書那枚硬幣——**這是 ch01 那枚公正硬幣**，ch02 用古典對稱性給了它 P(正)=1/2，現在輪到期望值來看它。

把單次擲幣寫成一個隨機變數：正面記為 1，反面記為 0（這叫伯努利隨機變數，ch11 會正式介紹這個名字）。它的期望值：

```text
E[X] = 1 · P(正) + 0 · P(反)
     = 1 · (1/2) + 0 · (1/2)
     = 1/2
```

E[X]=1/2。注意這個 1/2 完美示範了剛剛的陷阱：**這枚硬幣永遠不會給你 0.5。** 你擲一次，要嘛 1 要嘛 0，1/2 是擲很多次之後「平均每次得幾分」。重心圖像也對得上：尺的兩端（0 和 1）各放半公斤積木，平衡點當然在正中間 0.5。

那擲 n 次呢？令 Sₙ = n 次裡的正面總數。直覺會喊「n/2」，而且這次直覺是對的——但我先不告訴你為什麼這麼好算，因為這正是線性性要登場表演的地方。先記住答案：**E[n 擲的正面數] = n/2**。ch09 會接著問這個 n/2 會抖多大（變異數 n/4），ch11 會給它正式名字（二項分布），ch13 會用它演「比例收斂、差距發散」的全書最反直覺一頁。同一枚硬幣，我們才剝到第三層。

## 期望的線性性：本書最划算的工具

現在進這章的主菜。

線性性說的是這件事：

```text
E[X + Y] = E[X] + E[Y]
```

更一般地，對任意多個隨機變數、任意常數：

```text
E[a₁X₁ + a₂X₂ + ... + aₙXₙ] = a₁E[X₁] + a₂E[X₂] + ... + aₙE[Xₙ]
```

我知道你第一眼覺得這是廢話。「相加的平均等於平均的相加」，這在你算 average latency 的時候早就偷用過一百遍了——總延遲的平均，就是各階段延遲的平均相加。對，你的直覺對。

問題在於，你的直覺**會以為這需要那些階段彼此獨立**。畢竟一講到隨機變數相加，你滿腦子都是「要先確認它們獨立才能怎樣怎樣」（ch04 的陰影）。期望值的線性性最驚人的地方就在這裡：

> **它完全不需要 X 和 Y 獨立。** 不管 X 和 Y 怎麼相關、怎麼互相纏、共變異數是多少，`E[X+Y]=E[X]+E[Y]` 永遠成立。

讓我把這件事翻成你的語言，你會立刻知道它有多值錢。你估一個請求的「總耗時 = 排隊時間 + 處理時間 + 回傳時間」。這三段是獨立的嗎？才怪。系統一忙，排隊變長、處理也變慢、網路也塞——它們高度正相關，一起爛。**但你估平均總耗時的時候，照樣可以把三段的平均直接加起來。** 「平均總耗時 = 平均排隊 + 平均處理 + 平均回傳」，這條等式不在乎那三段多麼難分難捨。你一直在用這個性質，現在你知道它為什麼是對的，也知道它強在哪——它幫你繞過了整個「這些東西獨立嗎」的地獄。

為什麼不需要獨立？這裡給你**工程師的嚴謹**（每一步你都能口頭講出理由），不是揮手帶過。期望值的本質是「對所有可能結果按機率加權求和」。對每一個單一的結果 ω（一個具體的世界，硬幣怎麼落、信怎麼塞，全部定下來了），X(ω)+Y(ω) 就是兩個普通數字相加——這一步是國小算術，跟機率、跟獨不獨立一點關係都沒有。然後你對所有 ω 按機率加權求和，求和符號可以拆開：「(X+Y) 的加權和」拆成「X 的加權和」加「Y 的加權和」，這是 Σ 的線性，也是代數，不是機率。從頭到尾沒有任何一步問過「X 和 Y 獨立嗎」——因為根本沒用到。獨立性是在處理「乘積」`E[XY]` 時才需要的東西（那個要等 ch09、ch10），加法從來不需要。

這就是為什麼線性性是本書最划算的工具：**它讓你把一個複雜、糾纏、難算的大隨機量，拆成一堆簡單小隨機量的和，分頭算每個的期望，再加起來——完全不用管它們之間怎麼勾搭。** 下面這個招牌例子會讓你看到這有多誇張。

## 招牌震撼：信封配對，平均命中永遠是 1

這是這章我最想讓你能轉述給另一個工程師的東西。

問題（這是個三百年的老題，叫 matching problem，最早是 Montmort 1708 年研究一種叫 *jeu du treize* 的紙牌配對遊戲時提出的，他和 Nicholas Bernoulli 在 1713 年左右各自解出來，2026-06，已查證）：

> 你寫了 n 封信給 n 個不同的人，每封都該裝進寫了對應地址的信封。結果你喝多了，把 n 封信完全隨機地塞進 n 個信封，每個信封一封。請問：平均會有幾封信，剛好塞進了對的信封（信對人）？

先讓你的直覺猜一下。n=100 封信亂塞，平均幾封塞對？很多人會猜「應該很少吧，亂塞耶」，或者「會不會跟 n 有關，n 越大塞對的越多？」。

答案是 **1**。平均剛好命中 1 封。而且——這才是讓人頭皮發麻的地方——**不管 n 是 5、100、還是一億，答案都是 1。** 跟 n 完全無關。

我們用線性性把它算出來，你會發現它短到不像在解一個三百年的難題。

**第一步：定義指示變數。** 對第 i 封信，定義一個隨機變數 Xᵢ：

```text
Xᵢ = 1，如果第 i 封信剛好塞進對的信封（命中）
Xᵢ = 0，如果沒命中
```

這種只取 0 或 1 的隨機變數叫**指示變數（indicator variable）**，它是線性性的最佳拍檔。為什麼？因為一個指示變數的期望值，剛好就是「那件事發生的機率」：

```text
E[Xᵢ] = 1 · P(命中) + 0 · P(沒命中) = P(第 i 封命中)
```

**第二步：算單一封的命中機率。** 第 i 封信塞進對的信封，機率是多少？這封信被隨機塞進 n 個信封裡的某一個，每個信封等機率，其中只有一個是對的。所以：

```text
P(第 i 封命中) = 1/n
因此 E[Xᵢ] = 1/n
```

這對「每一封」都成立，因為對任何一封信來說，它落在自己那個正確信封的機率都是 1/n（不管別封怎麼塞）。

**第三步：總命中數就是把所有指示變數加起來。** 總共命中幾封？把每封的「中了沒」加總：

```text
總命中數 X = X₁ + X₂ + ... + Xₙ
```

這是關鍵的拆解：那個複雜的「總命中數」，被我拆成了 n 個簡單的 0/1 指示變數的和。

**第四步：線性性收網。**

```text
E[X] = E[X₁ + X₂ + ... + Xₙ]
     = E[X₁] + E[X₂] + ... + E[Xₙ]   ← 線性性，這裡是整個魔術
     = (1/n) + (1/n) + ... + (1/n)    ← n 個 1/n
     = n · (1/n)
     = 1
```

**E[總命中數] = 1。** n 約掉了。這就是為什麼答案跟 n 無關。

現在請你停下來，欣賞那條「線性性」那一步偷渡了什麼。**這些 Xᵢ 一點都不獨立。** 想想看：如果前 n−1 封信全都塞對了，那第 n 封信別無選擇，一定也塞對（剩最後一個信封一定是對的）。一封信塞對不對，會牽動別封的可能性——它們糾纏得一塌糊塗。如果你想老老實實處理這堆相關性，你會陷進一個叫**錯排（derangement）**的計數泥沼：要算「完全沒一封塞對」有幾種排法（這個數叫 Dₙ，最接近 n!/e 的整數），還要算「恰好命中 k 封」的排法數，那是一個會用到容斥原理、寫滿一整面白板的東西，三百年前 Montmort 和 Bernoulli 真的就是這樣硬幹的。

**但線性性完全繞過了這一切。** 它不在乎這些 Xᵢ 怎麼相關，它只要每一個的期望值（各自是 1/n），加起來就收工。你把一個需要錯排數、容斥原理的計數惡夢，變成了「n 乘以 1/n」的口算。

這就是我說它划算到不像話的意思。一個工具，讓你對著一堆纏死的相關隨機變數，假裝它們互不相干地分頭算期望，最後加起來還是對的——因為加法從不過問獨立。**我認為這是整本書 CP 值最高的一頁**，值得你把它背到能在白板上即興表演給人看。

順帶一提，如果你還想知道「那 P(完全沒命中) 是多少」，答案是 1/e ≈ 0.3679（n 大時）——又是那個 e，又是那條 1−1/e ≈ 0.63 的曲線（你在 ch03 算「至少一次」時見過它的近親）。但那需要錯排數，是另一個故事；期望值這條捷徑，根本不必碰它。

## 聖彼得堡悖論：期望值無限，卻沒人肯付多

線性性讓你覺得期望值無所不能。這一節要給你潑盆冷水：**期望值不是萬能的決策準則。** 有一個賭局，它的期望報酬是無限大，但你——以及任何一個正常人——都只願意付幾塊錢去玩。

這就是聖彼得堡悖論（St. Petersburg paradox），1738 年由丹尼爾·伯努利（Daniel Bernoulli）在聖彼得堡科學院的論文裡正式討論（題目本身更早由他堂兄 Nicolaus 在 1713 年提出，2026-06，已查證——landscape 也是這麼記的）。

賭局規則（原版講的是 Peter 和 Paul 對賭，賭金以 ducat 金幣計）：

> 莊家擲一枚公正硬幣，一直擲到出現正面為止。
> 第 1 次就正面，賠你 2 元；
> 第 2 次才正面，賠你 4 元；
> 第 3 次才正面，賠你 8 元；
> 第 k 次才正面，賠你 2ᵏ 元。
> 每晚一次出正面，獎金就翻倍。
> 請問：你願意付多少錢買一張入場券來玩這局？

我們用期望值算算這張入場券「該」值多少。第 k 次才出現正面，意思是前 k−1 次全是反面、第 k 次正面，機率是 (1/2)ᵏ。這時候賠 2ᵏ 元。把每一種情況的「機率 × 獎金」加起來：

```text
E[獎金] = Σ(k=1→∞) P(第 k 次才正面) · 2ᵏ
        = Σ(k=1→∞) (1/2)ᵏ · 2ᵏ

每一項是多少？
  k=1： (1/2)¹ · 2¹ = (1/2)·2 = 1
  k=2： (1/2)² · 2² = (1/4)·4 = 1
  k=3： (1/2)³ · 2³ = (1/8)·8 = 1
  ...每一項都剛好是 1！

E[獎金] = 1 + 1 + 1 + 1 + ... = ∞
```

每一項都精準地是 1（機率縮一半，獎金正好翻倍，相乘抵消），而這樣的項有無限多個，所以**期望報酬發散到無限大**。

照期望值的邏輯，這張入場券值無限多錢。你應該願意付 100 萬、付 1 億、付掉全部身家都還划算，因為期望報酬是 ∞。

但你會嗎？我不會。問一圈下來，大多數人只願意付 $10、$20，超過 $50 就覺得是盤子。**這就是悖論：一個期望值無限的賭局，理性的人卻只肯付一點點。** 你的直覺和期望值的計算公然打架。

這裡是這一節最重要、也最容易被講錯的一句話，請務必記住正確版（landscape 把它列為「常被誤傳」清單第 7 條）：

> **聖彼得堡悖論不是數學算錯。** 那個期望值真的就是無限大，加總沒有任何錯誤。悖論不在數學裡，在於「用期望值來做決策」這個準則本身——它在這裡失效了。

那為什麼會失效？丹尼爾·伯努利給的解法漂亮且深刻：**對你來說，錢的「價值」不等於錢的「金額」。** 第一個 100 萬會徹底改變你的人生，但當你已經有 10 億的時候，再多一個 100 萬幾乎沒感覺。錢帶來的「滿足感」隨著你已有的財富遞減——這叫**邊際效用遞減（diminishing marginal utility）**。伯努利提議用對數來描述：你在乎的不是金額 w，而是 ln(w) 這種「效用（utility）」。

關鍵在於：賭局裡那些「很晚才出正面」的情況，雖然獎金天文數字（2¹⁰⁰ 元之類），但它們的機率小到爆（(1/2)¹⁰⁰），而且那筆天文數字的錢對你的**實際效用**貢獻極小（你早就有花不完的錢了，再多也沒差）。一旦你算的是「期望效用」而不是「期望金額」，那些遠處的巨大獎金不再有無限的拉力，總和就收斂到一個有限的數——這個數就對應你真正願意付的價格。難怪沒人肯付太多：你的腦子裡算的從來不是金額，是效用。

這件事為什麼值得你停下來想：**它告訴你期望值是一個描述工具，不是一個決策神諭。** 「期望報酬最大」不等於「最該選」。一個工程師對這應該很有共鳴——你做決策從來不是只看「平均收益」，你還會看下行風險、看尾部、看「最壞會怎樣」。你不會因為某個操作「平均」會省 5% 成本就閉著眼睛上，如果它有 0.1% 的機率搞垮整個 production。聖彼得堡悖論是這種智慧最古老、最純粹的數學版本：**期望值只是故事的一半，下半場（它會抖多大、尾巴有多毒）是 ch09 和 ch12 的事。** 效用理論本身是個大坑，這裡就點到為止，延伸閱讀給你入口。

## 直覺的陷阱

期望值是個簡單概念，但它簡單到讓人輕敵，於是直覺在四個地方系統性地翻車。

| 陷阱 | 錯誤直覺 | 哪裡出錯 | 怎麼自我察覺 |
|---|---|---|---|
| 期望值＝最可能值 | 「期望壽命 80 歲，所以我大概會在 80 歲那年死」 | 期望值是機率加權的重心，不是眾數。重心被所有取值（含遠處的）共同拉扯，常常落在一個既非最可能、甚至非可能的位置（1.8 個小孩） | 你能不能講出一個「期望值根本不可能發生」的例子？講不出來＝你還把它當最可能值 |
| 平均是個典型值 | 「平均延遲 50ms，所以大部分請求大概 50ms」 | 長尾分布下，平均被少數巨值拉高，可能沒有任何請求真的接近平均。平均對偏斜分布是個糟糕的概括 | 你信平均、還是信中位數／P50？對延遲只報平均不報百分位＝你正要被尾巴打臉（ch12 詳述） |
| 線性性需要獨立 | 「這幾段耗時彼此相關，不能直接把平均加起來」 | `E[X+Y]=E[X]+E[Y]` 從不需要獨立。加法的期望永遠可拆，獨立性只在處理乘積 `E[XY]` 時才需要 | 你敢不敢對「高度相關的幾段耗時」直接加平均？不敢＝你把加法的規則跟乘法的搞混了 |
| 期望值＝該怎麼選 | 「這賭局／投資的期望報酬是正的（甚至無限），所以該全押」 | 期望值是描述，不是決策準則。聖彼得堡：期望無限但理性的人只付一點，因為你在乎的是效用與風險，不是裸期望 | 你做決策只看期望報酬、還是也看下行與尾部？只看期望＝你會在某個 0.1% 機率的災難上梭哈 |

第一個（期望壽命那句）特別值得拆，因為它在生活裡到處都是。「平均壽命 80」是一個期望值，它告訴你一大群人活到幾歲的平均，但它對「你個人會在幾歲死」幾乎沒說什麼——你可能 60 走、可能 95 走，這個分布又寬又偏，80 只是那團機率的重心。把總體的重心當成個體的預測，是期望值最常見的誤用，跟「把家庭平均 1.8 個小孩當成某一家真的生 1.8 個」是同一個錯。

## 紙上推演

純紙筆，能口算就口算，每題至少用兩個角度交叉驗一次。

### 推演一：用線性性算骰子點數和的期望 **[10 分鐘] ★**

擲 n 顆公正骰子，點數總和的期望值是多少？請用線性性算，**不准**去碰「總和的聯合分布」。先做 n=2，再推一般的 n。

#### 推演解答

先處理一顆骰子。令 Y 為單顆骰子的點數，取值 1 到 6，各 1/6：

```text
E[Y] = (1+2+3+4+5+6)/6 = 21/6 = 3.5
```

注意 3.5 又是一個「不可能出現的期望值」——骰子永遠骰不出 3.5，它是重心。（1 到 6 對稱，重心當然在正中間 (1+6)/2 = 3.5，這是第二個驗算角度。）

現在 n 顆骰子。令總和 S = Y₁ + Y₂ + ... + Yₙ，每個 Yᵢ 是第 i 顆的點數。直接上線性性：

```text
E[S] = E[Y₁] + E[Y₂] + ... + E[Yₙ]   ← 線性性，不問獨不獨立
     = 3.5 × n
```

n=2 時 E[S] = 7。**交叉驗算**：兩顆骰子的和，分布是個三角形，最高峰在 7（ch07 那張 PMF 表，和=7 的機率 6/36 最大）。這裡期望值剛好等於最可能值，是因為分布對稱——但這是巧合，別把它當通則（單顆骰子的 3.5 就不是任何可能值）。

這題的爽點：你**完全沒去算** n 顆骰子總和的分布長什麼樣（那會是一個很煩的東西），就直接拿到了期望值。線性性讓你跳過整個聯合分布。

### 推演二：聖彼得堡——算給自己看，再解釋你為何只付 $10 **[15 分鐘] ★★**

(a) 親手把聖彼得堡賭局的期望值前五項寫出來，確認每一項都是 1、總和發散。(b) 然後用「效用」的語言，講清楚為什麼你只願意付 $10——要講到你能轉述給一個只會喊「期望無限你還不全押」的朋友。

#### 推演解答

(a) 第 k 次才出正面：機率 (1/2)ᵏ，獎金 2ᵏ 元。

```text
k=1： (1/2)·2   = 1
k=2： (1/4)·4   = 1
k=3： (1/8)·8   = 1
k=4： (1/16)·16 = 1
k=5： (1/32)·32 = 1
...
E[獎金] = 1+1+1+1+1+... = ∞（無限多個 1 相加）
```

機率每次縮一半、獎金每次翻倍，乘起來永遠抵成 1。發散沒有任何算術錯誤——這是真的無限。

(b) 為什麼只付 $10？關鍵兩句話，講給朋友聽：

第一句：**那個無限大，是被「機率極小但獎金極大」的遠處項目撐起來的。** 要贏到 2³⁰ ≈ 十億，你得連續擲出 29 個反面，機率約十億分之一。這種事實際上幾乎不會發生在你這輩子玩的有限次裡。期望值把這些近乎不可能的天文獎金算了進去，但你的人生玩不了無限次。

第二句（更本質）：**你在乎的是錢的效用，不是金額。** 第一筆 100 萬改變人生，但賭局真正貢獻無限期望的那些「十億、千億」獎金，對你的實際效用幾乎沒有額外貢獻（你早就花不完了）。一旦改算「期望效用」（比如 ln(獎金)），遠處巨獎的拉力被效用的遞減壓垮，總和收斂到有限——那個有限值就對應你願意付的小錢。

一句話總結給朋友：「期望金額無限，是真的；但**期望值不是決策準則**，你該看的是期望效用和你承受得起的風險。沒人玩得起無限次，也沒人會因為多賺第十億個 100 萬而開心十倍。」

### 推演三：拆穿「期望壽命 80 所以我大概 80 歲死」 **[8 分鐘] ★**

這句話混淆了什麼？用本章的語言精確指出錯在哪，並給一個結構一模一樣的工程版誤用。

#### 推演解答

它犯的是本章的頭號陷阱：**把期望值（總體重心）當成個體的最可能結果（眾數），甚至當成個體的預測。**

拆解：

- 「期望壽命 80」是一個**期望值**：一大群人壽命的機率加權平均。它描述的是**總體**。
- 「我大概 80 歲死」是在問**個體**、而且問的是**最可能值**——「我最可能在哪一年死」。
- 這兩件事沒理由相等。壽命的分布又寬又偏（有人 30 歲意外、有人 100 歲），80 只是這團機率的重心，不是某個人「最可能」精準落點，更不是預測。就像家庭平均 1.8 個小孩，沒有一家真的生 1.8 個。

**工程版誤用**（結構完全相同）：「我們這支 API 平均延遲 50ms，所以大部分請求大概 50ms。」錯在哪：50ms 是延遲分布的重心（期望值、總體語言），但如果延遲是長尾（P50=20ms、P99=800ms），那個 50ms 被少數慢請求拉高，可能沒有任何一個請求真的接近 50ms——它既不是典型值（那是中位數 20ms），也不是你「大概會遇到」的值。把平均當典型值，跟把期望壽命當個人死期，是同一個錯。（這就是為什麼你該看 P50/P99 不該只看 average，ch12 會把這個坑挖到底。）

## 自我檢核

口頭自答，講得出來才算過：

1. 用重心的圖像解釋為什麼「期望值不是最可能值、甚至可能不是一個可能值」，並舉一個你自己想的例子（不准用 1.8 個小孩）。
2. 期望的線性性 `E[X+Y]=E[X]+E[Y]` 為什麼**不需要 X 和 Y 獨立**？從「期望就是按機率加權求和」出發，講出那個「加法可以拆、跟獨立無關」的理由。
3. 信封配對的命中期望為什麼是 1、而且跟 n 無關？把指示變數那四步講給另一個工程師聽，並指出那些指示變數其實**不獨立**、但線性性照樣成立。
4. 如果不用線性性、硬算信封配對，你會撞上什麼東西（這個東西叫什麼）？為什麼線性性能繞過它？
5. 聖彼得堡賭局的期望報酬為什麼是無限大（講出那個「每項都是 1」）？為什麼說它**不是數學算錯**？正確的解法（效用）在說什麼？
6. 為什麼期望值是個「描述工具」而不是「決策準則」？舉一個你工作中「光看期望值會做出錯誤決策」的情境。
7. 脊椎硬幣：E[單擲]=1/2 和 E[n 擲正面數]=n/2，這兩個數哪個可以用線性性瞬間拿到？怎麼拿？

## 延伸閱讀

- **丹尼爾·伯努利 1738 原始論文**〈Specimen Theoriae Novae de Mensura Sortis〉（英譯〈Exposition of a New Theory on the Measurement of Risk〉，Econometrica 1954 年的英譯流傳最廣）——聖彼得堡悖論與期望效用的源頭。讀第 17–19 節（悖論本身）和開頭談「同一筆錢對窮人富人價值不同」那段，三百年前的論證今天讀來依然清晰得驚人。
- **Stanford Encyclopedia of Philosophy,〈The St. Petersburg Paradox〉**（https://plato.stanford.edu/entries/paradox-stpetersburg/ ，2026-06 已查證）——把悖論的各種解法（效用、有限賭資、拒絕無限期望）整理得最乾淨的一篇，讀「Bernoulli's solution」一節即可掌握重點。確認它是效用問題、不是算錯。
- **指示變數與線性性**：任何一本好的機率入門（如 Blitzstein & Hwang,《Introduction to Probability》第 4 章）都會把「indicator random variables」當成一個獨立的技巧來教——值得讀的原因是它會給你更多「用線性性繞過硬計數」的招牌例子（不只信封配對），看完你會開始到處想用它。
- **錯排（derangement）/ Montmort matching problem**：MathWorld 或 Encyclopedia of Mathematics 的 derangement 條目——如果你想看看「硬算信封配對」到底有多痛（容斥原理、Dₙ 最接近 n!/e），對照之下會更感激線性性。讀到 P(完全沒命中)→1/e 那段，又會遇到那個無所不在的 e。
- **本書 ch09（變異數）**：期望值只說了「平均落在哪」，下一章問「它會抖多大」——E[單擲]=1/2 配上 Var=1/4，這枚硬幣才算被你看懂一半。**本書 ch12（連續分布與長尾）**：「平均會騙人」這章只挖了個洞，ch12 會讓你掉進去，看清為什麼長尾世界裡平均根本不該信。
