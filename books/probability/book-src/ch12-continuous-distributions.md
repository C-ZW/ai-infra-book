# ch12 — 分布動物園（下）：連續家族與長尾的警告

> **本章解決什麼問題**：ch11 帶你逛了離散家族——一次次數得出來的成敗計數。但你工作裡最常盯的那些數字其實是連續的：請求延遲、到達間隔、檔案大小，它們落在一條實數軸上、沒有「下一格」。這一章逛連續家族的三個主角：均勻分布（最大無知的連續版）、指數分布（等待時間，而且它有一個全書最反直覺的性質——無記憶性），還有跟你的儀表板每天打照面的常態分布（這裡先認個臉，「為什麼它無所不在」這個大哉問要留到 ch14 的中央極限定理才揭曉）。然後，本章最重要的一件事：帶你走進**長尾**的世界——一個「平均」會對你說謊、ch08 教你的期望值會失靈的世界。這是 Part III 分布動物園的下半場，也是你接下來進兩大定理（ch13、ch14）之前最後一塊拼圖：因為大數法則有個前提，而長尾正是那個前提崩掉的地方。

## 從你已知的出發

你打開 dashboard，看那條延遲曲線。你不會去問「下一個請求剛好是 47.3ms 的機率是多少」——這個問題的答案其實是 0，等下會講為什麼，而且這個 0 不是 bug。你會問的是「超過 200ms 的請求佔多少」「P99 是多少」。你問的全是區間、全是尾巴，從來不是單點。這就是連續隨機變數的世界：個別的點不重要，重要的是它怎麼分布在一條軸上。

你對連續分布其實已經有一整套身體直覺，只是沒人幫你命名過。我先把三個你早就在用的東西攤開：

第一，**到達間隔**。你做過 rate limiting、做過流量整形，你心裡有個模型：請求是「隨機」打進來的，兩個請求之間隔多久不固定。如果你假設請求「彼此獨立、以某個平均速率隨機到達」——這正是指數分布在描述的東西。等下你會看到它藏著一個會讓你停下來十分鐘的性質。

第二，**retry backoff 的 jitter**。你在重試延遲上灑一個 `random() * window` 的隨機抖動，為的是打散重試風暴、別讓所有 client 同一秒一起回來。那個 `random()`——在 `[0, window)` 之間每個值都等可能——就是均勻分布。你天天在 new 它。

第三，**尾端延遲**。你早就知道一件讓很多人吃過虧的事：服務延遲不是對稱的。P50 漂亮得很，50ms；P99 卻可能是 800ms。平均值夾在中間、誰都不代表。這條「中間還行、尾巴恐怖」的曲線，就是本章後半段那個長尾世界的入口。而「用平均延遲簽 SLA 會被尾巴打臉」這件事，我們會把它的數學骨架挖出來給你看。

連續分布要先處理一個離散世界沒有的怪事，我們從那裡開始。

## 密度不是機率：單點機率為 0，但事件還是會發生

ch07 已經點過這件事，這裡我們把它釘死，因為它是連續世界的入場券。

離散隨機變數（像 ch11 的硬幣正面數）你可以一格一格指派機率：P(X=0)、P(X=1)…加起來等於 1。連續隨機變數做不到。請求延遲可以是 47ms、47.3ms、47.314159…ms，軸上有無窮多個點，密密麻麻。如果你想給「剛好等於 47.3ms」這個單點一個正機率，哪怕只給一丁點，無窮多個點加起來就會爆破 1。所以唯一自洽的安排是：**每個單點的機率是 0。**

聽起來像悖論——機率 0 不是「不可能」嗎？但「延遲剛好是某個值」明明會發生啊。這裡要把兩個被你日常語言黏在一起的詞拆開：

> **機率 0 ≠ 不可能。** 在連續世界，「會發生」跟「機率為正」不是同一件事。

工程師最快聽懂的類比是浮點數。你早就學會**永遠不要寫 `if (x == 0.3)`**，要寫 `if (abs(x - 0.3) < eps)`。為什麼？因為「剛好相等」在浮點世界是個你不該指望的事件；你真正關心的、也真正可量的，是「落在一個小區間裡」。連續機率一模一樣：你不問「等於 47.3」，你問「落在 [47, 48) 裡」。後者才有正機率。

所以連續分布不用「機率質量」（每點一塊）來描述，改用**機率密度（probability density）**。密度函數 f(x) 不是機率，它是「機率的濃度」——每單位長度有多少機率。要拿到真正的機率，你得把密度在一段區間上**積起來**（積分就是「把一條曲線下的面積加總」，∫ 你大學看過，這裡只當工具用）：

```text
P(a ≤ X ≤ b) = ∫[a,b] f(x) dx      ← 密度下、a 到 b 之間的面積
P(X = 某一點)  = ∫[c,c] f(x) dx = 0  ← 寬度為 0 的區間，面積是 0
```

機率＝面積。這句話貫穿後面好幾章（ch14、ch15 會把它玩到極致，常態曲線下面積的微積分深入推導留給書架上的《馴服無限》ch15）。你現在只要記住一個翻譯：**密度的「高」告訴你哪裡更可能，但機率永遠是一段面積，不是一個高度。** 密度可以大於 1（它是濃度不是機率，沒有上限），但任何區間的面積不會超過 1。

有了這個入場券，逛動物園。

## 均勻分布：最大無知的連續版

均勻分布（uniform distribution）是連續家族裡最樸素的一隻。在區間 [a, b] 上，密度是一條水平線——每個值一樣可能，沒有偏好。

```text
密度 f(x)
  │
1/(b−a) ├───────────────┐
  │     │               │
  │     │               │
──┼─────┴───────────────┴──────► x
  0     a               b
```

那條水平線的高度是 1/(b−a)，不是隨便定的——因為整塊面積（一個寬 b−a、高 1/(b−a) 的矩形）必須等於 1。寬乘高 = (b−a) × 1/(b−a) = 1。✓ 機率守恆，自動滿足。

它「在量化什麼」？跟 ch02 古典機率的「等可能」是同一個哲學，只是搬到連續軸上：**當你對一個落在某範圍內的量一無所知、沒有任何理由偏好某個值勝過另一個，你就攤平它。** 這是無差別原則（principle of indifference）的連續版。你的 `random()`（標準函式庫那個回傳 `[0, 1)` 的）就是 Uniform(0, 1)，它是其他所有隨機數的原料：要生指數分布、常態分布、任何分布，底層都是先 new 一個均勻隨機數再去變形。

工程錨點你已經有了——retry 的 jitter。AWS 那篇講 exponential backoff 的經典文章建議你別用裸 backoff，要灑 jitter，而 full jitter 就是把下次等待時間設成 `random() * cap`，一個均勻隨機。為什麼非得均勻？因為你對「哪個重試時刻比較好」毫無資訊、也不想有偏好——你要的就是把客戶端攤平到整個時間窗，誰都別跟誰擠。最大無知，正是這裡最理性的選擇。

均勻分布還有個值得記的小事：它跟 ch02 的 Bertrand 悖論是親戚。「隨機」攤在連續空間上，必須講清楚「對哪個變數均勻」——對弦的端點均勻、對中點均勻、對半徑均勻，會給出三個不同的「均勻」、三個不同的答案。連續的等可能不是天上掉下來的，它依賴你怎麼參數化。這個坑 ch02 挖過，這裡只是提醒你它沒被填平。

均勻分布是熱身。下一隻才是這半章的明星。

## 指數分布與無記憶性：已經等了十分鐘的公車，不會比較快來

指數分布（exponential distribution）描述的是**等待時間**：一個「以固定平均速率隨機發生」的事件，距離它下一次發生還要等多久。它有一個參數 λ（讀作 lambda），叫**速率（rate）**——每單位時間平均發生幾次。等待時間的平均值是 1/λ：速率越快，等得越短，合理。

它的密度是一條從高處往右快速衰減的曲線：

```text
密度 f(x) = λ·e^(−λx)
  │
 λ├╲
  │ ╲
  │  ╲___
  │      ╲────___
──┼───────────────────────► x（等待時間）
  0
```

對工程師真正好用的不是密度，是**尾巴機率**（survival function）——「等超過時間 x 還沒發生」的機率。它長這樣，乾淨到值得背：

```text
P(X > x) = e^(−λx)      ← 「等了 x 還沒等到」的機率
```

直覺檢查一下：x=0 時，e⁰=1，「等超過 0 秒」當然必然成立；x→∞ 時，e^(−λx)→0，「永遠等不到」機率趨近 0。形狀對。這個 e^(−λx) 你其實見過——ch11 的卜瓦松分布裡那個 e⁻¹。它們不是巧合，是同一個過程的一體兩面：如果「每段時間發生幾次」服從卜瓦松，那麼「兩次之間隔多久」就服從指數。這個橋——卜瓦松過程、到達間隔、然後怎麼拿它去算排隊、算等待——是《佇列論》的主場，運算深度（見《佇列論》ch04、ch05）就好，本書不重推。本書要講的是它的**美**。

美在哪？盯著那條尾巴機率，做一個小代數，你會撞見一件冒犯直覺的事。

### 無記憶性：算給你看

問一個問題。你在公車站等公車，假設公車到達間隔服從指數分布。你已經等了 s 分鐘，公車還沒來。問：**接下來還要再等超過 t 分鐘的機率**，跟一個剛到站、什麼都沒等的人「要等超過 t 分鐘」的機率，哪個大？

直覺尖叫：當然是已經等了 10 分鐘的人比較虧、也比較該輪到他了吧？等越久不是該越快來嗎？

算算看。「已經等了 s、還要再等超過 t」這是個條件機率：在「已經等超過 s」的條件下，問「等超過 s+t」。用 ch04 的條件機率定義：

```text
P(X > s+t | X > s) = P(X > s+t 且 X > s) / P(X > s)
```

注意「X > s+t」本身就蘊含「X > s」（等超過 11 分鐘的人，當然也等超過了 10 分鐘），所以分子的交集就是 P(X > s+t)。代入指數的尾巴公式：

```text
                      P(X > s+t)        e^(−λ(s+t))
P(X > s+t | X > s) = ───────────── = ────────────────
                       P(X > s)           e^(−λs)
```

指數律的乘法在這裡幫你一個大忙：e^(−λ(s+t)) = e^(−λs) · e^(−λt)。把 e^(−λs) 約掉：

```text
       e^(−λs) · e^(−λt)
   = ──────────────────── = e^(−λt) = P(X > t)
            e^(−λs)
```

得到：

```text
P(X > s+t | X > s) = P(X > t)
```

讀出來：**「已經等了 s、接下來還要等超過 t」的機率，等於「從零開始要等超過 t」的機率。** 你等過的那 10 分鐘，完全沒有讓公車變得比較快來。系統把你忘了。這就叫**無記憶性（memorylessness）**。

我認為這是整章最該讓你停下來的一頁。停十分鐘，因為你的直覺正在騙你，而且騙得很有道理——只是錯了。

### 為什麼這麼反直覺，又為什麼是對的

你的直覺哪裡出錯？它偷偷把指數分布想成「人類等公車的真實經驗」——時刻表、班距、越接近發車越可能來。但指數分布描述的是一個**沒有時刻表、純隨機**的到達：每一個瞬間，事件發生的「傾向」都一模一樣，跟過去發生過什麼、等了多久，毫無關係。在這種世界裡，「等了 10 分鐘」不是「進度條走了一半」，它什麼進度都沒累積——因為根本沒有進度條。下一刻會不會來，只看下一刻自己，不看歷史。

放射性原子是最乾淨的真實例子。一個還沒衰變的原子，它「再撐 t 秒不衰變」的機率，跟一個剛產生的原子一模一樣。原子不會「老」。它沒有記憶。指數分布就是這種「不會老」的數學。

這裡有個漂亮的事實值得你轉述給另一個工程師：**指數分布是唯一的連續無記憶分布。** 不是「之一」，是唯一。任何一個連續的、正值的、滿足 P(X>s+t | X>s) = P(X>t) 的分布，數學上被逼著只能是指數。證明的骨架不難懂：無記憶性那條方程式等價於 g(s+t) = g(s)·g(t)（其中 g 是尾巴函數），而連續函數裡唯一滿足「相加變相乘」的就是指數函數 e 的某次方——這跟對數律「相乘變相加」是同一件事的鏡像。所以「無記憶」這個性質，幾乎是用一句話就把指數分布從所有連續分布裡單獨揪了出來。一個直覺性質，唯一決定了一整個分布的形狀，這種「性質反推形狀」的乾淨，是機率裡少見的奢侈品。

順帶把 ch11 接回來：離散世界也有一個唯一的無記憶分布，就是 ch11 的幾何分布——「擲到第幾次才第一次成功」。連續硬幣每一次都是新的開始，所以指數無記憶；離散硬幣每一擲也是新的開始，所以幾何無記憶。指數是幾何在連續時間的極限版。同一個「系統不記得你」的精神，一個跑在格子上，一個跑在實數軸上。

### 這個性質坑過你嗎？

坑過，而且你大概沒意識到。如果你假設到達間隔是指數的（很多排隊模型、很多模擬器的預設就是這樣），那你等於假設了「不管系統現在已經閒置/忙碌多久，下一個請求來不來的傾向都一樣」。這在「請求彼此獨立、沒有同步來源」時是很好的近似。但現實裡請求常常**不是**獨立的——cron job 整點齊發、重試風暴、快取同時過期，這些都會製造「成群結隊」的到達，把無記憶性打破。你拿指數模型去估那種流量的尾端，會低估擁塞。記住：無記憶是個漂亮的假設，不是物理定律；用它之前先問「我的到達真的沒有記憶嗎」。

## 常態分布：先認個臉，為什麼留到 ch14

連續家族的第三位主角，你早就認得它的長相——那條對稱的鐘形曲線，中間胖、兩邊瘦：

```text
密度
  │           .─''─.
  │         .'      '.
  │        /          \
  │      .'            '.
  │   _.'                '._
──┴──────────┼──────────────► x
            μ
```

常態分布（normal distribution，台灣統計用語，不是「正態」）有兩個參數，講出來你就懂：μ（mu，平均值）決定鐘形的中心在哪、σ（sigma，標準差，ch09 那個「不確定的大小」）決定它有多胖。μ 平移、σ 縮放，就這兩個旋鈕。寫成 X ~ N(μ, σ²)。

這裡我**故意不講**它的密度公式（那個帶 e^(−x²/2) 的東西），也**故意不講**它為什麼無所不在。因為「為什麼鐘形會出現在身高、測量誤差、考試成績、你把一萬個請求延遲平均起來的結果裡」——這個問題的答案是整本書最大的驚嘆點之一，它叫中央極限定理，它值得一整章（ch14），現在先講會破壞那個高潮。

但我得先在這裡幫你**埋一個對比**，因為它正是本章後半段的引爆點：

常態分布有一條**極瘦的尾巴**。它的密度在兩邊以 e^(−x²/2) 的速度往下掉——那個 x² 在指數上，意思是離中心越遠，機率不是變小，是**塌陷**。離平均 3 個標準差以外的事件，機率只剩 0.27%（ch15 會給你精確的 68-95-99.7）；離 5 個標準差以外，機率小到工程上你可以當它不存在。常態世界裡，極端值幾乎不發生。你的直覺——「離平均很遠的怪數據是異常、很罕見」——在常態世界裡是對的。SRE 的 3-sigma 告警就建立在這個信任上。

記住這個信任。因為本章最後要帶你去一個地方，那裡這個信任會徹底崩掉。

## 長尾與冪律：平均失去意義的世界

現在進本章的暗面，也是它最重要的貢獻：有一整類分布，常態的那套直覺在裡面**全錯**。

### 一個首富走進酒吧

有個統計學界講爛了的笑話，但它精準到值得再講一次。一間小酒吧裡坐了十個普通上班族，年收入都在台幣百來萬上下。這時世界首富走了進來、坐下。

現在這間酒吧裡所有人的**平均**年收入，瞬間變成幾百億。「這間酒吧的客人平均年收入幾百億」——這句話統計上完全正確，而且荒謬到毫無意義。沒有任何一個原本的客人變有錢，但「平均」這個字被一個人徹底綁架了。經濟學家 Krugman 很愛用這個例子講收入不平等（2026-06，已查證：「比爾·蓋茲走進酒吧」是這類教學的經典版本）。

對比一下：如果走進來的是一個「比平均高三個標準差」的人——在常態世界裡那已經是萬中選一的高個子了——他對平均的影響微乎其微。常態世界裡沒有人能一個人改寫平均，因為極端值被那條 e^(−x²/2) 的尾巴掐死了，根本不會出現大到那種程度的個體。

收入不是常態的。收入是**長尾**的。

### 冪律：尾巴衰減得太慢

長尾分布（長尾、heavy tail、肥尾 fat tail，幾個詞指向同一族現象）的數學核心是**冪律（power law）**。常態的尾巴用 e^(−x²) 塌陷；冪律的尾巴只用 x 的某個負次方往下走：

```text
P(X > x) ≈ x^(−α)      ← 冪律尾巴，α 是尾巴指數
```

差別有多大？指數/常態的尾巴是「指數級」變薄，冪律是「多項式級」變薄。多項式比指數慢得多得多。意思是：在冪律世界裡，超大的值不是不可能，而是**罕見但遠比你以為的常見**。一個比典型值大 100 倍的個體，在常態世界裡機率是天文數字分之一；在冪律世界裡，它只是「比較少見」而已，你遲早會撞到。

這族分布最有名的招牌叫 **80/20**：少數佔多數。20% 的人擁有 80% 的財富、20% 的客戶貢獻 80% 的營收、20% 的程式碼藏了 80% 的 bug。這裡有個常被講錯、landscape 特別標注的史實，我幫你校正：

> 義大利經濟學家 **Vilfredo Pareto** 在 1890 年代觀察到「約 20% 的人擁有約 80% 的土地」，這個**冪律觀察**歸他。但「**80/20 原則**」「**Pareto principle**」這個**名稱**，是品管顧問 **Joseph Juran 在 1940 年代**取的、並把它推廣到管理與品管領域——**不是 Pareto 自己取的**（2026-06，已查證）。Juran 還造了「vital few, trivial many（關鍵的少數、瑣碎的多數）」這個說法。所以下次有人說「這是 Pareto 提出的 80/20 法則」，你可以淡定補一句：觀察是 Pareto 的，這個名字是 Juran 五十年後貼上去的。

這個分布在你的系統裡到處都是，而且你早就靠它吃飯。**快取**就是冪律的直接受益者：少數熱點 key 佔了絕大多數請求，所以你只要把那 20%（甚至更少）的熱 key 留在記憶體，就能命中 80%＋的流量——這就是為什麼 cache 那麼小卻那麼有效。如果 key 的存取是均勻分布的（每個 key 一樣熱），快取幾乎沒用；正是因為存取冪律得厲害，快取才成立。檔案大小、城市人口、網站連入連結數、字詞出現頻率（Zipf 定律），全是冪律。你住在一個冪律的世界裡。

### 期望存在，但樣本平均不穩——給 ch13 的警告

現在來講本章最該讓你帶走的那句話，它直接連到下一章的大數法則。

先說清楚：本章的長尾**還沒到「期望值是無限大」那種極端**（冪律指數夠小的時候，連 ch08 的期望值都會發散到無限，那是更深的水，這裡點到、嚴格條件留延伸閱讀）。本章要警告的是一個更陰險、更日常的版本——**就算理論上的期望值好端端地存在，你手上那個「樣本平均」也可能完全不可信。**

差別在哪？ch13 的大數法則會告訴你「樣本平均會收斂到真實期望」，聽起來樣本平均是安全的。但大數法則收斂的**速度**，在長尾下慢到讓它在實務上失效。因為樣本平均隨時可能被你下一個抽到的巨值整個掀翻——首富隨時可能走進你的樣本。常態世界裡你抽 30 個樣本，平均就穩了；冪律世界裡你抽 30 萬個，平均還在跳，因為第 300,001 個可能比前面所有人加起來還大。

> **驚嘆點，請務必能轉述**：在長尾世界，「平均」不是一個穩定的、能代表群體的數，它是一個**被最大那筆觀測值牽著走的**數。你算出來的平均，主要在告訴你「目前為止抽到的最大值有多大」，而不是「典型情況長怎樣」。這就是為什麼講收入要用**中位數**不用平均、為什麼講延遲要看**百分位數**不看平均。中位數（首富走進酒吧，它紋風不動）對極端值免疫；平均沒有。

下面用兩組小資料把這件事算給你看——這是本章的 worked example。

## Worked example：兩個手算，一個無記憶、一個長尾

### 例一：指數無記憶性，數字驗證

取 λ = 0.5（平均等待 1/λ = 2 個時間單位）。用尾巴公式 P(X > x) = e^(−λx)，需要的指數值（標準值，背一下方便）：

```text
e^(−1)   ≈ 0.3679
e^(−0.5) ≈ 0.6065
e^(−1.5) ≈ 0.2231
```

**先算 P(X > 2)**——「從零開始，等超過 2 個單位」：

```text
P(X > 2) = e^(−0.5 × 2) = e^(−1) ≈ 0.3679
```

**再算 P(X > 3 | X > 1)**——「已經等了 1 個單位，還要再等超過 2 個單位（總共超過 3）」：

```text
P(X > 3 | X > 1) = P(X > 3) / P(X > 1)
                 = e^(−0.5×3) / e^(−0.5×1)
                 = e^(−1.5) / e^(−0.5)
                 = 0.2231 / 0.6065
                 ≈ 0.3679
```

**0.3679 = 0.3679。一模一樣。** 「已經等了 1 個單位、再等超過 2」跟「從零等超過 2」機率完全相等。你等過的那 1 個單位，蒸發了，系統不認帳。交叉驗算的方法二：直接用無記憶公式 P(X>3|X>1) = P(X>3−1) = P(X>2) = e^(−1) ≈ 0.3679，秒得同一答案。兩條路會合，無記憶性數字確認。✓

### 例二：長尾下，拿掉最大那筆，平均崩給你看

造一組 80/20 的小資料——五個「客戶營收」，刻意做成長尾（少數佔多數）：

```text
長尾資料：  2,  3,  4,  5,  186
```

（前四筆是普通客戶，第五筆是個鯨魚客戶——你的系統裡一定有這種人。）

```text
總和 = 2 + 3 + 4 + 5 + 186 = 200
平均 = 200 / 5 = 40
```

平均是 40。但**五個客戶裡有四個的營收低於 40**，「平均」根本不代表任何一個典型客戶。中位數呢？排序後中間那筆是 4——這才像話。平均（40）和中位數（4）差了十倍，這個巨大裂縫本身就是「你在看一條長尾」的警報。

現在做關鍵實驗：**拿掉最大那筆（186）**，重算：

```text
剩下：       2, 3, 4, 5
總和 = 14，   平均 = 14 / 4 = 3.5
```

平均從 **40 暴跌到 3.5**——掉了超過 91%。**一筆資料的有無，把整個平均砍掉九成。** 這就是「樣本平均不穩」的具體長相：你的平均幾乎完全是那一筆巨值撐起來的，它一走，平均就垮。

對照組——同樣五個數，但讓它「常態一點」（對稱、無極端值）：

```text
常態風資料： 38, 39, 40, 41, 42
總和 = 200，  平均 = 40
```

平均也是 40，跟長尾組故意湊成一樣。現在同樣**拿掉最大那筆（42）**：

```text
剩下：       38, 39, 40, 41
總和 = 158，  平均 = 158 / 4 = 39.5
```

平均從 40 變成 39.5——**只動了 1.25%**。幾乎沒感覺。

把兩組並排，結論一目了然：

```text
                     拿掉最大值前   拿掉最大值後   平均變動
─────────────────────────────────────────────────────────────
長尾   2,3,4,5,186        40            3.5         −91%  ← 平均被單筆綁架
常態   38,39,40,41,42     40            39.5        −1.25% ← 平均很穩
─────────────────────────────────────────────────────────────
```

同樣是「平均」，同樣的五個數量、同樣的起始平均 40，只因為一組是長尾、一組是常態，「拿掉一筆」的後果天差地別。**這裡了不起在哪、也是你要能轉述的**：在常態世界，平均是一個可信賴的群體代表，極端值動不了它（ch13 大數法則在這種世界跑得飛快）；在長尾世界，平均是一個建在沙上的數，最大那筆觀測值想讓它垮就讓它垮。同一個「算平均」的動作，在兩個世界裡一個是磐石、一個是流沙。你的延遲、你的營收、你的檔案大小，住在哪個世界，決定了「平均」這個字能不能信。

## 紙上推演

### 推演題一：把無記憶講成人話 **[15 分鐘]** ★★

「已經等了十分鐘的公車不會比較快來」這句話為什麼反直覺、又為什麼（在指數假設下）是對的？請用你自己的話講清楚兩件事：(a) 人的直覺為什麼會覺得「等越久該越快來」；(b) 指數分布的世界裡，是什麼東西「不存在」，導致這個直覺失效。不准用公式，講給一個沒學過機率的朋友聽。

#### 推演解答

(a) 人的直覺把等待想成「進度條」或「排隊」——時間在累積某種「該輪到我」的額度，等越久、額度越滿、越接近兌現。這在有時刻表的真實公車、在先到先服務的排隊裡是對的，所以直覺很頑固。

(b) 指數分布的世界裡，**「進度」這個東西根本不存在**。事件在每一個瞬間發生的傾向都一模一樣，跟過去等了多久完全脫鉤——系統沒有一個「記得你等了多久」的計數器。你等的那十分鐘沒有被存在任何地方、沒有累積成任何優勢，所以它當然不會讓下一刻更可能來。把它講成一句話：**指數分布描述的是「沒有記憶、沒有排隊、純隨機冒出來」的事件；既然系統不記得你來過，它就不可能因為你等久了而對你好一點。** 放射性原子不會老、不會「快衰變了」，是同一件事。

關鍵自我檢查：如果你發現自己在說「那它總得平衡一下吧」，你就掉回賭徒謬誤了——無記憶的意思就是「永遠不平衡，因為它根本不記帳」。

### 推演題二：「平均延遲 50ms」可能在說哪些謊 **[20 分鐘]** ★★

你的監控說「我們系統平均延遲 50ms」。請寫出**三個**情境，在每個情境裡這句話雖然數字正確、卻嚴重誤導使用者體驗。每個情境要講清楚：尾巴長什麼樣、誰被傷到、你該改看哪個指標。

#### 推演解答

三個情境（能講出兩個就算過關，三個是滿分）：

**情境一：雙峰。** 90% 的請求走快取、5ms 命中；10% 的請求 miss、打到後端、450ms。平均 = 0.9×5 + 0.1×450 = 4.5 + 45 = 49.5ms ≈ 50ms。漂亮的 50ms 底下，根本沒有任何一個請求「大約 50ms」——你的使用者要嘛爽快（5ms）、要嘛痛苦（450ms），沒有中間。被傷到的是那 10% miss 的人。該看：P90/P99，或乾脆看延遲的**直方圖**（一眼看出雙峰）。

**情境二：重尾。** P50=30ms、P90=60ms，但 P99=1.5 秒（GC 暫停、鎖競爭、冷連線）。少數巨值把平均往上拉到 50ms。被傷到的是那 1%——而那 1% 常常是發了最多請求的重度使用者，所以**以人計算**受害面遠大於 1%。該看：P99、P99.9，和**最大值**。

**情境三：平均被偶發尖峰污染但時間上集中。** 平日 P99 都很好，但每天備份視窗那半小時延遲全面飆到秒級。把一整天平均起來還是 50ms，因為壞的只有 30 分鐘。被傷到的是剛好在那半小時來的人。該看：延遲對**時間**的分布（時序圖），不是把時間抹平的單一平均。

共同教訓：平均是個會把尾巴、把雙峰、把時間結構全部抹平的指標。任何一個「平均看起來沒事」的系統，都值得你追問一句「那 P99 呢、那直方圖呢」。

### 推演題三：把「20% 的 key 佔 80% 流量」翻成冪律語言 **[15 分鐘]** ★★

你的 cache team 說「20% 的 key 佔了 80% 的請求流量」。(a) 這句話在描述哪一族分布？用本章的詞彙講。(b) 解釋這個分布形狀如何決定你的快取命中率——並說明如果存取改成均勻分布，快取會發生什麼事。

#### 推演解答

(a) 這是**冪律 / 長尾**分布（80/20 是它的招牌比例，名稱 Pareto principle 由 Juran 取，觀察由 Pareto 提出）。它在說 key 的「熱度」極度不均：少數熱 key 被狂打、絕大多數冷 key 幾乎沒人碰。「20% 的 key → 80% 流量」正是「少數佔多數」的長尾特徵。

(b) 快取命中率直接吃這個形狀的紅利。因為流量集中在少數熱 key 上，你**只要把那 20% 的熱 key 留在快取**（甚至更少，因為熱度本身在那 20% 裡可能又是 80/20），就能攔下 80%＋的請求——這就是為什麼一個容量遠小於 key 總數的 LRU 快取卻能有很高命中率：它自然會把熱 key 留住、把冷 key 淘汰。

對照：如果存取是**均勻分布**（每個 key 一樣熱、沒有熱點），那麼一個只裝得下 20% key 的快取，期望命中率就只有約 20%——因為下一個請求落在「快取裡那 20%」的機率就是 20%，沒有任何集中度可以利用。**快取之所以有效，本質上是在賭存取分布是長尾的；存取越長尾，同樣大小的快取越值錢。** 反過來，如果你的命中率莫名其妙地低，一個該懷疑的方向是：你的存取模式是不是比你以為的更均勻（例如被某種掃描、某種無熱點的隨機 key 打散了）。

## 自我檢核

口頭自答，講得出來才算過關：

1. 連續隨機變數「單點機率為 0」跟「這個值不會出現」為什麼不是同一件事？用浮點數比較的類比講一次。
2. 密度可以大於 1 嗎？如果可以，那它跟「機率不超過 1」怎麼不矛盾？（提示：濃度 vs 面積。）
3. 無記憶性用一句話是什麼意思？為什麼「已經等了很久」在指數分布裡不是有利資訊？
4. 為什麼說「指數分布是唯一的連續無記憶分布」——這個「唯一」憑什麼？（講出「相加變相乘只有指數函數能做到」的骨架即可。）
5. 常態分布的尾巴和冪律的尾巴，差別在哪？為什麼這個差別讓「離平均很遠＝異常」這條直覺在長尾世界失效？
6. 首富走進酒吧，為什麼平均年收入會爆炸、中位數卻紋風不動？這對你看延遲指標有什麼直接啟示？
7. 「我們平均延遲 50ms」這句話，至少講一個它在說謊的方式。
8. 為什麼快取在「存取是長尾」時特別有效、在「存取是均勻」時幾乎沒用？
