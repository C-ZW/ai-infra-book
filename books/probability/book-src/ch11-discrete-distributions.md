# ch11 — 分布動物園（上）：離散家族

> **本章解決什麼問題**：前面四章（ch07–10）你學會把隨機結果變成數字（隨機變數）、算它的長期平均（期望）、算它的抖動大小（變異數）、算兩個變數怎麼一起動（共變異）。但這些都是「對任意分布都成立」的通用工具。現在要做的事相反：認識幾個**具名的、反覆出現的**分布——它們不是隨便長出來的，而是某種固定情境的指紋。本章只收離散家族（結果是可數的、一顆一顆的）：伯努利、二項、幾何、卜瓦松。重點不是背公式，是看出它們其實是同一條血脈，而且最後那隻卜瓦松，會給你本章的驚嘆點——一大堆看起來毫不相干的「稀有計數」，竟然都長同一個形狀。連續家族（指數、常態、長尾）留給 ch12。

## 從你已知的出發

你其實已經跟這幾隻動物打過很多年交道，只是沒人幫你貼上學名。

你寫過重試邏輯。一次外部 API 呼叫，要嘛成功要嘛失敗——這是**伯努利（Bernoulli）**，一次成敗。你設了 `maxRetries: 100`，那這 100 次裡會成功幾次？這個「成功數」服從**二項（binomial）**分布。你關心的其實常常不是「成功幾次」，而是「第幾次才終於成功」——backoff 重試到第三次才通，那個「要等到第幾次」服從**幾何（geometric）**分布。

然後是最有意思的那隻。你盯著 Grafana 上「每分鐘 5xx 錯誤數」「每秒進來幾個請求」這種計數型 metric。它們有個共同點：在任何一個極短的瞬間，發生一次的機率都很低，但時間軸夠長、機會夠多，總量就積出來了。這種「大量機會、每次都罕見、加起來的計數」，幾乎一定可以用**卜瓦松（Poisson）**分布來近似。你設告警閾值時，心裡其實在問「正常情況下這個計數會飆到多少」——那條你憑經驗抓的上界，背後就是卜瓦松。

我先把本章的地圖攤開，因為這四隻動物不是平行並列的四個東西，它們是一條有方向的血脈：

```text
       伯努利              二項                幾何
     單次成敗      n 次獨立、數成功數    第一次成功要等幾次
    （1 個位元）  ──────────────►   （換個問法：問「等多久」）
         │              │                    │
         │              │ n→∞、p→0           │ 離散版「無記憶性」
         │              │ np=λ 固定          │ 預告 ch12 指數
         │              ▼                    ▼
         │           卜瓦松              （見 ch12 連續家族）
         │       稀有事件的普適形狀
         └──── 全都是「一連串獨立成敗」的不同問法 ───┘
```

伯努利是原子——一個位元、一次成敗。二項是「把 n 個位元加起來數有幾個 1」。幾何是「同一串位元，問第一個 1 出現在哪」。卜瓦松是二項在某個極限下的影子。看懂這條血脈，你就不用把它們當四個獨立公式背，而是當同一個故事的四種問法。

## 伯努利：一個位元的分布

最小的非平凡隨機變數，就是只有兩個值的那種。一次擲幣（正／反）、一次 API 呼叫（成功／失敗）、一封信是不是垃圾信（是／否）。把其中一個結果叫「成功」、編號 1，另一個叫「失敗」、編號 0，成功機率是 p。這就是**伯努利分布**，記成 X ~ Bernoulli(p)。它的 PMF（probability mass function，機率質量函數）短到一行：

```text
P(X = 1) = p          ← 成功
P(X = 0) = 1 − p      ← 失敗
```

就這樣。它窮到只有一個參數 p，但別小看它——它是整個離散家族的原子，後面三隻都是拿它疊出來的。

ch08 跟 ch09 其實已經把它的期望跟變異算過了，這裡複習一下、順便看出一個漂亮的小事實。期望值是「值乘以機率的加總」：

```text
E[X] = 1·p + 0·(1−p) = p          ← 期望就是成功機率本身
```

這很合理：如果成功記 1、失敗記 0，那長期平均下來就是成功的比例，也就是 p。變異數呢，用 ch09 的 Var(X) = E[X²] − (E[X])²。因為 X 只取 0 跟 1，X² 跟 X 一模一樣（0²=0、1²=1），所以 E[X²] = E[X] = p，於是：

```text
Var(X) = E[X²] − (E[X])² = p − p² = p(1−p)
```

`p(1−p)` 這個東西你盯著看一下，它在 p=1/2 的時候最大（等於 1/4），往兩端 p→0 或 p→1 的時候趨近 0。這完全符合直覺：一個幾乎一定成功（p≈1）或幾乎一定失敗（p≈0）的事件，幾乎沒有不確定性，變異數當然小；最不確定、最難猜的，是 50/50 那一刻。

**這是 ch01 那枚硬幣**：一枚公正硬幣的單擲，就是 Bernoulli(1/2)。E[X]=1/2、Var(X)=1/4、SD=1/2——這幾個數你在 ch08、ch09 已經見過，現在它們有了學名。脊椎硬幣的第一層，是一隻伯努利。

## 二項：把 n 個位元加起來

現在把那枚硬幣擲 10 次，問「出現幾次正面」。或者把那支 API 重試 100 次，問「成功幾次」。把同一個伯努利試驗**獨立重複 n 次**，數其中「成功」的總數，這個總數就服從**二項分布**，記成 X ~ Binomial(n, p)。

關鍵字是「獨立」跟「相同」——每次試驗成功機率都是同一個 p，而且彼此不影響。你的重試如果每次都是對同一個獨立服務、條件相同，那這個假設大致成立；但如果失敗是因為服務掛了（那接下來幾次也會跟著掛），那就**不獨立**了，二項分布會騙你。記住這個前提，它正是工程裡最常被偷偷違反的地方（ch04 講過的相關性故障，換個場景又回來了）。

二項分布的 PMF 是這樣的，我們不要先丟公式，先用「想得出來」的方式把它推出來：

要在 n 次裡剛好成功 k 次，得回答兩個問題。第一，**哪幾次成功**？n 次裡挑 k 次當成功的位置，有 C(n,k) 種挑法（n 取 k，組合數，ch03 複習過）。第二，**某一種特定的成敗序列**機率是多少？比方說「成、成、…、成（k 個）、敗、敗、…、敗（n−k 個）」，因為獨立，機率就是各次相乘：pᵏ·(1−p)ⁿ⁻ᵏ。而每一種有 k 個成功的序列，機率都一樣（不管成功擺哪幾個位置，總是 k 個 p 乘 n−k 個 (1−p)）。所以把「位置的種數」乘上「每種的機率」：

```text
                k       n−k
P(X = k) = C(n,k) · p · (1−p)        k = 0, 1, 2, ..., n
```

三個因子，你要能一個一個指著唸出它在幹嘛：C(n,k) 是「成功擺在哪幾次」的種數，pᵏ 是「那 k 次都成功」，(1−p)ⁿ⁻ᵏ 是「其餘 n−k 次都失敗」。沒有任何一項是憑空冒出來的。

期望跟變異不用重推——二項就是 n 個獨立伯努利相加，而 ch08 的期望線性性說「和的期望＝期望的和」（連不獨立都成立，這裡更是獨立），ch09 的「獨立和變異數相加」也直接套用。每個伯努利期望 p、變異 p(1−p)，加 n 個：

```text
E[X]   = n·p             ← n 次、每次平均貢獻 p 個成功
Var(X) = n·p(1−p)        ← 獨立，所以變異數直接相加
```

**這是 ch01 那枚硬幣**：n 擲公正硬幣的正面數 ~ Binomial(n, 1/2)，E=n/2、Var=n/4、SD=√n/2。脊椎硬幣的第三層。這個 √n/2 之後在 ch13、ch14 會變成全書最反直覺的一頁的主角，先記著它。

### Worked example：脊椎硬幣 Binomial(10, 1/2) 的完整 PMF 與直方圖

把硬幣擲 10 次，正面數從 0 到 10，每個的機率是 C(10,k)·(1/2)¹⁰ = C(10,k)/1024。分母都是 2¹⁰ = 1024（每次 2 種結果、10 次、獨立，總共 1024 種等可能序列），分子就是組合數。我手算列一遍 C(10,k)，你可以拿巴斯卡三角形對：1, 10, 45, 120, 210, 252, 210, 120, 45, 10, 1。加起來剛好 1024（交叉驗算：1+10+45+120+210+252+210+120+45+10+1 = 1024 ✓，這就是 Σ C(10,k) = 2¹⁰）。

```text

─────────────────────────────────────────
 0       1         1/1024            0.0010
 1      10        10/1024            0.0098
 2      45        45/1024            0.0439
 3     120       120/1024            0.1172
 4     210       210/1024            0.2051
 5     252       252/1024            0.2461   ← 最高，正中央
 6     210       210/1024            0.2051
 7     120       120/1024            0.1172
 8      45        45/1024            0.0439
 9      10        10/1024            0.0098
10       1         1/1024            0.0010
─────────────────────────────────────────
 合計  1024        1024/1024         1.0000   ← 機率總和為 1 ✓
```

把機率畫成橫條（每個 █ 約代表 0.005），你會看到一件值得停下來的事：

等等，k=8 那行畫太長了——我故意留這個錯給你抓：k=8 的機率是 0.0439，跟 k=2 一樣，應該跟 k=2 等長。對稱性是這張圖的鐵律，因為公正硬幣 p=1/2，「正面 k 次」跟「反面 k 次（也就是正面 10−k 次）」機率必然相等。正確的圖左右完全對稱，中間 k=5 最高，往兩邊一階一階矮下去。

這個形狀——**對稱、中間鼓、兩邊薄**——就是鐘形的雛形。現在只有 11 根柱子、還很粗糙，但你已經能看出端倪了。把 n 從 10 推到 100、1000，柱子愈來愈密、輪廓愈來愈滑，它會收斂到一條精確的鐘形曲線。**這正是 ch14 中央極限定理的劇透**，而且不是隨便哪條鐘形——歷史上第一個中央極限定理，就是棣美弗（de Moivre）1733 年在算這個二項分布的常態近似（照 landscape，ch14 細講）。脊椎硬幣從一隻伯努利，疊成二項，正在悄悄長成常態。你現在看到的這張對稱直方圖，是那條故事線的第一幀。

## 幾何：不問「成功幾次」，問「等到第幾次」

二項問的是「n 次裡成功幾次」。幾何問一個方向相反的問題：**一直試下去，第一次成功出現在第幾次？**

這在工程裡其實更貼近你的直覺。你的 backoff 重試不會傻傻跑滿 100 次——一成功就停了。你真正關心的是「平均重試到第幾次會通」。或者你在等一個罕見事件第一次發生：抽獎抽到第幾次中、壓測打到第幾個請求才觸發那個 race condition。這個「第一次成功的等待次數」服從**幾何分布**。

設每次成功機率 p、各次獨立。要「第 k 次才第一次成功」，意思是前 k−1 次全都失敗、第 k 次成功。因為獨立，直接相乘：

```text
P(X = k) = (1−p)^(k−1) · p        k = 1, 2, 3, ...
            └ 前 k−1 次全敗 ┘  └ 第 k 次成功
```

注意 k 可以是 1、2、3……一直到無窮——理論上你可能一直失敗、永遠等不到（雖然機率愈來愈小）。期望值（第一次成功平均要等幾次）是 **E[X] = 1/p**，這個我們待會在紙上推演用脊椎硬幣手算驗證它等於 2。直覺上也合理：成功機率 1/10 的事，平均要試 10 次才中第一次。

### 離散版的無記憶性：硬幣不記得你等了多久

幾何分布藏著本章第二個值得驚嘆的東西，而它正是賭徒謬誤（ch13 會專門處理）的數學根源。

假設你已經失敗了 10 次，還沒成功。問：「我已經等了 10 次了，接下來是不是該成功了？該輪到我了吧？」

數學的回答冷酷得很美：**不。你前面等了多久，完全不影響未來。** 用條件機率寫出來——「已經失敗 m 次的前提下，還要再等超過 n 次」的機率，等於「從頭開始還要等超過 n 次」的機率：

```text
P(X > m + n | X > m) = P(X > n)
```

我們手推一下為什麼。先算「至少要等超過 k 次」是多少——那等於「前 k 次全部失敗」，也就是 P(X > k) = (1−p)ᵏ。代進條件機率的定義（ch04：限縮到條件再標準化）：

```text
                       P(X > m+n)     (1−p)^(m+n)
P(X > m+n | X > m) = ─────────────── = ───────────── = (1−p)^n = P(X > n)
                        P(X > m)        (1−p)^m
```

那個 (1−p)^m 上下消掉了——**你過去累積的失敗，從分子分母一起被約掉，對未來毫無影響。** 這就是無記憶性（memorylessness）。每一次擲幣都是嶄新的、跟前面無關的一次，硬幣沒有記憶體、不存帳、不欠你一個正面。

這裡有個值得寫進書的硬事實：**幾何分布是唯一具有無記憶性的離散分布**（2026-06，已查證）。沒有別的離散分布能做到這件事。它在連續世界有個雙胞胎——指數分布，那是唯一無記憶的連續分布，是「等待時間」的版本（公車已經等了十分鐘也不會比較快來）。**ch12 會專門講指數的無記憶性之美**，這裡先讓你看到它的離散原型。一旦你內化了「每次都是全新的、過去被約分掉了」，你就永遠不會再相信「連輸五把該贏了」這種話——那種「該平衡了」的直覺，正是把無記憶的東西想成有記憶。

## 卜瓦松：稀有事件的普適形狀（本章的驚嘆點）

現在來看本章真正了不起的那隻。前面三隻都還算「你想得到」，卜瓦松不一樣——它會讓你停下來想「為什麼這麼多不相干的東西，竟然長一模一樣」。

先講它從哪來。卜瓦松不是憑空定義的，它是**二項分布在一個極限下的影子**。設想一個情境：機會非常多（n 很大），但每次機會發生的機率非常小（p 很小），而你只在乎「平均發生幾次」這個量，把它固定下來叫 λ（lambda），也就是 λ = n·p 保持不變。

這個情境到處都是。一天有 86400 秒（n 大），每一秒進來一個特定使用者請求的機率很小（p 小），但「平均每天幾個」是個穩定的數（λ）。一個城市有幾百萬人（n 大），每個人今天出某種罕見車禍的機率極小（p 小），但「平均每天幾起」很穩定。普魯士某個軍團有一堆士兵跟一堆馬（n 大），任一個士兵今年被馬踢死的機率微乎其微（p 小），但「平均每年踢死幾個」是個固定的數。

當 n→∞、p→0、而 np=λ 固定時，二項分布 Binomial(n, p) 收斂到一個只剩單一參數 λ 的分布，這就是**卜瓦松分布**：

```text
         e^(−λ) · λ^k
P(X = k) = ─────────────        k = 0, 1, 2, 3, ...
              k!
```

那個 e（≈2.71828）從哪冒出來的？這正是卜瓦松最迷人的地方——它是極限過程裡 (1−p)ⁿ 這一項在 n→∞ 時自然趨近 e^(−λ) 的結果（你大學在《馴服無限》看過 lim(1+x/n)ⁿ → eˣ，現在它在機率裡現身了）。我們不硬推這個極限（嚴格證明指向延伸閱讀），但下一節的 worked example 會用具體數字讓你**親眼看到**二項一步步逼近卜瓦松，逼到幾乎重合。

卜瓦松有個漂亮到不像話的性質：**它的期望跟變異數是同一個數，都等於 λ**。

```text
E[X] = λ          Var(X) = λ
```

這你可以從二項極限看出來：二項的 E=np=λ；二項的 Var=np(1−p)，但 p→0 時 (1−p)→1，所以 Var→np=λ。期望跟變異數黏在一起，這是卜瓦松的指紋。實務上這給你一個一眼判斷的工具：如果你的計數型 metric 大致是卜瓦松的，那它的標準差就該是 √λ。平均每分鐘 100 個錯誤，標準差就約 √100=10，所以看到 130 個（偏離 3 個標準差）就值得緊張，看到 110 個（1 個標準差內）是正常呼吸。SRE 的告警閾值，骨子裡常常就是這個 √λ。

### 為什麼這隻動物美：一堆不相干的計數，長同一個形狀

這才是驚嘆點，我要把它講到你能轉述。

卜瓦松的了不起，不在公式漂亮，而在它的**普適性（universality）**。剛剛那串例子——使用者請求、車禍、馬踢死人——它們之間沒有任何物理上的共通點。請求是封包，車禍是金屬撞擊，馬踢是生物學意外。它們唯一的共通結構是抽象的：**大量獨立的機會、每個機會本身極罕見、你只數總量。** 而光憑這個抽象結構，不管底層是什麼鬼東西，計數的分布**就一定長成卜瓦松那個形狀**，由單一個 λ 完全決定。

換句話說，卜瓦松不在乎你數的是什麼。它只在乎「稀有」跟「大量機會」這兩個條件。一旦滿足，細節全被抹掉，只剩 λ 存活。這跟 ch14 的中央極限定理是同一種震撼的離散版——個別事件完全不可測（哪個士兵、哪一天被踢死，沒人能預言），但總體的分布精確可測，而且**這個形狀對所有滿足條件的稀有計數都一樣**。人類竟然找得到一個橫跨封包、車禍、馬蹄的共同數學形狀，這件事本身就值得你坐下來想三分鐘。

歷史上把這件事釘死的，是兩個人。卜瓦松（Siméon Denis Poisson）本人在 **1837 年**的《關於刑事與民事審判機率的研究》裡，第一次寫下這個分布（他當時在算陪審團判決，照 landscape）。但真正讓它名留教科書的，是 60 年後的波特基維茨（Ladislaus Bortkiewicz）——他 **1898 年**寫了一本書叫《小數法則》（Das Gesetz der kleinen Zahlen），裡面分析了一份現在已經成為經典的資料：**普魯士軍隊裡被馬踢死的士兵人數**（照 landscape）。

這份資料美在哪？他收集了好幾個騎兵軍團、橫跨 20 年的紀錄，每個「軍團—年」是一筆觀測，總共 200 筆，記錄那一年那個軍團有幾個人被馬踢死。把這 200 筆按「死亡人數 0、1、2、3、4」分類數一數，跟卜瓦松（λ = 122/200 = 0.61，也就是平均每軍團每年踢死 0.61 人）預測的次數並排——我用 λ=0.61 親手把卜瓦松的預期次數算出來（e^(−0.61)·0.61^k/k! 再乘 200），對照他的觀測：

```text
 死亡數 k   觀測次數   卜瓦松預期（λ=0.61, ×200）
──────────────────────────────────────────────
   0          109          108.7
   1           65           66.3
   2           22           20.2
   3            3            4.1
   4            1            0.6
──────────────────────────────────────────────
   合計        200          約 200
```

你盯著這兩欄看。109 對 108.7。65 對 66.3。22 對 20.2。一個用最抽象的公式（e、階乘、一個叫 λ 的數）算出來的東西，竟然精準命中了真實普魯士士兵被馬踢死的歷史紀錄。馬踢死人這種事，個別來看完全是命運的玩笑——沒人能說哪個倒楣鬼哪一天會被哪匹馬踢中。但把它當成「大量機會、每次罕見」的計數來看，它服從一條鐵律，鐵到你能用一個希臘字母預測一百多年前的死亡分布。這就是「人類竟想得出這個」的那種震撼，而且它不是比喻、是一張你能對帳的真實數字表。

工程橋接：你的「每分鐘 5xx 錯誤數」「每秒到達請求數」用卜瓦松近似，背後是同一回事——大量極短瞬間（機會多）、每瞬間出事機率低（罕見）、你數總量。**Poisson 過程的時間軸推導**（把這個計數放到連續時間上、到達間隔、排隊長度）以及它在排隊系統的運算應用，**那是另一本書的主場**——（見《佇列論》ch05）。本書到此為止：我們只欣賞「為何稀有計數都長這個形狀」這件事本身的美，運算深度交給隔壁書架那本。

### Worked example：親眼看二項收斂到卜瓦松

光說「二項在極限會變卜瓦松」太抽象，我們用基準數字讓你親眼看到它逼近的過程。

設 n = 1000、p = 0.001。這正是「機會多、每次罕見」：一千次試驗、每次成功機率千分之一。所以 λ = np = 1000 × 0.001 = 1。我們來並排算二項 Binomial(1000, 0.001) 跟卜瓦松（λ=1）對 k=0、1、2 的機率。

先算二項那邊。P(X=0)：一千次全失敗，就是 (1−0.001)¹⁰⁰⁰ = 0.999¹⁰⁰⁰。這個數你心算不出來，但可以估：ln(0.999¹⁰⁰⁰) = 1000·ln(0.999) ≈ 1000·(−0.0010005) ≈ −1.0005，所以 ≈ e^(−1.0005) ≈ 0.3677。（這裡偷用了 ln(1−x)≈−x，正是「p 很小」讓近似成立的地方。）

卜瓦松那邊更直接。P(X=0) = e^(−1)·1⁰/0! = e^(−1)。這個常數值得你背起來，全書反覆出現：**e^(−1) ≈ 0.3679**。我交叉驗算一下這個值：e ≈ 2.71828，1/2.71828 = 0.36788，四捨五入 0.3679 ✓。

把幾個並排：

```text
   k     二項 Binomial(1000, 0.001)    卜瓦松（λ=1）      差距
────────────────────────────────────────────────────────────────
   0           0.3677                   e^(−1)  = 0.3679    0.0002
   1           0.3681                   e^(−1)  = 0.3679    0.0002
   2           0.1840                   e^(−1)/2 = 0.1839   0.0001
────────────────────────────────────────────────────────────────
```

兩邊幾乎重合，差距在小數第四位。注意一個漂亮的巧合：λ=1 的卜瓦松，P(0) 跟 P(1) **完全相等**，都是 e^(−1)。從公式看就懂——P(k=1) = e^(−1)·1¹/1! = e^(−1)，跟 P(k=0) = e^(−1)·1⁰/0! = e^(−1) 一模一樣（因為 λ¹/1! = 1¹/1 = 1 = 1⁰/0!）。然後 P(2) = e^(−1)·1²/2! = e^(−1)/2，恰好是一半。

這就是極限的力量：一個有兩個參數（n、p）、要算組合數跟次方的笨重二項分布，在 n 大 p 小的世界裡，塌縮成一個只有單一參數 λ、只用 e 跟階乘的輕巧卜瓦松。你算 Binomial(1000, 0.001) 要面對 C(1000, k) 這種噁心的組合數；換成卜瓦松 λ=1，一支手機計算機就解決了。卜瓦松不只是漂亮，它還是個**能用的近似工具**——稀有事件的計數，別硬算二項，直接套 λ=np 的卜瓦松。

## 直覺的陷阱

機率是人類直覺最常崩壞的領域，離散分布這幾隻動物各自埋了一個讓你掉坑的地方。逐個拆。

| 你以為 | 其實 | 怎麼自我察覺 |
|---|---|---|
| 「連續失敗很多次了，下次該成功了吧」 | 幾何分布**無記憶**：前面失敗 m 次被條件機率約分掉，未來跟從頭開始一模一樣。硬幣不欠你正面。 | 你心裡冒出「該輪到我了」「平衡一下」這種念頭時——那是賭徒謬誤的前身，你正把無記憶的東西想成有記憶。 |
| 「二項分布反正就是 n 次成敗，套公式就好」 | 公式的前提是**獨立且同機率**。失敗常常相關（服務掛了、後面跟著掛），這時二項會嚴重低估「全失敗」的機率。 | 你的「成功數」實際抖動比 np(1−p) 預測的大很多——那是獨立假設破了的徵兆（ch04 相關性故障的回音）。 |
| 「卜瓦松的 λ 是期望，跟變異數沒關係」 | 卜瓦松的**期望和變異數是同一個數 λ**。如果你的計數資料變異數遠大於平均，它就**不是**卜瓦松（這叫過度離散，over-dispersion），硬套會低估尾端風險。 | 算一下你計數 metric 的「樣本變異數 / 樣本平均」。卜瓦松該≈1。若遠大於 1（如突發流量、群聚故障），別用卜瓦松定告警。 |
| 「平均每天 0.61 人被馬踢死？那不可能，人是整數」 | 卜瓦松的 λ 是**長期平均**，不必是可能值（ch08 的回音：期望不是最可能值，也不必是可能值）。某一年踢死 0 人、某一年 2 人，平均出來 0.61。 | 你聽到「平均 1.8 個小孩」「平均 0.61 死亡」覺得荒謬時——你混淆了「平均」跟「某一次的實際值」。 |
| 「二項跟卜瓦松是兩種不同的東西」 | 卜瓦松**就是**二項在 n→∞、p→0、np=λ 的極限。它們是同一條血脈，不是兩隻無關的動物。 | 你把它們當成兩張要分別背的公式表——那代表你還沒看到那條「大量機會、每次罕見」的橋。 |

最值得標出來的是第一個——**賭徒謬誤的種子就埋在這裡**。無記憶性是這整章最反直覺、也最重要的一塊。人類的腦子演化來偵測模式、預期「平衡」，所以「連輸該贏了」感覺天經地義。但獨立事件沒有帳本、沒有記憶體、不存「欠你幾次」這種狀態。ch13 會把這個謬誤連同它的正解（大數法則是「稀釋」不是「修正」）完整攤開，這裡你先把無記憶性的代數骨架（那個被約掉的 (1−p)^m）刻進去。

## 紙上推演

純紙筆，零程式碼。每題先自己動手，再看推演解答。

### 推演題 1 **[10 分鐘]** ★

用幾何分布算「一枚公正硬幣，第一次出現正面平均要擲幾次」，並驗證它等於 2。**這是 ch01 那枚硬幣**。

#### 推演解答

公正硬幣每擲出正面（記為「成功」）的機率 p = 1/2。第一次成功的等待次數 X ~ Geometric(1/2)，期望值公式是 E[X] = 1/p = 1/(1/2) = 2。

但我不要你只套公式，我要你看到 2 是怎麼長出來的。期望值是「值乘以機率加總」：

```text
E[X] = 1·P(X=1) + 2·P(X=2) + 3·P(X=3) + ...
     = 1·(1/2) + 2·(1/4) + 3·(1/8) + 4·(1/16) + ...
```

P(X=k) = (1/2)^(k−1)·(1/2) = (1/2)^k，所以是 1·(1/2) + 2·(1/4) + 3·(1/8) + …。把前幾項加起來感受它往哪收斂：0.5 + 0.5 + 0.375 + 0.25 + 0.15625 + … = 0.5 + 0.5 + 0.375 + 0.25 + … 累加是 0.5、1.0、1.375、1.625、1.781…一路往 2 爬。這個級數 Σ k·(1/2)^k 的和正好是 2（標準的算術—幾何級數，《馴服無限》有；這裡接受結果）。

**交叉驗算（直覺法）**：第一擲就有 1/2 機率成功——一半的時候你一次就中。沒中（另一半）的話，因為無記憶性，你等於「重新開始等」，但已經花掉了一擲。寫成方程式：E = 1 + (1/2)·0 + (1/2)·E（第一擲必花，1/2 機率結束、1/2 機率回到原點重等）。解：E = 1 + E/2 → E/2 = 1 → E = 2 ✓。兩種方法都給 2。

直覺檢核：成功機率 1/2 的事，平均試 2 次中第一次，合理。成功機率 1/10 的事，平均試 10 次——1/p 的意義就是「機率的倒數＝平均等待次數」。

### 推演題 2 **[10 分鐘]** ★★

說明卜瓦松的 λ 同時是期望值也是變異數，並手算驗證一個具體例子（λ=2）的期望。

#### 推演解答

先講為什麼期望和變異數都是 λ。最乾淨的看法是走二項極限這條路：卜瓦松是 Binomial(n, p) 在 n→∞、p→0、np=λ 的極限。

- 二項的期望 = np = λ。極限不改變它，所以卜瓦松 E[X] = λ。
- 二項的變異數 = np(1−p)。當 p→0 時，(1−p)→1，所以 np(1−p) → np = λ。所以卜瓦松 Var(X) = λ。

期望和變異數在極限裡黏成同一個數，這就是卜瓦松的指紋。

手算驗證期望（λ=2）：E[X] = Σ k·P(X=k) = Σ k·e^(−2)·2^k/k!。k=0 那項是 0（被 k 乘掉），從 k=1 起：

```text
E[X] = e^(−2)·[ 1·(2¹/1!) + 2·(2²/2!) + 3·(2³/3!) + 4·(2⁴/4!) + ... ]
     = e^(−2)·[ 1·2 + 2·2 + 3·(8/6) + 4·(16/24) + ... ]
     = e^(−2)·[ 2 + 4 + 4 + 2.667 + ... ]
```

把括號裡的級數加起來會收斂到 2·e²（這是 λ·e^λ 的標準結果），所以 E[X] = e^(−2)·2·e² = 2 = λ ✓。

**自然頻率交叉驗算**：想像每分鐘平均 2 個錯誤，連續觀察一萬分鐘，總共約看到 20000 個錯誤，平均每分鐘 20000/10000 = 2。期望就是長期平均，當然是 λ。

### 推演題 3 **[10 分鐘]** ★★

你的系統每天平均出現 3 次 5xx 錯誤。用卜瓦松寫出「某天剛好 0 次」的機率，並解讀這個數。

#### 推演解答

「每天平均 3 次」就是 λ = 3。每天的錯誤數 X ~ Poisson(3)。「某天剛好 0 次」是 P(X=0)：

```text
           e^(−3)·3⁰     e^(−3)·1
P(X = 0) = ─────────── = ────────── = e^(−3)
              0!            1
```

（記住 0! = 1、3⁰ = 1，所以乾淨地剩下 e^(−3)。）

算 e^(−3) 的值：e³ = e·e·e ≈ 2.71828³ ≈ 20.09，所以 e^(−3) = 1/20.09 ≈ 0.0498，約 **5%**。

**交叉驗算**：e^(−3) = e^(−1)·e^(−1)·e^(−1) ≈ 0.3679³。0.3679² ≈ 0.1354，再乘 0.3679 ≈ 0.0498 ✓。兩條路都給約 0.0498。

**解讀**：平均每天 3 次錯誤的系統，大約每 20 天才會碰到一個「完全乾淨、零錯誤」的日子（1/0.0498 ≈ 20）。這很反直覺——「平均才 3 次、應該常常有零錯誤的好日子吧？」不，零錯誤的日子只有 5%。順帶看尾端：你也可以問「某天爆到 8 次以上」的機率，那會小得多，但**不是零**——偶爾的尖峰是卜瓦松的正常呼吸，不是異常。這正是設告警閾值的關鍵直覺：別把 λ 偶爾的正常上揚當成事故。標準差是 √3 ≈ 1.73，所以 3±1.73 大致是日常範圍，飆到 8（偏離約 3 個標準差）才真的罕見。

### 推演題 4 **[15 分鐘]** ★★★

口頭題（對另一個工程師講）：解釋為什麼「每分鐘錯誤數」「每秒請求數」「被馬踢死的士兵數」這三件毫不相干的事，會服從同一個分布。不准寫公式，只能用「大量機會、每次罕見」這條線講。講完再自我檢核：你有沒有把「為什麼是同一個形狀」講清楚，而不只是說「它們都是卜瓦松」。

#### 推演解答

要點清單（你的口述要命中這幾個，不必逐字）：

1. **先拆出共同的抽象結構**：這三件事表面毫無關係（封包、車禍金屬、馬蹄生物學），但抽掉表面，骨架一樣——有「大量獨立的機會」（一分鐘有無數個極短瞬間、一個軍團有一堆士兵跟馬天天相處），每個機會「本身極罕見」（某一瞬間剛好出錯、某個士兵某天剛好被踢死，機率微乎其微），而你「只數總量」（這一分鐘共幾次、這一年共幾個）。

2. **點出關鍵跳躍**：一旦只剩這個抽象骨架，**底層是什麼就不重要了**。數學只看見「大量機會 × 每次罕見」，看不見封包跟馬蹄的差別。所以它們的計數**必然**長成同一個形狀，而且這個形狀只由一個數 λ（平均發生幾次）決定——細節全被抹掉。

3. **這才是驚嘆點**：不是「它們碰巧都叫卜瓦松」，而是「人類找到了一個橫跨封包、車禍、馬蹄的共同數學形狀」。個別事件完全不可測（沒人能說哪個士兵哪天被踢），但總體分布精確到能對帳一百多年前的真實死亡紀錄。可測的總體從不可測的個體裡浮現——這是 ch14 那個更大震撼的離散預告。

4. **誠實標出邊界**（加分）：條件是「獨立」跟「稀有」。如果事件會群聚（一個錯誤引發一串連鎖故障），獨立就破了，計數會比卜瓦松更分散，這時硬套卜瓦松會低估尾端。知道工具什麼時候失效，比知道它什麼時候有效更重要。

常見錯路：只說「它們都是稀有事件所以都是卜瓦松」——這是把結論當解釋。聽的人會問「那為什麼稀有事件就長同一個形狀」，你得能回到第 2 點的「底層被抹掉、只剩 λ」才算真的講清楚。

5. 卜瓦松是哪個分布在什麼極限下的影子？把 n、p、λ 三者的關係講出來。

## 延伸閱讀

- **《佇列論》ch05（書架鄰書）**：本章只欣賞「卜瓦松為何是稀有計數的普適形狀」，但如果你想看它在**時間軸上展開**（Poisson 過程、到達間隔、排隊長度、Little's law 那一整套運算），那是隔壁那本書的主場。本章的卜瓦松是它的靜態快照，那本是動態影片。讀它的到達過程那幾節。
- **《馴服無限》ch13（書架鄰書）**：卜瓦松公式裡那個 e^(−λ) 的來歷——lim(1+x/n)ⁿ → eˣ——以及為什麼 e 會在「大量機會、每次罕見」的極限裡自然冒出來。想搞懂二項為何收斂到帶 e 的卜瓦松，回去把那個極限擦亮一次。
- **Bortkiewicz《Das Gesetz der kleinen Zahlen》（1898）的馬踢資料**：本章 worked example 那張表的原始出處。網路上「Prussian horse-kick data」是統計教學的經典資料集（如 R 的 `pscl::prussian`），值得自己拉下來、親手對一次觀測與卜瓦松預期——親眼看抽象公式命中真實死亡紀錄，比讀十遍定義都震撼（2026-06，資料集真實存在）。
- **本書 ch12（連續家族）**：幾何分布的無記憶性，在連續世界有個更美的雙胞胎——指數分布（公車不會因為你等久了就比較快來）。本章給你離散原型，ch12 給你連續版的完整欣賞，並警告你長尾世界裡「平均」如何失去意義。
- **本書 ch14（中央極限定理）**：你在 Binomial(10, 1/2) 直方圖裡看到的那個對稱鐘形雛形，在 ch14 會長成完整的常態曲線，而且你會知道為什麼幾乎所有「加總／平均」出來的東西都是鐘形。本章是那場全書最大驚嘆的第一幀。
