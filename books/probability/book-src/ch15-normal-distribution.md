# ch15 — 常態之美與機率即面積

> **本章解決什麼問題**：ch14 給了你一個宇宙級的結論——大量獨立隨機量的「和／平均」總是趨向同一條鐘形曲線，跟原料長相無關。但那一章只回答了「為什麼是鐘形」，沒回答「鐘形到底怎麼用」。這一章把那條曲線從一個哲學奇蹟，變成一把你能拿起來量東西的尺。你會學到常態分布的工作語言：標準化（把任何常態搬到同一把尺上）、z 分數、68-95-99.7 法則、還有那個在每篇論文裡陰魂不散的數字 **1.96**。更重要的是，這一章把一句話釘進你的骨頭裡——**機率就是面積**：連續分布的機率是曲線下的積分，你查 P99 其實就是在反查一條累積曲線。這是 Part IV「秩序從混沌浮現」的收尾（ch13 大數法則、ch14 中央極限、ch15 常態之美），收完它，你就帶著一把校準好的尺，走進 Part V 的統計推論。

## 從你已知的出發

你其實已經會用常態分布了，只是沒人告訴你那叫常態分布。

你看 dashboard 看了六年。某條延遲曲線，平時穩穩貼在某個值附近，偶爾跳一下。你心裡有一條看不見的「正常範圍」——在這條範圍裡的波動你連眉毛都不抬，超出去你才開始緊張。你判斷「這個尖峰算不算異常」用的不是絕對值，是**它離平常有多遠、遠到幾個身位**。這個「幾個身位」就是標準差，而你用「離中心幾個標準差」來判斷異常，就是這一章要給你語言的那套直覺。

更具體一點，三個你天天在做、其實全是這一章的東西：

第一，**P99**。你查一條延遲曲線的 P99，意思是「99% 的請求比這個值快、只有 1% 比它慢」。你有沒有想過你到底在查什麼？你在反查一條**累積曲線**：給定一個比例（99%），找出對應的門檻值。這條曲線在數學上叫 CDF（累積分布函數），而「機率＝累積曲線上的一段」這件事，正是本章的主梁。等下你會看到，P99 跟「標準常態的 99 百分位點」是同一個動作，只是換了把尺。

第二，**3-sigma 告警**。SRE 圈子裡有個土法煉鋼的告警規則：算出指標的平均和標準差，超過平均 ± 3 個標準差就告警。為什麼是 3？因為在常態假設下，落在 ±3σ 之外的機率只有 0.27%——大約千分之三，罕見到值得你半夜被叫起來。這個「3σ ≈ 千分之三」不是某人拍腦袋定的，是這一章 68-95-99.7 法則的最後一格。（先劇透一個陷阱：你的延遲指標往往**不是**常態的，是長尾的，所以這條規則會在尾巴上騙你——ch12 已經警告過，這一章我們把它講透。）

第三，**標準化（standardize）**。你比較兩個量綱完全不同的指標——比方說「這台機器的 CPU 比平常高了多少」跟「這台機器的延遲比平常高了多少」——你沒辦法直接比 70%（CPU）跟 200ms（延遲），單位都不一樣。但你可以各自問「它離自己的平常有幾個標準差」，這樣兩個數字就站到同一把尺上了。這個動作叫標準化，算出來的東西叫 **z 分數**，是這一章第一個、也是最有用的工具。

所以這一章不是教你新東西，是幫你已經會的東西裝上把手、刻上刻度。我們從那把尺開始。

## 標準化：把所有常態搬到同一把尺

常態分布有兩個參數：平均 μ（鐘的中心在哪）和標準差 σ（鐘有多胖）。寫成 X ~ N(μ, σ²)。麻煩在於，每個常態都長得不一樣——延遲是 N(50, 5²)、身高是 N(170, 7²)、考試分數是 N(60, 12²)，胖瘦位置全不同。如果每遇到一個常態就得重新算一遍它的面積，那這把尺一點都不通用。

高斯（Gauss）和後人留給我們最划算的一招，是發現**所有常態其實是同一條曲線**，只是被平移和縮放過。你只要會讀一條曲線——**標準常態 N(0, 1)**，中心在 0、標準差是 1——就能讀所有常態。怎麼搬過去？一個減法加一個除法：

```text
z = (x − μ) / σ      ← 把任何常態搬到 N(0,1) 的標準化公式
```

拆開看，這兩步你其實天天在做：

```text
x − μ        ← 先平移：「離中心多遠」，把鐘的中心對齊到 0
(x − μ) / σ  ← 再縮放：「以一個標準差為單位，遠了幾個身位」
```

減 μ 是「以平均為原點重新計算位置」；除 σ 是「換單位，把標準差當成新的『1』」。算完得到的 z，意思非常乾淨：**這個值在自己的分布裡，離平均有幾個標準差**。z = 2 就是「高出平均兩個標準差」，z = −1.5 就是「低於平均一個半標準差」。單位被洗掉了，剩下純粹的「幾個身位」。

這就是你比較 CPU 跟延遲時做的事。CPU 平常 50% ± 10%，現在 70%，那 z =(70−50)/10= 2.0。延遲平常 50ms ± 5ms，現在 60ms，那 z =(60−50)/5= 2.0。兩個都是「z = 2.0」——你現在可以說「這兩個指標異常的程度一樣」，儘管一個是百分比、一個是毫秒。**標準化就是把不同量綱的東西放到同一把尺上比較**，這是它了不起的地方，也是統計學裡到處都在用的動作（z-score、t-statistic、效果量，骨子裡都是這個減法加除法）。

到這裡你可能會問：搬過去之後呢？我手上有個 z = 2.0，然後咧？然後你就能查出「比它更極端的機率有多少」——而那個查的動作，是這一章真正的高潮。

## 機率即面積：你查 P99 就是在反查累積曲線

先把一句話釘死，這句話貫穿 Part III 到 Part V：

> **連續分布的機率，是密度曲線下的面積。**

ch12 已經鋪過這件事（密度不是機率、單點機率為 0、機率要靠積分把一段面積加起來）。這一章把它用到底。標準常態的密度函數長這樣：

```text
       φ(z) = (1/√(2π)) · e^(−z²/2)
```

別被嚇到，這條公式現在你只要會「讀」它的形狀，不用會推它（推導為什麼是這個樣子，等下單獨講一段，而且只給直覺）。重點是：**這條曲線下的面積，就是機率**。你想知道「z 落在某區間的機率」，就是去量那一段曲線下面的面積：

```text
P(a ≤ Z ≤ b) = ∫[a,b] φ(z) dz      ← a 到 b 之間、曲線下的面積
```

整條曲線下面的總面積是 1（機率守恆，跟 ch12 的均勻分布矩形面積=1 是同一個道理，只是這次曲線是彎的）。而那個 1/√(2π) 的係數，就是為了把面積調成剛好 1 而存在的——它是「歸一化常數」，把一條原本面積是 √(2π) 的曲線壓成面積 1。（為什麼是 √(2π) 而不是別的數，牽涉一個叫高斯積分的漂亮結果 ∫e^(−x²)dx = √π，π 怎麼會跑進來這裡我留一段給你看驚嘆，但**完整微積分推導指向《馴服無限》ch15「機率即面積」**，這本書不重講積分本身。）

現在把它接回你的工作。你查 P99 延遲，在做什麼？你在問：「面積累積到 99% 的那個門檻值在哪？」這就是反查**累積分布函數（CDF）**：

```text
F(x) = P(X ≤ x) = ∫[−∞, x] f(t) dt    ← 從負無窮積到 x，左邊累積的面積
```

CDF 是一條從 0 爬到 1 的累積曲線。**P99 就是反查這條曲線：給定 0.99，找出對應的 x。** 你每次在 Grafana 點開 p99，背後就是這個動作（雖然你的延遲不是常態的，是用實測樣本的經驗分布算的，但「反查累積曲線」這個邏輯一模一樣）。

我認為這是本章第一個值得你停下來的點。**機率這個抽象到不行的東西，被翻譯成了一件你能用尺量的事——面積。** 「某件事多可能發生」變成「某塊區域有多大」。一旦機率變成面積，所有微積分的工具（積分、求面積、反查累積值）瞬間全部可用。這不是比喻，是字面上的相等。人類花了一百多年才把「機運」這個玄學詞，磨成一塊可以拿圓規和積分去量的幾何圖形。

問題是：常態的這個積分**算不出封閉解**。∫e^(−z²/2)dz 沒有初等函數形式的答案（這是真的，不是我們懶得算）。所以歷史上大家怎麼辦?——查表。把標準常態的 CDF 預先算好，印成一張表，全世界共用。你現在用程式一行 `normalCDF(z)` 解決，但在沒有電腦的年代，那張 z 表是統計學家的聖經。下一節我們就來讀這張表最重要的三格。

## 68-95-99.7 法則：三格刻度，背起來終身受用

標準常態這張表，你不用背整張，只要背三格。這三格就是 SRE 圈子裡 sigma 告警的數學底，也是你判斷「幾個標準差算異常」的內建刻度：

```text
範圍        曲線下面積（落在內）   精確值       尾巴（落在外）
±1σ         約 68%                68.27%       約 32%（兩尾各 16%）
±2σ         約 95%                95.45%       約 4.6%（兩尾各 2.3%）
±3σ         約 99.7%             99.73%       約 0.3%（兩尾各 0.13%）
```

這三個數字（**68.27% / 95.45% / 99.73%**，照 landscape 的精確值）的意思是：一個常態隨機變數，有 68.27% 的機率落在「平均 ± 1 個標準差」裡，95.45% 落在 ±2σ 裡，99.73% 落在 ±3σ 裡。畫成圖：

```text
  φ(z) 標準常態密度（█ 的高度 = 機率密度，越高越可能）

                  ███
                ███████              ← 中間最厚（最可能）
              ███████████
            ███████████████
          ███████████████████
       ░░██████████████████████░░    ← 尾巴薄但永遠不碰到 0
   ░░░░░░░██████████████████████░░░░░░░
  ───┴────┴────┴────┴────┴────┴────┴───► z
    −3σ  −2σ  −1σ   0   +1σ  +2σ  +3σ

  落在 ±1σ 內：68.27%   ←─ 中間這段
  落在 ±2σ 內：95.45%   ←─ 再外推一圈
  落在 ±3σ 內：99.73%   ←─ 幾乎全部，尾巴只剩 0.27%
```

讀法很白話：**離中心越遠，那一段面積（機率）掉得越快**。從 ±1σ 到 ±2σ，多框進來的面積讓「落在外面」的機率從 32% 掉到 4.6%；再到 ±3σ，掉到 0.3%。這就是為什麼 3σ 是個好告警線——千分之三，夠罕見，但又不是罕見到永遠不觸發。如果你把告警線拉到 ±4σ（約十萬分之六），那你大概一年也收不到一次告警，異常都從你眼皮底下溜過去了。sigma 告警的本質，是你在「漏報」和「狼來了」之間，用這張表的刻度選一個平衡點。

這裡埋一個全書最常被寫錯的陷阱，先講清楚（landscape 把它列為作者最容易出錯的點之一）：

> **±2σ 不是 95%，是 95.45%。「恰好 95%」對應的是 ±1.96σ，不是 ±2σ。**

很多人把 95% 跟 2 個標準差畫等號，這是個方便的近似，但不精確。±2σ 框進來的是 95.45%，差了快半個百分點。那「恰好 95%」要框到哪？答案是 **±1.96σ**。這就帶出了那個你在每篇論文、每個信賴區間裡都會撞見的數字。

## 1.96 的來歷：那個陰魂不散的數字

打開任何一篇做統計的論文，「±1.96」這個數字遲早會跳出來。它不是某個物理常數，也不是誰的生日——它純粹是這張 z 表上一個特定刻度的座標。它的來歷乾淨到你會笑：

我們想要一個區間，讓常態隨機變數**有 95% 的機率落在裡面**。對稱地框——中間留 95%、兩邊各留 2.5%（5% 的尾巴平分到兩側）。那這個區間的邊界在哪？就是去問 z 表：「累積到 97.5%（也就是 95% + 左邊那條尾巴的 2.5%）的那個 z 是多少?」答案是 **1.959964…**，四捨五入成 **1.96**。

```text
        ┌──────── 95% ────────┐
        │                     │
   2.5% │                     │ 2.5%      ← 兩尾各留 2.5%
 ───────┼─────────────────────┼───────► z
     −1.96         0        +1.96
        └─ 95% 落在 ±1.96σ 之內 ─┘
```

所以 1.96 是「95% 雙尾」的臨界值。為什麼大家用 1.96 而不是更整齊的 2.0？因為 2.0 對應的是 95.45%，**多框進來了 0.45%**——如果你聲稱「95% 信賴」卻用 2.0，你其實偷偷給了自己 95.45% 的把握，名實不符。1.96 才是誠實的 95%。

順帶一個史實彩蛋（已查證，2026-06）：費雪（Fisher）在他 1925 年那本影響整個 20 世紀的《Statistical Methods for Research Workers》裡，表格給的就是更精確的 **1.959964**。後來大家圖方便取 1.96，少數人甚至圖更方便取 2。但這個「2」的習慣有點危險——它正是「±2σ ≈ 95%」這個誤解的源頭。下次你看到有人用 2.0 當 95% 臨界值，你心裡可以默默幫他補一句「其實是 1.96 啦」。

到這你已經有了讀常態的全套工具：標準化把任何常態搬到 N(0,1)、機率即面積讓你能查機率、68-95-99.7 給你三格速記刻度、1.96 給你 95% 的精確邊界。剩下兩個「美」的問題，我想帶你看一眼但不折磨你：這條曲線**為什麼長這樣**，以及它有一個近乎傲慢的**自我複製**性質。

## 鐘形為什麼長這樣：一瞥，不硬推

高斯函數的核心是 e^(−z²/2)。我們不推導它（推導要動到微積分和最大概似，指向延伸閱讀），但光是**讀**這個式子的結構，就能看懂鐘形的三個特徵，而且看懂之後你會覺得這個形狀幾乎是被逼出來的：

**第一，為什麼中間最高、左右對稱。** 指數上是 −z²/2，注意那個平方。z 和 −z 平方後一樣（2² = (−2)² = 4），所以曲線在 0 的左右是鏡像——**對稱來自平方**。而 z = 0 時，指數是 0，e⁰ = 1，這是函數的最大值；z 一往兩邊跑，−z²/2 越來越負，e 的負次方越來越小。所以中間鼓、兩邊塌，是平方加上負號的必然結果。

**第二，尾巴趨零有多快——快到驚人。** 這是常態最關鍵、也最容易被忽略的性質。尾巴衰減的是 e^(−z²/2)，注意指數裡是 **z 的平方**，不是 z 本身。這意味著尾巴掉得是**平方級的指數快**，遠比一般的指數衰減還凶：

```text
z = 1   →  e^(−0.5)   ≈ 0.607
z = 2   →  e^(−2)     ≈ 0.135      ← 才走兩步，掉到剩兩成
z = 3   →  e^(−4.5)   ≈ 0.011      ← 三步，剩百分之一
z = 4   →  e^(−8)     ≈ 0.00034    ← 四步，剩萬分之三
z = 5   →  e^(−12.5)  ≈ 3.7e−6     ← 五步，百萬分之四
```

每往外走一步，因為是平方，下降的力道越來越猛。這就是為什麼常態的尾巴「實質上」很快就沒了——5 個標準差之外的事，機率小到工程上可以當不存在。**這個「尾巴塌得飛快」正是常態跟長尾分布最關鍵的分界線。** ch12 警告過你：你的延遲是長尾的，它的尾巴衰減慢得多（冪律是多項式級的慢，不是平方指數級的快）。所以在常態世界，「5σ 之外＝幾乎不可能」是對的；但在長尾世界，你硬套這條規則，會被那條胖尾巴上一個本來就該來的極端值打到鼻青臉腫。**金融海嘯裡那些「百萬年一遇」卻三年來一次的事件，根因就是有人拿常態的瘦尾巴去描述一個胖尾巴的世界。** 這是 3-sigma 告警在重尾指標上會失靈的數學原因，記住它。

**第三，π 為什麼會在這裡。** 這是我覺得整條公式最魔幻的地方。φ(z) 前面那個 1/√(2π)——圓周率 π，那個量圓的東西——怎麼會跑進一個跟圓八竿子打不著的機率公式裡？答案藏在一個漂亮到值得單獨講的結果：高斯積分。要把這條鐘形曲線的總面積調成 1，你得先算出它原本的面積，而那個積分

```text
∫[−∞,∞] e^(−x²) dx = √π        ← π 從這裡冒出來
```

竟然等於 √π。π 在這裡出現的原因，是算這個積分最漂亮的方法要**把它平方、變成一個二維積分、然後換成極座標**——一換極座標，圓就進來了，π 就跟著進來了。**一個一維的、毫無圓味的積分，要繞到二維的圓上才算得出來，於是 π 留在了答案裡。** 我第一次看到這個推導的時候盯著它看了很久。一個關於「測量誤差怎麼分布」的函數，骨子裡藏著圓周率——這是那種會讓你覺得「數學底層是不是真的有某種統一性」的時刻。**完整推導（平方、二維積分、極座標換元）指向《馴服無限》**，本書點到這個驚嘆為止，不重講積分。

## 自我複製：常態相加還是常態

常態還有一個近乎傲慢的性質，一句話講完，但這句話分量很重：

> **兩個獨立的常態相加，結果還是常態。** 平均相加，變異數相加。

如果 X ~ N(μ₁, σ₁²)、Y ~ N(μ₂, σ₂²) 且兩者獨立，那麼 X + Y ~ N(μ₁+μ₂, σ₁²+σ₂²)。形狀不變，只是中心和胖瘦按規則合併（這叫常態的**穩定性**，stability property，已查證 2026-06）。注意「變異數相加」這條，正是 ch09 那個「獨立和的變異數相加、但標準差不相加」的老朋友又回來了——常態只是把它演到了極致：不只變異數聽話相加，連分布的**形狀**都原封不動。

這個性質為什麼了不起？因為它跟 ch14 的中央極限定理是一體兩面的。CLT 說「很多東西加起來會**趨向**常態」；而常態自己說「我加我自己，**直接就是**常態，連趨向都不用」。常態是這個「加法宇宙」裡的不動點——所有東西加總後都往它流，而它自己加自己原地不動。在所有分布裡，能做到「相加後形狀不變」的家族極少（它叫穩定分布族），常態是其中最乖、最常見的那個。你工程上把很多獨立的常態雜訊疊加（多個量測誤差累積），結果還是個漂亮的常態，不會變成什麼奇形怪狀——這個「疊加後仍可控」的性質，是常態之所以好用的隱形功臣。

好，工具齊了、美也看過了。現在回收脊椎硬幣，把這一章變成你手上能算的東西。

## 脊椎硬幣回收：用常態近似查「100 擲落在某區間」

**這是 ch01 那枚公正硬幣。** 我們一路升級它：ch02 它是古典 1/2、ch08–09 它有 E=1/2、Var=1/4、ch11 n 擲變成 Binomial(n, 1/2)、ch14 它標準化後趨向常態（而且這正是史上第一個中央極限定理，棣美弗 de Moivre 1733 年算的就是它）。現在 ch15，我們用常態這把尺去量它。

擲 100 次公正硬幣，正面數 X 服從 Binomial(100, 1/2)。由 ch08–09 的基準：

```text
μ = E[X]   = n·p     = 100 × 0.5  = 50      ← 平均出 50 個正面
σ² = Var(X) = n·p(1−p) = 100 × 0.25 = 25      ← 變異數 25
σ  = √25 = 5                                  ← 標準差 5 個正面
```

所以 X 近似 N(50, 5²) ——一個中心在 50、標準差 5 的鐘形。現在問一個你算離散二項要加總一堆組合數、但用常態近似只要查一格表的問題：**100 擲裡，正面數落在 45 到 55 之間的機率是多少？**

45 到 55 是「50 ± 5」，也就是「平均 ± 1 個標準差」。標準化兩個邊界：

```text
下界 z = (45 − 50) / 5 = −1.0
上界 z = (55 − 50) / 5 = +1.0
```

問題變成「標準常態落在 −1 到 +1 之間的機率」——這正是 68-95-99.7 法則的第一格：

```text
P(45 ≤ X ≤ 55) ≈ P(−1 ≤ Z ≤ 1) ≈ 68.27%
```

不用加總十一個組合數，直接讀表第一格，**約 68%**。這就是常態近似的省力之處：把一個離散的、要硬算的二項問題，搬到連續的常態尺上，查一格表就完事。棣美弗 1733 年發明這招的時候沒有電腦，硬算 Binomial(100, 1/2) 的中間段是會要人命的工程，常態近似是他的救命稻草。

## Worked example：標準化、查尾端，與一個埋給 ch18 的伏筆

帶兩個完整的手算，一個是純工程指標、一個是脊椎硬幣，最後埋一個伏筆給三章後的假設檢定。

### 例一：延遲超過 60ms 的比例

你的服務延遲近似常態，平均 50ms、標準差 5ms，寫成 N(50, 5²)。問：**超過 60ms 的請求佔多少比例？**（也就是 P(X > 60)。）

**第一步，標準化。** 把 60ms 搬到標準常態尺上：

```text
z = (x − μ) / σ = (60 − 50) / 5 = 2.0
```

60ms 在「平均 + 2 個標準差」的位置。

**第二步，查尾端面積。** 我們要的是「比 z = 2.0 更極端（更大）」那條尾巴的面積，也就是上尾機率 P(Z > 2.0)。查 z 表（或用 68-95-99.7 反推）：

```text
P(Z > 2.0) ≈ 0.0228   ≈ 2.3%      ← 標準常態 z=2 的上尾面積
```

所以**大約 2.3% 的請求會超過 60ms**。

**交叉驗算（用 68-95-99.7 反推）。** 我不想只是抄表上的 0.0228，用法則自己估一遍：±2σ 之內是 95.45%，所以**外面**是 4.55%，平均分到兩條尾巴，**一條尾巴是 4.55% / 2 ≈ 2.27%**。跟查表的 0.0228（2.28%）對得上（微小差異是 95.45% 的四捨五入造成）。✓ 兩種方法同一個答案，這個 2.3% 站得住。

工程上的意義：如果你的 SLA 是「P95 ≤ 60ms」（95% 的請求要在 60ms 內），那這個服務剛好踩線——只有 2.3% 超過，95% 線在 60ms 內，過關。但注意這整套推論吃了一個假設：**延遲是常態的**。真實延遲常是長尾的，右尾比常態胖得多，那條 2.3% 會被嚴重低估。**用常態算延遲尾巴是工程上的常見陷阱，下一節專門拆它。**

### 例二：脊椎硬幣，100 擲「≥ 60 正面」的機率

**還是 ch01 那枚硬幣。** X ~ Binomial(100, 1/2)，已知近似 N(50, 5²)。問：**擲 100 次，出現 60 個或更多正面的機率？**（P(X ≥ 60)。）

**標準化 60：**

```text
z = (60 − 50) / 5 = 2.0
```

又是 z = 2.0（這不是巧合——這個硬幣的標準差剛好是 5，跟例一的延遲一樣，所以「離平均 10」在兩個問題裡都是「2 個標準差」）。查上尾：

```text
P(X ≥ 60) ≈ P(Z ≥ 2.0) ≈ 0.0228 ≈ 2.3%
```

常態近似說：**一枚公正硬幣擲 100 次，出現 60 個以上正面，機率大約 2.3%。** 罕見，但遠不到不可能——大約每 44 次「100 擲實驗」就會碰到一次。

**埋伏筆（給 ch18）。** 這裡有一個誠實的細節我必須講，因為它正是三章後假設檢定的引子。常態近似給的 2.3% 是**近似值**，因為我們拿一條連續的鐘形去蓋一個離散的二項（正面數只能是整數，沒有 59.7 個正面這種東西）。如果你不嫌麻煩，硬算精確的二項機率：

```text
精確 P(X ≥ 60) = Σ(k=60→100) C(100,k)·(1/2)¹⁰⁰ ≈ 0.0284   ← 約 2.84%
```

精確值 2.84%，常態近似 2.28%，差了半個百分點——近似低估了一點。（如果你做「連續性校正」，把邊界從 60 挪到 59.5 再標準化，z = 1.9，上尾約 2.87%，幾乎完美命中精確的 2.84%——這是個讓近似更準的小技巧，細節留延伸閱讀。）

為什麼這半個百分點以後會要命？因為 ch18 要拿這個硬幣問一個尖銳的問題：「**觀察到 60/100 正面，這枚硬幣還算公正嗎？**」到時候會算「雙尾」機率——偏離 50 達到 10 以上（≥60 或 ≤40）的機率。常態近似給雙尾 ≈ 0.046，精確二項給 ≈ 0.057（這兩個數字現在先記著，basis 表裡有）。**一個在 0.05 門檻下面、一個在上面**——同一份證據，換個算法就跨過了「顯著」這條線。那一章你會親眼看到「p < 0.05」這條神聖門檻有多武斷。這一章的 2.84% vs 2.28%，就是那場戲的第一個音符。**記住這個半百分點的縫隙，ch18 見。**

## 直覺的陷阱

常態分布太好用、太漂亮，以至於它最大的危險就是**你會把它套到不該套的地方**。這一節的陷阱比別章更工程，因為你天天踩。

| 陷阱 | 錯誤直覺 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| **「常態無所不在，所以我的資料也是常態」** | CLT 說鐘形普適，那我手上這堆數應該也是鐘形吧 | 你把延遲、收入、檔案大小硬當常態，用 3σ 框「正常範圍」——但這些是**長尾**的，右尾比常態胖太多，你會把一堆本該出現的大值誤判成「異常」，告警炸成雪花 | 看分布圖。如果右邊拖一條長尾、平均被少數巨值拉得遠離中位數，它就不是常態。常態的平均≈中位數，且尾巴塌得飛快 |
| **「±2σ 就是 95%」** | 兩個標準差嘛，95%，整數好記 | 你聲稱「95% 信賴」卻用了 2.0，實際給了自己 95.45%；報出去的不確定性比你以為的鬆 | 凡是講「恰好 95%」就該用 **1.96**，不是 2。看到誰用 2.0 當 95% 臨界值，心裡警鈴要響 |
| **「機率 0 就是不可能 / 密度高就是機率高」** | 密度那麼高，那一點的機率一定大 | 把密度當機率讀。連續分布**單點機率永遠是 0**，密度只是「機率的濃度」，機率永遠是一段**面積**不是一個高度。密度甚至可以大於 1 | 提醒自己：問的永遠是區間（落在 [a,b]）不是單點（等於某值）。要機率就去積一段面積 |
| **「3σ 之外＝一定是異常」** | 千分之三耶，這麼罕見，肯定出事了 | 在長尾指標上，3σ 之外**本來就會頻繁出現**（胖尾巴的常態），你把正常的尾端波動當成事故，疲於奔命；反過來在常態指標上又可能漏掉真正該關心的 4σ、5σ | 先確認指標的分布長相。常態指標上 3σ 才是好告警線；長尾指標要改用百分位數（P99、P99.9）而不是 sigma |

最該記住的一句，也是這一章和 ch12、ch14 共同的告誡：

> **常態的尾巴塌得是平方指數級的快，這是它的祝福也是它的詛咒。** 祝福是「離平均遠＝幾乎不可能」這條乾淨規則；詛咒是你一旦把它套到尾巴更胖的真實世界（延遲、收入、市場、流量尖峰），就會系統性地、災難性地**低估極端事件**。常態不是萬物的形狀，它是「很多獨立小擾動相加」的形狀。你的資料符不符合那個前提，是你套常態之前該先問自己的第一個問題。

## 紙上推演

純紙筆，不准開計算機之外的東西。每題先自己算，再對解答。

### 推演題一：±2σ 到底是不是 95% **[10 分鐘]** ★

用 68-95-99.7 法則估「落在 ±2σ 之外」的比例，然後回答：為什麼大家做信賴區間用的是 1.96 而不是 2？把這兩個數字對應的「框進來的面積」各寫出來，說清楚差別。

#### 推演解答

±2σ 之內是 **95.45%**，所以**之外**是 100% − 95.45% = **4.55%**（兩條尾巴各 2.27%）。

而 95% 信賴區間要的是「恰好框進 95%、之外恰好 5%（每尾 2.5%）」。±2σ 框進來的是 95.45%，**多框了 0.45%**——尾巴只剩 4.55% 而不是 5%。要讓尾巴恰好是 5%（每尾 2.5%、累積到 97.5%），對應的臨界值是 **1.96**，不是 2.0。

所以：
- **1.96σ** → 框進 95.00%（誠實的 95%）
- **2.00σ** → 框進 95.45%（多給了自己 0.45% 的把握）

用 2.0 當 95% 臨界值，等於偷偷把信賴水準調高到 95.45% 卻仍宣稱 95%——名實不符。**1.96 才是「恰好 95%」的那把尺。** 常見錯路：以為「反正差不多」。在工程上 0.45% 可能無所謂，但在「這個藥有沒有效」「這個改動是不是真的提升」這種要下結論的場合，門檻的精確性就是誠實的底線。

### 推演題二：把一個工程指標標準化，判斷它多異常 **[15 分鐘]** ★★

你的 API 平常每分鐘錯誤數近似常態，平均 8 個、標準差 2 個。某分鐘你看到 **14 個**錯誤。

(a) 算它的 z 分數。(b) 用 68-95-99.7 估「正常情況下出現這麼多或更多錯誤」的機率。(c) 如果你的告警線設在 3σ，這次會不會觸發？(d) 一句話說明：如果錯誤數其實是長尾的（不是常態），你上面的判斷會怎麼偏。

#### 推演解答

**(a) 標準化：**

```text
z = (14 − 8) / 2 = 6 / 2 = 3.0
```

14 個錯誤在「平均 + 3 個標準差」的位置。

**(b) 上尾機率。** ±3σ 之內是 99.73%，之外是 0.27%，平分到兩尾，**上尾 ≈ 0.135%**（約千分之一點三五）。所以在常態假設下，「正常波動跑出 14 個以上錯誤」大約千分之一的機率——相當罕見。

**(c) 會觸發。** z = 3.0 剛好踩在 3σ 告警線上（≥ 3σ），觸發。這正是 3σ 告警的設計意圖：千分之三量級的罕見事件值得看一眼。

**(d) 長尾的偏。** 如果錯誤數其實是長尾的（比方說錯誤常常一陣一陣地爆，相關性故障讓它們不獨立、尾巴變胖），那「正常情況下跑到 14」的真實機率會**比 0.135% 高得多**——常態的瘦尾巴低估了胖尾巴世界的極端值。結果是：你以為抓到了一個千分之一的真異常，其實它在長尾世界裡沒那麼稀奇，可能只是尾端的正常抽動。**3σ 在長尾指標上會製造假警報**，這時候該換成看 P99.9 之類的百分位數門檻，而不是 sigma。

### 推演題三：口頭重講「機率即面積」 **[15 分鐘]** ★★★

闔上書，對著想像中的另一個工程師（或對空氣），用你自己的話講清楚三件事，每件 1–2 分鐘：(1) 為什麼連續分布裡「單點機率是 0」但事件還是會發生；(2) 「你查 P99」跟「反查 CDF」是同一件事——怎麼同法；(3) 標準化（z 分數）在做什麼、為什麼它能讓你比較兩個量綱不同的指標。講完對照下面的要點。

#### 推演解答

**(1) 單點機率為 0。** 連續軸上有無窮多個點，要是每個點都給正機率，加起來會爆破 1，所以唯一自洽的安排是每個單點機率 0。但機率 0 不等於不可能——機率永遠是一段**面積**（區間），不是一個點的高度。類比：浮點數你從不寫 `x == 0.3`，你寫 `abs(x − 0.3) < eps`，因為你關心的是「落在一個小區間」不是「剛好等於」。（要點：無窮多點 → 單點必為 0 → 機率是面積不是高度 → 浮點類比。）

**(2) P99 = 反查 CDF。** CDF 是一條從 0 爬到 1 的累積曲線，F(x) = 「比 x 小的機率」。查 P99 就是反過來用它：給定累積到 0.99，找出對應的門檻值 x。「正著用」是給 x 問機率，「反著用」是給機率問 x——P99 是後者。你在 Grafana 點 p99，背後就是這個反查。（要點：CDF 是累積面積曲線 → P99 是給比例反查門檻 → 跟 normalCDF 反函數同一個動作。）

**(3) 標準化。** z =(x − μ)/σ，先減平均（對齊中心到 0）、再除標準差（把「一個標準差」當成新單位）。算出來的 z 是「離平均幾個標準差」，量綱被洗掉了。所以 CPU 的 z=2 和延遲的 z=2 可以直接比——它們各自離自己的平常都是 2 個身位，異常程度相同，儘管一個是 %、一個是 ms。（要點：減 μ 對齊、除 σ 換單位 → z 是純粹的「幾個身位」→ 洗掉量綱所以能跨指標比較。）

如果這三段你都能不看書、不打結地講出來，這一章就過了。講不清楚的那一段，就是你還沒真懂的地方，回去重讀對應小節。

## 延伸閱讀

- **《馴服無限》ch15「機率即面積」**（你的書架）——本章一路把「機率＝曲線下面積」當工具用，卻沒推導積分，也沒講高斯積分 ∫e^(−x²)dx=√π 為什麼會冒出 π。那一章把這件事用微積分講透（平方、二維積分、極座標換元——π 從圓那裡進來）。如果你想看「π 怎麼跑進機率」的完整魔術，去讀它。本章和它是同一個面積，兩種講法。
