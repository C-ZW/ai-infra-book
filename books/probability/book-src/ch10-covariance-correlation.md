# ch10 — 共變異與相關：兩個變數的共舞，相關 ≠ 因果

> **本章解決什麼問題**：前三章我們把單一隨機變數榨乾了——期望（ch08）給它一個重心、變異數（ch09）給它一個寬度。但你的系統裡從來沒有「一個」指標孤零零地動：CPU 升的時候延遲也升、佇列一長錯誤率就漲。這一章處理**兩個隨機變數怎麼一起動**：共變異數（covariance）量它們同向還是反向、相關係數（correlation）把這個量壓進 −1 到 1 之間變成可比較的數。然後是本章的重頭戲、也是全書最被當口號濫用的一句話——**相關不是因果**。我們要把它從口號變成能解剖的東西：相關背後有三條岔路（巧合、共同原因、反向因果），而且 `ρ=0` 根本不代表兩件事無關。它接在 ch09 後面補完一條公式（`Var(X+Y)` 那個被你忽略的 `2Cov` 項），也為 ch16 隨機化、ch19 辛普森悖論與回歸均值埋下引信。

## 從你已知的出發

你早就在算相關，只是沒寫下那個希臘字母。

打開任何一個監控面板。你盯著兩條曲線：一條是請求速率，一條是 P99 延遲。流量一上來，延遲跟著爬；流量一退，延遲也鬆。你心裡那句「這兩個是連動的」——那就是相關。你甚至不用算，眼睛掃一眼兩條線的形狀，大腦就替你估了一個相關係數。

工程師對「相關」的直覺其實很強。你會說「錯誤率跟部署頻率有關」「快取命中率掉，DB 負載就漲」。這些全是在描述兩個隨機變數一起動的方向與強度。這一章前半就是把這個眼睛的直覺，變成一個能算、能比較、能寫進報告的數字。

但這一章真正的價值在後半，而且是那種你被它咬過的價值。

你一定遇過這種會議：有人指著兩條同漲的曲線說「你看，A 造成了 B」。然後你花了三天去改 A，B 一動也不動——因為 A 跟 B 根本不是因果，它們只是**同時被第三個東西（流量）推著走**。CPU 高和延遲高一起出現，不是 CPU 害的，是流量同時把兩個都頂上去了；你去降 CPU，延遲照樣爛。這就是 confounder（共同原因）。還有更氣人的：dashboard 上兩條曲線剛好都在禮拜五尖一下，有人就腦補出一個故事——但它們可能純粹是巧合，你盯著夠多的圖，總有兩張會剛好同步，這叫虛假相關（spurious correlation）。

這一章要給你的，是一套把「這兩個東西有關係」這句模糊的話拆開的工具：先量出相關有多強（ρ），再追問這個相關到底是怎麼來的（因果？共同原因？反向？還是純粹巧合？）。我認為後面這個追問，是一個資深工程師跟一個只會看圖喊「相關」的人之間，最關鍵的差距。

## 共變異數：兩個變數同向還是反向

先把問題擺正。我們有兩個隨機變數 X 和 Y，各自有自己的期望 `μ_X`、`μ_Y`。我想知道：當 X 比它的平均高的時候，Y 傾向比它的平均高，還是低？

ch09 量單一變數的離散，用的是「跟自己平均的偏差平方」`E[(X−μ_X)²]`。現在有兩個變數，自然的推廣是：把 X 的偏差乘上 Y 的偏差，再取期望。

```text
Cov(X,Y) = E[(X − μ_X)(Y − μ_Y)]
           ────────────────────────
唸法：「X 的偏差，乘上 Y 的偏差，的平均」
```

這個定義為什麼抓得到「同向反向」？看那個乘積 `(X−μ_X)(Y−μ_Y)` 的正負號：

```text
情況                          X−μ_X   Y−μ_Y   乘積
X 高於平均，Y 也高於平均       +       +       +    ← 同向
X 低於平均，Y 也低於平均       −       −       +    ← 同向（負負得正）
X 高於平均，Y 卻低於平均       +       −       −    ← 反向
X 低於平均，Y 卻高於平均       −       +       −    ← 反向
```

關鍵在那個負負得正。**兩個變數「同方向偏離各自平均」時，乘積永遠是正的**——不管是一起偏高還是一起偏低。反方向偏離時，乘積是負的。把所有這些乘積按機率加權平均（取期望），就得到 Cov：

- Cov > 0：X 和 Y 傾向**同向**動（一個高另一個也高）。正相關。
- Cov < 0：X 和 Y 傾向**反向**動（一個高另一個低）。負相關。
- Cov ≈ 0：沒有一致的線性同向／反向傾向。

注意一件事：`Cov(X,X) = E[(X−μ_X)(X−μ_X)] = E[(X−μ_X)²] = Var(X)`。共變異數是變異數的雙變數版；變異數只是「X 跟自己的共變異」。這不是巧合，是同一個東西。

### 一條更好算的公式

跟 ch09 的 `Var(X) = E[X²] − μ²` 一樣，共變異數也有個展開後比較好手算的版本：

```text
Cov(X,Y) = E[XY] − E[X]·E[Y]
           ─────────────────
推導：E[(X−μ_X)(Y−μ_Y)]
    = E[XY − μ_X·Y − μ_Y·X + μ_X·μ_Y]
    = E[XY] − μ_X·E[Y] − μ_Y·E[X] + μ_X·μ_Y     ← 期望線性性（ch08）
    = E[XY] − μ_X·μ_Y − μ_Y·μ_X + μ_X·μ_Y
    = E[XY] − μ_X·μ_Y                              ← 兩項抵成一項
```

白話：**「乘積的平均」減掉「平均的乘積」**。如果 X 和 Y 各走各的（乘積的平均剛好等於平均的乘積），這個差就是 0。差越大，連動越強。手算時用這條通常比逐點算偏差快。

## 共變異數的尺度問題，以及相關係數怎麼救它

共變異數有個討厭的毛病：**它有單位，而且這個單位很醜**。

如果 X 是延遲（毫秒）、Y 是 CPU（百分比），那 Cov(X,Y) 的單位是「毫秒 × 百分比」。這是什麼鬼東西？沒人知道。更糟的是，它會被尺度牽著走：你把延遲從毫秒改成秒，數字直接縮小一千倍，但兩個變數的「連動強度」明明一點都沒變。Cov = 50 跟 Cov = 50000 可能描述的是一模一樣的關係，只是換了單位。

所以 Cov 的**正負號**有意義（同向／反向），但它的**大小**沒辦法直接讀——你沒法說「Cov = 50 算強嗎」，因為不知道拿什麼比。

解法很漂亮：把它**標準化**，用各自的標準差去除，把單位和尺度全部消掉。

```text
ρ(X,Y) = Cov(X,Y) / (σ_X · σ_Y)
         ──────────────────────
ρ 唸 "rho"。分母把 X、Y 各自的「尺度」除掉，剩下純粹的連動強度。
```

這就是**相關係數**（correlation coefficient），ρ。它做的事，跟你在 ch09 把延遲標準化成 z 分數（`z = (x−μ)/σ`）是同一個念頭——除以 σ，把量綱洗掉，讓不同單位的東西可以放到同一把尺上比。Cov 是帶單位的原始連動，ρ 是洗乾淨、可比較的連動。

而 ρ 最神的地方，是它**永遠落在 −1 和 +1 之間**：

```text
−1 ≤ ρ ≤ +1
 ρ = +1  完美正相關：Y 是 X 的遞增直線函數（點全落在一條上升直線上）
 ρ = −1  完美負相關：Y 是 X 的遞減直線函數（點全落在一條下降直線上）
 ρ =  0  不相關：沒有線性連動
 |ρ| 越接近 1，點越貼近一條直線
```

`|ρ| ≤ 1` 不是規定，是能證明的（背後是柯西–施瓦茲不等式，本書不展開；當作工程師的嚴謹記住「標準化後的連動不可能超過完美」）。這個範圍讓 ρ 變成一個**可比較的純數**：ρ = 0.9 就是比 ρ = 0.3 連動得緊，不管 X、Y 各自是什麼單位、差幾個數量級——一個能寫進報告、能跨指標比較的「連動強度」。

```text
   ρ ≈ +1            ρ ≈ +0.5          ρ ≈ 0             ρ ≈ −1
   Y│      ·         Y│    · ·         Y│ ·  · ·         Y│·
    │    ·            │   ·· ·          │  · · ·          │  ·
    │   ·             │  · ··           │ · ·  ·          │    ·
    │  ·              │ ··  ·           │· ·  · ·         │      ·
    └──────── X       └──────── X       └──────── X       └──────── X
   貼直線上升         鬆散上升           看不出方向         貼直線下降
```

## Worked example：5 個點，手算 Cov 和 ρ

光看定義沒用，得親手算一次才有體感。給你五個點（想成五次測量，X 是某個負載指標、Y 是某個延遲指標）：

```text
點：  (1, 2)  (2, 4)  (3, 5)  (4, 4)  (5, 5)
```

**第一步：算兩個平均。**

```text
μ_X = (1+2+3+4+5)/5 = 15/5 = 3
μ_Y = (2+4+5+4+5)/5 = 20/5 = 4
```

**第二步：每個點算偏差，再把兩個偏差相乘。** 這一步是 Cov 的核心，慢慢來：

```text
點      X−3    Y−4    乘積 (X−3)(Y−4)
(1,2)   −2     −2     +4      ← 都低於平均，同向 → 正
(2,4)   −1      0      0      ← Y 剛好在平均上
(3,5)    0      1      0      ← X 剛好在平均上
(4,4)    1      0      0
(5,5)    2      1     +2      ← 都高於平均，同向 → 正
                      ─────
                 總和 = +6
```

**第三步：Cov = 乘積的平均。**

```text
Cov(X,Y) = 6/5 = 1.2          ← 正的，所以正相關（X 高 Y 傾向高）
```

到這裡只知道「正相關」，但 1.2 算強嗎？不知道，因為它帶尺度。繼續算 ρ。

**第四步：算兩個變異數（偏差平方的平均），開根號得標準差。**

```text
X 的偏差平方：(−2)² (−1)² 0²  1²  2²  = 4, 1, 0, 1, 4  → 總和 10
Var(X) = 10/5 = 2          σ_X = √2 ≈ 1.414

Y 的偏差平方：(−2)² 0²  1²  0²  1²  = 4, 0, 1, 0, 1  → 總和 6
Var(Y) = 6/5 = 1.2         σ_Y = √1.2 ≈ 1.095
```

**第五步：ρ = Cov / (σ_X·σ_Y)。**

```text
σ_X · σ_Y = √2 · √1.2 = √2.4 ≈ 1.549
ρ = 1.2 / 1.549 ≈ 0.775
```

**交叉驗算**：用展開公式 `Cov = E[XY] − μ_X·μ_Y` 對一次。

```text
E[XY] = (1·2 + 2·4 + 3·5 + 4·4 + 5·5)/5 = (2+8+15+16+25)/5 = 66/5 = 13.2
μ_X·μ_Y = 3·4 = 12
Cov = 13.2 − 12 = 1.2          ✓ 跟逐點算的 1.2 一致
```

**ρ ≈ 0.78**。怎麼讀這個數？正、而且相當強，但不是 1——點大致沿一條上升直線排，但有抖動（(4,4) 那個點往下叉了一下，把完美打了折）。如果這是你監控裡的兩個指標，你會說「這兩個明顯連動，但不是鐵板一塊」。這就是 ρ 給你的：一個能讀、能比較的連動強度。

注意整個過程**沒有任何因果**。我們只量了「它們一起動的程度」。X 害的、Y 害的、還是都被第三者害的——這個數字一個字都沒說。記住這個空白，它是本章後半的全部。

## ρ 只抓線性：強到爆的關係也能 ρ = 0

這是本章第一個會把你帶溝裡的地方，值得停十分鐘，因為你的直覺正要騙你。

很自然會以為：ρ = 0 就代表 X 和 Y 沒關係、互不影響。**錯。ρ = 0 只代表「沒有線性連動」，X 和 Y 之間可以有強到不能再強的關係，ρ 照樣是 0。**

看一個乾淨的反例。設 X 等機率取 `{−2, −1, 0, 1, 2}` 這五個值（對稱地散在 0 兩邊），然後令 `Y = X²`。

這個關係強不強？強到極致——**Y 完全由 X 決定**，你給我 X 我就能精準算出 Y，一點隨機都不剩。沒有比這更緊的關係了。來算 ρ：

```text
μ_X = (−2−1+0+1+2)/5 = 0           ← X 對稱，平均剛好 0

Cov(X,Y) = E[XY] − μ_X·μ_Y = E[X·X²] − 0·μ_Y = E[X³]

E[X³] = ((−2)³+(−1)³+0³+1³+2³)/5
      = (−8 − 1 + 0 + 1 + 8)/5
      = 0/5 = 0                     ← 正負完美抵銷
```

`Cov = 0`，所以 `ρ = 0`。一個 Y 完全被 X 決定的關係，相關係數是 0。

怎麼會這樣？畫出來就懂了：

```text
   Y│
   4│ ·                 ·       ← Y = X² 是一條拋物線
    │
   1│      ·       ·
   0│           ·
    └──┬──┬──┬──┬──┬── X
      −2 −1  0  1  2

X 從 −2 走到 0：X 升、Y 降（左半邊在下降）
X 從  0 走到 2：X 升、Y 升（右半邊在上升）
```

左半邊「X 高 Y 低」（負向），右半邊「X 高 Y 高」（正向），兩邊的貢獻**完美抵銷**，Cov 加起來是 0。ρ 量的是「整體有沒有一致的上升或下降直線趨勢」，而拋物線左升右降，沒有一致方向——所以線性連動為零。但 Y 跟 X 的關係明明強到 100%。

**這是本章最重要的一個陷阱**，因為它在工程裡天天發生。你的系統裡有一堆 U 形或倒 U 形關係：

- 執行緒池大小 vs 吞吐量：太少不夠用、太多搶資源 context switch 爆掉，中間有甜蜜點——倒 U。
- 快取大小 vs 整體延遲：太小命中率低、太大 GC／記憶體壓力反咬——U 形。
- 重試次數 vs 成功率：不重試會漏、重試太兇打垮下游引發重試風暴——倒 U。

這些關係**強得要命**，但如果你只算一個 ρ，很可能拿到接近 0 的數，然後得出「這兩個沒關係」的災難性結論。ρ 在這裡完全瞎了，因為它只有一隻眼睛，那隻眼睛只看得見直線。

所以記住這條鐵律：**ρ = 0 只能讓你說「沒有線性關係」，永遠不能讓你說「沒有關係」。** 真要判斷兩個變數有沒有關係，先畫散佈圖（scatter plot）——眼睛看得到拋物線、看得到 U 形，ρ 看不到。

## 獨立 ⟹ 不相關，但不相關 ⇏ 獨立

上面那個 `Y = X²` 的例子，順手解決了一個更深的問題——獨立和不相關的關係。ch04 我留了一句話：相關（correlation）是「相依強度的連續版」，獨立是「ρ=0 且更強」的條件，本章會講清楚「不相關 ⇏ 獨立」。現在來兌現。

先把兩個詞釘死：

- **不相關（uncorrelated）**：`ρ = 0`，也就是 `Cov(X,Y) = 0`。只說「沒有線性連動」。
- **獨立（independent）**：`P(X∩Y) = P(X)·P(Y)`（ch04 那條）。說的是「知道 X 對 Y 的判斷毫無影響」——任何形式的影響都沒有，不只線性。

兩者的關係是一個**單向箭頭**：

```text
獨立  ⟹  不相關          （成立）
不相關  ⇏  獨立          （不成立！反例就在上面）
```

**先看「獨立 ⟹ 不相關」為什麼成立。** 如果 X、Y 獨立，那 `E[XY] = E[X]·E[Y]`（獨立時乘積的期望等於期望的乘積，這是獨立的一個結果）。代進共變異公式：

```text
Cov(X,Y) = E[XY] − E[X]·E[Y] = E[X]·E[Y] − E[X]·E[Y] = 0
```

獨立必然不相關。乾淨俐落。

**再看「不相關 ⇏ 獨立」為什麼不成立——反例就是剛剛的 `Y = X²`。** 那裡 `ρ = 0`（不相關），但 X 和 Y 獨立嗎？絕對不獨立——`Y` 完全由 `X` 決定，知道 `X = 2` 我就 100% 確定 `Y = 4`。知道 X 對 Y 的判斷影響大到不能再大，這是相依的極端。所以：**不相關，卻高度相依。**

```text
   所有情況
   ├─ 不相關（ρ=0，沒線性連動）          ← 大圈
   │   ├─ 獨立（X 對 Y 完全沒資訊）       ← 小圈，被包在裡面
   │   └─ 不相關但不獨立 ◄── Y=X² 卡在這層夾心
   │                          （ρ=0 卻高度相依）
   └─ 相關（ρ≠0，有線性連動）

   獨立 ⊂ 不相關：獨立是更強的條件，被「不相關」整個包住。
   反過來不成立——不相關那圈裡，藏著一堆 ρ=0 卻相依的非線性關係。
```

獨立是**更嚴格**的條件：它要求 X 對 Y 沒有**任何**形式的資訊。不相關只要求沒有**線性**形式的連動。獨立的事一定不相關，但不相關的事可以暗藏一拖拉庫非線性的相依——拋物線、U 形、週期，全都可以 ρ = 0 但綁得死死的。

這在工程上的教訓跟上一節同一個：**你用 ρ = 0 不能推出「這兩個指標獨立、可以分開處理」。** 它們可能透過某個非線性關係（或某個 confounder）緊緊綁著，只是不是直線形狀，ρ 抓不到。把「不相關」當「獨立」用，是把一個弱結論偷渡成強結論——ch04 那個「假獨立」的病，這裡換個皮又出現一次。

## 補完 ch09：Var(X+Y) 那個被你略過的項

ch09 講變異數可加性時，我給的是「獨立情形」：`Var(X+Y) = Var(X) + Var(Y)`。現在有了共變異數，可以補上完整版——非獨立時，多一項。

```text
Var(X+Y) = Var(X) + Var(Y) + 2·Cov(X,Y)
                              ─────────────
                              ch09 偷偷假設了這項 = 0（即獨立／不相關）
```

推導不長，每步都能口頭講（工程師的嚴謹）：

```text
Var(X+Y) = E[((X+Y) − (μ_X+μ_Y))²]
         = E[((X−μ_X) + (Y−μ_Y))²]                ← 把偏差分開
         = E[(X−μ_X)² + 2(X−μ_X)(Y−μ_Y) + (Y−μ_Y)²]   ← (a+b)² 展開
         = E[(X−μ_X)²] + 2·E[(X−μ_X)(Y−μ_Y)] + E[(Y−μ_Y)²]   ← 期望線性性
         = Var(X)      + 2·Cov(X,Y)              + Var(Y)
```

那個 `2Cov` 就是 `(a+b)² = a² + 2ab + b²` 裡的交叉項 `2ab` 取了期望。當 X、Y 獨立（或只是不相關）時 Cov = 0，它消失，退回 ch09 的版本。

這項在工程上有血有肉的意義——它管的是**相關性如何放大或縮小總波動**：

- **Cov > 0（正相關）**：總和的波動**比各自相加更大**。兩個正相關的負載源加起來，尖峰會疊在一起、互相加碼。你以為合併兩個服務能「分散風險」，但如果它們正相關（同時被同一波流量打），總波動反而更兇。
- **Cov < 0（負相關）**：總和的波動**比各自相加更小**——這正是分散投資、負載對沖的數學。一個忙的時候另一個閒，加起來反而平穩。
- **Cov = 0**：剛好相加，ch09 的乾淨情形。

這也順手解釋了 ch09 那個 `√n` 法則的前提：n 個變數相加、SD 長 √n，這條**只在它們互不相關時成立**。如果你的 n 個「獨立」副本其實正相關（共享某個 confounder），它們的總波動會比 √n 大得多——又一次，相關性故障在數學上現形。你以為 n 個備援把波動稀釋了 √n 倍，正相關會讓這個稀釋大打折扣。

## 相關 ≠ 因果：把口號拆成三條岔路

好，進本章的招牌。「相關不是因果」這句話被講爛了，爛到失去意義。但它之所以重要，不是因為它是個咒語，而是因為**當你看到 X 和 Y 相關，背後其實有四種完全不同的世界可能成立**，而你光看那個 ρ 分不出是哪一種。

假設你量到 X 和 Y 強相關（ρ 很高）。可能的真相有：

```text
ρ(X,Y) 很高，背後可能是 ——

(1) X → Y          真因果：X 確實造成 Y
(2) Y → X          反向因果：其實是 Y 造成 X，你把箭頭看反了
(3) Z → X, Z → Y   共同原因（confounder）：第三者 Z 同時推動 X 和 Y
(4) 純屬巧合        虛假相關：兩條無關的線剛好同步

       這四種在資料裡長得一模一樣。ρ 分不出來。
```

這是整章我最想讓你記住的一張圖。**相關是對稱的**（`ρ(X,Y) = ρ(Y,X)`，從定義就看得出來，乘積不分先後），但**因果是有方向的**。你拿一個對稱的量（ρ），去推一個有方向的東西（因果），資訊天生不夠。一個 ρ = 0.9，可能是上面四種裡的任何一種，甚至是好幾種疊在一起。

逐條看，每條都配一個你會遇到的版本。

### 岔路一：純屬巧合（虛假相關）

最廉價的一種。資料夠多，你**一定**找得到剛好同步的兩條曲線，哪怕它們八竿子打不著。

有個叫 Tyler Vigen 的人做了個網站專收這種（《Spurious Correlations》，2026-06 查證），裡面有個經典案例：**美國緬因州的離婚率，和人均人造奶油（margarine）消費量，相關係數高達 0.99 以上**。0.99 比很多正經的預測模型還高。但離婚跟人造奶油顯然毫無關係——這純粹是在海量的時間序列裡撈出來的巧合。你拿夠多條每年下降的曲線，兩兩比，總有幾對長得幾乎一樣。

這在你的工作裡叫**多重比較**的工程版：dashboard 上幾十個指標，你眼睛掃過去，總會抓到兩條剛好一起尖的，然後大腦立刻替你編一個因果。但你抓到的，很可能只是幾十個指標裡注定會出現的那對巧合。ch19 會把這個「看夠多圖總有一張顯著」講成 p-hacking，這裡先聞個味道。

防禦：問「這個相關有機制嗎？我講得出 X 怎麼影響 Y 的故事嗎？」講不出機制、只有兩條同步的線——先當巧合，別動手。

### 岔路二：共同原因（confounder）

這是最常見、也最值錢的一種，因為它最會偽裝成因果。

教科書級的例子：**冰淇淋銷量和溺水死亡數，正相關**（2026-06 查證，這是統計教學的經典）。冰淇淋賣得多的時候，溺水的人也多。難道吃冰淇淋會讓人溺水？當然不是。真相是有個第三者同時推動兩者——**氣溫（夏天）**。天熱，大家買冰淇淋；天熱，大家去游泳，溺水自然變多。氣溫是 confounder，它躲在背後，同時牽著冰淇淋和溺水。

```text
        氣溫（夏天）  ← confounder，真正的共同原因
         ╱        ╲
        ╱          ╲
   冰淇淋銷量 ⋯⋯⋯ 溺水死亡數
              ↑
        看得到的相關（ρ 高）
        但兩者之間沒有箭頭——它們不互相造成
```

虛線是你在資料裡量到的相關；它很真。但那兩個變數之間**沒有因果箭頭**，箭頭全在 confounder 那裡。你拿走冰淇淋（禁賣），溺水數一點都不會降——因為冰淇淋從來就不是原因。

這就是本章開頭那個工程場景的數學：**CPU 高和延遲高相關，但 confounder 是流量。** 流量同時頂高了 CPU 和延遲。你去優化 CPU、降下來了，延遲卻沒動——因為 CPU 不是延遲的原因，它們是流量這個共同原因下的兩個兄弟。三天白幹。

confounder 的危險在於：它在資料裡跟真因果**長得一模一樣**——都是「X 和 Y 一起動」。你光看 ρ 永遠分不出「X 造成 Y」和「Z 同時造成 X 和 Y」。要分辨，得跳出資料、去想機制，或者——做實驗（下面講）。

### 岔路三：反向因果

箭頭看反了。你以為 X 造成 Y，其實是 Y 造成 X。

工程例子：你發現「重啟服務」和「服務不健康」高度相關，於是有人說「別亂重啟，重啟害服務變不健康」。箭頭反了——是服務先不健康，你才去重啟它（不健康 → 重啟，不是重啟 → 不健康）。或者「擴容」和「高負載」相關，被讀成「擴容導致高負載」——不，是高負載觸發了擴容。

反向因果常常一講就破，但在快速看圖的會議裡出奇地容易蒙混，尤其當你心裡已經有個想證明的故事時。防禦：問「會不會是反過來？我憑什麼確定箭頭朝這邊？」很多時候時間順序能救你（誰先發生），但小心——連時間順序都可能被 confounder 騙。

### 為什麼 A/B 隨機化能一刀斬斷這一切

那到底怎麼從相關走到因果？這是統計學最漂亮的一招，也是 ch16 的引線，這裡先給你直覺。

困境是：confounder（岔路二）躲在暗處，你甚至不知道它存在，怎麼防？你沒法把所有可能的第三者都列出來一一排除。

天才的解法是**隨機化（randomization）**：別去觀察 X 和 Y 自然怎麼動，而是**主動、隨機地**去設定 X，看 Y 跟著怎麼變。

```text
觀察（observational）：              隨機實驗（A/B）：
你看著 X 和 Y 自然變動               你丟硬幣決定每個單位進 A 組或 B 組
confounder Z 在背後同時推兩個        ──────────────────────────────────
分不出 X→Y 還是 Z→{X,Y}             硬幣跟任何 confounder 都無關
                                    → 兩組的所有 confounder 統計上自動均勻
                                    → 兩組唯一系統性差別只剩你動的那個 X
                                    → 這時 Y 的差別只能歸給 X = 因果
```

關鍵在那枚硬幣。你用**隨機**決定誰接受處理（A 組）、誰當對照（B 組），那麼任何 confounder——不管是氣溫、流量、使用者年齡，連你沒想到的、不知道存在的都算——在兩組之間都會**統計上均勻分布**。因為硬幣不認識任何 confounder，它不會偏心地把「高氣溫的日子」全塞進 A 組。於是兩組除了「你動的那個 X」之外，其他一切平均而言都一樣。這時候如果 Y 在兩組有系統性差別，那**只可能**是 X 造成的——confounder 被隨機化均勻掉了，反向因果被「是你主動設定 X」排除了，巧合被「夠大的樣本＋顯著性檢定」壓住了。

這就是為什麼 A/B test 能講因果、而看 dashboard 不能。dashboard 是觀察，你只是看著指標自然動，confounder 全躲在裡面；A/B 是實驗，你親手隨機地撥動一個變數，斬斷了它跟所有 confounder 的糾纏。**隨機化是人類發明的、把相關升級成因果的唯一可靠引擎**——我認為這是統計學最被低估的一個天才，ch16 會講透它。這裡先記住：那枚斬斷 confounder 的硬幣，就是 ch01 那枚硬幣，它在這裡又立了一功。

## 一段歷史：Galton 和 Pearson 怎麼把「相關」變成數字

「兩件事有關係」這個念頭，人類想了幾千年。但把它變成一個能算、有範圍、可比較的**數字**，是很晚、而且很意外的事。

主角是 Francis Galton（高爾頓，1822–1911）——達爾文的表弟，一個什麼都量的人。他本來在研究遺傳：父母身高跟子女身高有什麼關係？他畫了一堆親子身高的散佈點，發現點雲沿著一條直線散開，而且高個父母的孩子傾向比父母矮一點、矮個父母的孩子傾向比父母高一點，全都往中間「回歸」。這就是 ch19 要講的**回歸均值**，Galton 1886 年論文用的詞是「regression towards mediocrity（向平庸回歸）」（照 landscape）。

但 Galton 接著想：這條關係的**強度**能不能量化？1888 年，他發表了《Co-relations and their Measurement, Chiefly from Anthropometric Data》（〈共relation 及其測量，主要取自人體測量資料〉），登在 Proceedings of the Royal Society 第 45 卷（2026-06 查證）。這篇**第一次正式給出了相關係數**。有個小細節很可愛：他刻意把標題第一個字寫成「Co-relation」（共-關係）而不是「Correlation」，因為他覺得這個帶連字號的寫法更能傳達「兩個東西**互相**關聯」的意思（2026-06 查證）。他用的符號是字母 **r**——取自 reversion／regression（回歸）的字首（照 landscape）。所以今天你看到的相關係數記號 r，源頭是「回歸」這個字，而不是「相關」。

Galton 給了直覺和雛形，但數學上的嚴謹化、把它變成今天教科書那條「乘積動差相關係數（product-moment correlation coefficient）」公式的，是他的門徒 **Karl Pearson**（皮爾森，1857–1936）。Pearson 1896 年發表了相關與迴歸的第一個嚴格處理（照 landscape）。所以今天這個 r 的正式名稱叫 **Pearson's r**（皮爾森相關係數），紀念的是把它釘成精確公式的那個人。

這段歷史有個值得停一下的點：**「相關」這個概念，是先有了具體的數（Galton 在算親子身高）、才有了抽象的定義（Pearson 的公式）**，而不是反過來。而且它一出生就帶著「回歸」的胎記（符號 r、Galton 的研究動機）——相關和回歸均值是同一條線上的雙胞胎，ch19 會讓它們團聚。Galton 把一個幾千年的模糊念頭——「有關係」——壓成了一個介於 −1 和 1 之間的數。在那之前，「兩件事多有關」沒法比較、沒法吵架；在那之後，它變成了科學。這是把定性變定量的經典一役。

## 直覺的陷阱

| 你以為 | 實際上 | 怎麼自我察覺 |
|---|---|---|
| ρ = 0 代表兩個變數沒關係、互不影響 | ρ = 0 只代表**沒有線性連動**。Y=X²（拋物線）、U 形、倒 U 形的關係可以強到 100% 但 ρ = 0 | 你只算了一個 ρ 就下「無關」的結論——停。先畫散佈圖。眼睛看得到的拋物線，ρ 是瞎的 |
| 不相關（ρ=0）就等於獨立 | 獨立是**更強**的條件（X 對 Y 沒有任何形式的資訊）。獨立 ⟹ 不相關，但**反過來不成立**（Y=X² 就是不相關卻高度相依） | 你想把兩個指標「分開處理」因為它們 ρ=0——小心，它們可能透過非線性關係或 confounder 綁死，只是不是直線 |
| 兩條曲線一起動 ⟹ 一個造成另一個 | 還有三條岔路：反向因果（箭頭反了）、共同原因（confounder 同時推兩個）、純屬巧合（虛假相關）。ρ 是對稱的，分不出方向 | 你準備去改 A 來修 B 之前，問：「會不會是 B→A？會不會有個 Z 同時推 A 和 B？這有機制嗎還是巧合？」 |
| CPU 高造成延遲高（因為它們相關） | 很可能是 confounder（流量）同時頂高兩者。你降 CPU，延遲不動 | 你優化了 X、Y 卻沒跟著改善——這通常是 confounder 的指紋。X 不是因，是兄弟 |
| 相關係數越高，因果越強 | ρ 跟因果強度毫無關係。緬因州離婚率 vs 人造奶油 ρ>0.99，因果是零 | 你拿 ρ 的大小當因果證據——ρ 再高也只是「一起動得很整齊」，不證明誰造成誰 |
| 看 dashboard 找到的相關可以拿來做決策 | dashboard 是**觀察**，confounder 全躲在裡面。要談因果得做**隨機實驗**（A/B） | 你想從觀察資料推因果——除非你能隨機化，否則你最多只能說「相關」，講因果就是在賭 confounder 不存在 |

招牌謬誤點名：**「相關即因果」與它的孿生兄弟「ρ=0 即無關」**。前者把對稱的相關硬讀成有方向的因果；後者把「沒有線性」誤當「沒有關係」。這兩個是工程師看圖時最常栽的跟頭——一個讓你去修錯的東西（改了 confounder 的兄弟、白費三天），一個讓你錯放一個強關係（U 形被當成無關、甜蜜點調不到）。下次你想從兩條曲線得出任何結論前，先做兩個動作：(1) 畫散佈圖（防 ρ=0 陷阱），(2) 問「這背後是四種岔路的哪一種」（防相關即因果）。這兩個自問，比任何公式都護身。

## 紙上推演

> 全程紙筆，不寫程式。每題先自己推，再看解答。Cov 和 ρ 容易在偏差、平方、開根號這幾步手滑——每個數值至少用兩種方式對一次。

### 推演題 1：手算兩個指示變數的共變異 **[15 分鐘]** ★★

擲一顆公正骰子。定義兩個指示變數（indicator，事件發生為 1、否則 0）：

- `A = 1` 若點數 ≥ 4（即 4、5、6），否則 0。
- `B = 1` 若點數是偶數（即 2、4、6），否則 0。

(a) 算 `E[A]`、`E[B]`、`E[AB]`，用 `Cov(A,B) = E[AB] − E[A]E[B]` 求共變異數。
(b) 判斷 A、B 是正相關還是負相關，並用「同向／反向偏離平均」的直覺解釋這個正負號。
(c) 順便算 ρ（你會需要指示變數的變異數：`Var(A) = E[A²] − E[A]² = E[A] − E[A]²`，因為 `A² = A`）。

#### 推演解答

(a) 一顆骰子六個面等機率，逐面列表最不會錯：

```text
點數   A(≥4)  B(偶)  A·B
 1      0     0     0
 2      0     1     0
 3      0     0     0
 4      1     1     1     ← 同時為 1 只有 4 和 6
 5      1     0     0
 6      1     1     1
       ────  ────  ────
 和     3     3     2
```

```text
E[A]  = 3/6 = 1/2
E[B]  = 3/6 = 1/2
E[AB] = 2/6 = 1/3          ← AB=1 只在點數同時「≥4 且偶」，即 4、6 兩面

Cov(A,B) = E[AB] − E[A]E[B] = 1/3 − (1/2)(1/2) = 1/3 − 1/4 = 1/12 ≈ 0.083
```

(b) `Cov = +1/12 > 0`，**正相關**。直覺：「點數 ≥ 4」和「點數偶」這兩件事傾向同時發生（4 和 6 兩個面同時滿足，而且 6 個面裡有 2 個讓兩者同向為 1）。換成偏差語言——當點數落在 {4,6} 時，A 和 B 同時高於各自平均（偏差同號、乘積為正），這些正貢獻沒被完全抵銷，所以 Cov 是正的。直覺檢查：大的數比較容易是偶數嗎？在骰子上，≥4 的三個面是 {4,5,6}，其中兩個是偶數（多於一半）；而 <4 的 {1,2,3} 只有一個偶數。所以「大」和「偶」確實沾邊，正相關合理。

(c) 指示變數的變異數（用 `A²=A`）：

```text
Var(A) = E[A] − E[A]² = 1/2 − 1/4 = 1/4      σ_A = 1/2
Var(B) = E[B] − E[B]² = 1/2 − 1/4 = 1/4      σ_B = 1/2

ρ = Cov / (σ_A·σ_B) = (1/12) / (1/2 · 1/2) = (1/12)/(1/4) = 4/12 = 1/3 ≈ 0.33
```

**交叉驗算**：ρ 應落在 [−1,1]，1/3 合格；正號跟 Cov 一致。中等強度的正相關——合理，因為 {4,5,6} 裡只有 2/3 是偶數，連動不到鐵板一塊。

### 推演題 2：對「穿大鞋的人識字率高」找出 confounder **[10 分鐘]** ★★

有人拿一份全國人口資料，算出「鞋子尺碼」和「識字率」**強烈正相關**——鞋子越大的人，識字率越高。他得出結論：「腳大讓人聰明」或「多讀書腳會長大」。

(a) 指出這個相關背後最可能的 confounder。
(b) 畫出三變數的關係圖（誰指向誰）。
(c) 說明為什麼「把腳撐大」不會提升識字率——用 confounder 的語言講死它。

#### 推演解答

(a) confounder 是**年齡**（特別是「成人 vs 兒童」）。小孩腳小、識字率低（還沒學會）；成人腳大、識字率高。把大人小孩混在一起算，年齡同時推動了鞋碼和識字率，製造出一個假的「鞋碼↔識字」相關。

(b) 關係圖：

```text
            年齡        ← confounder（真正的共同原因）
           ╱    ╲
          ╱      ╲
      鞋子尺碼 ⋯⋯ 識字率
              ↑
        量到的強相關（ρ 高）
        但兩者之間沒有因果箭頭
```

(c) 鞋碼和識字率之間**沒有任何因果箭頭**——箭頭全在年齡那裡。那個相關是年齡同時牽動兩者「漏」出來的假象。證據：如果你**固定年齡**（只看成年人，或只看八歲小孩），鞋碼和識字率的相關會**消失**——因為在同一個年齡層裡，腳大的人並不比腳小的人更識字。這正是「分組看就翻盤／消失」的辛普森味道（ch19 的引線）。所以把誰的腳撐大，識字率一動都不會動：你動的是 confounder 的一個兄弟，不是原因。這跟「降 CPU 救不了延遲（因為 confounder 是流量）」是同一個病。

### 推演題 3：為什麼 ρ=0 不能下結論說兩者無關 **[15 分鐘]** ★★★

(a) 重做正文那個反例：X 等機率取 `{−2,−1,0,1,2}`、`Y = X²`，手算 `Cov(X,Y)` 確認 = 0。
(b) 用一句話解釋這個 Cov 為什麼會是 0（從「左半邊反向、右半邊同向、互相抵銷」講）。
(c) 舉一個**你工作中**的非線性關聯（U 形或倒 U 形），說明如果只看 ρ 會得出什麼災難性的錯誤結論。

#### 推演解答

(a) 逐點算（X 五個值各機率 1/5）：

```text
X     Y=X²   X−μ_X   Y−μ_Y    (X−μ_X)(Y−μ_Y)
            (μ_X=0) (μ_Y=2)
−2     4     −2       2        −4
−1     1     −1      −1        +1
 0     0      0      −2         0
 1     1      1      −1        −1
 2     4      2       2        +4
                              ──────
                         總和 = 0

Cov(X,Y) = 0/5 = 0     ⟹  ρ = 0
```

μ_Y 驗算：`(4+1+0+1+4)/5 = 10/5 = 2` ✓。μ_X = 0（對稱）✓。
**交叉驗算**（展開公式）：`Cov = E[XY] − μ_Xμ_Y = E[X³] − 0·2 = E[X³]`，而 `E[X³] = (−8−1+0+1+8)/5 = 0` ✓。兩法一致。

(b) 左半邊（X 從 −2 到 0）X 升 Y 降，貢獻負的乘積；右半邊（X 從 0 到 2）X 升 Y 升，貢獻正的乘積。拋物線對稱，左右兩邊的貢獻**大小相等、符號相反，完美抵銷**，所以 Cov = 0。但 Y 明明完全由 X 決定（給 X 就鎖死 Y）——關係強到極致，ρ 卻看不見，因為 ρ 只認直線、不認拋物線。

(c) 範例（你的答案會不同，重點是抓到「強關係但 ρ≈0」）：**執行緒池大小 vs 吞吐量**，倒 U 形——池太小，併發不足、吞吐低；池太大，context switch 和鎖競爭把吞吐拖垮；中間有個甜蜜點吞吐最高。如果你拿一堆不同池大小的測量算一個 ρ，很可能拿到接近 0 的數（左半邊正相關、右半邊負相關抵銷），然後得出「池大小跟吞吐量無關，隨便設」——災難。事實上它們關係強到你的容量規劃全靠它。正確做法：畫散佈圖看出倒 U、找甜蜜點，而不是信那個瞎了的 ρ。

### 推演題 4：把「相關即因果」的結論拆成四條岔路 **[15 分鐘]** ★★

口頭＋紙筆題。下面每個情境都有人從「相關」跳到了「因果」。對每一個，判斷它最可能是四條岔路（真因果 / 反向因果 / 共同原因 / 純屬巧合）的哪一種，並說怎麼驗證：

(i) 「我們發現用了新 SDK 的客戶端，崩潰率比較高，所以新 SDK 害的。」
(ii) 「冰淇淋賣得多的月份，溺水也多，所以該禁冰淇淋。」
(iii) 「重啟次數多的服務，不健康事件也多，所以別重啟。」
(iv) 「我們網站流量上升的那幾週，剛好某遠方國家的咖啡進口量也上升，市場有連動。」

#### 推演解答

沒有絕對標準答案（真相要靠資料和機制判斷），但合理的拆法：

(i) **可能是真因果，但要小心共同原因**。新 SDK 真的可能有 bug（X→Y）。但也可能是 confounder：願意嘗鮮裝新 SDK 的，往往是用 beta 通道、跑舊機型、或網路差的高風險使用者——是「使用者類型」這個 Z 同時推高了「裝新 SDK」和「崩潰率」。驗證：**做隨機實驗**——隨機把一部分使用者推新 SDK（而不是讓他們自選），比較崩潰率。隨機化斬斷「使用者類型」這個 confounder。

(ii) **共同原因（confounder = 氣溫/夏天）**。冰淇淋和溺水都被夏天頂高，彼此無因果。驗證：固定季節（只看七月內各週），相關會消失。禁冰淇淋救不了任何人。

(iii) **反向因果**。是服務先不健康、你才去重啟它（不健康 → 重啟），不是重啟害它不健康。驗證：看時間順序——不健康事件是在重啟**之前**還是之後發生？十之八九在之前。

(iv) **純屬巧合（虛假相關）**。流量和遠方咖啡進口量沒有任何可信機制，這是在海量時間序列裡撈到的同步（緬因離婚率 vs 人造奶油那一類）。驗證：問「有機制嗎？」——沒有；換個時間窗或換個國家，相關大概就垮了。別動手。

四題的共同解藥：**看到相關，先別接受箭頭。** 跑一遍「真因果？反向？confounder？巧合？」這張清單，再決定要不要、以及怎麼做實驗。能隨機化就隨機化——那是把相關升級成因果的唯一硬通貨。

## 自我檢核

口頭自答，講得清楚才算過。優先答「為什麼」。

1. 共變異數定義裡那個「兩個偏差相乘」`(X−μ_X)(Y−μ_Y)`，為什麼能抓到「同向／反向」？把「負負得正」那個機制講給人聽，不准只念公式。
2. 共變異數已經告訴你正相關還是負相關了，為什麼還要除以 `σ_X·σ_Y` 變成 ρ？ρ 比 Cov 多了哪兩個好處？（提示：尺度、可比較）
3. 「ρ = 0 所以這兩個指標沒關係」——這句話錯在哪？用 `Y=X²` 或一個你自己的 U 形例子講清楚 ρ「瞎」在哪裡。
4. 「獨立」和「不相關」差在哪？為什麼「獨立 ⟹ 不相關」但「不相關 ⇏ 獨立」？哪個是更強的條件，為什麼？
5. `Var(X+Y) = Var(X) + Var(Y) + 2Cov(X,Y)` 裡那個 `2Cov` 項，在工程上什麼時候會讓「合併兩個服務」的總波動**變大**、什麼時候**變小**？這跟 ch09 的 √n 法則前提有什麼關係？
6. 「相關不是因果」背後有四條岔路，講得出是哪四條嗎？為什麼一個 ρ 永遠分不出是哪一條？（提示：ρ 對稱、因果有方向）
7. 為什麼 A/B 隨機化能把「相關」升級成「因果」，而盯 dashboard 不能？那枚隨機的硬幣到底斬斷了什麼？
8. 你的系統裡「CPU 高和延遲高相關」——為什麼降 CPU 可能救不了延遲？這是四條岔路的哪一條，confounder 是誰？

## 延伸閱讀

- **Tyler Vigen，《Spurious Correlations》（網站 tylervigen.com 與同名書，2026-06 查證）**。本章「虛假相關」段的活教材。直接去看那些 ρ>0.99 卻荒謬絕倫的配對（緬因州離婚率 vs 人造奶油、各種離奇同步曲線）——看完你對「相關係數高」這件事會永久免疫，再也不會把高 ρ 當因果證據。當笑話讀，但記住那個寒意：你的 dashboard 上也有這種對子。
- **Francis Galton，〈Co-relations and their Measurement, Chiefly from Anthropometric Data〉，Proc. Royal Society 45 (1888)（2026-06 查證）**。相關係數的出生證明，相當短、可讀。看 Galton 怎麼從親子身高的散佈點，第一次把「有關係」壓成一個數，以及他為什麼堅持寫「co-relation」。歷史現場感十足——一個概念誕生的那一刻。
- **任何一本扎實機率入門的「共變異與相關」章**（如 Blitzstein & Hwang《Introduction to Probability》，網路有免費講義與課程影片）。如果你想把 `Cov`、ρ、`Var(X+Y)` 的代數練到反射、看 `|ρ|≤1` 的柯西–施瓦茲證明，看它的推導與例題。但記得本書立場：你要的是「為什麼 ρ 只抓線性、為什麼相關不是因果」的直覺，不是刷題。
- **Judea Pearl & Dana Mackenzie，《The Book of Why》（中譯《因果革命》）**。如果本章「相關 ≠ 因果」讓你癢起來、想知道怎麼**正式地**從資料推因果（而不只是隨機實驗），這本是現代因果推論的科普入口——confounder、do-演算、因果圖。本書刻意只到「三條岔路＋隨機化直覺」為止（多變量與因果框架是全域不涵蓋的洞），想往下走就讀它。
- **本書內部銜接**：「不相關 ⇏ 獨立」是 ch04「假獨立」陷阱的近親（獨立是 ρ=0 且更強）；隨機化斬斷 confounder 的完整威力見 ch16（抽樣的大膽）；confounder 製造「分組翻盤」的極致是 ch19 辛普森悖論；相關係數的孿生兄弟「回歸均值」（同樣源自 Galton、同樣用符號 r）也在 ch19——你會看到本章埋的這條線怎麼開花。
