# 寫作風格指南（內部文件，非書籍內容）

本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名（工作名）：《馴服隨機：機率與統計的直覺與驚嘆》。

## 這本書是什麼、不是什麼

- **是**：一本「欣賞與直覺」的書。目標是讓讀者親身感受人類如何在最不可預測的東西（隨機）裡找到鐵一般的秩序——看懂機率統計不是一堆公式與查表，而是「精確地與不確定共處」的語言；讀完有「人類竟然想得出這個」的震撼，**而且能用自己的話對另一個工程師講清楚震撼在哪**。
- **不是**：
  - 不是考試解題訓練——不刷組合計算、不背分布公式表、不出計算操練題。
  - 不是測度論機率教科書——σ-algebra、測度只在 ch22 收官時一句話帶過動機，不折磨讀者。
  - 不是統計工具教學——不教 R/SciPy、不教「怎麼跑 t 檢定」。p 值、信賴區間當作**欣賞與誠實理解**的對象（它在說什麼、它在哪裡崩壞），不是 how-to。
  - 不是轉職用書——純興趣，不服務 AI Infra 敘事；但工程橋接要讓它讀起來像母語，不是教科書的外語。
- **驗收標準（全書的六條主線，ch23 總收官口試）**：
  ① 機率到底在量化什麼（無知／長期頻率／信念——三種詮釋撐同一個數）；
  ② 秩序如何從混沌浮現（大數法則的**精確**意義：不是「偏差被修正」而是「被稀釋」）；
  ③ 中央極限為何讓鐘形無所不在（加總／平均的普適性）；
  ④ 貝氏：證據如何更新信念（先驗→後驗的機制與世界觀）；
  ⑤ 統計推論的大膽（從樣本推斷母體）與它的崩壞（p 值、複現危機）；
  ⑥ 人類對隨機的直覺在哪裡、為什麼系統性崩壞（招牌故障視角）。

## 數學記號規範（全書一致，附錄 A 是速查表；閱讀器不渲染 LaTeX，一律純文字 Unicode）

- **內文行內**用 Unicode：P(A)、P(A|B)、P(A∩B)、P(A∪B)、E[X]、Var(X)、σ、σ²、μ、ρ（相關係數）、Σ、∫、√、≈、≠、≤、≥、→、∞、~（服從，如 X ~ N(μ,σ²)）、∅、Aᶜ（餘事件）、x̄（樣本平均，用組合字元或寫「x̄」）、p̂（估計量，寫「p̂」或「p-hat」擇一一致——全書用「p̂」）。

4. `## 直覺的陷阱` — 本書招牌段落（故障視角的機率版，全書最重要的賣點之一，因為機率正是人類直覺最常崩壞的領域）：這個概念**最常被誤解成什麼**、錯誤直覺**會在哪一步把你帶溝裡**、**怎麼自我察覺**（什麼徵兆代表你其實沒懂）。用表格或條列。每章至少一個具名謬誤／悖論（賭徒謬誤、基率忽視、檢察官謬誤、辛普森悖論、信賴區間誤讀…）。
5. `## 紙上推演` — 動手做的形式（**全書純紙筆推演，不放任何程式碼**，2026-06-13 訪談定案）。每章 2~4 題，標注預估時間（如 **[15 分鐘]**）與難度（★~★★★）。題型：手算機率／期望／貝氏更新表、手推論證、找出錯誤論證的破綻、把白話翻成機率語言、用自然頻率法重算一個貝氏題、口頭重講一個悖論的解。題目後緊接 `### 推演解答` 給完整解答與討論（為什麼這樣想、常見錯路）。**Monte Carlo 的直覺可以用文字描述「想像擲一萬次」**，但不寫可執行程式碼；數值表一律手算示範。

## 深度標準

- **每章至少一個 worked example**：帶具體數字、一步一步算的完整過程（機率樹、貝氏更新表、收斂表、分布直方圖、信賴區間計算），不准跳步。outline 已為每章指定基準範例。
- **機率的「精確意義」優先於公式**：每個概念先回答「它在量化什麼問題」（機率在量化無知還是頻率？期望值是長期平均不是最可能值？信賴區間到底在保證什麼？），再給算法。禁止無動機的定義。
- **誠實標示嚴謹度**：每個論證明確標示自己是哪一級——「直覺版（嚴格證明本書不展開，指向延伸閱讀）」或「工程師的嚴謹：每步能口頭說出理由」。禁止「顯然」「易證」；禁止把直覺論證偽裝成證明。對 LLN／CLT 給直覺與條件、不給測度論證明。
- **數值自我複核**：所有數值（機率、期望、貝氏後驗、收斂表、信賴區間、p 值）寫章時必須自己重算一遍（心算／筆算／必要時口頭逐步驗證），不得抄自記憶。跨章基準數值以 outline 基準表與 landscape 的「數值基準複核」節為準。機率題尤其容易算錯——每個 worked example 至少用兩種方法或自然頻率法交叉驗算一次。
- **歷史事實照 landscape**：年份、歸屬、引文、軼事、悖論的精確陳述一律以 `_meta/landscape-2026-06.md` 為準；landscape 沒有的自行查證（每章 3~8 次 web 查證）並 hedge；傳說要標明是傳說。**特別注意 landscape 標 ⚠️ 的「常被誤傳」項目**（如熱手謬誤 2018 反轉、信賴區間誤讀、p<0.05 不是費雪的硬規則、蒙提霍爾依賴主持人行為假設）——這些正是「直覺的陷阱」段的好素材，務必用正確版。禁止憑記憶引用「名人名言」。
- **每個「驚嘆點」要能被重講**：本書的核心商品是震撼感。每章至少一處明確回答「這裡了不起在哪」——而且要寫到讀者能用自己的話轉述，不是堆形容詞。（範例：個體完全不可測但總體精確可測；鐘形的普適性不講原分布長相；信念可以有數學；你只問 1000 人就能知道兩億人。）
- **故障視角是賣點**：機率是人類直覺最常崩壞的領域，「直覺的陷阱」段要寫到位、要用 landscape 查證過的正確版。
- 寧深勿廣，不灌水。每章 5,000~9,000 中文字，依主題深度調整。

工程師對工程師。冷靜、直接、有觀點（可以寫「我認為這是整本書最反直覺的一頁」）。不行銷、不喊口號、不堆「太神奇了」式空洞感嘆——震撼要用內容本身造成，形容詞是結果不是手段。允許冷幽默（機率史充滿賭徒、釀酒廠統計員與被反例打臉的數學家）。數學不裝高深：每個符號出場都有白話直覺在旁邊；但也不假裝簡單——該慢的地方明說「這裡值得你停十分鐘，因為你的直覺正要騙你」。
