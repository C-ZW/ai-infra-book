# ch17 — 估計與信賴區間：誠實地說「我不確定多少」

> **本章解決什麼問題**：上一章（見 ch16）說了統計推論最大膽的事——只問 1000 人就敢推斷兩億人。但「敢推斷」之後馬上要面對一個誠實問題：你猜的這個數字，到底準不準？差多少？這章把「不確定」本身量化成一個可驗收的區間，就是信賴區間（confidence interval）。它是統計學送給工程師最實用的禮物，也藏著全書最招牌的一個誤讀——連 Milton Friedman 這種等級的人都當場踩進去過。下一章（見 ch18）的假設檢定，其實是同一套機器的另一個按法。

## 從你已知的出發

你報過無數次效能數字。「這個 API P50 延遲 50ms」「壓測跑出來吞吐量大概 8000 RPS」「這次最佳化把 RDS CPU 尖峰降了大約 40%」。

注意你每次講這種數字時，喉嚨裡都會卡一下，然後補一句修飾語。「**大概** 8000」「**降了大約** 40%」。那個「大概」「大約」是哪來的？

是因為你心裡清楚：你不是量了全宇宙所有請求，你只是跑了一段壓測、抽了一段時間的樣本。你報的 50ms 是**樣本**算出來的，真正的母體（這支 API 在所有可能負載下的真實延遲分布）你永遠看不到全貌。你的 50ms 是個猜測——一個有根據的猜測，但是猜測。

成熟的工程師會做一件事：他不只報那個點，他會給一個範圍。「平均延遲 50ms，誤差大概 ±3ms。」這句話比光講 50ms 誠實得多，因為它同時回答了兩件事：我猜是 50，而且我知道我能猜得多準。

恭喜，你已經在報信賴區間了。你只是沒人教你它叫這個名字，也沒人告訴你那個「±3ms」背後其實有一套精確到反直覺的數學在撐。

這章要做的就是把你工程直覺裡那個「±」變成嚴格的東西：它怎麼算出來、為什麼是這個寬度、增加樣本能把它縮多少（劇透：跟你「多跑幾次測試降噪」用的是同一條 √n 定律），以及最重要的——**這個區間到底在向你保證什麼**。最後這點，我認為是整本統計裡最多人講錯、而且講錯了還理直氣壯的一頁。

## 點估計：先給一個數，再承認它會晃

統計推論分兩步。第一步叫**點估計（point estimation）**：用樣本算出一個單一數字，當作對母體參數的猜測。

兩個你會反覆用到的點估計：

- **樣本平均（sample mean）** x̄ = (x₁ + x₂ + … + xₙ) / n，拿來估母體平均 μ。
- **樣本比例（sample proportion）** p̂ = (樣本裡符合條件的個數) / n，拿來估母體比例 p。

`p̂` 唸作「p-hat」，那個帽子 ^ 是統計學的慣例：戴帽子的是「估計出來的」，沒帽子的是「真實但你看不到的」。p 是真相，p̂ 是你的猜測。這個區分整章都很重要，後面那個招牌誤讀就是栽在「把帽子戴錯頭上」。

點估計本身有個誠實的麻煩：**它幾乎一定不等於真值**。

你從母體隨機抽 1000 人問支持率，算出 p̂ = 0.52。換一組 1000 人，可能算出 0.49。再換一組，0.51。p̂ 自己就是個隨機變數（random variable）——你每抽一次樣本，它就跳一個值。這正是上一章（見 ch16）抽樣分布（sampling distribution）的核心：統計量本身在抖。

所以光報一個點估計是不夠誠實的。50ms 這個數字看起來斬釘截鐵，但它底下是會晃的。問題是——晃多大？

### 標準誤：點估計自己的標準差

「晃多大」這件事，有名字，叫**標準誤（standard error，SE）**。

標準誤就是「你的點估計這個隨機變數的標準差」。聽起來繞口，拆開就清楚了：

- 母體裡單一個體有它的標準差 σ（個體之間差多少）。
- 但你報的不是個體，是 n 個的平均。平均比個體穩——穩多少？這就回收 ch09 那條 √n 法則了。

```text
SE(x̄)  = σ / √n          ← 樣本平均的標準誤
SE(p̂)  = √( p(1−p) / n )  ← 樣本比例的標準誤
```

第一條直接是 ch09 的結論：n 個獨立同分布的東西取平均，平均的標準差是個體標準差的 1/√n。平均比個體穩 √n 倍——這不是新東西，是大數法則（見 ch13）和中央極限（見 ch14）一路鋪下來的同一根骨頭。

第二條是比例的版本。比例其實是「0/1 指示變數的平均」——支持就記 1、不支持記 0，p̂ 就是這串 0 跟 1 的樣本平均。而一個機率為 p 的伯努利變數，變異數（variance）是 p(1−p)（這你 ch09 算過硬幣的 1/4，就是 p=1/2 時的 p(1−p)）。套進 σ/√n 就得到上面那條。

停一下，看一眼 p(1−p) 這個東西。它在 p=0.5 時最大（等於 0.25），往兩邊都變小。意思是：**民調最難估的，剛好是五五波的議題**。如果某議題支持率是 99%，你很容易估準（變異數小）；如果是真正的拉鋸戰 50/50，你的估計反而最不穩。這有點反直覺但完全合理——勢均力敵的東西，本來就最難分出勝負。後面算民調 ±3% 時，我們就用 p=0.5 這個「最壞情況」，因為新聞要報一個對所有議題都成立的保守誤差界。

## 信賴區間：把「我有多準」變成一段範圍

有了點估計（中心）和標準誤（晃動幅度），信賴區間的構造就幾乎是它自己長出來的。

中央極限定理（見 ch14）告訴你：當 n 夠大，你的點估計 p̂ 的抽樣分布近似常態（normal），中心在真值 p，標準差是 SE。而常態分布你 ch15 很熟了——它 95% 的質量落在中心 ±1.96 個標準差以內（**注意是 1.96 不是 2**，±2σ 其實是 95.45%，95% 對應的精確值是 1.96σ，這個全書一致）。

所以一個 95% 信賴區間長這樣：

```text
信賴區間 = 點估計 ± 1.96 × 標準誤
         = p̂ ± 1.96 × √( p̂(1−p̂) / n )
```

那個 1.96 是「95% 信賴水準」的臨界值。換信賴水準就換數字：

```text
信賴水準    臨界值
  90%        1.645
  95%        1.96     ← 最常用，民調與論文的預設
  99%        2.576
```

水準越高，臨界值越大，區間越寬。這是個誠實的取捨：你要越有把握蓋住真值，就得張開越大的網。要 99% 把握，網就得撐到 ±2.576 個標準誤。**沒有又窄又保險的午餐**——想兼得，唯一的路是增加 n（把標準誤本身縮小），這就是下一節的 √n。

注意一個容易混的地方：那個 ±1.96·SE 的整段寬度，正式叫**誤差界（margin of error）**，新聞講的「±3%」就是它。它是「半寬」，從中心到邊緣的距離。

### Worked example：民調 ±3% 的完整來歷（基準）

這是全書基準範例，每一步都手算給你看，不跳。

**設定**：全國民調，隨機抽 n = 1000 人，問某個五五波議題，假設真實支持率 p ≈ 0.5（最壞情況，給最大誤差界）。求 95% 信賴區間的半寬（誤差界）。

第一步，算標準誤裡面的變異數部分 p(1−p)：

```text
p(1−p) = 0.5 × (1 − 0.5)
       = 0.5 × 0.5
       = 0.25                  ← 五五波，變異數最大
```

第二步，除以 n：

```text
p(1−p) / n = 0.25 / 1000
           = 0.00025
```

第三步，開根號得標準誤：

```text
SE = √0.00025
   = 0.0158113...             ← 約 0.0158
```

這個 0.0158 怎麼來的，心算驗一下：0.0158² = 0.00024964 ≈ 0.00025，對得上。（或者：√0.25 = 0.5，√1000 ≈ 31.62，0.5 / 31.62 ≈ 0.01581，兩條路同一個數。）

第四步，乘上 1.96 得誤差界：

```text
誤差界 = 1.96 × 0.0158113
       = 0.030990...
       ≈ 0.0310
       ≈ ±3.1%
```

**交叉驗算**（這種數字必須複核）：用另一條路。1.96 × 0.0158 ——拆成 2 × 0.0158 = 0.0316，再減掉 0.04 × 0.0158 ≈ 0.00063，得 0.0316 − 0.00063 ≈ 0.03097。對得上 0.0310。

所以這支民調的 95% 信賴區間是 **52% ± 3.1%**，也就是大約 **[49%, 55%]**（如果 p̂ 抽出來是 0.52）。新聞標題說的「誤差 ±3%」，這 1000 個人、這 1.96、這 √(0.25/1000)，就是它全部的來歷。沒有黑魔法。

這裡藏著上一章那個震撼的數字版：**這個 ±3.1% 跟全國有兩億人還是兩百萬人，一點關係都沒有**。公式裡只有 n（樣本數），沒有 N（母體數）。母體只要「夠大到抽樣可視為獨立」，它有多大就無關了。誤差只跟你問了幾個人有關。這是真的，而且每次想到還是會愣一下。

### √n：誤差砍半的代價是樣本變四倍

現在你想把民調做精細點，誤差從 ±3.1% 壓到 ±1.55%（砍半）。要多少樣本？

直覺可能會說「樣本加倍吧」。錯。看公式：

```text
誤差界 = 1.96 × √( p(1−p) / n )
```

n 在根號**底下**。誤差跟 1/√n 成正比，不是跟 1/n。要把誤差砍半，得讓 √n 變兩倍，也就是 **n 變四倍**。

驗算一下，把 n 從 1000 提到 4000：

```text
SE(4000) = √( 0.25 / 4000 )
         = √0.0000625
         = 0.0079057...        ← 約 0.0079

誤差界   = 1.96 × 0.0079057
         = 0.015495...
         ≈ ±1.55%              ← 剛好砍半
```

對得上。1000 → 4000（四倍樣本）換來誤差 ±3.1% → ±1.55%（砍半）。

這條 √n 你其實天天在用，只是沒掛這名字。「這個 benchmark 噪音太大，多跑幾次取平均把誤差壓下來」——你壓的就是這個標準誤。跑 4 次平均，噪音降一半；想再降一半，得跑 16 次；再一半，64 次。**降噪的邊際成本是平方往上爬的**。

這也順手解釋了一個你可能困惑過的現象：為什麼民調公司不乾脆問十萬人、把誤差弄到 ±0.3%？因為從 ±3% 到 ±0.3%（降到十分之一），樣本要變一百倍——從 1000 變成 10 萬，成本爆炸，而換來的精度對「誰會贏」這種問題根本是過度精確。√n 讓「再準一點」變得越來越貴，所以大家停在「夠用就好」的 ±3%。工程上你也是這樣停的：壓測跑到誤差夠小能下決策就收手，不會為了第五位有效數字無限跑下去。

### Worked example：脊椎硬幣 60/100 的信賴區間（埋 ch18 伏筆）

回到全書脊椎——**這是 ch01 那枚硬幣**。

你拿一枚硬幣，懷疑它不公正。擲 100 次，出現 60 次正面。p̂ = 0.60。這枚硬幣真正的正面機率 p 是多少？給個 95% 信賴區間。

用觀察到的 p̂ = 0.60 算標準誤（這裡 p 未知，標準誤就用 p̂ 代入——這叫 Wald 區間，大樣本下夠用）：

```text
SE = √( p̂(1−p̂) / n )
   = √( 0.60 × 0.40 / 100 )
   = √( 0.24 / 100 )
   = √0.0024
   = 0.04899...              ← 約 0.049
```

驗一下：0.049² = 0.002401 ≈ 0.0024，對。

誤差界：

```text
1.96 × 0.049 = 0.09604 ≈ 0.096   ← 約 ±9.6%
```

所以 95% 信賴區間是：

```text
0.60 ± 0.096  =  [0.504, 0.696]
```

讀這個區間：你對這枚硬幣真實正面機率的估計範圍是大約 **[50.4%, 69.6%]**。

現在看那個下界——**0.504，剛剛好擦過 0.5**。一枚公正硬幣的 p 是 0.5，而 0.5 幾乎、但沒有完全落在你的 95% 信賴區間裡（差那麼一點點，0.5 < 0.504）。

這個「差一點點」不是巧合，它就是下一章（見 ch18）假設檢定的伏筆。「0.5 在不在 95% 信賴區間裡」這個問題，跟「在 0.05 顯著水準下能不能拒絕『硬幣公正』」這個問題，是同一枚硬幣的同一件事換句話說。60/100 這個資料剛好卡在邊界附近——公正與不公正的天平上，它落在一個讓你不太敢下結論的位置。ch18 會把這個「卡在邊界」算到小數點後面，你會看到它怎麼隨著不同算法在 0.05 兩側微微搖晃。先記住這個 [0.504, 0.696]，下一章我們回來算帳。

## 信賴區間到底在說什麼（全書招牌誤讀，停十分鐘）

好，前面都是熱身。現在這一節是整章、可能是整本書最重要的一頁。**你的直覺正準備騙你，而且騙得理直氣壯。**

你算出了一個 95% 信賴區間 [49%, 55%]。請你用一句話說它是什麼意思。

幾乎每個人——包括念過統計的人、包括用了一輩子信賴區間的科學家——脫口而出的是這句：

> 「真值有 95% 的機率落在 [49%, 55%] 這個區間裡。」

**這句話是錯的。** 不是有點不精確，是根本性地錯。而且它錯的方式，恰恰暴露了整套頻率派（frequentist）統計最反直覺、最了不起的設計。

錯在哪？錯在一個字：**機率是給隨機的東西用的，但這句話裡沒有隨機的東西了。**

拆給你看。這句話講的是「真值落在 [49%, 55%] 裡的機率」。但是：

1. **真值不是隨機的**。那個真正的支持率 p，是一個固定的、未知的常數。它就是某個確切的數字（比如說 0.518），只是你不知道而已。它不會今天 0.51、明天 0.53 地跳——它一直是那個值。它不是隨機變數，它沒有機率分布。

2. **區間 [49%, 55%] 也不再隨機了**。一旦你抽完樣、算完，這兩個端點就是兩個死掉的數字，釘在那裡。

所以「真值落在這個區間裡」這件事，**要嘛是真（真值確實在裡面），要嘛是假（不在）**。它不是機率 95%，它是機率 1 或機率 0——只是你不知道是哪個。就像「第 7 億位數的圓周率是不是 3」——答案是確定的，你只是還沒查。你不會說「圓周率第 7 億位是 3 的機率是 10%」，那個數字要嘛是 3 要嘛不是。

那 95% 是什麼意思？它**不是**在描述「這一條」區間。它在描述**製造區間的這套程序**：

> **正確版**：「如果我用同一套方法，重複抽樣、重複造區間，做非常多次，那麼長期下來，我造出的這些區間裡會有 95% 蓋住真值。」

95% 是**程序的長期命中率**，不是「這一條」的機率。隨機的是「抽樣這個動作」和「因此產生的區間」，不是真值。每次抽樣，區間都跳到不同位置（因為樣本不同），而那個固定不動的真值，被這些跳來跳去的區間蓋住的比例，長期是 95%。

我知道這個區別聽起來像在玩文字遊戲。它不是。我再換個畫面講一次，因為這頁值得你真的停下來。

### 用「拋圈圈」重講一次

想像一個固定的木樁釘在地上（這是**真值**，不動）。你閉著眼睛朝它拋圈圈，每個圈圈落在不同位置（這是每次抽樣造出的**區間**，會晃）。

「95% 信賴」的意思是：**你這個拋圈圈的手藝，長期有 95% 的圈圈會套中木樁。**

現在你拋出了一個圈圈，它已經落地了，停在那裡。問：「這個圈圈套中木樁的機率是多少？」

答案不是 95%。答案是——它要嘛套中了，要嘛沒套中。圈圈和木樁都不動了，沒有任何隨機的東西，去問「機率」是沒意義的。95% 是在描述你的**手藝**（程序），不是描述**這一拋**的結果。

```text
木樁固定 (真值 p)，圈圈會晃 (每次抽樣的區間)：
木樁 (真值) 永遠固定在下面這條虛線標出的位置、不動

  拋 1   [····●····]      套中  ✓
  拋 2      [····●····]   套中  ✓
  拋 3  [···●···]         套中  ✓
  拋 4           [··●··]  沒中  ✗   ← 這次運氣差，圈圈飄了
  拋 5    [····●····]     套中  ✓
  ......................................
            真值在這 (固定不動的木樁)

長期看：95% 的圈圈會套中木樁(=程序的命中率)
但「拋 4 套中的機率」這問法本身就壞掉了——它要嘛中要嘛沒中
```

你算出來的那一條 [49%, 55%]，就是「拋 4」或「拋 1」其中之一——但你**不知道是哪一個**，因為你看不到木樁在哪。你唯一知道的是：產出它的那隻手，長期命中率 95%。所以你選擇相信這一條也中了——但這是「信任程序」，不是「這一條有 95% 機率」。

### 這不是學究的吹毛求疵——Friedman 當場踩進去

這個誤讀有正式名字，叫**基本信賴謬誤（fundamental confidence fallacy）**，而且它跟信賴區間這個概念一樣老。

**奈曼（Jerzy Neyman）1937 年**正式定義了信賴區間（confidence interval）。就在他發表這套理論的場合——一場在美國農業部的演講——台下一位年輕經濟學家舉手提問，問的正是「那是不是表示真值有 95% 機率落在這區間裡？」奈曼當場糾正他：不，真值不是隨機變數，它是個未知的常數（2026-06，多源確認此軼事，細節依 landscape）。

那位被糾正的年輕人，叫 **Milton Friedman**——後來拿諾貝爾經濟學獎的那位。

我講這段不是要嘲笑誰。剛好相反：如果連 Friedman 這種等級的腦袋，第一次碰到都會直覺地踩進去，那這個誤讀就絕對不是「你不夠聰明」的問題。它是人類直覺對「機率」這個詞的預設用法跟頻率派的精確定義之間，一道結構性的裂縫。奈曼設計這套程序時，是**刻意**把隨機性放在「抽樣」上、把參數釘成常數的——這個選擇讓整套理論邏輯乾淨，代價就是它說出來的保證，永遠不是你直覺想聽的那句。

> **這裡了不起在哪**（能轉述給另一個工程師的版本）：頻率派做了一件很硬的事——它**拒絕**對「這一條具體的區間」說任何機率的話，因為一旦資料定了，這條區間是死的，沒有機率可言。它只願意對「程序」做保證。這很反人性（你就想知道「我手上這條到底準不準」啊），但它換來的是**不需要任何主觀先驗、純靠抽樣的隨機性就能給出鐵一般的長期保證**。你只要老老實實照程序做，長期 95% 命中，這個數字不靠任何人的信念、不靠任何先驗假設。這是它的紀律，也是它的驕傲。

### 給讀過 TLA+ 那本書的你：這是量詞順序

如果你讀過書架上《把系統寫成定理》（TLA+ 那本），這裡的精確意義你其實見過，只是換了衣服。

兩句話的差別，本質是**量詞順序**的差別：

- **錯誤讀法**：「給定這條算出來的區間，真值在裡面的機率 95%。」這是想對「∃ 出來的那一條具體區間」談機率。
- **正確讀法**：「對所有可能的真值（∀ p），這套程序產生的隨機區間蓋住它的機率是 95%。」機率是綁在「程序／抽樣」這個 ∀ 上的，不是綁在「這一條」上的。

如果你被 TLA+ 的 ∀∃ 攻防虐過，這個「保證是綁在哪個量詞上」的敏感度你已經有了。沒讀過也完全不影響——拋圈圈那張圖就夠了。

### 那「真值有 95% 機率在內」這句話，有沒有對的時候？

有。但你得換一套世界觀——換成**貝氏（Bayesian）**。

貝氏統計有個對應的東西，叫**可信區間（credible interval）**。貝氏的世界觀（回收 ch06）裡，未知參數**本身**就被當成隨機變數來對待——它有一個機率分布（後驗分布，posterior），描述「在看完資料後，你對這個參數的信念」。在這套框架下，你**真的可以**說：「給定我看到的資料，真值有 95% 機率落在這個可信區間裡。」那句被頻率派打叉的話，在貝氏這裡是合法的。

聽起來貝氏完勝？它給了你直覺想要的那句話。但天下沒有白吃的午餐——**代價是你得先掏出一個先驗（prior）**。可信區間要算，前提是你得先說「在看資料之前，你對這個參數的信念分布長什麼樣」。換不同先驗，算出來的可信區間就不同。頻率派的信賴區間不需要先驗（這是它的賣點），代價是它說不出那句直覺的話；貝氏的可信區間能說那句話，代價是它得先承諾一個先驗。

```text
                 信賴區間 (頻率派)       可信區間 (貝氏)
真值是什麼        固定的未知常數          有分布的隨機變數
隨機的是          抽樣/區間               你的信念(後驗)
95% 在說          程序的長期命中率        這一條的信念機率
「真值95%機率在內」 ✗ 錯                  ✓ 對
需要先驗嗎         不需要                 需要(這是代價)
```

兩個都對，只是回答的是不同問題、站在不同世界觀上。本書到這裡只點到為止——貝氏統計的完整方法（MCMC、共軛先驗那些）不在這本書的範圍，留給延伸閱讀。你只要記住：當有人理直氣壯說「真值有 95% 機率在這個信賴區間裡」，他要嘛講錯了，要嘛他其實在講一個貝氏可信區間卻沒發現自己偷換了世界觀（還偷偷塞了個先驗）。

## 一句話：那 t 分布是什麼

前面所有計算都偷偷用了一個假設：**樣本夠大**，所以中央極限定理生效、抽樣分布近似常態、可以用 1.96。

那樣本很小怎麼辦？比如你只跑了 8 次壓測。這時常態近似不夠好，而且還有個麻煩：標準誤公式裡的母體標準差 σ 你不知道，得用樣本自己估，這個「估出來的標準差」在小樣本下自己也很不穩，會讓區間比常態算的更寬一點才誠實。

解這個問題的分布叫 **t 分布（t-distribution）**，形狀像常態但尾巴更胖（反映「小樣本下我對自己的標準差估計也沒把握」這份額外的不確定）。它的故事很合這本書的調性：1908 年，一個在都柏林 Guinness 釀酒廠搞品管的人叫 **William Gosset**，天天面對小樣本（一批大麥、幾桶酒，常常只有十來個數據點），大樣本理論對他沒用。他推導出了這個小樣本的分布。但 Guinness 禁止員工具名發表（怕洩漏自家用統計做品管的商業機密），所以他用筆名「**Student**」投稿——這就是為什麼直到今天，全世界教科書上那個分布還叫「Student's t」，一個釀酒廠員工的化名（2026-06，多源確認；細節依 landscape）。

本書對 t 分布只點到這裡。樣本一大（幾十以上），t 分布就趨近常態，1.96 又回來了，你前面學的全部適用。完整的 t 分布理論不在這本書的範圍。你只要知道：**小樣本時區間要更寬，而把「更寬多少」算對的那個分布，是個釀酒廠員工用化名發表的**。機率統計史就是這麼多賭徒、釀酒員和被反例打臉的數學家。

## 直覺的陷阱

信賴區間是人類直覺崩壞的重災區。把這些釘在牆上：

| 你以為 | 其實 | 怎麼自我察覺 |
|---|---|---|
| 真值有 95% 機率落在這條區間裡 | 真值是常數、區間已固定，這一條要嘛中要嘛沒中。95% 是**程序**的長期命中率，不是這一條的機率 | 一聽到自己對「**這一條**算出來的區間」講「機率」，警報就該響——資料定了之後沒有隨機的東西可言 |
| 95% 信賴＝有 95% 的資料點落在區間內 | 完全是兩回事。信賴區間講的是**參數（如平均）**在哪，不是個別資料點散布的範圍。資料散布範圍叫預測區間，是另一種東西，通常寬得多 | 你發現自己把「平均的不確定」和「個體的離散」混為一談時 |
| ±2σ 就是 95% | 95% 對應的是 **±1.96σ**；±2σ 其實是 95.45%。差一點點但別寫錯 | 看到有人寫「2 個標準差 = 95%」，那是四捨五入後的口語，精確值是 1.96 |
| 兩個信賴區間有重疊＝兩組沒有顯著差異 | 不一定。區間重疊不能直接推論差異不顯著，要看的是「差值」的信賴區間，不是兩條各自的區間重不重疊 | 你想用「目測兩條 bar 的誤差線有沒有碰到」下結論時 |
| 增加樣本能讓區間想多窄有多窄、線性地窄 | 誤差跟 1/√n，不是 1/n。砍半要四倍樣本，再砍半要十六倍——越往後越貴 | 你以為「樣本加倍誤差就減半」時 |
| 信賴區間保證涵蓋了真值 | 它只保證**長期 95% 的區間**涵蓋。你手上這條，有 5% 的機會（指程序層面）根本沒蓋到真值，你還渾然不知 | 你把「95% 信賴」當成「幾乎確定」時——5% 不是 0，二十條裡有一條會落空 |

最後補一個常被忽略的致命前提：**這一切的前提是隨機抽樣**。如果你的樣本本身有偏（見 ch16 的 1936 Literary Digest 翻車），那信賴區間算得再漂亮都是垃圾——它只量「抽樣的隨機波動」這一種誤差，完全不量「你抽錯人」這種系統性偏誤。一個算到小數點後三位的窄區間，如果樣本是歪的，它只是「精確地錯」。誤差界小，不代表答案對。

## 紙上推演

### 推演題 1 — 四種讀法，只有一條對 **[15 分鐘] ★★**

某民調報告：候選人 A 的支持率 95% 信賴區間是 [47%, 53%]。下面四種讀法，哪一條對？另外三條各錯在哪？

1. 「A 的真實支持率有 95% 的機率落在 47% 到 53% 之間。」
2. 「如果用同樣方法重複很多次民調，約 95% 的民調算出的區間會蓋住 A 的真實支持率。」
3. 「95% 的受訪者支持率落在 47% 到 53% 之間。」
4. 「A 的真實支持率有 95% 的機率正好是 50%（區間中點）。」

口頭講出每條的判決與理由。

#### 推演解答

**只有第 2 條對。** 它正確地把 95% 綁在「程序／重複抽樣」上，講的是區間的長期命中率，不是某一條的機率。

第 1 條——**全章的招牌錯誤**。真實支持率是固定的未知常數，不是隨機變數；[47%, 53%] 這條已經算出來、固定了。所以「真值在這條裡」這件事要嘛真要嘛假（機率 1 或 0），談「95% 機率」沒意義。隨機的是抽樣這個動作，不是真值。要能合法講這句話，你得換成貝氏可信區間，而且得先掏一個先驗出來。

第 3 條——**把參數的不確定和資料的散布搞混了**。信賴區間講的是「A 的支持率（一個母體比例）」這個參數在哪，根本不是在描述受訪者怎麼散布。受訪者每個人不是「支持 47%」這種連續值，他要嘛支持要嘛不支持。這條根本是雞同鴨講。

第 4 條——**把中點當成真值的最可能單點還加上機率**。50% 只是這次的點估計，它幾乎一定不等於真值；而且對「真值正好等於某個單點」談 95% 機率，同樣犯了把常數當隨機變數的錯（連續量取到任一單點的機率還是 0，雙重錯誤）。

**常見錯路**：很多人會覺得第 1 和第 2 「不就一樣意思換句話說嗎」。不一樣。差別正是「機率綁在真值上（錯）還是綁在程序上（對）」。這個區別細到像在鑽牛角尖，但它是整套頻率派統計的地基——地基歪了，後面 ch18 的 p 值你會跟著一起誤讀。

### 推演題 2 — 把誤差砍半要多少樣本 **[10 分鐘] ★★**

某民調 n = 1000、誤差界 ±3.1%。老闆說：「我要誤差降到 ±1%。」需要多少樣本？先別急著算，先用 √n 直覺估，再驗算。

#### 推演解答

**直覺估**：誤差跟 1/√n。從 ±3.1% 到 ±1%，誤差要縮到約 1/3.1 ≈ 0.323 倍。誤差縮到 k 倍，n 要變 1/k² 倍。所以 n 要變約 (3.1)² ≈ 9.6 倍——大約 9600 人。

**驗算**：要的誤差界 0.01 = 1.96 × √(0.25/n)。

```text
0.01 / 1.96 = √(0.25 / n)
0.005102    = √(0.25 / n)
0.005102²   = 0.25 / n
0.00002603  = 0.25 / n
n           = 0.25 / 0.00002603
            ≈ 9604
```

約 **9604 人**。跟直覺估的 9600 對得上。

**討論**：從 1000 到 9604——將近十倍的樣本，只換來誤差從 ±3.1% 到 ±1%。這就是 √n 的殘酷：精度的邊際成本平方往上爬。也是為什麼「再準一點」在實務上經常不值得——多花十倍的錢問十倍的人，得到的「更確定」對大多數決策根本沒差。你壓測時也是同一個取捨：跑到誤差夠小能拍板就停，不會為了第五位有效數字無限燒機器。

### 推演題 3 — 脊椎硬幣再擲一輪 **[15 分鐘] ★★★**

**這是 ch01 那枚硬幣**。前面我們擲 100 次得 60 正，95% 信賴區間是 [0.504, 0.696]，下界擦過 0.5。

現在假設你不死心，又擲了 900 次，這 1000 次裡總共 600 次正面（比例還是 60%）。重算 95% 信賴區間。比例一樣是 60%，但結論的「力道」變了嗎？

#### 推演解答

p̂ 還是 0.60，但 n 從 100 變 1000。標準誤：

```text
SE = √( 0.60 × 0.40 / 1000 )
   = √( 0.24 / 1000 )
   = √0.00024
   = 0.015492...           ← 約 0.0155
```

誤差界：

```text
1.96 × 0.0155 = 0.03037 ≈ 0.030   ← ±3.0%
```

信賴區間：

```text
0.60 ± 0.030 = [0.570, 0.630]
```

**對照**：

```text
            p̂      區間                  0.5 在區間內？
60/100     0.60   [0.504, 0.696]        勉強擦邊在外(0.5 < 0.504)
600/1000   0.60   [0.570, 0.630]        0.5 遠在區間外(差了 7 個百分點)
```

**同樣 60% 的比例，結論的力道天差地別。** 100 次那組，0.5 幾乎擦著下界——你很難理直氣壯說「這硬幣不公正」，因為公正（0.5）就在門檻邊上晃。但 1000 次這組，0.5 被甩出區間整整 7 個百分點之外，區間窄到 [0.570, 0.630]，公正硬幣的世界（p=0.5）離這個區間遠得很。同樣的偏離比例，n 大十倍，你的把握強得多。

**埋的伏筆**：「0.5 在不在 95% 信賴區間裡」這個問題，下一章（見 ch18）會用假設檢定（hypothesis testing）的語言重講一遍——「在 0.05 顯著水準下能不能拒絕『硬幣公正』」。60/100 那組剛好卡在 0.05 的兩側微微搖晃（換個算法就跨過門檻），而 600/1000 那組會給出一個小到不行的 p 值。**比例不變、樣本變大，顯著性被放大**——這是 ch18 最重要的一課，而你現在已經從信賴區間這一側先看到它的影子了。同一枚硬幣，又懂了一層。

## 自我檢核

口頭自答，講不清楚就是還沒懂：

1. 標準誤是什麼的標準差？它跟「母體個體的標準差」差在哪？為什麼差了一個 √n？
2. 民調 ±3% 那個 3% 是怎麼算出來的？為什麼算的時候用 p=0.5？為什麼它跟全國有兩億人無關？
3. 「真值有 95% 機率落在這個信賴區間裡」這句話錯在哪？用「拋圈圈套木樁」講給一個堅信這句話的同事聽，他要怎樣才會點頭？
4. 95% 到底是在描述什麼東西的什麼性質？（提示：不是「這一條」，是某個更大的東西。）
5. 把誤差砍半要付出什麼代價？為什麼是這個代價而不是「樣本加倍」？這跟你「多跑幾次 benchmark 降噪」是同一條定律嗎？
6. 貝氏的可信區間為什麼**可以**合法地說「真值有 95% 機率在內」？它付出的代價是什麼？
7. 一個算到 ±0.5% 的超窄信賴區間，有沒有可能整個是錯的？什麼情況下會這樣？（提示：區間只量哪一種誤差、不量哪一種？）
8. 60/100 和 600/1000 同樣是 60% 正面，為什麼後者讓你對「硬幣不公正」有把握得多？

## 延伸閱讀

- **奈曼信賴區間誤讀**：Hoekstra et al.「Robust misinterpretation of confidence intervals」（2014, Psychonomic Bulletin & Review）——實測學生**與研究者**都系統性誤讀信賴區間的經典研究，看你不孤單。配 PMC 上「Continued misinterpretation of confidence intervals」（landscape 引的那篇）一起讀，把「為什麼這麼難改正」講透。
- **Neyman 1937 原始定義**：Wikipedia「Neyman construction」條目——想看信賴區間怎麼從「對所有真值都成立的程序」這個量詞結構長出來的，這條把建構講得最乾淨；也有 Friedman 軼事的脈絡。
- **Gosset 的故事**：The Conversation「The genius at Guinness and his statistical legacy」——釀酒廠、化名 Student、小樣本問題的來龍去脈，讀它配一杯黑啤最對味。
- **可信區間 vs 信賴區間**：Wikipedia「Credible interval」——想把貝氏那一側（「真值 95% 機率在內」為什麼在貝氏合法、代價是先驗）補完，從這條入門，再跳延伸的貝氏方法。
- **回到本書**：信賴區間是 ch16 抽樣大膽的「誠實度量」，是 ch18 假設檢定的另一面（同一套機器、不同按法）。讀完 ch18 再回來看本章脊椎硬幣那個 [0.504, 0.696]，你會發現它跟 p≈0.05 是同一件事。
