## 從你已知的出發

你其實一輩子沒「看過全部」過。

你估一支 API 的錯誤率，不是把這個月的幾億筆 log 一行行讀完。你開 sampling，每 100 筆抽 1 筆，或乾脆只看過去一小時。你算出 0.3%，就拿這 0.3% 當「整體的錯誤率」在報、設 alert 閾值、跟 PM 解釋。你篤定到根本不覺得這是個大膽的舉動。

容量規劃也是。你不可能真的把雙十一全量打給線上系統試試看——那叫事故不叫壓測。你開壓測環境灌一段**有限**的合成流量，看 P99 在哪裡崩，就敢簽「這套系統撐得住 X QPS」。拿一個親眼看過的小場景，推斷一個沒看過、賭不起的大場景。

金絲雀部署更直接。新版本先放 1% 流量進去——而且最好是**隨機**挑的 1%，不是「先給內部員工」也不是「先給某區域」。看那 1% 有沒有變糟，沒事再放大到 100%。你等於是說：這隨機的 1% 怎麼樣，全量大概就會怎麼樣。

這三件事——抽樣監控、壓測、金絲雀——背後是同一個信念，而這就是這一章的全部：**你可以只看一小部分，就對沒看過的整體下一個有誠實誤差界的結論，前提是那一小部分是隨機抽的。** 統計學把這個動作正式化、把「誤差界」算出來，並告訴你它什麼時候會整碗端走騙你。

這一章我們把這件你天天做的事翻過來看它的內臟。三個問題：母體跟樣本到底什麼關係、為什麼「隨機」才是魔法來源（而不是「樣本要長得像母體」）、還有全章最反直覺的一句話——**你問 1000 個人估全美國，跟問 1000 個人估你家社區，誤差幾乎一樣大。** 人口是兩億還是兩萬，幾乎不影響你需要的樣本數。第一次聽到的人沒有不皺眉的，我們會把它算到你心服口服。

## 母體與樣本：你想知道的，和你看得到的

先把四個詞釘死，因為接下來整個 Part V 都在這四個詞之間繞。

**母體（population）**：你真正想知道的那個整體。全美國的選民、這個月所有的 API 請求、線上系統能承受的全部負載情境。它通常大到你看不完，有時候根本是個抽象的「所有可能發生的請求」——理論上無窮多。

**樣本（sample）**：你實際拿到手、看得見的那一小撮。1000 個受訪者、抽出來的那 10000 筆 log、壓測打進去的那幾百萬個請求。

**參數（parameter）**：母體的某個你想知道的數字。全美國支持某候選人的**真實**比例 p、這個月**真實**的整體錯誤率。參數是**固定的常數**——它就是那個值，只是你不知道。這點之後會反覆咬你，先記著：真值不是隨機的，它只是未知。

**統計量（statistic）**：你從樣本算出來、用來猜參數的數字。樣本裡的支持比例 p̂（讀作 p-hat，全書統一這個寫法）、樣本裡算出的錯誤率。統計量是你**看得到**的，但它是**隨機的**——換一批樣本就換一個值。

```text
母體（你想知道，看不完）          樣本（你看得到，一小撮）
   參數 p（固定常數，未知）  ──抽樣──►  統計量 p̂（隨機，算得出）
   「真實支持率到底是多少」        「這 1000 人裡支持的比例」
                          ◄──推斷──
                          （拿 p̂ 反猜 p，這是 ch17 的事）
```

整個統計推論就是這張圖右下那條虛線箭頭：**拿你看得到的 p̂，反過來猜你看不到的 p。** ch13–15 走的是上面那條實線箭頭（已知 p，問 p̂ 會怎樣）；從這一章起我們全程逆流而上。

這裡有個容易滑過去、卻是整章樞紐的觀念，我要單獨拉出來講：**p̂ 是一個隨機變數。**

你算出來的「這 1000 人裡 52% 支持」，那個 52% 不是天上掉下來的鐵板一塊。它取決於你**剛好**抽到哪 1000 個人——換一組 1000 人，你會算出 51%、53%、或 49.5%。p̂ 每次都不一樣，它在跳。一個每次呼叫回傳不同值、但有固定行為剖面的東西——這不就是 ch07 講的隨機變數嗎？對，p̂ 就是一個隨機變數。

於是一個關鍵問題冒出來：p̂ 這個隨機變數，它的分布長什麼樣？它平均會落在哪、會抖多大？這個分布有個專門的名字，是整個統計推論的引擎。

## 抽樣分布：統計量自己也是一個隨機變數

**抽樣分布（sampling distribution）**：一個統計量（比如 p̂）在「重複抽樣」之下的機率分布。

這個概念第一次碰到會有點暈，因為它套了一層。我把它拆成一個想像實驗，用脊椎硬幣來做——**對，就是 ch01 那枚公正硬幣**，真實正面機率 p = 0.5，我們現在假裝不知道，要靠抽樣去估它。

想像你這樣做：

```text
第 1 把：擲 100 次硬幣，數正面 → 比如 53 個正面，p̂₁ = 0.53
第 2 把：再擲 100 次，重新數      → 比如 48 個正面，p̂₂ = 0.48
第 3 把：再擲 100 次              → 比如 51 個正面，p̂₃ = 0.51
   ⋮      （每一把都是一個全新的 n=100 樣本）
第 k 把：⋯
```

每一把給你一個 p̂。把這成千上萬個 p̂ 全部畫成一張直方圖——**這張直方圖就是 p̂ 的抽樣分布。** 它不是某一次的結果，是「如果你重複抽很多很多次，p̂ 會怎麼散開」的全貌。（純想像實驗，不寫程式碼；本書全程紙筆推演，你心裡擲一萬把就好。）

那這張圖長什麼樣？這就是前面四章的伏筆要回收的地方，而且回收得非常乾淨：

**第一，它的中心在哪？** 在 p = 0.5。因為每一把的 p̂ 期望值就是真實的 p（樣本比例平均而言不偏不倚地對準真值，這叫不偏估計，unbiased，ch17 細講）。這是大數法則在背後撐著（見 ch13）：樣本比例會往真實機率收斂。

**第二，它抖多大？** 這就是 ch09 那條 √n 法則（見 ch09）在回收。一次擲硬幣是伯努利隨機變數，變異數 p(1−p) = 0.5×0.5 = 0.25。n 個獨立的平均，標準差會被 √n 稀釋：

```text
p̂ 的標準差 = √( p(1−p) / n )
           = √( 0.25 / 100 )
           = √0.0025
           = 0.05               ← 這個數有個專名：標準誤
```

這個 0.05 太重要了，它有專屬名稱：**標準誤（standard error, SE）**。它就是「統計量這個隨機變數的標準差」——注意它跟一般標準差不是兩種東西，它**就是**標準差，只是特指「某個統計量在重複抽樣下的標準差」。標準誤是整個 Part V 的主角數字。

**第三，它是什麼形狀？** 鐘形。常態。這是中央極限定理在回收（見 ch14）：p̂ 本質上是 100 個 0/1 結果的平均，「大量獨立隨機量的平均」永遠趨向常態，不管原本那些 0/1 長得多不像鐘形。所以：

```text
p̂ ~ N( 0.5 , 0.05² )      ← 中心 0.5、標準誤 0.05 的常態
   = N( 0.5 , 0.0025 )
```

把這個分布畫出來，配上 ch15 的 68-95-99.7 法則（見 ch15），你立刻知道一把 100 次的硬幣抽樣會散成什麼樣：

```text
              p̂ 的抽樣分布  N(0.5, 0.05²)
                       ▒▒▒
                     ▒▒▒▒▒▒▒
                   ▒▒▒▒▒▒▒▒▒▒▒
                 ░░▒▒▒▒▒▒▒▒▒▒▒░░
              ░░░░░▒▒▒▒▒▒▒▒▒▒▒░░░░░
          ────┴────┴────┴────┴────┴────┴────┴──► p̂
            0.40 0.45 0.50 0.55 0.60
              │    │    │    │    │
             −2SE −1SE  p  +1SE +2SE

  ±1 標準誤（0.45 ~ 0.55）：約 68.27% 的樣本落在這
  ±2 標準誤（0.40 ~ 0.60）：約 95.45% 的樣本落在這
```

讀一遍這張圖你就懂了一件很實際的事：用 100 次硬幣去估「正面機率」，你大約有 95% 的把握，算出來的 p̂ 落在真值 ±0.10 之內（0.40 到 0.60）——單把 100 次的估計，誤差可能高達 ±10 個百分點。想更準？√n 法則告訴你怎麼換：n 變大、標準誤縮小，但縮得很慢——這是下一節那個反直覺結論的引擎。

先把這一節的驚嘆點講白：**統計量自己是個隨機變數，而它的分布——落在哪、抖多大、什麼形狀——完全被 ch13–15 那三條定理鎖死了。** 大數法則管中心、√n 管寬度、中央極限管形狀。你不是在黑暗裡瞎猜「我的樣本準不準」，你**精確地知道**自己會錯多少、以什麼形狀錯。這就是統計推論敢這麼大膽的底氣——不是賭運氣好，是它把自己會犯的錯算得清清楚楚。

## 為什麼樣本大小比母體大小重要得多

現在進這一章最反直覺的一頁，我認為這是整個 Part V 最容易讓工程師當場「蛤？」出聲的地方。

問題是這樣的：一份全國民調問 1000 個人，宣稱誤差 ±3%，而美國有大約 2 億選民。直覺上你會想——才問 1000 個就要代表 2 億人？那是二十萬分之一，太囂張了吧。如果母體只有 200 萬（某個城市），問 1000 個是千分之一，**比例上多抽了一百倍**，那準度應該高很多吧？

不對。誤差幾乎一模一樣。我們來算。

抽樣比例 p̂ 的標準誤是：

```text
SE = √( p(1−p) / n )
```

你盯著這條公式看——**裡面有母體大小 N 嗎？沒有。** 只有 n（樣本大小）和 p（比例）。母體有多大，根本沒進到這條式子裡。

民調 ±3% 是怎麼來的（這是全書基準，ch17 完整推）：取 p = 0.5（最保守，這時 p(1−p)=0.25 最大），n = 1000，95% 信賴用 1.96 個標準誤：

```text
SE      = √( 0.25 / 1000 ) = √0.00025 = 0.015811…
半寬   = 1.96 × 0.015811   = 0.030990…  ≈ ±3.1%
```

這個 ±3.1% 就是你在新聞上看到的「±3%」。它只跟 n=1000 有關。全美國 2 億人也好、某城市 200 萬人也好，這條式子算出來都是 ±3.1%。

「可是直覺上比例不一樣啊！」——直覺要的那個「母體大小有差」，其實是有的，它叫**有限母體校正（finite population correction, FPC）**，但它小到幾乎看不見。完整的標準誤是：

```text
SE = √( p(1−p) / n ) × √( (N − n) / (N − 1) )
                        └──── 這就是有限母體校正項 ────┘
```

那個校正項是個小於 1 的數，它會把標準誤稍微壓小一點（樣本佔母體越大、壓越多）。我們把三種母體大小代進去看：

```text
N = 2 億（2×10⁸），n=1000：
   校正項 = √( (2×10⁸ − 1000) / (2×10⁸ − 1) ) = 0.999998
   半寬  = 3.099% × 0.999998 ≈ 3.099%      ← 幾乎沒動

N = 200 萬（2×10⁶），n=1000：
   校正項 = √( (2×10⁶ − 1000) / (2×10⁶ − 1) ) = 0.999750
   半寬  = 3.099% × 0.999750 ≈ 3.098%      ← 還是幾乎沒動

N = 1 萬，n=1000（樣本佔了母體 10%！）：
   校正項 = √( (10⁴ − 1000) / (10⁴ − 1) ) = 0.94873
   半寬  = 3.099% × 0.94873 ≈ 2.940%       ← 這才看得出差別
```

看懂這張表你就懂了整件事。母體從 2 億掉到 200 萬——縮了 100 倍——校正項從 0.999998 變 0.999750，第四位小數才動；直覺以為的「母體小一百倍應該準很多」，一點感覺都沒有。只有當樣本**佔了母體相當大一塊**（這裡 1000 佔 1 萬的 10%），校正項才真的咬下去（0.949），讓誤差小一點點。

> **直覺糾正**：抽樣的準度幾乎只看你**舀了多少水**（樣本大小 n），不看**那鍋湯有多大**（母體大小 N）。除非那一瓢佔了整鍋湯的可觀比例（粗略超過 5%），否則母體大小可以當不存在。

為什麼？煮一大鍋湯要嚐鹹淡，你會**先攪勻**再舀一湯匙嚐——那一湯匙準不準，取決於湯匙多大和有沒有攪勻，跟那鍋湯是一公升還是一噸毫無關係。攪勻，就是隨機抽樣。

這就是統計學最大膽也最美的一件事，現在你能用自己的話講給另一個工程師聽了：**你不需要看完母體，甚至不需要看母體的一個固定比例。1000 個隨機樣本估兩億人，跟估兩萬人，一樣準。** 規模不是問題，攪勻才是問題。這也是抽樣監控划算的原因：流量翻 10 倍，你不用把抽樣量也翻 10 倍——只要絕對抽樣筆數 n 夠大，誤差就夠小（前提一樣是隨機抽，別只抽某台機器、某時段）。

## 隨機的魔法：代表性不靠「長得像」，靠「隨機」

上一節那鍋湯，全部關鍵字在「攪勻」。現在我要把這件事的份量講到位，因為它是整章——甚至整個統計推論——的命脈，而且它違反幾乎所有人的第一直覺。

你問一個沒學過統計的聰明人：「怎麼讓 1000 人代表兩億人？」他十之八九會說：「要挑得有代表性啊。照母體的男女比、年齡比、城鄉比去湊 1000 個人，讓樣本的組成**長得像**母體。」

這個直覺——「代表性 = 樣本長得像母體」——聽起來無懈可擊，但它其實是錯的，至少是危險的。真正讓樣本能代表母體的，**不是它長得像母體，而是它是隨機抽出來的。**

差別在哪？「照比例湊」這個動作，只能照你**想得到、量得到**的那幾個維度去湊——性別、年齡、地區。但母體在無數個你**沒想到、甚至沒法量**的維度上也有結構：政治冷感程度、生活作息、會不會接陌生來電、對某議題的隱性態度⋯⋯你照三個維度湊得再像，剩下那無窮多個維度你完全沒控制。而其中某一個，很可能正好跟你要估的東西高度相關，於是你的「代表性樣本」在那個維度上系統性偏掉，你還渾然不覺——因為你根本沒在看那個維度。

隨機抽樣的魔法就在這裡：**它不挑任何維度，所以它在「每一個」維度上——包括你沒想到的、量不到的全部維度——都自動趨於平衡。** 你不需要知道母體有哪些隱藏結構，隨機化替你一次擺平所有結構。這就是 ch10 那條線的高潮回收（見 ch10）：隨機化斬斷選擇偏誤、斬斷 confounder。你刻意挑選，就是用你的偏好（或無知）給樣本注入結構；你隨機抽，就是拒絕注入任何結構，讓母體的真實結構**按它本來的比例**自己流進樣本。

這也正是 A/B test 為什麼非得隨機分組不可（A/B 與因果的完整故事見 ch19）。你把使用者**隨機**丟進 A 組 B 組，不是因為隨機聽起來比較公平——是因為隨機化讓兩組在**所有**你想得到和想不到的維度上都自動相等（年齡、裝置、網速、心情⋯⋯全部），於是兩組之間**唯一**系統性的差別，就只剩你刻意施加的那個改動。差別只剩一個，因果就可推斷了。你花錢買隨機分組，買到的其實是「因果可推斷」這個極其昂貴的東西——confounder 被隨機化一次清空。

> **這裡了不起在哪**（請你能轉述）：你以為要讓小樣本代表大母體，得很費力地把樣本「設計得像」母體。恰恰相反——你越動手設計，越容易把自己的偏見偷渡進去。真正的代表性來自**放手**：擲骰子決定抽誰、親手放棄所有控制權，反而換來在無窮多個維度上的自動平衡。把控制權交給隨機，是你能對一個不了解其全貌的母體做的最聰明的一件事。

聽起來太好了，好到該起疑。沒錯，代價是：你**真的**得做到隨機，而「真的隨機」比你想的難非常多。下一節兩個真實故事，是這一章的故障視角，也是史上最貴的兩堂統計課。

## 兩場經典翻車：抽樣偏誤怎麼把人活活騙死

### 1936 年：問了兩百四十萬人，還是錯得一塌糊塗

1936 年美國總統大選，羅斯福（Roosevelt）對上共和黨的蘭登（Landon）。當時最權威的民調來自《文學文摘》（*Literary Digest*）——這本雜誌過去幾屆都猜對，信譽卓著。

這次他們玩很大：寄出**大約一千萬份**問卷，回收了**大約兩百四十萬份**。兩百四十萬！這在當時是史上最大規模的民調之一。樣本大成這樣，還能錯到哪去？

他們的預測：蘭登 57%、羅斯福 43%，蘭登大勝。

實際結果：羅斯福 62%、蘭登 38%。羅斯福不是贏，是輾壓——拿下 48 州裡的 46 州。《文學文摘》的預測偏了將近 19 個百分點，是大型民調史上最離譜的一次。這份雜誌的信譽碎了一地，沒幾年就停刊了。（事實照 landscape 與 2026-06 web 查證。）

兩百四十萬份樣本怎麼會錯得這麼慘？這就是上一節那句話的血淋淋反證：**樣本大小救不了你，如果你的抽樣根本不是隨機的。** 《文學文摘》的問卷名單從哪來？汽車登記資料和電話簿。1936 年是經濟大蕭條的尾巴，那個年代**買得起車、裝得起電話的人**，是社會裡相對有錢的一群——而有錢人那年明顯更偏向共和黨的蘭登。

他們的樣本框（sampling frame，你能接觸到的那群人）從根上就偏了。窮人——大蕭條裡更可能投給推新政的羅斯福的那一大群人——根本不在名單上，再大的樣本也抽不到他們。雪上加霜的是**無回應偏誤（non-response bias）**：寄出一千萬、只回兩百四十萬，那願意花力氣填問卷寄回去的兩成多人，本身又是另一層自我篩選，跟沉默的多數系統性不同。

偏誤（bias）和變異（variance）是兩種完全不同的錯，這是工程師最該帶走的一句話。把樣本變大，只能壓低**變異**（√n 法則，樣本越大估計越穩）；它對**偏誤**一點辦法都沒有。偏誤是「你的瞄準鏡裝歪了」——打一槍偏、打一萬槍還是全部往同一邊偏，而且因為彈著點很集中，你會更**自信**地相信那個錯誤的中心。《文學文摘》就是拿著兩百四十萬發子彈，極其精準、極其自信地打在了錯誤的靶心上。

對照組很諷刺：那一年有個叫蓋洛普（George Gallup）的廣告人，只用**大約五萬份**規矩的隨機樣本就猜中羅斯福贏；更秀的是，他事先用一個**只有三千人**、但從《文學文摘》自己名單裡抽出來的樣本，準確預言了《文學文摘》會錯、錯向哪邊。三千個瞄得準的，海電兩百四十萬個瞄歪的。現代民調業就此誕生。（2026-06 web 查證。）

這故事翻成你的語言：**抽樣監控如果只抽某一台機器、某一個時段、某一類請求，那你抽再多筆都是在精確地測量一個偏掉的子集。** dashboard 會給你一個漂亮、穩定、誤差很小的數字，而它系統性地不等於真相。樣本量大只會讓你更篤定地相信一個錯的值——這是監控最陰險的一種謊言，因為它穿著「資料量很大、很可信」的外衣。

### 二戰：你只看得到飛回來的飛機

第二個故事更精巧，它揭露的偏誤連「擴大樣本」都救不了，因為偏誤藏在「你能看到什麼」這件事本身。

二戰期間美軍想幫轟炸機加裝甲。裝甲很重，不能整架包起來，得挑要害補。軍方統計了從歐洲戰場**飛回來**的轟炸機，發現彈孔集中在機翼和機身，引擎和駕駛艙相對乾淨。結論呼之欲出：**彈孔最多的地方最危險，加裝甲在機翼和機身。**

統計研究小組（Statistical Research Group, SRG）的數學家亞伯拉罕·瓦德（Abraham Wald）說：不對，**裝甲要加在沒有彈孔的地方——引擎和駕駛艙。**（事實照 landscape §七與 2026-06 web 查證歸屬 Wald／SRG。）

為什麼？因為你統計的全是**飛回來**的飛機。一架飛機引擎或駕駛艙中彈，它就**飛不回來了**，根本不在你的資料裡。你看到引擎乾淨，不是因為引擎不容易中彈，而是因為**引擎中彈的飛機都墜毀了、不在你的樣本裡**。那些機翼滿是彈孔卻還飛得回來的飛機，恰恰證明了機翼中彈不致命。你樣本裡每一架，都是「中了這些彈還能活著回來」的倖存者；它們身上的彈孔，標示的不是「哪裡危險」，而是「哪裡中彈也沒關係」。

這就是**倖存者偏誤（survivorship bias）**：你的樣本被「能不能活到被你看到」這個條件偷偷過濾過了，而那個過濾條件正好跟你要研究的東西高度相關。墜毀的飛機——你最該分析的那批——從定義上就不會出現在你的資料集裡。它們是沉默的、看不見的、被結構性刪除的母體。

這個偏誤最毒的地方在於：**擴大樣本完全沒用。** 你再多統計一千架飛回來的飛機，每一架還是飛回來的，資料還是只有倖存者。問題不在樣本太小，在你的樣本框跟想要的母體之間被「倖存」這條件切了一刀，而你看不到那一刀。

工程師版的倖存者偏誤，你每天都泡在裡面：**你只有「沒當機的機器」的指標**（OOM 之後直接消失的容器，死前那幾秒可能根本沒上報，你看著一片綠說系統健康，但最該看的是已經不在面板上的死者）；**你只有「重試成功」的請求的延遲**（徹底放棄、被 retry 邏輯吃掉的請求延遲是無限大，沒進統計，你的 P99 是「成功者俱樂部」的 P99）；**你只訪談還在用產品的客戶**（想知道為什麼流失，卻只問得到沒流失的人——你在問飛回來的飛機為什麼會被擊落）。三個情境的共同結構：被「能不能活到被你看到」過濾過，而那個過濾條件正好跟你想研究的東西相關。

每次看一份資料下結論之前，瓦德的問題值得你刺自己一下：**這份資料裡，誰因為「沒活到能被我看到」而被悄悄刪掉了？** 沉默的那一群——墜毀的飛機、OOM 的容器、放棄的請求、流失的客戶——往往才是答案所在。

## 直覺的陷阱

抽樣這一章的直覺陷阱特別密，因為它同時踩中「大數的迷信」和「看不見的偏誤」兩種人類盲區。整理成表：

| 你的直覺以為 | 實際上 | 在哪一步把你帶溝裡 |
|---|---|---|
| 樣本越大越準，大到一定程度就一定可信 | 樣本大只壓**變異**不壓**偏誤**。抽樣方法歪了，再大的樣本只是更精準地測量一個錯的值 | 一看到「兩百四十萬份」就放下戒心，不去追問這些樣本**怎麼來的** |
| 要代表母體，得把樣本「設計得像」母體 | 代表性靠**隨機**不靠「長得像」。刻意設計只能控制你想得到的維度，反而偷渡你的偏見 | 在你沒想到、量不到的那個維度上系統性偏掉，而你因為沒看那維度，永遠不會發現 |
| 母體越大，需要的樣本越大 | 準度幾乎只看樣本大小 n，不看母體大小 N（除非樣本佔母體 >5%）。1000 人估兩億跟估兩萬一樣準 | 在「才問 1000 人怎麼代表兩億」這裡卡住、不信任正確的民調，或反過來覺得「母體小所以隨便問問就夠」 |
| 我手上的資料就是事實的全貌 | 你的資料被「能不能活到被你看到」過濾過（倖存者偏誤）。最該看的那群從定義上不在資料裡 | 看著飛回來的飛機補機翼、看著一片綠的 dashboard 說系統健康 |
| 願意回應的人 ≈ 隨機的人 | 願意填問卷／回訪談／留下來的人本身是自我篩選的一群（無回應偏誤） | 把「回收的兩成」當成「全體」，沉默的八成跟他們系統性不同 |

具名陷阱兩個，都來自上面的故事，務必能口頭重講：**倖存者偏誤（survivorship bias）**——只分析活下來的，把沉默的死者結構性刪除（瓦德的飛機）；**抽樣框偏誤／無回應偏誤**——你接觸得到的那群、和願意回應的那群，系統性地不等於母體（《文學文摘》的車主與電話用戶）。

怎麼自我察覺你已經中招了？幾個徵兆：

- 你開始用「樣本這麼大，不可能有問題」當論證——這句話幾乎總是偏誤的遮羞布。樣本大小從來不是抽樣品質的保證。
- 你說得出你的樣本**算出什麼**，卻說不清楚它**怎麼抽來的**、**漏掉了誰**。如果你沒法回答「誰不在我的資料裡」，你就還沒資格下結論。
- 你的估計值穩定得可疑——每次重抽都給你幾乎一樣的數。穩定（低變異）會偽裝成正確（低偏誤），但它們是兩回事。瞄歪的槍打出來也是一個很集中的彈著點。
- 你的樣本是「方便抽到的」——這台機器剛好有 log、這群使用者剛好在你的內測名單、這些客戶剛好還在。「方便樣本（convenience sample）」是偏誤的溫床，因為「方便」這個條件本身常常跟你要研究的東西相關。

## 紙上推演

### 推演題 1 ——「攪勻就好」要算給人聽 **[10 分鐘]** ★

一個工程師同事不信你：「你抽樣監控每 100 筆 log 抽 1 筆，估出錯誤率 0.3%。可是我們流量從每天 1 億筆漲到 10 億筆了，你的抽樣量也才跟著翻 10 倍，**佔整體的比例還是一樣小**，這估計憑什麼可信？」

請你用一句話＋一條公式，向他解釋為什麼「佔整體的比例」根本不是重點。提示：寫出標準誤公式，指出裡面有沒有「總流量 N」。

#### 推演解答

標準誤公式：

```text
SE = √( p(1−p) / n )
```

裡面只有 p（錯誤率）和 n（你**實際抽到的絕對筆數**），**沒有總流量 N**。所以決定你準度的，是你抽到的絕對筆數 n，不是它佔總量的比例。

流量從 1 億漲到 10 億，只要你的絕對抽樣筆數 n 夠大（比如維持在幾十萬筆），錯誤率估計的標準誤就一樣小——跟總流量是 1 億還是 10 億、跟你佔的比例是百萬分之一還是千萬分之一，**完全無關**（嚴格講有個有限母體校正項，但 n 相對 N 這麼小的時候它等於 1，可以無視）。

一句話版本：**「準不準看我舀了多少水（n），不看那鍋湯有多大（N）。只要攪勻了，一湯匙嚐一公升跟嚐一噸一樣準。」** 唯一要守住的不是比例，是「隨機抽」這件事——別只抽某台機器、某個時段、某類請求，否則 n 再大也是精準地測一個偏掉的子集。

### 推演題 2 ——倖存者偏誤的工程版獵捕 **[15 分鐘]** ★★

下面三個情境，各有一個倖存者偏誤。請對每個指出：(a) 你想知道的母體是什麼；(b) 你實際看到的樣本被什麼條件過濾了；(c) 被結構性刪掉、但其實最該看的是誰。

1. 你統計線上所有機器的記憶體使用曲線，發現平均很健康，於是把記憶體 quota 調低。
2. 你看 API 的延遲分布，P99 是 800ms，覺得還行，照這個簽 SLA。
3. 你做使用者訪談問「為什麼喜歡我們的 App」，受訪者一面倒地滿意，於是你判斷產品方向正確。

#### 推演解答

1. (a) 母體＝**所有**機器的記憶體行為，包括撐不住的。(b) 樣本被「**還活著、還在上報指標**」過濾——OOM 之後瞬間死掉的容器，它爆掉前那一下的曲線可能根本沒上報。(c) 最該看的是**那些已經 OOM 消失的機器**：它們才告訴你 quota 的真正下限在哪。看著倖存者的平均調低 quota，等於看著飛回來的飛機補機翼——你會讓更多機器墜毀。

2. (a) 母體＝**所有**請求的真實延遲，包括失敗的。(b) 樣本通常只含**成功回應**的請求；逾時、被 retry 吃掉、被 circuit breaker 擋掉的，它們的「延遲」是無限大或根本沒進統計。(c) 最該看的是**那些徹底失敗的請求**——它們的「延遲」是 ∞。你的 P99=800ms 是「成功者俱樂部」的 P99，真實的使用者體驗尾巴比這慘得多。

3. (a) 母體＝**所有**接觸過產品的人，含已流失的。(b) 樣本被「**還在用、所以還訪談得到**」過濾——流失的人早走了，不會出現在你的受訪名單。(c) 最該看的是**流失的使用者**：他們才知道產品哪裡讓人受不了。問還在用的人為什麼喜歡，就是在問飛回來的飛機為什麼會被擊落——你問錯了一整群人。

共同模式：每個情境裡，「能不能活到被你看到」這個過濾條件，都跟你想研究的東西（穩定性、延遲、流失原因）高度相關。這正是倖存者偏誤致命的結構。

### 推演題 3 ——抽樣分布的散布 **[15 分鐘]** ★★

脊椎硬幣（**對，就是 ch01 那枚公正硬幣**），真實 p=0.5，你不知道，要靠抽樣估。

(a) 你打算擲 n=400 次當一個樣本。寫出 p̂ 這個統計量的抽樣分布（中心、標準誤、形狀）。

(b) 用 68-95-99.7 法則，講出「大約 95% 的情況下，你這一把 400 次算出來的 p̂ 會落在哪個區間」。

(c) 跟正文 n=100 的情形（標準誤 0.05）比，n 翻成 4 倍，標準誤縮成幾分之幾？這驗證了哪條法則？

#### 推演解答

(a) 三件事各歸各的定理管：

```text
中心   ：E[p̂] = p = 0.5            ← 不偏（大數法則撐著，見 ch13）
標準誤 ：SE = √( p(1−p)/n )
           = √( 0.25 / 400 )
           = √0.000625
           = 0.025                ← √n 法則，見 ch09
形狀   ：常態                       ← 中央極限定理，見 ch14
所以   ：p̂ ~ N( 0.5 , 0.025² )
```

(b) ±2 個標準誤涵蓋約 95.45%（嚴格說 95% 對應 ±1.96 個標準誤，這裡用 ±2 的 95.45% 法則估）：

```text
0.5 ± 2 × 0.025 = 0.5 ± 0.05 = [0.45 , 0.55]
```

所以大約 95% 的情況下，你這一把 400 次算出來的 p̂ 會落在 0.45 到 0.55 之間（±5 個百分點）。

(c) n 從 100 變 400（4 倍），標準誤從 0.05 變 0.025——**縮成一半**。

```text
原 SE = √(0.25/100)  = 0.05
新 SE = √(0.25/400)  = 0.025      ← 剛好一半
```

這驗證了 √n 法則（見 ch09）：標準誤跟 √n 成反比。樣本翻 4 倍（√4 = 2），誤差只砍一半。要再砍一半（到 0.0125），得再翻 4 倍到 n=1600。**精度跟樣本量的平方根走，不是線性走——這就是為什麼把民調誤差從 ±3% 壓到 ±1.5% 要花四倍的錢**（ch17 完整展開這筆帳）。

### 推演題 4 ——口頭題：只問 1000 人怎麼代表兩億人 **[10 分鐘]** ★★

不准用任何公式，向一個堅信「才問 1000 人根本不可能代表兩億人」的朋友，講清楚為什麼可以。你的解釋必須回答他兩個質疑：「比例上那麼小怎麼可能準」和「萬一剛好抽到一群怪人怎麼辦」。

#### 推演解答

要點清單（能用自己的話把這幾點串成一段話就算過關）：

- **「準不準看舀多少、不看湯多大」**：用煮湯比喻。一鍋湯攪勻了，舀一湯匙嚐鹹淡，準度只看湯匙多大，跟那鍋湯一公升還是一噸無關。1000 人就是那一湯匙——它的絕對大小（1000）決定準度，母體是兩億還是兩萬幾乎不影響。比例小得嚇人（二十萬分之一）在數學上根本不進公式。

- **「攪勻 = 隨機抽」**：關鍵不是樣本「長得像」母體，是**隨機**抽。隨機化讓樣本在所有維度上——包括你想不到的——自動平衡，等於替你把那鍋湯攪勻。刻意去湊「代表性」反而會偷渡你的偏見。

- **回答「抽到一群怪人」**：會啊，**有可能**剛好抽歪——這正是為什麼民調要報「±3%」的誤差界。那個 ±3% 就是在誠實告訴你「抽歪到這個程度的事偶爾會發生」。統計不是保證每次都對，是**精確地量化它會錯多少、多常錯**。1000 人時，抽到嚴重不具代表性的怪樣本的機率，小到可以用那個誤差界框住。

- **反面警告（加分）**：但這一切的大前提是「**真的**隨機抽」。1936 年《文學文摘》問了兩百四十萬人還是錯了 19%，就因為名單（車主、電話用戶）從根上偏了。**所以該擔心的不是「樣本太小」，是「抽樣方法歪了」**——後者連兩百四十萬份都救不回來。

如果朋友還揪著「比例好小」不放，把上面有限母體校正那張表攤給他：母體從兩億縮到兩百萬，誤差界第四位小數才動。眼見為憑。

## 自我檢核

口頭自答，講得出來才算過關：

1. 母體、樣本、參數、統計量，四個詞各是什麼？哪些是固定常數、哪些是隨機的？（**為什麼** p̂ 是隨機變數而 p 不是？）
2. 「抽樣分布」是什麼東西的分布？它的中心、寬度、形狀分別被前面哪三章的哪三條定理鎖死？
3. 標準誤（standard error）跟一般的標準差是不同的東西嗎？它具體在量什麼？
4. 為什麼民調 ±3% 跟母體是兩億還是兩百萬幾乎無關？公式裡有沒有母體大小？什麼時候母體大小才真的有影響？
5. 「樣本要長得像母體才有代表性」——這句話錯在哪？真正帶來代表性的是什麼？為什麼隨機反而比刻意設計好？
6. 《文學文摘》問了兩百四十萬人還慘敗，推翻了哪個直覺？偏誤（bias）和變異（variance）為什麼是兩種不同的錯，擴大樣本能解決哪一種、不能解決哪一種？
7. 用瓦德的飛機講一遍倖存者偏誤，再舉一個你自己系統裡的版本。為什麼這種偏誤「擴大樣本」也救不了？
8. A/B test 為什麼非得隨機分組？你花錢買隨機分組，到底買到了什麼？（提示：跟 ch10 的 confounder 有關。）

## 延伸閱讀

- **Jordan Ellenberg,《How Not to Be Wrong》**（中譯《數學教你不犯錯》）——瓦德飛機彈孔那一章是全書開場，把倖存者偏誤講得比任何教科書都生動，讀第一章就值回票價。（2026-06 web 查證歸屬，Ellenberg 引述 Wald／SRG。）
- **維基百科「Literary Digest」與「The Literary Digest poll of 1936」條目**——把這場史上最貴的抽樣翻車的數字（一千萬寄出、兩百四十萬回收、蘭登 57 vs 羅斯福 43 的預測對上 62 vs 38 的實際）查清楚，順帶看蓋洛普如何用五萬人海電兩百四十萬人。
- **David Spiegelhalter,《The Art of Statistics》**（中譯《統計，讓數字說話》）——作者是把「資料怎麼來的比資料多大更重要」講得最清楚的當代統計學家之一；讀他談 sampling frame 那段。
- **Darrell Huff,《How to Lie with Statistics》**——1954 年的老書、薄薄一本，但「偏誤的樣本」那一章到今天一個字都不過時，每個看 dashboard 的人都該讀。把它當「統計版的反模式手冊」。
- **本書 ch17（估計與信賴區間）**——這一章把 p̂ 的抽樣分布建好了，下一章接著問：那我該怎麼**用**這個分布、誠實地說出「我估的值錯在多少之內」？民調 ±3% 的完整來歷在那裡，而且會拆穿一個全世界都在犯的信賴區間誤讀。
