# ch21 — 馬可夫鏈：無記憶的旅程與穩態之美

> **本章解決什麼問題**：前一章（見 ch20）我們把脊椎硬幣每擲變 ±1，得到隨機漫步——一條只會往左往右、沒有任何記憶的時間軸。馬可夫鏈把這件事推廣：狀態不再只是「位置 +1 / −1」，可以是「晴／雨」「健康／降級／當機」「這個網頁／那個網頁」；轉移也不再對稱，而是一張你自己定的機率表。本章要回答兩個問題：當「下一步只看現在、不看過去」這條規則成立時，長期會發生什麼？答案是一個叫**穩態分布**的東西——系統會忘記它從哪裡出發。這是隨機過程一瞥（ch20–22）的第二站，接在隨機漫步之後、布朗運動（ch22）之前。

## 從你已知的出發

你畫過狀態機。不是隱喻，是真的——你維運的每個服務心裡都有一張狀態圖：`HEALTHY → DEGRADED → DOWN → RECOVERING → HEALTHY`，circuit breaker 的 `CLOSED → OPEN → HALF_OPEN`，連線池的連線 `IDLE → ACTIVE → IDLE`，TCP 的那一大盤握手狀態。你習慣把這些畫成方框加箭頭，箭頭上寫「在什麼條件下會轉過去」。

現在做一件事：把箭頭上的「條件」換成「**機率**」。

「健康的服務，下一分鐘有 95% 還健康、4% 降級、1% 直接當掉」——這不是你平常的寫法，但它其實更誠實。你的服務會不會降級，從來不是一個乾淨的 if-else，而是一堆你看不見的微觀因素（某個慢查詢、某次 GC、某波突發流量）擲出來的結果。從外面看，它就是個帶機率的狀態轉移。

一旦你把狀態機配上轉移機率，你手上就是一條**馬可夫鏈（Markov chain）**了。而馬可夫鏈最反直覺、也最實用的一個結論是這樣的：

> 只要這張轉移表滿足兩個溫和的條件，那麼**不管你的系統現在是健康、降級還是剛當機重啟**，跑得夠久之後，它待在各個狀態的時間佔比都會收斂到**同一組數字**。

換句話說：你那台機器「長期有多少比例的時間是健康的」，**跟它今天早上開機時是什麼狀態無關**。它會忘記起點。

這句話如果你第一次聽覺得「廢話，本來就該這樣」，那你還沒真的懂它——因為它的兄弟版「最近故障比較多、接下來應該會平靜一陣」是徹頭徹尾的**賭徒謬誤**（見 ch13），而這兩句話聽起來像同一件事。本章很大一部分就是在劃清這條線。

先聲明跨書邊界，免得你期待錯方向：馬可夫鏈當作**排隊與效能運算的工具**，書架上那本《佇列論》已經寫透了——連續時間版（CTMC）、生滅過程、Jackson 排隊網路那些（見《佇列論》ch06–07、ch16）。本章不重推那些公式。本章只做一件《佇列論》沒做、也不該它做的事：講清楚穩態**為什麼美**。

## 馬可夫性質：把「歷史」整包丟掉的膽量

先把核心假設講到位，因為整章的力量全從這裡長出來。

一個隨機過程是「**馬可夫的（Markovian）**」，意思是：

> 給定**現在**的狀態，**未來**和**過去**獨立。

用條件機率寫（沿用 ch04 的記號）：設狀態序列是 X₀, X₁, X₂, …，那麼

```text
P(X_{n+1} = j | X_n = i, X_{n-1} = ..., ..., X_0 = ...)
        = P(X_{n+1} = j | X_n = i)
                                  ↑ 後面那一長串歷史，全部不影響
```

左邊：「已知我這一路走來的完整歷史，下一步去 j 的機率」。右邊：「只知道我現在在 i，下一步去 j 的機率」。馬可夫性質說這兩個**相等**。

停十分鐘，因為這裡值得你停。這個假設聽起來很弱（「下一步只看現在」），但它其實是一刀砍掉了**所有歷史**。它不是說「過去不重要」，而是說——**過去的所有資訊，已經被現在的狀態完整吸收了**。你想知道明天，只要看今天；今天怎麼來的、繞了多大一圈、走了多久，一概不必問。

你其實天天在跟這個假設打交道，只是沒這樣叫它。一個設計良好的 REST API 是**無狀態的（stateless）**：server 處理這個 request 不需要記得你上一個 request 是什麼，request 自己帶齊了所有 context。馬可夫性質就是隨機過程版的 stateless——**狀態本身就是完整的 context**，鏈不需要額外的記憶體去存歷史。這也是為什麼馬可夫鏈在計算上這麼乖：你只要存一張「從哪到哪」的轉移表，不用存一條會無限長的軌跡。

當然，工程師對「無記憶」這種承諾應該天生警惕——大多數真實系統其實**是**有記憶的（待會「直覺的陷阱」會專門修理這件事）。但先讓我們看看，當這個假設成立時，能榨出多漂亮的東西。

### 轉移矩陣：把整張狀態圖塞進一個方陣

如果狀態只有有限個（晴/雨、健康/降級/當機），那「下一步只看現在」這整套規則，可以**完整**寫進一個方陣，叫**轉移矩陣（transition matrix）**P。

`P[i][j]` = 從狀態 i，下一步跳到狀態 j 的機率。

規則只有一條，而且很自然：**每一列（row）加起來等於 1**。因為「我現在在 i，下一步總得去某個地方」，所有去處的機率必須湊滿 100%。這種每列和為 1、元素非負的矩陣有個名字叫**隨機矩陣（stochastic matrix）**，但名字不重要，重要的是它就是你那張狀態圖的另一種寫法——一個是給人看的箭頭圖，一個是給數學用的方陣，內容一模一樣。

拿我們本章的基準範例：天氣鏈。兩個狀態，晴（S）和雨（R）。假設：

- 今天晴，明天 80% 還晴、20% 轉雨；
- 今天雨，明天 40% 放晴、60% 繼續雨。

（雨天有點黏，這很合理——壞天氣傾向成群結隊。）

畫成狀態圖（ASCII，箭頭上是轉移機率）：

```text
       0.8                      0.6
      ┌───┐                    ┌───┐
      │   ↓                    │   ↓
    ┌──────┐    ──0.2──►     ┌──────┐
    │ 晴 S │                 │ 雨 R │
    │      │    ◄──0.4──     │      │
    └──────┘                 └──────┘
```

寫成轉移矩陣（列＝今天，行＝明天）：

```text
            明天晴    明天雨
今天晴  [    0.8      0.2   ]   ← 這一列加起來 = 1.0
今天雨  [    0.4      0.6   ]   ← 這一列也是 = 1.0
        P =
```

讀法：左上角 0.8 是「今天晴→明天晴」，右上角 0.2 是「今天晴→明天雨」。整張天氣的行為，就這四個數字。

那「**後天**」呢？這裡有個小小的、會讓你會心一笑的事實：如果今天晴，要算後天晴的機率，你得考慮兩條路徑——晴→晴→晴（0.8×0.8）加上晴→雨→晴（0.2×0.4）= 0.64 + 0.08 = 0.72。而這，恰好就是**矩陣乘法** P×P 的左上角元素。

換句話說：「走兩步的轉移機率」= P²，「走 n 步」= Pⁿ。馬可夫鏈把「時間往前走」這件事，變成了**矩陣連乘**。你不用一條一條軌跡去追蹤，矩陣的次方幫你把所有路徑的機率一次加總好。對一個天天在跟「狀態怎麼演化」打交道的工程師來說，這個翻譯本身就有點美：時間 = 次方。

## 穩態分布：系統最後停在哪裡（不是停，是停在一個比例上）

現在問核心問題。假設今天一定是晴天，我用一個機率向量表示「我對今天的認知」：π₀ = (1, 0)，意思是 100% 晴、0% 雨。

明天的分布是什麼？把這個向量乘上 P：

```text
π₁ = π₀ · P = (1, 0) · P
   = ( 1×0.8 + 0×0.4 ,  1×0.2 + 0×0.6 )
   = ( 0.8 , 0.2 )                      ← 明天 80% 晴、20% 雨
```

後天就是再乘一次 P，大後天再乘一次……每乘一次 P，時間往前一天。我們來看這個向量怎麼演化。下面是從「今天一定晴」(1, 0) 出發，逐天迭代的表（每個數字都是手算，等下推演題會帶你驗）：

```text
天數 n     P(晴)      P(雨)      與穩態 2/3 的差距
─────────────────────────────────────────────────
  0       1.0000     0.0000        +0.3333
  1       0.8000     0.2000        +0.1333
  2       0.7200     0.2800        +0.0533
  3       0.6880     0.3120        +0.0213
  4       0.6752     0.3248        +0.0085
  5       0.6701     0.3299        +0.0034
  …          ↓          ↓             ↓
  ∞       0.6667     0.3333         0
        (= 2/3)     (= 1/3)
```

它收斂了。而且不是亂收斂——它收斂到 (2/3, 1/3)。

看最右邊那欄：每往下一行，與穩態的差距大約**乘以 0.4**（0.3333 → 0.1333 → 0.0533 → 0.0213…，每次約 0.4 倍）。這個 0.4 不是巧合，是這條鏈的第二大特徵值（trace 0.8+0.6 減 1），它決定了「忘記起點」的速度。指數收斂，你在效能曲線上看過這種形狀——這就是為什麼短短五六天，初始的「100% 晴」就幾乎被洗乾淨了。

那 (2/3, 1/3) 是什麼？它叫**穩態分布（stationary distribution）**，記作 π。它的定義方程式漂亮到不可思議：

```text
            π · P = π
```

翻成白話：**你拿這個分布去乘轉移矩陣，乘完還是它自己。** 它是這個動力系統的不動點（fixed point）——一旦系統的分布長成 π 這個樣子，再往前走多少步，分布都不再變。晴天的比例維持 2/3、雨天維持 1/3，永遠。

注意「不變」的是**分布**，不是天氣本身。天氣每天還是照樣晴晴雨雨亂跳，個別的某一天你完全測不準。穩態說的是**總體**：跑一萬天，大約 6667 天是晴、3333 天是雨。**個體不可測、總體精確可測**——這是整本書一再出現的那個主旋律（見 ch13、ch14），馬可夫鏈在時間軸上把它又彈了一遍。

### 怎麼算出 (2/3, 1/3)：解 πP = π（手算）

不必把矩陣乘到天荒地老才看出極限。穩態可以直接解。設 π = (s, r)，s 是晴的長期比例、r 是雨的。π · P = π 攤開來是兩條方程式：

```text
晴的方程：  s = 0.8·s + 0.4·r      ← 「下一刻是晴」的兩個來源：本來晴沒走、本來雨放晴
雨的方程：  r = 0.2·s + 0.6·r
再加一條：  s + r = 1              ← 機率得湊滿（這條是 π 是分布的要求）
```

拿晴的方程來解：

```text
   s = 0.8 s + 0.4 r
   s − 0.8 s = 0.4 r
   0.2 s = 0.4 r
   s = 2 r                         ← 晴天是雨天的兩倍
```

代回 s + r = 1：

```text
   2r + r = 1  →  3r = 1  →  r = 1/3 ,  s = 2/3
```

得 **π = (2/3, 1/3)**。

工程師的嚴謹要求交叉驗算（這題尤其，機率手算最容易在某個小數點翻車）。把 π = (2/3, 1/3) 代回 π · P，看是不是還等於自己：

```text
π · P 的「晴」分量 = 2/3 × 0.8 + 1/3 × 0.4
                   = 1.6/3 + 0.4/3
                   = 2.0/3 = 2/3   ✓ 等於原本的 s

π · P 的「雨」分量 = 2/3 × 0.2 + 1/3 × 0.6
                   = 0.4/3 + 0.6/3
                   = 1.0/3 = 1/3   ✓ 等於原本的 r
```

對上了。乘完是它自己，πP = π 成立。而且這個 (2/3, 1/3) 跟上面那張迭代表收斂到的數字一致——兩條獨立的路（直接解方程式 vs. 一步步迭代）走到同一個答案，這是本書一貫的交叉驗算習慣。

### 為什麼「忘記起點」這件事美

到這裡你可能還覺得平淡，那讓我把那個驚嘆點挑明。

我從 (1, 0)「今天一定晴」出發，收斂到 (2/3, 1/3)。現在我換一個完全相反的起點：(0, 1)「今天一定雨」。再迭代一次給你看（手算，推演題會要你補完）：

```text
天數 n     從「一定晴」出發        從「一定雨」出發
            P(晴)    P(雨)         P(晴)    P(雨)
──────────────────────────────────────────────────
  0        1.000    0.000         0.000    1.000   ← 兩個起點天差地別
  1        0.800    0.200         0.400    0.600
  2        0.720    0.280         0.560    0.440
  3        0.688    0.312         0.624    0.376
  4        0.675    0.325         0.650    0.350
  5        0.670    0.330         0.660    0.340
  …          ↓        ↓             ↓        ↓
  ∞        0.667    0.333         0.667    0.333   ← 殊途同歸
```

兩條從相反極端出發的曲線，像兩滴墨水滴進同一杯水，最後**融成完全相同的灰色**。穩態 (2/3, 1/3) 不在乎你從哪裡來。

這就是了不起的地方，我認為這是整章最值得帶走的一句話：

> **這條鏈的長期行為，是它自己的性質（藏在轉移矩陣 P 裡），不是它出身的性質。** 起點的資訊，會被一步步指數地稀釋掉，直到完全消失。

把它翻回你的世界：你的服務「長期有 X% 時間健康」這個數字，是**它的轉移機率決定的**——是它的程式碼品質、依賴穩定度、流量特性決定的，**不是**它今天早上開機時是什麼狀態決定的。所以「重啟一台行為異常的機器」之所以常常有用，不是因為重啟改變了它的長期穩態（沒有，轉移矩陣沒變），而是因為它讓你**重新從乾淨狀態出發**——但只要轉移矩陣裡那個導致問題的機率還在，它終究會收斂回同一個有病的穩態。冷酷，但這正是穩態理論在告訴你的事：**治本要改的是轉移矩陣，不是初始狀態。**

### 收斂條件：什麼時候保證有唯一穩態（點到不證）

我得誠實標示嚴謹度：上面「不管從哪出發都收斂到同一個 π」這件事，**不是對所有馬可夫鏈都成立的**。它需要兩個條件。這裡只給直覺、不給證明（嚴格的遍歷定理本書不展開，指向延伸閱讀）：

| 條件 | 白話 | 壞掉的樣子 |
|---|---|---|
| **不可約（irreducible）** | 從任何狀態都走得到任何其他狀態（圖是連通的） | 鏈分裂成兩塊，掉進去 A 區就永遠出不來 B 區——「從哪出發」當然就有影響了 |
| **非週期（aperiodic）** | 回到原狀態的步數沒有固定的最大公因數 > 1 | 鏈死板地「奇數步在這、偶數步在那」來回擺，分布永遠在兩個值之間跳、不安頓 |

我們的天氣鏈兩條都滿足：晴雨互相到得了（不可約），而且晴→晴那條自環讓它不卡在固定週期（非週期）。所以它有唯一穩態，且從任何起點都收斂過去。這個「不可約 + 非週期 ⟹ 唯一穩態且必收斂」的結論，是馬可夫鏈理論的核心定理，行話叫**遍歷性（ergodicity）**。你只要記得它的精神：**只要鏈不會把自己困死、也不會死板地打拍子，它就會忘記起點。**

順帶一提，週期性那個「奇偶來回跳」的壞例子，你在 ch20 見過——對稱隨機漫步就是週期 2 的（每步必定改變位置的奇偶），所以它**沒有**穩態分布。馬可夫鏈把隨機漫步收編成一個特例，順便解釋了為什麼漫步那一章我們談的是「常返性」而不是「穩態」。

## 馬可夫本人：一場關於自由意志的數學鬥嘴

這套理論的誕生，背後有個比理論本身還好看的故事，而且它直接回應了「馬可夫性質到底在挑戰什麼」。

時間回到 1900 年代初的俄國。當時數學界有個被當成常識的信念：**大數法則（見 ch13）需要事件彼此獨立。** 雅各布·伯努利（Jacob Bernoulli）1713 年那個第一版大數法則（見 ch13）談的就是獨立試驗，大家想當然耳地以為「獨立」是不可或缺的前提。

這時莫斯科有位數學家**涅克拉索夫（Pavel Nekrasov）**——他出身神學院，相信數學能替神學服務——在 1902 年把這件事推向一個驚人的結論。他的論證大致是：自由意志的行為（人自願做的選擇）彼此之間沒有因果鏈、是獨立的；大數法則只對獨立事件成立；而社會統計資料（犯罪率、結婚率那些 Quetelet 式的數字，見 ch15）確實服從大數法則；**所以**——人的行為一定是獨立的、自由的。他想用機率論證明自由意志的存在。

聖彼得堡的**馬可夫（Andrey Markov）**，一個出了名脾氣硬、好戰、人送外號「憤怒的安德烈」的數學家，被這個論證激怒了。倒不是為了神學——他想戳破的是那個數學前提：**「大數法則必須要獨立」根本就是錯的。**

於是他在 **1906 年**動手，做了件漂亮的事：他建構出一串**彼此相依**的隨機變數——每一個的機率依賴前一個——然後**證明大數法則照樣成立**。他從最簡單的兩狀態系統下手（就是我們上面那種兩狀態鏈！），證明只要那四個轉移機率都落在 0 到 1 之間，系統演化下去，**每個狀態出現的頻率都會收斂到一個固定值**——也就是我們今天說的穩態。相依，但照樣有大數法則。涅克拉索夫的整個論證地基被抽掉了。（這個「1906 起手、兩狀態」的史實，照 landscape 與多源確認。）

故事最妙的是收尾。理論歸理論，馬可夫想要一個**真實世界裡相依事件鏈**的例子來示範，於是他在 **1913 年** 1 月 23 日，在聖彼得堡帝國科學院報告了一個你怎麼想都想不到的東西：他拿普希金（Pushkin）的長詩《**尤金·奧涅金**（Eugene Onegin）》，數了**前 20,000 個字母**，把每個字母標成「母音」或「子音」，然後分析這個序列。

（這裡務必照 landscape 的 ⚠️ 把年份分清楚：**理論工作是 1906，奧涅金母音子音的實證示範是 1913**，兩者常被混為一談，別都寫成同一年。）

他發現的數字很能說明問題。整篇 20,000 個字母裡，母音 8,638 個、子音 11,362 個，所以母音的整體出現率約 0.43。但如果你**看前一個字母**：母音後面再接母音的機率只有約 12.8%，而子音後面接母音的機率高達約 66.3%。換句話說，下一個字母是不是母音，**強烈地依賴前一個字母是什麼**——這正是一條活生生的、非獨立的馬可夫鏈。母音子音傾向交替，就像我們天氣鏈裡「雨天會黏著雨天」一樣，是有結構的相依。

我喜歡這個故事，因為它把整章的假設照亮了。馬可夫性質「下一步只看現在」聽起來像在**否認**相依——但馬可夫發明它，恰恰是為了**捕捉**相依。重點不是「事件互不影響」（它們明明互相影響），而是「**所有的影響，都濃縮在『當前狀態』這一個變數裡**」。下一個字母不獨立於整個歷史，但它只需要看**前一個字母**——歷史的其餘部分是多餘的。這就是馬可夫鏈又強又乖的祕密：它在「完全獨立（太簡單、不像真實世界）」和「依賴完整歷史（太複雜、算不動）」之間，找到了那個剛剛好的中間地帶。

## PageRank：把整個網際網路當成一條馬可夫鏈

如果你覺得母音子音太古典，這裡有個 1998 年的版本，而且你每天都在用它的後代。

**布林（Sergey Brin）與佩吉（Larry Page）**在 1998 年那篇奠定 Google 的論文裡，要解一個問題：網路上有上億個網頁，怎麼判斷哪個「重要」？他們的答案叫 **PageRank**，而它的核心是一個你現在已經認得的東西——**一條跑在整個網頁圖上的馬可夫鏈**。

想像一個**隨機衝浪者（random surfer）**：他現在停在某個網頁上，下一步，他隨機點擊這一頁上的某條連結，跳到下一頁；到了下一頁，再隨機點一條，再跳……（照 landscape 與原論文。）

這就是馬可夫鏈：

- **狀態** = 每一個網頁；
- **轉移機率** = 從這一頁，等機率地隨機跳到它連出去的任一頁；
- **馬可夫性質** = 衝浪者下一步去哪，只看他**現在**在哪頁，不看他怎麼繞到這裡的（無記憶）。

那「重要度」是什麼？就是**穩態分布**。讓這個衝浪者隨機亂點到天荒地老，他**待在每一頁的時間佔比**——也就是這條鏈的 π——就定義成那一頁的 PageRank。直覺很順：被很多頁連到的頁、被重要的頁連到的頁，衝浪者更容易繞回去、停留比例更高，π 值更大。重要度 = 你在這條巨型馬可夫鏈的穩態裡佔多重。

（只有一個工程補丁值得一提，因為它正好呼應上面的收斂條件：真實網頁圖會有「死胡同」——沒有對外連結的頁，衝浪者會卡死；也會有走不到的孤島，破壞不可約性。Brin 和 Page 的修法是給衝浪者一個小機率「不點連結、直接隨機傳送到任意一頁」，這個機率行話叫**阻尼因子（damping factor）**，他們設成 0.85——大約每點五次連結就隨機跳一次。這一補，整張圖立刻變得既不可約又非週期，唯一穩態的存在就有了保證。你看，前面那兩個抽象的收斂條件，到這裡變成了一行很具體的工程決策。）

讓這件事沉澱一下：Google 最初的排名，本質上是在解一條有**幾十億個狀態**的馬可夫鏈的 πP = π——跟我們上面手算 (2/3, 1/3) 的天氣鏈，是**字面上同一條方程式**，只是維度大了十億倍。同一個「乘完是自己的那個分布」的想法，從 1906 年憤怒的安德烈拿來打贏一場神學嘴砲，到 1998 年變成一家市值數兆的公司的地基。一條方程式，πP = π。我覺得這比任何形容詞都更能說明，為什麼穩態值得你停下來欣賞。

（順帶一提，馬可夫鏈還是現代很多東西的引擎，但本書只各點一句、不展開：貝氏計算裡用來逼近後驗的 **MCMC**（Markov Chain Monte Carlo，回收 ch06——它就是「設計一條穩態剛好是你想採樣的後驗分布的鏈，然後讓它跑」）；以及語音、生物序列裡的**隱馬可夫模型（HMM）**。想深究往延伸閱讀走。）

## 直覺的陷阱

馬可夫鏈最常被誤解的，不是數學，是它的**承諾範圍**——它到底保證了什麼、沒保證什麼。下面這幾條，每一條我都看過工程師（包括我自己）栽進去。

| 陷阱 | 錯誤直覺 | 為什麼錯 / 怎麼自我察覺 |
|---|---|---|
| **把穩態當成賭徒謬誤的靠山** | 「長期晴天佔 2/3，所以最近雨下太多了，接下來該放晴補回來。」 | 穩態說的是**長期比例**，不是「欠你的會還」。每天的轉移只看今天、不看「最近欠多少」（馬可夫性質！）。「該補回來了」偷偷假設了鏈有記憶——但它沒有。這是 ch13 賭徒謬誤穿了件狀態機外衣。**徵兆**：你開始講「平衡」「補回來」「該輪到」。 |
| **以為穩態是一個會停下來的終點** | 「系統最後會穩定在晴天。」 | π 不是一個狀態，是一個**分布**。系統永遠在晴雨之間亂跳，停不下來；穩下來的是**比例**。**徵兆**：你問「它最後會變成哪個狀態」——這問題本身就錯了，正確問法是「它長期待在各狀態的時間佔比」。 |
| **不檢查就假設馬可夫性質成立** | 「我把系統畫成狀態圖了，所以它是馬可夫鏈。」 | 畫得出狀態圖 ≠ 滿足馬可夫性質。如果「下一步」其實依賴**你怎麼走到這個狀態的**（歷史），它就不是馬可夫的。**徵兆**：你的轉移機率得看「上一個狀態是什麼」或「在這狀態待多久了」才填得出來——那歷史就沒被當前狀態吸收乾淨。 |
| **以為任何鏈都會收斂到唯一穩態** | 「跑久一定忘記起點。」 | 只有不可約 + 非週期才保證。可約的鏈（有出不來的死胡同、走不到的孤島）會記得起點；週期鏈（奇偶來回跳）永遠不安頓。**徵兆**：你的圖裡有「進得去出不來」的狀態，或有「一定要偶數步才回得來」的死板節奏。 |
| **指數收斂 vs. 已經收斂** | 「理論說會收斂，所以我量到的就是穩態。」 | 收斂是漸近的。我們天氣鏈每步只把誤差乘 0.4，算快了；但若第二特徵值是 0.999，要好幾千步才接近穩態。「最終會」不等於「現在已經」。**徵兆**：你拿一段太短的觀測就宣稱量到了長期行為。 |

最值得單獨拎出來講的是第一條和第三條。

**第一條（賭徒謬誤的狀態機版）** 危險在於它**聽起來很有道理**。穩態是真的、2/3 是真的，所以「應該回到 2/3 附近」感覺也是真的。陷阱在「應該」這個詞偷渡的因果方向：不是「因為偏離了 2/3，所以有股力量把它拉回來」（那是有記憶的彈簧），而是「**新的擲幣會把舊的偏離稀釋掉**」（見 ch13——稀釋，不是修正）。雨下了五天，第六天放晴的機率還是那固定的 40%，跟前五天一點關係都沒有。系統不欠你晴天。

**第三條（沒檢查就假設無記憶）** 是工程師最該警惕的，因為它最容易自我欺騙。舉個真實的反例：你的服務「在當機狀態」下一步會怎樣，**很可能依賴它『為什麼』當機、以及『當了多久』**——剛掛掉一秒鐘 vs. 已經掛了二十分鐘觸發了告警升級、有沒有 on-call 介入、是不是同一個根因第三次復發了——這些都是**歷史**，都不在「當前狀態 = DOWN」這一個標籤裡。一旦下一步依賴這些，馬可夫性質就破了，你算出來的穩態是錯的。重試的 exponential backoff 更是明擺著有記憶：第幾次重試決定了你等多久，「第幾次」就是歷史。**真實系統大多有記憶**；馬可夫鏈是一個常常很好用、但你每次都該回頭驗的近似。把它當成「我假設了無記憶」而不是「它就是無記憶」，這個區別會在某次事故覆盤時救你。

## 紙上推演

### 推演題 1：補完那張迭代表 **[15 分鐘] ★**

用本章天氣鏈 P = [[0.8, 0.2], [0.4, 0.6]]，從「今天一定雨」π₀ = (0, 1) 出發，**手算**接下來三天的分布 π₁、π₂、π₃。每步就是 (s, r) · P，也就是：

```text
新的 s = 0.8 · s + 0.4 · r
新的 r = 0.2 · s + 0.6 · r
```

算完跟正文那張雙起點對照表的右半邊對一下，看你有沒有走在「往 (2/3, 1/3) 收斂」的路上。

#### 推演解答

逐步代：

```text
π₀ = (0,     1    )
π₁: s = 0.8·0 + 0.4·1 = 0.4 ;  r = 0.2·0 + 0.6·1 = 0.6   → (0.400, 0.600)
π₂: s = 0.8·0.4 + 0.4·0.6 = 0.32 + 0.24 = 0.56
    r = 0.2·0.4 + 0.6·0.6 = 0.08 + 0.36 = 0.44            → (0.560, 0.440)
π₃: s = 0.8·0.56 + 0.4·0.44 = 0.448 + 0.176 = 0.624
    r = 0.2·0.56 + 0.6·0.44 = 0.112 + 0.264 = 0.376        → (0.624, 0.376)
```

每步機率和都該是 1（0.4+0.6、0.56+0.44、0.624+0.376——都對，這是你算對沒翻車的快速檢查）。和正文右半邊一致。注意它正從「下面」往 2/3≈0.667 爬，而「一定晴」那條是從「上面」往下掉——兩條從相反方向夾過來，這就是「忘記起點」的畫面。

### 推演題 2：驗證一個 3 狀態鏈的穩態 **[20 分鐘] ★★**

把你的服務畫成三狀態鏈：健康（H）、降級（D）、當機（C）。轉移矩陣（每列是「這個狀態下一步去哪」）：

```text
        →H      →D      →C
  H  [ 0.90    0.08    0.02 ]
  D  [ 0.50    0.40    0.10 ]
  C  [ 0.60    0.30    0.10 ]
```

有人宣稱穩態是 π = (0.84, 0.12, 0.04)（健康 84%、降級 12%、當機 4% 的時間）。**驗證**這個宣稱：算 π · P，看是不是等於 π。（你不需要去解方程式，只要代進去檢查——這比解快得多，也是工程上更常用的招：別人給你一個號稱的穩態，你只要乘一次就知道真假。）

#### 推演解答

算 π · P 的三個分量。π = (0.84, 0.12, 0.04)。

```text
→H 分量 = 0.84·0.90 + 0.12·0.50 + 0.04·0.60
        = 0.756 + 0.060 + 0.024 = 0.840   ✓ = 0.84

→D 分量 = 0.84·0.08 + 0.12·0.40 + 0.04·0.30
        = 0.0672 + 0.0480 + 0.0120 = 0.1272 ≈ 0.127

→C 分量 = 0.84·0.02 + 0.12·0.10 + 0.04·0.10
        = 0.0168 + 0.0120 + 0.0040 = 0.0328 ≈ 0.033
```

得到 π · P ≈ (0.840, 0.127, 0.033)。和宣稱的 (0.840, 0.120, 0.040) 比：H 分量精準命中，但 D 和 C 對不太上（0.127 vs 0.120、0.033 vs 0.040）。所以 **(0.84, 0.12, 0.04) 不是精確的穩態**，只是接近。

這題的教訓有兩層。第一，工程上「驗證一個號稱的穩態」（乘一次）遠比「解出穩態」（解聯立）省力——當別人塞給你一組數字說「這是長期分布」，你乘一下就能戳破。第二，這也示範了**為什麼**那組數字接近卻不對：它八成是某人迭代了幾步、看數字「差不多穩了」就抄下來——但「看起來穩了」不等於「滿足 πP = π」（呼應陷阱表最後一條）。真正的穩態得解方程式或迭代到收斂為止。（給好奇的你：這條鏈真正的穩態約是 (0.842, 0.122, 0.036)，三條方程式 + 機率和為 1 解得，留給你想練聯立的話自己玩。）

### 推演題 3：抓出馬可夫性質破掉的地方 **[15 分鐘] ★★**

下面三個系統，哪些可以**合理**建模成（離散時間、有限狀態）馬可夫鏈，哪些不行？逐一說出：如果不行，是「當前狀態」沒吸收掉哪一段歷史？

1. 一枚公正硬幣連續擲，狀態 = 這次正或反。
2. exponential backoff 重試：每次失敗後等待時間翻倍，狀態 = 「成功 / 失敗」。
3. 撲克牌不放回地一張張抽，狀態 = 這張牌的花色。

#### 推演解答

**1. 硬幣——是馬可夫鏈（而且是退化版）。** 下一擲正反的機率是固定的 1/2，根本不看現在這擲是什麼，更不看歷史。它甚至比一般馬可夫鏈還簡單：連「當前狀態」都不依賴（這種叫 i.i.d. 序列，是馬可夫鏈的特例）。轉移矩陣兩列都是 (0.5, 0.5)，穩態理所當然是 (0.5, 0.5)——脊椎硬幣的長期比例（見 ch13），用馬可夫的語言又講了一遍。

**2. backoff 重試——不是（用「成功/失敗」當狀態的話）。** 破綻在等待時間：第 1 次失敗等 1 秒、第 2 次等 2 秒、第 4 次等 8 秒……「下一步等多久」依賴**已經失敗過幾次**，而「失敗過幾次」是歷史，沒被「當前狀態 = 失敗」這個標籤吸收。兩次都標成「失敗」，但行為不同。**修法**：把狀態擴充成「(失敗, 第幾次重試)」——把那段歷史塞進狀態裡，馬可夫性質就回來了。這正是工程上常用的招：**當鏈不馬可夫，把缺的歷史加進狀態定義，讓它重新變馬可夫。** 代價是狀態變多。

**3. 不放回抽牌——不是。** 下一張是什麼花色的機率，依賴**前面已經抽掉了哪些牌**（紅心抽光了，下一張就不可能紅心）。整副牌的剩餘組成是歷史，「這張牌的花色」這個狀態完全沒記錄它。每抽一張，轉移機率就變一次。**對照**：如果是**放回**抽（抽完放回去洗），那剩餘組成永遠不變，就變成馬可夫鏈（其實退化成 i.i.d.，像硬幣那樣）。「放回 vs 不放回」恰好就是「有沒有記憶」的分水嶺——這個對照值得記著。

## 自我檢核

口頭自答，講得出來才算過。優先想「為什麼」。

1. 馬可夫性質一句話是什麼？「給定現在、未來與過去獨立」這句話裡，**獨立的是什麼、相對於什麼**？（提示：不是說事件互不影響，而是說歷史被現在吸收了。）
2. 穩態分布 π 滿足 πP = π。用大白話講，這條方程式在要求什麼？為什麼「乘完是自己」就代表「分布不再變」？
3. 「系統會忘記起點」——它忘記的到底是什麼、保留的又是什麼？（哪些性質是出身決定的、哪些是轉移矩陣決定的？）
4. 為什麼「最近雨下太多、該放晴了」是賭徒謬誤，但「長期晴天佔 2/3」是對的？這兩句話差在哪一個字？
5. 收斂到唯一穩態需要哪兩個條件？各自壞掉時會發生什麼？（一個記得起點、一個永遠在跳。）
6. 馬可夫 1906 年到底想反駁涅克拉索夫的什麼主張？他證明了什麼推翻它？（提示：跟「大數法則一定要獨立」有關。）
7. PageRank 的「重要度」對應到馬可夫鏈裡的哪個東西？阻尼因子在數學上是在搶救哪兩個收斂條件？
8. 給你一個系統，你要怎麼判斷「把它當馬可夫鏈」是不是合理的近似？如果不合理、又想救，有什麼招？（提示：把歷史塞進狀態。）

## 延伸閱讀

- **Brian Hayes, “First Links in the Markov Chain”, _American Scientist_(2013)**（2026-06 查證）。本章馬可夫鬥嘴故事的最佳單篇來源——涅克拉索夫的自由意志論證、馬可夫 1906 的反擊、1913 奧涅金實驗的來龍去脈，寫得像偵探小說。讀「the war on free will」與奧涅金那兩節。https://www.americanscientist.org/article/first-links-in-the-markov-chain
- **A. A. Markov, “An Example of Statistical Investigation of the Text _Eugene Onegin_…”（1913，David Link 2006 英譯）**（2026-06 查證）。想看本人原稿怎麼數那兩萬個字母、怎麼算母音子音轉移率，這是英譯本。歷史現場感十足。
- **Brin & Page, “The Anatomy of a Large-Scale Hypertextual Web Search Engine”（1998）**（照 landscape）。PageRank 原始論文，第 2.1 節把隨機衝浪者與穩態講得乾淨利落。讀那一節就好，後面是工程實作。http://infolab.stanford.edu/pub/papers/google.pdf
- **《佇列論》ch06–07、ch16**（書架鄰書）。馬可夫鏈當**運算工具**的去處：連續時間版（CTMC）、生滅過程、排隊網路怎麼算到達率與佇列長度。本章只給了穩態的直覺，要動手算系統的吞吐與延遲，去那裡。
- **見 ch20（隨機漫步）、ch22（布朗運動）**（本書）。回頭看：對稱隨機漫步是一條週期 2、沒有穩態的馬可夫鏈，這解釋了它為什麼談「常返」不談「穩態」。往前看：把馬可夫鏈的時間與狀態都連續化，就走向布朗運動。
- **Grinstead & Snell, _Introduction to Probability_（免費 PDF，第 11 章 Markov Chains）**（2026-06 查證）。想把「不可約 + 非週期 ⟹ 唯一穩態」從直覺補成證明、或多算幾條鏈練手，這章自足、例子多、不掉進測度論。
