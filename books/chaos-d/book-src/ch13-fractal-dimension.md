# ch13 — 碎維度：介於線與面之間的數

> **本章解決什麼問題**：ch12 給了你碎形（fractal）這個東西——自相似、放大之後還是它自己、無窮細節。你看著 Koch 雪花的鋸齒、看著奇異吸子那層層疊疊的紋路，心裡有個量化的衝動沒被滿足：它到底「有多碎」？這一章把這份直覺壓成一個數——維度（dimension）。震撼點藏在這裡：這個數**可以不是整數**。一條 Koch 曲線比直線更會佔空間、卻又填不滿一塊面，它的維度落在 1 和 2 之間，是 1.2619。本章先用「自相似維度」這把最乾淨的尺把 Koch、Cantor、Sierpiński 三角的維度從頭算出來，再升級到對任何形狀都管用的盒計數（box-counting）直覺——而盒計數的本質，就是你天天在看的 log-log 斜率。算完你會懂，為什麼 ch11 那隻勞侖次蝴蝶的維度是 2.06：它「比一張面多一點點」。

## 從你已知的出發

你其實早就用「分數維度」這個概念在量東西了，只是沒這樣叫它。

回想你最近一次看一張 log-log 圖。橫軸是負載（每秒請求數），縱軸是延遲的 P99；或者橫軸是檔案大小、縱軸是壓縮後大小；或者橫軸是資料量、縱軸是某個查詢的耗時。你在那張雙對數座標上看的是什麼？你看的是**斜率**。斜率 1，代表延遲跟負載線性走（負載翻倍、延遲翻倍）；斜率 2，代表它平方地爆（負載翻倍、延遲變四倍，你心裡一沉，知道某處藏了個 O(n²)）；斜率 0.5，代表它只開根號地長，相對溫和。

注意這件事的形狀：你沒有去記任何一個絕對數字，你記的是「當我把尺度放大一個倍數，被量的東西放大幾個倍數」——而且你習慣把這個關係寫成 `量 ∝ 尺度^斜率`，再用兩邊取對數、看斜率把它讀出來。這個「取兩次對數、讀斜率」的動作，**就是維度的定義**。維度不是「有幾個座標軸」這種小學答案，維度是一個**縮放律**：當你把觀察的尺度縮小到 1/k，你需要的「東西」變多 k 的幾次方——那個次方，就是維度。

```text
  量 ∝ (尺度)^D          ←  D 就是維度，是縮放的指數
  取對數：log(量) = D · log(尺度) + 常數
  → 在 log-log 圖上是一條直線，斜率 = D    ← 你天天在讀的那條線
```

一條直線，你用 1/k 長的小段去鋪，需要 k 段——`量 ∝ 尺度^1`，維度 1。一塊正方形，你用邊長 1/k 的小格去鋪，需要 k² 格——`量 ∝ 尺度^2`，維度 2。到目前為止維度都是整數，符合直覺。本章的全部反直覺，就發生在「需要的東西不是 k 的整數次方，而是 k 的 1.2619 次方」那一刻。那不是計算錯誤，那是有些形狀真的活在整數維度之間。

我要先把這層工程橋接講透，因為它是你接住後面所有怪數字的安全網。你之所以對「O(n²) 很糟」有身體反應，不是因為你背過 n² 比 n 大，而是因為你在腦子裡跑過那個縮放律：請求量翻倍、工作量變四倍，再翻倍、再變四倍——那個「翻倍對四倍」的固定比例，就是斜率 2 在說的話。同樣的，壓縮率告訴你「檔案放大十倍、壓縮後只放大三倍」，你立刻知道資料裡有大量可壓掉的冗餘，因為它沒有「滿」到每一位元都帶新資訊。**你讀這些斜率時，讀的從來不是某個點的絕對值，而是「尺度動一截、量動幾截」的那個比例**。把這個動作記牢——本章從頭到尾，維度就是這個比例，只是被套在「形狀的細節」而不是「演算法的耗時」上。

我認為這是整本書裡最容易被當成「文字遊戲」、結果卻一點都不是遊戲的一頁。它不是遊戲，因為它量的東西完全具體：一個形狀，當你把放大鏡的倍率調高，跳出來的新細節是以什麼速率增生的。線增生得慢（放大三倍、細節多三倍）、面增生得快（放大三倍、細節多九倍）、Koch 曲線增生的速率卡在中間（放大三倍、細節多四倍）。那個「四」逼出一個非整數的指數，而指數就是維度。我們慢慢來。

## 維度是一個縮放律：先把整數情況講死

在容許分數之前，先把「維度＝縮放指數」這件事在整數上焊牢，這樣分數出現時你才不會懷疑是不是定義被人動了手腳。

拿一個你信得過的東西：一條線段、一個正方形、一個立方體。對每一個，做同一件事——**把它縮小到原來的 1/2，問需要幾個這樣的縮小版才能拼回原物**。

```text
  物件        縮成 1/2 後   要幾份拼回原物   關係            指數 D
  ──────────────────────────────────────────────────────────────
  線段         半條          2 份            2 = 2¹           1
  正方形       半邊長        4 份            4 = 2²           2
  立方體       半邊長        8 份            8 = 2³           3
```

讀這張表的方法是：**份數 N = (放大倍率)^D**。每一份都縮小成原來的 s = 1/2，所以把一份放大回去要乘 1/s = 2 倍；而要拼回整個原物需要 N 份。於是 N = (1/s)^D。線段 N=2、1/s=2，所以 2 = 2^D 給 D=1；正方形 N=4、1/s=2，4 = 2^D 給 D=2；立方體 N=8、1/s=2，8 = 2^D 給 D=3。

把這條 N = (1/s)^D 兩邊取對數、解出 D，就得到本章的主角公式：

```text
  N = (1/s)^D
  ln N = D · ln(1/s)                    ← 兩邊取自然對數
  D = ln N / ln(1/s)                    ← 自相似維度（self-similarity dimension）

  其中  N   = 把整體切成幾份（每份都是整體的縮小複製）
        s   = 每一份相對整體的縮小比例
        1/s = 一份要放大幾倍才回到整體
```

（這裡只用到「log 是 exp 的反函數」這件你早就熟到不行的事——把指數從次方位置搬下來。對數的縮放直覺要從頭擦亮，見《馴服無限》談對數那章。）

關於底數：用 ln（自然對數）還是 log₁₀（常用對數）**完全不影響結果**，因為分子分母同時換底，那個換底常數約掉了。`ln4/ln3 = log4/log3 = 1.2619`，挑順手的用。本書統一用 ln。

現在你手上有把尺了。它對「正常」形狀給整數，這是健全性檢查（sanity check）：線段 1、正方形 2、立方體 3，跟直覺一致。下一節我們把同一把尺、一字不改地，套到 ch12 那些自相似的怪物身上——然後分數就掉出來了。

## 把尺套上 Koch、Cantor、Sierpiński

訣竅永遠是同一句話：**在這個形狀裡，找出「它由幾份自己的縮小複製組成」，數出 N 和 s，代進 D = ln N / ln(1/s)。** 自相似形狀的好處就是 N 和 s 一眼可見，不必量、不必估。

### Koch 曲線：1 和 2 之間的 1.2619

回到 ch12 的造法。Koch 曲線從一條線段出發，每一步把每一小段中間三分之一頂出一個尖角，於是**一段變四段、每段長度是原來的 1/3**。停在這裡——你要的兩個數已經到齊了。

```text
  Koch 曲線：放大三倍看，看到 4 份一模一樣的自己
  ──────────────────────────────────────────────
  N   = 4          ← 整條曲線由 4 個縮小複製拼成
  s   = 1/3        ← 每個複製是整體的 1/3 大
  1/s = 3          ← 一份放大 3 倍才回到整體

  D = ln N / ln(1/s)
    = ln 4 / ln 3
    = 1.3863 / 1.0986          ← ln4≈1.3863、ln3≈1.0986（自己複核過）
    ≈ 1.2619
```

讀這個 1.2619。它**大於 1**：Koch 曲線比一條老實的線段更會「佔地方」——你沿著它走，永遠在拐彎、在加塞細節，它的長度其實是無限的（每代乘 4/3，發散），這個「比線更滿」的事實，被維度抓成了「大於 1」。它又**小於 2**：它再怎麼皺，也只是一條曲線，圍不出、填不滿任何一塊有面積的區域，所以「比面少」，維度小於 2。

夾在 1 和 2 之間的那個 1.2619，量的正是這份「介於」：它比線多了多少、又離面差多遠。維度第一次不是在數座標軸，而是在**量一個形狀把空間塞得多滿**。這就是震撼點，而且它能被你重講給另一個工程師聽：Koch 曲線的維度 1.2619 的意思是——把觀察尺度縮小 3 倍，要看的細節變多 4 倍，而 4 不是 3 的整數次方，是 3 的 1.2619 次方。細節增生的速率，卡在「比線快、比面慢」。

停在這個 1.2619 上多想一下，因為它一次解決了兩個本來看起來互相打架的事實。第一，ch12 說過 Koch 曲線**長度無限**——每代乘 4/3,代數趨於無窮時長度發散。第二，本章說它維度只有 1.2619,一個老老實實的有限數。「無限長」配「維度 1.26」聽起來很矛盾：無限長的東西，維度不該很大嗎？不該。長度和維度量的是兩件不同的事——長度量「總共多少」，維度量「細節以什麼速率增生」。一條普通直線可以無限長（把它畫到無窮遠）,維度照樣是 1;Koch 的無限長，也撐不起比 1.2619 更高的維度，因為它每放大三倍只多生四份細節，這個**速率**就只到 1.2619。換句話說，維度根本不在乎你總量多大，它只在乎你「每縮小一級尺度、增生多少」。這個分辨，我留到第四題讓你親手把它講成一致的一句話；這裡先把伏筆埋下，你看到 1.2619 時心裡要同時裝著「它長度無限」這個事實，而且不覺得矛盾。

### Cantor 集：0 和 1 之間的 0.6309（小心，別跟 Sierpiński 對調）

Cantor 集（Cantor set）的造法更狠：從一條線段出發，每一步**挖掉中間三分之一**,只留左右兩段。於是**一段變兩段、每段長度是原來的 1/3**。

```text
  Cantor 集：放大三倍看,看到 2 份一模一樣的自己
  ──────────────────────────────────────────────
  N   = 2          ← 挖掉中段後,只剩左右 2 個縮小複製
  s   = 1/3        ← 每段是整體的 1/3 大
  1/s = 3

  D = ln 2 / ln 3
    = 0.6931 / 1.0986          ← ln2≈0.6931、ln3≈1.0986
    ≈ 0.6309
```

這個 0.6309 **小於 1**——它比一條完整的線「少」。合理：你一直在挖洞，挖到最後留下的是一堆無窮多、卻處處不連通的點（總長度趨近 0,但點的個數不可數）。它比孤零零的幾個點（維度 0）「多」，因為它在每個尺度上都還有結構、還在分岔成更多碎屑；但它又連不成線，所以「比點多、比線少」，維度落在 0 和 1 之間。

**這裡有一個 landscape 特別標紅的陷阱，我要你停一秒**:Cantor 是 0.6309(介於 0 與 1),Sierpiński 是 1.585（介於 1 與 2）。網路上不少摘要把這兩個值對調。記法很簡單，回到形狀本身，別背數字：Cantor 是在一條**線**上挖洞，挖剩下的當然「比線少」，維度<1;Sierpiński（下面就算）是在一塊**面**上挖洞，挖剩下的「比線多、比面少」，維度在 1 和 2 之間。**用形狀錨定數字，你永遠不會把它們搞反。**

### Sierpiński 三角：1 和 2 之間的 1.585

Sierpiński 三角(Sierpiński triangle)從一個實心三角形出發，每一步把它分成四個半邊長的小三角形、**挖掉正中間那個**,留下角上三個。於是**一個變三份、每份邊長是原來的 1/2**。

```text
  Sierpiński 三角:放大兩倍看,看到 3 份一模一樣的自己
  ──────────────────────────────────────────────
  N   = 3          ← 四個小三角挖掉中間,剩 3 個縮小複製
  s   = 1/2        ← 每份邊長是整體的 1/2
  1/s = 2

  D = ln 3 / ln 2
    = 1.0986 / 0.6931          ← ln3≈1.0986、ln2≈0.6931
    ≈ 1.585
```

注意這裡的 N 和 s 跟前兩個都不同：這次是分成**兩半邊長**(s=1/2、1/s=2),不是三等分。陷阱就在這：如果你慣性地以為「碎形都是三分」、把 1/s 寫成 3,會算出 ln3/ln3=1,完全錯。**每個碎形的 N 和 s 都要回它自己的造法去數，不能套公式套到忘了看圖。**

1.585 落在 1 和 2 之間，且比 Koch 的 1.2619 更靠近 2——這也對得上眼睛：Sierpiński 三角看起來確實比 Koch 曲線「更想填滿一塊面」，它是個鏤空的三角形、本來就是二維物件挖出來的；而 Koch 只是一條被揉皺的線。維度的大小，忠實地排出了「誰更會佔面」的順序。

把三個放一起對帳，這是本章你必須能默寫的一張表：

```text
  碎形             N   s     1/s   D = lnN/ln(1/s)   落在哪
  ────────────────────────────────────────────────────────
  Cantor 集         2   1/3    3     ln2/ln3 ≈ 0.6309   0 與 1 之間  (線上挖洞,比線少)
  Koch 曲線         4   1/3    3     ln4/ln3 ≈ 1.2619   1 與 2 之間  (皺的線,比線多)
  Sierpiński 三角   3   1/2    2     ln3/ln2 ≈ 1.585    1 與 2 之間  (鏤空的面,比線多比面少)
```

三個都用**同一把尺** D = ln N / ln(1/s) 算出，差別只在各自的 N 和 s。維度不是查表查來的神祕常數，是你從造法數出兩個數、代一個公式得到的——完全可手算、完全可複核。

## 盒計數：給不那麼乖的形狀一把通用的尺

自相似維度有個前提：形狀得**嚴格自相似**——由幾份一模一樣的縮小複製拼成，N 和 s 才數得出來。Koch、Cantor、Sierpiński 是數學家為了講課特地造出來的乖寶寶，造法寫死了「分幾份、縮多少」，所以 N 和 s 一翻造法就有。可是真實世界的碎形不乖。一段海岸線、一片雲、一條勞侖次軌跡，沒有人保證它「放大三倍剛好看到四份自己」——它們是**統計自相似**(放大後「長得像」但不完全一樣),不是嚴格自相似。對這些東西，你根本指不出「這是第幾份縮小複製、縮了多少」，「N 和 s」無從數起，前一節那把尺直接用不上。

這就像你手上那段監控資料畫出來的抖動曲線：它顯然有結構、顯然不是一條平滑的線，但它不是任何一個你能寫出造法的數學物件。你需要一把**不問造法、只問尺度**的尺——拿不同粗細的網格去蓋它、數它佔幾格，看格數隨網格變細怎麼長。這把尺對乖形狀和不乖形狀一視同仁，而且——你等下會看到——它對乖形狀算出來的數，跟自相似維度一模一樣。所以它不是另一個維度，它是同一個維度的**通用量法**。

需要一把不依賴自相似、對任何形狀都管用的尺。它叫**盒計數維度(box-counting dimension)**,而它的點子，你看一眼就會覺得「這不就是我看 log-log 圖的方式嗎」。

點子是：**拿方格紙蓋住這個形狀，數有幾格被它碰到；把格子縮小，再數一次。** 格子邊長叫 ε(epsilon),被碰到的格子數叫 N(ε)。看 N(ε) 怎麼隨 ε 縮小而增長。

```text
  同一段碎形(這裡用一條皺曲線示意),用越來越小的格子去蓋,數「碰到幾格」

  格邊長 ε = 1(粗格):              格邊長 ε = 1/2(細一倍):
  ┌──┬──┬──┬──┐                   ┌─┬─┬─┬─┬─┬─┬─┬─┐
  │  │░░│  │  │                   │ │░│░│ │ │ │ │ │
  ├──┼──┼──┼──┤                   ├─┼─┼─┼─┼─┼─┼─┼─┤
  │░░│░░│░░│  │                   │░│░│ │░│░│ │ │ │
  ├──┼──┼──┼──┤                   ├─┼─┼─┼─┼─┼─┼─┼─┤
  │  │  │░░│░░│                   │ │ │░│░│ │░│░│ │
  ├──┼──┼──┼──┤                   ├─┼─┼─┼─┼─┼─┼─┼─┤
  │  │  │  │░░│                   │ │ │ │ │░│░│░│░│
  └──┴──┴──┴──┘                   └─┴─┴─┴─┴─┴─┴─┴─┘
  碰到 N(ε)≈7 格                   碰到 N(ε)≈16 格(細節露出來,格數暴增)
  (░ = 曲線經過的格子)
```

關鍵在「暴增多快」。格子邊長砍半(ε→ε/2),被碰的格子數變幾倍？

- 如果被量的是一條**直線**:格子砍半，碰到的格數大約**翻倍**(×2)。`N ∝ (1/ε)^1`,維度 1。
- 如果是一塊**實心面**:格子砍半，碰到的格數大約**變四倍**(×4)。`N ∝ (1/ε)^2`,維度 2。
- 如果是一條**Koch 曲線**:格子縮成 1/3,碰到的格數變 **4 倍**（不是 3 倍、也不是 9 倍）。`N ∝ (1/ε)^1.2619`。

於是盒計數維度的定義，跟自相似維度長得一模一樣，只是把「N 份自相似複製」換成「N(ε) 個被碰到的格子」：

```text
  N(ε) ∝ (1/ε)^D
  → D = lim   ln N(ε) / ln(1/ε)         ← ε 趨近 0(格子無限細)
        ε→0

  實作版:把多組 (ln(1/ε), ln N(ε)) 點在 log-log 圖上,
          連成一條線,量它的斜率,斜率就是 D。
```

看到了嗎——**盒計數維度就是「log(格數) 對 log（1/格邊長）」這張 log-log 圖的斜率**。這正是你看 P99 對負載、壓縮率對檔案大小那張圖的同一個動作：在雙對數座標上讀一條直線的斜率。維度從來不是什麼異星概念，它是縮放律的指數，而你讀縮放律指數的肌肉記憶，早就練好了。

而且有件事值得你放心：對嚴格自相似的形狀，**盒計數維度算出來跟自相似維度完全相等**。你可以自己驗：Koch 曲線用邊長 (1/3)ⁿ 的格子去蓋，需要 4ⁿ 格（每細一代、格數乘 4）,於是 ln(4ⁿ)/ln(3ⁿ) = ln4/ln3 = 1.2619,跟前面一字不差。兩把尺在乖形狀上對上了帳，你才敢把盡計數這把通用尺，放心拿去量不乖的東西。

```text
  Koch 曲線盒計數,逐代複核(放 ```text,自己算過)
  ─────────────────────────────────────────────
  代數 n   格邊長 ε=(1/3)ⁿ   碰到格數 N(ε)=4ⁿ   ln N / ln(1/ε)
    1        1/3              4                 ln4/ln3 = 1.2619
    2        1/9              16                ln16/ln9 = 2ln4/2ln3 = 1.2619
    3        1/27             64                ln64/ln27 = 3ln4/3ln3 = 1.2619
    4        1/81             256               ln256/ln81 = 4ln4/4ln3 = 1.2619
  每一代的 (ln(1/ε), ln N) 點都落在同一條斜率 1.2619 的直線上 → 盒計數=自相似維度
```

## 把它接回奇異吸子：2.06 是什麼意思

繞了一圈，回到 ch11 那隻勞侖次蝴蝶。我們當時說它「是碎形」，但沒給數。現在給：它的維度約 **2.06**(更精確的 Kaplan–Yorke 估計 2.062;見 ch11,數值照 landscape 2026-06)。

先別管 2.06 怎麼算出來的（那要用 ch14 的 Lyapunov 指數，這裡先不展開、指向延伸閱讀）,只問：**2.06 這個數在說什麼？** 用本章剛練好的直覺讀它——維度是縮放律，2.06 介於 2 和 3 之間，所以這隻蝴蝶**比一張面多一點點、又遠遠不到一個實心立方體**。

```text
  維度 2.06 的吸子,逐字翻譯
  ─────────────────────────────────────────────
  > 2  : 它不只是一張光滑的曲面——軌跡層層疊疊地摺,
         摺出了比「面」更密的細節結構(那個「.06」就是多出來的碎)。
  ≈ 2  : 但它幾乎攤平在一個薄薄的曲面附近,
         不像實心物體那樣把三維空間填滿(那會是 3)。
  → 「一張被摺了無限多層、卻又薄到沒有體積的紙」
```

這就對上 ch11 留下的那個謎、也預告了 ch16:軌跡要同時做到**有界**（關在一個小盒子裡）、**永不重複**（非週期）、**不自交**(決定論不准同一點有兩個未來)——三件事看似矛盾，唯一的活路是**無限不重複地摺進一個碎形集合**。那個「摺進去的集合」薄到沒有體積（所以維度 <3）,卻又皺到比面更滿（所以維度 >2）,於是維度卡在 2.06。`.06` 不是測量誤差，是「摺疊」這件事留下的指紋：摺得越狠、塞進的細節越多，維度就越往 3 爬。ch16 會告訴你這個「摺」具體是怎麼做出來的；ch13 先讓你聽懂 2.06 在說一句什麼話。

而這也是為什麼「奇異吸子」當初被命名為 strange——它 strange 在幾何上，它的維度不是整數（Ruelle–Takens 1971 鑄這個詞，見 ch11）。混沌的形狀，需要一個介於整數之間的數，才量得準。

## 直覺的陷阱

碎維度是人類整數直覺崩得最徹底的地方之一。幾個具名的坑：

| 錯誤直覺 | 它會在哪一步把你帶溝裡 | 正確版（怎麼自我察覺） |
|---|---|---|
| 「維度一定是整數，1.2619 是算錯了」 | 你會一直找哪裡算錯、不肯接受結果，卡死在第一步 | 維度是**縮放律的指數**,沒有任何規定它得是整數；細節以非整數次方增生時，維度就是分數。徵兆：你心裡在反駁「哪有 1.26 個方向」——那是把維度誤當成「座標軸個數」，維度量的是**塞滿空間的密度**,不是軸數。 |
| 「把 D = lnN/ln(1/s) 背起來、每個碎形都套 N/3」 | Sierpiński 是分成 1/2 不是 1/3,你套成 ln3/ln3=1 就全錯 | N 和 s **必須回各自的造法去數**:Cantor(2,1/3)、Koch(4,1/3)、Sierpiński(3,1/2)。公式是同一個，N 和 s 每個形狀不同。徵兆：你算 Sierpiński 前沒先看「它怎麼分」就動筆。 |
| 把 Cantor(0.6309)和 Sierpiński(1.585)的值對調 | 你會說出「Cantor 維度 1.585」這種錯話，landscape 標紅的常見網路錯誤 | 回形狀：Cantor 在**線**上挖洞→比線少→<1;Sierpiński 在**面**上挖洞→比線多比面少→1~2。徵兆：你在背數字而不是看形狀。 |
| 「Koch 維度 1.26,所以它是一條『比較粗的線』」 | 你以為分數維度是「線變粗了」，其實它寬度為零 | Koch 曲線**沒有寬度**,它是被無限揉皺的零寬曲線；1.26 量的是「皺到比線更會佔地方」，不是粗細。徵兆：你想像它有粗細。 |
| 「維度是物體的固有屬性、跟我用多細的尺無關」 | 你以為量一次就定了 | 盒計數維度是 **ε→0 的極限斜率**;真實物體（海岸線、雲）在不同尺度區間斜率會變，只有在某個尺度範圍內近似為碎形。徵兆：你忘了維度是「縮放律」，而縮放律要跨尺度才看得出來。 |

最深的一個陷阱，值得單獨點：**別把碎維度當成「物體本身的內稟整數維度」的競爭者**。Koch 曲線「住在」二維平面上（它的拓撲維度仍是 1,它是一條曲線）,但它的碎維度是 1.2619。兩個維度不矛盾、量的是不同的事：拓撲維度量「它本質上是幾維的東西（一條線）」，碎維度量「它把所在空間塞得多滿」。混沌關心的、本章關心的，是後者。本章只給碎維度的**直覺**;它和 Hausdorff 維度（用測度嚴格定義的那個版本）的精確關係，本書不展開，想看嚴格定義見延伸閱讀。

## 紙上推演

### 第一題 ★ **[10 分鐘]**：每次分成 5 份、每份縮 1/3

有一個自相似碎形，造法是：每一步把每一段換成 **5 份**縮小的複製，**每份是原來的 1/3** 大。算它的維度，並說它落在哪兩個整數之間、為什麼。

#### 推演解答

直接數 N 和 s,代公式：

```text
  N   = 5          ← 每段換成 5 份
  s   = 1/3        ← 每份是 1/3 大
  1/s = 3

  D = ln N / ln(1/s)
    = ln 5 / ln 3
    = 1.6094 / 1.0986          ← ln5≈1.6094、ln3≈1.0986(自己複核)
    ≈ 1.4650
```

落在 **1 和 2 之間**。為什麼？跟 Koch 比一下就清楚：Koch 是「1/3 縮小、塞 4 份」，這題是「1/3 縮小、塞 5 份」——**同樣的縮小尺度下塞了更多份**,細節增生得更快，所以維度比 Koch 的 1.2619 更大(1.4650)。但 5 份還沒多到把面填滿：如果是「1/3 縮小、塞 9 份」那就是 ln9/ln3=2,剛好是一塊實心面（把一個正方形切成 3×3=9 個小正方形）。塞 5 份介於「線（3 份就 ln3/ln3=1）」和「面（9 份就 2）」之間，所以維度在 1 和 2 之間，且 1<1.465<2,合理。

常見錯路：把 1/s 寫成 5（看到「5 份」就以為 1/s=5）。1/s 是**縮小比例的倒數**,題目說縮成 1/3,所以 1/s=3,跟「幾份」無關。N 是份數(5)、1/s 是放大倍率(3),兩個數來自題目的不同地方，別混。

### 第二題 ★★ **[12 分鐘]**：2.06 在說什麼

不用任何計算，只用本章的維度直覺，向一個沒讀過這本書的後端同事解釋：勞侖次吸子的維度約 2.06,這個「2.06」到底在描述什麼？為什麼它**比 2 大、又比 3 小**這件事是有意義的，不是隨便一個落在中間的數？

#### 推演解答

要講的核心是「維度是縮放律，2.06 是塞滿空間的密度」，分三句：

1. **比 2 大的意思**:它不是一張光滑的曲面。如果它只是一張面，維度會剛好是 2。它的軌跡在相空間裡層層疊疊地摺，摺出了比「一張面」更密的細節——你放大去看，會看到面裡還有面、層裡還有層（像翻開一疊薄紙）。那個多出來的 `.06`,就是這些摺出來的額外細節貢獻的。

2. **比 3 小的意思**:但它又遠不到一個實心的三維物體。實心立方體維度是 3,會把一塊三維空間填滿。勞侖次吸子薄得幾乎沒有體積——它攤在一個薄薄的曲面附近，只是這曲面被摺了無限多層。所以它「比面多、比體少」，卡在 2 和 3 之間。

3. **為什麼這個「介於」有意義**:因為它精確對應一個物理事實——軌跡必須同時**有界**（關在盒子裡）、**永不重複**（混沌）、**不自交**（決定論）。要三者並存，軌跡只能無限不重複地摺進一個薄而皺的集合。維度 2.06 就是這個「薄(<3)而皺(>2)」的量化指紋。換個摺法更狠的系統，維度會更靠近 3。所以 2.06 不是隨便的中間值，它是這隻蝴蝶**摺得多狠**的度量。

可以收一句類比：「2.06 就像一張被揉到極致、卻薄到秤不出重量的紙——比一張平整的紙更會佔地方，但又永遠不是一塊實心的磚。」

常見錯路：說成「2.06 是因為它在三維空間裡畫圖、但只用了一部分」。那是把「它住在幾維空間」（那是 3）跟「它自己有多滿」（那是 2.06）混為一談——維度量的是後者。

### 第三題 ★★ **[15 分鐘]**：盒計數就是 log-log 斜率

你有一段監控資料畫成的曲線（先別管它是不是碎形）。你拿邊長 ε 的方格去蓋它、數碰到幾格 N(ε),得到下面四組數。用 log-log 斜率估它的盒計數維度，並判斷它「比較像線、還是比較像在填面」。

```text
  ε        N(ε)
  ────────────────
  1/2      5
  1/4      14
  1/8      38
  1/16     104
```

#### 推演解答

盒計數維度 = ln N(ε) 對 ln(1/ε) 的斜率。先把每組換成 (ln(1/ε), ln N):

```text
  ε      1/ε    ln(1/ε)         N(ε)   ln N(ε)
  ──────────────────────────────────────────────
  1/2     2     ln2  = 0.6931    5      ln5   = 1.6094
  1/4     4     ln4  = 1.3863    14     ln14  = 2.6391
  1/8     8     ln8  = 2.0794    38     ln38  = 3.6376
  1/16   16     ln16 = 2.7726    104    ln104 = 4.6444
```

取頭尾兩點算斜率（也可逐段算、取平均，差不多）:

```text
  斜率 = (4.6444 − 1.6094) / (2.7726 − 0.6931)
       = 3.0350 / 2.0795
       ≈ 1.4595
```

維度約 **1.46**。判斷：它在 1 和 2 之間，而且明顯**比一條直線（維度 1）更會佔地方**——格子每細一倍，碰到的格數增長得比「翻倍」更快（從 5 到 14 是 2.8 倍、不是 2 倍）,這正是「比線滿」的訊號。但它離填滿一塊面（維度 2,格數該每細一倍變四倍）還有距離。所以這段曲線「比線皺、但還沒滿到像在塗一塊面」——一個典型的碎形維度。

順帶一個工程橋接：你剛剛做的事，跟你看「P99 延遲對負載」的 log-log 圖、量斜率判斷是線性還是平方，**是同一個動作**。盒計數維度不是新技能，是你既有的 log-log 直覺套在「格數對尺度」上。

常見錯路：把 ε（會變小）當橫軸去算斜率，得到負數。橫軸要用 **ln(1/ε)**(隨格子變細而變大),維度才是正的斜率；這也呼應公式裡寫的是 ln(1/ε) 不是 ln ε。

### 第四題 ★★★ **[18 分鐘]**：為什麼 Koch 長度無限、維度卻有限

ch12 說 Koch 曲線長度無限（每代乘 4/3,發散）,本章又說它維度只有 1.2619（有限）。一個是無限、一個是有限，這兩件事矛盾嗎？把它們講成一致的一句話。(提示：長度是用「一維的尺（線段）」去量，維度是問「該用幾維的尺去量才量得出有限值」。)

#### 推演解答

不矛盾，而且它們講的是同一件事的兩面。關鍵是想清楚「用幾維的尺去量」：

- **用一維的尺（長度）去量 Koch**,結果是無限。因為 Koch 比一條真正的一維線「更滿」，你拿「線」這把太低維的尺去量它，尺永遠跟不上它增生的細節，量出來發散到無限。這正像你拿一條線去量一塊面的「長度」——當然無限（面裡塞得下無限長的線）。**用低於物體真實維度的尺去量，得無限。**

- **用二維的尺（面積）去量 Koch**,結果是零。Koch 曲線沒有寬度、圍不出面積，你拿「面積」這把太高維的尺去量它，量出來是 0。**用高於物體真實維度的尺去量，得零。**

- **那「對」的尺是幾維？** 既不是 1（量出無限）、也不是 2(量出零),而是夾在中間、唯一能量出**有限非零值**的那個維度——**1.2619**。碎維度的深層定義（這就是 Hausdorff 維度的精神，嚴格版見延伸閱讀）正是：**那個讓「測度」從無限掉到零的臨界維度**。低於它測度是無限、高於它測度是零、剛好在它測度是有限非零。

所以「長度無限」和「維度 1.2619」一點都不打架：長度無限，正是因為你用了一維的尺去量一個 1.2619 維的東西——尺太低維了，當然爆。維度 1.2619 恰恰是在告訴你「該換哪一維的尺，才量得出有限的量」。把這句話講出來，你就同時懂了「碎形為什麼長度無限」和「碎維度到底在定義什麼」。

常見錯路：覺得「無限長的東西維度不該只有 1.26、該很大」。長度大小和維度高低是兩回事——維度量的是「細節隨尺度增生的指數」，不是「總量多大」。一條普通直線可以無限長（維度仍是 1）,Koch 的無限長也撐不起更高的維度，因為它增生的**速率**（乘 4/3 對應的指數）就只到 1.2619。

- 跨書：log-log 斜率與冪律的同一種縮放律直覺，見《馴服隨機》談冪律(power law)那章；對數本身的縮放性質要從頭擦亮，見《馴服無限》談對數那章。維度的縮放指數、冪律的指數、對數座標的斜率，是同一個數學動作的三張臉。
