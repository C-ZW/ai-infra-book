# ch06 — 不動點與穩定：斜率小於一的法則

> **本章解決什麼問題**：ch05 把脊椎遞迴式 xₙ₊₁ = r·xₙ·(1−xₙ) 端上來，你在 cobweb（蛛網）圖上親眼看著 r=2.5 的軌跡收進一個值。但 ch05 只說「它收斂了」，沒回答三個你一定會追問的問題：它**停在哪**？**為什麼**停得住？換個 r 為什麼就停不住？這一章把這三問變成一條能手算、能一眼判定的法則——不動點（fixed point）是 x* = f(x*) 的解，而它吸不吸引人，全看曲線在交點處的**斜率絕對值有沒有小於 1**。這是整本穩定性分析的地基。把這條法則焊死，ch07 旋鈕一過 3、不動點突然留不住軌跡、世界一分為二，你會覺得是早就算到的、不是變魔術。

## 從你已知的出發

你一定見過一個服務「停不下來」。不是當機那種停不下來，是**震盪**那種：監控圖上某個量——副本數、佇列長度、連線池大小——不肯安靜地停在一個值，而是來回甩，甩出一條鋸齒。

最典型的是 autoscaler 的 flapping。你設了一條規則：CPU 高於某個門檻就加副本、低於就減。流量一抖，它加了一批；新副本起來後單機負載掉下去，它判定「太多了」又砍一批；砍完負載又頂上來，它再加……於是副本數在 N 和 2N 之間無止盡地跳，每次擴縮都付一次冷啟動的代價，使用者吃一次抖動。你那天 3am 被叫起來，做的第一件事多半是**把擴縮的反應力道調軟**：拉長冷卻時間、降低每次擴縮的幅度、加一點遲滯（hysteresis）。也就是——**降增益、加阻尼**。

調軟之後它就乖乖停在某個副本數了。你憑經驗知道一條鐵律：**修正的力道太強，系統會在目標值兩側過衝、來回甩、甚至越甩越大；力道夠軟，偏差才會一輪輪被吃掉、靜靜收斂。** 這條經驗你信，但你大概沒把它寫成一個數。本章要做的就是這件事：把「力道太強就震盪」變成一個你能代數計算、看一眼就知道穩不穩的判據。

兩個物理畫面先擺在這裡，整章都圍著它們轉：

```text
  穩定不動點 = 碗底的彈珠         不穩定不動點 = 筆尖朝下立著的鉛筆
        \         /                          |
         \       /                           |
          \. ● ./   ← 推一下，自己滾回來       ●  ← 碰一下，立刻倒，回不去
           \___/                          ___|___
       擾動被「壓回」                     擾動被「放大」
       吸引（attracting）                排斥（repelling）
```

兩個位置都是「平衡點」——理論上放準了都能停。差別在**對擾動的反應**：碗底的彈珠，你推它一下，它自己滾回去；倒立的鉛筆，你碰它一絲，它倒下去再也回不來。混沌穩定性分析的全部祕密，就藏在「擾動被壓回、還是被放大」這一念之差裡。而決定是壓回還是放大的那個量，是曲線在平衡點處的**斜率**。

## 不動點：先找出「停得下來的地方」

第一步，把「系統可能停在哪」找出來。**不動點（fixed point）** 的定義乾淨到不能再乾淨：餵進去等於吐出來、迭代下去不再變的那個值。

```text
        x* = f(x*)        ← 不動點：迭代一次（一百次、一萬次）都不動的值
```

工程直覺：不動點就是回授迴圈的**穩態候選**。autoscaler 真的停在某個副本數，是因為「當前負載算出來的目標副本數」剛好等於「當前副本數」——餵進去等於吐出來，這一輪算出的結果和上一輪一樣，於是它不動了。注意我說的是「候選」：找到不動點，只代表「這裡停得下來」，不代表「軌跡真的會去這裡」。會不會去，是穩定性的事，下一節算。這一節只負責把候選位置全部撈出來。

把脊椎遞迴式 f(x) = r·x·(1−x) 代進定義，解 x* = r·x*·(1−x*)，一步都不跳：

```text
  x*           = r·x*·(1 − x*)
  x* − r·x*·(1 − x*) = 0              ← 移到同一邊
  x*·[1 − r·(1 − x*)] = 0            ← 把公因子 x* 提出來
```

兩個因子相乘等於零，必有一個是零，這給我們兩個不動點：

```text
  因子一為零：x* = 0                    ← 「滅絕點」：空的系統，下一代還是空的
  因子二為零：1 − r·(1 − x*) = 0
              r·(1 − x*) = 1
              1 − x*     = 1/r
              x*         = 1 − 1/r     ← 「共存點」：人口穩定在某個正比例
```

第一個 x* = 0 永遠在那兒，跟 r 多大無關——沒有人口，餵 0 進去吐 0 出來。第二個 x* = 1 − 1/r 才是主角，它隨旋鈕 r 滑動：

```text
  r = 1.5  →  x* = 1 − 1/1.5  = 1 − 0.6667 = 0.3333
  r = 2.0  →  x* = 1 − 1/2.0  = 1 − 0.5000 = 0.5000
  r = 2.5  →  x* = 1 − 1/2.5  = 1 − 0.4000 = 0.6000   ← ch05 那條軌跡收斂的目標
  r = 3.0  →  x* = 1 − 1/3.0  = 1 − 0.3333 = 0.6667
  r = 3.2  →  x* = 1 − 1/3.2  = 1 − 0.3125 = 0.6875
```

有個邊界值得停一下：當 **r < 1** 時，1 − 1/r 算出來是**負數**，掉出了 [0,1] 這個合法區間。人口比例不可能是負的——所以 r<1 時共存點根本不存在，系統只剩 x*=0 一個去處。這跟生態直覺完全合拍：成長率 r 連 1 都不到，連自我替補都做不到，人口必然歸零。等下算穩定性時會看到，r<1 時 x*=0 確實是穩定的吸引點——族群穩穩地滅絕。

好，停得下來的地方找到了：x*=0 和 x*=1−1/r。但 ch05 你看到的是 r=2.5 的軌跡**真的**奔向 0.6、賴著不走；而我馬上要給你看 r=3.2 的軌跡，它明明也有個不動點 0.6875，卻死活不肯停在那裡。同樣是不動點，一個留得住軌跡、一個留不住——差別在哪？答案是斜率，下一節。

## 穩定判據：誤差每輪乘一個固定的數

把「碗底彈珠 vs 倒立鉛筆」翻成代數，只需要問一個問題：**這一輪偏離不動點一點點，下一輪偏離多少？**

設這一輪你離不動點有個小偏差 εₙ，也就是 xₙ = x* + εₙ。下一輪：

```text
  xₙ₊₁ = f(xₙ) = f(x* + εₙ)
        ≈ f(x*) + f′(x*)·εₙ      ← 在 x* 附近，把曲線當它的切線（一階近似）
        = x*     + f′(x*)·εₙ      ← 因為 x* 是不動點，f(x*) = x*

  所以  εₙ₊₁ = xₙ₊₁ − x* ≈ f′(x*)·εₙ
```

（這裡用到「導數＝瞬時斜率」——「x 動一點點，f 動幾倍」的那個倍率。生鏽的話，這就是把彎曲的曲線在一點附近**當直線看**的近似；要從頭擦亮見《馴服無限》談導數的章。）

讀最後一行，慢慢讀，它是本章一切的源頭：**下一輪的偏差，等於這一輪的偏差乘上 f′(x*)。** 那個 f′(x*) 就是「誤差倍率」——每過一輪，你離不動點的距離被乘上這個固定的數。再迭代下去：

```text
  ε₁ ≈ f′(x*)·ε₀
  ε₂ ≈ f′(x*)·ε₁ ≈ f′(x*)²·ε₀
  ε₃ ≈ f′(x*)³·ε₀
  …
  εₙ ≈ f′(x*)ⁿ·ε₀         ← 偏差被乘了 n 次同一個倍率
```

到這裡判據自己就跳出來了。一個數被自己連乘 n 次，命運只看它的絕對值跟 1 比：

```text
  |f′(x*)| < 1   →   εₙ → 0       穩定（吸引）：偏差一輪輪縮小，軌跡彈回不動點   = 碗底彈珠
  |f′(x*)| > 1   →   εₙ → ∞       不穩定（排斥）：偏差一輪輪放大，軌跡被甩開     = 倒立鉛筆
  |f′(x*)| = 1   →   εₙ 不縮不脹   臨界：一階近似看不出來，得問更高階——分岔要發生
```

**為什麼是絕對值？** 因為斜率可以是負的。f′(x*) 是負數時，εₙ 每輪會變號——偏差從正跳到負、再跳回正——軌跡在不動點兩側來回甩。但只要絕對值小於 1，每次甩過去的幅度都比上次小，最後照樣收進去，只是用「螺旋逼近」而不是「直線滑進」的方式。這正對應你調控制迴圈時「有點過衝、但會穩下來」那種手感。所以**符號決定逼近的姿勢（單調滑入 vs 來回螺旋），絕對值決定生死（收斂 vs 發散）**。判據裡的那個 |·| 不是裝飾，少了它你會把振盪式收斂誤判成發散。

到這裡，你心裡那條 autoscaler 的鐵律已經有了精確版：那個「修正力道」就是 f′(x*)，「力道太強就震盪」就是 |f′(x*)|>1。剩下的，是把這把尺套到脊椎遞迴式上，看數字長什麼樣。

## cobweb 的幾何：誰比對角線陡

代數有了，但這一章值得你停十分鐘的是它的**幾何**——因為一旦你在 cobweb 圖上看懂「斜率」，就再也不會忘記這條判據。

cobweb 圖怎麼擺：橫軸 xₙ、縱軸 xₙ₊₁，畫上曲線 y = f(x)，再畫上對角線 y = x。兩者的交點就是不動點——因為在交點上 y=f(x) 同時 y=x，於是 f(x)=x，正是不動點定義。迭代的軌跡在「曲線」和「對角線」之間階梯狀彈跳（往曲線量出下一個值、往對角線把它搬回橫軸、再往曲線……）。關鍵只有一句：**曲線在交點處比對角線陡，還是比對角線緩。**

對角線 y=x 的斜率永遠是 1。所以「曲線比對角線緩」就是 |f′(x*)|<1，「曲線比對角線陡」就是 |f′(x*)|>1。看圖：

```text
  情況 A：曲線在交點處比對角線「緩」（|f′(x*)|<1）→ 階梯收縮 → 穩定

      xₙ₊₁
        |              / y = x  ← 對角線，斜率恆為 1
        |            /
        |   曲線 __--●           交點處曲線比對角線「躺」
        |     _--‾  /|           （|斜率| < 1）
        |   _-  ┌──┘ |           軌跡一格一格縮進交點
        | _-   ┌┘    |           每一步離 x* 更近
        |/_____┴_____|______ xₙ
                    x*
        看點：曲線比對角線躺 → 反彈幅度一次比一次小 → 收進去（碗底彈珠）

  情況 B：曲線在交點處比對角線「陡」（|f′(x*)|>1）→ 階梯張開 → 不穩定

      xₙ₊₁
        |          |        / y = x
        |          |      /
        |   曲線    |    /          交點處曲線比對角線「立」
        |      ╲   |  /            （|斜率| > 1）
        |       ╲  |/             軌跡一格一格張開
        |        ╲╱|              每一步離 x* 更遠
        |________╱┴________ xₙ
                 x*
        看點：曲線比對角線立 → 反彈幅度一次比一次大 → 甩出去（倒立鉛筆）
```

這就是 |f′(x*)|<1 的純幾何意義，乾淨到不需要任何代數：**光看曲線和對角線在交點處誰比較陡，就知道不動點吸不吸引。** 對角線是那把固定的尺（斜率 1），曲線比它躺就壓縮誤差、比它立就放大誤差。分界線——「斜率絕對值剛好等於 1」——正是穩定崩潰的那一刻。記住這張圖，後面所有「不動點失穩」的故事，本質上都是「曲線在交點處變得比對角線陡了」。

## 把判據套上脊椎遞迴式：f′(x*) = 2 − r

現在算脊椎遞迴式的斜率。f(x) = r·x·(1−x)，先乘開再逐項微分，一步不省：

```text
  f(x)  = r·x·(1 − x)
        = r·x − r·x²            ← 乘開
  f′(x) = r − 2·r·x             ← 逐項微分：r·x 的導數是 r、r·x² 的導數是 2r·x
        = r·(1 − 2x)            ← 把 r 提出來
```

得到斜率函數 f′(x) = r(1 − 2x)。現在把非零不動點 x* = 1 − 1/r 代進去——這幾步建議你跟著筆算一遍，因為等下自我檢核會要你不看書重推：

```text
  f′(x*) = r·(1 − 2·x*)
         = r·(1 − 2·(1 − 1/r))      ← 代入 x* = 1 − 1/r
         = r·(1 − 2 + 2/r)          ← 括號內展開：2·(1 − 1/r) = 2 − 2/r
         = r·(−1 + 2/r)             ← 1 − 2 = −1
         = −r + 2                   ← 分配：r·(−1) = −r、r·(2/r) = 2
         = 2 − r                    ← 整理
```

整條曲線、整段 r，繞了一圈，不動點處的斜率收成乾乾淨淨的一個式子：

```text
        f′(x*) = 2 − r
```

漂亮得有點過分——所有 r 和 x 的糾纏全消掉了。把它塞進 |f′(x*)|<1 的判據，解出穩定區：

```text
  |2 − r| < 1
  −1 < 2 − r < 1                ← 拆絕對值
  −1 − 2 < −r < 1 − 2           ← 三邊各減 2
  −3 < −r < −1
   3 > r > 1                    ← 三邊乘 −1，不等式方向反轉（乘負數）
```

也就是：

```text
        1 < r < 3   ⟺   非零不動點 x* = 1 − 1/r 穩定（吸引）
```

把幾個關鍵 r 挑出來，每個都對得上前面的直覺：

```text
  r → 1⁺ ：f′(x*) = 2 − 1   = +1     ← |f′|=1，臨界下界；r<1 時 x* 出界，改由 x*=0 接管
  r = 2  ：f′(x*) = 2 − 2   =  0     ← 斜率全平！收斂快到離譜的「超穩定點」
  r = 2.5：f′(x*) = 2 − 2.5 = −0.5   ← 負斜率、絕對值 0.5 < 1，振盪式收斂（穩）
  r = 3  ：f′(x*) = 2 − 3   = −1     ← |f′|=1，臨界上界，穩定就此終結 → ch07 接手
```

r=2 那個 f′(x*)=0 特別值得玩味。斜率是 0，意思是不動點附近曲線是**平的**——一階近似下偏差被乘以 0，等於一步就被吃掉一大半，收斂快到不可思議。這叫**超穩定點（superstable point）**，是整個穩定區裡「阻尼最足」的那個 r。從 r=2 往兩邊走，斜率絕對值都變大（往 r=1 是 +1、往 r=3 是 −1），收斂越來越慢，直到兩端 |f′| 頂到 1，穩定崩潰。換句話說，**穩定區是有上下限的**：r 太小（<1）連存活都做不到，r 太大（>3）修正力道過猛而失穩，只有中間 1<r<3 這段，誤差才被壓縮。

也別漏了符號隨 r 的翻轉：r<2 時 f′(x*)=2−r>0（正斜率，軌跡單調滑進不動點）；r>2 時 f′(x*)=2−r<0（負斜率，軌跡來回振盪著進不動點）。所以 r=2.5 的軌跡會「這輪偏高、下輪偏低」地夾逼 0.6，不是直直滑進去——這個細節 worked example 會親眼驗給你看，它正是 ch05 那張迭代表為什麼忽高忽低的根由。

順手把滅絕點 x*=0 也算了：f′(0) = r·(1 − 0) = r。所以 |f′(0)|=r，r<1 時 x*=0 穩定（族群穩穩滅絕），r>1 時 x*=0 失穩（一丁點人口就被放大、逃離 0）。兩個不動點剛好在 r=1 交班：r<1 滅絕點吸引、共存點出界；r>1 滅絕點排斥、共存點接棒當吸引點——一路撐到 r=3，共存點也撐不住為止。

## Worked example：r=2.5 穩、r=3.2 失穩，手算到穿

判據是一回事，親手把兩個 r 算穿、而且跟 ch05 的迭代表對上，又是一回事。下面兩個情況的數字我都重算過、保留 4 位小數。

**情況一：r=2.5。** 先算不動點與斜率：

```text
  x*       = 1 − 1/2.5 = 1 − 0.4 = 0.6
  f′(x*)   = 2 − 2.5            = −0.5
  |f′(x*)| = 0.5 < 1             ← 判定：穩定，且負斜率（預言振盪式逼近）
```

判據說穩定、且來回振盪。手動迭代驗（這是 ch05 那條遞迴式，x₀=0.1，每步重算一次）：

```text
  n   xₙ        計算 xₙ₊₁ = 2.5·xₙ·(1 − xₙ)                   離 x*=0.6 的偏差 εₙ
  0   0.1000    2.5 × 0.1000 × 0.9000 = 0.2250               −0.3750
  1   0.2250    2.5 × 0.2250 × 0.7750 = 0.4359               −0.1641   ← 還是負，在縮
  2   0.4359    2.5 × 0.4359 × 0.5641 = 0.6147               +0.0147   ← 變號了！跨過 x*
  3   0.6147    2.5 × 0.6147 × 0.3853 = 0.5921               −0.0079   ← 又翻回負，幅度更小
  4   0.5921    2.5 × 0.5921 × 0.4079 = 0.6038               +0.0038
  5   0.6038    2.5 × 0.6038 × 0.3962 = 0.5981               −0.0019
  6   0.5981    2.5 × 0.5981 × 0.4019 = 0.6010               +0.0010
        … 偏差絕對值一路縮、正負交替 → 螺旋收進 0.6
```

兩件事一次對上：(1) 它**收斂到 0.6**，與 x*=1−1/2.5=0.6 分毫不差；(2) 從第 2 步起偏差開始**正負交替**（−0.1641 → +0.0147 → −0.0079 → +0.0038 → …），正是 f′(x*)=−0.5 那個負斜率預言的振盪式逼近。再看一個漂亮的細節：越靠近不動點，每輪偏差比上一輪的比例就越接近 |−0.5|=0.5（−0.0079 → +0.0038，約一半；−0.0019 → +0.0010，約一半）。為什麼遠的時候不準、近的時候才準？因為 εₙ₊₁≈f′(x*)·εₙ 是**切線近似**，遠離不動點時曲線不像它的切線，越靠近才越像。這就是 ch05 那張收斂表的機制：它不是「碰巧」收斂，是 |f′(x*)|=0.5<1 逼它收斂、−0.5 的負號逼它螺旋著收。

**情況二：r=3.2。** 同樣先算：

```text
  x*       = 1 − 1/3.2 = 1 − 0.3125 = 0.6875
  f′(x*)   = 2 − 3.2              = −1.2
  |f′(x*)| = 1.2 > 1               ← 判定：不穩定（排斥）
```

判據說不穩定。先強調一件容易被跳過的事：x*=0.6875 **仍然是貨真價實的不動點**——餵 0.6875 進去，吐出來還是 0.6875（驗：3.2 × 0.6875 × 0.3125 = 0.6875，對）。它不是「不存在」，它是「**排斥**」：只要起點不是精準的 0.6875，哪怕差一丁點，偏差就被每輪乘上 −1.2 而放大，軌跡被推離。手動迭代看它跑去哪（x₀=0.5）：

```text
  n   xₙ        計算 xₙ₊₁ = 3.2·xₙ·(1 − xₙ)
  0   0.5000    3.2 × 0.5000 × 0.5000 = 0.8000
  1   0.8000    3.2 × 0.8000 × 0.2000 = 0.5120
  2   0.5120    3.2 × 0.5120 × 0.4880 = 0.7995
  3   0.7995    3.2 × 0.7995 × 0.2005 = 0.5129
  4   0.5129    3.2 × 0.5129 × 0.4871 = 0.7995
  5   0.7995    3.2 × 0.7995 × 0.2005 = 0.5130
  6   0.5130    3.2 × 0.5130 × 0.4870 = 0.7995
        … 它不收進 0.6875，而是穩穩地在 0.5130 和 0.7995 兩值間來回跳
```

軌跡沒停在不動點，而是停進了一個**在兩個值間交替的循環**——0.5130、0.7995、0.5130、0.7995……這就是 ch07 要正式登場的 2-cycle（週期 2 軌道）。本章只交代一件事：**為什麼 x*=0.6875 留不住軌跡**——因為 |f′(x*)|=1.2>1，它從吸引翻成了排斥。r 從 2.5 升到 3.2，跨過了 r=3 那道 |f′|=1 的牆，不動點失穩，世界準備一分為二。分裂後為什麼是「跳兩個值」、為什麼是「加倍」而不是「加一」，留給 ch07。

兩個情況對帳一下，整章濃縮成這張表：

```text
  r      x* = 1−1/r    f′(x*) = 2−r    |f′(x*)|     判定        軌跡實況
  2.5    0.6000        −0.5            0.5 < 1      穩定        螺旋收斂到 0.6
  3.2    0.6875        −1.2            1.2 > 1      不穩定      跳成 2-cycle（0.5130↔0.7995），留不住
```

同一條遞迴式、同一個公式 f′(x*)=2−r，只因為旋鈕從 2.5 轉到 3.2，一個收斂、一個失穩。**穩定性不是系統的固有性質，是旋鈕的函數**——這是本章我最想塞進你腦子的一句話。你的 autoscaler 穩不穩，從不是「它本來就穩／本來就會 flapping」，而是你把那個誤差倍率（增益）調到了 1 的哪一邊。

## |斜率|<1 就是「迴路增益<1」：控制工程的同一條法則

把這章的 |f′(x*)|<1 和你的控制迴圈擺在一起看，你會發現它們根本是同一條法則換了套衣服。

一個比例（proportional）回授迴圈——P 控制器——的精神是：量出當前偏差，乘一個增益（gain）Kp，當作這一輪的修正量。整條迴圈走一圈，偏差被放大或縮小的那個總倍率，叫**迴路增益（loop gain）**。控制工程的入門鐵律是：

- **迴路增益絕對值 < 1**：擾動繞迴圈一圈後變小，一輪輪被吃掉，系統穩穩坐回設定點。
- **迴路增益絕對值 > 1**：擾動繞一圈後變大，一點點偏差越放越大，系統開始過衝、來回甩、震盪甚至發散。
- **增益調太兇**：這正是工程師熟到不行的場景——Kp 一路往上加，系統反應越來越快，過了某個點開始 overshoot，再上去就持續振盪。教科書調 PID 的標準動作之一，就是把 Kp 往上推到輸出剛開始振盪，那一刻迴路增益就頂到了臨界。

把這三條和本章對齊：脊椎遞迴式在不動點附近，「迴路增益」就是 f′(x*)=2−r。

```text
  控制迴圈                         脊椎遞迴式
  ─────────────────────────       ─────────────────────────
  迴路增益 G                       誤差倍率 f′(x*) = 2 − r
  |G| < 1 → 穩、收回設定點          |f′(x*)| < 1 → 穩、收進不動點（1 < r < 3）
  |G| > 1 → 震盪、過衝、發散         |f′(x*)| > 1 → 失穩、跳成週期軌（r > 3）
  增益太高 → flapping              r 過 3 → 不動點留不住 → 2-cycle
  加阻尼／降增益 → 重新穩住          把 r 調回 1<r<3 → 重新穩住
```

所以你那天 3am「降增益、加阻尼」把 autoscaler 救回來，和「把 r 從 3.2 調回 2.5」讓脊椎重新收斂，**在數學上是同一個動作**：把那個決定誤差被放大還是被壓縮的局部倍率，從 1 的外面挪回 1 的裡面。增益太高就 flapping，和 r 過 3 不動點留不住，是同一件事的兩個說法。下次你在調某個會震盪的控制迴圈，心裡可以默念：我在找 |f′|<1 的那一側。

（一個要守住的邊界：這條 |·|<1 是**離散**迭代的判據。連續時間的微分方程系統判據不同——看特徵值的**實部是否為負**，不是模是否小於 1。本書脊椎是離散迭代，所以一律用「模<1」；別把兩套混用。連續時間的版本指向《馴服無限》談微分方程穩定性的地方。）

## 一維只是特例：多維看 Jacobian 特徵值

脊椎遞迴式只有一個 x，所以「斜率」就是一個數 f′(x*)，判據就是一句 |f′(x*)|<1。但真實系統常是多維的——Lorenz 系統三維（ch11 會見到），你的服務有好幾個彼此耦合的狀態量。多維時「斜率」升級成一個矩陣：Jacobian（雅可比矩陣），它記錄「每個方向的偏差，會怎麼影響每個方向的下一輪偏差」。

判據的**形狀完全沒變**，只是「斜率」換成「矩陣的特徵值」：

```text
  一維：不動點穩定 ⟺ |f′(x*)| < 1
  多維：不動點穩定 ⟺ Jacobian 在該點的「所有」特徵值，模（絕對值）都 < 1
```

特徵值就是「矩陣往某個特徵方向把向量拉伸幾倍」的那個倍率（見《矩陣是動詞》談特徵值的章）。模小於 1，代表那個方向的偏差被壓縮；所有特徵方向都被壓縮，整個鄰域就被吸回不動點。只要**有一個**特徵值的模大於 1，那個方向的偏差就被放大，不動點就失穩——這跟一維裡 |f′|>1 失穩是同一回事，只是現在「失穩」可能只發生在某些方向上（這正是後面奇異吸子「某些方向壓縮、某些方向拉伸」的伏筆）。一維的 f′(x*) 不過是這套理論退化到一個方向時的特例——一維的「唯一特徵值」就是 f′(x*) 本身。這裡只給直覺、不推導；要看線性化分析在多維怎麼算，去《矩陣是動詞》。

## 局部的尺、全域的尺：ch14 的伏筆

最後埋一顆種子，因為本章這把 |f′(x*)|，會在 ch14 長成全書最核心的一個數。

注意本章我們問的是一個**很侷限**的問題：軌跡待在**某一個**不動點附近、走**一輪**，偏差被乘幾倍？答案是 f′(x*)，一個跟位置綁定、靜止的數。但混沌系統的軌跡**不待在不動點**——它一路亂跑，每一步都踩在曲線的不同位置，每個位置的斜率 f′(xₙ) 都不一樣。那要怎麼問「偏差長期被放大還是壓縮」？

答案是把「每一步當下的 ln|f′(xₙ)|」沿著整條軌跡**長期平均**起來，這個平均值就是 Lyapunov 指數 λ（ch14 主角）：

```text
  ch06 這一章：|f′(x*)|        ← 局部、靜止：一個不動點、一輪的誤差倍率
  ch14 的 λ ：ln|f′(xₙ)| 的長期平均  ← 全域、運動：軌跡一路跑，沿途斜率的對數平均
```

λ>0（平均下來在放大）就是混沌、λ<0（平均下來在壓縮）就是穩定——和本章 |f′|>1 失穩、|f′|<1 穩定是同一把尺，只是從「站在不動點量一次」變成「跟著軌跡跑一路、沿途量、取平均」。**ch06 是那把尺的局部靜止版，ch14 是它的全域運動版。同一把尺。** 把這句話記住，等你到 ch14 看到 r=4 時 λ=ln2≈0.6931（每步誤差約翻倍），你會認得它就是 f′(x*) 那個倍率的長大版本。這也順帶把本書中央那條張力擺上檯面：同一條確定的式子，旋鈕在 1<r<3 是「誤差被壓回」的乖巧不動點，旋鈕到 r=4 卻是「誤差被翻倍」的敏感混沌——敏感與秩序，本就藏在同一個 f′ 的不同取值裡。

## 紙上推演

### 推演題 1 ★ **[10 分鐘]**

給 r=2.8。(a) 算非零不動點 x* 與斜率 f′(x*)；(b) 判定穩定還是不穩定，並說它會以「單調」還是「振盪」的方式逼近不動點；(c) 不做完整迭代，只用斜率，估「離不動點的偏差大約幾輪縮一半」，並說明為什麼這比 r=2.5 慢。

#### 推演解答

(a) 直接套公式：

```text
  x*     = 1 − 1/2.8 = 1 − 0.3571 = 0.6429
  f′(x*) = 2 − 2.8              = −0.8
```

(b) |f′(x*)| = 0.8 < 1 → **穩定**。斜率是**負的**（−0.8），所以偏差每輪變號 → **振盪式**逼近：軌跡在 0.6429 兩側來回跳著收進去，不是直直滑進。

(c) 每輪偏差乘 |−0.8|=0.8。要縮到一半，問 0.8ⁿ ≈ 0.5：

```text
  0.8¹ = 0.8000
  0.8² = 0.6400
  0.8³ = 0.5120     ← 最接近 0.5
  0.8⁴ = 0.4096
```

所以**大約 3 輪偏差縮一半**。這比 r=2.5 慢——r=2.5 時每輪乘 0.5，**一輪**就縮一半。原因：r=2.8 更靠近失穩邊界 r=3，斜率絕對值 0.8 更大、阻尼更弱、收斂更慢。常見錯路：忘了取絕對值，看到 −0.8 就喊「不穩定」；或忘了負斜率代表振盪，誤以為單調滑入。

### 推演題 2 ★★ **[15 分鐘]**

口頭（寫下要點）解釋：**為什麼 r=3 是臨界點？** 不要只說「因為 |f′|=1」。要講清楚三層：這在 cobweb 圖上長什麼樣、為什麼一階近似在這裡失效、以及這個臨界為什麼預告了一場**質變**而不只是「收斂變慢」。

#### 推演解答

要點，三層：

1. **代數層**：r=3 時 f′(x*)=2−3=−1，所以 |f′(x*)|=1，剛好踩在穩定判據 |f′|<1 的邊界上。

2. **cobweb 幾何層**：|f′|=1 表示曲線在交點處跟對角線**一樣陡**（只是斜率是 −1、方向相反）。階梯既不收縮也不張開——偏差每輪乘 −1，幅度**不變、只變號**。軌跡繞著不動點打轉，既不掉進去、也不甩出來，卡在生死之間。

3. **一階近似失效層**：判據 εₙ₊₁≈f′(x*)·εₙ 是把曲線當切線的**一階**近似。當 |f′(x*)|=1，一階項剛好「平手」，真正決定命運的是被我們丟掉的**更高階**項（曲線的彎曲）。所以光看斜率（一階資訊）已經判不出穩不穩——這就是「臨界」的精確意思：**一階資訊用罄**。

4. **為什麼是質變不是量變**：r<3 時軌跡收進**一個**值；r 一過 3，不動點失穩，軌跡改去**兩個**值之間跳（2-cycle），長期行為從「週期 1」變「週期 2」。這是定性改變（分岔），不是「收斂慢一點」這種量變。r=3 是這場分裂的起跑線（細節 ch07）。

常見錯路：把 r=3 講成「系統開始發散／爆掉」。不對。它不發散（x 永遠在 [0,1]，被 (1−x) 擁擠項壓著），它是從「停一點」翻成「跳兩點」。**失穩 ≠ 爆炸。**

### 推演題 3 ★★ **[15 分鐘]**

有人主張：「既然 r=3.2 時 x*=0.6875 是不動點，那我只要把初始值**精準設成** 0.6875，系統就會永遠待在 0.6875——這不就『穩定』了嗎？」這個論證哪裡有破綻？它和「穩定」這個詞的真正意思差在哪？順帶：這跟你在分散式系統裡追 bug 為什麼要 bit-identical 重放，有什麼血緣？

#### 推演解答

破綻在於混淆了「不動點」和「**穩定的**不動點」。論證前半是對的：在數學的理想世界裡，精準的 0.6875 餵進去確實永遠吐出 0.6875（3.2×0.6875×0.3125=0.6875）。但這不叫「穩定」。

**穩定（stability）講的是對擾動的抵抗力**：把起點放在 0.6875 **附近**（不是精準在上面），軌跡會不會自己回到 0.6875？|f′(x*)|=1.2>1 的答案是「不會，反而被推得更遠」。所以 0.6875 是個**排斥**不動點——就是本章開頭那支倒立的鉛筆：理論上有平衡位置，但任何一絲擾動都讓它倒下。穩定的不動點才像碗底的彈珠，推一下會自己滾回去。

工程血緣：在真實的浮點與有限精度世界裡，「精準的 0.6875」這個前提多半根本給不起——0.6875 = 0.5 + 0.1875 = 11/16 剛好是 2 的冪之和，還能精確存；但絕大多數不動點（像 0.6429）是無限循環或無理的，浮點必然帶捨入誤差 εₙ。對**排斥**不動點，這個最後一位的捨入誤差會被每輪乘上 1.2 放大，幾十輪後就把你推離不動點、推進那個 2-cycle。這正是 SDIC（敏感依賴，ch03）的近親，也是為什麼分散式追 bug 要 bit-identical 重放：在不穩定的系統裡，最後一位小數的差別會被放大成完全不同的軌跡，於是「同樣的輸入」在兩台機器上跑出不同結果，bug 復現不了。「理論上精準就停得住」在工程上等於「停不住」——因為精準，是你給不起的。

常見錯路：以為「是不動點」就蘊含「會穩定停留」。本章整章就在拆這個——不動點是穩態的**必要**條件、不是**充分**條件；充分條件是 |f′(x*)|<1。

### 推演題 4 ★★ **[12 分鐘]**

把脊椎遞迴式換掉，改看另一個一維映射 g(x) = x + r·x·(1−x)（這是 logistic 的「連續時間離散化」近似，r 是步長）。(a) 它的非零不動點還是 x*=1 嗎？先解出來。(b) 算 g′(x) 與 g′(x*)。(c) 用 |g′(x*)|<1 判斷：步長 r 要多大才會讓這個不動點失穩？這題想讓你看到：**穩定判據的「形狀」不挑映射，f′(x*)=2−r 那個漂亮結果是 logistic 專屬，但 |f′(x*)|<1 這條法則是通用的。**

#### 推演解答

(a) 解不動點 g(x*)=x*：

```text
  x* + r·x*·(1 − x*) = x*
        r·x*·(1 − x*) = 0          ← 兩邊減 x*
```

r≠0，所以 x*·(1−x*)=0，給 x*=0 或 **x*=1**。非零不動點是 x*=1（注意：跟 logistic 的 1−1/r 不同，這個映射的不動點不隨 r 動）。

(b) 微分：

```text
  g(x)  = x + r·x·(1 − x) = x + r·x − r·x²
  g′(x) = 1 + r − 2·r·x = 1 + r·(1 − 2x)
  g′(x*) = g′(1) = 1 + r·(1 − 2) = 1 + r·(−1) = 1 − r
```

(c) 套判據 |g′(x*)|<1：

```text
  |1 − r| < 1
  −1 < 1 − r < 1
  −2 < −r < 0
   0 < r < 2          ← 步長 r 在 (0,2) 內，x*=1 穩定；r ≥ 2 失穩
```

所以這個映射在步長 **r=2** 時不動點失穩（|g′|=1）。重點不是「2」這個數，而是整套流程**一模一樣**：找 x*=g(x*)、算 g′(x*)、套 |g′(x*)|<1。logistic 算出 f′(x*)=2−r、在 r=3 失穩；這個映射算出 g′(x*)=1−r、在 r=2 失穩——**結果的數字是各映射專屬，但「斜率絕對值<1」這條判據是通用的**。這也預告了 ch08 的鐵律：不同的映射，沿著同一條路（倍週期）通往混沌，撞上同一個 Feigenbaum 常數——通用的從來不是某個數，是某種**結構**。

常見錯路：(1) 想當然把不動點寫成 1−1/r（那是 logistic 的，這個映射不是）；(2) 微分時漏掉前面那個 +x 的導數 1，算成 g′=r(1−2x)（少了那個 1，臨界值就全錯了）。

## 自我檢核

口頭自答，講得出來才算過關，講不清就是還沒懂：

1. 不動點的定義是什麼？為什麼說它是回授迴圈的「穩態**候選**」而不直接是「穩態」？
2. 穩定判據 |f′(x*)|<1 是怎麼從「偏差每輪乘 f′(x*)」一路推出來的？為什麼非取絕對值不可（少了 |·| 會誤判什麼）？
3. 在 cobweb 圖上，「曲線比對角線緩」和「比對角線陡」分別對應什麼軌跡行為？分界線為什麼是斜率絕對值等於 1（而不是 0）？
4. 把 f′(x)=r(1−2x) 代入 x*=1−1/r 得 f′(x*)=2−r——這幾步你能闔上書、徒手重推一遍嗎？
5. 為什麼穩定區是 1<r<3 這個**有上限**的區間？r 太大為什麼反而失穩（用「增益太高就 flapping」的工程直覺講給另一個工程師聽）？
6. r=2 為什麼叫超穩定點？f′(x*)=0 在 cobweb 圖上意味著什麼、收斂為什麼最快？
7. r=2.5 的軌跡為什麼是「忽高忽低」螺旋收斂、而不是直直滑進去？這跟 f′(x*) 的什麼性質有關？
8. 「x* 是不動點」和「x* 是**穩定**不動點」差在哪？用 r=3.2 的 x*=0.6875 舉例說明，這個差別會怎麼在浮點世界裡坑你。

## 延伸閱讀

- **Strogatz,《Nonlinear Dynamics and Chaos》第 10 章（One-Dimensional Maps）** — 本章「不動點＋|f′|<1 判據＋cobweb」的標準教科書版，把線性化穩定性與分岔講得最乾淨利落。讀 10.1–10.3，讀完你會發現本章只是它的工程口語版。
- **math.libretexts.org，*Mathematical Biology*（Chasnov）的 Logistic 章節** — 線上免費，從人口模型角度推 x*=1−1/r 與 f′(x*)=2−r 的穩定性，含完整代數步驟，適合把本章推導再獨立核一次。（https://math.libretexts.org，站內搜 Logistic / fixed point stability）
- **Wikipedia「Cobweb plot」** — cobweb 圖的純幾何說明與動畫直覺：階梯收縮 vs 張開、螺旋進 vs 螺旋出。配本章兩張 ASCII 示意一起看，「斜率 vs 穩定」的對應會更立體。（https://en.wikipedia.org/wiki/Cobweb_plot）
- **control.com，「Proportional-only Control」一章** — 從控制工程那一側講「增益太高就振盪」，正好和本章 |f′(x*)|<1 隔空對照。看完你會徹底相信「調 P 控制器」和「轉 r 這個旋鈕」是同一個數學動作。（https://control.com/textbook/closed-loop-control/proportional-only-control/）
- **本書 ch07** — 旋鈕一過 3，這個失穩的不動點分裂成 2-cycle，再分裂成 4-cycle……倍週期級聯與招牌分岔圖。本章 r=3.2 那個「跳兩個值」在那裡會被講透：為什麼是加倍、新的穩定週期從哪來。
- **《矩陣是動詞》談特徵值的章** — 多維不動點穩定性＝Jacobian 特徵值的模是否全部 <1。本章一維的 f′(x*) 是它的特例；想看線性化分析在多維怎麼算，去那裡。
- **《馴服無限》談導數與微分方程穩定性的章** — 「導數＝瞬時斜率」的根，以及連續時間系統「看實部正負」（而非「模<1」）的對照判據。本章只用到離散版，連續版去那裡補。
