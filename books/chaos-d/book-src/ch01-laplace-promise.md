## 從你已知的出發

先講一個你每天都在依賴、卻可能沒給它取過名字的東西：**純函數**。

同樣的輸入，永遠給同樣的輸出。沒有藏在背後的時鐘、沒有讀全域變數、沒有偷偷打 API。`f(3)` 今天回 7，明天回 7，在你的機器回 7，在 CI 跑也回 7。你寫單元測試時賴的是它，你做 snapshot 測試時賴的是它，你之所以能對一段邏輯「推理」而不是「祈禱」，靠的就是它。

純函數是一種承諾：**未來完全由現在決定，中間沒有任何縫隙讓意外鑽進來。** 把這個承諾從一個函數放大到整個宇宙，你就得到了哲學上的「決定論（determinism）」。

你對這個承諾的信任，遠比你以為的深。再舉幾個你身上現成的例子：

- **可重現建置（reproducible build）**：同一份 source、同一組 toolchain 版本、同一份 lockfile，建出 byte-identical 的產物。整個供應鏈安全的論證——「我能驗證這顆 binary 真的來自這份 source」——成立的前提，就是這條建置流程是決定論的。哪天它不可重現了，你會花一整天去抓那個讓它「飄」的非決定來源（某個 timestamp、某個 map 的迭代順序、某個併發）。
- **replay determinism**：event sourcing 把同一串 event 從頭重放，得到同一個 state。你之所以敢用「重放」當作除錯與審計手段，是因為你相信這個重放是決定論的——同樣的 event 序列，同樣的結果。
- **PRNG**：偽隨機數產生器，給定 seed，就吐出一整串「看起來很亂」但完全寫死的數字。你壓測時固定 seed 好讓 case 可重現，靠的正是「它其實一點都不隨機」這件事。

注意這三個全是「決定論」在工程裡的化身。你不只懂決定論，你天天在它身上下注。

那麼問題來了。如果一個系統是決定論的——同輸入同輸出、過去完全決定未來——它是不是就**可預測**？

你的直覺會立刻說：當然。我能重放，不就等於我能預測嗎？

**這一章存在的唯一理由，就是先讓你把這句「當然」說出口、說得理直氣壯。** 因為這整本書，就是要證明這句「當然」是錯的——不是錯在決定論，而是錯在那個「就等於」。決定論和可預測，是兩件不同的事。它們在你寫的小程式裡剛好總是一起出現，於是你把它們焊成了一個概念。混沌做的事，是把這個焊點敲開。但這要等到第 3 章才動手。本章先把承諾立穩。

## 牛頓的時鐘：承諾的物理版本

承諾的源頭在物理。

牛頓（Newton）在 1687 年的《自然哲學的數學原理》之後，人類第一次有了一套規則，能用同一組方程式同時管住「最大的天體」和「掉下來的蘋果」。給你行星此刻的位置和速度，加上萬有引力定律，你就能往前推算它一千年後在天上的哪個點——而且推得準到能預報日食。

這套成功太驚人，於是十八世紀慢慢長出一個世界觀，後人給它取了名字叫「鐘錶宇宙（clockwork universe）」：整個宇宙是一台造好上緊發條的鐘，零件按固定的力學定律一格一格走，未來的每一刻原則上都已經寫在現在的狀態裡了。

這裡先插一句誠實話，免得你日後被人糾正。「鐘錶宇宙」這個比喻，**多半是後世（萊布尼茲那一代的論戰、以及十八、十九世紀的詮釋者）貼到牛頓身上的，而不是牛頓自己掛在嘴邊的招牌**；歷史學界甚至有「找不到一處牛頓明確把宇宙稱作鐘錶機械的鐵證」的說法（2026-06 查證，史學上仍有討論）。牛頓本人留了一個位子給上帝去偶爾校正這台鐘。所以正確的講法是：「鐘錶宇宙」是一個**思想的氣候**，是牛頓力學成功之後瀰漫開來的決定論信心，不是某一句牛頓語錄。我們在意的不是它歸誰，而是它表達的那個承諾本身。

那個承諾可以濃縮成一個框：

```text
  ┌─────────────────────────────────────────────┐
  │  鐘錶宇宙的承諾                               │
  │                                              │
  │  規則固定（力學定律不變）                    │
  │       ＋                                     │
  │  現在給定（所有零件此刻的狀態）              │
  │       ⇒                                      │
  │  未來盡在其中（原則上算得出每一刻）          │
  └─────────────────────────────────────────────┘
```

把這個框翻成你的話：**規則是純函數、初始狀態是輸入，於是整段未來就是這個函數的輸出，一路 replay 下去就是了。** 同一個直覺，物理版叫鐘錶宇宙，工程版叫 replay determinism。它們是同一個信念。

## 拉普拉斯的智者：承諾被寫成一句話

把這個承諾講到最徹底、最有名的人，是拉普拉斯（Pierre-Simon Laplace）。

1814 年，他在《機率的哲學試論》（*Essai philosophique sur les probabilités*）的導論裡寫下一段話。注意這本書的標題——是一本講「機率」的書。拉普拉斯的立場其實是：機率不是世界真的不確定，而是**我們無知**的度量。世界本身是決定論的；我們之所以要用機率，只是因為我們不夠聰明、資料不夠全。為了把這個立場講清楚，他先描繪了一個假想中「完全不無知」的存在，當作對照組。經典英譯大意是這樣的：

```text
  「一個智者（an intelligence），若在某一刻知道驅動自然的所有力、
    以及組成自然的萬物的所有位置——而且其智力足夠強大，能把這些
    資料拿去分析——那麼，從最巨大的天體到最微小的原子，它都能
    用同一道公式涵蓋其運動；對它而言，沒有什麼是不確定的，
    未來會如過去一樣，呈現在它眼前。」
```

這就是後世所稱的「拉普拉斯惡魔（Laplace's demon）」。

但這裡要守住一個事實，因為連很多科普文都搞錯：**拉普拉斯本人用的詞是「智者／智能（intelligence）」，他從沒寫過「惡魔（demon）」。** 「惡魔」是後人加上去的綽號（2026-06 查證：這是後世評論者的潤飾，不是拉普拉斯原文用語）。這不只是用詞潔癖。把它叫「惡魔」會讓你不自覺把它想成某種超自然、可疑、註定要出錯的東西；但拉普拉斯的原意完全相反——這個智者是**完美理性的化身**，是「如果你把鐘錶宇宙的承諾推到極限，會得到什麼」的具象答案。它不可疑，它是當時最自信的科學世界觀的代言人。我們全書要做的，不是說這個智者很傻，而是要找出：就算給它無限算力、無限精度，它在哪裡仍然會撞牆。

把這個智者需要的三樣東西列清楚，等一下你會發現混沌砍的就是其中一樣：

```text
  拉普拉斯的智者要能算盡未來，需要三件事：

  (1) 完整的初始狀態  ── 此刻每個粒子的位置與動量，一個不漏
  (2) 完整的規則      ── 驅動一切的力學定律
  (3) 足夠的分析能力  ── 把 (1)(2) 拿去算的無限算力

  承諾：(1) ＋ (2) ＋ (3)  ⇒  過去與未來，盡在眼前
```

工程師讀到這裡通常會直覺去攻擊 (3)：「無限算力？現實哪有，所以承諾本來就是空話。」**這個攻擊是對的，但它沒打到要害，而且會害你錯過全書最反直覺的那一刀。** 因為混沌的震撼恰恰在於：**就算我們大方地把 (3) 送給它——假設真的有無限算力——這個智者照樣會輸。** 輸的點不在算力，在 (1) 那個「完整」上、在這套規則對「完整」這件事有多麼苛刻的要求上。但這要 Part I 後面才慢慢揭開。本章你只需要把承諾的三件套記牢，並且先別替它求饒——讓它站在最強的姿態上，這樣它待會兒倒下才有意義。

## 混沌的一句話定位（先給座標，機制留到後面）

既然全書要拆這個承諾，你得先知道拆它的那個東西叫什麼。給你一句話定義，後面所有章都在把它講深：

> **混沌（chaos）＝對初始條件敏感的確定性系統。**

逐字拆給你看，因為這句話的每個字都有人搞錯：

- **確定性（deterministic）**：它是純函數，沒有任何隨機項，同初值給出同軌跡。混沌**不是**亂數、**不是**雜訊、**不是**「世界本身擲骰子」。這一點之反直覺，值得我說重一點：一個完全沒有隨機成分、規則寫死的系統，可以製造出你用任何統計檢定都會誤判成「隨機」的行為。混沌的「亂」是被算出來的亂。
- **對初始條件敏感（sensitive to initial conditions）**：兩個近到幾乎分不出的起點，隨著時間演化，差距會被指數放大，直到兩條軌跡完全不像。起點差一根頭髮，幾步之後天差地別。

把兩個字併起來：**一條完全確定的規則，可以把「我量不準起點」這件小事，放大成「我完全測不準終點」。** 這就是混沌，也就是拉普拉斯承諾斷掉的那一環的預告——智者要的是「完整的初始狀態」，而現實裡的測量永遠帶一點點誤差，混沌系統會把那一點點誤差吹成風暴。

注意：**這裡定義的是混沌「是什麼」，不是它「怎麼運作」。** 為什麼誤差會指數放大、放多快、什麼樣的系統會這樣——這些機制全部留到後面（敏感依賴的真正意義在 ch03；量化放大速率的 Lyapunov 指數在 ch14；製造它的拉伸與摺疊機制在 ch16）。本章只給座標，不給引擎。

順手立一張「是／不是」對照表，這張表會在全書反覆回來打你直覺的臉，你現在先掃一眼：

```text
  混沌「是」……                  混沌「不是」……
  ────────────────────────────  ──────────────────────────────
  確定性的（無隨機項）          隨機的（不是世界在擲骰子）
  原則上完全可算                「無法計算」「沒有方程式」
  對初始條件指數敏感            「微小原因必然造成某個特定大果」
  有結構、有形狀（Part III）    一團毫無秩序的噪音
  服從跨系統的鐵律（Part II）   每個亂系統各亂各的
  認識論上不可長期預測          本體論上不存在「真相」
```

右欄那些「不是」，每一條都是後面某章「直覺的陷阱」段要正面對決的庸俗版誤解。你現在不用全懂，只要先有個印象：混沌幾乎在每一個關節都跟你的第一直覺相反。這就是為什麼這本書值得慢慢讀——你的直覺正準備在接下來每一章騙你一次。

## 這裡反直覺在哪：把承諾和它的反命題並排

我把本章的整個張力濃縮成一組對照，這是你讀完 ch01 最該帶走、且要能對另一個工程師複述的東西：

```text
  承諾（你的第一直覺，本章立的標靶）：
      決定論  ⇒  可預測
      「規則寫死、起點給定，未來就算得出來。我能 replay，就能預測。」

  反命題（全書要證明的，從 ch03 開始動手）：
      決定論  ⇏  可預測
      「規則寫死、起點給定，未來仍然測不準——
       因為起點永遠帶一點誤差，而某些系統會把那點誤差指數放大。」

  關鍵：被砍斷的不是「決定論」，是中間那個「⇒」。
        系統照樣完全決定論，照樣每一步都算得出來；
        垮掉的只是「算得出來 ⇒ 我能在實務上預測它」這個推論。
```

了不起的地方在於：**這兩句話可以同時為真。** 系統 100% 決定論（你可以一步步算、可以 replay、沒有任何隨機），同時 100% 不可長期預測（你永遠量不到那個「完整無誤差」的起點，而誤差會爆炸）。決定論談的是「世界是不是唯一地往下走」（本體論，世界自己的事）；可預測談的是「我們能不能算出它往哪走」（認識論，我們的事）。它們是兩根獨立的軸，平常綁在一起只是巧合。混沌就是那個讓兩根軸分家的反例。（這三根軸——決定論、隨機、可預測——會在 ch04 正式拆開成四象限，本章先讓你聞到味道。）

哲學圈把這個區分講得很乾脆：「對混沌與決定論的混淆，多半來自把決定論等同於可預測性。」（史丹佛哲學百科，2026-06 查證原文如此。）你今天讀完這一章，就是要把這個焊死的等號，先在腦子裡敲出一條裂縫。

## 一個玩具決定論系統：親眼看「完全可算」

光講承諾太空。我要你親眼看一個決定論系統「完全可算」是什麼樣子——而且我刻意挑一個你最熟的：**線性同餘產生器（LCG，linear congruential generator）**，PRNG 最古老的一種。

規則是一條整數遞迴式：

```text
  xₙ₊₁ = (a·xₙ + c) mod m
```

我選一組小到能手算的參數：a = 5、c = 3、m = 16，起點 x₀ = 1。一步步手算（每步先算 a·xₙ + c，再對 16 取餘數）：

```text
  n │ xₙ │ 計算 a·xₙ + c = 5·xₙ + 3       │ mod 16 │ xₙ₊₁
  ──┼────┼────────────────────────────────┼────────┼──────
  0 │  1 │ 5·1  + 3 = 8                    │   8    │   8
  1 │  8 │ 5·8  + 3 = 43                   │  11    │  11    ← 43 = 2·16 + 11
  2 │ 11 │ 5·11 + 3 = 58                   │  10    │  10    ← 58 = 3·16 + 10
  3 │ 10 │ 5·10 + 3 = 53                   │   5    │   5    ← 53 = 3·16 + 5
  4 │  5 │ 5·5  + 3 = 28                   │  12    │  12    ← 28 = 1·16 + 12
  5 │ 12 │ 5·12 + 3 = 63                   │  15    │  15    ← 63 = 3·16 + 15
  6 │ 15 │ 5·15 + 3 = 78                   │  14    │  14    ← 78 = 4·16 + 14
  7 │ 14 │ 5·14 + 3 = 73                   │   9    │   9    ← 73 = 4·16 + 9
```

序列：1 → 8 → 11 → 10 → 5 → 12 → 15 → 14 → 9 → …

盯著這串數字看。**1, 8, 11, 10, 5, 12, 15, 14, 9** 完全看不出規律——它跳上跳下，沒有遞增、沒有週期一眼可見的影子，你把它印出來給人看，多數人會說「這是隨機數」。可是它哪有半點隨機？我沒擲一次骰子。每一個數都是上一個數套同一條死規則算出來的，過程攤在你眼前，你自己驗得了每一步。**這就是決定論承諾兌現的樣子：規則固定、起點給定，整串未來一個不差地被釘死了。**

而且這個玩具完美示範了拉普拉斯承諾的另一半——可重放。換你接手：我告訴你「現在 x = 5」，你不需要知道前面發生過什麼，就能往下接 12, 15, 14, 9…，跟我算的一字不差。狀態決定未來，這就是 replay determinism 的數學骨架。你的 PRNG seed、你的 event sourcing，骨子裡都是這條遞迴式的放大版。

這裡停十秒，記住兩件事，因為第 3 章會在這兩件事上動刀：

1. 這個系統**算得出來**——拉普拉斯的智者會非常滿意，給它起點，它把整串未來一次寫完。
2. 這個系統的輸出**看起來很亂**——但「看起來亂」和「真的不可預測」是兩回事。LCG 看起來亂，可它一點都不難預測：你知道規則和當前值，下一個值是確定的，誤差不會放大（整數運算，沒有「一點點誤差」可言）。

**伏筆（明寫，請記住這句）**：第 3 章我會換一條長相非常像的遞迴式——同樣是「把當前狀態餵回固定規則、算出下一步」的家族——但它跑在實數上，而且我會給它兩個只差千分之一的起點。你會親眼看到，同一個「玩具決定論系統」的框架，這次**算得出來、卻測不準**：兩條軌跡前幾步幾乎重疊，幾步之後就分道揚鑣到完全不像。那條式子（脊椎式 `xₙ₊₁ = r·xₙ·(1 − xₙ)`）**第 5 章正式介紹**，是貫穿全書的主角；ch03 只是先借它一用，讓你看見「可算 ≠ 可預測」這一刀。

換句話說：LCG 是「決定論 ⇒ 可預測」這條承諾**還成立**的版本（看起來亂但其實乖）；下一章開始，你會遇到承諾**斷掉**的版本（同樣決定論，卻把測量誤差吹成風暴）。本章你站在承諾這一側，把它看清楚、看自信，這樣往下走才知道自己失去了什麼。

## 直覺的陷阱

混沌是人類直覺最常崩壞的領域，所以每章我都用這一段，把「最容易掉進去的坑」攤開。本章只有一個坑，但它是全書的母坑——後面所有的坑都是它的子孫。

| 庸俗版（直覺會這樣想） | 正確版（守住這個） | 你會在哪一步掉進去 |
|---|---|---|
| 決定論的系統就是可預測的系統，兩者是同義詞。 | 決定論是「世界唯一地演化」（本體論）；可預測是「我們能不能算出來」（認識論）。是兩根獨立的軸，只是在簡單系統裡剛好同進同出。 | 你寫的小程式裡，決定論和可預測**總是**一起出現，於是你把「能 replay」直接讀成「能預測」，把兩個概念焊死成一個。 |
| 既然世界是決定論的，那只要算力夠強、模型夠好，原則上什麼都能預測。 | 對某些系統，**就算給無限算力，仍然測不準**——因為瓶頸不在算力，在「起點永遠量不到無限精度」加上「誤差會指數放大」。算力救不了你。 | 你下意識去攻擊承諾三件套裡的 (3) 算力，覺得「現實算力有限所以承諾本來就成立不了」，於是錯過真正的要害在 (1) 那個「完整無誤差的初始狀態」。 |
| 一個系統的輸出「看起來很亂」，它就是不可預測的／隨機的。 | 「看起來亂」是觀察者的感受，跟「可不可預測」「是不是隨機」是三件事。LCG 看起來很亂，卻完全可預測、完全不隨機。 | 你把「我看不出規律」誤當成「它沒有規律」。本章的 LCG 就是反例：你看不出規律，它卻被一條死規則釘死。 |

**怎麼自我察覺你掉進去了？** 一個徵兆：當你說「這系統是決定論的，所以理論上可以預測」時，如果你心裡那個「所以」說得毫不猶豫、覺得根本不用論證——那你大概就站在坑邊了。這本書讀完，你對這個「所以」應該會本能地停頓一下，問自己：「等等，它對初始條件敏感嗎？」這個停頓，就是你學會混沌的標誌。

（注意：本章只立框架。「決定論／隨機／可預測」三軸如何正式拆成四象限、各自坐落在哪，是 ch04 的工作；「看起來像雜訊的東西到底是混沌還是真雜訊、怎麼分辨」是 ch17 的工作。這裡點到為止。）

## 紙上推演

全書純紙筆，不跑任何程式。下面兩題，拿張紙，動手。

### 推演題 1：手算 LCG，再驗證「狀態決定未來」 ★ **[10 分鐘]**

用本章的 LCG：`xₙ₊₁ = (5·xₙ + 3) mod 16`。

(a) 從 x₀ = 7 開始，手算到 x₅，列成迭代表。
(b) 算到後面你會回到某個出現過的值，序列開始循環。從你 (a) 算出的數字裡，找出這條軌跡的循環長度（週期）。
(c) 不看前面的歷史，只給你「現在 x = 12」，請接著往下寫三個值。然後回頭驗證：這三個值，和你從 x₀=7 一路算下來經過 12 之後的那三個值，是否一模一樣？這說明了什麼？

### 推演解答

(a) 逐步手算（先算 5·xₙ + 3，再 mod 16）：

```text
  n │ xₙ │ 5·xₙ + 3        │ mod 16 │ xₙ₊₁
  ──┼────┼─────────────────┼────────┼──────
  0 │  7 │ 5·7  + 3 = 38   │   6    │   6    ← 38 = 2·16 + 6
  1 │  6 │ 5·6  + 3 = 33   │   1    │   1    ← 33 = 2·16 + 1
  2 │  1 │ 5·1  + 3 = 8    │   8    │   8
  3 │  8 │ 5·8  + 3 = 43   │  11    │  11    ← 43 = 2·16 + 11
  4 │ 11 │ 5·11 + 3 = 58   │  10    │  10    ← 58 = 3·16 + 10
  5 │ 10 │ 5·10 + 3 = 53   │   5    │   5    ← 53 = 3·16 + 5
```

序列：7 → 6 → 1 → 8 → 11 → 10 → 5 → 12 → 15 → 14 → 9 → …（後面接上正文那串）。

(b) 因為狀態只有 16 種可能（0–15），序列遲早會撞回出現過的值、然後永遠循環——這是任何有限狀態決定論系統的鐵則（鴿籠原理）。從 x₀=7 出發，繼續往下算會回到某個前面出現過的值。重點不是背出週期數字，而是看出**「有限狀態 ＋ 決定論 ⇒ 必定最終週期」**：因為下一步只看當前值，一旦某個值重複，後面整段就照抄一遍。（這個觀察在 ch07「倍週期」會以另一種面貌回來，那裡的「週期」活在實數系統上，行為精細得多。）

(c) 從「現在 x = 12」往下：5·12+3 = 63 = 3·16+15 → **15**；5·15+3 = 78 = 4·16+14 → **14**；5·14+3 = 73 = 4·16+9 → **9**。所以接 12, 15, 14, 9。回頭對照正文那串「…5, 12, 15, 14, 9…」——**一模一樣**。這說明：**未來只由當前狀態決定，跟「怎麼走到這裡」的歷史無關。** 這正是 replay determinism 的核心——你不需要重放全部歷史，只要拿到正確的當前狀態快照，就能接著往下算出完全相同的未來。你的 event sourcing 之所以能用「快照 ＋ 後續 event」加速重放，靠的就是這條性質。

### 推演題 2：用自己的話定義決定論，並舉一個「決定論卻難預測」的親身例子 ★★ **[15 分鐘]**

(a) 不准用「隨機」這個詞，用一句話定義「決定論」。
(b) 從你自己的系統裡，舉一個**確實是決定論、但你實務上很難預測它行為**的真實例子。說清楚：它哪裡算決定論？難預測又難在哪裡？
(c) 想一想：你那個例子的「難預測」，是因為「算力／資訊不夠」，還是因為某種「敏感放大」？（這題不要求標準答案，要的是你開始用這把尺去量自己的系統。）

### 推演解答

(a) 一個可接受的定義：**決定論＝系統的完整當前狀態，唯一地決定了它的整段未來；同樣的狀態，永遠演化出同樣的未來，中間沒有任何縫隙容意外介入。** （注意我繞開了「隨機」——決定論的正面定義是「唯一演化」，不是「沒有隨機」；「沒有隨機」只是它的一個推論。）

(b) 最常見、也最漂亮的工程例子是**依賴時序的並行 bug（race condition）**。整個系統其實是決定論的：每一條指令、每一次記憶體存取、每一個排程決定，在底層都遵循確定的規則，沒有任何超自然的隨機。但你**實務上幾乎無法預測**它什麼時候會炸——因為結果敏感依賴於一個你無法完整掌握、也無法穩定重現的「初始與中途狀態」：執行緒被作業系統排程的精確時刻、cache 的冷熱、那一瞬間機器上還跑著什麼。換一台機器、加一行 log（它改變了時序！），bug 就不重現了。這就是工程師最熟悉的「決定論卻測不準」的活樣本——也是為什麼分散式系統除錯要追求 bit-identical 重放：你想做的，正是把那個「完整初始狀態」鎖死，好讓決定論的可重放性回到你手上。

其他可接受的例子：浮點運算順序不同導致結果飄移（決定論，但敏感於求值順序）；快取淘汰行為（決定論，但敏感於存取序列）；分散式系統裡訊息到達順序造成的不同最終狀態。

(c) 引導：race condition 的難預測，**兩種成因都有，但主成因是「敏感」**——你缺的不是算力，是「完整且可重現的初始狀態」。哪怕給你無限算力，只要你抓不到那個精確到奈秒的排程時序，你還是測不準。**這個「瓶頸在初始狀態的完整性、不在算力」的直覺，正是混沌的核心。** 你已經在你的系統裡見過它了，只是沒給它取這個名字。本書接下來要做的，就是把這個你早就遇過的現象，講成一門有鐵律的學問。

## 延伸閱讀

- **史丹佛哲學百科〈Chaos〉**（plato.stanford.edu/entries/chaos）—— 讀開頭關於「決定論 ≠ 可預測性」的論述。本章「承諾 vs 反命題」的哲學骨架就來自這裡；它那句「對混沌與決定論的混淆多半來自把決定論等同於可預測性」值得抄在你筆記第一頁。
- **維基百科〈Laplace's demon〉**（en.wikipedia.org/wiki/Laplace's_demon）—— 看那段 1814 原文的英譯全文，並注意它明確指出「demon」是後人加的綽號、拉普拉斯原文用的是「intelligence」。本章守住的史實在此可一鍵複查。
- **拉普拉斯《機率的哲學試論》英譯本 *A Philosophical Essay on Probabilities***（archive.org 有公版全文）—— 讀導論前幾頁就好。你會發現這位決定論最強代言人，寫的其實是一本講「機率即我們無知之度量」的書——這個反差本身就值得玩味，也預告了 ch04 要拆的「決定 vs 機率」。
- **James Gleick《Chaos: Making a New Science》（1987）第 1 章**—— 全書混沌科普的經典開場。把它當作這趟旅程的散文版序曲；它的「蝴蝶」故事我們會在 ch03 用更精確的史實版本重講（Gleick 流傳甚廣，但若干細節後來有更準的考據）。
- **OpenLearn〈The restless Universe〉決定論一節**（open.edu，搜 "Mechanics and determinism"）—— 一頁講清楚牛頓力學如何長出決定論世界觀、以及「鐘錶宇宙」這個比喻的來歷與爭議。本章對「鐘錶宇宙別直接歸給牛頓」的謹慎，這裡有更完整的脈絡。
