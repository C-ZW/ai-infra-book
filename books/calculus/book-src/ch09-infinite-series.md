## 從你已知的出發

在 Node.js 裡敲 `Math.sin(0.5)`，回來 `0.479425538604203`。停一秒：這個數是哪來的？

你的 CPU 會做的事情少得可憐：加、減、乘、除、比較、搬資料。指令集裡沒有「sin」這種東西（舊式 x87 確實有一條 fsin 指令，但因為精度問題早就被軟體實作取代，而它內部一樣是靠多項式與查表；2026-06）。sin 的定義是幾何的——單位圓上的縱座標——而幾何不是演算法。盤點一下你徒手就能算到任意精度的函數家族：多項式（只要加法和乘法）、有理函數（再加一個除法）。就這樣，名單結束。eˣ、ln、sin、cos 全都不在裡面：它們的定義是極限、是圓、是「增長率等於自身」（見 ch05），沒有一個告訴你「給我 0.5，我給你數字」的具體步驟。

但你其實早就用過本章的主角，只是只用了第一項。容量規劃的線性外插——「照這個斜率，三個月後流量會到……」——翻成 ch05 的語言就是 f(x+h) ≈ f(x) + f′(x)·h：用「位置＋斜率」冒充函數本身，在小範圍內好用得很。泰勒級數只是不肯停在第一項的線性外插：再抄彎度、再抄彎度的變化、一路抄下去。

剩下的問題是工程味十足的：抄到第幾項才夠？這跟你定 SLO 的思路一模一樣——先講好容差（雙精度浮點大約 16 位有效數字，這是天花板），再決定花幾項預算買到它。本章結尾你會看到，你機器裡的 `sin` 對這題的回答精確到令人感動。

## 把函數抄成多項式

想法很單純：在一個點附近（取 x = 0），造一個多項式，讓它跟 f「長得一樣」。什麼叫長得一樣？一階一階對齊：

- 對齊**位置**：兩者在 0 的值相同；
- 對齊**斜率**：一階導數相同（ch04 的導數）；
- 對齊**彎度**：二階導數相同（ch06 用它判凹凸）；
- 對齊**彎度的變化**：三階導數相同……

每多一項，多抄一階導數。假設 f 在 0 附近可以寫成 f(x) = c₀ + c₁x + c₂x² + c₃x³ + …（先假裝可以——這個假設值多少錢，本章後半會回頭追究），那每個係數都被逼到只有一種選擇：

```text
代 x = 0：      f(0)   = c₀                     ← 其餘項全帶 x，歸零
微分一次再代 0：f′(0)  = c₁                     ← c₀ 被微掉，c₂x² 以上仍帶 x
再微分一次：    f″(0)  = 2·c₂        → c₂ = f″(0)/2
第三次：        f‴(0)  = 3·2·c₃      → c₃ = f‴(0)/6
第 n 次：       f⁽ⁿ⁾(0) = n!·cₙ      → cₙ = f⁽ⁿ⁾(0)/n!
```

n! 是階乘（factorial）：n·(n−1)·…·1。它不是憑空出現的——xⁿ 被微分 n 次，指數 n、n−1、…、1 一路掉下來相乘，n! 就是「微分 n 次」這個動作留下的腳印。於是：

f(x) = f(0) + f′(0)·x + f″(0)·x²/2! + f‴(0)·x³/3! + …

這就是泰勒級數（Taylor series）。名字掛在 Brook Taylor 身上（1715 年《Methodus Incrementorum Directa et Inversa》發表一般形式），但 Gregory、牛頓、萊布尼茲、Johann Bernoulli、de Moivre 各自都摸到過變體，「泰勒級數」這個稱呼要到十八世紀末才通行（2026-06 查證）。素描的比喻很貼切：第一筆抓位置，第二筆抓朝向，第三筆抓弧度——筆數越多，畫像在落筆點附近越逼真。

## 三條級數：eˣ、sin、cos

**eˣ 是最乾淨的一次抄寫。**ch05 說它是「宇宙最自戀的函數」——每一階導數都是自己，所以在 0 點的每一階導數值都是 e⁰ = 1。整排係數全是 1/n!：

eˣ = 1 + x + x²/2! + x³/3! + x⁴/4! + … = Σ(n=0→∞) xⁿ/n!

兩個 sanity check，各花十秒：

```text
檢查一（結構）：逐項微分
  x³/3! 微分 → 3x²/3! = x²/2!     ← 每一項往前挪一格，整列火車不變
  所以這條級數微分後等於自己——「是自己的導數」寫在臉上 ✓

檢查二（數值）：代 x = 1，部分和應該逼近 e
  1 → 2 → 2.5 → 2.6667 → 2.7083 → 2.7167 → 2.7181 → 2.71825 → …
  e ≈ 2.71828 ✓（自算複核）
```

順便對帳：ch05 的複利路線 (1+1/n)ⁿ 爬到 n = 365 才到 2.7146，誤差還在千分位；級數路線八項就進到 2.71825。同一個 e，級數這條路快得多——這就是「可計算的長相」的份量。

**sin 與 cos 用同一招，靠的是微分的循環。**sin → cos → −sin → −cos → sin，週期 4；在 0 點取值就是 0, 1, 0, −1, 0, 1, …循環。塞進 cₙ = f⁽ⁿ⁾(0)/n!，一半的係數歸零，剩下的正負交錯：

```text
sin x = x − x³/3! + x⁵/5! − x⁷/7! + …      ← 只有奇數冪，+ − 交錯
cos x = 1 − x²/2! + x⁴/4! − x⁶/6! + …      ← 只有偶數冪，+ − 交錯
```

不硬磨推導，做三個結構檢查：sin 只有奇數冪——奇函數本來就該如此（sin(−x) = −sin x），偶數項全死是對的 ✓；逐項微分 sin 的級數，得到的恰好是 cos 的級數（x⁵/5! → x⁴/4!）✓；只取第一項就是 sin x ≈ x——物理課的小角度近似，也就是你的線性外插 ✓。

該見真章了。用前幾項算 sin(0.5)，每一步攤開（自算複核）：

```text
0.5³ = 0.125        0.125/3!     = 0.125/6        = 0.020833333…
0.5⁵ = 0.03125      0.03125/5!   = 0.03125/120    = 0.000260417…
0.5⁷ = 0.0078125    0.0078125/7! = 0.0078125/5040 = 0.0000015501…
```

| 取到第幾項 | 部分和 | 與真值的誤差 |
|---|---|---|
| 1（x） | 0.500000000 | 2.1×10⁻² |
| 2（−x³/3!） | 0.479166667 | 2.6×10⁻⁴ |
| 3（＋x⁵/5!） | 0.479427083 | 1.5×10⁻⁶ |
| 4（−x⁷/7!） | 0.479425533 | 5.4×10⁻⁹ |

真值 sin(0.5) = 0.479425539…。每多一項，誤差掉兩個多數量級。而且交錯級數（alternating series）有個紅利：部分和輪流高估、低估，真值被夾在相鄰兩個部分和之間——表上看得到，0.479166667 與 0.479427083 確實夾住真值——所以截斷誤差不會超過你丟掉的第一項（這對「項遞減趨零的交錯級數」成立；直覺版，本書不證，但你剛剛用眼睛驗收過一輪）。「展開到第幾項夠用」於是變成一道誠實的預算題：要 10⁻⁹ 的容差，四項；要頂到雙精度天花板，七、八項。

歷史補一筆：sin 與 cos 的級數，南印度 Kerala 學派的 Madhava（約 1350–1425）比歐洲早了兩百多年就寫下，還把 π 算到 11 位小數。不過 Madhava 本人的著作已經佚失，這些成果是經由約 1550 年的《Yukti-Bhasa》等後世 Kerala 文本轉述歸於他的；目前也沒有證據顯示它們傳入歐洲——主流觀點視之為獨立的先驅成就，而非歐洲微積分的源頭。

**回答開場的問題。**你機器裡的 `sin`（fdlibm 一脈，絕大多數 libm 的共同祖先）做兩件事：第一步，範圍縮減（range reduction）——sin 有週期與對稱，任何輸入都能摺回 [−π/4, π/4]；對 sin(1000) 而言這一步才是主菜（先去掉 159 圈整圈）。第二步，在這個小區間上用一條 13 次多項式逼近，原始碼註解白紙黑字寫著誤差界 ≤ 2⁻⁵⁸，比雙精度的一個最小刻度還小；係數長得幾乎就是泰勒係數，但末幾位經過調整，把整段區間的最大誤差再壓低（2026-06 查證 fdlibm 原始碼；各平台 libm 細節不一，「縮範圍＋多項式」的骨架則是通則）。所以「計算機怎麼算 sin」的答案是字面的：sin 在你的機器裡**就是**一個多項式，加上一次摺紙。泰勒級數不是教科書玩具，是出廠標配。

## 收斂半徑：分身的有效範圍

eˣ、sin、cos 這三條級數有個被你默默享用的奢侈性質：丟任何 x 進去都收斂。你可能以為這是常態。不是。

回到 ch03 的幾何級數：1/(1−x) = 1 + x + x² + x³ + …，成立的條件是 |x| < 1。牆就在 x = 1——函數本身在那裡爆炸（分母歸零），級數誠實地跟著死，這還算講理。真正有教育意義的是 x = 2：函數值 1/(1−2) = −1，完全健康；級數的部分和卻是 1, 3, 7, 15, 31, …，自信地衝向無窮。注意它的姿態——級數不會報錯、不會丟例外，它給你的是**自信滿滿的垃圾**。「在哪裡成立」從來不是法律免責條款，是生死線。

一般的事實（直覺版，嚴格理論本書不證）：每條冪級數（power series）都有一個收斂半徑（radius of convergence）R——|x| < R 時收斂，|x| > R 時發散，邊界上各案各判。eˣ、sin、cos 的 R = ∞；幾何級數的 R = 1。

再看一個詭異案例：f(x) = 1/(1+x²)。這函數在整條實數線上光滑、有界、無懈可擊。把 −x² 代進幾何級數，得到它的級數 1 − x² + x⁴ − x⁶ + …——半徑卻只有 1。x = 2 時函數值好端端的是 1/5，級數卻發散。牆在哪裡？沿著實數線看，什麼障礙都沒有。答案要到複數平面才看得見：x = ±i 時分母 1 + x² = 0。收斂半徑是「到最近一個爆點的距離」，而爆點可以躲在你看不見的維度裡。本書不展開複變理論，但記住這幅畫面：複數不是等一下 Euler 公式才現身的客串演員，它一直在後台管事。

兩枚伏筆。其一：在牆外硬要把級數「加出一個值」，會發生什麼？那是 ch12 整章的詐騙現場。其二：本章一路默認「光滑的函數就乖」——ch13 的怪獸會把這個直覺整個拆掉。最後一件實務事：就算半徑無限大，半徑內也不是人人平等——離展開點越遠，收斂越慢。慢多少？章末的 Python 小實驗讓你親手量。

## 調和級數：第一面牆

現在回頭追究「無限多項加起來」這件事本身。你對它的直覺大概是 ch01 養成的：backoff 間隔 1/2 + 1/4 + 1/8 + … = 1，無限多步、有限總量。一路順風到現在，你的直覺很可能已經偷偷升級成一條定律：「項趨於 0 的級數會收斂」。

請面對：

1 + 1/2 + 1/3 + 1/4 + 1/5 + …

這叫調和級數（harmonic series）。項趨於 0，毫無疑問。部分和爬得極慢，越爬越慢，看起來百分之百是要收斂的樣子。它發散。最早的證明早於微積分三百年——約 1350 年，巴黎的 Nicole Oresme，方法是分組：

```text
1 + 1/2 + (1/3 + 1/4) + (1/5 + 1/6 + 1/7 + 1/8) + (1/9 + … + 1/16) + …

第 1 組（2 項）：1/3 + 1/4             ≥ 1/4 + 1/4   = 1/2    ← 實際是 0.5833
第 2 組（4 項）：1/5 + 1/6 + 1/7 + 1/8 ≥ 4 × 1/8     = 1/2    ← 實際是 0.6345
第 3 組（8 項）：1/9 + … + 1/16        ≥ 8 × 1/16    = 1/2
第 k 組（2^k 項）：每一項都 ≥ 組尾那項 1/2^(k+1)，
                   整組 ≥ 2^k × 1/2^(k+1) = 1/2
```

每組都至少貢獻 1/2，而組數無限。取到第 2ⁿ 項時，部分和 ≥ 1 + n/2——要多大有多大。發散，證畢。

注意這個論證的成色：每一步都只是「把每項換成更小的數」的不等式，沒有任何極限玄學，十四世紀的工具就夠用——這是貨真價實的證明，不是直覺版。（這個結果其後湮沒，十七世紀才由 Mengoli 與 Jacob Bernoulli 重新證明。）

這就是第一面牆：**項趨於 0 是收斂的必要條件，但遠遠不充分。**幾何級數收斂靠的從來不是「項在變小」，而是「項以固定比率變小」——每項砍半，尾巴的總重量被壓成有限。1/n 也在變小，但太客氣：慢到每一段「走到下一個 2 的冪次」的路上，堆起來的項都超過 1/2。「變小」不夠，要「變小得夠快」。

慢到什麼程度（自算複核）：部分和要超過 10，需要 12,367 項；要超過 100，照 ln 估計需要約 1.5×10⁴³ 項（分組論證給的保證是 2¹⁹⁸ ≈ 4×10⁵⁹ 項一定夠——保證打得很寬，實際更早到站，但兩個數字都同樣荒謬）。部分和的成長節奏其實就是對數：

| n | Hₙ（前 n 項部分和） | ln n |
|---|---|---|
| 10 | 2.9290 | 2.3026 |
| 100 | 5.1874 | 4.6052 |
| 1000 | 7.4855 | 6.9078 |

差值穩定收向 0.5772（這個常數叫 Euler–Mascheroni 常數 γ，認得即可）。Hₙ ≈ ln n——如果你想起 ch07 的黎曼和，對：Hₙ 就是 f(x) = 1/x 曲線下面積的粗切版，而 ∫₁ⁿ (1/x) dx = ln n 是 ch08 的 FTC 一行解。這幅圖 ch12 會正式入帳。

這面牆還有個工程版的雙胞胎。用單精度浮點（float32）老老實實逐項加調和級數：加到第 2,097,151 項之後，部分和永遠卡在 15.4037——從那一項起，每個 1/n 都小於部分和的浮點解析度，加了等於沒加（自算複核：純 Python 模擬 float32 逐項加法）。一個數學上發散的級數，被有限精度「加到收斂」。螢幕上的數字不動，從來就不是收斂的證據——收斂的定義只有一個：部分和的極限存在（ch03）。牆後面是一整片雷區，ch12 專章開挖。

## Basel 問題：十八世紀的大膽

調和級數差一步就收斂。把每項平方：

Σ(n=1→∞) 1/n² = 1 + 1/4 + 1/9 + 1/16 + …

先確認它真的收斂——剛被打過臉，這次不靠感覺：

```text
n ≥ 2 時：1/n² < 1/(n(n−1)) = 1/(n−1) − 1/n          ← 拆成相鄰兩項的差

Σ(n=2→N) 1/n² < (1 − 1/2) + (1/2 − 1/3) + … + (1/(N−1) − 1/N)
              = 1 − 1/N < 1                           ← 中間全部對消（telescoping）

所以部分和 < 1 + 1 = 2：遞增、有上界。
```

遞增又有上界的數列必定收斂——這句話聽起來理所當然，背後其實是實數完備性在兜底（ch14 的主場，這裡先記一筆帳）。好，收斂，和不超過 2。**但等於多少？**

這就是 Basel 問題，以 Bernoulli 家族與 Euler 共同的家鄉巴塞爾命名，懸了將近一個世紀，當時第一線的數學家全都試過、全都沒拿下（2026-06 查證）。難處有二。其一，收斂慢得折磨人（自算複核）：S₁₀ = 1.5498、S₁₀₀ = 1.6350、S₁₀₀₀ = 1.6439——誤差大約以 1/n 縮小，想直接加出六位小數要上百萬項，手算時代等於免談。其二更根本：就算你加出 1.644934，你也認不出這個數是誰。

Euler 的第一步非常工程：先把答案**量測**出來。1731 年（論文編號 E20）他發明了級數加速技巧，用幾十項的工作量逼出六位小數 1.644934（2026-06 查證；後來又推進到十幾位）。第二步才是傳世的大膽。本章推過 sin 的級數；Euler 盯著它想：有限多項式由根決定——p(x) 的根是 r₁, …, rₙ，它就能寫成 c·(1 − x/r₁)…(1 − x/rₙ)。而 sin x 的根人人都知道：0, ±π, ±2π, ±3π, …。**如果 sin 像一個「無限長的多項式」一樣服從這條定理呢？**

```text
級數的長相：sin x / x = 1 − x²/3! + x⁴/5! − …               ← 除掉 x，去掉根 0
按根分解：  sin x / x “=” (1 − x²/π²)(1 − x²/4π²)(1 − x²/9π²)…
                                                             ← 每對根 ±nπ 併成一個因子
展開右邊，只追蹤 x² 項（從一個因子拿 −x²/(n²π²)，其餘全拿 1）：
    x² 的係數 = −(1/π² + 1/(4π²) + 1/(9π²) + …) = −(1/π²)·Σ 1/n²
左邊 x² 的係數 = −1/3! = −1/6

對齊兩邊：(1/π²)·Σ 1/n² = 1/6   →   Σ(n=1→∞) 1/n² = π²/6
```

π²/6 = 1.6449340668…——跟量測值 1.644934 嚴絲合縫。1735 年，二十八歲的 Euler 在聖彼得堡宣讀這個結果（論文 E41，卷標 1734/35），一夕成名。

誠實標示，這裡一個字都不能含糊：**這個論證在當年不及格，Euler 自己知道。**「根決定因式分解」是有限多項式的定理，無限版沒有任何定理背書——而且漏洞有方向可指：eˣ·(sin x)/x 跟 (sin x)/x 的根一模一樣，照同樣邏輯該有同樣的分解，多出來的 eˣ 哪去了？Euler 的對沖手段有兩道：數值對帳（六位小數咬合，靠運氣矇到的機率極低），以及交叉驗證——同一招還能算出 Σ1/n⁴ = π⁴/90 等一整族結果，全部與數值吻合。1741 年他補上當時被普遍接受的更嚴謹論證；至於「sin 的無限乘積分解」本身合法，要等到十九世紀 Weierstrass 的因式分解定理才真正補票（2026-06 查證）。

值得停十分鐘的地方：左邊是純粹的整數帳本——1、4、9、16 的倒數；右邊冒出 π。圓周率憑什麼出現在整數的世界裡？順著論證看，答案很乾脆：**π 是從 sin 的零點走私進來的。**級數的世界與圓的世界在這裡第一次握手。這也是十八世紀風格的縮影：形式操作大膽衝鋒、數值複核殿後、嚴格性欠條後補。這套風格戰果輝煌——但同一個 Euler，用同一種膽量去碰發散級數的時候，就沒這麼好運了。勝利與翻車並陳，是 ch12 的戲。

## Euler 公式：指數與旋轉接上線

把本章的三條級數排在一起，看出一件事：

```text
eˣ    = 1 + x + x²/2! + x³/3! + x⁴/4! + x⁵/5! + x⁶/6! + …   ← 全部是 +
cos x = 1     − x²/2!          + x⁴/4!          − x⁶/6! + …   ← 偶數項，+ − 交錯
sin x =     x          − x³/3!          + x⁵/5!         − …   ← 奇數項，+ − 交錯
```

cos 拿走了 eˣ 的偶數項，sin 拿走了奇數項——材料一模一樣，差的只是正負號的節奏：+ + − − + + − −，以 4 為週期。哪裡見過週期 4？i 的冪次：i² = −1，所以 i, −1, −i, 1, i, …——乘兩次翻號，乘四次回家。這不是巧合的長相，這是邀請函。

把 x = iθ 代進 eˣ 的級數（θ 是實數）：

```text
e^{iθ} = 1 + iθ + (iθ)²/2! + (iθ)³/3! + (iθ)⁴/4! + (iθ)⁵/5! + …

用 i² = −1、i³ = −i、i⁴ = 1 逐項整理：

       = 1 + iθ − θ²/2! − iθ³/3! + θ⁴/4! + iθ⁵/5! − θ⁶/6! − …

按「帶不帶 i」分成兩堆：

實部：1 − θ²/2! + θ⁴/4! − θ⁶/6! + …    ← 正是 cos θ 的級數
虛部：θ − θ³/3! + θ⁵/5! − …            ← 正是 sin θ 的級數

所以  e^{iθ} = cos θ + i·sin θ
代 θ = π：cos π = −1、sin π = 0
所以  e^{iπ} = −1，亦即  e^{iπ} + 1 = 0
```

五個常數：e（增長）、i（旋轉，馬上解釋）、π（圓）、1、0——各自來路不同，在一條等式裡會合。

誠實標示，請逐字讀：在代入 iθ 之前，「e 的虛數次方」這幾個字**沒有任何定義**——eˣ 的老定義（連乘也好、ch05 的極限也好）對虛數毫無著力點。把 iθ 塞進一條為實數推導的級數，是形式操作（formal manipulation）：合法性沒有事先保證，跟 Euler 對 sin 動的手腳是同一族。現代數學的處理乾脆俐落：反過來，**拿這條級數當定義**——e^{iθ} 的意思就是把 iθ 代進級數所得的和（複數級數的收斂性可以嚴格建立，複數冪的完整理論本書不展開）。所以 e^{iπ} + 1 = 0 有一半是定義的選擇。真正神奇的不是「它被證明出來」，而是這個定義跟既有的世界咬合得天衣無縫——看一眼就懂。

乘以 i，在複數平面上是逆時針轉 90°：1 → i → −1 → −i → 1，四步回到原點。而 cos θ + i·sin θ 是單位圓上角度 θ 的那個點——所以 e^{iθ} 的意思是：**指數函數吃進一個角度，吐出一個旋轉。**

```
                虛軸
                 │
               i ●          ● e^{iθ} = cos θ + i·sin θ
                 │        ╱
                 │      ╱  半徑 1
                 │    ╱
                 │  ╱  夾角 θ
    ●────────────┼────────────●──── 實軸
   −1            │            1
                 │
              −i ●
                 │
```

再給一個微分視角（級數逐項微分，跟 eˣ 同一招）：d/dθ e^{iθ} = i·e^{iθ}——速度等於位置乘上 i，也就是**速度永遠垂直於位置、長度不變**。這正是等速圓周運動的定義。ch05 的自戀函數到了複數世界，自戀成一個圓；θ = π 走半圈，正好停在 −1 上——e^{iπ} = −1 的幾何讀法就這麼平實。這條「指數＝旋轉」的通道，是 ch10 振動方程的捷徑、ch11 傅立葉把任何訊號拆成圓的地基。歷史歸檔：系統性的表述出自 Euler 1748 年的《Introductio in analysin infinitorum》；等價的對數形式 Roger Cotes 在 1714 年更早寫過，Euler 的貢獻是指數形式與系統運用。

順帶一提，這條推導你應該親手做一次——「紙上推演」第 2 題等你，那是全書最划算的二十分鐘。

**題 1 [15 分鐘] ★★** — 用 eˣ 的級數估 e^0.5：(a) 取前三項算出數值。(b) 不查真值，先回答：這個估計偏高還是偏低？為什麼 sin 的截斷誤差會交錯換邊、這裡卻永遠同一邊？(c) 用「下一項」當下界、用一條幾何級數蓋住尾巴當上界，把誤差夾出範圍，再對照真值 1.6487213 驗收。

**題 2 [20 分鐘] ★★（全書最划算的二十分鐘）** — 闔上書。白紙上寫下 eˣ 的級數，先在角落列出 i¹ 到 i⁸ 的對照表，然後代入 x = iθ，自己把實部與虛部整理出來，認出它們是誰，最後代 θ = π。做完回答一個問題：整條推導裡，哪一步是「形式操作」？

**題 3 [10 分鐘] ★★（口頭題）** — 一位同事說：「每一項都趨近於零欸，加起來怎麼可能無限大？」用 Oresme 的分組把他講服。限制：不准寫部分和的公式，只能用「打包」的語言。講完對照解答的要點清單自評。

### 推演解答

**題 1**　(a) 1 + 0.5 + 0.5²/2! = 1 + 0.5 + 0.125 = 1.625。

(b) 偏低。x > 0 時 eˣ 的級數每一項都是正的，丟掉的尾巴是一堆正數，截斷必定低估——而且永遠低估，不管你截在第幾項。sin 的級數正負交錯，每多一項就跨過真值換一邊，截斷誤差才會高低擺盪。「級數的符號結構」決定「誤差的行為」，這個對應值得記住。

(c) 下界：尾巴的第一項就是 0.5³/3! = 0.0208333，後面還全是正的，所以誤差 > 0.0208333。上界：從第四項起，相鄰兩項的比是 0.5/(n+1) ≤ 0.5/4 = 0.125，所以尾巴 < 0.0208333 × (1 + 0.125 + 0.125² + …) = 0.0208333/(1 − 0.125) = 0.0238095——ch01 的幾何級數在這裡當天花板用。驗收：1.6487213 − 1.625 = 0.0237213，果然落在 (0.0208333, 0.0238095) 之間（自算複核）。你剛剛親手用過的「拿幾何級數蓋住尾巴」，正是各種收斂判別法的核心直覺——本書不開判別法目錄，但最重要的一款你已經會了。

**題 2**　對照表：i¹ = i、i² = −1、i³ = −i、i⁴ = 1，之後循環（i⁵ = i、…、i⁸ = 1）。代入整理後，不帶 i 的項是 1 − θ²/2! + θ⁴/4! − …（cos θ），帶 i 的項是 θ − θ³/3! + θ⁵/5! − …（sin θ）；代 θ = π 得 e^{iπ} = −1 + i·0。自我檢查點：整理途中正負號應該出現 + + − − 的節奏——那是 i 在打拍子；如果你的符號亂掉，多半是 (iθ)³ = i³θ³ = −iθ³ 這類地方漏了負號。「形式操作」在第一步：把為實數推導的級數對虛數使用，當下沒有任何合法性保證——現代的補救是反過來拿級數當 e^{iθ} 的定義。至於為什麼划算：二十分鐘的代數，買到「指數＝旋轉」這條貫穿 ch10（振動）、ch11（傅立葉）以及半個現代工程（交流電、訊號處理、量子力學）的通道。

**題 3**　模範答案要點清單（必須命中全部五點）：

1. 先承認他的直覺來源：他見過的收斂級數（backoff 的 1/2ⁿ）項確實趨於 0——但箭頭方向反了：收斂的級數項必趨於 0；項趨於 0 的級數不一定收斂。
2. 打包：從 1/3 開始，按「走到下一個 2 的冪次」打包——先 2 項一包、再 4 項、再 8 項，包包都打得滿。
3. 每包估重：包裡每一項都不小於包尾那項，所以每包總重至少 2ᵏ × 1/2ᵏ⁺¹ = 1/2。
4. 收尾一擊：他手上有無限多個重量 ≥ 1/2 的包裹，部分和想到多高就到多高——沒有天花板，就是發散。
5. 對比衰減速度：1/2ⁿ 每項砍半（固定比率），1/n 越砍越客氣——「變小」不夠，要「變小得夠快」；快慢的分界線，這兩條級數一邊站一個。

### Python 小實驗

同一條 sin 級數，在 x = 0.5 與 x = 3 各取前 8 個部分和，親手量「離展開點的距離」值多少項（純標準函式庫）：

```python
# Taylor partial sums of sin around 0: x - x^3/3! + x^5/5! - ...
# How many terms do we need at x = 0.5 vs x = 3.0?
import math

def partial_sums(x, n_terms):
    s, term, out = 0.0, x, []
    for k in range(n_terms):
        s += term
        out.append(s)
        term *= -x * x / ((2 * k + 2) * (2 * k + 3))  # next odd-power term
    return out

for x in (0.5, 3.0):
    true = math.sin(x)
    print(f"x = {x}   true sin(x) = {true:.10f}")
    print(f"{'terms':>5}  {'partial sum':>14}  {'|error|':>9}")
    for k, s in enumerate(partial_sums(x, 8), start=1):
        print(f"{k:>5}  {s:14.10f}  {abs(s - true):9.1e}")
    print()
```
