# ch06 — 最佳化：斜率為零的地方

> **本章解決什麼問題**：ch04 抓住了瞬間（導數的定義），ch05 把它變成可以機械操作的代數——但到目前為止，導數只是「被算出來」，還沒做過事。這一章是導數的第一份正式工作：在所有可能裡找最好的。費馬的觀察（山頂的切線是平的）把「無限多個候選點」壓縮成「解一條方程式」；方程式解不出來時，柯西的梯度下降把它退一步變成「往坡下走」。Part II 到此收工，ch07 換到微積分的另一半動作：累積。

## 從你已知的出發

你做過這件事：用 k6 壓測找一個服務的最佳並發數。併發 100，吞吐量上升；200，再升；400，P99 延遲爆掉、吞吐反而掉下來。你在 200 和 400 之間來回收緊，直到出現那個熟悉的手感——**再加就變差，再減就浪費**。autoscaling 的步距、worker pool 的大小、cache 的 TTL，你都調過同一種東西：一個「動一下也不會更好」的點。

注意你判斷「到頂了」的依據。不是吞吐量的絕對數字——你事先根本不知道上限是多少——而是**鄰居都比它差**這個局部特徵。最佳點的身分證不是「值很大」，是「往兩邊動都變差」。把這句話翻成微積分，就是本章的主角：在最高點，斜率是零。

還有一件你概念上已經知道的事：機器學習的訓練是「往坡下走」（梯度下降）。這章補上它的數學身世——你會發現它跟十七世紀一位法國法官的業餘觀察是同一條血脈。

## 山頂上，切線是平的

想像一條光滑的山稜線。站在任何一個斜率為正的點，往右踏一步會更高，所以那裡不是山頂；斜率為負，往左一步會更高，也不是山頂。山頂唯一的可能：**切線躺平，斜率為零**。

```
              ─────          ← 山頂：切線躺平，f′ = 0
             *     *
            *       *
          *           *      ← 右半邊 f′ < 0：往左走會更高
        *               *
      *                   *  ← 左半邊 f′ > 0：往右走會更高
    ────────────┬────────────
                x*
```

這個觀察可以寫成論證（工程師的嚴謹：每一步能口頭說出理由；「夠小時貼近」那一步藏著 ch03 的極限挑戰賽，正式版 ch14 一行寫完）：

```text
設 x* 是內點的極大值，且 f 在 x* 可微，記 L = f′(x*)。

若 L > 0：差商 (f(x*+h) − f(x*))/h 在 h 夠小時貼近 L > 0（ch04 的定義）。
  取夠小的 h > 0：差商 > 0 且 h > 0，所以 f(x*+h) > f(x*)。
  右邊有更高的點，x* 不是極大值——矛盾。
若 L < 0：取夠小的 h < 0，同樣得到 f(x*+h) > f(x*)——矛盾。
所以 L = 0。
```

最早把這件事寫成**方法**的人是費馬（Fermat）——本業是土魯斯的法官，數學是下班後的事。他在 1636 年前後流傳的手稿《求極大與極小的方法》（Methodus ad disquirendam maximam et minimam）裡給出一套機械流程：成果可追溯到 1629 年，經由梅森（Mersenne）的通信圈流傳，身後 1679 年才正式出版——比牛頓與萊布尼茲的微積分早了將近半個世紀。他的招數是把 f(x+e) 和 f(x)「擬等」起來（他自創的詞：adequality），整理式子、約掉 e、然後把剩下的 e 直接丟掉。是的，這正是一百年後 Berkeley 嘲笑的「消逝量的鬼魂」（見 ch04）——費馬比牛頓更早養了這隻鬼，而且用它辦案辦得有聲有色。

更早還有一次「目擊紀錄」：克卜勒（Kepler）1615 年的《酒桶的新立體幾何》（Nova stereometria doliorum vinariorum）。他買酒時不服酒商用一根斜插的量尺就敢報價，回家算了九十多種旋轉體的容積，途中注意到兩件事：奧地利酒桶的比例幾乎正好是同樣量尺讀數下容積最大的形狀；而且**在最大值附近，尺寸怎麼動、容積幾乎不變**。「最大值附近函數變化最慢」——這就是「山頂是平的」的第一次書面登場，比費馬的方法還早二十年。

**這裡了不起在哪**：在費馬之前，「找最大」是跟無限多個候選點比大小——你得贏過整條曲線。費馬把它換成「解 f′(x) = 0」：一條方程式，有限個解。**無限大的搜尋空間，塌縮成一張有限的嫌疑犯名單**。這是本書會反覆出現的馴服手法：ch03 把「無限多步」收斂成一個可驗收的極限，這裡把「無限多個候選」壓成幾個嫌疑犯。順帶一提，費馬後來把同一招指向大自然：光在介質間走的是「最省時」的路徑（1662 年的最短時間原理，折射定律從最佳化裡直接掉出來）——大自然自己也在解最佳化問題。

但注意我的用詞：**嫌疑犯名單，不是判決書**。f′ = 0 是「極值的必要條件」——山頂一定平，平的不一定是山頂。它會誤報（平的但不是極值），它的前提（內點、可微）破掉時也會漏報（是極值但抓不到）。三個經典反例在「直覺的陷阱」開庭；先記住完整的辦案流程：

1. **抓駐點**（stationary point）：解 f′(x) = 0 的點。
2. **抓不可微點**：f′ 不存在的點（尖角、斷裂）。
3. **抓邊界點**：定義域的端點。
4. **全部帶回 f(x) 比大小**——比完才有判決。

還有一句要誠實說：「最大值一定存在」這件事本身不是白來的——連續函數在閉區間上才保證有最大值（極值定理），這個保證的地基是實數的完備性（見 ch14）。在開區間或不連續的世界裡，嫌疑犯可能全體無罪，因為根本沒有罪案。

## 彎的方向：f″ 的直覺

名單上有了嫌疑犯，怎麼快速分辨山頂和谷底？看**二階導數** f″——斜率的斜率。f′ 告訴你地面平不平，f″ 告訴你地面往哪邊彎。

```
   皺眉（f″ < 0）          微笑（f″ > 0）
       ***                 *         *
     *     *                *       *
    *       *                *     *
   *         *                 ***
   山頂住在這種彎裡         谷底住在這種彎裡
```

直覺版一句話：f″ < 0 表示斜率一路遞減——左邊還在升、右邊已經在降，駐點夾在中間就是局部山頂；f″ > 0 反過來是谷底。這個判斷夠用了，本書不操練正式的「二階導數判別法」；只提醒一句：f″ = 0 時二階資訊也沉默（x³ 和 x⁴ 在 0 都是 f″ = 0，命運卻不同），這時別查表，回去直接看 f′ 的正負或函數值本身。

## 圍籬問題：一次完整的辦案

經典題，完整走一遍流程。手上有 40 公尺圍籬，圍一塊長方形的地，怎麼圍面積最大？

設一邊長 x，周長固定 40，所以另一邊是 (40 − 2x)/2 = 20 − x。定義域：0 ≤ x ≤ 20（端點是圍籬塌成一條線的退化情況）。目標函數與一階條件：

```text
A(x)  = x(20 − x) = 20x − x²
A′(x) = 20 − 2x            ← (20x)′ = 20；(x²)′ = 2x（見 ch04）；逐項相減（見 ch05）
A′(x) = 0  ⇒  20 − 2x = 0  ⇒  x = 10
```

辦案流程四步走：駐點 x = 10；A 是多項式、處處可微，沒有不可微嫌疑犯；邊界 x = 0 與 x = 20。帶回比大小：

```text
A(10) = 10 × 10 = 100      ← 駐點
A(0)  = 0 × 20  = 0        ← 邊界：圍籬塌成線
A(20) = 20 × 0  = 0        ← 邊界：另一條線
```

判決：x = 10，兩邊都是 10——**正方形勝出**，面積 100 平方公尺。二階複核：A″(x) = −2，處處小於零，整條曲線都在皺眉，所以唯一的駐點必是全域山頂。

```
A(x)
100 ┤          ***
    │        *     *
 75 ┤      *         *
    │    *             *
 50 ┤   *               *
    │  *                 *
 25 ┤ *                   *
    │*                     *
  0 └───────────────────────►
     0         10          20   x
```

兩個值得多看一眼的地方。第一，**對稱性早就暗示了答案**：把 x 換成 20 − x，問題一個字都沒變，所以如果最佳解唯一，它必須落在對稱中心 x = 10。對稱性論證一行破案——但它需要「解唯一且存在」當前提；微分給的是不靠對稱、不靠運氣的通法。第二，這題的約束（周長 40）被我們直接代入消掉了；當約束沒辦法這樣消掉時（例如約束是一條曲面），有一把叫 Lagrange 乘數的工具處理「在約束面上躺平」的條件——多變數領域，本書不展開，延伸閱讀有路標。

## 解不出方程式的時候：往坡下走

圍籬問題能破案，是因為 A′(x) = 0 是條一次方程式，國中生都會解。但現代的最佳化問題長這樣：損失函數有上億個變數，f′ = 0 寫得出來、解不出來。怎麼辦？

退一步：解不出「斜率為零的地方」，就**沿著斜率往下走**，走到斜率自己變成零為止。這個想法歸於柯西（Cauchy）1847 年的一篇短文——他當時在處理天文計算裡的聯立方程，給出的方法今天叫**梯度下降**（gradient descent）。規則只有一行：

```text
xₙ₊₁ = xₙ − η·f′(xₙ)       ← 站在 xₙ，沿下坡方向踏一步
```

為什麼是「減」：斜率為正表示右邊是上坡，下坡在左邊，所以要往負方向走；斜率為負則反過來。步子的大小由 η（**學習率**，learning rate）乘上斜率決定——坡越陡步子越大，坡越平步子越小，**內建煞車**。

拿全章的基準例子手跑一遍：f(x) = x²，f′(x) = 2x，η = 0.1，起點 x₀ = 3。先把更新式化簡：

```text
xₙ₊₁ = xₙ − 0.1·(2xₙ) = (1 − 0.2)·xₙ = 0.8·xₙ    ← 整個演算法縮成一個乘數
```

| n | xₙ | f(xₙ) = xₙ² |
|---|------|---------|
| 0 | 3 | 9 |
| 1 | 2.4 | 5.76 |
| 2 | 1.92 | 3.6864 |
| 3 | 1.536 | ≈ 2.3593 |
| 4 | 1.2288 | ≈ 1.5099 |
| 5 | 0.98304 | ≈ 0.96637 |

f 每一步都在降，而且降得很有紀律：x 每步乘 0.8，f 每步剛好乘 0.64。換句話說 xₙ = 3·0.8ⁿ——**梯度下降在這個例子裡就是一條幾何數列**，跟你 exponential backoff 的重試間隔（見 ch01）是同一族。它永遠走不到 0，但 ch03 教過你這不叫失敗：lim(n→∞) xₙ = 0，要多近有多近。工程上的停止條件也跟 ch03 一樣務實：|xₙ₊₁ − xₙ| < eps 就收工。

接著看這個演算法最著名的故障模式。因為更新式縮成了 xₙ₊₁ = (1 − 2η)·xₙ，整個行為由一個乘數決定：

| η | 乘數 1 − 2η | 行為 |
|------|-------|------|
| 0.01 | 0.98 | 收斂，但龜速爬行 |
| 0.1 | 0.8 | 單調收斂（上表）|
| 0.5 | 0 | 一步到位（拋物線特例）|
| 0.8 | −0.6 | 跳過谷底、左右震盪，但越盪越小，收斂 |
| 1.0 | −1.0 | 永遠在 3 與 −3 之間擺盪 |
| 1.1 | −1.2 | 越盪越大，發散（紙上推演親手算）|

收斂條件就是 |1 − 2η| < 1，也就是 0 < η < 1。太小：η = 0.01 要走約 110 步才達到 η = 0.1 十步的效果（0.98ⁿ 對上 0.8ⁿ）。太大：每一步跨過谷底摔到對面更高的坡上，越摔越遠。誠實標示：「一個乘數看穿一切」只對 f(x) = x² 成立（它的更新式剛好是線性的）；一般函數只在谷底附近近似如此——但「太大震盪、太小龜速、中間剛好」這張相圖是普遍的。

現在回頭看你調過的東西。ML 設定檔裡那個 learning rate，字面意義就是這個 η：訓練曲線炸開是乘數絕對值大於一，loss 像蝸牛是乘數太貼近一。autoscaling 的 scale-up 步距太猛，副本數在高低水位之間 flapping——那就是 η 太大的震盪，你只是沒叫過它的數學名字。k6 壓測的 ramp 策略、二分收緊的步距，全是同一個權衡：**步長換穩定**。你不是今天才認識梯度下降，你是今天才知道它姓什麼。

## 多維一瞥：梯度是最陡的方向

真實的損失函數不是一條曲線，是一片上億維的地形。一維時斜率是一個數；多維時，每個方向各有一個斜率，把它們打包成一個向量，就叫**梯度**（gradient），記作 ∇f。它有一個本章只給直覺、不給證明的關鍵性質：**梯度指向上坡最陡的方向**，所以負梯度就是最速下降的方向。

想像起霧的山坡。你看不到全景，只感覺得到腳下地面的傾斜——於是你朝腳感最陡的下坡方向踏一步，再感覺、再踏。這就是高維梯度下降的全部：看不到地圖，只摸得到局部坡度，照走。

訓練一個模型，就是在這片地形上做這件事：損失函數對上億個參數的梯度，由連鎖規則（見 ch05）一層層算出來——這套系統化的記帳法叫反向傳播，數學上就是連鎖規則的反向應用，1986 年 Rumelhart–Hinton–Williams 的 Nature 論文使它普及（反向模式的自動微分更早就有人做出來；機制細節見 ch15）。把血脈排開：**費馬 1636 年的平頂觀察＋柯西 1847 年的下坡走法＋萊布尼茲的連鎖規則，就是 2026 年每一次模型訓練的引擎室**。沒有新魔法，只有三百年的舊觀察換上 GPU。

多維還帶來一個新角色：**鞍點**（saddle point）——馬鞍的中心，沿一個方向看是谷底、沿另一個方向看是山頂，梯度照樣是零。維度越高，「每個方向都剛好同時往上彎」越難得，所以高維地形裡的駐點被普遍認為大多是鞍點而非谷底（深度學習文獻的共識級說法；2026-06，本書未逐篇查證原始文獻）。這正是嫌疑犯名單直覺在高維的回聲：訓練「卡住了」未必是掉進局部最小，常常只是在鞍點附近的平原上磨蹭。

## 直覺的陷阱

f′ = 0 這條線索太好用，好用到大腦會自動把「必要條件」升級成「充分條件」。三個反例開庭，每個都是五秒鐘能畫出來的圖。

**案例一：誤報——f(x) = x³ 在 x = 0。** f′(x) = 3x²，所以 f′(0) = 0，嫌疑犯名單上有它。但左邊 f(−0.1) = −0.001 比 f(0) = 0 小、右邊 f(0.1) = 0.001 比它大——它兩頭不討好也兩頭不輸，根本不是極值，只是上坡途中的一個平台。

```
            *
           *
          *
        **
      **          ← x = 0 附近：切線躺平，但只是樓梯的平台
    **
   *
  *
 *
```

**案例二：漏報——f(x) = |x| 在 x = 0。** 這是不折不扣的最小值，但左斜率 −1、右斜率 +1，切線根本不存在（見 ch04 的差商定義：左右極限不相等）。費馬的論證需要「可微」當前提，尖角不簽這份合約，所以解 f′ = 0 永遠抓不到它。

```
 *           *
  *         *
   *       *
    *     *
     *   *
      * *
       *          ← x = 0：谷底，但這裡沒有切線可言
```

**案例三：漏報——邊界。** 把圍籬問題反過來問：面積最小怎麼圍？嫌疑犯名單一個字不變（駐點 x = 10、邊界 0 和 20），但判決翻盤：最小值在兩個端點，A = 0。注意 A′(0) = 20，斜率根本不是零——因為在邊界你只能往一邊動，「往兩邊動都變差」的論證直接垮掉。只解 f′ = 0 就交卷的人，這題零分。

| 錯誤直覺 | 會把你帶溝裡的那一步 | 修正 |
|---|---|---|
| f′ = 0 就是極值 | 把 x³ 的平台當谷底，對著鞍點宣布勝利 | f′ = 0 只是「地面平」；用 f″ 或直接比函數值定罪 |
| 極值一定在 f′ = 0 處 | 漏掉 \|x\| 的尖角、漏掉端點 | 名單三來源：駐點＋不可微點＋邊界 |
| 局部最小＝全域最小 | 在雙谷地形裡找到一個谷就停手 | 一階、二階條件全是局部證詞；全域要另外比 |
| 梯度下降總會收斂 | 學習率調大「只是快一點」 | 乘數絕對值過一就發散；先看 η 再看玄學 |
| 訓練卡住＝局部最小 | 對著鞍點平原加 epoch | 高維駐點多半是鞍點；卡住的地形不只一種 |

怎麼自我察覺還沒懂：解完 f′ = 0 就交卷、沒有反射性地問「端點呢？不可微的點呢？」；說不出 x³ 在 0 那個點的名字（駐點，但非極值）；以為調大學習率只影響速度不影響成敗。出現任何一條，回來重讀辦案流程。

## 紙上推演

**題 1 [10 分鐘] ★**　對 f(x) = x³ 在 x = 0：算出 f′ 並驗證 f′(0) = 0；再解釋為什麼它不是極值，並說出這種點的正式名字。

**題 2 [15 分鐘] ★★**　把學習率改成 η = 1.1，從 x₀ = 3 重跑五步迭代表（每步算 xₙ 與 f(xₙ)），描述你看到的行為；然後找出讓這個迭代「一步到位」的 η 值。

**題 3 [10 分鐘] ★**　兩數之和為 20，乘積最大是多少？用一階條件解，然後指出它為什麼跟圍籬問題是同一題。

### 推演解答

**題 1**　f′(x) = 3x²，f′(0) = 0，駐點成立。但 3x² 在 0 的兩側都大於零——函數在左右兩邊都在上升，0 只是上升途中切線剛好躺平的一瞬。直接比值更乾脆：f(−0.1) = −0.001 < f(0) = 0 < f(0.1) = 0.001，左邊有更小、右邊有更大，兩種極值都當不成。名字：**駐點但非極值**（一維常稱「有水平切線的反曲點」；多維的近親就是鞍點）。常見錯路：去查 f″(0) = 0 然後說「判別法失敗，所以沒有極值」——不對，f″ = 0 是二階資訊沉默，不是判無罪；x⁴ 在 0 同樣 f″ = 0，卻是貨真價實的最小值。沉默時回去看 f′ 的正負或函數值。

**題 2**　更新式：xₙ₊₁ = xₙ − 1.1·(2xₙ) = (1 − 2.2)·xₙ = −1.2·xₙ。

| n | xₙ | f(xₙ) = xₙ² |
|---|--------|---------|
| 0 | 3 | 9 |
| 1 | −3.6 | 12.96 |
| 2 | 4.32 | 18.6624 |
| 3 | −5.184 | ≈ 26.874 |
| 4 | 6.2208 | ≈ 38.698 |
| 5 | −7.46496 | ≈ 55.726 |

行為：每一步都跳過谷底摔到對面，而且 |x| 每步放大 1.2 倍、f 每步放大 1.44 倍——震盪發散。對照 η = 0.1 的表：同一個演算法、同一個起點，只差一個超參數，一個優雅收斂、一個直奔無窮。「一步到位」：要乘數 1 − 2η = 0，即 **η = 0.5**——一步從 3 跳到 0。誠實註明這是拋物線的特例：這個「最聰明步長」其實偷用了 f″ = 2 這個二階資訊（用二階資訊選步長是另一族演算法的起點，本書不展開）。

**題 3**　設兩數為 x 與 20 − x，乘積 P(x) = x(20 − x)——跟圍籬問題的 A(x) 一字不差。P′(x) = 20 − 2x = 0 給 x = 10，乘積 10 × 10 = 100；端點 P(0) = P(20) = 0。為什麼是同一題：圍籬問題裡「周長 40 的長方形」翻譯出來就是「兩邊長之和為 20」，約束代入消掉之後，目標函數一模一樣。數學不在乎你嘴上說的是公尺還是純數——**最佳化問題的皮各有不同，骨架常常相同**，而建模就是剝皮找骨的手藝。

## 自我檢核

口頭自答，講得出來才算過：

1. 「內點極值＋可微 ⇒ f′ = 0」的反證怎麼走？斜率不為零時，往哪邊走會更高？
2. 為什麼說 f′ = 0 給的是嫌疑犯名單而不是判決書？三種失效（誤報、尖角漏報、邊界漏報）各舉一個五秒內畫得出來的例子。
3. f″ 在這個故事裡是什麼角色？「斜率的斜率」這句話拆開來講是什麼意思？
4. 圍籬問題如果把「最大」改成「最小」，答案搬到哪裡去了？這件事暴露了辦案流程的哪一步絕對不能省？
5. 梯度下降的更新式為什麼是減號？學習率太大時，幾何上發生了什麼事（用「跨過谷底」講一遍）？
6. 你自己維運過的系統裡，哪個參數扮演 η？它「震盪」的時候，監控圖上長什麼樣？
7. 用 30 秒對另一個工程師講完這條血脈：費馬 1636 → 柯西 1847 → 2026 年的模型訓練，每一站各貢獻了什麼？
