# ch11 — 傅立葉：把任何訊號拆成圓

> **本章解決什麼問題**：Part IV 走到第三步。ch09 把函數攤開成無限級數、ch10 讓微分方程開始預測世界，本章是語言長大的最後一躍：傅立葉宣稱**任何**週期訊號都能拆成純頻率的無限疊加。你會親眼看著最光滑的正弦疊出帶稜角的方波、看到 ch07 的積分搖身變成量測儀器、看到一個怎麼壓都壓不掉的 9% 過衝——然後看著「等於」兩個字把整個數學逼進嚴格化危機。本章是 Part IV 的終點，也是 Part V 的引橋。

## 從你已知的出發

打開 Grafana，把 QPS 曲線拉成一週的視圖。你的眼睛會自動把它讀成幾層：白天高、半夜低的**日週期**；週末整體塌下去的**週週期**；底下一條緩緩上升的**趨勢**（業務在長）；最上面一層毛毛的**雜訊**。你做容量規劃時心裡的那句「扣掉日夜起伏，基線其實在漲」，就是在做訊號分解——你只是沒寫下數學。

再看一眼音樂播放器的 EQ：三十幾根柱子，每根代表一段頻率的能量。一首歌是一條隨時間擺動的波形，EQ 卻把它顯示成「每個頻率各有多少」——同一個訊號的兩種長相：**時域（time domain）**是「每個時刻值是多少」，**頻域（frequency domain）**是「每個頻率佔多少」。你天天在這兩個視角之間切換，切得毫不費力。

本章要回答的問題是：這種「拆成頻率」的直覺，到底是修辭，還是數學？工程師的本能會說「對夠乖的訊號，大概可以吧」。傅立葉（Fourier）說的是：**任何**週期訊號，**精確地**等於純頻率的疊加，而且每個頻率該佔多少，算得出來、答案唯一。這個「任何」惹出的麻煩，比微積分史上任何一個字都大。

## 一個關於熱的問題，與一個過頭的答案

場景在十九世紀初的法國。傅立葉研究的是熱：一根鐵棒受熱後，溫度分布怎麼隨時間演化？這是一條微分方程的事——溫度的時間變化率，由溫度分布的空間彎曲程度決定（定律寫的是變化率，不是軌跡：ch10 的世界觀）。熱方程本身點到為止，本書不解它。

關鍵是傅立葉的解題策略。他發現這條方程對**純正弦形狀**的溫度分布簡單到不像話：正弦形狀不變，只是整體按指數衰減，而且頻率越高衰減越快。所以他的算盤是：把**任意**的初始溫度分布拆成正弦的疊加，讓每個正弦各自衰減，再加回來。整個計畫成立的前提只有一個——任何初始分布都拆得開。於是他直接宣稱：

**任何週期函數，都等於 sin 與 cos 的無限和。**

任何。包括帶跳躍的、帶稜角的、你隨手畫的。當時的數學大老們不買帳，而他們的理由非常有道理：sin 與 cos 是世界上最光滑的函數——無限次可微、每一階導數都溫文儒雅；有限個光滑函數相加，還是光滑的，這是定理。那憑什麼疊加**無限**個之後，能長出方波的直角跳躍？光滑性去哪了？這個質疑你應該覺得耳熟：它跟 ch09「多項式無限疊加居然能變出 sin」是同一種震驚，但這次賭注更大——這次連**不連續**的函數都號稱拆得開。

歷史照行程表走：1807 年 12 月 21 日，傅立葉向法蘭西科學院提交熱傳導備忘錄；拉普拉斯（Laplace）、蒙日（Monge）傾向刊行，但拉格朗日（Lagrange）反對——主因正是三角級數的處理方式——論文未獲出版。1811 年科學院以熱傳導為題懸賞，傅立葉的修訂稿得了獎，評語仍然是不夠嚴格。直到 1822 年（拉格朗日已過世），他才出版《熱的解析理論》（Théorie analytique de la chaleur），把整套方法公諸於世。一個物理問題的解法被數學的守門人擋了十五年——而雙方都有道理：傅立葉的方法是對的，拉格朗日的質疑也是對的。這筆帳要到本章結尾才開始清算。

## 親眼看：光滑的波疊出方角

先說書名裡的「圓」：一個純頻率 sin x，就是等速圓周運動投在一條軸上的影子（ch09 用 e^{iθ} 把指數與旋轉接上了線）。把訊號拆成 sin 與 cos，等於把它拆成一堆半徑不同、轉速不同的圓。本章用「波」的畫面就夠，圓的畫面留給延伸閱讀的動畫。

主角登場。方波（square wave）：前半週期 +1、後半週期 −1，工程師最熟的訊號——clock、PWM、開關。它在 x = 0、π、2π 處硬生生跳躍，是「最不光滑」的代表。傅立葉級數（Fourier series）給它的拆法是：

```text
方波(x) = (4/π) · ( sin x + sin 3x/3 + sin 5x/5 + sin 7x/7 + … )
```

只有奇數倍頻率，振幅按 1/n 衰減。係數哪來的，下一節算給你看；先看疊加的過程。取前 1 項：

```
    │        ********
 +1 ┤     ***        ***
    │   **              **
    │ **                  ***
  0 ┼*───────────────────────*───────────────────────*
    │                         ***                  **
    │                            **              **
 −1 ┤                              ***        ***
    │                                 ********
    └─────────────────────────────────────────────────
    0                        π                       2π
```

一條正弦，振幅 4/π ≈ 1.27——中段甚至衝得比方波的 +1 還高。看不出方波的影子。加到前 3 項（sin x + sin 3x/3 + sin 5x/5）：

```
    │   **              **
 +1 ┤  *  **************  **
    │ *
    │*                      *
  0 ┼────────────────────────*────────────────────────
    │                         *                      *
    │                                               *
 −1 ┤                          **  **************  *
    │                            **              **
    └─────────────────────────────────────────────────
```

頂部開始壓平、側壁開始立起來，但平頂上有漣漪、跳躍兩旁翹起兩隻「耳朵」。前 5 項：

```
    │  *                  **
 +1 ┤ * ******************
    │                       *
    │*
  0 ┼────────────────────────*────────────────────────
    │                                                *
    │                         *
 −1 ┤                            ****************** *
    │                          **                  *
    └─────────────────────────────────────────────────
```

平頂更平、側壁更陡，整體已經像一個方波了。光滑的波，真的在疊出方角——拉格朗日輸了一半（為什麼只輸一半，Gibbs 那節見分曉）。

數值對帳。取最容易算的點 x = π/2（方波正中央，真值 1）：

```text
x = π/2 時，各項的值：
  sin(π/2)    = 1
  sin(3π/2)/3 = −1/3        ← sin(3π/2) = −1
  sin(5π/2)/5 = +1/5        ← sin(5π/2) = +1

前 3 項部分和 = (4/π)(1 − 1/3 + 1/5)
             = (4/π)(13/15)
             = 52/(15π)
             ≈ 1.10347       ← 真值是 1，高了約 10%
```

把更多部分和排成表（寫章時逐項複算）：

| 取前 n 項 | 部分和在 x = π/2 | 與真值 1 的差 |
|---|---|---|
| 1 | 1.27324 | +0.27324 |
| 2 | 0.84883 | −0.15117 |
| 3 | 1.10347 | +0.10347 |
| 4 | 0.92158 | −0.07842 |
| 5 | 1.06305 | +0.06305 |
| 50 | 0.99363 | −0.00637 |

上下擺盪、越夾越緊，收向 1。順帶一個漂亮的對帳：「方波在 π/2 等於 1」整理一下就是 1 − 1/3 + 1/5 − 1/7 + … = π/4——著名的 Madhava–Leibniz 級數。傅立葉級數隨手吐出一條圓周率公式，這種「不同領域在你沒注意時握手」的瞬間，是這門語言長大的標誌。

## 正交性：積分變成探測器

現在補上欠的帳：係數是怎麼算出來的？表面上這是個絕望的問題——未知數有無限多個（b₁, b₂, b₃, …），你只有一條等式。按工程直覺，這得解一個無限大的聯立方程。

傅立葉的做法漂亮到值得你停十分鐘。先定義兩個訊號的**內積（inner product）**：相乘，再對一個週期積分——∫[0,2π] f(x)·g(x) dx。把它讀成「相關性讀數」：兩個訊號同步起伏，乘積偏正、讀數大；毫無默契，正負抵銷、讀數歸零。關鍵事實：

**不同頻率的內積是零。**

這就是正交（orthogonal）——「垂直」的推廣：兩個向量垂直時內積為零，兩個頻率「互不干擾」時也是。本書驗證一例、其餘標明不證（一般情形用同樣的對稱性或積化和差可證）。看 ∫[0,2π] sin x · sin 2x dx：不用積，畫出乘積 sin x · sin 2x 就結案了——

```
    │            ┊                       ┊
    │     ++++   ┊                       ┊   ++++
    │    +    ++ ┊                       ┊ ++    +
    │  ++       +┊                       ┊+       ++
  0 ┼++──────────-─────────-----─────────-──────────++
    │            ┊-       -     -       -┊
    │            ┊ -    --       --    - ┊
    │            ┊  ----           ----  ┊
    └─────────────────────────────────────────────────
    0           π/2                     3π/2         2π
```

四瓣，正、負、負、正。以 x = π/2 的虛線為鏡，把左右兩點 x 與 π−x 配對：sin x 的值不變（對 π/2 對稱），sin 2x 的值變號（sin(2(π−x)) = −sin 2x），乘積一正一負、兩兩抵銷——前兩瓣面積互相吃掉。後兩瓣以 x = 3π/2 為鏡，同樣的論證再走一遍。總積分：0。沒有任何硬算，純對稱性。

反過來，**自己對自己**的內積不是零：∫[0,2π] sin 3x · sin 3x dx = ∫ sin²3x dx——平方永遠 ≥ 0，沒有負瓣可抵，積出來是 π（sin² 的平均值是 1/2，乘上週期長 2π）。

這兩件事合起來，就是一台探測器。想從 f 裡讀出 b₃：拿 sin 3x 去「照」它——

```text
假設  f(x) = b₁ sin x + b₂ sin 2x + b₃ sin 3x + …
兩邊同乘 sin 3x，對一個週期積分（積分範圍都是 [0,2π]，省略不寫）：

∫ f(x)·sin 3x dx
  = b₁·∫ sin x · sin 3x dx     ← = 0，不同頻率，隱形
  + b₂·∫ sin 2x · sin 3x dx    ← = 0，隱形
  + b₃·∫ sin 3x · sin 3x dx    ← = b₃·π，只有它留下痕跡
  + b₄·∫ sin 4x · sin 3x dx    ← = 0，隱形
  + …                          ← 全部歸零

⇒  b₃ = (1/π) ∫[0,2π] f(x)·sin 3x dx
```

無限多個未知數，**一次積分讀出一個，彼此互不干擾**——不解任何聯立方程，每個係數各自結帳。這是本章第一個值得轉述的驚嘆點：ch07 的積分本來是算面積的苦工，在這裡變成了量測儀器；EQ 的推桿之所以能一根一根獨立動，地基就是正交性。

誠實標示：上面那步「逐項積分」，對有限和天經地義，對**無限**和卻是需要定理背書的操作（這正是 ch14 要清算的帳之一）——傅立葉的時代沒有人覺得這要證。先記下這筆，繼續往前。

對方波跑一次探測器，看 4/π 是哪來的：

```text
b₁ = (1/π) ∫[0,2π] 方波(x)·sin x dx
   = (1/π) ( ∫[0,π] (+1)·sin x dx + ∫[π,2π] (−1)·sin x dx )
   = (1/π) ( 2 + 2 )           ← ∫[0,π] sin x dx = 2（FTC 一行，見 ch08）；
   = 4/π ≈ 1.2732                 後半段 sin x 為負，乘 −1 再貢獻 2
```

同樣的機器搖出 b₂ = 0（偶數頻率全滅）、b₃ = 4/(3π)、b₅ = 4/(5π)……整條級數就是這麼來的。

## Gibbs 現象：壓不掉的 8.95%

回去看部分和圖上那兩隻「耳朵」——跳躍點旁邊翹起的小尖角。直覺說：項數加多，耳朵會跟漣漪一起被磨平。畫 50 項、500 項，你會發現耳朵**變窄、貼向跳躍點，但不變矮**。

數字（2026-06 自行複核）：

```text
部分和的最高峰 → (2/π)·Si(π) ≈ 1.17898    ← n→∞ 的極限高度
過衝量         =  1.17898 − 1 = 0.17898
佔整個跳躍（−1 到 +1，高度 2）的比例 = 0.17898 / 2 ≈ 8.95%
```

（Si 是正弦積分函數，Si(π) ≈ 1.85194——本書當常數用，不展開。）注意分母的約定：有些文獻以**半跳**（高度 1）為分母，寫成 17.9%——兩種寫法都在流通，本書一律以**整個跳躍**為分母：過衝 ≈ 9%，精確值 8.95%。這就是 Gibbs 現象（Gibbs phenomenon）：只要訊號有跳躍，截斷的傅立葉級數就在跳躍旁過衝約 9%，加多少項都壓不掉。本章結尾的 Python 小實驗會讓你親手量到它。掛名的歷史順帶一提：最早其實是 Henry Wilbraham 在 1848 年發現的，Gibbs 1899 年在《Nature》上重新描述，Maxime Bôcher 1906 年分析並命名（2026-06 查證）——數學史的命名權經常頒給講得最響亮的人。

工程上你撞過它：示波器上方波邊緣的 ringing、JPEG 圖裡文字邊緣的蚊狀雜訊、濾波器對階躍訊號的振鈴——都是同一隻耳朵。截掉高頻＝用部分和代替全級數，稜角旁就長出 Gibbs。

但真正重要的是它逼你把話說精確。「部分和收斂到方波」到底是什麼意思？固定任何一點 x（跳躍點除外），部分和的數列確實收斂到方波值——**逐點**成立。可是「整條曲線最大誤差」永遠卡在 0.179 左右不動，因為耳朵一直在，只是越站越靠近跳躍點。「每一點各自收斂」與「整條曲線一致地貼上去」是**兩種不同強度的收斂**——這個區分在 1807 年根本不存在於任何人的腦中，而它正是 ch14 嚴格化的核心戰場之一。無限疊加第一次當著你的面表演詭異行為：每一點都乖，整體卻永遠有個 9% 的鬼影。

## 「等於」是什麼意思：嚴格化的引信

現在清算開頭那筆帳。傅立葉寫下「方波 = (4/π)(sin x + sin 3x/3 + …)」，這個等號的右邊是無限和——而 ch03 與 ch09 教過你：無限和不是加法，是**部分和數列的極限**。所以這句話的精確意思是：「對每個 x，部分和數列收斂，而且極限恰好等於方波在 x 的值。」每個字都需要證明，而且不全是真的：

- 在跳躍點 x = π，級數的每一項 sin(nπ) 都是 0，部分和恆等於 0——但方波在那裡的值要嘛 +1 要嘛 −1。級數固執地給出兩側的**中點**，誰也不偏。
- 狄利克雷（Dirichlet）1829 年在 Crelle 期刊（第 4 卷）給出第一個嚴格的收斂定理：函數夠乖（分段平滑一類的條件，細節本書不展開）時，傅立葉級數逐點收斂，跳躍點收斂到兩側中點。「任何函數」第一次有了經得起盤問的版本。
- 那不乖的函數呢？黎曼（Riemann）接棒。要問「哪些函數能被三角級數表示」，得先回答「哪些函數能被積分」——他在 1854 年的資格論文（正是 ch12 重排定理出處的那篇）裡給出了今天叫**黎曼積分**的定義（2026-06 查證）。你在 ch07 學的積分，它的嚴格版是被傅立葉級數逼出來的。
- 再往深挖：如果兩條三角級數處處收斂到同一個函數，它們的係數必然相同嗎？這是康托爾（Cantor）接到的問題。他 1870 年證出唯一性，接著追問「允許在一些例外點上失效呢？」——為了刻畫「例外點集可以多大」，他在 1870–72 年的系列論文裡引入了點集的概念。**集合論是從傅立葉級數的例外點集裡長出來的。**
- 而直覺最後的堡壘也會塌：1873 年 du Bois-Reymond 構造出一個**連續**函數，其傅立葉級數在某些點上發散（2026-06 查證）——連「連續就夠乖」都不成立。

把鏡頭拉遠看這條因果鏈：一根鐵棒怎麼變涼的工程問題，逼出了「函數是什麼」「積分是什麼」「無限集合是什麼」的全面重建。這是本章第二個值得轉述的驚嘆點，也是這本書接下來三章的劇本：無限和會詐騙（ch12）、無限疊加會生出怪獸——劇透一句，ch13 的 Weierstrass 函數本身就是一條三角級數——而整棟大樓最後要重打地基（ch14）。你現在站在 Part V 的門口。

## 它現在就在你的手機裡

故事的現代結局短得不成比例。1965 年 Cooley 與 Tukey 發表快速傅立葉變換（Fast Fourier Transform, FFT），把算 N 點頻譜的代價從 ~N² 壓到 ~N·log N（雛形想法可追溯到 Gauss 的手稿，2026-06 查證）。從此「拆頻率」從數學家的紙上工程變成每秒在你手機裡跑幾百萬次的副程式：你拍的每張 JPEG 把影像切成 8×8 小塊、做離散餘弦變換（DCT，傅立葉的近親）、再粗暴地丟掉高頻係數——「人眼對高頻不敏感」變現成壓縮率；你串流的每首歌、語音通話的每一幀、頻譜圖上的每根柱子、監控系統裡的週期性偵測，全是傅立葉係數。1807 年被拒稿的那個主張，今天是數位文明的底層協定之一。點到為止。

## 紙上推演

**題 1 [15 分鐘] ★★（口頭題）** — 用「探測器」比喻，把「為什麼每個頻率的係數可以各自結帳、不用解聯立方程」講給一個沒讀過本章的工程師。必須講到：內積是什麼、正交是什麼、探測器為什麼只對自己的頻率有反應。講完用解答的要點清單自評。

**題 2 [10 分鐘] ★★** — 看著本章三張部分和圖預測：項數加倍之後，跳躍旁的「耳朵」會變矮嗎？先說出你的理由，再回答第二問：既然耳朵不消失，「加更多項」到底改善了什麼？

**題 3 [15 分鐘] ★★** — 把「這條監控曲線有日週期、有週週期、還在緩慢成長」翻成傅立葉語言：(a) 基本週期該選多長？(b) 日週期會出現在第幾號諧波？只有那一根嗎？(c) 「緩慢成長的趨勢」放得進這個框架嗎？放不進的話，工程上怎麼辦？

### 推演解答

**題 1**　模範答案要點清單（你的版本必須命中全部五點）：

1. 兩個訊號的「內積」＝相乘後對一個週期積分，讀成相關性讀數：同步起伏讀數大，毫無默契正負抵銷歸零。
2. 不同頻率的內積是 0（正交）——sin 2x 在 sin 3x 的探測器上完全隱形；本章用對稱性驗過一例（乘積的正負瓣兩兩鏡像抵銷）。
3. 自己對自己不是 0——sin² 永遠 ≥ 0，沒有負瓣可抵，積出 π：探測器只對自己的頻率有反應。
4. 所以拿 sin nx 掃過訊號：其他成分全數歸零，只剩 bₙ·π，一次積分讀一個係數。像按 key 直接取值，不像解方程——每個頻率一條獨立通道，不串台。
5. 誠實附註：中間「逐項積分」那步對無限和需要定理背書（見 ch14）——傅立葉自己沒證，後人補的票。

**題 2**　不會變矮。耳朵的峰值高度收斂到 (2/π)·Si(π) ≈ 1.17898，過衝固定在跳躍高度的 8.95%，與項數無關；項數加倍改變的是耳朵的**寬度**（大約減半）與**位置**（更貼近跳躍點）。第二問：改善的是「逐點」與「整體平均」的表現——固定任何一個離跳躍有距離的點，誤差會趨於 0（耳朵終究會掃過它、不再回來）；誤差的總面積（能量）也趨於 0。卡死不動的只有「整條曲線的最大誤差」。一句話總結：**逐點收斂成立，一致貼合不成立**——能分清這兩件事，你已經站在 ch14 門口了。

**題 3**　(a) 取基本週期 T = 一週（訊號裡最長的循環）。(b) 週週期＝第 1 號諧波；一週裝 7 個日循環，所以日週期＝第 7 號諧波。但日內形狀不是純正弦（午餐尖峰、晚間高峰、凌晨低谷），所以能量不只在第 7 號——還會散在 14、21 號等倍頻上：諧波是「形狀不圓滑」的指紋，正如方波的能量散在 3、5、7 倍頻。(c) 放不進。sin 與 cos 的任何疊加都是週期函數，「一路往上長」的趨勢不週期，硬塞會把能量抹進所有頻率（你會看到一坨假頻譜）。工程做法：先去趨勢（detrend）——扣掉移動平均或線性擬合——再對殘餘做頻率分解。時間序列工具都長成「trend + seasonal + residual」三件套，原因在此。

### Python 小實驗

親手量 Gibbs 的耳朵。對方波的部分和（前 1、5、50 項）在半個週期上掃峰值（純標準函式庫，跑完約一兩秒）：

```python
# Square-wave Fourier partial sums and the Gibbs overshoot.
# Square wave: +1 on (0, pi), -1 on (pi, 2*pi); jump height = 2.
# Partial sum: S_N(x) = (4/pi) * [sin x + sin 3x/3 + ...] (N terms).
import math

def partial_sum(x, n_terms):
    return (4 / math.pi) * sum(
        math.sin((2 * j + 1) * x) / (2 * j + 1) for j in range(n_terms)
    )

GRID = 20000  # sample points on (0, pi); peak sits near x = pi/(2N)

print(f"{'N':>3} {'peak height':>12} {'overshoot':>10}  (% of jump = 2)")
for n in (1, 5, 50):
    peak = max(partial_sum(math.pi * i / GRID, n) for i in range(1, GRID))
    print(f"{n:>3} {peak:>12.6f} {peak - 1:>10.6f}  {(peak - 1) / 2:.2%}")

print("limit: (2/pi)*Si(pi) = 1.178980 -> overshoot 8.95% of the jump")
```

預期輸出（CPython 3，寫章時實際執行）：

```text
  N  peak height  overshoot  (% of jump = 2)
  1     1.273240   0.273240  13.66%
  5     1.182328   0.182328  9.12%
 50     1.179013   0.179013  8.95%
limit: (2/pi)*Si(pi) = 1.178980 -> overshoot 8.95% of the jump
```

讀法：N = 1 的「峰」就是單一正弦的振幅 4/π ≈ 1.2732；N = 5 時峰值已經降到 1.1823；N = 50 時 1.1790——然後就停了。從 5 項到 50 項，峰值只動了小數點後第三位，它收斂的目的地是 1.17898，**不是 1**。想看得更兇，把迴圈改成 `for n in (50, 500, 5000)`（記得把 GRID 等比例加大，否則取樣網格會漏掉越來越窄的峰——數值實驗自己也有解析度陷阱）：三個峰值會擠在 1.1790 附近不動如山。過衝不死，它只是搬家到離跳躍點更近的地方。
