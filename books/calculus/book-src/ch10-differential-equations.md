# ch10 — 微分方程：會變的世界的文法

> **本章解決什麼問題**：ch08 把微分與積分接成一台引擎，ch09 給了函數另一種長相，但到目前為止，這套語言都在描述「已經畫好的曲線」。本章是世界觀的轉向：自然定律幾乎從不直接給你軌跡，它給的是變化率——dy/dt = f(y, t)——軌跡得靠數學自己長出來。你會看到同一個方程的四張面孔、F = ma 為什麼是微分方程、sin 與 cos 怎麼從一條沒有任何圓的彈簧裡跳出來（ch11 傅立葉的地基），以及公式解不存在時的最後手段：Euler 法——你寫過的 tick 迴圈。

## 從你已知的出發

如果你寫過任何一種模擬的更新迴圈——遊戲伺服器的 world update、物理引擎、甚至只是一個會自己演化狀態的排程器——你寫過的大概是這個形狀：

```text
每個 tick（間隔 dt 秒）：
    position += velocity * dt      ← 用「此刻的變化率」直線外推一小步
    velocity += accel * dt
```

恭喜：你解微分方程已經好幾年了，只是沒人告訴你這件事的名字叫 Euler 法（本章結尾正式相認）。注意這段程式碼的奇怪之處——它從頭到尾**沒有寫出軌跡**。沒有任何一行告訴你物體十秒後在哪裡；程式裡只有「下一瞬間怎麼變」的規則，軌跡是跑出來的。

第二個你做過的事：容量預測。「假設流量每月成長 15%，照這個趨勢外插六個月」——「每月成長 15%」是一句關於**變化率**的假設，外插的動作就是在解一條微分方程。第三個：你調過 EMA 平滑的 alpha、看過「每秒固定比例的 session 流失」「一批快取條目隨時間批量過期」這類指數衰減曲線——本章會證明，EMA 字面上就是一條微分方程的離散版，而你調的 alpha 就是步長（紙上推演第 3 題）。

這章要做的，是把你天天在寫的這些東西接上它們的母語：定律寫的是變化率，不是軌跡。

## 定律寫的是變化率，不是軌跡

克卜勒（Kepler）從第谷的觀測數據裡擬合出「行星軌道是橢圓」——那是在描述**軌跡**，像從監控撈出歷史曲線然後手畫趨勢線。牛頓（Newton）寫下的是完全不同種類的句子：每一瞬間，行星的加速度由它當下受的力決定。這句話不告訴你行星明年在哪。它只告訴你：**給定現在的狀態，下一瞬間怎麼變。**

用你的母語講：自然界沒有公開「未來的 event log」，它公開的是 transition function——給定 state，告訴你 state 怎麼更新。微分方程（differential equation）就是連續時間的 transition function：一條方程，未知數不是一個數、而是一整個函數 y(t)，方程說的是這個函數的變化率與它自身（以及時間）的關係：

```text
dy/dt = f(y, t)        ← 「變化率是狀態的函數」：會變世界的通用文法
```

「解」一條微分方程，意思是把這個 transition function 展開成完整的軌跡——從局部規則還原全局歷史。最簡單的情況你其實已經解過：如果變化率只跟時間有關、跟狀態無關（dy/dt = g(t)），答案就是 y = ∫g，這正是 ch08 反導數的工作。真正的新遊戲從這裡開始：**變化率依賴 y 自己**——輸出回頭餵進規則，回授（feedback）出場，事情立刻變得又豐富又危險。

這個世界觀可以畫出來。以 dy/dt = y 為例：在平面上每一點 (t, y) 插一支小箭，斜率等於該處的 y——這張圖叫方向場（slope field）。方程本身就是這整片箭頭；「解」是一條處處順著箭頭流的曲線：

```
 y
 2 ┤  | | | | | | | |     ← y = 2 這排：斜率 2，箭陡
 1 ┤  / / / / / / / /     ← y = 1 這排：斜率 1
 0 ┼  ─ ─ ─ ─ ─ ─ ─ ─     ← y = 0：斜率 0，躺平不動
−1 ┤  \ \ \ \ \ \ \ \     ← y = −1：斜率 −1，往下逃
   └──────────────────► t
   （ASCII 只能示意：真實的箭斜率連續變化，越遠離 0 越陡）
```

同一片箭頭場，從不同起點放開手，會滑出不同的曲線。這就是本章反覆出現的結構：**定律給的是一族軌跡，初始條件挑出其中一條。**為什麼自然定律長這樣？因為局部規則簡單、全局軌跡複雜。萬有引力一行字寫得完；行星軌道是橢圓加攝動加（後面會看到的）混沌。自然的「程式碼」很短，「執行結果」很長——而微分方程就是那段原始碼。

## 一個方程的四張面孔：dy/dt = ky

整個家族裡最簡單的一支，是變化率正比於現量：

| 面孔 | 方程 | k 的身分 |
|---|---|---|
| 培養皿裡的菌落 | dy/dt = ky（k > 0） | 每個個體單位時間的繁殖率 |
| 連續複利 | dy/dt = ry | 年利率——ch05 的故事重演 |
| 放射性衰變 | dy/dt = −λy | 每顆原子單位時間的衰變機率 |
| 咖啡冷卻 | dT/dt = −k(T − T_room) | 散熱效率——紙上推演第 1 題拆它 |

前三張臉一眼就是同一個方程，只差 k 的正負。第四張需要卸個妝：冷卻定律說「散熱速率正比於**溫差**」——這是牛頓 1701 年匿名發表於《Philosophical Transactions》的觀察（論文〈Scala graduum caloris〉，2026-06 查證）。令 y = T − T_room（把座標原點搬到室溫），因為室溫不動，y′ = T′，方程變成 dy/dt = −ky。卸了妝，還是同一張臉。四個八竿子打不著的系統共用一條方程，因為它們共用同一個機制：**每一份存量，都以固定速率產生（或消滅）變化**——菌落裡每隻菌都在分裂、帳戶裡每塊錢都在生息、樣本裡每顆原子都可能衰變、溫差的每一度都在推動散熱。

怎麼解它？這本書不教解法目錄，我們用 ch05 的庫存反推。庫存裡只有一個「導數等於自己」的原料：(eᵗ)′ = eᵗ。現在需要「導數等於自己的 k 倍」——拿連鎖規則改裝：

```text
要找：y′ = k·y 的 y
試 y = e^(kt)：
    (e^(kt))′ = e^(kt)·k = k·y     ← 連鎖規則：外層 eᵗ、內層 kt（見 ch05）✓
放大 C 倍也行：y = C·e^(kt)：
    (C·e^(kt))′ = C·k·e^(kt) = k·y                                      ✓
```

y = C·e^(kt) 全家都是解。C 是誰？代 t = 0：y(0) = C·e⁰ = C——**C 就是初始狀態**。方向場的話又應驗了一次：方程給一族曲線，y(0) 挑出那一條。

還剩一個誠實的問題：有沒有別的解躲在這一族之外？有個漂亮的論證可以封死後門。設 y 是**任何**滿足 y′ = ky 的函數，看 u = y·e^(−kt)：

```text
u′ = y′·e^(−kt) + y·(−k)·e^(−kt)    ← 乘法規則＋連鎖規則（見 ch05）
   = (y′ − ky)·e^(−kt)
   = 0                              ← 因為 y′ = ky
```

u 的變化率處處是 0，所以 u 是常數（直覺版：變化率永遠為零的量不會動；嚴格證明要靠均值定理，本書不展開）。u = C 即 y = C·e^(kt)——一條漏網之魚都沒有。指數函數不是這個方程「碰巧的解」，它是被這個方程**唯一指定**的解，這正是 ch05 說 e 是「增長率正比現量」所有系統的原生語言的精確意思。

順帶表演全書唯一一次「解法技巧」——分離變數（separation of variables），看一眼就好：

```text
dy/dt = ky
dy/y  = k·dt           ← 把 dy、dt 當可搬動的小量（萊布尼茲記號的便道）
∫ dy/y = ∫ k·dt        ← 兩邊各自積分
ln y  = kt + c         ← 左邊用 (ln y)′ = 1/y（見 ch05）；右邊用 FTC（見 ch08）
y     = eᶜ·e^(kt) = C·e^(kt)
```

ch04 警告過 dy/dx 不是分數，這裡每一步「搬運」背後其實都有定理背書（換元積分，本書不展開）——把它當高效的助記法用，算完代回原方程驗證即可。技巧到此為止：這本書關心的是語言，不是題庫。

衰變那張臉值得多看一眼，因為它給了你一個天天在用的概念。y = y₀·e^(−λt) 掉到一半需要多久？解 e^(−λ·t½) = 1/2，得 t½ = ln 2/λ ≈ 0.6931/λ——**半衰期**，而且它與現量無關：從 100 克掉到 50 克，跟從 2 克掉到 1 克，花的時間一樣。碳-14 的半衰期約 5730 年（2026-06 查證），考古定年就是拿這條方程當時鐘倒著讀。工程版你也認得：如果每秒有固定比例的 session 離線、固定比例的快取條目失效，線上人數和命中率走的就是同一條 e^(−λt)，你的「流量掉一半要多久」直覺，就是半衰期。

## F = ma：寫下定律，還得發明解它的數學

把全人類最有名的物理公式重新讀一次。F = ma 的 a 是加速度——位置對時間的二階導數。所以這條「定律」其實是：

```text
m·x″ = F(位置, 速度, 時間)      ← 力 = 質量 × 加速度，攤開是一條二階微分方程
```

（牛頓自己的寫法是 ẍ——ch04 那套點記號，專職對時間微分。）F = ma 不是一條代公式，它是一條微分方程：給定每一瞬間的力，它規定位置函數 x(t) 必須滿足的局部關係。想知道砲彈落點、行星位置？你得從這條方程把 x(t) **解**出來。

這就是牛頓真正的成就為什麼是兩件事，缺一不可：他寫下了定律（運動定律＋萬有引力），**同時發明了把定律變成軌跡的數學**（微積分）。1687 年的《Principia》把這兩件事接通：從「引力與距離平方成反比」這條局部規則出發，克卜勒花多年從數據擬合出來的橢圓軌道被**推導**了出來。軌跡第一次不是觀測的總結，而是定律的定理。

這套文法最戲劇性的一次驗收在 1846 年。天王星的軌道老是對不上計算，Le Verrier 反過來問：如果偏差來自一顆沒人見過的行星，它得在哪裡？他用攝動計算反解出位置，8 月 31 日向法蘭西科學院提交預測，並去信柏林天文台的 Galle——9 月 23 日**當晚**，Galle 與 d'Arrest 就在距預測位置約 1° 之內找到海王星，前後不到一小時。一顆行星在被望遠鏡看到之前，先被紙筆從 F = ma 裡算了出來——「筆尖上發現的行星」這個說法流傳至今。（英國的 Adams 獨立做過類似計算，但他的解多次變動、實際位置距其預測約 12°，現代科學史已下修他「共同預測者」的地位。）

同一支筆尖，幾十年後也撞上了牆：Poincaré 為瑞典國王 Oscar II 懸賞而寫、1890 年刊出的得獎論文證明，三體問題的方程寫得下、卻沒有一般的公式解，而且軌跡對初始條件極度敏感——這是混沌理論的起點（一句話帶過，本書不展開；見延伸閱讀）。記住這個落差：**能寫下定律，不保證能解出軌跡。**這個落差正是本章最後一節、也是你的 tick 迴圈存在的理由。

## 彈簧振子：三角函數從圓裡跳出來

換一條方程，獎品完全不同。一顆小球掛在彈簧上，拉離平衡點的位移叫 x，彈簧的回復力與位移成正比、方向相反（單位挑到所有常數都是 1）。套 F = ma：

```text
x″ = −x        ← 哪個函數，二階導數是自己的負號？
```

查 ch05 的庫存：sin′ = cos、cos′ = −sin。連微分兩次：

```text
(sin t)″ = (cos t)′ = −sin t   ✓
(cos t)″ = (−sin t)′ = −cos t  ✓
```

兩個都中。事實上 x = A·cos t + B·sin t 全家都是解——二階方程有**兩個**自由常數，因為你得同時指定初始位置與初始速度才能釘死一條軌跡：代入可得 x(0) = A、x′(0) = B。拉開 1 單位、靜止放手（A = 1、B = 0），小球的位置就是 x = cos t，永遠振動下去。

現在停下來感受這件事有多奇怪。ch05 裡，sin 與 cos 是**圓**的座標——一個點繞單位圓，水平影子是 cos、垂直影子是 sin。但彈簧上沒有圓。沒有任何東西在旋轉，只有一顆小球在一條直線上來回。方程卻把圓召喚了出來：彈簧的運動恰好是等速圓周運動的投影。三角函數的真實身分不是「關於三角形的函數」，而是「**滿足 x″ = −x 的函數**」——圓只是它們最早被人類撞見的地方。

而 x″ = −x 無所不在。任何系統在穩定平衡點附近，回復力都可以局部線性化（ch05 的世界觀：放大夠近，什麼都是直線）成「正比於偏移、方向相反」——於是鞦韆的小角度擺動、吉他弦、LC 電路裡的電荷、地震中大樓的搖晃、空氣中的聲波，全是這條方程的變奏（一般形式 x″ = −ω²x，解是 sin(ωt) 與 cos(ωt)，ω 是頻率）。**三角函數從圓裡跳出來之後，住進了所有會振動的東西裡。**這就是 ch11 的地基：既然振動世界的原子是 sin 與 cos，傅立葉那句「任何訊號都能拆成 sin 與 cos 的疊加」就從瘋話變成了值得一試的猜想。

這條方程還自帶一個帳本，等一下的 Python 實驗會用到。令 v = x′，定義 E = x² + v²（能量，差一個係數不影響結論）：

```text
E′ = 2x·x′ + 2v·v′      ← 連鎖規則＋和規則（見 ch05）
   = 2x·v + 2v·(−x)     ← x′ = v、v′ = x″ = −x
   = 0                  ← 能量守恆：真實的彈簧不多不少
```

真解的能量一毫不差地守恆。記住這個 0——數值方法守不守得住它，是一個會痛的故事。

## 不會解就一步步走：Euler 法

dy/dt = ky 有公式解，彈簧有公式解——它們是特權階級。Poincaré 已經告訴你：一般的方程**沒有**公式解，而且這是常態不是例外。那定律還有什麼用？

歐拉（Euler）的答案（1768 年《Institutionum calculi integralis》，2026-06 查證）：不會解，就走。方程在每一點都告訴你變化率——那就相信它一小步。在一小段 h 裡把變化率當常數（ch05 的局部線性，又是它），直線外推：

```text
y(t + h) ≈ y(t) + h·f(y, t)      ← 順著方向場的箭頭，直走 h 秒，再換箭
```

拿基準題完整跑一遍：dy/dt = y、y(0) = 1，步長 h = 0.1，走到 t = 1。每一步把「此刻的 y」當變化率，前四步逐步展開（自算複核）：

```text
y₁ = y₀ + 0.1·y₀ = 1      + 0.1×1      = 1.1
y₂ = y₁ + 0.1·y₁ = 1.1    + 0.1×1.1    = 1.21
y₃ =               1.21   + 0.1×1.21   = 1.331
y₄ =               1.331  + 0.1×1.331  = 1.4641
```

每一步都是「乘上 (1 + 0.1)」：yₙ₊₁ = (1.1)·yₙ。十步走完：

| 步 n | tₙ | yₙ |
|---|---|---|
| 4 | 0.4 | 1.4641 |
| 5 | 0.5 | 1.61051 |
| 6 | 0.6 | 1.77156 |
| 7 | 0.7 | 1.94872 |
| 8 | 0.8 | 2.14359 |
| 9 | 0.9 | 2.35795 |
| 10 | 1.0 | **2.59374** |

第 10 步的精確值是 (1.1)¹⁰ = 2.5937424601。真實答案是多少？這條方程我們有特權知道：y = eᵗ，y(1) = e ≈ 2.71828。誤差約 0.1245——不光彩，但方向對了。把步長縮成 h = 0.01（走一百步）：(1.01)¹⁰⁰ ≈ 2.7048，誤差縮到約 0.0135。步長再縮十倍，誤差再縮約十倍（Python 實驗會給整張表）：Euler 法**收斂**，只是收得樸素——誤差大約與 h 同階。

你應該認出這串數字了。(1 + 1/10)¹⁰、(1 + 1/100)¹⁰⁰——這就是 ch05 的複利表。ch05 說過複利表是「切細、執行、取極限」家族的操作、ch10 會再見到一模一樣的形狀——就是這裡：**在 dy/dt = y 上跑 Euler 法，每一步就是結算一次利息；步長趨近 0，就是連續複利；e 不是被算出來的，是被一步一步走出來的。**兩章在同一張表上會師。

然後是本章的相認時刻。把 Euler 法寫成程式：

```text
每個 tick（dt 秒）：
    position += velocity * dt
    velocity += accel(state) * dt
```

這就是本章開頭那段你寫過的 tick 迴圈，逐字不差。遊戲伺服器每秒 60 次地在數值積分牛頓的運動方程；物理引擎是一台微分方程求解器；你的容量外插是在徒手跑一條 dy/dt = ky。Euler 法樸素、一階、誤差不小——但它是所有數值方法的祖先，而且它早就在你的程式碼裡上班了。它的兩個真實陷阱——步長太大會震盪發散、長時間模擬會悄悄漏帳——分別留給紙上推演第 2 題和 Python 實驗，讓你親手撞一次。

|---|---|---|
| 「學微分方程＝背解法目錄」 | 課堂愛教有公式解的題，但那是精選的特權階級；絕大多數方程（三體只是最有名的）沒有封閉解，職業火力都在定性分析與數值方法上 | 看到解不出的方程，你以為是自己（或數學）失敗了，而不是「這很正常，換工具」 |
| 「能寫下定律＝能預測未來」 | 寫得下 ≠ 解得出 ≠ 算得準：混沌系統把初始量測的微小誤差指數放大，再小的步長也救不回來 | 你以為只要量測更準、機器更快，預報就能無限延伸 |
| 「dy/dt = ky 的解就是 e^(kt)」 | 漏掉 C：方程給的是一族曲線，初始條件才挑出一條。沒有 y(0)，你的「解」回答不了任何具體問題 | 你寫下的解代 t = 0 之後對不上現量，或你說不出 C 的物理身分 |
| 「步長越小，模擬一定越準」 | 紙上是對的（誤差 ~ h），機器上不是：步數變多、成本上升、浮點捨入誤差開始累積（ch04 的割線表翻過同一輛車，ch15 算總帳）；而實務上先殺死你的多半是反方向——步長太大直接震盪發散 | 你從沒用兩個步長跑同一個模擬比對過結果 |
| 「模擬跑起來像那麼回事＝模擬是對的」 | naive Euler 的彈簧每個 tick 悄悄注入能量，短期看不出來，長期軌道整個飛掉——錯得非常安靜 | 你的模擬沒有任何守恆量檢查（能量、動量、總帳對齊） |

## 紙上推演

**題 1 [15 分鐘] ★★** — 一杯 90°C 的咖啡放在 25°C 的房間裡。(a) 把「咖啡冷卻」翻譯成微分方程：自己立出形式、解釋每個符號與正負號的物理意義。(b) 不解方程，先從方程本身讀出：溫度曲線一開始陡還是後來陡？最後趨向哪裡？(c) 用變數代換把它化成 dy/dt = −ky 的標準臉，寫出通解，並用初始條件釘死常數。

**題 2 [20 分鐘] ★★** — 對 dy/dt = −y、y(0) = 1：(a) 用 h = 0.1 手算 Euler 法三步，對照精確值 e^(−0.3)。(b) 把步長換成 h = 1.5 與 h = 2.5，各走四步，描述發生了什麼。(c) 解釋「步長太大會跳過頭甚至震盪」的機制，並找出穩定的步長條件。提示：把更新式整理成「每步乘一個固定倍數」。

**題 3 [15 分鐘] ★★★** — 監控系統的 EMA 平滑通常寫成 yₙ₊₁ = yₙ + α·(xₙ − yₙ)（x 是原始訊號，y 是平滑值）。(a) 它是哪條微分方程的 Euler 離散化？α 對應什麼？（提示：dy/dt = k(x − y)。）(b) 用這條方程解釋：α 太小、太大、大於 1 時平滑曲線各是什麼行為？(c) 本章哪一張「面孔」跟它是同一條方程？

### 推演解答

**題 1**　(a) 牛頓冷卻定律：散熱速率正比於咖啡與環境的溫差——

```text
dT/dt = −k·(T − 25)      ← T：咖啡溫度（°C）；k > 0：散熱效率
```

負號是方向盤：T > 25 時右邊為負、溫度下降；如果有一杯 10°C 的冰咖啡，T − 25 < 0、右邊為正、溫度上升——同一條方程自動管加熱與冷卻，這是「寫對正負號」的獎賞。(b) 不解方程就能讀出形狀：溫差越大、變化率越大，所以**一開始最陡**；隨著 T 靠近 25，右邊趨近 0，曲線越來越平、漸近貼向 25 但永遠不穿過（變化率在 25 處歸零，到站就熄火）。你看過的服務重啟後錯誤率「收斂回基線」的曲線就是這個形狀。(c) 令 y = T − 25（溫差），室溫是常數所以 y′ = T′，方程變成 y′ = −ky，通解 y = C·e^(−kt)。初始條件：t = 0 時 T = 90，即 y(0) = 65 = C。所以

```text
T(t) = 25 + 65·e^(−kt)      ← 驗算：T(0) = 25 + 65 = 90 ✓；t→∞ 時 T→25 ✓
```

常見錯路：把方程立成 dT/dt = −kT（忘了減環境溫度）——那杯咖啡會冷到絕對零度以下，你的模型在物理審查就被退件了。

**題 2**　(a) h = 0.1：yₙ₊₁ = yₙ − 0.1·yₙ = 0.9·yₙ，三步：1 → 0.9 → 0.81 → 0.729。精確值 e^(−0.3) ≈ 0.7408，誤差約 0.012，方向對、略偏低。(b) 更新式是 yₙ₊₁ = (1 − h)·yₙ。h = 1.5：每步乘 −0.5，得 1 → −0.5 → 0.25 → −0.125 → 0.0625——正負交替地「震盪收斂」，但真解 e^(−t) 從頭到尾是正的，這個負值在物理上常是胡說（負的溫差？負的存量？）。h = 2.5：每步乘 −1.5，得 1 → −1.5 → 2.25 → −3.375 → 5.0625——震盪**發散**，模擬爆炸。(c) 機制：Euler 法拿「步首的變化率」用滿一整步，但對衰變方程，y 在步內一直在縮、變化率也該跟著縮——步長太大，等於用過期的陡坡衝過頭，直接越過平衡點摔到對面。穩定條件：|1 − h| < 1，即 0 < h < 2；h 超過 1 就開始跳過頭變號，超過 2 就萬劫不復。對照 ch06：梯度下降在 f(x) = x² 上的更新式是乘 (1 − 2η)、條件 0 < η < 1——**同一隻動物**。學習率與步長，是同一個權衡在兩個領域的化名。

**題 3**　(a) 對 dy/dt = k·(x(t) − y) 做步長 Δt 的 Euler 離散：yₙ₊₁ = yₙ + Δt·k·(xₙ − yₙ)。對照 EMA 定義式，α = k·Δt——**alpha 就是「追趕速率 × 步長」**。方程的白話：平滑值以正比於落差的速度追原始訊號，差多少、追多快。(b) α 太小：k 小、時間常數 1/k 大，y 追得極慢，曲線平滑但嚴重落後（你看到的尖峰是十分鐘前的）。α 適中：及時又平滑。α 接近 1：幾乎不平滑，y 貼著 x 抖。α > 1：更新倍數 (1 − α) 變負——平滑值繞著訊號正負過衝，這正是題 2 的 h > 1 同款翻車，只是這次翻在你的 dashboard 上。(c) 拿 x 當常數（訊號不動）看：y′ = k(x − y) 就是牛頓冷卻定律，y 是咖啡、x 是室溫。**EMA 是一杯不斷追著室溫跑的咖啡**；你每次調 alpha，都是在決定那杯咖啡的散熱效率。

### Python 小實驗

兩個實驗裝在一個檔案裡：上半段量 Euler 法的收斂速率，下半段讓 naive Euler 在彈簧上跑長時間、看 ch10 正文承諾過的那個守恆量被它弄丟（純標準函式庫）：

```python
# Euler's method on dy/dt = y, y(0) = 1, integrated to t = 1.
# Exact answer is e; watch the error shrink roughly like h (first order).
import math

def euler_to_e(h):
    y = 1.0
    for _ in range(round(1 / h)):
        y += h * y                  # the tick loop: y += rate * dt
    return y

print(f"{'h':>6} {'Euler y(1)':>12} {'error':>10}")
for h in (0.1, 0.01, 0.001):
    y = euler_to_e(h)
    print(f"{h:>6} {y:12.6f} {math.e - y:10.6f}")

# Bonus: naive Euler on the spring x'' = -x  (x=1, v=0).
# Energy E = x^2 + v^2 should stay exactly 1.0. Watch it drift.
x, v, h = 1.0, 0.0, 0.1
print(f"\n{'t':>4} {'energy x^2+v^2':>15}")
for n in range(1, 601):
    x, v = x + h * v, v - h * x     # RHS uses old x, v: naive Euler
    if n % 150 == 0:
        print(f"{n*h:4.0f} {x*x + v*v:15.4f}")
```

預期輸出（CPython 3 實跑複核；浮點運算確定性，常見平台應可重現）：

```text
     h   Euler y(1)      error
   0.1     2.593742   0.124539
  0.01     2.704814   0.013468
 0.001     2.716924   0.001358
```

讀法：h 縮十倍，誤差就縮約十倍——一階收斂的體感。基準數字全數對帳：(1.1)¹⁰ ≈ 2.5937、(1.01)¹⁰⁰ ≈ 2.7048，h = 0.001 給 2.716924，一路爬向 e ≈ 2.71828。代價同樣是十倍：誤差每多一位精度，迴圈就多跑十倍步數——「好方法用同樣預算多換好幾位精度」是數值方法這門工藝的主題，本書點到為止。

```text
   t  energy x^2+v^2
  15          4.4484
  30         19.7885
  45         88.0275
  60        391.5834
```

這是會痛的那張表。真實彈簧的能量恆等於 1（正文推過 E′ = 0）；naive Euler 跑到 t = 60，能量變成 **391 倍**——振幅放大近 20 倍，小球越盪越瘋。而且這不是浮點問題，是方法本身在漏帳，漏得還很有規律：每一步把能量精確放大 (1 + h²) 倍——

```text
x′ = x + hv、v′ = v − hx
x′² + v′² = x² + 2hxv + h²v² + v² − 2hxv + h²x² = (1 + h²)·(x² + v²)
```

交叉項相消，留下 (1 + h²)。h = 0.1 時每個 tick 注入 1% 能量，600 步就是 (1.01)⁶⁰⁰ ≈ 391.6——跟表上的數字一位不差。短期模擬看不出來（前幾秒一切如常），長期軌道整個飛掉，錯得非常安靜——這就是「模擬看起來對」騙人的方式，也是為什麼老練的模擬工程師一定盯著守恆量看。一句話帶過出路：存在專門保住能量結構的步法（辛積分器，symplectic integrator）——最便宜的一款只是把上面兩行更新調換順序（先更新 v、再用新 v 更新 x），能量就近乎守恆，遊戲物理引擎用的正是這類方法（2026-06 查證；機制本書不展開，見延伸閱讀）。
