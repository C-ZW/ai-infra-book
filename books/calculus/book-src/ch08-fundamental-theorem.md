# ch08 — 微積分基本定理：面積＝斜率反過來

> **本章解決什麼問題**：到上一章為止，這本書其實是兩本書——Part II 講斜率（微分：局部、切線、相減再相除），ch07 講面積（積分：全域、累積、相乘再加總），兩條故事線沒有共用任何一個零件。本章證明它們是同一件事的正反面：累積量的變化率＝當下的高度。這條定理把 ch07 的極限苦工換成一行端點相減，補上 ch02 結尾那個卡了 1800 年的缺口——「通法」。它是全書的最高點，也是 ch09 之後一切引擎的點火器。慢慢讀。

## 從你已知的出發

你要回答一個再日常不過的問題：過去一小時，這個服務總共吃了多少個請求？

兩條路。第一條：把這一小時每一秒的 QPS 全部加總——3,600 個取樣點，各自乘上窗寬、疊起來。你認得這個動作：這是 ch07 的黎曼和，矩形一條一條疊。第二條：讀兩次 counter，`C(now) − C(now − 1h)`，相減，收工。

你每次都走第二條，而且從來不覺得自己做了什麼了不起的事。但停一秒：第二條路憑什麼合法？「整段時間的總量」是無限多個瞬間流量的累積——不折不扣的積分；而你只摸了端點兩下，中間那三千五百九十八個取樣點看都沒看。

合法的原因藏在 counter 的出身裡：每個請求進來，counter 就加一。這個系統裡有人在**即時維護一個「變化率恰好等於 QPS」的量**，所以總量不必加總，端點相減就是答案。本章要證明這件事的數學版，而且兇得多：對任何連續的 f，這樣的「counter」**永遠存在**；而且常常不需要任何系統幫你維護——拿 ch05 的微分規則表反著查，就能憑空把它寫出來。曲線下面積，ch07 要用 Σk² 公式硬扛的那種苦工，從此變成端點相減。

這條定理叫**微積分基本定理**（Fundamental Theorem of Calculus，下稱 FTC）。「基本」不是「初級」，是「地基」——整棟微積分大樓的承重牆。

## 這兩件事憑什麼有關係？

先把張力擺到桌面上。讀到這裡，你手上有兩台機器，請誠實盤點它們的零件：

| | 微分（ch04–06） | 積分（ch07） |
|---|---|---|
| 問的問題 | 這一點有多陡？ | 這一段累積了多少？ |
| 視野 | 局部——一個點加上它的鄰居 | 全域——整段區間一起算 |
| 核心動作 | 相減、相除、取極限 | 相乘、加總、取極限 |
| 幾何形象 | 切線 | 面積 |
| 計算體驗 | 流水線——ch05 的規則表機械化量產 | 手工業——每題都是一場極限硬仗 |

連出身都無關：切線問題與面積問題是希臘人留下的兩個**獨立**難題，兩千年來各走各的路。阿基米德算弓形面積（見 ch02）時，沒有任何理由想到切線；費馬找極值（見 ch06）時，也沒有任何理由想到面積。

所以請你把這個問題真的問出口：**這兩件事憑什麼有關係？**一個是除法的極限，一個是乘法加總的極限。除法和乘法互逆，這是定義；但「極限化的除法」和「極限化的乘法加總」憑什麼互逆？這聽起來像範疇錯誤，像在問 hash 函數和排序演算法誰是誰的反函數。

本章的主張正是這個聽起來不可能的句子：**求面積是取導數的反操作**。算一條曲線下的面積——理論上要把無限多塊矩形加總的那件事——等於把 ch05 的微分規則表**反著查一次**。如果這個主張沒有讓你覺得哪裡怪怪的，請回去再讀一遍上面那張表；覺得怪是讀懂的前提，因為接下來的證明就是把「怪」一步一步拆掉的過程。

## 把面積變成函數：A(x)

第一步是一個視角轉換，整個證明成敗在此。

ch07 問的是「從 0 到 1 的面積」——一個**數**。現在改問：「從 0 到 x 的面積」，讓右端點變成一根可以滑動的滑桿。面積隨著 x 改變，於是它不再是一個數，是一個**函數**：

**A(x) = ∫₀ˣ f(t) dt　——　累積函數（accumulation function）**

先處理一個會癢的地方：積分號裡面為什麼是 t 不是 x？因為 x 已經被滑桿用掉了。t 是從 0 掃到 x 的「跑腿變數」，x 是決定掃到哪裡停的參數——寫成程式你天天在區分這兩者：

```text
def A(x):                  ← x 是參數：滑桿的位置
    total = 0
    for t in 0 .. x:       ← t 是迴圈變數：從 0 跑到 x 的那條腿
        total += f(t) · dt
    return total
```

迴圈變數和上限參數共用名字是新手 bug；∫₀ˣ f(x) dx 就是那個 bug 的數學版。

A 是個什麼樣的函數？拿三個算得出來的例子摸一下手感（每個都重算過）：

| f(t) | A(x) 哪來的 | A(x) | 取 A 的導數（ch04／ch05） |
|---|---|---|---|
| 2（常數） | 長方形：高 2、寬 x | 2x | 2 |
| t | 三角形：底 x、高 x | x²/2 | x |
| t² | ch07 的黎曼和代數，把上限 1 換成 x 重跑一遍 | x³/3 | x² |

第三列多說一句：ch07 用 Σk² 公式算出 ∫₀¹ t² dt = 1/3，同一套代數把上限換成 x，每一步照抄，極限變成 x³/3——不是用本章還沒證明的定理偷算的，是上一章的苦工誠實換來的。

現在看最右欄。三次，**每一次**，A 微分回去都恰好是 f 本人：2x 的導數是 2，x²/2 的導數是 x，x³/3 的導數是 x²。一次是巧合，兩次是可疑，三次是模式：**面積函數的導數，看起來就是被積的那條曲線**。憑什麼？

## 一條細條：七步證明

這一節是全書的心臟。本書的驗收標準（ch01 開宗明義的那六條）裡，FTC 這條靠的就是你能不能**不看書重講這個證明**——所以慢慢走，每一步都停下來確認你能說出理由。

問題：A′(x) 是什麼？按 ch04 的定義，先寫差商的分子 A(x+h) − A(x)——「滑桿從 x 推到 x+h，面積多了多少」。畫出來：

```
f(t)
  │                            *
  │                          *
  │                       *  ←─ 曲線 y = f(t)，越過細條頂端繼續走
  │                    *  ┌─┐
  │                 * ::: │ │
  │               * ::::: │ │←─ 細條面積 = A(x+h) − A(x)
  │             * ::::::: │ │
  │           * ::::::::: │ │
  │         * ::::::::::: │ │
  │       * ::::::::::::: │ │
  │     * :::: A(x) ::::: │ │
  │   * ::::::::::::::::: │ │
  └───────────────────────┴─┴───── t
  0                       x x+h
```

新增的面積是一條寬度 h 的**細條**。它有多大？粗看：高約 f(x)、寬 h，所以細條 ≈ f(x)·h。這個「約等於」就是整條定理的種子，但「約等於」不能拿來證明——把細條放大，誠實面對它：

```
 把細條放大（曲線在條內仍會起伏）：

      ┌───────────────┐ ←─ 上蓋矩形：高 M，面積 M·h
      │            *::│
      │         *:::::│
      │      *::::::::│
      ├───*───────────┤ ←─ 下蓋矩形：高 m，面積 m·h
      │:::::::::::::::│
      │:::: 細條 :::::│
      │:::::::::::::::│
      └───────────────┘
      x               x+h
```

曲線在條內從某個最低高度 m 爬到某個最高高度 M。細條的真實面積（曲線以下的部分）裝得下整個高 m 的矩形、又被高 M 的矩形整個裝下——三明治成形：m·h ≤ 細條 ≤ M·h。

先用數字感受一下這個三明治怎麼收緊。取 f(t) = t²、x = 0.5（這裡 f(0.5) = 0.25），A(x) = x³/3 是上一節從 ch07 搬來的。逐步縮小 h（每格手算複核過）：

| h | m（條內最低） | (A(x+h) − A(x))/h | M（條內最高） |
|---|---|---|---|
| 0.1 | 0.25 | 0.30333 | 0.36 |
| 0.01 | 0.25 | 0.25503 | 0.26010 |
| 0.001 | 0.25 | 0.25050 | 0.25100 |

每一列，中間那欄都乖乖被左右夾住；h 縮小，上下兩片麵包同時壓向 0.25 = f(0.5)，中間的差商無路可逃。把這個過程寫成一般的證明，七步，一步一行：

```text
目標：f 連續時，A′(x) = f(x)

第 1 步　A(x+h) − A(x) = 細條面積
        ← 0 到 x+h 的面積減 0 到 x 的面積，剩下 x 到 x+h 那一條
第 2 步　令 m = f 在 [x, x+h] 上的最低高度，M = 最高高度
        ← 條很窄，曲線在條內仍會起伏，先把起伏框住
第 3 步　m·h ≤ 細條面積 ≤ M·h
        ← 細條裝得下高 m 的矩形、又被高 M 的矩形裝下（上圖三明治）
第 4 步　m ≤ (A(x+h) − A(x))/h ≤ M
        ← 三邊同除以 h（h > 0，不等號方向不變）
第 5 步　h → 0：細條縮向一條線，m 與 M 都被擠向 f(x)
        ← 全場唯一用到「f 連續」的一步——曲線在 x 附近不跳，
          條內的最低點與最高點都逃不出 f(x) 的任意小鄰域
第 6 步　夾擠（ch03 的老朋友）：(A(x+h) − A(x))/h → f(x)
第 7 步　左邊正是導數的定義（ch04）：A′(x) = f(x)　∎
```

工程師的嚴謹，補兩句。其一：上面假設 h > 0；h < 0 時細條長在 x 左側，論證鏡像成立——兩側都得會合才配叫導數（ch04 的左右夾）。其二：第 5 步是唯一的命門。f 若在 x 點跳躍，m 和 M 會分別被兩側的高度扯住、永遠會合不了——紙上推演第 2 題會把這個破壞例做給你看；連續性的完整故事在 ch14。

把結論唸出來：**累積量的變化率＝當下的高度**。事後看，它近乎理所當然——面積本來就是高度一條一條疊出來的，疊加的速度當然就是現在這條的高度；counter 增長的速率，當然就是現在的 QPS。整個證明只是把這句直覺裡的「當然」換成經得起 ch03 挑戰賽的步驟。最深的定理往往長這樣：證明完之後你覺得它本來就該對——但在證明之前，切線和面積可是兩個沒有任何共用零件的世界。我認為這七步是整本書最美的一段路。

## 定理的兩張臉

剛才證的是 FTC 的第一張臉：

**第一張臉（微分形式）**：f 連續時，A(x) = ∫₀ˣ f(t) dt 可微，且 A′(x) = f(x)。——先積分、再微分，回到原函數：微分把累積還原成高度。

但開頭承諾的是「端點相減」那招。它是第二張臉，從第一張臉走三步就到。先取個名字：導數恰好等於 f 的函數，叫 f 的**反導數**（antiderivative）。第一張臉說：A 本人就是一個反導數。可是你用 ch05 規則表反查出來的，通常是另一個——比如查到 F(x) = x³/3，誰保證它跟「真正的面積函數」是同一個東西？走這三步：

```text
設 F 是你反查出來的任何反導數（F′ = f），A(x) = ∫[a,x] f(t) dt
（起點從 0 換成一般的 a，七步證明一字不改——起點只決定 A 從哪裡歸零）

(A − F)′ = A′ − F′ = f − f = 0      ← 第一張臉上場
導數處處為 0 ⇒ A − F 是常數 C       ← 速度恆為 0 的車不會移動
                                      （直覺版；嚴格版要均值定理，本書不證）
A(x) = F(x) + C
代 x = a：0 = F(a) + C ⇒ C = −F(a)   ← A(a) 是「從 a 到 a 的面積」= 0
代 x = b：∫[a,b] f(t) dt = F(b) − F(a)　∎
```

**第二張臉（求值形式）**：只要 F′ = f，就有 ∫[a,b] f(t) dt = F(b) − F(a)。——隨便哪一個反導數，端點相減就是整段面積。

兩張臉為什麼是同一枚硬幣：第一張臉是**存在性**——每個連續函數都有反導數，而且面積函數本人就是；第二張臉是**可用性**——反導數彼此只差一個常數，所以隨手抓一個來相減，常數自己消掉。一張是引擎，一張是操作介面。那個惡名昭彰的「+C」也在這裡領到身分：反導數從來不是一個函數，是一整族彼此平移的函數——而端點相減的結構恰好讓你永遠不需要知道自己抓到族裡的哪一個。

## 同一題三算：總對帳

現在收割。全書的脊椎問題——y = x² 從 0 到 1 的曲線下面積——第三次、也是最後一次登場：

```text
第三算（FTC）：
  反查：F(x) = x³/3，驗證 F′(x) = 3x²/3 = x²  ✓   ← ch05 冪規則反著用
  相減：∫₀¹ x² dx = F(1) − F(0) = 1/3 − 0 = 1/3
```

一行。對帳表（這是同一題：ch02 的弓形換上座標衣服後正是這塊面積）：

| 算法 | 方法 | 工作量 | 適用範圍 |
|---|---|---|---|
| 第一算：窮盡法（ch02） | 內接三角形逐層修補＋有限和恆等式＋雙重歸謬 | 一位史上級天才，外加一套只適用拋物線的客製幾何 | 這一題——每換一個圖形，從零重新發明 |
| 第二算：黎曼和（ch07） | 切 n 條矩形、Σk² 公式加總、取 n→∞ 的極限 | 半頁代數，外加一條你得先知道的求和公式 | 任何夠乖的函數——但每題重打一場極限硬仗 |
| 第三算：FTC（本章） | 反查微分規則表得 x³/3，端點相減 | 一行 | 任何找得到反導數的函數——一張表打天下 |

**這裡了不起在哪**，請讀準確：不在「算得快」。牛頓（Newton）與萊布尼茲（Leibniz）的天才，不在算出某一塊面積——那件事阿基米德兩千年前就辦到了——而在看出**所有面積問題是同一個問題**：找反導數。從此每一條微分規則（ch05 整章）自動翻面變成一條積分規則，「每題請一位天才」的手工業，一夜之間變成「查同一張表」的工業。ch02 結尾那個 1800 年的卡關，卡的就是這條通法。這是技術革命的標準長相：不是把舊工作做快十倍，是讓一整類工作換了形狀。

誠實的另一面，記兩筆。第一，FTC 不是萬靈丹：反導數不保證查得到——e^(−x²) 的反導數可以證明不是任何初等函數（本書不證），這時你乖乖退回 ch07 的數值積分。黎曼和沒有白學：它是積分的**定義**，也是 FTC 失靈時的 fallback path。第二，找反導數本身有一整套技巧目錄（教科書的「積分技巧」章節）——本書全域不教，欣賞引擎不需要考技師執照。

## 誰看見了全貌：從幾何珍玩到演算法

照慣例把功勞記對人——這次的記法本身就有數學教訓。

FTC 不是某天從天上掉下來的。第一個**公開出版**的初步幾何版本，出自蘇格蘭人格雷果里（James Gregory）1668 年的《Geometriae pars universalis》；更一般的幾何版證明，出自巴羅（Isaac Barrow）約 1670 年的《Lectiones Geometricae》（第 10、11 講）。順帶修正一個通俗說法：「巴羅是牛頓的老師」——精確版本是，巴羅是劍橋首任盧卡斯教授（Lucasian Professor）、牛頓的前任與提攜者，正式授業關係的證據其實薄弱。提攜倒是有憑有據：1669 年他把二十六歲牛頓的手稿《De Analysi》轉寄給倫敦的柯林斯（John Collins），信裡稱作者「極年輕……但在這些事情上有非凡的天才」；同年十月他辭去教席，推薦牛頓接任（2026-06 查證）。

那麼為什麼革命不是 1668 年爆發？因為格雷果里和巴羅手裡的定理，包在歐幾里得式的幾何語言裡——一條關於切線截距與面積曲線的比例性質，證明嚴謹、結論正確，**威力被包裝吞掉了**。在他們眼裡這是一顆幾何珍玩；沒有座標代數的火力支援，看不出它其實是一台引擎。

牛頓與萊布尼茲各自獨立看見了全貌。牛頓 1669 年的《De Analysi》已經把「求面積＝反微分」當成計算方法系統使用（這份手稿私下流傳了四十多年，遲至 1711 年才出版——優先權之爭的火種之一，那場官司的難看細節見 ch04）。萊布尼茲一邊從巴羅的書得到實質幫助（2026-06 查證）、一邊把定理鑄進記號裡：∫ 與 d 這對符號（手稿 1675 年，見 ch04／ch07）根本就是照著「互為反操作」設計的，定理在記號層面讀起來像約分。兩人共同完成的事是同一件：把一條幾何性質，升級成「所有面積問題的通用演算法」，再配上完整的規則系統。

數學教訓比歷史八卦值錢：**看見一個事實，和看出這個事實是做什麼用的，是兩種不同等級的天才**。你在工程史上見過同款劇情——關鍵的發現早就躺在某篇論文裡，改變產業的是把它做成系統的人。

## counter、rate()、increase()：定理住在你的監控系統裡

最後把開頭那座橋走滿。這是全書最重要的一座橋，值得一格一格踩。

| 監控系統 | 微積分 | 對應 FTC 哪張臉 |
|---|---|---|
| counter `C(t)` | 累積函數 A(t)——系統即時維護的反導數 | 主角本人 |
| `rate(C[w])` | A′(t)：差商的極限（ch04 對過帳） | 第一張臉：微分把累積還原成高度 |
| `increase(C[w])` | ∫ 窗口內 QPS dt：整段累積量 | 第二張臉：端點相減＝整段面積 |

逐格驗收。counter 是累積函數：每個請求 counter 加一，C(t) 就是「從開機到 t 的請求面積」。`rate()` 是它的導數：取樣窗口上的差商，窗口縮小的極限就是瞬時 QPS——這半邊 ch04 講過。新領的是另外半邊：`increase(C[1h])` 給你一小時的總請求數，而 Prometheus 文件白紙黑字說 increase 就是 rate 乘上窗口秒數的語法糖（2026-06 查證）——它**不加總任何取樣**，本質上拿窗口兩端的 counter 值相減（外加 reset 補償與端點外插，所以結果可能不是整數）。「不用加總、端點相減就是整段累積」——這句話就是 FTC 的第二張臉，每天在你的 dashboard 上執行。

連「+C」都有對應物。counter 的絕對值毫無意義——process 重啟歸零、不同 instance 起點不同——有意義的永遠是**差**。反導數差一個任意常數而無妨，理由一模一樣：你永遠只用端點差，基準點是多少根本不重要。數學家的「+C」和你對 counter reset 的淡定，是同一個直覺。

離散版你也早就寫過：**前綴和**（prefix sum）。先花 O(n) 建好 S[i] = a[0] + … + a[i−1]，之後任何區間和都是 S[j] − S[i]，O(1) 回答。建前綴和＝積分，相鄰差 S[i+1] − S[i] = a[i]＝微分，「區間和＝端點差」＝FTC。你刷題時把它當小聰明背下來的那一招，是基本定理的整數版。

所以結帳：你不是「將來會用到」FTC——你的監控棧每一天都在執行它：counter 維護累積、`rate()` 取變化率、`increase()` 收整段帳，三位一體。ch01 起這本書反覆說「你早就在用微積分」，這句話在本章兌現到頂。接下來換方向：有了引擎，這門語言要開始長大——把函數拆成無限級數（ch09）、讓定律寫成變化率（ch10）、把訊號拆成圓（ch11），全部踩在這條定理上。

## 直覺的陷阱

FTC 最大的危險不是難懂，是**太好用**——用順手之後，你會忘記它是一條需要證明的定理，從此講不出它哪裡神奇。

| 錯誤直覺 | 它把你帶去哪條溝 | 自我察覺的徵兆 |
|---|---|---|
| 「積分的定義就是反微分」 | 把定理當定義。積分的定義是黎曼和的極限（ch07）；「面積能用反導數算」是要證明的驚人事實——當成定義，FTC 在你眼裡變成廢話 | 你說不出 FTC「到底哪裡了不起」，或覺得本章標題的問題很無聊 |
| 「微積分互逆就像乘除互逆，平凡」 | 乘除互逆寫在定義裡；這裡是兩個**各自獨立定義**的極限過程恰好互逆 | 你講不出微分和積分各自的定義，只記得「它們互相抵銷」 |
| 「A(x) 裡的 t 是手滑寫錯」 | 迴圈變數和上限參數混用，∫₀ˣ f(x) dx 這種式子寫出來不覺得怪 | 你說不出 t 和 x 各自扮演什麼角色 |
| 「+C 是老師用來扣分的迂腐」 | 看不見反導數是一整族函數，也看不見端點相減為什麼恰好讓常數無害 | 你說不出「為什麼算 F(b) − F(a) 時不必管 C」 |
| 「有了 FTC，黎曼和過時了」 | 反導數查不到時（e^(−x²)）生產環境裡跑的是數值積分；FTC 是快取命中，黎曼和是 fallback | 你以為「所有積分都解得出來」 |
| 「FTC 對任何函數都成立」 | 第 5 步的連續性假設被你吃掉了；f 一跳，A 就長角 | 你說不出證明的哪一步用了連續、跳躍點上會發生什麼（推演第 2 題） |

## 紙上推演

**1. 秒殺與苦工的帳** **[10 分鐘]** ★
用 FTC 算 ∫₀¹ x dx 與 ∫₀² x³ dx：寫出反導數、驗證它微分回去是對的、端點相減。然後對帳：前者對照幾何（三角形）與你在 ch07 推演做過的 n = 4 黎曼和；後者請寫出「如果走 ch07 的路」需要哪些裝備，感受一次工作量的落差。

**2. 把定理弄壞** **[15 分鐘]** ★★
取跳躍函數 f(t)：t < 1 時值為 0，t ≥ 1 時值為 1。寫出 A(x) = ∫₀ˣ f(t) dt 的明確公式，畫出 A 的圖形，指出它在 x = 1 處長什麼樣。然後回答：七步證明在哪一步、為什麼壞掉？

**3.【全書最重要的一題】把證明講出來** **[30 分鐘]** ★★★（口頭題）
不看書，把 FTC 的細條證明完整講給另一個工程師聽（真人最好；沒有就對著錄音講，或寫逐字稿）。從「面積變成函數」起手，到「累積量的變化率＝當下的高度」收尾。講完用解答的要點清單逐條自評。這一題是本書對自己的最終驗收之一：它過了，這本書對你就值回票價；它沒過，本章還沒讀完。

### 推演解答

**1.**

```text
∫₀¹ x dx：反導數 F(x) = x²/2，驗證 F′(x) = 2x/2 = x ✓
          F(1) − F(0) = 1/2 − 0 = 1/2
對帳：三角形面積 = (1 × 1)/2 = 1/2 ✓
      ch07 推演的 n = 4 黎曼和：左和 0.375 < 1/2 < 右和 0.625 ✓

∫₀² x³ dx：反導數 F(x) = x⁴/4，驗證 F′(x) = 4x³/4 = x³ ✓
          F(2) − F(0) = 16/4 − 0 = 4
```

苦工版的裝備清單：你需要 Σk³ = n²(n+1)²/4 這條公式（比 Σk² 更冷門），然後算右和 (2/n)·Σ(2k/n)³ = 4(n+1)²/n²——n = 4 給 6.25、n = 10 給 4.84、n = 100 給 4.0804，最後取極限得 4。每一步都做得到，但你已經知道答案是一行：這就是第三算革命的體感。常見錯路：F(2) 算成 2³/4 = 2（忘了反導數是四次方）；驗證那步偷懶跳過——反查最大的好處就是**答案自帶免費的正確性檢查**，微分回去對不上就是查錯了。

**2.** 跳躍前不累積、跳躍後等速累積：

```text
f(t) = 0（t < 1）；1（t ≥ 1）
A(x) = 0（x ≤ 1）；x − 1（x > 1）
```

```
A(x)
  │                      *
  │                    *
  │                  *      ← x > 1：A(x) = x − 1，斜率 1
  │                *
  │              *
  └**************┴────────── x
  0              1
```

A 在 x = 1 處有一個**角**：左斜率 0、右斜率 1。注意兩件事。第一，A 處處連續——積分是平滑化操作，把 f 的「跳躍」磨成 A 的「角」，粗糙度降一級但沒有歸零；微分反過來，是放大粗糙度的操作。第二，壞掉的位置在第 5 步：h > 0 時細條全在跳躍右側，f 恆為 1，m = M = 1，差商穩穩被夾向 1；h < 0 時細條落在跳躍左側，f 一路是 0，這段累積一點都沒長——A(1+h) = A(1) = 0，差商直接是 0（端點 t = 1 只是一個點，對面積沒有貢獻，扯不動結果）。一邊釘在 1、一邊釘在 0，**會合失敗**——左右極限 1 ≠ 0，A′(1) 不存在，而 f(1) = 1 沒有東西等於它。連續性不是律師條款，是第 5 步「m 與 M 被擠向同一個值」的供電來源；連續性的正式定義與它能買到什麼保證，見 ch14。

**3.** 模範答案要點清單（你的版本必須命中全部六點，順序可以自己編排）：

1. **升級視角**：把「從 0 到 x 的面積」變成函數 A(x)，x 是滑桿——沒有這步，後面連「面積的導數」這句話都說不出口。
2. **細條**：A(x+h) − A(x) 是一條寬 h 的細條——大面積減小面積，剩中間那條。
3. **三明治**：細條被 m·h 與 M·h 夾住（裝得下矮矩形、被高矩形裝下），m、M 是條內最低與最高的高度。
4. **同除以 h**：差商被 m 與 M 夾住。
5. **取極限並點名連續性**：h → 0 時條縮成線，**因為 f 連續**，m 和 M 同被擠向 f(x)，夾擠定差商極限——A′(x) = f(x)。
6. **收尾一句話＋為什麼值錢**：「累積量的變化率＝當下的高度」；再由「反導數彼此只差常數」走到 F(b) − F(a)——所以面積問題＝反查微分表（或用 counter／rate／increase 收尾也算命中）。

常見錯路：從「微分和積分互逆」開始講——那是結論，不是論證，等於用待證的事證明自己；忘記點名連續性在哪一步上工（要點 5 的後半）；說出「把 h 設成 0」（回去重讀 ch04）；只證了第一張臉卻宣稱「所以端點相減」（少了「差常數」那塊墊腳石）。

## 自我檢核

口頭自答，講得出來才算過關：

1. 在 FTC 出場之前，微分和積分各自的定義是什麼？定義層面它們共用任何零件嗎——「這兩件事憑什麼有關係」為什麼是個真問題而不是修辭？
2. 累積函數 A(x) 是什麼？「把面積變成函數」這個視角轉換為什麼是整個證明的前提？積分號裡的 t 和外面的 x 各是什麼角色？
3. 七步證明，不看書走一遍（對，就是推演第 3 題——它重要到值得在檢核清單再點一次名）。
4. 定理的兩張臉各自說什麼？從第一張臉走到第二張臉，中間墊了哪塊石頭？
5. 「+C」為什麼在端點相減時無害？它在你的監控系統裡的對應物是什麼？
6. 同一題三算的對帳表，憑記憶重建：三種算法的方法、工作量、適用範圍。然後回答：FTC 是「技術革命」而不只是「更快的技巧」，差別在哪一句話？
7. f 在某點跳躍時，A 在那一點長什麼樣？證明的哪一步、因為什麼壞掉？
8. `increase()` 憑什麼可以不加總任何取樣就給出整段增量？這句話翻成數學，是 FTC 的哪張臉？
