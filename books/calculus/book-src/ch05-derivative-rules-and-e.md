# ch05 — 導數的代數：規則為什麼長那樣

> **本章解決什麼問題**：ch04 把導數定義成差商的極限，但如果每個函數都得回到定義硬算一次極限，微分就只是個漂亮的想法，不是工具。這一章把直覺規則化成可演算的代數——乘法規則來自一張矩形圖、連鎖規則來自縮放器串接——並且在「哪個底數最自然」的追問裡撞見 e。讀完你應該能**不背公式地重新發明**這些規則，並看懂微分的世界觀：局部線性。它的應用在 ch06（最佳化），它的另一半在 ch07（積分）。

## 從你已知的出發

你做容量規劃時心裡有一條曲線：負載低時 CPU 隨 QPS 緩慢爬升，接近飽和時急轉直上。整條曲線明顯非線性，但你還是敢說「QPS 再加 10%，CPU 大概再加 7%」——因為你只在**目前工作點附近的小區間**外插，而任何光滑曲線在夠小的區間裡都近似直線。這個你天天在用的工程直覺，就是本章的世界觀：局部線性。

第二個你已經會的東西是換匯鏈。台幣換美元、美元再換日圓，總匯率是兩段匯率**相乘**——多層轉換的比率會層疊。連鎖規則（chain rule）就是這件事的微分版。

第三個是失控的指數：重試風暴、cache 雪崩、複利。它們共同的形狀是「變化速度跟現量成正比」——量越大、長得越快。這一章會問一個精確的問題：哪個底數讓「增長率」恰好**等於**「自身」？答案是 e，而它最早是從複利表裡被人撞見的。

## 局部線性：微分的世界觀

ch04 的定義是 f′(x) = lim(h→0) (f(x+h)−f(x))/h。把這條式子反過來讀，就得到本章的引擎：

```text
f(x+h) ≈ f(x) + f′(x)·h        ← h 夠小時，誤差比 h 還小一個等級
```

白話：**在夠小的範圍內，函數就是一條直線**，斜率是 f′(x)。你把任何光滑曲線放大、再放大，它終究會直到看不出彎：

```
原始尺寸            放大 10 倍          放大 100 倍
│        *          │        *          │        *
│       *           │       *           │       *
│     **            │      *            │      *
│   **              │    **             │     *
│ **                │  **               │    *
│*                  │**                 │   *
└─────────────      └─────────────      └─────────────
  彎得很明顯          弧度變淺            看起來就是直線
```

「放大到變直」（zoom until straight）不只是比喻——它是整個微分學的視角。導數不是曲線的某種神祕屬性，它就是**放大之後那條直線的斜率**。本章接下來的每一條規則，都是這個視角的推論：函數在局部是直線，直線只有斜率一個自由度，所以組合函數的導數可以用「斜率的代數」算出來。

拿數字確認「誤差比 h 還小一個等級」不是空話。取 f(x) = x²、工作點 x = 3（ch04 的老地方，f′(3) = 6），線性近似是 f(3+h) ≈ 9 + 6h（自算複核）：

```text
h = 0.1 ：真值 3.1²  = 9.61    線性近似 9.6    誤差 0.01
h = 0.01：真值 3.01² = 9.0601  線性近似 9.06   誤差 0.0001
```

h 縮成 1/10，誤差縮成 1/100——近似差的是 h² 等級的量，比 h 本身又小一級。記住這個 h²：下一節矩形圖右下角那塊「小角」，就是它的幾何身世。

兩個誠實的註記。第一，「光滑」是前提：f(x) = |x| 在 x = 0 放大一萬倍還是一個尖角，左邊斜率 −1、右邊斜率 +1，沒有「那條直線」可言——尖點處不可微。第二，更兇的事在後面：ch13 會端出一個**連續但處處是尖刺**的怪獸函數，無論放大到什麼尺度都不會變直——「放大到變直」這幅本章賴以維生的圖像，會被它整個打破。現在先記住：這個世界觀好用，但它有適用範圍，而範圍的邊界比直覺以為的近得多。

## 和規則一句話，乘法規則一張圖

和規則只值一句話：(f+g)′ = f′ + g′，兩個量各自變化、變化量相加，沒有任何驚喜。

乘法規則才是故事的開始。先猜猜看：(f·g)′ 等於 f′·g′ 嗎？這個猜想自然到連萊布尼茲（Leibniz）本人都先寫錯——1675 年秋的手稿裡他一度寫下 d(uv) = du·dv，幾天後才自己修正（2026-06 查證：此說見於數學史料整理與其手稿研究，細節未能完全查證）。檢驗很便宜：取 f = g = x，那 f·g = x²，導數該是 2x（見 ch04）；但 f′·g′ = 1·1 = 1。錯得離譜，而且錯的形狀很有資訊量——正確答案跟 x 有關，猜想卻是常數，代表猜想把「f 和 g 自身的大小」整個弄丟了。

正確的規則要從一張圖長出來。把 f(x)·g(x) 看成一塊矩形的面積：邊長分別是 f 和 g（把 x 固定，先省略不寫）。現在讓 x 動一點點（增量 h），兩條邊各自變長 Δf = f(x+h)−f(x) 與 Δg = g(x+h)−g(x)。面積多出來的部分是什麼形狀？

```
        g(x)        Δg
   ┌──────────────┬──────┐
   │              │      │
 f │     f·g      │ f·Δg │
   │              │      │
   ├──────────────┼──────┤
Δf │     g·Δf     │ 小角 │
   └──────────────┴──────┘
```

新增面積拆成三塊：右邊一條直條 f·Δg、下面一條橫條 g·Δf、右下角一小塊 Δf·Δg。完整推導（工程師的嚴謹：每一步都能口頭說出理由）：

```text
Δ(fg) = (f+Δf)(g+Δg) − f·g
      = f·Δg + g·Δf + Δf·Δg          ← 展開，三塊面積各就各位

除以 h，讓 h→0：

(fg)′ = lim(h→0) [ f·(Δg/h) + g·(Δf/h) + Δf·(Δg/h) ]
      = f·g′    + g·f′    + 0·g′      ← Δg/h→g′、Δf/h→f′、Δf→0
      = f′g + fg′
```

關鍵在第三項——**小角為什麼可以丟**。「因為它很小」不是理由：三塊面積在 h→0 時統統趨於 0，照這個標準全都該丟。真正的理由是：我們除的是**一個** h，而小角是**兩個**小量的乘積。除以 h 只消耗掉其中一個（Δg/h 趨向 g′，是個有限數），剩下的 Δf 仍然趨於 0，所以整項是「趨於 0 的量 × 有限的量」，極限是 0。兩條邊條則不同：各自只含**一個**小量，被 h 除掉之後剩下的是 f·g′ 與 g·f′，活了下來。這就是極限在背後工作的樣子——它不是把「小東西」一律抹掉，而是精確地區分「跟 h 同階的小」與「比 h 更高階的小」。（這裡用了「極限可以拆開相加相乘」的運算定律，以及「可微保證 Δf→0」——本書當直覺接受，嚴格版見 ch14 一句話。）

拿具體數字驗一次，順便跟 ch04 對帳。取 f = g = x，在 x = 3、h = 0.1：

```text
面積增量 = 3.1² − 3² = 0.61
兩條邊條 = 3×0.1 + 3×0.1 = 0.60
小角     = 0.1×0.1      = 0.01
除以 h：0.61/0.1 = 6.1 = 6（邊條的貢獻）+ 0.1（小角的貢獻）
```

ch04 的割線斜率表在 h = 0.1 時給的數字正是 6.1。現在你知道那個「6.1 = 6 + 0.1」裡多出來的 0.1 是什麼了：**它就是矩形圖右下角那一小塊，除以 h 之後的殘骸**。h 繼續縮小，殘骸跟著消失，留下 6。我認為這是本章最值得停下來的一刻——兩條完全不同的路（數值逼近表、幾何拆面積）在同一個數字上會合，而且互相解釋了對方的零頭。

## 連鎖規則：串接的縮放器

複合函數 z = f(g(x)) 是一條管線：x 進去，先被 g 加工成 y = g(x)，再被 f 加工成 z = f(y)。局部線性說：在工作點附近，每一級加工都只是一個**縮放器**——輸入抖動 1 單位，g 這一級把它放大 g′(x) 倍，f 那一級再放大 f′(y) 倍。串接起來，總縮放倍率是兩級相乘：

```text
dz/dx = dz/dy · dy/dx        ← 比率層疊：總倍率 = 各級倍率相乘
```

這跟換匯鏈一模一樣：假設 1 美元換 32 台幣、1 台幣換 4.7 日圓，那 1 美元換 32×4.7 = 150.4 日圓——中間貨幣只是過站，倍率直接相乘。連鎖規則說的就是：**變化率是局部的匯率**。

實際拆一題，並且用第二條路驗算。取 z = y²、y = x³，即 z = (x³)²：

```text
dz/dy = 2y = 2x³        ← 外層：y 抖 1，z 抖 2y
dy/dx = 3x²             ← 內層：x 抖 1，y 抖 3x²
dz/dx = 2x³·3x² = 6x⁵

驗算：z = (x³)² = x⁶，直接微分得 6x⁵ ✓
```

兩條路同一個答案——連鎖規則不是新的真理，只是把「先合成再微分」變成「先微分再相乘」的便道。（順帶誠實一句：嚴格證明還得處理內層增量恰好為 0 的邊角情況，本書不展開——縮放器圖像在光滑函數上是對的，但它是直覺版。）

現在回收 ch04 埋的伏筆。萊布尼茲記號讓 dz/dy · dy/dx「看起來像分數約分」：dy 上下對消，剩下 dz/dx。誠實地說：**這不是除法**。ch04 講過，dy/dx 是一整個極限的名字，不是 dy 除以 dx 兩個量——這裡沒有任何代數約分發生，真正發生的是上面那個「縮放器串接」的極限論證。但請注意這件事有多了不起：萊布尼茲設計的記號，讓一條需要證明的定理**看起來像一行不證自明的代數**。記號替你思考、替你記憶、甚至替你猜出正確的結果——這是史上最成功的記號設計之一，也是它用了三百多年還沒退役的原因。代價是你得記住它的適用邊界：一階時「約分」屢試不爽，但二階導數 d²y/dx² 就**不能**再當分數搬弄了——指數的位置根本不對稱，那是記號在提醒你它只是記號。順帶一句敬意：牛頓（Newton）的流數記號 ẋ 在這種多層串接裡無法表達「對誰微分」，這是萊布尼茲記號在歐陸勝出的原因之一；本書的工作記號維持 f′ 與 dy/dx（見 ch04）。

## 哪個底數最自然：在複利表裡撞見 e

輪到指數函數。對 aˣ 直接套定義：

```text
(aˣ)′ = lim(h→0) (aˣ⁺ʰ − aˣ)/h
      = aˣ · lim(h→0) (aʰ − 1)/h     ← aˣ 跟 h 無關，整個提出來
```

剩下的極限跟 x 完全無關，只跟底數 a 有關——它就是 aˣ 在 x = 0 處的斜率，記作 C(a)。所以**每個指數函數的導數都是它自己乘一個常數**：(aˣ)′ = C(a)·aˣ。常數是多少？手算逼近（h 取小，自算複核）：

| 底數 a | h=0.1 | h=0.01 | h=0.001 | 趨向 |
|---|---|---|---|---|
| 2 | 0.7177 | 0.6956 | 0.6934 | 0.6931… |
| 10 | 2.5893 | 2.3293 | 2.3052 | 2.3026… |

底數 2 的指數函數，增長率是自身的 0.6931 倍；底數 10 則是 2.3026 倍。兩個常數都醜，而且來路不明（謎底在下一節揭曉）。自然的問題是：**有沒有一個底數，讓常數恰好等於 1**——增長率不多不少、就等於自身？

C(a) = 1 的意思是：h 很小時 aʰ ≈ 1 + h，也就是 a ≈ (1+h)^(1/h)。令 h = 1/n，這個神祕底數應該是 (1 + 1/n)ⁿ 在 n→∞ 的極限。（直覺版：這裡偷偷交換了兩個極限的順序，嚴格論證本書不做。）

這個式子，歷史上不是從微分來的，而是從錢來的。1683 年，雅各布·白努利（Jacob Bernoulli）研究複利問題：本金 1 元、年利率 100%，結算次數越多越好嗎？一年結一次得 2 元；半年結一次，每次 50%，得 (1.5)² = 2.25 元。結算越密、賺得越多——但有上限嗎？把表算下去（全部數字自算複核）：

| 結算頻率 | n | (1 + 1/n)ⁿ |
|---|---|---|
| 一年一次 | 1 | 2 |
| 半年一次 | 2 | 2.25 |
| 每月一次 | 12 | 2.6130 |
| 每天一次 | 365 | 2.7146 |
| 每小時量級 | 10000 | 2.71815 |
| 連續複利 | →∞ | e ≈ 2.71828 |

表的形狀本身就有戲：從每月到每天，多結算 353 次，只多賺 0.10 元；從每天到「連續複利」，再多賺不到半分錢。數列在爬，但越爬越無力——它在逼近一個天花板。白努利證明了這個極限存在、且夾在 2 與 3 之間，但算不出它的值（2026-06 查證）。這個天花板後來被叫做 e = 2.71828…。

注意這件事的奇怪程度：一個從**複利**裡冒出來的數，恰好就是讓**導數等於自身**的底數。同一個 e 守著兩道門。把橋走明白一點：「增長率等於現量」翻成複利語言，就是「每一個瞬間都以 100% 年利率對當下餘額計息」。把一年切成 n 段，每段利率 1/n、餘額乘上 (1+1/n)，n 段下來就是 (1+1/n)ⁿ；切得越細，越接近「每瞬間都在計息」的理想，極限就是 e。所以複利表撞上 e 不是巧合——它是在**數值地執行**「導數等於自身」這個性質，跟 ch04 用割線表逼切線是同一族的「切細、執行、取極限」操作（ch10 的 Euler 法會再見到一模一樣的形狀）。於是：

```text
(eˣ)′ = eˣ
```

eˣ 是自己的導數——宇宙最自戀的函數：它的變化率就是它自己，它越大就長越快，長越快就更大。數值體感：(e^0.001 − 1)/0.001 = 1.0005，常數真的是 1（自算複核）。這個性質讓 e 成為「變化速度跟現量成正比」的所有系統的原生語言：人口、放射衰變、複利、你的重試風暴與 cache 雪崩——ch10 會把這件事升級成微分方程的世界觀。

順帶交代名字。e 這個符號是歐拉（Euler）取的：1727/28 年一篇未出版的火砲論文裡首次使用，1731 年 11 月 25 日給哥德巴赫（Goldbach）的信中見諸文字，1736 年《Mechanica》first 見於印刷品；他為什麼選 e 這個字母，史料無解——可以確定的是，那不是「e for Euler」式的自我命名（2026-06 查證）。

## ln：e 的反面

eˣ 回答的問題是「以自然速率成長 x 單位時間後，變成幾倍」。把問題反過來——「要變成 y 倍，需要多久」——答案就是自然對數（natural logarithm）ln y。它是 eˣ 的反函數：e^(ln y) = y。

ln 的導數不用重新苦算，連鎖規則立刻變現。對恆等式 e^(ln x) = x 兩邊微分：

```text
e^(ln x) · (ln x)′ = 1        ← 左邊用連鎖規則：外層 eˣ、內層 ln x
(ln x)′ = 1/e^(ln x) = 1/x
```

一個指數家族的函數，導數卻是 1/x 這個樸素的有理函數——這條等式在 ch08 之後還會回來扮演要角。

現在揭曉上一節的謎底。任何底數都能改寫成 e 的世界：aˣ = e^(x·ln a)。連鎖規則：

```text
(aˣ)′ = e^(x·ln a) · ln a = ln a · aˣ
```

那個來路不明的常數 C(a)，**從頭到尾就是 ln a**。對照前表：ln 2 ≈ 0.6931、ln 10 ≈ 2.3026，跟手算逼近的數字嚴絲合縫。底數 2 的「討厭常數」不是雜訊，它是 2 距離「自然底數」有多遠的精確度量。

「自然」這個詞也有出處：1668 年墨卡托（Nicholas Mercator）在《Logarithmotechnia》中首次使用「logarithmus naturalis」（2026-06 查證）。為什麼它配叫自然？常用對數以 10 為底，是遷就人類的十根手指；ln 的底數是被數學自己選出來的——選它，所有公式裡的常數都變成 1。

## 結果速查：多項式與 sin/cos

矩形的故事可以升維。x³ 是邊長 x 的立方體體積：邊長多 h，體積增量 ≈ 三片面 3x²·h，再加上邊條與小角——它們都是兩個以上小量的乘積，除以 h 後照樣歸零。所以 (x³)′ = 3x²。一般的冪規則 (xⁿ)′ = n·xⁿ⁻¹ 是同一個故事：n 維方塊有 n 片「面」，每片貢獻 xⁿ⁻¹·h（嚴格版用二項式展開，本書不展開）。

sin 與 cos 給幾何直覺草圖（直覺版，本書不硬證；嚴格版需要 lim(h→0) sin h/h = 1，那是 ch14 等級的工具）。看 sin 曲線上切線斜率怎麼變：

```
 1 ┤      ****            ← x=π/2：切線水平，斜率 0
   │    **    **
   │   *        *
 0 ┼──*──────────*──────────*──► x
   │ 0            *        *  2π
   │               *      *
-1 ┤                **  **
   │                 ****
    （x=0 處斜率 1；x=π 處斜率 −1——逐點讀出來的正是 cos x）
```

x = 0 處曲線最陡、斜率 1；爬到 π/2 攻頂、切線水平、斜率 0；下坡到 π 時斜率 −1。把「斜率」這個新函數逐點描出來，你描的就是 cos x。數值抽查一個點：x = 0 處的差商 sin(0.001)/0.001 = 0.9999998，確實貼著 cos 0 = 1（自算複核）。更俐落的說法：一個點等速繞單位圓，位置是 (cos θ, sin θ)，速度向量是位置向量轉 90°，即 (−sin θ, cos θ)——縱座標的變化率（sin 的導數）恰好是 cos，橫座標的變化率是 −sin。圓的旋轉對稱，是這對導數互咬的真正原因（ch10 的彈簧振子會讓它們從圓裡跳出來）。

本章戰果結算：

| 函數 | 導數 | 一句話理由 |
|---|---|---|
| c（常數） | 0 | 不動的東西沒有變化率 |
| xⁿ | n·xⁿ⁻¹ | n 維方塊的 n 片面 |
| eˣ | eˣ | 被「增長率＝自身」選出的底數 |
| aˣ | ln a · aˣ | aˣ = e^(x·ln a) ＋ 連鎖規則 |
| ln x | 1/x | e^(ln x) = x 兩邊微分 |
| sin x | cos x | 繞圓速度向量＝位置轉 90°（直覺版） |
| cos x | −sin x | 同上 |
| f + g | f′ + g′ | 變化各自累加 |
| f · g | f′g + fg′ | 矩形的兩條邊條；小角陣亡 |
| f(g(x)) | f′(g(x)) · g′(x) | 縮放器串接，倍率相乘 |

退一步看這張表的歷史份量。ch02 說過，阿基米德之後卡了一千八百年，因為每個問題都是一次性的手工藝。這張表加上三條組合規則，意味著**微分變成了演算法**：任何由冪、指數、對數、三角組合出來的函數，都能機械地、不需要靈感地取導數——拆層、查表、相乘相加，像在跑一台確定性的狀態機。「天才的洞見」變成「人人可學的技術」，這正是牛頓與萊布尼茲勝過所有先驅的地方（見 ch08），也是 ch06 之後一切應用的底氣。

## 直覺的陷阱

| 錯誤直覺 | 它怎麼把你帶溝裡 | 自我察覺的徵兆 |
|---|---|---|
| (fg)′ = f′·g′ | 把「f 與 g 自身的大小」弄丟——邊條面積跟邊長成正比，這資訊在 f′·g′ 裡蒸發了。連萊布尼茲都先犯過這個錯 | 用 f = g = x 測一下：得 1，但 (x²)′ = 2x。一秒鐘的測試，永遠值得做 |
| 「小角很小所以可以丟」 | 「很小」不是論證——h→0 時三塊面積都很小。照這標準整個導數都該是 0 | 你說不出「除以 h 之後**誰**還趨於 0、**為什麼**」，就代表你在背結論，極限沒有真的在你腦裡工作 |
| dy/dx 是分數，可任意約分移項 | 一階時恰好都對，於是你越用越大膽——直到二階的 d²y/dx² 或多變數的偏導數時整個爆掉 | 每次「約分」前問自己：這步對應的極限敘述是什麼？說不出來，就是越界了 |
| 局部線性對所有連續函數成立 | 「連續曲線放大就會變直」——|x| 在 0 點放大一萬倍還是尖角；ch13 的怪獸更是處處皆尖 | 畫切線前先問：左右差商一致嗎？「連續」與「可微」之間的鴻溝，比直覺寬一個宇宙（見 ch13） |
| e 只是個要背的常數 2.718… | 把 e 當神祕數字背，你永遠講不出它哪裡「自然」，也接不上 ch10 的微分方程 | 有人問「為什麼不用 2 或 10 當底」你答不上來——答案應該是：只有 e 讓增長率恰好等於自身，其他底數都拖著一條 ln a 的尾巴 |

## 紙上推演

**推演 1【10 分鐘，★】** 用乘法規則的矩形圖推 (x·x)′，畫出圖、標出三塊面積，並對照 ch04 用差商極限算出的 2x。兩條路的「零頭」各是什麼？

**推演 2【15 分鐘，★★】** 用連鎖規則拆 (x²+1)³ 的導數。要求：先口頭說出每一層縮放器各放大多少倍，再寫式子；最後把 (x²+1)³ 整個展開、直接微分，驗證兩條路一致。

**推演 3【15 分鐘，★★】** 解釋「為什麼用 2 當底數的指數函數，導數會多出一個討厭的常數」：那個常數是什麼？它有幾種身分？為什麼 e 沒有這條尾巴？講到一個只讀過 ch04 的工程師能聽懂為止。

### 推演解答

**推演 1**：取 f = g = x，矩形是邊長 x 的正方形。x 多走 h，兩條邊條各是 x·h，小角是 h²：

```text
Δ(x·x) = x·h + x·h + h² = 2x·h + h²
除以 h：2x + h  →(h→0)→  2x
```

對照 ch04 的差商：((x+h)² − x²)/h = (2xh + h²)/h = 2x + h，**逐項相同**——差商裡的 2x 就是兩條邊條、那個「+h」就是小角除以 h 的殘骸。兩條路不只是答案相同，根本是同一筆帳的兩種記法。常見錯路：把小角直接寫成 0 再除以 h——順序錯了，要先除、再取極限，否則你解釋不了 ch04 的表裡為什麼出現 6.1 而不是 6。

**推演 2**：管線是 x → y = x²+1 → z = y³。口頭版：x 抖 1 單位，內層把它放大 2x 倍；y 抖 1 單位，外層把它放大 3y² 倍；串起來總倍率 3y²·2x。寫下來：

```text
dz/dx = 3(x²+1)² · 2x = 6x(x²+1)²

驗算：(x²+1)³ = x⁶ + 3x⁴ + 3x² + 1
直接微分 = 6x⁵ + 12x³ + 6x = 6x(x⁴ + 2x² + 1) = 6x(x²+1)² ✓
```

抽查一個點增加信心：x = 1 時兩式都給 24。常見錯路：忘記內層的 2x（把複合函數當單層函數微分）——用「每層各自報倍率」的口頭流程就不會漏，因為管線有兩級，倍率就必須有兩個。

**推演 3**：套定義，(2ˣ)′ = 2ˣ · lim(h→0) (2ʰ−1)/h。那個極限是 2ˣ 在 x = 0 處的斜率，數值逼近給 0.7177（h=0.1）→ 0.6934（h=0.001）→ 0.6931…。它的第二個身分在 ln 一節揭曉：2ˣ = e^(x·ln 2)，連鎖規則給 (2ˣ)′ = ln 2 · 2ˣ——常數就是 ln 2 ≈ 0.6931。白話版：指數函數的導數永遠是「自己 × 在零點的斜率」；底數 2 太小、在零點爬得太慢（斜率只有 0.6931），底數 10 太大、爬太快（2.3026），只有 e 校準到斜率恰好 1，於是導數就是自己、一根尾巴都不拖。其他底數的「討厭常數」其實是連鎖規則的內層倍率 ln a——它在精確報告你離自然底數多遠。
