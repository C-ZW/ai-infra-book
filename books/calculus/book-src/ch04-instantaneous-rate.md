## 從你已知的出發

你每天都在量「瞬間變化率」，只是監控系統幫你把哲學問題藏起來了。

Prometheus 的 `http_requests_total` 是一個 counter：只增不減的累積請求數，叫它 C(t)。你想看的 QPS 不是 counter 本身，是它的變化率，所以你寫 `rate(http_requests_total[5m])`——它做的事情本質上就是拿窗口兩端的差除以窗口長度：

```text
QPS ≈ (C(t) − C(t−w)) / w     ← w 是取樣窗口，例如 300 秒
```

這個式子等一下會有正式名字：差商（difference quotient）。現在問一個你在調 dashboard 時真的遇過的問題：w 該多小？`[5m]` 太鈍，尖峰被抹平；`[1m]` 即時一點；`[10s]` 更即時、但曲線開始毛躁。你想要的其實是「此刻的 QPS」——把 w 縮到 0。但 w = 0 時，分子是 0、分母是 0，整個式子當場死亡。「瞬間 QPS」這個詞，嚴格說根本不存在：任何單一瞬間裡，新增請求數是 0，經過時間也是 0。

你天天面對的這個工程張力，就是本章的主題。數學家把 w（接下來叫 h）一路縮下去，問的不是「h = 0 時式子等於多少」（無意義），而是「h 趨近 0 時，式子逼近哪裡」。這一字之差，就是兩千年。

（至於窗口太小時雜訊爆炸——這件事在純數學的浮點世界裡也會發生，本章結尾的 Python 小實驗會讓你親眼看到，ch15 再算總帳。）

## 「瞬間速度」本來是一句自相矛盾的話

速度 = 走過的距離 ÷ 花掉的時間。這個定義天生只對「一段時間」有意義：一小時 60 公里、一秒 3 公尺。現在問：「箭在這一瞬間飛多快？」一瞬間裡，箭走過的距離是 0，經過的時間是 0。按定義，瞬間速度 = 0/0。

0/0 為什麼無意義？回到除法的定義：a/b = c 的意思是 a = b·c。那 0/0 = c 要求 0 = 0·c——任何 c 都成立。0/0 不是「沒有答案」，是「答案太多」：它本身不攜帶任何資訊。更糟的是，長得像 0/0 的極限，答案真的各不相同：

```text
lim(h→0) 5h/h  = 5     ← 分子分母都趨近 0，極限是 5
lim(h→0) h²/h  = 0     ← 也都趨近 0，極限卻是 0
lim(h→0) h/h²  = ∞     ← 還是 0/0 的長相，這次直接爆掉
```

三個式子都是「0/0 形狀」，極限分別是 5、0、無窮。所以資訊不在「0/0」這個形狀裡，而在**分子分母各自怎麼趨近 0 的路徑**裡。這種式子叫未定式（indeterminate form）——「未定」的意思是：光看形狀定不下來，得看內容。

這正是芝諾（Zeno）第三支箭瞄準的位置（見 ch01）。飛矢悖論說：任一瞬間，箭都佔據著與自身相等的空間，所以那一瞬間它沒在動；飛行不過是瞬間的集合；所以箭從頭到尾不動。兩千年來沒有人能正面回答「瞬間裡的運動是什麼」，因為「瞬間速度」這個概念當時根本沒有定義——你不能用一個未定義的詞打贏辯論。本章結尾我們會回來收掉這支箭。

## 第一條路：割線斜率表，用數字逼

把問題從物理搬到幾何，好算得多。取 f(x) = x²，問：這條拋物線在 x = 3 的地方「有多陡」？

陡峭程度就是斜率，但斜率需要兩個點。一個點畫不出線——這是「瞬間速度需要兩個時刻」的幾何翻版。那就老實給兩個點：(3, 9) 和旁邊一點 (3+h, (3+h)²)，h 是一小段水平距離。穿過這兩點的直線叫割線（secant line），它的斜率是：

```text
割線斜率 = ((3+h)² − 9) / h     ← 高度差除以水平差，國中數學
```

h ≠ 0 時這是一次誠實的除法，沒有任何玄學。現在讓 h 縮小，一步一步算：

```text
h = 1   ：斜率 = (4² − 9)/1        = (16 − 9)/1        = 7
h = 0.1 ：斜率 = (3.1² − 9)/0.1    = (9.61 − 9)/0.1    = 6.1
h = 0.01：斜率 = (3.01² − 9)/0.01  = (9.0601 − 9)/0.01 = 6.01
```

把更多行（包括從左邊靠近的負 h）排成表：

| h | 另一點 3+h | 割線斜率 |
|---|---|---|
| 1 | 4 | 7 |
| 0.1 | 3.1 | 6.1 |
| 0.01 | 3.01 | 6.01 |
| 0.001 | 3.001 | 6.001 |
| −0.001 | 2.999 | 5.999 |
| −0.01 | 2.99 | 5.99 |
| −0.1 | 2.9 | 5.9 |
| −1 | 2 | 5 |

兩側同時夾向 6。畫成圖：h 縮小時，B 點沿著曲線滑向 A，割線繞著 A 旋轉，停靠的位置就是切線（tangent line）——「只在這一點貼著曲線的那條線」。

```
       y
       │                       *
       │                      *
(3+h)² ┤┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄● B = (3+h, (3+h)²)
       │                  *┊
       │                *  ┊   Δy = (3+h)² − 9
       │              *    ┊
     9 ┤┄┄┄┄┄┄┄┄┄┄┄● A┄┄┄┄┄┊
       │          *┊←─ h ─→┊
       │        *  ┊       ┊
       │     *     ┊       ┊
       └──────────────────────── x
                   3      3+h
```

但是注意：這張表再長，也只是證據，不是證明。你算到 h = 10⁻⁹，也只證明了那一個 h 的割線斜率；「斜率逼近 6」仍然是肉眼的歸納。這是 ch02 的老問題：個案手藝堆不出通法。而且表裡藏著一個太過整齊的規律——斜率看起來永遠恰好是 6 + h（7 = 6+1、6.1 = 6+0.1、5.99 = 6+(−0.01)）。這麼整齊，背後一定有代數原因。

## 第二條路：代數約簡，h 約掉的瞬間

把割線斜率直接展開，不代任何數字：

```text
((3+h)² − 9) / h
  = (9 + 6h + h² − 9) / h     ← 展開 (3+h)² = 9 + 6h + h²
  = (6h + h²) / h             ← 9 和 −9 抵銷
  = h(6 + h) / h              ← 分子提出公因數 h
  = 6 + h                     ← h ≠ 0，所以可以約掉
```

割線斜率**恆等於** 6 + h——只要 h ≠ 0。表格的祕密當場揭曉：h = 1 給 7、h = 0.1 給 6.1、h = −0.01 給 5.99，全部就是 6 + h。數值那條路一格一格逼近的目的地，代數這條路一眼讀出來：

```text
lim(h→0) (6 + h) = 6     ← 兩條路在 6 會合
```

請注意整個流程裡 0/0 從未發生。每一步的合法性都經得起盤問：

- 約掉 h 的那一步，前提是 h ≠ 0——而我們考慮的每一條割線本來就要求 h ≠ 0（兩個點要是不同的點）。沒有除以零。
- 最後一步不是「把 h 代成 0」。如果代 0，最初的除法就死了。我們問的是：6 + h 在 h 趨近 0 時逼近什麼。答案是 6——而且這次不是歸納，是 ch03 那種可以打挑戰賽的極限：你出多小的誤差 ε，我都能給出夠小的 h 範圍，讓 6 + h 與 6 的距離小於 ε（這裡 |（6+h）− 6| = |h|，所以 h 的範圍直接取 ε 就行——這場挑戰賽簡單到近乎客氣）。

這就是本章標題「把 0/0 變合法」的真正意思——**不是給 0/0 賦一個值，而是把問題換掉**：先在 h ≠ 0 的安全區做完所有代數，把惹禍的 h 約掉，再對剩下的乾淨式子取極限。0/0 被繞開了，不是被馴服了——馴服的是「瞬間」。

值得停十分鐘的地方在這裡。「瞬間速度」原本是個沒有定義的詞，人類的解法不是回答「瞬間裡怎麼運動」（這個問題到今天也沒有答案，而且不需要有），而是直接**定義**：瞬間變化率 ＝ 越縮越小的窗口上平均變化率的極限。這不是發現了一個藏起來的真相，是發明了一個經得起挑戰賽檢驗的定義，然後整個力學、整個現代科學就蓋在這個定義上。不回答形上學問題、只給出可驗收的操作——這種勝利方式，做工程的你應該覺得眼熟。

## 導數：給這個極限一個名字

把上面的動作寫成通用形式，就是全書最重要的定義之一。函數 f 在點 x 的導數（derivative）：

f′(x) = lim(h→0) (f(x+h) − f(x))/h

分子是輸出的變化，分母是輸入的變化，整個分式叫差商——它就是你在 `rate()` 裡看到的那個結構。導數＝差商的極限。對 f(x) = x² 把一般的 x 留著重算一遍：

```text
f′(x) = lim(h→0) ((x+h)² − x²) / h
      = lim(h→0) (2xh + h²) / h      ← x² 抵銷，剩下含 h 的項
      = lim(h→0) (2x + h)            ← h ≠ 0，約掉
      = 2x                           ← h 趨近 0，2x + h 逼近 2x
```

（最後一步用了「極限可以拆開常數項與 h 項」這類運算直覺——工程師的嚴謹：每步能口頭說出理由；嚴格版的極限運算定律見 ch14。）

f′(x) = 2x。對帳：f′(3) = 6，跟兩條路會合的答案一致。但別錯過這一步真正的升級：**導數不是一個數，是一台機器**——函數進、函數出。x² 進去，2x 出來；2x 把 x² 在每一點的陡峭程度整批描述完畢：x = 0 處斜率 0（谷底是平的）、x = −3 處斜率 −6（左半邊下坡）、x 越大坡越陡。ch02 的阿基米德每換一個點就要重新施工一次；現在一條公式管所有點。這就是通法與手工藝的差距，也是 ch05 整章要展開的世界。

## 兩套記號，兩個發明人，一場難看的官司

你會在所有書裡看到兩套導數寫法，它們來自兩個獨立的發明人，值得各認識一次。

**牛頓（Newton）**：1665–66 年倫敦大瘟疫，劍橋關閉，二十出頭的牛頓回到 Woolsthorpe 老家，在這段日子裡發展出「流數法」（method of fluxions）的核心想法——他把隨時間流動的量叫 fluent、它的變化率叫 fluxion，記作 ẋ（字母上加一點）。系統性的論文 1671 年就寫完了，但他生前不肯出版，拖到 1736 年（他死後）才刊行；點記號則是 1690 年代初才定型、1693 年經 Wallis 首次印行（2026-06 查證）。《自然哲學的數學原理》（Principia）1687 年出版。至於蘋果砸頭——傳說等級的軼事，當故事聽即可。

**萊布尼茲（Leibniz）**：獨立發明了同一套微積分，而且是個記號天才。1675 年 10 月 29 日的手稿裡他首次寫下積分號 ∫（拉丁文 summa 的拉長 S，故事見 ch07）；同年 11 月 11 日首次把 ∫ 與 dx 連用。d 記號 1684 年隨論文〈Nova Methodus pro Maximis et Minimis〉首次印在《Acta Eruditorum》上——這是史上第一篇公開發表的微分學論文；∫ 則到 1686 年才見於出版品。他的導數寫法是 dy/dx：y 的微小變化比上 x 的微小變化，「比率」的直覺直接寫在臉上。

順帶補一筆：今天最常用的撇號 f′(x) 通常冠上拉格朗日（Lagrange）的名字；查證顯示這記號更早已見於歐拉（Euler）筆下、由拉格朗日普及（2026-06，細節未能完全查證）。

本書兩套並用，分工如下：

- **f′(x)**：強調「導數是一台機器、是一個函數」時用它；點值最清楚——f′(3) = 6，乾淨俐落。
- **dy/dx**：強調「兩個變化量的比率」時用它；在連鎖規則（見 ch05）、微分方程（見 ch10）和單位對帳（公尺/秒）的場合，它好用到不公平。但它有個危險的副作用，「直覺的陷阱」一節再算帳。
- **ẋ**：物理書裡見到認得即可——專指對時間微分。

然後是那場官司。兩人都活著時就撕破臉：誰先發明微積分？萊布尼茲向皇家學會申訴，而當時的皇家學會會長正是牛頓本人。他親自挑選了一個全由己方支持者組成的調查委員會（沒有任何歐陸數學家），1712 年作成報告、年底以《Commercium Epistolicum》刊行，結論判萊布尼茲剽竊——而這份報告實質上由牛頓本人起草與主導，萊布尼茲從頭到尾沒有獲得答辯機會。牛頓後來還匿名發表書評，吹捧這份「公正調查」。裁判兼球員，科學史上最著名的一例。現代史學的共識是：兩人各自獨立發明，牛頓在先、萊布尼茲先發表。這場官司最大的輸家其實是英國數學——通行的說法是，英國數學家出於對牛頓的忠誠死守 ẋ 記號、與歐陸的萊布尼茲記號圈隔絕，在分析學上落後了將近一個世紀，直到 1812 年劍橋分析學會（Babbage 等人創立）才把歐陸記號迎回來（敘事輪廓 2026-06 查證；落後程度的量化見仁見智）。記號不只是語法糖，它決定你能想多遠——寫過程式的你不需要被說服。

## Berkeley 的鬼魂：當年的批評是對的

1734 年，柏克萊主教（George Berkeley）出版《The Analyst》，對微積分的邏輯地基發起總攻。他的攻擊點極準：牛頓計算流數時，先讓增量不等於 0（才能除），算完又把它當 0 丟掉（才能得到乾淨答案）——同一個量在同一個論證裡，既不是零、又是零。第 XXXV 節的原句是整本書最鋒利的一刀：

> And what are these Fluxions? The Velocities of evanescent Increments? And what are these same evanescent Increments? They are neither finite Quantities nor Quantities infinitely small, nor yet nothing. May we not call them the Ghosts of departed Quantities?
>
> （那這些流數到底是什麼？消逝增量的速度？那這些消逝增量又是什麼？它們既不是有限量，不是無窮小量，也不是無。我們何不稱之為消逝之量的鬼魂？）

誠實的結論：**以當時拿得出的論證，Berkeley 是對的。**牛頓和萊布尼茲都給不出「無窮小」的一致定義，他們的微積分算出來的答案全對、邏輯地基確實是空的。注意你剛剛在本章走過的路——先在 h ≠ 0 的世界做完除法、約掉 h、最後對 6 + h 取極限——從頭到尾沒有任何量「既是零又不是零」，正好繞開了 Berkeley 的炮口。這套乾淨的走法是後人（Cauchy 到 Weierstrass 一脈）花了一百多年補出來的地基，正式版在 ch14；同樣在 ch14 你會看到一個漂亮的歷史反轉：無窮小本身最終也被 Robinson 嚴格化平反。本書先讓你享受乾淨的版本，但要記得：這份乾淨在 1734 年還不存在，而指出國王沒穿衣服的人推動了數學史。

## 收掉 ch01 的飛矢

現在回頭結案。芝諾的論證：①任一瞬間，箭佔據與自身相等的空間，所以那一瞬間箭是靜止的；②飛行時間由瞬間組成；③所以箭整段飛行都靜止。

極限定義的拆解：第①步的前半其實**永遠是對的**——單一瞬間的快照裡，本來就沒有運動可言，一張照片裡讀不出速度。芝諾錯的是把「快照裡讀不到運動」偷換成「運動不存在」。這個偷換能成立，靠的是一個未言明的假設：「瞬間速度」必須是「發生在那個瞬間內部的某種運動」。

導數的定義正面拒絕了這個假設。v(t) = lim(h→0) (s(t+h) − s(t))/h——瞬間速度不是瞬間裡的運動，而是「以 t 為中心、越縮越小的時間窗上的平均速度」的極限。速度不住在單一快照裡，它住在快照與鄰居的關係裡：是位置在 t 附近如何變化的性質。一張快照讀不出速度，一疊快照的差才讀得出。

所以「箭在每個瞬間都是靜止的」這句話，在極限語言裡甚至算不上錯——它是無意義：「靜止」也是一種速度（v = 0），同樣需要鄰居才能定義，單一瞬間裡既談不上動、也談不上不動。芝諾用一個未定義的概念推出了悖論；極限給了定義，悖論失去立足點。三支箭至此全數落地：二分與阿基里斯死於部分和的極限（見 ch03），飛矢死於導數的定義。ch01 開的局，收掉了。

## 直覺的陷阱

| 錯誤直覺 | 它會把你帶去哪裡 | 自我察覺徵兆 |
|---|---|---|
| 「導數是 0/0 算出來的值」 | 對 0/0 做代數，或以為最後一步是「把 h 代成 0」 | 你的演算過程裡出現「令 h = 0」這種句子 |
| 「dy/dx 是兩個很小的數相除」 | 單層使用多半無害，但你會在 d²y/dx²（完全不像分數）和多變數場合翻車 | 你開始單獨移項 dy、dx，卻說不出哪個定理背書 |
| 「h 夠小之後 6+h 就是 6」 | 把極限當成「過程的最後一項」（ch03 的老病） | 你說出「h 最終變成 0」 |
| 「數值表收斂了就等於證明」 | 把有限個案的歸納當成論證 | 你覺得算到 h = 10⁻⁹ 比代數展開更有說服力（本章實驗還會告訴你：算到 10⁻⁹ 連數值本身都不可信） |
| 「切線就是只碰曲線一點的線」 | 古希臘的定義，在反曲處失效——x³ 在 x = 0 的切線就穿過曲線 | 你答不出「x³ 在原點的切線是哪條」 |

自我檢測法：上面任何一格，如果你需要重看正文才能說出「為什麼錯」，該節就還沒讀完。

## 紙上推演

**題 1 [25 分鐘] ★★** — 對 f(x) = x³ 在 x = 2 重演本章的兩條路：(a) 手算割線斜率表，h = 1、0.1、0.01；(b) 代數展開 ((2+h)³ − 8)/h 並約簡；(c) 指出兩條路會合在哪個數，再把 x 留成一般式，推出 f′(x)。

**題 2 [10 分鐘] ★★（口頭題）** — 把飛矢悖論和它的解講給一個只懂國中數學的人，讓他服氣。限制：不准使用「極限」「導數」這兩個詞——你只能用「越縮越小的時間窗上的平均速度」這類白話。講完用下面解答的要點清單自評。

**題 3 [15 分鐘] ★★★** — 「導數就是 dy 除以 dx，兩個很小的量相除。」這句話有問題。(a) 按本章的定義，式子裡有沒有叫 dy、dx 的物件？「很小的量相除」會給出什麼（提示：6 + h）？(b) 為什麼這句話又「幾乎」是對的——它是哪個合法操作的危險縮寫？

### 推演解答

**題 1**　先備好展開式：(2+h)³ = 8 + 12h + 6h² + h³。

(a) 割線斜率表：

```text
h = 1   ：(3³ − 8)/1       = (27 − 8)/1          = 19
h = 0.1 ：(2.1³ − 8)/0.1   = (9.261 − 8)/0.1     = 12.61
h = 0.01：(2.01³ − 8)/0.01 = (8.120601 − 8)/0.01 = 12.0601
```

(b) 代數路：

```text
((2+h)³ − 8) / h
  = (12h + 6h² + h³) / h      ← 8 與 −8 抵銷
  = 12 + 6h + h²              ← h ≠ 0，約掉
```

(c) 對帳：12 + 6(1) + 1² = 19 ✓；12 + 0.6 + 0.01 = 12.61 ✓；12 + 0.06 + 0.0001 = 12.0601 ✓。極限是 lim(h→0)(12 + 6h + h²) = 12。一般化：((x+h)³ − x³)/h = 3x² + 3xh + h² → f′(x) = 3x²；代 x = 2 得 12 ✓。常見錯路：二項式展開漏掉 6h² 那一項（漏了它，h = 1 那行就對不上帳）；只做正的 h——建議補一行 h = −0.01：12 − 0.06 + 0.0001 = 11.9401，從左側一樣夾向 12。

**題 2**　模範答案要點清單（你的版本必須命中全部五點）：

1. 先讓步：一張快照裡確實看不到運動——芝諾這一半是對的。
2. 指出速度的本性：速度從來都是「一段時間裡的平均」，「一瞬間的速度」原本是個沒有意義的問題。
3. 給一個可操作的比喻：行車記錄器抓兩格相鄰畫面，位置差除以時間差，就是那一小段的平均速度。
4. 給出新定義：把兩格之間的時距越縮越短，平均速度會穩定靠向一個數——我們**約定**那個數叫「這一瞬間的速度」。
5. 點破偷換：芝諾把「快照裡讀不到運動」換成了「運動不存在」；但運動是快照**之間**的關係，不是快照**之內**的東西。

**題 3**　(a) 沒有。本章的定義裡只有一整個極限 lim(h→0) (f(x+h) − f(x))/h，式子裡不存在名叫 dy 或 dx 的獨立物件。「兩個很小的量相除」不管多小，給出的都是某條割線的斜率 6 + h，不是 6；而如果「小到等於 0」，除法本身非法。導數是這串割線斜率的**目的地**，不是其中任何一筆除法的結果。(b) 它是「Δy/Δx 在 Δx 趨近 0 時的極限」的縮寫。萊布尼茲的記號故意長成分數的樣子，因為它在很多場合**表現得**像分數（連鎖規則見 ch05、分離變數見 ch10）——但每一次「約分」其實都是一個定理在背書，不是除法。把縮寫當實體，單層使用多半無害；說不出背書定理還繼續移項，就是翻車的開始。補一刀：無窮小後來真的被嚴格化了（Robinson 非標準分析，見 ch14），萊布尼茲的直覺最終平反——但那是一整套新地基，不是素樸版的「兩個小量相除」。

### Python 小實驗

數學說：h 越小，割線斜率越接近 6。浮點說：你再說一遍試試。跑這段（純標準函式庫）：

```python
# Secant slope of f(x) = x^2 at x = 3, with shrinking h.
# Mathematically the slope (f(3+h) - f(3)) / h equals 6 + h,
# so it should approach 6 as h -> 0. Watch what floats actually do.

def f(x):
    return x * x

x = 3.0
exact = 6.0

print(f"{'h':>7}  {'secant slope':>20}  {'|slope - 6|':>11}")
for k in range(13):
    h = float(f"1e-{k}")          # 1, 0.1, 0.01, ..., 1e-12
    slope = (f(x + h) - f(x)) / h
    print(f"{h:7.0e}  {slope:20.15f}  {abs(slope - exact):11.1e}")
```

預期輸出（CPython 3、IEEE 754 雙精度；這些位數在常見平台上應可逐位重現——浮點運算是確定性的——若你的平台末幾位不同，看趨勢不看位數）：

```text
      h          secant slope  |slope - 6|
  1e+00     7.000000000000000      1.0e+00
  1e-01     6.100000000000012      1.0e-01
  1e-02     6.009999999999849      1.0e-02
  1e-03     6.000999999999479      1.0e-03
  1e-04     6.000100000012054      1.0e-04
  1e-05     6.000009999951316      1.0e-05
  1e-06     6.000001000927568      1.0e-06
  1e-07     6.000000087880153      8.8e-08
  1e-08     5.999999963535174      3.6e-08
  1e-09     6.000000496442226      5.0e-07
  1e-10     6.000000496442226      5.0e-07
  1e-11     6.000000496442226      5.0e-07
  1e-12     6.000533403494046      5.3e-04
```

讀法：誤差先乖乖跟著 h 縮小（誤差 ≈ h，因為斜率恰是 6 + h），在 h ≈ 10⁻⁸ 附近觸底（誤差約 3.6×10⁻⁸），然後**反轉惡化**——h = 10⁻¹² 時誤差暴漲回 5×10⁻⁴，比 h = 10⁻⁴ 的時候還爛。

原因是災難性消去（catastrophic cancellation）：分子 f(3+h) − f(3) 是兩個都約等於 9 的數相減。雙精度浮點在 9 附近的解析度（相鄰可表示數的間距）約 1.8×10⁻¹⁵，分子只能落在這個網格的格點上。h = 10⁻¹² 時分子理應約 6×10⁻¹²，整個量只剩三千多個網格——有效數字被減法吃得只剩三、四位，再除以 h 也救不回來。表中 10⁻⁹ 到 10⁻¹¹ 三行給出一模一樣的斜率，正是這種量化網格造成的浮點巧合（每次跑都一樣，但別在它身上找數學規律）。

兩個收穫：第一，數學極限活在無限精度的世界，浮點機器不在那裡——「h 越小越準」在真實機器上是錯的，最佳窗口卡在中間（你調 `rate()` 窗口時的直覺，數學自己也躲不掉）；第二，最佳 h 為什麼恰好落在 10⁻⁸ 這個量級、以及怎麼用更聰明的差分公式做得更好，是 ch15 的戲份——這張表到時候會原封不動地回來。
