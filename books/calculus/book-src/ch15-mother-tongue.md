## 引擎室：梯度下降與反向傳播

你盯著訓練曲線（loss 對 step 的折線）往下掉的時候，看的其實是 ch06 那張迭代表的億維版本：每個 step，系統算一次坡度、往坡下挪一步、記下新的 loss。ch06 的玩具版你手算過——f(x) = x²、學習率 η = 0.1、從 x₀ = 3 出發，每步乘 0.8，五步走到 0.98304，f 每步乖乖乘 0.64。真實的損失函數不是一條曲線，是一片地形：上億個參數是座標、loss 是海拔，訓練就是在起霧的山坡上摸著局部坡度往下走（ch06 的多維一瞥）。

只剩一個問題：億維的坡度——梯度——憑什麼算得起？答案是 ch05 的連鎖規則，外加一套聰明的記帳順序。這套記帳法叫**反向傳播**（backpropagation）。

神經網路是一條很長的複合函數管線：輸入進去，一層一層加工，最後吐出 loss。ch05 講過複合函數的全部祕密——**比率層疊**：每一級加工在工作點附近都是一個縮放器，總倍率＝各級倍率相乘，dz/dx = dz/dy · dy/dx。反向傳播做的事一句話講完：**從輸出端開始，把各級倍率一路往回乘，沿途記帳**。看迷你版，兩級管線 f(x) = (3x+1)²，拆成 u = 3x+1、f = u²，在 x = 1 算 df/dx：

```text
前向傳遞（先把每一級的值算出來，x = 1）：
  u = 3·1 + 1 = 4
  f = u²    = 16

反向傳遞（從輸出端把比率往回乘）：
  df/df = 1                          ← 起點：f 對自己，倍率 1
  df/du = 1 × 2u = 1 × 8 = 8         ← 過「平方」級：這一級的倍率是 2u
  df/dx = 8 × 3       = 24           ← 過「×3 再 +1」級：這一級的倍率是 3
```

比率沿著鏈傳播的樣子：

```
前向（值往右流）  ：x = 1 ──[×3 再 +1]──► u = 4 ──[平方]──► f = 16
反向（比率往左乘）：24 = 8×3 ◄──[倍率 3]── 8 = 1×2u ◄──[倍率 2u = 8]── 1
```

用 ch04 的第二條路驗算：f(x) = (3x+1)² = 9x² + 6x + 1，直接微分得 f′(x) = 18x + 6，代 x = 1 得 24 ✓。兩條路會合——反向傳播不是新的真理，是把「先合成再微分」換成「先微分再相乘」的那條 ch05 便道，加上固定從輸出端走回來的紀律。

真實網路跟這個玩具的差別只有兩個。第一是規模：x 不是一個數，是上億個參數；每級「倍率」不是一個數，是一整箱偏導數——把倍率推廣到矩陣的那門手藝叫矩陣微積分，本書不進門，延伸閱讀有地圖。第二是方向是精打細算過的：從輸出端往回乘，每個中間量的比率只需要算一次、供下游所有參數共用，整箱億維梯度的成本跟前向跑一遍同一個量級——倒著記帳不是品味，是演算法上的勝負手（本書講到直覺為止）。

最後把歷史的帳記對（ch06 排過，這裡收口）：「往坡下走」是柯西（Cauchy）1847 年的方法；反向傳播在數學上就是連鎖規則的反向應用，1986 年 Rumelhart–Hinton–Williams 的 Nature 論文使它**普及**——普及，不是發明，反向模式的自動微分更早就有人做出來。所以這句話可以放心說死：**現代 AI 的引擎室裡是 ch05 與 ch06**。費馬的平頂、柯西的下坡、萊布尼茲的比率層疊——沒有新魔法，只是三百年的舊觀察換上了 GPU。

## 控制迴圈：PID——誤差的三種時態

恆溫器怎麼把室溫釘在 22 度？巡航定速怎麼把車速釘在 100？K8s 的 HPA 怎麼把 CPU 使用率釘在 60%？同一個問題形狀：有個你控制得了的輸出（加熱功率、油門、副本數），有個你想釘住的量測值，中間隔著一個會亂動的世界。

定義誤差 e(t) ＝ 目標值 − 量測值。工業界統治級的答案叫 **PID 控制器**（PID controller），輸出只有一行：

```text
u(t) = Kp·e(t) + Ki·∫₀ᵗ e(s) ds + Kd·e′(t)
       現在的誤差    過去的累積        未來的趨勢
```

P、I、D 三個字母不是神祕縮寫：字面上就是比例（proportional）、積分（integral）、微分（derivative）——ch07 與 ch04 的名字直接焊在型號上。三項是誤差的三種時態：

- **P 看現在**：誤差多大就用多大力修——差 3 度開大火，差 0.5 度開小火。簡單有效，但有個結構性缺陷：只靠 P，多半留下一個永遠修不掉的固定偏差（穩態誤差，steady-state error）。為什麼，推演第 2 題用紙筆推給你看。
- **I 看過去**：把開機以來的誤差整段積起來。只要誤差持續偏同一邊，這筆帳就持續長大、輸出持續加碼，直到誤差被壓回 0 為止；而且歸零之後，「維持輸出所需的力道」被積分整筆記在帳上——它就是 ch08 的累積函數。消滅穩態誤差的是它；代價是它有慣性，會衝過頭。
- **D 看未來**：誤差的變化率。誤差正在快速縮小？先收力，免得過衝——D 是煞車。代價：微分會放大高頻雜訊（ch04 取樣窗口問題的原班人馬），推演第 3 題算給你看。

你技術棧裡最有名的「恆溫器」是 K8s 的 HPA，它的演算法真的只有一行（2026-06 查證 Kubernetes 官方文件）：

```text
desiredReplicas = ⌈currentReplicas × (currentMetric / desiredMetric)⌉
```

誠實拆帳：這是一顆**純 P 控制器**——只看現值比例，沒有 I 項、沒有 D 項；比值落在 1.0 的 ±0.1 容差內就不動作（你認得這個結構：ε 容差，ch03）。所以 HPA 教科書級地展示了 P 控制的脾氣：步距太猛，副本數就在高低水位之間 flapping——ch06 說過，這就是學習率太大的震盪，同一條數學；它靠縮容穩定窗口這類機制壓震盪，而不是靠 D 項（2026-06 查證，細節以官方文件為準）。控制理論是一整門學科，本書到此為止——但 P、I、D 三個字母你從此不會再看錯：那是微積分的三個核心動作，焊在每一台恆溫器、每一台無人機、每一支 3D 印表機的噴頭裡。

## 機率即面積：你的 P99 每天都在積分

「P99 延遲 230 ms」的正式語意是：隨機抽一個請求，延遲不超過 230 ms 的機率是 0.99。要讓這句話有意義，你需要一個分布——而連續分布的語言，是面積。

連續隨機變數的規矩：給一條機率密度函數（probability density function, pdf），曲線本身不是機率，**曲線下的面積才是**——落在 [a, b] 的機率＝密度曲線在 [a, b] 上的積分，整條曲線下的總面積恰好是 1。最有名的密度是標準常態分布（standard normal distribution）的鐘形曲線：

φ(x) = e^(−x²/2)/√(2π)

（分母那個 √(2π) 唯一的工作是讓總面積恰好等於 1；這個歸一化常數本身是個著名的積分結果，本書不推。）

接著是本節真正的橋。把面積從左端一路累積過來，定義「值 ≤ x 的機率」，這叫累積分布函數（cumulative distribution function, CDF）。看清楚它的構造：**CDF 就是 ch08 的累積函數 A(x)，一字不改**。於是 FTC 自動上工：密度是 CDF 的導數、CDF 是密度的積分——機率論的 pdf／CDF，跟你監控系統的 rate／counter 是同一對搭檔，連互逆關係都是同一條定理。

那 P99 是什麼？是 **CDF 的反查**：找出讓 CDF 達到 0.99 的那個位置。你的監控系統每天在做的事：histogram 把延遲樣本裝進桶（bucket）——這就是把密度曲線切成粗矩形（ch07 的切片）；`histogram_quantile(0.99, …)` 在桶邊界之間做線性內插、反查出 P99（2026-06 查證 Prometheus 文件）——在一條粗糙版的 CDF 上倒著讀。所以這句話是字面意義的：**你的延遲百分位數每天都在積分**——先把樣本積成 CDF，再反查它。順帶領一個監控冷知識的數學解釋：兩個儀表板的 P99 對不上，多半是桶切法與窗口不同——兩條粗糙 CDF 各自內插出各自的答案，誰都沒「錯」，它們本來就都是估計。

常態分布還附贈一組你背過的數字：**68-95-99.7 法則**——距平均 ±1σ、±2σ、±3σ 的面積分別約 68.27%、95.45%、99.73%。第一個數字哪來的？它就是 ∫[−1,1] φ(x) dx。麻煩在於 φ 的反導數可以證明不是初等函數（ch08 提過 e^(−x²) 那一族）——FTC 的快取在這裡 miss，乖乖退回 ch07 的黎曼和。本章第二個 worked example：四條矩形，徒手摸這個 68%。

```
 φ(x)
0.4 ┤         * * *
    │        ┌──┬──┐
0.3 ┤     ┌──┤  │  ├──┐
    │     │  │  │  │  │
0.2 ┤   * │  │  │  │  │ *
    │  *  │  │  │  │  │  *
0.1 ┤ *   │  │  │  │  │   *
    │*    │  │  │  │  │    *
    └─────┴──┴──┴──┴──┴─────── x
         −1     0     1
```

```text
切法：[−1, 1] 切 4 條，每條寬 0.5，矩形高度取中點的 φ 值（中點法）：

  φ(0.25) = e^(−0.25²/2)/√(2π) = e^(−0.03125)/2.50663
          = 0.96923/2.50663 = 0.38667     ← 內側兩條（中點 ±0.25）
  φ(0.75) = e^(−0.75²/2)/√(2π) = e^(−0.28125)/2.50663
          = 0.75484/2.50663 = 0.30114     ← 外側兩條（中點 ±0.75）

面積 ≈ 0.5 × (0.30114 + 0.38667 + 0.38667 + 0.30114)
     = 0.5 × 1.37562
     = 0.68781
```

精確值是 0.68269——統計課那個 68.27% 的出處。四條矩形、實際只算兩個 φ 值，誤差 0.0051，不到一個百分點。想更進一步，推演第 4 題用梯形法重算一次，結尾有一個漂亮的會師等你。「機率是面積」不是修辭：你剛剛用 ch07 的方法、ch02 的精神，徒手算出了教科書要你背的那個數。

## 數值現實：在離散機器上模擬連續數學

ch04 結尾欠你一筆帳，現在結。那個 Python 實驗：算 f(x) = x² 在 x = 3 的割線斜率，h 從 1 一路縮到 10⁻¹²。數學說誤差該一路縮小（斜率恰是 6 + h，誤差就是 h）；機器給你看的是——誤差先乖乖跟著 h 掉，在 h ≈ 10⁻⁸ 觸底（約 3.6×10⁻⁸），然後反轉惡化，h = 10⁻¹² 時暴漲回 5.3×10⁻⁴，比 h = 10⁻⁴ 的時候還爛。

ch04 給了定性原因（災難性消去），這裡補定量的：兩股誤差在拔河。

```text
總誤差 ≈ 截斷誤差 ＋ 捨入誤差
          ～h 階      ～ε/h 階

截斷誤差：數學自己的鍋——割線不是切線，差商比真導數多出 ～h 階的尾巴。
捨入誤差：機器的鍋——f(x+h) 與 f(x) 是兩個幾乎相等的數，相減時有效位數
          被吃掉，殘渣再除以 h、被放大 1/h 倍。ε 是機器精度，
          雙精度 ε ≈ 2.2×10⁻¹⁶。
```

h 大，截斷誤差吃掉你；h 小，捨入誤差吃掉你。兩股力量平衡的位置在 **√ε 量級**——雙精度下約 10⁻⁸（對 e(h) = h/2 + 2ε/h 求最小值即得；推導見延伸閱讀）。回頭看 ch04 那張表：觸底恰好在 10⁻⁸。那不是 bug，是這條公式的肖像。聰明一點的公式買得到更多：中央差分 (f(x+h) − f(x−h))/2h 的截斷誤差是 h² 階——兩側對稱，h 階的尾巴自己抵銷——最適步長變成 **ε^(1/3) 量級**（約 6×10⁻⁶），觸底時的誤差也更小。換公式，比硬調步長值錢。

積分與微分方程那一側，同一齣戲換個劇本。ch07 的實驗：矩形法誤差 ～1/n、梯形法 ～1/n²——同樣的計算預算，好公式多換好幾位精度。ch10 的 Euler 法：誤差 ～h（步長 0.1 算 e 差約 0.12，步長 0.01 差約 0.013），步長砍十倍、誤差砍十倍、成本漲十倍；而且步數一多，捨入開始累積，長時間模擬還會悄悄漏帳（彈簧振子的能量漂移）。每個數值方法都活在同一個三角裡：**精度、成本、穩定性——挑兩個**。

把視角拉高。微積分的定義全部蓋在「h 要多小有多小、n 要多大有多大」上（ch03 的挑戰賽）；浮點世界裡 h 有地板、n 有預算，理想假設必然漏水。但這不是「數學沒用」——恰恰相反：是數學告訴你地板在哪裡（√ε、ε^(1/3)）、誤差以什麼速率縮小（1/n vs 1/n²）、什麼時候該換公式。**在離散機器上模擬連續數學，是一門誠實的工藝**：它從不假裝誤差不存在，只承諾把誤差控制在你指定的容差內。你該認得這個句型——它就是 ε 的句型（ch03），整本書的起點。

## 你已經在裡面了

收尾不喊口號，盤點事實。本章每一個系統的核心迴圈裡，焊的都是同一組零件：

- 會變的量，問「變多快」——導數（ch04）：`rate()`、D 項、梯度。
- 會變的量，問「累積了多少」——積分（ch07）：`increase()`、I 項、CDF。
- 兩個問題互為反操作——FTC（ch08）：counter／rate、pdf／CDF。
- 用「怎麼變」推「會變成怎樣」——微分方程（ch10）：tick 迴圈、容量外插。
- 把變化拆成頻率——傅立葉（ch11）：頻譜、季節性分解。
- 對無限的每一次操作都驗收誤差——極限與級數（ch03、ch09、ch12）：eps 比較、浮點求和、步長地板。

微積分是「會變的量」的通用介面：只要一個量隨另一個量變，這套語言就接管描述權——而你的職業生涯整天在跟會變的量打交道。所以這本書從第一章講到現在的那句話，最後一次、用最強的形式說：你不是「將來有一天會用到」微積分，**你已經在裡面了**。差別只在：以前你是這門語言的母語使用者——說得流利、不識文法；十四章讀完，你識字了。ch16 做最後一件事：把整個故事，用你自己的話，完整講一遍。

| 「backprop 是深度學習發明的新數學」 | 把引擎室神祕化，論文裡的梯度符號變成黑魔法；其實機制是 ch05 的連鎖規則，1986 年普及的是記帳法與規模，不是數學 | 你說不出 backprop 跟連鎖規則的差別在哪（答案：數學上沒有差別，差別在記帳方向與規模） |

## 紙上推演

**題 1 [15 分鐘] ★** — 挑一個你真實維運過的指標（QPS、佇列深度、記憶體用量、錯誤數……自選），用本章的語言指認：(a) 它本身是速率型還是累積型？(b) 它的導數是哪個現成查詢？(c) 它的積分是哪個現成查詢？(d) 對每個答案做單位對帳（每秒？個？個×秒？）。

**題 2 [20 分鐘] ★★** — 巡航定速上坡，維持等速需要固定油門 u*。(a) 純 P 控制 u = Kp·e 之下，車速穩定後誤差 e 是多少？為什麼不可能是 0？(b) 加上 I 項：u = Kp·e + Ki·A(t)，A(t) = ∫₀ᵗ e(s) ds。假設系統穩定在某個常數誤差 eₛ ≠ 0，用 A(t) 的行為推出矛盾。(c) 誤差歸零之後，維持油門的力道由誰出？用 ch08 的累積函數語言回答。

**題 3 [15 分鐘] ★★** — 量測值疊上高頻雜訊 n(t) = δ·sin(ωt)，δ 很小、ω 很大。(a) 它對 P 項的污染多大？(b) 對 D 項呢——手算 n′(t) 的振幅？(c) 換成離散差商 (e(t) − e(t−h))/h，雜訊最壞貢獻多大？把結論接回 ch04：這跟 `rate()` 窗口縮小曲線變毛、跟割線實驗 h 觸底反彈，是不是同一件事？

**題 4 [20 分鐘] ★★** — 把 68% 再算一次，這次用梯形法：節點 0、±0.5、±1，密度值 φ(0) = 0.39894、φ(0.5) = 0.35207、φ(1) = 0.24197。(a) 算出梯形和。(b) 跟正文的中點法 0.68781、精確值 0.68269 並排：誰高估、誰低估、為什麼？提示：φ 在 [−1, 1] 上是凹的——而它的反曲點恰好就在 ±1。(c) 加碼：算 (2×中點 ＋ 梯形)/3，看看你撞見了什麼。

### 推演解答

**題 1**（以 `http_requests_total` 示範，你的指標照樣造句）　(a) 累積型（counter）：只增不減，數學身分是累積函數 A(t)（ch08）。(b) 導數＝`rate(http_requests_total[5m])`，單位「個/秒」——差商的工程近似（ch04）。(c) counter 本身就已經是積分；現成的區間版是 `increase(...[1h])`，單位「個」——FTC 第二張臉，端點相減＝整段累積。若你挑的是速率型（gauge，如 CPU 使用率）：導數得自己造（`deriv()` 或差商），積分是 `avg_over_time × 窗長` 這類「面積」查詢。(d) 單位對帳是防呆關鍵：微分除以秒、積分乘上秒，量綱對不上就是指認錯了。常見錯路：把 gauge 餵給 `rate()`——對非單調的量取「reset 補償過的變化率」，語意整個跑掉。

**題 2**　(a) 穩定 ⇒ 油門是常數 Kp·eₛ；維持等速 ⇒ 這個常數必須恰好等於 u*。所以 eₛ = u*/Kp ≠ 0：**P 控制器要靠殘留誤差才擠得出持續輸出**——誤差一歸零、輸出就歸零、車就減速，誤差又被拉回來。調大 Kp 只能把 eₛ 壓小，壓不到 0；而 Kp 太大就是 ch06 的學習率過大，換來震盪。(b) 設誤差穩定在 eₛ ≠ 0，則 A(t) = ∫₀ᵗ e ≈ eₛ·t 線性長大、沒有上界——ch08 說過，累積函數對常數輸入的反應就是一條直線——於是 u = Kp·eₛ + Ki·eₛ·t 也無上界，油門踩穿地板，與「系統穩定」矛盾。穩定狀態只剩一種可能：eₛ = 0。**常數誤差騙得過比例，騙不過積分**：你欠的每一秒它都記著。(c) A′ = e（FTC 第一張臉）：誤差歸零＝累積函數停止變動，但它**停在累積出來的高度上**——上坡需要的力道 u*/Ki 被 A 整筆記在帳上，由 I 項持續出力。P 看不見的歷史，I 用面積記住了。（帳外一筆：這份記憶也有病——目標突然大改時，積分會先把舊帳放完才回頭，這叫積分飽和，工程上要另外治，本書不展開。）

**題 3**　(a) P 項看到的污染振幅就是 δ——小，無害。(b) n′(t) = δω·cos(ωt)，振幅 δω：**微分把雜訊放大 ω 倍**。δ = 0.01、ω = 1000，污染變成 10，訊號可能整個被淹掉；頻率越高放大越兇，而雜訊偏偏住在高頻。(c) 離散版：雜訊對分子的最壞貢獻是 2δ（兩端各抖一次、方向相反），除以 h 得 2δ/h——窗口越小，放大倍率越大。三件事是同一條數學：D 項怕雜訊、`rate()` 窗口縮小曲線變毛（ch04 開頭）、割線實驗 h 太小誤差反彈（ch04 實驗、本章 √ε）——全是「差商的分母在放大分子裡的污染」。解法也同源：低通濾波、拉大窗口、換更穩的差分公式——全是用「反應變鈍」換「輸出變穩」。

**題 4**　(a) 梯形法，兩端點各算半權重：

```text
T = 0.5 × (φ(−1)/2 + φ(−0.5) + φ(0) + φ(0.5) + φ(1)/2)
  = 0.5 × (0.12099 + 0.35207 + 0.39894 + 0.35207 + 0.12099)
  = 0.5 × 1.34506
  = 0.67253
```

(b) 並排：中點法 0.68781 高估（＋0.0051），梯形法 0.67253 低估（−0.0102），精確值 0.68269 被夾在中間。方向不是巧合：φ 在 (−1, 1) 內是凹的（彎口朝下）——凹弧上，中點矩形的頂在中點處切住曲線、兩側多吃進面積，所以高估；梯形的弦掛在曲線下方，所以低估。而 ±1 恰好是 φ 的反曲點——鐘形曲線由凹轉凸的位置就在 ±1σ：σ 不是隨便畫的刻度，是曲線自己的幾何關節（也是這題切在 [−1, 1] 特別乾淨的原因）。(c) (2×0.68781 + 0.67253)/3 = 2.04815/3 = 0.68272，跟精確值 0.68269 只差約 3×10⁻⁵——你剛剛重新發明了辛普森法（Simpson's rule）：把高估方與低估方按 2:1 調和，凹凸造成的主誤差項恰好對消，四條切片換到五位準確。ch07 的承諾「好方法多換好幾位精度」，這次是你親手兌現的。
