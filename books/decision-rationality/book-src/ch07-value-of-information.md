# ch07 — 資訊的價值：要不要再驗一次

> **本章解決什麼問題**：前兩章把「不確定」變成一個可更新的數（ch05 後驗、ch06 先驗從哪來），然後接上效用矩陣算出最適行動。但真實決策還缺一步：在按下去之前，你常常還能**多蒐集一點證據**——再驗一次、多跑一輪壓測、多放一批流量。問題是證據不免費。本章把「要不要再蒐集資訊」算成一個期望值問題，給出資訊的價值（value of information）的精確意義，並引出全書最乾淨的一個洞見：**一條不會改變你決策的資訊，它的價值精確等於零。** 你在 Part II 的最後一章，下一章起轉向「人實際怎麼選」。

## 從你已知的出發

你每天都在替資訊定價，只是沒寫成式子。

要上線一個有風險的改動。你已經跑過一輪 CI、過了 code review，心裡的把握大概七成。現在你站在一個岔路口：**直接全面上線**，還是**先放 1% 流量、看半小時的指標、再決定要不要全面上線或回滾**？

漸進式放量不是免費的。它要多花你半小時盯盤、佔一個工程師的注意力、把上線時程往後推、還可能因為「只有 1% 流量」而暴露不出真正的問題。所以你心裡其實在算一筆帳——這半小時的漸進式放量，**值不值**它的成本？

而你判斷「值不值」的真正依據，多數人講不清楚，但你做對了：**這批放量，有沒有可能讓我換一個動作？** 如果放量結果無論好壞，你都會繼續全面上線（因為你已經決定「就算有點小問題也先上、邊上邊修」），那這半小時的漸進式放量在**決策上**一文不值——你只是在買心安，不是在買資訊。反過來，如果放量有相當機率會讓你「看到不對勁就回滾」，那它就值錢，因為它可能讓你避開一場全面上線事故。

```text
   漸進式放量的決策價值 = 它「可能讓你換一個更好的動作」的程度

   無論放量結果如何，你都會全面上線  →  決策價值 = 0（只剩心安價值）
   放量有機會讓你改成回滾             →  決策價值 > 0（買到了避開事故的機會）
```

把這個直覺擦亮、寫成式子、再證明它精確成立——就是本章。你會發現你維運生涯裡那句口頭禪「這個監控加了也不會改變我們做什麼，純粹好看」，其實是一句嚴格的決策論命題：**不改變行動的資訊，期望價值是零。** 這不是修辭，是可以算出來、而且算出來真的是 0 的。

順帶說一個你早就有的更深直覺：資訊蒐集**永遠不會讓你更糟**（在期望意義上）。多知道一點，最差也就是「這資訊沒用、我照原樣決策」，不會因為多看了一眼反而做出更壞的選擇——前提是你**理性地用**這個資訊。這條「資訊價值 ≥ 0」後面會證，它和你「多測一輪總沒壞處」的工程直覺是同一件事。

## 資訊值錢，若且唯若它可能讓你換動作

先把概念釘死，再上數字。

決策理論對「資訊的價值」有一個冷酷而精確的定義，它和「資訊量大不大」「資訊有不有趣」**完全無關**，只和一件事有關：**這條資訊會不會改變你最後採取的行動。**

為什麼？回想 ch05 的決策結構：你手上有一個後驗信念，把它乘上效用矩陣，算出一個最適行動 a*，然後執行 a*，拿到一個期望效用。現在有人要賣你一條資訊。這條資訊能幫你的唯一管道是：**它更新你的後驗，更新後的後驗可能指向一個不同的、更好的行動。** 如果它更新完後驗、你算一遍、最適行動還是同一個 a*——那你做的事一模一樣，拿到的期望效用一模一樣，這條資訊對你的決策**毫無貢獻**。

```text
資訊改變決策的唯一路徑：

   資訊 D  →  更新後驗 P(θ|D)  →  重算 a* = argmax_a Σ P(θ|D)·u(a,θ)
                                         │
                          ┌──────────────┴──────────────┐
                          ▼                             ▼
              a* 變了（指向更好的行動）        a* 沒變（還是原來那個）
                          │                             │
                  資訊有正價值                    資訊價值 = 0
              （買到了「換更好動作」的機會）   （你做的事完全一樣）
```

這就是本章的軸心，我認為它是整本書最違反工程師直覺的一頁——因為我們的本能是「資訊越多越好」「能測就測」。但決策論告訴你：**資訊的價值不在資訊本身，在它和你的行動集之間的關係。** 同一條資訊，對「行動會因它而改變」的人值錢，對「無論如何都做同一件事」的人一文不值。價值不是資訊的屬性，是「資訊 × 決策處境」的屬性。

把它翻成你的語言：一個監控指標的價值，不取決於它多精確、多即時，而取決於**它報警時你會不會因此做出不同的動作**。如果某條 dashboard 線無論漲跌你都不會改變任何部署決定，它的決策價值是零——不管它畫得多漂亮、取樣多密。（它可能還有別的價值，比如事後 debug 的回溯價值、或團隊的心理安定，但那是另一回事，不是這裡定義的「決策資訊價值」。）

## 兩把尺：完全資訊（EVPI）與樣本資訊（EVSI）

「資訊的價值」要量化，得先問「哪種資訊」。決策論有兩把尺，一把量理論上限，一把量真實證據。

**第一把：完全資訊期望價值（expected value of perfect information, EVPI）。** 想像有個神諭（clairvoyant），在你行動前**準確無誤地**告訴你世界的真實狀態（到底有病還是健康、這次部署到底會不會出事）。有了它，你每次都能挑那個狀態下最好的行動，再也不會猜錯。EVPI 就是「有神諭」與「沒神諭、只能靠現有信念決策」之間，期望效用的差額——也就是你**最多**願意為「消除這個不確定」付多少。

```text
EVPI = (有完全資訊時的期望效用)  −  (沒有資訊、用現有信念的最適期望效用)
     = EV|PI − EV
```

EVPI 是**上限**：再厲害的檢查、再多輪的壓測，都不可能比「直接知道答案」更值錢。它回答一個很實用的問題——「這個不確定，最理想情況下值得我花多少去消除？」如果 EVPI 算出來只有 3 塊錢，那你就別花 10 塊去買任何檢查了，再準的檢查也不划算。這個概念由 Ronald Howard 在 1966 年的論文〈Information Value Theory〉（*IEEE Transactions on Systems Science and Cybernetics*, SSC-2(1):22–26）形式化（2026-06 查證，來源見延伸閱讀）。Howard 的關鍵貢獻是：把資訊的價值同時和**機率結構**與**後果（效用）**綁在一起算——這正是它跟 Shannon 資訊理論的分野，後者只量機率結構、不管後果。

**第二把：樣本資訊期望價值（expected value of sample information, EVSI）。** 真實世界沒有神諭，你拿到的是**不完美**的證據：一次會出錯的檢查、一輪只覆蓋部分情境的壓測、一批只有 1% 流量的漸進式放量。EVSI 量的是「做這一次不完美觀測」帶來的期望效用增益。它由 Howard Raiffa 與 Robert Schlaifer 在 1961 年的《Applied Statistical Decision Theory》推廣（2026-06 查證）。

兩者的關係很直觀，也是一個好用的自我複核：

```text
   0  ≤  EVSI（一次不完美觀測）  ≤  EVPI（神諭級的完美資訊）
   └┬┘                            └┬┘
    │                              └ 上限：完美資訊也不可能更值錢
    └ 下限：資訊在期望意義上永不為負
```

任何真實檢查的價值，夾在 0 和 EVPI 之間。算完 EVSI 如果發現它比 EVPI 還大，你一定哪裡算錯了——這是寫章時我自己用的檢查，等下會看到脊椎的數字剛好印證（EVSI≈14.0 < EVPI=15.0）。

## 為什麼資訊價值永遠 ≥ 0：一個你直覺早就懂的證明

工程師會問：憑什麼說「多知道一點不會更糟」？萬一資訊把我**誤導**了呢？

關鍵在「理性地用」這四個字。理性決策者拿到資訊後，會重算最適行動——而他**永遠保留「無視這條資訊、照原計畫做」的選項**。所以最差的情況是：資訊沒用，他照原樣決策，期望效用不變（增益 0）。只要資訊偶爾能指向更好的行動，增益就是正的。一個「永遠有保底選項、偶爾能更好」的東西，期望值不可能是負的。

```text
   拿到資訊後，你總是可以選擇「忽略它、做原本要做的 a*」
   ⇒ 有資訊時的最適期望效用  ≥  無資訊時的期望效用
   ⇒ 增益（資訊價值）≥ 0

   注意「期望意義上」這五個字：某一次的某個結果可能讓你
   事後懊悔（資訊騙了你、害你改了動作結果更糟）；但「平均
   over 所有可能結果」一定 ≥ 0。資訊價值是事前的期望、不是
   事後的保證。
```

這是**工程師的嚴謹**等級的論證（每一步你都能口頭說出理由），不是形式證明——但它對。嚴格版（資訊在期望效用意義下的次模性與非負性）見 Howard 1966 與 Raiffa–Schlaifer 1961，本章不展開。

擋一個常見誤解：「資訊價值 ≥ 0」說的是**決策價值**，不是「蒐集成本」。資訊本身的期望價值非負，但蒐集它**要花錢**（時間、流量、注意力）。淨值 = 資訊價值 − 蒐集成本，這個**可以是負的**。所以「該不該再驗一次」的真正判據是：**EVSI（這次檢查的決策價值）有沒有大於做這次檢查的成本？** 資訊價值非負，不代表蒐集永遠划算——它只代表「如果免費，多測一輪不會虧」。

## worked example：脊椎檢查線——要不要做第二次檢查？

把脊椎的檢查線請回來。設定和 ch05 完全一致（盛行率 1%、敏感度 99%、偽陽 5%），你已經驗出第一次陽性，後驗是 16.7%（=1/6，機率機制見《馴服隨機》ch05，本書刻意用 1% 盛行率讓後驗落在「值得決策」的區間）。效用矩陣也沿用全書基準：

```text
脊椎檢查的效用矩陣（全書統一）

                  狀態：有病          狀態：健康
   行動：治療        −10               −20
   行動：不治療      −100               0

   讀法：治療有病＝小代價（−10，治對了但有副作用）；
        治療健康＝白挨副作用（−20）；
        不治療有病＝漏掉大病（−100，最慘）；
        不治療健康＝什麼都沒發生（0，最好）。
```

**第 0 步：沒有第二次檢查時，最適行動是什麼？** 這是 ch05 算過的，先複算一遍當基準（後驗 p=1/6）：

```text
   EU(治療 | 後驗 1/6)   = (1/6)·(−10) + (5/6)·(−20)
                        = −1.6667 − 16.6667 = −18.3333

   EU(不治療 | 後驗 1/6) = (1/6)·(−100) + (5/6)·(0)
                        = −16.6667 + 0     = −16.6667

   −16.6667 > −18.3333  ⇒  最適行動 = 不治療，期望效用 = −16.67
```

意外吧——驗出陽性，最適行動竟是**不治療**。因為後驗只有 1/6，多數陽性者其實健康，「治療健康者白挨副作用」的累積代價壓過了「漏掉那 1/6 病人」。這是 ch05 的結論，本章拿它當「沒有新資訊」的基準線：**期望效用 = −16.67。**

**第 1 步：做第二次（獨立）檢查，它有兩種結果。** 第二次檢查同樣 sens=99%、偽陽 5%。把「16.7% 後驗」當作第二次檢查的**新先驗**（這正是 ch06 講的「後驗變下一次先驗」的貝氏循環），算第二次檢查各自結果的機率與更新後的後驗：

```text
   第二次檢查的結果機率（先驗 = 1/6）：
     P(2nd 陽性) = 0.99·(1/6) + 0.05·(5/6) = 0.1650 + 0.0417 = 0.2067
     P(2nd 陰性) = 1 − 0.2067 = 0.7933

   第二次陽性後，更新後驗：
     P(病|2nd 陽) = 0.99·(1/6) / 0.2067 = 0.1650 / 0.2067 ≈ 0.7984  ← 兩次陽性，疑慮大增
   第二次陰性後，更新後驗：
     P(病|2nd 陰) = 0.01·(1/6) / 0.7933 = 0.001667 / 0.7933 ≈ 0.0021 ← 一陽一陰，幾乎排除
```

自我複核：兩個分支機率 0.2067 + 0.7933 = 1.0000 ✓。兩個分支的後驗也合理——兩次陽性把疑慮推到 ~80%，一陽一陰把疑慮幾乎清零到 ~0.2%。

**第 2 步：每個分支各自的最適行動。** 在新後驗下重算 EU，挑大的：

```text
   分支 A：第二次也陽性（機率 0.2067，後驗 0.7984）
     EU(治療)   = 0.7984·(−10) + 0.2016·(−20) = −7.984 − 4.032 = −12.02
     EU(不治療) = 0.7984·(−100) + 0.2016·(0)   = −79.84
     ⇒ 最適 = 治療，期望效用 = −12.02      ← 翻轉了！原本不治療，現在治療

   分支 B：第二次陰性（機率 0.7933，後驗 0.0021）
     EU(治療)   = 0.0021·(−10) + 0.9979·(−20) = −0.021 − 19.958 = −19.98
     EU(不治療) = 0.0021·(−100) + 0.9979·(0)  = −0.21
     ⇒ 最適 = 不治療，期望效用 = −0.21
```

注意分支 A：第二次檢查讓最適行動從「不治療」**翻轉**成「治療」。這就是資訊值錢的源頭——它**可能讓你換一個動作**。

**第 3 步：把兩個分支加權，得到「做第二次檢查」的期望效用。**

```text
   EU(做第二次檢查) = P(2nd陽)·(分支A最適) + P(2nd陰)·(分支B最適)
                   = 0.2067·(−12.02) + 0.7933·(−0.21)
                   = −2.485 + (−0.167)
                   = −2.65
```

**第 4 步：EVSI = 差額。**

```text
   EVSI = EU(做第二次檢查) − EU(不做、沿用基準)
        = (−2.65) − (−16.67)
        = +14.02

   ⇒ 第二次檢查的決策價值 ≈ 14.0（單位同效用）。為正，因為它
     有 20.67% 的機率（驗出第二次陽性）把你的行動從「不治療」
     翻成「治療」，避開漏診那 0.80 機率病人的 −100 慘格。
```

複核 EVSI ≤ EVPI：等下算 EVPI = 15.0（見推演題 2），而 14.0 < 15.0 ✓——第二次檢查很接近完美資訊的上限，因為它真的相當準。

結論：在這組數字下，**只要第二次檢查的成本低於 ~14（同單位），就值得做。** 這就是「要不要再驗一次」的精確答案——不是憑感覺，是算出來的。

## worked example：同一條資訊，價值精確為零

上面 EVSI = 14 是因為資訊**翻轉了行動**。現在示範本章的招牌洞見：把效用矩陣換一組，讓行動**無論第二次結果如何都不會翻轉**——你會看到 EVSI **精確等於 0**，不是「接近 0」，是代數上嚴格的 0。

換一個情境：假設這個病「漏掉的代價」其實沒那麼可怕——比如它是一種拖一陣子再治也無妨的慢性病，漏診代價只有 −15（而不是 −100）。其餘格子不變：

```text
變體效用矩陣（漏診代價從 −100 降到 −15）

                  狀態：有病          狀態：健康
   行動：治療        −10               −20
   行動：不治療       −15                0
```

先看沒有第二次檢查時（後驗 1/6）：

```text
   EU(治療)   = (1/6)·(−10) + (5/6)·(−20) = −1.67 − 16.67 = −18.33
   EU(不治療) = (1/6)·(−15) + (5/6)·(0)   = −2.50
   ⇒ 最適 = 不治療，期望效用 = −2.50
```

現在做第二次檢查，看兩個分支的最適行動（分支機率與後驗和前面一樣，0.7984 與 0.0021）：

```text
   分支 A（第二次陽性，後驗 0.7984）：
     EU(治療)   = 0.7984·(−10) + 0.2016·(−20) = −12.02
     EU(不治療) = 0.7984·(−15) + 0.2016·(0)   = −11.98
     ⇒ 最適 = 不治療（−11.98 > −12.02），剛好沒翻轉！

   分支 B（第二次陰性，後驗 0.0021）：
     EU(治療)   = 0.0021·(−10) + 0.9979·(−20) = −19.98
     EU(不治療) = 0.0021·(−15) + 0.9979·(0)   = −0.03
     ⇒ 最適 = 不治療
```

兩個分支的最適行動都是**不治療**——和「沒有第二次檢查」時一模一樣。資訊來了，你卻什麼都不會改。算 EVSI：

```text
   EU(做第二次檢查) = 0.2067·(−11.98) + 0.7933·(−0.03)
                   = −2.476 + (−0.024) = −2.50

   EVSI = (−2.50) − (−2.50) = 0.000000   ← 精確的零
```

為什麼**精確**是零，不是「碰巧很接近」？因為當每個分支的行動都是同一個（不治療），那「做第二次檢查的期望效用」其實就是「不治療這個固定行動的期望效用，先按第二次結果分組再加權平均」——而按子集分組再加權平均，數學上就是把它**還原回**不分組的整體平均（這是條件期望的全期望公式／law of total expectation，機率機制見《馴服隨機》談期望值）。換句話說：

```text
   行動不翻轉  ⇒  你只是把同一個行動的效用「先拆組、再平均回去」
              ⇒  和不拆組的基準期望效用恆等
              ⇒  EVSI 代數上 ≡ 0，與檢查多準、結果怎麼分布都無關
```

這是我認為本章最該讓你「啊」一聲的地方，請用自己的話把它記住：**只要一條資訊不可能讓你換動作，它的決策價值就不是「很小」，而是恆等於零——再精確、再昂貴、再費工的檢查也一樣。** 一個能驗出八成把握的高級檢查，和一條毫無資訊量的雜訊，在「不會改變你動作」的情境下，決策價值完全相等：都是 0。

回到漸進式放量：如果你已經鐵了心「就算放量爆了也先全面上線、邊上邊修」，那這批漸進式放量——不管它多精密、覆蓋多少情境——決策價值精確是零。你不是在蒐集資訊，你是在做一件讓自己心安、但不影響任何動作的儀式。值不值得做這個儀式是另一回事（團隊安心也有價值），但別騙自己說那是「為了更好的決策」。

## 直覺的陷阱

決策正是人類直覺與規範理論最常打架的領域，本章的陷阱尤其反直覺，因為它和工程師的職業本能正面衝突。

**陷阱一（具名）：「資訊越多越好。」** 這是本章要擋掉的頭號錯誤直覺。

```text
   錯誤直覺在哪一步把你帶下溝：

   你的本能：「多測一輪、多加一個監控、多驗一次，總比沒有好。」
   → 第一步就錯了：你把「資訊」當成了無條件的好東西。
   → 真相：資訊的決策價值，取決於它會不會改變你的行動。
           不改變行動的資訊，決策價值精確是零，而蒐集它有成本。
   → 於是「資訊越多越好」變成「持續為零價值的資訊付費」。
```

工程版的下場你很熟：dashboard 上一百條沒人會據以行動的線、每天響兩百次沒人會因它改變任何動作的告警（這正是 alert fatigue 的決策論根源——告警的決策價值是零，成本卻是注意力的持續流失）。

**陷阱二：把「資訊有趣」當「資訊有價值」。** 一條資訊可能很有趣、很反直覺、讓你「喔原來如此」——但如果它不改變你接下來做什麼，它的**決策**價值仍是零。求知慾和決策價值是兩回事；混淆它們，你會把大量時間花在「知道了很爽但用不上」的探測上。

| | 大眾版（錯誤直覺） | 正確版（本書守住的） |
|---|---|---|
| 資訊的價值來自 | 資訊本身（多、準、即時） | 資訊 × 決策處境（它會不會改變你的行動） |
| 不改變決策的資訊 | 「總有點價值吧」 | 決策價值**精確**為零（不是很小，是 0） |
| 「多測一輪」 | 「總沒壞處」 | 資訊價值 ≥ 0 但蒐集有成本；淨值可為負 |
| 資訊價值 ≥ 0 的意思 | 「每次蒐集都划算」 | 只說「免費時不虧」；要不要蒐集看 EVSI vs 成本 |
| 有趣的資訊 | 「有趣就值得查」 | 有趣 ≠ 有決策價值；不改變行動就是零 |

**怎麼自我察覺你掉進去了：** 當你想加一個監控、想再測一輪、想再查一個數據，先問自己一句殺人的話——**「如果這個結果出來是 X，我會做什麼？如果是 Y，我又會做什麼？」** 如果兩個答案一樣（無論結果如何我都做同一件事），停手：這條資訊的決策價值是零，你只是在買心安或滿足好奇。如果兩個答案不同，那它就值錢，值得算一下 EVSI 夠不夠付成本。這一問，是 EVSI 的口袋版，幾乎不花時間，卻能擋掉你大半「為測而測」的衝動。

## 紙上推演

全書純紙上推演，紙筆即可，不必碰任何程式。數值請自己手算一遍，別抄。

### 推演題

**第 1 題（★★，15 分鐘）** 不算任何數，用你自己的話論證：**一條無論結果如何都不會改變你決策的資訊，它的決策價值為什麼是零？** 要求你的論證能讓另一個工程師點頭，至少觸及兩點：(a) 資訊幫你的唯一管道是什麼；(b) 為什麼「行動不翻轉 ⇒ 期望效用恆等於基準」是代數上的等式、不是近似。然後舉一個你工作中「為了心安而蒐集、其實零決策價值」的監控指標或測試，並指出：如果要讓它變得有價值，你的決策邏輯得怎麼改？

**第 2 題（★★★，25 分鐘）** 用脊椎檢查線的**原始**效用矩陣（漏診 −100 那組）和後驗 1/6，算 **EVPI**（完全資訊的價值）。提示：神諭會準確告訴你「有病」或「健康」。分四步：(1) 沒有資訊時的最適期望效用（本章 worked example 第 0 步已給，−16.67）；(2) 如果神諭說「有病」，你會挑哪個行動、拿到多少效用？說「健康」呢？(3) 把這兩個「知道真相後的最佳效用」按真實機率（病 1/6、健康 5/6）加權，得到 EV|PI；(4) EVPI = EV|PI − (沒資訊的最適)。算完後回答：你算出的 EVPI 是否 ≥ 本章 worked example 的 EVSI≈14.0？為什麼一定該如此？

**第 3 題（★★，15 分鐘）** 本章 worked example 顯示：原始矩陣（漏診 −100）下第二次檢查 EVSI≈14（值得做），但漏診代價降到 −15 的變體下 EVSI=0（白做）。**同一個檢查、同樣準，價值卻從 14 掉到 0。** 用一句話解釋這個落差的根源在哪（在檢查身上，還是在別處？），然後把它翻成一個你維運的場景：描述一個「同一個監控，在情境 A 值得、在情境 B 完全多餘」的例子，並點出是什麼變了讓它從值得變多餘。

### 推演解答

**第 1 題：** 合格論證如下。

(a) 資訊幫你的**唯一**管道是：它更新你的後驗，更新後的後驗可能指向一個不同的、更好的行動。資訊不會憑空增加任何結果的效用（−100 的格子還是 −100），它能做的只有「讓你重新選一格」。所以如果它選出來的還是同一格，它什麼都沒做。

(b) 為什麼是**等式**而非近似：假設無論第二次結果是陽是陰，最適行動都是同一個（譬如都「不治療」）。那麼「做第二次檢查的期望效用」=「不治療這個固定行動的效用，先按第二次結果分組、再按結果機率加權平均」。但把一個量先分組再加權平均，等於不分組直接平均（全期望公式：E[X] = E[E[X|分組]]，機率機制見《馴服隨機》談期望值）。所以它**恆等於**「不治療在原後驗下的期望效用」，也就是沒有第二次檢查的基準值。兩者相減，代數上嚴格為 0——不依賴檢查多準、結果機率怎麼分布。常見錯路：以為「資訊量總有一點，所以價值總大於零一咪咪」——錯，決策價值看的是行動有沒有翻轉，和資訊量（資訊理論意義的 bits）是兩個正交的東西；第二次檢查的「資訊量」明明不小（它會大幅改變你的後驗信念），但只要這個信念改變不足以**跨過行動翻轉的門檻**，決策價值就是零。

你的零價值監控例子（如「一條無論漲跌都不會改變部署決定的 dashboard 線」）要讓它有價值，唯一的辦法是：**讓某個結果觸發一個不同的動作**——例如「這條線超過閾值就自動回滾」。一旦它能改變行動，它的 EVSI 就從零變正。沒有「不同的動作」掛在後面的資訊，再美也是零。

**第 2 題：** EVPI 計算。

```text
   (1) 沒資訊的最適期望效用（後驗 1/6）= −16.67（不治療）。

   (2) 神諭說「有病」→ 在「有病」這一列挑最好的：
         u(治療,病)=−10 vs u(不治療,病)=−100  ⇒ 挑治療，得 −10
       神諭說「健康」→ 在「健康」這一列挑最好的：
         u(治療,健康)=−20 vs u(不治療,健康)=0  ⇒ 挑不治療，得 0

   (3) EV|PI = P(病)·(知道有病時的最佳) + P(健康)·(知道健康時的最佳)
            = (1/6)·(−10) + (5/6)·(0)
            = −1.6667 + 0 = −1.6667

   (4) EVPI = EV|PI − (沒資訊的最適)
           = (−1.6667) − (−16.6667)
           = +15.00
```

EVPI = 15.0。它 ≥ worked example 的 EVSI≈14.0 ✓，**必然**如此：完全資訊（神諭，每次都猜對）是任何不完美檢查的上限，沒有任何真實檢查能比「直接知道答案」更值錢。第二次檢查的 14.0 很接近 15.0 的上限，因為這個檢查確實很準（敏感度 99%）——但它仍有偽陽/偽陰，所以差那 1.0。常見錯路：(i) 在第 (3) 步用「後驗 1/6」當機率去加權——錯，神諭場景下要用**真實的狀態機率**（病 1/6 是當前後驗對「有病」的信念，這裡正確；重點是別把「治療」的決策權重混進來）；(ii) 忘記減去基準，直接把 EV|PI=−1.67 當成 EVPI——EVPI 是**差額**，不是有資訊後的絕對效用。

**第 3 題：** 落差的根源**不在檢查身上**——同一個檢查、同樣準、同樣的後驗更新能力。根源在**效用矩陣**：原始矩陣裡「漏診 −100」這個慘格，讓「治療 vs 不治療」的最適行動對後驗敏感（後驗夠高就該翻成治療），於是第二次檢查能把後驗推過翻轉門檻、產生價值；變體矩陣把漏診降到 −15，使「不治療」幾乎在所有後驗下都最優，行動門檻被推到檢查推不到的地方，於是再準的檢查也翻不動行動，價值歸零。一句話：**資訊的價值不是資訊的屬性，是「資訊 × 效用結構」的屬性。**

工程版例子（一種合格答案）：一個「磁碟使用率」告警。情境 A：服務寫入量大、磁碟滿了會直接當機且難恢復（「漏掉 = 大災難」），這時告警有價值，因為它會讓你在某個閾值觸發擴容/清理——行動會翻轉。情境 B：同一個服務改成有自動擴容 + 寫入量極小、就算滿了也只是降級無傷（「漏掉 = 小事」），這時無論告警響不響你都不會手動做什麼——同一個告警，決策價值掉到零。**變的不是告警（還是那個告警），是它後面那格「漏掉的代價」和「會不會因它翻轉動作」。**

## 自我檢核

口頭自答，講得出來才算過關。優先回答「為什麼」。

1. **為什麼**一條資訊的價值，取決於它會不會改變你的行動，而不取決於它本身多精確、多即時？資訊幫你的唯一管道是什麼？
2. 用你自己的話講清楚：為什麼「不改變決策的資訊」價值是**精確**的零，而不是「很小」？（提示：行動不翻轉時，期望效用為什麼恆等於基準？）
3. EVPI 和 EVSI 各量什麼？為什麼任何真實檢查的 EVSI 一定夾在 0 和 EVPI 之間？舉一個你會用 EVPI 當「上限篩選」的工程判斷（先看上限值不值得，再決定要不要找具體檢查）。
4. 「資訊價值 ≥ 0」說的是決策價值還是淨值？為什麼「多測一輪總沒壞處」這句話**只在資訊免費時**才對？
5. 脊椎檢查線：第一次陽性後最適行動是「不治療」，但第二次也陽性後翻成「治療」——**為什麼**第二次檢查值錢（EVSI≈14）？它值錢的源頭是哪個分支、哪個翻轉？
6. 你想加一個新監控/再測一輪時，那句「口袋版 EVSI」的自問是什麼？它怎麼幫你當場判斷這個資訊有沒有決策價值？
7. 「資訊有趣」和「資訊有決策價值」差在哪？舉一個你查過、知道了很爽但對任何決定都沒影響的數據。
8. 把本章和 ch05 接起來：ch05 教你「後驗 × 效用 → 行動」，ch07 教你「要不要先去拿更多後驗」。為什麼說 ch07 是 ch05 的「前一步」、而不是另一個獨立主題？

## 延伸閱讀

- **Ronald A. Howard, "Information Value Theory" (1966)**（*IEEE Transactions on Systems Science and Cybernetics*, SSC-2(1):22–26；概覽見 https://en.wikipedia.org/wiki/Expected_value_of_perfect_information ）：EVPI 與「資訊價值＝機率 × 後果」的原始形式化。讀它你會看懂為什麼這套理論刻意和 Shannon 資訊理論分家——後者只量 bits、不管你拿這些 bits 做什麼決策。
- **Wikipedia, "Expected value of sample information"**（https://en.wikipedia.org/wiki/Expected_value_of_sample_information ）：EVSI 的定義、與 EVPI 的關係、以及「資訊價值非負」性質的乾淨陳述。Historical foundation 一段點明 Raiffa–Schlaifer 1961 的源頭，適合把本章兩把尺坐實。
- **Howard Raiffa & Robert Schlaifer,《Applied Statistical Decision Theory》(1961)**：EVSI 與貝氏決策分析的奠基之作。不必通讀，但若你想看「決策樹倒推 + 資訊價值」的完整工程化版本，這是源頭；本書 ch15 序貫決策會再用到它的決策樹倒推直覺。
- **《馴服隨機》ch05（貝氏定理）**：本章第二次檢查的後驗更新（16.7% → 79.8% 或 0.2%）背後的機率機制在那本書。本章只用結果、不重推；注意該書盛行率 0.1%、本書 1%，是刻意差異。
- **《馴服隨機》談期望值與全期望公式**：本章「行動不翻轉 ⇒ EVSI 恆為零」的代數核心，是條件期望的全期望公式 E[X]=E[E[X|分組]]。想看它為什麼成立、不只是接受它，去那本書把「先分組再平均＝直接平均」這條補實。
- **《如何不騙自己》談實驗設計**：本章把「要不要蒐集資訊」算成期望值，正是實驗設計的決策論骨架——一個實驗值不值得做，看它能不能改變你接下來的行動。那本書從「制度怎麼防自欺」的角度看同一件事，和本章互指。
