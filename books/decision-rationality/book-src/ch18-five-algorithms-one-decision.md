## 從你已知的出發

你做過 postmortem。一場事故結束、止血了、服務回來了，然後你坐下來寫那份文件。

好的 postmortem 有一個你大概沒明說但一直在用的結構：它不是只記「發生了什麼」，它從**好幾個視角**看同一件事。時間線視角（這幾個小時逐分鐘發生了什麼）、根因視角（最底層那個 bug 是什麼）、偵測視角（為什麼我們花了四十分鐘才發現）、止血視角（我們做的哪個動作真的有效、哪個只是巧合）、流程視角（這個流程哪裡該補一道閘）。

這五個視角看的是**同一場事故**。它們不互相競爭——你不會問「時間線視角和根因視角哪個是對的」，那是個笨問題。它們是五副不同的眼鏡，每副讓你看見別副看不見的東西，合起來你才真正懂那場事故。一份只有時間線、沒有流程視角的 postmortem，是漏掉了一半；一份只罵根因、不看偵測延遲的，下次還會被同一類問題打。

這一章就是全書那個賭注的 postmortem。同一個下注的決定——擲一枚公正硬幣，正面贏 NT$200、反面輸 NT$100——我們有五副眼鏡：期望值、期望效用、貝氏期望效用、展望理論、有限理性。你已經一副一副戴過了。本章把它們並排，看每副各看見什麼。而且——這是最關鍵的——我要你別問「哪副是對的」。那跟「時間線和根因哪個對」一樣，是個笨問題。

不過這裡有一個 postmortem 那個比喻不完全對的地方，正好是全書的核心。事故的五個視角都在描述**同一個層次**的事實（都在說「實際發生了什麼」）。而這五副決策的眼鏡，分屬**兩個不同的層次**：前面幾副有的在說「你**該**怎麼選」（規範），有的在說「人**實際**怎麼選」（描述）。它們看見不同答案——不是因為其中一副算錯了，而是因為它們在回答**兩個不同的問題**。這道分層，就是全書追了十七章的那道裂縫。最後一章，我們把它對帳到底。

## 五副眼鏡，先各自報到

開始對帳前，先讓五副眼鏡各自講一句它是誰、看的是什麼層次的問題。這不是複習定義——是定位它們各自站在「規範 vs 描述」的哪一邊。

```text
五副眼鏡的身份證

  ①  期望值（EV）           規範·最粗   長期平均報酬。問「可重複下無限次，平均賺賠多少」。
                                       對單次、尾部巨大的決策會騙你（聖彼得堡）。

  ②  期望效用（EU）         規範·完整   把金額換成效用 u(x)，凹效用＝風險趨避。
                                       幾條 vNM 公理逼出的唯一演算法。問「該怎麼選」。

  ③  貝氏期望效用          規範·帶證據  後驗信念 × 效用矩陣 → 最適行動。
                                       ②的延伸：不確定可更新，再接效用算行動。

  ④  展望理論（PT）         描述         人實際怎麼算：參考點、損失趨避、機率權重、框架。
                                       問「人怎麼選」，不問「該怎麼選」。

  ⑤  有限理性（satisficing）跨層·元      不對 L 算一個同型的數。問「這個決定值不值得算到底」。
                                       設一條夠好門檻、達標就停。是「算到哪停」，不是「算什麼」。
```

最後一列你要特別留心。①到④都對那個賭注**算出一個可比的東西**——一個數字（接還是拒），或一個行動（治療還是不治療）。⑤不一樣。satisficing 不會給你「賭注 L 的 satisficing 值是多少」這種東西，因為它根本不是那個層次的問題。ch13 已經把這件事講透了：satisficing 退後一步，問的是「**這個 L 的決定，值不值得我一路算到展望理論這麼細**」。前四副是「算什麼」，第五副是「算到哪裡停」。把⑤當成第五個跟前四競爭的答案，是這一章最容易犯、後面「直覺的陷阱」會專門拆的誤讀。

好，五副眼鏡都報到了。戴上去，看那個賭注。

## 對帳：賭注線，四副眼鏡各算一次

先把那個賭注用全書統一的彩券記法寫出來，這是 ch01 立起、後面每一章都回來碰過的脊椎：

```text
脊椎賭注  L =（+200, 0.5; −100, 0.5）    單位 NT$
```

下面是全書的核心對帳。每一格的數字我都在寫這章時重算過一遍（不抄記憶），算式攤開、不跳步。

### ① 期望值（ch02）：+50，接

```text
E[L] = Σ pᵢ·xᵢ
     = 0.5·(+200) + 0.5·(−100)
     = 100 − 50
     = +50                      ← 正的 ⇒ 期望值眼鏡說「接」
```

期望值眼鏡看見的是：如果你能把這個賭重複下無限次，平均每次淨賺 NT$50。在**可重複**的世界裡這是對的、有用的（大數法則讓平均收斂，機制見《馴服隨機》ch08／ch13）。但 ch02 那道種子裂縫就在這裡——多數人面對這個 +50 的賭，**拒絕**。期望值眼鏡解釋不了這個拒絕，它甚至會說拒絕的人「算錯了」。它沒算錯。是這副眼鏡看不到下一副眼鏡看得到的東西。

### ② 期望效用（ch03）：8.66 < 10，拒

把金額換成效用。全書統一的凹效用是 u(x)=√(x+100)，財富基準 W=100。

```text
u(不賭)  = √(0+100)             = √100  = 10
EU(L)    = 0.5·u(+200) + 0.5·u(−100)
         = 0.5·√(200+100) + 0.5·√(−100+100)
         = 0.5·√300 + 0.5·√0
         = 0.5·17.3205 + 0
         = 8.6603               ← 8.66 < 10 ⇒ 期望效用眼鏡說「拒」

確定等值  u(CE) = 8.6603  ⇒  CE = 8.6603² − 100 = 75.0 − 100 = −25.0
風險溢酬  = E[L] − CE = 50 − (−25) = 75
```

期望效用眼鏡看見了第一副看不見的：那 NT$100 的潛在損失，砍進的是效用曲線**陡的那一段**（因為 u 是凹的、邊際遞減），而那 NT$200 的潛在收益，落在效用曲線**平的那一段**。同樣的機率，賠掉比賺到更傷。於是 EU=8.66 < u(不賭)=10，拒絕。

這裡是全書第一個「規範之美」的高潮，值得你停一下重講一遍：**拒絕一個正期望值的賭，在凹效用下不是膽小、不是不理性——它正是理性**（守住 ch03 的 T4）。而且這個「該拒」不是我隨口規定的，它是 vNM 四公理（完備、遞移、連續、獨立）逼出來的唯一演算法。你只要不想被人用一串交換牽著鼻子無限抽水（money pump），你的選擇就**必須**等價於在最大化某個期望效用。規範眼鏡的力量在這裡：它不是建議，是幾條你不會反對的公理逼出的強制。

確定等值 CE=−25 是另一個有用的讀數：這個賭對你（在這條效用下）的價值，等同於「確定拿到 −25」——你寧可確定賠 25 也不要玩這個期望值 +50 的賭。中間那 75 的差距（風險溢酬）就是你為了**避開不確定**願意付的價。

### ③ 貝氏期望效用（ch05）：換到檢查線，最適行動是「不治療」

賭注線到這裡已經被兩副規範眼鏡看過了。但脊椎有**兩條線**——還有那次檢查。第三副眼鏡是②的延伸：當不確定可以靠證據更新時，先算後驗、再接效用矩陣。這條線我們搬到檢查上算，因為它正是為「信念可更新」設計的。

脊椎檢查：盛行率 1%、敏感度 99%、偽陽率 5%。先算「驗出陽性後，到底多可能真有病」。

```text
後驗（貝氏公式）
  P(病|陽性) = P(陽|病)·P(病) / [P(陽|病)·P(病) + P(陽|健)·P(健)]
            = (0.99·0.01) / (0.99·0.01 + 0.05·0.99)
            = 0.0099 / (0.0099 + 0.0495)
            = 0.0099 / 0.0594
            = 0.16667 ≈ 16.7%

自然頻率交叉驗算（1 萬人）
  100 人有病  → 99 人真陽
  9900 人健康 → 495 人偽陽
  陽性共 594 人，其中真病 99 人 → 99/594 = 16.67%   ✅ 兩法一致
```

（這個盛行率 1%／後驗 16.7% 是本書刻意選的；機率機制與另一組基率見《馴服隨機》ch05，那裡用盛行率 0.1%、後驗 1.94%，差別是讓本書的後驗落在「值得拿來做決策」的區間。貝氏定理本身怎麼推，指向該書、本書不重推。）

注意 16.7% 這個數字本身就反直覺：檢查敏感度 99%，但驗出陽性後真有病的機率只有六分之一。這是基率忽視的反面——你的後端直覺其實懂這個：alert 命中規則很準，但真故障的基率低，所以收到 alert 時真有事的機率可能低得驚人（見 ch05 的 alert fatigue 橋接）。

但貝氏眼鏡的關鍵不只是後驗。決策＝**後驗 × 效用**。光看 16.7% 不能決定要不要治療——還要看「漏掉一個病人」跟「白治一個健康人」各有多痛。接上全書統一的效用矩陣：

```text
效用矩陣          狀態=病          狀態=健康
  治療            −10              −20
  不治療          −100              0

EU(治療|陽性)   = 0.1667·(−10) + 0.8333·(−20) = −1.667 − 16.667 = −18.33
EU(不治療|陽性) = 0.1667·(−100) + 0.8333·0     = −16.67

最適行動  a* = argmax{ −18.33 , −16.67 } = 不治療   ← −16.67 > −18.33
```

貝氏眼鏡看見了：在**這組**效用與這個後驗下，居然是「不治療」期望效用較高。為什麼？因為 16.7% 還不夠高到讓「漏掉病人的 −100」壓過「白治健康人的 −20 在 83.3% 上反覆累積」。這不是說「別治病」——它說的是「在這個後驗、這個損失矩陣下，最適行動恰好是不治療」。把 u(不治療,病) 從 −100 改成 −500（漏掉的代價更高），最適行動立刻翻成治療（ch05 的紙上推演算過）。

這一格的驚嘆點，是它把「決策＝後驗×效用」這件事釘死：**漏看任一邊都會做錯決定**。只看後驗（16.7% 不高，不治療吧）可能對也可能錯，要看效用矩陣才知道；只看效用（漏病很慘，治療吧）忽略了後驗，也可能浪費。第三副眼鏡的貢獻，是把「信念」與「代價」這兩件事**強制乘在一起**。

### ④ 展望理論（ch08／ch09）：−11.8 < 0，拒（但機制不同，且框架可翻）

回到賭注線。前三副是規範眼鏡（你該怎麼選）。第四副換層次了——展望理論問的是「人**實際**怎麼算」。它不用「最終財富的效用 u(x+100)」記帳，它用「相對參考點的得失 v(x)」記帳。參考點＝現狀 0：+200 是得、−100 是失。

價值函數（Tversky–Kahneman 1992 形式）：v(x)=x^α（x≥0）、v(x)=−λ·(−x)^β（x<0）。用 TK1992 的中位參數 α=β=0.88、λ=2.25，先把機率權重簡化成 w(p)=p（待會點出加上它的方向）：

```text
v(+200) = 200^0.88 = exp(0.88·ln200) = exp(0.88·5.2983) = exp(4.6625) ≈ 105.9
v(−100) = −λ·100^0.88 = −2.25·exp(0.88·ln100) = −2.25·exp(4.0526) = −2.25·57.5 ≈ −129.5

PT 值 ≈ 0.5·(+105.9) + 0.5·(−129.5) = 52.95 − 64.75 = −11.8  < 0   ⇒ 拒
```

⚠️ **這些參數務必 hedge（2026-06）**：α=β=0.88、λ=2.25 是 TK1992 的一組常被引用的中位估計，**不是物理常數**。近期 meta-analysis（Brown 等，2024）以隨機效應給出 λ≈1.31，並指 2.25「可能偏高」；依領域範圍約 1.8–2.7（見 landscape T6）。損失趨避**現象本身**跨文化、跨領域穩健，但**具體係數值變異很大**。這不是小事——它足以翻轉這格的結論，下面立刻示範。

第四副眼鏡看見的，跟第二副（期望效用）**結論一樣（拒），機制卻完全不同**。期望效用說「拒，因為邊際遞減（凹效用）」；展望理論說「拒，因為**損失本身被放大**（|v(−100)|=129.5 > v(+200)=105.9，即使機率相等）」。這是兩條不同的數學路徑通到同一個「拒」。

而且展望眼鏡看得到前三副完全看不到的兩件事：

**第一，結論對參數敏感到危險。** 把 λ 換成 meta-analysis 的 1.31 重算：

```text
v(−100) = −1.31·57.5 ≈ −75.3
PT 值 ≈ 0.5·105.9 + 0.5·(−75.3) = 52.95 − 37.65 = +15.3  > 0   ⇒ 變成「接」
```

同一個賭、同一套展望理論、只換一個參數，結論從「拒」翻成「接」。這正是為什麼 landscape 把這條標成全書最該 hedge 的一項——**規範眼鏡（期望效用）給的「拒」是公理逼出的、穩；描述眼鏡（展望理論）給的「拒」依賴一個有再現性爭議的經驗參數、不穩。** 兩個「拒」看起來一樣，地基的硬度天差地別。寫到這裡我認為這是整張對帳表最容易被忽略、卻最重要的一頁：別把描述模型的數字當規範結論的硬度來用。

**第二，框架一翻，答案可翻。** 加上機率權重 w(p)（小機率被高估、中大機率被低估，倒 S 型）只是讓 p=0.5 附近的權重略微偏移、方向上不改變這個賭的結論。但**框架**會。同一個賭，問成「你願不願意冒險博取 +200」（得框架，多數人傾向風險趨避、拒），跟問成「你要不要保住現狀、避免可能的 −100」（失框架，多數人傾向風險愛好），可以得出相反選擇（ch10 的反射效應）。規範眼鏡（②③）對措辭**免疫**——邏輯等價的問法該給同一個答案（描述不變性）。描述眼鏡（④）對措辭**敏感**——而且它精確預測往哪個方向翻。這既是規範失效的鐵證、也是描述的勝利。

### ⑤ 有限理性（ch13）：不算數，問「值不值得算到底」

前四副都對那個賭注交了一個可比的東西：+50（接）／8.66<10（拒）／檢查線最適行動（不治療）／−11.8<0（拒）。第五副不交。

satisficing 退後一步，看著前面這四欄密密麻麻的算式，問一個**元層次**的問題：

```text
前四副在回答：「這個 L，該怎麼算、算出來是接還是拒？」
satisficing 在回答：「這個 L 的決定，值不值得我一路算到展望理論這麼細？」
```

對那個賭注，多數人連期望值都沒掏出來算就拒絕了。satisficing 說這**未必不理性**：如果「再多算三套理論」能省下、能改變的東西，比不上「算它們的力氣」，那麼「達到夠好的把握就停手」本身就是元層次上的最優（守住 ch13 的 T10——satisficing 不是偷懶，把搜尋成本算進去後它可能就是最佳化）。

所以 satisficing 在對帳表上是**獨立的一列、標成「定性／不同類」**，格子裡不放數字。它不是第五個跟前四競爭的答案——它是「算到哪裡停」的判斷，不是「算什麼」的結果。前四副是「該往桌上擺哪些算式」，第五副是「這桌算式什麼時候該收掉、去做別的事」。把它當第五個競爭答案，等於問「剪刀的兩片刀刃哪片是對的」——它們不在比同一件事。

## 中央張力的最終一句話

全書反覆叩問同一句話：**規範理性（你該怎麼選）與描述現實（人實際怎麼選）之間，那道穩定的裂縫。** 走到這裡，可以給它一個收束的答案了。ch17 已經做了哲學收束，這裡用對帳表把它釘成一句能被工程師轉述的話：

那道裂縫**不是 bug，是兩套都對的數學在描述兩件不同的事。**

表②（期望效用）回答「該怎麼選」——它是規範，地基是公理，它的「拒」硬。表④（展望理論）回答「人怎麼選」——它是描述，地基是實驗數據，它的「拒」依賴一個會變的參數。當人系統性地偏離規範（Allais 兩題、框架翻轉、損失趨避），這**既不證明規範錯**（公理還是公理，違反它還是會被 money pump），**也不證明人笨**（偏離有結構、可預測方向、可建模——這正是它了不起的地方，守住 T8）。它只證明了一件樸素的事：**「人該怎麼選」和「人實際怎麼選」是兩個不同的問題，各有各的、各自漂亮的數學。**

把這句話翻成你的母語：規範＝你 design doc 裡寫的「我們應該這樣權衡」；描述＝code review 真實發生時人**實際**怎麼拍板。兩者有穩定的落差。好的工程流程不假裝這落差不存在，也不罵人「怎麼不照 design doc」——它**設計來補這道落差**（就像 checklist 補人腦的捷思、runbook 補你凌晨三點的記憶力）。承認描述性限制、用規範性流程去補，這是全書留給你最實用的一件事。

## 直覺的陷阱

收官這章的陷阱，是針對「讀完全書」這個狀態本身的——你現在最可能犯的兩個錯。

| 陷阱 | 錯誤直覺帶你下溝的那一步 | 怎麼自我察覺 | 正確版 |
|---|---|---|---|
| **「五種演算法＝五個答案，要挑一個」** | 看到表上 +50（接）／8.66<10（拒）／−11.8<0（拒）給出不同結論，本能想問「那到底哪個對」、想選一個當「正解」 | 你發現自己在問「決策理論的標準答案是接還是拒」——這個問題本身就錯了。就像問「時間線視角和根因視角哪個是 postmortem 的正解」 | 五副眼鏡看**同一件事的不同面向**。①看長期平均、②看公理逼出的規範、④看人實際怎麼算。它們答案不同因為**問題不同**，不是因為其中一個算錯 |
| **「學了這些就會做更好的決策」** | 以為「懂了展望理論、懂了損失趨避」＝以後就不會被它影響了；把規範知識當成描述性偏向的解藥 | 你下次照樣對 regression 的恐懼壓過對新功能的渴望、照樣被失框架推著賭——**知道病名不等於免疫**。如果你以為讀完就免疫了，那正是沒讀懂 | 規範知識**不自動修正**描述性偏向（呼應 ch17）。專家一樣會框架翻轉、一樣有確定性效應。要補它得靠**流程**（checklist、預先承諾、外部視角），不是靠「我知道了所以我不會」 |

第二個陷阱我想多說一句，因為它是全書最反諷的地方。你花了十八章學「人怎麼系統性偏離規範」。學完之後最危險的念頭是：「好，現在**我**不會了。」——這念頭本身就是一種偏誤（高估自己的免疫力）。展望理論的參數在**你**身上一樣跑。康納曼自己——拿了諾貝爾、寫了一輩子這個的人——公開說過他研究了一輩子這些偏誤，自己照樣會掉進去（這是他常被引用的自述，我把它當態度而非逐字引文）。正確的收穫不是「我免疫了」，是「我知道我會偏向哪、所以我替自己設了流程去補」。runbook 不是因為你笨，是因為你知道凌晨三點的你記性不可靠。去偏流程同理。

**題 1（★★，[15 分鐘]）—— 五分鐘電梯版**

想像你和一個沒讀過這本書的資深工程師同搭電梯。用**五分鐘、不寫任何公式**，對他講清楚：同一個下注的決定（公正硬幣，正面贏 200、反面輸 100），為什麼「規範說該拒、人實際也拒、但兩個拒不是同一回事」，而且這不是矛盾。先自己講一遍（出聲或寫下逐字稿），再看解答對照。

### 推演解答

一個能過關的講法大致是這樣（你的用詞可以不同，但這幾個轉折必須在）：

> 「有個賭，丟公正硬幣，正面給你 200、反面你賠 100。平均算下來每次賺 50，所以『期望值』說你該接。但多數人會拒——包括你。
>
> 這裡有兩個**不同**的理由解釋這個拒，這是重點。
>
> 第一個是**規範**的（你該怎麼選）：賠掉 100 對你的『痛』，比賺到 200 的『爽』更大——不是因為金額，是因為錢的價值邊際遞減（第一個一萬塊比第一百個一萬塊重要）。把這個算進去，拒絕這個賭其實是**理性**的。這套理論還是從幾條你不會反對的公理逼出來的，硬。
>
> 第二個是**描述**的（人實際怎麼算）：人腦不是算最終財富，是算『相對現狀的得失』，而且**賠錢的權重天生被放大**（差不多兩倍多，但這個數字學界還在吵，別當常數）。所以也得出拒。
>
> 兩個『拒』看起來一樣，其實一個在說『你**該**這樣』、一個在說『人**就是**這樣』。它們不是同一個答案重複兩次，是兩套各自漂亮的數學在回答**兩個不同的問題**。這就是為什麼明明結論一樣，我還要分開講——因為換個賭它們會吵起來。」

**常見錯路**：(1) 只講一個理由（通常只講損失趨避），漏掉「規範 vs 描述是兩層」——那等於沒講出全書的點。(2) 把第二個理由講成「人很笨所以算錯」——錯，描述模型不是在說人笨，是在精確刻畫人怎麼算（守住 T7/T8）。(3) 把 λ≈2.25 講成定論——要 hedge。能把「兩個拒不同源、且不矛盾」這層講出來，才算過。

**題 2（★，[10 分鐘]）—— 改觀三點**

不准翻書。憑記憶寫下：讀完這本書，你對「理性」這個詞的理解，改變了哪**三點**？每點一句話。寫完再看解答的參考清單對照（你的三點不必跟它一樣，但檢查你的有沒有踩到這幾個層次）。

### 推演解答

一份夠格的清單會碰到下面這些（任三點且言之成理即可）：

```text

  · 風險趨避不是膽小：凹效用下拒絕正期望值的賭是理性的（不是性格缺陷）。
  · 期望值不是該付的價：聖彼得堡——期望值無限不代表該無限下注。
  · 風險 ≠ 不確定：能測機率的「風險」和測不出的「模糊」是兩件事，
    對模糊額外保守也是理性人會犯的（不是算錯）。
  · 不改變決策的資訊，價值精確等於零：再多資訊若不換行動就是零價值。
```

**常見錯路**：把三點全寫成「我學到了某某理論的公式」——那是「學到什麼內容」不是「對理性改觀」。改觀指的是**對『理性是什麼』這個詞的理解**變了，而不是多背了幾條定義。如果你的三點抽掉具體理論名字後就什麼都不剩，再想一遍。

**題 3（★★，[20 分鐘]）—— 把你最近一個真實決策過五副眼鏡**

挑一個你最近做過的、帶不確定性的真實技術決策（要不要 rollback、要不要先 canary 再全面上線、選哪個資料庫、要不要再跑一輪壓測）。把它**逐副眼鏡**過一遍：① 期望值眼鏡會怎麼算？② 期望效用眼鏡（你的隱含效用是凹的嗎，哪個損失砍在陡的那段）？③ 如果有可更新的證據（canary 結果、壓測數據），貝氏眼鏡的後驗×代價是什麼？④ 描述眼鏡：你當時**實際**是被什麼推著決定的（參考點？對 regression 的損失趨避？某人措辭的框架）？⑤ satisficing：你那時其實算到哪就停了，那個停手點合理嗎？

### 推演解答

沒有標準答案——這題的價值在「過一遍」本身。但一份做對了的答案會有這幾個特徵，拿來自檢：

```text
做對了的徵兆

  · ① 和 ② 你能說出差別：期望值眼鏡可能說「接」（平均划算），
     但你的效用是凹的（一次線上事故的痛 >> 省下的時間的爽），所以 ② 說「別賭」。
     如果你的①②結論不同，你抓到了全書的核心張力在你自己決策裡的版本。
  · ③ 你能指出「我當時其實有可更新的證據卻沒好好用」，或「canary 就是我的後驗更新」。
  · ④ 最有價值：你誠實寫出「我**規範上**算出該 X，但我**實際**做了 Y，因為……」
     ——那個「因為」往往是參考點、損失趨避、或某人的框架。
  · ⑤ 你能說出「我算到 ___ 就停了」，並判斷那個停手點是合理的 satisficing
     還是偷懶（把搜尋成本誠實算進去，它划算嗎）。
```

**常見錯路**：(1) 五副眼鏡全給同一個答案——多半是你沒真的切換層次，把規範和描述混成一團了。好的決策案例裡，①②④至少有兩副會給出**不同**的讀數（這正是有東西可學的訊號）。(2) 第④副跳過或美化——「我當時很理性地考量了所有因素」幾乎一定是事後合理化。誠實的④會讓你有點不舒服，那才對。

## 自我檢核

口頭自答，講得出來才算過：

1. 為什麼表上①（期望值，接）和②（期望效用，拒）對同一個賭給出相反結論？哪一個「算錯了」？（提示：都沒錯，它們在算不同的東西。）
2. 表②（期望效用，拒）和④（展望理論，拒）結論一樣——為什麼我說「別被這個一樣騙了」？兩個「拒」的機制差在哪、地基的硬度差在哪？
3. 在檢查線（後驗 16.7%）上，為什麼光知道「16.7%」還不能決定要不要治療？貝氏眼鏡逼你乘上的另一半是什麼？
4. 把展望理論的 λ 從 2.25 換成 1.31，那個賭的結論翻了。這件事告訴你「規範結論的硬度」和「描述結論的硬度」差在哪？
5. 為什麼 satisficing 在對帳表上不放數字、被標成「定性／不同類」？它回答的問題跟前四副差在哪一個層次？
6. 「人系統性偏離規範」既不證明規範錯、也不證明人笨——用你自己的話講，那它證明了什麼？
7. 「不改變決策的資訊，價值精確等於零」——舉一個你工作中為了心安而蒐集、其實零決策價值的例子，並說明它為何是零。
8. 讀完全書你最該警惕的那個念頭是什麼？（提示：跟「我現在免疫了」有關。）為什麼那個念頭本身就是個偏誤？

## 延伸閱讀

- **Kahneman, *Thinking, Fast and Slow*（2011）** — 康納曼把展望理論、雙系統、各種偏誤寫給一般讀者的集大成之作。本書收的是「偏離的數學描述」；想看「偏離的心理機制與作者本人的反省」（包括他承認自己研究一輩子照樣中招），讀這本，尤其談 prospect theory 與 framing 的章節。心理機制的深入則指向《思考的陷阱》。
- **von Neumann & Morgenstern, *Theory of Games and Economic Behavior*（1944；公理化在 1947 二版附錄）** — 規範那一側的原典。期望效用公理的形式內容在 1947 二版的附錄 "The Axiomatic Treatment of Utility"（常被誤記成 1944，見 ch03）。不必通讀，看附錄感受「幾條公理逼出唯一演算法」的規範之美。
- **Brown, Imai, Vieider & Camerer 等，損失趨避 meta-analysis（*Journal of Economic Psychology*，2024）** — 對 λ≈2.25 的再現性檢驗，隨機效應給 λ≈1.31（CI 約 1.10–1.53），指 TK1992 的 2.25 可能偏高。讀它來校準你對「展望參數不是常數」的直覺（2026-06；這是全書最該定期回頭看是否更新的一條）。
- **Stanford Encyclopedia of Philosophy, "Decision Theory" 與 "The St. Petersburg Paradox"** — 規範決策理論與聖彼得堡的權威概念地圖。想把全書的規範地基（期望效用、公理、聖彼得堡是效用問題不是算術錯）放進更大的哲學脈絡，從這兩條目入手。
- **本書內部回看** — 想重溫某一格：規範地基回 ch03（vNM 公理、凹＝風險趨避）；裂縫的描述模型回 ch08–ch12（四支柱與被掀翻的公理對照表在 ch12）；有限理性回 ch13（satisficing 為何是元層次最佳化）；哲學收束回 ch17（理性三角、規範與描述的和解）。這張對帳表是它們的總帳，不是替代品。
