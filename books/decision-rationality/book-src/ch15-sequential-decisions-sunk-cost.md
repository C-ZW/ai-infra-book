# ch15 — 序貫決策：決策樹與沉沒成本

> **本章解決什麼問題**：到上一章為止，本書算的幾乎都是「單步」決策——站在一個岔路口，把後驗乘上效用、挑一個最適行動，按下去。但真實決策很少只有一步。你常常是「先做一件事、看結果、再決定下一件事」：先漸進式放量、看指標、再決定全面上線或回滾。本章把單步推到多步——決策樹（decision tree）與它的倒推（backward induction），講清楚「現在的選擇要替未來保留選項」是什麼意思；然後處理一個天天在 code review 裡出現、規範理論卻一刀切的東西：**已經投進去的成本（sunk cost），在往前看的決策裡該被完全忽略。** 這是 Part IV 的最後一章，下一章起轉入收束。

## 從你已知的出發

你做 rollout 的時候，腦子裡跑的就是一棵決策樹，只是沒畫出來。

一個有風險的改動要上線。你不會閉著眼睛全面上線。你會：**先放 1% canary → 觀察一段時間的錯誤率與延遲 → 看到沒問題就放大到 10%、再全面上線；看到不對勁就回滾。** 這一串「做一步、看結果、再決定下一步」就是序貫決策（sequential decision）。它和單步決策的根本差別在於：**你現在這一步的選擇，會改變你下一步能做什麼、以及你下一步知道什麼。**

放 canary 這個動作本身不直接賺錢——它要花你時間、佔注意力、把上線往後推。它的價值全在於：它**先給你一個觀測**，讓你帶著更新過的信念再做第二個決定。這正是 ch07「資訊的價值」的序貫版——先探測、再決定，而探測值不值得，取決於它可能讓你換一個更好的動作。

你也天天在 review 裡看到序貫決策的另一面，而且看到它**出錯**：

```text
   「這個專案我們已經做三個月了，現在砍掉太可惜。」
   「都買了一年的 license 了，硬著頭皮也要把它用起來。」
   「重寫了一半，回頭成本太高，只能繼續往前。」
```

這三句話有一個共同的結構：**用「已經投進去多少」當作「要不要繼續」的理由。** 規範理論對這件事的態度非常冷酷——已經花掉、且收不回來的錢，在「接下來該怎麼做」這個問題裡，**權重應該是零**。三個月也好、一年 license 也好、重寫一半也好，它們都已經發生、不會因為你接下來的選擇而改變，所以它們**不該**進入「往前看」的計算。你只該問一件事：**從現在這一刻起、忽略過去，繼續 vs 停手哪個的未來期望報酬更高？**

這聽起來簡單到像廢話，但它是本章最反人性的一頁。你的直覺會拼命把沉沒成本拉進來，而且拉得很有道理的樣子。本章後半就是要把這個直覺拆開，看清楚它在哪一步把你帶溝裡。

## 決策樹：把「做一步、看一步」畫成一棵樹

序貫決策的標準工具是決策樹。它只有三種節點，你全都熟：

```text
   決策節點（你選）        機運節點（自然/不確定選）       葉子（報酬）
       ┌─┐                      ○                          數字
       │ │  你主動挑一條分支   ╱│╲   每條分支標機率         葉子上的
       └─┘                    ╱ │ ╲  （加起來=1）           期望效用
     方框＝你能控制          圓圈＝你不能控制              一條路走完的報酬
```

- **決策節點（方框）**：你能控制。從這裡岔出去的每條分支，是你的一個可選行動，你會挑期望效用最高的那條。
- **機運節點（圓圈）**：你不能控制。從這裡岔出去的每條分支是一個不確定的結果，各帶一個機率（這些機率是你的後驗信念）。你不能挑，只能**取期望**。
- **葉子**：一條路從頭走到尾的最終報酬（效用）。

對工程師最直白的對應：方框＝你的 `if`／你主動下的指令（ship / rollback / canary）；圓圈＝環境給你的回應、你只能觀測不能決定（這版到底有沒有 bug、canary 會不會報警）。整棵樹就是把「先做哪個動作、會遇到哪些結果、之後再做哪個動作」全攤開。

### 倒推：從葉子往回算

決策樹怎麼解？答案是**倒推（backward induction，又叫 rollback）**——從最右邊的葉子開始，往左一路把樹「收掉」：

```text
   遇到機運節點（○）：把它的所有分支「取期望」
       → 用機率加權各分支的值，得到這個圓圈節點的期望效用

   遇到決策節點（□）：把它的所有分支「取最大」
       → 你會選期望效用最高的那條，這個方框節點的值＝最高的那條

   一路往左收，直到根節點 → 根的值就是整棵樹的期望效用，
                            收的過程中標下來的「每個方框該選哪條」就是最適策略
```

注意這兩個動作的**不對稱**，這是整個方法的靈魂、也是最常被搞錯的地方（見下面〈直覺的陷阱〉）：**機運節點取期望（你不能挑，只能平均），決策節點取最大（你能挑，挑最好的）。** 把機運節點也「取最大」——也就是「假設運氣總會站在我這邊」——是序貫決策最常見的災難。

為什麼要從後往前？因為「現在該怎麼選」取決於「之後會怎麼走」。你沒法先決定第一步、再決定第二步——第一步的好壞要看它把你帶到的處境裡、第二步能做多好。所以必須先把「未來每個處境下我會怎麼做、能拿多少」算清楚，再回頭評價現在的選擇。這跟你寫遞迴是同一個腦回路：要算 `solve(n)`，先把 `solve(n+1)` 的結果備妥。倒推就是決策樹上的動態規劃；完整的動態規劃／Bellman 方程本書不展開（只到倒推直覺），數學機器指向決策分析教科書（見延伸閱讀）。

## Worked example：先 canary、還是直接全面上線？

把脊椎的精神搬到一個 rollout 上，用一棵兩階段決策樹算到底。數字是我自己設計、自己算過的，你可以跟著複核。

**處境**。一個有風險的改動要上線。世界有兩個你看不見的狀態：

```text
   θ = good（這版沒 bug）   先驗 P(good) = 0.9
   θ = bad （這版有 bug） 先驗 P(bad)  = 0.1
```

報酬以效用點數計（負數＝損失，數字示意、單位無所謂，只看相對大小）：

```text
   直接全面上線（ship）：  good → 0（平安上線）
                           bad  → −100（全面上線事故、半夜被 page、回滾＋善後）
   回滾/不上（roll）：  −25（無論 good/bad；延後上線、白做一場的固定代價）
```

**第一個問題：不做任何觀測，該 ship 還是 roll？** 這是個單步決策，照 ch05 的做法算：

```text
   EU(ship) = P(good)·0 + P(bad)·(−100)
            = 0.9·0 + 0.1·(−100) = −10

   EU(roll) = −25

   −10 > −25 ⇒ 沒有額外資訊時，理性選擇是【直接 ship】，期望效用 −10。
```

直覺上對：bad 只有一成機率，雖然爆起來很痛（−100），但攤平後（−10）還是比白白回滾（−25）划算。**這就是「盲射」的基準線。**

**現在引入 canary（探測）。** 你可以先對 1% 流量放量，它給你一個不完美的訊號 s ∈ {報警, 沒事}：

```text
   P(報警 | bad)  = 0.9   （真有 bug，小流量觀察多半會抓到）
   P(報警 | good) = 0.1   （沒 bug 也可能誤報，偽陽）
```

canary 不免費，先記它的成本是 c（後面討論 c 多大才值得）。放完 canary、看到訊號後，你**再決定**ship 還是 roll。這就是第二階段——序貫的精髓在這裡：**第二步的決定，建立在第一步買到的觀測上。**

先把 canary 兩種訊號各自的機率與後驗算出來（這是 ch05 的貝氏更新，機制指向《馴服隨機》ch05）：

```text
   P(報警) = P(報警|bad)·P(bad) + P(報警|good)·P(good)
           = 0.9·0.1 + 0.1·0.9 = 0.09 + 0.09 = 0.18
   P(沒事) = 1 − 0.18 = 0.82

   後驗 P(bad | 報警) = P(報警|bad)·P(bad) / P(報警)
                      = 0.09 / 0.18 = 0.50
   後驗 P(bad | 沒事) = P(沒事|bad)·P(bad) / P(沒事)
                      = (0.1·0.1) / 0.82 = 0.01 / 0.82 ≈ 0.0122
```

自然頻率交叉驗算（想像 1000 次這樣的上線）：

```text
   1000 次上線 ─┬─ 100 次真有 bug ─┬─ 90 次 canary 報警
                │                  └─ 10 次 canary 沒抓到（漏報）
                └─ 900 次沒 bug  ──┬─ 90 次 canary 誤報（偽陽）
                                   └─ 810 次 canary 沒事

   報警 = 90（真）+ 90（誤）= 180 次 → 其中真有 bug 90 次 → 90/180 = 50.0%  ✓
   沒事 = 10（漏）+ 810       = 820 次 → 其中真有 bug 10 次 → 10/820 ≈ 1.22% ✓
```

兩法一致。canary 報警時「真有 bug」的機率從先驗的 10% 跳到 50%；canary 沒事時則掉到約 1.2%。這就是 canary 替你買到的東西——**把模糊的 10% 信念，分裂成「報警的高風險堆」與「沒事的低風險堆」。**

現在畫樹、倒推。先在每個第二階段的決策節點上算 ship vs roll：

```text
   【canary 報警，後驗 P(bad)=0.50】
       EU(ship) = 0.50·0 + 0.50·(−100) = −50
       EU(roll) = −25
       −25 > −50 ⇒ 報警就【回滾】，這個方框的值 = −25

   【canary 沒事，後驗 P(bad)=0.0122】
       EU(ship) = 0.9878·0 + 0.0122·(−100) = −1.22
       EU(roll) = −25
       −1.22 > −25 ⇒ 沒事就【全面上線】，這個方框的值 = −1.22
```

決策節點取最大——這兩個方框分別收成 −25 和 −1.22。再往左碰到 canary 的機運節點（圓圈），取期望：

```text
   做了 canary 後（先不計 canary 成本 c）的期望效用
     = P(報警)·(報警那格的最佳值) + P(沒事)·(沒事那格的最佳值)
     = 0.18·(−25) + 0.82·(−1.22)
     = −4.5 + (−1.0) = −5.5
```

整棵樹收完，長這樣：

```text
                         ┌─ ship ──○─ good 0.9878 → 0
            ┌─ 沒事 0.82 ─□        └─ bad  0.0122 → −100   EU(ship)=−1.22  ◄選
            │  P(bad)=.012└─ roll ──────────────── −25
   做canary─○ (−c)
   □        └─ 報警 0.18 ─□─ ship ──○ ... EU(ship)=−50
   │           P(bad)=.50 │         └─ roll ─────── −25   ◄選
   │                      └─ roll ─────────────────────── −25
   │
   └─ 不做canary，直接 ─□─ ship ──○─ good 0.9 → 0
                         │         └─ bad 0.1 → −100   EU(ship)=−10  ◄選
                         └─ roll ──────────────────────── −25

   做 canary 分支期望（未計成本）= −5.5
   不做 canary（直接 ship）       = −10
```

**最後一步：canary 值不值它的成本 c？** 把「做 canary」與「不做、直接 ship」比：

```text
   做 canary 的淨期望效用 = −5.5 − c
   不做（直接 ship）       = −10

   做 canary 划算  ⟺  −5.5 − c > −10  ⟺  c < 4.5
```

也就是說，**這次 canary 帶來的資訊價值恰好是 4.5 個效用點**（這正是 ch07 的 EVSI——樣本資訊期望價值：有觀測時的最佳期望效用 −5.5，減去無觀測時的最佳期望效用 −10，差額 4.5）。代幾個成本看：

```text
   c = 3  → 做 canary 淨 −8.5  >  直接 ship −10  ⇒ 該做 canary
   c = 5  → 做 canary 淨 −10.5 <  直接 ship −10  ⇒ 別做，直接 ship
```

讀到這裡停十秒，因為這是序貫決策最漂亮的地方：**canary 的價值不是「它讓我更安心」，是一個能算出來的數字（4.5）——而且這個數字精確等於「它能讓我換一個更好動作」的期望收益。** 當 canary 太貴（c≥4.5），即使它技術上完全有效、報警準確率很高，理性的選擇仍是不做、直接賭那一成。資訊有效 ≠ 資訊值得買，這條 ch07 的洞見在序貫樹上又出現一次。

## 序貫決策的真正禮物：保留選項

上面那棵樹還藏著一個比「算出 4.5」更深的東西。回頭看：**直接 ship 的時候，你把一整條未來路徑鎖死了**——按下全面上線，要是踩到那一成的 bad，你就得吃滿 −100。但**先 canary**，等於你**先不做承諾、先買一個觀測、把「之後要不要全面上線」這個選擇權保留到資訊更多的時候再行使**。

這就是選項價值（option value）：**保留「之後再決定」的彈性，本身就有價值**，即使你最後可能根本沒用到那個彈性。在上面的例子裡，82% 的情況 canary 沒事、你照樣全面上線，看起來 canary「白做了」；但它值錢的理由全在那 18% 報警、讓你及時回滾的情況——你買的是「壞情況發生時能轉向」的權利。

工程師對這個直覺其實熟到不行，只是平常不叫它 option value：

```text
   可回滾的部署  >  不可回滾的部署     ── 即使你大多數時候不會回滾
   feature flag  >  寫死的 if          ── 保留「之後關掉」的權利
   藍綠部署、影子流量、漸進放量          ── 全都是花成本買「之後能轉向」
   不把資料庫 migration 寫成不可逆       ── 保留「回到上一個 schema」的選項
```

每一條都是在付出當下的成本（多一套環境、多一段 flag 邏輯、多一步），去買「未來在更壞或更明朗的情境下還能改主意」的權利。這也接回 ch04 的模糊與不確定：當你連機率都估不準（Knightian uncertainty），保留彈性比押注一個你不信的數字更穩——選項價值在「不確定越大」時越值錢。當對手會針對你的選擇反應時，「保留選項」與「故意鎖死選項以示承諾」之間還有更深的張力，那已經溢出單人決策、屬於賽局的地盤（見《賽局》談承諾與可信威脅）。

## 沉沒成本：規範理論最冷酷的一刀

現在來處理本章開頭那三句 review 裡的話。先把規範立場講到不能再清楚，再看人為什麼總是違反它。

**沉沒成本（sunk cost）＝已經付出、且無論你接下來怎麼選都收不回來的成本。** 規範理論的判決只有一句：

> 在「往前看」的決策裡，沉沒成本的權重是零。你只該比較**從現在起、各個選項的未來增量成本與報酬**；過去花了多少，與此無關。

為什麼？因為決策的定義就是「在可選的未來裡挑一個」。沉沒成本對**每一個**未來選項都一樣大（它已經發生、不隨你的選擇改變），所以它在選項之間**不造成差別**——而決策只關心差別。一個對所有選項都加同一個常數的項，不會改變哪個選項最大。它在數學上就是可以從 argmax 裡約掉的常數。

用一個乾淨的數字看清楚。一個內部專案，**已經投了 80 點**（人月、成本，隨你）。現在面臨「繼續 vs 砍掉」：

```text
   繼續：還要再投 40 點，有 50% 機率成功（成功值 50、失敗值 0）
       往前看的期望效用 = 0.5·50 + 0.5·0 − 40 = 25 − 40 = −15
   砍掉：往前看的期望效用 = 0   （不再投入、也不再產出）

   −15 < 0  ⇒  規範上該【砍掉】。
```

注意這個算式裡**完全沒有那 80 點**。它不出現，因為它已經沉了——繼續也好、砍掉也好，那 80 點都回不來，它對「繼續 vs 砍掉」這個比較毫無作用。規範理論說：忘掉它，往前看，−15 < 0，砍。

但人腦會這樣算：

```text
   「我們已經投了 80，現在砍掉等於這 80 全部打水漂。
     再投 40、有一半機會做成、把 80 救回來——拼一把吧。」
```

這就是沉沒成本謬誤（sunk cost fallacy）。它的錯誤**不在算術**——它算得出「再投 40 有一半機會」——錯在**把一個不該進入比較的項（已沉的 80）偷偷塞進了「繼續」的理由**，彷彿繼續能「挽回」那 80。但繼續挽回不了任何東西：那 80 已經不存在於你的可選未來裡，無論你選哪邊它都一樣。把「不想浪費已花的 80」當作「繼續」的論據，是在用一個常數去偏袒一個選項——這就是被帶溝裡的那一步。

這個謬誤有個著名的別名叫**協和號效應（Concorde effect / Concorde fallacy）**——典故是英法合造的協和號超音速客機，明知商業上注定虧損，兩國政府仍以「已經投入巨資、現在停就全白費」為由續砸。這個詞一般歸給 Richard Dawkins 與 Jane Brockmann／Tamsin Carlisle 在 1970 年代研究動物行為時引入，用來描述動物（與人）依「已投入」而非「未來收益」做投資決定的傾向（此歸屬為常見轉述，2026-06 我無法即時連網覆核確切出處與年份，請當軼事看待、不作為硬事實引用）。名字來自哪不重要，結構是普世的：**用過去的投入替未來的決定背書。**

要把沉沒成本誠實地擺進決策樹，正確做法是：**把它畫成根節點之前、所有分支共有的一格**，看清楚它對任何一個分支都一樣大、因此在分支之間互相抵銷、不影響倒推的結果：

```text
              ┌── 繼續 ──○─ 成功 0.5 → +50
   −80 ───────┤          └─ 失敗 0.5 →   0     （這條再 −40）
   (已沉，    └── 砍掉 ─────────────────  0
    共有)        ▲
                 │ −80 在「繼續」與「砍掉」上一樣大
                 ▼ 它落在分叉【之前】，倒推時對「該選哪條」毫無作用
                   → 正確的比較只看分叉【之後】：−15 vs 0 → 砍
```

一旦把它畫在分叉之前，你就會直接看到：它不在任何一條分支的**差額**裡。會被沉沒成本綁架的人，腦中其實是把那 −80 錯誤地只掛在「砍掉」那條（彷彿砍掉才「賠掉」80、繼續就「沒賠」），製造出一個假的不對稱。畫對了樹，假不對稱就消失了。

我認為這是本章最值得帶走的一句：**規範理論不是叫你「別在乎已經花的努力」——它是指出，已經花的努力在「接下來怎麼做最好」這個問題上，數學上就是個約得掉的常數。** 在乎它（捨不得、可惜）是人之常情，是描述性事實；但讓它**改變你往前看的選擇**，是規範性的錯誤。這正是全書那道裂縫又一次現身：人實際怎麼算（把沉沒成本算進去）vs 該怎麼算（約掉它）。

## 直覺的陷阱

| 陷阱 | 錯誤直覺帶你下溝的那一步 | 正確版 / 怎麼自我察覺 |
|---|---|---|
| **沉沒成本謬誤（協和號效應）** | 「已經投了這麼多，現在停就全白費了，再拼一把把它救回來。」把已沉的成本當成「繼續」的理由。 | 已沉成本對每個未來選項都一樣大，是 argmax 裡約得掉的常數（landscape T12）。**自我察覺**：問自己「假如這個專案是別人昨天剛丟到我桌上、我沒投入過任何東西，我現在會啟動它嗎？」答案若是「不會」，你就是被沉沒成本綁架了。 |
| **倒推時把機運節點當決策節點** | 算未來那一步時，假設「運氣會站我這邊」「到時候我總有辦法」，等於把圓圈節點也「取最大」。 | 機運節點**取期望**（你不能挑、只能平均），決策節點才**取最大**。把不確定的未來當成「最好情況」會系統性高估每一條前瞻路徑的價值，讓你做出過度樂觀的承諾。**自我察覺**：你的計畫是否依賴「到那時候應該就……」這種對運氣的隱性假設？ |
| **把「保留選項」當浪費** | 「canary 八成沒事，純粹拖時間」「feature flag 大多數時候不會關，多此一舉」。 | 選項價值是「壞情況發生時能轉向」的期望收益；它的價值來自你**沒用到**的那些分支裡的最壞情況被你避開了。**自我察覺**：你是否用「結果沒出事」回頭論證「當初不該留退路」（後見之明把選項價值抹掉）？ |
| **可惜 ≠ 該繼續** | 把「捨不得、可惜」這種真實的情緒，當成繼續的**論據**。 | 可惜是描述性事實（人會痛），不是規範依據。痛歸痛，往前看的算式裡不該有它。**自我察覺**：你給出的理由裡，有沒有「不然就浪費了」這個句式？它是沉沒成本的指紋。 |

## 紙上推演

### 題 1（★★★，**[25 分鐘]**）兩階段決策樹倒推

換一組數字，自己把上面的 worked example 重走一遍。處境：上線一個改動，先驗 P(bad)=0.2、P(good)=0.8。報酬：直接 ship → good 0、bad −80；回滾 → −20（不分狀態）。可先做一個整合測試（integration probe），訊號 s∈{報警, 沒事}，P(報警|bad)=0.75、P(報警|good)=0.2。

(a) 不做 probe，該 ship 還是 roll？期望效用多少？
(b) 算 P(報警)、P(沒事) 與兩個後驗 P(bad|報警)、P(bad|沒事)。
(c) 對每個訊號算出第二階段該 ship 還是 roll。
(d) 倒推算「做 probe（先不計成本）」的期望效用，並求 probe 的資訊價值（EVSI）：probe 成本最多到多少還值得做？

### 推演解答

**(a) 不做 probe：**

```text
   EU(ship) = 0.8·0 + 0.2·(−80) = −16
   EU(roll) = −20
   −16 > −20 ⇒ 直接 ship，基準期望效用 = −16。
```

**(b) 訊號機率與後驗：**

```text
   P(報警) = 0.75·0.2 + 0.2·0.8 = 0.15 + 0.16 = 0.31
   P(沒事) = 0.69
   P(bad|報警) = 0.15 / 0.31 ≈ 0.4839
   P(bad|沒事) = (0.25·0.2) / 0.69 = 0.05 / 0.69 ≈ 0.0725
```

（自然頻率複核，1000 次：bad 200 → 報警 150、漏 50；good 800 → 誤報 160、沒事 640。報警 310 中 150 真 bad → 0.484 ✓；沒事 690 中 50 真 bad → 0.0725 ✓。）

**(c) 第二階段：**

```text
   報警（P(bad)=0.484）：EU(ship)=0.484·(−80)=−38.7；EU(roll)=−20 → 回滾（−20）
   沒事（P(bad)=0.0725）：EU(ship)=0.0725·(−80)=−5.8； EU(roll)=−20 → 全面上線（−5.8）
```

**(d) 倒推與 EVSI：**

```text
   做 probe（未計成本）= 0.31·(−20) + 0.69·(−5.8)
                       = −6.2 + (−4.0) = −10.2
   EVSI = (有 probe 的最佳) − (無 probe 的最佳) = −10.2 − (−16) = 5.8
   ⇒ probe 成本 < 5.8 才值得做。
```

**常見錯路**：(1) 把 (a) 的基準誤當成 roll（−20）而非 ship（−16），算出的 EVSI 會偏大——基準必須取「無資訊時的**最適**行動」，這裡是 ship。(2) 第二階段在「報警」格直接 ship（沒跟 roll 比），漏掉了 probe 的全部價值正來自「報警時改去 roll」。

### 題 2（★★，**[15 分鐘]**）把沉沒成本約掉

一個重構做了一半，**已投入 60 點**（不可回收）。現在「完成它」還要再投 50 點，完成後預期帶來 70 點的長期收益（確定，無不確定性）；「砍掉、回到舊架構」則不再投入、收益 0。

(a) 規範上該完成還是砍掉？算式裡那 60 點該放哪？
(b) 一位同事說「都做一半了，再撐一下把 60 賺回來」。指出他算式裡多了什麼。

### 推演解答

**(a)**

```text
   完成的往前看期望效用 = 70 − 50 = +20
   砍掉的往前看期望效用 = 0
   +20 > 0 ⇒ 該【完成】。已投入的 60 點【不進入算式】，因為它對「完成」與「砍掉」一樣大、約得掉。
```

注意這題的結論是「該完成」——這很重要：忽略沉沒成本**不等於**「半途的東西都該砍」。它只是說「用未來增量算」。這裡未來增量為正（70−50=+20），所以完成；題 1（前一節 worked example 的 80/40 版）未來增量為負（−15），所以砍。**同樣是忽略沉沒成本，結論可能是繼續、也可能是停手——取決於往前看的增量，不取決於過去投了多少。**

**(b)** 他把「賺回那 60」當成完成的理由。但完成的真實收益是未來的 70、代價是未來的 50，淨 +20，與「賺回 60」無關——那 60 回不來，完成也好砍掉也好它都是沉的。他在算式裡多塞了一個「挽回 60」的虛項；剛好這題該完成，所以他**結論對、理由錯**——這種「對的結論配錯的理由」最危險，因為下次未來增量是負的時候，同一套錯理由會讓他繼續燒錢。

### 題 3（★★，**[15 分鐘]**）選項價值的口頭題

不用算，用自己的話回答：

(a) 一個「可一鍵回滾」的部署，比「不可回滾」的部署多值多少？這個「多值的部分」在什麼情況下才被你真正用到？
(b) 為什麼「這次 canary 沒事、所以當初不該浪費時間做 canary」是後見之明的謬誤？

### 推演解答

**(a)** 多值的部分＝選項價值＝「壞情況發生時能轉向」的期望收益。它只在**壞情況真的發生**的那些分支裡被行使（那 18% 報警、或上線後出事要回滾的情況）。在好情況分支裡它確實「沒被用到」——但這不代表它沒價值，正如保險在沒出險的年份「沒用到」不代表保險沒價值。你買的是事前的期望保障，不是事後保證每次都用得上。

**(b)** 因為決策的價值要用**事前的期望**評斷，不能用**事後的單一結果**回頭評斷。canary 的價值是 over 所有可能訊號的加權（報警時讓你避開事故、沒事時讓你放心全面上線）；某一次「剛好沒事」只是落在 82% 的那條分支上，不能反推「當初的期望價值是零」。用單次結果否定事前決策，正是把機運節點當成「我早該知道」——典型的後見之明偏誤（其心理機制見《思考的陷阱》談後見之明）。

## 自我檢核

口頭回答，講不清楚就是還沒懂：

1. 決策樹的三種節點各是什麼？倒推時機運節點和決策節點的處理為什麼**不一樣**（一個取期望、一個取最大）？把這個不對稱講給另一個工程師聽。
2. 為什麼非得「從葉子往回算」、不能「從根往前算」？這跟你寫遞迴的哪個直覺是同一件事？
3. 用自己的話說：canary 的資訊價值（4.5）精確等於什麼？為什麼 canary「技術上有效」不保證「值得做」？（接回 ch07）
4. 沉沒成本為什麼在規範上是「argmax 裡約得掉的常數」？用「對每個未來選項都一樣大」這句話解釋。
5. 「忽略沉沒成本」是不是等於「半途的專案都該砍」？用題 2 與 worked example 兩個結論相反的例子說明差別。
6. 選項價值是什麼？為什麼「結果沒出事、所以當初不該留退路」是錯的推理？
7. 沉沒成本謬誤的錯誤**不在算術**——那它錯在哪一步？「假如這是別人昨天剛丟給我的、我會不會啟動它」這個自我檢測為什麼有效？

## 延伸閱讀

- **Howard Raiffa,《Decision Analysis: Introductory Lectures on Choices under Uncertainty》(1968)**——決策樹與倒推（rollback）的經典入門；想看「先探測再決定」如何系統化成樹，讀它的決策樹章節。Raiffa 也是 ch07 EVSI 的共同奠基者（Raiffa–Schlaifer 1961）。
- **Hal Arkes & Catherine Blumer, "The Psychology of Sunk Cost",《Organizational Behavior and Human Decision Processes》35 (1985): 124–140**——沉沒成本謬誤的代表性實驗論文（劇院季票、滑雪票那些經典情境就出自這裡）；想看「人確實會被沉沒成本綁架」的乾淨證據，讀它的實驗設計（2026-06，期刊與年份為常見引用，連網覆核從略）。
- **Stanford Encyclopedia of Philosophy,「Decision Theory」**（https://plato.stanford.edu/entries/decision-theory/）——序貫決策、動態一致性、與本書其他主題的關係的權威概覽；想把單步決策接到多步的哲學脈絡，讀它的 sequential / dynamic choice 段。
- 沉沒成本與協和號效應的**心理機制**（為何明知不該卻做不到、與損失趨避如何交織）見《思考的陷阱》談沉沒成本與承諾升級；本章只給規範判決（該約掉）與決策樹畫法。
- 當你的序貫選擇面對的是**會反應的對手**（會針對你的回滾策略調整攻擊、或需要「故意鎖死選項」來建立可信承諾）時，單人決策樹不夠了，見《賽局》談重複賽局、承諾與可信威脅——這是 ch16 要正式交棒的邊界。
