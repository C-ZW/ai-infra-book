## 從你已知的出發

先講一個你大概設過幾十次的東西：一個閾值。

**版本一：流量尖峰的自動擴容。** 你給服務設一條 autoscaling 規則——CPU 超過 70% 就加機器。你怎麼定 70% 這個數？你心裡其實在做一個期望成本最小化：閾值定太低，平常養一堆閒置機器，浪費錢；定太高，遇到尖峰時擴得太慢，使用者吃到延遲、甚至超時。你拿歷史流量分布當「自然」，算「在這個分布下，哪個閾值的期望成本最小」。這是一個**漂亮的單人決策問題**——而且 Part I 到 Part IV 整套機器都適用。流量尖峰是雙十一、是某篇文章上了熱門、是一個 cron job 撞在一起。它們**不在乎你的閾值定在哪**。你把閾值從 70% 調到 75%，明天的尖峰不會因此變大一點來「對付」你。自然是冷漠的。

**版本二：一條 rate limiter，擋的是濫用者。** 同樣是設一個閾值——每個 IP 每分鐘最多 100 次請求，超過就限流。表面上跟版本一是同一道題：閾值定太鬆，惡意流量打穿你；定太嚴，誤傷正常使用者。你可能會想「那我也拿歷史濫用流量的分布，算一個期望成本最小的閾值」。

**這裡有一個你不能再忽略的差別：對面那個濫用者，會看你的閾值，然後改他的打法。** 你把單一 IP 的上限壓到 100，他換成 IP 池、每個 IP 打 99 次，剛好踩在你線下。你加上「同一 ASN 聚合計數」，他把流量灑到多個雲廠商。你上行為指紋，他開始模仿正常使用者的節奏。**你每動一次閾值，他的「機率分布」就變一次**——而且是朝著最讓你難受的方向變。

停在這裡想一下這個差別有多本質。版本一裡，那個讓你不確定的東西（流量分布）是**固定的、給定的**，你在一個不動的地形上找最低點。版本二裡，地形本身**會對你站的位置反應**——你一站上某個點，最低點就跑到別處去了。前者你可以用期望值掃過所有閾值、挑期望成本最小的那個；後者你掃出來的那個「最優閾值」，會在你部署上線的當天失效，因為對手把它讀走了。

你早就在工程直覺裡分過這兩種東西，只是沒給它們名字。**容量規劃**、**重試與超時**、**快取大小**、**對自然故障的告警閾值**——這些對面是冷漠的自然，是單人決策。**安全防護**、**反濫用**、**反作弊**、**競價/配額面對會套利的使用者**、**威脅建模**——這些對面是一個會反應的人，是賽局。本章要做的，就是把這條你早有的直覺，升級成一條清楚的界線，並說清楚：越過這條線，這本書的工具會以什麼方式失效。

## 一句話的分界：自然不「針對」你，對手會

把整本書的脊椎拿出來最快。脊椎賭注是這樣的：

```text
脊椎賭注 L =（+200, 0.5; −100, 0.5），單位 NT$
擲一枚公正硬幣：正面贏 200、反面輸 100。
期望值 E[L] = 0.5·(+200) + 0.5·(−100) = +50（ch02 算過）
```

從 ch02 到 ch12，我們對這個賭注做了各種事——算期望值、套凹效用、用展望理論重估。但這一路有一個前提**從來沒被質疑過**：那枚硬幣是**公正**的，而且它**不會看你押哪面**。

現在改一個字。把「公正硬幣」換成「莊家」。

```text
脊椎賭注的兩個世界

  世界 A（自然／單人決策）
    對面是一枚公正硬幣。P(正)=0.5 固定，與你的選擇無關。
    你押正、押反、不押——硬幣的機率都是 0.5。
    → 你可以用期望值 / 期望效用 / 展望理論算「該不該玩」。

  世界 B（對手／賽局）
    對面是一個莊家，他「設」賠率、而且看得到你的傾向。
    你若總押正，他就調整讓正面少賠或少出。
    P(正) 不再是固定的 0.5，而是「莊家對你的反制」的函數。
    → 沒有一個固定的機率可以代進期望值；你算出的最優押法，
      會被莊家讀走、然後反制掉。
```

差別只有一行：**世界 A 的機率是給定的常數，世界 B 的機率是對手對你策略的回應。** 這一行，就是這本書的邊界。

我把它濃縮成一句你可以記一輩子的話：**自然不會「針對」你，對手會。**

- 自然（natural uncertainty）：硬幣、流量尖峰、盛行率、硬碟故障率、地震。它有機率分布，這個分布**不隨你的決策改變**。你的整套單人決策機器（期望值 → 效用 → 貝氏 → 資訊價值）就是為這種對手設計的。
- 對手（strategic opponent）：莊家、攻擊者、套利者、競價對手、談判桌另一邊的人。他有自己的目標，而且**會觀察你的策略、再選對他最有利的回應**。你的「機率分布」對他來說不是固定的——它是你選了什麼之後，他**最佳化**出來的反應。

這就是為什麼 ch04 講「不確定」時，我特別把 Knight 的「風險 vs 不確定」跟「對手」分開講。Knight 的不確定是**機率測不出來**（新依賴沒有歷史故障數據）——但它仍然是一個「不針對你」的自然，只是你不知道它的分布。對手是另一個維度的東西：**就算機率測得出來，只要它會對你反應，你代進去的那個機率下一秒就過期了。** 模糊（ambiguity）是「我不知道分布」；賽局是「分布會看我出牌」。兩件不同的事。

## 為什麼期望值對「會反應的對手」會系統性失靈

這一節值得你停五分鐘，因為它是本章的核心洞見，而且它是**質變不是量變**。

單人決策的整套邏輯，骨架是這一步（ch05 給過）：

```text
單人決策的核心動作（對自然）
   a* = argmax_a  Σ_θ  P(θ) · u(a, θ)
        └─────────────┬──────────────┘
         在「狀態 θ 的固定機率分布 P(θ)」下，
         挑出期望效用最高的行動 a*
```

這一步能成立，**完全靠 P(θ) 是個跟 a 無關的常數**。你掃過所有行動 a，每個 a 配上同一張機率表 P(θ)，算期望效用，挑最高的。整個過程像在一張固定的地形圖上找最高點。

現在把自然換成對手。對手不是一個「狀態 θ」，他是一個**也在做 argmax 的人**。他的行動會構成你的「狀態」，而他選哪個行動，取決於**他預期你會選什麼**。於是那張機率表 P(θ) 變成了 P(θ | a)——對手的「分布」是你行動的函數。式子崩了：

```text
對手不是固定的 P(θ)，而是一個也在 argmax 的人

   你想算：   a* = argmax_a  Σ_θ  P(θ | a) · u(a, θ)
                                   └──┬──┘
                              對手的分布依賴你的 a

   但對手同時在算：
              θ* = argmax_θ  (對手自己的效用 u_對手(a, θ))
                              而他的 a 是「他猜你會選的」

   → 你要算 a 得先知道 P(θ|a)，
     但對手要決定 θ 得先猜你的 a，
     你猜他、他猜你、你猜他猜你……無窮回退。
```

看到那個無窮回退了嗎？**這不是一個更難算的期望值，這是一個結構不同的問題。** 單人決策是「在固定地形上找最高點」，這是個最佳化問題，有現成答案。賽局是「我站的點會改變地形、而地形也在改變我該站的點」——它不是最佳化，它要找一個**互相一致的不動點**：一組策略，使得「在對方這樣選的前提下，我這樣選是最優的；而在我這樣選的前提下，對方那樣選也是最優的」。沒有人想單方面改變。這個不動點就是**納許均衡（Nash equilibrium）**。

我特別強調「不動點」這三個字，因為這正是質變所在。單人決策的解是 argmax 的輸出（一個最佳化的答案）；賽局的解是一個**互相最佳化的固定點**（一個方程組的自洽解）。把賽局問題硬用單人期望值解，等於把一個求不動點的問題，當成一個求極值的問題——你會得到一個「假設對手不動」的最優解，然後對手一動，它就錯了。

均衡概念怎麼求、混合策略長什麼樣、不動點為什麼一定存在（Nash 用 Brouwer/Kakutani 不動點定理證的），這些是《賽局》的主體，本章不展開（見《賽局》談納許均衡與混合策略）。本章只要你帶走一件事：**當對面會反應，你要找的不是「最優行動」，是「互相最優的不動點」——而期望值機器只會給你前者。**

### 混合策略其實是「對隨機對手的單人決策」

有一個地方賽局跟本書的工具會短暫握手，值得點一下，因為它能幫你看清楚邊界。

賽局裡常出現**混合策略（mixed strategy）**——你不固定出某一招，而是**按某個機率分布隨機出招**（猜拳不固定出石頭，而是各 1/3）。為什麼要隨機？因為一旦你的出招可預測，會反應的對手就能反制你；隨機化是讓對手「無法針對」的手段。

這裡的連結是：**一旦對手的混合策略「定下來了」（比方均衡裡他就是各 1/3），那麼從你的角度，他就退化成了一個「固定機率分布的自然」**——你面對的變回一個單人決策，用本書的期望值就能算「對他這個固定隨機分布，我的最優回應是什麼」。

```text
賽局與單人決策的接縫

  完整賽局：你和對手都在選策略，互相反應 → 求納許均衡（不動點）

  但「固定住對手的混合策略」這一刀切下去：
    對手 = 一個給定機率分布的隨機產生器（不再對你反應）
    → 你這一邊變回單人決策：a* = argmax_a Σ P(對手出招)·u(a,·)
    → 這時 ch02–ch07 的整套期望值機器又能用了

  關鍵：均衡之所以是均衡，正是因為
    「在對方那個分布下，你的最優回應 也 讓對方那個分布成為最優」
    ——兩個單人決策互相咬合的不動點。
```

換句話說，納許均衡可以看成**兩個單人決策問題互相咬合**：在均衡點上，每一邊都在「對方策略固定」的假設下做自己的單人最佳化，而且這個假設恰好自洽。本書教你的是「咬合的那半邊」；《賽局》教你的是「怎麼讓兩半邊咬合、不動點存不存在、怎麼找」。這就是「決策↔賽局是同一枚硬幣的兩面」的精確意思——不是兩個無關的領域，是同一個最佳化動作，差在「對方會不會也在做同一件事」。

## 越界三件事：承諾、訊號、謝林點

單人決策的世界裡，有些動作根本沒有意義；它們只在「有另一個會推理的對手在看」時才存在。我各用一句話帶過，讓你認得它們是「越界訊號」——一旦你的問題裡冒出這些東西，你已經在賽局裡了，該交棒。深入都指向《賽局》。

**承諾（commitment）。** 在單人決策裡，「綁住自己未來的手腳」永遠是壞事——選項越多越好（這正是 ch15 的選項價值，option value）。但在賽局裡，**故意減少自己的選項可以是優勢**：把退路燒掉，讓對手相信你「不得不」強硬，於是對手讓步。經典畫面是把橋燒了讓自己的軍隊無法撤退——對單人決策這是自殘，對賽局這是讓威脅可信。工程版：公開承諾一個 deprecation 日期（綁死自己，逼下游遷移）、不可逆的 API 棄用。**「自縛手腳反而更好」這個念頭一旦合理，你就出界了。**

**訊號（signaling）。** 當對手不知道你的「型」（你是強是弱、你的成本是高是低），你可以用一個**對你來說便宜、對假裝者來說昂貴**的動作來可信地傳遞資訊。單人決策裡你不會「為了讓某個觀察者相信什麼」而行動——自然不會被你說服。賽局裡這是核心：你採取一個只有「真材實料的你」才付得起的動作，讓對手據此更新對你的信念。工程版：開源你的核心、跑一個誰都看得到的長期 SLA 紀錄——這些是對「會評估你的對手/客戶」發的可信訊號。

**謝林點（focal point / Schelling point）。** 當兩邊**想協調但不能溝通**，某些選項會因為「顯著、自然、雙方都猜得到對方會猜」而被同時選中——不是因為它更優，是因為它**突出**。經典例子：兩個人要在一座沒約地點的城市碰面，很多人會不約而同去最有名的地標或正午的車站大廳——不是那裡最好，是那裡「大家都會想到大家都會想到那裡」。這個概念出自湯瑪斯·謝林（Thomas Schelling）1960 年的《The Strategy of Conflict》（謝林 2005 年與 Robert Aumann 共獲諾貝爾經濟學獎；此處為背景，細節以《賽局》為準）。單人決策裡「顯著性」毫無意義——自然不會跟你協調。工程版：大家不約而同把預設埠號設成某個「整數」、HTTP 200 當「成功」的共識、版本號從 1.0.0 起跳——這些協調的錨點都是謝林點。**「我們需要一個雙方都猜得到對方會選的點」這個需求一出現，你在賽局裡。**

把這三個放一起看，它們有一個共同特徵：**都假設對面有一個會推理、會更新信念、會跟你互動的腦。** 自然沒有腦。所以這三件事在純單人決策裡全都不存在。它們是最好的「越界探測器」。

## worked example：同一個「設一個閾值」問題的兩個版本

這是本章的核心 worked example。同一句話——「設一個閾值」——對「自然」和「對手」是兩道結構不同的題。我把兩邊都一步步算給你看，重點是讓你**親眼看到單人期望值在版本二會給出一個錯誤的、會被對手讀走的答案**。

設定：你要為一個 API 端點設一條「單一來源每分鐘最多 N 次請求」的限流線。超過就擋。簡化成只有兩個候選閾值供比較：**寬鬆 N=100** 與 **嚴格 N=20**。代價有兩種：擋到正常使用者（誤殺，每次成本 c_誤 = 1）、放過該擋的壞流量（漏網，每次成本 c_漏 = 5，漏網比誤殺貴，因為它直接傷到系統）。

### 版本一：對手是「自然」（流量尖峰／無意識爆量）

這裡「壞流量」是無意識的——某個寫壞的 client 在迴圈裡狂打、一次行銷活動帶來的尖峰。它**不會因為你設了哪個閾值而改變打法**。假設歷史數據給了固定的分布：

```text
版本一：對自然，固定分布（與你的閾值無關）

  每分鐘流量裡：
    正常使用者：通常 ≤ 20 次/分；只有 5% 的正常使用者會短暫衝到 20–100 次
    無意識爆量：固定就是「狂打到 200 次/分」，不管你閾值設多少

  閾值 N=100（寬鬆）：
    誤殺正常使用者 ≈ 0（正常人很少破百）        → 誤殺成本 ≈ 0·c_誤 = 0
    放過 100 次爆量（200 裡擋掉 100、放過 100） → 漏網成本 = 100·c_漏 = 100·5 = 500
    期望成本 ≈ 0 + 500 = 500

  閾值 N=20（嚴格）：
    誤殺 5% 衝到 20–100 的正常人，設這群每分鐘有 40 人次被誤殺
                                                → 誤殺成本 = 40·c_誤 = 40·1 = 40
    放過爆量 (200 裡擋掉 180、放過 20)          → 漏網成本 = 20·c_漏 = 20·5 = 100
    期望成本 ≈ 40 + 100 = 140

  → N=20 期望成本 140 < N=100 的 500，選 N=20。✓
```

這個答案是**對的、而且穩定的**。為什麼穩定？因為那個爆量的 client 不會看你選了 N=20 然後改成「每分鐘剛好打 20 次」來鑽你——它是個無腦迴圈，明天還是狂打 200。你算出的 N=20 明天照樣最優。這就是單人決策的好處：**地形不動，最高點就一直在那。**

### 版本二：對手是「攻擊者」（會讀你的閾值、再反制）

同一條限流線，但壞流量的來源換成一個**會適應**的濫用者。他的目標是「在不被擋的前提下，盡量多打」。關鍵：**他會把請求量壓到剛好踩在你閾值線下。**

```text
版本二：對攻擊者，分布是「你閾值的函數」（他會反制）

  你設 N，攻擊者就把每來源的請求量壓到 N（剛好不被擋）、
  再用足夠多的來源把總量補回去（IP 池／多 ASN／多雲廠商）。
  → 不管你 N 設多少，他「每來源打 N 次、不被擋」永遠成立。

  你掃單人期望值，會掃出什麼？
    你會以為：N 越小，每個來源能打的越少，漏網越少 → 把 N 壓到很小最好。
    例如你照版本一的邏輯選 N=20，算出「漏網被壓到 20/來源」、期望成本看起來很低。

  但這是「假設攻擊者不動」算出來的——他動了：
    你把 N 壓到 20，他就用 10 倍的來源、每個打 19 次 → 總漏網量反而 不變甚至更大，
    而你把 N 壓到 20 同時 誤殺了一大票衝到 20–100 的正常使用者（成本 40 那一塊狂漲）。
    → 你「最優化」出來的 N，淨效果是「自己誤殺、對手照打」。

  正確的問法（賽局）：
    這不是「在固定分布下挑 N」，而是
    「我選 N、他選(每來源量, 來源數)來反制、我再選…」的不動點。
    解不是一個閾值數字，而是一組互相最優的策略
    （通常要配合『提高他增加來源的成本』——驗證、信譽、行為指紋——
      把問題從『純閾值』拉回一個他無法零成本反制的維度）。
    這已經是賽局的地盤（見《賽局》談策略互動與機制設計）。
```

把兩個版本並排，差別一目了然：

```text
            版本一（自然）              版本二（對手）
  分布      固定 P(壞流量)             P(壞流量 | 你的 N)，隨你的選擇而變
  解的型態  argmax_N 期望成本（極值）   互相最優的不動點（均衡）
  穩定性    明天照樣最優               你一上線，對手就讀走、反制
  正確工具  本書（ch02–ch15）          賽局（見《賽局》）
  用錯的後果 —                         你「最優化」自己誤殺、對手照打
```

**自我複核**：版本一的兩個期望成本——N=100 是 0 + 100·5 = 500，N=20 是 40·1 + 20·5 = 40 + 100 = 140，140 < 500，選 N=20，算術正確。版本二的重點不在算出一個漂亮的數字，而在**證明「用版本一那套去算版本二會給出一個會被反制的錯誤答案」**——這正是 T13 的核心：把會反應的對手當固定機率，會系統性低估他。注意版本二我**故意沒有**給出一個「正確的最優 N」，因為單靠一個閾值數字本來就不是賽局問題的解；解是一組策略加上「改變賽局結構」（提高對手反制的成本）的機制設計，那是《賽局》的事。

這就是交棒的精確時刻：**當你發現「我算出的最優解，會因為對手讀走它而失效」，你手上的就不再是決策問題，而是賽局問題。** 把鑰匙交過去。

## 邊界不是非黑即白：同一個系統會在兩邊之間漂移

我得誠實補一句，免得你把這條界線用得太硬。現實系統裡，「自然」和「對手」常常是**同一個系統在不同時刻的兩種面貌**，界線會漂移：

- 一個 API 今天的流量 99% 是無意識的正常使用（自然），直到有人發現它能被套利——那一刻，同一個系統從決策問題**變成**賽局問題。你的脊椎裡那個「要不要出現了會套利的對手」的時刻，往往來得很安靜。
- 反過來，一個被攻擊的系統，當你用機制設計（驗證、配額、信譽）把對手反制的成本拉高到「他不值得反應」之後，他的最優策略**退化成固定**——你又可以暫時用單人期望值近似他了（接回前面「固定住對手的混合策略」那一刀）。

所以更精準的判準不是「這個系統是決策還是賽局」，而是一個你每次設計時都該問的問題：**「我這個選擇，會不會改變對面那個東西的最優反應？」** 會 → 賽局；不會 → 決策。流量尖峰不會因為你的閾值而改變它的最優反應（它沒有最優反應，它無腦）；攻擊者會。這一問，比「分類這個系統」可靠得多，因為它跟著**對手有沒有腦**走，而不是跟著系統的名字走。

```text
判準（每次設計時自問一次）

  「我這個選擇，會不會改變對面那個東西的『最優反應』？」

     不會  → 對面是自然（沒有最優反應，或反應與你無關）
            → 單人決策：ch02–ch15 的整套機器適用

     會    → 對面是對手（會讀你的選擇、再反制）
            → 賽局：求互相最優的不動點，交棒《賽局》

  注意：同一系統會在兩邊漂移。出現「會套利的人」那一刻就越界了。
```

這也正好把本書的中央張力拉到一個新層面。前面十二章的張力是「規範（該怎麼選）vs 描述（人實際怎麼選）」。本章的張力換了個軸：**單人理性 vs 互動理性**。你可以是個完美的單人決策者——每個期望值都算對——卻在面對一個會反應的對手時系統性地輸，因為你解錯了問題的**種類**。理性不只是「算得對」，還包括「認得出你在哪一種問題裡」。這是 ch17 要接手的線。

## 直覺的陷阱

決策↔賽局的邊界，是工程師最容易踩錯的地方之一，因為單人期望值太好用、用順手了就會想套到所有問題上。守住 landscape 的 T13。

| 陷阱 | 錯誤直覺長什麼樣 | 它在哪一步把你帶溝裡 | 正確版（怎麼自我察覺） |
|---|---|---|---|
| **把會反應的對手當固定機率**（具名陷阱，T13） | 「我拿攻擊流量的歷史分布算一個期望成本最小的閾值就好。」 | 你把 P(壞流量) 當成跟你的選擇無關的常數，代進 argmax——但它其實是 P(壞流量 \| 你的閾值)。你解的是「假設對手不動」的最優，對手一動就失效。 | 對手的分布是你策略的函數。**徵兆**：你算出一個「最優解」，但隱隱覺得「上線當天就會被繞過」——那個直覺是對的，你在用單人工具解賽局。 |
| **以為賽局只是「更難的決策」** | 「賽局就是對手比較聰明、機率比較難估的決策嘛，多算幾層就好。」 | 你以為差別是量（更難算），其實是質（不同的解概念）。單人決策求極值（argmax），賽局求不動點（均衡）。多算幾層期望值不會收斂到均衡——你猜他、他猜你，無窮回退。 | 均衡是**互相最優的自洽點**，不是「算得更深的 argmax」。**徵兆**：你的推理變成「我猜他猜我猜他……」——這個無窮回退就是你越界的信號，該換工具。 |
| **以為混合策略「就是個機率分布、跟自然一樣」** | 「對手按某機率出招，那不就跟硬幣一樣？我代期望值就好。」 | 對——但**只在均衡已經定下來、對手的混合策略固定之後**才對。在那之前，那個機率分布本身是賽局解出來的、會隨你而變。你跳過了「均衡怎麼來的」直接用它的結果。 | 固定住的混合策略確實能當「自然」算（那一刀），**但那個分布是賽局給的，不是你能拿歷史數據估的**。徵兆：你想「估」對手的出招機率——自然的機率可以估，理性對手的均衡機率要解。 |
| **把「承諾／自縛手腳」當成壞事** | 「選項當然越多越好（option value），綁死自己一定虧。」 | 這在單人決策裡完全正確（ch15），但你把它無條件帶進了有對手的世界。對會推理的對手，**可信地減少自己的選項**能改變他的最優反應，反而有利。 | 選項價值是**單人**真理；有對手時，承諾能讓威脅/讓步可信。徵兆：你發現「如果我能讓對方相信我別無選擇，他就會讓步」——這個念頭只在賽局裡成立，你已經出界。 |

最值得記的一句自我察覺：**只要你的推理裡開始出現「他會怎麼想我會怎麼做」這種互相猜測的回退，你就已經不在這本書的地盤了。** 自然不會猜你。一旦對面開始猜你，交棒。

## 紙上推演

全部紙上推演、口頭自答，不需要跑任何東西。

### 推演題

**第 1 題（★★，[15 分鐘]）判斷下列各情境屬「單人決策（對自然）」還是「賽局（對對手）」，並用一句話說出判準。**

1. 你給資料庫連線池設大小，依據是歷史 QPS 分布。
2. 你給反爬蟲設請求頻率閾值，對面是會調整爬取節奏的競品爬蟲。
3. 你決定要不要為一次罕見的硬碟批次故障多買備援。
4. 你在一場 SLA 賠償談判裡決定開價，對面是會還價的客戶。
5. 你給金絲雀部署設「錯誤率超過 X% 就自動回滾」的閾值，X 依據過往發布的錯誤率分布。
6. 你設一個促銷折扣碼的使用上限，對面是會用腳本批量套利折扣的羊毛黨。

**第 2 題（★★★，[20 分鐘]）口頭題：解釋「為什麼自然的不確定可以用期望值、會反應的對手不行」。** 要求你不只說結論，而是說清楚「期望值那一步在對手情境裡具體斷在哪」。

**第 3 題（★★，[15 分鐘]）舉一個你系統裡「從決策變成賽局」的時刻**——某個本來對著無意識流量的閾值/規則，在某一刻出現了「會套利的對手」之後，為什麼舊的「最優設定」失效了？你當時是怎麼（或該怎麼）換工具的？

---

### 推演解答

**第 1 題解答。** 判準只有一句：**「我這個選擇，會不會改變對面那個東西的最優反應？」**

```text
  1. 連線池大小 ← 歷史 QPS 分布          → 單人決策
     （QPS 不會因為你把池子調大就「針對」你變大；自然。）
  2. 反爬蟲閾值 ← 會調整節奏的競品爬蟲    → 賽局
     （爬蟲會讀你的閾值、改節奏踩線；對手有最優反應。）
  3. 罕見硬碟批次故障的備援              → 單人決策
     （硬碟不會因為你買了備援就改變故障率；自然，且這是 ch07 資訊/ch15 序貫的料。）
  4. SLA 賠償談判開價 ← 會還價的客戶      → 賽局
     （對面會對你的開價反應、還價；談判是賽局原型。）
  5. 金絲雀自動回滾閾值 ← 過往錯誤率分布  → 單人決策
     （新版本的 bug 不會「針對」你的回滾閾值；自然。這是 ch15 序貫決策。）
  6. 折扣碼使用上限 ← 會套利的羊毛黨      → 賽局
     （羊毛黨會讀上限、用腳本剛好踩線繞過；對手。機制設計題。）
```

常見錯路：把第 3、5 題誤判成賽局，因為「故障/bug 感覺像在跟我作對」。但故障和 bug **沒有腦、沒有目標、不會讀你的設定再反制**——它們是自然，只是讓你不爽而已。判準看的是「對面有沒有會最佳化的腦」，不是「我有沒有被它害到」。反過來，第 2、4、6 的共同特徵是對面**有一個會看你出牌再出牌的人**。

**第 2 題解答。** 完整版要點：

期望值的核心動作是 `a* = argmax_a Σ_θ P(θ)·u(a,θ)`。這一步能成立，**唯一靠的就是 P(θ) 是一個跟你的行動 a 無關的固定分布**——你才能對每個候選 a 配同一張機率表、比期望效用、挑最高。

對「自然」，這個前提成立：硬幣的 0.5、QPS 的分布、故障率，都不會因為你選了哪個 a 而改變。地形不動，argmax 找得到、而且明天還在。

對「會反應的對手」，這個前提**斷掉**：對手的「分布」是你行動的函數 P(θ|a)——你選 a，他看到，再選對他最有利（對你最不利）的回應。於是：

- 你要算 a 得先知道 P(θ|a)，但 P(θ|a) 取決於對手猜你選什麼 a；
- 對手要決定回應得先猜你的 a，而你的 a 又取決於你猜他的回應……
- → 無窮回退，沒有「固定的 P(θ)」可以代進 argmax。

所以斷點是精確的：**期望值假設分布固定，對手讓分布變成你策略的函數，這個假設就垮了。** 解法不再是求極值（argmax），而是求一個互相最優的不動點（納許均衡）——這是質變。

常見錯路：回答「因為對手比較聰明/難預測」。這沒抓到要害——難預測的自然（Knight 不確定、模糊）你還是可以用期望效用框架近似（ch04）。真正的要害不是「難」，是**分布依賴你的選擇**這個結構，它讓 argmax 這一步在數學上失去定義對象。

**第 3 題解答（示範一條典型答案，你的會不同）。** 一個常見的真實版本：

```text
  從決策變成賽局的時刻：API rate limit 遇上薅羊毛

  起點（決策）：
    某 API 設了「每 token 每分鐘 60 次」的限流，依據是正常客戶的歷史用量分布。
    這時對面全是無意識的正常使用——閾值 60 是個漂亮的單人期望成本最小解。

  越界的那一刻（出現會套利的對手）：
    有人發現這個 API 的回應可以被轉賣/濫用，開始用大量免費帳號、
    每個帳號剛好打 59 次/分（踩在線下），用帳號數量把總量補上去。
    → 你的「最優閾值 60」失效了：壓低閾值只會誤殺正常客戶，
      對手換更多帳號照打。你解的不再是「挑一個閾值」。

  換工具（從單人期望值 → 機制設計）：
    不再糾結閾值數字，而是改變賽局結構——提高「多開帳號」的成本：
    手機驗證、付費門檻、帳號信譽分、行為指紋。
    把問題從「對手能零成本反制的純閾值」拉到「對手反制要付代價」的維度。
    （這已經是《賽局》談機制設計的地盤。）
```

常見錯路：在越界之後，繼續「再調一次閾值」——這是把賽局問題當決策問題硬解的典型症狀，你會在「調閾值 → 被繞過 → 再調閾值」的迴圈裡轉圈，因為**你動的維度正是對手能零成本反制的維度**。跳出迴圈的方法是換維度（讓對手的反制變貴），那是機制設計、是賽局。

## 自我檢核

口頭自答，講得出來才算過關。優先問「為什麼」。

1. 用一句話說出單人決策和賽局的分界——**為什麼**「自然不針對你、對手會」就足以分開這兩類問題？
2. 期望值的核心動作 `argmax_a Σ P(θ)·u(a,θ)` 裡，**哪一個前提**在面對會反應的對手時會斷掉？斷掉之後，解的「型態」從什麼變成什麼？
3. 「賽局只是更難的決策」錯在哪？「更難」是量變，賽局的差別**為什麼**是質變？（提示：argmax vs 不動點。）
4. 混合策略什麼時候可以「當成自然的固定機率」用本書的期望值算、什麼時候不行？那個固定的機率分布是**估**出來的還是**解**出來的？
5. 承諾（故意減少自己的選項）在單人決策裡是壞事（違反 option value），**為什麼**在賽局裡反而可能是優勢？舉一個工程例子。
6. 你怎麼用一個問題就判斷「這是決策還是賽局」？為什麼這個問題比「給系統分類」可靠？
7. 為什麼說「同一個系統會在決策和賽局之間漂移」？什麼事件會讓一個決策問題**變成**賽局問題？反過來，什麼做法能讓賽局問題**退化回**決策？
8. 本章把全書中央張力換了一個軸（從「規範 vs 描述」換到「單人 vs 互動」）——你怎麼用自己的話講清楚：一個每個期望值都算對的人，**為什麼**仍可能在賽局裡系統性地輸？

## 延伸閱讀

- **Thomas Schelling，《The Strategy of Conflict》（1960）**：謝林點、承諾、可信威脅的原典；讀第 2–3 章對「承諾為何是優勢」最有感。謝林 2005 與 Robert Aumann 共獲諾貝爾經濟學獎（背景，2026-06）。
- **John von Neumann & Oskar Morgenstern，《Theory of Games and Economic Behavior》（1944；期望效用公理化在 1947 二版附錄）**：賽局與單人效用的共同源頭——同一本書同時奠定了你前面學的 vNM 效用（ch03）和賽局。讀它能看見「決策↔賽局是同一枚硬幣兩面」不是比喻，是字面史實。
- **Stanford Encyclopedia of Philosophy，「Game Theory」條目**（plato.stanford.edu/entries/game-theory/）：納許均衡、不動點、混合策略的嚴謹但可讀的入門；你想知道「均衡為什麼一定存在」時看這條（深入指向《賽局》全書）。
- **本套姊妹書《賽局：當理性互相碰撞》**：本章交棒的去處——納許均衡的求解、混合策略、重複賽局與承諾、機制設計，全在那裡（見《賽局》談策略互動與機制設計）。本章只負責立界碑、不展開。
- **回看本書 ch04（風險不是不確定）與 ch15（序貫決策）**：ch04 幫你分清「模糊（分布測不出）」和「賽局（分布會反應）」是兩件不同的事；ch15 的選項價值（option value）正是本章「承諾為何在賽局裡反而有利」的對照組——同一個動作，在單人世界是資產、在賽局世界可能該主動放棄。
