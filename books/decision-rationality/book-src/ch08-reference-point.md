注意這張圖的形狀變了。前兩個 Part 我們在搭一座橋——從「該怎麼選」的最小公設，一步步搭到貝氏決策與資訊的價值。那座橋很美，美在它幾條無害的公理（完備、遞移、連續、獨立）就逼出唯一的演算法：最大化期望效用。現在我們要走到橋的另一端，回頭看看橋上**真正走過的人**留下的腳印——而那些腳印，沒有一個踩在橋面正中央。

我認為 Part III 是整本書最反直覺、也最值得一個工程師停下來的一段。因為它不是在說「人會犯錯」。它在說一件更難消化的事：**人偏離那座漂亮的橋，偏得很穩定、很有方向、可以寫成另一套數學。** 那套數學就是展望理論（prospect theory），本章先拆它的第一根、也是最底層的支柱：參考點。

## 從你已知的出發

先講一個你每季都在經歷、卻可能沒當成「決策理論」看的東西：SLA 的 9。

假設你維運的服務上一季可用度做到 99.9%。這一季掉到 99.5%。三個 9 變兩個半，數字上是 0.4 個百分點。你心裡那一刀，痛。你會開檢討會、會被問「發生什麼事」、會花一整個 sprint 把它救回去。

現在換一個情境。同一個服務，上一季是 99.1%，這一季升到 99.5%。同樣是 0.4 個百分點，方向相反。你的感受呢？大概是「不錯，有進步」，輕飄飄地過去，可能連 standup 都懶得提。

停一下。**這兩個情境的「最終狀態」是同一個——99.5%。** 一個讓你開檢討會，一個讓你點個頭就過。如果你是按「最終可用度狀態」估值的——按 ch03 那套 u(最終財富) 的記帳法——這兩件事的效用應該一模一樣，因為終點一樣。但你的反應天差地別。

差在哪？差在你心裡那個「零」放在哪。第一個情境，你拿 99.9% 當零，99.5% 是**損失**。第二個情境，你拿 99.1% 當零，99.5% 是**收益**。同一個終點，從不同的零看出去，一個是賠、一個是賺，估值完全不同。

這就是參考點（reference point）。你不是在對「99.5% 這個狀態」估值，你是在對「相對於上次水準的變化」估值。績效考核是「比去年」不是看絕對值；你對一次 regression 的恐懼遠大於對「多上線一個功能」的渴望；你判斷一次發布「成功還失敗」用的是「達標沒有」而不是「絕對交付了多少」——這些全都是同一件事：**你的腦子記帳記的是變化，不是狀態。**

工程師最容易低估這件事，因為它太自然了，自然到你不覺得它是個「選擇」。但它是。下一節我們把它跟 ch03 那套規範記帳法擺在一起，你會看見一道清楚的裂縫。

## 兩套記帳系統：u(狀態) 對上 v(變化)

回到脊椎賭注——ch01 立起、ch02 算過期望值、ch03 算過期望效用的那一個：

```text
脊椎賭注 L =（+200, 0.5; −100, 0.5），單位 NT$
擲一枚公正硬幣，正面贏 200、反面輸 100。
期望值 E[L] = 0.5·(+200) + 0.5·(−100) = +50（ch02）
```

ch03 是怎麼記這筆帳的？它用一條凹效用 u(x)=√(x+W)，財富基準 W=100，把賭注的兩個結果**換算成最終財富**再估值：

```text
ch03 的記帳法：u(最終財富)
  接賭、贏 → 最終財富 = 100 + 200 = 300 → u = √300 = 17.32
  接賭、輸 → 最終財富 = 100 − 100 = 0   → u = √0   = 0
  不賭     → 最終財富 = 100             → u = √100 = 10

  EU(接賭) = 0.5·17.32 + 0.5·0 = 8.66 < 10 = u(不賭)
  ⇒ 拒絕（風險趨避，ch03 已算）
```

注意這套記帳的核心動作：它把「+200」「−100」這些**變化量**，先加到你的家底 W=100 上，變成「最終你手上有多少錢」（300、0、100），再對那個**最終狀態**估值。效用是「你站在哪」的函數，不是「你往哪走了一步」的函數。

展望理論說：人不是這樣記的。人記的是**你往哪走了一步**。它換了一個記帳單位——不是最終財富 u(x+W)，而是相對參考點的變化 v(Δ)：

```text
展望理論的記帳法：v(變化)，參考點＝現狀（你現在的 100，當成零）
  接賭、贏 → 變化 = +200（一筆「得」）→ v(+200)
  接賭、輸 → 變化 = −100（一筆「失」）→ v(−100)
  不賭     → 變化 =    0（沒動）        → v(0) = 0

  ⇒ 評估的是 v(+200) 對上 v(−100)，不是 √300 對上 √0。
```

兩套系統的差別，不在「凹不凹」，而在**坐標原點放哪**。ch03 把原點釘死在「財富 = 0」（一無所有），所有狀態都從那裡量起；展望理論把原點放在「你現在站的地方」（現狀），所有東西都從那裡量得失。一個是絕對坐標，一個是相對坐標。

這個差別聽起來小，後果很大。我先給你一個會卡住直覺的觀察，下一節用數字攤開：在 ch03 的絕對坐標裡，「你現在有多少家底 W」會直接改變你接不接賭（基準表特別挑 W=100 而不是 W=1000，就是因為 W 一變，結論就從「拒絕」翻成「接受」——這正是絕對坐標的特徵）。而在展望理論的相對坐標裡，只要參考點是現狀，「+200 對上 −100」這筆帳跟你家底多少**幾乎無關**——你算的永遠是「從這裡往上 200、往下 100」。

```text
裂縫的第一道縫（本章只立、ch09 才算到底）

  ch03（規範）：對 u(x+W) 估值      ← 你站在哪（絕對狀態）
  展望（描述）：對 v(Δ) 估值        ← 你往哪走了一步（相對變化）

  兩套都得出「拒絕脊椎賭注」——但走的是不同的路：
    ch03   靠「凹」（邊際遞減：贏到的第 200 元沒輸掉的第 100 元那麼重）
    展望   靠「失比得更陡」（損失趨避，ch09 的主角）

  同一個拒絕、兩種機制。這就是 ch12 要回答的：
  「描述」和「規範」殊途同歸，到底證明了什麼？
```

本章的工作到此其實只有一句話：**把估值的對象從「狀態」搬到「變化」。** 損失趨避（失更陡）是 ch09，機率權重也是 ch09，框架是 ch10。但這一搬，是後面三章所有戲劇的地基——沒有參考點，就沒有「得」和「失」之分，沒有得失之分，就沒有損失趨避、沒有反射效應、沒有框架。所以我把它單獨放一章。

## worked example：同一個終點，兩個參考點，兩種估值

光說「人記變化不記狀態」還太抽象。我們用一個乾淨的數字把它逼出來。這個例子刻意**不用**脊椎賭注（脊椎賭注留到 ch09 用損失趨避重算），先用一個沒有不確定性的純記帳例子，把「參考點依賴」這件事本身看清楚。

情境：一筆操作之後，你**最終手上確定有 NT$1100**。沒有賭、沒有機率，就是個確定的終點。問題是：這 1100 對你來說，是一筆「得」還是一筆「失」？

答案完全取決於你拿什麼當零。

```text
最終財富固定 = 1100，從兩個不同參考點記帳

  情境 A：參考點 = 1000（你本來預期會有 1000）
    變化 = 1100 − 1000 = +100  → 這是一筆「得 100」
    感受：賺到了，開心。

  情境 B：參考點 = 1200（你本來預期會有 1200，臨時被砍了一刀）
    變化 = 1100 − 1200 = −100  → 這是一筆「失 100」
    感受：虧了，難受。

  ⇒ 同一個 1100，A 是賺、B 是虧。終點一樣，估值相反。
```

現在把這兩種記帳，跟 ch03 的「u(最終財富)」記帳並排，看清楚它們是不同的系統。我用 ch03 的基準效用 u(x)=√(x+100) 把絕對記帳也算出來（這裡 x 是淨變化、家底 100 已含在公式裡，最終財富 1100 對應 x=1000）：

```text
三套記帳並排（終點都是「最終 1100」）

  ┌──────────────────┬─────────────┬───────────────────────────┐
  │ 記帳系統         │ 估的是      │ 1100 這個終點的值          │
  ├──────────────────┼─────────────┼───────────────────────────┤
  │ ch03 規範 u(x+W) │ 最終狀態    │ u = √(1000+100) = √1100    │
  │                  │（絕對坐標） │   ≈ 33.17（一個固定的數）  │
  ├──────────────────┼─────────────┼───────────────────────────┤
  │ 展望 v，參考 1000│ 變化 +100   │ v(+100)：一筆「得」        │
  │                  │（相對坐標） │   → 估成正值               │
  ├──────────────────┼─────────────┼───────────────────────────┤
  │ 展望 v，參考 1200│ 變化 −100   │ v(−100)：一筆「失」        │
  │                  │（相對坐標） │   → 估成負值（且更陡，ch09）│
  └──────────────────┴─────────────┴───────────────────────────┘

  複核 √1100：33² = 1089，33.2² = 1102.24，故 √1100 ≈ 33.17 ✓
```

讀這張表的重點不在那個 33.17（它是個準確但無關緊要的數），重點在**它只有一個值**。ch03 的記帳法看「最終 1100」就是 √1100，跟你「本來預期多少」毫無關係——它不知道也不在乎你的參考點，因為它根本沒有參考點這個零件。

而展望理論的兩行，**同一個 1100 給出符號相反的估值**：參考點 1000 時是正的「得」，參考點 1200 時是負的「失」。差別純粹來自「你把零放哪」。這就是參考點依賴（reference dependence）的精確意思：

> 兩個在「最終財富」上完全相同的結果，可以因為參考點不同而被估成一得一失——這在 ch03 的規範記帳法裡是不可能發生的，因為那裡根本沒有「參考點」這個自由度。

這正是 Tversky 與 Kahneman 後來（1991，"Loss Aversion in Riskless Choice"，QJE）把參考點依賴講得最白的地方：價值的載體是相對參考點的**得與失**，不是淨資產狀態；偏好可以在兩個「最終財富完全相同」的選項之間翻轉，只因為它們相對參考點被編碼成不同的得失（2026-06 查證）。

我認為這是整本書最該停十分鐘的一頁。不是因為數學難——它一點都不難，就是個減法。難的是承認：**你以為你在對「結果」估值，其實你一直在對「相對某個你沒明說的零點的變化」估值，而那個零點是可以被移動的。** 後面整個 Part III 的戲劇——損失趨避、反射效應、框架效應——全都長在這個「零點可以被移動」上。ch10 會讓你看到：別人只要用措辭幫你把零點挪一下，你的選擇就翻轉，而你完全不會察覺。

## 參考點通常是現狀，但不總是

上一節為了乾淨，把參考點直接給定（1000 或 1200）。真實情況是：參考點從哪來？

Kahneman 與 Tversky 1979 的原始說法是：**參考點通常對應「當前的資產狀態」**，也就是現狀（status quo）（2026-06 查證，Econometrica 47:263–291）。這也是脊椎賭注的預設——你現在站在現狀，賭注的 +200/−100 就是相對現狀的得失。本書全書凡用展望理論重算脊椎賭注，參考點一律取**現狀 0**（基準表定案）。

但「通常是現狀」不是「永遠是現狀」。1991 年那篇 riskless choice 的論文裡，Tversky 與 Kahneman 把參考點講得更鬆：它常是現狀，但也可以是**期望、抱負、規範、或社會比較**（2026-06 查證，QJE 106:1039–1061）。對工程師來說這非常好懂——同一個結果，把零點挪一下，得失就翻面：

| 參考點來源 | 工程版的例子（同一結果、零點不同） |
|---|---|
| 現狀 | 「相比現在的延遲」——這次部署讓 p99 升了還是降了 |
| 期望／承諾 | 「相比 SLA 承諾的 99.9%」——做到 99.95 是達標、99.85 是違約，哪怕兩者差距很小 |
| 抱負／目標 | 「相比這季 OKR 設的目標」——同一個交付量，目標訂高了就是「沒達標」 |
| 社會比較 | 「相比隔壁團隊」——同樣的 incident 數，別人少你就難受 |

這裡有一個誠實標示要先打在前面：**參考點怎麼被決定，是描述心理學的開放問題，本書不深入。** 為什麼人會選現狀當零、為什麼期望有時會取代現狀、參考點怎麼隨時間漂移（你昨天的損失變成今天的新基準——快樂跑步機）——這些是「人腦怎麼運作」的心理機制，指向《思考的陷阱》談錨定與參考點。本書的工作是更窄、也更乾淨的一件事：**一旦參考點給定，得失就確定，估值就照 v(變化) 算。** 我們收的是這套記帳法的**數學後果**，不是它的心理成因（這條交棒線，ch12 會正式畫清楚——展望理論的數學由本書接手，心理機制指向偏誤書）。

但即使只收數學後果，有一件事必須當場說清楚，否則整個 Part III 會被誤讀——這就是下一節。

## 這是描述，不是規範——別把 v 當成 u

整本書的中央張力，在這一章第一次以「正面描述模型」的形式出現，所以這條線必須劃得格外清楚。

ch03 的效用 u(x) 是**規範**的：它在回答「你**該**怎麼選」。那套理論說，如果你接受四條無害的公理（完備、遞移、連續、獨立），你的選擇就**等價於**在最大化某個 u 的期望值——這是個「應該」，違反它你會被 money pump 牽著鼻子抽水（ch03）。

展望理論的價值函數 v(Δ) 是**描述**的：它在回答「人**實際**怎麼選」。它不主張「你應該按參考點記帳」，它只是觀察到「人就是按參考點記帳」，並且把這個觀察寫成一條可以預測方向的數學。它沒有公理逼你接受它，它有的是實驗——一大堆「正常聰明的人系統性地這樣選」的實驗。

```text
別把這兩個 v 和 u 搞混（這是 Part III 最容易出的錯）

  u(x)   規範      「你該怎麼算」    對最終財富、凹、公理逼出來的
  v(Δ)   描述      「人實際怎麼算」  對參考點變化、得凹失凸、實驗歸納出來的

  把 v 當成「正確的 u」 → 你會以為展望理論在說「人該按參考點記帳」（錯）
  把 u 當成「人真在算的」→ 你會以為 ch03 在描述真人（錯，它是規範）
```

這個區分為什麼這麼重要？因為一旦混淆，你會掉進兩個方向相反的坑。一個坑是把展望理論讀成「人很笨，居然不按最終財富算」——這是 T8 要守住的陷阱，下一節細談。另一個坑更隱蔽：把展望理論讀成「所以參考點記帳才是對的，期望效用過時了」——這同樣是誤讀，ch12 會正面回答。**參考點依賴是一個關於「人怎麼算」的事實，不是一個關於「該怎麼算」的主張。** 描述一座橋上的腳印偏哪邊，不等於主張「橋面該重畫到腳印那邊」。

我把這句話放在這裡，是因為它是整個 Part III 的閱讀說明書：接下來四章，每當你看到「人會這樣」「人會那樣」，問自己一句——這是在說「人實際怎樣」（描述）還是「人該怎樣」（規範）？展望理論講的永遠是前者。它的了不起，不在於它推翻了什麼，而在於它證明了**人偏離規範的方式本身是有結構的、可建模的、可預測方向的**——這比「人會犯隨機的錯」深刻得多，也比「人很笨」尊重人得多。

## 直覺的陷阱

參考點這個概念，因為太貼近日常，反而最容易被兩種方向相反的直覺帶下溝。守住 landscape 的正確版（T7、T8）。

| 陷阱 | 錯誤直覺長什麼樣 | 它在哪一步帶你下溝 | 正確版（怎麼自我察覺） |
|---|---|---|---|
| **把 v(Δ) 當成規範 u(x)**（T7 的本章版） | 「展望理論給了一條更好的價值函數，所以以後估值就該用 v、按參考點記帳。」 | 把一個**描述**模型（人怎麼算）讀成**規範**主張（該怎麼算）。一旦這樣讀，你會開始用 v 去「指導」決策，等於把人的系統性偏差當成目標。 | v 是描述、u 是規範，兩套數學描述**兩件不同的事**。徵兆：當你想說「展望理論告訴我們**應該**怎麼決策」時——停。它只告訴你人**實際**怎麼決策。「該怎麼決策」的問題，規範理論（ch03）還沒被推翻。 |
| **把參考點依賴當「人很笨」**（T8） | 「人居然不看最終財富、只看相對變化，這明明是個 bug，是非理性。」 | 把一個**穩定、可預測方向**的描述性事實，當成「缺陷／隨機錯誤」的指控。一旦這樣讀，你會錯過全章的重點——重點不是「人錯了」，是「人偏離的方式有結構」。 | 參考點依賴跨情境、跨文化穩定，方向可預測（同一終點，零點挪向高處就變損失、挪向低處就變收益），可以寫成數學。它了不起正在於此：偏離不是噪音，是訊號。徵兆：當你想說「人怎麼這麼蠢」時——換個問法，「人偏的方向能不能預測」，能，那它就不是蠢，是規律。 |
| **以為自己沒有參考點**（隱形零點） | 「我做決策看的是絕對的最終狀態，不會被『相對於什麼』影響。」 | 你確實有一個參考點，只是它隱形——通常是現狀或某個你沒明說的預期。看不見它，你就無法察覺它被移動。 | 自我察覺法：拿任何一個你最近的「成功/失敗」判斷，問「我是拿什麼當零點？」——上季水準？SLA 承諾？OKR 目標？隔壁團隊？通常你會發現那個零點是可以換的，而換了之後同一個結果就從「失敗」變「成功」。ch10 會證明：別人能用措辭幫你換零點，你還察覺不到。 |

第三個陷阱是這章最該帶走的自我檢查工具。**「我的零點放在哪？它能不能被換到別處？」**——這一問，是你拆穿後面框架效應（ch10）的鑰匙。

## 紙上推演

純紙筆推演，不要碰任何程式碼。每題標難度與時間，先自己算/想，再讀解答。

---

**題 1（★，10 分鐘）**：把脊椎賭注 L =（+200, 0.5; −100, 0.5）分別用「最終財富」與「參考點變化」兩種記帳法各寫一次。用 ch03 基準 u(x)=√(x+100)、W=100，列出兩套記帳各自評估的對象（不必算到底，重點是寫清楚「估的是什麼」）。然後回答：這兩套記帳，差別到底在哪一個零件上？

**題 2（★★，15 分鐘）**：某季你的服務可用度。情境甲：上季 99.92%，這季 99.70%。情境乙：上季 99.48%，這季 99.70%。
(a) 兩個情境的「最終狀態」一樣嗎？
(b) 用參考點記帳，這季的 99.70% 在甲、乙各是一筆「得」還是「失」？參考點各是多少？
(c) 如果有人按 ch03 的 u(最終可用度) 估值，這兩個情境的效用該一樣還不一樣？為什麼真人的反應不一樣？這道落差正是本章的「裂縫」——用一句話講清它。

**題 3（★★，15 分鐘）**：最終財富固定為 NT$1100。
(a) 參考點 = 900 時，這是得還是失？變化多少？
(b) 參考點 = 1300 時，這是得還是失？變化多少？
(c) 用一句話說明：為什麼「最終財富一樣、估值卻能相反」這件事，在 ch03 的規範記帳法裡**不可能**發生？（提示：ch03 的記帳法少了哪個零件？）

---

### 推演解答

**題 1 解答**

兩套記帳並排：

```text
最終財富記帳（ch03 規範，u(x)=√(x+100)）：
  估的對象 = 最終財富的效用
    贏 → 最終財富 300 → u = √(300+100)? 不對，先看清楚 x 的定義。
```

這裡要小心一個常見錯路：u(x)=√(x+100) 的 x 是**淨變化量**（賭注的 +200 / −100），W=100 已經寫在 +100 裡。所以正確代入是：

```text
最終財富記帳（ch03）：x = 淨變化
  贏 → x = +200 → u = √(200+100) = √300 = 17.32
  輸 → x = −100 → u = √(−100+100) = √0 = 0
  不賭 → x = 0 → u = √(0+100) = √100 = 10
  估的對象：把變化加進家底後的「最終財富效用」（絕對坐標）

參考點變化記帳（展望，參考點 = 現狀 0）：
  贏 → 變化 = +200 → v(+200)（一筆「得」）
  輸 → 變化 = −100 → v(−100)（一筆「失」）
  不賭 → 變化 = 0 → v(0) = 0
  估的對象：相對現狀的「得失」（相對坐標）
```

差別在哪個零件？**零點（參考點）。** ch03 把零點釘在「財富 = 0」，所有結果換算成最終財富再估值（u 是最終狀態的函數）；展望把零點放在「現狀」，所有結果換算成相對現狀的得失再估值（v 是變化的函數）。兩套都會得出「拒絕」，但 ch03 靠的是 u 的凹性（邊際遞減），展望靠的是 v 對失更陡（損失趨避，ch09 才算到底）。常見錯路：把 u(x)=√(x+100) 的 x 當成「最終財富」直接代 300/0/100——那會算成 √400、√100、√200，全錯。x 是變化，W=100 在公式裡。

**題 2 解答**

(a) 最終狀態一樣：兩個情境這季都是 **99.70%**。

(b) 參考點記帳：

```text
情境甲：參考點 = 上季 99.92%
  變化 = 99.70 − 99.92 = −0.22 個百分點  → 一筆「失」（退步了）

情境乙：參考點 = 上季 99.48%
  變化 = 99.70 − 99.48 = +0.22 個百分點  → 一筆「得」（進步了）
```

同一個 99.70%，甲是失、乙是得，純粹因為零點（上季水準）不同。

(c) 按 ch03 的 u(最終可用度) 估值，兩情境**該一樣**——因為終點同樣是 99.70%，u 只看終點、不看你從哪來。但真人的反應不一樣：甲會開檢討會、乙會點頭過，因為真人按參考點記帳，甲是損失、乙是收益。一句話講清裂縫：**規範記帳法（看終點）說這兩件事一樣，真人的記帳法（看相對上季的變化）說一個是退步一個是進步——人估的是變化，不是狀態。** 這就是參考點依賴，也是本章的整道縫。

**題 3 解答**

```text
最終財富固定 = 1100

(a) 參考點 = 900：
    變化 = 1100 − 900 = +200  → 「得 200」

(b) 參考點 = 1300：
    變化 = 1100 − 1300 = −200 → 「失 200」
```

(c) 一句話：在 ch03 的規範記帳法裡，「最終財富一樣、估值相反」**不可能**發生，因為那套記帳法的價值只是最終財富的函數 u(1100)——同一個 1100 永遠對應同一個 u 值，它**沒有「參考點」這個零件**可以移動。要讓同一個終點估出相反的值，你必須先有一個能被移動的零點；而那個零點，正是規範記帳法刻意不要、展望理論刻意加上的東西。常見錯路：以為「參考點 1300 時失 200」是因為「1100 比較少」——不是，1100 是固定的，變的是你拿來比的零點。得失是「終點減零點」，動的是零點。

## 自我檢核

口頭自答，講得清楚才算過。優先回答「為什麼」。

1. 用一句話對另一個工程師講清楚「參考點」：人估值估的是什麼、不是什麼？為什麼這跟 ch03 的 u(最終財富) 是**兩套不同的記帳系統**，而不只是「同一套的不同寫法」？
2. SLA 從 99.9% 掉到 99.5%，跟從 99.1% 升到 99.5%，最終狀態一樣。為什麼你的痛/喜完全不對稱？這個不對稱用「參考點」怎麼解釋？
3. 「同一個最終財富 1100，可以被估成一得一失」——這件事在 ch03 的規範記帳法裡為什麼**不可能**發生？（答案要點到「缺了哪個零件」。）
4. 為什麼本章一再強調「v 是描述、u 是規範」？把 v 當成「更好的 u」會掉進哪個坑？把 u 當成「人真在算的」又會掉進哪個坑？
5. 「參考點依賴是人很笨的證據」——這句話錯在哪？正確版怎麼講才不會把「穩定可預測的偏離」誤當成「隨機的缺陷」？
6. 參考點「通常是現狀」，但還可能是什麼？舉一個你工作裡「同一個交付結果、因為參考點不同（現狀／SLA 承諾／OKR 目標／隔壁團隊）而感受相反」的例子。
7. 本章只立了參考點這一根支柱，刻意沒算「失比得更陡」（損失趨避）。為什麼說「沒有參考點，就沒有損失趨避、沒有框架」？參考點為什麼是 Part III 的地基？

## 延伸閱讀

以下連結在 2026-06 查證過，優先放權威來源。

- **Kahneman, D. & Tversky, A., "Prospect Theory: An Analysis of Decision under Risk"（*Econometrica* 47(2):263–291, 1979）**（https://www.econometricsociety.org/publications/econometrica/1979/03/01/prospect-theory-analysis-decision-under-risk）— 展望理論的出生證明，本章參考點概念的原典。讀 value function 一節：它如何定義在「相對參考點的偏離」上、並指出參考點「通常對應當前資產狀態」。這是 Econometrica 史上被引用最多的論文，原文比任何二手轉述都清楚。
- **Tversky, A. & Kahneman, D., "Loss Aversion in Riskless Choice: A Reference-Dependent Model"（*Quarterly Journal of Economics* 106(4):1039–1061, 1991）**（http://bear.warrington.ufl.edu/brenner/mar7588/Papers/tversky-kahneman-qji1991.pdf）— 把「參考點依賴」講得最白的一篇：兩個在最終財富上完全相同的選項，可以因參考點不同而偏好翻轉。本章那張「同一個 1100、兩種估值」的表，理論依據就在這。注意它把參考點放寬到不只現狀（也可以是期望、抱負、規範）。
- **Wikipedia, "Prospect theory"**（https://en.wikipedia.org/wiki/Prospect_theory）— 想要參考點、損失趨避、機率權重、編輯/評估兩階段的快速地圖，這條最省事；可對照 1979 原文交叉核對「concave for gains, convex for losses, steeper for losses」這三句。讀完回到 ch09，那三句裡的後兩句（損失趨避、得失曲率）才會被算到底。
- **參考點依賴的心理機制**——本章只收了參考點的**數學後果**（一旦給定，得失就確定）。人腦為什麼選現狀當零、參考點怎麼隨時間漂移（快樂跑步機）、為什麼你察覺不到自己的零點——這些**心理機制**指向《思考的陷阱》談錨定與參考點。本書與它是同一道裂縫的兩面：本書接數學，它接心理（這條交棒線 ch12 正式畫清）。
- **凹效用的規範版**——本章對照的 u(x+W) 那套絕對坐標記帳法，完整內容在 ch03（vNM 公理、凹＝風險趨避）。想看「凹」的微分直覺（邊際遞減為什麼是導數遞減），點到為止指向《馴服無限》談導數與凹凸。
