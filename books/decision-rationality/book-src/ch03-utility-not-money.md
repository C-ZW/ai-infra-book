# ch03 — 期望效用：為何效用不是金額

> **本章解決什麼問題**：上一章我們看見期望值在聖彼得堡賭局前崩潰——平均報酬無限，沒人願意付無限的賭金。這一章把崩掉的東西補回來：把「金額」換成「效用（utility）」，期望值就變成期望效用（expected utility, EU）。這個小動作有兩個我認為是全書最美的後果。其一，凹效用函數一出現，「拒絕一個期望值為正的賭」立刻從「膽小、不理性」變成「嚴格理性」——這正是脊椎賭注那道裂縫第一次被精確命名。其二，幾條你根本不會反對的公理（vNM 四公理），居然能**逼出唯一一種**理性的演算法：期望效用最大化。本章是 Part I「理性的地基」的承重牆，也是全書「規範之美」的第一個高潮；ch04 接著會問「如果連機率都測不出來呢」。

## 從你已知的出發

你早就在用效用，只是沒這樣叫它。

凌晨三點 P0 當機，跟離峰時段的同一個服務掛掉，帳面上「損失的金額」可能一模一樣——都是少賺幾個小時的營收。但你心裡很清楚這兩件事**完全不是同一個重量**。凌晨三點的當機是 on-call 被叫醒、是睡眠債、是隔天整個團隊的判斷力打折、是 SLA 條款裡那個會被觸發的賠償門檻。同樣的金額，**對你的真實成本**天差地別。「真實成本函數」這個你天天在心裡跑的東西，就是效用函數，不是帳面數字。

再看一個你更熟的形狀。容量規劃時，你寧可過度配置（over-provision）一點點——多開兩台機器的成本是線性的、可預期的、不痛不癢的；但**配置不足**導致的雪崩式故障，成本不是線性的，它會在某個臨界點突然爆炸。於是你心裡那條「賠掉 vs 賺到」的曲線是**不對稱**的：少賺一塊錢沒什麼，但在最糟的時候多賠一塊錢可能要命。「同樣多的錢，賠掉比賺到更傷」——這句工程直覺，數學上就叫**凹（concave）**，就叫**邊際效用遞減（diminishing marginal utility）**。你對快取命中率的既視感也是同一件事：命中率從 50% 拉到 60% 爽度很高，從 98% 拉到 99% 邊際效益小得多。同樣多的「增量」，越往上越不值錢——這就是凹。

所以這一章不是要教你一個新東西，是要把你**已經在用的直覺**接上一個三百年前就寫好的數學，並且讓你看見：這個接法不是隨便接的，它幾乎是被逼出來的唯一接法。

## 從金額到效用：那個改變一切的小動作

回憶 ch02 的期望值定義。對彩券 `L = (x₁, p₁; x₂, p₂; …)`，期望值是把**金額**加權平均：

```text
E[L] = Σ pᵢ · xᵢ        ← 把「金額 xᵢ」用機率加權
```

聖彼得堡告訴我們這個量會誤導：它可以無限，但理性的人只願付有限。Daniel Bernoulli（丹尼爾·伯努利）1738 年在聖彼得堡科學院的期刊上給的解法簡單到近乎作弊——他說，理性的人最大化的不是金額的期望，是**效用**的期望。把上面那條式子裡的 xᵢ 換成 u(xᵢ)：

```text
EU(L) = Σ pᵢ · u(xᵢ)    ← 把「效用 u(xᵢ)」用機率加權
```

u 是**效用函數（utility function）**：一個把金額映射到「對你的價值」的函數。Bernoulli 主張 u 大致是對數的（u(x)≈ln x），財富越多，多一塊錢帶來的效用越小。這條對數曲線的尾巴壓得很扁，於是聖彼得堡那條「靠極大但極不可能的獎金撐起來」的無限尾巴，在效用世界裡被壓成有限的——理性的人願付有限，矛盾消失。（這是 Bernoulli 1738 的解，照事實基準；要注意悖論本身最早是 Nicolaus Bernoulli 1713 年寫給 Montmort 的信裡提出的，名字來自發表地聖彼得堡，不是 Daniel 提出的——見 ch02。）

我請你停在這裡十秒，因為這個小動作的份量很容易被滑過去。**它沒有否定期望值的數學，它改寫了「期望值是對什麼東西取的」。** 平均報酬發散是對的；但「該付多少」問的是效用，不是金額。整本書要追的那道裂縫，種子就埋在這個替換裡：金額是世界的客觀屬性，效用是你的——一旦決策算的是效用，「理性」就有了個人的形狀。

## 凹效用＝風險趨避：把脊椎賭注算到底

現在我們把這套工具第一次套到全書脊椎那個賭注上。

脊椎賭注（全書統一）：`L = (+200, 0.5; −100, 0.5)`，單位 NT$。擲一枚公正硬幣，正面贏 200、反面輸 100。ch02 算過它的期望值：

```text
E[L] = 0.5·(+200) + 0.5·(−100) = +100 − 50 = +50
```

期望值為正 +50。一個「期望值最大化」的決策者會毫不猶豫接受。但多數人會拒絕——包括你，多半也會。這不是膽小，是理性，我現在用效用把它算給你看。

全書統一用這條凹效用：`u(x) = √(x + 100)`。這裡的 100 是**財富基準 W=100**——你不是從零開始賭，你身上本來就有相當於 100 單位的東西墊底，效用是建在「最終財富」上，不是建在「變化量」上（這個區別在 ch08 展望理論會變成主角，那裡的價值函數恰恰相反，是建在變化上的）。√ 是凹的：邊際遞減，跟你的容量規劃直覺同形。

逐步算（每一步我都自己複核了一遍，不跳步）：

```text
不賭的效用（保持現狀，財富 = 100）：
  u(不賭) = u(0)            ← 「不賭」= 財富變化 0，最終財富仍是 W=100
          = √(0 + 100)
          = √100
          = 10

賭的期望效用（兩個結果各代進 u，再加權）：
  贏的那邊：u(+200) = √(200 + 100) = √300 = 17.3205   ← √300 ≈ 17.3205（校驗常數）
  輸的那邊：u(−100) = √(−100 + 100) = √0   = 0         ← 輸掉 100，最終財富歸零，效用 0

  EU(L) = 0.5·u(+200) + 0.5·u(−100)
        = 0.5·17.3205 + 0.5·0
        = 8.6603 + 0
        = 8.6603

比較：
  EU(L) = 8.66   <   u(不賭) = 10
  ────────────────────────────────
  期望「效用」較低 ⇒ 理性的選擇是「不賭」
```

看清楚發生了什麼：**期望「金額」是 +50（賭比較好），期望「效用」卻是 8.66 < 10（不賭比較好）。** 同一個賭注，兩套演算法給相反的答案。而給「不賭」這個答案的那一套，正是規範理論說你**該**用的那一套。

關鍵在那個 √0 = 0。輸掉 100 把財富打到底，效用直接歸零；而贏到 300 的效用只有 17.3，**不到** 10 的兩倍。因為曲線是凹的，越往上漲得越慢——贏的那邊帶來的效用增量（17.3−10=7.3），補不滿輸的那邊砍掉的效用（10−0=10）。機率對半，但效用的損失比效用的收益重。於是 EU 拉到 10 以下，理性的人收手。

這就是**風險趨避（risk aversion）的精確意義**：不是性格、不是膽量，是「u 是凹的」這件事的純數學後果。背後是一條叫 Jensen 不等式的東西——對凹函數 u，`u(E[L]) ≥ E[u(L)]`：先平均再取效用，永遠不小於先取效用再平均。風險趨避就是這個不等式，一句話。（嚴格證明本書不展開，指向延伸閱讀與《馴服無限》談凹凸與導數；這裡給的是「工程師的嚴謹」：每一步我都能口頭說出理由——凹 ⇒ 邊際遞減 ⇒ 損失端的效用降幅大於收益端的效用升幅 ⇒ EU 被拉低。）

我認為這是本書到目前為止最值得你停下來的一頁。三百年的爭論——「拒絕正期望值的賭到底理不理性」——答案是：**看你的 u 凹不凹。** 凹，就理性。風險趨避不需要辯護，它本來就是最大化期望效用的結論。

### 確定等值與風險溢酬：把「值多少」反解出來

既然賭注 L 的期望效用是 8.66，我們可以反問一個很實際的問題：**多少確定的錢，跟這個賭注一樣好？** 這個量叫**確定等值（certainty equivalent, CE）**——一筆「拿了就走、不賭」的確定金額，它的效用恰好等於賭注的期望效用。

```text
定義：u(CE) = EU(L)
求解：√(CE + 100) = 8.6603        ← u(CE) = EU(L) = 8.66
      CE + 100 = 8.6603²
      CE + 100 = 75.0             ← 8.6603² ≈ 75.0（自我複核：8.66² = 74.99…≈75）
      CE = 75.0 − 100
      CE = −25.0
```

讀這個 −25：這個期望值 **+50** 的賭注，對你的價值等於「確定收到 −25」，也就是**確定賠 25**。你會花錢請別人把這個賭注拿走。然後是風險溢酬（risk premium）：

```text
風險溢酬 = E[L] − CE = (+50) − (−25) = +75
```

75 是你為了「躲開這個不確定」願意放棄的期望金額。期望值明明是 +50，你卻甘願把它整個吐出來、再倒貼 25——這 75 的縫，就是風險趨避用金錢標出來的價碼。你在容量規劃裡多開的那兩台機器、買保險時付的那筆保費，本質上都是在付這個風險溢酬。它不是浪費，是凹效用下理性地花錢買確定。

### 對照一次：ln 效用怎麼解聖彼得堡

為了把 ch02 的線收回來，這裡用對數效用快速對照一次聖彼得堡（全書只在這裡用一次 ln，其餘一律用 √）。聖彼得堡賭局：擲硬幣到首次正面，第 n 次正面付 2ⁿ。期望「金額」是 `Σ (1/2ⁿ)·2ⁿ = Σ 1 = ∞`，發散。但若你最大化的是 `u = ln(財富)`，假設你身上本有財富 w，玩這局的期望效用是：

```text
EU = Σ(n=1→∞) (1/2ⁿ) · ln(w + 2ⁿ)     ← 每項：機率 1/2ⁿ × 贏到 2ⁿ 後的對數效用
```

關鍵：ln(w + 2ⁿ) 大約隨 n 線性成長（ln(2ⁿ)=n·ln2），而前面的機率 1/2ⁿ 是指數衰減。指數壓線性，級數**收斂**——期望效用是有限的。有限的期望效用對應有限的願付金額。聖彼得堡的「無限」全部來自金額尾巴，一進對數效用就被馴服了。這就是「效用不是金額」最古老的證據，也是 Bernoulli 1738 的原始洞見。

（一句點到、不展開：「該下多少注才不會破產」是另一個問題——Kelly 準則處理它，本書只當理論試金石、不教投注，細節指向延伸閱讀。）

## vNM 四公理：幾條無害的規矩逼出唯一的演算法

到目前為止我用的是 Bernoulli 的版本——「人最大化某個 u 的期望」。但這留了一個尷尬的洞：**憑什麼是 u 的期望？** 為什麼不是 u 的中位數、u 的最差情況、u 的某種別的混合？Bernoulli 沒回答這個。回答它的是 1944 年的馮紐曼（John von Neumann）與摩根斯坦（Oskar Morgenstern）。

他們的書《Theory of Games and Economic Behavior》第一版是 1944 年出的；但**從公理推導期望效用的那段形式內容，是 1947 年第二版才補進附錄**（"The Axiomatic Treatment of Utility"）。科普常寫成「1944 年提出期望效用公理」，更精確是「1944 出書、1947 二版附錄給公理化」（事實基準 T2，引用時守住這個年份）。

他們做的事，用工程的話講，是**從規格反推出唯一實作**。他們先寫下四條對「你的偏好」的約束——四條你看了會覺得「廢話，當然」的規矩。然後證明：只要你的偏好同時滿足這四條，你的選擇就**必然等價於**在最大化某個效用函數的期望。不是「可以這樣建模」，是「你**只能**這樣，否則你違反了你自己同意的某條規矩」。這個結論叫 **vNM 表現定理（representation theorem）**。

四條公理（白話版，原文精確界定指向延伸閱讀）：

| 公理 | 白話 | 違反它會怎樣 |
|---|---|---|
| 完備性（completeness） | 任何兩個彩券你都能比：要嘛偏好 A、要嘛偏好 B、要嘛覺得一樣好。不准說「沒法比」。 | 你「無法決定」——連賽局都還沒開始 |
| 遞移性（transitivity） | 偏好 A 勝 B、B 勝 C ⇒ 必須偏好 A 勝 C。不准繞成一個圈（A>B>C>A）。 | 被當成 money pump 無限抽水（見下） |
| 連續性（continuity） | 若 A 勝 B 勝 C，那存在某個機率 p，使「p 機率拿 A、否則拿 C」的混合彩券剛好跟 B 一樣好。——偏好沒有「無限好／無限壞」的斷層。 | 偏好出現無法被任何數字表示的斷崖 |
| 獨立性（independence） | 若 A 勝 B，把同一個第三彩券 C 用同樣機率混進 A 和 B，偏好不該改變：混 A 的勝混 B 的。——共同的部分不該影響比較。 | Allais／Ellsberg 正是斷在這條（ch04／ch11） |

讀這張表時，請帶著一個我認為很關鍵的觀察：**這四條沒有一條提到「期望」或「平均」。** 完備、遞移、連續、獨立——全是關於「你怎麼排序偏好」的結構性規矩，沒有一條偷偷塞進「你要算期望值」。然而表現定理證明：滿足這四條，你的選擇就**自動**等價於最大化某個 u 的**期望**。期望這個運算不是被規定的，是被這四條逼出來的。

這就是我說的「規範之美」。你不是被命令去算期望效用；你是被問「你同不同意這四條」，你說同意，然後數學告訴你：那你已經在算期望效用了，你沒有別的選擇。（這裡我必須誠實標示嚴謹度：以上是**直覺與精神**，不是證明。表現定理的嚴格證明本書不展開，給直覺、指向延伸閱讀。我不會用「顯然」或「易證」蒙混——這個定理一點都不顯然，它是 20 世紀決策論的地基之一。）

### money pump：違反公理為什麼會「漏錢」

「四條公理你不會反對」這句話需要一點骨頭，否則只是修辭。最好捏的骨頭是遞移性——違反它的代價可以直接用錢量出來，這叫 **money pump（抽水幫浦）**。

假設某人的偏好繞成一個圈：A > B > C > A（嚴格偏好，circular）。我來當那個牽你鼻子的人：

```text
起點：你手上拿著 C
  ① 你偏好 B 勝 C  → 我說「付我 1 塊，我把 C 換成 B 給你」→ 你樂意付，因為 B 對你更好
       現在你拿 B，我口袋 +1
  ② 你偏好 A 勝 B  → 我說「再付 1 塊，把 B 換成 A」→ 你樂意付
       現在你拿 A，我口袋 +2
  ③ 你偏好 C 勝 A  → 我說「再付 1 塊，把 A 換成 C」→ 你樂意付
       現在你拿 C，我口袋 +3 …… 而你又回到了起點 C
  ↻ 回到①，無限循環。每轉一圈你付我 3 塊，東西卻回到原點。
```

注意每一步你都是**按自己的偏好**在行動、每一步都覺得自己換得更好——可是繞完一圈，你手上的東西沒變，錢卻少了 3 塊，而且可以無限榨下去。這就是違反遞移性的「代價」：你會被你自己的偏好牽著走進一個無底洞。（web 查證確認：money pump 只能榨**有循環的嚴格偏好**——A>B>C>A 這種；遞移但偶爾「沒法比」的偏好不一定能這樣榨，這是文獻裡的細緻處，本書點到為止。）

money pump 把「公理不是品味問題」這件事講死了：你**可以**選擇有循環的偏好，但那等於在自己身上裝一個漏錢的洞。理性的最小公設不是某個教授的審美偏好，是「不要讓人能無限榨你」的自保。同一個精神在貝氏那邊叫 Dutch book（不自洽的信念會被組合成穩賠的賭局），那是 ch06 的事——你會看到「公理 ⇒ 不可被套利」這個結構在決策論裡反覆出現。

## 直覺的陷阱

決策正是人類直覺跟規範理論最常打架的地方，本章有兩個最常見的坑，兩個都很滑。

| 錯誤直覺 | 在哪一步把你帶溝裡 | 正確版（守住） |
|---|---|---|
| 陷阱一：把效用當金額——「期望值為正就該接」 | 看到 E[L]=+50>0 就說「接啊，期望值正的」——漏掉了 u 的凹性，等於偷偷假設 u 是線性的 | 該算的是 EU 不是 E。u 凹時 EU(L)=8.66 < u(不賭)=10，拒絕才是最大化期望效用。金額是世界的、效用是你的，兩者不是同一個量。 |
| 陷阱二：以為風險趨避是「膽小／不理性」（事實基準 T4） | 把「拒絕正期望值的賭」讀成性格缺陷，覺得「理性的人就該全接」 | 在凹效用下，拒絕正期望值賭注是嚴格理性（Jensen：u(E[L])≥E[u(L)]）。風險趨避＝u 凹這件事的純數學後果，不是膽量問題。「全接」反而是假設你風險中性（u 線性），那才是特例。 |

**怎麼自我察覺你掉進去了**：

- 陷阱一的徵兆——你發現自己在用「期望值是正的」當作「該接受」的**唯一**理由，卻說不出你的 u 長什麼形狀。一旦你停下來問「那如果輸的那邊不是 −100 而是 −10000 呢」，你會立刻不想接了——這就是你心裡的 u 其實是凹的證據，你只是剛才把它當線性了。
- 陷阱二的徵兆——你在心裡偷偷鄙視「不敢賭」的人。但若你真信「拒絕正期望值＝不理性」，那你應該接受**任何**期望值為正的賭，包括「99% 機率破產換 1% 機率翻一千倍、但期望值勉強為正」的那種。你不會接。所以你其實是風險趨避的，你只是沒承認。風險趨避不是別人的毛病，是凹效用下每個正常人的理性常態。

第三個值得提防的小坑：**把「u 凹」當成「u 一定是 √ 或 ln」**。√ 和 ln 只是本書選的兩條方便的凹曲線，不是物理定律。真正的規範內容是「u 凹 ⇒ 風險趨避」，至於你的 u 凹得多厲害（風險溢酬多大）是你個人的事，沒有標準答案。別把教學用的具體函數誤當成普世常數——這個提醒在 ch09 講展望理論參數時會變得更重要。

## 紙上推演

全部純紙筆手算，不要去跑任何程式。算完跟解答對，特別注意「常見錯路」。

### 推演題

**題 1（★★，20 分鐘）**：給效用函數 `u(x) = √(x + 100)`（財富基準 W=100）與脊椎賭注 `L = (+200, 0.5; −100, 0.5)`。手算：(a) E[L]；(b) EU(L)；(c) 確定等值 CE；(d) 風險溢酬。然後用一句話說明：為什麼期望值說「接」、期望效用說「拒」，哪個是規範上正確的，為什麼。

**題 2（★★，口頭，15 分鐘）**：不准用 Jensen 不等式的名字，用你自己的話，向一個只懂「期望值」的工程師同事解釋：「為什麼**凹**效用就一定**風險趨避**？」要求他聽完能自己舉一個例子。

**題 3（★★★，25 分鐘）**：造一個你工作中「明明期望值為正、你卻不會接」的真實賭注（例如某個有正期望收益但有小機率釀成大故障的架構變更），把它寫成彩券記法 `L = (好結果, p; 壞結果, 1−p)`。(a) 指出在什麼金額假設下它的期望值為正；(b) 論證你拒絕它這件事，蘊含你的隱含效用函數是凹的；(c) 額外問：你的 u 在「壞結果」那一端凹得特別厲害，這對應到什麼工程現實？

---

### 推演解答

**題 1 解答**：

```text
(a) 期望值：
    E[L] = 0.5·(+200) + 0.5·(−100) = 100 − 50 = +50

(b) 期望效用：
    u(+200) = √(200+100) = √300 = 17.3205
    u(−100) = √(−100+100) = √0  = 0
    EU(L)  = 0.5·17.3205 + 0.5·0 = 8.6603

    對照不賭：u(0) = √(0+100) = √100 = 10
    EU(L) = 8.66 < 10 ⇒ 拒絕

(c) 確定等值 CE：u(CE) = EU(L) = 8.6603
    √(CE+100) = 8.6603
    CE+100 = 8.6603² = 75.0
    CE = −25.0

(d) 風險溢酬：E[L] − CE = (+50) − (−25) = +75
```

哪個正確：**期望效用（拒）是規範上正確的那一個。** 因為理性的選擇是最大化你**對結果的價值**（效用）的期望，不是最大化帳面金額的期望。金額的期望 +50 完全沒考慮「輸掉 100 把財富打到 0」對凹效用造成的重傷；效用的演算法考慮了，所以它說拒。

**常見錯路**：(1) 把 u(−100) 算成「負的」——不對，u(−100)=√(−100+100)=√0=0，不是負數，重點是財富變化 −100 讓**最終財富**歸零，效用降到 0 而非變負。(2) CE 反解時忘記減 100，把 8.66²=75 直接當成 CE——CE 是金額變化，要 75−100=−25。(3) 把「期望效用比較低」說成「賭比較危險所以不賭」——這是對的結論但錯的理由，正確理由是「EU 較低」這個冷冰冰的數字，危險感只是它的副產品。

**題 2 解答（要點）**：凹的意思是「邊際遞減」——同樣多的金額，越往上加，帶來的效用增量越小。現在賭一把「等機率贏 a 或輸 a」：贏的那邊你從現在的位置往上爬 a，但因為凹、上面那段爬得**慢**，賺到的效用少；輸的那邊你往下掉 a，但下面那段**陡**，掉掉的效用多。機率對半，可是「失去的效用」比「得到的效用」大，所以期望效用一定低於現狀——於是你拒絕。一句話：**凹 ⇒ 損失端的效用降幅大於收益端的升幅 ⇒ 公平甚至有利的賭也會讓期望效用下降 ⇒ 風險趨避。** 同事該舉的例子：「我現在有 100 萬，給我『等機率變 200 萬或變 0』我不要——因為從 100 萬掉到 0 的痛，遠大於從 100 萬漲到 200 萬的爽。」

**常見錯路**：說成「因為可能會輸所以怕」——這是心理描述不是數學論證，沒解釋「為什麼明明期望值打平甚至為正還是拒」。關鍵字必須是「邊際遞減 ⇒ 兩端不對稱」，不是「怕輸」。

**題 3 解答（示範一種）**：

```text
賭注：一次風險偏高的資料庫 schema 遷移（不停機方案）
  好結果（順利）：省下未來 6 個月的維運痛苦，估值 +600（單位：某個你的效用幣）
  壞結果（遷移中資料損毀）：估值 −1000
  機率：你評估 p(順利)=0.9

(a) 期望值 = 0.9·(+600) + 0.1·(−1000) = 540 − 100 = +440 > 0  ← 期望值為正
(b) 但你不會接（你會先做雙寫、影子讀、可回滾，把壞結果機率壓到很低才動）。
    「期望值為正卻拒絕」⇒ 你的 u 不是線性的；你對 −1000 的厭惡
    遠超過 +600 的喜愛的兩倍弱（0.9:0.1 的機率本來偏向好結果），
    這只可能是因為 u 在損失端陡、在收益端緩 ⇒ u 凹。
(c) u 在「資料損毀」端凹得特別厲害，對應的工程現實是：
    資料損毀往往不可逆、會連鎖（信任崩塌、合規問題、客戶流失），
    它的真實成本是超線性的——這正是「凹效用」在最糟那端的具體模樣。
```

**常見錯路**：把「我不接」直接當成「期望值其實是負的我算錯了」——不，期望值真的是正的 +440，你拒絕它**不是**因為算錯期望值，而是因為你最大化的根本不是期望值。這正是本章的整個重點：拒絕正期望值賭注是理性的，前提是你的 u 凹。

- **von Neumann & Morgenstern,《Theory of Games and Economic Behavior》（1944；公理化在 1947 二版附錄）**：原典。不必硬啃全書，知道「期望效用公理化是 1947 二版補的附錄」這個年份，本身就能讓你在酒桌上糾正一個常見誤傳。
- **Stanford Encyclopedia of Philosophy，"Dutch Book Arguments"** 與 money pump 相關條目：把「違反公理 ⇒ 可被穩賺套利」的精神講透；本章 money pump 與 ch06 的 Dutch book 是同一個結構的兩個面孔，先讀這裡能把兩章串起來。
