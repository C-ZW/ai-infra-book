# ch17 — 理性是什麼：規範與描述的和解

> **本章解決什麼問題**：全書到這裡，規範的地基（Part I–II：期望效用、貝氏決策、資訊的價值）已經立穩，描述的裂縫（Part III：展望理論）已經攤開，有限理性（Part IV）已經把「最佳化」放回一顆成本有限的腦，ch16 又畫清了「對手會反應時就交棒給賽局」的邊界。剩下一個沒回答的大問題壓在全書頭上：**「理性」這個詞到底指哪個——你「該」怎麼選（規範），還是你「實際」怎麼選（描述）？** 既然人系統性地偏離規範，那規範還算數嗎？人是不是就「不理性」？這章把全書的中央張力做一次哲學收束，給出最終回答；並把「理性三角」——決策、偏誤、科學方法這三本書的分工——講清楚，明寫本書承擔的招牌跨書連結。下一章（ch18）是收官，用脊椎決策把五種演算法對帳一遍。

## 從你已知的出發

你寫過 design doc，也跑過 postmortem。這兩件事，正好對應「理性」的兩種意思，而你早就同時在用、只是沒給它們起名字。

**Design doc 是規範的。** 你在裡面寫「這個服務應該怎麼做容量規劃」「故障時應該按什麼順序降級」「這個權衡應該偏向可用性還是延遲」。你寫的是**該怎麼做**——一套在「如果大家都照著走」的理想前提下，能導出好結果的程序。它不描述任何人實際做了什麼，它規定一個標準。

**Postmortem 是描述的。** 事故發生後你坐下來復盤，誠實記錄**人實際上怎麼做了**：on-call 在凌晨三點看到一片紅，憑印象判斷是 DB（先驗太強），漏看了 deploy 時間軸（基率忽視），在壓力下選了風險更大的緊急回滾而不是穩妥的漸進式回退（損失趨避加框架效應）。postmortem 不罵人——好的 postmortem 假設「換成任何人在那個情境下都可能這樣」，它描述真實的決策過程，包括它怎麼偏離了 design doc 寫的理想。

現在問你一個問題：**design doc 和 postmortem 衝突的時候，是哪一個錯了？**

直覺上你想說「都沒錯」，但講不清楚為什麼。如果 design doc 是對的（人該那樣做），那 postmortem 記錄的偏離不就證明人「沒做對」？如果 postmortem 是對的（人就是會那樣做），那 design doc 寫的理想不就是不切實際的空話？

這正是全書的中央張力，穿上了工程的外衣：**規範（你該怎麼選）與描述（人實際怎麼選）之間那道穩定的裂縫，到底是誰錯了？** 我的答案——也是這章要慢慢說服你的答案——是：**兩個都對，因為它們根本在回答兩個不同的問題。** design doc 在回答「什麼程序能導出好決策」，postmortem 在回答「真實的腦在真實的成本下實際跑了什麼程序」。它們不是同一個問題的兩個答案（那才會有對錯），它們是兩個問題各自的正確答案。一個好的工程團隊不是「讓 postmortem 向 design doc 看齊」，而是**承認兩者之間有一道穩定的、可預測方向的裂縫，然後設計流程去補它**——這就是 runbook、checklist、review、預先承諾存在的全部理由。

這章就是把這個工程直覺，升級成關於「理性」的一個完整立場。

## 理性的兩個意思：程序的，與實質的

先把「理性」這個被用爛的詞拆成兩半。這個拆法不是我發明的，它是 Herbert Simon（賽門）的——就是 ch13 那位有限理性的提出者。Simon 區分了**程序理性（procedural rationality）**與**實質理性（substantive rationality）**（這是他廣為人知的一組對照，常見於他談有限理性的著作；本書取其精神，不糾結確切出處頁碼）。

```text
   理性的兩種意思（Simon 的區分）

   實質理性（substantive）            程序理性（procedural）
   ──────────────────────            ──────────────────────
   問「結果好不好」                   問「過程對不對」
   ──────────────────────            ──────────────────────
   一個選擇是理性的，                  一個選擇是理性的，
   若它「客觀上」達成了目標            若它「用對的方法」做出來——
   （達到了最好的結果）                即使結果不盡如人意

   隱含假設：                          隱含假設：
   ・所有選項都看得到                  ・腦／機器的算力有限
   ・所有後果都算得出                  ・資訊有成本、不齊全
   ・無限算力、無限時間                ・時間有限

   ⇒ 古典經濟學的「理性人」            ⇒ Simon 有限理性的「理性人」
     用的是這個意思                      用的是這個意思
```

這個區分一立起來，很多糾纏不清的爭論就解開了。

**「人不理性」這句話，混淆了這兩個意思。** 當行為經濟學家說「人系統性地違反期望效用」，他說的是人偏離了**實質理性**的某個標準（期望效用最大化，那個假設無限算力的理想）。但這完全不蘊含人**程序上**不理性——一個在資訊與算力都有限的腦袋裡跑得相當不錯的決策程序（用捷思、用 satisficing、用損失趨避當快速警報），在它真實的約束下可能是**程序理性**的，哪怕它的輸出偏離了那個無限算力的理想。

我用一個工程比喻把這件事釘死。**實質理性像是「假設無限記憶體、無限 CPU、零延遲網路」下寫出來的演算法**——理論上最優，O(最好)。**程序理性像是在真實機器上、有記憶體上限、有 GC 暫停、有網路逾時的約束下，能實際跑完並給出夠好答案的演算法。** 你不會因為一個演算法不是「假設無限資源下的理論最優」就說它「寫錯了」——如果它在真實約束下表現良好，它就是好程式。把人腦的決策說成「不理性」，常常就是犯了「拿無限資源的理論最優去羞辱一個在真實約束下跑得不錯的系統」這個錯。

這給了我們和解的第一塊地基：**規範理論（期望效用）是「無限資源下的實質理性理想」；有限理性是「真實成本下的程序理性理想」；展望理論是「真實的腦實際跑了什麼程序」。三者不是互相打架，它們在回答三個不同的問題。** 下一節我把這三層攤開。

## 三層，不是三個對手

把全書出現過的「理性」整理成三層。關鍵在於：它們不是「三個候選答案要選一個對的」，而是**三個不同的問題各有各的答案**。這是和解的核心，請慢一點。

```text
   三層理性：三個不同的問題，不是三個對手

   ┌─────────────────────────────────────────────────────────────────┐
   │ 第一層：規範·無限資源                                           │
   │   問題：在「能看到所有選項、算得出所有後果」的理想下，          │
   │         該怎麼選？                                              │
   │   答案：期望效用最大化（vNM 公理逼出的唯一演算法）                │
   │   身分：理想的標竿。不描述任何真實的腦，是一把尺。              │
   │   本書出處：Part I–II（ch03 vNM、ch05 貝氏決策、ch07 資訊價值） │
   ├─────────────────────────────────────────────────────────────────┤
   │ 第二層：規範·有限資源                                           │
   │   問題：把「搜尋與計算本身有成本」算進去，該怎麼選？            │
   │   答案：satisficing／設 aspiration level／夠好就停（修正規範）  │
   │   身分：把成本納入後的「該怎麼做」——仍是規範，但更務實。        │
   │   本書出處：Part IV（ch13 有限理性、ch14 多屬性、ch15 序貫）    │
   ├─────────────────────────────────────────────────────────────────┤
   │ 第三層：描述·真實的腦                                           │
   │   問題：人「實際上」怎麼選？                                    │
   │   答案：展望理論（參考點、損失趨避、機率權重、框架）            │
   │   身分：描述模型。說人怎麼算，不說人該怎麼算。                  │
   │   本書出處：Part III（ch08–12）                                 │
   └─────────────────────────────────────────────────────────────┘
   └─────────────────────────────────────────────────────────────────┘
   它們之間的關係（注意都不是「對／錯」）：
     第一層 → 第二層：把資源成本加進去，理想被「修正」（不是被推翻）
     第一層 vs 第三層：理想與現實之間有「裂縫」（不是誰錯了）
     第二層 ←→ 第三層：satisficing 既是修正規範、又貼近真實的腦
                       （ch13 說它是「規範與描述的橋」就在這）
```

盯著最後那塊「它們之間的關係」看，因為這裡藏著全書最容易被講錯的一句話。

**第一層和第二層之間，是「修正」不是「推翻」。** 把搜尋成本算進去之後，「找全域最大」不再是無條件的最優——在元層次上（把找答案的代價也算進帳），「夠好就停」可能才是真正的最佳化（ch13 的 T10）。所以第二層沒有否定第一層，它只是給第一層加了一個之前被理想化掉的約束項（成本）。這就像你不會說「考慮了 GC 暫停的演算法分析推翻了演算法理論」——它是在更真實的成本模型下的同一套最佳化思維。

**第一層和第三層之間，是「裂縫」不是「對錯」。** 這是全書反覆敲的那句話，現在給它最精確的版本：**期望效用（第一層）回答「該怎麼選」，展望理論（第三層）回答「人怎麼選」，這是兩個不同的問題，所以它們的答案不同不構成矛盾，正如「地圖該畫成什麼樣」和「人實際怎麼走路」不同不構成矛盾。** 一張正確的地圖不會因為「行人實際上會抄近路、闖紅燈」就變成錯的地圖；行人的真實路徑也不會因為「不符合地圖規劃的路線」就變成錯的走法。它們是兩件事。

這就是和解。它不是「各打五十大板、和稀泥」，它是一個精確的主張：**規範與描述描述的是兩件不同的事，所以一個系統性偏離規範的描述事實，邏輯上不能反駁規範、也不能證明人笨。** 下兩節分別把這兩個「不能」說透。

## 為什麼「人偏離規範」不證明規範錯了

這是全書中央張力的最終回答之一，也是最容易被滑過去的一步邏輯。慢。

有一種很自然的推論，聽起來幾乎像常識：「既然正常、聰明、受過訓練的人都系統性地違反期望效用（Allais、Ellsberg 一路證過來），那這套規範理論一定有問題吧？真實的人都不照它走，它憑什麼說自己是『該怎麼做』的標準？」

這個推論偷渡了一個前提：**「真實的人怎麼做」可以決定「人該怎麼做」。** 把這個前提攤在陽光下，你會發現它很可疑——它等於說「凡是人普遍會犯的錯，就不算錯」。

用一個你熟悉的領域戳破它。**人類普遍、系統性、可預測地不會心算 137 × 256。** 這是一個關於人的穩定描述事實。但它證明「137 × 256 = 35072 這個算術規範錯了」嗎？當然不。它只證明「人腦不是為大數乘法優化的硬體」。算術規範（乘法該怎麼算）和人類描述事實（人實際算不出來）是兩回事；後者再穩定，也撼動不了前者。**「人做不到 X」從來不蘊含「X 不是該做的」**——除非你已經偷偷假設了「該做的＝人能輕鬆做到的」，而那恰恰是要被質疑的前提。

```text
   「人偏離規範 ⇒ 規範錯」這個推論斷在哪

   前提（描述）：人系統性地違反期望效用       ← 真（Allais/Ellsberg）
   想推出（規範）：所以期望效用不是好規範      ← 不跟著來

   斷裂處：從「是」（人實際怎樣）跳到「應」（人該怎樣）
           需要一座橋：「凡人普遍會犯的就不算錯」
           ——而這座橋一攤開就站不住
             （反例：人普遍算不出 137×256，
              不證明這道乘法的正確答案錯了）

   ⇒ 描述事實再穩定，也不能單獨反駁一個規範主張。
     要反駁規範，得從「規範自己的公理不值得遵守」下手，
     不能從「人不遵守它」下手。
```

那規範能不能被反駁？**能，但要從正確的方向。** 你不能因為「人不遵守獨立公理」就說獨立公理是錯的規範；你只能去論證「獨立公理這條公理本身不值得一個理性主體遵守」。這是 Allais 本人和許多後來者真正在做的事——他們不是說「人違反它所以它錯」（那是上面那個壞推論），他們是說「仔細想想，獨立公理作為一條規範要求，在某些情境下未必是我反思後願意被它約束的」。這是一場**規範層次**的辯論（公理該不該要），不是描述事實能裁決的。

這正好回扣 ch11 結尾 Savage 的故事。Savage 自己在 Allais 兩題上選了不一致的答案（描述：他和大家一樣），但他**沒有因此放棄公理**——他反思後仍認為那幾條公理是他願意被約束的，於是把自己的選擇當成需要矯正的偏差改了回去。Savage 的態度精確地示範了上面這條邏輯：**「我違反了它」只是一個描述事實，要不要因此放棄這條規範，取決於我反思後還認不認這條公理，而不取決於我（或大家）有沒有違反它。** 你可以不同意 Savage 的最終選擇（Allais 就不同意），但你必須承認他把「是」和「應」分得很乾淨。這就是和解要求於你的紀律。

## 為什麼「人偏離規範」不證明人笨

另一個方向的誤讀，同樣自然，同樣錯：「好吧，就算規範沒錯，那人系統性違反它，總證明了人挺笨/挺不理性的吧？」

不。這是把第三層（描述）誤當成「一份錯誤清單」。展望理論不是「人類決策 bug 大全」，它是**對一個在真實約束下運作的系統的精確刻畫**，而那個系統往往運作得相當好。把全書的描述發現重新看一遍，每一條「偏離」背後都有一個在真實世界裡站得住腳的理由：

```text
   每個「偏離」背後，都有一個真實約束下的好理由

   描述發現（偏離規範）        真實約束下，它在解決什麼
   ────────────────────       ──────────────────────────────
   損失趨避（失比得重 λ 倍）   生存環境裡，「失去」常常不對稱地致命
                              （餓死 vs 多吃一頓）——對失更敏感是
                              便宜又救命的快速警報（ch09）
   ────────────────────       ──────────────────────────────
   參考點依賴（算變化不算狀態）狀態的絕對值常常無從感知，但「比上次好/
                              壞」便宜可得——用變化當訊號是低成本策略
                              （ch08）
   ────────────────────       ──────────────────────────────
   滿意即可（satisficing）     把搜尋成本算進去後，「夠好就停」可能
                              就是最優；窮舉全選項才是真的不划算
                              （ch13、T10）
   ────────────────────       ──────────────────────────────
   機率權重（高估小機率）       對「極小機率但極嚴重」的事件（捕食者、
                              災難）保持過度警覺，在生存上的代價不對稱
                              ——寧可錯防一千（ch09）
```

這不是說「所以人其實全對」——展望理論的這些機制在**它們演化出來的環境之外**確實會出系統性的錯（在現代金融市場裡損失趨避讓你太早賣掉賺錢的、死抱賠錢的；機率權重讓你同時買樂透又買保險）。重點是更細緻的一句：**這些偏離不是隨機的愚蠢，是有結構、有方向、可預測、且在原生約束下有功能的策略，被搬到了它沒被優化的新環境裡。** 把它叫「人笨」會錯過全部重點——重點恰恰是它**不笨**（有結構、可建模、能預測方向），只是**錯位**（被用在它沒被優化的場合）。

這正是 ch08 的 T8、ch12 的 T7 守的那條線：**參考點依賴、損失趨避不是「人很笨」的指控，是穩定、可預測方向的描述性事實。** 一個能被 v(x)=−λ(−x)^β 這麼乾淨地刻畫、還能預測 Allais 違反往哪個方向發生的系統，配不上「笨」這個字。會說它笨的人，是拿第一層的尺（無限資源理想）去量第三層的系統（真實約束的腦），量出落差就喊「笨」——這恰恰是前面那個「拿無限資源理論最優羞辱真實系統」的錯誤的另一個版本。

所以中央張力的最終回答，完整版是這樣兩句：

> **人系統性偏離規範，既不證明規範錯（因為「是」推不出「應」），也不證明人笨（因為偏離是真實約束下有結構、有功能的策略）。它只證明了一件事：規範與描述是兩套都對的數學，在描述兩件不同的事——一個在說「無限資源下該怎麼選」，一個在說「真實的腦實際怎麼選」。那道裂縫不是 bug，是兩個不同問題各自正確的答案之間，本來就該有的距離。**

我認為這是全書最值得你帶走的一頁。它不只是關於決策理論——它是一種看待「理想」與「現實」落差的成熟姿態：落差不是要消滅的醜聞，是要理解和管理的結構。

## 理性三角：三本書，三個角度，同一個失誤

到這裡，本書自己的故事講完了。但「理性」這個題目比一本書大——本書是「推理六書」家族的一員，它和兩本姊妹書構成一個招牌結構，我把它叫**理性三角**。這一節是這個三角的總述（總綱意義上本書承擔的招牌跨書連結）。

三本書面對的是**同一個決策失誤**，但各站一個角度，回答三個不同的「怎麼」：

```text
   理性三角：同一個失誤，三個角度

                    本書《在不確定中下注》
                    ＝規範基準（該怎樣）
                         ╱  ╲
                        ╱    ╲
                       ╱      ╲
        《思考的陷阱》╱        ╲《如何不騙自己》
        ＝心理機制   ╱          ╲ ＝制度防線
        （怎麼錯）  ╱            ╲（怎麼防自欺）
                  ●──────────────●

   ─────────────────────────────────────────────────
   角            問的問題              本書關係
   ─────────────────────────────────────────────────
   決策（本書）   規範上「該」怎麼算？  我是規範基準，
                 偏離的數學描述是？     展望理論的數學在我這
   ─────────────────────────────────────────────────
   偏誤          心理上「為何」會錯？   我把心理成因
   《思考的陷阱》 雙系統？捷思怎麼跑？   指向它（ch08/09/12/13）
                 為何察覺不到？
   ─────────────────────────────────────────────────
   科學方法      制度上「怎麼」防自欺？ 我把制度防線
   《如何不騙自己》同儕審查、預先註冊、  指向它（ch05/06）
                 可重現——怎麼設計      ＝貝氏確證面互指
                 流程不靠個人意志力
   ─────────────────────────────────────────────────
```

用一個具體失誤把三角串起來——就用全書反覆出現的**基率忽視**（ch05 那個：alert 命中率高、真故障基率低，你收到 alert 時其實多半是偽陽，但你把「有 alert」當成「真有事」）：

```text
   同一個失誤「基率忽視」，三本書各說一段

   ① 規範上該怎麼算（本書 ch05）：
      P(真故障|alert) = P(alert|故障)·P(故障)
                        ───────────────────────────
                        P(alert|故障)·P(故障) + P(alert|沒事)·P(沒事)
      ——後驗要把「故障的基率」乘進去，不能只看敏感度。
        把脊椎檢查的數字搬過來：盛行率 1%、敏感度 99%、偽陽 5%
        ⇒ 後驗只有 16.7%（ch05）。規範說：別把 99% 當成 16.7%。

   ② 心理上為何會錯（《思考的陷阱》談雙系統與基率）：
      快思（系統一）抓住生動的「alert 響了！」，
      基率是個抽象、無聊、不生動的背景數字，被自動忽略。
      ——這是心理機制，本書不展開，指向它。

   ③ 制度上怎麼防（《如何不騙自己》談制度防線）：
      不靠個人記得算貝氏——把基率「烤進」流程：
      alert 旁邊自動顯示「此規則歷史真陽率」，
      runbook 要求先看基率再行動。
      ——這是制度設計，本書不展開，指向它。
```

三段話對應三本書，缺一不可：

- 只有 ① 規範，你知道**該怎麼算**，但不知道**為什麼明知故犯**（凌晨三點你還是把 alert 當真），也不知道**怎麼讓整個團隊穩定地不犯**。
- 只有 ② 心理機制，你知道**為什麼會錯**，但沒有「對的演算法」當標竿（連「錯」都定義不出來），也沒有制度去攔截。
- 只有 ③ 制度防線，你有流程，但不知道流程**該把什麼烤進去**（要烤進「先乘基率」這條規範），也不理解**它在補哪個心理漏洞**。

這就是為什麼這三本書是一個三角而不是三本各自獨立的書：**規範定義了什麼叫對，心理機制解釋了為什麼會偏離，制度防線提供了不靠意志力的補救。** 本書是這個三角的規範角——我負責把「該怎麼算」和「偏離的數學長相」寫透，另外兩個角明確交棒給姊妹書。

工程上你早就活在這個三角裡。**規範**＝你的 design doc 與演算法分析（該怎麼權衡）；**心理機制**＝你知道人在壓力下會慌、會錨定、會被措辭帶著走（為什麼 review 攔得住一個人攔不住的錯）；**制度防線**＝你的 runbook、checklist、code review、預先承諾的回滾條件（不靠任何人凌晨三點還清醒）。**好的工程流程，就是承認人腦的描述性限制（第三層），用規範性的工具（第一層）去補，靠制度（科學方法那本的地盤）讓這個補償不依賴個人狀態。** 你不靠記憶力，你靠 runbook——這句話本身就是整個理性三角的工程版總結。

## 謙遜的收束：知道自己偏向哪，就能補它

和解不是終點，它有一個非常實用的下游。如果你接受了「規範與描述之間有一道穩定、可預測方向的裂縫」，那麼一個直接的推論是：**你可以預先知道自己的決策會往哪個方向偏，於是可以針對性地設計補償。**

這是描述模型「可預測方向」這個性質（T8）最值錢的地方。一個隨機的錯誤無法補償（你不知道它下次往哪偏）；但一個**有方向**的偏離可以——你知道它總往那邊偏，就在那邊架一道欄杆。

```text
   把可預測的偏向，翻成可設計的補償

   你會穩定偏向哪（描述）         架什麼欄杆（規範性流程）
   ──────────────────────       ──────────────────────────────
   損失趨避：太怕一次當機，       預先承諾的回滾/熔斷條件——
   會死守不該守的舊系統           「滿足 X 就回滾」事先寫死，
   （ch09）                      別等事故當下用恐懼決策
   ──────────────────────       ──────────────────────────────
   框架效應：被 RFC 措辭、        刻意把同一決定用「得」和「失」
   被「99% 成功」帶著走          兩種框架各問一次，看答案是否翻轉
   （ch10）                      （翻轉＝你被框架推著走了）
   ──────────────────────       ──────────────────────────────
   沉沒成本：捨不得砍已投入       review 時強制問「如果今天從零開始、
   三個月的專案（ch15）          忘掉已花的，還會選它嗎」——
                                把「往前看」烤進流程
   ──────────────────────       ──────────────────────────────
   基率忽視：把 alert 當真        告警旁自動標真陽率、
   （ch05）                      runbook 要求先看基率
   ──────────────────────       ──────────────────────────────
   過度自信／隧道視野            外部視角：找一個沒投入的人
                                （描述事實）                  看（pre-mortem：「假設它失敗了，
                                                              事後我們會說是因為什麼」）
```

注意這張表的每一行都是同一個動作：**用第一層的規範工具（事先算好的條件、強制的提問、外部的尺），去攔截第三層那個可預測的偏向。** 這就是 ch12 說的「好的流程是設計來補這道落差」的具體化。預先承諾（事先寫死回滾條件）、外部視角（找局外人看）、checklist（把規範烤進步驟）、pre-mortem（強迫想像失敗）——每一個都是在「人會往那邊偏」這個描述知識之上，架一道規範的欄杆。

這就是「謙遜」的精確意思——不是「我承認我會犯錯」這種空泛的自謙，而是**具體到知道自己會往哪個方向犯、然後在那個方向上預先架好欄杆**。一個重要的警告（它也是 ch18 的核心陷阱）：**讀完這本書、知道了這些規範演算法，並不會自動修好你的描述性偏向。** 你讀完 ch09 不會因此就不損失趨避了——知識住在第一層，偏向住在第三層，兩層不會因為你「懂了」就自動對齊。康納曼（Kahneman）自己一輩子研究這些，據他坦言也仍會掉進自己研究的陷阱（這是被廣泛轉述的說法，hedge）。**所以補償必須是制度性的、外部的、不依賴你當下意志力的——這正是為什麼要靠 runbook 而不是靠記憶力，靠 review 而不是靠自律。** 規範知識的用途不是「升級你的腦」，是「讓你設計出補你腦的流程」。

## 直覺的陷阱

這章是哲學收束，最容易踩的陷阱也是觀念性的——把兩個不同層次的東西硬塞進一個對立。

| 陷阱 | 錯誤直覺長怎樣 | 它在哪一步帶你下溝 | 正確版（守住） |
|---|---|---|---|
| **把「描述」與「規範」當成對立的兩個答案**（T7 的全書版） | 「展望理論和期望效用，總有一個是對的吧？既然人實際照展望理論走，那期望效用大概是被推翻的舊理論」 | 你預設了「同一個問題只能有一個對的答案」，於是把兩個**不同問題**的答案擺到擂台上分勝負。但「該怎麼選」和「實際怎麼選」不是同一個問題——讓它們打架就像問「地圖和行人的真實路線哪個對」 | 它們回答兩個不同問題：期望效用答「無限資源下該怎麼選」（規範），展望理論答「真實的腦怎麼選」（描述）。兩個都對，因為它們不在競爭同一個位子。一個描述事實不能反駁一個規範主張（「是」推不出「應」），所以「人這樣選」推不出「期望效用被推翻」 |
| **把「理性」等同於「冷靜計算／壓抑情緒」** | 「理性的人就是不帶感情、純靠算的人；損失趨避那種情緒反應就是不理性」 | 你把「理性」窄化成**實質理性**的一種刻板印象（無限算力的冷血計算機），於是凡是省算力、帶情緒的快速策略都被你打成「不理性」——但這恰恰是程序理性所肯定的東西 | 理性有兩個意思：實質理性（結果對）和程序理性（過程對）。在真實約束下，一個用情緒當快速警報（損失趨避）、用捷思省算力的腦，可能是**程序理性**的——它在它的成本約束下做了對的事。把「理性＝冷血計算」當唯一意思，會讓你把所有務實的、有限資源下的好策略都誤判成缺陷 |

第三個更隱微的陷阱：**以為「和解」＝「都對，所以怎樣都行／無所謂對錯」。** 這是把和解誤讀成相對主義。怎麼自我察覺你掉進去了？徵兆是你開始說「反正規範也只是一種觀點、人愛怎麼選怎麼選」——這就走過頭了。和解的主張很精確、一點都不相對主義：**在「該怎麼選」這個問題上，規範理論有它嚴格的對錯（違反公理就能被 money pump 抽乾，ch03）；在「實際怎麼選」這個問題上，描述模型有它經驗的對錯（預測錯方向就是壞模型）。** 兩個問題各自都有硬標準，和解只是說「別拿其中一個問題的標準去評判另一個問題的答案」。它讓對錯**各歸其位**，不是取消對錯。把和解讀成「怎樣都行」，等於把兩個都有硬標準的問題，攪成一鍋沒有標準的爛泥——那不是這章的主張，那是這章要防的誤讀。

## 紙上推演

純紙筆推演，不必跑任何程式。這章是觀念題，重點在你能不能把區分講清楚。

### 推演題

**題 1（★★，20 分鐘）** 拿一個你親身犯過的決策錯誤（基率忽視、沉沒成本、被框架帶走、損失趨避死守舊系統……挑一個真實的），把它拆成理性三角的三欄：① **規範上該怎麼算**（這個決定的「正確演算法」是什麼，引本書哪一章）；② **心理上為何會錯**（你當時的腦在跑什麼快速策略，為什麼它在那個情境下出錯——這欄指向《思考的陷阱》，你只要點出「這屬於心理機制」即可，不必深究）；③ **制度上怎麼防**（設計一個不依賴你當下意志力的流程，讓下次同樣的人不會再犯）。要求：三欄都要具體到「換一個同事在我的位子也會這樣」的程度，不要寫成自我批評。

**題 2（★★★，20 分鐘）** 口頭／書面題。向一個堅持「行為經濟學證明了人是不理性的、所以那些『該怎麼選』的數學理論都是象牙塔廢話」的同事，把這個立場駁倒。要求你的反駁裡必須清楚出現兩步：(a) 指出他從「人實際怎麼選」（描述）跳到「該怎麼選的理論沒用」（規範）時，偷渡了什麼前提；(b) 用一個「人普遍做不到、但該做的事不因此改變」的非決策例子（如算術），讓他看見那個前提站不住。

**題 3（★★，15 分鐘）** 區分程序理性與實質理性。給定情境：你的服務在凌晨被一個你從沒見過的錯誤打掛，你在資訊嚴重不全、且必須 5 分鐘內動作的壓力下，選了一個「夠安全、能立刻止血」的回滾，事後發現它不是「理論上最優」的處置（如果你有完整資訊和無限時間，有個更省事的修法）。問：(a) 用「實質理性」的尺量，你的決定理性嗎？(b) 用「程序理性」的尺量呢？(c) 這兩個答案不同，說明了「理性」這個詞的什麼？

### 推演解答

**題 1 解答。** 以「基率忽視導致誤判 incident」為範例（你可換成自己的真實案例，結構一樣）：

```text
   失誤：P0 alert 響了，我直接當成真故障，緊急回滾了上一個 deploy，
        事後發現是監控規則偽陽，回滾本身還引發了二次抖動。

   ① 規範上該怎麼算（本書 ch05 貝氏決策）：
      收到 alert，正確動作是先算後驗 P(真故障|alert)，
      把「這條規則歷史真陽率」（基率）乘進去——
      若這條規則 10 次有 8 次是偽陽，後驗低到不該無腦回滾。
      規範說：別把 P(alert|故障) 當成 P(故障|alert)（基率忽視）。

   ② 心理上為何會錯（指向《思考的陷阱》，本欄只點到）：
      凌晨三點、P0 的紅色很生動，「快做點什麼」的壓力壓過
      「先估個基率」的冷靜——這屬於心理機制（雙系統、生動性
      壓過抽象基率），不是本書地盤，見《思考的陷阱》談基率與雙系統。

   ③ 制度上怎麼防（指向《如何不騙自己》，設計外部欄杆）：
      ・告警卡片自動帶上「此規則近 30 天真陽率：20%」
      ・runbook 第一步硬性要求：「回滾前確認真陽率 > 閾值」
      ・高風險處置需第二人 ack（外部視角，攔截單人隧道視野）
      ——不依賴 on-call 凌晨還記得算貝氏。
```

常見錯路一：把第①欄寫成「我應該更小心」——那不是規範演算法，是空泛的決心。規範欄必須給出**具體的演算法**（先乘基率算後驗），這才是本書教你的東西。常見錯路二：三欄寫成同一件事的三種說法。它們必須是**三個不同層次**：①是「對的演算法」、②是「為何腦跑不出對的演算法」、③是「怎麼用流程繞過腦的限制」。如果你的三欄能互相替換，就是還沒拆乾淨。

**題 2 解答。** 反駁「人不理性所以該怎麼選的理論沒用」：

> (a) **你偷渡了一個前提：「人實際怎麼做」能決定「人該怎麼做」。** 你從「行為經濟學發現人系統性違反期望效用」（這是真的，描述事實），跳到「所以那些規範理論沒用」（這是規範主張），中間需要一座橋：「凡是人普遍會犯的，就不算錯／就不該被要求。」這座橋你沒明說，但你整個論證靠它撐著。而它一攤開就垮。
>
> (b) **反例（非決策的）：人類普遍、穩定地心算不出 137 × 256。** 這是個鐵打的描述事實——隨便找一百個人，幾乎沒人能立刻口算對。那麼，這證明「137 × 256 = 35072 這條算術規範是廢話、是象牙塔」嗎？顯然不。它只證明「人腦不是為大數乘法優化的硬體」。算術規範（該等於多少）和人類能力（算不算得出來）是兩回事；後者再普遍，也撼不動前者。
>
> 同理：人違反期望效用（描述）撼不動「無限資源下該怎麼選」（規範）。要說期望效用沒用，你得去論證「它的公理本身不值得遵守」——那是一場規範辯論（可以打，Allais 就在打），但**不能靠「人不遵守它」來打贏**。「人做不到」從來不等於「不該做」。

常見錯路：把反駁停在「人也有理性的一面啦」這種和稀泥。那沒擊中要害。要害是那個**從「是」滑到「應」的邏輯斷裂**，以及那個「人普遍做不到 ⇏ 不該做」的非決策反例——用算術（或任何「明顯該做但人做不好」的例子）把斷裂照亮，對方就無路可退。

**題 3 解答。**

```text
   (a) 用「實質理性」量：嚴格說，不夠理性。
       實質理性問「結果是不是最優」。你的回滾不是
       「有完整資訊＋無限時間下的理論最優處置」，
       所以以這把尺，它落在最優之下。

   (b) 用「程序理性」量：是理性的，甚至是漂亮的。
       程序理性問「在你真實的約束下，你用的程序對不對」。
       約束是：資訊嚴重不全、5 分鐘死線、二次故障代價高。
       在這組約束下，「選一個夠安全、能立刻止血的處置」
       正是對的程序（這就是 satisficing：設一個
       『夠安全＋能止血』的門檻，達標就動手，不追求最優）。
       事後才知道有更省事的修法，不影響當下程序的理性——
       因為當下你不可能知道（資訊不全是約束、不是你的錯）。

   (c) 兩個答案不同，說明「理性」是個有歧義的詞，至少兩個意思：
       實質（結果對）與程序（過程對）。把人腦的決策說成
       「不理性」，常常只是拿實質理性那把尺（隱含無限資源）
       去量一個在真實約束下程序良好的系統——量出落差就喊
       「不理性」。但在它真實的成本約束下，它做了對的事。
```

常見錯路：覺得「(a) 不理性、(b) 理性」是矛盾、一定有一個答錯了。不矛盾——這正是本題的點：**同一個決定，用兩把不同的尺（兩種理性的意思）量，得到兩個都對的答案。** 認得出「我們在用哪把尺」，比爭「到底理不理性」有用得多。這也是整章和解的微縮版：先問「你在問哪個問題」，再談對錯。

## 自我檢核

口頭自答，講得出來才算過關。優先答「為什麼」。

1. 用 design doc 和 postmortem 各自對應「規範」和「描述」，跟另一個工程師解釋為什麼「兩者衝突時」不是「誰錯了」。
2. 程序理性和實質理性差在哪？用「假設無限資源的演算法理論最優」vs「真實機器約束下跑得好的程式」這個比喻講一遍。
3. 三層理性（規範·無限資源／規範·有限資源／描述·真實的腦）各回答什麼問題？為什麼說它們「不是三個對手」？第一層到第二層是「修正」、第一層到第三層是「裂縫」——這兩個關係差在哪？
4. 「人系統性偏離規範」為什麼**不**證明規範錯了？那個壞推論在從「是」到「應」的哪一步斷掉？舉一個非決策的例子（如算術）說明「人做不到 ⇏ 不該做」。
5. 「人系統性偏離規範」為什麼**不**證明人笨？挑損失趨避或機率權重，講出它在真實約束下解決了什麼問題、為什麼「有結構、可預測方向」配不上「笨」這個字。
6. 理性三角是哪三本書、各站哪個角度？用「基率忽視」這個失誤，講出三本書各說的那一段（規範該怎麼算／心理為何錯／制度怎麼防），並說明為什麼缺任何一角都不完整。
7. 「知道了這些規範演算法，就會做出更好的決策」——這句話哪裡不對？為什麼補償必須是制度性、外部、不依賴當下意志力的？
8. 「規範與描述和解了」會不會就是「怎樣都對、無所謂對錯」的相對主義？為什麼不是？（提示：和解讓對錯**各歸其位**，不是取消對錯。）

## 延伸閱讀

- **Herbert A. Simon, 1955/1956 與其後關於有限理性的著作** — 程序理性 vs 實質理性的區分出自 Simon 對「古典經濟學理性人」的長期批判。想看「理性的兩個意思」的源頭，讀他談 bounded rationality 與 procedural rationality 的文章（如收於其論文集者）。本書取其精神，確切篇章請依其著作集查證。
- **Stanford Encyclopedia of Philosophy,「Bounded Rationality」與「Decision Theory」條目** — 把「規範 vs 描述」「程序 vs 實質」這幾組區分講得很清楚、有哲學深度又不掉書袋。讀「normative vs descriptive」那段，正是本章中央張力的學術版。（landscape 交叉核對過的權威來源）
- **Daniel Kahneman,《Thinking, Fast and Slow》（中譯《快思慢想》）的結語部分** — 康納曼本人對「知道了偏誤也未必能改、所以要靠外部與制度補償」這個謙遜收束的第一手陳述。讀最後幾章談「兩個自我」與如何用外部視角補償；本章「靠 runbook 不靠記憶力」的精神出處之一。
- **《思考的陷阱》談雙系統與捷思** — 理性三角的「心理機制」角。本章只講偏離的數學描述與規範後果，人**為什麼**會偏離、**為什麼察覺不到**，屬於心理機制，見那本書。本章是理性三角招牌連結的總收束點。
- **《如何不騙自己》談制度防線與貝氏確證** — 理性三角的「制度防線」角。怎麼設計同儕審查、預先註冊、可重現性等流程，讓「不自欺」不依賴個人意志力，見那本書；它與本書的貝氏決策在「信念更新」這件事上互指。
- **《馴服隨機》談機率的詮釋** — 本章三層裡的「機率」（後驗、基率）怎麼算，機制在那本書。本章只用結果，把機率機制指向它。
