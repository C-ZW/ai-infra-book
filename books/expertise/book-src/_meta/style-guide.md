# 寫作風格指南（內部文件，非書籍內容）

本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名（工作名）：《專家是怎麼煉成的：一萬小時的真相與刻意練習的科學》（id `expert`）。最終書名於 outline gate 由作者定案，撰章時以工作名為準。

## 這本書是什麼、不是什麼

- **是**：一本「欣賞與直覺」的書。目標是讓讀者親身看懂**專業能力是怎麼長出來的**——把「一萬小時」這個被傳壞的整數，還原成它真正的出處與真正的限制；看懂「刻意練習（deliberate practice，又譯精深練習）」不是「練很久」，而是「在能力邊緣、有即時回饋、可重複、針對特定弱點的一個緊繃迴圈」；讀完有「原來進步的引擎不是時數、而是這個迴圈的品質，而且我能把它套到任何技能上」的清醒感，**而且能用自己的話對另一個工程師講清楚迷思被簡化掉了什麼**。
- **不是**：
  - 不是自助手冊／勵志書——不賣「照做就能變天才」的承諾，不開「七天速成」的食譜。本書談的是**機制與證據**：刻意練習為什麼有效、有效到什麼程度、在哪些領域失效、天賦與硬限制在哪裡仍然擋路。讀者讀完得到的是**判斷力**（會用四問診斷任何練習），不是一張訓練菜單。
  - **不是醫療／治療／戒斷／神經介入手冊**。本書談大腦與認知的機制（如預測誤差訊號、記憶鞏固、心智表徵）純粹是為了解釋「練習為何能改變表現」；**本書不提供任何醫療建議、不診斷、不開藥、不給戒斷或治療方案、不建議補品或腦力訓練產品**。凡涉及健康、用藥、臨床狀況，一律明說「這超出本書範圍，請諮詢專業人士」。談 ADHD、睡眠、神經可塑性時只談「機制是什麼、科學共識到哪」，**絕不給「你該怎麼做」的醫療指示**。
  - 不是教科書——不刷認知心理學的名詞表、不背實驗清單、不出考題。實驗與理論的歷史敘事**服務於機制理解**，不是編年史；統合分析、複製研究只取「它推翻了什麼直覺、留下什麼結論」，不抄方法學細節。
  - 不是轉職用書——純興趣，不服務 AI Infra 敘事。但工程橋接要讓它讀起來像母語：讀者的 profiling／測量驅動優化、紅綠燈式單元測試回饋、code review 當外部回饋、把大技能拆成可測量子技能逐項攻克——這些是本書最划算的橋，要用滿但不喧賓奪主。
  - **不是純紙上推演書，但也沒有真實世界自我追蹤實驗**：本書的「動手做」一律是**思想實驗／推演**（預測一個研究結果再對答案、找出一個壞論證的破綻、把一個白話宣稱翻成機制、設計一個假想實驗、口頭重講一條因果鏈）。**不要叫讀者寫睡眠日記、記憶測驗、自我追蹤練習時數**——這既越過了「不下醫療/行為建議」的界線，也不是本書要的東西。本書要的是看懂機制、能診斷別人的宣稱，不是改造讀者的生活。

- 驗收標準（全書六條主線，finale 章總收官逐條口試）：
  1. **刻意練習的精確四要件定義**：踩在能力邊緣（剛好夠難、不舒適）、有即時且明確的回饋、可重複（針對同一弱點反覆磨）、針對一個具體弱點（不是泛泛地「多做」）。能用這四問判定任何「練習」是刻意練習還是空轉。
  2. **「一萬小時」數字的真實來源與被簡化成什麼**：它是 Ericsson 1993 小提琴研究中「最佳組到 20 歲的**平均**累積獨自練習時數」，**不是門檻、不是保證、原文無『法則』一詞**；「法則」框架是 Gladwell《Outliers》2008 造的，Ericsson 本人在《Peak》2016 親自反駁。
  3. **心智表徵（mental representation）為何是專家能力的核心**：專家不是練出更強的通用腦，而是練出更精細的**領域特定**心智表徵（棋手的盤面組塊、醫師的病徵模式）；Chase–Simon 的隨機盤面實驗是最乾淨的證據。
  4. **刻意練習到底解釋了多少表現變異、領域差異多大**：Macnamara 2014 統合分析——遊戲 26%、音樂 21%、運動 18%、教育 4%、職業 <1%；領域落差本身就是論點。並呈現 Ericsson 陣營與 Macnamara 陣營的爭議，不選邊。
  5. **天賦與硬限制並未消失**：基因、工作記憶、起步年齡（敏感期）等「練習量之外」的因素真實存在；刻意練習很重要，但不是唯一、也不是決定性的全部。
  6. **怎麼替任一技能設計自己的緊繃迴圈**：把四問落地成「量測→診斷弱點→設計可重複的邊緣練習→即時回饋→再量測」的迴圈（讀者的 RDS CPU 降 40% 招牌敘事就是一次這樣的迴圈）。

## 讀者是誰

唯一讀者：Chewei（陳哲偉），6 年以上資深後端工程師（TypeScript／Node.js／NestJS、AWS、K8s、分散式系統、效能工程、可觀測性），政大資科所選修過演算法與機器學習、淡江資工學士。神經科學／心理學從零起步，但有大量系統工程直覺可橋接。招牌敘事資產：profiling 驅動讓尖峰 RDS CPU 降 40%（量測→診斷→優化→數據佐證——這正是一次刻意練習迴圈的工程版）。

下面三欄是寫作時校準深度的依據：

**不需要解釋**（讀者已精通，可直接當橋接錨點）：
- 效能工程：profiling／測量驅動優化、找瓶頸、上下界夾擠、火焰圖式的「哪段最熱」。
- 回饋迴圈：單元測試的紅綠即時回饋、TDD、CI gate、code review 當外部回饋來源。
- 分而治之：把一個大技能／慢查詢拆成可測量子技能／子問題逐項攻克（慢 SQL 拆成 index／查詢計畫／連線池逐一調）。
- 系統直覺可借用的詞彙：快取失效與命中率、GC、排程器、批次維護窗、backpressure、正回饋迴圈、event sourcing、idempotency、TD/prediction-error（RL 的時序差分與預測誤差，他選修過 ML）、variable-ratio reinforcement（變動比率增強）。
- checklist／協定／迴圈化思維（他偏好把方法固化成迴圈）。

**快速複習即可**（看過或聽過但生鏽）：
- 統計直覺：變異被解釋的百分比（R²／效應量的白話）、相關 vs 因果、樣本與母體、為什麼平均會誤導。一句話喚醒、不從頭教。
- RL 的 TD/預測誤差骨架（他選修過 ML，知道「預期與實際的落差驅動更新」這個形狀），本書借它類比 dopamine 預測誤差時可一句話接上。
- 統合分析（meta-analysis）是什麼（把多份研究的效應量彙整）——白話帶過即可。

**要從零教**（本書主體，神經科學／心理學）：
- 刻意練習的精確定義與四要件、它與「一般練習／累積經驗時數」的差別。
- 心智表徵（mental representation）、組塊（chunking）、專家的知覺優勢。
- 表現變異的歸因（刻意練習解釋多少、其餘留給什麼）、kind vs wicked 學習環境、高效度環境與可靠直覺的條件。
- 遠遷移（far transfer）為何幾乎不存在、近遷移、腦力訓練 app 的證據。
- 敏感期（sensitive period）vs 強版關鍵期、成人神經可塑性的真實程度。
- 天賦的角色：工作記憶、起步年齡、遺傳對「練習反應度」的影響——這些都從零講機制與證據，但每處都橋接到讀者的系統直覺。

**跨書邊界（本書完全獨立，不引用其他書）**：本書系四本主題書（專家養成／其餘三本）**互不引用、各自完整**。即使讀者書架上有 ML 相關書，本書談 TD/預測誤差也只當「讀者已有的直覺」一句話接上，**不寫「見某書某章」**。本書不依賴任何外部章節。

## 記號與數字規範（全書一致；閱讀器不渲染 LaTeX，一律純文字）

- **效應量／百分比照基準**：所有「刻意練習解釋 X% 變異」的數字、所有時數、樣本數、年份，以 `_meta/landscape-2026-06.md` 為唯一錨點，**照抄、不四捨五入到好記整數、不憑記憶**。Macnamara 2014 五個百分比必須**逐一標領域**（遊戲 26%／音樂 21%／運動 18%／教育 4%／職業 <1%），**禁止給單一平均值帶過**——落差就是論點。
- **「一萬小時」的鐵律寫法**：每次提「一萬小時」都要**立刻接**「這是 Ericsson 1993 最佳小提琴組到 20 歲的**平均**累積獨自練習時數，不是門檻、不是保證」。這是全書脊椎的支點，不得省略這個限定。
- **嚴謹度標籤（每個宣稱必帶其一）**：`✅ 穩固共識`／`🟡 主流假說但有爭議`／`🥊 重大爭議（雙方並陳、不選邊）`／`❌ 這是迷思，正解是…`。對應 landscape 的四級符號。撰章時把這個分級寫進正文語氣，讓讀者隨時知道腳下是岩盤還是流沙。
- **脊椎四問的標準排版**：把任何一次「練習」丟進四問，用 ```text 圍欄區塊排成一張診斷表（四列：能力邊緣？／即時回饋？／針對哪個具體弱點？／可重複？），逐問打勾或打叉。這是全書脊椎視覺，**ch 早期定版、後續章照抄變形**（套到不同技能、套到讀者的 RDS 案例、套到一個「假刻意練習」的反例上）。
- **多步機制鏈／論證骨架**放 ```text 圍欄區塊，每步右側可加「← 為什麼」短註。禁止把長推導硬塞進行內。
- 時效性／可爭議事實標時點 `（2026-06）`，並 hedge（見「事實正確性」）。

## 每章骨架（必須遵守，標題名稱逐字照抄——一致性掃描 lint_book.py 會 grep 這些標題，順序固定）

1. 章首：`# chNN — 章名：副標`，緊接 blockquote：
   `> **本章解決什麼問題**：…（這章在全書地圖的哪裡、為什麼存在、跟前後章的關係，3-5 行）`
   每個 Part 的第一章在 blockquote 後放**全書 ASCII 路線圖**（基準圖在 outline.md，照抄並把「◄你在這裡」標到本 Part；CJK=2 欄對齊），讀者偏好架構圖先行。
2. `## 從你已知的出發` — 橋接段：把本章核心概念錨定在讀者的後端／系統工程實戰直覺上，要**具體**（profiling 找瓶頸、單元測試紅綠回饋、code review 外部回饋、把慢查詢拆成子問題逐一調、RL 預測誤差、快取命中率…），不要泛泛而談。
3. 核心內容（多個 `##` section，深入機制與直覺）。歷史與實驗敘事服務於機制理解，不是編年史。
4. `## 直覺的陷阱` — 本書招牌段落（故障視角的科普版）：這個主題**最常被誤解成什麼科普迷思**、錯誤直覺**會在哪一步把你帶溝裡**、**怎麼自我察覺**（什麼徵兆代表你其實被話術騙了）。用表格或條列。這四個主題迷思密度極高，這段是全書最有價值的段落之一——例：「練滿一萬小時就會成專家」「練習越久越強，時數就是一切」「腦力訓練 app 能讓你整體變聰明」「過了某歲就學不會了／大腦定型」「天賦不存在，努力決定一切」（反向迷思也要拆）——這些都必須正面拆解。
5. `## 紙上推演` — 動手做的形式（**思想實驗／推演為主，無 Python、無自我追蹤實驗**）。每章 2~4 題，每題標難度（★~★★★）與預估時間（**[NN 分鐘]**）。題型：預測一個研究的結果再對答案、找出一個壞論證的破綻、把一句白話宣稱翻成機制、設計一個假想實驗、口頭重講一條因果鏈、把某個「練習」丟進脊椎四問判定它是不是刻意練習。題目後緊接 `### 推演解答` 給完整解答與討論（為什麼這樣想、常見錯路、研究怎麼說）。
6. `## 自我檢核` — 5~8 個問題，**口頭自答**：講得出來才算過關，講不清楚＝還沒懂。深度對齊「能不能把這個概念講給另一個工程師聽」，**優先出「為什麼」而不是「是什麼」**。
7. `## 延伸閱讀` — 真實連結（以 `_meta/landscape-2026-06.md` 驗證過的為優先），每條一句話說明為什麼值得讀、讀哪一段。沒把握的連結標「（未驗證）」。

## 深度標準

- **每章至少一個 worked example**：本書系的 worked example 是「**一條走到底的機制鏈**」或「**一個真實研究的數字逐步解讀**」，不准跳步。具體形式：
  - 把一個統合分析的百分比走一遍（如「26% 變異被解釋」是什麼意思、那剩下的 74% 去哪了、為什麼職業 <1% 卻不代表練習沒用）。
  - 把一條機制鏈走到底（如 Chase–Simon 隨機盤面實驗：正常盤面大師近乎完美重建 → 隨機盤面降到接近生手 → 因此優勢來自領域特定模式再認、不是通用記憶力）。
  - 把一個複製研究的數字並排（如 2019 複製：最佳組 8,224 小時 vs 良好組 9,844 小時、差異不顯著 → 推翻的是「時數乾淨對應水準」這個強版本，不是「練習無用」）。
  - 把脊椎四問套到讀者自己的招牌敘事（RDS CPU 降 40%：量測＝profiling、即時回饋＝CPU 指標、具體弱點＝某條熱查詢、可重複＝改完再量）走一遍，證明那是一次刻意練習迴圈。
- **嚴謹度的誠實標示**：每個宣稱明確標示自己是哪一級——`✅ 穩固共識`／`🟡 主流假說但有爭議`／`🥊 重大爭議（雙方並陳）`／`❌ 這是迷思，正解是…`。**禁止「顯然」「眾所周知」「易見」**；禁止把單一研究的結論講成定論；禁止把暢銷書的修辭當成科學共識。
- **重大爭議呈現雙方、不替讀者選邊**：凡 landscape 標 🥊 的（刻意練習解釋多少、定義可否證偽、敏感期硬度與成人天花板），**兩造都要寫、標時點（2026-06）、不下單一斷言**。寫法是「A 陣營主張…，理由是…；B 陣營回應…，理由是…；爭議的核心其實是…」——把爭點本身講清楚，比假裝有答案更有價值。
- **誠實對待 Gladwell 與暢銷化**：不要把 Gladwell 寫成騙子或造假——他是**通俗化過程中的放大失真**（好故事壓過了原研究的保留條件）。本書的價值正在於把失真還原回去，靠的是把證據攤清楚，不是罵人。
- **每個概念給「它在解什麼問題」**：刻意練習解什麼（為什麼「練很久」不夠）、心智表徵解什麼（專家的優勢到底是什麼）、kind/wicked 解什麼（為什麼有些領域練不出可靠專業）、敏感期解什麼（年齡到底擋了什麼）。禁止無動機的定義。
- **脊椎每章復現**：每章至少一處把本章內容收回脊椎四問——「這跟『踩在能力邊緣／即時回饋／具體弱點／可重複』哪一問有關？」「這個迷思之所以成立，是因為它把『副產品（時數）』誤當成了『原因（迴圈品質）』。」脊椎是同一組診斷問句，反覆出場、反覆磨利。
- 寧深勿廣，不灌水。每章 5,000~9,000 中文字，依主題深度調整（爭議厚的章如 Macnamara 統合分析、敏感期，可偏厚）。

## 事實正確性（神經科學／心理學科普的零信任守則）

- 神經科學／心理學科普充斥**被過度簡化、被推翻、或仍有爭議**的「事實」。凡是版本、數字、效應量、年份、歸屬、樣本數，一律當作**可能過時或被誤傳**，必須查證並標注爭議。
- 歷史與時效性事實（年份、人物、引文、論文歸屬、書籍版本、連結、效應量數字）以 `_meta/landscape-2026-06.md` 為事實基準；本章特定細節**自行用 WebSearch/WebFetch 查證（每章 3~8 次）**並 hedge。**不要從二手科普轉抄、不要憑記憶**。
- 重大修正（推翻既有基準）需**兩個獨立來源**；修正後更新 landscape → 依基準同步所有引用章 → 在 maintenance.md 掃描日誌記一行。
- **必須守住的高風險事實**（最易被科普寫錯，務必對照 landscape）：
  1. 「一萬小時」是 Ericsson 1993「最佳小提琴組到 20 歲的**平均**累積獨自練習時數」，**不是門檻、不是保證、原文無『法則』一詞**。
  2. 「法則」框架是 Gladwell《Outliers》2008 造的；Ericsson《Peak》2016 親自反駁，且 2020-06-17 卒於 Tallahassee。漂亮的反諷——**別寫成 Ericsson 自己主張法則**。
  3. Macnamara 2014 五個百分比逐一標領域，別給平均、別張冠李戴（最常見錯：把職業 <1% 或某格說成全部）。
  4. 2019 複製**沒有推翻刻意練習有用**，推翻的是「時數乾淨對應水準、解釋幾乎全部」這個強版本（最佳組 8,224h 反而比良好組 9,844h 練得少、不顯著；效應從約 48% 縮到約 26%）。**別寫成「刻意練習被證明無效」**（那是過度反向）。
  5. 遠遷移幾乎不存在＋腦訓 app：Owen 2010 Nature（n=11,430，只進步在受訓任務）；**共識聲明是 2014（73 人）、Simons 回顧是 2016**——年份與性質別合併、別寫錯。
  6. kind/wicked 是 **Hogarth**（2001/2015）；「高效度環境才有可靠直覺」是 **Kahneman & Klein 2009**——別混掛給 Ericsson 或某本暢銷書。
  7. 用「**敏感期**」而非「關鍵期」，**兩面講**：成人可塑性終生在（駁強關鍵期），但敏感期效應在某些能力真實（駁「年齡無關」）。**不對「成人能追到多少」下單一斷言**。
  8. chunking 的「約 50,000 組塊」是 Chase–Simon 的**數量級估計**、非精測，要標明。
- 引文與名言：禁止憑記憶引用「名人名言」。Gladwell 的「神奇數字」措辭坊間略有出入，引用時寫「大意是…」、別假裝逐字。
- 延伸閱讀只放真實存在的連結；沒把握標「（未驗證）」。本書連結多為論文（PubMed／PMC／期刊原文）與訃聞／權威百科，優先給原始來源而非二手科普。

## 語氣

工程師對工程師。冷靜、直接、有觀點（可以寫「我認為這是整本書最重要的一張表」「這個迷思之所以難殺，是因為它太好記了」）。**不販賣神話、不喊「太神奇了」、不喊「太厲害了」**——震撼要靠把機制講清楚造成，形容詞是結果不是手段。允許冷幽默。

對「一萬小時」這種被傳壞的迷思，語氣是**還原者**而非揭發者：不嘲諷信了的人（這個數字好記又勵志，誰都會上鉤），而是耐心把「副產品（時數）被誤當成原因（迴圈品質）」這條線一步步拆給讀者看，拆到他自己會用四問去防身。對 🥊 爭議，語氣是**誠實的中立**：把兩造的最佳論證都寫到位，讓讀者看見「這題目前真的沒有定論、而爭點本身很有意思」，而不是偷偷站隊再假裝客觀。對天賦／硬限制，語氣是**不販賣勵志也不販賣宿命**：練習很有用、但不是萬能；基因與起步年齡真實存在、但不是判決書。每個符號、每個百分比出場都有白話直覺在旁邊；該慢的地方明說「這裡值得你停十分鐘，因為等一下整本書的判斷力都靠它」。
