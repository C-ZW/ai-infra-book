# ch04 — 「誰最強」是測不準的：評測政治學

> **本章解決什麼問題**：ch03 交給你五變數這把尺，下一步該拿它去逐家拆解了——但在進 Part II 之前，得先處理一個會毀掉整本書的天真：「排行榜上分數最高的，就是最強的模型。」這章在「同一張藍圖、不同養法」地圖上的位置很特別：它不歸因任何一家，而是先卸掉「排名＝真相」這個錯誤前提，安裝一副誠實的比較眼鏡。它回答的不是「哪一家為什麼強」，而是更前置的「『強』這件事，到底測不測得準」。結論先講：截至 2026-06，沒有單一冠軍、排名每幾個月洗牌、benchmark 會被污染也會飽和——能穩定辨識的是各家的**風格**，不是名次。這個 caveat 是全書的脊椎，後面每一家的「以…著稱」都掛在它上面。

## 從你已知的出發

你看過太多漂亮的平均延遲了。

你跑 k6 壓測一個 NestJS 服務，報告吐出來：平均回應時間 80 毫秒。漂亮。你正要把這個數字貼進週報，但你停了下來——因為你被 p99 坑過。你把分位數攤開來看：p50 是 45 毫秒，p95 是 120 毫秒，p99 是 **2,300 毫秒**。那條尾巴在燒。平均之所以漂亮，是因為它把一小撮災難性的慢請求，溶進了一大堆飛快的請求裡。那 1% 的使用者，每一次操作都在等兩秒半，而你的「平均 80 毫秒」對他們的痛苦一無所知。

你早就學會：**單一數字會騙人。** 一個 summary statistic 是把一整個分布壓成一個點，壓的過程必然丟掉資訊——而丟掉的那部分，往往正是你最該擔心的部分。所以你看壓測報告從不只看平均：你看分布、看尾巴、看這個負載**是在什麼場景下打的**（讀多寫少？冷快取還是熱快取？併發 100 還是 10,000？）。同一個服務，換一種流量形狀，數字可以天差地遠——報告底下那行小字「scenario: ...」比那個大大的平均數字重要得多。

現在把這套直覺，原封不動搬到 AI 模型的排行榜上。

「GPT-5.5 在某 benchmark 拿 92 分」——這就是那個漂亮的平均延遲。它是一個 summary statistic，把「這個模型在數學、程式、長文、多語言、agentic 任務上的千百種表現」壓成了一個點。它沒告訴你這 92 分是在什麼**場景**下測的、分布的尾巴有沒有在燒、這道題模型是**真的推理出來**還是**背過答案**。本章要做的，就是教你用看 k6 報告的眼睛去看 AI 排行榜：別信那個大數字，去看分布、看場景、看它怎麼測出來的。一個排行榜，就是一份只印了平均值、藏起了 p99 和 scenario 的壓測報告。

## 排行榜上那些數字，各自在量什麼

先把場上的主要量尺認清楚。截至 2026-06，業界在看的 benchmark 大致分三種**類型**——記住「三種類型」這件事，是本章最重要的工具，後面會反覆用到。

```text
評測的三大類型（量的東西根本不同）

  類型一：靜態學科考      類型二：人類偏好擂台    類型三：agentic 實戰
  (static academic)      (human-preference)     (agentic suite)
  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
  │ MMLU-Pro     │      │ LMArena      │      │ SWE-bench    │
  │ GPQA Diamond │      │ (前 Chatbot  │      │  Verified    │
  │ AIME         │      │  Arena)      │      │ Terminal-    │
  │              │      │              │      │  Bench       │
  └──────────────┘      └──────────────┘      └──────────────┘
  固定題庫、對答案       真人盲投、算 Elo       真實任務、看成敗
  量「會不會」          量「人比較喜歡誰」     量「做不做得成事」
  弱點：會被背、會飽和   弱點：偏好≠正確       弱點：難建、跑得慢
```

- **MMLU-Pro**：硬版的多選知識測驗（約 12,000 題、每題 10 個選項）。它量的是「跨學科知識答得對不對」。截至 2026-06 它**已經接近飽和**——前沿模型擠在 88~94% 這個窄帶裡，彼此差幾分，根本分不出高下。
- **GPQA Diamond**：研究所等級的科學推理題（生物、物理、化學的「Google 都查不到答案」那種）。它本來是用來把模型考倒的，但截至 2026-06 前沿模型已經逼近 ~94%，**也在飽和邊緣**。
- **AIME**：美國高中數學競賽題。reasoning 模型（見 ch14）在這上面已經接近滿分。
- **SWE-bench Verified**：這是**agentic coding 的關鍵量尺**——拿真實 GitHub 上的 Python issue（一個經人工驗證過的 500 題子集），讓模型像工程師一樣讀 codebase、改程式、跑測試，看它**真的把 issue 修好沒有**。這量的不是「會不會」，是「做不做得成」。
- **LMArena**（2026-01-28 從 Chatbot Arena / LMSYS 改名為「Arena」）：完全不同的玩法。它讓真人同時看兩個匿名模型的回答、盲投誰比較好，再用下西洋棋那套 **Elo** 把所有對戰結果換算成排名。它量的不是對錯，是**人類偏好**。
- **Terminal-Bench、ARC-AGI-2**：較新的 agentic／抽象推理量尺，因為新，還沒被刷飽和。

光是把這張表看懂，你就該起疑了：MMLU-Pro 量知識、LMArena 量偏好、SWE-bench 量任務成敗——**這三個根本不是同一個維度**。問「哪個模型最強」就像問「哪個資料庫最快」：你不先講清楚是 OLTP 還是 OLAP、是點查還是全表掃描、是讀多還是寫多，這個問題根本沒有答案。「最強」會依任務裂開：**coding 強 ≠ 數學強 ≠ 長上下文強 ≠ 多語言強**。一個在 AIME 屠榜的模型，可能在 SWE-bench 上修不好一個真實的 bug。

## 兩道裂縫：背答案（污染）與擠成一團（飽和）

就算你只盯著其中一種類型的分數看，那個數字本身還有兩道結構性的裂縫，會讓它系統性地騙你。

**第一道裂縫：污染（contamination）——分數量到的是「背誦」，不是「推理」。**

benchmark 的題目都是公開的，掛在網路上。而模型的 pretraining（見 ch02）是把幾乎整個網路吞下去。你看出問題了嗎——**考卷在考試前就被吞進了考生的腦子裡。** 當測驗的題目（甚至答案）洩漏進了訓練資料，模型在考場上做的就不是「推理」，而是「回憶」：它見過這道題，背得出答案。分數被灌水了，但它量到的能力是假的。這對你不陌生——這就像你用一份**訓練時就看過的測試集**去評估一個機器學習模型，準確率高得嚇人，一上線就垮，因為它根本沒學會「泛化」，只學會了「記住這幾筆」。

**第二道裂縫：飽和（saturation）——尺已經量不出差異了。** 當 MMLU-Pro 上前幾名都擠在 88~94%，這把尺對「分辨前沿模型」這件事已經失效了。剩下那幾分的差距，落在量測誤差的範圍裡（這正是下一節要接的機率問題），不再是能力的真實訊號。一把所有人都考 90 分以上的考卷，分不出班上的前三名——它只證明這份考卷對這群人太簡單了。這也是為什麼業界得不斷推出 GPQA、ARC-AGI-2 這些更難的新尺：舊尺被爬到頂了。

這兩道裂縫合起來，就解釋了那個你一定觀察過的怪象：**排名每幾個月就洗一次牌。** 一部分是真的——各家確實在快速進步、輪流推出新旗艦。但另一部分是假的：新模型發表時的「屠榜」分數，常常摻了對熱門 benchmark 的過擬合（針對考古題最佳化），等到有人拿一份**全新、沒外洩過的題目**去重測，分數就縮水了。一個排名的保鮮期，常常只有幾週。

## 招牌個案：GSM1k 撕開的那 13 分

把「污染」從一個抽象的擔憂，變成一個有名有姓、帶真實數字的故事——這是 Scale AI 在 2024 年做的一件漂亮事。

有一個經典的小學數學應用題 benchmark 叫 **GSM8k**，當時幾乎每家模型的技術報告都會秀自己在上面的高分。但它太紅了、題目在網路上到處都是——也就是說，它是污染的高風險區。Scale AI 的研究者起了個疑：這些高分裡，有多少是「真的會算」，有多少是「背過題目」？

他們的做法，本質上是你做 ML 評估時最樸素也最有效的一招——**換一份模型絕對沒見過的新測試集**。他們請人手工出了 **GSM1k**：1,205 道全新的小學數學題，刻意做到跟 GSM8k 同樣的風格、同樣的難度分布，但**一題都沒在網路上出現過**。然後拿同一批模型，在新舊兩份考卷上各考一次，看分數掉多少。掉得越多，代表它在 GSM8k 上的高分越是「背」出來的。

結果（Scale AI 的論文，arXiv:2405.00332，2024）很說明問題：

- **Mistral 與 Phi 這兩個家族，幾乎在每個尺寸上都系統性地掉分，最多掉到約 13 個百分點。** 它們在熱門的 GSM8k 上看起來很能算，換到沒見過的 GSM1k 就露了餡——這 13 分，量到的是背誦，不是推理。
- 對照組：**GPT、Claude、Gemini、Llama 這幾個家族，新舊考卷之間幾乎沒掉分（差距 <3%）。** 它們是真的會算。
- 更狠的證據：研究者發現模型「能不能背出 GSM8k 原題」的傾向，跟它「新舊考卷的分差」是正相關的（Spearman r²≈0.32）——越是背得出原題的模型，掉分掉得越兇。這幾乎是污染的指紋。

這個個案的價值不在於羞辱某幾家（注意：那是 2024 年的快照，各家後來都修正了流程），而在於它給了你一把**反污染的檢驗刀**：看到任何 benchmark 高分，問一句「拿一份它沒見過的同型題重測，分數還在嗎？」——這正是你做 ML 不會用訓練集當測試集的同一條紀律。它也順手示範了本章的一個側論點：**會背答案的，常是被緊逼著刷分的較小模型**；最前沿的大模型反而泛化得好——但這結論一樣有保存期限，別當定律。

## 接上你的機率書：為什麼小樣本排名不可靠

這一節把你書架上《馴服隨機》那本接過來——你已經有這套直覺，這裡只是喚醒它，套到排行榜上。

LMArena 的 Elo 排名，本質是**抽樣**。它不是上帝視角下「A 模型客觀比 B 強多少」，而是「在投過票的這群人、這批 prompt 上，A 贏 B 的次數比較多」。這裡藏著兩個你熟得不能再熟的陷阱：

- **抽樣偏誤（sampling bias）**：投票的是誰？多半是會上 LMArena 的科技愛好者，他們的偏好不等於全人類的偏好。問的是什麼 prompt？如果擂台上的 prompt 偏向某類任務（比如閒聊多、硬核 coding 少），擅長那類任務的模型就佔便宜。**樣本不是母體**——你抽樣的那群人和那些題目，決定了排名長什麼樣。這跟你做 A/B test 時最怕的「流量沒分均、被某個地區的尖峰污染了結果」是同一個病。
- **小樣本不可靠（信賴區間）**：一個剛上架、只被投了幾百票的模型，它的 Elo 帶著一個很寬的不確定性。排名第 3 和排名第 7，如果它們的信賴區間大幅重疊，那這個「第 3 vs 第 7」的差別**在統計上可能根本不存在**——就像你拿 5 個請求的延遲樣本去估 p99，那個數字基本是噪音。好的排行榜會標出信賴區間（Elo 的 ± 範圍），而多數轉述排名的新聞會把那個 ± 砍掉，只留一個看起來很確定的名次給你。

把這兩件事合起來，你就有了對抗「排名＝真相」最鋒利的武器：**任何一個名次，背後都是一次抽樣；問抽了誰、抽了什麼、樣本夠不夠大、誤差棒有沒有重疊。** 你不需要會算 Elo 的公式，只需要保留「這是抽樣、抽樣會騙人」這個警覺——這本書要你帶走的是警覺，不是公式。

## 那該怎麼誠實地比較？三角驗證

卸掉了天真，總得給一副能用的眼鏡。截至 2026-06，業界的最佳實務不是「找一個最權威的 benchmark」——**沒有這種東西**——而是**三角驗證（triangulate）**：

> 同時看三種**不同類型**的訊號——一個靜態學科考（MMLU-Pro／GPQA）、一個人類偏好擂台（LMArena）、一個 agentic 實戰套件（SWE-bench／Terminal-Bench）——**只有當三者一致指向同一個結論時，那才是真訊號。**

為什麼這招有效？因為這三種類型的**弱點不一樣，所以不會一起被同一個花招騙過**。針對考古題刷分的污染，能灌水靜態學科考，但騙不過真人盲投的偏好擂台，也修不好 SWE-bench 上一個真實的 bug。一個只會討好人類評審、實際能力空虛的模型，能在 LMArena 拿高分，但會在 SWE-bench 上原形畢露。**當一個花招只能騙過其中一種尺，三尺一交叉，它就藏不住了。** 這跟你做系統可觀測性的直覺一模一樣：你從不只信一個指標，你交叉看 metrics、logs、traces——因為任一個單獨都能說謊，但要它們**同時**用一致的方式說謊，極難。三角驗證就是把這套 observability 紀律，套到「評估一個模型」上。

這也正是為什麼本書從頭到尾**拒絕下絕對排名**。不是因為謹慎到迂腐，而是因為「最強」這個提問本身就是壞掉的：它假設有一個一維的、跨任務、跨時間都成立的冠軍，而現實是多維的、會洗牌的、量起來帶誤差棒的。本書能誠實交付的，不是名次，是**成因的結構**——這也是為什麼接下來八章，每一家我都只寫「截至 2026-06，以…著稱／在…benchmark 上領先」，並當場附上那個 benchmark 的但書，絕不寫「X 是最強的模型」。

## 成因拆解

本章的現象不是「某一家為什麼強」，而是「為什麼『最強』這件事本身測不準」。所以這節的歸因，不是歸到某一家身上，而是歸到**評測這件事的結構性現實**——這些現實又恰好對應到五變數的某幾格在「製造比較的混亂」。

**(a) 五格歸因（這次歸的是「為什麼測不準」這個現象）**

| 變數 | 角色 | 為什麼 |
|---|---|---|
| 資料 | ✓ 主因 | 污染的根源在這格：benchmark 題目是公開網路文本，pretraining 把它一起吞了，考卷洩進了訓練資料，分數量到背誦而非推理。 |
| 後訓練·對齊 | △ 次因 | 針對熱門 benchmark 過擬合（刷榜）發生在後訓練；對齊出的「討喜風格」會灌水 LMArena 的人類偏好票，偏好≠正確。 |
| 算力·基礎設施 | （空白） | 算力決定模型多強，但不直接決定「強不強測得準」——它不是這個現象的成因。 |
| 研究文化·品味 | △ 次因 | 各家對「該誠實揭露污染、還是挑漂亮 benchmark 報」的取捨，是文化差異；也決定了一家會不會沉迷刷榜。 |
| 產品取向 | △ 次因 | 「強」依任務裂開（coding≠數學≠長文），而模型被養成擅長哪類任務，往往是被它要服務的產品壓出來的——所以拿單一 benchmark 排名，本就在拿錯維度比。 |

一句話收束：**「最強」測不準，主因記在〔資料〕（污染）這一格，次因散在〔後訓練〕（刷榜、討喜）、〔研究文化〕（揭不揭露）、〔產品取向〕（強項依任務裂開）；唯一穩定可測的不是名次，是各家的風格。**

**(b) 影子：三角驗證也有它的代價與盲點。** 別把三角驗證當成萬靈丹——它一樣有影子。其一，**它治不了三尺同時失效**：如果一個任務類型（比如多模態、超長上下文、小眾語言）根本還沒有好的 benchmark，三角驗證就無從交叉，你只剩下傳聞。其二，**它治不了系統性偏誤**：如果三種尺剛好都偏袒同一類能力（例如都偏英文、都偏短任務），它們會一致地騙你，而「一致」反而讓你更信。其三，**它很貴、很慢**：跑齊三類評測、尤其 agentic 套件要實際執行任務，成本和時間都遠高於跑一個多選題庫——這正是為什麼大眾媒體圖省事，只轉述一個 LMArena 名次。換句話說，誠實的比較**本身就有成本**，而便宜的不誠實永遠更好傳播。

**(c) 何時不成立：這是截至 2026-06 的快照，不是定律。** 評測這個領域本身在快速演化。今天接近飽和的 MMLU-Pro／GPQA，是因為模型追上了尺——明天會有更難的新尺把前沿模型重新拉開差距（ARC-AGI-2 就是這種嘗試）。今天「較小模型才會背答案、大模型泛化好」的觀察（GSM1k 個案），也只是 2024 的快照，隨各家流程改進會變。甚至「benchmark 會被污染」這件事，也催生了反制（私有題庫、動態出題、防洩漏協定），長期可能局部緩解。唯一不會過期的，是那條後設原則：**任何單一數字都在壓縮資訊，壓掉的那部分往往最關鍵——所以永遠看分布、看場景、交叉驗證。** 這條，跟你看 k6 報告的紀律一樣，不會過時。

## 紙上推演

### 推演題1 — 把「最強測不準」講給另一個工程師聽 [10 分鐘] ★

你同事看到新聞標題「某模型登上 LMArena 第一」，興奮地說「那它就是現在最強的模型了」。用看 k6 壓測報告的比喻，三句話內讓他明白為什麼這句話有問題。

#### 推演解答

關鍵是把「排名＝真相」對應到「平均延遲＝服務體驗」這個他一定懂的錯誤。可以這樣講：「『LMArena 第一』就像壓測報告上那個 80 毫秒的平均延遲——它是一個被壓扁的數字，藏起了分布和場景。第一，LMArena 只量『人類偏好』這一個維度，不量 coding、不量數學，就像平均延遲不告訴你 p99；第二，它是抽樣（一群科技宅、一批 prompt 投出來的），換群人換批題，名次就會動，就像換個流量場景數字就變；第三，它的 Elo 帶誤差棒，第 1 和第 3 的信賴區間可能重疊，差別在統計上可能不存在。所以正確的讀法是『截至這週，它在人類偏好這一項領先』，不是『它最強』。」常見誤解：以為「綜合排行榜」就綜合掉了這些問題——不，把不同維度硬塞進一個名次，只是把問題藏得更深。

### 推演題2 — 設計一個「看起來公平、其實偏袒某家」的評測 [20 分鐘] ★★★

請你當一個不老實的評測機構，設計一份「看起來中立客觀、其實系統性偏袒 A 家模型」的 benchmark。你會在哪幾個環節動手腳？這些偏袒藏在哪裡、為什麼難被一眼看穿？

#### 推演解答

偏袒可以藏在評測 pipeline 的每一個環節，而且每一個都能用「我們只是做了個合理選擇」來辯護——這正是它陰險的地方：

- **選題（資料）**：題目類型偏向 A 家的強項。A 家擅長 coding 弱在多語言？那就讓題庫裡 70% 是程式題、刻意少放非英文題。對外只說「我們聚焦實用任務」。
- **出題來源（污染）**：選用一批 A 家**可能在訓練資料裡見過**的公開題庫，而對手家沒見過。表面上「大家考同一份卷」很公平，實則 A 家在「開卷」。
- **評分標準（後訓練偏好）**：如果用「人類評審打分」，就找偏好 A 家輸出風格（比如愛長篇、愛 markdown、愛先講結論）的評審群；或者乾脆用 A 家自己的模型當「裁判」（LLM-as-judge）——模型對自家風格的輸出有系統性偏好，這是已知效應。
- **prompt 格式**：用 A 家模型偏好的提示格式、system prompt、few-shot 範例去問所有模型。同一道題，換個問法，不同模型表現可以差很多——你只要把「標準問法」定成 A 家的舒適區。
- **報告呈現（小樣本＋砍誤差棒）**：樣本數壓小到剛好讓 A 家領先、又還沒小到啟人疑竇；報告裡只印名次和分數，把信賴區間拿掉，讓「A 領先 1.5 分」看起來是鐵的事實，而非噪音。

難被看穿，是因為**每一步都是「合理範圍內的選擇」**，沒有一步是公然造假——偏袒是這些選擇**累積**出來的系統性傾斜。這也回頭證明了為什麼要三角驗證：上面這套花招能騙過「這一份」靜態學科考，但你拿 A 沒佔便宜的人類偏好擂台和 agentic 實戰一交叉，傾斜就露餡了。常見誤解：以為「公開題目、公開流程」就等於公平——透明只防得了赤裸的造假，防不了藏在一連串合理選擇裡的結構性偏袒。

### 推演題3 — 找破綻：「這個模型在 GPQA 拿 94 分，已經接近博士水準」 [10 分鐘] ★★

這句話（常見於模型發表新聞）至少有兩處可以開刀。指出來。

#### 推演解答

- **破綻一（飽和）**：GPQA Diamond 截至 2026-06 已逼近飽和——前沿模型全擠在 ~94%。所以「94 分」不代表這家特別強，代表這把尺對前沿模型已經量不出差異了。在一份大家都考 94 的卷子上考 94，是「跟大家一樣」，不是「鶴立雞群」。新聞把「飽和」誤讀成「登頂」。
- **破綻二（污染／背誦≠理解）**：94 分量的是「在這份固定題庫上答對的比例」，不直接等於「博士水準的科學推理能力」。如果題目有洩漏，分數摻了背誦；就算沒洩漏，「會在多選題裡選對答案」和「能像博士那樣提出假設、設計實驗、面對開放問題」是不同的能力——benchmark 量的是前者，新聞偷換成了後者。
- **加分破綻（單一數字）**：一個 GPQA 分數只是一個維度。要判斷「接近博士水準」這種大話，至少得三角驗證——它在需要真實多步推理的 agentic 任務上、在真人盲評上是否一致地強？單尺定生死，本身就是錯的。

常見誤解：把「在某個固定考試上的高分」直接等同於「具備該考試想衡量的真實能力」——這正是 Goodhart 定律（當一個指標變成目標，它就不再是好指標）在 AI 評測上的現身。

## 自我檢核

1. 為什麼「哪個模型最強」這個提問本身是壞掉的？用「哪個資料庫最快」來類比，講清楚問題出在哪。
2. benchmark 污染（contamination）的因果鏈是什麼——為什麼「題目公開」加上「pretraining 吞網路」會讓分數量到背誦而非推理？這跟「拿訓練集當測試集」哪裡像？
3. 飽和（saturation）和污染是兩回事。如果不分清楚，你會怎麼誤讀一個 92 分？兩者各自讓那個數字怎麼騙你？
4. 三種評測類型（靜態學科考／人類偏好擂台／agentic 實戰）各量什麼、各自的弱點是什麼？為什麼「弱點不一樣」正是三角驗證有效的原因？
5. 為什麼三角驗證**不是**萬靈丹？至少講出一種它也治不了的情況。
6. 從機率的角度：LMArena 的一個名次，為什麼說它「本質是抽樣」？抽樣偏誤和小樣本（信賴區間重疊）各自怎麼讓「第 3 vs 第 7」變得不可信？
7. GSM1k 個案裡，那「最多 13 個百分點」的掉分量到的是什麼？為什麼 GPT／Claude／Gemini／Llama 幾乎沒掉、而 Mistral／Phi 掉很多——這說明了什麼，又不能過度推論成什麼？
8. 如果排名每幾個月就洗牌，本書憑什麼還說「各家的差異會持續」？穩定的到底是什麼（提示：不是名次）？

## 延伸閱讀

- **Scale AI, "A Careful Examination of LLM Performance on Grade School Arithmetic"（GSM1k 論文，arXiv:2405.00332, 2024）** — 本章招牌個案的原始出處。讀「new benchmark」與「overfitting」那兩節，看他們怎麼用一份全新題集把污染從猜測變成數字（最多 13% 的掉分、Spearman 相關）。這是你日後檢驗任何高分的範本。
- **"A Survey on Data Contamination for Large Language Models"（arXiv:2502.14425, 2025）** — 想把污染這件事看全面，這篇綜述把各種污染型態與偵測方法整理成一張地圖。挑「detection methods」一節讀，建立「換新題重測」之外的其他驗污手段直覺。
- **LMArena（arena.ai，前 Chatbot Arena / LMSYS）** — 親手去看那個 live 排行榜，特別注意每個 Elo 旁邊的 **± 信賴區間**，感受一下「重疊的名次差別在統計上不存在」是什麼意思。本章機率那節的活教材。
- **《馴服隨機：機率與統計》（你書架上）** — 抽樣偏誤、信賴區間、為什麼小樣本排名不可靠，那本把這幾個概念講到底。本章只喚醒、不重講；要追根究柢回那本的抽樣與信賴區間章節。
- **landscape §F（本書內部基準，截至 2026-06）** — 各 benchmark 當前的飽和狀態與「三類型三角驗證」最佳實務的原始整理。日後 benchmark 名次過期時，先回這裡對基準。
