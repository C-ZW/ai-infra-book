# ch12 — Alibaba Qwen：開放權重廣度與多語言縱深

> **本章解決什麼問題**：上一章拆完 DeepSeek 用效率創新撼動「訓練很貴」的成本敘事。這一章換 Alibaba（阿里巴巴）的 Qwen，回答五變數記分卡裡「**產品取向**」與「**資料**」這兩格如何共同主導一家的性格——為什麼「**全尺寸開放權重**」（從邊緣裝置到資料中心都有對應大小）、「**多語言縱深**」與「Hugging Face 上的高採用度」會長在 Qwen 身上。本章的招牌因果鏈只有一條，但會講到底：**阿里雲要做生態，於是把「每個裝置層級都出一款模型」當成分發策略，加上深厚的中文／亞洲語言資料，催出了 Hugging Face 上的採用飛輪**。開源 vs 閉源的哲學全貌留給 ch13，多模態留給 ch17；本章只負責 Qwen 這一家的成因，不越界。

## 從你已知的出發

回想你那套**自建的多層 cache library**——全團隊採用的那一套。它能變成事實標準，不是因為它在某個 microbenchmark 上跑分最快，而是因為它**覆蓋了大家所有的使用場景**：要極輕量、只想包一層記憶體快取的，有對應的用法；要中間層、帶 Redis 後援的，也有；要企業級、帶失效策略與多區一致性的，還是同一套 API。新同事不必先學一堆新東西、也不必在「我這個小服務該不該引入這麼重的東西」之間掙扎——**從最小到最大，每個部署點都有他剛好需要的那一號**。一旦這件事成立，採用率自己滾起來：越多人用，越多人在它上面長出工具、寫文件、踩坑回報，它就越難被替換。**覆蓋全部署點本身就是護城河**，這和「單點最快」是兩回事。

Qwen 對「模型尺寸」做的，幾乎就是你對 cache library 做的同一件事，只是把場景換成「模型要在哪裡跑」。當很多前沿實驗室只關心「我的旗艦能不能在某個 benchmark 上贏」時，Qwen 反過來問：**從一支手機、一台筆電，到一整櫃 GPU，每個算力層級的開發者，能不能都拿到一款『剛好這個大小、權重可下載、授權可商用』的 Qwen？** 本章要說的就是：Qwen 之所以在開放權重生態裡份量這麼重，主因不在它某一款旗艦贏了誰，而在它**鋪滿了整條尺寸階梯**——再加上多語言這個別家較難複製的資料縱深。

## 以什麼著稱：全尺寸開放權重、多語言、亞洲市場縱深（截至 2026-06）

先講「是什麼」，標好日期戳，再回頭問「為什麼」。

截至 2026-06，Qwen（阿里雲的大模型系列）以三件事著稱：**開放權重的廣度**（尺寸種類比幾乎任何一家都齊全）、**多語言能力**，以及**中文／亞洲市場的貼合度**；而且它是 **Hugging Face 上下載量最高的模型家族之一**。一個一個看，並把「已出貨的耐久事實」和「每月在動的點版本」分清楚——這是 Qwen 這家最該小心的地方。

- ✅ **Qwen3**（2025 年）是一條完整的尺寸階梯，這是本章最該記住的耐久事實：
  - **邊緣級**（手機／嵌入式）：0.6B、1.7B
  - **筆電級**（本機跑得動）：4B、8B
  - **資料中心級**：32B、235B、480B
  - 其中含 **MoE（Mixture-of-Experts，稀疏專家）** 變體，招牌是 **Qwen3-235B-A22B**——總參數 235B、但每個 token 只啟動約 22B（A22B 就是「active 22B」的意思），還有更小的 Qwen3-30B-A3B。多數採 **Apache 2.0** 授權。
  - 它支援 **119 種語言與方言**——這是 Qwen3 的招牌數字之一。
- 🟡 點版本一路往前滾：**Qwen3.5**（2026-02-16，含一個 397B-A17B 的 MoE）、**Qwen3.6-Plus**（2026-04-02，上 Model Studio）、**Qwen3.6-35B-A3B**（2026-04-17，在 Hugging Face／ModelScope 上開放權重，Apache 2.0）。
- 🟡 採用度：截至 2026-01 的整理，Qwen 在 Hugging Face 上累積**超過 7 億次下載**、衍生出**十多萬款（一說超過 20 萬款）衍生模型**，是平台上下載量最高的開放模型家族；甚至有來源指出 2025-12 單月下載量，超過其後八個熱門家族的總和。⚠️ 這類彙整數字每月在變、口徑不一，當成「採用度極高」的量級佐證讀，別把任一個精確值當定論。

> ⚠️ 點版本名（Qwen3.5 / 3.6 / 3.7-Max⋯⋯）每月都在洗，這裡**不押點版本號**。穩定可講的事實是三件：「**尺寸的廣度**」「**開放權重（多為 Apache 2.0）**」「**多語言（119 種）**」（截至 2026-06）。這和 ch05 對 OpenAI 點版本、ch06 對 Gemini 點版本的處理一致——記結構，不記正在動的數字。

注意這裡的措辭分寸。「下載量最高之一」「尺寸最齊全之一」是**可數的分發事實**，能講；但「Qwen 是最強的開放模型」這種絕對排名不能講——名次會洗牌、會被便宜化的對手追上（見 ch04）。我們談的是**廣度與分發**，不是某個 benchmark 上的名次。

## 招牌一：一條鋪滿的尺寸階梯（從手機到資料中心）

為什麼「尺寸齊全」值得當招牌、而不是一句「他們模型很多」帶過？因為它解決的是一個你天天在面對、外行卻看不見的問題：**部署點不是一個，是一條連續譜**。

你維運過的系統就是這樣：同一個邏輯，在邊緣（CDN/裝置端）、在中型服務、在重型批次任務裡，對資源的容忍度天差地遠。一個只有 512MB 記憶體的 sidecar 容器，和一台 8 卡 GPU 的訓練機，需要的「同一件事」是完全不同的兩號。模型也一樣——而且因為模型的記憶體佔用主要由參數量決定（你那本 AI Infra 書把這條算過：權重要常駐 GPU 記憶體，KV cache 隨上下文長度再疊上去），**「我這個場景跑得動多大的模型」是一道硬約束，不是一道偏好題**。

Qwen 的策略，就是把這條約束**一格一格填滿**。把它畫成你熟的部署譜：

```text
部署點            典型約束              Qwen3 對應一號（截至 2026-06）
─────────────────────────────────────────────────────────────
手機／嵌入式      記憶體 KB~GB 級       0.6B / 1.7B（邊緣級）
筆電／單機        本機 CPU/小 GPU       4B / 8B（筆電級）
單卡～多卡伺服器  一張到數張 GPU        32B（dense）
資料中心／高品質  多卡、要省啟動算力    235B-A22B（MoE，總大但每 token 只啟動約 22B）/ 480B
```

這張表就是本章第一個被講透的因果個案，我把它叫做「**全尺寸開放 → 採用飛輪**」：

```text
每個部署點都出一款（邊緣→筆電→資料中心，權重可下載、Apache 2.0 可商用）
   │  開發者「拿來即用」的摩擦趨近於零（不必先評估該不該引入）
   ▼
任何算力預算的人都能上車 → 海量 fine-tune、衍生模型、教學、工具長出來
   │  Hugging Face 上 7 億+ 下載、十多萬款衍生模型（截至 2026-01，🟡）
   ▼
生態變厚：教程、量化版、推論最佳化、整合都先支援 Qwen
   │  「預設就試 Qwen」變成開發者的肌肉記憶（先發者地位自我強化）
   ▼
採用度本身變護城河：不是某款最強，是「整條階梯都在這裡、生態都圍著它」
```

你對這個飛輪的形狀應該有強烈既視感——它和你那套 cache library 變成全團隊標準的路徑同構，只是規模放大到全球開發者。**護城河不是「最強的那一款」，是「從最小到最大、權重能下載、而且大家都在它上面長東西」**。這正是五變數第五格「產品取向」最直白的演示：阿里雲不是先養出一款模型再去找使用者，而是**用「每個層級都出一款開放權重」這個分發策略，把開發者生態當作要養成的東西**——模型矩陣是被這個生態目標反過來捏出形狀的。

值得補一句 MoE 在這裡的角色，因為它接得回你已有的直覺。資料中心級那一號 235B-A22B 是 MoE：總參數很大（容量大、品質有本錢），但每個 token 只「點亮」其中約 22B 的專家子網路（推論時的啟動算力小）。在你的語言裡，這像極了你做過的**冷熱分層**或**按需路由**——資料全在那裡（容量），但每次請求只打到你真正需要的那一小塊（成本）。MoE 讓 Qwen 能在「階梯頂端」同時要到高容量與可控的服務成本，而不必逼開發者吞下一個每個 token 都要算滿 235B 的怪物（MoE 的跨家全貌與機制細節是 ch16 的事，這裡只記它在「全尺寸」這條招牌裡扮演什麼角色）。

## 招牌二：多語言縱深（119 種語言不是順手的副產品）

第二個招牌是**多語言**。截至 2026-06，Qwen3 宣稱支援 **119 種語言與方言**，涵蓋中文、英文、歐語，到低資源語言。中文／亞洲語言的貼合度尤其是它的口碑強項。

要把這格講透，得先說清楚**多語言為什麼不是「順手就有」的東西**，否則它聽起來只像一個行銷數字。一個模型「見過什麼世界」，幾乎完全由它的**預訓練資料配比**決定（這是五變數第一格「資料」的本體）。如果一家實驗室的語料絕大多數是英文網路文本，那它的中文、阿拉伯文、東南亞語言能力就是「能講但不深」——不是它不想強，是它**沒在那個世界裡長大**。多語言縱深要花真金白銀：你得有那些語言的高品質語料、得在配比裡認真地分給它們權重、還得在後訓練時用那些語言的資料把指令遵循與性格也調出來。這是一條結構性的成本，不是一個開關。

Qwen 的結構性優勢恰好落在這裡。它背靠阿里巴巴——一家**業務根基在中文與亞洲市場**的公司。把這條因果鏈攤開：

```text
阿里巴巴的業務重心在中文／亞洲市場（電商、雲、企業服務）
   │  天然地接觸到大量中文與亞洲語言的真實語料與使用場景
   ▼
預訓練配比裡，中文／亞洲語言拿到的權重遠高於「以英文為主」的實驗室
   │  資料這格的配比，決定了模型「在哪個世界裡長大」
   ▼
中文／亞洲語言的能力是「長出來的縱深」，不是事後翻譯接上去的薄層
   │  再把多語言鋪到 119 種，覆蓋低資源語言
   ▼
亞洲市場與多語言場景的貼合度，成為別家較難快速複製的差異化
```

這條鏈的關鍵字是「**較難快速複製**」。架構（藍圖）誰都抄得到、訓練法也會擴散，但**「你在哪個語言世界裡長大」是資料配比決定的，而資料配比根植於你是誰、你服務誰**——一家以英文消費市場為重心的實驗室，不會無緣無故把語料的大頭分給東南亞語言。所以多語言這格，是 Qwen 一個帶有「身世」性質的優勢：它源自阿里巴巴的市場位置（產品取向），落實在預訓練配比（資料），兩格扣在一起才成立。

## 為什麼是 Qwen？用五變數層層回答

招牌講完了，現在回答「為什麼是這家」。一個一個變數過。

**產品取向（第五格，主因）**——這是 Qwen 最強的一格，也是整章因果的源頭。背後是**阿里雲的商業誘因**：它要做的是雲生意，而開放權重是一套**分發策略**——把「每個裝置層級都出一款」的模型矩陣免費放出去（多為 Apache 2.0、可商用），讓開發者生態圍著 Qwen 長起來，再用代管 API（**Model Studio**）與雲端算力把這份採用度變現。注意這和 ch08 要拆的 Meta 邏輯有微妙差別，也和 ch09 的 Mistral 不同（這裡只記一筆，避免越界）：Qwen 的開放，從一開始就**和一個要賣算力的雲深度綁定**——開放權重不是理念的終點，是把開發者導進阿里雲生態的入口。「全尺寸開放」之所以是 Qwen 先做到這個齊全度，不是別家技術做不到，是**別家當時沒有『用一條完整尺寸階梯去鋪生態』這種雲廠商式的產品取向**。

**資料（第一格，主因）**——多語言縱深就是這格在說話。背靠阿里巴巴的中文／亞洲業務，Qwen 的預訓練配比天然地給了中文與亞洲語言更高的權重，再鋪到 119 種語言。這是「見過什麼世界」的差異，根植於公司的市場位置，**別家較難快速複製**——它和產品取向（第五格）扣在一起：你服務誰，決定了你餵模型什麼。

**算力·基礎設施（第三格，次因）**——Qwen 背後是**阿里雲**的算力（具體規模未公開，⚠️）。要訓練一整條從 0.6B 到 480B 的階梯、還要每月迭代點版本，沒有一個自家雲撐著是不可能的。但算力在這裡是次因不是主因，因為它是「**能做到**」的必要條件，不是「**為什麼是這家想用全尺寸開放去鋪生態**」的原因——阿里雲讓 Qwen 練得起這條階梯，但不是阿里雲讓它**想到**要用開放權重鋪生態（那是第五格的判斷）。

**後訓練·對齊（第二格，次因）**——Qwen3 是 hybrid reasoning（混合推理），可以切換「思考／不思考」模式（reasoning 的跨家全貌見 ch14）。後訓練讓每一號模型都「聽話、好用、會推理」，多語言的指令遵循也靠這格調出來。但它是**執行**那條全尺寸開放策略的手段——讓階梯上每一號都堪用——不是做出策略的源頭，所以記次因。

**研究文化·品味（第四格，次因）**——「把尺寸鋪滿、快速迭代、緊跟開放權重社群」本身是一種研究文化取向：高頻發版、密切回應社群、願意把多種尺寸都好好訓而不是只顧旗艦。這份品味放大了前兩格的效果，但它服務於「鋪生態」這個產品目標，不是獨立的源頭。

## 成因拆解

### (a) 五格歸因

| 變數 | 角色 | 為什麼 |
|---|---|---|
| 資料 | ✓ **主因** | 背靠阿里巴巴中文／亞洲業務 → 預訓練配比給中文／亞洲語言更高權重 → 119 種語言的**多語言縱深**，別家較難快速複製。 |
| 後訓練·對齊 | △ 次因 | hybrid reasoning ＋ 把階梯上每一號都調得堪用、多語言指令遵循；是**執行**全尺寸開放的手段。 |
| 算力·基礎設施 | △ 次因 | 阿里雲撐起「訓一整條 0.6B→480B 階梯＋每月迭代」（規模未公開，⚠️）；是「能做到」的必要條件，非「為什麼先想到」。 |
| 研究文化·品味 | △ 次因 | 高頻發版、緊跟開放權重社群、願意把多尺寸都好好訓；放大前面幾格，但服務於鋪生態的產品目標。 |
| 產品取向 | ✓ **主因** | 阿里雲的商業誘因：開放權重是**分發策略**——每個裝置層級都出一款（多 Apache 2.0）鋪生態，再用 Model Studio API ＋雲算力變現，催出 Hugging Face 採用飛輪。 |

一句話收束：**Qwen 強在「全尺寸開放權重的廣度」與「多語言縱深」，因為產品取向（阿里雲用全尺寸開放當分發策略鋪生態）＋資料（中文／亞洲語言配比的縱深）；算力與後訓練是把這兩格落實的次因。影子是——地緣信任門檻、與最頂尖閉源前沿的差距、以及中國實驗室共通的評測可信度爭議。**

### (b) 影子

同一組讓它強的選擇，也讓它暴露在對應的弱點上：

- **地緣政治與信任門檻**。Qwen 來自中國實驗室這個「身世」，本身就是一道結構性的採用門檻——在某些政府、受監管產業、對資料主權敏感的企業眼裡，「中國背景的模型」即使權重完全開放、可離線部署，仍會被列入採購限制或合規審查。諷刺的是，**開放權重原本是用來降低採用摩擦的，但對這群使用者，provenance（來自誰）的摩擦蓋過了 openness 的便利**。這正是 ch09 歐洲「主權 AI」訴求的鏡像——對某些買家，「這模型是誰養的」比「它多開放、多便宜」更先決。
- **與最頂尖閉源前沿的差距**。全尺寸開放鋪的是**廣度**，但廣度不等於拿下「最頂尖那一格」。截至 2026-06，最受矚目的閉源前沿（如 ch05 的 OpenAI、ch06 的 Google）在某些高難任務上仍被視為第一梯隊，而 Qwen 的招牌是「**夠好、全尺寸、開放、便宜**」，不是「在最難的 benchmark 上絕對領先」。把廣度誤讀成「樣樣都頂尖」，就踩到一個常見的陷阱——廣度與巔峰是兩個不同的軸。
- **評測可信度爭議**。中國實驗室（不只 Qwen）普遍面對一種外部質疑：公布的 benchmark 分數是否有資料污染（contamination）、是否針對熱門榜單過擬合。這裡的立場不是替誰背書或定罪，而是套用 ch04 的方法——**任何家、任何榜單的數字都要分任務看、看有沒有飽和與污染**；對處於地緣張力中的實驗室，這種審視只會更嚴。對它是不公平的額外稅，但這道稅真實存在。
- **開放權重的商業模式張力**。把旗艦級的權重免費放出去，意味著**競爭對手、甚至下游雲廠商都能拿去自架、不付你一毛錢**（這個張力 ch08 的 Meta、ch11 的 DeepSeek 也都有，是開放權重路線的共通影子）。Qwen 的解法是綁阿里雲（Model Studio 代管變現），但這條變現鏈能不能長期撐住「免費放出最好的權重」的成本，是個未定的問題（ch13 把開放 vs 閉源的商業模式攤開拆）。

### (c) 何時不成立

這個「強項」在以下情況會被打臉，提醒你這是**截至 2026-06 的快照、不是定律**：

- **換買家／換場域**：「開放權重 ＋ 全尺寸 ＋ 便宜」對獨立開發者、新創、研究者是巨大的吸引力；但對受地緣與合規約束的政府與大型企業，provenance 門檻可能讓這些優點**完全失效**——同一組強項，在不同買家眼裡的價值天差地遠。
- **換軸看強弱**：「尺寸最齊全之一、下載量最高之一」是分發事實；但**換到「某個最難任務上誰絕對最強」就不保證領先**。截至 2026-06，coding/agentic 口碑常掛 Anthropic、超長上下文掛 Google、最大消費分發掛 OpenAI——「誰最強」要分任務，沒有單一冠軍（見 ch04）。
- **被便宜化／被趨同推翻**：全尺寸開放這套打法，別家也在學（開放權重的廣度本身就是易擴散的優勢，ch20 會講「哪些差異會收斂」）。今天 Qwen 的尺寸齊全度是領先群之一，但這條護城河不像算力或專有資料那樣難填——點版本號、下載量排名、生態份額都每幾個月在動，本章所有數字（119 種語言、235B-A22B、7 億+ 下載）都是 2026-06 前後的切片，幾個月後就該重掃。

## 紙上推演

### 推演題1 [10 分鐘] ★

用你自己的話，對另一個工程師講清楚「全尺寸開放 → 採用飛輪」這條因果鏈：為什麼「從 0.6B 到 480B 每個尺寸都出一款、權重可下載」這件事，會變成 Qwen 在開放權重生態裡的護城河？護城河到底是哪個東西？

#### 推演解答

關鍵是**把它和「最強的那一款」分開**。Qwen 的優勢不是某一號在某個 benchmark 上贏了誰（那種領先會洗牌），而是它**鋪滿了整條部署譜**：手機級（0.6B/1.7B）、筆電級（4B/8B）、資料中心級（32B/235B-A22B/480B），權重可下載、多為 Apache 2.0 可商用。於是任何算力預算的開發者都能「拿來即用」、摩擦趨近於零，海量 fine-tune、衍生模型、教程、量化版、推論最佳化都先圍著 Qwen 長（截至 2026-01 的整理：Hugging Face 上 7 億+ 下載、十多萬款衍生模型，🟡）。生態越厚，「預設就試 Qwen」越變成肌肉記憶。**護城河不是「最強的一款」，是「整條階梯都在、權重能下載、而且生態都圍著它長」**——這對應五變數的產品取向（主因）。常見誤解是把 Qwen 的份量讀成「它的旗艦最強」；錯，穩定的優勢是**廣度 ＋ 開放 ＋ 由此滾出的生態飛輪**。對帳你自己：你那套 cache library 能成為全團隊標準，也不是因為某個 microbenchmark 最快，是因為它覆蓋了所有部署點、大家都在它上面長東西——同一個飛輪。

### 推演題2 [12 分鐘] ★★（翻譯）

把「Qwen 的中文／多語言特別強」這個現象，翻譯回五變數記分卡——主因落在哪一格？再說清楚：為什麼這格的優勢「**別家較難快速複製**」，而像「reasoning 模式」這種東西卻很快就被各家抄齊了？

#### 推演解答

主因落在**第一格（資料）**，而且和第五格（產品取向）扣在一起。一個模型「見過什麼世界」由預訓練的**資料配比**決定；Qwen 背靠阿里巴巴的中文／亞洲業務，天然接觸到大量該語言世界的真實語料，於是配比裡中文／亞洲語言拿到的權重，遠高於一家以英文消費市場為重心的實驗室——這份縱深是「長出來的」，不是事後翻譯接一層。為什麼難複製？因為**資料配比根植於「你是誰、你服務誰」**：一家英文市場為重心的公司，不會無緣無故把語料大頭分給東南亞語言；要追上，對手得真的去取得那些語言的高品質語料、重做配比、重做多語言後訓練——這是結構性成本，不是一個開關。對照之下，「reasoning 模式」（讓模型推論時多想一段）是一種**方法**，方法會擴散：OpenAI 開了頭（o1，2024-09），DeepSeek 證明可便宜複現（R1，2025-01），一年內變成桌面門檻（ch14）。常見誤解是把「資料優勢」和「技巧優勢」混為一談——技巧（架構、訓練法）會被抄齊並收斂，**資料配比這種帶『身世』的優勢卻黏得多**（ch20 會把「哪些差異持續、哪些收斂」講成一條主線）。

### 推演題3 [15 分鐘] ★★★（找破綻）

有人說：「Qwen 是 Hugging Face 上下載量最高的開放模型家族之一，又支援 119 種語言，所以它就是最強的開放模型。」請指出這句話裡的**三個**破綻。

#### 推演解答

**破綻一：「下載量最高」是分發事實，不是能力排名。** 下載量、衍生模型數（截至 2026-01：7 億+ 下載、十多萬款衍生，🟡）量的是**採用度與生態厚度**，這是可數的；但「最強」是**能力排名**，會洗牌、會被便宜化的對手追上。把「最多人用」偷換成「最強」，等於把分發指標誤當成品質指標——這正是 ch04 一再拆的混淆（LMArena 量人類偏好、SWE-bench 量 agentic coding，下載量哪個都不量）。**破綻二：「廣度」不等於「巔峰」。** 全尺寸開放鋪的是覆蓋面，但截至 2026-06，最頂尖的某些高難任務口碑仍在閉源前沿（ch05/ch06）；Qwen 的招牌是「夠好、全尺寸、開放、便宜」，不是「最難 benchmark 上絕對領先」——廣度與巔峰是兩個軸，樣樣都有不代表樣樣都頂。**破綻三：對中國實驗室的數字，要多一層污染與過擬合的審視。** benchmark 分數可能受資料污染（contamination）或針對熱門榜過擬合影響（這是對整類實驗室的共通質疑，不是定罪某一家）；而且「119 種語言」是**支援廣度**的宣稱，不等於每種語言都同等深——低資源語言的實際品質得分語言看。正確講法是：「截至 2026-06，Qwen 以**開放權重的廣度**與**多語言（119 種）**著稱、是 Hugging Face 上採用度最高的家族之一，但這是分發與覆蓋面的事實，不是某個任務上的絕對能力排名，且各家、各榜的分數都要分任務看、看飽和與污染。」

## 自我檢核

講得出來才算過關；講不清楚就是還沒懂。

1. **為什麼**「Qwen 下載量最高之一」是個能數人頭的分發事實，而「Qwen 是最強的開放模型」不是？兩者差在哪？（提示：採用度 vs 能力排名）
2. 「全尺寸開放 → 採用飛輪」這條鏈，**如果**Qwen 只出一款旗艦、不鋪滿從 0.6B 到 480B 的階梯，會斷在哪一環？哪一格優勢會連帶縮水？
3. 為什麼說 Qwen 的多語言縱深「**別家較難快速複製**」，而 reasoning 模式卻很快被各家抄齊？把這個對比掛回五變數（資料 vs 方法）講一遍。
4. 為什麼把「開放權重」說成 Qwen 的**分發策略**而不只是理念？阿里雲在這條策略裡靠什麼變現？
5. 在 Qwen 的五格歸因裡，為什麼**阿里雲算力只是次因不是主因**？（提示：必要條件 vs 「為什麼是這家先想到用全尺寸開放鋪生態」）
6. Qwen 強項的**影子**是什麼？為什麼「開放權重原本要降低採用摩擦」，對某些買家反而**provenance 的摩擦蓋過 openness 的便利**？舉一類這樣的買家。
7. **如果不**用「截至 2026-06／以⋯⋯著稱／分任務看」的句式，而直接寫「Qwen 是最強的開放模型」，會出哪幾個問題？（提示：絕對排名、廣度≠巔峰、評測污染）
