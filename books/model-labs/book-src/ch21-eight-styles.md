# ch21 — 同一張藍圖，八種風格：五變數總帳與一套帶得走的判斷框架

> **本章解決什麼問題**：這是全書的總收官。從 ch01 那個反直覺的謎題（食譜是一篇 2017 年的公開論文，八家卻養出天差地遠的菜色）出發，走完八家拆解（ch05–12）與七個橫切面（ch13–19），現在把所有「強在 X，因為〔哪幾格〕，影子是 Y」結成一張**五變數總帳**。本章不引進任何新事實——全是回收，只標「見 chNN」。它要驗收全書四個主張，並把那把分析的尺交到你手上：**以後看到任何一款新模型發表，你能自己用五變數判斷它，而不是等別人告訴你誰最強。**

## 從你已知的出發

你一定讀過大系統的 postmortem（事故複盤）。一份好的 postmortem，重點從來不是「記住那天凌晨三點 RDS CPU 飆到幾趴、哪個 pod 第幾次重啟」——那些數字過一個月就沒人記得了。真正帶得走的，是**那套分析的骨架**：先看流量是不是異常、再看相依服務有沒有抖、再看自己的配置與容量留了多少餘裕、最後看當時的運維紀律有沒有破口。骨架穩了，下次換一個全新的事故，你照樣能在十分鐘內定位到「這次的鍋記在哪一格」。

這本書就是一份關於「八家大模型為什麼各擅勝場」的 postmortem，而這一章是它的**結語頁**。前面二十章給了你一堆數字——9 億 WAU、2M token 上下文、671B 總參數、560 萬美金、555,000 顆 GPU——這些全是 2026-06 的切片，三個月後一半會過期（landscape 的脆弱事實清單列得清清楚楚）。如果你讀完只記住這些數字，那這本書就白讀了，因為它們正是最不該背的部分。

你該帶走的，是那套**和你做瓶頸診斷一樣的排查骨架**：面對任何一款模型，逐格問「它的強項記在資料、後訓練·對齊、算力·基礎設施、研究文化·品味、產品取向這五格的哪幾格？影子又是什麼？」——這才是這本書真正的商品。這一章就來把這把尺磨利、交到你手上。

## 先把四個主張收齊

全書從 ch01 立下四個主張，每一章都在替它們累積證據。在攤開總帳之前，先一句一句驗收——你現在應該能對另一個工程師把每一句講清楚了。

**主張一：架構不是護城河。** Transformer 是 2017 年的公開論文（〈Attention Is All You Need〉），attention 怎麼算全世界都讀得到（ch01）。截至 2026-06，八家前沿模型本質上都是 decoder-only Transformer ＋ self-attention，零件清單高度重疊：RoPE ＋（GQA 或 MLA）＋（通常加）MoE（landscape §A）。你能在週末照著刻一個小版本的東西，按定義就不是別人領先的祕密。護城河沒有消失，它**搬了家**——搬到資料、算力、後訓練這些論文裡讀不到的格子。

**主張二：模型是「養」出來的，不是「寫」出來的。** 沒有人逐行規定「使用者問 X 就回 Y」（ch01–02）。各家做的是設計一個成長環境——餵什麼資料、用什麼目標最佳化、給多少算力——能力是從這個環境裡**長出來的產物**。這就是為什麼差異可以拆解：你不是在比較八份不同的程式碼，你是在比較八種不同的**養法**。

**主張三：五個變數解釋一切差異。** 養法可以拆成五格：**資料 × 後訓練·對齊 × 算力·基礎設施 × 研究文化·品味 × 產品取向**（ch03 定調）。這五格是全書的記分卡，每一章的 `## 成因拆解` 都拿它來歸因。它們**交互作用、少有單一主因**——往往是兩三格咬合（如 Google 的算力＋研究＋產品三格互相強化）。

**主張四：「最強」測不準，「風格」才穩定。** 沒有單一冠軍。排行榜每幾個月洗牌、benchmark 會被污染、頂端 benchmark 已經飽和到分不出前沿模型（ch04，landscape §F）。穩定可辨識的，是各家的**取向與性格**：OpenAI 的產品先行、Google 的長上下文、Anthropic 的對齊執念、DeepSeek 的效率逼出。能力會局部收斂（互相 distill、互相抄），但**成因結構是結構性的**，會持續（ch20）。

四句話收齊了。現在把八家的收束句結成一張帳。

## 五變數總帳：八家 × 五格

這張表是全書八個 `## 成因拆解` 的歸因總覽，每一格與 ch05–12 各章一致（✓＝主因、△＝次因、留白＝不是這家的招牌成因）。它不是排行榜——**沒有哪一行比哪一行高**，它只回答「每一家的強項記在哪幾格」。

```text
            資料   後訓練   算力     研究文化  產品
                   ·對齊   ·基礎設施 ·品味     取向
OpenAI       △     △       △        ✓         ✓      (ch05)
Google       △     ·       ✓        ✓         ✓      (ch06)
Anthropic    ·     ✓       △        ✓         △      (ch07)
Meta         △     ·       △        △鬆動     ✓★     (ch08)
Mistral      △     △       △        ✓         ✓      (ch09)
xAI          ✓     ·       ✓        △         △      (ch10)
DeepSeek     ·     ✓       ✓逼出    △         ·      (ch11,＋架構✓)
Qwen         ✓     △       △        △         ✓      (ch12)

✓主因  △次因  ·不是招牌成因
★ Meta 的產品取向是「同一格、輸入變了、輸出反轉」(見下)
DeepSeek 另把「架構」(MLA/MoE) 獨立記一筆主因，視為算力格的延伸 (ch11)
```

讀這張表有三個重點，每一個都呼應一個全書主張：

**第一，主因落點各不相同，這正是「風格穩定」的證據。** 把每一行的 ✓ 位置攤開看：OpenAI 落在「研究文化＋產品」，Google 落在「算力＋研究文化＋產品」三格咬合，Anthropic 落在「後訓練·對齊＋研究文化」，xAI 落在「算力＋資料」，DeepSeek 落在「算力（劣勢逼出）＋後訓練＋架構」，Qwen 落在「資料＋產品」。**沒有兩家的主因組合一樣**——這不是巧合，是各家結構性條件（在哪個國家、背靠哪個產品、受什麼管制、押什麼研究方向）不同的必然結果。風格之所以穩定，因為這些條件不會一夜改變。

**第二，「產品取向」這一格出現最多次，且是 Meta 最戲劇化的個案。** 八家裡有五家（OpenAI、Google、Meta、Mistral、Qwen）把產品取向記為主因。這呼應 ch03 的洞見：**模型被養成什麼形狀，往往是被它要服務的產品壓出來的。** 最極端的是 Meta（ch08）——同一格「產品取向」，輸入變了，輸出就反轉：不靠賣模型賺錢時，開放權重換生態話語權；要用 30 億用戶平台與專有資料變現時，就把前沿收回閉源（Muse Spark，2026-04，標為轉折非定局）。ch01 預告的「送出權重＝送出部分護城河」這個影子，在 2026 應驗了。

**第三，「研究文化·品味」是最難被抄走的一格。** 它在 OpenAI、Google、Anthropic、Mistral 都是主因或關鍵。為什麼難抄？因為資料可以爬、算力可以買、後訓練法會被論文擴散，但「先押 test-time compute 而不是先做別的」「把安全當第一性原則願意養一支短期不賺錢的可解釋性團隊」「把工程效率擰到極致」——這些是**判斷與取向**，藏在人才密度與價值觀裡，沒有 API 可以下載（ch20 說的結構性護城河，這格是核心）。

## 把八句收束話排成一列

總帳表是橫看，這裡縱看——把八家各自的「強在 X，因為〔哪幾格〕，影子是 Y」一句句排出來，這是全書收束句式的最終累積（措辭與各章一致）：

- **OpenAI**（ch05）：強在「定義產品形狀」與「開創 reasoning 範式」，因為**研究文化·品味＋產品取向**（算力與資料是放大的次因）；影子是先行者的包袱——範式一旦公開就被便宜化（DeepSeek R1 證明可便宜複現），且「最強但最貴且封閉」被開放權重從下方夾擊。
- **Google**（ch06）：強在長上下文與原生多模態，因為**算力·基礎設施＋研究文化·品味＋產品取向**三格咬合（自研 TPU 垂直整合 → 長序列成本可控 → 敢開別家不敢開的上下文，是被講透的因果鏈）；影子是太寬的戰線與消費者端「起個大早趕個晚集」的節奏。
- **Anthropic**（ch07）：強在 coding/agentic 可靠度與對齊·可解釋性研究底，因為**後訓練·對齊＋研究文化·品味**（Constitutional AI 的自我修正迴路外溢成 agentic 能力）；影子是安全取向的速度/開放代價，與相對對手的規模/算力天花板。
- **Meta**（ch08）：強在「締造了開放權重生態」，因為**產品取向**（用開放換生態與人才）＋算力；但 2026 把前沿收回閉源，同樣因為**產品取向**（改用 30 億用戶平台變現）；影子是送出權重＝送出部分護城河，這個影子應驗了。
- **Mistral**（ch09）：強在「以小搏大的效率」與「歐洲主權的開放權重」，因為**研究文化·品味＋產品取向**（主權定位壓出開放）；影子是規模天花板，與上下夾擊的生存壓力。
- **xAI**（ch10）：強在「後發者的加速度」與「即時時效性」，因為**算力·基礎設施＋資料**（自建 Colossus 極速堆叢集＋X 即時消防水管）；影子是蠻力的能源/成本/品質代價，與即時資料、內容治理的雜訊與爭議。
- **DeepSeek**（ch11）：強在「用一整疊效率發明把達標所需算力往下壓、還開放權重」，因為**算力·基礎設施的劣勢逼出＋架構（MLA/MoE）＋後訓練（GRPO）**三者咬合；影子是成本數字（560 萬美金）被過度簡化的傳播、出口管制的長期天花板。
- **Qwen**（ch12）：強在「全尺寸開放權重的廣度」與「多語言縱深」，因為**產品取向＋資料**（阿里雲用全尺寸開放當分發策略＋中文/亞洲語言配比的縱深）；影子是地緣信任門檻、與最頂尖閉源前沿的差距、評測可信度爭議。

把這八句連起來讀一遍，你會發現——**沒有一句話在說「誰最強」**，每一句都在說「強在哪個面向、記在哪幾格、代價是什麼」。這就是全書要訓練的閱讀方式：把「排名問題」換成「歸因問題」。

## 帶得走的工具：看到新模型發表，怎麼用五變數自己判斷

這是這本書留給你的真正商品——一套你以後拿著就能用的**判斷協定**。下次某家發表新模型、科技媒體一片「史上最強」的標題時，先別急著信，跑這套流程。它的形狀跟你做瓶頸診斷時那張排查清單一模一樣：不是憑感覺，是逐項過。

```text
新模型發表 → 五變數判斷協定 (跑一遍約 15 分鐘)
  │
  ├─[0] 先剝掉行銷層 ─────────────────────────────────
  │     問：這是「以…著稱」還是「就是最強」?
  │     凡是無時效的絕對排名 → 自動打折 (ch04)。
  │     發表方自報的 benchmark 數字 → 標「自報、待第三方複現」。
  │
  ├─[1] 招牌強項是什麼? ──────────────────────────────
  │     先用一句話寫下「它以做好『什麼任務』著稱」
  │     (coding? 長上下文? 多語言? 低成本? 即時時效?)
  │     —— 不是「它很強」,是「它強在哪個軸」。
  │
  ├─[2] 逐格歸因: 這個強項記在哪幾格? ────────────────
  │     資料      —— 它見過什麼別家沒見過的世界? (專有語料/多語言)
  │     後訓練·對齊 —— 有沒有招牌訓練法? (新 RL? 新對齊?)
  │     算力·基礎設施 —— 自研晶片/自建叢集/被管制逼出效率?
  │     研究文化·品味 —— 押了什麼別家沒先押的方向?
  │     產品取向   —— 它要服務誰? 這個強項是不是被產品壓出來的?
  │     → 標 ✓主因 / △次因 / 留白,通常 1–3 格是主因。
  │
  ├─[3] 影子是什麼? (這格必填,不准只寫優點) ──────────
  │     同一組讓它強的選擇,讓它弱在/暴露在哪? (ch 各家(b))
  │     例: 開放權重 → 商業模式問題; 蠻力算力 → 能源/成本代價。
  │
  └─[4] 標日期 + 何時不成立 ──────────────────────────
        寫下「截至 YYYY-MM」,並問: 換什麼任務/被哪家/過多久
        這個判斷會被推翻? (ch 各家(c)) 記住它是快照不是定律。
```

這套協定的精神，全在 ch04 那句「**換任務軸**」——「強」要分任務（coding ≠ 數學 ≠ 長文 ≠ 多語言），任何把不同軸壓成一個數字的排名都該打折。最佳實踐是三角驗證：一個靜態學術評測（MMLU-Pro/GPQA）＋一個人類偏好競技場（LMArena）＋一個 agentic 套件（SWE-bench），**三者一致才是真訊號**（landscape §F）。一個只在自報 benchmark 上漂亮的新模型，跟你那份「平均延遲漂亮但 p99 爆掉」的 k6 壓測報告是同一回事。

## 成因拆解

本章的「現象」是**全書本身**——八家為什麼各擅勝場。所以這個招牌段落，就是上面那張五變數總帳表的元層次反思，外加全書的影子與失效條件。

### (a) 五格歸因（全書層次）

把五格當成「差異成因」本身來歸因，問「整個產業的差異，主要由哪幾格驅動」：

| 變數 | 全書層次的角色 | 為什麼 |
|---|---|---|
| 資料 | ✓ 主因 | 各家最不公開的祕密，也是 xAI（X 活水）、Qwen（中文縱深）的招牌成因；資料牆與版權訴訟正逼產業轉向合成/可驗證資料（ch18）。 |
| 後訓練·對齊 | ✓ 主因 | 對齊方法的演化（RLHF→Constitutional AI→RLVR/GRPO，ch15）是 Anthropic、DeepSeek 招牌的根；同一預訓練底靠它養出不同性格。 |
| 算力·基礎設施 | ✓ 主因 | 「能不能養出某類模型」的硬約束（ch16）；Google（TPU）、xAI（Colossus）、DeepSeek（被管制逼出效率）的招牌都落在這格。 |
| 研究文化·品味 | ✓ 主因 | 最難被抄走、最結構性的一格（ch20）；決定誰先押 reasoning、誰先押對齊、誰先押效率。 |
| 產品取向 | ✓ 主因 | 八家裡五家的招牌主因；模型被養成什麼形狀，常被它要服務的產品壓出來——Meta 的反轉是最戲劇化的證明（ch08/ch13）。 |

全書層次的結論：**五格全是主因，沒有一格可以省。** 這正是 ch03 的核心——差異是多格交互作用的產物，少有單一主因。任何「某家強只因為他資料好/算力多」的單因解釋，都是過度簡化。

### (b) 影子

這本書本身、以及它教你的這套框架，也有影子，不能照單全收：

- **快照的影子**。全書所有版本號、排名、價格、GPU/TPU 數量都是 **2026-06-13 的切片**。landscape 列了 12 條最脆弱的事實（旗艦版本、各家算力數字、DeepSeek 成本、Colossus 規模…），明說「~2026-09 會過期」。這本書的**框架**（五變數、養 vs 寫、影子必寫）是耐久的；它的**數字**是會爛的。把兩者分清楚，否則你會拿一張過期的帳去判斷新局。
- **歸因的影子**。五變數是一把好用的尺，但它是**人為的切法**，不是物理定律。真實成因常常糊在格與格之間（DeepSeek 的「架構」就跨在算力格邊上，得另記一筆）。框架的價值在「逼你逐格問、逼你寫影子」，不在「每個現象都有唯一正確的歸格」。把它當排查清單用，別當標準答案背。
- **不可證偽的影子**。「研究文化·品味」這格最難被抄走，但也最難被驗證——它幾乎無法 A/B 測試，容易變成「凡是解釋不了的就推給文化」的萬能託辭。用它時要特別誠實：能指出具體押注（如「先押 test-time compute，2024-09 的 o1」）才算數，泛稱「他們文化好」就是偷懶。

### (c) 何時不成立

這本書的核心主張，在什麼情況下會被打臉？

- **若非 Transformer 範式勝出**：主張一（架構不是護城河）的前提是「八家共用 decoder-only Transformer 藍圖」。若哪天 Mamba/狀態空間模型這類挑戰者真的在前沿勝出（截至 2026-06 仍未撼動），「同一張藍圖」就得改寫——這是 **2026-06 的快照，不是定律**（ch01 已標）。
- **若差異全面收斂**：主張四（風格穩定）賭的是成因結構是結構性的。但 ch20 也承認趨同壓力真實存在（互相 distill、reasoning 已成桌面門檻而非差異化）。若哪天算力、資料、研究品味都被某種方式抹平，「八種風格」可能塌成「幾種風格」。框架仍能用，但帳本會更短。
- **若出現本書沒料到的新變數**：五變數是 2026-06 的切法。若未來冒出一個現在還看不清的成因軸（如某種全新的資料合成範式、或地緣政治直接重畫算力版圖），就該給記分卡加一格——框架要能進化，否則它自己就成了過期的快照。

## 紙上推演

這是全書的壓軸動手做。前面二十章你都在讀別人的歸因；這一題，換你獨立跑一遍完整的五變數拆解。

### 推演題1

**[10 分鐘] ★** 口頭重講。不看書，向另一個工程師把全書四個主張一句句講清楚（架構不是護城河、模型是養出來的、五變數解釋差異、最強測不準風格才穩定），每一句**至少給一個具體證據**（哪一家、哪個現象）。講不出證據的那一句，就是你還沒真懂的那一句。

#### 推演解答

四句話的證據錨點（你的版本只要意思到、證據對就算過）：

1. **架構不是護城河**——證據：八家都是 decoder-only Transformer ＋ self-attention，零件清單高度重疊（RoPE＋GQA/MLA＋MoE，landscape §A）；2017 公開論文人人能讀。護城河搬到資料/算力/後訓練三格。
2. **模型是養出來的**——證據：沒人寫「問 X 回 Y」，能力是設好成長環境（資料、目標、算力）後長出來的（ch02）。對比你寫 cache library 是逐行寫死。
3. **五變數解釋差異**——證據：八家主因落點各不相同（Google 算力＋研究＋產品三格咬合、Anthropic 後訓練＋研究、xAI 算力＋資料…），沒有兩家一樣。
4. **最強測不準、風格穩定**——證據：MMLU-Pro/GPQA 已飽和到分不出前沿、排名每幾個月洗牌、benchmark 會污染（ch04）；但「Anthropic 對齊執念、Google 長上下文、DeepSeek 效率」這些風格年復一年穩定。

常見誤解：把第四句講成「所以沒辦法比較模型」。不對——能比較，但要**分任務軸＋三角驗證＋標日期**，不是不能比，是不能用一個無時效的數字比。

### 推演題2

**[25 分鐘] ★★★** 完整實戰（壓軸題）。下面是一則**虛構的**新模型發表稿，請你拿「五變數判斷協定」從頭到尾跑一遍：剝行銷層 → 寫招牌強項 → 逐格歸因 → 寫影子 → 標日期與失效條件。

> 〔虛構發表稿〕**2026 年 8 月，某東南亞主權雲端商「NusantaraAI」發表開放權重模型 Garuda-1**。新聞稿稱：「Garuda-1 是**全球最強的東南亞語言模型**，在我們自建的 SEA-Bench 上得分超越所有對手，原生支援 15 種東南亞語言，採 MoE 架構（總 200B / 活躍 20B），Apache 2.0 開放權重，跑在我們與當地政府合建的主權資料中心上。」

#### 推演解答

跑協定，一步一步：

**[0] 剝行銷層**：「全球最強的東南亞語言模型」是**無時效的絕對排名**，自動打折（ch04）——改讀成「截至 2026-08，以東南亞語言著稱」。「在我們自建的 SEA-Bench 上超越所有對手」是**自報 benchmark**，而且是自家出的題：高度可疑（自家評測天然偏袒自家，ch04 的評測政治學）；標「自報、待第三方在獨立評測上複現」。剝完行銷，真正可信的事實只剩：開放權重、Apache 2.0、MoE 200B/20B、15 種東南亞語言、主權資料中心。

**[1] 招牌強項（分軸）**：以「**東南亞多語言**」＋「**開放權重＋主權部署**」著稱——不是「它最強」，是它強在這兩個軸。

**[2] 逐格歸因**：
- **資料 ✓主因**——15 種東南亞語言的縱深，是別家（多以英文/中文為重）較難快速複製的專有語料優勢。這格的邏輯跟 Qwen 的中文縱深（ch12）一模一樣：背靠在地業務 → 配比給在地語言更高權重。
- **產品取向 ✓主因**——「東南亞主權雲」的定位由上往下壓出整套形狀（開放權重＋Apache 2.0＋與政府合建主權資料中心）。這跟 Mistral 的「歐洲主權」（ch09）是同一個劇本：主權定位壓出開放。
- **算力·基礎設施 △次因**——自建主權資料中心是「能訓、能服務」的必要條件，但（若體量不大）也可能是規模天花板的來源（Mistral 的影子，ch09）。是執行條件，不是「為什麼強在東南亞語言」的成因。
- **後訓練·對齊 留白／△**——發表稿沒提任何招牌訓練法創新，暫不記為主因。
- **研究文化·品味 △次因**——選 MoE、選開放權重是合理的工程品味，但沒有「先押了別家沒押的方向」的證據，記次因。

**[3] 影子**（必填）：
- 開放權重 → **商業模式問題**（DeepSeek/Qwen 共通影子，ch11/12）：送出權重後怎麼變現？
- 主權定位 ＋ 較小體量 → **規模天花板**與「夾在開放權重大廠與閉源巨頭之間」的生存壓力（Mistral 影子，ch09）。
- 主權／在地背景 → 跨區採用的**地緣信任門檻**與評測可信度爭議（Qwen 影子，ch12）。
- 自家出題的 SEA-Bench → **評測污染/偏袒**風險，分數要打折。

**[4] 日期＋失效條件**：標「截至 2026-08」。何時不成立——（a）換任務軸：它在東南亞語言著稱，不代表 coding/數學/長上下文也領先；（b）被哪家：若 Qwen/Google 補上東南亞語料，資料縱深的護城河會被侵蝕；（c）過多久：SEA-Bench 一旦被別家拿去刷或污染，「超越所有對手」立刻失效。

**結出收束句**：*Garuda-1 強在「東南亞多語言縱深」與「主權開放權重」，因為資料（在地語言配比）＋產品取向（主權定位壓出開放）；算力與研究品味是次因。影子是商業模式、規模天花板、地緣信任門檻與自家評測的可信度。截至 2026-08，待第三方複現。*

常見誤解：被「全球最強」「超越所有對手」帶著走，直接信了排名。整套協定的第一步「剝行銷層」就是專門擋這個——**自報的最強，等於沒說**。你能完整跑出這一題，代表這本書的商品你已經帶走了：你不再需要別人告訴你誰最強，你自己有一把尺。

### 推演題3

**[15 分鐘] ★★** 對帳收官。把這套「五變數判斷協定」對應回你自己做技術選型/架構評審的經驗（如評估要不要引入一個新的 cache 方案、或要不要採購某個代管服務）。你那套評估流程裡，有沒有對應到「剝行銷層、逐格歸因、必寫影子、標失效條件」的步驟？哪一步是你以前常漏掉的？

#### 推演解答

對得上的地方：好的技術選型本來就跟這套協定同構。「剝行銷層」≈ 你看 vendor 的 benchmark 時自動打折、要求自己場景的 PoC 數字；「逐格歸因」≈ 你拆「這個方案強在吞吐？延遲？運維成本？生態？」而不是聽「它很快」；「標失效條件」≈ 你問「這個優勢在我們的流量形狀下還成立嗎、量級長到十倍時呢」。

最常被漏掉的，幾乎都是「**必寫影子**」這一步。人在被一個亮點說服時，天然會略過代價——就像只看「平均延遲降了」卻沒問「p99 呢、failover 時呢、團隊學習成本呢」。這本書把「影子必寫」設成 `## 成因拆解` 的硬性段落，就是要對抗這個人性弱點：**任何只有優點的評估，都還沒做完。** 把這個紀律帶回你的架構評審——每個「強在 X」後面，強制接一句「影子是 Y」——你的選型會穩很多。這也正是全書留給你最實用的一條紀律。
