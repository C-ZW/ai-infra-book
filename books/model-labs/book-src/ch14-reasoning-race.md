# ch14 — 推理模型競賽：o1 之後的範式轉移

> **本章解決什麼問題**：上一章（ch13）把開源 vs 閉源拆成兩種世界觀；這一章換一個橫切面——**推理模型（reasoning model）**。2024 年底，OpenAI 用 o1 做了一件以前沒人當共識的事：讓模型在回答前先「想」一段，而且**越想越聰明**。這在「同一張藍圖、不同養法」地圖上不是某一家的招牌，是一個**全行業在一年內集體跟進**的範式轉移——所以它是 theme 章，不是 company 章。本章回答三個「為什麼」：為什麼「想久一點」會更聰明（它動了 ch02 講的哪一條 scaling 軸）、為什麼是 OpenAI 先做出來而 DeepSeek 讓它變便宜、以及為什麼到 2026 年「會推理」已經是桌面門檻而不是差異化。對齊方法的機制（RLHF→Constitutional AI→RLVR 那條譜系）留給 ch15，本章只用到它的結論。

## 從你已知的出發

回想 ch02 我們一起盯著的那條 scaling 線——Kaplan 2020 那張「投入 vs loss」跨七個數量級的趨勢線。當時整個產業沿著它爬的方式只有一種：**把模型練得更大、餵更多資料、砸更多訓練算力**。x 軸是「訓練期投入」，往右走就更強。這條軸我們在 ch02 已經講透了，它是「養模型」這件事第一條被發現、被信仰的軸。

現在我要你切換到一個你維運時天天在做、但和訓練毫不相干的取捨。你有一個跑批的 job——比如那條 event-driven 結算 pipeline 裡，某個 CronJob 觸發、丟進 SQS、由冪等消費者處理的批次。你心裡很清楚一件事：**同一段邏輯，你給它多少資源、跑多久，結果的品質是可以不一樣的。** 你可以走 fast path——少抽樣、近似算、趕快吐一個「夠用」的數字，延遲低但精度將就；你也可以走 heavy path——多花 CPU 時間、跑更完整的對帳、把邊角 case 都掃過，延遲高但結果紮實。這個選擇不在「你怎麼訓練這套程式」（程式碼是同一份），而在「你**執行時**願意給它多少時間」。**這是一條完全獨立於『程式怎麼寫』的軸：執行期的算力換結果的品質。**

把這兩件事擺在一起，你就摸到本章的全部。在 o1 之前，模型的「強」只掛在第一條軸上——訓練期投入。一個問題不管難或簡單，模型都是「想都沒想」就開始一個 token 一個 token 吐答案（你的推論書裡那套 prefill、decode、KV cache，吐答案的成本是固定的）。o1 做的事，等於是把你那條「fast path vs heavy path」的取捨，**第一次裝進了模型本身**：讓它在吐最終答案前，先在內部生成一大段「草稿思路」，把更多 token、更多**推論期**算力花在「想」上。一條全新的 scaling 軸被打開了——不是「訓練時練更大」，是「**推論時想更久**」。本章就是要拆清楚這條新軸怎麼來、為什麼每一家都被逼著裝上它，以及它的帳記在哪。

## 兩條 scaling 軸：reasoning 到底新在哪

先把最容易混淆的事講清楚：reasoning 範式同時動了**兩條**軸，這是它被稱為「範式轉移」而不只是「一個新功能」的原因。

**第一個轉移：監督式 → 強化學習（supervised → RL）。** ch02 講的 pretraining 是「猜下一個字、猜錯就修一點」——本質是模仿資料裡已經寫好的答案，這叫監督式學習。o1 的訓練多了一層不一樣的東西：用**強化學習（RL）** 訓練模型在**思維鏈（chain-of-thought，CoT）** 上做隱性搜尋。白話講：不再是「照抄人類寫好的推理過程」，而是讓模型自己生成一長串思路、嘗試多種策略、回頭檢查、走錯了退回來，然後**用『最後答案對不對』當訊號去獎勵那些有效的思路**（這套「用可驗證的對錯當獎勵」的方法叫 RLVR，它的機制與在對齊譜系裡的位置是 ch15 的主題，這裡只借它的結論）。模型不是被教會推理的，是被**逼著自己摸索出**一套推理習慣的——這又是一次「養，不是寫」（ch02）。

**第二個轉移，也是更反直覺的一個：scaling 推論算力，不只是訓練算力。** 這是 o1 真正驚人的地方。OpenAI 發現 o1 的效能會隨著**兩種**算力同時上升：訓練期的 RL（第一條軸，舊的），以及**推論期讓它「思考」的算力**（test-time compute，全新的第二條軸）。同一個練好的模型，你在它回答時讓它「想更久」（生成更長的思路、探索更多分支），答案品質就更高。有人形容這「補上了 Bitter Lesson 缺的一塊、開出一條全新的算力擴展軸」——在這之前，產業相信「要更強只能在訓練上砸更多」；o1 之後，**推論期的算力第一次成為一條可以換取智慧的軸**。

這正好是你那條「fast path vs heavy path」取捨的放大版，而且現在有了名字：

```text
舊的唯一一條軸（ch02）：訓練期投入 ↑ → 能力 ↑
   砸在「養」的階段，一次性，養完就定型

o1 開的第二條軸：推論期「思考」算力 ↑ → 答案品質 ↑
   砸在「用」的階段，每一次問答都要重新付
   ≈ 你的 job 走 heavy path：多給 CPU 時間換更紮實的結果
```

讀懂這條新軸的重量：它把「智慧」的一部分從**訓練成本**搬到了**推論成本**。以前你想要更聰明的答案，得等下一代更大的模型（重新訓練、天文數字的一次性投入）；現在你可以拿**同一個模型**，在當下這一次問答多花點算力、讓它多想一會兒，就換到更好的答案。這對你這種做系統的人有一個直接的味道——它是**延遲換品質的取捨，而且這個旋鈕現在握在推論端**。你太熟這種取捨了：你的多層 cache 在「命中率 vs 新鮮度」之間轉旋鈕、你的 autoscaling 在「成本 vs 延遲」之間轉旋鈕。o1 做的，是給模型也裝了一個這樣的旋鈕，刻度叫「想多久」。

## 為什麼是 OpenAI 先做、DeepSeek 讓它變便宜

「為什麼是 OpenAI 先做出來」這個問題，ch05 已經拆過根（先押 test-time compute 是它第四格「研究文化·品味」的招牌），本章不重複那條因果，只記事實當這一節的起點：截至 2026-06，OpenAI 於 **2024-09**（o1-preview，2024-09-12）推出 o1，業界普遍視為第一個主流的 reasoning model（✅）。它**開了頭**。

本章真正要講透的，是**接下來那一拳**——因為「OpenAI 開了頭」如果只是「又一家做了個強模型」，這章就不值得存在。讓 reasoning 從「OpenAI 的招牌」變成「全行業範式」的關鍵事件，是 **DeepSeek-R1**（截至 2026-06，2025-01-20 發表，✅）。

要懂 R1 為什麼是引爆點，先還原 o1 剛出時整個產業的心理狀態。o1 很強，但 OpenAI **沒有公開配方**——它連模型「想了什麼」（那段 CoT）都對使用者藏起來，更別說訓練方法。所以 2024 年底的共識是一種模糊的敬畏：「reasoning 顯然有用，但這多半又是一個只有 OpenAI 那種等級的算力與機密配方才養得出來的東西。」reasoning 看起來像是**貴的、封閉的、難複製的**。

R1 一次打碎了這三個假設。DeepSeek 不但做出了能對標 o1 的開放權重推理模型，還**把配方寫進論文公開**：核心是用 **RL ＋ 可驗證的獎勵**（答案對不對、程式能不能編譯，這種客觀訊號），搭配它招牌的 **GRPO**（Group Relative Policy Optimization，一種砍掉 critic 網路、靠「一組樣本互相比較」來省記憶體的 RL；GRPO 的歸屬與機制是 ch11/ch15 的事，這裡只記它是 R1 的引擎）。這套配方便宜、清楚、可複現——R1 一出，開源社群幾天內就冒出「Open R1」這類完整復現專案（截至 2026-06，✅）。**「reasoning 能用便宜得多的方式重現」這個認知，在 2025 年初被坐實了。**

這就是本章被講透的成因個案，我把它叫做「**範式的便宜化**」：

```text
2024-09  OpenAI o1：證明「想久一點＝更聰明」可行
   │  但配方封閉、CoT 藏起來 → 看起來貴、難複製
   ▼
產業共識：reasoning 有用，但多半是「只有巨頭養得起」的東西
   │
2025-01  DeepSeek-R1：開放權重 ＋ 把配方寫進論文
   │  RL＋可驗證獎勵＋GRPO，便宜、清楚、可複現
   ▼
幾天內 Open R1 等復現專案冒出
   │  「reasoning 能便宜重現」被坐實
   ▼
門檻從「巨頭專屬」塌成「人人可做」→ 全行業在一年內集體跟進
```

這條鏈解釋了一件事：**為什麼 reasoning 的擴散速度，比歷史上任何一個模型能力都快**。RLHF 從 InstructGPT（2022）到全行業普及花了一兩年；但 reasoning 從 o1（2024-09）到「人手一個」只花了大約一年——因為 R1 在中間做了「把配方便宜化、公開化」這一步，等於替全行業把複製成本砍到地板。這也順帶呼應了 ch13 開源 vs 閉源的張力：**一個開放權重的玩家，可以靠『公開配方』把一個閉源先行者的護城河在數月內填平。** OpenAI 開創的紅利吃在最前面，但它定義的範式，被 DeepSeek 用開放的方式變成了公共財。

> ⚠️ 一個必須擋掉的誤讀：R1 讓 reasoning「便宜化」，**不等於**「養前沿模型很便宜」。R1 站在 DeepSeek 自家已經練好的 V3 base model 上，而 V3 那個 base 本身是巨大的投入（ch02 提過它 671B 參數餵 14.8 兆 token；那個「560 萬美金」的成本主張及其但書留給 ch11 拆）。R1 證明的是「**把一個夠好的 base 改造成推理模型**這一步可以很便宜」，不是「從零養一個前沿模型很便宜」。把這兩件事混為一談，就跟把「560 萬美金」當成「養一個前沿模型只要 560 萬」一樣，是抽掉前提到處用。

## 到 2026 年，「會推理」是桌面門檻

範式擴散到什麼程度？把 landscape 的表攤開來看（截至 2026-06，各家措辭照 landscape，✅/🟡）：

| 家 | reasoning 的形態（截至 2026-06） | 備註 |
|---|---|---|
| OpenAI | GPT-5「Thinking」／reasoning-effort 旋鈕（none/low/medium/high/xhigh） | o-series 已收進 GPT-5 router，自動決定快答或深想 |
| Google | Gemini「Deep Think」 | 是 Gemini 3 線裡的一個**推理模式**，不是獨立模型 |
| Anthropic | Claude extended thinking（Opus/Fable 線） | 以 agentic/coding 口碑著稱 |
| DeepSeek | R1 → 收進 V3.1/V4 的推理能力 | 開放權重、GRPO 訓練 |
| Mistral | Magistral（已併入 Mistral Small 4） | 歐洲效率派 |
| xAI | Grok 的 reasoning 模式 | 速度取向 |
| Qwen | Qwen3 hybrid reasoning | **一個 toggle 開關**，可切「想/不想」 |
| Meta | Muse Spark（推理，閉源） | 宣稱平行推理／「思路壓縮」⚠️ 未公開驗證 |

讀這張表，重點**不是**誰的 reasoning 最強（那是 ch04 已經立規矩、會洗牌、分不出絕對名次的事），而是**這張表本身的密度**：八家全到齊。一個 2024 年底還讓人敬畏的能力，2026 年變成了「**沒有就落隊**」的標準配備。

更值得玩味的是它「裝上去的形狀」開始趨同。注意表裡反覆出現的兩個詞：**旋鈕**（OpenAI 的 effort dial）和 **toggle**（Qwen 的 hybrid 開關），以及「**模式而非獨立模型**」（Google Deep Think、OpenAI 把 o-series 收進 GPT-5 router）。這是一個產品取向倒逼出來的收斂：早期 reasoning 是一個獨立的、慢的、貴的模型，使用者得自己選「我要用會思考的那個還是快的那個」；但對一個要服務上億人的產品來說，這是糟糕的體驗（ch05 講 GPT-5 router 時拆過同一個邏輯）。所以各家不約而同地把 reasoning 收斂成**同一個模型裡可調的一檔**——你可以把旋鈕轉到 0（快答、省錢）、也可以轉到 xhigh（深想、貴但紮實）。這正是你那條「fast path / heavy path 同一份程式碼、執行時選一檔」的取捨，被各家**獨立地**重新發明了一遍。

把這件事接回全書主張：reasoning 一旦成為桌面門檻，它就**不再是區分各家風格的軸**了。它從「差異化」退化成「基礎建設」——就像 ch20 會講的，能被互相抄走、會收斂的東西終究退場為背景，真正的差異得退回到別處（資料、算力、產品分發、研究品味的下一個賭注）。**reasoning 是「同一張藍圖、不同養法」這個主張的一次完美演示：配方一旦公開，技巧就會擴散趨同，差異不在這裡。**

## 成因拆解

本章拆的不是「某一家為什麼強」，而是一個橫切面的問題：**為什麼 reasoning 在一年內成為全行業標配**。但記分卡照用——看這五格在「範式擴散」這件事裡各扮演什麼角色。

### (a) 五格歸因：為什麼 reasoning 成為全行業標配

| 變數 | 角色 | 為什麼 |
|---|---|---|
| **資料** | △ 次因 | reasoning 的 RL 訓練偏愛**可驗證**的領域（數學、程式——對錯有客觀標準）。哪一家有乾淨的、可自動判分的資料／合成資料，誰就更好做 RLVR。資料這格決定了「在哪些任務上推理長得起來」（資料牆與合成資料的全貌→ch18）。 |
| **後訓練·對齊** | ✓ **主因** | 範式的本體就在這格。從監督式模仿，轉到「用 RL＋可驗證獎勵讓模型自己摸索推理」（RLVR/GRPO），是後訓練方法的一次躍遷。誰先把這套後訓練配方跑通、誰把它便宜化，直接決定了擴散速度（譜系機制→ch15）。 |
| **算力·基礎設施** | △ 次因 | reasoning 把一部分算力需求從訓練端搬到**推論端**——每一次「深想」都要付推論算力。誰的推論基礎設施便宜（自研晶片、效率架構），誰就更撐得起 test-time compute 的帳（算力全貌→ch16）。 |
| **研究文化·品味** | ✓ **主因** | 「先押 test-time compute 這條軸值得賭」是純粹的方向判斷（OpenAI，ch05）；「把配方公開、便宜化」是另一種品味（DeepSeek）。這格解釋了**為什麼是這兩家先後點燃了範式**——最難被抄走的不是配方，是「當初為什麼敢這樣賭」。 |
| **產品取向** | △ 次因 | 「把 reasoning 收斂成同一模型裡的一檔旋鈕/toggle」是產品判斷：要服務上億人，不能逼使用者手動選模型。產品取向解釋了**範式裝上去的形狀**為什麼各家趨同（自動路由、藏起複雜度）。 |

一句話收束：**reasoning 之所以一年內成為全行業標配，主因在「後訓練·對齊」（RLVR/GRPO 這套新後訓練配方的躍遷）＋「研究文化·品味」（OpenAI 先押這條軸、DeepSeek 把它便宜公開）；算力（帳搬到推論端）、資料（偏好可驗證領域）、產品取向（收斂成旋鈕）是把它放大與定形的次因。影子是——成本與延遲的暴增、以及「會推理」被誤當成「會可靠」。**

### (b) 影子：同一組讓它強的選擇，代價記在哪

reasoning 不是只有上行。同一套「讓它想久一點」的選擇，直接長出三個對應的弱點——這是你這種做系統的人最該先看的故障視角：

- **成本與延遲的暴增，而且方向會反轉。** 「深想」＝生成一大段內部 CoT，那段思路是**真的要算、要付錢、要等**的 token。一個簡單問題（「2+3=?」）如果模型硬要長篇推理，你付的是「過度思考（overthinking）」的稅——慢、貴，**而且不一定更準**。這不是嘴上說說：2025 年的研究發現所謂「定價反轉」——大約兩成的模型比較裡，一個「更便宜」的模型因為吐了太多思考 token，實際**比貴的模型還燒錢**（截至 2026-06，🟡 來自多篇 2025 研究）。對你這種要算 p99 延遲與單位成本的人，這是個熟悉的噩夢：**你以為打開了一個品質旋鈕，其實同時打開了一個成本與延遲的水龍頭，而且兩者不一定成正比**。這正是各家急著做「自動路由、簡單問題別硬想」的原因。
- **reasoning ≠ reliable（會推理不等於會可靠）。** 一段長長的、看起來很有條理的 CoT，給人一種「它真的在嚴謹推理」的強烈說服力——但那段思路可以是**事後合理化**，可以引一步錯的中間結論卻照樣自信地走下去（研究稱誤差會沿著推理鏈累積、甚至把模型帶離正確路徑）。更危險的是：CoT 讓**錯誤看起來更可信**，因為它包裝得像論證。會「想」不代表想得對；它只是更會把（對的或錯的）結論講得頭頭是道。
- **評測過擬合（eval overfitting）。** reasoning 模型在數學競賽（AIME）、研究生科學（GPQA）這類**有標準答案、可自動判分**的 benchmark 上表現耀眼——但這正是它被 RL 獎勵的領域，難免有「為考試而生」的味道。ch04 已經拆過 benchmark 污染與飽和：當一個能力的訓練訊號（可驗證獎勵）和它的評測指標（可驗證 benchmark）高度重疊，漂亮的分數有多少是「真的更會推理」、有多少是「更會考這種試」，是分不清的。reasoning 的招牌數字，要扣這層折扣來讀。

### (c) 何時不成立

提醒你這是**截至 2026-06 的快照、不是定律**：

- **當任務不可驗證時，這套配方會退色。** reasoning 的便宜化高度依賴「可驗證獎勵」——答案對不對、程式能不能跑，有客觀訊號才好做 RL。一旦換到**沒有標準答案**的任務（創意寫作、開放式判斷、價值權衡），「想久一點＝更準」的保證就鬆掉了，因為你連「更準」都沒法自動判分（這也是合成資料在這些領域有 model collapse 風險的同一個根，→ch18）。
- **當「想更久」撞到收益遞減時。** test-time compute 這條軸不是無限的——2025 年已有研究指出大型推理模型存在「test-time scaling 高原」：想到某個程度後，再多給算力，品質不再明顯上升，只剩成本上升。這條新軸和 ch02 那條訓練軸一樣，是**有斜率、會彎、會撞牆的曲線，不是提款機**。
- **當它從差異化退化成背景時（這正在發生）。** 截至 2026-06，八家全有 reasoning（見上表）——一個人人都有的東西，就不再是任何一家的「強項」。今天值得寫進一家招牌的，已經不是「有沒有 reasoning」，而是「reasoning 之外還押對了什麼」。範式的紅利有保鮮期，o1 開創者的光環已經被「人手一個」稀釋掉了。

## 紙上推演

### 推演題1：對帳——把 test-time compute 對應到你的 fast path / heavy path [12 分鐘] ★★

你在結算 pipeline 裡做過「同一份邏輯、fast path 走近似、heavy path 走完整對帳」的取捨。請對著它，講清楚 test-time compute（讓模型「想更久」）對應的是這個取捨的哪一面，以及這個對應**在哪裡會破**（哪個地方 fast/heavy 的比喻撐不住）。

#### 推演解答

**對應得相當乾淨：test-time compute 就是把你的「fast path / heavy path」旋鈕裝進模型。** 同一份「程式碼」（同一個練好的模型權重，沒有重新訓練），執行時你選一檔——旋鈕轉到低（快答、省算力、延遲低、品質將就）≈ fast path；轉到高（生成長 CoT、多花推論算力、延遲高、品質紮實）≈ heavy path。關鍵是這條軸**獨立於訓練**：就像你的 heavy path 不是靠「重寫程式」換來的，是靠「執行時多給 CPU 時間」換來的；reasoning 的品質提升也不是靠「練一個更大的模型」，是靠「推論時多想一會兒」。這就是 o1 那條「第二條 scaling 軸」的全部直覺。

**比喻在哪裡破（這是重點，兩處）**：

1. **你的 heavy path 是你寫死的、可解釋的**——你知道它多花的 CPU 時間具體去算了什麼（多抽了哪些樣本、補對了哪幾筆帳），因果透明、有作者。模型的「深想」不是：那段 CoT 是**長出來的**（ch02），沒有人寫「遇到這類題就這樣推」，所以你**無法保證**多花的算力真的用在刀口上——它可能在「過度思考」一個簡單問題、把算力燒在無用的繞路上（這就是 overthinking 稅）。你的 heavy path 不會對「2+3=?」突然跑十分鐘，模型會。

2. **你的 heavy path 走完，結果通常更對**（你多對了一次帳）。模型的「想更久」**不保證更對**——長 CoT 可能是事後合理化、可能沿著一步錯的中間結論累積誤差。你的比喻裡「多花時間 → 更紮實」幾乎是因果鐵律；在 reasoning 裡它只是**傾向**，而且只在可驗證的領域裡比較成立。

常見誤解：把 test-time compute 想成「免費的智慧升級——反正想久一點就更好」。錯。它是一個**會反咬的延遲/成本旋鈕**（定價反轉、overthinking 稅），而且它換來的「更好」有保鮮期與適用範圍（撞高原、不可驗證任務退色）。把它當你維運過的任何一個「成本 vs 品質」旋鈕看就對了——沒有哪個這種旋鈕是只有好處、可以無腦往上轉的。

### 推演題2：反事實——如果 test-time compute 根本沒用，2025 年會長什麼樣？ [12 分鐘] ★★★

反事實假設：假設 o1 那條「推論時想更久 → 答案更好」的軸**根本不成立**（多花推論算力，品質就是不會上升）。只改這一個事實，2025–2026 的格局會怎麼不同？這個反事實揭露了 reasoning 範式的什麼本質？

#### 推演解答

如果第二條軸不存在，那麼「要更強」就**只剩 ch02 那一條訓練軸**——練更大、餵更多、砸更多訓練算力。後果一條條推：

1. **o1 不會是「範式轉移」，頂多是「一個調得不錯的模型」。** 它之所以震撼，正是因為它開了一條**新軸**；軸若不存在，o1 就只是又一個沿舊軸往上爬的點，不值得全行業掉頭。

2. **DeepSeek-R1 的引爆力會消失。** R1 的意義是「把這條新軸的配方便宜化、公開化」；軸若無用，「便宜複現一個沒用的東西」沒有任何衝擊力，2025 年初那場「reasoning 能便宜重現」的集體認知重置不會發生。

3. **整個產業會更死守「scaling 就是把模型練更大」。** 而那條路在 2025–2026 正好撞上兩堵牆：資料牆（高品質人類文字見底，→ch18）和訓練成本暴漲（→ch16）。少了 test-time compute 這條「不必重新訓練就能換智慧」的逃生路，產業會在「越練越大、越來越貴、資料卻見底」的死巷裡更早撞牆。

4. **「智慧的成本結構」不會改變。** reasoning 範式做的一件深層的事，是把一部分「變聰明」的成本從**一次性訓練**搬到**每次問答的推論**。軸若不存在，這個搬移不會發生，智慧就還是「只能靠下一代更大模型」的、被訓練成本鎖死的東西。

**這個反事實揭露的本質**：reasoning 範式的核心價值，**不在「模型會推理」這個能力本身**（那只是表象），而在它**打開了第二條可以換取智慧的軸**，並因此給了一個「訓練軸快撞牆」的產業一條新的成長路徑。是「多開一條軸」這件事改變了格局，不是「會想」這個功能。這也解釋了為什麼它擴散得這麼快、這麼徹底——一條新的成長軸，對一個正在撞牆的產業，是救命的。

### 推演題3：找破綻——「我們的模型會推理，所以它更可靠」 [10 分鐘] ★★

某家發表會上說：「我們的新模型會做長鏈推理（會展示完整思路），所以它的答案更可靠、更值得信任。」用本章的東西，指出這句話**至少兩個**破綻。

#### 推演解答

1. **「會推理」≠「會可靠」（reasoning ≠ reliable）。** 一段長長的、有條理的 CoT 給人強烈的「它在嚴謹思考」的說服力，但那段思路可以是**事後合理化**，也可以引一步錯的中間結論卻自信地走下去（誤差沿鏈累積）。更糟的是 CoT 讓**錯誤看起來更可信**——它把結論包裝得像論證。所以「展示了完整思路」恰恰可能讓你**更難**發現它錯了，因為它說得頭頭是道。會想不代表想得對。

2. **「展示思路」本身可能是表演。** 模型展示給你看的 CoT，不保證就是它「真正」用來得到答案的計算過程——它可以生成一段看起來合理、但其實是為了好看而產出的思路。所以「我給你看了我的推理」不等於「我的推理是健全的」，這是兩件事。

3.（加分）**它把可驗證任務的優勢，偷渡成全領域的可靠。** reasoning 在數學、程式這種**可自動判分**的任務上確實更強（那是它被 RL 獎勵的領域）；但「更可靠」這個全稱主張，在沒有標準答案的任務（開放判斷、價值權衡）上根本沒有保證——那裡連「對不對」都沒法驗證，更談不上「想久一點更可靠」。把可驗證領域的戰績，講成「所以它整體更可信」，是抽掉前提的過度推銷。

常見誤解：把「看得到思路」當成「思路可信」。透明度（你看得到它想什麼）和正確性（它想得對不對）是**兩個獨立的軸**——CoT 給了你前者，完全不保證後者。對你這種讀過 observability 的人，這個區分很自然：**一個系統吐出詳細的 log，不代表它的邏輯就是對的；log 只是讓你『看得到』，不等於『它對』。**

## 自我檢核

口頭自答，講不清楚就是還沒懂：

1. reasoning 範式同時動了**兩條** scaling 軸——哪兩條？為什麼說第二條（test-time compute）比第一條更反直覺？（提示：智慧的成本從訓練端搬到了哪裡）
2. 用你的 fast path / heavy path 取捨，對另一個工程師講清楚 test-time compute 是什麼。然後講出這個比喻**在哪裡會破**（提示：你的 heavy path 可解釋且通常更對，模型的「深想」兩者都不保證）。
3. **為什麼**是 DeepSeek-R1（而不是 o1）讓 reasoning 從「巨頭招牌」變成「全行業範式」？R1 公開的那一步，具體打碎了哪三個假設？
4. 「R1 證明 reasoning 可以很便宜」**不等於**「養前沿模型可以很便宜」——這兩句話的差別在哪？（提示：R1 站在誰的肩膀上）
5. **如果**test-time compute 這條軸根本不成立，2025 年的產業會怎麼不同？這個反事實說明了 reasoning 範式的核心價值其實是什麼？
6. 為什麼說「會推理不等於會可靠」？長 CoT 為什麼反而可能讓錯誤**更難**被發現？
7. 截至 2026-06，八家全有 reasoning——這對「reasoning 還算不算某一家的差異化招牌」說明了什麼？（提示：會被抄走、會收斂的東西，最後退場成什麼）
8. **如果不**做「自動路由、簡單問題別硬想」，直接讓模型對每個問題都深想，會踩到哪兩個本章講過的故障？（提示：定價反轉、overthinking 稅）
