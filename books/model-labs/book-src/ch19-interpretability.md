# ch19 — 我們造出看不懂的東西：可解釋性與安全

> **本章解決什麼問題**：這是全書最 sci-fi 的一個橫切面。前面十八章，我們一直說「模型是**養**出來的、不是**寫**出來的」（ch02）——這一章要把這句話最反直覺的後果攤開來講清楚：**一個訓練完的前沿模型，內部在算什麼，連造出它的人都不完全看得懂。** 沒有原始碼、沒有「遇到 X 就回 Y」的規則，只有幾千億個浮點數。本章回答三件事：(1) 為什麼會這樣——能力是長出來的，所以沒有可讀的源碼；(2) 各家拿什麼去面對這個黑箱——**可解釋性（interpretability，「模型的神經科學」）、red-teaming、evals/安全研究**，而且這是一個**整個產業共同在做的研究領域**，不是某一家的專利；(3) 各家在「安全 vs 速度／開放」上的取向差異——**講取捨，不褒貶**。Anthropic 這一家在可解釋性上的個別故事已在 ch07 講過，本章只把它當「其中一個玩家」、放回產業全景裡看。橋接：可解釋性≈你對一個讀不懂原始碼的黑箱系統做 **observability**——拿探針去戳它。影子：可解釋性**還很早期**（別過度推銷）、安全與能力／速度的張力、「安全」也可能是行銷。

## 從你已知的出發

你維運過你**沒有完整讀懂原始碼**的系統。

想想你接手過的某個遺留服務，或某個第三方的黑箱元件：你不可能逐行讀懂它每一條分支，但你**照樣得讓它穩定上線**。你怎麼做的？你不會因為讀不懂就放棄——你**插探針**。你加 trace、拉 metrics、埋 log，在它的關鍵路徑上插滿觀測點，然後**從外部行為反推內部狀態**。你那條自建的 **FluentBit pipeline** 做的就是這件事：你把一個你沒法逐行讀懂的系統，變成一個你**能觀測、能在它出事時看見徵兆**的系統。你接受了一個務實的前提——「我不需要讀懂每一行，我需要的是在它要爆之前能看見儀表板上的紅燈」。

這一章要講的，是把這個處境**放大到極限**的版本。前沿大模型不是「你沒讀懂的原始碼」——它**根本沒有原始碼**。沒有人寫過「遇到金門大橋就這樣回」，能力是從海量資料裡**長出來的**（ch02）。所以它不是「難讀的程式」，它是一個**幾千億參數、誰也沒寫過源碼的神經網路**。你那套「讀不懂就插探針做 observability」的本能，在這裡升級成一整個研究領域，有個野心勃勃的名字——**可解釋性（interpretability）**，也被形容為「**模型的神經科學**」。

而 observability 的精神在這裡也原封不動地成立：**你插探針不是為了讀懂每一行，是為了在它要做出危險的事之前，能看見徵兆。** 這就把「可解釋性」和「安全」這兩件事，從一開始就綁在了一起。

## 為什麼「連造它的人都不全懂」

先把這個最反直覺的前提講到讓你信服。一個訓練好的 GPT、Gemini、Claude、Llama，到底「不懂」在哪？

回想 ch02 講的訓練本質：你**沒有**逐條寫規則。你設好一個成長環境（資料、最佳化目標、算力），讓模型透過「預測下一個 token」這個簡單目標，在幾千億參數上**自己長出**能力。訓練結束，你拿到的是什麼？是一個巨大的數字陣列——**幾千億個浮點數權重**。它能寫程式、能翻譯、能推理，但這些能力**不對應任何一段你能讀的程式碼**。沒有一個函式叫 `translateToFrench()`，沒有一行寫著 `if topic == "金門大橋"`。能力是**分散地、糾纏地**編碼在那片數字海裡的。

這跟你寫過的任何系統都本質不同。你的 cache library 再複雜，它的每一個行為都**有一行你（或某個人）寫過的程式碼負責**——出了 bug，你能 git blame 到那一行。前沿模型沒有這種東西可以 blame。它表現出某個行為（比如突然諂媚、或在某類問題上幻覺、或被某種提示「越獄」），你**找不到「那一行」**，因為根本沒有那一行。

更難的是兩件結構性的事，它們是「看不懂」的真正根源：

- **疊加（superposition）**：模型內部的神經元**不是一個蘿蔔一個坑**。一個神經元往往**同時參與了很多個互不相關的概念**（這叫多義性 polysemanticity）——同一個神經元可能對「金門大橋」「某種程式語法」「某種情緒」都有反應。為什麼？因為模型要用有限的神經元表達遠多於神經元數量的概念，它就學會了把概念「壓縮疊加」在一起。這讓你**沒法靠「讀單個神經元」來理解模型**——每個神經元都是一團糊在一起的訊號。
- **湧現（emergence）**：很多能力不是被設計出來的，是**規模到了某個點突然冒出來的**。沒有人寫過、也沒有人預測得準它何時出現。能力既然是「湧現」的、不是「設計」的，自然也就沒有設計文件可讀。

> 對你的直覺：這就像一個**極端版的分散式系統湧現行為**。你維運高併發系統時見過——沒有任何一個元件「負責」整體的雪崩，雪崩是無數元件交互**湧現**出來的，你在任何單一服務的程式碼裡都找不到「雪崩」這個邏輯。前沿模型把這件事推到極致：**它的每一個能力都是湧現的，沒有一行程式碼「負責」它**。你習慣的 git blame、讀源碼、設中斷點，在這裡全部失效。

所以「連造它的人都不全懂」不是一句行銷話術，也不是工程師謙虛——它是這種「養出來而非寫出來」的造物方式**必然的代價**。你用「設環境讓能力自己長」換到了「沒有可讀源碼」。這就是為什麼需要一整個「神經科學」去**事後**研究它。

## 可解釋性在做什麼：把黑箱當大腦解剖

既然沒有源碼可讀，可解釋性研究就反過來——**把訓練好的模型當成一個已經存在的大腦，拿工具去解剖它、研究它的內部結構**。這就是「模型的神經科學」這個比喻的由來：神經科學家也面對一個沒有源碼的大腦，他們的辦法是插電極、做掃描、找出「哪一群神經元負責什麼」。可解釋性做的是同一件事，對象換成神經網路。

截至 2026-06，這個領域有兩個可以具體點名的代表性方向（✅ 歸屬與年份以 landscape 為準）：

- **稀疏自編碼器 / 單義性（sparse autoencoders / monosemanticity，2024）**——這是對付上面那個「疊加／多義性」難題的辦法。既然一個神經元是一團糊在一起的訊號，那就**用一個輔助的小網路（稀疏自編碼器，SAE），把這團糊訊號拆解成一個個乾淨、可解讀的「特徵」（feature）**。一個特徵不再是「一個神經元」，而是「一個跨越很多神經元、但只對應**單一概念**的方向」——例如「金門大橋」「程式碼裡的某種 bug」「諂媚式的奉承」各是一個特徵。Anthropic 2024 從 Claude 3 Sonnet 裡用這招抽出了**數百萬個**這樣的特徵（細節在 ch07）。
- **電路追蹤 / 歸因圖（circuit tracing / attribution graphs，約 2025-03）**——找到「有哪些特徵」只是第一步。下一步是追蹤**這些特徵之間怎麼連、資訊怎麼一步步流過模型**，畫出類似電路圖的東西。這等於從「標出大腦裡有哪些功能區」進到「畫出這些區之間的神經迴路」。

> 對你來說，這件事的橋接精準得驚人：**這就是對黑箱系統做 observability。** SAE 抽特徵≈你在一個沒有 log 的服務裡**埋進結構化的觀測點**，把混在一起的內部狀態拆成你看得懂的維度；電路追蹤≈你做**分散式 tracing**，把一個 request 怎麼流過你的微服務鏈、在哪一跳被改寫，一段段串起來。你做 observability 從不指望「讀懂每一行」——你指望的是「在足夠多的關鍵點插探針，從外部反推內部」。可解釋性是同一種精神，只是被戳的對象是一個誰也沒寫過源碼的神經網路。

**這是一個產業級的研究領域，不是某一家的專利**——這點本章要特別強調，因為 ch07 講 Anthropic 時容易給人「這是 Anthropic 的東西」的錯覺。實際上：⚠️/✅（部分一手、部分二手，2026-06）**OpenAI** 也做過大規模 SAE，在 GPT-4 級模型裡找到可解讀特徵（例如一個「humans have flaws」的特徵）；**Google DeepMind** 2024 用 SAE 分析 Gemma 2、並公開了相關工具（Gemma Scope）；學界更是有一整片 mechanistic interpretability 的社群在跑。各家投入的**力度**與**對外講述的多寡**確實不同（Anthropic 講得最大聲，見 ch07），但「想搞懂黑箱內部」這件事，是**整個前沿產業共同面對、共同在推**的研究戰線。

### 招牌個案：把「金門大橋」這個特徵調到最大會怎樣

抽象的「特徵」聽起來還是很玄。讓我用一個能把「神經科學」變具體的真實實驗收尾——這也是這個領域最廣為人知的一次公開展示。

✅（Anthropic, 2024，從 Claude 3 Sonnet）研究者用 SAE 在模型內部找到了一個專門對「**金門大橋**」有反應的特徵。然後他們做了一件關鍵的事：**不是去改提示詞、也不是重新訓練模型，而是直接在模型推論的當下，把這個「金門大橋」特徵的活化值「夾」到最大**（這叫 activation steering，操弄推論期的內部活化）。結果：模型開始**逢人就說自己是金門大橋**，無論你問它什麼，它都能把話題繞回金門大橋，甚至會表達對橋、對舊金山的情感。這個被刻意「調過」的版本後來被暱稱為 **Golden Gate Claude**。

為什麼這個小實驗如此重要？因為它一次證明了三件原本只是假說的事：

1. **模型內部真的有「可指認」的概念表徵**——「金門大橋」不是糊在整個網路裡無法定位的，它對應到一個**具體、可被找到的方向**。
2. **這個表徵是可以被外科手術式操弄的**——你可以精準地把「某一個概念」調強或調弱，而不動其他部分。這從「觀測」跨進了「干預」，正是真正理解一個系統的標誌（你能可控地改變它，才算真的懂它一部分）。
3. **它把「可解釋性」和「安全」直接接了起來**——同樣的技術，把「金門大橋」換成「諂媚」「製造生化武器的意圖」「欺騙」，你就有機會**在模型內部偵測、甚至抑制**這些危險特徵，而不必等它在輸出端出事。（同類研究也展示了用特徵去**降低有 bug 的程式碼生成、減輕偏見、偵測有害輸出**。）

> 對你的直覺：這就像你終於在一個黑箱服務裡，找到了那個一直找不到的「**feature flag**」——而且不只能讀它的值，還能在 runtime 把它扳開或關掉，然後親眼看到下游行為跟著變。從「我只能觀測」到「我能定位並干預」，這是 observability 的聖杯。Golden Gate Claude 就是可解釋性領域第一次在前沿模型上、對著一個玩笑般的特徵，公開摸到了這個聖杯的邊。

但**請記住「邊」這個字**——後面的影子段落，我們會誠實地把這個成就的真實成熟度標清楚。能定位「金門大橋」這種**鮮明、具體**的特徵，距離「能完整讀懂模型每一個決定」**還非常遠**。

## 另外兩條防線：red-teaming 與 evals

可解釋性是「打開黑箱往裡看」的那條路，野心最大、也最不成熟。在它成熟之前，產業靠的是另外兩條**更務實、更黑箱外部**的防線。它們的共同精神依然是你熟的：**既然讀不懂內部，那就在外部把它往死裡測。**

- **Red-teaming（紅隊測試）**：組一支人（或用另一個模型）扮演攻擊者，**主動想盡辦法讓模型出事**——誘導它說出製造危險物的步驟、繞過安全限制（越獄 jailbreak）、產生有害或違法內容。這完全就是你熟的**滲透測試／混沌工程**搬到模型上：你不等它在生產環境出事，你在上線前**主動製造最壞情況**，看它在哪裡崩。red-teaming 的價值不在「證明模型安全」（你永遠證明不了），而在「**找出它已知的破口**」，這跟你做 chaos engineering 的心態一模一樣——主動注入故障，比被動等故障好。
- **Evals（評測／安全評測）**：建立一整套標準化的測試集，**量化**模型在各種維度上的行為——不只是 ch04 講的能力 benchmark（MMLU、SWE-bench…），還包括**安全向**的評測：它有多容易被越獄、會不會諂媚（sycophancy）、在高風險領域（生化、網路攻擊）能提供多少危險協助、會不會欺騙。這就是你的**自動化測試套件 ＋ CI 把關**搬到模型上：每訓一版新模型，先跑一遍安全 evals，不過關的維度就是已知風險。

⚠️ 但這兩條防線有一個你做安全工程時最熟的根本限制：**測試只能證明 bug 存在，不能證明 bug 不存在**。red-teaming 找到的是**你想得到去測的**攻擊；evals 量的是**你已經定義出來的**風險維度。對一個能力會湧現、行為會在你沒測過的角落冒出來的系統，「跑過了所有 evals」**遠不等於**「安全」——它只等於「在我們已知的測試集上沒翻車」。這就是為什麼可解釋性（直接看內部）仍然是聖杯：因為只有真的看懂內部，你才可能發現那些**你根本不知道該去測的**風險。red-teaming 和 evals 是「黑箱外部的防線」，可解釋性是想「把箱子變透明」——前兩者是現在能用的，後者是還在路上的。

## 各家的安全取向差異：講取捨，不褒貶

現在進到本章一個需要極度小心的部分：**各家在「安全」上的取向確實不同——但這是一個取捨光譜，不是好人壞人榜。** 本書的態度（ch04 立的規矩）是講成因、講取捨，不替任何一家的安全敘事背書、也不唱衰任何一家。

把它放回**五變數**來看，安全取向差異主要落在第四格**研究文化·品味**與第五格**產品取向**：

- 有些家把**安全與對齊當第一性原則、長期養可解釋性團隊**（這個取向在 ch07 講透了，是研究文化品味這格的押注）。它的代價——速度與開放可能讓步：投在安全研究上的人力與算力，不會讓下一版在 benchmark 上多幾分；某些能力可能因安全考量而延後釋出或加更多限制。
- 有些家更偏**速度取向**（ch10 的 xAI 是這一極的代表）：快速堆算力、快速迭代、公開試錯，安全治理相對輕。它的代價——更容易出內容治理爭議、更依賴上線後的補救。
- 有些家在**開放權重 vs 閉源**這條軸上的選擇（ch13 的主題），本身就**內含一個安全論述**——而且**兩個方向都有自己的安全故事**，這正是「沒有絕對對錯」的最好證明：
  - **閉源派**的安全論述：權重不公開，才能控制誰能用、能避免被拿去做危險用途（防濫用、防越獄後直接拿到原始能力）。
  - **開放權重派**的安全論述：權重公開，全世界研究者才能一起檢查、一起做可解釋性、一起找破口（防黑箱、防單一公司說了算）。

> 關鍵的反直覺：**同一個「開放」選擇，在一邊是安全的理由、在另一邊是不安全的理由。** 閉源派說「公開＝危險（誰都能濫用）」，開放派說「公開＝安全（誰都能監督）」。你沒法說哪邊「對」——它們**量的是不同的風險**（濫用風險 vs 集中／黑箱風險）。這跟你在工程上權衡「把內部工具開源 vs 鎖在公司內」是同一種兩難（見 ch13）：開源讓更多人幫你抓 bug，但也讓攻擊者看得更清楚。**安全不是一個可以最大化的單一數值，是一組互相拉扯的取捨。**

所以，當你看到任何一家高調談自己的安全投入，正確的讀法不是「所以這家最有良心／最虛偽」，而是去問：**它在「安全 vs 速度／開放」這個光譜上站哪裡？它選的這個位置，讓它強在哪、暴露在哪？** 這才是本書要的「成因視角」。下一節的影子，就是來把這個取捨的代價講白。

## 成因拆解

### (a) 五格歸因

本章是**主題章**，拆的不是某一家，而是「**為什麼有些家在可解釋性／安全上投入特別多**」這個橫切現象。歸因的重點：這主要是一個**研究文化·品味**的押注，被**產品取向**部分塑形；資料與算力是次因（它們是安全工作的成本，不是它的成因）。

| 變數 | 主/次 | 為什麼 |
|---|---|---|
| 資料 | （次因／背景） | 安全評測需要建構危險場景的測試資料；red-teaming 會產生大量「攻擊—防禦」資料回頭餵對齊訓練。但資料不是「為什麼有家願意投入安全」的成因，是執行安全工作的素材。留白偏次。 |
| **後訓練·對齊** | **△次因** | 可解釋性與 red-teaming 的成果會回流到後訓練——找到「諂媚特徵」可以幫忙抑制它、red-team 找到的越獄可以補進對齊資料。對齊是安全研究的**下游受益者與執行面**，但「為什麼投入」的根不在這格。 |
| 算力·基礎設施 | △次因 | 跑大規模 SAE、訓練解釋用的輔助網路、跑海量安全 evals，都要燒算力。算力決定你**做得起多大規模的可解釋性**，但它是成本約束，不是動機來源。 |
| **研究文化·品味** | **✓主因** | **這是核心。願意長期養一支「短期不賺錢、不會讓 benchmark 多幾分」的可解釋性／安全團隊，是一種價值取向上的押注（這個判準在 ch07 講透）。「該把安全擺多前面、該為它讓多少速度」——這完全是研究品味的選擇，也是各家差異的根源，且最難被抄走。** |
| **產品取向** | **✓主因** | **你服務誰，部分決定你需要多重的安全。要把模型推進企業、政府、高風險領域（醫療、金融、生化研究）的家，安全與可審計性是賣點與門檻；純消費級、速度優先的家，安全的商業誘因相對弱。開放 vs 閉源（ch13）的選擇也內含不同的安全論述。產品形狀，部分壓出了安全投入的形狀。** |

> 註：跟 company 章不同，這章的因果是「**研究文化品味（主因，押不押安全這個方向）＋ 產品取向（主因，要服務誰）→ 各家在『安全 vs 速度／開放』光譜上站不同位置**」。安全不是某一家的「強項」，是整個產業共同面對的約束；各家的差異，在於它們**選擇站在這個取捨光譜的哪一點**。

一句話收束：**「為什麼有些家在可解釋性／安全投入多」主要因為〔研究文化·品味（願長期押短期不賺錢的方向）＋產品取向（要服務高風險／企業／開放生態，安全成為門檻或論述）〕兩格主導；後訓練與算力是執行面的次因。影子是——可解釋性還很早期（遠不能完整讀懂模型）、安全與能力／速度的張力（投安全＝讓速度）、以及「安全」本身可能被當成行銷話術。**

### (b) 影子

把資源大量投向可解釋性與安全的這組選擇，同時也讓投入者（和整個產業）暴露在三個地方：

- **可解釋性還很早期——別過度推銷，這是最大的影子。** ⚠️ Golden Gate Claude 很迷人，但它能漂亮地定位的是**鮮明、具體**的特徵（一座橋、一種明顯情緒）。對於「模型在這次回答裡到底走了哪條推理路徑、為什麼這次幻覺、為什麼這次被越獄」這種**真正關鍵**的問題，目前能看懂的仍是模型內部**極小的一塊**，遠不是「已經能完整讀懂模型」。更冷靜的訊號是：⚠️（2025–26，部分二手）有研究指出 SAE 在大規模、嚴格的評測上**並沒有穩定打敗簡單基線**（在概念偵測、行為操控等任務上），Google DeepMind 的可解釋性團隊據報因此**調降了對 SAE 路線的優先級**。這不代表可解釋性沒前途，而是提醒你：**這是一個還在劇烈試錯、方法會被推翻的早期領域**，任何「我們已經能讀懂模型了」的說法，現階段都該被高度懷疑。本章自己的態度必須一致：講方法、講野心，但**不替任何成熟度敘事背書**。
- **安全與能力／速度是真實的張力，不是免費的美德。** 投在可解釋性、red-teaming、安全 evals 上的每一份人力與算力，**不會**讓下一版模型在 SWE-bench 上多幾分。把安全擺前面的家，在「最新最強、最快上線」這條賽道上，結構性地會比純速度取向的對手**讓出一點身位**（ch07 甚至看到具體的容量代價）。反過來，把速度擺前面的家，換來迭代快，但暴露在更多上線後的治理爭議。**這是一個你沒法兩頭全拿的取捨**——任何宣稱「我們又最安全又最快又最開放」的敘事，都值得你追問它在哪一頭其實做了讓步。
- **「安全」可以是真投入，也可以是行銷話術——而且外界極難分辨。** 這接回 ch18 結尾埋的同一個問題：當一個優勢**無法被外部證偽**時，你怎麼分辨它是真實的結構性領先、還是講得好聽的故事？「我們很重視安全」這句話，正是這種難證偽的主張之一——可解釋性研究確實能餵養一個漂亮的安全敘事，但「有對外講安全」和「真的把安全擺在能力前面、並付出速度代價」是兩回事。判準（ch07 給過）：看這個投入**短期賺不賺錢**、看它**有沒有真的讓出速度／能力**，而不是看它對外講得多響。一個誠實的讀者，對所有家的安全敘事都該保留這層懷疑——**包括那些你直覺上比較信任的家**。

### (c) 何時不成立

本章描述的圖像（黑箱不可解、可解釋性是早期前沿、安全是取捨光譜），會在什麼情況下被打臉？

- **被可解釋性的突破打臉**：如果某天 SAE 的後繼方法、或某種全新的解釋技術真的成熟到能**可靠地、可擴展地**讀懂模型在做關鍵決定時的內部過程，那「連造它的人都不全懂」這句話的力道就會減弱——黑箱會變成半透明箱。截至 2026-06 這還遠未發生（見上面的 SAE 質疑），但這是一個**正在被猛攻**的方向，可能在本書的保鮮期內就有局部突破。本章標的是 2026-06 的成熟度快照，**不是定律**。
- **被監管／事故打臉**：本章把安全寫成各家自選的「取捨光譜」。但如果出現重大安全事故、或強監管（歐盟 AI Act 之類）落地，安全可能從「研究品味的選擇」變成「**人人都得做的合規門檻**」——那時各家在這條軸上的差異會被法規拉平，「投不投安全」不再是品味問題，而是能不能營業的問題。屆時這一章的「光譜」會被壓縮成「及格線」。
- **被「安全外溢成能力」打臉**：本章（為了講取捨）強調了安全 vs 能力的張力。但 ch07 給過一個反向的真實案例——Anthropic 把對齊當第一性原則，結果**外溢**成了 coding/agentic 的可靠度。也就是說，「安全」與「能力」不總是此消彼長，某些安全工作（如讓模型更聽話、更可控、更少胡說）**本身就是能力**。所以「投安全＝犧牲能力」這個取捨，在某些維度上會被「安全即能力」打臉——這也提醒你，五變數從來是**交互作用**，少有乾淨的單向取捨。
- **本章所有具體事實都是 2026-06 的快照**：可解釋性的方法（SAE 的興衰）、各家的安全取向、監管狀態，每一項都在快速變動。穩定的是「**模型是養出來的、所以沒有可讀源碼、所以需要事後的神經科學去研究它**」這個**結構**；會過期的是任何一個具體的方法成熟度、團隊優先級、與某一家當下的安全姿態。

## 紙上推演

### 推演題1 — 對帳：把「可解釋性」對應回你對黑箱系統做 observability **[12 分鐘]** ★★

不准用 ML 術語，把「可解釋性研究在做什麼」整套對應回你維運過的一個**具體**經驗：對一個你**沒有完整原始碼**的系統做 observability。要對應到三層——**觀測、定位、干預**——並說出這個類比在哪裡會**失準**（類比的極限）。

#### 推演解答

最貼的對應是你接手一個讀不懂源碼的遺留服務、或一個第三方黑箱元件，然後靠你那條 **FluentBit pipeline** 把它變得可觀測。三層對應：

1. **觀測（你埋 log/metrics ↔ 可解釋性抽「特徵」）**：你不指望讀懂每一行，你在關鍵路徑插探針，把混在一起的內部狀態拆成你看得懂的維度。SAE 抽特徵做的就是這件事——把一個「一團糊」的神經元活化，拆成「金門大橋」「諂媚」這種乾淨、可讀的維度。
2. **定位（你做分散式 tracing ↔ 電路追蹤）**：光有 metrics 不夠，你還要知道一個 request 怎麼流過你的服務鏈、在哪一跳被改寫。電路追蹤／歸因圖就是模型版的 tracing——追蹤資訊怎麼一步步流過模型、哪些特徵連到哪些。
3. **干預（你扳 feature flag ↔ activation steering）**：observability 的聖杯是不只能讀、還能在 runtime 改。Golden Gate Claude 就是這個——研究者在推論當下把「金門大橋」特徵夾到最大，模型行為立刻跟著變。能可控干預，才算真的懂了一部分。

**類比在哪裡失準（這才是這題的重點）**：你的遺留服務**有源碼**——你讀不懂只是因為沒時間／太亂，但「那一行」**存在**，理論上 git blame 得到。前沿模型**根本沒有那一行**——它的行為是幾千億參數湧現的，沒有任何一段程式碼「負責」某個行為。所以可解釋性比 observability **難一個量級**：你做 observability 是「補上沒讀的源碼的外部視圖」，可解釋性是「**為一個從不存在源碼的東西，從零發明出一套源碼的概念**」。這就是為什麼它還這麼早期。

常見誤解：把可解釋性想成「就是更高級的 log」。不是——log 是把**已經被寫進程式的事件**輸出來；可解釋性是去**逆向發現那些從沒被任何人寫過、卻自己長出來的結構**。前者是讀，後者是考古。

### 推演題2 — 找破綻：「模型可解釋性已經解決了，我們已經能讀懂模型」 **[10 分鐘]** ★★★

有人看了 Golden Gate Claude 的 demo，興奮地說：「你看，他們都能精準定位『金門大橋』這個概念、還能扳動它了——可解釋性已經基本解決，我們已經能讀懂模型內部了。」請你用 90 秒，**不否定**這個成就的真實價值、但精準指出這句話過度推銷在哪。

#### 推演解答

完整的拆法分三刀：

1. **尺度刀（能定位的是冰山一角）**：Golden Gate Claude 能漂亮定位的是**鮮明、具體**的特徵——一座橋、一種明顯情緒。但對「模型這次為什麼幻覺、為什麼被這個提示越獄、走了哪條推理路徑」這種**真正關鍵**的問題，目前能看懂的仍是內部**極小一塊**。從「能指認金門大橋」到「能完整讀懂模型每一個決定」之間，差了不知道多少個數量級。「能定位一個概念」≠「能讀懂模型」。

2. **證據刀（方法本身正在被質疑）**：別把一個漂亮 demo 當成領域成熟。⚠️（2025–26）有嚴格評測指出 SAE 在大規模任務上**並沒有穩定打敗簡單基線**，Google DeepMind 的可解釋性團隊據報還**調降了 SAE 的優先級**。一個其核心工具正在被同行質疑、優先級被調降的領域，離「已經解決」非常遠。這是一個**還在劇烈試錯**的早期前沿，不是收工的成熟技術。

3. **校準刀（別翻案翻過頭）**：但也別因為上面兩刀就全盤否定——Golden Gate Claude **確實**第一次在前沿模型上證明了「內部有可指認、可干預的概念表徵」，這是真實且重要的進展，把可解釋性從哲學思辨變成了可操作的工程。正確姿態是「**這是一個有真實突破、但遠未成熟、且方法仍在被推翻的早期領域**」，而不是非黑即白的「已經解決」或「全是噱頭」。

常見誤解：一種是看到漂亮 demo 就以為領域成熟（忽略尺度與方法質疑）；另一種是聽到「SAE 被質疑」就全盤否定可解釋性（忽略 Golden Gate Claude 的真實突破）。兩種都沒讀懂——關鍵永遠是**校準成熟度**：有真進展，但離終點極遠。

### 推演題3 — 找破綻：「某家高調談安全，所以它不是最安全（就是公關）、就是在自我設限拖慢自己」 **[12 分鐘]** ★★★

群組裡兩派吵起來。A 說：「某家天天談安全，根本是公關話術、搶道德高地。」B 說：「就算是真的，那它把安全擺這麼前面，純粹是自我設限、拖慢自己，活該被速度派超車。」請你指出**這兩種說法各自的破綻**，並給出一個更誠實的讀法。

#### 推演解答

兩派都把一個**取捨**讀成了**一個簡單的對錯判斷**，方向相反但都太草率。

**A 的破綻（「就是公關」）**：A 假設「有對外講安全」=「假的、只是公關」。但這個結論**無法從『有講』推出來**——這正是 ch18/本章反覆講的「難證偽」問題。「我很重視安全」確實是難證偽的主張，但難證偽**不等於假**。要判斷真假，A 該用的判準是：這家的安全投入**短期賺不賺錢**？它**有沒有真的讓出速度／能力**作為代價（ch07 的容量代價就是一個真讓出速度的具體證據）？光憑「它有講」就斷定「是公關」，跟光憑「它有講」就相信「很安全」，是**同一種**懶惰——都跳過了驗證。

**B 的破綻（「純粹自我設限拖慢自己」）**：B 假設安全與能力是乾淨的此消彼長——投安全＝純損失速度。但 ch07 給過反例：把對齊當第一性原則，**外溢**成了 coding/agentic 的可靠度。某些安全工作（讓模型更聽話、更可控、更少胡說、更可審計）**本身就是能力**，尤其在企業／高風險產品上，可審計性是**賣點與門檻**而非純成本。所以「投安全＝活該被超車」這個推論也站不住——它忽略了「安全即能力」的維度，也忽略了不同產品取向下安全的商業價值不同。

**更誠實的讀法**：安全不是一個可以最大化的單一數值，是一組**互相拉扯的取捨**。正確的問題不是「這家安不安全／是不是公關」，而是：**它在『安全 vs 速度／開放』光譜上站哪裡？這個位置讓它強在哪、暴露在哪？** 把安全擺前面的家，換來企業／高風險場景的信任與某些「安全即能力」的外溢，代價是讓出一點速度身位；速度派換來迭代快，代價是更多治理爭議。**沒有絕對的對錯，只有取捨的位置與它的影子。** 而且這層懷疑要對**所有**家一視同仁——包括你直覺上比較信任的那家。

常見誤解：把安全當成一條可以排名的單軸（「誰最安全」）。安全是多風險維度的拉扯（濫用風險 vs 集中／黑箱風險、能力 vs 可控…），不同家**量的是不同的風險**——這正是開源派與閉源派能用「開放」這同一件事、各自講出一套安全論述的原因。

## 自我檢核

口頭自答，講不清楚就是還沒懂：

1. 為什麼說「連造它的人都不全懂」一個訓練好的模型？把「沒有源碼可讀」這件事跟 ch02 的「養出來而非寫出來」接起來講——並用「疊加／多義性」和「湧現」各講一句，說明看不懂的**結構性根源**在哪。
2. 可解釋性為什麼被叫「模型的神經科學」？SAE/單義性（2024）和電路追蹤（約 2025-03）各解決了上一步的什麼問題？用你對黑箱系統做 observability 的經驗（觀測→定位→干預）把這兩者對應一遍。
3. **Golden Gate Claude 這個實驗，為什麼一次證明了三件事**（可指認、可干預、接上安全）？它最了不起的是哪一步——為什麼「能干預」比「能觀測」更關鍵？
4. 為什麼說「可解釋性是**整個產業**共同在做的研究領域、不是某一家的專利」？（提示：OpenAI/GPT-4、Google DeepMind/Gemma 2 都做過 SAE）這跟 ch07 給人的印象差在哪、為什麼要特別澄清？
5. red-teaming 和 evals 跟可解釋性差在哪（黑箱外部 vs 想把箱子變透明）？為什麼「跑過了所有安全 evals」**遠不等於**「安全」？用你「測試只能證明 bug 存在、不能證明 bug 不存在」的直覺講。
6. **為什麼開源派和閉源派能用『開放』這同一件事、各自講出一套安全論述？** 它們各自量的是什麼風險？這怎麼證明「安全不是一個可以排名的單軸」？
7. 「可解釋性已經解決了」這句話過度推銷在哪？怎麼**既肯定 Golden Gate Claude 的真實突破、又指出它離成熟極遠**？（提示：SAE 被質疑、Google DeepMind 調降優先級）「校準成熟度」和「非黑即白」差在哪？
8. 面對一家高調談安全的公司，為什麼「就是公關」和「很安全」是**同一種懶惰**？判斷它是真投入還是行銷，正確的判準是什麼（短期賺不賺錢、有沒有真讓出速度）？為什麼這層懷疑要對**所有**家一視同仁？

## 延伸閱讀

- **Templeton et al., "Scaling Monosemanticity"（Anthropic, 2024，從 Claude 3 Sonnet 抽特徵）** — https://transformer-circuits.pub/ ——「模型的神經科學」與 **Golden Gate Claude** 的源頭論文。讀它怎麼用稀疏自編碼器把糊在一起的神經元拆成數百萬個可解讀「特徵」、又怎麼靠「夾住一個特徵」做出 Golden Gate Claude——本章招牌個案的一手出處（Anthropic 個別故事見 ch07）。
- **Anthropic, circuit tracing / attribution graphs（約 2025-03）** — https://transformer-circuits.pub/2025/july-update/index.html ——可解釋性的下一步：不只找特徵，而是追蹤資訊怎麼一步步流過模型、畫出電路圖。讀它對「我們連自己造的東西都看不全懂」這件事的**誠實面對**，體會這個領域的野心與謙卑。
- **A Survey on Sparse Autoencoders（arXiv:2503.05613）** — https://arxiv.org/html/2503.05613v3 ——SAE 的**產業級綜述**。讀它印證「這不是一家的事」：OpenAI、Google DeepMind、學界都在跑 SAE，以及這條路線當前的能耐與**侷限**——本章「別過度推銷成熟度」那段的依據。
- **"Sparse Autoencoders Find Highly Interpretable Features in Language Models"（OpenReview）** — https://openreview.net/forum?id=F76bwRSLeK ——可解釋性社群的代表性論文之一。讀它怎麼從「神經元是一團糊（多義性）」這個問題出發，建立「為什麼需要 SAE 去拆特徵」的直覺。
- **OpenAI/Anthropic 的 SAE 研究對照整理（Arize AI）** — https://arize.com/blog/llm-interpretability-and-sparse-autoencoders-openai-anthropic/ ——把 OpenAI 與 Anthropic 在 SAE 上的工作放一起對照的可信整理。讀它建立「可解釋性是**多家共推**的研究戰線」這個本章核心立場。
- **Kaplan et al. 2020 / Chinchilla（接回 ch02）** — https://arxiv.org/abs/2001.08361 ——回頭讀 scaling laws，理解「能力沿一條曲線**長**出來」這件事——正是它讓模型「沒有源碼可讀」、進而**需要**事後的可解釋性。把「為什麼造出看不懂的東西」這個本章前提，接回「模型是養出來的」這個全書地基。
