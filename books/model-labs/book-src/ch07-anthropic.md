# ch07 — Anthropic：對齊優先、可解釋性與 agentic/coding 口碑

> **本章解決什麼問題**：上一章拆完 Google 的「算力·基礎設施」主導性格。這一章換 Anthropic，回答的是一條反直覺的因果：一家把**安全與對齊當第一性原則**的實驗室，為什麼反而長出了「聽話、好用、適合 agent/coding」的口碑？本章的招牌因果鏈只有一條，但會講到底——**Constitutional AI 用「給憲法→自我批評→自我修正」的迴路，把對齊變成可規模化的工程，而這套「照規範自我修正」的訓練取向，外溢成了 coding/agentic 任務上的可靠度**。對齊方法的跨家譜系（RLHF→Constitutional AI→RLVR）留給 ch15；可解釋性與安全的跨家全貌留給 ch19。本章只負責 Anthropic 這一家，與其他七家同等對待、不當全書主角。

## 從你已知的出發

你在團隊推過一件事，回報遠超它的技術含量：**把規範寫成一份 lint／規則檔，讓 CI 自動把關**。在那之前，「程式碼風格該怎樣、哪些寫法不准」是靠資深的人在 code review 裡逐條挑——這很慢、很主觀、而且**不可規模化**：團隊一變大，人工 review 就成了瓶頸，挑到的東西還因人而異。你做的事是把「我們團隊認為對的寫法」抽成一份明文的規則檔，然後讓 CI 在每個 PR 上自動跑。從此把關這件事**不再依賴某個資深工程師當下有沒有空、心情好不好**——規則寫在檔案裡，機器照著執行，整個團隊踩著同一條底線進場。

這裡有個關鍵轉折，值得你停一下：你不是把「人」換成「機器」，你是把**「人的判斷」先固化成一份文件，再讓自動化去執行那份文件**。規則檔本身是人寫的、是人的價值判斷；CI 只是不知疲倦地照著它跑。這個「**把判斷寫成憲法，再讓系統照憲法自動把關**」的形狀，正是本章主角 Anthropic 最招牌的方法——Constitutional AI（憲法式 AI）——的精神。差別只在，他們把關的不是程式碼風格，是**模型的行為**：哪些回答算有害、哪些算誠實、哪些算有幫助。傳統做法是請一大群人逐條標註「這個回答好不好」（人工 code review 的等級），Anthropic 的做法是先寫一份「憲法」，讓**模型照著憲法自己批評、自己修正自己的回答**（CI 自動把關的等級）。

本章要拆清楚兩件事：第一，這套「對齊」的方法到底怎麼運作；第二——也是更反直覺的——**為什麼這種對安全的執著，最後反而養出了 coding 與 agentic 任務上的好口碑**。

## 以什麼著稱：coding/agentic 可靠度，與對齊·可解釋性的研究底（截至 2026-06）

先講「是什麼」，標好日期戳，再回頭問「為什麼」。

截至 2026-06，Anthropic 的 Claude 系列以兩組東西著稱，要分開講，因為它們屬於不同層次：

- **🟡 reputation 層（最招牌的口碑）**：**coding 與 agentic 任務的可靠度**。具體的旗艦產品面是 **Claude Code**（在終端機裡把寫程式的工作整段委派給模型）與 computer-use（讓模型操作電腦）。口碑集中在「**長程多步驟的自主性**」（long-horizon multi-step autonomy）與「**指令遵循**」（instruction-following）——也就是：給它一個橫跨很多步、要它自己規劃、用工具、檢查自己成果的髒任務，它跑得相對穩。
- **✅ 研究底層（可被引用的硬事實）**：**Constitutional AI / RLAIF**（Bai et al., 2022-12，arXiv:2212.08073），以及一條真實的**可解釋性（interpretability）研究線**——被形容為「**模型的神經科學**」。

當前的模型版本，要分清楚「已穩定出貨」與「非常新、還在變動」：

- ✅ **Claude Opus 4.8**（2026-05-28 發表）——定位在複雜推理、長程 agentic coding、高自主性工作的頂層。
- ✅ **Claude Sonnet 4.6**（2026-02-17）——coding 接近 Opus，但指令遵循與工具可靠度更好；✅ **Claude Haiku 4.5**（2025-10-15）——最快、最便宜的一層。
- 🟡 **Claude Fable 5 ＋ Claude Mythos 5**（2026-06-09 發表，號稱「Mythos-class」、Anthropic 至今最強）。Fable 5 是帶安全護欄的公開版；Mythos 5 是同一個模型、拆掉部分護欄、限制給 **Project Glasswing**（先是資安專業人士，後續才開放給部分生物研究者）。

## 招牌方法：Constitutional AI 的「給憲法→自我批評→自我修正」迴路

這是本章要講透的因果個案。先把這套方法**整條鏈拆開**，因為它就是 Anthropic 性格的源頭。

傳統的對齊主流是 **RLHF**（人類回饋強化學習，源頭是 OpenAI 2022-03 的 InstructGPT，見 ch05/ch15）：請大量真人去比較「A 回答跟 B 回答哪個比較好／比較無害」，用這些人類偏好訓練一個 reward model（獎勵模型），再用它去 RL 那個語言模型。問題出在哪？出在**「請大量真人」這一步不可規模化、又昂貴、又不一致**——這正是你做 lint/CI 之前，那種「靠資深工程師逐條人工 code review」的痛，只是放大到「要請人去讀海量可能有害的內容並打分」的尺度。

Constitutional AI（Anthropic, 2022-12）的核心改動，就是把「人逐條打分」換成「**模型照一份明文憲法自評**」。它分兩段，分別用到對齊的兩道基本工序，先用一兩句把這兩個詞講清楚（你後面會反覆碰到）：**SFT（監督式微調，supervised fine-tuning）**＝直接給模型「標準的輸入→輸出」配對，讓它照著模仿；**RL（強化學習，reinforcement learning）**＝讓模型自己生出多個版本，再由一套評分機制給獎懲，逼它往高分的方向調整。（完整對齊方法譜系見 ch15。）有了這兩個詞，下面這兩段對照你的 CI 直覺會很好懂：

```
第一段 SL-CAI（監督式：自我批評 → 自我修正）
  給模型一個（可能會誘出有害回答的）提問
        |  模型先給出初版回答
        v
  「拿一條憲法原則當尺，批評自己剛剛的回答哪裡違規」
        |  自我批評（self-critique）
        v
  「照這條批評，把回答改寫成更符合原則的版本」
        |  自我修正（self-revision），反覆換不同原則多輪
        v
  收集「修正後的好回答」當教材，去微調模型（SFT）

第二段 RL-CAI / RLAIF（用 AI 回饋取代人類回饋）
  讓上一段微調後的模型，對同一個提問生成兩個回答
        |  另一個「裁判模型」照一條憲法原則挑「哪個比較無害」
        v
  這些「AI 給的偏好標註」訓練出 preference model（取代人工標註）
        |  preference model 當 reward
        v
  用它去 RL 主模型 → 成品（對齊度高、且幾乎沒用到人工逐條標註）
```

把它對回你的世界：**憲法 = 那份 lint/規則檔（人寫的價值判斷，固化成明文）；自我批評→自我修正 = CI 在每個 PR 上自動跑規則、把不合規的地方標出來並要求改；RLAIF = 你不再請資深工程師逐條 review，而是讓自動化照規則檔把關**。人類的角色從「逐條標註每一個回答」往上移了一層——變成「**寫好那份憲法**」（截至 2026 的報導，這份憲法據稱已從 2023 年的約 2,700 字，長到約 23,000 字）。人還在，但人做的是「定規範」這件高槓桿的事，不是「逐條執行」這件不可規模化的事。這就是 RLAIF（AI 回饋強化學習）省掉人工標註的真正意義。

這條鏈，就是 Anthropic 整個性格的種子。下一節要回答的，是這顆種子怎麼長成「coding/agentic 好用」這個看似不相干的果實。

## 為什麼對齊投入會外溢成「聽話、好用、適合 agent/coding」

這是本章最反直覺、也最值得講透的一段。直覺上，「重視安全」聽起來像是會讓模型**更綁手綁腳、更愛拒絕、更難用**——怎麼會反而長出 coding/agentic 的好口碑？關鍵在於：**你想讓模型「安全」，必須先讓它具備的那些底層能力，恰好就是 agent/coding 最吃的能力**。一格一格看這個外溢：

- **「照規範自我修正」≈ coding/agentic 最核心的迴圈**。Constitutional AI 訓練模型做的事是：給一條原則 → 檢查自己的輸出有沒有違反 → 不對就改。把「原則」換成「這段程式要通過的測試／這個任務的需求」，這**就是**寫程式與跑 agent 的核心迴圈：產出 → 自我檢查 → 修正 → 再檢查。一個被大量訓練成「會拿一把尺檢查自己、發現不對會回頭改」的模型，天生就比較適合「給它一個多步驟任務、讓它自己規劃、用工具、檢查成果」的 agentic 場景。**對齊訓練順手鍛鍊出來的「自我審視與修正」肌肉，剛好是 agent 的核心肌肉。**

- **「指令遵循」是對齊與好用的共同底座**。要讓模型「安全」，第一件事是它得**真的聽懂、並照著你給的原則做**——一個連指令都遵循不好的模型，你連「別產生有害內容」都叫不動它。所以對齊投入會大量強化「指令遵循」（instruction-following）這個能力。而「指令遵循好」恰恰就是「好用、聽話」的白話定義，也是 agentic 任務的命脈：agent 要跨很多步不跑偏，靠的就是每一步都穩穩照指令走。**為了安全而鍛鍊的指令遵循，直接兌現成『聽話好用』。**

- **長程任務的「不跑偏」與安全的「不出事」是同一種可靠度**。一個會在第 30 步突然忘記目標、亂用工具、或自信地胡謅的模型，既是 agentic 的災難，也是安全的災難。Anthropic 把「在長程、高自主的情境下仍然穩」當成第一性的追求——這個追求**不分它是為了安全還是為了好用**，因為在長程任務裡，這兩者是同一件事。這就是為什麼 Claude 的口碑集中在「long-horizon multi-step autonomy」：那正是安全執念和 agentic 需求**指向同一個目標**的地方。

這就是本章那條被講透的因果鏈：

```
把安全/對齊當第一性原則
      |
      v  方法選擇：Constitutional AI（照憲法自我批評→自我修正）
訓練出「會拿一把尺檢查自己、發現不對就回頭改」的模型
      |
      v  這套「自我審視+修正+強指令遵循」能力
      v  恰好是 agent/coding 最核心的肌肉
外溢：長程多步驟自主、指令遵循、工具可靠度的口碑
      |
      v  產品取向把它收成開發者面的產品
Claude Code / computer-use（terminal-delegated coding、操作電腦）
```

注意這條鏈的方向：**不是「Anthropic 想做 coding 模型，所以去做了」，而是「Anthropic 為了安全去鍛鍊某些能力，這些能力外溢成了 coding/agentic 的可靠度」**。強項是對齊執念的**副產品**，不是直接瞄準的靶。這就是「模型是養出來的、不是寫出來的」（ch02）在這一家身上最漂亮的演示：你設定一個「凡事先求安全、求可控、求能照規範自評」的成長環境，**長出來的產物**之一，竟是一個特別適合委派髒活的 agent。

## 另一條招牌：可解釋性研究——「模型的神經科學」

Anthropic 的第二個研究招牌，要單獨講，因為它是一種**罕見的研究文化品味**，而不是一個產品功能。

訓練完的模型權重，有一個外行很難相信的事實：**連造出它的人，都不完全看得懂它內部在算什麼**（這個橫切面的全貌是 ch19 的主題，本章只講 Anthropic 這一家在做什麼）。一個訓練好的模型就是幾千億個浮點數，沒有人逐行寫過「遇到 X 就回 Y」——能力是長出來的，所以也沒有現成的「原始碼」可讀。Anthropic 投了一支相當深的團隊，專門做一件事：**把這個黑箱當成大腦，拿探針去研究它的內部結構**。這就是為什麼它被形容為「模型的神經科學」。

為什麼非得做到這一步？因為前面那套 RLHF／Constitutional AI 能塑形的，終究只是模型的**表面行為**——回覆禮不禮貌、拒不拒絕有害請求。但它**測不出模型內部有沒有作弊**：模型可能學會了 reward hacking（鑽獎勵機制的漏洞拿高分，而非真的做對），或是偽裝對齊——表面上順從、討好（sycophancy），骨子裡卻在隱蔽地欺瞞。你只看輸出，分不出「真的對齊」和「學會了演得像對齊」。而當你要把一個模型放進**長程 agent 任務**（讓它自己跑幾十步、用工具、改檔案），這個分不出來就致命了：你最怕的正是它「表面聽話、暗地亂來」。要排除這種風險，光看行為不夠，你被逼著必須**打開黑箱、看模型內部到底在想什麼**——這就是把「安全」當第一性原則時，邏輯上必然推導出來的下一步。可解釋性不是錦上添花的學術興趣，是「要對長程自主的模型有信心」這個需求，逼出來的科學投入。下面兩個里程碑，就是這條投入路上的成果。

兩個可被引用的里程碑（✅）：

- **稀疏自編碼器 / 單義性（sparse autoencoders / monosemanticity，2024）**——Templeton et al. 2024 從 Claude 3 Sonnet 裡，用一種叫「字典學習」的技巧，把混在一起的神經元活化，拆解成一個個**可解讀的「特徵」（feature）**（例如某個特徵專門對「金門大橋」有反應）。這解決的是「模型內部一個神經元同時管很多無關概念（多義性）」這個讓人看不懂的問題。
- **電路追蹤 / 歸因圖（circuit tracing / attribution graphs，約 2025-03）**——再進一步，不只找出「有哪些特徵」，而是追蹤**這些特徵之間怎麼連、資訊怎麼一步步流過模型**，畫出類似電路圖的東西。

對你來說，這件事有一個非常貼切的橋接：**這就是對黑箱系統做 observability**。你維運過你沒法逐行讀懂的系統——你不會因為讀不懂就放棄，你會插探針、加 trace、拉 metrics，從外部行為反推內部狀態（你那條自建 FluentBit pipeline 做的就是這件事）。可解釋性研究是同一種精神，只是被戳的對象是一個幾千億參數、誰也沒寫過原始碼的神經網路。為什麼這算「研究文化品味」（五變數第四格）而不是隨便哪家都會做的事？因為它**短期內幾乎不直接賺錢**——它不會讓下一版模型在 benchmark 上多幾分。願意長期養一支「看不懂就想辦法看懂」的團隊，本身就是一種價值取向上的押注，而這種押注**最難被抄走**：你可以下載別人的開放權重，但你抄不走一個累積多年的研究品味與團隊。

> ⚠️ 一個誠實的但書（接回 ch19）：可解釋性還很早期，目前能看懂的是模型內部極小一塊，遠不是「已經能完整讀懂模型」。而且「安全」與「可解釋性」也可能被任何一家當成行銷話術——本書的態度是講方法、講取捨，不替任何一家的安全敘事背書。

## 算力·基礎設施：刻意的 multi-cloud（不自研晶片、也不主要靠 Nvidia）

講完研究與方法，補上算力這格，因為它的形狀很特別、也直接連到本章的影子。

截至 2026-06，Anthropic 是一家**刻意 multi-cloud 的前沿實驗室**，而且它押注的主力，**既不是自研晶片、也不是主要靠 Nvidia GPU**：

- ✅ **AWS 是主力**。**Project Rainier**（AWS，印第安納州）約 50 萬顆 **Trainium2** 晶片；Anthropic 使用超過 **100 萬顆 Trainium2**；擴大後的合約規模達到「最多 **5GW** 的 Trainium 算力」＋ 十年期 **1,000 億美元**的 AWS 承諾。
- ✅ **同時也用 Google TPU**（2025-10 的合約，規模達「最多 100 萬顆 TPU」、2026 進入多 GW 等級）。

把這兩條放在一起，Anthropic 的算力身份就清楚了：它**橫跨 AWS Trainium ＋ Google TPU**。對照前兩章——OpenAI 深綁 Azure（ch05）、Google 自研 TPU 垂直整合（ch06）——Anthropic 走的是第三條路：**不自己做晶片（不像 Google），也不把賭注主要押在 Nvidia 那條最擁擠的路上，而是綁在兩家雲廠商的自研／加速晶片上**。這背後是一種風險分散的品味（不被單一供應商鎖死），但代價是：它沒有 Google 那種「模型工程師和晶片工程師同桌設計」的最底層垂直整合紅利——它的晶片，終究是 AWS 和 Google 設計的，不是它自己的。

## 成因拆解

### (a) 五格歸因

| 變數 | 角色 | 為什麼 |
|---|---|---|

一句話收束：**Claude 強在 coding/agentic 可靠度與對齊·可解釋性的研究底，因為〔研究文化·品味（安全為第一性、可解釋性執念）＋後訓練·對齊（Constitutional AI 的自我修正迴路外溢成 agentic 能力）〕兩格主導；算力（multi-cloud）與產品取向（偏開發者）是次因。影子是——安全取向的速度/開放代價，以及相對對手的規模/算力天花板與容量晃動。**

### (b) 影子

同一組讓它強的選擇，也讓它暴露在對應的弱點上：

- **安全取向的速度與開放代價。** 凡事先過安全與對齊這一關，意味著**多一道工序、慢一點出手**；對齊得越緊，模型有時也越容易「過度拒絕」（該答的也婉拒），這是把安全擺第一的結構性副作用。更明顯的是**開放性的代價**：Anthropic 的前沿模型是封閉權重的——你下載不到、改不了、不能 on-prem 自己跑（開源 vs 閉源的兩種世界觀見 ch13）。而最強的 Mythos 5 甚至**刻意限制存取**（只給 Project Glasswing 的資安專業人士、後續部分生物研究者）——這正是「把安全擺第一」最直接的開放代價：能力越強，它越**不**讓你自由取用。
- **規模與算力相對對手的天花板。** 比起 Google（自研 TPU 垂直整合）、OpenAI（Azure＋Stargate 5,000 億）、xAI（自建 Colossus）、Meta（自有大量 GPU），Anthropic **不擁有自己的晶片、也沒有自己的超大自建叢集**——它的算力是租來的（AWS Trainium ＋ Google TPU）。這是一種風險分散，但也意味著它在「能堆多大、堆多快」這條軸上，結構性地受制於雲廠商的供給與排期，缺少 Google 那種最底層的協同設計紅利。
- **容量晃動（capacity wobble）。** 這個影子在 2026-06 就具體應驗了：最新的 **Fable 5 在 2026-06-23 從訂閱方案被下架**、改成用量計費，官方理由是**容量限制、不是安全事件**。一家算力靠租、又把資源大量投在安全與可解釋性研究上的實驗室，在「最新最強模型的供給穩定度」上，會比手握自研晶片或超大自建叢集的對手更吃緊。**把錢和算力投向安全的同一個選擇，也讓它在尖峰供給上更容易捉襟見肘。**

### (c) 何時不成立

這個「強項」在以下情況會被打臉，提醒你這是**截至 2026-06 的快照、不是定律**：

- **換任務軸**：「coding/agentic 可靠度」是 reputation，不是全能冠軍。截至 2026-06，OpenAI 同樣以 agentic coding 著稱、超長上下文掛在 Google、低成本開放權重掛在 DeepSeek——「誰最強」要分任務，沒有單一冠軍（ch04 已立此規矩）。把「以 coding 著稱」偷換成「coding 最強」就踩線了。
- **被便宜化／開放權重夾擊**：當 DeepSeek、Qwen 等用開放權重＋極低價格逼近「夠用」的 agentic 品質時，Anthropic「強但封閉且不便宜」的位置會被擠壓——這跟 ch05 OpenAI 面對的下方夾擊是同一種結構性壓力。
- **被時間推翻**：點版本號、排行榜名次每幾個月洗牌；本章所有版本（Opus 4.8、Fable 5/Mythos 5）與容量晃動（Fable 5 在 2026-06-23 下架）都是 2026-06 的切片，尤其 Fable 5/Mythos 5 是**非常新、極易過期**的事實，三個月後就該重掃。

## 紙上推演

### 推演題1 [12 分鐘] ★★（口頭重講因果）

用你自己的話，對另一個工程師講清楚：為什麼一家**把安全當第一性原則**的實驗室，反而會養出「coding/agentic 好用」的口碑？這中間的因果是什麼？這個強項是「直接瞄準的靶」還是「副產品」？

#### 推演解答

關鍵是看出「安全所需的能力」和「agent 所需的能力」**高度重疊**。要讓模型安全，你得先讓它（1）真的聽懂並照著給定的原則做（指令遵循）、（2）會拿一把尺檢查自己的輸出、發現不對會回頭改（自我批評→自我修正，這正是 Constitutional AI 訓練的核心動作）、（3）在長程、高自主的情境下不跑偏。而這三件事——強指令遵循、自我審視與修正的迴圈、長程不跑偏——**恰恰就是 coding 與 agentic 任務最吃的核心肌肉**。所以對齊投入會**外溢**：為了安全鍛鍊的能力，順手兌現成了 agentic 的可靠度。重點措辭：這個強項是**副產品**，不是直接瞄準的靶——Anthropic 不是「想做 coding 模型所以去做」，而是「為了安全去鍛鍊某些能力，這些能力外溢成 coding 口碑」。常見誤解是把「重視安全」直覺地讀成「綁手綁腳、愛拒絕、難用」；那個直覺只看到「過度拒絕」這個影子，沒看到「安全所需的底層能力恰好是好用的底層能力」這條主線。這對應五變數的研究文化·品味（把安全當第一性）與後訓練·對齊（Constitutional AI）兩格主導。

### 推演題2 [15 分鐘] ★★（翻譯：把現象對應回五變數）

把「Anthropic 願意長期養一支做可解釋性（『模型的神經科學』）的團隊」這個現象，試著歸到五變數的某一格，並說明為什麼是那一格、而不是別格。再回答：這格的東西為什麼「最難被抄走」？

#### 推演解答

主要歸到第四格**研究文化·品味**。判斷依據：可解釋性研究**短期內幾乎不直接賺錢、也不會讓下一版模型在 benchmark 上多幾分**——它不是產品取向（第五格）驅動的（沒有要服務的產品在逼它做），也不是算力（第三格）或資料（第一格）能解釋的；願意把資源投到一個「短期沒回報、純粹想搞懂黑箱」的方向，這是一種**價值取向上的押注**，正是「研究文化·品味」這格的本體。為什麼最難被抄走？因為前面幾格的東西多半可擴散：架構是公開論文、訓練技巧會被複現、開放權重可以直接下載。但「品味」是一個團隊累積多年的方向判斷力與價值取向——你抄得到別人**做出來的結果**（甚至下載權重），卻抄不走「**為什麼這家當初敢長期押這個短期不賺錢的方向**」。常見誤解是把可解釋性歸到「產品取向」或「安全行銷」；可解釋性確實能餵養安全敘事，但它的根是研究品味，而且本書提醒過（接 ch19）：可解釋性還很早期、「安全」也可能是行銷，所以歸因時要看「**這個投入短期賺不賺錢**」這個判準，而不是看它的對外說法。

### 推演題3 [12 分鐘] ★★（對帳：你的系統取捨 ↔ Anthropic 的取捨）

把你推「lint/規則檔＋CI 自動把關」這件事，和 Anthropic 的 Constitutional AI 對帳：兩者在「把人的判斷固化成文件、再讓自動化執行」這個結構上哪裡同構？又有哪裡**不一樣**（別把類比推過頭）？

#### 推演解答

**同構處**：兩者都是把「原本靠資深人力逐條把關」這件不可規模化的事，往上抽一層——先把判斷固化成一份**明文規範**（你的 lint 規則檔 ≈ Anthropic 的憲法），再讓**自動化去執行那份規範**（CI 自動跑 ≈ RLAIF 讓模型照憲法自我批評/修正/互評）。人的角色都從「逐條執行」上移到「定規範」這件高槓桿的事——這正是為什麼 Constitutional AI 能省掉大量人工標註（RLAIF 的意義）。**不一樣、不能推過頭的地方**：你的 CI 是**確定性**的——同一份程式碼、同一條規則，CI 每次給的結果一模一樣，規則命中與否是黑白分明的。Constitutional AI 裡負責「執行憲法」的是**另一個語言模型**——它對「哪個回答比較無害」的判斷是**機率性、會出錯、本身也需要被對齊的**。也就是說，你的把關者（CI）是可靠的死規則，Anthropic 的把關者（裁判模型）是一個**也可能犯錯、也可能被鑽漏洞的模型**。這個差異很關鍵：它就是為什麼「對齊」永遠不是一勞永逸的——你用一個模型去對齊另一個模型，等於用一個會犯錯的把關者去把關（這也是 ch15 對齊演化、ch19 安全會回收的張力）。常見誤解是把 RLAIF 想成「全自動、不需要人了」；其實人退到了「寫憲法」這層，而且裁判模型本身的可靠度，仍是整套方法的軟肋。

## 自我檢核

講得出來才算過關；講不清楚就是還沒懂。

1. **為什麼**「把安全當第一性原則」反而會外溢成「coding/agentic 好用」？把那條因果鏈（安全所需能力 ↔ agent 所需能力的重疊）講一遍，並說清楚這個強項是「副產品」還是「直接瞄準的靶」。
2. Constitutional AI 的「給憲法→自我批評→自我修正」迴路，**如果**拿掉「憲法」這份明文規範，會變回什麼？（提示：對回你做 lint/CI 之前的人工 code review）為什麼說 RLAIF 的真正意義是「把人的角色從逐條標註上移到寫憲法」？
3. 為什麼說「可解釋性研究」要歸到五變數的**研究文化·品味**這格，而不是產品取向？判準是什麼？（提示：這個投入短期賺不賺錢）為什麼這格「最難被抄走」？
4. Anthropic 的算力走「刻意 multi-cloud（AWS Trainium＋Google TPU）」這條路，和 Google 自研 TPU 垂直整合（ch06）相比，**少了**什麼紅利？**如果不這樣**、改成自研晶片或自建超大叢集，它的影子會怎麼變？
5. 「容量晃動」（Fable 5 在 2026-06-23 被下架）這個影子，為什麼和「把資源大量投向安全與可解釋性、算力又靠租」這組選擇是同一件事的兩面？
6. 你用 CI 把關程式碼，和 Anthropic 用一個「裁判模型」照憲法把關回答，最關鍵的差別在哪？為什麼這個差別意味著「對齊不是一勞永逸的」？
7. **如果不**用「截至 2026-06／以…著稱」的句式，而直接寫「Claude 是最強的 coding 模型」，會出什麼問題？OpenAI 同樣以 agentic coding 著稱這件事，為什麼正好說明「以…著稱＋日期戳」的講法更站得住？
