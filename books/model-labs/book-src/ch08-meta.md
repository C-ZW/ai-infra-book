# ch08 — Meta：開放權重的締造者，與 2026 的轉向

> **本章解決什麼問題**：我們繼續拿著五變數這把尺逐家拆解。前三家（OpenAI、Google、Anthropic）都是閉源前沿的代表；Meta 不一樣——它曾是**開放權重最大的推手**（Llama），把模型權重免費送出去，養出整個產業最龐大的生態系。但 2026-04，它做了一件全書最戲劇化的事：**轉向閉源前沿**（Muse Spark）。本章回答兩個「為什麼」——為什麼是 Meta 締造了開放權重生態，以及為什麼是 Meta 在 2026 親手把它收掉。ch01 預告過的那句「送出權重＝送出部分護城河」，在這一章不再是抽象命題，而是真實發生的事件。開源 vs 閉源的哲學全貌留給 ch13。

## 從你已知的出發

你做過一件很多後端工程師都做過、也都後悔過一半的事：把一個內部好用的東西，開放給全公司用。

具體一點——你寫的那套**多層 cache library**。一開始它只是你自己服務裡的一個模組，解決你自己的問題。後來別的團隊看到了，問「能不能也用」，你把它抽成共用套件，發到公司內部的 registry，寫好 README，全團隊採用。那一刻你得到的東西很實在：**它成了標準**。新專案預設就接你這套，別人遇到 cache 問題第一個想到你，你在團隊裡的技術影響力，是靠這個套件的採用率撐起來的——採用率本身變成了一種護城河。

但你大概也嚐過另一半。當十幾個團隊都依賴你這套 library，**維護成本就不再是你一個人的事，是十幾個團隊的事**。有人用法刁鑽，有人卡在你沒料到的 edge case，有人基於它做了一整條產品線、然後反過來要求你不能隨便改 API。某天你會冒出一個念頭：當初要是沒開放，我現在是不是更自由？要不要把核心收回來、只留一個閹割版給外面用？

把「cache library」換成「模型權重」，把「全公司」換成「全世界的開發者」，這就是 Meta 從 2023 到 2026 走過的整條弧線。它用 Llama 的開放權重，做了一件當時沒有任何前沿大廠敢做的事——**把養好的模型免費送出去**，結果締造了整個產業最大的生態系。然後在 2026，當維護成本、競爭代價、變現誘惑同時超過了開放的回報，它做了和你那個念頭一模一樣的決定：**把前沿收回來**。本章就是要拆清楚這兩個決定各自的成因，記在五變數的哪幾格上——而它最特別的地方在於，**它是全書唯一一家，你能在同一章裡，看著它的強項與強項的影子同時上演**。

## 招牌：它締造了開放權重的生態系

先講事實，標好日期戳。截至 2026-06，Meta 在 LLM 產業裡的歷史定位非常清楚：**它是開放權重路線最大的締造者**。從 2023 年的 Llama 開始，Meta 反覆做一件當時違反產業直覺的事——把訓練好的模型**權重直接公開、讓人下載**。不是開個 API 讓你呼叫，是把養好的模型本體交到你手上，你可以下載、可以改、可以在自己的機器上跑、可以拿去微調成你要的樣子。

這件事的份量，要從「權重」到底是什麼講起。回想 ch02 的核心隱喻：模型是**養**出來的，不是**寫**出來的。那養成的結果——那條花了海量資料與算力長出來的能力——全部濃縮在一組數字裡，就是**權重（weights）**。閉源廠商（OpenAI、Google、Anthropic）把權重鎖在自己的伺服器後面，你只能透過 API 租用它的能力，碰不到本體。Meta 做的事，等於把這組「養成的成果」整包送出去。在你的工程直覺裡，這大概等同於：別人只給你一個 SaaS 端點，Meta 直接把整個服務的 Docker image ＋ 所有權重檔丟給你，說「拿去，自己部署，隨便改」。

這個決定養出了什麼？一個**飛輪**，而且是和 ch05 OpenAI 那個「規模即訊號」飛輪結構不同、但同樣強大的飛輪：

```text
免費送出權重（Llama 系列）
   │  開發者下載 → on-prem 部署、微調成領域模型
   ▼
龐大的衍生生態：成千上萬的 fine-tune、量化版、工具鏈
   │  大學、新創、企業都在 Llama 上做研究與產品
   ▼
Llama 變成「開放權重」的事實標準與通用語
   │  整個社群的工具、教學、最佳實踐都繞著它長
   ▼
生態採用率本身變成護城河：不是模型最強，是「大家都在它上面開發」
```

你應該一眼就認出這個形狀——它和你那套 cache library 成為團隊標準的路徑**同構**。護城河不是「Llama 是 benchmark 最強的模型」（它通常不是，截至 2026-06 前沿榜首一向在閉源那幾家之間洗牌），而是「**整個開放權重社群的預設選擇是它**」。研究者要做實驗、新創要做產品、企業要 on-prem 一個自己能掌控的模型，第一個拿起來的就是 Llama。這種「**人人都在你的地基上蓋房子**」的影響力，是 API 租用模式買不到的。

這裡放一個帶日期戳的具體例子，也是本章第一條被講透的因果鏈。截至 landscape 記錄的 2025-04，Meta 推出 **Llama 4「herd」**——第一代原生多模態、第一代採用 **MoE（混合專家）** 的 Llama，包含 **Scout**（17B active、16 個專家、宣稱可達 1,000 萬 token 上下文）與 **Maverick**（17B active、128 個專家）。注意這裡的數字讀法：MoE 的「17B active」是說**每個 token 實際只走 17B 參數的路徑**，但模型總參數遠大於此——這正是 ch01 提過的 MoE 稀疏設計（總參數巨大、實際算的少），讓「巨大的模型」能用「可負擔的推論成本」跑起來。Meta 把這樣一個原生多模態、可在自家硬體上跑的模型，**權重公開**——對整個開放權重社群來說，這等於又一次免費發了一套高階地基。

## 但「開放」從來都帶著繩子

這裡要立刻打掉一個常見的浪漫想像，因為它是理解 2026 轉向的關鍵前提：**Llama 的「開放」從來就不是純粹的開源（open source）。**

本書的用語規範區分得很清楚（見 style-guide）：「**開放權重（open-weight）**」是說你能下載權重、能拿去用；「**開源（open source）**」在嚴格定義（OSI）下，意味著連訓練程式碼、訓練資料、授權都自由無限制。Llama 一直只是前者，而且連這個「前者」都帶著繩子。截至 landscape 記錄，**Llama 4 社群授權（Community License）** 至少綁了兩條：

- **700M MAU 天花板**：如果你的產品在 Llama 4 發布前一個月的月活躍使用者超過 **7 億**，你不能直接拿去商用，必須**另外向 Meta 申請授權**，而且是否給、由 Meta「全權裁量」。這條看似冷僻，其實精準地指向 Meta 真正提防的對象——只有極少數巨頭（包括它的直接競爭者）會撞到 7 億這條線。
- **EU 地區限制**：發布當下，**法律註冊地在歐盟（EU）的組織被禁止使用**這批多模態模型。Meta 給的理由是 EU AI Act 帶來的「法規不確定性」。

把這兩條放在一起看，你會發現一件很有工程感的事：**這份授權不是「自由軟體」的精神，是一份精心設計的、帶條件的開放**。它的邏輯和你開放 cache library 時心裡的盤算其實一樣——「我願意讓大多數人免費用，因為採用率對我有利；但我會在條款裡留一手，防著那些用我的東西來打我的人」。所謂「開放」，從第一天起就是一個**有策略的選擇**，不是一個信仰。理解了這一點，2026 的轉向就不會顯得突兀——它不是從「理想主義」墮落到「現實主義」，而是同一套**現實主義的成本計算**，在輸入變了之後，得出了相反的結論。

## 2026 的轉向：把前沿收回來

現在講全書最戲劇化的那一幕。把它讀成一個**轉折、而非定局**——這是 landscape 反覆叮囑的（這是整份 landscape 裡標記為「最新鮮、最易變」的實驗室故事）。

轉向分兩步。**第一步是擱置。** 截至 landscape 記錄，Llama 4 最大的那一隻——**Behemoth**，約 **2T（兩兆）參數** 等級、定位為「教師模型」——在 2025-04 發布時是「訓練中（in training）」，但之後**實質上被擱置了，從未公開釋出**。原因據報是在 2T 這個規模上踩到了工程的牆：**MoE 的路由（routing）** 在訓練中途出了問題、干擾了專家的專業化分工，**chunked-attention（分塊注意力）** 在分塊邊界留下盲點、傷害了長篇推理——Meta 對「把它做出來、收益是否值得」失去了信心。它的準確狀態是「**announced, not released**（宣布了、沒釋出）」，而非「已推出」。

> ⚠️ Behemoth 的「擱置」沒有正式宣布取消，是「預告了卻一直沒釋出」的狀態（截至 2026-06）；2T 規模的 MoE 路由與 chunked-attention 問題是據報的技術原因，不是 Meta 官方完整覆盤。版本與細節照 landscape，標日期戳。

**第二步，也是真正的轉向：閉源前沿。** 截至 landscape 記錄的 **2026-04-08**，Meta 推出 **Muse Spark**——它的**第一款專有、閉源權重的前沿模型**，出自新成立的 **Meta Superintelligence Labs（MSL，超智慧實驗室）**，由 **Alexandr Wang**（Scale AI 共同創辦人，Meta 為此投資與招攬，由他出任首席 AI 官）領軍。關鍵的一句話：**你下載不到 Muse Spark 的權重。** 它只透過 Meta 自家 app 與受限的 API 預覽提供，現在驅動著 Meta AI app。這一步，結束了 Meta **長達七年「在前沿維持開放權重」的身分**。

把這兩步放在一起，你就看懂了這條因果鏈的完整形狀：

```text
七年「前沿開放權重」身分（Llama 線）
   │  前沿訓練成本暴漲 ＋ 對手（含中國廠商）靠 Llama 搭便車做產品
   │  ＋ 3B 用戶平台（FB／IG／WhatsApp）＋ 專有使用者資料的變現拉力
   ▼
策略反轉：前沿那一檔，權重不再公開
   │  Behemoth（~2T）擱置 → 不再用「最大的開放模型」當旗艦
   ▼
Muse Spark（2026-04，MSL／Alexandr Wang，閉源）→ 驅動 Meta AI app
   │
   ▼
ch01 預告的影子應驗：送出權重＝送出部分護城河，於是把控制權收回來
```

為什麼會反轉？landscape 給了三條成因，每一條都對應你那套 cache library 的某個煩惱：

- **前沿訓練成本暴漲**。養一個 2T 等級的前沿模型，成本已經高到「免費送出去」變成一件難以承受的事。Meta 2026 的資本支出（capex）被引述在 **約 1,150–1,350 億美元** 的量級（⚠️ 這是估計、爭議數字，不同來源差異大）。當每一代模型都要燒掉這個量級的錢，「養好就免費送」的慷慨，對股東就越來越難交代——就像當你那套 library 的維護吃掉你一半的時間，你會開始算「這個開放到底值不值」。
- **怕對手靠 Llama 搭便車**。開放權重最尷尬的一點是：**你的對手也能下載**。landscape 點名 Meta 擔心**中國廠商直接拿 Llama 做商用產品**——你花了上百億美元養出來的能力，被別人零成本拿去做成跟你競爭的東西。那條 700M MAU 條款，就是這個恐懼的早期版本；2026 的轉向，是這個恐懼升級成了戰略決定。對應你的處境：你最怕的不是別的團隊用你的 cache library，是某個團隊**基於它做了一條跟你打對台的產品線**。
- **3B 用戶平台 ＋ 專有資料的變現拉力**。Meta 手上有別家沒有的東西——**Facebook、Instagram、WhatsApp 合計約 30 億用戶**的平台，以及這些平台上獨有的專有使用者資料。把前沿模型閉源、綁進這些 app（Muse Spark 現在驅動 Meta AI app），用平台與專有資料變現，是一條開放權重給不了的路。當你有自己的「產品」可以餵、可以變現，「把模型送出去當公共財」的吸引力自然下降——這是五變數第五格（產品取向）在說話：**Meta 終於決定，模型要服務的是自己的產品，不是整個開放社群。**

## 為什麼是 Meta？用五變數層層回答

招牌與轉向都講完了，現在回答「為什麼這整條弧線會發生在 Meta 身上」。一格一格過。

**產品取向（第五格，主因）**——這是貫穿前後兩個決定的同一條主軸，所以它是主因。先看開放這一段：Meta 不是雲廠商（沒有像 Azure、Google Cloud、阿里雲那樣靠賣 API 算力賺錢的動機），也不是純研究機構。它的本業是**社群平台與廣告**，模型對它而言一開始**不是要拿去賣的產品**——所以「把模型免費送出去、用開放生態換產業話語權與人才吸引力」對 Meta 是划算的，反正它不靠賣模型賺錢。但這個邏輯在 2026 翻轉了：當 Meta 決定要用前沿模型**驅動自家 app、靠 30 億用戶平台與專有資料變現**，模型第一次變成了「**要服務自家產品的核心資產**」——而核心資產你不會免費送人。同一格（產品取向），輸入變了（從「不靠模型賺錢」變成「要靠模型驅動產品變現」），輸出就從「開放」反轉成「閉源」。**這正是五變數框架最漂亮的地方：不是 Meta 變心了，是它要服務的產品形狀變了，於是模型被壓出了相反的形狀。**

**算力·基礎設施（第三格，次因）**——Meta 是世界上最大的 GPU 買家之一。截至 landscape 記錄，它在 Llama 訓練年代公開的目標是約 **35 萬顆 Nvidia H100**（⚠️ 這是 2024 年代的數字，且為目標／估計值，非當前精確量），2026 的 capex 量級已如前述。這格是次因不是主因，理由和 ch05 OpenAI 一樣：**充沛的算力是「能養出 Llama、也能養出 Muse Spark」的必要條件，但它不是「為什麼 Meta 選擇開放、又選擇轉向」的原因**。算力決定你做不做得到，產品取向決定你想不想做、用什麼形狀做。

**研究文化·品味（第四格，次因，且正在鬆動）**——Meta 的 AI 研究有一個獨特的底色：**FAIR（Facebook AI Research）的開放發表傳統**。在 Yann LeCun 的影響下，FAIR 長期信奉「開放研究、開放發表」的學術文化，這是 Llama 願意開放權重的文化土壤之一。但 2026 的轉向，本身就是這個文化在**鬆動**的訊號：成立一個全新的、與舊 FAIR 文化不同的 **Meta Superintelligence Labs**，由外來的 Alexandr Wang 領軍、產出閉源模型——這是組織層面的「換血」，是品味這一格正在被產品壓力重新塑形的證據。所以這格記次因，且要標明它**正在改變**。

**資料（第一格，次因）**——這格在轉向後變得重要。開放權重時代，Meta 的資料優勢不突出；但轉向閉源、要靠 app 變現之後，**FB／IG／WhatsApp 上的專有使用者資料** 成了它區別於閉源對手的潛在武器（landscape 提到 Muse Spark 明確倚賴專有使用者資料）。它是次因，因為它是被第五格（產品取向轉向）**啟動**的——先有「要用自家平台變現」的決定，專有資料這格才被點亮。

**後訓練·對齊（第二格，留白／不相關）**——本章的因果核心是「開放 vs 閉源」這個戰略選擇，不是某種對齊方法的高下。Meta 當然也做後訓練，但它**不是這條弧線的主因或次因**，這格在本章留白（對齊方法的跨家譜系見 ch15）。

## 成因拆解

### (a) 五格歸因

| 變數 | 角色 | 為什麼 |
|---|---|---|
| 資料 | △ 次因 | 轉向後，FB／IG／WhatsApp 的**專有使用者資料**成為閉源變現的武器（Muse Spark 倚賴它）；但它是被「產品取向轉向」點亮的**結果**。 |
| 後訓練·對齊 | — | 本章核心是「開放 vs 閉源」的戰略選擇，不是對齊方法高下；這格不相關（→ ch15）。 |
| 算力·基礎設施 | △ 次因 | ~35 萬顆 H100（⚠️ 2024 年代目標值）、2026 capex ~1,150–1,350 億（⚠️ 爭議）撐住養模型；是「能做到」的必要條件，非「為什麼選這條路」。 |
| 研究文化·品味 | △ 次因（**鬆動中**） | FAIR 開放發表傳統是 Llama 開放的文化土壤；但成立 MSL、由 Alexandr Wang 領軍產出閉源模型，是這格**正在被產品壓力重塑**的證據。 |
| 產品取向 | ✓ **主因** | 貫穿前後的同一主軸：**不靠賣模型賺錢時 → 開放換生態話語權**；**要用 30 億用戶平台與專有資料變現時 → 閉源收回控制權**。同一格、輸入變了、輸出反轉。 |

一句話收束：**Meta 強在「締造了開放權重生態」，因為產品取向（不靠賣模型、用開放換生態與人才）＋算力（自有大量 GPU）；但 2026 它把前沿收回閉源（Muse Spark），同樣因為產品取向（改用 30 億用戶平台與專有資料變現）。影子是——送出權重＝送出部分護城河，這個影子在 2026 應驗了。**

### (b) 影子

這一章的影子最特別：**它不是「將來可能發生」的代價，而是「已經上演」的事件**。ch01 預告過「護城河搬了家、送出去的就收不回浪漫」，這裡是它的真實版本。

- **送出權重＝送出部分護城河（已應驗）**。開放權重最強的地方——人人能下載、能改、能基於它做產品——也正是它最致命的地方：**你的對手同樣能下載**。Meta 花上百億美元養出來的能力，被全世界（包括它最提防的競爭者）零成本拿走。當「別人靠我的地基蓋房子」從「擴大我的影響力」變成「養肥我的對手」，這條護城河就從資產變成了負債。2026 的轉向，就是 Meta 承認了這個影子、並試圖把它堵起來。**這是全書唯一一個「強項的影子在書寫期間就應驗成真」的個案**——也是為什麼這一章被選來把 ch01 的抽象命題演成真實事件。
- **轉向後的信任落差與社群反彈**。當一家以「開放」立身、靠開放贏得整個社群好感與人才吸引力的公司，突然把前沿收回閉源，它面對的是一種**特殊的信任損失**：用了你七年、把工具鏈與最佳實踐全繞著 Llama 長出來的開放社群，會覺得被背叛。這種反彈不是 benchmark 分數能補回來的——**開放的紅利吃在前面，轉向的代價也記在這裡**。對帳你那套 cache library：當你某天宣布核心要閉源、只留閹割版給外面用，最先跳腳的，正是那些當初最信任你、把產品壓在你套件上的團隊。
- **閉源前沿能否追上，仍未定**。Meta 在開放權重時代是無可爭議的締造者，但在閉源前沿這條新賽道上，它是**後進者**，要正面對撞早已深耕的 OpenAI、Google、Anthropic。一個很誠實的訊號：截至 2026-06，連 Alexandr Wang 自己都把 Muse Spark 形容為「**還不在領先前沿模型的層級**」、稱它是一道「**開胃菜（appetizer）**」（⚠️ 個人公開說法，非正式 benchmark 定位）。換句話說，這個轉向**賭的是未來**，現在還遠遠談不上成功——這正是為什麼 landscape 要我們把它寫成「**2026 轉折、非定局**」。

### (c) 何時不成立

提醒你這是**截至 2026-06 的快照、不是定律**：

- **轉向可能不徹底，甚至回擺**。Meta 並沒有宣布永遠不再出開放權重模型。完全有可能它走「兩條腿」——前沿閉源（Muse Spark 線）、同時繼續釋出較小的開放權重模型（就像 Google 一邊閉源 Gemini、一邊開放 Gemma，這種並行策略 ch13 會拆）。本章描述的是**前沿那一檔**的轉向，不是 Meta 從此和開放權重一刀兩斷。
- **「締造者」是歷史定位，不是當前排名**。說 Meta 締造了開放權重生態，是可考的歷史事實；但這**不等於** Meta 當前在任何 benchmark 上領先。截至 2026-06，前沿榜首在閉源那幾家之間洗牌，開放權重的當前話語權也有 DeepSeek、Qwen、Mistral 在競逐（見 ch11、ch12、ch09）——「最強」要分任務、分時間，沒有單一冠軍（ch04 已立此規矩）。
- **被時間推翻**。Muse Spark 的閉源轉向是 2026-04 才發生的事，是整份 landscape 標記為**最新鮮、最易變**的故事。三個月後就該重掃：轉向有沒有「黏住」？有沒有新的開放 Llama 出現？Muse Spark 在獨立 benchmark 上的真實定位如何？本章所有數字（2T Behemoth、35 萬 H100、1,150–1,350 億 capex、30 億用戶）都是切片，會過期。

## 紙上推演

### 推演題1 [10 分鐘] ★

用你自己的話，對另一個工程師講清楚這條因果鏈：為什麼「把模型權重免費送出去」這個看起來像在做慈善的決定，反而能成為 Meta 的護城河？護城河到底是哪個東西？

#### 推演解答

關鍵和 ch05 一樣——**把護城河和「最強模型」分開**。Llama 通常不是 benchmark 榜首（前沿那幾家是閉源的），所以護城河絕不是「我的模型最強」。它是這個飛輪：免費送出權重 → 開發者下載、on-prem、微調 → 長出成千上萬的衍生模型與工具鏈 → 整個開放權重社群的預設選擇變成 Llama → **採用率本身成為護城河**。護城河是「**全世界都在你的地基上蓋房子**」這件事，對應五變數的產品取向（主因）。常見誤解是把開放權重讀成「做慈善／理想主義」；其實它是一筆**精算過的策略投資**——Meta 不靠賣模型賺錢，所以用免費權重換生態話語權、人才吸引力與產業標準地位，對它划算。對帳你自己的系統：你把 cache library 開放給全團隊，得到的不是「最好的程式」這個名聲，是「**大家預設都接你這套**」這個結構性影響力——這就是採用率即護城河。

### 推演題2 [15 分鐘] ★★★（翻譯＋反事實）

2026-04，Meta 把前沿模型從開放（Llama）轉成閉源（Muse Spark）。請做兩件事：(1) 把「開放→閉源」這個反轉**翻譯**回五變數，指出主因落在哪一格、為什麼是「同一格的反轉」而不是「換了主因」；(2) **反事實**：如果 Meta 像 Google 一樣本身就是個靠賣雲端 API 賺錢的公司，這個轉向還會發生嗎？

#### 推演解答

**(1) 翻譯。** 主因是第五格**產品取向**，而且關鍵洞察是——**前後兩個相反的決定，是同一格驅動的**。開放階段：Meta 不靠賣模型賺錢（本業是社群平台與廣告），所以「免費送模型換生態話語權」划算。閉源階段：Meta 決定要用前沿模型驅動自家 app、靠 30 億用戶平台與專有資料變現，模型第一次變成「要服務自家產品的核心資產」，核心資產不會免費送人。**不是換了主因，是同一格（產品取向）的輸入變了（從『不靠模型賺錢』變成『要靠模型驅動產品變現』），輸出就從開放反轉成閉源。** 這題的教訓是五變數框架的精髓：**模型的形狀（開放或閉源）是被它要服務的產品反過來壓出來的**。

**(2) 反事實。** 如果 Meta 本來就是靠賣雲端 API 賺錢的公司（像 Google Cloud、阿里雲），那它**從一開始可能就不會那麼大方地開放前沿權重**——因為開放會直接侵蝕它賣 API 的生意。換句話說，**正是「不靠賣模型賺錢」這個前提，才讓 Meta 成為開放權重最大的推手**；而 2026 的轉向，恰恰是這個前提開始改變（要用平台變現）的結果。所以這個轉向的根，不在「Meta 變壞了」，在「Meta 的變現模式變了」。常見誤解是把它讀成理想主義的墮落；它其實是**同一套現實主義計算在不同輸入下的不同輸出**。

### 推演題3 [12 分鐘] ★★（找破綻）

有人說：「Llama 是開源的，所以它完全自由、沒有任何使用限制，這就是開放權重的精神。」請指出這句話裡的兩個破綻。

#### 推演解答

**破綻一：「開放權重」不等於「開源」。** 本書用語嚴格區分：**開放權重（open-weight）** 是「你能下載權重、能用」；**開源（open source）** 在 OSI 嚴格定義下要連訓練程式碼、資料、授權都自由。Llama 一直只是前者，它**沒有**公開訓練資料與完整訓練流程，所以把它叫「開源」本身就不精確——除非連訓練碼都開，否則嚴格說只能叫「開放權重」、不能叫「開源模型」。**破綻二：「完全自由、沒有任何使用限制」是錯的。** Llama 4 社群授權至少綁了兩條繩子（截至 landscape 記錄）：產品月活躍使用者超過 **7 億** 必須另外向 Meta 申請授權（由 Meta 全權裁量），以及發布當下**禁止法律註冊地在歐盟的組織使用**這批多模態模型。所以「開放」從第一天起就是**有條件、有策略的開放**，不是無限制的自由——而正是這種「精算過的開放」，才讓 2026 的閉源轉向顯得順理成章，而非突兀的背叛。正確講法是：「Llama 是**帶授權限制的開放權重**模型，截至 2026-06 Meta 已把前沿那一檔（Muse Spark）轉為閉源。」

## 自我檢核

講得出來才算過關；講不清楚就是還沒懂。

1. **為什麼**「把模型權重免費送出去」對 Meta 是一筆划算的策略投資，而對 Google、阿里雲這類靠賣 API 算力賺錢的公司可能就不划算？（提示：本業變現模式）
2. 「開放權重」和「開源」差在哪？為什麼說 Llama 叫「開源」並不精確？它的授權綁了哪兩條繩子？
3. 用一句話講清楚 2026 轉向的核心：為什麼說「開放→閉源」這個反轉**不是換了主因，而是同一格（產品取向）的輸入變了」**？
4. ch01 預告過「送出權重＝送出部分護城河」。**為什麼**這個影子會在 2026 對 Meta 應驗？「別人靠我的地基蓋房子」在什麼時候從資產變成負債？
5. **如果**當初 Llama 沒有那條「700M MAU 必須另外申請授權」的限制，會少防到誰？這條冷僻條款其實在提防什麼？
6. Meta 在開放權重時代是「締造者」，但在閉源前沿這條新賽道上是「後進者」。**為什麼**這讓「閉源前沿能否追上 OpenAI／Google」仍是未定之數？（提示：Alexandr Wang 自己怎麼形容 Muse Spark？）
7. **如果不**把這一章寫成「2026 轉折、非定局」，而直接斷言「Meta 已經放棄開放權重了」，會出什麼問題？為什麼這個轉向特別需要「轉折、非定局」這層但書？
