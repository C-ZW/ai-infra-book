# ch16 — 投影與最小平方：當方程式無解時

> **本章解決什麼問題**：前一章（ch15）把內積（inner product／dot product）找回來，於是「長度」「角度」「正交」都變回可以算的代數。本章把這套工具用在一個你天天遇到、但線代課很少老實講的情境上——**方程式無解的時候，怎麼辦？** 不是丟一句「無解」就走人，而是**誠實地給出最接近的答案**。這就是投影（projection）與最小平方（least squares）：把一個夠不到的目標，垂直地投到你夠得到的子空間上，取那個最近的點。本章是 Part V「正交與最佳近似」的正中央——前面 ch15 給了正交這把尺，後面 ch17（正交基與 QR）會給更穩的算法、ch18（對稱與譜定理）會把投影矩陣的結構講透。本章的驚嘆點只有一個，但它很大：**我們會解一個無解的方程，而且解得理直氣壯。** QR 解最小平方在數值上的優勢留 ch17；最小平方的統計意義（為什麼它是「最好」的估計、高斯—馬可夫定理）留《馴服隨機》；正則化（regularization）只在最後點一句、指向延伸。

開始前先把全書一律遵守的台灣慣例釘死一次：**行（直行，column）是矩陣縱向的一排、列（橫列，row）是橫向的一排**——跟中國大陸相反（見 landscape 與 ch05）。本章的主角之一是「行空間（column space）」，指的是矩陣那些**直行**張成（span）的空間；請一路把「行」讀成 column。

## 從你已知的出發

你已經做過最小平方，只是沒人告訴你那是線性代數。

**dashboard 上那條趨勢線。** 你盯著一張延遲（latency）監控圖，散落著上百個量測點——抖動、雜訊、偶爾的尖刺。你拉一條趨勢線（trend line／regression line）穿過去，想看「整體是往上還是往下」。那條線幾乎不會精準通過任何一個點，但它「最不偏」——它讓所有點到線的偏差平方加總最小。**那就是最小平方回歸，一字不差。** 你做這件事的時候在做的，是把「一堆互相矛盾的資料」壓成「一條最委屈但最公道的直線」。本章要告訴你：這條線是怎麼從一個**無解的方程組**裡，被線性代數逼出來的——而且它是**唯一**最近解，不是亂湊的近似。

**約束太多、互相打架，求一個最不委屈的折衷。** 你設計過這種系統：十條約束（constraint），但只有三個旋鈕（自由度）。十條約束塞不進三個旋鈕——這個系統**過約束（over-constrained）**，嚴格意義上「無解」。但你不會兩手一攤，你會找一組旋鈕設定，讓「被違反的約束的總懲罰最小」。這正是**超定系統（overdetermined system）的最小平方解**：方程比未知多、湊不出精準解，於是退而求「最接近滿足所有方程」的那一組。投影就是這件事的幾何版——把一個夠不到的目標，正交地投到你夠得到的範圍裡。

**「再也擠不出資訊」的訊號。** 這個直覺最關鍵，記住它。最小平方解的標誌是：**殘差（residual，也就是沒被解釋掉的偏差）垂直於你的模型能表達的所有方向。** 殘差一旦正交（orthogonal）於你的子空間，就代表「我已經把資料裡所有跟我的模型對得上的成分全榨乾了，剩下的這點偏差，跟我的任何一個旋鈕都不相關，再怎麼調都擠不出來」。垂直＝榨乾＝最佳。如果殘差還跟某個旋鈕方向有夾角（內積不為 0），那就表示「往那個方向再推一點還能更好」，你就還沒到最佳。**正交是「我盡力了」的代數證據。** 這句話是本章的脊樑，後面每一步都在把它說清楚。

把三個直覺收成一句帶進本章：**當你想要的東西不在你能到達的範圍內，最誠實的回應不是放棄，而是投影——垂直地落到最近的可達點上，讓殘差正交於你能做的一切。** 這就是投影與最小平方。

## 投影：子空間裡離你最近的那個點

先把「投影」這件事的幾何講乾淨，從最小的情形——投影到一條線——開始，因為所有更高維的投影都是它的放大版。

### 把一個點投到一條線上

給一條過原點的線，方向是單位向量 û（暫且設 ‖û‖=1，長度為 1）。再給平面上任意一個點 b（這裡 b 是個向量）。問：**這條線上，離 b 最近的點是哪個？**

直覺先行：你拿一支筆，從 b 往線上垂直放下去，落腳的那個「垂足（foot of perpendicular）」就是最近點。為什麼是垂足而不是別的點？因為從 b 到線上任一點的距離，是直角三角形的斜邊；斜邊永遠比直角邊長；而**垂足對應的那條連線，正好是唯一的直角邊**（它垂直於線），所以它最短。「最近」與「垂直」是同一件事的兩面——這是畢氏定理（Pythagorean theorem）的直接後果（ch15 用內積把畢氏找回來了）。

把它算出來。最近點 p 一定在線上，所以 p = c·û（某個純量 c 倍的 û）。關鍵條件是**殘差 b − p 垂直於線**，也就是垂直於 û：

```text
(b − p) ⊥ û
⟺ (b − c·û) · û = 0           ← 正交＝內積為 0（ch15）
⟺ b·û − c·(û·û) = 0
⟺ b·û − c·1 = 0               ← û 是單位向量，û·û = ‖û‖² = 1
⟺ c = b·û
```

所以 **p = (b·û)·û**。投影量 c＝b 在 û 上的內積，這正是 ch15 講的「內積就是投影長」——現在我們把那個投影長乘回方向，得到投影點本身。一句話：**b 在一條線上的投影，是「b 跟這條線方向的內積」乘以那個方向。** 殘差 b − p 則是 b 垂直於線的那一截，它扛著「這條線解釋不了的部分」。

（如果線的方向向量 a 不是單位長，公式就帶上歸一化：p =（(b·a)/(a·a)）·a。分母 a·a＝‖a‖² 把長度的影響除掉。脊椎那條對角線方向 (1,1) 長度是 √2 不是 1，等一下會用到這個版本。）

### 從一條線到一個子空間

把「線」換成「子空間（subspace）」——可能是平面、可能是更高維的行空間——故事**一模一樣**，只是「垂直於一條線」升級成「垂直於整個子空間」。

設子空間是某個矩陣 A 的**行空間（column space）**：A 的那些直行張成的所有線性組合（ch10）。給一個目標向量 b。投影 p 是行空間裡離 b 最近的點，它一定能寫成 A 的行的某個組合，也就是 **p = A x̂**（x̂ 是組合係數，待求）。最近的條件還是那一句：

```text
殘差 b − p = b − A x̂  垂直於整個行空間
```

「垂直於整個行空間」是什麼意思？行空間是 A 的每一行張出來的，所以「垂直於整個行空間」⟺「垂直於 A 的每一行」⟺「殘差跟 A 的每一行內積都是 0」。把「A 的每一行分別跟殘差做內積」打包成一個矩陣運算，就是 **Aᵀ(b − A x̂) = 0**（Aᵀ 的每一列正好是 A 的每一行，所以 Aᵀ 乘一個向量＝「拿 A 的每一行去跟它做內積」——這是轉置 transpose 最有用的一個讀法）。展開：

```text
Aᵀ(b − A x̂) = 0
Aᵀb − AᵀA x̂ = 0
AᵀA x̂ = Aᵀb              ← 正規方程（normal equations）
```

這條 **AᵀA x̂ = Aᵀb** 叫**正規方程（normal equations）**。名字裡的「normal」不是「正常」的意思，是幾何裡「**法向／垂直**（normal＝perpendicular）」的意思——它整條式子在說的就是「殘差是法向的（垂直於行空間的）」（這個命名出自高斯，2026-06，見下文歷史）。記住這個詞源，你就再也不會把正規方程當成一條要背的怪公式：**它字面上就是「讓殘差垂直」這個幾何條件的代數寫法。**

投影矩陣（projection matrix）也順手寫出來：解出 x̂ =（AᵀA）⁻¹Aᵀb，所以投影點 p = A x̂ = **A(AᵀA)⁻¹Aᵀ · b**。那一坨 P = A(AᵀA)⁻¹Aᵀ 就是把任意 b 打到行空間的**正交投影矩陣**。本章不堆這個一般公式的性質（P²=P、Pᵀ=P 等留 ch18 順帶收），對 2D／簡單情形，我們直接用上一節那條 p =（(b·a)/(a·a)）·a 就夠，不繞 (AᵀA)⁻¹。一般公式放這裡，是讓你知道「投到線上」和「投到子空間」是同一個東西的兩個尺寸。

## 最小平方：解一個無解的方程

現在把投影接到本章的驚嘆點上。

### 為什麼 Ax=b 會無解

回顧 ch07 的行視角（column picture）：解 Ax=b，就是問「能不能用 A 的那幾行（columns），加權湊出 b」。**有解 ⟺ b 落在 A 的行空間裡。** 當方程比未知多（A 又高又瘦，超定系統），行空間通常只是高維空間裡的一個薄薄的子空間，而 b——帶著量測雜訊的真實資料——**幾乎不可能剛好落在那個薄子空間上**。於是 Ax=b 無解。不是你算錯，是 b 真的不在行空間裡，A 的那幾行怎麼組合都湊不出它。

線代課到這裡常常就停了：「無解，結束。」但這在工程上是不可接受的回答。你的趨勢線、你的過約束系統、你的感測器校正——資料永遠帶雜訊，方程永遠對不齊，如果「無解就放棄」，那最小平方這門幾百年的手藝就不存在了。**正確的問法不是「能不能精準湊出 b」，而是「行空間裡哪個點最接近 b」。**

### 把無解變成「最近」

既然 b 不在行空間裡、湊不出來，那就退一步：在行空間裡找一個**最接近 b** 的點 p＝Ax̂，用它代替那個夠不到的 b。這個 p 就是 b 在行空間上的**投影**——上一節剛算過，它由正規方程 **AᵀA x̂ = Aᵀb** 決定。

這就是**最小平方**：

```text
Ax = b   無解（b 不在行空間裡）
   ↓  退而求其次：找 x̂ 使 Ax̂ 最接近 b
解 AᵀA x̂ = Aᵀb   ← 正規方程，一定有解
```

「最接近」用什麼量？用**距離的平方**——殘差 b − Ax̂ 的長度平方 ‖b − Ax̂‖² 最小。為什麼是平方而不是絕對值？因為「最短距離＝垂直」這件事是內積（平方範數）給的（畢氏定理），平方距離才讓「最近」乾淨地等價於「正交」。最小化 ‖b − Ax̂‖² 的那個 x̂，正好就是讓殘差垂直於行空間的那個 x̂——這就是「最小**平方**（least **squares**）」名字的來源，也是為什麼正規方程能給出最佳解。

**這就是驚嘆點，把它講清楚給另一個工程師聽：** 我們面對一個無解的方程 Ax=b，沒有假裝它有解、也沒有兩手一攤，而是**把目標 b 垂直地投影到我們夠得到的行空間上，取那個最近的落腳點，解出湊到落腳點所需的係數 x̂**。x̂ 不是「Ax=b 的解」（那不存在），它是「Ax 最接近 b 的解」——而且因為「最近點」在凸的子空間上**唯一**（垂足只有一個），x̂ 是**唯一**的最佳答案，不是諸多近似裡隨便挑的一個。**我們解了一個無解的方程，給的還是唯一最近解。** 我認為這是整個 Part V 最該讓你「啊哈」的一頁。

### worked example：三個點配一條直線

把它落到具體數字上，從頭算到尾、每一步驗證。這是本章的核心 worked example，也是全書的最小平方基準例（脊椎之外的固定配角，跨章一致）。

**問題。** 三個資料點 (1,1)、(2,2)、(3,2)，想配一條直線 ŷ = a + b·x（a 是截距 intercept、b 是斜率 slope）。三點**不共線**（你可以心算：從 (1,1) 到 (2,2) 斜率是 1，從 (2,2) 到 (3,2) 斜率是 0，兩段斜率不同），所以**沒有任何一條直線能同時穿過三個點**——這就是一個無解的方程組，最小平方上場。

**第一步：寫成 Ax=b。** 「直線通過三個點」要求：

```text
a + b·1 = 1      （點 (1,1)）
a + b·2 = 2      （點 (2,2)）
a + b·3 = 2      （點 (3,2)）
```

未知是 (a, b) 兩個，方程是三條——三條塞進兩個自由度，超定。寫成矩陣（每一列是一個點的方程，第一行全是 1 對應截距 a、第二行是 x 值對應斜率 b）：

```text
A = | 1  1 |        x = | a |        b = | 1 |
    | 1  2 |            | b |            | 2 |
    | 1  3 |                            | 2 |
```

（注意這裡符號有點殘忍：方程組裡的 b 是右邊那個目標向量 (1,2,2)ᵀ，跟直線斜率 b 撞名了。下文凡指目標向量我都寫「向量 b」、凡指斜率我都寫「斜率 b」，請看上下文。這是最小平方文獻的傳統撞名，忍一下。）

這個 Ax=b 無解：b=(1,2,2)ᵀ 不在 A 兩行 (1,1,1)ᵀ 與 (1,2,3)ᵀ 的行空間（那是 ℝ³ 裡的一個平面）上。

**第二步：組正規方程 AᵀA x̂ = Aᵀb。** 先算 AᵀA（Aᵀ 是 2×3，A 是 3×2，乘起來 2×2）：

```text
        | 1  1 |
Aᵀ = | 1 1 1 |   A = | 1  2 |
       | 1 2 3 |       | 1  3 |

AᵀA = | 1·1+1·1+1·1   1·1+1·2+1·3 | = | 3   6 |     ← 左上＝第一行自己的內積＝1²+1²+1²=3
      | 1·1+2·1+3·1   1·1+2·2+3·3 |   | 6  14 |     ← 右下＝第二行自己的內積＝1²+2²+3²=14
```

（讀一下 AᵀA 的意義：它的每個元素是「A 的某一行 · A 的某一行」——AᵀA 就是 A 各行兩兩內積排成的表，所以它必對稱。左上 3＝(1,1,1)·(1,1,1)、右下 14＝(1,2,3)·(1,2,3)、非對角 6＝(1,1,1)·(1,2,3)＝1+2+3。對稱矩陣，預告 ch18。）

再算 Aᵀb：

```text
Aᵀb = | 1 1 1 | | 1 |   | 1+2+2  |   | 5  |     ← (1,1,1)·(1,2,2)=5
      | 1 2 3 | | 2 | = | 1+4+6  | = | 11 |     ← (1,2,3)·(1,2,2)=11
                  | 2 |
```

正規方程：

```text
| 3   6 | | a |   | 5  |
| 6  14 | | b | = | 11 |
```

**第三步：解這個 2×2 系統。** 用 ch08 的 2×2 逆公式或直接消去。先算行列式 det(AᵀA)＝3·14 − 6·6 ＝ 42 − 36 ＝ **6**（非零，所以 AᵀA 可逆、解唯一——只要 A 的行線性獨立，AᵀA 就可逆，這是最小平方有唯一解的條件）。用 2×2 逆 (1/det)|[14,−6],[−6,3]|：

```text
| a |   1   | 14  −6 | | 5  |   1   | 14·5 + (−6)·11 |   1   | 70 − 66 |   1  | 4 |   | 2/3 |
| b | = ─── |        | |    | = ─── |                | = ─── |         | = ─ |   | = |     |
        6   | −6   3 | | 11 |   6   | (−6)·5 + 3·11  |   6   | −30 + 33|   6  | 3 |   | 1/2 |
```

**截距 a = 2/3、斜率 b = 1/2。** 最小平方配適線是 **ŷ = 2/3 + (1/2)x**。

**第四步：代回算殘差、驗殘差和＝0、驗殘差正交。** 深度標準要求自驗，這裡做三重驗算。先算三個配適值（把 x=1,2,3 代進 ŷ=2/3+x/2）：

```text
x=1：ŷ = 2/3 + 1/2 = 4/6 + 3/6 = 7/6      殘差 r₁ = y − ŷ = 1 − 7/6 = −1/6
x=2：ŷ = 2/3 + 1   = 4/6 + 6/6 = 10/6=5/3  殘差 r₂ = 2 − 5/3 = 6/3 − 5/3 = 1/3
x=3：ŷ = 2/3 + 3/2 = 4/6 + 9/6 = 13/6      殘差 r₃ = 2 − 13/6 = 12/6 − 13/6 = −1/6
```

**驗算一（殘差和＝0）：** 含截距項的最小平方，殘差必定加總為零（因為截距那一行全是 1，殘差正交於它就是「殘差 · (1,1,1) = 0」＝殘差和為 0）：

```text
r₁ + r₂ + r₃ = −1/6 + 1/3 − 1/6 = −1/6 + 2/6 − 1/6 = 0   ✓
```

殘差和為 0，這是「配適線不系統性地偏高或偏低」的代數見證——這正是含截距最小平方的招牌性質。

**驗算二（殘差正交於另一行 (1,2,3)）：** 殘差也必須垂直於 A 的第二行（x 值那行），否則「往斜率方向再推一點還能更好」：

```text
殘差 · (1,2,3) = (−1/6)·1 + (1/3)·2 + (−1/6)·3
              = −1/6 + 2/3 − 1/2
              = −1/6 + 4/6 − 3/6 = 0   ✓
```

兩個內積都是 0 ⟺ 殘差 (−1/6, 1/3, −1/6) 垂直於 A 的整個行空間 ⟺ 這就是投影、就是最小平方解。**這兩個 0 就是「我盡力了、再也擠不出資訊」的代數證據。**

**驗算三（投影點真的在行空間上）：** 配適值向量 p = (7/6, 5/3, 13/6) 應該＝Ax̂，且 b = p + 殘差。檢查 p + r =（7/6−1/6, 5/3+1/3, 13/6−1/6）=（1, 2, 2）= 向量 b ✓。b 被乾淨地拆成「行空間裡的投影 p」加「垂直於行空間的殘差 r」——這是 b 相對於行空間的正交分解（orthogonal decomposition），畢氏定理在這裡成立：‖b‖² = ‖p‖² + ‖r‖²（殘差與投影垂直）。

三重驗算全過。**這條 ŷ = 2/3 + (1/2)x 不是亂猜的近似，是唯一讓殘差垂直於模型的最近解。**

### 把它看見：三條垂直殘差

### 一段歷史：高斯、勒讓德，與一場優先權之爭

最小平方法的誕生跟一顆失蹤的小行星有關。1801 年穀神星（Ceres）被發現又隨即被太陽擋住失蹤，高斯（Carl Friedrich Gauss）用一套「讓觀測殘差平方和最小」的方法，從稀少的觀測資料把它的軌道算了出來、預測了它會再出現的位置——而且算準了（2026-06，見 landscape；高斯在 1809 年的《天體運動論》*Theoria Motus* 裡系統性地處理最小平方）。

但**最小平方法的首次發表不是高斯**——是法國數學家**勒讓德（Adrien-Marie Legendre）在 1805 年**，發表在一本關於彗星軌道的書的附錄裡（標題就叫 *Sur la méthode des moindres carrés*，「論最小平方法」，2026-06 查證）。高斯 1809 年發表時聲稱自己 1795 年就在用這套方法了，這讓勒讓德很不爽——他寫信給高斯說「優先權的主張不該在沒有先前發表佐證的情況下提出」，後來公開反駁。**這是數學史上著名的優先權之爭之一**（地位僅次於牛頓對萊布尼茲的微積分之爭，2026-06）。

這段歷史我刻意用謹慎的措辭：**最小平方的「首次發表」歸勒讓德（1805），高斯「宣稱」更早（1795）但無發表佐證**——優先權爭議至今未有定論，寫的時候別替任何一方下判決（landscape 對此標 ⚠️ 提醒措辭）。不過有一個術語確實是高斯留下的、而且正中本章主題：**「正規方程（normal equations）」裡的「normal」就是高斯用的，取「法向／垂直（perpendicular）」之意**——殘差垂直於模型張成的空間（2026-06 查證）。所以你每次寫下 AᵀA x̂ = Aᵀb，都在重複高斯兩百年前那個幾何洞見：**最佳，就是讓殘差變成法向的（normal／垂直的）。**

（迴歸（regression）作為一條趨勢線、它在統計上「估計什麼、為什麼是最好的估計」，是另一本書的主場——見《馴服隨機》談迴歸與趨勢線的那幾章。本章只講線代這一側：投影與最近點。軟連結、不依賴。）

## 直覺的陷阱

最小平方「機械上會算」的人很多，「知道自己在算什麼」的人少。下面四個是你最可能踩的，每個附「怎麼自我察覺」。

| 陷阱 | 錯誤直覺長什麼樣 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| **把「無解」當成「算錯」** | 解 Ax=b 解不出來，以為自己消去消錯、或矩陣輸入錯 | 反覆重算、懷疑工具，卻不知道「無解」是超定系統的常態 | 先數維度：方程比未知多（A 又高又瘦）＋資料帶雜訊 ⟹ b **幾乎一定**不在行空間裡 ⟹ 精準解本來就不存在。這時要做的不是 debug，是改問「最近解」。**無解是物理事實，不是計算錯誤。** |
| **以為最小平方是「亂猜的近似」** | 「反正湊不準，隨便找條差不多的線就好」 | 不信任結果、不知道它有什麼保證；換個方法得不同答案也不警覺 | 最小平方是**唯一最近解**，不是諸多近似裡隨便挑的：行空間是凸的，b 到它的垂足**只有一個**。x̂ 由正規方程**唯一**決定（只要 A 的行獨立）。它有硬保證——殘差正交、距離最小——不是拍腦袋。**最近 ≠ 隨便近。** |
| **忽略「殘差⊥子空間」的幾何** | 只會套 AᵀA x̂=Aᵀb 出數字，問「為什麼是這條式子」答不出來 | 遇到變形（加權最小平方、不同子空間）就不會推，只能背公式 | 正規方程**字面上**就是「殘差垂直於 A 每一行」：Aᵀ(b−Ax̂)=0 ⟺ A 各行跟殘差內積全為 0。記住「normal＝法向＝垂直」，整條式子就從「要背」變成「看就懂」。能自答「為什麼殘差要垂直」（垂直＝最近＝榨乾），你才真懂。 |
| **用正規方程而忽略它數值上不穩** | 「AᵀA x̂=Aᵀb，組出 AᵀA、解掉，收工」 | A 病態（接近奇異）時，**AᵀA 的條件數是 A 的平方**——誤差被放大到平方倍，解可能爛掉（呼應 ch08 條件數、用逆矩陣解方程的罪） | 正規方程在「理論上對、數值上脆」：組 AᵀA 這個動作本身就會把病態放大一輪。徵兆是 AᵀA 的行列式接近 0、或解對輸入微擾極敏感。**正解是不要顯式組 AᵀA，改用 QR 分解解最小平方**——這是 ch17 的主題，也是「不要算逆、要分解」那條鐵律在最小平方上的現身。本章先讓你知道這個雷在哪。 |

把這四個陷阱收成一句你能口頭講的：**最小平方解的是一個「本來就無解」的方程，給的是「唯一」最近解（靠殘差垂直保證），而正規方程雖然是這件事的乾淨代數寫法、卻是數值上該避開的算法（ch17 給穩的）。** 講得出這句，你就跨過了「會算」到「懂」的那道線。

## 紙上推演

### 推演題

**第 1 題 ★ [10 分鐘]——把一個點投影到一條線，算垂足與殘差，驗正交**
把點 b=(3, 1) 投影到方向 a=(1, 1) 的那條過原點直線上（a 不是單位向量，用帶歸一化的公式 p =（(b·a)/(a·a)）·a）。(a) 算投影點（垂足）p；(b) 算殘差 b − p；(c) 驗證殘差 ⊥ a（內積為 0）。

**第 2 題 ★★ [12 分鐘]——過原點直線的最小平方斜率**
兩個資料點 (1, 1)、(2, 3)，配一條**過原點**的直線 ŷ = m·x（只有斜率 m、沒有截距）。(a) 寫成 Ax=b 的形式（A 是 2×1）；(b) 用正規方程 AᵀA m̂ = Aᵀb 解出 m̂；(c) 算兩點的殘差並驗證殘差正交於 A 那一行。（提示：A=[[1],[2]]，這裡 AᵀA 是 1×1 純量，正規方程退化成一條算式。）

**第 3 題 ★★ [8 分鐘]——口頭題**
用你自己的話，向另一個工程師解釋：**最小平方為什麼是「無解時的最近解」而不是「亂猜的近似」？** 要點到（i）為什麼方程無解、（ii）為什麼最近點唯一、（iii）殘差垂直代表什麼。

**第 4 題 ★★★ [12 分鐘]——找出論證的破綻**
某人主張：「既然 Ax=b 無解，那 AᵀA x̂ = Aᵀb 應該也無解——你不過是在兩邊各乘了一個 Aᵀ，無解乘什麼還是無解。」這個結論是錯的（正規方程恰恰**一定有解**）。指出這個論證錯在哪——為什麼「Ax=b 無解」推不出「AᵀA x̂=Aᵀb 無解」。

### 推演解答

**第 1 題。** b=(3,1)、a=(1,1)。

(a) 先算內積：b·a = 3·1 + 1·1 = 4；a·a = 1·1 + 1·1 = 2。投影點：

```text
p = (b·a / a·a)·a = (4/2)·(1,1) = 2·(1,1) = (2, 2)
```

垂足 p=(2,2)。

(b) 殘差 b − p = (3,1) − (2,2) = (1, −1)。

(c) 驗證殘差 ⊥ a：

```text
(b−p)·a = (1)·1 + (−1)·1 = 1 − 1 = 0   ✓
```

殘差 (1,−1) 垂直於方向 (1,1)（它們正是脊椎 S 的兩個特徵方向，互相正交——ch11 算過、ch15 講透）。垂足對了：(2,2) 在線上（是 (1,1) 的 2 倍）、連線 (1,−1) 垂直於線，這就是「最近」。

**第 2 題。** 點 (1,1)、(2,3)，配 ŷ = m·x（過原點）。

(a) 「直線過兩點」要求 m·1 = 1、m·2 = 3，寫成 Ax=b：

```text
A = | 1 |     x = | m |     b = | 1 |
    | 2 |                       | 3 |
```

這無解：一條過原點直線不可能同時滿足 m=1（第一點要斜率 1）和 m=3/2（第二點要斜率 1.5）。

(b) 正規方程 AᵀA m̂ = Aᵀb，這裡 A 是 2×1、AᵀA 是 1×1 純量：

```text
AᵀA = | 1 2 | | 1 | = 1·1 + 2·2 = 5        ← ＝ a·a，A 那一行自己的內積
              | 2 |
Aᵀb = | 1 2 | | 1 | = 1·1 + 2·3 = 7
              | 3 |
5·m̂ = 7  →  m̂ = 7/5 = 1.4
```

最小平方斜率 **m̂ = 7/5**。（合理性檢查：它落在兩點各自要求的斜率 1 與 1.5 之間，偏向資料點較遠的那個——符合「折衷」直覺。）

(c) 殘差：配適值 ŷ₁ = 1.4·1 = 7/5、ŷ₂ = 1.4·2 = 14/5。

```text
r₁ = 1 − 7/5 = −2/5
r₂ = 3 − 14/5 = 15/5 − 14/5 = 1/5
```

驗證殘差正交於 A 那一行 (1,2)：

```text
r·(1,2) = (−2/5)·1 + (1/5)·2 = −2/5 + 2/5 = 0   ✓
```

殘差垂直，最近解確認。（注意：這題**沒有截距項**，所以殘差和 r₁+r₂ = −2/5+1/5 = −1/5 **不為零**——「殘差和為 0」是含截距最小平方的特性，過原點模型沒有那行全 1 的列，就沒這保證。這也提醒你第 4 題那種「殘差必正交於 A 的每一行」才是普適的，殘差和為 0 只是「其中一行剛好是全 1」的特例。）

**第 3 題（口頭題要點）。** 一個好的回答串起三件事：

- **為什麼無解**：方程比未知多（超定）＋資料帶雜訊 ⟹ 目標 b 不在 A 的行空間裡 ⟹ A 的行怎麼組合都湊不出 b ⟹ 精準解不存在（不是算錯）。
- **為什麼最近點唯一**：行空間是個（過原點的）平坦子空間，從 b 放一條垂線下去，垂足只有一個——「最近」由垂直保證，垂足唯一，所以最佳 x̂ 唯一（只要 A 的行獨立）。不是諸多近似裡挑一個，是**唯一**那個。
- **殘差垂直代表什麼**：殘差正交於行空間 ⟺「資料裡所有跟模型對得上的成分都被榨乾了，剩下的偏差跟我的任何旋鈕都不相關，再調也擠不出來」。垂直＝盡力了。如果殘差還跟某方向有夾角，就還能更好、還沒到最佳。

收尾一句：**最小平方不是「猜一條差不多的線」，是「在我夠得到的範圍內，唯一那個離目標最近的點」，而且它帶著硬保證（殘差垂直、距離最小）。**

**第 4 題。** 論證錯在「無解乘什麼還是無解」——這句對**保持方程不變地兩邊乘一個可逆矩陣**才成立，但乘 Aᵀ **不是可逆變換**，它**改變了方程的解集**。

具體說：Ax=b 無解，意思是 b 不在 A 的行空間裡。但乘上 Aᵀ 之後，AᵀA x̂ = Aᵀb 問的是**完全不同的問題**——它不再要求「精準湊出 b」，而是要求「殘差 b−Ax̂ 垂直於行空間」。Aᵀ 這個動作把 b 沿著「垂直於行空間」的方向**壓掉了**（Aᵀ 的零空間正是左零空間，它把垂直於行空間的成分吃掉）：b 不在行空間裡的那一截（殘差），被 Aᵀ 乘掉變成 0，剩下的 Aᵀb 只保留 b 在行空間上的投影資訊。所以 **Aᵀb 一定落在 AᵀA 能到達的範圍裡**，正規方程**必有解**（A 行獨立時還唯一）。

一句話：乘 Aᵀ 不是「無傷大雅地兩邊同乘」，而是「把問題從『精準解』偷換成『最近解』」——它故意丟掉了那截湊不出來的殘差。**無解乘一個會降維的東西，可以變有解**，因為你乘掉的正好是「之所以無解」的那一截。這也再次說明：正規方程不是 Ax=b 的等價變形，是一個**新的、退而求其次的**最佳化問題的解。

### 動手生圖

本章的圖（三個資料點、最小平方配適線 ŷ=2/3+(1/2)x、三條垂直殘差）由以下腳本產生。它同時就是你的小實驗：跑它、改它、重生它。

```python
# ch16 figure: least-squares line through three points (1,1),(2,2),(3,2).
# Fit y_hat = 2/3 + (1/2)x. Three vertical dashed residual segments show each
# point's perpendicular (in the x-axis sense) distance to the line; residuals
# -1/6, +1/3, -1/6 sum to zero. This is "solving an unsolvable equation".
from pathlib import Path
import numpy as np
import matplotlib
matplotlib.use("Agg")          # headless; no display needed
import matplotlib.pyplot as plt

OUT = Path(__file__).resolve().parent / "out" / "ch16-least-squares.svg"
OUT.parent.mkdir(parents=True, exist_ok=True)

xs = np.array([1.0, 2.0, 3.0]); ys = np.array([1.0, 2.0, 2.0])   # the data
a, b = 2.0 / 3.0, 1.0 / 2.0                                       # intercept, slope
fit = lambda x: a + b * x

fig, ax = plt.subplots(figsize=(6, 5))
xl = np.array([0.3, 3.7])
ax.plot(xl, fit(xl), color="#2471a3", lw=2.0, label="fit  y=2/3+(1/2)x")
for x, y in zip(xs, ys):                                          # vertical residuals
    ax.plot([x, x], [y, fit(x)], color="#c0392b", ls="--", lw=1.4)
    ax.annotate(f"r={y-fit(x):+.2f}", (x, (y + fit(x)) / 2),
                color="#c0392b", fontsize=8, xytext=(6, 0),
                textcoords="offset points", va="center")
ax.scatter(xs, ys, color="#222", zorder=5, label="data points")
ax.scatter(xs, fit(xs), facecolors="none", edgecolors="#2471a3", zorder=5, label="fitted")

ax.set_title("Least squares: closest line to 3 non-collinear points", fontsize=10)
ax.set_xlabel("x"); ax.set_ylabel("y")
ax.set_xlim(0, 4); ax.set_ylim(0, 3); ax.grid(True, color="0.9", lw=0.6)
ax.legend(loc="lower right", fontsize=8)
fig.savefig(OUT, bbox_inches="tight")
print("wrote", OUT)            # build_figures.py reads this
```

**預期輸出**：一張寬一點的散點圖。三個黑點 (1,1)、(2,2)、(3,2)；一條藍色配適線 ŷ=2/3+x/2；三條紅色虛線段從每個黑點垂直連到線上的藍色空心圈（配適值）。標註顯示三個殘差 r=−0.17、r=+0.33、r=−0.17（即 −1/6、+1/3、−1/6），加起來為 0。線從左下往右上、斜率 0.5。

**改參數看什麼**（把概念玩活）：

- **加一個離群點看配適線怎麼被拉**：把資料改成四個點，加一個離群值，例如 `xs = np.array([1.,2.,3.,4.])`、`ys = np.array([1.,2.,2.,8.])`（第四點 (4,8) 遠高於趨勢），然後**重算 a、b**——別忘了配適線會被這個離群點**狠狠往上拉**（最小平方對離群值敏感，因為它懲罰的是距離**平方**，遠的點權重特別大）。你可以手動把 `a, b` 改成新解、或直接用 `a, b = np.polyfit(xs, ys, 1)[::-1]` 讓 numpy 算，看那條線如何為了遷就一個離群點而背叛其他三個。這就是「最小平方對離群值不穩健（not robust）」的視覺現身——也是為什麼實務上有人改用最小**絕對**值（懲罰距離不平方）或穩健回歸。
- **把三點改成共線**：把 `ys` 改成 `np.array([1.,2.,3.])`（三點完全共線、斜率 1），你會看到三條紅殘差線**全部消失**（殘差全為 0）——因為這時 Ax=b **真的有解**（b 就在行空間裡），最小平方退化成精確解。投影到一個「目標剛好已經在裡面」的子空間，投影就是它自己。

## 自我檢核

口頭自答；講得出來才算過關，卡住就回到對應段落。

1. **把一個向量投影到一個子空間，是在做什麼？** 在子空間裡找離它**最近**的點（垂足）；最近 ⟺ 殘差垂直於子空間（畢氏：垂直邊比斜邊短）。
2. **正規方程 AᵀA x̂ = Aᵀb 從哪來？「normal」是什麼意思？** 從「殘差 b−Ax̂ 垂直於 A 的每一行」來：Aᵀ(b−Ax̂)=0 展開即得。normal＝法向／垂直（高斯的命名），整條式子就是「殘差垂直」這個幾何條件的代數寫法。
3. **為什麼 Ax=b 會無解？** 超定（方程比未知多）＋資料帶雜訊 ⟹ b 不在 A 的行空間裡 ⟹ A 的行湊不出 b。無解是物理事實，不是算錯。
4. **最小平方為什麼是「無解時的最近解」而不是亂猜？**（本章必答）因為它是 b 到行空間的**正交投影**——垂足唯一（最近由垂直保證）、x̂ 由正規方程唯一決定（A 行獨立時）。它有硬保證（殘差正交、距離最小），不是諸多近似裡隨便挑一個。
5. **「殘差垂直於子空間」在直覺上代表什麼？** 代表「資料裡所有跟模型對得上的成分都榨乾了，剩下的偏差跟任何一個旋鈕都不相關、再調也擠不出來」。垂直＝盡力了；殘差還有夾角＝還能更好。
6. **含截距的最小平方，殘差和為什麼是 0？** 因為截距對應 A 裡全是 1 的那一行，殘差正交於它就是「殘差 · (1,1,…,1) = 0」＝殘差總和為 0。過原點模型沒這行，殘差和不一定為 0（但仍正交於 A 的每一行）。
7. **正規方程理論上對，為什麼數值上要小心？** 組 AᵀA 會把 A 的條件數**平方**，A 一病態，解就可能爛掉（呼應 ch08）。正解是別顯式組 AᵀA，改用 QR 分解（ch17）——「不要算逆、要分解」在最小平方上的現身。
8. **「Ax=b 無解，所以 AᵀA x̂=Aᵀb 也無解」錯在哪？** 乘 Aᵀ 不是可逆變換、會改變解集——它把 b 垂直於行空間的那截（殘差）壓成 0，故意丟掉「之所以無解」的部分，於是新方程必有解。正規方程不是 Ax=b 的等價變形，是一個新的最佳化問題的解。
