# ch17 — 正交基與 QR：好座標的代價

> **本章解決什麼問題**：上一章（ch16）你學會把一個向量投影到子空間、用最小平方在「方程式無解時」誠實地給最近答案；但你大概也注意到了——投影公式、解正規方程，都還是在「解線性方程組」。本章換一個更聰明的座標系，讓這些麻煩消失。核心是一個樸素的奢侈品：**單範正交基底（orthonormal basis）**——一組互相垂直、長度都是 1 的座標軸。在這種座標下，一個向量的座標不必解方程，直接拿它跟各軸做內積就有了（ch04 那個「換基底要解線性組合」的痛點被一筆勾消）；投影也退化成逐項內積相加（ch16 一下變簡單）。**Gram–Schmidt** 是把任意一組獨立向量「扶正」成正交的標準程序——幾何上就是一個一個把新向量身上「已經有的成分」剃掉。把這個程序寫成矩陣，就是 **QR 分解** A=QR。最後也是最實用的一句：QR 解最小平方比 ch16 的正規方程**數值上穩得多**——這就是 ch08 埋下的那句「不要算逆矩陣、要做分解」的正解。本章是 Part V 倒數第二章；對稱矩陣的譜定理留 ch18、把這一切推廣到任意矩陣的 SVD 留 ch19、Householder 等數值 QR 演算法本章只一句指向延伸。

開始前把全書一律遵守的台灣慣例釘死一次：**行（直行，column）是矩陣縱向的一排、列（橫列，row）是橫向的一排**——跟中國大陸相反（見 landscape 與 ch05）。本章常講「把 S 的兩行做 Gram–Schmidt」，那個「行」一律指直行 column，也就是 S 的兩個行向量 (2,1) 與 (1,2)。用詞上我們講「單範正交」（orthonormal，台灣慣例，亦作標準正交）；講「特徵向量」不寫「本徵向量」。

## 從你已知的出發

你每天都在跟「座標軸選得好不好」打交道，只是沒這樣命名它。

**一組「互不洩漏」的乾淨欄位。** 想像你在設計一張監控表，欄位是 cpu、mem、qps、latency。如果某兩個欄位高度相關——比方 qps 一漲 cpu 就跟著漲——那它們其實在重複講同一件事，你看 cpu 的數字時，一部分資訊「洩漏」進了 qps，反之亦然。你會想要一組**互不洩漏**的欄位：每一欄都只報它自己那塊、彼此不重疊。線代裡，「兩個方向互不洩漏」有一個精確的代數判準——**內積為 0，正交**（ch15）。一組互相正交、又各自正規化成長度 1 的座標軸，就是**單範正交基底**：每根軸只管自己的一個維度，彼此乾乾淨淨。這是「去相關（decorrelation）」最純粹的幾何版本，也是 ch20 PCA 要找的東西的雛形。

**embedding 對齊到一組正交方向。** 你知道 cosine similarity 是在問兩個 embedding 向量方向多像（ch02、ch15）。如果你手上的特徵維度彼此糾纏（不正交），算相似度時會被重複計算的成分污染。把特徵「轉到一組正交軸上」再比，每個維度的貢獻才是乾淨、可加的。Gram–Schmidt 做的就是這件事：給你一堆糾纏的方向，**逐個把它們身上跟前面那些軸重疊的成分剃掉**，剩下純粹屬於這個方向的新東西，正規化，得到下一根乾淨的軸。這是「把糾纏的特徵剃成獨立的」的字面操作，PCA 的精神前奏（ch20）。

**解最小平方的「穩定版」。** ch16 你用正規方程 AᵀAx̂=Aᵀb 解了一條迴歸線。能算，但如果你寫過數值敏感的東西，你對「先算 AᵀA 再解」會有種不安——這一步把矩陣的「病態程度」**平方**放大了（等一下會精確講為什麼）。實務上的數值線代庫（LAPACK、NumPy 的 `lstsq`）解最小平方時，預設走的不是正規方程，而是 **QR 分解**（或 SVD，ch19）。QR 就是這一章的主角：它把 A 拆成「一個正交矩陣 Q 乘一個上三角矩陣 R」，解最小平方時誤差只被放大 κ(A) 倍而不是 κ(A)² 倍（2026-06，見後文與延伸閱讀）。**這就是 ch08 那句「不要算逆矩陣、要做分解」落到地上的樣子。**

把這三個直覺收成一句帶進本章：**正交基底是「一組互不洩漏的乾淨座標軸」，它讓座標、投影、解方程全部變簡單；Gram–Schmidt 是把任意基底扶正成正交的標準動作；QR 是它的矩陣記法，也是工程上解最小平方最該信任的工具。** 好座標不是免費的——你得花一道 Gram–Schmidt 把它做出來。但做出來之後，後面全部省事。這就是副標「好座標的代價」。

## 單範正交基底為什麼是天堂

先講清楚為什麼值得大費周章。一組基底 {q₁, q₂, …, qₙ} 叫**單範正交（orthonormal）**，如果它同時滿足兩個條件：

```text
正交（orthogonal）：  qᵢ · qⱼ = 0   當 i ≠ j   ← 兩兩垂直、互不洩漏
單範（normal/unit）：  qᵢ · qᵢ = 1            ← 每根都長度 1
```

合起來可以寫成一條漂亮的式子 `qᵢ · qⱼ = δᵢⱼ`（i=j 時 1、否則 0，δ 是 Kronecker delta）。標準基底 {ê₁, ê₂} 就是最熟悉的單範正交基底——兩根座標軸本來就垂直、長度 1。本章要做的，是把「任意一組歪七扭八的獨立向量」也整理成這種乾淨形式。

為什麼這麼想要它？因為它一次消掉了前面好幾章的麻煩。

### 好處一：座標 = 直接做內積，不必解方程（消掉 ch04 的痛點）

回憶 ch04 的痛點。給一組**一般**基底 {b₁, b₂}，要求某個向量 v 在這組基底下的座標 (c₁, c₂)，你得解線性組合 `c₁b₁ + c₂b₂ = v`——一個線性方程組。基底向量越多、越歪，這個方程越討厭。

但如果基底是**單範正交**的 {q₁, q₂}，座標直接掉出來，不必解任何方程。為什麼？假設

```text
v = c₁ q₁ + c₂ q₂        ← v 是這組正交軸的某個線性組合
```

兩邊跟 q₁ 做內積，看右邊會發生什麼：

```text
v · q₁ = c₁ (q₁·q₁) + c₂ (q₂·q₁)
       = c₁ · 1      + c₂ · 0          ← 正交！q₂·q₁=0 那一項整個消失
       = c₁
```

那個 `c₂ (q₂·q₁)` 項因為 q₂⊥q₁ 而**蒸發**了——這就是「互不洩漏」的威力：問 q₁ 的座標時，q₂ 完全不來插嘴。同理 `c₂ = v·q₂`。所以：

```text
v 在單範正交基底下的座標 cᵢ = v · qᵢ        ← 一個內積就是一個座標，不解方程
```

這是本章第一個「天堂」。**在好座標系裡，讀座標只是做內積。** 你之後會看到，傅立葉分析（把訊號拆成各頻率的成分）就是這件事——正弦／餘弦函數構成一組正交基底，每個頻率的係數就是訊號跟那個正弦函數的內積（見《圓的影子》傅立葉的門口；函數也是向量、也有內積，ch22 一句點到）。**「算係數＝做內積」之所以成立，全靠基底正交。**

### 好處二：投影 = 逐項內積相加（消掉 ch16 的麻煩）

ch16 的投影要解正規方程，因為基底一般不正交。但若子空間有一組單範正交基底 {q₁, …, qₖ}，把任意向量 b 投影到這個子空間，公式退化成最樸素的形式——**各方向的座標算出來、各自貼回去、加總**：

```text
proj(b) = (b·q₁) q₁ + (b·q₂) q₂ + … + (b·qₖ) qₖ
```

每一項 `(b·qᵢ) qᵢ` 是 b 在第 i 根軸上的投影分量（內積給座標、再乘回單位向量給回向量），全部加起來就是 b 在子空間裡的影子。**沒有矩陣求逆、沒有解方程，只有 k 個內積。** 對照 ch16 那個要解 AᵀAx̂=Aᵀb 的正規方程——差別全在「基底正不正交」。這也順帶解釋了為什麼大家拚命想要正交基底：投影是最小平方、迴歸、訊號分解的核心動作，把它變成「逐項內積相加」等於把後面一大票計算都變簡單了。

（一句嚴謹度標示：上面兩個推導是「工程師的嚴謹」——每步你都能口頭說出理由，關鍵只用到 qᵢ·qⱼ=δᵢⱼ 把交叉項消掉。沒有偷工，也沒有需要 Axler 級別證明的地方。）

## Gram–Schmidt：把歪的扶正

現在的問題反過來：你手上通常**沒有**現成的正交基底，只有一組普通的獨立向量（比如某個矩陣的幾個行向量）。Gram–Schmidt 程序就是把它們**逐個扶正**成單範正交的標準動作。

幾何直覺先行，這是整個程序的靈魂：**一個一個處理向量，每拿到一個新的，就把它身上「已經被前面那些軸涵蓋掉的成分」剃掉，剩下的就是這個方向帶來的、前面沒有的新東西；正規化它，成為下一根乾淨的軸。**

「剃掉已有的成分」用的就是 ch16 的投影。對兩個向量 a₁、a₂ 的情形（本章主角，2 維），程序是：

```text
第一根：  e₁ = a₁ / ‖a₁‖                      ← 第一個方向無人在前，直接正規化
剃投影：  w₂ = a₂ − (a₂·e₁) e₁                 ← 把 a₂ 在 e₁ 方向上的成分剃掉
第二根：  e₂ = w₂ / ‖w₂‖                       ← 剩下的就垂直於 e₁ 了，正規化
```

中間那一步是關鍵。`(a₂·e₁) e₁` 是 a₂ 投影到 e₁ 上的影子（ch16：座標 a₂·e₁ 乘回單位向量 e₁）——也就是 a₂ 身上「跟 e₁ 重疊」的部分。把它從 a₂ 減掉，剩下的 w₂ **必然垂直於 e₁**。為什麼必然？驗一下：

```text
w₂ · e₁ = (a₂ − (a₂·e₁) e₁) · e₁
        = a₂·e₁ − (a₂·e₁)(e₁·e₁)
        = a₂·e₁ − (a₂·e₁)·1          ← e₁ 已正規化，e₁·e₁=1
        = 0                           ← 剃乾淨了，w₂ ⊥ e₁
```

這不是巧合也不是運氣——**「減掉投影」這個動作的定義就是為了讓殘差垂直**（這正是 ch16「殘差⊥子空間」那條性質的再現）。所以 Gram–Schmidt 一定會吐出正交的結果，每一步都把新向量身上的「舊成分」剃乾淨。向量多於兩個時就重複：拿第三個，把它在 e₁、e₂ 兩個方向的投影都剃掉，剩下的正規化成 e₃，依此類推。

### 脊椎 S 第一面的正交化：把 S 的兩行扶正

把脊椎矩陣 **S = [[2,1],[1,2]]**（ch01 登場、ch05 讀成兩個基向量去向、ch11 解出特徵值 3 與 1 的那同一個 S）的**兩個行向量** a₁=(2,1)、a₂=(1,2) 做 Gram–Schmidt，每步驗算。這是本章的核心 worked example，不跳步。

**第一步：正規化 a₁ 得 e₁。** 先算長度：

```text
‖a₁‖² = 2² + 1² = 4 + 1 = 5      →   ‖a₁‖ = √5
e₁ = a₁ / ‖a₁‖ = (2, 1) / √5 = (2/√5, 1/√5)
```

（驗 e₁ 長度：(2/√5)² + (1/√5)² = 4/5 + 1/5 = 1 ✓，是單位向量。）

**第二步：算 a₂ 在 e₁ 上的投影，把它剃掉。** 投影分量可以直接用 a₁ 算（不必先正規化，比例一樣）：

```text
a₂ · a₁ = (1)(2) + (2)(1) = 4              ← 這就是 ch15／outline 基準：(2,1)·(1,2)=4
‖a₁‖² = 5
proj = (a₂·a₁ / ‖a₁‖²) a₁ = (4/5)(2, 1) = (8/5, 4/5)    ← a₂ 身上跟 a₁ 重疊的成分
```

`(8/5, 4/5)=(1.6, 0.8)` 就是「a₂ 在 a₁ 方向上的影子」。把它從 a₂ 剃掉：

```text
w₂ = a₂ − proj = (1, 2) − (8/5, 4/5) = (1 − 8/5, 2 − 4/5) = (−3/5, 6/5)
```

**第三步：正規化 w₂ 得 e₂。** 先算 w₂ 長度：

```text
‖w₂‖² = (−3/5)² + (6/5)² = 9/25 + 36/25 = 45/25 = 9/5    →   ‖w₂‖ = 3/√5
e₂ = w₂ / ‖w₂‖ = (−3/5, 6/5) / (3/√5) = (−3/5, 6/5) · (√5/3) = (−1/√5, 2/√5)
```

（中間那步：(−3/5)·(√5/3) = −√5/5 = −1/√5；(6/5)·(√5/3) = 2√5/5 = 2/√5。）

**第四步：驗算——正交、各長 1（深度標準要求自驗，內積最容易算錯）。**

```text
正交： e₁·e₂ = (2/√5)(−1/√5) + (1/√5)(2/√5)
            = (−2 + 2)/5 = 0/5 = 0                    ✓  e₁ ⊥ e₂
長度： ‖e₁‖² = (2/√5)² + (1/√5)² = 4/5 + 1/5 = 1        ✓
       ‖e₂‖² = (−1/√5)² + (2/√5)² = 1/5 + 4/5 = 1       ✓
```

三個檢查全過。**S 的兩行被扶正成單範正交基底 e₁=(2/√5, 1/√5)、e₂=(−1/√5, 2/√5)。** 注意一件事：原本 a₁、a₂ 的夾角是 θ≈36.87°（ch15：cosθ=0.8），現在 e₁、e₂ 被扶成剛好 90°。Gram–Schmidt 把那個 36.87° 的「糾纏」一刀剃成了乾淨的直角。

把這個結果看見——下面這張圖把上面四步畫成一張：

## QR 分解：Gram–Schmidt 的矩陣記法

剛才那一連串向量操作，可以**完全用一個矩陣等式**裝起來，這就是 QR 分解。把 a₁、a₂ 當成一個矩陣 A 的兩個行（A 就是 S）：

```text
A = | 2  1 |        a₁ = (2,1) 是第一行、a₂ = (1,2) 是第二行
    | 1  2 |
```

Gram–Schmidt 的結果——兩根單位軸 e₁、e₂——並排成一個矩陣 **Q**（它的行就是 e₁、e₂）。因為 e₁、e₂ 單範正交，**Q 是正交矩陣**（QᵀQ=I，列／行互相單範正交；正交矩陣＝純旋轉或反射、不改長度與角度，ch18 會再深用）。那 R 是什麼？反推一下：Gram–Schmidt 的每一步都在說「原向量 = 已扶正的軸的某個線性組合」。把這些組合係數排起來就是 R，而且 R 一定是**上三角**——因為 a₁ 只用到 e₁（a₂ 還沒生出來），a₂ 用到 e₁ 跟 e₂，第 k 個原向量只會用到前 k 根新軸，所以下三角全是 0。

R 的元素就是「原行向量跟各新軸的內積」`Rᵢⱼ = eᵢ · aⱼ`（這正是好處一：在 Q 這組正交基底下，aⱼ 的座標就是內積）。對 S 算出來：

```text
R₁₁ = e₁·a₁ = (2/√5)(2) + (1/√5)(1) = (4+1)/√5 = 5/√5 = √5
R₁₂ = e₁·a₂ = (2/√5)(1) + (1/√5)(2) = (2+2)/√5 = 4/√5
R₂₁ = e₂·a₁ = (−1/√5)(2) + (2/√5)(1) = (−2+2)/√5 = 0       ← 下三角為 0，意料之中
R₂₂ = e₂·a₂ = (−1/√5)(1) + (2/√5)(2) = (−1+4)/√5 = 3/√5
```

所以脊椎 S 的 QR 分解是：

```text
        | 2/√5   −1/√5 |              | √5    4/√5 |
Q =     | 1/√5    2/√5 |        R =   |  0    3/√5 |

A = S = Q R
```

**代回驗證 QR=S**（深度標準：worked example 至少代回驗算一次）：

```text
QR = | 2/√5  −1/√5 | | √5   4/√5 |
     | 1/√5   2/√5 | |  0   3/√5 |

第 (1,1) 項： (2/√5)(√5) + (−1/√5)(0)      = 2 + 0          = 2   ✓
第 (1,2) 項： (2/√5)(4/√5) + (−1/√5)(3/√5) = 8/5 − 3/5      = 5/5 = 1   ✓
第 (2,1) 項： (1/√5)(√5) + (2/√5)(0)        = 1 + 0          = 1   ✓
第 (2,2) 項： (1/√5)(4/√5) + (2/√5)(3/√5)   = 4/5 + 6/5      = 10/5 = 2  ✓

QR = | 2  1 | = S    ✓
     | 1  2 |
```

精準回到 S。**QR 分解就是把 Gram–Schmidt 的全部資訊裝進「一個正交矩陣 × 一個上三角矩陣」。** R 的對角元 √5、3/√5 還順帶記錄了每根軸正規化前的長度（‖a₁‖=√5、‖w₂‖=3/√5）——R 把「每個向量帶來多少新長度」攤在對角線上。

（一句歷史：把「正交化一組向量」這件事掛上 Gram 與 Schmidt 兩個名字，是後人的命名。Jørgen Pedersen Gram 是丹麥數學家、1883 年在最小平方的脈絡用過；Erhard Schmidt 是德國數學家、1907 年在積分方程裡用過，他自己承認方法跟 Gram 的本質相同——而更早 Laplace、Cauchy 就已經在用了，這又是一個「名字晚於方法」的線代慣例，跟 ch08 的高斯消去同款（2026-06，見延伸閱讀）。「QR 分解」與把它做穩的 Householder 反射矩陣則更晚——1958 年 Householder 才把它引進矩陣計算，本章不展開、見延伸。）

## QR 解最小平方：為什麼比正規方程值得信任

這是本章最該帶走的工程結論，也是 ch08「不要算逆、要做分解」與 ch16 正規方程的合流點。

回憶 ch16：超定系統 Ax=b（方程比未知多、b 不在行空間裡）無解時，最小平方解 x̂ 滿足**正規方程**：

```text
AᵀA x̂ = Aᵀb               ← ch16 的解法：先組 AᵀA，再解這個方程
```

能算，但有一個數值上的隱憂，而且是**致命**的隱憂。一個矩陣有多「病態」（小擾動會被放大多少，ch08 的條件數 condition number κ(A)）決定了你解出來的答案有多可信。問題在於 `AᵀA` 這一步——**它把條件數平方了**：

```text
κ(AᵀA) = κ(A)²            ← 病態程度被平方放大（2026-06，見延伸閱讀）
```

直覺上為什麼會平方：AᵀA 的奇異值是 A 的奇異值的平方（ch19 會講奇異值；這裡先用結論），而條件數＝最大奇異值/最小奇異值，平方之後比值也平方。後果很實在：如果 A 的條件數是 10⁶（已經不算太誇張的病態），那 AᵀA 的條件數就是 10¹²——你在用 double（約 16 位有效數字）算，等於先燒掉 12 位精度才開始解，剩下 4 位，最小平方解的低位數字全是雜訊。**「它可逆、AᵀA 也可逆」不代表「你解得準」**——這正是 ch08 病態矩陣那一節的故障在最小平方裡的現身。

QR 怎麼避開這一刀？把 A=QR 代進 Ax=b，利用 Q 是正交矩陣（QᵀQ=I、且 Q 不改長度，所以 min‖Ax−b‖ 等價於 min‖QRx−b‖ 等價於 min‖Rx−Qᵀb‖）：

```text
最小平方解 x̂ 滿足：  R x̂ = Qᵀb
```

R 是**上三角**，這個方程用**回代（back-substitution）**從最後一個未知數往上一個一個解出來，根本不用求逆、不用組 AᵀA。關鍵是：**全程沒有出現 AᵀA，條件數只停在 κ(A)、沒有被平方**。誤差被放大 κ(A) 倍而不是 κ(A)² 倍（2026-06，見延伸閱讀）。對上面那個例子，QR 燒掉 6 位精度、留 10 位——比正規方程的 4 位好太多了。

所以工程結論一句話：**解最小平方，預設走 QR（或 SVD，ch19），不要手算 AᵀA 再解、更不要算 A⁻¹。** 你用的 NumPy `numpy.linalg.lstsq`、MATLAB 的反斜線 `\`、LAPACK 的最小平方常式，骨子裡都是 QR 或 SVD，不是正規方程——它們替你做了正確的選擇（2026-06）。**這就是 ch08 那句「用逆矩陣解方程是數值上的罪、實務上做分解」的完整、可操作版本：分解指的就是 QR（與 ch19 的 SVD）。**

（誠實補一句代價，呼應副標：QR 比正規方程貴——運算量大約兩倍（2026-06）。但對絕大多數問題，多花一倍時間換回不被平方放大的精度，是極划算的交易。只有當你確定矩陣良態、又極度在意速度時，正規方程才偶爾值得。這就是「好座標的代價」——你付出一道分解，買到數值上的安心。）

## 直覺的陷阱

正交基底與 QR 是「觀念簡單、細節咬人」的典型。下面五個是你（機械操作沒問題、語意生鏽的資深工程師）最可能踩的，每個都附「怎麼自我察覺」。

| 陷阱 | 錯誤直覺長什麼樣 | 會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|---|
| **以為任意基底都一樣好** | 「不就是一組能張成空間的向量嗎，正不正交有差嗎」 | 用一般（非正交）基底時，每算一個座標、每做一次投影都得解線性方程組；基底一歪、又病態，數值還會爛掉 | 正交基底下**座標＝內積、投影＝逐項內積相加**，完全不解方程（本章好處一、二）。基底正不正交，差的是「一個內積」還是「一個線性方程組」。覺得算座標好麻煩，多半是因為你的基底沒正交。 |
| **Gram–Schmidt 順序無關** | 「反正都是那組向量，先扶誰後扶誰結果一樣」 | 換個順序，得到的 e₁、e₂ 是**不同的**正交基底；做完發現跟同事的 Q、R 對不上，以為有人算錯 | Gram–Schmidt **結果依賴輸入順序**：第一個向量直接決定第一根軸（e₁=a₁/‖a₁‖），換 a₁ 當第一個，e₁ 就變另一條線、後面全跟著變。兩個答案都對、都是合法的正交基底，只是不同組。順序是程序的一部分，不是可有可無。 |
| **classical Gram–Schmidt 數值穩** | 「公式推導得乾乾淨淨，照著算就對了」 | 在病態或高維情形，**classical（教科書那版）Gram–Schmidt 會悄悄丟失正交性**——算出來的 e₁、e₂ 內積不是 0 而是個不小的數，你卻渾然不覺 | 教科書版（一次把所有投影都用原始向量算）在浮點下會因「相減幾乎平行的向量」產生災難性抵消，正交性損失正比於 κ(A)²；**modified Gram–Schmidt**（每剃掉一個投影就立刻更新工作向量、再算下一個）損失只正比 κ(A)，穩得多（2026-06，見延伸閱讀）。徵兆：算完驗 qᵢ·qⱼ 不夠接近 0。本章手算數字漂亮，是因為是 2 維又良態；上規模就得用 modified 版（或直接用 Householder QR）。 |
| **把 QR 當「只是一種分解」** | 「QR 就是把 A 拆成 Q 乘 R，一個寫法而已」 | 遇到最小平方還是反射性地去組 AᵀA 解正規方程，把條件數平方放大、燒掉一半精度還不知道 | QR 的**殺手級用途是解最小平方**：R x̂ = Qᵀb 回代即得，全程不碰 AᵀA、條件數不被平方。把 QR 只當「分解寫法」會錯過它最值錢的地方——它是 ch08「不要算逆」那句話的正解。 |
| **以為「正交」就是「無關／獨立」** | 把正交（內積=0）跟線性獨立、統計獨立混為一談 | 在 ch20 PCA、或處理相關特徵時，誤把「正交」當「統計獨立」，或反過來以為獨立向量自動正交 | 三件不同的事：**線性獨立**（誰都不是別人的線性組合，ch03）是最弱的；**正交**（內積=0）更強，正交一定獨立、獨立不一定正交（a₁、a₂ 獨立但不正交，夾角 36.87°）；**統計獨立**（聯合分布可分解）又是另一回事（線代版的「不相關≠獨立」見《馴服隨機》ch10）。Gram–Schmidt 把「獨立」升級成「正交」，靠的就是剃掉重疊成分這道工。 |

把第一、四個陷阱合成一句你能口頭講的：**正交基底不是「換個寫法」，是把座標、投影、解方程從『解線性方程組』降級成『做幾個內積』的真實省事；而 QR 把這個省事用在最小平方上，避開了正規方程平方放大條件數的數值災難。** 這兩件事講清楚，你就抓住了本章的全部價值。

## 紙上推演

### 推演題

**第 1 題 ★ [10 分鐘]——為什麼正交基底下座標就是內積**
給單範正交基底 {q₁, q₂}（qᵢ·qⱼ=δᵢⱼ）與一個向量 v=c₁q₁+c₂q₂。(a) 證明 c₁=v·q₁（把 v 跟 q₁ 做內積，逐項展開，指出哪一項因正交而消失）；(b) 用一句話說明：如果基底**不**正交，這個「座標＝內積」的捷徑為什麼就壞了。

**第 2 題 ★★ [15 分鐘]——對一組新向量做 Gram–Schmidt 並驗證正交**
對 a₁=(3,0)、a₂=(1,2) 做 Gram–Schmidt。(a) 算 e₁；(b) 算 a₂ 在 e₁ 上的投影、剃掉、得 w₂；(c) 正規化得 e₂；(d) 驗證 e₁·e₂=0、‖e₁‖=‖e₂‖=1。（這組刻意讓 a₁ 躺在 x 軸上，看正交化怎麼把 a₂ 的橫向成分剃掉。）

**第 3 題 ★★ [12 分鐘]——順序影響結果**
把第 2 題的兩個向量**交換順序**：先扶 a₂=(1,2)、再扶 a₁=(3,0)。算出新的 e₁'、e₂'，並指出它跟第 2 題的 e₁、e₂ 是不是同一組。用這個結果反駁「Gram–Schmidt 順序無關」。

**第 4 題 ★★★ [12 分鐘]——QR 為什麼比正規方程值得信任**
某人說：「解最小平方，正規方程 AᵀAx̂=Aᵀb 跟 QR 解 Rx̂=Qᵀb 數學上等價，所以隨便用哪個都行。」這句話「數學上等價」沒錯，但結論錯了。(a) 指出兩者在**數值上**的關鍵差別（提示：條件數）；(b) 解釋為什麼「組 AᵀA」這一步是罪魁；(c) 用一句話把它連回 ch08「不要算逆矩陣解方程」。

### 推演解答

**第 1 題。**

(a) 把 v=c₁q₁+c₂q₂ 跟 q₁ 做內積，內積對加法與純量倍是線性的（ch15），逐項展開：

```text
v · q₁ = (c₁q₁ + c₂q₂) · q₁
       = c₁ (q₁·q₁) + c₂ (q₂·q₁)
       = c₁ · 1      + c₂ · 0          ← q₁·q₁=1（單範）、q₂·q₁=0（正交）
       = c₁
```

那個 `c₂(q₂·q₁)` 項因為 q₂⊥q₁ 而整個消失——這就是答案：**c₁=v·q₁**。同理 c₂=v·q₂。

(b) 如果基底不正交，q₂·q₁≠0，那個交叉項**不會消失**：`v·q₁ = c₁(q₁·q₁) + c₂(q₂·q₁)`，右邊同時牽扯 c₁ 和 c₂，你得不到 c₁ 的單獨值——必須把兩條這種式子湊成方程組一起解。正交性是「讓交叉項蒸發、座標各自獨立掉出來」的唯一理由。失去正交，就退回 ch04 解線性組合的老路。

**第 2 題。** a₁=(3,0)、a₂=(1,2)。

(a) ‖a₁‖=√(9+0)=3，e₁=(3,0)/3=**(1,0)**。

(b) 投影：

```text
a₂·a₁ = (1)(3)+(2)(0) = 3，  ‖a₁‖²=9
proj = (3/9)(3,0) = (1/3)(3,0) = (1,0)
w₂ = a₂ − proj = (1,2) − (1,0) = (0,2)        ← 橫向成分 (1,0) 被剃掉，只剩縱向
```

（看出來了嗎——a₁ 躺在 x 軸上，所以「剃掉 a₂ 在 e₁ 上的投影」就是剃掉 a₂ 的 x 分量，剩純 y 分量 (0,2)。）

(c) ‖w₂‖=√(0+4)=2，e₂=(0,2)/2=**(0,1)**。

(d) 驗證：e₁·e₂=(1,0)·(0,1)=0 ✓；‖e₁‖=1、‖e₂‖=1 ✓。這組正交化的結果剛好就是標準基底 {(1,0),(0,1)}——因為 a₁ 本來就沿 x 軸，Gram–Schmidt 把第二根扶成了 y 軸。

**第 3 題。** 交換順序：先 a₂=(1,2)、後 a₁=(3,0)。

```text
e₁' = a₂/‖a₂‖ = (1,2)/√5 = (1/√5, 2/√5)
a₁ 在 e₁' 上投影： a₁·a₂ = (3)(1)+(0)(2)=3，  ‖a₂‖²=5
  proj = (3/5)(1,2) = (3/5, 6/5)
w₂' = a₁ − proj = (3,0) − (3/5,6/5) = (12/5, −6/5)
‖w₂'‖ = √(144/25 + 36/25) = √(180/25) = (6√5)/5 = 6/√5
e₂' = (12/5,−6/5)/(6/√5) = (2/√5, −1/√5)
```

新基底是 e₁'=(1/√5, 2/√5)、e₂'=(2/√5, −1/√5)，**跟第 2 題的 {(1,0),(0,1)} 完全是不同的兩根軸**（雖然也單範正交，驗 e₁'·e₂'=(2−2)/5=0 ✓）。第一個向量決定第一根軸的方向，換了第一個，整組就換了。**所以「Gram–Schmidt 順序無關」是錯的——順序變，正交基底就變，兩組都對但不相同。**

**第 4 題。**

(a) 數值上的關鍵差別是**條件數**：正規方程要組 `AᵀA`，而 `κ(AᵀA)=κ(A)²`——病態程度被平方放大；QR 解 `Rx̂=Qᵀb` 全程不碰 AᵀA，條件數只停在 κ(A)、不被平方（2026-06）。

(b) 「組 AᵀA」是罪魁，因為一旦把條件數平方，你在有限精度（double 約 16 位有效數字）下會多燒掉一倍的精度位數。A 條件數 10⁶ 時，AᵀA 是 10¹²，等於先丟 12 位才開始解，最小平方解的低位全是雜訊。「數學上等價」是在無限精度的理想世界；真實的浮點世界裡，先平方再解跟不平方直接解，精度差一大截。

(c) 連回 ch08：用 A⁻¹（或 AᵀA 這種會放大病態的中間量）解方程是「數值上的罪」；**正解是做分解——對最小平方，那個分解就是 QR（或 ch19 的 SVD）**，回代即得、條件數不被平方。

### 動手生圖

本章的圖（對 S 兩行 a₁=(2,1)、a₂=(1,2) 做 Gram–Schmidt：灰色斜輸入、橘色被剃掉的投影 (8/5,4/5)、紅藍兩根正交單位軸 e₁ e₂）由以下腳本產生。它同時就是你的小實驗：跑它、改它、重生它。

```python
# ch17 figure: Gram-Schmidt on the spine S=[[2,1],[1,2]] columns a1=(2,1), a2=(1,2).
# e1 = a1/|a1|; subtract a2's projection onto e1 (the (8/5,4/5) component) to get the
# orthogonal residual, then normalize to e2. Shows the two slanted inputs, the shaved-off
# projection, and the resulting orthonormal e1, e2 (e1.e2=0).
from pathlib import Path
import numpy as np
import matplotlib
matplotlib.use("Agg")          # headless; no display needed
import matplotlib.pyplot as plt

OUT = Path(__file__).resolve().parent / "out" / "ch17-gram-schmidt.svg"
OUT.parent.mkdir(parents=True, exist_ok=True)

a1 = np.array([2.0, 1.0]); a2 = np.array([1.0, 2.0])      # the two columns of S
e1 = a1 / np.linalg.norm(a1)                              # first orthonormal vector
proj = (a2 @ a1) / (a1 @ a1) * a1                         # projection of a2 onto a1 = (8/5,4/5)
resid = a2 - proj                                         # orthogonal residual = (-3/5,6/5)
e2 = resid / np.linalg.norm(resid)                        # second orthonormal vector

def arrow(v, col, lw=2.2, ls="-"):
    ax.annotate("", xy=v, xytext=(0, 0),
                arrowprops=dict(color=col, width=lw, headwidth=9, linestyle=ls))

fig, ax = plt.subplots(figsize=(6, 6))
arrow(a1, "#7f8c8d", 1.6); ax.text(2.05, 0.9, "a1=(2,1)", color="#7f8c8d", fontsize=9)
arrow(a2, "#7f8c8d", 1.6); ax.text(0.55, 2.0, "a2=(1,2)", color="#7f8c8d", fontsize=9)
arrow(proj, "#d68910", 1.4); ax.text(1.6, 0.45, "proj=(8/5,4/5)", color="#d68910", fontsize=8)
ax.plot([proj[0], a2[0]], [proj[1], a2[1]], color="#d68910", lw=1.0, ls=":")  # a2 - proj
arrow(e1, "#c0392b"); ax.text(0.92, 0.30, "e1", color="#c0392b", fontsize=11)
arrow(e2, "#2471a3"); ax.text(-0.62, 0.85, "e2", color="#2471a3", fontsize=11)
ax.plot([0, resid[0]], [0, resid[1]], color="#2471a3", lw=1.0, ls="--", alpha=0.6)  # residual dir

ax.set_title("Gram-Schmidt on S columns: e1.e2 = %.0f  (orthonormal)" % (e1 @ e2), fontsize=10)
ax.set_xlim(-1.2, 2.6); ax.set_ylim(-0.6, 2.4); ax.set_aspect("equal")
ax.axhline(0, color="0.6", lw=0.6); ax.axvline(0, color="0.6", lw=0.6)
ax.grid(True, color="0.9", lw=0.6)
fig.savefig(OUT, bbox_inches="tight")
print("wrote", OUT)            # build_figures.py reads this
```

**預期輸出**：一張圖。灰色兩根斜箭頭是輸入 a₁=(2,1)、a₂=(1,2)（夾角約 36.87°）；橘色箭頭是投影 proj=(8/5,4/5)=(1.6,0.8)，橘色點線從 a₂ 垂直落到它上面（被剃掉的成分）；紅色 e₁、藍色 e₂ 是最後的兩根單位軸，沿藍虛線方向的 e₂ 與 e₁ 互成直角。標題印出 `e1.e2 = 0`，確認正交。確認三個數值：proj=(1.6,0.8)、e₁=(2/√5,1/√5)≈(0.894,0.447)、e₂=(−1/√5,2/√5)≈(−0.447,0.894)（你都能手算對上）。

**改參數看什麼**（把概念玩活的地方）：

- **換 a₂ 看正交化結果怎麼變**：把 `a2` 改成 `np.array([0.,3.])`（純縱向）跑跑看——a₁=(2,1) 不變，但 a₂ 換了，剃掉的投影分量、得到的 e₂ 都會不同（e₁ 一樣，因為 e₁ 只由 a₁ 決定）。體會「e₂ 是『a₂ 剃掉 a₁ 成分後的剩餘方向』」。
- **交換 a₁ a₂ 的順序看 e₁ 變成誰**：把兩行對調 `a1=np.array([1.,2.]); a2=np.array([2.,1.])`。現在 e₁ 沿 (1,2) 而不是 (2,1)——**整組正交基底跟著換了**（這就是「順序影響結果」第 3 題的視覺版）。兩組都正交、都對，但不是同一組。
- **換成已經正交的輸入看 Gram–Schmidt 什麼都不做**：把 `a2` 改成 `np.array([-1.,2.])`（它已經垂直於 a₁=(2,1)，內積 −2+2=0）——投影 proj 會是 (0,0)，w₂=a₂，Gram–Schmidt 只做了正規化、沒剃任何東西。輸入已正交時，這道程序退化成單純的「各自正規化」。

## 自我檢核

口頭自答；講得出來才算過關，卡住就回到對應段落。

1. **單範正交基底是什麼？它好在哪？** 一組兩兩垂直（內積=0）、各自長度 1 的座標軸。好處：座標＝直接做內積、投影＝逐項內積相加，全部不必解線性方程組（消掉 ch04、ch16 的麻煩）。

2. **為什麼正交基底下座標就是內積？**（本章必答）因為 v=Σcᵢqᵢ 跟某個 qⱼ 做內積時，所有 i≠j 的交叉項都因 qᵢ·qⱼ=0 而消失，只剩 cⱼ(qⱼ·qⱼ)=cⱼ。正交讓交叉項蒸發、座標各自獨立掉出來；基底不正交，交叉項不消、就得解方程組。

3. **Gram–Schmidt 在做什麼？用一句幾何的話。** 逐個處理向量，每拿到一個新的，就把它身上「已被前面那些軸涵蓋掉的成分」用投影剃掉，剩下的純粹新方向正規化成下一根軸。靠「減掉投影」保證殘差垂直於前面所有軸。

4. **為什麼 a₂ 減掉它在 e₁ 上的投影後，一定垂直於 e₁？** 因為 w₂·e₁ = a₂·e₁ − (a₂·e₁)(e₁·e₁) = a₂·e₁ − a₂·e₁ = 0（用到 e₁·e₁=1）。「減掉投影」這個動作的定義就是為了讓殘差正交——這正是 ch16「殘差⊥子空間」的再現。

5. **QR 分解的 Q 和 R 各是什麼？R 為什麼是上三角？** Q 的行是 Gram–Schmidt 扶正出的單範正交軸（Q 正交，QᵀQ=I）；R 的元素 Rᵢⱼ=eᵢ·aⱼ 是原向量在新軸下的座標。R 上三角，因為第 k 個原向量只用到前 k 根新軸（後面的軸還沒生出來），所以下三角全 0。

6. **Gram–Schmidt 的結果跟輸入順序有關嗎？** 有關。第一個向量直接決定第一根軸，換順序就換出不同（但同樣合法）的正交基底。順序是程序的一部分。

7. **QR 為什麼比算逆矩陣（或正規方程）解最小平方更值得信任？**（本章必答）正規方程要組 AᵀA，把條件數平方放大（κ(AᵀA)=κ(A)²），在有限精度下多燒一倍精度；QR 解 Rx̂=Qᵀb 全程不碰 AᵀA，條件數只停在 κ(A)、回代即得。這就是 ch08「不要算逆、要做分解」的正解——分解指的就是 QR（與 ch19 的 SVD）。

8. **「正交」跟「線性獨立」是同一件事嗎？** 不是。正交（內積=0）比線性獨立（誰都不是別人的線性組合）強：正交一定獨立，獨立不一定正交（a₁、a₂ 獨立但夾角 36.87°、不正交）。Gram–Schmidt 就是把「獨立」升級成「正交」的那道工。
