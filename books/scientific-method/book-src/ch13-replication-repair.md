# ch13 — 重複的力量：複製、預註冊、開放科學如何修補危機

> **本章解決什麼問題**：ch09 告訴你再現性危機是什麼、為什麼會發生——p 值被誤用、p-hacking、一批效應複製率只有約三分之一到四成。那是「壞消息本身」。本章只問一件事：**知道了，然後呢？** 科學社群這十幾年用什麼制度把這個漏洞補起來——複製、預註冊、註冊報告、開放資料、多實驗室合作。本章在全書地圖落在 Part IV「科學擋不住的、與它如何修補」：ch11 講科學擋不住個人的偏心，ch12 講它如何贏過偽科學，本章講它如何修自己的傷。本章還藏著一個反直覺的好消息——**危機被公開揭露、被吵得沸沸揚揚，這本身就是糾錯機制在運作的證據**，不是科學破產的訃聞。脊椎透鏡：塞麥爾維斯的結果其實**被複製過**，但缺機制讓複製也說服不了人（這個遺憾接 ch14）。

## 從你已知的出發

你寫過 flaky test。

那種跑十次綠八次、紅兩次，重跑一下又綠了的測試。你心裡很清楚：**一次綠不算數**。一次綠可能是真的通過，也可能是某個 race condition 剛好沒撞上、某個外部服務剛好沒抽風、某個 timeout 剛好沒踩線。你判斷一個 test「真的穩」，靠的不是某一次的結果，是它**反覆跑、反覆綠**——你甚至會在 CI 裡把可疑案例跑個 100 次，看失敗率收斂到哪。你在用重複過濾僥倖。

你也說過、或聽人說過那句工程界最有名的鬼話：「在我機器上會動。」

這句話之所以是鬼話，是因為它**無法被複製**。一個結果如果只在某一台機器、某一個人手上、某一個下午成立，換個環境就垮，那它就不是一個可靠的事實，是一次僥倖（或一個沒被發現的隱藏依賴）。可重現建置（reproducible build）這整套工程實踐，存在的唯一理由就是：**讓「能跑」這件事脫離特定環境的偶然，變成任何人在任何乾淨環境都能重現的事實。**

如果你接受這兩件事——「一次綠不算數」「在我機器上會動不算數」——那你已經懂了本章的全部直覺。再現性危機說穿了，就是科學界發現自己一直在拿 flaky test 的單次綠當結論發表，而且沒人去重跑。本章那些修補制度——複製、預註冊、開放資料——逐一對應你早就在用的工程習慣：把案例多跑幾次、先凍結驗收標準再實作、把環境和資料攤開讓別人能重現。差別只在，科學界補上這些習慣花了一百年，代價是好幾代研究者的職涯與好幾櫃子沒人看過的失敗結果。

> 一句話回扣 ch09（不重講）：再現性危機的**診斷**（p-hacking 機制、複製率約三分之一到四成那個壞消息）歸 ch09；本章只管**處方**。需要時我會寫「（見 ch09）」，不在這裡重算複製率。

## 為什麼「複製」是防自欺的核心，而不只是一道手續

先把複製（replication）這個詞講精確，因為它常被誤解成「重看一遍同一篇論文的數據對不對」——那叫**重分析（reanalysis）**或**計算再現（computational reproducibility）**，是檢查算術，重要但不是這裡的主角。本章講的**複製**，是另一群人、用另一批受試者、（理想上）在另一個實驗室，**從頭再做一次**這個實驗，看效果還在不在。

為什麼這件事是防自欺的核心？因為單次結果裡，**僥倖和真相長得一模一樣**。

回憶 ch09 的算術（不重推）：如果一個效應其實不存在（虛無假設為真），你做一次實驗，仍有約 5% 的機率純靠運氣得到 p < 0.05 的「顯著」結果。再加上 p-hacking 的研究者自由度，這個偽陽率還會被推得更高。問題在於——**從單獨一篇論文裡，你無法分辨它到底是那 5% 的運氣，還是真效應。** 兩者印出來的 p 值、信賴區間、星號，看起來完全一樣。

複製就是那個分辨器。它的邏輯極其簡單，簡單到近乎暴力：

```text
若效應是真的：
  做一次 → 顯著
  再做一次（新樣本、新實驗室）→ 多半還是顯著
  再做一次 → 還是顯著
  ……重複會「累積」

若效應是僥倖（那 5% 的運氣）：
  做一次 → 碰巧顯著
  再做一次 → 運氣用完了，多半不顯著
  再做一次 → 又不顯著
  ……重複會「過濾掉」它

所以：能被獨立複製的效應，是真效應的強訊號；
      複製不出來的「顯著」，多半是當初那次的運氣。
```

這跟你把 flaky test 跑 100 次是**同一個動作**。一次綠分不出「真通過」和「race 剛好沒撞」；跑 100 次，真通過會穩定綠，race 會穩定地偶爾紅——重複把「穩定的真」和「偶發的假」分開了。

這裡有一個值得停下來想十分鐘的點：**複製的防自欺力量，不來自任何一次實驗做得多嚴謹，而來自「獨立性」**。同一個人、同一個團隊、用同一套可能有偏的流程，重做十次，十次都顯著——這幾乎沒有增加多少可信度，因為如果偏誤藏在流程裡（某個 p-hacking 習慣、某個量測的系統偏差），它會在每一次重做裡**忠實地重現**。真正有鑑別力的，是**別人**、用**別的樣本**、在**別的實驗室**重做。獨立性高的複製，才能把「藏在某個團隊習慣裡的系統性自欺」暴露出來。

（這跟 ch15 要講的「獨立趨同 vs 回音室」是同一個直覺的兩個面向：十個互相抄的人說同一句話，不等於十份獨立證據。複製要有意義，必須獨立。）

## 危機之前，科學長什麼樣：複製為什麼一度幾乎不存在

要理解這些修補制度在補什麼洞，得先看危機之前那個系統有多漏。

危機之前的科學發表，有一套**獎勵結構**，而這套結構幾乎是專門設計來**懲罰複製、獎勵僥倖**的——雖然沒人是故意的，這正是它可怕的地方（呼應 ch11：制度的失靈不需要壞人，只需要沒人去堵的漏洞）。

```text
危機前的獎勵結構（沒人故意設計成這樣，但它就是長這樣）

  新奇的、顯著的、漂亮的正面結果  →  登上頂級期刊  →  升等、拿經費、出名
  複製別人的研究（尤其複製成功）  →  「你只是重做別人的」→ 期刊不收、沒人引用
  複製失敗（推翻別人）            →  得罪同行、難發表、被質疑「你是不是沒做好」
  自己的實驗沒做出顯著效果        →  塞進抽屜，不投稿（誰要看 null result？）
                                      ↑ 這就是「檔案抽屜問題」file drawer problem
```

把這張表看懂，你就懂為什麼危機會發生，而且為什麼它**註定**會發生：

- **沒人複製**：因為複製不能發表、不算貢獻。一個結果發表後，可能幾十年沒有任何人去獨立重做——那 5% 的運氣就這樣一直掛在文獻裡，被當成事實引用。
- **檔案抽屜問題（file drawer problem）**：做出「沒效果」的人不投稿，期刊也不收。於是文獻裡**只剩下顯著的結果**——這叫發表偏誤（publication bias，ch16 會正式收進失敗模式目錄）。想像有 20 個實驗室各自獨立研究同一個其實不存在的效應，純靠運氣大約有一個會做出 p < 0.05，那一個發表了、上了期刊，另外 19 個 null result 全進了抽屜。讀者只看到那一篇，以為「有效應」。**文獻本身就是被僥倖篩選過的樣本。**
- **真誠的 p-hacking**（見 ch09，不重講）：研究者自由度讓真誠的人也能無意間把偽陽率推高，而當時沒有任何制度逼你事先講清楚你打算怎麼分析。

所以危機不是「一群壞科學家造假」。它是一套**結構性地放大偽陽、過濾掉糾錯**的系統，在運轉了幾十年後，被一群人決定去做那件沒獎勵的事——大規模複製——終於暴露出來。下一節講的每一道修補制度，都是在拆這張獎勵結構表裡的某一格。

## 預註冊：先凍結驗收標準，再看資料

如果你只能記住本章一個修補制度，記這個。因為它最像你的工作，也最直接地堵住 ch09 那個「真誠的人也會 p-hack」的漏洞。

**預註冊（preregistration）**：在**看到資料之前**，把你的假設、你要量哪個指標、樣本要收多少、收到什麼條件就停、你打算怎麼分析、什麼叫成功——全部寫下來，戳上時間戳，公開存檔（存到像 OSF 這類公開登記處）。事後你做出來的分析，跟這份預先凍結的計畫一比對，所有「事後才決定的自由度」就無所遁形。

它堵的是哪一種自欺？**事後挑分析**。

```text
沒有預註冊（研究者自由度全開）：
  收資料 → 邊看邊決定：
            量哪個指標好（測了 5 個，報最顯著那個）
            收到多少人才停（看到顯著就停，叫 optional stopping）
            要不要排除離群值（排掉對自己不利的那批）
            分幾組、怎麼切（切到出現顯著的那個切法）
  → 每一個「事後決定」都偷偷抬高偽陽率，而且自己不一定察覺

有預註冊（自由度事先凍結）：
  先寫死：假設 = H，主要指標 = X，樣本 = 200，停止規則 = 收滿就停，
          分析 = 預先指定的那一個，成功 = p<0.05 在 X 上
  → 戳時間戳、公開存檔
  → 然後才收資料、跑「那一個」分析
  → 事後想換指標/換分析？可以，但必須標明是「探索性」，不能假裝是「驗證性」
```

你已經在做這件事了，名字叫**先寫驗收標準、再實作**。

想想一個健康的 A/B test 流程跟一個有毒的流程差在哪。有毒的：上線新版本，盯著 dashboard，哪個指標先動就講哪個，一直跑到某個指標達到顯著就喊「贏了」收手。這就是 ch09 講的偷看（peeking）＋多重比較，是教科書級的 p-hacking。健康的：**上線前**就寫死——主要指標是轉換率（不是事後挑的留存、停留時間、點擊率裡最好看那個），樣本量算好要收多少（不是看到顯著就停），停止規則固定，什麼叫贏事先定義。然後才放量，才看那一個指標。

這兩個流程的差別，**和預註冊與不預註冊的差別，是同一件事**。預註冊就是把「驗收標準」從「實作之後可以偷偷改」變成「實作之前已凍結、改了會被看見」。你不會接受一個工程師在跑完壓測之後才宣布「喔其實我們的目標一直是 P95 不是 P99」——因為你知道那叫**移動球門**。科學界花了很久才把「移動球門」這件在工程界天經地義不能幹的事，變成制度上不能幹的事。

> 誠實標示嚴謹度（工程師的嚴謹）：預註冊不是萬靈丹，每一步都能講出理由地說它**不**保證什麼——它不保證實驗設計本身是好的（你可以預註冊一個爛實驗）、不保證樣本有代表性、不能阻止有人事後偷偷不照計畫做（它讓偏離**可被看見**，不是物理上不可能）。而且 2024–2025 有個值得記下的反諷（2026-06）：當「有沒有預註冊」本身變成一個品質徽章、變成期刊加分項，有些人會為了拿徽章而預註冊一份空泛到沒有約束力的計畫——工具一旦變成指標，就開始被玩弄（這正是 ch09／ch16 的 Goodhart 式自欺在制度層的回聲）。預註冊堵的是「事後挑分析」這一格，不是全部。

## 註冊報告：在還沒有結果之前，就決定要不要發表

預註冊堵住了研究者那一端的自由度。但獎勵結構表裡還有一格沒拆：**期刊只收漂亮的正面結果**。這格不拆，發表偏誤就還在——你預註冊得再乾淨，做出 null result 還是會被期刊退稿、塞回抽屜。

**註冊報告（registered reports）** 拆的就是這格。它把同儕審查切成兩半：

```text
傳統發表流程：
  做實驗 → 拿到結果 → 寫論文 → 投稿 → 審稿人看「結果」決定收不收
                                        ↑ 結果不漂亮 → 退稿 → 進抽屜（發表偏誤）

註冊報告流程：
  寫研究計畫（問題＋方法＋分析＋預註冊）→ 投稿
    → 審稿人在「還沒有任何結果」時審「這個問題重要嗎、方法夠嚴謹嗎」
    → 通過 → 期刊「原則上接受」(in-principle acceptance)：
              不管結果是正是負、顯著不顯著，只要你照計畫做，就保證刊登
    → 然後才做實驗、拿到結果（正負都登）
```

這個設計的精妙在於它**同時**堵了兩個自欺：審稿人在看不到結果的情況下評判方法，就不會被一個漂亮的正面結果迷惑而放水（也不會因為結果挑戰了自己的立場而刁難——這是 ch11 動機性推理的制度解藥）；而「原則上接受」這個承諾，意味著**做出 null result 的人不再被懲罰**，抽屜被打開了。一個事先被判定「問題重要、方法嚴謹」的實驗，做出「沒效果」也是有價值的知識——它告訴你那個效應可能不存在。

對照你的工程直覺：這像是**在實作之前先讓 reviewer 批准 RFC 與測試計畫，並承諾「只要你照這個計畫做、把測試誠實跑完，不管綠紅都收進主幹的紀錄」**。你不會因為某次壓測結果不好看就把它從紀錄裡刪掉假裝沒測過——那是在污染你自己團隊的知識庫。註冊報告就是科學界版本的「null result 也要進紀錄」。

## 開放資料與多實驗室複製：把獨立性做大

剩下兩道修補制度，目標都是把前面講的「獨立性」這件事**規模化**。

**開放資料／開放材料（open data / open materials）**：把原始資料、分析程式碼、實驗材料全部公開。它堵的是「你說有就有、沒人能查」這種封閉自欺。一旦資料公開，任何人都能重跑你的分析（計算再現），任何人都能拿你的材料去做獨立複製（不必猜你到底怎麼做的）。這對你來說毫無神秘感——這就是**開源 ＋ 可重現建置**：把環境、依賴、流程全攤開，讓任何人在乾淨環境都能重現，「在我機器上會動」失去藏身之處。

**多實驗室複製（multi-lab / many-labs）**：與其等某個孤單的研究生去複製一篇論文（沒獎勵、樣本又小），不如組織幾十個實驗室，用同一份預註冊協定、同時複製同一個效應，匯總成一個巨大樣本。這是複製的工業化版本，把「獨立性」這個變數一次拉到很高——幾十個不同地點、不同人群、不同主試者，如果效應在這麼多獨立條件下都穩定出現，它就很難是某個團隊的僥倖或習慣偏差。

這裡可以放一個本書最該誠實的具體案例，因為它正好示範了「複製如何過濾僥倖」。**自我耗竭（ego depletion）** ——「意志力像肌肉，用了會累」這個 1998 年提出、紅遍二十年、被寫進無數暢銷書的效應——在大型多實驗室預註冊複製下幾乎站不住（2026-06）：

```text
自我耗竭效應的複製史（截至 2026-06，需 hedge）

  1998   原始研究提出，效果看起來很大、很漂亮，紅了二十年
  2016   一個 23 實驗室、2000+ 人的預註冊多實驗室複製
         → 沒有顯著效果（效果量基本上趴在零附近）
  後續   另一個多實驗室複製找到「統計顯著」但效果量極小（d ≈ 0.1，
         小到實務上幾乎沒意義，且做法仍有爭議）

  解讀（誠實版，別過度伸張）：
    這不證明「意志力一定不存在」，
    但它強烈說明：原始那個又大又漂亮的效應，
    很可能是「小效應 ＋ 發表偏誤 ＋ 研究者自由度」放大出來的。
    重點不是「心理學都假」（那是 ch09 T7 的陷阱），
    是「重複把一個撐了二十年的僥倖過濾掉了」——這正是糾錯機制在運作。
```

注意我在這裡的措辭有多小心，這是刻意的（守住 landscape C-replication 的紀律）：我不說「自我耗竭被證明是假的」，我說「大型預註冊複製未能支持它原本宣稱的那種效果（2026-06）」。因為複製本身也可能設計有限、條件不同；科學的誠實要求對複製失敗也保持同樣的暫定態度（這是 ch10「科學的真永遠暫定」在這裡的回聲）。但**方向**很清楚：一個沒被獨立複製就紅了二十年的效應，禁不起複製的拷問。

## 脊椎透鏡：塞麥爾維斯其實被複製過——但複製救不了他

現在換這個透鏡看脊椎。**這是脊椎案的塞麥爾維斯洗手案，現在換「複製」透鏡看。**

前面幾章你看到的塞麥爾維斯，是一個有觀察（兩病房死亡率差，每千人版 98.4‰ vs 36.2‰）、有準隨機對照（單雙日交替分配，見 ch05）、有戲劇性介入效果（氯化石灰洗手後，1848 年第一病房全年降到約 1.27%，見 ch01）的人。

本章要補上一個常被遺漏的事實：**他的結果被複製過。**

塞麥爾維斯離開維也納後，在匈牙利的醫院（佩斯的聖羅胡斯醫院）推行同樣的洗手措施，產褥熱死亡率**再次**下降。這不是同一個病房的同一批數據——是**另一個地點、另一批產婦、另一個時間**重做了一次介入，效果重現。用本章的語言：這是一次獨立複製，而且成功了。按理說，這應該是壓垮反對者的證據——一次降死亡率可能是僥倖，兩個不同醫院都降，僥倖的解釋就很難成立了。

然後，最痛的一點來了：**複製成功，並沒有讓他被接受。**

```text
塞麥爾維斯案 × 本章各修補制度的對帳

  制度          他有沒有做到              結果
  ─────────────────────────────────────────────────────────
  複製          有（佩斯醫院再次降死亡率） 效果重現，仍被否決
  獨立性        部分（不同醫院、他本人推） 主試者仍是他，獨立性不足
  開放資料/溝通  幾乎沒有（發表慢、文筆差） 數據沒有有效流通、說服力打折
  預註冊        不適用（時代沒有此概念）   ——
  機制          完全沒有（無細菌理論）     ← 致命傷，複製也補不了這一格
```

這張表逼出本章最重要、也最反直覺的一課：**複製是必要的，但不是充分的。** 塞麥爾維斯把「複製」這一格做到了（在當時的條件下難能可貴），結果依然被整個醫界否決二十年。為什麼？因為他缺的不是「再多一次證據」，他缺的是**機制**——他說不出「為什麼洗手有效」。沒有細菌理論（Pasteur／Koch 要到他死後才立起），他的複製成功在反對者眼裡只是「又一個無法解釋的巧合」，而一個無法解釋的巧合，無論重現幾次，都很容易被當時的瘴氣典範「解釋掉」（這就是 ch04 提的 Duhem–Quine：反例總能用調整輔助假設來吸收）。

這裡藏著一個給現代科學的警告，也是本章接 ch14 的橋：**我們今天如此推崇複製、把它當再現性危機的解藥，會不會反過來低估了「光有複製不夠」？** 一個效應被複製一百次，如果完全沒有機制解釋它為什麼成立，它仍然是脆弱的——你不知道它在什麼條件下會失效、會不會其實是某個所有複製都共享的隱藏混淆。塞麥爾維斯的悲劇不是「他的證據不夠」，是「他的證據缺了機制這一塊」。複製過濾僥倖，機制提供理解——兩者都要。這一塊，ch14 接著講。

## 「危機」這個詞，其實是好消息

把本章收在一個會讓你愣一下的觀點上。

「再現性危機」這個詞聽起來像訃聞——好像科學病了、垮了、不可信了。媒體很愛這個敘事，因為「科學破產」比「科學在做正常的自我維護」好賣。但如果你把本章從頭讀到這裡，你應該已經看出來：**危機被揭露、被命名、被吵、被催生出一整套修補制度——這整個過程，本身就是科學糾錯機制在公開運作的證據。**

對照一下「不會認錯」的體系（這是 ch12 的核心對比）。占星、順勢療法、各種偽科學，它們沒有「再現性危機」——不是因為它們的結果都複製得出來，恰恰相反，是因為**它們從不複製、從不公開檢討、從不認錯**。一個從不檢查自己錯在哪的體系，當然永遠不會有「危機」——它只是一直錯下去而沒人去量。**會爆出危機，是因為有人在認真地量自己對不對。** 偽科學的「歲月靜好」才是真正該擔心的。

```text
兩種體系面對「我可能錯了」

  科學（危機後）：       公開複製 → 發現一批效應站不住 → 命名危機
                       → 改制度（預註冊/註冊報告/開放資料/多實驗室）
                       → 「危機」= 糾錯正在運作的噪音

  偽科學/不認錯的體系：   不複製 → 不檢查 → 永遠沒有「危機」
                       → 安靜 = 從來沒在量自己對不對
                       → 「歲月靜好」= 糾錯機制根本不存在
```

當然，這份樂觀要誠實地打折（不造神也不唱衰）。糾錯**正在**運作，不代表已經修好——有研究指出（2026-06），那些已經被證明複製不出來的結果，在危機被揭露多年後**仍然被照常引用**，而且引用時很少有人提它複製失敗。改制度容易，改整個學術界的引用習慣與獎勵結構難。所以正確的態度是：危機是科學健康的證據（它在糾錯），同時糾錯遠未完成（積習難改）。這份「在修、但還沒修完」的張力，比「科學破產」或「科學萬歲」任何一個單音都更接近真相——也更接近你對自己系統的態度：發現一個 class of bug 並建立了防護，是好事；但你不會因此宣布「我們再也不會有這種 bug 了」。

回到中央張力：**科學不是證明真理，是有系統地把自己被騙的機會壓到最低。** 本章講的每一道制度——複製過濾僥倖、預註冊凍結球門、註冊報告打開抽屜、開放資料消滅「在我機器上會動」、多實驗室拉高獨立性——堵的都是同一類自欺：**「一次僥倖被當成穩定事實」**。而「危機」這個詞最深的一層意思是：科學是少數會主動去找自己哪裡騙了自己、找到了還大聲說出來的人類事業。這件事本身，值得你對它多一分敬意——和多一分監督。

## 直覺的陷阱

| 陷阱 | 錯誤直覺 | 在哪一步帶你下溝 | 正確版 |
|---|---|---|---|
| **危機 = 科學破產**（守 ch09 T7） | 「複製率只有三分之一到四成，所以心理學／科學都是假的、不可信」 | 把「一批特定效應未穩健複製」過度概括成「整個領域都假」；又或反過來宣稱「全部都假」 | 那是**對特定一批研究**的複製率，而且它催生了制度修正——**危機是糾錯機制在運作，不是破產**。誠實版兩邊都不站：不說「都假」，也不說「沒事」 |
| **複製成功 = 結論確立** | 「我重做一次也顯著，所以這效應坐實了」 | 忽略了**獨立性**——同一個人用同一套可能有偏的流程重做，偏誤會忠實重現 | 有鑑別力的是**獨立**複製（別人、別的樣本、別的實驗室）；自己重做十次，不等於十份獨立證據 |
| **預註冊 = 不准探索** | 「凍結了分析計畫，就不能做任何計畫外的分析了」 | 把預註冊誤解成枷鎖，於是抗拒它 | 預註冊不禁止探索，它只要求你**誠實標明**哪些是事先說好的「驗證」、哪些是事後發現的「探索」。探索性發現完全可以做、可以報，只是不能假裝成驗證性結果 |
| **預註冊 = 萬靈丹** | 「只要預註冊了，這研究就可信」 | 把一道防線當成全部，放鬆對設計本身的審視 | 預註冊只堵「事後挑分析」這一格——你可以預註冊一個爛實驗、不具代表性的樣本。而且當「有沒有預註冊」變成品質徽章本身，它就開始被玩弄（2026-06）。它是必要防線之一，不是品質保證 |
| **複製夠了就行** | 「只要能反覆複製，就不需要知道為什麼」 | 用塞麥爾維斯的反例：他的結果被複製過，仍被否決——複製過濾僥倖，但補不了**機制**那一格 | 複製是**必要不充分**。沒有機制，一個複製一百次的效應仍是脆弱的（不知何時失效、是否共享隱藏混淆）。複製＋機制才完整（接 ch14） |

**具名陷阱（本章主推）——「複製成功就坐實了」的獨立性盲點**：這是工程師特別容易踩的，因為我們習慣「重跑一次過了就安心」。但科學複製的力量**全部來自獨立性**。你自己把 flaky test 重跑 100 次仍然綠，這有意義——因為環境的隨機性提供了某種獨立性。但如果你的 test 之所以綠是因為它 mock 掉了那個真正會出問題的依賴，那你重跑一萬次都綠、上線照樣炸——因為一萬次都共享同一個「mock 掉真依賴」的系統性偏差。自我察覺的方法：問「我這 N 次重複，有沒有換掉那個最可能藏著偏差的環節？」如果每一次都共享同一個可疑環節（同一個人、同一套流程、同一個 mock），那 N 次重複的可信度增益遠小於你以為的。

## 紙上推演

### 推演題

**題 1（★★，[15 分鐘]）—— 把 A/B test 改造成「預註冊」式**
你的團隊要驗證「新版推薦演算法會不會提升使用者留存」。目前的習慣流程是：上線新版，每天看 dashboard，盯著留存、轉換率、停留時間、點擊率、客訴數這五個指標，哪個先動就先講哪個，跑到某個指標達到顯著就喊「新版贏了」收手。請設計一個「預註冊」式的版本，明確寫出你會在**上線前凍結哪些東西**，並指出原本流程的每一步分別在 p-hack 哪個漏洞。

**題 2（★★，[12 分鐘]）—— 為什麼「危機被揭露」是健康的證據**
你的朋友看到新聞標題「研究發現：超過六成心理學經典實驗無法重現，科學陷入信任危機」，下結論說「看吧，科學跟算命沒兩樣，都是唬人的」。用本章與 ch12 的工具，寫一段話反駁他——但要**誠實**，不能滑到另一個極端（「科學完美無瑕」）。

**題 3（★★★，[20 分鐘]）—— 複製成功，為什麼還不夠？**
塞麥爾維斯的洗手結果在佩斯醫院被獨立複製、效果重現，他卻仍被醫界否決。請論證：(a) 為什麼「再多一次複製」救不了他；(b) 一個被複製一百次、但完全沒有機制解釋的現代效應，可能在哪些情況下仍然是錯的或脆弱的（至少舉兩種）；(c) 這對「我們今天是不是太推崇複製、低估了機制」這個問題，給你什麼啟示。

### 推演解答

**題 1 解答。**

預註冊式版本——上線前（看到任何資料之前）凍結這五樣，戳時間戳存檔（哪怕只是存進團隊的 RFC／實驗登記文件）：

```text
上線前凍結（驗收標準先行）：
  1. 主要指標：留存率（7 日）        ← 只此一個是「驗證性」，事先指定
     （其餘四個若要看，標明為「探索性」，結果只能當線索不能當結論）
  2. 假設與方向：新版 ≥ 舊版留存，預期提升 ≥ X%（事先定 X）
  3. 樣本量／實驗時長：事先用功效分析算好要收多少、跑幾天（不看資料）
  4. 停止規則：收滿預定樣本/時長才看結果（不准「達標就停」）
  5. 成功定義：主要指標上 p < 預定門檻，且效果量 ≥ 實務顯著的下限

原流程的每一步在 p-hack 什麼：
  「盯五個指標、哪個先動講哪個」 → 多重比較（測 5 個，總有一個碰巧顯著）
  「跑到顯著就停」              → optional stopping / 偷看（peeking），灌爆偽陽率
  「哪個先動先講」              → 事後挑指標（HARKing：事後把假設改成「正好顯著」那個）
```

常見錯路：(1) 以為「預註冊就不能看另外四個指標了」——不對，可以看，只是必須標明是探索性、結論不能建立在它們上（見陷阱表第三列）。(2) 凍結了主要指標卻忘了凍結**停止規則**——只要還能「達標就停」，偽陽率照樣被灌爆，這是最常被漏掉的一格。(3) 把「探索性發現」寫進結論假裝是驗證——這就是 HARKing，正是預註冊要堵的。

**題 2 解答。**

一段誠實的反駁，骨架如下：

「那個『六成無法重現』的數字（實際上約三分之一到四成複製成功，見 ch09），是科學界**自己去複製、自己量出來、自己公開**的——這恰恰是算命做不到的事。算命從來沒有過『再現性危機』，不是因為它的預言都準，是因為**它從不去檢查自己準不準、從不公開複製、從不認錯**。一個會爆出危機的體系，是因為有人在認真量自己對不對；一個歲月靜好的體系，往往是根本沒在量。而且科學爆出危機後做了什麼？它發明了預註冊、註冊報告、開放資料、多實驗室複製去修——糾錯機制公開運作。所以正確的結論不是『科學跟算命一樣』，正好相反：**會自曝其短並動手修補，正是科學跟算命最不一樣的地方。**」

但要誠實打折（不能滑到「科學完美」）：「同時也得承認，糾錯**還沒修完**——複製失敗的結果至今仍被照常引用（2026-06），獎勵結構積習難改。所以結論是『科學在修、但還沒修完』，不是『科學沒問題』。」

常見錯路：被朋友激到，反向過度辯護成「科學從不犯錯／危機是假議題」——那就把 ch09 揭露的真問題給否認掉了，也違反本書「不造神」的紀律。誠實的反駁必須兩邊都不站：不接受「都假」，也不接受「沒事」。

**題 3 解答。**

(a) 為什麼再多一次複製救不了塞麥爾維斯：他被否決的關鍵不是「證據量不夠」，是**缺機制**。在瘴氣典範下，一個無法解釋為什麼成立的相關，無論重現幾次，都能被反對者用「調整輔助假設」吸收掉（Duhem–Quine，見 ch04／ch08）——「那只是巧合」「那家醫院還有別的不同」「你的數據有問題」。複製增加的是「效應真實存在」的信心，但它**完全不觸及**「為什麼存在」這個問題；而當時要說服人接受一個違反主流理論的主張，缺的正是後者。複製過濾僥倖，但它對「機制空白」這一格無能為力。

(b) 一個被複製一百次但沒有機制的現代效應，仍可能錯/脆弱的情況（舉兩種）：

```text
情況一：所有複製共享同一個隱藏混淆
  一百個實驗室都用同一套有偏的量測工具/受試者池/實驗範式，
  那個偏差在每一次複製裡忠實重現 → 一百次「成功複製」其實是
  一百次重現同一個系統性錯誤（獨立性是假的）。
  沒有機制，你無從預判「這個共享環節會不會就是混淆源」。

情況二：效應真實，但適用域未知（無機制 = 無法預測何時失效）
  效應在所有測過的條件下都成立，但你不知道「為什麼」，
  所以你不知道它在沒測過的條件下還成不成立。
  一旦推廣到新環境（新人群、新尺度、新情境），可能突然失效，
  而你毫無預警——因為你從沒有過一個能告訴你「邊界在哪」的機制。
```

(c) 啟示：今天把複製當再現性危機的解藥，是對的、必要的——但若因此以為「只要複製得出來就萬事大吉」，就會低估機制。塞麥爾維斯案是這個低估的反向預警：他**有**複製、**缺**機制，結果不被接受、也（在反對者的懷疑裡）始終帶著「會不會是別的原因」的陰影。理想的科學結論需要**複製（過濾僥倖）＋機制（提供理解、劃定適用域、抵抗『其實是別的原因』的質疑）**兩者兼備。這正是 ch14 的主題：機制之必要。

常見錯路：把這題答成「所以複製沒用」——剛好相反，複製是必要的，題目要的是看出它**不充分**。另一條錯路是只答 (a) 的史實、不抽象到 (b) 的一般原理——本書要的是能遷移到你自己系統的直覺，不是背塞麥爾維斯的傳記。

## 自我檢核

口頭自答，講得出來才算過關。優先答「為什麼」。

1. 為什麼「一次顯著結果」分不出「真效應」和「5% 的運氣」？複製靠什麼機制把這兩者分開？（提示：跟 flaky test 跑 100 次同構）
2. 複製的防自欺力量，**核心**來自「做得多嚴謹」還是「獨立性」？為什麼自己重做十次，不等於十份獨立證據？
3. 危機之前那套「獎勵結構」，為什麼是**結構性地**放大偽陽、過濾糾錯？「檔案抽屜問題」是這張表裡的哪一格？
4. 預註冊堵的是哪**一種**自欺？它跟你「先寫驗收標準再實作」是同一件事的哪個對應？它**不**保證什麼？
5. 註冊報告跟普通的「先預註冊再做」差在哪？「原則上接受」這個承諾，拆掉了獎勵結構表裡的哪一格？
6. 為什麼說「再現性危機被揭露」反而是科學健康的證據？這個論點要怎麼**誠實地打折**，才不會滑成「科學完美無瑕」？
7. 塞麥爾維斯的結果**被複製過**，為什麼複製成功仍救不了他？這對「複製是不是萬靈丹」說明了什麼？（接 ch14）
8. 本章在堵哪一種自欺？用中央張力的句型講一遍（「一次僥倖被當成穩定事實」這條主線怎麼貫穿複製／預註冊／註冊報告／開放資料／多實驗室？）

## 延伸閱讀

- **Open Science Collaboration, "Estimating the Reproducibility of Psychological Science"（*Science*, 2015；OSF: osf.io/ezcuj）**：再現性危機的引爆論文，複製 100 項研究的那一篇。本章談「修補」，這篇是「診斷」的原始出處（細節歸 ch09）；讀它的方法論部分，看「大規模獨立複製」實際怎麼組織。
- **Center for Open Science（OSF, osf.io）與 Registered Reports（cos.io/rr）**：預註冊與註冊報告的實際平台與規範。想看「先凍結驗收標準」在科學界長什麼樣、註冊報告流程怎麼跑，看這裡的指引（工程師會覺得親切：很像 RFC＋測試規格先行）。
- **Simmons, Nelson & Simonsohn, "False-Positive Psychology"（*Psychological Science*, 2011）**：研究者自由度／p-hacking 的奠基文。本章的「為什麼要凍結分析計畫」是它的反面——讀它你會懂預註冊到底在防什麼（機制細節見 ch09）。
- **"Crisis or self-correction: Rethinking media narratives about the well-being of science"（PMC5856501）**：直接論證「危機 vs 自我糾錯」這個敘事張力的文章。本章最後一節的觀點來自這條線，想看更完整的兩面論述讀它。
- **Hagger 等的多實驗室自我耗竭複製（2016）與後續 Many Labs 複製**：本章自我耗竭案例的出處。讀它感受「多實驗室預註冊複製」如何把一個紅了二十年的效應拷問到趴下——同時注意作者們措辭多謹慎（沒人宣稱「證明意志力不存在」），這份謹慎本身就是 ch10「科學的真永遠暫定」的示範（引用前查 2026 最新狀態）。
- **《馴服隨機》ch19**：複製率、多重比較、發表偏誤這些數字「怎麼算」的統計機制。本章只講制度修補、刻意不算，要補計算肌肉去那本（談「統計如何崩壞」的那章）。
- **《思考的陷阱》談動機性推理／自我合理化的那章**：為什麼真誠的研究者會抗拒預註冊、會無意間 p-hack——那是心理機制，歸偏誤書。本章只講「制度怎麼從外部堵這個偏心」，不講「大腦內部為何這樣」。
