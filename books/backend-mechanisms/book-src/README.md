# 機制之下

**後端與分散式系統如何運作、為何如此**

這是一本可以坐下來讀的後端與分散式系統技術書。它不是速查表——每一章帶你**真正讀懂一個機制**：它解什麼問題、底下怎麼運作、買到什麼保證、付什麼代價、在哪裡會壞、以及為什麼最後是這個形狀。讀完一章，你腦中應該有一條理解的脈絡，而且能向另一個工程師把這個機制的細節講清楚。

## 這本書的脊椎

每一章底下都是同一個問題：**你想要什麼保證（guarantee），你願意付什麼取捨（trade-off）。** 共識買到「絕不腦裂」，代價是少數派停寫；快取買到低延遲，代價是一致性窗口；非同步複製買到寫入低延遲，代價是切換時可能丟資料。本書的寫法，是讓這些取捨**作為「故事裡的後果」浮現**——先看到天真做法在哪裡破掉，機制怎麼補，補了之後又欠下什麼新的債。

## 寫給誰

一般資深後端／軟體工程師（約 3–8 年實戰）：會寫程式，HTTP／資料庫／交易／並發／雲端都用到工作級，但對底層「會用、卻沒系統化想透」。本書補的正是那一層，講到機制級。不預設你讀過分散式系統理論（不假設讀過 Paxos／FLP／向量時鐘）。

## 怎麼讀

全書依主題分成 8 個 Part，從**判斷與推動**這兩副眼鏡開始，接著走過訊息交付、資料與一致性、連線與 API、並發與韌性、分散式系統核心、架構與發布、觀測與營運文化。各章大致可獨立閱讀；需要前置機制時，章內會用「（見〈章名〉）」指路。每章結尾的「延伸閱讀」收的是外部的一手來源（RFC、論文、官方文件），想往下深挖時照著走。

時效性事實（版本、授權、產品狀態、預設值）以 2026-06 為基準、隨文標注；技術會變，讀到這類數字時記得它有保鮮期。
