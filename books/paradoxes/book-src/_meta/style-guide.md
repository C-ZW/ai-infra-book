# 寫作風格指南（內部文件，非書籍內容）

本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名（工作名）：《沒說出口的那句：反直覺難題與它們藏起來的假設》。

## 這本書是什麼、不是什麼

- **是**：一本「悖論欣賞」的書。挑二十幾個歷史上真正引發爭論的反直覺難題（蒙提霍爾、賭徒輸光、紅藍眼睛…），每一章重演當年的震撼——一個聽起來理所當然的答案，被嚴謹推理推翻——然後把推翻它的**那一句沒人說出口的假設**攤到桌上。讀完讀者要能：對另一個人講清楚「錯的不是數學，是一句沒說出口的前提」，而且自己動手把正確答案算出來。
- **不是**：
  - 不是解題手冊——不刷題型、不背公式表；每個悖論是一個完整的故事，不是一道練習。
  - 不是嚴格測度論／邏輯教科書——需要的工具（條件機率、吸收態、共同知識運算子、測度）在用得上時當場引入，把嚴謹度誠實標示，不折磨讀者。
  - 不是某個人的故事——**全書面向一般讀者，不寫入任何作者個人經歷、不做職涯敘事、不假設讀者的職業**。橋接一律用普世的機制對機制類比，不用某個行業的私有經驗當錨點。
- **驗收標準（全書七條主線，末章總收官）**：①一個悖論由哪三步構成（直覺的答案／它偷偷加上的假設／嚴謹重建）；②「沒說出口的那句」是什麼意思、怎麼在野外辨認；③條件與資訊怎麼騙人（一個動作洩漏了什麼）；④長期與邊界怎麼騙人（吸收態、無限期望）；⑤共同知識為什麼不等於人人都知道；⑥聚合、傳遞性、隨機的定義怎麼在暗處塌掉；⑦讀者能不能把任一章的震撼用自己的話重講。

## 脊椎（貫穿全書的單一主線）

**每一個悖論都因為一句沒說出口的假設而崩，解法就是把那句話攤到桌上。** 這是全書的骨。它有兩個外顯裝置，每章都要出現：

1. **重演歷史震撼**（敘事引擎）：每章開場重建當年的真實爭論——誰給了自信的錯答案、誰反對、爭論多久。歷史服務於「讓讀者先掉進同一個陷阱」，不是編年史。
2. **「那句沒說出口的話是：___」**（解法母題）：每章的 `## 直覺的陷阱` 段落必須以這一句收尾，明確寫出這個悖論偷偷加上、卻從沒明講的前提。這是全書的招牌節拍，末章會把所有「那句話」收成一張假設類型總表。

脊椎不是一個數字，是一套解剖法：**直覺答案 → 找出被偷渡的假設 → 嚴謹重建**。ch01 教這套解剖法，末章回收，中間每章各示範一次。

## 讀者是誰

面向一般受過教育、對數學好奇的讀者。**不假設任何職業或專業背景。**

- **預設已具備（大學入門機率起點）**：基礎機率（樣本空間、事件、P(A)）、條件機率與 P(A|B) 的記號、獨立、期望值 E[X]=Σ pᵢxᵢ 的定義、基礎組合（C(n,k)、排列）、基礎微積分（極限、簡單求和與積分、幾何級數 Σ rⁿ）。這些可以直接使用，第一次出現給一句話喚醒，不從零教。
- **要當場教（本書主體）**：貝氏更新的直覺、吸收馬可夫鏈與破產機率、共同知識運算子與 k 階知識、公開宣告為何改變賽局、效用 vs 期望值、非傳遞關係、測度與不可測集、指示性資訊（indexical）與可信度。凡是用得上的進階工具，在第一次需要它的那一章當場引入，並標明「更嚴格的版本見 chNN／本書只做到直覺層」。
- **禁止**：不得假設讀者是工程師／某職業；不得用私有行業經驗當橋接錨點；不得寫「你在工作中一定遇過…」這種預設讀者身分的句子。

## 語言規範

- 繁體中文，台灣慣用語。數學名詞中英並用：第一次出現給「中文（English）」，之後可只用中文——條件機率（conditional probability）、期望值（expected value）、吸收態（absorbing state）、共同知識（common knowledge）、貝氏定理（Bayes' theorem）、測度（measure）。
- 人名用通行中譯＋原文，第一次出現附原文：貝特朗（Bertrand）、帕斯卡（Pascal）、費馬（Fermat）、惠更斯（Huygens）、白努利（Bernoulli）、辛普森（Simpson）、阿曼（Aumann）、路易斯（Lewis）、諾齊克（Nozick）、艾爾加（Elga）、巴拿赫（Banach）、塔斯基（Tarski）、班佛（Benford）、孔多塞（Condorcet）、布雷斯（Braess）、帕隆多（Parrondo）、瑪麗蓮·沃斯·莎凡（Marilyn vos Savant）。
- **禁止簡體字與大陸詞彙**：機率（不是「概率」）、資訊（不是「信息」）、預設（不是「默認」）、記憶體、演算法、若且唯若（不是「當且僅當」）。
- 第二人稱「你」可用來對讀者說話（引導讀者跟著推理），但**不得用來假設讀者的身分或經歷**。

## 數學記號規範（全書一致，附錄 A 是速查表；閱讀器不渲染 LaTeX，一律純文字）

- **內文行內**用 Unicode：P(A|B)、E[X]、Σ、∫、√、≈、≠、≤、≥、→、∞、½、⅓、⅔、πᵢ。分數在行內優先寫成 2/3、1/1000；關鍵結論可用 ⅔ 這類 Unicode 分數強調。
- 條件機率一律寫 `P(A|B)`；貝氏定理寫 `P(H|E) = P(E|H)·P(H) / P(E)`。
- 求和寫 `Σ(n=1→∞) 2ⁿ·(1/2ⁿ)`；期望值寫 `E[X] = Σ pᵢxᵢ`。
- **多行推導**放在 ` ```text ` 圍欄區塊裡，等號對齊、每步右側可加「← 為什麼」短註。禁止把長推導硬塞進行內；本書不迴避數學，長推導要完整、不跳步，但每一步都要有一句白話理由。
- 圖：優先用**程式生成圖**（見下節）；純結構關係（歸納表、決策樹、路線圖）可用 ASCII 圖，必須通過 `check_diagrams.py`（CJK=2 欄寬）。曲線圖不追求精確比例，追求形狀直覺。

## 程式生成圖規範

- 每張圖是一支自足的 `figures/chNN-name.py`（matplotlib，Agg backend），輸出到 `figures/out/chNN-name.svg`；用 `build_figures.py figures/` 一次跑完。程式與註解一律英文，每支 ≤ 60 行。
- 內文以 `![說明](figures/out/chNN-name.svg)` 內嵌。**圖說（caption）內不得出現 `]` 字元**（會讓打包器不內嵌圖片，2026-06 教訓）；需要區間時用「0 到 1」而非「[0,1]」。
- 圖說要交代「看點」：這張圖要讓讀者看出什麼（例如「切換後的長條是留下的兩倍——因為主持人的動作把機率灌到沒開的那扇門」）。
- 每章視主題配 0～2 張圖：機率樹、蒙地卡羅長條圖、漫步路徑、破產機率曲線、共同知識歸納樹、辛普森反轉散點、非傳遞骰子循環、弦悖論三種取法。純邏輯章（意外絞刑、兩位將軍）可以零張圖，用 ASCII 歸納表即可。

## 每章骨架（必須遵守，標題名稱照抄——一致性掃描會 grep 這些標題）

1. 章首：`# chNN — 章名：副標`，接著 blockquote：
   `> **本章解決什麼問題**：…（這章在全書地圖的哪裡、屬於哪個悖論家族、跟前後章的關係，3-5 行）`
   每個 Part 的第一章在 blockquote 後放全書 ASCII 路線圖（基準圖在 outline.md，照抄並把「◄ 你在這裡」標到本章）。
2. `## 從你已知的出發` — 開場：重建歷史場景，把悖論的設定講清楚，讓讀者先給出那個**自信的錯答案**，並說明它為什麼聽起來理所當然。這裡不揭曉正解。
3. 核心內容（多個 `##` section）：重演爭論 + 嚴謹推導 + 圖 + 揭曉正解。歷史與數學交織，但**歷史服務於數學直覺**。至少一個 worked example，帶具體數字、一步一步算到底、不跳步。
4. `## 直覺的陷阱` — 本書招牌段落：這個悖論**偷偷加上了哪一句沒說出口的假設**、錯誤直覺**在哪一步被那句話帶溝裡**、**怎麼自我察覺**。用表格或條列。本段**必須**以定型句收尾：
   `> **那句沒說出口的話是**：____________。`
5. `## 紙上推演` — 動手做，每章 2~4 題，標注預估時間（如 **[15 分鐘]**）與難度（★~★★★）。題型：手算機率表、把白話設定翻成數學、找出一個錯誤論證的破綻、造一個變體並預測結果、把某個證明口頭重講。題目後緊接 `### 推演解答` 給完整解答與討論（為什麼這樣想、常見錯路）。
   **章節內文不放任何可執行程式碼區塊**（不要 `### Python 小實驗`、不要 ```python）。視覺化一律用 figures/ 的程式生成圖（內嵌 SVG）；模擬「結果」與數值步驟用 ```text 圍欄呈現，不是可跑的程式碼。
6. `## 自我檢核` — 5~8 個問題，**口頭自答**：講得出來才算過關。優先出「為什麼」而不是「是什麼」；至少一題是「這個悖論那句沒說出口的假設是什麼」。
7. `## 延伸閱讀` — 真實連結（以 `_meta/landscape-history.md` 驗證過的為優先），每條一句話說明為什麼值得讀。

## 深度標準

- **每章至少一個 worked example**：帶具體數字、一步一步算的完整過程（機率表、破產機率代入、貝氏更新、歸納逐日表），不准跳步。outline 已為每章指定基準範例與基準數字。
- **本書不迴避數學**：該推導就推導完整，該給封閉解就給（破產機率 1−i/N、貝氏後驗、幾何級數和）。但每個符號出場都要有白話直覺在旁邊；嚴謹度要誠實標示——「直覺版」或「完整證明」或「本書只做到 sketch，嚴格版見延伸閱讀」。禁止「顯然」「易證」，禁止把直覺論證偽裝成證明。
- **數值自我複核**：所有數值（機率、期望值、破產機率、天數）寫章時必須自己重算一遍，不得抄自記憶。跨章基準數值以 outline 基準表為準，引用時必須與基準表一字不差。
- **歷史事實照 landscape-history**：年份、歸屬、引文、軼事一律以 `_meta/landscape-history.md` 為準；沒有的自行 WebSearch/WebFetch 查證（每章 3~8 次）並 hedge；傳說要標明是傳說。禁止憑記憶引用名人名言或「一萬封讀者來信」這類數字——照 landscape。
- **每個震撼點要能被重講**：本書的核心商品是「原來錯在這裡」的恍然。每章至少一處明確回答「那句沒說出口的話到底是什麼、為什麼藏得住」，而且要寫到讀者能用自己的話轉述。
- 寧深勿廣，不灌水。每章 5,000~9,000 中文字，依主題深度調整。

## 橋接契約（客觀、不疲勞的類比）

- 類比一律**機制對機制**，不用日常生活軟性比喻，不用第二人稱套關係（不寫「就像你早上…」）。每章至多一個外部類比，且要明確標錨點與邊界（類比自哪個概念、在哪裡成立、在哪裡失效）。
- 本書最強的橋接來源是**書內已定義的概念**：後面的悖論優先橋接回前面章節（例如三囚犯←→蒙提霍爾、泥巴小孩←→紅藍眼睛、檢察官謬誤←→基率謬誤）。跨章引用寫「（見 chNN）」。
- 「不橋接、直接教」是合法且常見的選擇：沒有強結構對應的悖論，以及末章，直接教即可。橋接來源分配見 outline 的 Bridge Registry，每個來源全書至多用 3 次。

## 事實正確性

- 歷史與時效性事實（年份、人物、出版品、著名爭論的數字）以 `_meta/landscape-history.md` 為事實基準；本章特定細節自行用 WebSearch/WebFetch 查證（每章 3~8 次）。
- 數學內容（定義、定理、計算）不依賴 landscape，但必須自我複核；常被誤傳的事實（蒙提霍爾主持人規則、聖彼得堡的首倡者、熱手的 Miller–Sanjurjo 修正、共同知識 vs 相互知識）以 landscape 查證版為準。
- 延伸閱讀只放真實存在的連結；沒把握標「（未驗證）」。

## 格式與語氣

- Markdown。主要層級 `##`，次層 `###`。多行推導用 ` ```text ` 區塊；Python 用 ` ```python ` 區塊。表格用於機率表、對照表、陷阱清單、歸納逐日表。跨章引用寫「（見 chNN）」，不寫死檔名。
- 語氣：冷靜、直接、有觀點（可以寫「我認為這是全書最漂亮的一句沒說出口的話」）。不行銷、不喊口號、不堆「太神奇了」式空洞感嘆——震撼要用內容本身造成。允許冷幽默。該慢的地方明說「這裡值得你停十分鐘」。
