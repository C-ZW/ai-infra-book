# ch05 — 連續統的怪事：線、面、空間一樣多

> **本章解決什麼問題**：上一章（見 ch04）對角論證證明了 ℝ 數不完，逼出一個比 ℵ₀ 嚴格大的新基數 𝔠 = |ℝ|。但這個新基數本身極度反直覺：一條短短的線段，竟然和整條無限長的直線一樣多；一條一維的線，竟然和整個二維平面一樣多。這章把 𝔠 的這幾件怪事正面攤開，給出明確的一一對應、誠實標示它們哪裡不完美、並守住一個關鍵分寸——這些怪事一個都不破壞「𝔠 > ℵ₀」。維度不增加基數，但可數到不可數仍然是真正的跳躍。順帶用基數一句話秒殺「超越數幾乎是全部」，留下存在性與構造性鴻溝的伏筆（見 ch12）。

## 從你已知的出發

你寫過這個東西，只是沒給它取名字。

假設你要把二維地理座標 `(lat, lon)` 壓成一個一維的排序鍵，讓地理上相鄰的點在 B-tree 裡也盡量相鄰——這樣範圍查詢才不會掃整張表。標準作法叫 **Z-order**（又稱 Morton code）：把 `lat` 和 `lon` 的位元交錯排列。`lat = a₁a₂a₃…`、`lon = b₁b₂b₃…`，交錯成一個鍵 `a₁b₁a₂b₂a₃b₃…`。一個整數，編碼了兩個座標。PostGIS、Iceberg、Databricks 的 data skipping 底下都躺著這招或它的表親 Hilbert curve（空間填充曲線）。

你做這件事的目的是工程的——locality、index、少掃幾個 partition。但你其實在親手示範一件集合論裡最讓人不敢相信的事：**平面上的點，並不比直線上的點多。** 一個 (x, y) 可以被忠實地編碼成一個數 z，這正是「線和平面一樣多」的構造本體。

康托爾（Cantor）在 1877 年發現這件事時，寫了一句後來人人引用的話給戴德金（Dedekind）：「**je le vois, mais je ne le crois pas**」——我看見了，卻不敢相信。連發現者本人都覺得難以置信。這一章就是要把這份難以置信走清楚：先看「短線段＝整條直線」，再看「線＝平面」，然後守住分寸——這些一一對應全都成立，但 𝔠 > ℵ₀ 依然成立。維度可以白送，可數那一道牆卻翻不過去。

還有一個你天天碰、卻沒意識到的事實會在這章登場：你寫的每一行能跑的程式碼，幾乎一定是個「無聊」的代數數運算；但你隨手在數線上戳一個點，那個實數幾乎一定是超越數。我們認識的數和真正存在的數，幾乎是兩個不相交的世界。

## 第一件怪事：有界線段和整條直線一樣多

先把目標講白。開區間 (0, 1) 是一條長度有限的線段——量得出長度 1。整條實數線 ℝ 無限長——量不出長度。直覺尖叫：整條線「明明多得多」。

但「多少」的定義不是長度，是能不能一一對應（見 ch01）。所以正確的問法是：能不能造一個 f：(0, 1) → ℝ，不重（單射）不漏（滿射）？

能。而且不只一個。我給你兩個，並且都驗證到底。

### 構造一：x ↦ x/(1−|x|)（先把 (0,1) 搬到 (−1,1)）

這個公式定義在 (−1, 1) 上比較對稱，所以分兩步。

第一步，把 (0, 1) 線性搬到對稱的 (−1, 1)：

```text
s: (0,1) → (−1,1),  s(t) = 2t − 1
```

t=0⁺ 對到 −1⁺，t=1⁻ 對到 1⁻，t=0.5 對到 0。線性、嚴格遞增（斜率 2 > 0），值域剛好掃滿 (−1,1)，所以是 (0,1) ↔ (−1,1) 的一一對應——逆映射 t = (x+1)/2 寫得出來，不重不漏，這一步不必停留。

第二步，把 (−1, 1) 攤開成整條 ℝ：

```text
g: (−1,1) → ℝ,  g(x) = x / (1 − |x|)
```

來驗證 g 真的是一一對應，逐項走，不跳步：

```text
端點行為  x → 1⁻：分母 1−x → 0⁺，分子 x → 1，  g(x) → +∞
          x → −1⁺：分母 1−|x| → 0⁺，分子 x → −1， g(x) → −∞
中點      x = 0：g(0) = 0/1 = 0
單調性    在 [0,1) 上 g(x) = x/(1−x)，分子增、分母減 ⇒ 嚴格遞增
          在 (−1,0] 上 g(x) = x/(1+x)，同理嚴格遞增
          x=0 兩段接得上（都過 0）、連續 ⇒ 整個 (−1,1) 上嚴格遞增
不漏      g 連續、嚴格遞增、從 −∞ 掃到 +∞ ⇒ 每個實數都被某個 x 命中（滿射）
不重      嚴格遞增 ⇒ x₁ ≠ x₂ ⇒ g(x₁) ≠ g(x₂)（單射）
```

不漏那一步用到「連續函數從 −∞ 掃到 +∞ 之間每個值都取得到」——這是 ℝ 的完備性（中間值定理），本書不展開，它的根在《馴服無限》（見《馴服無限》ch14，ℝ 從 ℚ 的構造）。

再給逆映射當確認，逆映射存在且能寫出來，是「真的一一對應」最硬的證據：

```text
反解 y = x/(1−|x|)：
  x ≥ 0 時 y = x/(1−x) ≥ 0 ⇒ x = y/(1+y)
  x < 0 時 y = x/(1+x) < 0 ⇒ x = y/(1−y)
合起來：g⁻¹(y) = y / (1 + |y|)
```

g⁻¹ 把整條 ℝ 收回 (−1, 1)，這正是你前面那個工程公式的本尊——把無界的座標壓進有界的盒子。複合 g ∘ s 就是 (0,1) → ℝ 的一一對應。**一條長度為 1 的線段，和無限長的直線，點一樣多。**

### 構造二：tan（給一個更短的）

如果你信任三角函數，這個一行就完事：

```text
h: (−1,1) → ℝ,  h(x) = tan(πx/2)
```

x → 1⁻ 時 πx/2 → π/2⁻，tan → +∞；x → −1⁺ 時 → −∞；x=0 時 tan 0 = 0。tan 在 (−π/2, π/2) 上嚴格遞增、連續、值域是整個 ℝ。同樣是一一對應，逆映射是 (2/π)·arctan。

兩個構造殊途同歸，都在說同一件事。我偏好寫構造一，因為它沒有「相信三角函數的值域剛好是整條 ℝ」這個隱藏假設——分式版每一步都看得見。

> **這裡了不起在哪**：你可以拿掉一條直線「無限長」這個賣點，把它整條塞進一段量得出長度的線段裡，一個點都不漏、一個點都不重。長度（一種測度）和「多少」（基數）從此分家——這是這一章你最該帶走的一句話，下面還會再撞它一次。

## 第二件怪事：線和平面一樣多（康托爾 1877）

這件事比上一件更難接受。一維的線 ℝ 和二維的平面 ℝ²，基數一樣大。

先把問題縮到單位正方形與單位線段（用上一節的工具，(0,1) ↔ ℝ，所以證 |(0,1)| = |(0,1)²| 就等於證 |ℝ| = |ℝ²|）。我們要造一個對應，把正方形裡的一個點 (x, y) 編成線段裡的一個點 z，並且反過來能還原。

康托爾 1877 年給戴德金的信裡用的就是**小數位交錯**——和你寫 Z-order 時交錯位元，是同一招。

### 交錯構造（先給直覺，再戳破它）

把 (x, y) ∈ (0,1)² 寫成小數：

```text
x = 0.a₁ a₂ a₃ a₄ …
y = 0.b₁ b₂ b₃ b₄ …
```

交錯成一個數：

```text
z = 0.a₁ b₁ a₂ b₂ a₃ b₃ a₄ b₄ …
```

具體跑一遍。設 x = 0.1234…、y = 0.5678…：

```text
x = 0. 1   2   3   4  …
y = 0.  5   6   7   8 …
        ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓
z = 0. 1 5 2 6 3 7 4 8 …    ← 取 x 一位、取 y 一位、輪流
```

要還原？把 z 的奇數位刮出來拼回 x、偶數位拼回 y，就回到 (x, y)。看起來完美：一個 z 唯一決定一對 (x, y)，一對 (x, y) 唯一給一個 z。

**但它不是完美的一一對應。我必須誠實標示這一點，這也是康托爾原始構造真正的技術核心。**

問題出在小數的**雙重表示**：0.4999… = 0.5000…，同一個實數有兩種小數寫法。為了讓 x ↦ 小數表示「是個函數」（一個 x 只配一個寫法），我們規定一律取**不以無窮多個 9 結尾**的那個寫法（即取 0.5000… 而非 0.4999…）。約定之後 x ↦ (aₙ) 是良定義的單射。但麻煩搬家了，沒消失：

```text
取 z = 0.1 9 0 9 0 9 0 9 …   （交錯拆開）
奇數位 → x = 0.1 0 0 0 …  = 0.1
偶數位 → y = 0.9 9 9 9 …  = 0.999… —— 但我們禁止 9 結尾！
```

這個 z 反推出來的 y 違反了「不准全 9 結尾」的約定，也就是說：**存在合法的 z（線段上的點），它不是任何「合法 (x, y)」交錯出來的像。** 交錯映射作為「(0,1)² → (0,1)」這個方向看，**不是滿射**——這正是戴德金當年回信指出的破綻，康托爾原始函數確實漏掉了一小撮點。

### CSB 來補：不完美的兩半各取一半

別把它修補成「完美交錯」——那會越補越醜（要分情況處理連續 9，繞死人）。正確的工程作法是換工具。

我們手上其實有兩個方向都乾淨的**單射**：

```text
方向一  (0,1)² → (0,1)：用上面的交錯（避開全 9 約定後它是單射，雖然不滿射）
方向二  (0,1) → (0,1)²：把 z 配成 (z, ½)，塞進正方形的一條水平線上（不同 z 給不同的 (z,½)，是單射）
```

兩個方向都是單射，於是——**Cantor–Schröder–Bernstein 定理（CSB）**直接保證存在一個真正的一一對應 (0,1)² ↔ (0,1)，不必我們手工把交錯修到完美。CSB 說的就是：兩向單射 ⇒ 一樣多（|A| ≤ |B| 且 |B| ≤ |A| ⇒ |A| = |B|）。它是怎麼從兩個單射拼出一個一一對應的、它的歸屬史為什麼很亂，留到 ch07 正式講（見 ch07）。這裡你只需要知道：**「幾乎一一對應」靠 CSB 升級成「真的相等」是合法的、而且是標準作法。**

於是 |ℝ| = |ℝ²| = 𝔠。**線和平面一樣多。** 康托爾那句「我看見了卻不敢相信」，指的就是這一步。

> 一個歷史的分寸（照本書事實基準）：通俗版愛把這句話渲染成「康托爾被自己的結果嚇傻了」。比較精確的學術考據認為，他當時對結果相當有把握（他覺得自己動搖了「維度」這個幾何基礎），那句話更接近「我看到證明成立、卻還想再三確認沒出錯」的心情。所以我寫「連他自己也覺得難以置信」，不寫「他嚇傻了」。

### 推廣到 ℝⁿ，以及一道不能跨的牆

同一招可以重複：要把 (x, y, w) ∈ ℝ³ 編成一個數，就三路交錯 `a₁b₁c₁a₂b₂c₂…`，一樣用 CSB 補。歸納下去：

```text
|ℝ| = |ℝ²| = |ℝ³| = … = |ℝⁿ| = 𝔠   （任意有限維 n）
```

維度，完全不增加基數。一維、二維、一百維，點一樣多。

**但這裡有一道牆，務必看清楚它在哪**，否則你會得出全錯的結論「所以一切無限都一樣大」：

```text
ℵ₀  <  𝔠 = |ℝ| = |ℝ²| = … = |ℝⁿ|
↑                                  ↑
這一步是真正的跳躍              這一排全部相等
（ch04 對角論證證的，翻不過去）  （維度白送，CSB 隨手補）
```

把線升到平面、平面升到空間——基數紋風不動。但從可數（ℵ₀）升到不可數（𝔠）——基數真的跳了一級，而且這一跳是對角論證鎖死的，沒有任何交錯把戲翻得過去。

為什麼維度免費、可數那道牆收費？直覺上：交錯把「兩個 ℵ₀ 長的數字串」併成「一個 ℵ₀ 長的串」——可數加可數還是可數（見 ch03 的 ℕ×ℕ ↔ ℕ），位數的個數沒變，所以基數沒變。但對角論證造出的那個新序列，是用「跟列表每一列都作對」逼出來的，它逃出了任何 ℵ₀ 長的列舉——這不是位數的併合能搆到的層級。**加乘塌縮、跳級靠別的東西**（取冪，見 ch06／ch07）。

## 第三件怪事：超越數幾乎是全部，但你一個都點不名

這一節短，但它是這章最值得回味的一拳，而且只花一句話的基數論證。

回憶 ch03：整係數多項式可數，每個多項式只有有限個根，可數個可數集的聯集還是可數，所以**代數數（能當某個整係數多項式的根的數）是可數的**——ℵ₀ 個。

而 ch04 剛證了 **ℝ 不可數**——𝔠 個。

把這兩件並排：

```text
ℝ  =  代數數  ∪  超越數
𝔠  =   ℵ₀    ∪   ？
```

如果超越數也是可數的，那 ℝ = 可數 ∪ 可數 = 可數，和「ℝ 不可數」矛盾。所以——

**超越數（transcendental number，不是任何整係數多項式根的實數）必然不可數，而且是 𝔠 個。**

把它翻成人話：數線上代數數只有 ℵ₀ 個、超越數有 𝔠 個，後者嚴格、無可比擬地多。**你閉著眼睛往數線上戳一個點，戳到代數數的「機會」是 ℵ₀ 比 𝔠——幾乎不可能。隨手一個實數，幾乎一定是超越數。**

這裡藏著本書最深的一個對比，我請你停十秒：

```text
存在性： 超越數幾乎是全部（𝔠 個），這個論證三行就證完——而且不指認任何一個。
構造性： 證明某個「具體的數」是超越數，極難。
         e 是超越數，Hermite 1873 才證出來。
         π 是超越數，Lindemann 1882 才證出來。
         比康托爾 1874 的存在性證明，晚了快十年、難了好幾個量級。
```

康托爾 1874 的論文（用的是區間套法，不是對角線——見 ch04 守住這個分野）順手就證明了「每個區間裡都有超越數」，卻**一個都沒指出來**。它說「它們幾乎填滿了數線」，同時說「我不告訴你任何一個長什麼樣」。

這就是**存在性與構造性的鴻溝**：「存在」可以靠數數量（基數）逼出來，「給我看一個」卻可能難上加難。一個習慣「存在就要給得出 `pick()` 實作」的工程師，會在這裡第一次感到不舒服——這份不舒服我們會養大它，一路養到選擇公理（見 ch12）和直覺主義（見 ch17）。

> **這裡了不起在哪**：人類先證明了「絕大多數實數是超越數」，整整一代人之後才好不容易抓出 π、e 兩個具體例子。先知道「幾乎全是」，再知道「哪一個是」——知識的順序和你的直覺完全相反。這是純粹「用基數數數量」帶來的、最便宜也最震撼的一個結論。

## 一句話的 2^ℵ₀

順帶把一個記號接上，正式處理留給後面。

把 (0,1) 裡的實數寫成二進位 0.c₁c₂c₃…，每個 cᵢ ∈ {0,1}，差不多就是一個「ℕ → {0,1} 的無窮二元序列」，也就是 {0,1}^ℕ 的元素（ch04 對角論證打的就是這個集合）。而 ℕ 的每個子集 ↔ 一個特徵序列（第 n 位是 1 ⟺ n 在子集裡），所以 {0,1}^ℕ ↔ 𝒫(ℕ)，個數寫成 2^ℵ₀。把這些「差不多」用 CSB 補成真正的相等之後（同樣是 ch07 的工作）：

```text
𝔠 = |ℝ| = |(0,1)| = |{0,1}^ℕ| = |𝒫(ℕ)| = 2^ℵ₀
```

全書同一個基數 𝔠，五種寫法。這裡你只要先記住「𝔠 也叫 2^ℵ₀」這件事。為什麼取冪 2^ℵ₀ 會嚴格大於 ℵ₀（而加乘不會）——那是冪集定理（見 ch06）和基數算術（見 ch07）的主場。

## 直覺的陷阱

這一章的反直覺密度全書數一數二，正因為如此，錯誤直覺也特別多。逐條拆。

| 錯誤直覺 | 它會在哪一步把你帶溝裡 | 怎麼自我察覺 |
|---|---|---|
| 「線段比直線短，怎麼可能一樣多」 | 你拿**長度**（測度）當「多少」用了。線段量得出長度、直線量不出，這是測度的差別；但「多少」是基數，由一一對應定義。g(x)=x/(1−\|x\|) 把整條線塞進線段、一點不漏——多少相同。 | 你一講出「比較短／比較長」就是徵兆：你在比測度，不是比基數。停下來問「我能不能造一個一一對應」，不要問「哪個比較長」。這一拳呼應 ch03（稠密 ≠ 多）：順序、長度、密度都是別的性質，和「多少」無關。 |
| 「線升成平面，基數當然變大（多了一個維度的點）」 | 你以為維度是一種「乘上去的量」。其實 ℝ² 的點數是 𝔠·𝔠，而無限基數 𝔠·𝔠 = 𝔠（加乘塌縮，見 ch07）。位元交錯就是這個塌縮的具體見證——兩串數字併成一串，位數沒變多。 | 你預期「n 維比 1 維多 n 倍」之類的話，就是徵兆。有限世界 ℝⁿ「大 n 倍」是測度直覺；基數世界裡 ℝⁿ 和 ℝ 一樣大。 |
| 「既然線=平面，那 ℵ₀ 和 𝔠 八成也只是寫法不同、其實一樣大」 | 把「維度免費」過度推廣成「所有無限一樣大」，直接抹掉 ch04 的整章成果。 | 看清楚那道牆畫在哪：相等的是 𝔠=\|ℝⁿ\| 這一排，跳級的是 ℵ₀ < 𝔠 那一步——後者是對角論證鎖死的，任何交錯都翻不過去。維度白送 ≠ 可數也白送。 |
| 「位元交錯是完美的一一對應，我寫 Z-order 都用得好好的」 | 工程上 Z-order 用在**有限位元**（如 32-bit lat × 32-bit lon），有限位元沒有「全 9／全 1 結尾」的雙重表示問題，所以工程版真的是雙射。但無窮小數版有雙重表示，交錯**不是滿射**，要 CSB 補。 | 你說「交錯就是雙射、證完了」就是徵兆：你把有限版的乾淨偷渡到無窮版。誠實的證明在無窮版必須承認交錯只是單射、靠 CSB 升級。 |
| 「超越數很多，但既然 π、e 都是超越數，看來不難找」 | 倖存者偏差。你認得的超越數，全是數學家費大力氣一個一個證出來的少數樣本（π 等到 1882、e 等到 1873）；你「認得」這件事本身就篩掉了絕大多數。 | 你能脫口而出的數（整數、分數、√2、π、e、log…）幾乎全是代數數或少數被證出的超越數。真正「典型」的實數是個你寫不出、念不出、沒有名字的超越數——而它們才是 𝔠 那一大坨。 |

## 紙上推演

純紙筆。每題先自己動手，再看解答。解答裡標了常見錯路。

### 推演一：造一個 (−1,1) ↔ ℝ 的一一對應並驗證 ★ **[15 分鐘]**

不准用本章正文的 g 和 h 原封照抄。自己挑一個公式（提示：tan 或分式都行，但要把驗證走完），證明它是 (−1, 1) → ℝ 的一一對應：寫出端點行為、單調性、為什麼不漏、為什麼不重，最好附上逆映射。

### 推演二：線段比直線短，怎麼會一樣多 ★ **[10 分鐘]**

不准用「就是有個一一對應啊」打發。要對一個「線段明明比較短」的人，把「為什麼短不蘊含少」講到他服氣。明確分開三件事：長度是什麼、多少是什麼、它們為什麼無關。一段話，能口頭講出來。

### 推演三：交錯到底漏了什麼 ★★ **[20 分鐘]**

針對正文的小數位交錯映射 (x, y) ↦ z = 0.a₁b₁a₂b₂…（約定小數一律取「不以全 9 結尾」的寫法）：

(a) 自己找一個**具體的** z ∈ (0,1)，它不是任何合法 (x, y) 交錯出來的像（即：把它拆開後，某個座標違反「不准全 9 結尾」的約定）。

(b) 用一句話說清楚：這證明了交錯映射缺了什麼性質（單射？滿射？），所以為什麼非得請 CSB 來補。

### 推演四：為什麼幾乎所有實數都是超越數，我們卻幾乎只認得代數數 ★★ **[15 分鐘]**

口頭題。把這個對比講清楚：超越數有 𝔠 個（不可數）、代數數只有 ℵ₀ 個（可數），所以「典型」的實數是超越數；但你能叫出名字的數幾乎都是代數數或極少數被證出的超越數。明確點出兩個關鍵詞：**存在性 vs 構造性**，以及為什麼「我認得它」這件事本身有偏差。

---

### 推演解答

**推演一解答。** 以分式版為例（tan 版同理）：取 f(x) = x/(1−x²)。

```text
端點   x → 1⁻：分母 1−x² → 0⁺，分子 → 1，  f → +∞
       x → −1⁺：分母 → 0⁺，分子 → −1，    f → −∞
中點   f(0) = 0
單調   f′(x) = (1+x²)/(1−x²)²，在 (−1,1) 上恆正 ⇒ 嚴格遞增
不漏   連續、嚴格遞增、值域掃過 (−∞,+∞) ⇒ 滿射（中間值定理）
不重   嚴格遞增 ⇒ 單射
```

（這個 f 比正文的 g 更對稱，逆映射要解二次式 x²·y + x − y = 0、取 (−1,1) 內那根，能寫但較煩；若你選了正文的 g(x)=x/(1−|x|)，逆映射乾淨是 y/(1+|y|)。兩者都對。）

**常見錯路**：只證了「嚴格遞增 ⇒ 單射」就收工，忘了證滿射。單調只給你單射；要滿射，得明確指出值域是整條 ℝ（連續＋端點趨於 ±∞＋中間值定理）。漏了滿射，你只證了「(−1,1) 嵌進 ℝ」，沒證「一樣多」。

**推演二解答。** 三件事分開講：

- **長度**是一種**測度**：問「這段佔了數線多長」。(0,1) 長度 1，ℝ 長度無限。這是真的不一樣。
- **多少**是**基數**：問「能不能和另一個集合一一配對」。和測度毫無關係。
- 它們為什麼無關：g(x)=x/(1−|x|) 把整條 ℝ 一點不漏、一點不重地塞進 (−1,1)。如果「短就該少」，這個塞法根本塞不下——但它塞下了。所以「短」（測度小）和「少」（基數小）是兩件事，一個不蘊含另一個。

一句話版本：**長度是「佔多長」，多少是「能不能配對」；一條短繩子上的點，可以和一條長繩子上的點一根一根配完，配不完才叫多，配得完就叫一樣多。** 呼應 ch03：稠密、順序、長度都是別的性質，全與基數無關。

**常見錯路**：用「無限就是無限，所以都一樣」搪塞——這是錯的（ch04 證了無限不只一種），而且沒回答問題。問題問的是「短為什麼不蘊含少」，答案是「短是測度、少是基數」，不是「無限都一樣」。

**推演三解答。**

(a) 取 z = 0.1009090909…（即 0.10 後面 09 無限循環）。拆開：

```text
奇數位（第 1,3,5,…位）：1, 0, 0, 0, … → x = 0.1000… = 0.1     ← 合法
偶數位（第 2,4,6,…位）：0, 9, 9, 9, … → y = 0.0999… = 0.1     ← 但寫法是全 9 結尾
```

y 的這個小數寫法以無窮多個 9 結尾，違反了「一律取不以全 9 結尾的寫法」的約定。也就是說，沒有任何**合法**的 (x, y)（兩座標都用標準寫法）會交錯出這個 z——這個 z 在交錯映射的像之外。

（任何「奇數位或偶數位形成全 9 尾」的 z 都是這種反例；上面只是最好寫的一個。）

(b) 這證明交錯映射**不是滿射**（值域漏掉了像 z 這樣的點），雖然它是單射。「兩向各有單射、但任一向都不易直接做成雙射」正是 CSB 的用武之地：(0,1)²→(0,1) 有單射（交錯）、(0,1)→(0,1)² 有單射（z ↦ (z, ½)），CSB 保證存在真正的一一對應，免去把交錯硬修到滿射的苦工。

**常見錯路**：以為「奇偶位拆開」永遠拆得回去、所以交錯是雙射。拆得回去只說明它是**單射**（不同 z 拆出不同 (x,y)）；雙射還要**滿射**，而滿射正是雙重表示破壞掉的那一半。

**推演四解答。** 要點清單：

1. 代數數可數（ℵ₀）：整係數多項式可數、每個有有限多根、可數個可數集的聯集可數（見 ch03）。
2. ℝ 不可數（𝔠）：對角論證（見 ch04）。
3. 兩者相減：若超越數可數，則 ℝ = 代數數 ∪ 超越數 = 可數 ∪ 可數 = 可數，矛盾。故超越數不可數，且為 𝔠 個。
4. 「典型」實數是超越數：ℵ₀ 相對於 𝔠 幾乎可忽略，隨手取一個實數幾乎一定落在超越數那 𝔠 一坨裡。
5. **存在性 vs 構造性**：「超越數幾乎是全部」三行證完（康托爾 1874）；但證**某個具體數**超越極難（e 等到 1873、π 等到 1882）。存在可以靠數數量逼出，指認卻可能難上加難。
6. **「我認得它」的偏差**：你能叫出名字的數，全是被人類特別命名／特別證明出來的少數樣本——這個篩選本身就排除了絕大多數無名的超越數。所以「我認得的幾乎都是代數數」和「實數幾乎都是超越數」並不矛盾，前者是樣本偏差。

**常見錯路**：把「超越數很多」說成「所以超越數很好找」。剛好相反——多到 𝔠 個，卻幾乎沒有一個有名字。多寡（基數）和可指認（構造）是兩回事，這正是這題要你體會的鴻溝。

## 自我檢核

口頭自答，講得出來才算過。優先答「為什麼」。

1. 「多少」和「長度」差在哪？為什麼一條短線段能和整條直線一樣多，這不矛盾？
2. 給我一個 (0,1) ↔ ℝ 的明確一一對應，並說出你怎麼知道它不漏（滿射）。
3. 康托爾用什麼招把 (x, y) 編成一個數？這招和你寫過的哪個工程技巧是同一件事？
4. 小數位交錯為什麼**不是**完美的一一對應？哪個環節壞了、靠什麼補回來？
5. |ℝ| = |ℝ²| = … = |ℝⁿ| 全部相等，這有沒有推翻「𝔠 > ℵ₀」？那道「翻不過去的牆」畫在哪？為什麼維度免費、可數收費？
6. 為什麼「超越數幾乎是全部」三行就證完了，「證某個具體數超越」卻可能難到等一代人？這個對比有個名字，叫什麼？
7. 為什麼我們認得的數幾乎都是代數數，這和「實數幾乎都是超越數」並不矛盾？

## 延伸閱讀

- **Fernando Q. Gouvêa, "Was Cantor Surprised?"（*American Mathematical Monthly*）**：對「je le vois, mais je ne le crois pas」那封 1877 年信的學術考據，論證康托爾當時對結果其實相當有把握。讀它校正「康托爾被嚇傻」的通俗渲染——讀第 1–2 節即可掌握要點。（2026-06 線上可查）
- **Wikipedia, "Cantor's first set theory article"**：1874 年論文的內容與脈絡，把「代數數可數 ⇒ 超越數存在且不可數」這條非構造存在論證講得很清楚；同時守住「1874 是區間套、不是對角線」的分野。
- **Wikipedia, "Z-order curve"**：位元交錯（Morton code）在資料庫與空間索引的工程實作，正是本章 |ℝ|=|ℝ²| 構造的有限位元近親；讀「Applications」一節看它怎麼在 data lake 裡保 locality。
- **Wikipedia, "Lindemann–Weierstrass theorem"**：π、e 為什麼是超越數的近代證明背景（Hermite 1873 證 e、Lindemann 1882 證 π）。讀它感受「存在性 1874 就有、具體指認卻要再等近十年且難得多」這道鴻溝。
- **《馴服無限》ch14**：ℝ 從 ℚ 的構造（Dedekind 切割／Cauchy 序列）與完備性。本章「連續、嚴格遞增、值域是整條 ℝ」用到的中間值定理，根就在那裡；本書不展開，要補請讀那一章。
