# ch18 — 技術棧裡的無限：可數性、對角線與不可判定

> **本章解決什麼問題**：前十七章你跟著康托爾從「一樣多」一路走到「數學困住自己」。一路上我在每章開頭都丟給你一個工程錨點——hash、generator、composite key、bitmask、停機問題——當作理解的把手。現在到了還債的時候：這一章把那些把手全部收成一張總表，並且證明它們不是「像」集合論，而是**字面上同一件事**。本章不開任何新的集合論概念，整章就是一座橋，只回收不挖坑。你會看到可計算性的邊界、型別系統的極限、機率為什麼非要 σ-代數不可——全都坐落在康托爾畫的那條可數/不可數線上。讀完這章，ch19 才好把整本書收成一條你能口頭講完的故事。

## 從你已知的出發

過去十七章，每一章的「從你已知的出發」都在做同一件事：拿你寫過的某段 code，說「看，這就是某個集合論概念」。我列一下我用過哪些把手：

- ch01：資料庫 join on unique key、完美 hash——一一對應。
- ch02：hash table 的 rehash／resize——希爾伯特旅館的房間移位。
- ch03：composite key 把 (table, row) 壓成一個鍵、序列化——配對函數。
- ch04：寫不出 generator——不可列舉＝不可數的計算版；而且我當場承諾「對角論證跟停機問題是同一招」。
- ch06：n 個 feature flag → 2ⁿ 種組態——冪集的指數爆炸。
- ch10：`getAllSetsNotContainingThemselves()`、無 WHERE 又自我 JOIN 的查詢——羅素悖論。
- ch12：你寫 `pick()` 必須給實作、但 AC 只保證選擇器存在——非構造性。
- ch13：`volume(A ∪ B) = volume(A) + volume(B)` 在病態集合上崩掉——非可測集。
- ch16：停機判定器 H、對角程式 D(D)——對角線回到計算自己身上。

每次我都說「先記住，後面回收」。這一章就是「後面」。但我要先把一件事說清楚，因為它決定了你該用什麼態度讀這章：**這些不是教學用的類比。** 類比是「A 像 B，記住 A 比較好懂 B」。我要主張的更強——**它們是同一個數學物件穿了不同衣服**。當你寫 `Map<K, V>` 並假設 key 不碰撞，你用的「不碰撞」就是單射，沒有「像」的成分，它字面上是單射的定義。當你發現「沒有一個靜態分析器能判定所有程式的所有行為」，你撞上的就是對角論證，不是「類似對角論證的東西」。

這個區別重要，因為它改變了這本書對你的意義。如果集合論只是「像」你的工作，那讀完就放回書架。但如果可計算性的邊界**本來就是**可數/不可數的邊界、型別系統的不可判定**本來就是**對角線——那你過去六年天天在那條線附近工作，只是沒人告訴你那條線叫什麼名字。康托爾在 1874 到 1891 之間畫的那條線，圖靈 1936 在計算這一側重新畫了一遍，你每天在它的工程後果裡謀生。這章把名字還給你。

我們從那張總表開始，再逐一把幾條最深的橋走透。

## 最深的一句：幾乎所有函數都不可計算

這節要證一個會讓你重新看待「寫程式」這件事的命題。先把它寫出來，再慢慢拆：

> **程式的全體是可數的（ℵ₀），但「ℕ→{0,1} 的函數」的全體是不可數的（𝔠）。所以幾乎所有函數，都沒有任何程式能算出來——它們存在，但你寫不出。**

這句話我要你能自己重講，所以分三步走，每步都是你已經會的東西。

### 第一步：程式可數

一個程式是什麼？對你來說是一個原始碼檔案——一串字元。關鍵在「一串」：程式是**有限長**的字串，寫在一個**有限**的字母表上（ASCII、Unicode 的某個有限子集、隨便你選一個固定的字符集）。這兩個「有限」是全部的力量來源。

把所有程式按長度分堆：長度 1 的程式有幾個？字母表有 k 個字元，長度 1 的字串最多 k 個——**有限**。長度 2 的最多 k² 個——有限。長度 n 的最多 kⁿ 個——有限。於是「所有程式」＝「長度 1 的 ∪ 長度 2 的 ∪ 長度 3 的 ∪ …」＝**可數多個有限集的聯集**。ch03 證過：可數個可數集（這裡甚至是有限集）的聯集還是可數。所以——

```text
所有程式的集合 = ⋃_{n=1}^∞ {長度 n 的合法程式}
              = 可數多個有限集的聯集
              = 可數（ℵ₀）              ← ch03 的定理直接套用
```

更具體地給你一個列舉（generator）：先列出所有長度 1 的程式（字典序）、再長度 2 的、再長度 3 的……每個程式都是有限長，所以一定排在某個有限位置被點到。這就是一個替「所有程式」寫的 generator——程式可數，鐵證。

注意這裡用到的全是 ch02–03 的家當：能排成一列＝可數，有限字串可以字典序排，可數個有限集聯集可數。**程式可數**這件事，骨子裡就是「程式是有限資訊」這一句話的後果。

### 第二步：函數不可數

現在看「函數」。我取最樸素的一種：從 ℕ 映到 {0,1} 的函數，f: ℕ → {0,1}。這種函數就是「對每個自然數，回答 0 或 1」——你可以想成一個無限長的 boolean array，第 n 格是 f(n)。

但這正是 ch04 的主角！「ℕ→{0,1} 的函數」的全體，就是無限二元序列集 {0,1}^ℕ，ch04 用對角論證證明了它**不可數**（𝔠）。我不重證（這章不開新坑），但提醒你它怎麼證的：假設這些函數能排成一列 f₁, f₂, f₃, …，造一個對角函數 f\*(n) = 1 − fₙ(n)，它跟每個 fₙ 在第 n 個輸入上不同，所以不在列表裡，矛盾。

```text
所有 f: ℕ → {0,1} 的集合 = {0,1}^ℕ = 不可數（𝔠）    ← ch04 對角論證
```

### 第三步：對不起來，所以幾乎全部寫不出

把兩步併起來。一邊是程式，可數，ℵ₀ 個。另一邊是函數，不可數，𝔠 個。每個程式至多算出一個函數（你把程式跑在輸入 n 上、看它輸出 0 還 1，就得到一個 f）。所以「可被某程式算出的函數」最多跟程式一樣多——**至多 ℵ₀ 個**。但函數總共有 𝔠 個，而 𝔠 > ℵ₀（ch04 證的嚴格不等）。

```text
可計算的函數 ≤ 程式數 = ℵ₀
所有函數          = 𝔠 > ℵ₀
⇒ 不可計算的函數 = 𝔠 − ℵ₀ = 𝔠（還是不可數那麼多）
```

ℵ₀ 個程式，去蓋 𝔠 個函數，連邊都沾不到。一一對應**不可能**——鴿籠原理的無限版：可數多個程式塞不進不可數多個函數，必然絕大多數函數沒有任何程式對應。而且不是「少數漏網」，是漏掉了 𝔠 那麼多個——**幾乎所有函數都不可計算**。可計算的那 ℵ₀ 個，在 𝔠 的汪洋裡是測度為零的一小撮。

停下來感受一下這句話的重量。你寫程式六年，腦子裡「函數」這個詞幾乎等同於「我能寫出來的東西」。但這個證明說：你能寫出來的函數，在所有函數裡是**滄海一粟**，可數 vs 不可數的那種一粟。絕大多數從 ℕ 到 {0,1} 的對應關係，沒有公式、沒有演算法、沒有任何有限長的描述能生成它——它們只是「存在」，存在在那個不可數的集合裡，永遠在你的編輯器之外。

這正是 ch05 超越數那句話的精確回聲：代數數可數、實數不可數，所以「隨手丟一個實數幾乎一定是超越數」，但具體指出某個數超越很難。這裡是同一道牆的計算版：可計算函數可數、所有函數不可數，所以「隨手指一個函數幾乎一定不可計算」，但具體寫出一個不可計算函數要費點勁。也是 ch12 非構造性的回聲——存在 ≠ 給得出。康托爾的對角線在這裡第 N 次發威：它不只說「有比 ℵ₀ 大的無限」，它說「在你的職業正中央，可寫的東西是可數的稀有品，不可寫的東西是不可數的常態」。

### Worked example：手造一個具體的不可計算函數

「幾乎所有函數不可計算」是個存在性陳述（用基數秒殺），它沒指出**哪一個**不可計算——又是 ch12 那種「證明存在卻不給例子」的非構造味道。但我們可以更進一步，**具體寫下一個**不可計算函數，用的正是 ch16 的停機問題對角線。這是這節的硬貨，一步一步走。

先把程式列舉好。第一步我們已經有了：所有程式可數，排成一列 P₀, P₁, P₂, …（用上面的字典序列舉，第 n 個程式叫 Pₙ）。現在我先給你一個**寫不出矛盾的失敗嘗試**——這個失敗本身是這節最該學的一課，所以我故意先走錯一步。定義一個函數 g: ℕ → {0,1}：

```text
        ⎧ 1   若程式 Pₙ 跑在輸入 n 上「會停機」
g(n) =  ⎨
        ⎩ 0   若程式 Pₙ 跑在輸入 n 上「不停機」
```

白話：g 拿第 n 個程式去跑「它自己的編號 n」當輸入，停就答 1、不停就答 0。g 是個合法函數——對每個 n，「Pₙ(n) 停不停」是個有確定答案的事實（要嘛停、要嘛不停，沒有第三種），所以 g(n) 每個值都唯一確定。g 完全**良定義**。

你可能想：那直接證 g 不可計算不就好了？假設 g 可計算、由某程式 P_d 算出。問 P_d(d) 停不停——若停，g(d)=1，P_d(d) 輸出 1，一致；若不停……但 P_d 要輸出值就得停，所以它會停。**逼不出矛盾。** g 只是「忠實報告每個 Pₙ(n) 停不停」，它就是它報告的那個函數，安安靜靜，沒頂撞任何人，自然不會自相矛盾。

這就是教訓：**對角線不是「把事實列成表」就會自動出矛盾，矛盾來自你造的對象去頂撞每一列。** ch04 的 s\* 翻轉每一位、ch10 的 R 問自己屬不屬於自己、ch16 的 D(D) 反抗判定器——它們的定義裡都寫死了「**跟那個聲稱涵蓋我／判定我的東西作對**」。一個只「報告」的 g 不會爆，會爆的是「假設有人能算我，我就專門跟它唱反調」。

所以正確的收尾不是去硬擰 g，而是**接線到 ch16 已經擰好的那條對角線**。g 若可計算，會發生什麼？

```text
若 g 可計算 ⇒ 算 g(n) 就能判定「Pₙ(n) 停不停」
          ⇒ 由此可組裝出一個萬能停機判定器（任意 P、任意 x 都能判停不停）
          ⇒ 違反 ch16（停機問題不可判定，那是靠 D(D) 反抗判定器證的）
          ⇒ 矛盾。所以 g 不可計算。∎
```

g 自己「報告事實、不唱反調」，所以單看 g 擰不出矛盾；但「能算 g」這個**假設**會讓你造出停機判定器，而停機判定器的不存在，ch16 已經用對角線替我們證好了。換句話說，g 不可計算的「作對」沒有消失——它**外包給了 ch16**。我們因此**具體寫下了**一個不可計算函數：g(n)＝「Pₙ(n) 停不停」，良定義、每個值有確定答案，就是沒有程式算得出它。存在且指得出，但寫不出。

（嚴謹度標示：「程式可數、函數不可數⇒幾乎全部不可計算」是**完整證明**，骨架就在上面三步，你能自己重講。「g 不可計算」我**借用了 ch16 已證的停機問題**當引理，沒有從頭重證停機問題——那是 ch16 的硬證，這裡只接線。）

## 對角線家族：CS 裡到處是它的後代

ch16 已經把對角論證、Cantor 定理、羅素悖論、停機問題串成「同一條對角線」。這節把它在電腦科學裡的後代再點幾個——點到為止，因為它們的細節超出本書，但你要認得出它們的血統。每一個都是「假設有個萬能的東西，造一個專門跟它作對的對象，逼出矛盾」這同一招。

**Rice 定理（Rice's theorem）**——這是停機問題的一般化，也是「沒有萬能靜態分析器」的精確版。它說：**程式的任何非平凡的語義性質（semantic property），都不可判定。**（2026-06 查證）拆解這句話的兩個關鍵詞：

- **語義性質**：關於程式**行為／算出的函數**的性質（「這程式對所有輸入都停嗎？」「它算的是不是常數函數？」「它會不會印出密碼？」），相對於**語法性質**（「原始碼裡有沒有 `if`？」「有幾行？」）。語法性質你的 linter 隨手就判，語義性質才是 Rice 管的。
- **非平凡**：這性質不是「對所有程式都成立」、也不是「對所有程式都不成立」——也就是真的有程式有它、有程式沒它。

合起來：只要你問的是「程式**幹了什麼**」而不是「程式**長什麼樣**」，而且這問題的答案不是恆真恆假，那就**沒有一個程式能對所有程式判定它**。停機問題（「會停嗎」）只是其中一個特例。Rice 定理是你身為工程師天天撞牆的數學原因：為什麼沒有完美的死碼偵測、沒有完美的「這函數有沒有副作用」分析、沒有能判定「兩段 code 是否等價」的工具。它們不是還沒做出來，是**對角線證明了做不出來**。你的型別檢查器、靜態分析器之所以只能保守地給近似答案（誤報、漏報、或拒絕一些其實沒問題的程式），根子就在這。

它的證明也是對角線：假設有個萬能判定器能判某個非平凡語義性質，你就能用它造一個停機判定器，而停機判定器不存在（ch16），矛盾。同一招，第 N 次。

**不可壓縮字串（Kolmogorov 複雜度）**——一句話的版本：一個字串的 Kolmogorov 複雜度，是「能生成它的最短程式」的長度。**幾乎所有字串都不可壓縮**——這也是個數數論證（counting argument），和對角線同源於「鴿籠」：長度 n 的字串有 2ⁿ 個，但長度小於 n 的描述（短程式）少於 2ⁿ 個（2⁰+2¹+…+2ⁿ⁻¹ < 2ⁿ），所以至少有一個長度 n 的字串沒有更短的描述，而且絕大多數都沒有。（2026-06 查證）這就是「不存在完美的無損壓縮演算法」的數學底——你的 gzip 對隨機資料無能為力，不是工程不力，是 2ⁿ 個東西塞不進少於 2ⁿ 個短描述。注意這跟「程式可數、函數不可數」是同一個鴿籠精神：**短描述比被描述的對象少，所以多數對象沒有短描述。**（順帶：「K(x) ≥ |x| 的 x 存在」可證，但「指出某個具體不可壓縮字串」又是不可計算的——存在 vs 構造的牆再次出現。）

**型別系統的不可判定片段**——夠強的型別系統（能編碼任意計算的那種，例如帶某些進階特性的型別層）裡，「這個程式型別檢查得過嗎」會變成不可判定，因為型別檢查器被迫去模擬任意計算、於是繼承了停機問題。這是為什麼編譯器設計要在「型別系統的表達力」和「型別檢查保證會終止」之間做取捨——你想要更強的型別保證，就逼近那條對角線；想要編譯一定停，就得砍掉表達力。Curry–Howard 對應（命題即型別、證明即程式，ch17 點過）讓這件事更深：型別檢查＝證明檢查，而「能不能證」的極限就是 ch16 的不完備。

把這幾個收一句：**停機問題、Rice 定理、不可壓縮字串、型別系統的不可判定，全是對角論證的直系後代。** 你在 CS 課本、編譯器文件、壓縮演算法的極限裡讀到的每一個「這做不到」，回溯三代，祖先都是康托爾 1891 那條翻轉對角線的線。

## 可數可加性與機率：為什麼非要 σ-代數

這條橋接回 ch13（非可測集）並指向書架上的《馴服隨機》。我只走「為什麼機率的可加性要可數而不是任意」這一刀，測度論本身不展開（那是機率書的地盤）。

你對「面積／體積可加」有很強的直覺：兩塊不重疊的區域，總面積＝各自面積相加。寫成式子：

```text
不重疊 ⇒  μ(A ∪ B) = μ(A) + μ(B)          （有限可加性）
```

機率也一樣：互斥事件的機率相加，P(A 或 B) = P(A) + P(B)。但機率論（和測度論）要求的更強——**可數可加性（countable additivity，σ-additivity）**：可數無限多個互斥事件，機率也相加：

```text
A₁, A₂, A₃, … 兩兩互斥 ⇒  μ(⋃ₙ Aₙ) = Σₙ μ(Aₙ)     （可數可加性）
```

為什麼非要把它加強到「可數無限項也能相加」？ch04 的陷阱清單裡有一條是「可數可加就是有限可加的無限版」——那是錯的，可數可加**嚴格更強**，而機率論真的需要這個更強的版本。理由有兩面：

**一面是「需要它」**：機率裡所有有趣的東西都是可數無限的極限。「丟硬幣直到第一次正面」可能丟 1 次、2 次、3 次、……，這些情況有可數無限多種，你要把它們的機率加起來得到 1，靠的就是可數可加。大數法則、收斂、分布函數——全都是可數無限項的求和或取極限。有限可加性給不出這些（它只管有限多項），整個機率論會垮掉。這條接《馴服隨機》——機率即面積、可數可加性是那本書的地基，細節去那裡。

**另一面是「不能更強」**：那為什麼不乾脆要求「**任意**可加性」——任意多個（包括不可數多個）互斥集合都能相加，豈不更威？因為**任意可加性會自爆**，而引爆它的正是 ch13 的非可測集。回想 Vitali 集（ch13 的 worked example）：在 [0,1] 上把實數按「差是有理數」分成等價類、用 AC 各挑一個代表元湊成 V。V 的可數多個有理平移版鋪滿 [0,1]、兩兩不交、每個平移版「大小一樣」（平移不變）。現在問 V 的測度 μ(V)：

```text
[0,1] ⊆ ⋃_{q∈ℚ∩[−1,1]} (V + q) ⊆ [−1,2]   （可數多個平移版兩兩不交，且夾住 [0,1]）
若可數可加成立：1 = μ([0,1]) ≤ Σ_q μ(V+q) = Σ_q μ(V) ≤ 3   （平移不變 ⇒ 每項都等於 μ(V)）
    若 μ(V) = 0：Σ = 0，但要 ≥ 1     矛盾
    若 μ(V) > 0：Σ = +∞，但要 ≤ 3    矛盾（可數多個相等正數相加發散）
⇒ V 沒有任何合法測度值——V 不可測。                    （2026-06 查證）
```

這就是 ch13 的核心一刀，現在你看清它跟可加性的關係了：**正是「可數可加 ＋ 平移不變 ＋ μ([0,1])=1」這三條一起，逼出「V 不能有測度」。** 那唯一的出路是：承認「不是每個集合都能有體積／機率」，把測度只定義在一個**夠用但不貪心**的集合家族上——這個家族就是 **σ-代數（σ-algebra）**：對補集、對**可數**聯集封閉的集合族。σ 這個字母就是「可數」的意思——σ-代數恰好配合可數可加性，封閉到可數操作為止、不再往上要求不可數，這樣既蓋住所有你關心的事件，又**剛好把 Vitali 那種病態集合擋在門外**。

收一句：**機率為什麼活在 σ-代數上、為什麼可加性是可數而非任意——是 ch13 非可測集這個怪物逼出來的防禦工事。** 可數可加是「強到夠用」，任意可加是「強到自爆」，σ-代數是劃定戰線的那條牆。這跟 ch11 的 ZFC 是同一個故事的兩個版本：天真地「給每個集合一個體積」會爆（非可測集），就像天真地「給每個性質一個集合」會爆（羅素悖論）——兩邊的解法都一樣，**限制**你能對哪些對象做操作。σ-代數是測度論的分離公理。（為什麼這族要對可數聯集封閉、Borel/Lebesgue 怎麼建——《馴服隨機》。）

## 序列化＝Gödel numbering：把結構編成數

最後一條短橋，接 ch16 的 Gödel 編碼。你每天 serialize：把一個巢狀的 object、一棵 AST、一個 protobuf message，編成一條位元組串或一個整數鍵。你從沒覺得這有什麼了不起——它就是「把結構攤平成一串資料」。

哥德爾 1931 做的事，字面上就是這個。他要讓數學系統能談論「自己的公式和證明」，於是給每個符號一個數、用配對函數（ch03！）把一串符號編成一個自然數、再把一串公式（一個證明）編成一個更大的自然數。**Gödel numbering 就是替「公式與證明」這個資料結構寫的 serializer。** 一旦公式變成數，「這個公式可證明」就變成一個「關於數」的算術命題，數學系統就能在自己內部談論自己的可證明性——對角線於是有了下手的地方，G「我不可證」才造得出來（ch16）。

這條橋的點睛是：**「把程式碼當資料」不是一個比喻，它是 Gödel numbering、是 quine、是反射、是序列化，全部同一件事。** 你寫 `JSON.stringify(fn.toString())` 把一個函數變成字串再存起來的那一刻，你在做哥德爾 1931 做的事——把「會做事的東西」編成「能被當資料處理的東西」。哥德爾的洞見不是「能編碼」（你天天編碼），是「**一旦能編碼，系統就能談論自己，自我參照就成立，對角線就能回到系統自己身上**」。序列化是平凡的，序列化讓自我參照成為可能是不平凡的——這就是 ch16 的全部戲劇。

配對函數（ch03）→ 序列化 → Gödel numbering → 自我參照 → 對角線回到數學自己（ch16）。這條鏈，你在 ch03 編 composite key 時就踏上了第一格。

**題 2 ★★ [15 分鐘]** 把一個你用過的工具對應到本書一個概念，並證明是字面同一件事。

**題 4 ★★★ [20 分鐘]** 為什麼那個對角函數要「作對」而不能只「報告」。

### 推演解答

**題 1。**
(a) regex 是有限字串、寫在有限字符集上。按長度分堆：長度 n 的 regex 最多 kⁿ 個（k＝字符集大小），有限。所有 regex ＝ 可數多個有限集的聯集 ＝ 可數（ℵ₀）。給列舉：先列長度 1 的（字典序）、再長度 2 的、……每個 regex 在有限位置被點到。跟本章「程式可數」一字不差，只是把「程式」換成「regex」。

(b) {0,1}\*（所有有限二元字串）本身可數（同上，按長度分堆）。一個「語言」是 {0,1}\* 的一個子集，所以「所有語言的集合」＝ 𝒫({0,1}\*)。由冪集定理（ch06），|𝒫(可數集)| > |可數集|，而且 𝒫(可數無限集) 的基數是 𝔠（2^ℵ₀，ch06/ch07）——**不可數**。所以語言有 𝔠 個。

(c) 可數多個 regex（ℵ₀），去描述不可數多個語言（𝔠），ℵ₀ < 𝔠，鴿籠的無限版：絕大多數語言沒有 regex 描述。把「regex」換成「程式」結論更強——程式也只有 ℵ₀ 個（本章第一節），所以**有語言（其實是幾乎所有語言）無法被任何程式描述／辨識**。這跟本章「程式可數、函數不可數⇒幾乎所有函數不可計算」**是同一個論證**：可數的描述者，蓋不住不可數的被描述對象。語言 ≈ 函數（語言↔特徵函數，n∈L ⟺ f(n)=1），完全同構。

**題 2。**（示範一個答案——以 hash 為例；你選別的工具同理。）
(a) hash：把每個 key 對應到一個 bucket，我假設「不同 key 不會撞到同一個 bucket」（理想無碰撞）且「每個 bucket 都被用到」（填滿）。
(b) 對應 ch01 的一一對應（bijection）：key 集合 ↔ bucket 集合。
(c) 並排定義——「無碰撞」＝「不同 key 的 hash 不同」＝「k₁≠k₂ ⇒ h(k₁)≠h(k₂)」，這**逐字**是單射（injection）的定義；「填滿」＝「每個 bucket 都有某個 key 映過去」＝「∀bucket ∃key, h(key)=bucket」，這**逐字**是滿射（surjection）的定義。單射＋滿射＝一一對應。所以「理想 hash」不是「像」一一對應，它的兩個性質就是一一對應的兩個定義條件，逐條對上。反駁「只是比喻」：比喻會在某處不對應（hash 還有 load factor、rehash 等），但「無碰撞＋填滿＝bijection」這個核心是定義級的相等，不是相似——load factor 那些是工程附加，不影響 bijection 的核心同一性。（若你選了 type checker：對應 ch11/ch16，type checking＝證明檢查，過不過＝spec 滿不滿足；選 loop 終止：對應 ch09，ranking function＝往良序集遞減。）

**題 3。**
(a) 用到可數可加性的步驟：`1 = μ([0,1]) ≤ Σ_{q} μ(V+q)`——這裡 q 跑遍 [−1,1] 裡的有理數，有**可數無限**多個，把可數無限項的測度加起來、再用單調性夾住聯集的測度，正是可數可加性。有限可加性只允許「有限多項相加」，到不了這一步。
(b) 若只有有限可加性：你只能說「**有限**多個平移版的測度相加 ＝ 它們聯集的測度」，但 V 的平移版有可數無限多個才鋪滿 [0,1]，有限多個鋪不滿。於是你無法把「整個 [0,1] 的測度 1」拆成「無限多個相等的 μ(V) 之和」，那個「0 加不出 1、正數加成無窮」的矛盾就造不出來——V 在純有限可加的世界裡可以僥倖不出事。矛盾的引擎就是「可數無限個相等項求和」這件有限可加做不到的事。
(c) 下界（不能更弱）：機率需要可數可加，否則大數法則、極限、「丟到第一次正面」這類可數無限情形算不了，整個理論垮（接《馴服隨機》）。上界（不能更強）：任意可加（含不可數項）會被 Vitali 引爆——平移不變＋任意可加＋μ([0,1])=1 不相容。所以可數可加是「強到夠用、又還沒自爆」的唯一甜蜜點，σ-代數就是把可加性鎖在「可數」這一級的容器。

**題 4。**
(a) 「g(n)＝Pₙ(n) 停不停」良定義，因為對每個 n，「Pₙ(n) 停不停」是個有確定答案的客觀事實（停或不停，無第三種），所以 g 對每個輸入有唯一值。但**良定義 ≠ 可計算**——g 的值「存在且確定」，不代表有程式算得出它。光是 g 自己，並不自相矛盾：g 就是「那個記錄了所有對角停機事實的函數」，它安安靜靜地存在，沒有頂撞任何人，所以你無法靠「g 跟自己矛盾」來證明它不可計算。它只是「報告事實」。
(b) 「作對」結構：ch04 的 s\* 在第 n 位**翻轉**第 n 列（s\*(n)=1−sₙ(n)，跟第 n 列作對）；ch10 的 R={x:x∉x} 問「R 屬不屬於自己」、答案兩邊都頂撞自己；ch16 的 D(D)＝「若判定器說我停我就不停」，**反抗判定器**。共同點：對象的定義裡寫死了「**跟那個聲稱涵蓋我／判定我的東西作對**」——不是被動報告，是主動唱反調。矛盾來自「唱反調」，不來自「事實本身」。
(c) 乾淨骨架：g(n)＝「Pₙ(n) 停機則 1、否則 0」。假設 g 可計算 ⇒ 存在程式算 g ⇒ 給定任意程式 P 與輸入 x，我可以（先找到 P 在列舉中的編號、或直接改造）用 g 判定「P(x) 停不停」⇒ 我造出了萬能停機判定器 ⇒ 違反 ch16（停機問題不可判定）⇒ g 不可計算。這裡「作對」藏在哪？——藏在 **ch16 已經替我們做完的那一步**：停機問題的不可判定本身，就是靠 D(D) 跟判定器作對證出來的。本章的 g 不必重造對角線，它**接線**到 ch16 那條已經擰好的對角線上。所以乾淨版的「作對」不是消失了，是外包給了 ch16。這正是回收章的本分：不重證，接線。

## 自我檢核

口頭自答，講得出來才算過關。這章的過關標準是「能不能把橋講成『同一件事』，而不是『很像』」。

1. 「程式可數但函數不可數，所以幾乎所有函數不可計算」——不看書把這三步講一遍（程式為何可數、函數為何不可數、為何推出幾乎全不可計算）。卡在哪步，那步就是你還沒懂的。
2. 為什麼「程式可數」骨子裡只是「程式是有限長字串」這一句話的後果？換個更強的程式語言能讓程式變不可數嗎？為什麼不能？
3. 「幾乎所有函數不可計算」會不會讓你寫的程式變得沒意義？為什麼不會？（關鍵：有意義的函數幾乎都落在可計算那 ℵ₀ 個裡。）
4. Rice 定理在說什麼？「非平凡語義性質」這六個字每個詞是什麼意思？為什麼它蘊含「沒有完美的靜態分析器」、卻不蘊含「靜態分析器都沒用」？
5. 停機問題、Rice 定理、不可壓縮字串、型別系統的不可判定——它們的共同祖先是什麼？挑一個，講它怎麼是「對角線的後代」。
6. 機率為什麼要可數可加而不是有限可加（下界）、又為什麼不能任意可加（上界）？「任意可加會自爆」是被誰引爆的？σ-代數在這故事裡扮演什麼角色？
7. 「序列化＝Gödel numbering」——你天天 serialize，哥德爾 1931 做的字面上就是這個。那哥德爾不平凡的洞見到底是哪一步（提示：不是「能編碼」）？
8. 【本章收線題】挑三條橋（一一對應、對角論證、非可測集，或你自選），對每一條講清楚「它不是像我的工具、它字面上就是」。講到能反駁「這些都只是教學類比」。
