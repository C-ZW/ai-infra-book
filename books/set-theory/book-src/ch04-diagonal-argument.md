## 從你已知的出發

你寫過 generator。一個 generator 的本事是：對一個可能無限的集合，承諾「你一直跟我要，我最終會把每個元素都吐給你一次」。可數（countable）這個詞，翻成你的母語，就是「存在這樣一個 generator」——能排成一列 a₁, a₂, a₃, …，每個元素都在某個有限位置出現。ch02 到 ch03 你做的全是這件事：替 ℤ 寫 generator（0, 1, −1, 2, −2, …）、替 ℚ 寫 generator（對角蛇行）、甚至替代數數寫 generator（列舉多項式、再列舉根）。每次你都成功了，於是你開始相信：只要我夠聰明，總能替任何集合寫出 generator。

現在我換個問法，這個問法會決定整本書的走向：**有沒有一個集合，無論你多聰明，都寫不出 generator？** 不是「我們還沒找到」，是「可以證明這個 generator 不可能存在」。

對一個工程師，這個問句有個熟悉的形狀。你大概聽過停機問題（halting problem）：沒有一個程式能判定任意程式會不會停。那個證明的核心招數，跟本章要走的對角論證，**是同一招**——你想列出一份完整清單（所有判定器、所有實數、所有無限位元串），我就構造一個專門跟你清單上每一列作對的東西，逼出矛盾。本章先把這招在最乾淨的場景裡走透；ch16 再把它搬到「可證明性」上，停機問題會在那裡完整兌現。先記住這句話：**「寫不出 generator」＝「不可列舉」＝「不可數」的計算版。** 對角論證是這句話第一次被證出來。

我選的場景是「所有無限長的二元序列」。你每天處理位元串，這個對象沒有任何神秘感：

```
一個無限二元序列就是一個函數 s: ℕ → {0,1}
寫出來像：1 0 1 1 0 0 1 0 ...     （永不結束）
所有這種序列的集合，記作 {0,1}^ℕ
```

工程上你會說：這是「無限長的 boolean array」的全體。每一位獨立地是 0 或 1，序列無限長。問題就一句：**這個集合可數嗎？能不能排成一列？** 你前三章的直覺會喊「應該可以吧，每個元素都是規規矩矩的位元串」。這個直覺，下面會被精確地、無可挽回地推翻。

## 先把張力上滿：為什麼你會以為它可數

值得停十秒，因為等一下整個證明的力道，全來自你現在有多相信「它可數」。

ℚ 的列舉那麼反直覺都成功了——ℚ 在數線上稠密，任兩個有理數之間還塞著無限多個，看起來「比 ℕ 多無限倍」，結果還是可數。代數數更誇張：所有整係數多項式的所有根，π 你還沒碰、但 √2、黃金比、所有你寫得出根式的數全在裡面，照樣可數。到 ch03 結束，你手上的規律是：

> 凡是「每個元素都能用有限資訊描述、且能系統性地排序」的集合，都可數。

無限二元序列看起來完全符合。每一位就是一個 bit，序列就是一串 bit，還有什麼比這更「規矩」？你甚至可以開始排：先排「最後全是 0」的（這些對應有限二進位數，可數）、再排有理數展開的、再排……

問題就埋在這個「再排……」裡。你能排的，永遠是「有某種有限描述」的序列。但 {0,1}^ℕ 裡的序列**不需要有任何描述**——它可以是「每一位都隨機擲硬幣決定」的那種，沒有公式、沒有規律、沒有有限的生成規則。這種序列佔了絕大多數，而你的 generator 思路天生只能碰到「描述得出來」的那些。康托爾要證的，正是：不管你的清單塞進了多少規矩或不規矩的序列，**清單之外永遠剩下至少一個**。

## 對角論證：完整版（本書硬證之首）

這是工程師的嚴謹級別：每一步我都會說出理由，你要能口頭複述每一步為什麼非如此不可。論證用反證法（proof by contradiction）。

**假設**：{0,1}^ℕ 可數。依定義，這表示存在一個列舉——能把所有無限二元序列排成一列：

```
s₁, s₂, s₃, s₄, ...    （每個 sₙ 是一個無限二元序列；ℕ 把它們編號）
```

「可數」的全部內容就在這裡：每個序列都拿到了一個自然數編號，沒有遺漏。如果這個假設能撐到底而不矛盾，那 {0,1}^ℕ 就真的可數。我們要做的是讓它撐不住。

**把列表攤成一張表**。每個 sₙ 是一串位元，逐位寫出來，第 n 列就是 sₙ。我用 sₙ(k) 表示第 n 個序列的第 k 位。給一個具體的假想列表（你的列舉可能長得完全不同，這只是為了看清結構——論證對任何列舉都成立）：

```text
        位 1   位 2   位 3   位 4   位 5  ...
s₁  =     1     0     1     1     0   ...
s₂  =     0     0     1     0     1   ...
s₃  =     1     1     0     1     1   ...
s₄  =     0     1     1     1     0   ...
s₅  =     1     0     0     1     0   ...
 :                                   ·.
```

**標出對角線**。看 sₙ 的第 n 位——sₙ(n)——這些就是左上到右下的對角線。我把它們框起來：

```text
        位 1   位 2   位 3   位 4   位 5  ...
s₁  =   [ 1 ]   0     1     1     0   ...   ← 取第 1 位：1
s₂  =     0   [ 0 ]   1     0     1   ...   ← 取第 2 位：0
s₃  =     1     1   [ 0 ]   1     1   ...   ← 取第 3 位：0
s₄  =     0     1     1   [ 1 ]   0   ...   ← 取第 4 位：1
s₅  =     1     0     0     1   [ 0 ] ...   ← 取第 5 位：0
 :                                   ·.
對角線位元 = 1, 0, 0, 1, 0, ...
```

**每一位都翻轉**，造一個新序列 s*。規則：s* 的第 n 位，定義成對角線第 n 位的相反——

```text
s*(n) = 1 − sₙ(n)        （0 翻成 1，1 翻成 0）
```

對上面這張表，對角線是 1, 0, 0, 1, 0, …，翻轉後：

```text
對角線位元 =  1   0   0   1   0  ...
s*        =  0   1   1   0   1  ...   ← 每一位都跟對角線相反
```

注意 s* 是一個合法的無限二元序列：每一位都明確是 0 或 1（因為 sₙ(n) 不是 0 就是 1，1 減掉它還是 0 或 1）。所以 s* ∈ {0,1}^ℕ。既然我們假設列表收錄了**所有**無限二元序列，s* 必須出現在列表的某一列。問題是：哪一列？

**s* 不在列表中**。這是整個證明的引爆點，慢慢走。假設 s* 出現在第 m 列，也就是 s* = sₘ。那麼這兩個序列每一位都該相等，特別是第 m 位：

```text
s*(m) = sₘ(m)            ← 若 s* 真的就是第 m 列，第 m 位必須相等
```

但 s* 的造法寫死了第 m 位：

```text
s*(m) = 1 − sₘ(m)        ← s* 的定義
```

把兩式併起來：

```text
sₘ(m) = 1 − sₘ(m)
2·sₘ(m) = 1
sₘ(m) = 1/2              ← 但 sₘ(m) 只能是 0 或 1，不可能是 1/2
```

矛盾。換個更白話的講法你會更有感：s* 跟第 m 列**在第 m 位上故意不同**（這正是「翻轉」的用意）。所以 s* 不可能等於 sₘ。而 m 是任意的——對**每一個** m，s* 都跟第 m 列至少在第 m 位不一樣。於是 s* 跟列表裡的**每一列**都不同。

**收網**。s* 是一個合法的無限二元序列，卻不在「收錄了所有無限二元序列」的列表裡。這直接打臉了「列表收錄了所有序列」這個假設。假設爆了。

**結論**：{0,1}^ℕ 不可能被排成一列。它**不可數**。∎

這就是對角論證。我認為這是我見過最划算的一頁紙：一個假設、一條對角線、一次翻轉，就證明了一種無限嚴格大於另一種。它不需要任何 ℝ 的構造、不需要極限、不需要拓樸，純粹是「翻轉對角線」這個動作逼出來的。

### 為什麼非「翻轉對角線」不可

工程師看證明，光看懂不夠，要看出「為什麼這樣設計」。s* 的造法有兩個約束，缺一不可：

- **它必須跟每一列都不同**——否則它就在列表裡，沒矛盾。怎麼保證跟「無限多列」都不同？太貴了，除非有個統一的機制。對角線給了這個機制：只要在第 n 位跟第 n 列作對，就一口氣跟所有列都岔開了。一個位置擋一列，對角線剛好把「列的編號」和「位的編號」對齊，一次掃完。
- **它必須是合法序列**——每一位都得是確定的 0 或 1。翻轉（1 − sₙ(n)）保證了這點：對角線位元是什麼是確定的，翻轉後也是確定的。

換句話說，對角線是「同時跟可數多個對象作對」的最省力構造。你想列出無限多個東西、卻被一條穿過所有列的對角線各個擊破。記住這個形狀——ch06 的冪集定理、ch10 的羅素悖論、ch16 的不完備與停機問題，全是這同一條對角線換皮。

## 換皮成實數：(0,1) 區間不可數

無限二元序列數不完，跟「實數數不完」差幾步？很近，但有一個技術陷阱必須誠實處理，這也是科普最常偷懶的地方。

把 (0,1) 之間的實數寫成十進位小數 0.d₁d₂d₃…。對角論證照搬：假設 (0,1) 可數，排成一列 r₁, r₂, r₃, …，攤成表，取對角線數字 dₙₙ（第 n 個數的第 n 位小數），每一位改掉，造一個新小數 r*——它跟每個 rₙ 在第 n 位不同，所以不在列表中，矛盾。

聽起來一樣乾淨。但有個坑：**同一個實數可以有兩種十進位寫法**。0.4999999… = 0.5000000…（這個等式本身見《馴服無限》ch03，那裡用三種方式證它）。所以「r* 跟每個 rₙ 在某一位的數字不同」**不保證** r* 跟 rₙ 是不同的數——它們可能是同一個數的兩種長相。如果你不管這個，證明就有洞，而且是真的洞，不是吹毛求疵。

修法很簡單，誠實地避開兩個惹禍的數字 0 和 9（雙重表示永遠涉及一串 9 變一串 0）。造 r* 時，每一位這樣選：

```text
r* 的第 n 位 d*ₙ：
    若 dₙₙ = 5，就取 d*ₙ = 6
    否則         取 d*ₙ = 5
```

這樣 r* 的每一位都只會是 5 或 6——永遠不碰 0 和 9，所以 r* 不可能是「某個數的另一種寫法」，它只有這一種長相。同時 d*ₙ ≠ dₙₙ（5 對 6、6 對 5 都不等，其餘數字對 5 也不等），保證 r* 跟第 n 個數在第 n 位**數字確實不同**，又避開了雙重表示，所以 r* 跟 rₙ 是**不同的數**。於是 r* ∈ (0,1) 卻不在列表裡，矛盾。

**走一個具體例子**。假設列表前幾個數是：

```text
r₁ = 0. 3 1 4 1 5 ...     對角線取第 1 位：3
r₂ = 0. 7 5 2 0 8 ...     對角線取第 2 位：5
r₃ = 0. 1 0 5 0 0 ...     對角線取第 3 位：5
r₄ = 0. 6 6 6 5 7 ...     對角線取第 4 位：5
r₅ = 0. 9 0 0 0 2 ...     對角線取第 5 位：2
對角線數字 = 3, 5, 5, 5, 2, ...
```

套規則（是 5 改 6、否則改 5）：

```text
對角線 =  3   5   5   5   2  ...
r*     =  5   6   6   6   5  ...      r* = 0.56665...
```

驗證：r* 的第 1 位是 5 ≠ 3（r₁ 的第 1 位），所以 r* ≠ r₁；第 2 位是 6 ≠ 5（r₂ 的第 2 位），所以 r* ≠ r₂；……每一位都對得上，而且 r* 全由 5 和 6 組成，沒有雙重表示問題。r* 是一個貨真價實、不在列表裡的 (0,1) 實數。

於是 (0,1) 不可數。再加一句話就推到整條 ℝ：(0,1) 跟 ℝ 一樣大（一個明確的一一對應，例如 tan(π(x−½)) 把 (0,1) 拉伸到整條實線——注意直接拿 tan(x) 套 (0,1) 只到 (0, tan 1)，要先把 (0,1) 平移縮放到 (−π/2, π/2) 才掃得到整條 ℝ，細節 ch05 給），所以 **ℝ 不可數**。

## 「不只一種無限」第一次成為定理

把這一章的結論放到全書脊椎上。ℕ 可數，|ℕ| = ℵ₀。我們剛證了 ℝ（或等價地 {0,1}^ℕ、(0,1)）不可數。「不可數」精確的意思是：它跟 ℕ 之間**不存在**一一對應，而且不是因為它比 ℕ 小——ℕ 明明可以單射進 (0,1)（n ↦ 0.000…1，把 1 放在第 n 位），所以 (0,1) 至少跟 ℕ 一樣大。一樣大被排除、又至少一樣大，剩下唯一的可能是：

```text
ℵ₀ = |ℕ| < |ℝ| = 𝔠
```

這個 𝔠（fraktur c，continuum，連續統）是一個**嚴格大於 ℵ₀** 的無限基數。全書後面 |ℝ| = |(0,1)| = |{0,1}^ℕ| = |𝒫(ℕ)| = 2^ℵ₀ 都是同一個 𝔠（ch05/ch06/ch07 逐步補嚴格）。

這就是這一章了不起的地方，而且我要你能用自己的話講出來：在康托爾之前，「無限」是一個詞、一個模糊的「永遠數不完」，伽利略（ch01）看到一一對應卻只敢說「無限不能比大小」就收手了。康托爾不收手——他先把「一樣多」操作化成一一對應（ch01），然後用對角論證**證明**了 ℝ 跟 ℕ 一樣多是假的。「有些無限比別的無限大」從此不是哲學感嘆，是有證明的數學事實。**「不只一種無限」這句書名，在這一頁從修辭變成定理。** 你可以對另一個工程師說：我能證明給你看，實數比自然數多——而且只用一條對角線。這是雞皮疙瘩該起來的地方，靠的不是驚嘆號，是上面那條沒有漏洞的推理。

## 一句話的伏筆：這跟停機問題是同一招

不展開（ch16 才完整），但現在點明，因為它是本書最划算的橋。

你想證明「不存在萬能的停機判定器 H」。停機問題的證明是這樣的：假設 H 存在，能對任意程式 p 與輸入 x 判定 p(x) 會不會停。把所有程式排成一列（程式是有限字串，可數，ch18 細講）攤成表——第 n 列是第 n 個程式、各欄是它對各輸入的行為。然後造一個對角程式 D：D 讀進一個程式 p，去問 H「p(p) 會停嗎」，如果 H 說會停，D 就故意死循環；如果 H 說不停，D 就故意停下來。D **在對角線上跟每個程式作對**。把 D 餵給它自己——D(D) 會停嗎？兩種答案都自相矛盾。

看出來了嗎？這就是 s* 跟第 n 列在第 n 位作對，只是把「位元」換成「程式對輸入的行為」、把「翻轉 0/1」換成「H 說停我就不停」。**對角論證是「你列不出所有實數」，停機問題是「你列不出所有程式的停機行為」——同一條對角線。** 「不可列舉」就是「不可數」的計算版。記住這句話，ch16 兌現。

## 直覺的陷阱

對角論證是本書最常被「聽懂了其實沒懂」的證明。下面每一條都是真實會把人帶溝裡的錯誤直覺，以及怎麼自我察覺你中了哪一條。

| 錯誤直覺 | 它會在哪一步把你帶溝裡 | 怎麼自我察覺、怎麼破 |
|---|---|---|
| 「s* 造好了，那就把它加進列表第一列不就好了？」 | 你以為這修補了列表。 | 加進去後**對角線就變了**，對新列表重跑一次，又造出一個新的 s*' 不在裡面。你補一個我造一個，永遠補不完——這恰恰證明了不可數，不是反駁。徵兆：你覺得「修一下就好」，代表你還沒接受「對任意列表都失敗」。 |
| 「s* 其實排在表的第 ∞ 列。」 | 你以為列表有個「最後一列」能藏 s*。 | **沒有第 ∞ 列。** 列表由 ℕ 索引，每一列的編號都是一個有限的自然數，∞ 不是自然數、不是任何一列的編號。s* 跟第 1, 2, 3, … 每一個**有限編號**的列都不同，這就窮盡了所有列。徵兆：你說出「第無限列」「最後一列」，代表你把 ℕ 想成有終點了。 |
| 「對角線只穿過列表的一部分，漏掉的列怎麼辦？」 | 你以為對角線管不到所有列。 | 對角線第 n 位處理第 n 列，n 跑遍所有自然數，**每一列都有它專屬的一位**被作對。沒有列被漏掉——第 17 列在第 17 位被擋，第 10^9 列在第 10^9 位被擋。 |
| 「這只證明了『這個特定列表』不完整，換個聰明的列表就行了。」 | 你以為問題出在列舉方式不夠好。 | 證明**沒有用到列表的任何特殊性質**——對角線翻轉對「任何」排法都造得出反例。這不是「這次沒排好」，是「排不出來」。徵兆：你開始想「如果我先排無理數再排有理數……」，代表你還在找更聰明的 generator，但證明已經堵死了所有 generator。 |
| 「無限 + 1 還是無限，所以所有無限一樣大，對角論證一定哪裡錯了。」 | 你用 ℵ₀ 的算術（ch02 旅館）去否定一個比 ℵ₀ 大的東西。 | 「+1 不變」對 ℵ₀ 成立，但那是**基數相等**的陳述，不是「所有無限基數相等」。對角論證恰恰證明了存在一個無限基數 𝔠，連 ℵ₀ 都套不進去。旅館塞得下可數多人，塞不下 {0,1}^ℕ 那麼多人——這正是本章的內容。 |
| 十進位版：「對角線改一位就造出新數了。」（漏掉雙重表示） | 你跳過了 0.4999…=0.5000… 的坑。 | 「數字不同」不等於「數不同」。必須限制改出來的位只落在安全集合（如本章的 5/6），避開 0 和 9，才保證造出的是真的不同的數。徵兆：你的證明裡沒提雙重表示，那它有洞。 |

最後一條我要單獨強調：**對角論證證的是「跟每一列都不同」，不是「比每一列都大」或任何順序性質。** 它是純粹的「不相等」論證。很多人下意識想成「造一個更大的數」，那是錯的方向——s* 可能比表裡某些序列「小」（如果你硬給序列定個序），但它跟每一列都**不等**，這就夠了。

## 紙上推演

純紙筆。沒有 Python——對角論證的驚嘆全在推理，不在跑出來的數字。

**題 1 ★ [10 分鐘]** 手跑一次對角論證。
給定下面這張假設的 4 列二元序列表，請：(a) 寫出對角線位元；(b) 用翻轉規則 s*(n) = 1 − sₙ(n) 造出 s*；(c) 逐列驗證 s* 跟第 1、2、3、4 列各在哪一位不同。

```text
        位1  位2  位3  位4 ...
s₁  =    0    1    1    0  ...
s₂  =    1    1    0    1  ...
s₃  =    1    0    0    0  ...
s₄  =    0    0    1    1  ...
```

**題 2 ★★ [20 分鐘]** 對 𝒫(ℕ)（自然數的所有子集）自己跑一次對角論證。
提示：一個子集 A ⊆ ℕ 可以用它的「特徵序列」表示——第 n 位是 1 代表 n ∈ A、是 0 代表 n ∉ A（這個對應 ch06 會正式化）。所以 𝒫(ℕ) 跟 {0,1}^ℕ 是同一個東西的兩種說法。請假設 𝒫(ℕ) 可數、排成一列子集 A₁, A₂, A₃, …，**不用特徵序列、直接用集合語言**構造一個子集 D，使 D 跟每個 Aₙ 都不同，並說明 D 是怎麼「在第 n 個位置跟 Aₙ 作對」的。寫出 D 的定義。

**題 3 ★★ [15 分鐘]** 拆解一個常見的錯誤論證。
你的同事說：「對角論證有問題。你造出的 s* 不在列表裡，那我就把 s* 排在原列表所有元素之前，當新的第 1 列，後面接原來的 s₁, s₂, …。現在 s* 在列表裡了，矛盾就沒了，所以 {0,1}^ℕ 其實可數。」請指出他錯在哪，並說明為什麼他的「修補」反而又一次證明了不可數。

**題 4 ★★★ [15 分鐘]** 為什麼十進位版要避開 0 和 9？
有人用十進位版對角論證，造新數時規則是「對角數字若是 0 就改成 1，否則把它減 1」。請構造一個具體情境（給一個小列表），讓這個規則造出的 r* 雖然「每一位數字都跟對角線不同」，卻**等於列表裡的某個數**，從而證明這個規則有漏洞。然後說明本章的「是 5 改 6、否則改 5」規則為什麼不會出這個問題。

### 推演解答

**題 1。**
(a) 對角線取 sₙ 的第 n 位：s₁ 第 1 位 = 0，s₂ 第 2 位 = 1，s₃ 第 3 位 = 0，s₄ 第 4 位 = 1。**對角線位元 = 0, 1, 0, 1。**
(b) 每一位翻轉：**s\* = 1, 0, 1, 0**（後面還有無限位，但前 4 位由這張 4×4 表決定；完整列表會繼續，s* 也繼續翻轉）。
(c) 逐列驗證——
- 跟 s₁（0,1,1,0）比：s* 第 1 位 = 1，s₁ 第 1 位 = 0，**在第 1 位不同**。
- 跟 s₂（1,1,0,1）比：s* 第 2 位 = 0，s₂ 第 2 位 = 1，**在第 2 位不同**。
- 跟 s₃（1,0,0,0）比：s* 第 3 位 = 1，s₃ 第 3 位 = 0，**在第 3 位不同**。
- 跟 s₄（0,0,1,1）比：s* 第 4 位 = 0，s₄ 第 4 位 = 1，**在第 4 位不同**。

每一列都被它專屬的那一位擋掉了。注意 s* 在**別的**位上可能跟某列相同（例如 s* 第 1 位 = 1 也等於 s₂ 的第 1 位），這完全不要緊——對角論證只需要「至少一位不同」，而對角線保證了「第 n 位」這個專屬位置。這是新手最容易看漏的地方：你不需要每一位都不同，你需要每一**列**都有一位被作對。

**題 2。**
照提示，第 n 位作對，翻譯回集合語言就是：**對「n 在不在 Aₙ 裡」這件事作對**。所以定義

```text
D = { n ∈ ℕ : n ∉ Aₙ }
```

白話：把所有「不屬於自己那一列子集」的 n 收進 D。這個 D 跟每個 Aₘ 都不同，因為它們在「m 這個元素」上的歸屬恰好相反——
- 若 m ∈ Aₘ，則依 D 的定義 m ∉ D，所以 D ≠ Aₘ（在 m 這一點上分歧）。
- 若 m ∉ Aₘ，則依 D 的定義 m ∈ D，所以 D ≠ Aₘ（同樣在 m 這一點上分歧）。

不論哪種情況，D 跟 Aₘ 至少在元素 m 的歸屬上不同，所以 D ≠ Aₘ。對所有 m 都成立 ⇒ D 不在列表中 ⇒ 矛盾 ⇒ 𝒫(ℕ) 不可數。

這個 D 就是 s* 的集合版：第 n 位翻轉 ⟺ 「n 屬不屬於 Aₙ」反過來。記住 D = { n : n ∉ Aₙ } 這個形狀——它跟 ch06 Cantor 定理的對角集 D = { x : x ∉ f(x) }、跟 ch10 羅素悖論的 R = { x : x ∉ x } 是同一個東西的不同打扮。你在這裡已經摸到了那條會貫穿全書的對角線。

**題 3。**
他的錯誤：把 s* 插進新列表後，**新列表的對角線跟舊的不一樣了**。對角論證不是針對「某個固定列表」一勞永逸的——它是個構造程序：給我**任何**列表，我就現場造一個不在其中的 s*。對他的新列表（s*, s₁, s₂, …）重跑一次，取**新對角線**（新第 1 列 = s* 的第 1 位、新第 2 列 = s₁ 的第 2 位、…），翻轉，造出 s*'，s*' 不在新列表裡。

所以他的「修補」一點也沒修補——他每塞一個進去，我就再造一個漏網的。這個「補一個漏一個、永遠補不完」**正是不可數的本質**：不是某張清單漏了，是**沒有任何清單能不漏**。他以為自己在反駁，其實在親手把證明又演示了一遍。常見錯路的根源是把「可數」誤解成「我能不斷往清單加東西」——但可數要求的是「存在一份**一次到位、涵蓋全部**的清單」，而對角論證證明這種清單不存在。

**題 4。**
這個規則「對角數字減 1（若是 0 就變成 9）」就會出事。把列表第一個數安排成 r₁ = 0.5000…（也就是 0.5），對角線安排成 5, 0, 0, 0, …：

```text
r₁ = 0. 5 0 0 0 ...        對角線第 1 位 = 5
r₂ = 0. 9 0 9 9 ...        對角線第 2 位 = 0
r₃ = 0. 4 4 0 4 ...        對角線第 3 位 = 0
r₄ = 0. 2 2 2 0 ...        對角線第 4 位 = 0
 :   （之後每一列第 n 位都安排成 0）
對角線 = 5, 0, 0, 0, ...
```

套規則（減 1，0 變 9）：第 1 位 5→4，之後每位 0→9，造出 r* = 0.4999…。逐位看，r* 每一位都跟對角線不同（4≠5、9≠0），照理該「不在列表裡」。但 **0.4999… = 0.5000… = r₁**——r* 跟 r₁ 數字逐位全不同，卻是同一個數。規則沒擋住「全 9 結尾對上全 0 結尾」這種雙重表示，r* 撞回了 r₁，證明就破了。這就是為什麼非避開 0 和 9 不可。

本章的「是 5 改 6、否則改 5」為什麼安全：造出的 r* 每一位都只會是 5 或 6，**永遠不出現 0、也永遠不出現 9**。雙重表示一定涉及「一串 9 結尾」對上「一串 0 結尾」，而 r* 兩者都碰不到，所以 r* 只有唯一一種十進位長相，「數字不同」就鐵定蘊含「數不同」。這就是為什麼安全集合要避開 0 和 9——不是迷信，是堵死雙重表示這唯一的漏洞。

## 歷史校正：1874 不是 1891

寫到這裡必須擋住科普第一大錯，這也是你跟別人講這個故事時最容易講錯的地方。

康托爾**第一個**證明 ℝ 不可數的論文是 **1874 年**的〈論全體實代數數的一個性質〉，它用的是**區間套（nested intervals）論證，不是對角線**：給定任何一個聲稱列舉了所有實數的數列，他在數線上套出一串越縮越小的閉區間、每個區間都避開列表裡越來越多的數，這串區間的交點是一個實數，卻不在列表裡——同樣得到不可數，但機制完全不同。這篇 1874 論文還順手證明了超越數存在（代數數可數、實數不可數，差集必然非空且不可數），是非構造性存在證明的早期經典（ch05 接續）。

著名的**對角論證是 1891 年**的另一篇〈論流形論中的一個初等問題〉，相隔 17 年。它更短、更一般——一般到 ch06 能原封不動套到任意集合 A 上證明 |A| < |𝒫(A)|，這是區間套做不到的。後世講解常把 1891 的對角線「回填」進 1874 的故事，99% 的科普因此把「康托爾用對角線證明實數不可數」直接繫到 1874。**錯。要寫對角線就標 1891，要寫 1874 就說區間套。** 本書這一章講的是對角線，所以是 1891。

附帶提醒，免得你跟著一起浪漫化：康托爾的集合論在當時飽受克羅內克（Kronecker）等人反對，克羅內克主張一切數學要能化約成對整數的有限步驟（這條線 ch17 會深入），他確實曾阻撓康托爾投稿。但坊間「克羅內克逼瘋了康托爾」是被嚴重渲染的迷思——康托爾患的是雙相情感障礙（病因在生理），學術壓力是誘因不是病因。講這段歷史時克制、尊重，別寫成「天才被迫害逼瘋」的戲劇。
