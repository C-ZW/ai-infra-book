# 寫作風格指南（內部文件，非書籍內容）

本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名（工作名）：《不只一種無限：集合論與無限的直覺與驚嘆》。最終書名於 outline gate 由作者定案，撰章時以工作名為準。

## 這本書是什麼、不是什麼

- **是**：一本「欣賞與直覺」的書。目標是讓讀者親身感受人類如何學會**比較無限**——看懂集合論不是一堆 ∈、∪、∩ 的符號操作，而是「先把『一樣多』講清楚、再用它去丈量無限」這一連串驚心動魄的發現；讀完有「人類竟然想得出這個、而且竟然證得出無限有大小」的震撼，**而且能用自己的話對另一個工程師講清楚震撼在哪**。
- **不是**：
  - 不是考試解題訓練——不刷集合代數習題、不背公理表、不出符號操練題。
  - 不是嚴格公理化教科書——ZFC 公理在 ch11 當「故事」講（為什麼需要這條、它擋掉什麼），不是逐條形式推導；證明只保留「短而致命」的骨架（對角論證、Cantor 定理、CSB、羅素悖論），長證明明說「本書不證、指向何處」。
  - 不是轉職用書——純興趣，不服務 AI Infra 敘事；但工程橋接要讓它讀起來像母語，不是教科書的外語。集合論與**可計算性**的血緣（對角論證＝停機問題、可數＝可列舉）是本書最划算的橋，要用滿但不喧賓奪主。
- 驗收標準（全書的六條主線，ch19 總收官逐條口試）：
  1. 「一樣多」的真正定義是什麼（**一一對應 bijection**），為什麼它比「數出來」更基本、能用在無限上。
  2. 為什麼無限**不只一種**——可數（ℵ₀）與不可數（𝔠）的分界，對角論證怎麼證的。
  3. 冪集定理：為什麼**沒有最大的無限**，無限有無窮多級。
  4. 為什麼天真集合論會自爆（羅素悖論），ZFC 與選擇公理怎麼重建地基、又付出什麼代價（Banach–Tarski）。
  5. 連續統假設為什麼「**沒有答案**」——獨立性是什麼意思，哥德爾與科恩各證明了什麼。
  6. 這一切怎麼活在計算裡——對角論證＝停機問題、可數＝可列舉、無限的分級就是可計算性與機率測度的邊界。

## 數學記號規範（全書一致，附錄 A 是速查表；閱讀器不渲染 LaTeX，一律純文字）

- **集合與映射**：∈ ∉ ⊆ ⊊ ∪ ∩ ∅ ℕ ℤ ℚ ℝ ℂ；映射 f: A → B；一一對應寫 `A ↔ B`（存在 bijection）或 `f: A → B 為一一對應`。
- **基數記號**：集合 A 的基數寫 `|A|`；ℵ₀（aleph-null）＝ |ℕ|；𝔠（fraktur c，連續統）＝ |ℝ|；冪集 `𝒫(A)`；`2^|A|` ＝ |𝒫(A)|。比較大小寫 `|A| = |B|`、`|A| < |B|`、`|A| ≤ |B|`。
- **序數記號**：ω（小寫 omega，第一個無限序數）；ω+1、ω·2、ω²、ω^ω、ε₀；序型寫「order type」。**0 的約定**：序數脈絡（ch08 起）採 von Neumann 慣例 ω = {0,1,2,…}，明說與前面「點名用 1,2,3」是同一個 ℕ、|ℕ|=ℵ₀ 不變（見基準表「ℕ 的約定」）。
- **對角論證的標準排版**：把「一列東西」寫成一張表（第 n 列 = 第 n 個被列舉的對象，逐位列出），對角線元素標出來、再「每一位都改掉」造出新對象——用 ```text 區塊畫表，對角線用註記指出。這是全書脊椎視覺，ch04/ch06/ch16 共用同一張表的變形。
- **連續統假設**寫 `CH：2^ℵ₀ = ℵ₁`（即 𝔠 = ℵ₁，ℵ₀ 與 𝔠 之間沒有別的基數）；GCH 寫 `2^ℵₐ = ℵₐ₊₁`。
- **多行推導／證明骨架**放在 ```text 圍欄區塊裡，每步右側可加「← 為什麼」短註。禁止把長推導硬塞進行內。
- 不依賴 Unicode 罕見字渲染失敗時的備援：ℵ 若顯示有疑慮，行內可寫 `aleph-0`，但正文統一用 ℵ₀（閱讀器 Apple 平台字型可顯示，已驗證 calc/prob 書使用 ∫ Σ 等同級字元無虞）。

## 每章骨架（必須遵守，標題名稱照抄——一致性掃描 lint_book.py 會 grep 這些標題）

1. 章首：`# chNN — 章名：副標`，接著 blockquote：
   `> **本章解決什麼問題**：…（這章在全書地圖的哪裡、為什麼存在、跟前後章的關係，3-5 行）`
   每個 Part 的第一章（ch01、ch04、ch07、ch10、ch14、ch17）在 blockquote 後放全書 ASCII 路線圖（基準圖在 outline.md，照抄並把「◄你在這裡」標到本 Part），讀者偏好架構圖先行。
2. `## 從你已知的出發` — 橋接段：把本章核心概念錨定在讀者的工程實戰上，要具體（hash 一一對應、generator 可列舉、composite key 編碼、partition 代表元、SQL 自我參照、TLA+ spec/ranking function、停機問題），不要泛泛而談。
3. 核心內容（多個 `##` section，深入直覺與機制）。歷史敘事與數學內容交織，但**歷史服務於數學直覺**，不是編年史。
4. `## 直覺的陷阱` — 本書招牌段落（故障視角的數學版）：這個概念**最常被誤解成什麼**、錯誤直覺**會在哪一步把你帶溝裡**、**怎麼自我察覺**（什麼徵兆代表你其實沒懂）。用表格或條列。例：「無限＋1 還是無限，所以所有無限一樣大」「對角線造出的數早就在列表裡」「可數可加就是有限可加的無限版」「選擇公理顯然成立，何必當公理」「獨立＝我們還沒證出來」——這些都是必須正面拆解的常見錯路。
5. `## 紙上推演` — 動手做的形式（紙筆推演為主）。每章 2~4 題，標注預估時間（如 **[15 分鐘]**）與難度（★~★★★）。題型：手造一個一一對應、手跑一次對角論證、找出一個錯誤論證的破綻、把白話翻成集合語言、口頭重講一個證明。題目後緊接 `### 推演解答` 給完整解答與討論（為什麼這樣想、常見錯路）。**本書純紙筆，無 Python 小實驗段**（集合論的驚嘆在論證不在數值；少數可選的「想像實驗」併入推演題即可）。
6. `## 自我檢核` — 5~8 個問題，**口頭自答**：講得出來才算過關，講不清楚＝還沒懂。問題對齊「能不能把這個概念講給另一個工程師聽」的深度，優先出「為什麼」而不是「是什麼」。
7. `## 延伸閱讀` — 真實連結（以 `_meta/landscape-2026-06.md` 驗證過的為優先），每條一句話說明為什麼值得讀、讀哪一段。

## 深度標準

- **每章至少一個 worked example**：帶具體東西、一步一步走的完整過程（一個明確的一一對應的構造、一張對角線表的逐位構造、一次基數算術、一次序數比較、一次悖論的三段論拆解），不准跳步。outline 已為每章指定基準範例。
- **嚴謹度的誠實標示**：每個論證明確標示自己是哪一級——「直覺版」「工程師的嚴謹：每步能口頭說出理由」「完整證明（本書唯一硬證的少數定理之一）」或「本書不證，指向 …」。**禁止「顯然」「易證」**；禁止把直覺論證偽裝成證明。對角論證、Cantor 定理、CSB（敘述＋骨架）、羅素悖論這幾個「短而致命」的證明要**完整到讀者能自己重講**。
- **概念自我複核**：所有「構造」（一一對應、列舉、對角線、序數算式）寫章時必須自己走一遍驗證（這個對應真的不重不漏嗎？這個列舉真的每個元素都會被點到嗎？對角線真的跟每一列都不同嗎？），不得抄自記憶。跨章基準（ℵ₀、𝔠、CH 的寫法、ℕ 約定、對角線標準排版）以 outline 基準表為準。
- **歷史事實照 landscape**：年份、歸屬、引文、軼事一律以 `_meta/landscape-2026-06.md` 為準；landscape 沒有的自行查證（每章 3~8 次 web 查證）並 hedge；傳說要標明是傳說。**特別守住**：1874 不可數性證明（套疊區間法）≠ 1891 對角論證——兩者是不同證明，常被混為一談；康托爾「我看見了卻不敢信」是 1877 給戴德金的信、講 |ℝ|=|ℝ²|；希爾伯特「沒有人能把我們逐出康托爾為我們創造的樂園」是 1926。禁止憑記憶引用「名人名言」。
- **每個「驚嘆點」要能被重講**：本書的核心商品是震撼感。每章至少一處明確回答「這裡了不起在哪」——而且要寫到讀者能用自己的話轉述，不是堆形容詞。
- 每個概念給「它在解什麼問題」：一一對應解什麼、對角論證解什麼、公理化解什麼、獨立性證明解什麼。禁止無動機的定義。
- 寧深勿廣，不灌水。每章 5,000~9,000 中文字，依主題深度調整（地基章 ch10/ch11/ch15 可偏厚）。

## 事實正確性

- 歷史與時效性事實（年份、人物、引文、書籍版本、連結）以 `_meta/landscape-2026-06.md` 為事實基準；本章特定細節自行用 WebSearch/WebFetch 查證（每章 3~8 次）。
