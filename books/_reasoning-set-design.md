### 2.1 書架上既有、可指向的鄰書（不重推）

## 3. 共用寫作契約（六本的 style-guide 都必須內含；細節各書可加，但這些是硬底線）

### 3.1 章節骨架（標題**照抄**——Tier-1 gate `lint_book.py` 會 grep，順序不可改）

每章依序必含這五個 `##` 標題（中間可插自訂 `##` 內容段）：

1. `## 從你已知的出發` — 橋接段，把本章概念錨到讀者後端實戰（要具體，不泛泛）。
2. `## 直覺的陷阱` — 招牌段（故障視角）：這概念最常被誤解成什麼、錯誤直覺在哪步把你帶溝裡、怎麼自我察覺。用表格或條列，每章至少一個具名陷阱。
   （lint 接受的方言：`## 直覺的陷阱`／`## 陷阱與防禦`／`## 故障模式與防禦`等；**本套六本統一用 `## 直覺的陷阱`**。）
3. `## 紙上推演` — 每章 2~4 題，標難度（★~★★★）與時間（如 **[15 分鐘]**）。題後緊接 `### 推演解答`（完整解答＋常見錯路）。**全套不放任何要讀者去跑的程式碼。**
4. `## 自我檢核` — 5~8 題，口頭自答；優先問「為什麼」不是「是什麼」。
5. `## 延伸閱讀` — 真實連結（以該書 landscape 驗證過的優先），每條一句話說明值得讀什麼。

章首格式：`# chNN — 章名：副標`，接 blockquote `> **本章解決什麼問題**：…（3–5 行，本章在全書地圖哪裡、為何存在、跟前後章關係）`。
每個 Part 的首章在 blockquote 後放全書 ASCII 路線圖（基準圖在該書 outline，照抄並標「◄ 你在這裡」）——讀者偏好架構圖先行。

### 3.2 lint 硬底線（每章寫完自己對照）

- 五骨架標題齊全且順序正確；`## 紙上推演` 段內要有 `### 推演解答`、★ 難度、`NN 分鐘` 時間。
- 每章 ≥1 個 ` ``` ` fenced 區塊（worked example／ASCII 圖／推導表，純興趣書一律有手算示範，天然滿足）。
- 每章 CJK ≥2500 字（硬下限，會 fail），建議 5000~9000。寧深勿廣。
- **零簡體字與大陸詞彙**（lint 逐字元比對）。台灣用語見 §3.3。

### 3.3 語言規範（六本一致）

- 繁體中文台灣慣用語。技術／數學名詞中英並用：第一次「中文（English）」，之後可只用中文。
- 第二人稱「你」直接對讀者說話。自然口語，**不要翻譯腔**。
- 禁簡體字、禁大陸詞彙。常見地雷對照（台灣用語在前）：
  機率（不用「概率」）、變異數（不用「方差」）、常態分布（不用「正態分布」）、
  資訊（不用「信息」）、預設（不用「默認」）、程式碼（不用「代碼」）、物件（不用「對象」）、
  品質（不用「質量」當 quality）、最佳化／優化、函數（數學脈絡）、隨機（不用「随机」）、
  質數（不用「素數」？——**注意：台灣數學界「質數」「素數」並用，本套統一用「質數」**）、
  公因數／最大公因數 GCD（不用「公約數」）、餘數／同餘、賽局（不用「博弈」當形容；書名仍可用「賽局理論」，內文「賽局」「博弈論」皆可，**本套統一賽局／賽局理論**）。
- 數學記號：閱讀器**不渲染 LaTeX**，一律純文字 Unicode（≈ ≠ ≤ ≥ → ∞ √ Σ ∏ ∈ ∉ ⊂ ∀ ∃ ¬ ∧ ∨ ⇒ ⇔ ≡ |·| ⌊⌋ π ε δ λ μ σ）。
  多行推導／矩陣／決策樹／真值表進 ` ```text ` 區塊，對齊、每步右側可加「← 為什麼」短註。禁止把長推導硬塞行內。

### 3.4 深度與誠實標準（六本一致）

- 每章 ≥1 個 worked example：帶具體數字、一步步算、不跳步、放 ` ```text ` 區塊。outline 為每章指定基準範例。
- **嚴謹度誠實標示**：每個論證標明哪一級——「直覺版（嚴格證明指向延伸閱讀／本書不證）」或「工程師的嚴謹：每步能口頭講出理由」。禁「顯然」「易證」；禁把直覺論證偽裝成證明。
- **數值／邏輯自我複核**：所有數字表、報酬矩陣、機率計算、模算術、真值表，寫章時自己重算一遍，不得抄記憶。
- **史實照 landscape**：年份、歸屬、引文、軼事一律以該書 `landscape-2026-06.md` 為準；landscape 沒有的自行查證（每章 3~8 次 web）並 hedge；傳說標明是傳說。禁憑記憶引用名人名言。
- **每個驚嘆點要能被重講**：每章至少一處明確回答「這裡了不起／反直覺在哪」，且寫到讀者能用自己的話轉述，不是堆形容詞。
- 語氣：工程師對工程師。冷靜、直接、有觀點。不行銷、不喊口號、不堆空洞驚嘆——震撼用內容本身造成。允許冷幽默。

### 3.5 各書特別敏感的脆弱事實（landscape 必須查證並標 ⚠️／時點）

- **數論與質數**：⚠️**已知最大質數**（梅森質數，GIMPS；2024-10 發現 M52 = 2^136279841−1；landscape 必須查 2026-06 是否仍是最大、有無新發現並標時點）；孿生質數猜想進展（張益唐 2013 有界間隙→Maynard/Tao 收緊到 246）；RH 仍未解；最大已知孿生質數。這些是全套最時效敏感項。
- **邏輯謬誤與認知偏誤**：⚠️**再現性危機**重創一批名研究——self／ego depletion、社會促發（priming）、power posing、面部回饋、Dunning–Kruger 的重新詮釋等。凡引用某偏誤的經典實驗，必須查 2026 是否已被質疑／未能複製，並誠實標示（這本身就是與《科學方法》呼應的教材）。禁止把「未複製成功」的效果當鐵打事實。
- **科學方法**：⚠️再現性危機數據（Open Science Collaboration 2015 心理學複製率約 36–40%）、p 值爭議（ASA 2016 聲明）、塞麥爾維斯史實細節（1847 維也納總醫院、產褥熱死亡率數字、未被接受、1865 過世）。數字與年份查證。
- **博弈論**：史實歸屬——von Neumann–Morgenstern 1944《賽局理論與經濟行為》、Nash 1950/1951、Axelrod 重複賽局電腦競賽 1980（tit-for-tat 奪冠）、ESS Maynard Smith & Price 1973、機制設計 Hurwicz/Maskin/Myerson（2007 諾貝爾）。
- **哥德爾不完備**：史實——哥德爾 1931〈論《數學原理》及相關系統中形式上不可判定的命題〉、希爾伯特 1900/1928 計畫、Turing 1936 停機、Gentzen 1936 用超限歸納證 PA 一致性。**精確陳述守住**（兩條定理的條件：一致、可遞迴公理化、夠強表達基本算術）——這是最容易被科普寫錯的地方，務必精確、並在「直覺的陷阱」收哥德爾濫用。
- **決策與理性**：Kahneman–Tversky 展望理論 1979、Allais 1953、Ellsberg 1961、聖彼得堡（Daniel Bernoulli 1738）、vNM 1944、Savage 1954、Simon 有限理性（1978 諾貝爾）、Kahneman 2002 諾貝爾。展望理論的某些參數估計有再現性討論，引用實驗數字時 hedge。

## 4. P1 各書 outline 的種子（must-cover／must-NOT-cover 骨幹；P1 agent 據此展開成完整逐章規格）

> 以下只給「擁有權核心＋邊界＋脊椎落點」骨幹，P1 agent 要把它擴成完整 outline（每章 goal／必涵蓋／不要碰＋指向／橋接／worked example／推演題／插圖），並寫出跨章基準表（報酬矩陣數值、機率、模數、定理條件等）。

**1. 賽局《當理性互相碰撞》**：囚徒困境登場（同一張 2×2，定義報酬、優勢策略）→ 納許均衡（為何 (背叛,背叛) 是均衡卻雙輸）→ 純策略不存在時的混合策略（剪刀石頭布、配對銅板）→ 零和與極小極大（von Neumann）→ 重複賽局與合作的演化（Axelrod、tit-for-tat、未來陰影）→ 演化賽局與 ESS（鷹鴿賽局）→ 協調賽局與謝林點 → 訊號與承諾 → 機制設計（反過來設計報酬讓理性導向好結果，如拍賣、投票）→ 收官把囚徒困境七層重看。邊界：單人期望效用指向《決策》、混合策略期望值機率指向《機率》。

**2. 數論《乘法的原子》**：整除與質數定義 → 算術基本定理（唯一分解，**埋哥德爾伏筆**）→ 質數無窮（歐幾里得建構式論證，**非反證法**——常見科普錯誤，守住 30031=59×509 反例；全書第一個「美得想哭」的證明）→ 篩法與質數有多稀疏 → 模算術／同餘（時鐘算術，工程橋接 hash／環形緩衝）→ 費馬小定理、歐拉定理 → 輾轉相除 GCD 與貝祖 → 質數分布 π(x) 與質數定理 PNT（x/ln x，指向《微積分》）→ 孿生質數與質數間隙（張益唐）→ 黎曼假設一瞥（質數的音樂）→ RSA 一瞥（質數的實用）→ 收官質數作為亂與律共存。插圖：Ulam 螺旋、π(x) vs x/ln x、模時鐘、質數間隙散點。邊界：機率啟發指向《機率》、解析部分指向《微積分》、編碼應用 forward 指向《哥德爾》。

**3. 哥德爾《這句話無法被證明》**：形式系統是什麼（公理＋規則＝機械符號遊戲，工程橋接型別系統／證明即程式）→ 希爾伯特之夢（把數學完全機械化、可判定）→ 真 vs 可證的差別 → 自我指涉與說謊者悖論 → 哥德爾編碼（把句子變數字，**引用數論 FTA**）→ 對角化引理（造出說「我不可證」的句子 G）→ 第一不完備定理（G 真但不可證）→ 第二不完備定理（系統證不了自己一致）→ 停機問題與可計算性的孿生（Turing）→ 哥德爾的（誤）用與正確邊界（「直覺的陷阱」重頭戲）→ 收官真理溢出證明。邊界：質因數分解機制指向《數論》、濫用作為謬誤指向《偏誤》。

**4. 偏誤《思考的陷阱》**：雙系統 System 1/2（快思慢想，工程橋接 cache 命中 vs 冷路徑計算）→ 捷思的雙面性（可得性、代表性、錨定——對的工具錯的場合）→ 形式謬誤（肯定後件、否定前件）→ 非形式謬誤目錄（稻草人、假兩難、滑坡、訴諸權威／人身、循環論證…）→ 機率直覺崩壞（基率忽視、合取謬誤 Linda、賭徒謬誤——機制指向《機率》）→ 確認偏誤與動機性推理 → 框架效應與損失趨避（描述面，數學面指向《決策》）→ 社會與記憶偏誤 → 再現性危機：哪些經典偏誤其實沒站穩（誠實，呼應《科學方法》）→ 去偏誤能做到什麼、做不到什麼 → 收官「你還是會上當，但你知道在哪上當」。邊界：偏誤的規範基準指向《決策》、制度防線指向《科學方法》。

**5. 科學方法《如何不騙自己》**：塞麥爾維斯洗手案登場（觀察到的死亡率差異）→ 觀察不等於理論（理論負載）→ 歸納的問題（休謨：太陽明天會升起嗎）→ 可證偽（Popper：科學的分界）→ 假說與對照實驗（控制變因、隨機化、雙盲，橋接 A/B test／canary）→ 因果 vs 相關（混淆變因，橋接「相關性故障」）→ 貝氏確證（證據如何更新信念，指向《決策》《機率》）→ 科學作為社群（Kuhn 典範、Lakatos 研究綱領、為何塞麥爾維斯被否決）→ 統計的陷阱（p 值、p-hacking、再現性危機）→ 科學的真 vs 數學的真（對照《哥德爾》）→ 收官科學＝系統化的防自欺。邊界：心理偏誤指向《偏誤》、信念更新數學指向《決策》《機率》、形式系統真理觀對照《哥德爾》。

**6. 決策《在不確定中下注》**：一個帶不確定性的決策登場（賭注／醫療檢查）→ 期望值與它的陷阱（聖彼得堡悖論）→ 期望效用與 vNM 公理（為何效用不是金額，凹函數＝風險趨避）→ 不確定 vs 風險（Knight、Ellsberg 模糊趨避）→ 貝氏決策與信念更新（醫療檢查的偽陽性，機制指向《機率》）→ 資訊的價值（要不要再做一個檢查）→ 展望理論（人實際怎麼選：參考點、損失趨避、機率權重——描述 vs 規範的裂縫）→ Allais 悖論 → 有限理性與捷思（Simon，橋接《偏誤》）→ 多屬性決策與序貫決策 → 收官同一個決策五種算法重看。邊界：對手會反應的決策指向《博弈》、破壞理性的心理機制指向《偏誤》、證據更新的信念面與《科學方法》互指、機率機制指向《機率》。

## 6. 審查規則（self-review ＋ agy 第二意見；2026-06-20 使用者定案）

**這條規則同時適用 P1（元檔）與 P2（逐章）。** 每本書 P1 元檔寫完、每章 P2 寫完，都跑這套：

1. **自我審查**（焦慮工程師視角）：跨書邊界是否與本綱 §2 一致、脊椎落點是否真的貫穿每個 Part、landscape 脆弱事實是否查證並標時點、骨架標題是否照抄可過 lint、台灣用語是否乾淨、是否有「無動機定義」或「偽裝成證明的直覺」、數值/邏輯是否自我複核。

2. **agy 第二意見**——**模型固定 `Gemini 3.5 Flash (Medium)`**（使用者 2026-06-20 指定；不是 High，不是別家）：
   - **多次呼叫、不是一次**：P1 一本一通（六本平行）、P2 一章一通（agy 用量無上限，放心大量平行）。
   - **指令範本**：`/Users/cheweichen/.local/bin/agy --model "Gemini 3.5 Flash (Medium)" -p "<內容>"`。內容用**絕對路徑**或直接把檔案內容餵進去（agy 相對路徑會寫進它自己的 workspace）。
   - **時效**：agy `--print-timeout` 預設 5 分鐘；Flash Medium 對大 prompt（~14KB）約 11–24 秒，**給 ≥290s timeout、別提早 kill**（早期誤把「還在跑」當「卡死」殺掉）。Pro/Thinking 系列慢很多，本規則不用。
   - **並行 ≤3**：大 prompt 平行 >3 會 throttle；逐章 fan-out 每批 3 個＋`--print-timeout 9m`。
   - **內容過濾 fallback（2026-06-21 教訓）**：Gemini 會對某些章「靜默內容過濾、回空」（非逾時——同尺寸他章能跑＝內容問題非長度）。退路鏈：跨家族 `codex exec -m gpt-5.2-codex` → 若 codex 不可用（如 token 撤銷）改用 Claude agent 對抗式自審（每數字重算、每史實 web 查證）。**別讓被過濾的章靜默跳過其他章都過的閘**。lint_book 現在也掃 `_meta/*.md`＋書根 `CLAUDE.md`（簡體字）、有大陸詞 ERROR/WARN 兩級，語言資料抽到 `tools/md-reader/lang/<code>.json`（語言無關引擎）。

3. **裁決鐵律——VERIFY-BEFORE-APPLY（本規則的核心，2026-06-20 教訓）**：
   **agy 的每一條發現都是「假設」不是「判決」**，尤其它常只看摘要/digest，會自信地誣告。**每條都要先對照完整原始檔重新驗證，真的才改，假的駁回並寫一句理由。** 禁止盲套 agy 的結論。
   - 實測偽陽性類型（看到要警覺）：把**強制骨架**（★ 難度、`NN 分鐘`、五段標題）當「教科書化漂移」；記錯數學條件（如說 RSA 要 `gcd(e,n)=1`，其實是 `gcd(e,φ(n))=1`）；把**正確**的式子誣為錯（如 `u(x)=√(x+W)` 是最終財富效用、卻被說成建在變化量）；把**刻意的跨書邊界設計**（哥德爾濫用歸偏誤書、機率機制指向機率書）當「重推鄰書」；把章骨架裡看不到的工程橋接當「缺工程脈絡」。
   - 真陽性要修（agy 確實會抓到真錯）：細微數學（TFT 門檻 δ≥(T−R)/(R−S)=2/3 而非 grim 的 1/2）、史實年份（Lucas 1961 非 1959）、脆弱事實精確度（梅森質數是「第 52 個**被發現的**」非「確定第 52 個」）、脊椎斷在某 Part、核心遺漏（溯因推理、規劃謬誤）、招牌案例數字不誠實（塞麥爾維斯 18.27% 是單月非全年）。
   - **corroboration＝信心**：同一條被 ≥2 次獨立指出（不同模型或 self-review＋agy 都點）→ 高度可信、優先處理。
   - 採納標準：與本綱衝突時以本綱為準；單一來源不推翻 landscape 事實基準，重大修正需兩個獨立查證（沿用 skill P4）。

4. 整合修正後才往下走（P1→P2 fan-out；P2 逐章修完才進 P3 lint）。
