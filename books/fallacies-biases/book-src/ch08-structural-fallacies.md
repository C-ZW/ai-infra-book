# ch08 — 結構型謬誤：假兩難・滑坡・循環・不當類比（含哥德爾濫用）

> **本章解決什麼問題**：上一章（ch07）的攻擊型謬誤錯在「打錯了對象」——稻草人打一個被扭曲的立場、人身攻擊打人不打論點。這一章換一組更難抓的非形式謬誤，它們**不攻擊誰，只是在論證的骨架裡偷偷塞了一塊沒被證成的木頭**：把選項硬塞成只有兩個（假兩難）、把一條沒人證明過的因果鏈當成必然會發生（滑坡）、把結論藏進前提裡再「推」出來（循環論證）、用一個只在表面像的類比硬套出結論（不當類比）。它們都靠 ch06 教的「重建隱藏前提」才抓得到——錯誤不在表面那句話，在沒講出來的那塊預設。本章最後收一個跨書招牌案例：把哥德爾不完備定理亂套到社會、心靈、後現代論述上的「哥德爾濫用」，它正是「不當類比」最戲劇的標本（定理到底說什麼指向《這句話無法被證明》，本書只負責講「為何那是濫用」）。

## 從你已知的出發

你開過那種會。架構討論到一半，有人把白板一拍，說：「我們現在就兩條路——要嘛**整個重寫**，要嘛**等它自己爛掉**。選吧。」

你心裡有點不對勁，但一時說不上來。會議的氣氛已經被這句話框死了：你要嘛站「重寫派」，要嘛站「躺平派」，沒有第三個位置可以站。於是討論變成兩派吵架，沒人去問那個更該問的問題——**「誰說只有這兩條路？」**

這就是**假兩難（false dilemma，亦稱非黑即白）**，本章四個謬誤裡你最常在會議室裡親眼看到的一個。它的陰險之處不在它說錯了什麼，而在它**沒說**的那一塊：明明還有「先抽出最痛的那個模組重寫、其餘維持」「加一層 anti-corruption layer 隔離爛掉的部分」「先補測試凍結行為再漸進重構」這些選項，但它們被那句「就兩條路」一筆抹掉了。你的注意力被鎖在它擺出來的兩個盒子裡，沒人去檢查盒子外面。

把這個感覺記住，因為本章四個謬誤共用同一個機制：

- **假兩難**偷渡的是「選項就這些」這個沒被證成的前提；
- **滑坡**偷渡的是「A 一定會連鎖導致 Z」這條沒被證成的因果鏈；
- **循環論證**偷渡的是「結論本身」——它被藏進前提，再被「推導」出來，等於用 A 證明 A；
- **不當類比**偷渡的是「這兩個東西在這個關鍵點上也一樣」這個沒被檢查的相似性。

你在 code review 裡天天碰到它們的工程變體：「不加快取系統就會垮」（假兩難——其實還有限流、分頁、非同步）、「開了這個例外整個架構就崩」（滑坡——要證成那條連鎖）、「這設計好，因為它是最佳實踐；它是最佳實踐，因為它好」（循環）、「微服務就像樂高，拼起來就好」（不當類比——樂高的介面是標準化的，你的服務不是）。這些句子的共同點是：**表面那句話聽起來很有力，真正出問題的是它沒講出來、要你自己去重建的那塊預設。** 本章就是教你把那塊預設挖出來。

## 一個共同的診斷工具：先重建，再檢查骨架

在拆四個謬誤之前，先把武器擦亮。ch06 講過一件事：非形式謬誤大多藏在**沒講出來的前提（enthymeme，省略式論證）**裡。攻擊型謬誤（ch07）你還能在表面句子裡抓到「他扭曲了我的立場」；結構型謬誤更滑，因為表面那句話**邏輯結構往往是有效的**——問題出在一個沒被說出口、也沒被證成的前提上。

所以對付這一章的謬誤，標準動作是兩步：

```text
步驟一：重建論證 —— 把白話翻成「前提 → 結論」，補出沒講出來的隱藏前提
步驟二：檢查骨架 —— 對那塊隱藏前提問一句：「它被證成了嗎？還是被偷渡了？」
                    │
                    ├─ 假兩難  ：隱藏前提 =「選項只有這些」← 被證成了嗎？
                    ├─ 滑坡    ：隱藏前提 =「每一步都會必然觸發下一步」← 證成了嗎？
                    ├─ 循環    ：隱藏前提 =「結論」本身 ← 那不是前提，是要證的東西
                    └─ 不當類比：隱藏前提 =「相似的那點正好支撐結論」← 真的相似嗎？
```

注意這跟 ch05 的形式謬誤（肯定後件那種）**機制不同**。形式謬誤錯在**推論形式本身無效**——`P→Q`、看到 `Q` 就斷定 `P`，這跟內容完全無關，純粹是結構壞了（見 ch05）。結構型的非形式謬誤不一樣：它的**推論形式可以是有效的**，壞掉的是某個前提的真假與證成狀態。換句話說——

> 形式謬誤是「**形式**壞了，內容對不對無所謂」；本章的結構型謬誤是「**前提**壞了或沒被證成，形式可能還好端端的」。別把它們混為一談——這是全書主線之一（形式 vs 非形式的分界，見 ch05、ch06，ch20 收官會總對帳）。

這也是為什麼這一章難抓：你的系統一（System 1，見 ch01）對「論證的表面說服力」反應很快——一句「就兩條路，選吧」聽起來果斷、清楚、有力，系統一給它打了「合理」的標籤就放行了；而要抓出它的毛病，得靠系統二（System 2）慢慢去做「重建隱藏前提」這件費力的事。系統一不會替你做這一步，它只負責「聽起來對不對」，不負責「前提證成了沒」。本章每一個謬誤，最後都會回扣到這條脊椎：**系統一憑表面說服力放行，系統二該做的骨架檢查被偷懶蓋過。**

## 假兩難：把光譜壓成開關

**假兩難**：把實際上有多個選項（甚至是連續光譜）的情況，硬說成「只有兩個、而且互斥」的選擇，逼你二選一。

**它原本對在哪？** 二分法不是天生有罪。有些情況真的就是二選一：一個 feature flag 要嘛開要嘛關、一筆交易要嘛 commit 要嘛 rollback、一個布林值沒有第三態。在這些**真的互斥且窮盡**的場合，「只有兩個選項」是對描述、是有用的簡化——它幫你把決策收斂，不必窮舉。系統一愛二分法，因為二分省力：兩個盒子比一條光譜好處理得多。

**它在哪翻車？** 翻在把一個**本來是光譜或多選項**的情況，謊報成「窮盡且互斥的兩個」。真實世界的工程決策幾乎都是光譜：重構不是「全重寫 vs 不動」，中間有十種漸進策略；可用性不是「100% vs 垮掉」，中間是一條 SLA 曲線；安全不是「完全鎖死 vs 完全開放」，中間是一整套權衡。假兩難的傷害是它**把你看不見的選項從你的視野裡刪掉了**——你不是選錯，你是根本沒看到還有別的選項可選。

```text
假兩難的解剖：

  表面句：「不重寫就只能等死。」
       │
       ▼  重建（補出隱藏前提）
  前提1（明說）：不重寫 → 系統會爛死
  前提2（偷渡）：選項「只有」重寫 和 等死 兩個   ← 沒被證成！
  結論：所以必須重寫
       │
       ▼  檢查骨架
  攻擊點不是前提1，是前提2：誰證明過選項只有這兩個？
  反例（補出被刪掉的選項）：
    - 抽出最痛的單一模組重寫，其餘維持（絞殺榕模式）
    - 加 anti-corruption layer 隔離爛掉的邊界
    - 先補特性測試凍結現狀，再小步重構
    - 評估「再撐 N 個月」的實際成本，可能根本不必動
       │
       ▼  一旦第三、第四選項被攤開，「就兩條路」這個前提自己就垮了
```

**怎麼自我察覺？** 聽到「不是 A 就是 B」「要嘛…要嘛…」「沒有別的選擇」「你到底站哪邊」這類措辭時，問一句最便宜的反制問題：**「真的只有這兩個嗎？第三個選項是什麼？」** 多數假兩難撐不過這一問——因為它本來就靠「沒人去問」活著。工程現場最常見的假兩難是「快 vs 好」「安全 vs 易用」「現在做 vs 永遠不做」——它們幾乎總是光譜被壓成了開關。

## 滑坡：沒被證成的連鎖必然

**滑坡（slippery slope）**：主張「一旦走了第一步 A，就會無可避免地一路滑到災難性的 Z」，而中間那條「A→B→C→…→Z」的連鎖，每一環的必然性**都沒被證成**。

這個謬誤最需要小心，因為它有一個**長得幾乎一樣、但完全正當**的雙胞胎：**合理的後果推估**。後果推估是工程的日常美德——「如果我們在這裡放寬型別檢查，下游就可能傳進 null，於是那個沒做空值防禦的函式會炸」，這是有根據的因果推理，每一環你都能指出機制。差別不在「有沒有連鎖」，在**連鎖的每一環是否被證成**。

```text
合理的後果推估  vs  滑坡謬誤 —— 差在「每一環的證成」

  合理後果推估：A → B → C
    每一環都附機制：「為什麼 A 會導致 B」有具體、可檢驗的理由
    機率誠實：用「可能/很可能」，不假裝必然
    例：「放寬這個型別 → 下游可能收到 null（因為沒人擋）→ 那個函式會 NPE（因為它沒防禦）」
        每一步都指得出機制，且承認是機率不是鐵律

  滑坡謬誤：A ⇒ B ⇒ C ⇒ … ⇒ Z（災難）
    連鎖被當「必然」，但每一環的因果都沒給理由
    用「一定、必然、無可避免」掩蓋「其實只是有可能、而且越往後越不可能」
    例：「開這一個 API 例外 → 大家都會要例外 → 規範形同虛設 → 整個架構崩潰」
        每個箭頭都跳過了「為什麼一定會這樣」，且機率被連乘卻當成不變
```

**為什麼滑坡在數學上特別站不住？** 這裡有個你做機率直覺時會更熟的點（機制指向《馴服隨機》，本書只給直覺）：就算每一環「真的有可能」觸發下一環，比如每環有 70% 機率，**一條五環的鏈整體成真的機率是 0.7⁵ ≈ 0.17**——只剩一成七。滑坡謬誤的修辭手法，正是把這一連串「各自不到百分百」的機率，偷偷當成「每一環都必然」再串起來，於是「有點可能」被放大成「一定會」。鏈越長，這個偷渡越離譜——可是聽的人感覺不到，因為系統一不會自動幫你把那串機率乘起來，它只接收到結尾那個嚇人的 Z。

**它原本對在哪？** 對在「預警」這個功能。很多真實風險就是連鎖式的，先攔住第一步確實比事後收拾便宜。滑坡修辭之所以有說服力，是因為它寄生在一個真實的認知需求上：我們確實該為「這個小決定的下游後果」操心。它翻車的地方只有一個——**把該被證成的連鎖偷換成不必證成的必然**。

**怎麼自我察覺？** 對每一個「然後就會…」的箭頭單獨發問：**「這一步導致下一步，機制是什麼？機率多高？有沒有煞車？」** 真實的滑坡（如制度設計裡的破窗效應）經得起這個盤問、答得出機制；謬誤的滑坡答不出來，只會把音量調大。一個身為焦慮工程師的我特別在意的點：滑坡的反面——「**這只是個小改動，不會有任何下游影響**」——也是一種錯，那是把連鎖的機率謊報成零。誠實的版本永遠在中間：**列出連鎖的每一環、給每一環一個誠實的機率與機制，該設煞車的地方設煞車。**

## 循環論證：用結論證明結論

**循環論證／乞題（circular reasoning / begging the question）**：把要證明的結論，當成前提偷偷塞進論證裡，於是「推導」出來的東西，其實只是你一開始就假設了的東西。用 A 證明 A。

先澄清一個常被搞混的術語（2026-06 查證）：「**乞題（begging the question）**」嚴格說是「結論被當作前提之一預設了」；「**循環論證（circular reasoning）**」常指「為支持某主張而提出的理由，其實只是該主張的換句話說」。多數來源把後者當成前者的一種形式，學界對是否該嚴格區分有不同意見。本書統一把它們當同一類謬誤的兩種說法處理——它們的病灶是同一個：**論證沒有從「已知為真」走到「待證」，它原地踏步。**

循環論證最難抓，因為它的**邏輯形式是完美有效的**。「A，所以 A」這個推論，從前提到結論百分之百保真——前提真則結論必真（廢話，它們是同一句）。它一點都不違反邏輯規則。它壞在哪？壞在它**沒有提供任何資訊**：一個有效但循環的論證，前提的可信度完全等於結論的可信度，所以它對「我該不該相信結論」這件事**零貢獻**。這正好呼應 ch06 的核心區分——**有效（valid）不等於有用、不等於健全（sound）**：循環論證是「有效但沒用」的極端標本。

```text
循環論證的解剖（把它攤平就現形）：

  論證：「這個設計是對的，因為它遵循了最佳實踐。」
        「為什麼遵循最佳實踐就對？」
        「因為最佳實踐就是被公認為對的做法。」
       │
       ▼  把它寫成前提→結論
  前提：這個設計遵循最佳實踐
  隱藏前提：凡最佳實踐 = 對的做法（這就是把「對」塞進前提）
  結論：這個設計是對的
       │
       ▼  檢查：結論「對」是否獨立於前提被證成？
  沒有。「最佳實踐」這個詞本身就把「對」打包進去了。
  整個論證等於：「它對，因為它對。」
  —— 它沒給你任何「為什麼這個實踐在你這個脈絡下真的對」的理由
```

**它原本對在哪？** 嚴格說循環論證沒有「對的版本」——它不是被錯置的捷思，是論證結構的空轉。但有個近親是正當的：**用前提去定義術語**不算循環（「我把『可用』定義為 SLA ≥ 99.9%」是約定，不是論證）。會踩雷的是把一個**有實質內容的主張**包裝成「定義」，再從定義裡「推」出那個主張——例如把「微服務更好」偷塞進「現代架構」這個詞的定義裡，然後說「我們該用現代架構，因為微服務更好」。

**怎麼自我察覺？** 最有效的測試：**把論證的前提和結論並排，問「前提的可信度，是不是已經預設了結論的可信度？」** 如果為了相信前提，你得先相信結論，那它就是循環。工程現場最常見的循環是訴諸一個把結論打包進去的詞——「最佳實踐」「業界標準」「現代做法」「乾淨的程式碼」——這些詞本身就偷藏了「好」的判斷，用它們當理由就是在用結論證結論。下次有人用「這才是 clean code」當理由時，問他：「先不要用『clean』這個詞，告訴我這樣寫具體解決了什麼問題？」——抽掉那個打包了結論的詞，循環就現形了。

## 不當類比：表面像，關鍵點不像

**不當類比（faulty analogy，亦稱弱類比/錯誤類比）**：因為 X 和 Y 在某些方面相似，就推論它們在**另一個關鍵方面也相似**——而那個關鍵方面，恰恰是兩者**不一樣**的地方（2026-06 查證術語）。

類比是工程師的母語。我們靠類比理解新系統（「Kafka 像一個持久化的 log」）、靠類比設計（「這個 retry 機制借鏡 TCP 的指數退避」）。好的類比是強大的思考工具。不當類比的問題不在「用了類比」，在**把類比當成了證明**。

```text
類比的解剖：X 和 Y 共享性質 {a, b, c}，問能不能推出共享 d？

  好類比（啟發用）：
    X、Y 在 {a,b,c} 上像 → 「也許」在 d 上也像 → 去驗證 d
    結論：「值得一試」，類比是假設的來源，不是結論的證明

  不當類比（當證明用）：
    X、Y 在 {a,b,c} 上像 → 「所以」在 d 上一定也像   ← 跳步
    病灶：d 恰恰落在 X、Y 的「關鍵不同點（relevant disanalogy）」上
    例：「微服務就像樂高積木，拆開重組就好」
        共享：模組化、可組合（這部分像）
        要推的 d：「重組很容易、介面自動相容」
        關鍵不同點：樂高的凸點是標準化的物理介面；
                    你的服務介面是各自演化的、有版本、有狀態、有網路分區
        → d 正好落在最不像的地方，類比在這裡斷掉
```

**它原本對在哪？** 對在**生成假設**。類比是人類遷移知識最有效的機制之一——把已知領域的結構借到未知領域，省下從零摸索。系統一極擅長「這個像那個」的快速配對（這其實是代表性捷思的近親，見 ch04：靠「像不像」做判斷）。在「我該怎麼開始理解這個新東西」的脈絡，類比給對方向的機率很高。

**它在哪翻車？** 翻在**從「像」滑到「所以一定也像」這一步**——尤其當你要推論的那個性質，正好落在兩者的關鍵差異上。系統一只算「整體像不像」，不會自動去檢查「我要推的這一點，到底是落在相似區還是相異區」。這就是不當類比的下溝點：**相似度被當成了相關性。** 兩個東西在十個無關緊要的點上像，不會讓它們在那個你真正在乎的點上也像。

**怎麼自我察覺？** 對每個被當論據的類比，做一個動作：**明確寫出「這個類比要我接受的那個性質 d 是什麼」，再問「X 和 Y 在 d 這一點上，是真的像，還是恰好不像？」** 把焦點從「整體像不像」挪到「要推的那一點像不像」，不當類比立刻現形。code review 裡的版本：每次有人說「這就像 X 一樣」，把它改寫成「在『要推的這件事』上，它跟 X 真的一樣嗎？」

## 哥德爾濫用：不當類比的戲劇化標本（跨書連結）

現在把不當類比推到極端，看一個本書叢集的招牌案例：**哥德爾濫用（Gödel misuse）**。

哥德爾（Kurt Gödel）的不完備定理是 20 世紀邏輯學的高峰之一。它的精確陳述、條件與證明，本書**不展開**——那是姊妹書《這句話無法被證明》（godel）的主場（指向它談不完備定理的精確陳述）。本章只負責一件事：講清楚**為什麼把它套到社會、心靈、後現代論述上，是一種濫用**——而且它正是「不當類比」的教科書級標本。

先給定理一個**最低限度、夠用就好**的素描（精確版見《這句話無法被證明》）：哥德爾第一不完備定理大致說，任何**夠強、一致、且其公理可被機械枚舉的形式系統**（足以表達基本算術的那種），都存在一個它**既無法證明、也無法否證**的真命題。注意這句話裡每一個加粗的限定詞——「形式系統」「公理可機械枚舉」「足以表達算術」——它們是定理成立的**前提條件**，不是裝飾。

濫用是怎麼發生的？典型句式是：「**哥德爾證明了任何系統都有它自己無法把握的真理，所以**——理性是有限的／法律無法自洽／心靈超越機器／沒有絕對真理／我的後現代論點成立。」這每一句，都是把一個**只對特定形式系統成立**的數學結果，類比到一個**根本不是形式系統**的對象上。

```text
哥德爾濫用 = 不當類比 + 訴諸權威，雙重病灶：

  定理成立的對象 X：形式系統
    具備：形式語言、明確公理集、形式推論規則、可機械枚舉的定理集
       │
       │  濫用者的類比跳躍：「社會/心靈/法律/真理 也是一種『系統』」
       ▼
  被亂套的對象 Y：社會、人類心靈、法律、後現代論述…
    不具備：沒有形式語言、沒有明確公理、沒有形式推論規則、
            沒有「定理」這種可機械枚舉的東西
       │
       ▼  關鍵不同點（relevant disanalogy）：
  定理的全部前提條件，在 Y 上一個都不滿足
  → 「Y 也是系統，所以哥德爾定理適用於 Y」是不當類比：
     X 和 Y 共享的只是「系統」這個被偷換的鬆散日常詞，
     而定理依賴的「形式系統」是個極嚴格的技術詞 —— 這同時也是「偷換概念」
       │
       ▼  外加第二層病灶：訴諸權威（見 ch07）
  「哥德爾這麼偉大、定理這麼深奧」被當成結論的背書，
  但權威只在他證明的範圍內相關；套到範圍外，權威不再相關 → 訴諸（不相關）權威
```

這個診斷不是我發明的。法蘭森（Torkel Franzén）2005 年的《Gödel's Theorem: An Incomplete Guide to Its Use and Abuse》（2026-06 查證）正是系統性地整理這些誤用，並指出核心病灶：被亂套的對象**根本不是形式系統**，缺了形式語言、公理集、形式推論規則、可機械枚舉的定理集，因此定理**根本不適用**。更早，物理學家索卡（Alan Sokal）與布里克蒙（Jean Bricmont）1998 年的《Fashionable Nonsense》（*Intellectual Impostures*，2026-06 查證）也有專章處理後現代學界對哥德爾與混沌理論等的濫用，舉了把不完備定理當哲學萬用鑰匙的具體例子。

> **這裡值得停一下，因為它反直覺在哪很值得講。** 哥德爾濫用之所以特別有迷惑性，不是因為套用者笨，而是因為「系統」這個詞在日常語言裡太好用、太通用——社會是個系統、公司是個系統、生態是個系統。系統一聽到「形式系統有不可證的真理」和「社會是個系統」，飛快地把兩個「系統」對接起來，給出「所以社會也有不可把握的真理」這個聽起來深刻的結論。它**感覺**像一個洞見，因為它借用了一個真實而深刻的數學定理的權威。但那個對接點——「系統」這個詞——在兩處的意思天差地別：一個是有形式語言和公理的精確技術對象，一個是日常的鬆散比喻。**整個濫用，就建在這一個被偷換的詞上。** 這同時是不當類比（關鍵點不像）、偷換概念（同一個「系統」兩種意思，見 ch09 談 equivocation）、和訴諸不相關權威（見 ch07）的三重疊加——它是非形式謬誤的一個豪華組合包。

要拆穿它，動作還是那個：**把類比要你接受的那一點寫出來，再檢查它落在相似區還是相異區。** 哥德爾濫用要你接受的是「Y 也受不完備定理約束」，而定理的全部前提條件在 Y 上都不成立——它精準地落在最不像的地方。一句話的反制：**「你說的『系統』，有形式語言和公理集嗎？沒有的話，哥德爾定理就不適用——它講的是形式系統，不是任何叫『系統』的東西。」**

## 雙系統透鏡：這四個謬誤共用的下溝點

回扣脊椎（見 ch01）。本章四個謬誤都是非形式謬誤，它們的雙系統故事**和形式謬誤（ch05）不同、也和捷思偏誤（ch02–04）不完全相同**，值得單獨說清楚：

```text
非形式・結構型謬誤的雙系統路徑：

系統一（快、自動、永遠開）
   ├─ 收到一段論證，評估「它聽起來說服力夠不夠」
   ├─ 對「表面說服力」敏感：果斷的二分、嚇人的連鎖、
   │   權威的詞彙、貼切的類比 —— 一律給「合理」標籤
   └─ 不做「重建隱藏前提」這件費力的事 —— 它沒這個功能 ← 關鍵

系統二（慢、費力、會偷懶）
   ├─ 大多數時候：被表面說服力說服，省力放行 ✗（謬誤通過）
   └─ 偶爾介入：動手重建論證、把隱藏前提挖出來、檢查它證成了沒 ✓（抓到）
```

這裡要守住兩條誠實邊界：

第一，**這不是「捷思被錯置」那種偏誤**（不像可得性、代表性那樣是某個省力工具用錯場合）。結構型謬誤更像是**系統二該做的工作沒做**——重建隱藏前提是純費力的系統二活，系統一不會代勞，它只能告訴你「聽起來合理」。所以這類謬誤的防禦，幾乎全靠「逼自己動手重建」這個流程，光靠「更小心」沒用（去偏誤的邊界見 ch18）。

第二，**這也不是形式謬誤那種「結構本身無效」。** 假兩難、滑坡、不當類比的推論形式往往是有效的——問題在某個前提沒被證成；循環論證更是百分百有效卻零資訊。所以你不能用 ch05 的「檢查推論形式」抓它們，得用 ch06 的「重建前提、檢查健全性」。**形式謬誤查形式，非形式謬誤查前提**——這把鑰匙這一章四次都用得上。

而中央張力在這一章以一種特別安靜的方式出現：**這四個謬誤你在別人嘴裡一聽就覺得「怪怪的」，在自己嘴裡卻常常用得理直氣壯。** 你會在 PR 裡寫「不重構就只能等死」（假兩難）、會說「這是 clean code」（循環）、會講「微服務像樂高」（不當類比）——而當下你不會覺得自己在犯謬誤，你覺得自己在講道理。**在錯的脈絡用一個聽起來有力的論證結構，而且你察覺不到自己正在用**——這就是全書反覆叩問的那句話，在結構型謬誤這一層的樣子。

## 直覺的陷阱

本章四個謬誤最常被誤解、講錯、或自以為懂其實沒懂的地方如下。每條都附「徵兆」——出現它代表你正踩在溝邊。

```text
陷阱 A（具名）：把「合理的後果推估」誤判成滑坡謬誤
  常見誤解：只要看到「A 會導致 B 會導致 C」就喊「滑坡！」
  正確版：滑坡謬誤的病灶是「連鎖每一環沒被證成、卻當必然」，
          不是「有連鎖」本身。每一環都給得出機制與機率的後果推估
          是正當推理，不是謬誤。差別在「證成」不在「有沒有鏈」。
  自察徵兆：你用「滑坡」當駁斥標籤，卻沒去檢查對方每一環到底有沒有理由——
            那你自己正在用「貼標籤」取代「重建」。

陷阱 B：把循環論證當成「無效的推論」
  常見誤解：「循環論證邏輯上錯了。」
  正確版：循環論證的推論形式是『完美有效』的（A→A 保真）。
          它壞在『沒提供資訊／前提預設了結論』，不是『形式無效』。
          這正是 ch06「有效 ≠ 有用 ≠ 健全」的極端標本。
  自察徵兆：你想用「這推論不成立」駁循環論證——駁錯點了。
            該說的是「這論證有效但空轉，前提已預設結論」。

陷阱 C：訴諸謬誤 —— 「他犯了謬誤，所以他的結論錯」
  常見誤解：抓到對方用了假兩難/滑坡/不當類比，就斷定他的結論是錯的。
  正確版：指出謬誤只證明「這個『論證』沒成功支持結論」，
          不證明「結論本身為假」（訴諸謬誤/fallacy fallacy，見 ch06、landscape T10）。
          一個爛論證的結論，仍可能剛好是對的——只是這個論證沒能證明它。
  自察徵兆：你從「你的論證有謬誤」一路滑到「所以你錯了」——多走了一步。
            正確的收尾是「請給一個不靠這個謬誤的理由」。

陷阱 D：把「用了類比」本身當成謬誤
  常見誤解：「你這是類比，所以不算數。」
  正確版：類比是合法且強大的思考工具（生成假設、解釋、設計）。
          只有「把類比當證明、且關鍵點不像」才是不當類比。
          一句「這只是類比」就駁斥，跟一句「這只是相關」一樣偷懶（見 ch09、landscape T11）。
  自察徵兆：你駁斥一個類比卻說不出「它要推的那一點，具體哪裡不像」。

陷阱 E：把假兩難的兩個選項當成「窮盡」
  常見誤解：對方只給兩個，你就在這兩個裡面挑。
  正確版：假兩難的解藥不是「選對的那個」，是「先質疑選項集合本身完不完整」。
          二分法只在『真的互斥且窮盡』時成立（feature flag 開/關），
          多數工程決策是光譜，不是開關。
  自察徵兆：你發現自己在兩個選項裡糾結很久——先停下來問「是不是還有第三個」。
```

陷阱 A（滑坡 vs 合理後果推估）和陷阱 C（訴諸謬誤）是這章你最該帶走的兩個。它們有個共同教訓：**抓謬誤本身也可能變成一種謬誤。** 學會這些名字之後，最大的副作用是到處貼標籤——看到連鎖就喊滑坡、抓到謬誤就宣布對方全錯。真正的功夫不在「認出謬誤的名字」，在「動手重建論證、檢查那塊隱藏前提到底證成了沒」。名字是診斷的起點，不是結論。

以下題目純紙上推演（本書全程不放任何要你去跑的程式碼）。先自己想，再看解答。

**推演題 1（★★，[20 分鐘]）— 假兩難 + 滑坡的混合拆解。**
下面是一段（虛構但逼真的）技術決策論述。請你：(a) 重建它，補出每一個沒講出來的隱藏前提；(b) 標出它同時用了哪些本章謬誤、各在哪一句；(c) 給出至少兩個被它刪掉的選項，並對它的滑坡鏈逐環問「機制是什麼、機率多高」。

> 「我們要嘛現在就全面禁止任何人直接改 production 資料庫，要嘛就接受遲早會有人手滑刪掉全部資料。而且只要我們開一個『緊急情況可以手動改』的例外，下次大家都會說自己是緊急情況，然後例外變常態，然後我們的所有變更管控形同虛設，整個系統的可靠性就會徹底崩潰。所以唯一負責任的做法就是全面禁止、零例外。」

**推演題 2（★，[10 分鐘]）— 抓循環。**
判斷下面三句話哪幾句是循環論證，並對循環的那幾句，指出「結論被偷塞進哪個詞或哪個前提」。
(1)「這個架構是對的，因為它是業界標準。」「為什麼業界標準就對？」「因為大家都採用對的做法，這就成了標準。」
(2)「這個 API 回應慢，因為它的 P99 延遲很高。」
(3)「他的程式碼品質好，因為他寫的就是高品質的程式碼。」

**推演題 3（★★★，[25 分鐘]）— 哥德爾濫用的「為何不適用」拆解。**
有人在一場技術哲學閒聊裡說：「哥德爾不完備定理證明了任何系統都有它自己無法驗證的盲點，所以任何測試套件都不可能完整、QA 本質上是徒勞的。」請你不依賴定理的精確陳述（那指向《這句話無法被證明》），只用本章的「不當類比拆解法」，寫出這段話為何是哥德爾濫用：(a) 定理成立的對象需要哪些前提條件；(b) 「測試套件」這個對象是否滿足；(c) 這個類比要你接受的那一點 d 是什麼、它落在相似區還是相異區；(d) 給一句話的現場反制。

### 推演解答

**題 1 解答。**

(a) 重建（補隱藏前提）：

```text
論述拆解：

  主張1（假兩難）：
    明說前提：不全面禁止 → 遲早有人手滑刪光資料
    偷渡前提：選項「只有」全面禁止 和 放任手滑 兩個   ← 沒證成
    結論：必須全面禁止

  主張2（滑坡）：
    A：開一個「緊急可手動改」的例外
    ⇒ B：大家都宣稱自己是緊急情況
    ⇒ C：例外變常態
    ⇒ D：變更管控形同虛設
    ⇒ Z：系統可靠性徹底崩潰
    偷渡：每一個 ⇒ 都被當必然，無一給出機制與機率   ← 沒證成
```

(b) 謬誤標定：第一句「要嘛全面禁止、要嘛接受刪光」是**假兩難**；中段「開例外 → 大家都說緊急 → 例外變常態 → 管控失效 → 可靠性崩潰」是**滑坡**；結尾「唯一負責任的做法」又收回到假兩難（把光譜壓成單一選項）。

(c) 被刪掉的選項（攻破假兩難）：權限分級（只有特定角色能改、且需雙人覆核）、所有手動變更強制留審計 log 並事後 review、提供安全的 migration 工具讓人不必直接碰 DB、緊急通道但綁定事後 postmortem。逐環盤問滑坡：「開例外 → 大家都宣稱緊急」的機制是什麼？如果例外需要審批與留痕，這一環的機率就被煞車壓低；「例外變常態 → 管控失效」之間有沒有監控與稽核這道煞車？每一環都能加煞車，整條「必然崩潰」的鏈就垮了。

**常見錯路**：(1) 只抓到滑坡、沒抓到頭尾的假兩難——這段話的兩個謬誤是疊用的，假兩難負責把你框進兩個盒子，滑坡負責把其中一個盒子塗黑。(2) 反過來犯訴諸謬誤——「他用了假兩難，所以全面禁止一定是錯的」。不對：**全面禁止有可能恰好是這個團隊當下的合理選擇**（如果風險真的極高、人力極缺），只是這段論述沒能證明它——它靠謬誤而非靠理由得到結論。正確收尾是「請給一個不靠假兩難和滑坡的理由」。

**題 2 解答。**
(1) **是循環**。「對 → 因為是業界標準 → 因為大家採用對的做法成了標準」——「對」被偷塞進「業界標準」的定義裡，整個論證等於「它對因為它對」。
(2) **不是循環，但要小心**——它接近「換句話說」：「回應慢」和「P99 延遲很高」幾乎是同一件事的兩種講法，所以它沒解釋**為什麼**慢（沒給根因）。它不算典型循環論證（沒有把一個實質主張包裝成它自己的理由去說服你「該相信它」），比較像同義反覆的描述。誠實判法：如果有人用 (2) 來「論證該不該優化」，那它就空轉了；如果只是描述現象，那它只是沒資訊、不是謬誤。**這題的價值在邊界**——循環論證和「同義反覆描述」的界線。
(3) **是循環**，而且是最赤裸的版本：「品質好，因為他寫的是高品質的程式碼」——前提就是結論換句話說，零資訊。

**常見錯路**：把 (2) 也斬釘截鐵判成循環。它的病灶其實更輕（資訊不足/同義反覆），把它和 (1)(3) 那種「結論被打包進前提」混為一談，就抓不準循環論證的真正病灶——**前提的可信度預設了結論的可信度**。(2) 沒有這個預設關係，它只是沒往前走。

**題 3 解答。**

(a) 定理成立的對象需要的前提條件（最低限度素描，精確版指向《這句話無法被證明》）：一個**形式系統**——要有形式語言、明確且可機械枚舉的公理集、形式推論規則、以及「定理」這種可被機械枚舉的對象；而且要**夠強**（足以表達基本算術）、**一致**。

(b)「測試套件」滿足嗎？**不滿足**。測試套件不是一個有形式語言和公理集的形式系統；它是一組具體的檢查案例，沒有「從公理機械推導定理」這回事。「測試覆蓋不完整」是個真問題，但它的成因是工程現實（你寫不完所有 case、需求會變、組合爆炸），跟哥德爾定理講的「形式系統記憶體在不可判定的真命題」是**完全不同**的機制。

(c) 類比要你接受的那一點 d：「測試套件**必然**有它**原則上**無法驗證的盲點，所以 QA 徒勞」。它落在**相異區**——定理的「不完備」是關於形式系統內部的可證性的精確結果，套到「測試寫不寫得完」上，借的只是「不完整」這個日常詞的鬆散共鳴。而且就算測試不可能窮盡，也完全推不出「QA 徒勞」——這還多疊了一個假兩難（要嘛完整、要嘛徒勞，刪掉了「不完整但極有價值」這個真實選項）。

(d) 一句話反制：**「測試套件不是形式系統，沒有公理和形式推論規則，哥德爾定理講的是形式系統的可證性，不是『東西能不能被測完』——而且就算測不完，也不等於 QA 沒用。」**

**常見錯路**：跑去爭論哥德爾定理的精確內容（那是《這句話無法被證明》的場子，你不需要、也不該在這裡比誰更懂定理）。本章的拆解法只要求你做不當類比的標準動作——**寫出要推的那一點 d，檢查它落在相似還是相異區**——根本不必精通定理就能拆穿濫用。這正是哥德爾濫用最諷刺的地方：拆穿它不需要懂哥德爾，需要的是懂類比。

## 自我檢核

口頭自答；講得出來才算過關，講不清楚代表還沒懂。優先回答「為什麼」。

1. 本章四個謬誤（假兩難、滑坡、循環、不當類比）共用什麼機制？用一句話講出它們偷渡的東西各是什麼。
2. 為什麼本章的結構型謬誤**不能**用 ch05 抓形式謬誤的方法（檢查推論形式）來抓？該用什麼方法？
3. 滑坡謬誤和「合理的後果推估」差在哪一個關鍵點？給一個你工作中「看起來像滑坡、其實是正當後果推估」的例子。
4. 為什麼說循環論證的「邏輯形式是完美有效的」？那它到底壞在哪？這怎麼呼應 ch06 的「有效 ≠ 健全」？
5. 不當類比的下溝點是哪一步？為什麼「用了類比」本身不是謬誤、「整體很像」也不足以支撐結論？
6. 哥德爾濫用為什麼同時是不當類比、偷換概念、和訴諸權威三重病灶？被偷換的那個關鍵詞是哪一個？
7. 「訴諸謬誤（fallacy fallacy）」是什麼？為什麼「他犯了假兩難，所以他的結論一定錯」這句話本身也是個謬誤？
8. 為什麼這四個謬誤「你在別人嘴裡一聽就怪，自己用卻理直氣壯」？這跟系統一只評估「表面說服力」、不做「重建隱藏前提」有什麼關係？

## 延伸閱讀

- Franzén, T. (2005). *Gödel's Theorem: An Incomplete Guide to Its Use and Abuse*. A K Peters（https://en.wikipedia.org/wiki/G%C3%B6del%27s_incompleteness_theorems#Misconceptions）。哥德爾濫用的權威整理；讀它對「被亂套的對象根本不是形式系統」的論證，看本章「為何不適用」的拆解從哪來。定理本身的精確陳述指向《這句話無法被證明》。
- Sokal, A. & Bricmont, J. (1998). *Fashionable Nonsense: Postmodern Intellectuals' Abuse of Science*（*Intellectual Impostures*）。後現代學界濫用哥德爾、混沌、拓樸的具體案例集；讀「Gödel/混沌」相關章節看「借用科學權威」如何運作（2026-06 查證；語氣辛辣，當案例庫讀、結論自己判）。
- Internet Encyclopedia of Philosophy「Fallacies」條目（https://iep.utm.edu/fallacy/）。非形式謬誤的系統性目錄，含假兩難、滑坡、乞題、弱類比的學術定義；當作交叉查證術語的起點，比一般網路清單嚴謹。
- Walton, D. *Slippery Slope Arguments*（牛津，1992）。專書級地處理「滑坡何時是謬誤、何時是正當論證」；本章「合理後果推估 vs 謬誤滑坡」的分界，學界討論的深水區在這裡。
- 《這句話無法被證明》談不完備定理的精確陳述與條件。本章只給哥德爾定理的最低限度素描和「為何被濫用」，**定理到底說什麼、前提條件的精確意義、證明的骨架**指向它——讀完那本，你拆哥德爾濫用會更有底氣。
- 《如何不騙自己》談相關與因果、可證偽。本章的「德州神槍手」近親與「相關當因果」放在 ch09，制度怎麼防自欺指向那本；結構型謬誤的「重建論證」是個人層次的防線，制度防線在那本。
