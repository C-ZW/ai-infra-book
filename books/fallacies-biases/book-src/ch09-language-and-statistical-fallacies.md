# ch09 — 語言與統計型謬誤：模稜兩可・德州神槍手・相關當因果

> **本章解決什麼問題**：非形式謬誤的第三組，也是 Part IV 的最後一站。前兩章的謬誤錯在「攻擊錯對象」（ch07）和「論證骨架偷渡未證成的預設」（ch08）；這一組錯在更隱密的地方——**語言在句子中途偷偷換了意思**，或**統計訊號被當成了因果結論**。它們特別危險，因為每一步看起來都是「合理的句子」「真實的數字」，沒有明顯的扭曲、沒有明顯的假設，你得停下來盯著「這個詞前後是同一個意思嗎」「這個關聯撐得起這個因果嗎」才抓得到。本章也是通往 Part V（機率直覺崩壞）的橋：相關當因果與德州神槍手，是你對「資料說了什麼」的直覺第一次系統性翻車，而 Part V 會把這種翻車推到機率的核心（相關係數與檢定的演算法指向《馴服隨機》《如何不騙自己》，本章只給「為何直覺會這樣讀錯」）。

## 從你已知的出發

你看過太多這種句子，而且你自己講過：

> 「我們上線新版之後，P99 從 800ms 降到 300ms，**所以**新版的優化有效。」

念起來毫無破綻。數字是真的——你 dashboard 上就是這樣。時間順序也對——先上線，後變快。結論聽起來理所當然。

然後三週後你才發現：那次上線，**同時**換了一批新機器、Redis 剛好做完一次 failover 回到主節點、而且那天是週日流量低谷。新版的程式碼可能根本沒動到那條熱路徑。你把一個「同時發生」當成了「因為它」。

這就是本章的主角之一——**相關當因果（correlation ≠ causation）**。它在你工作裡天天跑：

- **歸因陷阱**：「改了 A、同時 B 好了」→「A 修好了 B」。但你那次部署改了不只 A。
- **dashboard 撈顯著**：你盯著二十條曲線，總有一條剛好在你做某件事之後動了，你就指著它說「看，有效」——這是**德州神槍手**（先射箭、再畫靶）。
- **偷換概念**：「這個服務是 stateless 的，**所以**沒有狀態問題」——`stateless`（不在本機保存 session 狀態）和「沒有任何狀態相關的故障」是兩個意思，但同一個「狀態」這個詞讓它們在句子裡滑過去了。這是**模稜兩可（equivocation）**。

這三類謬誤有個共同的狡猾之處：**它們不靠扭曲事實，靠的是「語言的滑動」和「對資料的過度解讀」。** 句子裡的每個詞都是真的、dashboard 上的每個數字都是真的——錯在詞與詞之間、在數字與結論之間那個沒人盯著的接縫。系統一（System 1，見 ch01）對「句子通不通順」「敘事合不合理」反應極快，會直接放行；系統二該做的「這個詞前後同義嗎」「這個關聯有幾種解釋」的檢查，太費力，常被偷懶蓋過。本章就是要把那幾個接縫一個一個拆給你看。

## 模稜兩可：同一個詞，中途換了意思

先講語言型的兩個謬誤，它們最乾淨、機制最清楚。

**模稜兩可／偷換概念（equivocation）**：一個論證裡，同一個詞（或詞組）在不同步驟用了**不同的意思**，靠「字面相同」讓推論看起來連貫，其實前提和結論根本講的是兩件事（2026-06 查證定義）。

它原本不是謬誤——一詞多義是自然語言的常態，你天天靠上下文消歧。`lock` 在「資料庫鎖」和「螺絲鎖」是兩個意思，你從不混淆，因為脈絡夠清楚。模稜兩可的翻車，發生在**同一段論證裡脈絡沒換、詞義卻偷偷換了**，於是「字面連貫」騙過了「意義連貫」。

教科書級的例子（2026-06 查證）：

```text
前提一：法律的懲罰是坐牢。
前提二：那枚火箭違反了「重力定律」。
結論　：所以那枚火箭該去坐牢。

偷換點：「定律 / law」
  前提一的 law = 人類社會的法律（會被違反、有罰則）
  前提二的 law = 自然律（描述性、無所謂「違反」與「罰則」）
  → 同一個字「law」，前後是兩個意思 → 結論不成立
```

工程版的偷換更不容易抓，因為它戴著技術術語的面具：

```text
「這個服務是 stateless 的，所以它沒有狀態問題（state-related bug）。」

偷換點：「狀態 / state」
  stateless 的 state = 「不在本機行程記憶體裡保存跨請求的 session 狀態」
  「狀態問題」的 state = 「任何跟可變狀態有關的故障」
  → 但 stateless 服務照樣會撞到外部狀態（DB、cache、訊息佇列）的
    競態、髒讀、stale cache、分散式一致性問題
  → 「不在本機存 session」推不出「沒有任何狀態相關故障」
```

你聽過這句話、可能也點過頭。它的危險正在於**第一個「狀態」是真的、第二個「狀態」聽起來也是真的**，兩個之間的滑動沒有任何警報。

**怎麼自察？** 模稜兩可的指紋是：**一個關鍵詞在論證裡出現兩次以上，而你能替它的兩次出現各寫一個不同的定義。** 抓它的動作就是——把那個詞的每一次出現都換成它**當下的精確定義**，重讀一遍。如果換完之後論證垮了，就是偷換。上面的火箭，換成「自然律」就垮了；stateless 換成「不在本機存 session」就垮了。

### 合成與分割：把部分的性質硬套給整體（或反過來）

模稜兩可的近親，是一對方向相反的謬誤（2026-06 查證）：

- **合成謬誤（fallacy of composition）**：因為**每個部分**有某性質，就斷定**整體**也有。
- **分割謬誤（fallacy of division）**：因為**整體**有某性質，就斷定**每個部分**也有。

```text
合成（部分 → 整體）：
  「這台機器的每個元件 P99 都 < 10ms，所以整條請求鏈 P99 < 10ms。」
  錯在哪：延遲會疊加、會有排隊與尾端放大；
         整體的 P99 不是各部分 P99 的簡單加總或最大值
         （尾端延遲在串接時會被放大——你很熟）。

分割（整體 → 部分）：
  「這個服務整體可用性 99.99%，所以它依賴的每個下游也有 99.99%。」
  錯在哪：整體可用性可能靠重試、降級、多副本「撐」出來，
         個別下游可能遠低於 99.99%；整體的性質是「湧現」出來的，
         不是每個部分都各自具備。
```

為什麼這是「對的捷思／錯的脈絡」（見 ch02 框架）？因為**很多性質確實會從部分傳遞到整體**——每個零件都很輕，整台確實偏輕；每行程式碼都通過 lint，整個檔案確實通過 lint。系統一學到「部分如此、整體多半如此」這條捷思，平均夠用。它翻車在**湧現性質（emergent property）**上：延遲的疊加、可用性的乘積、合唱的和聲、塞車的形成——這些是「整體才有、部分沒有」或「部分有、整體被稀釋／放大」的性質。系統一不會自動標記「這是不是一個會湧現的性質」，於是照搬了傳遞規則。

> **注意這裡和 ch08 的分界。** 合成／分割看起來像「結構」問題，但它**不是形式謬誤**（不是肯定後件那種「與內容無關的結構無效」，見 ch05）。它是不是謬誤，**取決於那個性質本身會不會在部分與整體之間傳遞**——也就是取決於**內容**。「每個零件輕→整台輕」常常對；「每個元件快→整體快」常常錯。同樣的論證骨架，換個性質就從有效變無效。這正是非形式謬誤的本質：你得重建「這個性質會不會傳遞」這個被省略的前提，才判得出來（承 ch06 隱藏前提）。

## 德州神槍手：先射箭，再畫靶

接下來進統計型。第一個是這章名字最好記的——**德州神槍手謬誤（Texas sharpshooter fallacy）**。

名字來自一個畫面：一個德州人對著穀倉牆壁亂開一堆槍，然後走過去，**在彈孔最密的那一叢周圍畫上靶心**，宣稱自己是神槍手。據查（2026-06），這個比喻最早出現在科學文獻、用來警告流行病學家別誤讀「疾病群聚」——一位流行病學家在 1977 年談霍奇金氏淋巴瘤群聚的論文裡用了這個故事；「德州神槍手」當成一個**謬誤名稱**被固定下來是更晚的事（約 1990 年代，這個命名時點請當作背景、非關鍵事實）。

謬誤的本體是：**先看到資料、再從資料裡挑出一個剛好顯著的樣式，然後假裝那個樣式是事先預測的、是有意義的**——同時忽略所有不合樣式的資料（那些「沒中靶」的彈孔）。

```text
正常的推論順序（先有靶，再射）：
  1. 先提出假設（畫好靶心）：「我猜 A 會降低延遲。」
  2. 蒐集資料（開槍）。
  3. 看資料是否落在預測上（中靶與否）。

德州神槍手（先射，再畫靶）：
  1. 先看一大堆資料（一牆彈孔）。
  2. 從裡面挑出剛好顯著／剛好好看的那一叢。
  3. 事後畫一個靶心套住它，宣稱「看，這就是我說的樣式」。
     → 忽略：你本來有多少個「可能顯著」的位置可挑
       （你掃過二十條曲線，總有一條會動）
```

工程現場你天天做這件事：

- 上線後你盯著 dashboard 二十條指標，**總有一條**在上線之後動了——你指著它說「優化有效」。但你沒算：二十條裡隨機就會有幾條碰巧波動（這牽涉「多重比較」，機制指向《如何不騙自己》談多重比較與 p-hacking）。
- A/B test 同時看十個次群（行動／桌機／各地區／各方案），**某一個次群**剛好顯著，你只報那一個。剩下九個不顯著的，你沒提——那些是沒中靶的彈孔。
- 事後 postmortem 找「根因」，你在一大堆 log 裡挑出一條「時間剛好對得上」的事件，宣稱找到了——但你沒問：有多少條事件的時間都大致對得上？

> **這裡值得停十分鐘，因為德州神槍手和「正常的探索性分析」長得幾乎一樣，差別只在一個動作的順序。** 探索資料、發現一個有趣的樣式——這完全沒問題，是科學的起點。**變成謬誤的那一刻，是你把「事後挑出來的樣式」當成「事前就成立的證據」來下結論**，而不去做下一步：拿一份**新的、獨立的**資料，事先把這個樣式當假設、再驗一次（中靶才算數）。挑樣式是免費的，驗證才要付錢。德州神槍手就是不付那筆錢，直接把免費挑到的當成付過錢的。

德州神槍手和 ch13 要講的確認偏誤是親戚但不同：確認偏誤是「我先有假設、只去找支持它的證據」；德州神槍手是「我先沒假設、在資料裡撈一個剛好顯著的、再倒推成假設」。一個是帶著靶去找符合的彈孔，一個是對著彈孔畫靶。

## 相關當因果：兩件事一起動，不代表一個推動另一個

這章的重頭戲，也是你工作裡最常犯的——**相關當因果（correlation ≠ causation）**。

它的本體：**A 與 B 在資料上一起變動（相關），不足以斷定「A 造成 B」。** 因為「A 與 B 相關」這同一筆觀察，至少還有三種同樣能解釋它的替代故事，而你的直覺會自動跳過它們、直接挑那個敘事最順的「A 導致 B」。

```text
觀察到：A 與 B 一起變動（相關）

  ┌─ 解釋①  A → B        A 真的造成 B（你直覺的預設、敘事最順）
  │
  ├─ 解釋②  B → A        反向因果：其實是 B 造成 A
  │                       （你把因果方向裝反了）
  │
  ├─ 解釋③  C → A 且 C → B   第三變因（共同原因 / confounder）：
  │                       有個 C 同時推動 A 和 B，A 與 B 之間沒有直接因果
  │
  └─ 解釋④  巧合         在夠多的變數裡，總有些會剛好一起動
                         （連到德州神槍手；spurious correlation）
```

四種解釋，**同一筆相關資料全都相容**。直覺的問題不是「想不到」這些替代解釋，而是**根本沒去想**——系統一拿到「A、B 一起動」就直接輸出「A 造成 B」，因為這個敘事最省力、最像因果。把那三條替代分支補出來，是系統二的活，而它常常沒被叫醒。

幾個有名的真實例子（2026-06 查證），專治「A 造成 B」的直覺：

- **冰淇淋銷量** 和 **溺水／暴力犯罪率** 高度相關。但不是冰淇淋讓人溺水——是**氣溫（第三變因 C）**同時推高了兩者：天熱大家吃冰、也下水游泳／在外活動。這是解釋③（共同原因）。
- 有人蒐集出 **人均起司消費量** 和 **被床單纏住而死亡的人數** 之間高達 0.9 以上的相關（Tyler Vigen 的「偽相關」蒐集）。這純粹是解釋④（巧合）：在你掃過成千上萬條時間序列時，總會撞出幾對碰巧同向走的——這正是德州神槍手在時間序列上的化身。

工程版的相關當因果，每一個你都遇過：

```text
「上線新版後 P99 降了，所以新版有效。」（解釋①的直覺預設）
  替代②（反向）：未必。但想想——是不是「因為流量先降了（B），
                 我們才敢上線（A）」？有時部署時機本身被狀況決定。
  替代③（第三變因）：那次上線同時換了機器 / Redis failover 完成 /
                 剛好週末低谷——C 同時造成「上了新版」和「變快」。
  替代④（巧合）：你那天剛好只看 P99 這條，沒看它前一週本來就在抖。

「加了這個 cache，錯誤率就下來了，所以 cache 修好了它。」
  替代③：同一次 release 是不是也修了那個 retry 邏輯？
  替代②：是不是錯誤率本來就在回落（前一次尖峰在退燒），
         你只是剛好在退燒時加了 cache？（這條會在 ch16 回歸均值再深挖）
```

> **誠實邊界（守住 landscape T11）：「相關不蘊含因果」也別矯枉過正。** 相關確實不蘊含因果，但相關**是**因果的必要證據之一——沒有相關，通常就沒有因果。所以「這只是相關所以沒意義」也是一種懶惰的駁斥，跟「相關就是因果」一樣不負責任。正確的態度不是一句「相關≠因果」就收工，而是**去查那三條替代分支**：有沒有第三變因？方向會不會反？樣本夠不夠大到排除巧合？要真正談因果，你需要的是**對照實驗**（控制住第三變因、隨機分組）——那是《如何不騙自己》的地盤（談對照實驗與因果推論的方法論）；本書只負責讓你看清「為何直覺會直接跳到①、漏掉②③④」。

附帶兩個常被一句話帶過、但同屬「語言滑動」的小謬誤：**訴諸自然（appeal to nature）**——「天然的所以好」（天然的砒霜也毒）；**訴諸新潮／訴諸傳統（appeal to novelty / tradition）**——「新的所以好」或「老做法所以對」。它們的共同毛病和模稜兩可一脈相承：把一個**描述性**屬性（天然／新／老）偷渡成一個**評價性**結論（好／對），中間那一步「為什麼這個屬性蘊含好」從沒被證成。

## 把雙系統透鏡套回這一章

回扣脊椎（見 ch01）。這一章的謬誤分兩種機制，別混講：

```text
語言型（模稜兩可、合成/分割、訴諸自然/新潮）
  S1：對「句子通不通順、字面連不連貫」反應極快 → 字面相同就放行
  S2 該做、卻偷懶沒做的：把關鍵詞的每次出現換成精確定義、檢查是否同義
  → 偏誤 = 對的捷思（靠字面與脈絡消歧，平均夠用）用在了
           「同段論證中途偷換詞義」的錯脈絡

統計型（德州神槍手、相關當因果）
  S1：對「敘事順不順、有沒有一條看起來動了的曲線」反應極快
      → 挑出最順的因果故事 / 最顯著的那叢彈孔，直接放行
  S2 該做、卻偷懶沒做的：補出②③④替代解釋、問「我事先預測過嗎、
                         我掃過多少個可能顯著的位置」
  → 偏誤 = 對的捷思（在乾淨環境裡「一起動」常代表「有關係」）
           用在了「多變數、可事後挑選、有共同因」的錯脈絡
```

兩種都是「對的捷思／錯的脈絡」（這是認知偏誤的回扣方式，不是形式謬誤——後者的回扣方式不同，見 ch05/ch06）。中央張力照舊成立：**你不會感覺到自己正在偷換概念或挑彈孔。** 那句「stateless 所以沒狀態問題」說出口時感覺很對；那條「上線後 P99 降了所以有效」寫進 postmortem 時感覺天經地義。察覺不到，正是它危險的地方。

## 直覺的陷阱

這一章的謬誤特別會偽裝成「合理的推論」，以下是最常見的幾個翻車點。

```text
┌──────────────────────────────────────────────────────────────────┐
│ 陷阱 A：「相關≠因果」喊完就收工（矯枉過正版，landscape T11）        │
│   誤解：學會「相關不是因果」後，看到任何相關就一句「這只是相關」    │
│         打發掉，當成已經破解。                                      │
│   下溝點：你把「不蘊含」當成「無關」。相關其實是因果的必要線索，    │
│           「只是相關所以沒意義」和「相關就是因果」一樣偷懶。        │
│   自察：問自己「我有沒有真的去查②反向、③第三變因、④巧合，           │
│         還是只是用一句口號逃避思考？」破解相關不是否認它，          │
│         是把它當成『該去追三條替代分支』的起點。                    │
├──────────────────────────────────────────────────────────────────┤
│ 陷阱 B：把德州神槍手當成「不能做探索性分析」                        │
│   誤解：以為「事後在資料裡找樣式」本身就是謬誤，於是不敢探索資料。  │
│   下溝點：探索資料、發現樣式完全沒問題，那是科學的起點。            │
│           謬誤只發生在「把事後挑到的樣式，當成事前成立的結論」。    │
│   自察：問「我是在『產生假設』還是在『下結論』？產生假設後，        │
│         我有沒有拿份新的獨立資料再驗一次？」挑樣式免費、驗證要付費。│
├──────────────────────────────────────────────────────────────────┤
│ 陷阱 C：以為偷換概念只發生在「明顯不同」的詞上                      │
│   誤解：覺得 equivocation 是「law=法律 vs 自然律」這種一眼看穿的。  │
│   下溝點：最危險的偷換，兩個意思非常接近、共用一個技術術語          │
│           （state、availability、test「通過」、「安全」）。         │
│   自察：把關鍵詞的每次出現都換成它『當下的精確定義』再讀一遍，      │
│         換完論證若垮了，就是偷換。                                  │
├──────────────────────────────────────────────────────────────────┤
│ 陷阱 D：把合成/分割謬誤誤當成「形式謬誤」（與 ch05/ch08 的分界）    │
│   誤解：覺得「部分→整體」是個固定的無效結構，像肯定後件。           │
│   下溝點：它是不是謬誤『取決於性質會不會傳遞』——是內容問題，        │
│           不是結構問題。「每個零件輕→整台輕」常對；                 │
│           「每個元件快→整體快」常錯。同骨架、換性質就翻面。         │
│   自察：問「這個性質是會在部分與整體間傳遞的，還是會湧現/稀釋的？」 │
└──────────────────────────────────────────────────────────────────┘
```

具名陷阱（本章招牌）：**陷阱 A「相關≠因果喊完就收工」**——這是工程師讀過幾篇科普後最容易得的病。你以為學會了破解因果謬誤，其實只是學會了一句新口號，用它逃避「真的去追三條替代分支」這件累活。守住 landscape T11 的正確版：相關不蘊含因果，但相關是因果的必要證據；破解它是去查，不是去喊。

## 紙上推演

純紙筆，不寫程式。算完自己複核一遍。

---

**題 1（★，[10 分鐘]）抓偷換。** 下面三句各藏一個「同詞兩義」的偷換。指出被偷換的那個詞、以及它前後各是什麼意思。

1. 「使用者**有權**說任何話，所以他這樣說是**對的**。」
2. 「這個 endpoint 的測試**通過**了，所以這個功能是**對的**（正確的）。」
3. 「這套設計很**安全**，因為它跑在公司內網——內網是**安全**的。」

**題 2（★★，[15 分鐘]）德州神槍手 vs 正當探索。** 你上線一個快取優化後，盯著 25 條 dashboard 指標，發現「東南亞地區、Android、付費用戶」這個次群的轉換率上升了 1.8%（其他 24 條沒明顯變化）。你在週會上報告「快取優化提升了東南亞 Android 付費用戶的轉換」。

(a) 這是不是德州神槍手？錯在哪一步？
(b) 如果你**真的**想知道這個次群有沒有被優化影響，下一步該做什麼？
(c) 把這題和「正當的探索性分析」劃清界線：探索到這個次群，本身有錯嗎？

**題 3（★★★，[20 分鐘]）拆相關當因果。** 一段真實風味的監控敘事：

> 「我們把 API 的 connection pool 從 50 開到 200 之後，那一週的 5xx 錯誤率從 0.8% 降到 0.2%。把 pool 開大解決了 5xx。」

(a) 列出至少**三個**「pool 開大」以外能解釋「5xx 下降」的替代解釋，分別對應反向因果②、第三變因③、巧合④。
(b) 要真正確認是「pool 開大」造成的，你需要做什麼樣的觀察或實驗（指向《如何不騙自己》的對照實驗思路，描述即可）？
(c) 為什麼「pool 開大→5xx 降」這個敘事在系統一裡那麼順、那麼難不信？

---

### 推演解答

**題 1 解答。**

1. 偷換的詞是「**權利／對（right）**」。第一個「有權」是 **權利／被允許**（你被容許做某事）；第二個「對」是 **道德上正確／良善**。「被容許說」推不出「這樣說在道德上是對的」——這是模稜兩可的教科書例（2026-06 查證）。常見錯路：覺得這句「聽起來怪怪的但說不上來哪錯」——說不上來，正是因為此例源自英文「right」的雙關（right = 權利 / right = 正確），翻譯成中文後「有權」與「對的」字面已無重疊，但語義滑動仍在；中文讀者感覺哪裡不對勁，卻難以精確指出換了什麼詞。
2. 偷換的詞是「**通過／對**」。「測試通過」＝**這幾個被寫出來的測試案例沒紅**；「功能是對的」＝**符合需求、所有情況都正確**。測試通過只覆蓋你寫到的 case（綠燈 CI 不等於功能正確，這條 ch06 講過健全性 vs 有效性）。常見錯路：把「測試通過」當成「功能正確」的同義詞——這正是偷換得逞的證據。
3. 偷換的詞是「**安全（safe / secure）**」。第一個「安全」泛指**整體沒有資安風險**；第二個「內網是安全的」其實只支持**「不直接暴露在公網」**這個窄義。內網照樣有橫向移動、內部憑證外洩、SSRF、供應鏈等風險——「在內網」推不出「沒有資安風險」。換成精確定義再讀，論證就垮了。

**題 2 解答。**

(a) **是德州神槍手。** 錯在：你掃過 25 條指標（25 個「可能中靶」的位置），事後挑出唯一動了的那一叢，事後畫靶說「優化提升了它」。在 25 條隨機波動的指標裡，碰巧有一兩條顯著動了，本來就會發生（這牽涉多重比較，機制指向《如何不騙自己》）。你把「事後挑出來的」當成「事前預測的結論」。

(b) **拿一份新的、獨立的資料，事先把這個次群當假設，再驗一次。** 具體做法（描述即可）：把「東南亞 Android 付費用戶轉換上升」當成一個**預先註冊的假設**，在**下一段時間**或一個**乾淨的 A/B 分流**裡只看這一個指標，看它是否重現。中靶才算數。

(c) **探索到這個次群，本身完全沒錯**——那是產生假設的正當起點（破陷阱 B）。錯的不是「發現它」，是「在發現它的同一份資料上、把它當成已證實的結論去下判斷」。挑樣式免費，驗證才要付費；謬誤是不付那筆驗證費。

**題 3 解答。**

(a) 三個替代解釋：

```text
②反向因果：是不是「那一週流量／壓力本來就低（B 先降）」，
           你才有餘裕去調 pool（A）？甚至是某次故障退燒後，
           團隊順手把 pool 也調了——錯誤率的下降另有其因。
③第三變因：那一週是不是同時做了別的事？——換了上游逾時設定、
           修了一個 retry 風暴、下游某依賴自己恢復了、
           或那週剛好沒有大促。C 同時造成「你調了 pool」與「5xx 降」。
④巧合／回歸均值：前一週 0.8% 是不是本來就是一個尖峰？
           尖峰之後本來就傾向回落（這條 ch16 回歸均值會深挖），
           你只是剛好在它回落時調了 pool。
```

(b) 要談因果，得**控制住其他變因**：理想是**對照**——例如把流量分成兩組，一組 pool=200、一組 pool=50，**同時段、同條件**比較 5xx；或在沒有別的變更的窗口裡，單獨改 pool 再觀察、並把它改回去看 5xx 是否回升（可逆操弄）。關鍵是排除②③④，這是對照實驗的精神（方法論指向《如何不騙自己》）。

(c) 因為這個敘事有**清楚的時間順序**（先調 pool、後 5xx 降）、有**現成的因果故事**（pool 太小→連線排隊→逾時→5xx，這機制本身合理），而且**是你做的**（你想要自己的操作有效）。系統一拿到「時間順序＋合理機制＋我做的」就直接輸出「我修好了它」，三條替代分支根本沒被叫醒。難不信，是因為它又順又恭維你。

## 自我檢核

口頭自答；講得出來才算過關，講不清楚代表還沒懂。優先回答「為什麼」。

1. 用一句話講：模稜兩可（equivocation）和稻草人（見 ch07）差在哪？（提示：一個偷換**詞義**、一個扭曲**對方立場**。）
2. 為什麼合成／分割謬誤**不是**形式謬誤（不像肯定後件，見 ch05）？「同樣的論證骨架、換個性質就從對變錯」這句話怎麼說明它是內容問題、不是結構問題？
3. 德州神槍手和「正當的探索性資料分析」差在**哪一個動作的順序**？為什麼「探索到一個樣式」沒錯、「在同一份資料上把它當結論」才錯？
4. 給定「A 與 B 相關」，把四種解釋（A→B、B→A、共同因 C、巧合）各用一個你工作中的例子講出來。為什麼系統一會自動只挑第一個？
5. 「這只是相關所以沒意義」這句話為什麼也是一種謬誤式的偷懶（landscape T11）？正確的態度是什麼？
6. 為什麼最危險的偷換概念，發生在兩個意思**很接近**、共用一個技術術語時（如 `stateless` 的 state）？你怎麼用「換成精確定義再讀一遍」抓出它？
7. 德州神槍手和確認偏誤（見 ch13）都跟「挑證據」有關，但機制不同——差在哪？（提示：一個帶著靶找彈孔，一個對著彈孔畫靶。）
8. 為什麼「上線後指標變好所以是我的功勞」這個敘事在系統一裡特別難不信？（至少講出兩個讓它變順的因素。）

## 延伸閱讀

- 維基百科「Texas sharpshooter fallacy」條目（https://en.wikipedia.org/wiki/Texas_sharpshooter_fallacy）。讀它的「穀倉牆／畫靶」比喻與流行病學疾病群聚的起源段落；當交叉查證起點，命名年代（約 1977 比喻、後續定名）以原始論文為準、別當鐵打事實。
- 維基百科「Equivocation」條目（https://en.wikipedia.org/wiki/Equivocation）。一詞兩義的標準例（包含「law＝法律 vs 自然律」「right＝權利 vs 正確」）；看它如何把「字面連貫」和「意義連貫」拆開。
- Scribbr「Correlation vs. Causation」（https://www.scribbr.com/methodology/correlation-vs-causation/）。讀它對第三變因（confounder）、反向因果、巧合三條替代解釋的整理；當成「拿到相關後該追哪三條分支」的清單。
- Tyler Vigen《Spurious Correlations》（https://www.tylervigen.com/spurious-correlations）。一整牆「高相關但純屬巧合」的時間序列（起司消費 vs 床單纏死、人造奶油 vs 離婚率）；看完你會對「掃夠多變數就會撞出假相關」有直覺——這正是德州神槍手在時間序列上的化身。
- 《如何不騙自己》談對照實驗與多重比較。本章只講「為何直覺把相關讀成因果、把事後挑的樣式當結論」；**怎麼用對照實驗排除第三變因、p-hacking 與多重比較的方法論**指向它。
- 《馴服隨機》談相關與獨立。本章不碰相關係數與檢定的演算法；**相關到底怎麼量、為何「無相關」不等於「獨立」**指向它。
