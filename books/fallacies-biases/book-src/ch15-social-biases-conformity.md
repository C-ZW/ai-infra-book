# ch15 — 社會偏誤與從眾：別人怎麼想，怎麼改變你怎麼想

> **本章解決什麼問題**：到這裡為止，本書談的偏誤幾乎都關在一顆腦袋裡——你的系統一（S1）跑了一個捷思、你的系統二（S2）沒攔下來（見 ch01）。但你不是在真空裡推理的。你在 code review 裡推理、在 postmortem 會議裡推理、在 on-call 群組裡推理——身邊有別人，而別人會改變你的判斷。這一章把雙系統透鏡架到**社會脈絡**上：當「大家都這麼說」「這是他造成的」這類社會訊號餵進 S1，捷思被放大、被扭曲，產生一批新的翻車——從眾、團體迷思、把別人的失誤怪到他「性格」上。這一章也是本書誠實層的前哨：你會看到，社會心理學最有名的幾個效應，數字比你以為的軟得多。機率怎麼算不在這裡（指向《馴服隨機》）；「人在賽局裡為什麼會有非理性的公平偏好」也不在這裡（指向《賽局》），本章只負責社會訊號如何改寫你的判斷。

## 從你已知的出發

你一定經歷過這個場景。一個 PR 擺在那裡，設計有點不對勁——某個邊界條件沒處理、某個抽象洩漏了。你正要在 comment 框裡敲下質疑，然後你滑到上面，看到團隊裡最資深的那位已經按了 approve，留了一句「LGTM, nice work」。

你的手指停了一下。

那一瞬間發生的事，就是這一章的全部。你原本要說的話沒變、你看到的問題沒變、程式碼一個字元都沒變。變的只有一個訊號：**一個你尊敬的人，公開表態說它沒問題**。而這個訊號，足以讓你開始懷疑是不是自己看錯了，足以讓那句質疑從「我要說」降級成「算了，可能是我多慮」。最後那個 bug 就這樣帶著兩個 approve 上了線。

把這件事拆開，你會看到本章的兩條主線同時在跑：

- **從眾（conformity）**：別人的判斷成了你判斷的輸入。不是因為他們給了你新證據（那是理性的），而是因為「跟大家一樣」這件事本身，對 S1 來說就是一個「安全、省力、八成沒錯」的訊號——於是 S1 把它當答案放行了。
- **基本歸因謬誤（fundamental attribution error）**：等這個 bug 炸了，postmortem 上有人說「就是他寫的時候不夠仔細」。一句話把整起事故的因果指向**那個人的性格**（粗心），而不是**那個情境**（review 流程沒有強制跑某個檢查、那天他在趕另一個 deadline、CI 根本沒覆蓋那條路徑）。對人不對系統。

這兩條主線有一個共同點，也是這一章要你帶走的核心：**它們不是新的捷思，是你已經認識的那些 S1 捷思，被「社會」這個維度放大或轉向了**。可得性（ch03）、代表性（ch04）這些工具還是那些工具，只是現在輸入端多了「別人在想什麼」「該怪誰」這類社會訊號，輸出就歪到新的方向去。你一個人 debug 時，S1 已經夠會騙你了；放進一個房間、加上幾個同事，它有了新的騙法。

還有一件你天天在做、卻沒注意到它是同一回事的：你 review 別人 code 時，一眼就把問題歸到「他不懂這個模式」；你自己寫出一模一樣的 bug 時，第一反應是「那天 spec 改來改去、時間又趕」。同一個錯誤，怪別人時怪到「他這個人」，輪到自己時怪到「當時那個狀況」。這個不對稱，本章後半會專門拆。

## 從眾：別人的答案，怎麼變成你的答案

先把最有名的標本擺上來，也順手把它的數字校準到誠實的版本——這正是本書招牌的誠實。

1950 年代，所羅門·阿希（Solomon Asch）做了那個後來進每一本心理學教科書的線段實驗（Asch 1956；我在寫這章時回查過設定與後續複製，見延伸閱讀）。設定很乾淨：

```text
阿希線段任務（Asch 1956）

  白色硬紙板一張卡：一條「標準線」。
  另一張卡：三條線 A、B、C，其中只有一條跟標準線一樣長，
            而且「一樣長的那條」明顯到不會看錯——
            單獨作答時錯誤率約 1% 以下（這題本身毫無難度）。

  房間裡 7~9 個人輪流大聲報答案。
  受試者不知道：除了他，其他全是套好招的「暗樁」。
  關鍵幾輪，暗樁們一致地、自信地報出一個「明顯錯誤」的答案。

  問題：真正的受試者，會跟著報錯，還是堅持自己眼睛看到的？
```

結果是這個實驗之所以有名的原因：**在暗樁集體報錯的那些關鍵輪次裡，受試者大約有三分之一（約 37%）的反應跟著報了錯**；而整個實驗下來，約四分之三的受試者**至少有一次**屈服於群體、報出他眼睛明明知道是錯的答案。明顯到單獨作答幾乎沒人會錯的題目，只因為旁邊幾個人異口同聲說別的，三分之一的反應就倒戈了。

這裡要立刻打誠實的預防針，因為「37%」常被當成鐵打的人性常數來說嘴，它不是：

```text
阿希從眾，誠實的數字（2026-06 查證）

  ・那個 37%，是「關鍵輪次的反應」層級、不是「人」層級——
    並非 37% 的人每次都從眾，而是從眾反應佔關鍵輪次約三分之一。
  ・只要「有一個暗樁倒戈、報出正確答案」，從眾率就大幅崩塌
    ——人需要的不是多數同意，是「全體一致」的那道牆；
    一道裂縫就夠你站住。
  ・改成「私下寫答案、不公開報」，從眾也大幅下降
    ——可見大半是「不想當眾跟大家不一樣」的社會壓力，
    不全是「真的相信自己看錯了」。
  ・後續複製數字飄移很大：Perrin & Spencer 1981 在英國工程系學生身上
    幾乎測不到從眾（趨近於零）；Bond & Smith 1996 的跨文化整合分析
    （133 個研究、17 國）發現從眾自 1950 年代以來在美國樣本下降，
    且集體主義文化的從眾普遍高於個人主義文化。
    → 從眾真實存在，但「有多強」高度取決於年代、文化、與情境設計。
```

我堅持把這段放在 worked example 之前，是因為它本身就是一個關於本書的教訓：**連「人會從眾」這種看起來天經地義的事，它的效果量都會隨脈絡大幅變動，沒有一個放諸四海的數字**。這正是 ch17 再現性危機那一章要全面展開的態度——任何偏誤數字都該配一個「在什麼條件下、多大」的標籤。

### 機制：兩種從眾，S1 都覺得划算

從眾為什麼會發生？阿希自己訪談受試者時就分出了兩種，後來社會心理學把它叫做「資訊性」與「規範性」影響。用雙系統透鏡（ch01）重看，兩種對 S1 來說都是「划算的捷徑」：

```text
兩種從眾，與它們各自「對的捷思 / 錯的脈絡」

┌──────────────────┬────────────────────────────┬────────────────────────────┐
│ 類型             │ 對的捷思（原本省了什麼）   │ 翻車的脈絡                 │
├──────────────────┼────────────────────────────┼────────────────────────────┤
│ 資訊性從眾       │ 「這麼多人都這樣判，       │ 當「多數」其實沒有         │
│ （我以為他們     │   他們大概知道些我         │ 獨立資訊（大家只是         │
│   知道得比我多） │   不知道的事」——在你       │ 互相抄、或都被同一個       │
│                  │   資訊不足時，參考         │ 錯誤來源餵），「多數」     │
│                  │   多數判斷是理性的省力。   │ 不再是證據，你卻還         │
│                  │                            │ 把它當證據。               │
├──────────────────┼────────────────────────────┼────────────────────────────┤
│ 規範性從眾       │ 「跟群體一致能維持         │ 當「正確」比「合群」       │
│ （我不想當眾     │   歸屬、避免衝突」——       │ 更重要時（安全審查、       │
│   跟大家不一樣） │   在多數社交場合，         │ 上線決策），你為了         │
│                  │   合群確實划算。           │ 不尷尬而放掉一個           │
│                  │                            │ 你其實看得到的問題。       │
└──────────────────┴────────────────────────────┴────────────────────────────┘
```

注意這張表沒有一格寫「人很懦弱」或「人很笨」。資訊性從眾在資訊不足時是**理性的**——別人的判斷確實常常是有用的證據，這正是同儕評審、群眾智慧能work的基礎。它翻車的點非常精確：**當那個「多數」並不獨立——大家其實只是互相看、或者都源自同一個有問題的判斷——它就不再帶新資訊，可你的 S1 還把「人數多」當成「證據強」**。這在工程裡有個你熟悉的名字：當一個錯誤假設在 incident 群組裡被第一個人喊出來，後面每個人附和都不是新證據，只是回聲；但十個人附和聽起來就是比一個人有把握。人數被誤當成證據——這是資訊性從眾最乾淨的翻車。

回到開頭那個 PR：資深工程師的 approve，對你 S1 來說既是資訊性（「他比我懂，他說沒問題大概真沒問題」）又是規範性（「他都approve了我還挑刺，會不會顯得我找碴」）。兩種影響疊在一起，那句質疑就咽回去了。而你當下的感覺**不是**「我屈服於社會壓力」，是「嗯，可能真的是我多慮」——中央張力又一次現身：**你被別人改變了判斷，卻感覺是自己改變了主意。**

## worked example：把「對人不對系統」的 postmortem 翻過來

從眾講完，把另一條主線——基本歸因謬誤——用一個完整的例子走透。這是本章我認為最該帶走的一頁，因為它直接決定你的 postmortem 寫出來是有用的還是有毒的。

基本歸因謬誤（fundamental attribution error，這個名字是社會心理學家羅斯（Lee Ross）1977 年取的；也叫對應偏誤 correspondence bias）說的是：**我們解釋別人的行為時，系統性地高估「他這個人怎樣」（性格、能力、態度），低估「他當時處在什麼情境」。** 看到一個人在路上對店員發飆，你的 S1 第一時間蹦出「這人脾氣真差」，而不是「他可能剛收到很糟的消息」。性格解釋現成、生動、好歸檔；情境解釋要你去想當時的處境，費力，S2 又懶得動。

把這把歪掉的尺套到一份真實的 postmortem 上：

```text
原始 postmortem 結論（對人不對系統）

  事故：支付服務在週五晚間高峰回傳大量 500，持續 23 分鐘。
  根因（原稿）：「工程師 A 在 PR #4821 裡把重試上限從 3 改成 30,
                 沒注意到這會在下游慢的時候放大流量、打垮連線池。
                 A 不夠仔細,審查時也沒人抓到。」
  行動項（原稿）：「提醒 A 以後改重試參數要更小心。
                   加強 code review 的責任心。」
```

讀起來像不像很多你看過的 postmortem？「不夠仔細」「更小心」「責任心」——全部指向**人的性格**。現在一步一步把它翻成「對系統不對人」，每一步標出基本歸因謬誤在哪裡把因果指錯了：

```text
逐步重寫：把因果從「人」移回「系統」

步驟 1：A「不夠仔細」這句話,藏了什麼沒講出來的情境前提?
        → 為什麼一個「把 3 改成 30」這種高風險改動,
          能在沒有任何護欄攔截的情況下一路上線?
        ✔ 自我複核：把「他粗心」換成提問,情境因素就浮出來了。

步驟 2：列出當時的情境因素(基本歸因謬誤讓我們略過的那些)：
        ・重試上限沒有設硬上限或告警,改成任意值都能合併。
        ・CI 沒有「重試 × 下游延遲」的壓力測試,這條路徑沒被覆蓋。
        ・PR 在週五下午五點合併,reviewer 也在趕自己的東西。
        ・「30」這個數字在 diff 裡只是一行,沒有任何視覺警示。
        ✔ 自我複核：每一條都是「換一個人來、同樣會中招」的系統洞。

步驟 3：套一個檢驗——「換掉這個人,事故還會發生嗎?」
        → 會。任何工程師在同樣的無護欄環境下,
          都可能合併一個放大流量的改動而不自知。
        → 既然換人不能解決,根因就不在「這個人」,在系統。
        ✔ 自我複核：這一步是區分「真‧個人因素」與「歸因謬誤」的關鍵。

步驟 4：重寫根因與行動項(對系統不對人)：
        根因：重試參數缺乏硬上限與變更告警;CI 未覆蓋
              「重試放大 × 下游降級」的交互;高風險配置改動
              在 diff 中沒有視覺標記,使 reviewer 難以察覺。
        行動項：① 為重試上限設硬上限(如 ≤5)+ 超過需二人核准。
                ② CI 加入下游延遲注入測試,覆蓋重試放大路徑。
                ③ 高風險配置改動在 PR 自動打標籤,強制提示 reviewer。
```

對照兩個版本：原稿學到的教訓是「A 要更小心」——下次換個人、或 A 在另一個忙碌的週五，一模一樣的事還會發生，因為**系統一點沒變**。重寫版學到的是三道護欄——它把「人」從因果鏈裡拿掉，換成「在這個系統裡，任何人都可能犯的、而且應該被攔下來的事」。

驚嘆點、也是你該能轉述給另一個工程師的是這句：**基本歸因謬誤最貴的代價不是冤枉了某個人，是它讓你停止找系統的洞。** 一旦你的 S1 滿足於「就是他粗心」這個現成的性格解釋，S2 就不會再費力去問「為什麼系統允許這件事發生」——於是你修了一個人的「態度」（修不動，也修錯了對象），卻漏掉了那個下次會用別的人重演一遍的結構缺陷。這就是為什麼「無咎責 postmortem（blameless postmortem）」不是企業文化的客套，是**對抗基本歸因謬誤的工程化護欄**（這也預告了 ch18：對抗偏誤靠改流程，不靠叫人「更有責任心」）。

### 行為者—觀察者：對自己手下留情

基本歸因謬誤還有一個你天天在犯、卻最難察覺的鏡像版：**怪別人時怪性格，輪到自己時怪情境**。這叫行為者—觀察者不對稱（actor–observer asymmetry）。

```text
同一件事,兩套歸因

  別人的服務掛了          →「他們架構爛、不懂容錯」      (性格/能力)
  我的服務掛了            →「上游突然抽風、需求又改」    (情境)

  別人 PR 有 bug          →「他不熟這塊」                (能力)
  我 PR 有 bug            →「那天 spec 改三次、時間超趕」(情境)

  別人 on-call 漏接告警   →「他不夠上心」                (態度)
  我 on-call 漏接告警     →「那個告警噪音太多、半夜睡死」(情境)
```

你看自己時，你**知道**所有那些情境因素——你親身在裡面，那些壓力、那些改來改去的需求、那個吵到你麻木的告警，對你來說是鮮活的背景。你看別人時，你**看不到**他的背景，只看到結果（服務掛了），於是 S1 抓最現成的解釋：他這個人。資訊的不對稱（你有自己的背景、沒有別人的背景）直接餵出歸因的不對稱。

這裡又要上本書的誠實。行為者—觀察者不對稱長期被當成社會心理學的鐵律寫進教科書，但 2006 年馬萊（Bertram Malle）做了一個整合分析（173 個研究），結果出乎意料：**把各種條件平均起來、再修正發表偏誤後，這個不對稱的整體效果幾乎趨近於零**；它只在很特定的條件下才穩定出現（例如解釋的是負面事件、或行為者被描述得很特異）。

我要小心地說清楚這代表什麼、不代表什麼（這正是 ch17 教的區分）：

```text
行為者—觀察者不對稱,誠實版（2026-06 查證）

  ✗ 不能說：「人怪別人怪性格、怪自己怪情境」是鐵打定律。
            ——大規模整合分析顯示,平均效果趨近於零。
  ✗ 也不能說：「這效果是假的、不存在」。
            ——它在特定條件(如負面事件)下仍會出現。
  ✓ 該說：這個不對稱「沒有原本以為的那麼普遍、那麼強」,
          高度受事件正負向、行為者特異性等脈絡調節。
  ⚠ 但「基本歸因謬誤」本身(對『他人』高估性格)作為一種傾向
     仍相對站得住——只是它也受文化調節(集體主義文化較少出現),
     並非無條件普世。
```

所以本章我給你的不是「記住這條鐵律」，而是一個更穩的、可操作的版本：**當你要把一起事故歸因到「某人就是怎樣怎樣」之前，先做那個步驟 3 的檢驗——「換一個人來，還會發生嗎」**。這個動作不依賴任何一個搖晃的效果量是否成立，它只依賴一個你一定同意的邏輯：如果換人還會發生，那根因就不在人。

## 直覺的陷阱

社會偏誤這塊地雷特別多，因為它的素材（從眾、旁觀者、團體迷思）大多來自社會心理學最戲劇化、也最常被通俗化扭曲的那批研究。逐個拆，守住 landscape 的正確版。


最該停下來十秒的是**陷阱 ①**。凱蒂·吉諾維斯案是整個「旁觀者效應」進入大眾文化的引信，而它的流傳版本——「38 個鄰居冷眼旁觀」——是一個被原始新聞報導誇大、後來被學者考證打回去的都市傳說。我特別點它，是因為它示範了本書一再強調的事：**一個生動、符合「人很冷漠」這個現成敘事的故事，會繞過你的查證、直接進駐你的記憶**（這正是 ch03 可得性捷思的社會版——生動的故事被高估為真）。當你下次想引用「研究證明旁觀者很冷漠」時，先問一句：我引的是實驗室的對照研究，還是那個被新聞放大的吉諾維斯故事？

第二該停的是**陷阱 ③ 團體迷思**。這個詞太好用了——「我們團隊陷入團體迷思」聽起來既專業又有解釋力——以至於很容易忘記它在科學上站得多不穩。它源自詹尼斯（Irving Janis）1972 年對幾起政治決策災難（如豬玀灣事件）的事後個案分析，**幾乎沒有對照實驗支撐**，而且它的核心概念定義模糊到難以嚴格檢驗。我不是要你別用這個詞——它確實抓到了一個真實的群體現象——而是要你用它時誠實：它是一個**有啟發性的描述框架**，不是一個**被實驗確立的因果機制**。這跟團體層面真正站得比較穩的東西（如 Asch 從眾、少數人異議能瓦解一致性）是不同等級的證據。

## 紙上推演

全部用紙筆。不要碰鍵盤，這一章沒有任何要你去跑的東西。

---

**第 1 題（★，[10 分鐘]）：在 code review 裡抓從眾**

回想一次你（或你看過別人）在 code review／設計討論裡，因為「資深的人先表了態」而把原本要提的意見咽回去的時刻。把它寫成三行：(1) 你原本看到的問題是什麼；(2) 改變你的那個「社會訊號」是什麼；(3) 你當下感覺自己是「被壓力影響」還是「被說服了」。然後回答：這是資訊性從眾、規範性從眾，還是兩者都有？你怎麼分辨？

---

**第 2 題（★★，[15 分鐘]）：把一份 postmortem 從「對人」翻成「對系統」**

拿一個你經歷過或想像得出的事故，先用「對人不對系統」的方式寫一句根因（含至少一個性格／能力歸因，如「他不夠仔細」）。然後套本章 worked example 的步驟 1–4，把它重寫成「對系統不對人」：列出被略過的情境因素、做「換人還會不會發生」檢驗、給出至少兩道改環境的護欄。最後一句：基本歸因謬誤在原稿的哪個詞上把因果指錯了？

---

**第 3 題（★★，[15 分鐘]）：自己給自己設計一個從眾的「裂縫」**

本章說「一個暗樁倒戈就能讓從眾大幅崩塌」。把這個機制反過來用：設計一個你團隊可以實際採用的**會議流程小改動**，讓「不同意」變得更容易說出口（提示：想想「先各自寫、再公開」「指定一個唱反調的角色」「資深的人最後才表態」這類做法各自在阻斷哪一種從眾）。說明你選的那個改動，是在對付資訊性從眾還是規範性從眾，還是兩者。

---

**第 4 題（★★★，[20 分鐘]）：誠實標註一個社會偏誤**

從本章四個「常被當鐵律」的效應裡挑一個（Asch 從眾、團體迷思、行為者—觀察者不對稱、吉諾維斯/旁觀者效應），不查書，先憑記憶寫下你**以前相信的版本**；然後寫下本章給的**誠實版**（效果有多強、受什麼調節、哪部分站得住哪部分被打折）。最後回答：這個「打折」屬於「效果不存在」還是「效果較小／受脈絡調節」？為什麼這個區分很重要（連到 ch17）？

---

### 推演解答

**第 1 題。** 檢查點：你有沒有把「我原本看到的問題」和「改變我的社會訊號」分開寫清楚？很多人寫不出第一項——因為一旦從眾發生，連「我原本看到了問題」這件事都會被記憶悄悄抹掉（「其實本來也沒那麼確定吧」），這正是中央張力：你察覺不到自己被改了。分辨資訊性 vs 規範性的關鍵問句：**「如果這個資深的人是私下、只對我一個人說『我覺得沒問題』，我會不會同樣改變主意？」** 如果會，偏向資訊性（你真的把他的判斷當證據）；如果你發現自己主要在意的是「當眾反駁他會不會尷尬」，那就有強烈的規範性成分。常見錯路：把所有從眾都歸成「我太軟弱」——這違反本書規矩。資訊性從眾在資訊不足時是理性的省力；它翻車只在「那個多數其實不獨立、不帶新資訊」時。

**第 2 題。** 標準答案是你自己的事故，但結構固定。原稿那句性格歸因（「他不夠仔細」）就是基本歸因謬誤指錯因果的地方——它把一個由系統缺陷（沒有護欄、沒有覆蓋、沒有視覺警示）決定的結果，歸到了一個人的特質上。「換人還會不會發生」這個檢驗是解題核心：只要答案是「會」，根因就不在人。常見錯路：(1) 護欄寫成「提醒大家更小心」——那還是在改人，不是改環境，等於沒翻過來；(2) 走到另一個極端，完全否認任何個人因素——這也不對，本章的點不是「人永遠無辜」，是「先排除系統因素、再談個人因素」，因為 S1 的預設恰恰相反（先抓人）。真正的個人因素（如蓄意繞過流程）是存在的，只是它遠比 S1 以為的少。

**第 3 題。** 這題沒有唯一解，檢查的是你有沒有把「流程改動」對準「正確的從眾類型」。三個常見做法各打哪一種：

```text
流程改動            主要對付              機制
─────────────────────────────────────────────────────────
先各自寫、再公開    規範性從眾           作答時還沒看到別人的答案,
                                          就沒有「當眾跟大家不同」的
                                          壓力——等於改成「私下作答」,
                                          正是 Asch 裡讓從眾崩塌的條件。

指定唱反調的角色    一致性(unanimity)   制度性地製造一個「暗樁倒戈」,
(devil's advocate)                       讓房間裡永遠有一道裂縫,
                                          瓦解「全體一致」這道牆。

資深的人最後表態    資訊性 + 規範性       資淺的人先說,他們的判斷
                                          不被資深者的早期表態錨定
                                          (連到 ch04 錨定),也減少
                                          「他都說了我還反對」的壓力。
```

常見錯路：選了「指定唱反調角色」卻說它在對付規範性從眾——它主要對付的是「全體一致」這個觸發條件（提供異議來源），跟「資訊性 vs 規範性」是不同的切法。能講清楚「我這個改動是在阻斷哪一步」就算過關。

**第 4 題。** 以行為者—觀察者不對稱為例的標準解：
- 以前相信的版本：「人怪別人怪性格、怪自己怪情境，這是穩固的社會心理學定律。」
- 誠實版：Malle 2006 對 173 個研究的整合分析顯示，修正後**平均效果趨近於零**，只在特定條件（如負面事件、行為者被描述得特異）下穩定出現。
- 這個打折屬於哪一類：屬於「**效果較小／受脈絡強烈調節**」，**不是**「效果不存在／造假」。它在某些條件下仍會出現，只是沒有原本以為的那麼普遍、那麼強。
- 為什麼區分重要：把「未能穩定複製」誤讀成「這效果是假的」，會讓你從「過度相信」直接跳到「全盤否定」，兩個都是錯的（這正是 ch17 的核心鐵律）。誠實的態度是給每個效應一個帶條件的標籤：「在 X 條件下、約 Y 強度」，而不是非真即假。常見錯路：把所有「沒站穩的社會心理學效應」一鍋端說成「都是造假」——這既不公平也不準確，多數是效果縮水或受脈絡調節，不是學術不誠實。

## 自我檢核

口頭回答，講不清楚就是還沒懂。優先回答「為什麼」。

1. 從眾的兩種機制（資訊性、規範性）各自「對的捷思」對在哪、「錯的脈絡」錯在哪？用你自己的 code review 經驗各舉一例。
2. 為什麼說「一個暗樁倒戈就能讓從眾大幅崩塌」這件事，比「37% 從眾率」這個數字更值得你記住？它對你設計會議流程有什麼直接啟示？
3. 基本歸因謬誤在一份「對人不對系統」的 postmortem 裡，具體是在哪個詞、哪一步把因果指錯了？「換一個人來還會不會發生」這個檢驗為什麼能抓出它？
4. 行為者—觀察者不對稱「平均效果趨近於零」（Malle 2006）——這是不是說「人怪別人怪性格、怪自己怪情境」根本沒發生過？「效果較小／受脈絡調節」和「效果不存在」差在哪、為什麼這個差別是本書的命脈？
5. 凱蒂·吉諾維斯「38 個冷漠目擊者」為什麼是個該被你警惕的故事？它怎麼示範了 ch03 可得性捷思的社會版（生動的故事如何繞過查證）？
6. 「團體迷思」這個詞，你會在什麼場合用、用的時候要加什麼誠實的但書？它跟 Asch 從眾在「證據強度」上差在哪？
7. 這一章的社會偏誤，跟前面 ch03–04 那些「一個人的捷思」是全新的東西，還是舊捷思被社會維度放大／轉向的結果？舉一個例子說明這個延續性。
