# ch11 — 合取謬誤：Linda 與「更具體反而更可能」

> **本章解決什麼問題**：這是 Part V「機率直覺崩壞」的第二站。ch10 拆了基率忽視——人忽略事前比例、只盯著「像不像」。這一章要給你整本書裡我認為最反直覺的一頁：**你在一句話後面加上一個細節，讓它更具體，多數人就會判它「更可能」——明明多一個條件只會讓機率更低，不可能更高**。這不是新的偏誤，是 ch04 那個代表性捷思在機率脈絡的又一次翻車現場，而且翻得比基率忽視還難看：基率忽視至少是「忽略了一個數字」，合取謬誤是「違反了一條連小學生都同意的包含關係」。機率公式怎麼證——P(A∩B) ≤ P(A) 為何恆成立——是《馴服隨機》的地盤，本章不重推；我只負責讓你看清**人的直覺在哪一步把「更像」誤當成「更可能」、為什麼當下毫無自覺，以及這個效果其實沒有你以為的那麼鐵**。

## 從你已知的出發

凌晨三點，PagerDuty 把你叫起來。某個服務的 P99 飆高、錯誤率上升。你睡眼惺忪，腦子裡蹦出一個假設。比較下面兩句，哪一句你會覺得「對，八成就是這個」：

```text
假設 A：「是上游某個依賴出問題了。」
假設 B：「是上游那個剛上線的 recommendation 服務，
         在晚間流量高峰時，因為快取雪崩、
         打爆了它後面的 DB 連線池，
         拖垮了我們的同步呼叫。」
```

老實說，B 讀起來**就是**比 A 可信。它有畫面、有機制、有時間點、有因果鏈。你會點頭：「對，很可能就是這樣。」A 太空泛了，空泛到讓你不安。

但停十秒——這裡你的直覺正要騙你，而且你不會有感覺。

假設 B 是假設 A 的**子集**。B 描述的每一種情況，都是 A 描述的情況（「上游某個依賴出問題」）外加一大串額外條件：剛上線、晚間高峰、快取雪崩、打爆連線池、同步呼叫。每多一個「而且」（AND），這個聯合事件成立的機率就只會**更低或持平，絕不可能更高**。A 是大圈、B 是大圈裡的一小塊。一小塊不可能比包著它的大圈更可能發生。

可是你的感覺剛好相反——你覺得 B 更可能。這個「更具體、更有故事性的假設讀起來更可信，但機率上必然更低」的落差，就是這一章的主角：**合取謬誤**（conjunction fallacy）。它每天在你的 incident channel 上演：越具體、越「像」某個你見過的故障原型的假設，越「像」對的，於是越被你押注——而它每多一個 AND，命中所有條件的機率只會更小。

你已經有了拆它需要的全部零件。你知道 P(A∩B) ≤ P(A) 這條規則的存在（style-guide 說你不需要被教這個）。你知道「子集不可能比母集更大」是常識。你甚至寫過一堆 `WHERE a AND b` 的查詢，加一個條件只會讓回傳的列數變少或不變、絕不會變多——這就是合取律的工程版，你天天在用。這一章奇怪的地方不是「規則難」，而是：**你完全同意這條規則，卻在 incident 現場、在 Linda 題前面，照樣違反它，而且違反的當下感覺自己很理性。**

## Linda 是誰：本書最有名的一頁

先把這一章的標本擺上檯面。1983 年，特沃斯基（Amos Tversky）與康納曼（Daniel Kahneman）在《Psychological Review》（90:293–315）發表了後來被引用到爛的「Linda 問題」（我在寫這章時回查過設定與數字，見本章延伸閱讀）。題目大致是這樣：

```text
給受試者一段人物描述：

  「Linda 31 歲，單身，直言不諱，非常聰明。
   她主修哲學。學生時代她深切關注歧視與社會正義議題，
   也參加過反核遊行。」

然後問：下面哪一個比較可能？
  (a) Linda 是銀行員。
  (b) Linda 是銀行員，而且積極參與女性主義運動。
```

結果：**約 85% 的受試者選 (b)**——「銀行員而且女性主義者」比「銀行員」更可能。

這個 85% 是 1983 年原始研究在特定措辭下的數字，我要先打一個誠實的預防針：它**不是**「人類在任何表述下都這樣」的鐵律，後面〈直覺的陷阱〉會專門拆這件事（landscape T6）。但先把這個原始結果放在你眼前，因為它違反的東西非常乾淨：

(b)「銀行員 ∧ 女性主義者」是 (a)「銀行員」的子集。每一個「既是銀行員又是女性主義者」的 Linda，都首先**是個銀行員**。所以「銀行員」這個集合**包含**「女性主義銀行員」這個集合。一個被包含的子集，機率不可能大於包含它的母集。用記號寫：

```text
令 B = {Linda 是銀行員}
令 F = {Linda 是女性主義者}

(a) 的事件 = B
(b) 的事件 = B ∩ F        （既是銀行員、又是女性主義者）

合取律（恆真，證明指向《馴服隨機》）：
        P(B ∩ F) ≤ P(B)
        ──────────────  即  P(b) ≤ P(a)，永遠成立
```

所以正確答案在邏輯上**沒有模糊空間**：(a) 至少和 (b) 一樣可能，通常更可能。判 (b) 更可能，是直接違反機率的合取律。可 85% 的人——包括統計學訓練良好的受試者、修過機率課的研究生——照樣選了 (b)。

> 合取律 P(A∩B) ≤ P(A) 為什麼恆成立、怎麼從機率公理證出來，是《馴服隨機》的地盤（見《馴服隨機》談集合與機率公理）。本書不重推，只用結果。我這一章要回答的是另一個問題，一個機率課不會教你的問題：**為什麼一個你一旦看見就同意的規則，你在當下卻違反了它、而且毫無自覺？**

## 機制：代表性捷思又一次「拿錯尺」

合取謬誤不是新偏誤，它是 ch04 那個代表性捷思（representativeness heuristic）在機率脈絡的翻車。機制和基率忽視（ch10）同源——同一個引擎、不同的翻車口。

回憶 ch04 的核心：你的系統一（S1，見 ch01）被問「X 屬於哪一類、有多可能」時，不會去做機率計算，它會偷偷把問題換成一個更好答的：「**X 跟這一類的原型有多像？**」然後用「像不像」的程度當「有多可能」的答案。在分類日常物件時這把尺好用得不得了。問題在於——它量的是相似度，不是機率，而這兩者可以被拆開。

現在把這把尺套到 Linda 上。那段描述（直言、關心社會正義、反核遊行）刻意寫得**很像「女性主義者」的原型**，而**一點都不像「銀行員」的原型**（銀行員給人的刻板印象是保守、安分、不上街）。於是 S1 評估兩個選項時，跑的根本不是機率，是相似度：

```text
S1 偷偷把問題換掉了：

  你以為自己在回答：哪個「更可能」（機率）
  S1 實際在回答的是：哪個「更像描述裡的 Linda」（相似度）

  選項 (a)「銀行員」
     → 跟 Linda 的描述像不像？不太像。（Linda 不像個銀行員）→ 低分

  選項 (b)「銀行員 + 女性主義者」
     → 跟 Linda 的描述像不像？「女性主義者」這部分超像！→ 拉高總分

  於是 S1 判定：(b) 比 (a) 更「像 Linda」 → 誤當成 (b) 更「可能」
```

關鍵的反直覺點在這裡：**加上「女性主義者」這個條件，在「相似度」這把尺上是加分，在「機率」這把尺上卻是減分。** 多一個細節讓 (b) 更貼合 Linda 的原型（相似度上升），同時多一個 AND 讓 (b) 的機率必然下降。兩把尺的方向**正好相反**。S1 用了相似度那把尺，於是把一個機率更低的選項判成了「更可能」。

這就是為什麼我說它是「對的捷思、錯的脈絡」的最佳標本（這是全書脊椎，見 ch01、ch02 的雙系統透鏡）。代表性捷思在「判斷像不像」時是對的工具——Linda 確實更像女性主義者，這個判斷沒錯。它翻車是因為被錯置到「判斷機率」這個它不適用的脈絡：在這裡，多一個條件讓「更像」和「更可能」分道揚鑣，而 S1 抓了「更像」那一條路走。

### 為什麼 S2 沒攔下來

按 ch01 的雙系統透鏡，這裡 S2（那個慢的、會算的、本該介入的系統）該做的事很簡單：注意到 (b) 是 (a) 加一個條件，套用合取律，判 (a) ≥ (b)。這對 S2 來說是小學程度的運算。

可是 S2 沒做。為什麼？因為 S1 已經給了一個**感覺很有把握**的答案。(b)「就是更像 Linda」這個感覺是直接、生動、有說服力的。S2 收到這個「答案已經有了、而且很篤定」的訊號，就執行了 ch01 講的「省力預設放行」——它沒有感覺到有需要介入的地方。**S2 偷懶不是因為它算不出來，是因為它根本沒被叫醒**：沒有任何警報告訴它「這裡有個合取關係要檢查」。合取律不會自己跳出來提醒你它適用，你得主動想到「欸，(b) 是不是 (a) 的子集」——而在故事的牽引下，你想不到。

這帶我們到全書的中央張力，它在這裡以合取謬誤的形式出現：**你違反合取律的當下，感覺不到自己在違反**。你不會覺得「我忽略了 (b)⊆(a)」，你只會覺得「(b) 顯然比較合理」。漏掉一個檢查，不會在意識裡留下痕跡——你沒辦法察覺一件你「沒做」的事。

## worked example：把兩個選項畫成集合

口說無憑，把它畫出來。下面這張集合包含圖，是我認為整章最值得你盯著看十秒的東西。我在寫的時候自己重走了一遍，確認方向沒錯。

```text
所有可能的 Linda（樣本空間）

   ┌──────────────────────────────────────────────┐
   │                                                │
   │     B = 銀行員（選項 a）                       │
   │   ┌──────────────────────────────┐             │
   │   │                              │             │
   │   │   B ∩ F = 銀行員 ∧ 女性主義  │             │
   │   │   （選項 b）                 │             │
   │   │   ┌────────────────┐         │             │
   │   │   │////////////////│         │             │
   │   │   │////  這一塊  ///│  ← (b)  │  ← (a) 是   │
   │   │   │//// 是 (a) 的 //│         │     整個 B  │
   │   │   │//// 一小部分 //│         │             │
   │   │   └────────────────┘         │             │
   │   │                              │             │
   │   └──────────────────────────────┘             │
   │                                                │
   └──────────────────────────────────────────────┘

   (b) 是 (a) 裡面的一塊斜線區域。
   一塊區域的面積，不可能大於包著它的整個區域。
   ∴ P(b) ≤ P(a)，無一例外。
```

現在一步一步走，每步都自我複核：

```text
步驟 1：(b) 描述的 Linda，必然也滿足 (a) 嗎？
        是。「銀行員而且女性主義者」的人，首先是個銀行員。
        → 所以 (b) 的每一個成員，都是 (a) 的成員。
        → 集合上：(B ∩ F) ⊆ B。      ✔ 自我複核：子集關係成立

步驟 2：子集的機率，能大於母集嗎？
        不能。機率是「佔樣本空間的比例」，
        子集佔的那塊面積，不可能比包含它的母集那塊面積大。
        → P(B ∩ F) ≤ P(B)。          ✔ 自我複核：這就是合取律

步驟 3：那為什麼直覺判反了？
        因為直覺量的不是面積（機率），是「離 Linda 的原型多近」。
        加上「女性主義者」讓 (b) 在原型上更近 → 相似度上升。
        但相似度上升的同時，集合在縮小 → 機率下降。
        兩件事方向相反，S1 跟著相似度走，就翻了。

結論：唯一正確答案是 (a)「銀行員」至少和 (b) 一樣可能。
      判 (b) 更可能 = 把一塊斜線區域說成比整個大框更大。
```

驚嘆點在哪、值得你能用自己的話轉述給另一個工程師的是這句：**多給一個細節，讓事情聽起來更具體、更可信、更「對」——但每一個「而且」都在把集合縮小、把機率往下壓。你的「更可信」感和真實機率，在這裡是反向的。** 凌晨三點那個 B 假設之所以危險，正是這個：它最具體、最有故事、最像你上次見過的那次故障，所以你最想押它；而它要全部條件同時成立的機率，恰恰是所有假設裡最低的。

## 直覺的陷阱

合取謬誤是個漂亮的標本，但它周圍埋了好幾顆地雷，踩中任何一顆你就把這一章學歪了。逐個拆，並守住 landscape 的正確版。

```text
┌────────────────────────────────────────────────────────────────┐
│ 陷阱清單（合取謬誤）                                           │
├────────────────────────────────────────────────────────────────┤
│ 誤解 ①「85% 是鐵打的人性常數」                                 │
│   正確版：85% 是 1983 原始研究、在特定措辭下的數字。           │
│   改用頻率版（「100 個像 Linda 的人裡，幾個是…」）違反率掉到   │
│   約 20–40%；用下注措辭（不提『機率』、把『而且』講死）掉到    │
│   約 10–50%。效果真實、但受表述強烈調節——不是「人在任何問法    │
│   下都這樣」。（landscape T6，2026-06）                        │
│                                                                │
│ 誤解 ②「合取謬誤證明人是壞掉的機率機器」                       │
│   正確版：不是。代表性在「判斷像不像」時是對的工具，只是被     │
│   錯置到「判斷機率」。而且部分受試者其實把 (a) 讀成了「銀行員  │
│   但不是女性主義者」（語用學上的合理解讀）——若真這樣解讀，     │
│   選 (b) 就未必違反邏輯。措辭引發不同解讀，是這題的一大爭點。  │
│                                                                │
│ 誤解 ③「這只是個心理學玩具，跟我工作無關」                     │
│   正確版：它就是你 incident 假設越具體越想押的那個傾向。每多   │
│   一個「而且」（剛上線 ∧ 高峰 ∧ 快取雪崩 ∧ …），命中全部的     │
│   機率只會更低；最有畫面的假設，往往是最不可能全中的那個。     │
│                                                                │
│ 誤解 ④「把它跟基率忽視當同一回事」                             │
│   正確版：同一個引擎（代表性），不同翻車口。基率忽視＝忽略     │
│   事前比例（ch10）；合取謬誤＝違反子集包含。別混講。           │

```

最該停下來的是**誤解 ①**。我認為這是這一章最重要的誠實：Linda 題常被當成「看，人類就是這麼不理性」的鐵證拿來說嘴，但它的違反率對你**怎麼問**極度敏感。

特沃斯基與康納曼自己在 1983 年就報告過：把問題從「機率」改成「頻率」措辭——不是問「Linda 是銀行員的機率比較大，還是銀行員兼女性主義者的機率比較大」，而是問「**在 100 個符合這段描述的人裡，有幾個是銀行員？有幾個是銀行員兼女性主義者？**」——違反率會大幅下降（我在寫這章時回查過：頻率版通常掉到約 20–40%，下注範式約 10–50%，但不會歸零、仍可靠存在，見延伸閱讀）。吉仁澤（Gerd Gigerenzer）與赫特維希（Ralph Hertwig）這一派把這當成核心論點：人對「單一事件的機率」這種不自然的問法很不在行，但換成「頻率」這種我們演化上熟悉的格式，直覺就回來不少——這跟 ch10 自然頻率法救基率題是同一個道理。

這件事的意思**不是**「合取謬誤是假的」。它真實、可重複、跨各種受試者出現。它的意思是：**這個偏誤有多嚴重，一部分取決於你用哪種措辭召喚了哪個捷思**。「機率」這個字眼模糊、可被多重解讀，給了 S1 用相似度抄捷徑的空間；「100 人裡有幾個」這種頻率問法把問題釘死在數數上，逼 S2 去看集合。措辭如何召喚不同的捷思，本身就是個關於認知的教材——也是為什麼我堅持引用那個 85% 時一定要標「這是原始措辭下的數字」。

## 紙上推演

全部用紙筆，不要碰鍵盤。算術手算，集合用畫的。

---

**第 1 題（★，[10 分鐘]）：把直覺翻成集合**

不要用 Linda。用一個你自己工作裡的例子：找一個「加了細節聽起來更可信、但機率其實更低」的 incident 假設或產品判斷。把它寫成 (a)（粗版）和 (b)（細版 = 粗版 ∧ 一個額外條件），畫出兩者的集合包含關係，並用一句話講清楚「為什麼 (b) 讀起來更可信、機率卻更低」。

---

**第 2 題（★★，[15 分鐘]）：頻率版重講**

把 Linda 題改寫成**頻率版**：用「在 100 個符合這段描述的人裡」開頭，重寫 (a) 和 (b) 兩個問句。然後用自己的話解釋：為什麼光是把「機率」換成「100 人裡有幾個」，多數人就不再犯合取謬誤了？這對你「怎麼問一個問題」有什麼啟示？（提示：S1 用哪把尺，跟你怎麼問有關。）

---

**第 3 題（★★，[15 分鐘]）：手算一個合取掉多快**

不要去推合取律的證明（那是《馴服隨機》的事）。只用乘法直覺，手算下面這個 incident 假設鏈，感受「每多一個 AND，機率掉多快」。假設這五個條件**彼此獨立**（現實未必，但這裡只為看數量級），各自的事前機率如下，手算逐步聯合機率，放 ` ```text ` 對帳：

```text

  剛上線的服務出問題      P = 0.5
  發生在晚間流量高峰       P = 0.3
  且是快取雪崩            P = 0.2
  且打爆了 DB 連線池       P = 0.4
  且拖垮我們的同步呼叫     P = 0.5
```

**第 4 題（★★★，[20 分鐘]）：雙面性口頭重講**

不寫字，對著空氣（或一隻橡皮鴨）講 3 分鐘：代表性捷思在 Linda 題裡，「對的捷思」對在哪、「錯的脈絡」錯在哪。要明確講出「相似度」和「機率」這兩把尺在加上『女性主義者』時為何方向相反。再接一句：為什麼引用「85%」時必須誠實標註它受措辭調節。

### 推演解答

**第 1 題。** 標準答案是你自己的例子，但檢查點固定：(b) 必須真的是 (a) 加條件（(b) ⊆ (a)），不能是兩個不相干的假設。常見錯路：舉的 (b) 其實**不是** (a) 的子集，而是換了一個假設——那不是合取謬誤，是別的東西。正確的長相一定是「(a) 而且還……」。一個乾淨的例子：(a)「這次延遲是下游慢」；(b)「這次延遲是下游那個新版本的 gRPC 客戶端在連線數超過 200 時的 bug 導致的」。(b) 顯然是 (a) 的一小塊，卻因為具體、有機制而讀起來更可信。畫圖時，(b) 的框一定要整個畫在 (a) 的框**裡面**。

**第 2 題。** 頻率版範例：

```text
「想像 100 個符合上面這段描述的人。
   其中有幾個是銀行員？        ____ 人
   其中有幾個是銀行員兼女性主義者？ ____ 人」
```

為什麼這樣問就少犯了？因為「100 人裡有幾個」這種問法把問題釘死成**數數**：你被迫想像一群人，然後在「銀行員」這群裡圈出「也是女性主義者」的那一小撮——這時子集關係幾乎是用看的就看出來了（圈出來的那撮，當然不會比整群銀行員多）。而「哪個機率大」這種單一事件的問法是抽象的、模糊的，給了 S1 空間用「哪個更像 Linda」偷偷把問題換掉。啟示：**問題的措辭會決定你召喚哪個捷思**。同樣一件事，問「機率」召喚相似度捷思（容易翻車），問「100 個裡幾個」召喚數數（逼出 S2）。這跟 ch10 自然頻率救基率題是同一招——也呼應 ch18 去偏誤的核心：改變表述（改環境）比叫人「更小心」（改人）有效。常見錯路：以為頻率版能讓違反率歸零——不會，只是大幅下降（landscape T6），效果仍可靠存在。

**第 3 題。** 手算：

```text
五個條件全中（獨立假設下，連乘）：
  0.5 × 0.3 × 0.2 × 0.4 × 0.5
= 0.5 × 0.3            = 0.15
= 0.15 × 0.2          = 0.03
= 0.03 × 0.4          = 0.012
= 0.012 × 0.5         = 0.006

  → 「五個條件全中」的機率 ≈ 0.006，也就是 0.6%。

對照「只要第一個條件」：
  P(剛上線的服務出問題) = 0.5 = 50%

差幾倍：0.5 / 0.006 ≈ 83 倍。
```

✔ 自我複核：0.5×0.3=0.15、×0.2=0.03、×0.4=0.012、×0.5=0.006，重算一致。

啟示：那個最具體、最有故事的假設 B（五個條件全中）只有約 0.6% 機率，比「上游某依賴出問題」這個粗假設低了約 83 倍。**直覺把最不可能的那個排到最前面去查。** 正確的查法是反過來：先驗證粗的、機率高的那一層（「真的是上游嗎？」），逐層加條件、逐層縮小，而不是一頭栽進那個最香的完整故事。常見錯路：以為現實中這些條件獨立——通常不獨立（高峰常伴隨雪崩），所以真實聯合機率未必是單純連乘；但「每多一個 AND、機率單調不增」這個方向**永遠**成立，數量級的教訓不變。（精確的相依機率怎麼算，指向《馴服隨機》談條件機率。）

**第 4 題。** 檢查點：你有沒有講出「兩把尺方向相反」這個核心？「對的捷思」＝代表性在判斷「Linda 像不像女性主義者」時是對的，她確實更像，這個相似判斷沒問題。「錯的脈絡」＝把這把相似度尺借去量「有多可能」。加上「女性主義者」這個條件，在相似度尺上是**加分**（更貼原型），在機率尺上是**減分**（多一個 AND、集合縮小）；S1 跟著相似度走，於是把機率更低的選項判成更可能。誠實標註那段：「85%」是 1983 原始措辭下的違反率，改頻率版或下注版會明顯下降，所以它證明的是「在某些問法下人傾向用相似度代替機率」，不是「人類在任何情況下都這麼不理性」。常見錯路：把它講成「人類很笨、不懂機率」——這違反本書規矩（謬誤不是愚蠢，是錯置的捷思）。
