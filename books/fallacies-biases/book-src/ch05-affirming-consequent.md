在進去之前，我想先講清楚這一章和前四章在機制上是**兩種不同的東西**，因為這正是本書最容易被讀者（也包括寫書的我自己）含糊帶過的分界線。

前四章講的是**認知偏誤**：S1 跑了一個原本對的捷思（可得性、代表性、錨定），被你拖到一個它本來不該負責的脈絡，於是給出系統性偏掉的答案。那套故事的框架是「對的工具、錯的場合」。

這一章講的是**形式謬誤**，它**不適用**那套故事。肯定後件不是「一個被用錯場合的捷思」——它是一個**邏輯結構**，這個結構本身無效，跟你拿它去推什麼內容毫無關係。你不能說「肯定後件在某個脈絡是對的」，因為它在**任何**脈絡都不是有效的演繹。等一下我們會看到，雙系統透鏡在這裡只回扣一個方向：**為什麼 S1 會覺得一個無效推論看起來很合理**。注意這句話的精確措辭——不是「肯定後件是個捷思」，而是「S1 對邏輯形式不敏感，憑表面合理度放行了一個無效的形式」。這兩句話差很多，本章從頭到尾守住這個差別。

## 從你已知的出發

`if/then` 是你的母語。你一天寫幾十個條件判斷，閉著眼睛都不會錯。但我要說的是：你寫得出正確的 `if`，不代表你在**讀別人的因果敘述**時不會把 `if` 反過來——而且你不會有感覺。

來看一個你每天都在做、而且每天都在犯這個錯的場景：debug。

你的服務出事了，症狀是 **502 暴增**。你心裡馬上冒出一個假設：「上游那個會逾時的服務又掛了。」這個假設不是憑空來的，它背後有一條你信得過的因果規則：

```text
規則：若 上游服務逾時（P），則 我們會吐 502（Q）       P → Q
```

這條規則是對的。上游逾時確實會讓你吐 502。現在你觀察到 **502 暴增（Q 出現了）**，於是你的結論是：

```text
觀察：502 暴增（Q）
結論：所以一定是上游逾時（P）                          Q，∴ P ← 等一下
```

停在這裡。你剛剛做的這一步，叫**肯定後件（affirming the consequent）**。它是一個**形式謬誤**。

問題出在哪？`P → Q` 只保證「上游逾時 ⇒ 會有 502」這個方向。它**完全沒有保證**反過來「有 502 ⇒ 一定是上游逾時」。502 可以有一百種原因：你自己的程式 panic、連線池耗盡、記憶體爆掉、某個 nginx 設定錯了、DNS 解析失敗⋯⋯每一種都會吐 502，每一種都跟上游逾時沒半點關係。`P → Q` 為真，**不蘊含** `Q → P` 為真。

你之所以會直接跳到「一定是上游」，不是因為你邏輯爛，而是因為這個假設**最好想**（可得性，ch03，上次出事就是它）、**最像**那個熟悉的故障原型（代表性，ch04）。捷思在這裡負責「為什麼你偏偏想到這個假設」；但「`P → Q` 加上 `Q`，所以 `P`」這個**推論動作本身**的無效，是純粹的邏輯結構問題，跟你選了哪個假設無關。換成任何別的假設，這個倒推一樣無效。

這就是形式謬誤跟前四章偏誤的分水嶺：**偏誤決定你把哪個內容填進去，形式謬誤決定那個推論的骨架能不能成立**。本章只管骨架。

## 一條規則，四種推論：兩個有效、兩個謬誤

把所有東西收乾淨。一條條件命題 `P → Q`（讀作「若 P 則 Q」），你可以對它做四種推論。只有四種，不多不少，因為你能「肯定／否定」前件 P，也能「肯定／否定」後件 Q，2×2＝4。

```text
前提：P → Q（若 P 則 Q）
─────────────────────────────────────────────────────────────
名稱                          已知      結論      有效?    為什麼
─────────────────────────────────────────────────────────────
肯定前件 modus ponens   (MP)  P         ∴ Q       有效     P 成立 ⇒ Q 被規則保證
否定後件 modus tollens  (MT)  ¬Q        ∴ ¬P      有效     Q 沒發生 ⇒ P 不可能成立
肯定後件 affirming cons.(AC)  Q         ∴ P       無效     Q 可由別的原因來
否定前件 denying ante.  (DA)  ¬P        ∴ ¬Q      無效     P 不成立 Q 仍可由別的原因來
─────────────────────────────────────────────────────────────
```

這張表是這一章的全部。我下面的任務只有兩件事：(1) 讓你**信服**前兩種有效、後兩種無效，不是靠背、是靠你自己能在腦裡走一遍；(2) 讓你看到自己其實天天在犯後兩種。

先講有效的兩個，它們很短。

**肯定前件（MP）**：規則說「若 P 則 Q」，現在 P 真的成立了，那 Q 一定跟著成立——因為規則就是這麼保證的。這就是你寫 `if (P) { doQ() }` 的全部信任基礎。沒有爭議。

**否定後件（MT）**：規則說「若 P 則 Q」，現在我觀察到 **Q 沒發生**。那 P 能成立嗎？不能。因為如果 P 成立了，規則會強迫 Q 發生；可是 Q 偏偏沒發生，所以 P 一定沒成立。這個推論工程師其實非常熟，它就是**否證**的骨架：「如果是上游逾時，監控一定會看到上游的 timeout 計數上升；可是上游 timeout 計數是平的（¬Q）；所以這次不是上游逾時（¬P）。」你每次用「如果是 X，就該看到 Y；沒看到 Y，所以不是 X」排除一個假設，你用的就是 MT。它有效。

現在是兩個謬誤，這才是重點。

**肯定後件（AC）**：規則「若 P 則 Q」，觀察到 **Q 發生了**，於是斷定 **P 成立**。錯在哪？前面 debug 的例子已經演過：`P → Q` 是單行道，它沒鋪反方向的路。Q 可以有別的來源，所以「Q 出現」不能反推「P 成立」。

要徹底說服自己一個形式無效，最乾淨的辦法是**找一個反例**——一個讓前提全真、結論卻假的填法。內容選得越白癡越好，因為形式謬誤跟內容無關，用白癡內容反而看得最清楚：

```text
P → Q 填成：若 下雨（P），則 地是濕的（Q）           這條規則沒問題
已知       ：地是濕的（Q）
結論（AC） ：所以 下過雨（P）                          ← 無效

反例：有人剛拿水管澆了花。
      前提「若下雨則地濕」仍然真、「地是濕的」也真，
      但「下過雨」是假的。前提全真、結論假 ⇒ 推論無效。
```

只要存在**一個**這種反例，這個推論形式就不可能是有效的，因為「有效」的定義就是「前提真則結論必真、不容許任何例外」。澆花的水管就是那個例外。

**否定前件（DA）**：規則「若 P 則 Q」，觀察到 **P 沒成立（¬P）**，於是斷定 **Q 也不會發生（¬Q）**。一樣用同一條下雨規則打它：

```text
P → Q 填成：若 下雨（P），則 地是濕的（Q）
已知       ：今天沒下雨（¬P）
結論（DA） ：所以 地不會濕（¬Q）                       ← 無效

反例：今天沒下雨，但有人拿水管澆了花。
      地照樣是濕的。前提全真、結論假 ⇒ 推論無效。
```

注意 AC 和 DA 其實是同一個錯的兩種長相：它們都偷偷假設了「`P → Q` 也保證 `Q → P`」「`P` 是 `Q` 的**唯一**原因」。一旦 Q 還有別的來源（澆花的水管、別的故障原因），這個唯一性假設就破了，兩種推論一起垮。AC 是「看到結果就咬定那個原因」，DA 是「那個原因沒出現就斷定結果不會有」——它們是把單向蘊含當成雙向的兩面。

我把這個自我複核做完整一次，因為 style-guide 要求我不准跳步、要自己重走一遍：四種形式，MP 與 MT 我找不到任何反例（前提真時結論被強制為真），AC 與 DA 我各自用「澆花的水管」這同一個反例就打穿了。四種都對帳完畢，跟我記憶裡的標準答案一致——但我不是靠記憶下結論的，是靠剛剛那兩個反例。

## 為什麼這四個長得這麼像，你卻分得出對錯——的時候

值得停十分鐘的地方在這裡：四種推論在自然語言裡長得**幾乎一樣**，都是「若…則…，而且…，所以…」。你的 S1 在讀的時候，分辨它們靠的不是邏輯結構，而是「**聽起來順不順**」。而「順」這個感覺，剛好在 AC 和 DA 上最會騙你。

為什麼 AC 特別順？因為現實世界裡，很多 `P → Q` 的關係**確實接近雙向**。「若發燒則體溫高」幾乎可以反過來說。「若這個 API 回 200 則請求成功」在多數情境也近似雙向。S1 從這些「剛好雙向」的例子裡學到一個粗略的捷思：看到 Q 就往 P 想。這個捷思在「P 真的是 Q 唯一原因」的脈絡裡會給對答案——但它**不是**「肯定後件這個形式有效」，它是「S1 憑經驗賭 P 和 Q 雙向」。一旦 Q 有別的來源，賭就輸了，而你不會收到任何「我剛剛做了無效推論」的內在警報。

這就帶到本章唯一的雙系統回扣，而且我要非常小心地措辭，因為這正是 style-guide 反覆強調、最容易講歪的地方：

```text
雙系統透鏡（ch01）在形式謬誤上的回扣 —— 只有一個方向

  ✗ 不要說：肯定後件是「一個用錯場合的捷思」。
            （它不是捷思，是一個無效的演繹形式，在任何場合都無效。）

  ✓ 要說的是：S1 對「邏輯形式」不敏感。
            它讀論證時靠「聽起來合不合理」打分，不檢查結構。
            AC 和 DA 在自然語言裡聽起來很合理（因為很多現實關係剛好近似雙向），
            於是 S1 給它「感覺對」、放行。
            S2 本該做的結構檢查（找反例、問「Q 有沒有別的來源」），被偷懶蓋過了。
```

換句話說：偏誤的回扣是「哪個捷思在 S1 跑、原本對在哪、在哪翻車」；形式謬誤的回扣是「**為什麼 S1 會覺得一個無效推論看起來有效**」。前者問的是內容怎麼填錯，後者問的是結構檢查為什麼沒做。這是兩個不同的問題，這一章只回答後者。

我認為這是一個比它表面更深的觀察：你的邏輯能力本身沒問題（你寫 `if` 從不寫反），出問題的是「在讀一段自然語言敘述時，S2 有沒有被叫醒去檢查結構」。多數時候沒有。而下一節的瓦森實驗，就是把這件事擺到桌上、量給你看。

## 瓦森的四張卡片：連純結構檢查，九成的人都做錯

1966 年，瓦森（Peter Wason）設計了一個後來變成認知科學經典的小實驗（瓦森四卡片選擇任務，Wason selection task）。它的厲害之處在於：它**只**考你的形式推理，沒有機率、沒有基率、沒有需要背的知識——純結構。然後它告訴你，抽象版的正解率**低於 10%**（多數整理給出 4%–20% 這個區間，視措辭與樣本而定，這裡採 landscape 的「<10%、約九成答錯」並 hedge 為「依研究在 4%–20% 間」；2026-06 查證）。

題目是這樣。桌上四張卡片，每張卡一面是字母、一面是數字。你看到朝上的四面是：

```text
   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐
   │  E  │   │  K  │   │  4  │   │  7  │
   └─────┘   └─────┘   └─────┘   └─────┘
```

規則是：**「若一張卡的一面是母音，則它的另一面是偶數。」** 用我們的記號，令 P＝「這面是母音」、Q＝「另一面是偶數」，規則就是 `P → Q`。

問題：**為了檢驗這條規則有沒有被違反，你最少必須翻開哪幾張卡？**

絕大多數人翻 **E 和 4**。先別往下看，感受一下你自己的直覺指向哪兩張。

正確答案是 **E 和 7**。我們一張一張對帳，每張卡對應上面那張四推論表裡的一格：

```text
卡片   朝上面    這對應規則的哪一端        翻它能做什麼              該翻?
──────────────────────────────────────────────────────────────────────
 E     母音(P)   前件成立                  背面若是奇數 → 直接違反規則   要翻 ✓
 K     非母音    前件不成立(¬P)            規則對非母音沒說話 → 翻了沒用  不翻
 4     偶數(Q)   後件成立                  背面是母音或子音都不違規      不翻 ✗
 7     奇數(¬Q)  後件不成立(¬Q)            背面若是母音 → 直接違反規則   要翻 ✓
──────────────────────────────────────────────────────────────────────
```

把它對回四推論：

- 翻 **E** ＝ 檢查 **MP** 的前件那一端：母音必須配偶數，背面一翻若是奇數，規則當場破。有效、該做。
- 翻 **7** ＝ 檢查 **MT** 那一端：規則說母音⇒偶數，那麼「不是偶數（奇數）」就**不能**是母音（否則違規）。翻 7 若背面是母音，規則當場破。這正是否定後件 MT 的力量——它是唯一能從「Q 不成立」這端抓到違規的牌。有效、該做。
- 不翻 **4**：4 是偶數（Q 成立）。翻它想幹嘛？想看背面是不是母音？可是規則**沒說**「偶數一定要配母音」。背面是母音不違規，是子音也不違規。翻 4 等於想用 **AC**（看到 Q 就回頭查 P）去驗證規則——而 AC 無效，翻它得不到任何能判違規的資訊。**多數人翻 4，翻的正是肯定後件那張無效的牌。**
- 不翻 **K**：K 不是母音（¬P）。規則對非母音完全沒承諾，背面是什麼都不違規。想翻 K ＝ 想用 **DA**（前件不成立就推後件），一樣無效。

看出那個對稱了嗎？**人們最想翻的 E 和 4，剛好是「能證實規則的兩張」；該翻卻沒人翻的 7，是「能否證規則的那張」。** 瓦森本人把這個現象解讀為**確認偏誤的形式版**：人傾向去翻「能讓規則成立」的牌，而不去翻「能讓規則破掉」的牌。但要嚴守本章與 ch13 的分工——確認偏誤的**動機面與詞源**（瓦森更早的 1960 年 2-4-6 任務）是 ch13 的主述，這裡只把四卡片當作一個冷冰冰的證據：**人連這種沒有任何內容干擾的純結構檢查，都系統性地偏向選能證實的卡、漏掉那張靠 MT 才抓得到的否證卡。**（2-4-6 任務見 ch13。）

這裡有個讓人坐立難安的細節，我認為是本章最值得記住的一頁：**換個脈絡，同一個邏輯結構，正解率會暴漲。** 把規則換成「若一個人在喝啤酒（P），則他必須年滿 18 歲（Q）」，卡片換成「喝啤酒／喝可樂／22 歲／16 歲」，請你當查緝的警察抓違規——這時多數人輕鬆翻對「喝啤酒」和「16 歲」（即 P 與 ¬Q）。Griggs & Cox 1982 的這類具體版把正解率從抽象版的約兩成拉到約 74%（2026-06 查證）。

這件事的反直覺在於：**邏輯結構一模一樣（都是檢查 `P → Q` 要翻 P 和 ¬Q），難度卻天差地別。** 你的形式推理能力不是一個穩定的、隨時可調用的模組；它高度依賴題目用什麼內容包裝、有沒有把問題框成一個你熟悉的「抓違規」社會場景。抽象符號喚不醒 S2 去做結構檢查，「抓偷喝酒的未成年人」就喚得醒。這恰恰印證了本章的雙系統回扣：S1 不檢查結構，而能不能把 S2 叫起來做結構檢查，靠的不是邏輯本身，是包裝。

順帶守一個下一章會展開的陷阱②（訴諸謬誤 / fallacy fallacy）：發現對方用了肯定後件，**不等於**對方的結論錯。它只說明「這個論證沒能證明結論」。結論可能恰好為真，只是你得用別的、有效的方式去支持它。這條留給 ch06 細講，這裡先埋。

## 紙上推演

### 推演題一（★，[10 分鐘]）：把白話翻成結構，標出推論型

下面三段話各是一段日常／工程推理。把每段翻成 `P → Q` 的結構，標出它用的是 MP／MT／AC／DA 哪一種，並判定有效或謬誤。

```text
(a) 「如果這次部署有問題，健康檢查一定會變紅。健康檢查現在是綠的，
     所以這次部署沒問題。」

(b) 「資深工程師寫的程式碼通常品質高。這段程式碼品質很高，
     所以八成是資深工程師寫的。」

(c) 「如果使用者按了登出，session 就會被清掉。這個使用者沒按登出，
     所以他的 session 還在。」
```

#### 推演解答

**(a)** 規則：若 部署有問題（P）→ 健康檢查變紅（Q）。已知：健康檢查是綠的，即 ¬Q。結論：部署沒問題，即 ¬P。形式：`¬Q，∴ ¬P` ＝ **否定後件（MT）＝有效**。

但這裡要加一句工程師的誠實註記：MT 在邏輯上有效，**前提是那條規則真的成立**——「部署有問題就一定會讓健康檢查變紅」。如果健康檢查根本沒覆蓋這次部署改動的那塊（規則其實是假的），那結論再有效也不可信。**有效 ≠ 可信**，這個分界是 ch06 的主題。本題只問形式：形式是有效的 MT。

**(b)** 規則：若 資深工程師寫（P）→ 品質高（Q）。已知：品質高，即 Q。結論：資深工程師寫，即 P。形式：`Q，∴ P` ＝ **肯定後件（AC）＝謬誤**。反例隨手可得：一個用心的新人、或一段被 linter／reviewer 救過的程式碼，品質一樣高，但 P 為假。品質高有別的來源，所以「品質高」不能反推「資深寫的」。

**(c)** 規則：若 按了登出（P）→ session 被清（Q）。已知：沒按登出，即 ¬P。結論：session 還在，即 ¬Q。形式：`¬P，∴ ¬Q` ＝ **否定前件（DA）＝謬誤**。反例：session 過期、被管理員強制踢、伺服器重啟導致 session store 清空——這些都讓 session 不在，但使用者沒按登出。「沒按登出」不蘊含「session 還在」，因為 session 消失有別的途徑。

常見錯路：(a) 很多人會憑「結論聽起來對」就放過去而不標形式，但本題它**剛好**是有效的 MT，重點是你要能說出「為什麼有效」而不是「感覺對」；(c) 最容易被誤判成有效，因為「沒登出所以還在」聽起來理所當然——這正是 DA 最會騙人的地方，它在「P 是 Q 的唯一原因」這個錯誤假設下才成立，而現實裡 session 消失從來不只一條路。

### 推演題二（★★，[15 分鐘]）：debug 現場的肯定後件

回想（或設想）一次你 debug 的經驗，符合這個模式：你看到某個症狀，立刻咬定某個原因，查了半天才發現原因不對、繞了遠路。把它寫成 `P → Q` 結構，指出你在哪一步犯了肯定後件，再寫出一個「用否定後件去否證假設」的替代流程。

#### 推演解答

這題沒有標準答案，但有標準的**骨架**。範例：

```text
症狀：API 延遲飆高（Q 出現了）
你心裡的規則：若 資料庫慢（P）→ API 延遲飆高（Q）       P → Q（這條對）
你的推論：延遲飆高了（Q），所以一定是資料庫慢（P）       Q,∴P ＝ AC，謬誤
你做的事：盯著 DB 的慢查詢 log 找了 40 分鐘，沒找到。

問題：延遲飆高有別的來源——GC 停頓、下游逾時、連線池滿、
      某個剛上線的 N+1。Q 不只 P 一個來源，所以 Q 不能反推 P。

替代流程（用 MT 主動否證每個假設）：
  假設「是 DB 慢」⇒ 若真，DB 的 P99 query time 一定也飆。
  去看 DB P99：平的（¬Q'）。
  ∴ 不是 DB（¬P）。               ← 一個 MT，30 秒排除一個假設
  接著對「GC 停頓」「下游逾時」各做一次同樣的 MT 否證。
```

關鍵差別：肯定後件是「看到結果、往最順的原因撲過去、只找支持它的證據」；否定後件式的 debug 是「對每個候選原因問『如果是它，我**必定**會看到什麼？』，然後去看那個東西**在不在**——不在就果斷排除」。後者把「找證實」換成「找否證」，一個 MT 三十秒砍掉一個假設，比死盯一個 AC 假設快得多。（這條「主動找否證」的思路，動機面會在 ch13 確認偏誤展開；本章只取它的形式版。）

常見錯路：把替代流程寫成「再找一個假設來肯定」——那只是換一張 AC 的牌翻，沒有跳出謬誤。真正的解法是改用 MT 的方向：問「若 X 為真，必有什麼可見後果？沒看到就排除 X」。

### 推演題三（★★，[15 分鐘]）：講清楚瓦森四卡片

不看上面的對帳表，用你自己的話對另一個工程師講清楚：規則「若一面是母音則另一面是偶數」（卡片 E／K／4／7），該翻哪兩張、為什麼，以及為什麼多數人翻錯的那張（4）對應的是哪個無效推論。

#### 推演解答

標準講法：規則是 `P → Q`，P＝母音、Q＝偶數。要檢驗一條 `P → Q` 規則有沒有被違反，你只需要翻**兩種卡**：前件成立的（P）和後件**不**成立的（¬Q）。

- 翻 **E**（P）：母音這面背後必須是偶數，翻開若是奇數，當場抓到違規。
- 翻 **7**（¬Q）：7 是奇數（非偶數）。規則說母音必配偶數，反過來說「非偶數」就不准是母音；翻開若是母音，當場抓到違規。這張靠的是否定後件 MT——「Q 不成立 ⇒ P 不能成立」。
- 不翻 **4**（Q）：偶數背後是母音或子音都不違規（規則沒說偶數一定配母音）。想用「看到偶數就回頭驗母音」＝肯定後件 AC，無效，翻了得不到判違規的資訊。多數人翻 4，翻的就是這張 AC 的牌。
- 不翻 **K**（¬P）：非母音規則沒管，翻了沒用（對應 DA）。

為什麼多數人錯：人傾向去翻「能讓規則成立／看起來相關」的牌（E 和 4），而漏掉那張唯一能**否證**規則、要靠 MT 才抓得到的 7。這是確認偏誤的形式版——但記得，這裡只取它「人連純結構檢查都偏向找證實」的形式面，動機與詞源在 ch13。

常見錯路：(1) 翻 E 和 4（最經典的錯，把 AC 那張當成該翻）；(2) 翻 E、4、7 三張（多翻了沒用的 4，雖然 7 對了）；(3) 把「該翻 7」的理由說成「7 是奇數所以可疑」——理由不夠精確，正確理由是「7＝¬Q，而檢驗 `P→Q` 必須查 ¬Q 那端會不會偷偷配上 P」。能把「為什麼是 ¬Q 那張」講清楚，才算真懂。

## 自我檢核

口頭自答，講不清楚就是還沒懂。優先回答「為什麼」。

1. 為什麼「肯定後件是一個用錯場合的捷思」這句話是錯的？形式謬誤和前四章的偏誤，在機制上差在哪？
2. 給定 `P → Q`，為什麼 MP 和 MT 有效、AC 和 DA 無效？你能不能不靠背、用「`P → Q` 是單行道、Q 可以有別的來源」這一句話把四個都推一遍？
3. 「找一個反例」為什麼足以證明一個推論形式無效？為什麼反例的內容選得越白癡越好？
4. 雙系統透鏡在這一章只回扣一個方向，是哪個方向？為什麼不能說「肯定後件是 S1 跑的一個捷思」、只能說「S1 對邏輯形式不敏感、憑表面合理度放行」？
5. 瓦森四卡片題，該翻哪兩張？多數人翻錯的那張（4）對應哪個無效推論？沒人翻的那張（7）靠哪個有效推論才抓得到違規？
6. 為什麼把規則換成「喝酒要滿 18 歲」、正解率就從約兩成跳到約七成？同一個邏輯結構，難度差別說明了你的形式推理能力是什麼性質的東西？
7. 「他犯了肯定後件，所以他的結論一定錯」——這句話本身為什麼也是個錯誤？（這條接 ch06。）
8. debug 時，你怎麼把一個肯定後件式的歸因（看到症狀就咬定原因）改寫成否定後件式的否證流程？
