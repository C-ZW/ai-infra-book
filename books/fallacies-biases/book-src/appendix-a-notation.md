# 附錄 A — 記號與術語速查

> 這是一張**速查表**，不是教學。每個記號、每條公式都假設你已在正文某章遇過它；這裡只負責讓你在忘記「P→Q 裡那個箭頭到底是條件還是推導、P(A|B) 的條件在哪一邊、MP 和 MT 哪個有效」時，能在十秒內查回來，並標出它**第一次正式登場**的章。要從零理解某個概念，回該章；要核對一個符號的確切意義或一個設定數字，留在這裡。

閱讀器不渲染 LaTeX，全書一律純文字 Unicode 記號（內文行內就是 P→Q、P(A|B)、P(A∩B) ≤ P(A) 這種寫法）。所以這張表也是純文字的——你看到的形狀就是書裡的形狀。

## 一、全書脊椎一眼定位

全書二十章反覆套用的是**同一具雙系統透鏡**，先把它釘在這裡，後面所有記號都繞著它轉。

```text
脊椎：雙系統透鏡（ch01 登場，被重看七層直到 ch20）

  系統一（S1）                       系統二（S2）
  ──────────────────────────────     ──────────────────────────────
  快、自動、平行                       慢、刻意、序列
  省力、永遠開著                       費力、容易累
  給「感覺對」的答案                   能算出「正確」的答案（如果逼它）
  工程橋接：cache 命中、熱路徑         工程橋接：冷路徑、慢查詢、人工覆核

  認知偏誤 = S1 跑了一個捷思           S2 沒攔下來 → 錯置到不該用的脈絡
  形式謬誤 = 邏輯結構本身無效          S1 憑「聽起來合理」就放行 → S2
  （與內容、與捷思無關）               沒做結構檢查
```

這具透鏡的兩個誠實邊界，一定要記住（見 ch01、landscape T1）：
- 「系統一／系統二」是教學**隱喻**，不是腦裡有兩個小人，不對應固定腦區。
- 雙系統只解釋「為何 S1 覺得一個偏誤或無效推論看起來合理」，**不是每一種思考陷阱都是「錯置的捷思」**——形式謬誤錯在邏輯結構、與捷思無關。

## 三、同形記號歧義專節（重要）

本書借用的符號不多，但有幾個在不同脈絡常指不同東西，或與書架上鄰書的用法衝突。查一個符號前先確認你查的是哪個用法，否則會把對的意義填到錯的格子裡。

```text
→（箭頭）—— 本書有兩個常見用法，靠上下文區分：               ch05
   用法一：條件命題「若…則」：P → Q，表示「若 P 為真則 Q 為真」；
           這是邏輯形式，和 P、Q 的內容無關。
   用法二：推導或「導致」：寫在論述裡的「A → B」表示「A 導致 B」
           或「A 之後推導出 B」。
   判別法：放在「P → Q」這個命題形式裡的 → ＝條件命題；
           放在論述句子裡描述因果或推論步驟的 → ＝推導箭頭。

P（大寫）—— 本書有三個身分：                                  ch05/ch10
   ① 命題邏輯的命題變數（P、Q、R）：ch05–ch08 的邏輯章裡。
   ② 機率函數 P(·)：ch10–ch12 的機率章裡，P(A) 表示 A 的機率。
   ③ 縮寫 MP（modus ponens 肯定前件）裡的 M。
   判別法：跟 → ∧ ∨ ¬ 在一起的大寫字母是命題變數；
           跟括號 P(·) 在一起的是機率函數；MP/MT/AC/DA 是整體縮寫。

λ（lambda）—— 本書：損失趨避係數，約 2（精確值依研究而異，     ch14
   見 landscape §4.2）。本書只取直覺，數學指向《在不確定中下
   注》。易撞：在《馴服隨機》等書，λ 可能是速率參數（指數分布
   的 λ）、特徵值等，意義完全不同。看到 λ 先問：這是「損失趨避
   係數」還是別本書的某個參數？

P(A|B) 與 P(B|A) —— 本書的核心概念之一，兩者絕對不等。       ch10
   P(A|B)：已知 B 發生後，A 的條件機率（貝氏後驗的形狀）。
   P(B|A)：已知 A 發生後，B 的條件機率（似然度的形狀）。
   混淆這兩個是「檢察官謬誤」（ch10）：把「P(DNA 匹配|無罪)
   極低」誤讀成「P(無罪|DNA 匹配)」——兩個數字可以差幾個數量
   級。在告警偵測的工程脈絡：P(告警響|系統正常) 就是偽陽率；
   P(系統正常|告警響) 才是你真正想知道的，兩者差很遠。
```

## 四、四種條件推論速查

這張表是 ch05 的壓艙石，也是全書邏輯部分最不能混淆的四行。給定前提 P → Q，四種推論的有效性如下：

```text
四種條件推論速查（前提一律：P → Q）
─────────────────────────────────────────────────────────────────────────────
名稱         縮寫   已知     結論    有效?   為何
─────────────────────────────────────────────────────────────────────────────
肯定前件     MP     P        ∴ Q    有效    P → Q 成立且 P 成立，Q 必然真
modus ponens
─────────────────────────────────────────────────────────────────────────────
否定後件     MT     ¬Q       ∴ ¬P   有效    若 Q 為假，則 P 必假（對位律）
modus tollens
─────────────────────────────────────────────────────────────────────────────
肯定後件     AC     Q        ∴ P    無效    Q 真不保證 P 真（P → Q 成立不
affirming              （謬誤）           蘊含 Q → P 成立；反例：所有
consequent                              貓都是動物，看到動物不代表是貓）
─────────────────────────────────────────────────────────────────────────────
否定前件     DA     ¬P       ∴ ¬Q   無效    P 假不保證 Q 假（P 不是 Q 的
denying              （謬誤）           唯一充分條件；反例：若下雨則
antecedent                             路濕，沒下雨路不一定不濕）
─────────────────────────────────────────────────────────────────────────────
```

工程防呆記法：MP 和 MT 是「一頭一尾」（前件/後件各取一個）；AC 和 DA 是「各取反方向的」——AC 拿 Q 推 P、DA 拿 ¬P 推 ¬Q，兩個都是謬誤。在 debug 脈絡：「症狀 B 出現就斷定是原因 A」＝肯定後件（AC，謬誤）；「原因 A 沒發生就說症狀 B 不會出現」＝否定前件（DA，謬誤）。

## 五、機率與統計記號速查

```text
記號              意義                                        首次章
─────────────────────────────────────────────────────────────────────────────
P(A)              事件 A 的機率，0 ≤ P(A) ≤ 1               ch10
P(A|B)            條件機率：B 已發生下 A 的機率               ch10
P(A∩B)            合取機率：A 與 B 同時發生                   ch11
P(A∩B) ≤ P(A)    合取律（恆真）：子集不可能比母集更可能      ch11
P(H|D)            後驗：見到資料 D 後，假設 H 為真的機率      ch10
P(D|H)            似然：假設 H 成立時，觀測到資料 D 的機率    ch10
d                 效果量（effect size）：衡量偏誤效應的強度   ch17
[95% CI]          95% 信賴區間                                ch17
```

貝氏定理（**只引用、不在本書推導**，機制指向《馴服隨機》）：

```text
P(H|D) = P(D|H) × P(H) / P(D)

  P(H)   = 先驗：看到資料前，H 為真的基率
  P(D|H) = 似然：H 成立時，看到資料 D 的機率
  P(D)   = 邊際：見到資料 D 的總機率（正規化常數）
  P(H|D) = 後驗：你真正想知道的——見到 D 後 H 為真的機率

完整推導見《馴服隨機》；本書只用結果與直覺。
```

## 六、同形記號歧義補充：捷思雙面性判讀法

本書有一張判讀法貫穿全書，用來統一回答「這個捷思哪裡對、哪裡翻車」。附錄 A 把它釘在這裡，各章提到時直接回查：

```text
「捷思雙面性」判讀法（ch02 登場，全書套用）

  捷思名稱 │ 它原本對在哪個脈絡            │ 它在哪個脈絡翻車
  ─────────┼───────────────────────────────┼──────────────────────────────
  可得性   │ 常見的確實容易想起來，        │ 生動、近期、情緒化的被高估；
  捷思     │ 頻率代理成立（ch03）          │ 媒體放大的被高估（ch03）
  ─────────┼───────────────────────────────┼──────────────────────────────
  代表性   │ 「像不像原型」常是有用的分類  │ 「像不像」被錯當「多可能」；
  捷思     │ 線索，節省認知資源（ch04）    │ 忽略基率與樣本大小（ch04/10/11）
  ─────────┼───────────────────────────────┼──────────────────────────────
  錨定     │ 第一個出現的數字常有參考價值，│ 無意義的錨（如亂轉的輪盤）也拖
           │ 減少搜尋空間（ch04）          │ 動估計，無充分調整（ch04）
  ─────────┼───────────────────────────────┼──────────────────────────────
  徵兆：答案來得太快太順、沒感覺在算 → 可能正走在捷思的熱路徑上
```
