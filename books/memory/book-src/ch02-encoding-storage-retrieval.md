## 從你已知的出發

你每天都在處理一條資料的生命週期：寫進去、存著、讀出來。你不會把這三件事混為一談，因為你知道它們各自會用不同的方式壞。

- 一筆訂單**沒寫進資料庫**（請求在邊界就被丟掉、欄位 validation 失敗、訊息根本沒進 queue），跟
- 一筆訂單**寫進去了但之後被覆蓋／衰減**（被另一筆 update 蓋掉、TTL 到期被 evict、磁碟壞軌），跟
- 一筆訂單**還在那裡但你讀不出來**（index 沒命中、query 條件寫錯、cache 回了 stale 值）

——這是三種**不同的故障**，要用三種不同的方法 debug。你絕不會看到「讀不到訂單」就直接斷定「資料沒寫進去」，因為那會讓你去修錯的地方。

記憶系統完全是同一回事，只是大多數人——包括工程師——在談自己的記憶時，把這三關**全部混成一句「我忘了」**。「我忘了他那天穿什麼」可能是：當時你**根本沒編碼**那個細節（注意力不在那）、編碼了但**痕跡衰減／被干擾**蓋過、或痕跡還在但**這次提取勾不出來**（換個線索就想起來了）。三種狀況，機制不同、能不能救不同、會不會被「補一個假的」也不同。

這章要做的就是把你 debug 資料管線的那套紀律，搬到記憶上：先把管線畫清楚，知道每一關**各自**怎麼壞，你才不會把「沒寫進去」誤診成「讀不出來」，也才不會錯過本章最重要的一件事——記憶這條管線的**讀取（提取）這一步本身會回寫**。在你的資料庫裡，一次 `SELECT` 不會改掉資料；在記憶裡，一次認真的回憶會。這條我們留到本章後半，先把三關卡立起來。

橋接錨點先講明，後面會用滿：

- **編碼＝有損壓縮（lossy 編碼）**：經驗進來時不是無損存檔，是像 JPEG 丟掉高頻細節、保留輪廓。
- **提取＝讀取時重算（read-through，不是讀 immutable blob）**：你不是把一段完整影片讀回來，是拿輪廓**現場重建**細節。
- **提取的回寫＝read-repair**：這次重建出來的版本，會被寫回去當成「原始記憶」——這是脊椎，本章兌現一半（現象層），分子層留到 ch09。

## 三道關卡：把記憶拆成一條管線

先給最素樸的版本。一段記憶從發生到被你想起來，至少要過三關：

```text
       ┌──────────┐      ┌──────────┐      ┌──────────────────┐
經驗 → │ 編碼     │  →   │ 儲存     │  →   │ 提取             │ → 「回憶」
       │ encoding │      │ storage  │      │ retrieval        │
       │ 寫進去   │      │ 存著     │      │ 讀出＋重建＋回寫 │
       └──────────┘      └──────────┘      └──────────────────┘
         有損壓縮          衰減／干擾          現場重算，非讀檔
```

注意第三個框比前兩個寬——這不是排版意外，是本章的論點：提取**不只是讀**。我們一關一關走。

### 第一關 編碼：在入口就已經是有損的

編碼是「經驗變成記憶痕跡」的那一步。關鍵事實，反直覺但站得很穩：**編碼從一開始就是有損的，而且丟什麼、留什麼，由你當時的注意力和既有理解決定，不是由「事件本身有多重要」決定。**

你不是錄下整個場景的每個畫素。你的感官頻寬遠大於你能編碼進長期記憶的量，中間有一個巨大的瓶頸：注意力。沒被注意到的，根本沒進入編碼——這不是「存了一個模糊版」，是**根本沒寫**。經典的演示是「你描述得出一枚硬幣的正面圖案嗎」——你看過它幾千次，但從沒**為了記住**而注意過它，所以從沒編碼進去。讀不出來，因為從來沒寫進去。

更重要的是**留下來的是什麼**。這裡要引一個全書會反覆用的區分（ch07 會再用一次）：**要旨（gist）vs 逐字（verbatim）**。

- **逐字痕跡（verbatim）**：表層、精確的細節——他**確切**說了哪幾個字、襯衫**確切**是哪個藍。
- **要旨痕跡（gist）**：語意、要點、評價——他**反對**方案 B、語氣**不耐**。

這種雙重痕跡的編碼機制有個正式的理論框架，叫**模糊痕跡理論（Fuzzy-Trace Theory，Brainerd & Reyna）**：記憶不是單一表徵，而是同時編碼了逐字與要旨兩種痕跡（2026-06）。它最重要的一個預測是：**逐字痕跡衰減得比要旨痕跡快得多**——過了一兩天，逐字（他到底說了哪幾個字）多半就沒了，但要旨（他反對、他不耐）還在，而且還能被進一步詮釋。

這條預測直接解釋了你的日常經驗：你記得「他很生氣的**大意**」卻記不得「他說的**逐字**」。而這正是麻煩的開端——當提取要求你給出逐字（「他**確切**怎麼說的？」），而逐字痕跡早就衰減了，你的大腦不會空手而回，它會**用還在的要旨去現編一句逐字**。模糊痕跡理論的另一半預測就在這裡：**要旨會滋生與「意義」一致的假記憶**（ch07 的 DRM 就是這個機制的純化版）。

把這條接回工程橋：編碼就像 JPEG 壓縮。高頻細節（逐字）被丟掉，輪廓（要旨）被保留。你之後「看」到的細節，是解碼器**根據輪廓現算的**，不是原圖裡的畫素。差別只在於——JPEG 解碼器至少是確定性的、每次解出來一樣；你的記憶解碼器每次重建會受當下的線索、情緒、被問的方式影響，每次可能不一樣（ch05、ch10 會看到這有多嚴重）。

> 這裡先按一下脊椎：**編碼丟掉的細節，後面是「讀出來」的，還是「重建出來的」？** 答案是後者。但這只是第一層。真正的轉折在第三關。

### 第二關 儲存：痕跡會衰減，也會被別的記憶蓋過

假設一段記憶成功編碼了。它在儲存階段也不是高枕無憂。儲存階段的失真有兩種主要型態：

**衰減（decay）**：痕跡隨時間變弱。這是 ch03 會專門處理的「遺忘曲線」的領域，這裡只立一句——遺忘曲線量的是**保留量隨時間衰減**，它是「資料變淡」這條相對好懂的線。

**干擾（interference）**：這條更隱蔽，也更接近你的工程直覺。記憶失真的一大來源不是痕跡「消失」，而是**被別的記憶蓋過或擠掉**。兩個方向：

- **逆向干擾（retroactive interference）**：**新**記憶干擾**舊**記憶。你換了新車牌，過一陣子想不起舊車牌——新的把舊的蓋掉了。像新的寫入覆蓋了同一個 key 的舊值。
- **順向干擾（proactive interference）**：**舊**記憶干擾**新**記憶。你習慣了舊密碼，改了新密碼後老是打成舊的——舊的卡在 cache 裡，妨礙你命中新的。

干擾的工程鏡像很直接：cache key 碰撞、LRU 把不常用的舊項目擠出去、相似的 key 互相污染查詢結果。重點是——**干擾下的記憶往往「還在」，只是取不到或被蓋過了**。這帶出一個關鍵區分，下一關會兌現。

### 第三關 提取：讀出 ＋ 重建 ＋（這是重點）回寫

提取是把儲存的東西「拿出來」變成一段你意識得到的回憶。三關卡裡，這一關最被誤解，也是本章存在的理由。

**第一件事：提取不是讀回一段完整影片，是用要旨現場重建。** 你儲存的是稀疏的要旨＋幾個鉤子（gist + a few hooks），不是一段連續畫面。提取時，大腦拿這些稀疏的東西，**展開**成一段「完整、連貫、有細節」的回憶。那些細節——他的確切表情、那句話的確切措辭、牆上的海報——很多是**這次提取當下生成的**，不是儲存裡讀出來的。而你**主觀上完全分不出來**哪些是讀出來的、哪些是現編的，因為兩者在意識裡長得一模一樣（為什麼分不出來，是 ch13「來源監控」的主題，這裡先埋）。

**第二件事，也是全書最關鍵的一句：提取不是唯讀的。** 在你的資料庫裡，一次 `SELECT` 不改任何東西——讀取是冪等的、無副作用的。記憶不是。一次**認真的、生動的回憶**，會把那段記憶帶回一個**不穩定（labile）狀態**，然後重新存回去——而存回去的，是**這次重建出來的版本**，包含這次現編的細節、這次被問話帶進來的東西。

這就是脊椎的工程鏡像：**read 觸發 write**。它精確對應你熟悉的 **read-repair**——讀取時順手把「讀到的版本」寫回 source of truth。差別（這個差別很重要，ch09 會展開）在於：read-repair 是**每次讀都觸發、機械且可靠**；記憶的回寫**有邊界條件**，不是每次提取都觸發（記憶太舊、太強、提取方式不對就可能不進入不穩定狀態）。所以類比是**腳手架**，不是證明——以實證為準，類比破在哪要明說。

這一關的分子機制（提取 → labile → 蛋白質合成窗口 → 重新穩定）是 ch09 的事，本章不展開分子鏈。本章只要你接受這個**現象層**的主張：**提取會改寫記憶**。為什麼現在就要你接受？因為它把脊椎從「一個漂亮的隱喻」變成了「一個可檢驗的主張」——如果提取真的會回寫，那我們應該能用「問話的措辭」去污染記憶（ch05 證明了，能）、能在反覆回憶後讓記憶系統性地漂移（ch10、ch12 證明了，會）。整本書後半都在兌現這一關。

## 三關卡各對應一類錯誤：別把它們混為一談

把三關卡立起來之後，最有實戰價值的一件事是：**「我忘了」其實是三種完全不同的故障，混在一起講就會誤診。** 這正是你 debug 資料管線時最在意的紀律——先定位是哪一層壞了，再決定怎麼修。

| 哪一關出錯 | 記憶現象 | 工程鏡像 | 能不能救／救法 |
|---|---|---|---|
| **編碼失敗** | 從沒注意到，根本沒進去（硬幣圖案） | 請求在邊界就被丟掉、根本沒寫入 | **救不了**——資料從未存在，再怎麼提示也沒用 |
| **儲存衰減** | 痕跡隨時間變弱（ch03） | TTL 到期、項目被 evict | 難救——東西真的淡掉了 |
| **儲存被干擾** | 還在，但被別的記憶蓋過 | LRU 擠掉、key 碰撞 | 部分可救——換條路徑可能繞過干擾 |
| **提取失敗** | 痕跡還在，但這次勾不出來（話到嘴邊） | index 沒命中、query 條件不對 | **常可救**——換個線索就出來了 |

最後一列藏著記憶研究裡一個很乾淨的區分，值得你停一下：**「可得（available）」≠「可取（accessible）」**。一段記憶可能**還在儲存裡（available）**，但此刻**取不到（not accessible）**——因為你手上的提取線索不對。圖爾文（Endel Tulving）的**編碼特定性原則（encoding specificity principle，Tulving & Thomson 1973）**講的就是這件事：一個提取線索要有用，它得**在編碼當時也在場**；提取時的線索愈能對上編碼時的脈絡，愈勾得出來（2026-06）。所以「換個房間就想起來」「回到當時的心情就想起來」不是玄學——是線索匹配。

這個區分為什麼對全書重要？因為它把「遺忘」和「重建錯誤」徹底分開（ch03 會把這把刀磨利）：

- **提取失敗**是「資料還在但取不到」——這是**遺忘**的一種，東西沒變、只是勾不出來。
- 而本書真正反直覺的東西，是另一回事：**提取勾出來了，但勾出來的是個被改寫的版本**——這是**重建錯誤**，東西**變了**。

科普最常把這兩件事混為一談：「記憶會出錯」聽起來像一句話，其實是兩種機制（取不到 vs 被改寫），後果天差地別。一個是你**沒**想起來，一個是你**很有信心地想起了一個假的**。後者才是這本書要起雞皮疙瘩的地方。

順帶把工作記憶（working memory）和長期記憶的分界用一句話劃清，不展開：上面三關卡講的是**長期記憶**；工作記憶是你此刻意識裡握著的那一小撮、秒級就會掉的東西（像幾秒鐘的 buffer），容量小、不持久，它的容量模型不是本書主題（見 ch08 對多重記憶系統的處理）。

## Worked example：一段「同事昨天在會議上的發言」，逐關拆給你看

抽象講完了，走一條完整的鏈。這是本章的 worked example——一條走到底、不跳步的機制鏈。

場景：昨天的會議，同事阿凱針對方案 B 講了一段話。今天有人問你：「阿凱昨天到底怎麼說的？」你開口複述。我們逐關標出，你「複述」出來的每一塊，是哪一關的產物。

**編碼當時（昨天會議上）你實際存進去的，大概只有：**

```text
存進去的（稀疏、有損）            沒存進去的（注意力不在／衰減快）
─────────────────────────       ─────────────────────────────────
gist：阿凱反對方案 B            verbatim：他「確切」用了哪幾個字
gist：語氣不耐、有點衝          他講了幾分鐘、語速多快
鉤子：他敲了一下桌子（顯眼）    他當時的確切坐姿、表情細節
鉤子：「成本」這個詞出現過      其他人當下的反應
```

注意：你存的是**要旨＋兩三個顯眼的鉤子**，逐字幾乎沒存。這完全符合模糊痕跡理論——逐字痕跡衰減快，過一天多半就剩要旨了（2026-06）。

**今天提取時，大腦做了什麼：** 它拿這幾塊稀疏的料，**重建**出一段「完整」的回憶。它必須生成你被要求的逐字句子和細節，而那些東西儲存裡沒有，於是它從兩個地方補：

```text
重建出來的「回憶」               這一塊的來源
──────────────────────────     ──────────────────────────────
「阿凱說方案 B 成本太高、         ← gist（反對＋成本）＋你對阿凱的
 一定會超支，他受不了了」            schema（「他一向龜毛、愛抓成本」）
                                   現編出的一句逐字（儲存裡沒有）
「他講的時候皺著眉、很激動」       ← gist（不耐）＋鉤子（敲桌子）放大成
                                   的表情細節（多半是現編的）
「他還說了句『這根本不可行』」     ← 很可能是你的 schema 補的，阿凱
                                   未必說過這句（最危險的一塊）
```

逐步算帳：你「複述」的這段話裡，**只有兩三塊是儲存的**（反對、成本、不耐、敲桌子），**其餘的逐字句子和表情細節，是這次提取當下生成的**——由要旨＋你對阿凱的 schema 拼出來。而你主觀上覺得這整段都是「我記得的」，分不出哪塊是讀的、哪塊是編的。

**最後一刀（脊椎在此兌現）：** 你今天這次複述，不是無副作用的讀取。你剛剛**很有信心地、出聲地**重建了一個版本——其中包含「這根本不可行」這句阿凱可能沒說過的話。下次再有人問，你會更傾向複述出**這個現編版本**，而且更有信心，因為它剛被你提取、回寫、強化了一次。**這次 read，觸發了一次 write。** 你對「阿凱昨天怎麼說」的記憶，剛剛被你自己這次回憶**改寫**了一點點——而你毫無察覺。

把這條鏈和你的工程直覺對齊：這不是讀一個 immutable blob。這是一次 **read-through cache miss 後的重算**（用稀疏的料重建完整結果），而且**重算結果被寫回了 source of truth**（read-repair），下次讀到的是被改過的版本。最終一致性下的 source-of-truth 漂移——只是這裡漂移的是你的過去，而且沒有版本戳能讓你發現它漂了。

## 直覺的陷阱

記憶這個主題迷思密度極高。本章對應的三關卡層面，最常見的錯誤直覺如下。每條標出**錯在哪一關**、會把你帶溝裡的那一步、以及怎麼自我察覺。

| 常見直覺 | 錯在哪 | 正解 | 怎麼自我察覺 |
|---|---|---|---|
| 「**我忘了**＝資料還在只是想不起來」 | 把三種故障混成一種 | 「忘了」至少是三件事：**沒編碼**（從沒進去）／**衰減或被干擾**（淡了或被蓋）／**提取失敗**（在但勾不到）。三者機制與後果都不同 | 問自己：當時我**有沒有注意**到這件事？沒注意＝編碼從沒發生，再多提示也救不回來 |
| 「重要的事一定會**被好好記下來**」 | 編碼關 | 編碼由**注意力**決定，不由「事件重要性」決定。你沒注意的，再重要也沒進去（硬幣圖案、你天天用的東西） | 你描述得出每天用的某個熟悉物件的細節嗎？描述不出＝看過千次但從未編碼 |
| 「**提取就是讀取**，讀一下不會改變記憶」 | 提取關（**本書核心**） | 提取**不是唯讀**——認真回憶會把記憶帶回不穩定狀態、回寫一個重建版本。read 觸發 write | 你有沒有一段「講了很多遍」的往事？講愈多遍愈確信、細節愈豐富——那豐富的細節很可能是歷次重講**現編並回寫**的（ch10、ch09） |
| 「我記得的**細節**，就是當時**存進去**的細節」 | 編碼＋提取關 | 多數細節是提取時用**要旨＋schema 現編**的（逐字痕跡早衰減了）。鮮明的細節 ≠ 真實存過的細節 | 一段記憶愈是「細節鮮明、歷歷在目」，愈值得懷疑——流暢的細節是**重建得很順**的副產物，不是高保真原檔的證據（ch07、ch10） |
| 「**話到嘴邊（tip-of-the-tongue）**＝這個記憶不見了」 | 提取關 | 話到嘴邊恰恰證明記憶**還在**：你能取到它的語意、甚至首字母、音節數，只是**取不到完整發音**——是純粹的提取失敗，不是儲存丟失 | 想不起一個名字但「就在嘴邊」、能說出它的開頭音——這就是「可得但暫時不可取」的活教材 |
| 「記憶是一個**統一的東西**，要嘛記得要嘛忘了」 | 跨關 | 記憶是**多階段、多痕跡**的：你可以記得要旨忘了逐字、記得有這件事但搞錯細節、能力學會了卻不記得學過（ch08 HM） | 你能不能「知道有這回事但完全想不起內容」？這種半記得狀態本身就證明記憶不是一個開關 |

**話到嘴邊（tip-of-the-tongue, TOT）值得多停一句**，因為它是「提取失敗 ≠ 儲存丟失」最乾淨的證據。研究的主流看法：TOT 狀態下，你成功取到了詞的**語意與詞彙層（lemma）**訊息——你知道它的意思、知道它幾個音節、甚至知道它開頭那個音——失敗的只是 **lemma 到語音（phonology）的映射**（2026-06）。換成工程話：**你的 index 命中了、metadata 都查到了，只有指向實際 blob 的那個 pointer 解不開。** 資料明明在，就是這一跳取不到。這正是「資料還在但取不到」的純化案例——也是為什麼「忘了」絕不能一律當成「資料不見了」。

## 紙上推演

每題標難度與預估時間。先自己想，再看解答。

### 推演 1（★★，[15 分鐘]）給「我清楚記得那天他穿藍襯衫」設計一個能分辨的假想實驗

你的朋友堅稱「我清楚記得面試那天主管穿藍襯衫」。問題：這個「藍襯衫」細節，是**面試當下編碼進去的**，還是**後來提取時補上的**？設計一個假想實驗，把這兩種可能分開。提示：你能不能在事件之後，悄悄給一半的人「接觸到藍襯衫」這個資訊，看它會不會被當成「原始記憶」？

#### 推演解答

這題其實是在讓你獨立發明出**誤導訊息效應的實驗設計**（ch05 會看到 Loftus 真的這樣做了）。一個乾淨的設計：

1. 讓所有受試者看同一段面試影片，影片裡主管穿的是**灰**襯衫（事實鎖定）。
2. 隨機分兩組。事後（比如一週後），給**實驗組**讀一段「摘要」，裡面夾帶一句「主管穿著藍襯衫走進來」；**控制組**讀一段不提襯衫顏色的摘要。
3. 再過一段時間，問所有人：「主管那天穿什麼顏色的襯衫？」

**邏輯**：影片裡是灰的，所以任何人答「藍」都不可能是「編碼進去的真實細節」——影片裡根本沒有藍。如果實驗組答「藍」的比例**顯著高於**控制組，那這個「藍」就是**事後資訊被提取階段寫進記憶的**，而非編碼當時存的。差幅（實驗組 − 控制組）就是「事後措辭污染記憶」的量。

**為什麼這樣設計才乾淨**：關鍵是用一個**已知為假**的細節（影片是灰的）。如果你用真實顏色去問，你永遠分不出受試者是「真的記得」還是「猜對了」。用一個影片裡不存在的細節，任何「記得」它的人都暴露了重建的痕跡。

**常見錯路**：直接問「你記得是藍的還是後來才知道的？」——沒用。因為受試者**主觀上分不出來**（這是 ch13 來源監控的主題）。你不能靠問受試者來源，得靠**操弄**＋**對照組**。

接回脊椎：實驗組那個「藍」，是 read（事後讀摘要）觸發的一次 write（寫進了「面試記憶」）。控制組沒有這個 write，所以乾淨。

### 推演 2（★，[10 分鐘]）把「話到嘴邊」歸到三關卡的哪一關，並說明理由

你想不起某部電影男主角的名字，但「就在嘴邊」——你知道他演過哪幾部、知道名字是兩個字、甚至覺得開頭是「ㄎ」的音。請問：這是編碼、儲存、還是提取出了問題？用一句話對另一個工程師說清楚。

#### 推演解答

**提取失敗**——而且是「儲存完好、提取失敗」最乾淨的例子。

理由鏈：

- **不是編碼失敗**——你顯然編碼過這個名字（你認得他、知道他的作品），資料進去過。
- **不是儲存丟失**——你還能取到一大堆**關於它**的東西（語意：他的作品；部分形式：兩個字、開頭的音）。如果痕跡真的衰減消失了，你連這些都取不到。
- **是提取失敗**——你取到了語意層和部分形式層（lemma），只差**完整發音那一跳**（lemma → phonology 的映射沒解開）（2026-06）。

對工程師的一句話：**「就像 index 命中了、metadata 全查到了，只有指向實際 blob 的那個 pointer 解不開——資料在，就這一跳取不到。」**

延伸自我察覺：這正是為什麼換個線索（看到他另一部片、有人提到第一個字）常常「啪」一下就解開了——你補上的那個額外線索，剛好接通了缺的那一跳（呼應編碼特定性：對的線索才勾得出）。

### 推演 3（★★★，[20 分鐘]）口頭題——為什麼說「提取不是唯讀」是整本書的轉軸

不看書，對一個只聽過「記憶會變淡」的工程師解釋：為什麼「提取不是唯讀的」這一句，是整本書最關鍵的轉折？講到他能複述。

#### 推演解答

要點清單（講出來、不是背）：

1. **先給對方的預設**：他現在腦裡的模型是「記憶像存檔——可能會變淡（衰減），但讀它不會改它」。這個模型裡，讀取是安全的、無副作用的，就像 `SELECT`。

2. **戳破它**：記憶的提取**不是 `SELECT`，是 read-repair**。一次認真的回憶會把那段記憶帶回**不穩定狀態**，然後把**這次重建出來的版本**（含這次現編、這次被問話帶進來的細節）**回寫**回去。read 觸發了 write。

3. **為什麼這是轉軸（最重要）**：如果提取**只是**讀，那記憶最多就是「會變淡的存檔」——不準是因為**丟了東西**。但如果提取會**回寫**，那記憶會**長出原本沒有的東西**、會被「怎麼問」這件事**主動改寫**、會在反覆回憶中**系統性漂移而你毫無察覺**。前者是「資料變少」，後者是「資料變了、還變得你更有信心」。**整本書後半（誤導、植入、閃光燈、法庭）全都建立在「提取會回寫」這一句上**——沒有這一句，那些現象無法解釋。

4. **守防線**（別講過頭）：「會回寫」不等於「記憶全不可信」。回寫**有邊界條件**、不是每次提取都觸發；多數日常記憶大致準確。這一句說的是「可被污染、鮮明 ≠ 正確」，不是「一切都是編造」。

5. **收尾的一問**（讓對方複述）：「所以，當你下次很有信心地回憶一件往事——你是在**讀出**它，還是在**讀的同時改寫**它？」他若能把這一問用自己的話再說一遍，就過了。

## 自我檢核

口頭自答，講得出來才算過。優先答「為什麼」。

1. 「我忘了」至少是哪三種不同的故障？為什麼把它們混為一談會讓你「修錯地方」？（對照你 debug 資料管線時怎麼先定位故障層）
2. 為什麼說編碼是「有損的」？決定「留什麼、丟什麼」的是事件的重要性，還是別的東西？
3. 要旨（gist）和逐字（verbatim）哪個衰減得快？這個差別怎麼解釋「你記得他生氣的大意、卻記不得他的原話、而且會現編一句原話」？
4. 「可得（available）」和「可取（accessible）」差在哪？話到嘴邊（tip-of-the-tongue）證明的是這兩者中的哪一種狀態？
5. 提取失敗（資料還在但取不到）和重建錯誤（取出來但被改寫了），是不是同一回事？後果差在哪？
6. **為什麼說「提取不是唯讀」是這整本書的轉軸？** 如果提取真的只是唯讀，本書後半的哪些現象就無法成立？
7. 把本章 worked example（阿凱的發言）講一遍：你「複述」出來的內容，哪幾塊是儲存的、哪幾塊是這次提取現編的？這次複述對你的記憶做了什麼？
8. 脊椎一問：本章哪一個具體機制，把「每次 read 觸發一次 write」從隱喻變成了可檢驗的主張？（提示：它在 ch09 兌現分子層）
