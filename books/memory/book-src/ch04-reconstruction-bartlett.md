# ch04 — 記憶是重建：Bartlett、schema 與你腦補的空白

## 從你已知的出發

你維護過用 **event sourcing（事件溯源）** 的系統：狀態不是存一張快照原樣讀回，而是存一連串事件，要用的時候**重播（replay）**事件序列、把當前狀態重新算出來。這個架構你很熟，也知道它的甜頭——任何歷史狀態都能重建、稽核日誌天生齊全。

現在把這套架構往壞處想一遍——這正是「焦慮工程師」的功課。重播要正確，前提是**事件流完整**。如果中間有幾個事件遺失了（網路封包遺失、上游沒寫進去、保留期過了被清掉），重播會怎樣？理想情況是直接報錯停下。但很多真實系統不會——它們會**用預設值補上缺失的欄位**，讓重播「跑完」，給你一個看起來完整、其實有一段是 default 填出來的狀態。更糟的是：跑完之後，你手上那個物件**分不出哪些欄位是真事件算出來的、哪些是 default 補的**——它們長得一模一樣，沒有一個 flag 標著「這格是猜的」。

人腦的提取就是這個壞掉的 event sourcing。經驗進來時，編碼階段做的是**有損壓縮**（見 ch02）——保留要旨（gist）、丟掉大量細節，等於只留下稀疏的事件流。等你回憶時，大腦重播這串殘缺的事件，缺的細節用一個模板補上——這個模板就是本章的主角：**基模（schema）**。基模是大腦對「一類情境通常長怎樣」的預設值。提取出來的那段「完整」回憶，一部分是真事件重播的，一部分是基模 default 填的，而你**主觀上分不出來**——就跟那個跑完的狀態物件一樣，沒有欄位標著「這格是腦補的」。

這就是巴特利特（Frederic Bartlett, 1886–1969）在 1932 年想講的事，只是他沒有 event sourcing 這個詞。本章要做兩件事：一是把這個機制講到你能用自己的話對另一個工程師複述；二是誠實處理一個讓工程師最不舒服的科學狀態——**這個結論是對的，但它最早的那批「壓測」做得很爛**。

> **這裡先把脊椎插一根樁**：基模補洞，就是「提取不是唯讀」的一半理由。當你回憶並把腦補的細節說出口、甚至寫下來，這段被基模污染的版本還會回寫進記憶（再鞏固，見 ch09），下次提取時它就是新的「原始檔」。本章談的是補洞這個 **write 的內容從哪來**；補完之後那個 write 怎麼被存回去，是後面的事。

## 巴特利特在解什麼問題：把「記憶＝倉庫」換成「記憶＝重建」

要看懂巴特利特為什麼是個突破，先看他反對的是什麼。

巴特利特之前，記憶研究的主流形象是艾賓浩斯（見 ch03）那種——把記憶當**倉庫**：你存進去一批無意義音節，過一陣子量它衰減掉多少。這個框架預設了一件事：**記憶是被動儲存的東西，研究記憶＝研究它怎麼隨時間掉量**。在這個框架裡，「錯誤的回憶」就是「掉了一部分量」，如同硬碟壞軌——資料缺一塊。

巴特利特覺得這個框架漏掉了真實記憶最有趣的部分。真實生活裡，人記的不是無意義音節，是**有意義的故事、場景、人**；而人記錯的方式也不是「缺一塊」那麼單純——人會**主動把記憶改成另一個樣子**，改成一個更合理、更熟悉、更符合自己預期的版本。這不是倉庫掉量能解釋的，這需要一個「會主動加工」的機制。

他的解法是把研究材料換掉。他不給受試者無意義音節，給他們一則**陌生的故事**——一則美洲原住民的民間故事《鬼魂的戰爭》（The War of the Ghosts）。這則故事對 1932 年的英國劍橋受試者來說充滿異質元素：獵海豹、出征打仗時同行的夥伴其實是鬼魂、人中箭卻不覺得受傷、最後黎明時一個黑色的東西從垂死者口中冒出來。故事不照英國人習慣的因果邏輯走，很多情節在英國文化裡「沒道理」。

選這則故事是整個設計的關鍵，值得你停下來想為什麼。**如果記憶是被動倉庫，那受試者忘掉的應該是隨機的細節——這裡缺一句、那裡漏一個名字。但如果記憶是主動重建，那受試者「記錯」的方向應該有系統——他們會把陌生的東西改成熟悉的東西。** 換句話說，這則故事是一個能把兩種理論分開的判別器：扭曲的**方向**會告訴你是哪一種機制在運作。

結果是後者。受試者反覆回憶這則故事時，回憶不是隨機掉量，而是**有方向地往「英國人覺得合理的版本」漂移**。巴特利特把這種漂移歸納成幾種型態，我們下一節逐一拆。

> **嚴謹度標記**：「記憶是受基模驅動的重建」這個**核心洞見**——✅ 是當代記憶科學的穩固共識。但巴特利特 1932 那批**具體實驗**——⚠️ 方法很弱（本章後面專門處理）。這兩件事要分開記：結論穩、原始證據弱。

## schema：大腦的 default 模板，從哪借來的

「schema（基模）」這個詞，巴特利特不是自己造的——他**借自神經學家亨利·海德（Henry Head, 1861–1940）**。這個來歷講出來對你特別有用，因為它一講就懂。

海德研究的是**身體姿勢感**。你閉著眼睛也知道自己的手舉到哪、頭歪向哪一邊——這個「知道」靠的不是即時重新測量每個關節，而是大腦維護著一個**身體模型**，把過去的感覺經驗整合成一個持續更新的「我現在大概是什麼姿勢」的基準。新的感覺進來時，是拿來跟這個模型比對、更新模型，而不是從零重算。海德把這個身體模型叫 schema。

巴特利特把這個概念從身體搬到記憶：大腦對「一類情境通常長怎樣」也維護著模型——一間餐廳通常有什麼、一場葬禮通常怎麼進行、一個出征的勇士通常會幹嘛。回憶一段經歷時，大腦不是從零讀回每個細節，而是**拿這個模型當骨架、把少數真實片段掛上去，缺的地方用模型的預設值填**。

用你的話講：**schema 就是反序列化時遇到缺欄位填的 default。**

```text
回憶一段經歷的過程（壞掉的 event sourcing）：

  稀疏的真實片段（編碼時保留的 gist）   ← 真資料，但很少
        │
        ▼
  套上對應情境的 schema（餐廳/葬禮/出征…）  ← 模板，提供結構與預設值
        │
        ▼
  缺的細節用 schema 的 default 補上        ← 這些是「現編」的
        │
        ▼
  輸出一段「完整、合理」的回憶            ← 真片段＋default 混在一起
        │
        ▼
  你主觀上分不出哪些是真片段、哪些是 default ← 沒有欄位標「這格是猜的」
```

這個機制平常是**功能不是 bug**——這點要講清楚，否則你會以為大腦設計得很爛。schema 補洞讓你能用極稀疏的編碼（省記憶體）重建出可用的回憶，而且絕大多數時候補得**大致對**：你回憶「上週三的午餐」，補出來的「我坐在常坐的位子、用了平常那支湯匙」八成是對的，因為你的午餐 schema 就是從一堆真實午餐統計出來的，default 值本來就接近真值。**這正是脊椎的防線**：重建論說的是記憶**可被污染、鮮明≠正確**，不是「記憶全是編造、全不可信」——多數日常回憶的 default 值剛好接近真值，所以大致準確。

問題只發生在**真值偏離 default 值**的時候——比如那則故事裡的「靈魂出竅」對英國 schema 來說是個離群值，default 沒有這一格，於是被改成最接近的熟悉概念。下一節就看這種扭曲長什麼樣。

## 三種扭曲：同化、削平、銳化

巴特利特把重建造成的系統性扭曲歸納成幾種型態。本書鎖三個，你要能分辨它們、不要混用：

| 扭曲型態 | 機制 | 在故事裡的樣子 |
|---|---|---|
| **同化（assimilation）** | 把陌生元素**改成**符合自己 schema 的熟悉元素 | 「獵海豹（hunting seals）」被改成「釣魚（fishing）」；「獨木舟（canoe）」被改成「船（boat）」——換成英國受試者生活裡有的東西 |
| **削平（leveling）** | 把不符 schema、抓不住的細節**丟掉**，故事變短變簡 | 那些「沒道理」的超自然情節、奇怪的專有名詞，一次次再生中逐漸消失，故事整體縮水 |
| **銳化（sharpening）** | 把少數**符合預期、抓得住**的元素**放大、強調**，甚至補上原本沒有的因果連接 | 故事被重組成一個有清楚起承轉合的「英國式」敘事，因果被補得比原文更明確 |

注意這三者是**同一個機制的三個面向**，不是三個獨立規則。它們全都源自同一件事：**用 schema 當篩子**。符合 schema 的留下、被強化（銳化）；不符的要嘛被改成符合的（同化）、要嘛被丟掉（削平）。篩完，輸出一個「更像英國故事」的版本。

> **一個歸屬上的小心眼**：你可能在別處看過「削平／銳化／同化」被掛在奧爾波特與波斯特曼（Allport & Postman）的二戰謠言研究名下。那是後來的事——他們用**連續再生**研究謠言怎麼傳變形，沿用並推廣了這組詞彙。這組扭曲型態的**根**在巴特利特的記憶重建研究。本書把它歸給巴特利特、用在記憶上；謠言傳播是它的延伸應用，不是出處。

### worked example：走一遍《鬼魂的戰爭》的扭曲

不要停在「會扭曲」這種抽象話。拿故事裡一個具體元素，逐次再生地走一遍，看 schema 怎麼一步步咬它。我們拿「靈魂出竅／鬼魂」這條線（這是故事裡對英國受試者最異質的部分）：

```text
原文（巴特利特給的材料）：
  一名年輕人跟著一群人去打仗，作戰中聽見有人喊「那個年輕人中箭了」，
  但他不覺得痛、也不覺得受傷——暗示同行的其實是鬼魂。
  他回家講述經歷，到了黎明，一個黑色的東西從他口中冒出來，他就死了。

逐次再生的漂移（schema 開始咬）：

  第 1 次再生：
    「他不覺得痛」這個怪點被保留，但受試者開始覺得不對勁    ← 異質元素還在，但已有張力
        │
        ▼
  中間幾次再生（同化 + 削平）：
    「鬼魂」這個超自然框架不符英國 schema → 被淡化或拿掉    ← 削平：抓不住的丟掉
    「中箭卻不痛」被改寫成更合理的版本（受傷但撐著）        ← 同化：改成符合因果的
    「黑色的東西從口中冒出」被改成「他吐血」或直接刪掉      ← 同化／削平
        │
        ▼
  後期再生（銳化）：
    剩下的情節被重組成「一個人去打仗、受傷、回家、死了」    ← 銳化：補上清楚因果
    一個合邏輯的英國式故事，但原文的核心怪味全沒了          ← schema 篩完的成品
```

關鍵在最後一步的對比：受試者**並不覺得自己在亂改**。他們主觀上是在「盡力回憶」——只是回憶這個動作本身就是重建，而重建用的篩子是他們的文化 schema。他們以為自己在 read，其實 read 觸發了一連串 schema 驅動的 write（生成了原文沒有的細節）。**這就是本章脊椎的兌現點**：那個「他吐血」的細節，不是從記憶倉庫讀出來的，是這次回憶當下用 schema 現編、然後說出口的。

**這證明了什麼、又限制了什麼**，下一節必須誠實交代——因為這個經典示範，方法上其實漏洞百出。

## 誠實時間：核心對，但原始證據很弱

這一節是本書「事實紀律」的試金石，也是給你這種工程師看的。一個漂亮的 demo 跑通了，不代表它壓測過、不代表它的數字可信。巴特利特 1932 就是一個**結論對、但壓測沒做**的經典案例。

### 兩個方法、別混用

先把術語釘死，因為科普很愛混：

- **重複再生（repeated reproduction）**：**同一個人**在不同時間點多次回憶同一則故事（15 分鐘後、幾天後、幾個月後、甚至多年後）。《鬼魂的戰爭》的經典示範主要用這個。
- **連續再生（serial reproduction）**：**一人傳一人**——A 讀了故事後回憶給 B，B 的版本傳給 C，C 傳給 D……像傳話遊戲（telephone game）。

兩者都會展示扭曲，但它們**測的不是同一件事**：重複再生測「一個人的記憶隨時間怎麼漂」，連續再生測「訊息在人際傳遞鏈裡怎麼變形」（這也是為什麼它後來被拿去研究謠言）。把結論互相套用是錯的——一條人際傳遞鏈累積的扭曲，不能直接拿來說「你個人的記憶會扭曲這麼多」。**本書一律分清楚這兩個詞。**

### 方法學的洞

巴特利特那批實驗，用現代標準看，問題很實在：

1. **指示語鬆散**。他沒有明確、標準化地告訴受試者「請力求準確、不確定的就別寫」。受試者很可能把任務理解成「把這個故事**講出來**」——而講故事本來就會潤飾、補因果、讓它順口。那麼觀察到的「扭曲」，有多少是**記憶真的被重建了**，有多少只是**受試者在禮貌地把空白用合理猜測填滿、好把故事講完整**？這兩者在他的設計裡分不開。
2. **需求特徵（demand characteristics）**。受試者可能隱約感覺到「實驗者想看故事怎麼變」，於是不自覺配合——這會放大扭曲。
3. **沒有統計**。樣本小、報告以質性描述為主，沒有現代意義的效應量。

### Gauld & Stephenson 1967：把洞補起來，結論就軟了

最關鍵的後續，是高爾德與史蒂芬森（Gauld & Stephenson, 1967）的概念性複製。他們做了一個簡單但致命的操弄：**明確要求一組受試者「只回憶你確定的部分、不要猜」**。

結果：**一旦要求力求準確，扭曲就大幅減少。** 那些戲劇性的重建錯誤，相當一部分是在「允許你猜」的氛圍下的禮貌性填空——把准許猜測的許可拿掉，記憶其實比巴特利特展示的準得多。

這個結果不是推翻重建論，而是**校準**它。它告訴我們兩件事：

- **扭曲是真的**——大腦確實會用 schema 補洞、會同化會削平。Gauld & Stephenson 沒讓扭曲歸零，只是讓它變小。
- **但扭曲的量取決於你被怎麼要求回憶**——同一段記憶，在「隨便講」的指示下重建得很厲害，在「力求準確」的指示下重建得少。記憶的可塑性是**有條件的**，不是一個固定常數。

> **這是 ch09 邊界條件的第一次預演**：你會在再鞏固那章看到同樣的母題——「提取會打開改寫窗口」不是無條件的鐵律，而是**取決於提取的方式**（被動再暴露 vs 預測誤差式提取，見 ch09）。這裡也一樣：**取決於提取時的指示語**。記憶系統不是「每次 read 都觸發等量 write」的機械裝置，觸發多少 write、看你怎麼 read。把這個「有條件」的圖像記住，它是全書避免滑進「記憶全不可信」這種廉價虛無主義的關鍵。

### 所以我們為什麼還相信重建論？

因為**證據的重心不在巴特利特身上**。如果「記憶是重建的」只靠《鬼魂的戰爭》一個方法鬆散的 1932 古典示範撐著，那它頂多是個有趣假說。它之所以是穩固共識，是因為**後來一整批方法嚴格得多的研究**獨立地、用不同手法把它釘死了：

- 蘿芙托斯的誤導訊息實驗（見 ch05）——有控制組、有量化誤報率、操弄乾淨。
- DRM 典範（見 ch07）——大腦在**零外部誤導**下自己生成假記憶，內生重建的乾淨示範。
- 直到記憶準確度的「上限人類」HSAM 者照樣被植入假記憶（見 ch14）。

巴特利特提出了**正確的框架**，但**證明它的擔子**是後人挑的。身為工程師，你應該對這種科學狀態感到熟悉而不是不安：一個架構提案（design doc）方向對、但提案人附的 benchmark 很爛，這不代表架構錯——它代表你該去看後來別人做的、更嚴謹的壓測。本書接下來四章（ch05–ch07＋ch14）就是那些壓測。

## 直覺的陷阱

| 迷思／錯誤直覺 | 錯在哪一步 | 正解 |
|---|---|---|
| **「重建證明了所有記憶都是編造的、都不可信」** | 把「會被污染」滑坡成「全是假的」。Gauld & Stephenson 顯示力求準確時扭曲就小很多；多數日常回憶的 schema default 接近真值、大致準確（❌→正解） | 重建論說的是記憶**可被污染、鮮明≠正確**，**不是**「記憶全不可信」。守住這條，否則整本書會被讀成虛無主義 |
| **「schema 補洞是大腦的設計缺陷」** | 把一個高效設計誤判成 bug。沒有 schema，你得用海量編碼存每個細節，且無法歸納新情境 | schema 補洞**多數時候是功能**——用稀疏編碼重建可用回憶、且 default 通常接近真值。它只在真值偏離 default 時才出錯 |
| **「巴特利特用嚴謹實驗證明了記憶是重建的」** | 把「結論對」誤當成「原始證據強」。1932 那批實驗指示語鬆散、無統計、扭曲量取決於指示語 | 核心洞見✅穩固，但**原始實驗⚠️很弱**；我們信它靠的是後來更嚴格的研究（Loftus、DRM），不只靠《鬼魂的戰爭》 |
| **「連續再生＝重複再生，都是『記憶會變』」** | 把兩個測不同東西的方法混為一談 | **重複再生**＝同一人多次回憶（測個人記憶漂移）；**連續再生**＝一人傳一人（測人際傳遞變形）。結論不可互套 |
| **「扭曲是固定的——記憶就是會扭曲這麼多」** | 把可塑性當常數。同一段記憶在不同指示語下扭曲量差很大 | 扭曲量**有條件**——取決於提取方式／指示語（ch09 邊界條件的預演）。這正是避免「記憶必壞」誤解的關鍵 |
| **「schema 是靜態的知識資料庫」** | 把它想成一張查表。海德與巴特利特強調的是**動態**模型——每次經驗都在更新它 | schema 是**動態的、持續被經驗更新**的組織結構，不是凍結的查找表；它既塑造回憶、也被新回憶塑造 |

自我察覺的徵兆：如果你發現自己想說「所以記憶都是假的、我什麼都不能信了」——停。你正滑進第一個陷阱。正確的姿態是工程師式的**校準**：記憶在哪些條件下會漂、漂多少、怎麼判斷哪些細節最可能是 default 補的（提示：越是「太合理、太符合你對那個人/那類場景的預期」的細節，越可疑——那正是 schema 最會出手的地方）。

## 紙上推演

### 推演一（★，[10 分鐘]）：把一句白話翻成 schema 機制鏈

把這句白話宣稱翻成一條一步步的因果鏈：「**一個目擊者把一個他沒看清楚的陌生場景，記成了一個熟悉的版本。**」要求每一步標出是哪種扭曲（同化／削平／銳化），並指出哪一步是「現編」（write）。

#### 推演解答

一條合格的鏈大概長這樣：

```text
① 事件發生時編碼匆忙、現場陌生 → 只存下稀疏 gist        ← 編碼有損（ch02）
② 提取時大腦套上最接近的熟悉 schema（「這類場景通常…」） ← 重建啟動
③ 沒看清的陌生細節，被改成 schema 裡的熟悉對應物        ← 同化（write）
④ 抓不住、不符 schema 的細節被丟掉                      ← 削平
⑤ 少數符合預期的元素被強調、補上合理因果               ← 銳化（write）
⑥ 輸出一段「清晰、合理」的回憶，目擊者主觀上以為是讀出來的 ← 真片段＋default 混合
```

常見錯路：(1) 只說「他記錯了」而不指出**方向**——重點是扭曲有系統地往「熟悉」漂，不是隨機。(2) 漏掉第⑥步的「主觀上分不出」——這是整個機制最反直覺、也最危險的地方（伏筆來源監控，見 ch13）。第③和第⑤步是 write：那些細節是這次提取**生成**的，不是倉庫裡讀出來的。

### 推演二（★★，[15 分鐘]）：找出壞論證的破綻

有人說：「**巴特利特的《鬼魂的戰爭》實驗證明了所有記憶都是編造的、目擊證詞一律不可信。**」這個論證至少有三個破綻，把它們找出來。

#### 推演解答

三個破綻，由近到遠：

1. **過度延伸（最嚴重）**。「會被重建扭曲」≠「全是編造、一律不可信」。這是脊椎防線。多數日常記憶的 schema default 接近真值、大致準確；Gauld & Stephenson 1967 還顯示「力求準確」時扭曲大幅減少。把「可被污染」講成「全假」是滑坡。

2. **單一弱研究撐不起全稱結論**。就算結論方向對，巴特利特 1932 本身方法鬆散（指示語不標準、無統計、需求特徵）——拿一個 demo 級的研究去支撐「**所有**記憶」「**一律**不可信」這種全稱句，論證強度遠遠不夠。要支撐重建論，得搬出後來更嚴格的證據（Loftus、DRM），而那些證據說的也不是「全不可信」。

3. **材料外推問題**。《鬼魂的戰爭》是刻意挑的**極端陌生**材料（異文化、超自然、反英國因果），它放大扭曲是設計使然。把這種極端材料的扭曲量直接外推到「一般目擊者看到的尋常場景」，是把實驗條件當成普遍結論——尋常、符合 schema 的場景，重建反而補得相對準。

附帶一個工程類比可以幫你記：這就像拿一個**刻意構造的最壞輸入**（adversarial input）讓系統出包，然後宣稱「這系統處理**任何**輸入都會出包」。最壞輸入證明的是「存在會出包的輸入」，不是「全部輸入都出包」。

### 推演三（★★★，[20 分鐘]）：設計一個能分開「重建」與「禮貌填空」的假想實驗

巴特利特的致命弱點是：他觀察到的扭曲，分不出是**記憶真的被重建改寫了**，還是受試者**只是在被准許猜測的氛圍下禮貌地填空把故事講完**。請設計一個假想實驗，把這兩者分開。（提示：想想 Gauld & Stephenson 動了哪個變數。）

#### 推演解答

核心思路：**把「指示語／回憶氛圍」當自變數操弄，看扭曲量是否隨之改變**——如果扭曲是「真的記憶重建」，它不該因為你叫人小心一點就大幅消失；如果扭曲主要是「禮貌填空」，那把猜測的許可拿掉，它就該大幅下降。

一個可行的假想設計：

```text
同一則陌生故事，受試者隨機分三組，延遲時間相同：

  A 組（寬鬆指示）：「把這個故事講出來」          ← 准許/暗示可以補完整
  B 組（嚴格指示）：「只寫你確定的、不確定就留空」 ← 拿掉猜測許可
  C 組（標記指示）：「正常回憶，但凡是你『不太確定、 ← 讓受試者自標來源
                    可能是自己補的』的部分標星號」

量測：每組的扭曲量（同化/削平/銳化的次數），以及 C 組被標星號的內容
      事後與原文比對的命中率。

預測與判讀：
  • 若 A 扭曲 ≫ B → 大量「扭曲」其實是寬鬆指示下的禮貌填空（Gauld 結果）
  • 但 B 仍有一定殘留扭曲 → 那部分是「真·重建」，叫人小心也壓不掉
  • C 組若「被標星號處」的扭曲率明顯高於「沒標星號處」→ 受試者部分
    能察覺自己在補（但不會全察覺，否則重建就不危險了）→ 伏筆來源監控（ch13）
```

討論：這個設計的精神就是 Gauld & Stephenson 做的事——他們動了 A vs B 這個變數，發現 B 的扭曲大幅變小，因此證明巴特利特原始設計把「重建」和「禮貌填空」混在一起了。C 組是我加的延伸，目的是測「受試者對自己重建的自覺程度」——而本書後面會告訴你，這個自覺**很不可靠**（自信與準確脫鉤，見 ch10；來源無標籤，見 ch13）。常見錯路：只設計「測扭曲量」卻不操弄指示語——那你就重蹈巴特利特覆轍，永遠分不出兩種來源。

## 自我檢核

口頭自答，講得出來才算過。優先答「為什麼」。

1. **為什麼**巴特利特要刻意挑一則異文化、不合英國因果邏輯的故事當材料，而不用一則尋常故事？（提示：扭曲的**方向**是判別器。）
2. schema 到底是什麼？用一個你熟悉的工程概念（反序列化填 default／event sourcing 補缺失事件）把它講給另一個工程師聽。
3. 同化、削平、銳化各是什麼？它們**為什麼是同一個機制的三個面向**而不是三條獨立規則？
4. 重複再生和連續再生差在哪？它們**測的是不是同一件事**？為什麼結論不能互套？
5. **為什麼**說巴特利特「結論對、但原始證據弱」？Gauld & Stephenson 1967 動了哪個變數、發現了什麼、這對重建論是**推翻**還是**校準**？
6. 「記憶是重建的」——這句話**為什麼不等於**「所有記憶都不可信」？你怎麼向一個聽完就想說「那我什麼都不能信了」的人守住這條防線？
7. 回到脊椎：受試者把「黑色的東西從口中冒出」記成「他吐血」——這是 read 還是 read 觸發了 write？那個「吐血」的細節是哪一步生成的？

## 延伸閱讀

- **Bartlett, F. C. (1932). _Remembering: A Study in Experimental and Social Psychology_.** 原典。不必通讀，重點看《鬼魂的戰爭》那則故事原文與他對「重建」「schema」的論述——你會發現他講的比多數二手轉述都更謹慎、更像在提假說而非下定論。
- **McGraw Hill,「Reconstructive Memory & Schema Theory」**（https://www.mheducation.ca/blog/series-classic-learning-science-reconstructive-memory-schema-theory/）。本章 landscape 來源之一，把重建記憶與 schema 理論講得清楚、適合校對「課本版說法」，但記得課本版常省略方法學爭議。
- **Gauld, A. & Stephenson, G. M. (1967).「Some experiments relating to Bartlett's theory of remembering.」**（_British Journal of Psychology_）。本章「結論對、證據弱」這一節的關鍵後續——讀它怎麼用一個指示語的操弄就讓扭曲大幅縮水。是本書「重建有條件」母題的源頭之一。
- **「Bartlett revisited: Direct comparison of repeated reproduction and serial reproduction techniques」**（_Journal of Applied Research in Memory and Cognition_, 2014；https://www.sciencedirect.com/science/article/abs/pii/S2211368114000485）。直接比較兩種方法的現代研究——如果你想把「重複再生 vs 連續再生別混用」這件事弄到很實，讀這篇。

> **本章在全書地圖的位置**：你現在懂了大腦**自發**用 schema 補洞、為什麼會扭曲、以及這個結論本身的證據強弱。下一章（ch05）把張力上滿——不再是大腦自己補洞，而是**外人用一個動詞主動寫進你的記憶**：誤導訊息效應，全書高潮之一，也是「read 觸發 write」最乾淨的示範。
