# 寫作風格指南（內部文件，非書籍內容）

本文件是全書的寫作契約。每一章都必須遵守，否則全書會失去一致性。
本書書名（工作名）：《大腦怎麼編造過去：記憶作為重建的科學》（id=mem）。最終書名於 outline gate 由作者定案，撰章時以工作名為準。

脊椎一句話：**記憶不是錄影回放，而是每次提取都重新「建構」**——因此可被污染、植入、扭曲，而且鮮明度與正確度脫鉤。脊椎把「錄影帶」隱喻換成「每次 read 都會觸發一次 write」。全書反覆問同一個問題：**「這段記憶是被讀出來的，還是被重新建構（並可能在這次回憶中被改寫）的？」**

---

## 這本書是什麼、不是什麼

- **是**：一本「欣賞與直覺」的書。目標是讓一位資深後端工程師親身**看懂記憶的運作機制**——記憶不是硬碟上一個檔案被原封讀回，而是一條**每次都要重新跑一遍的重建管線**；讀完能用自己的話對另一個工程師講清楚：為什麼一段「鮮明到不可能錯」的記憶可以是錯的、假記憶怎麼被植入、提取為什麼會反過來改寫記憶本身。讀完有「原來我對自己過去的記憶，有一大半是這次回憶當下現編的」這種冷靜的震撼。
- **不是**：
  - **不是教科書**——不背腦區表、不刷神經解剖名詞、不出選擇題。神經機制（海馬迴、杏仁核、再鞏固分子鏈）只在「它解釋了哪個記憶現象」時才出場，不為了完整而完整。
  - **不是自助手冊／記憶力課程**——不教你怎麼考高分、不賣記憶宮殿速成法。間隔效應、測試效應、記憶宮殿會講（它們是少數**真穩**的結論，見 §深度標準），但講的是**機制為什麼有效**，不是「七天練成過目不忘」。
  - **不是醫療／治療／戒斷指南**——**本書明確不提供任何醫療、心理治療、創傷處理、用藥（含 propranolol/普萘洛爾）、戒斷或自我診斷建議。** 凡涉及 PTSD、創傷記憶、再鞏固療法、恢復記憶等臨床主題，本書只談**機制與科學證據的現況與爭議**，並在相關章節明文標注「這是科學討論，不是臨床指引；任何個人狀況請找合格專業人員」。談 propranolol 一定連著講「臨床療效未定論」（見 landscape §五），絕不寫成「這樣做可以抹除壞記憶」。
  - **不是自我追蹤實驗書**——**沒有 Python、沒有真實世界的自我實驗**。不要叫讀者寫睡眠日誌、記憶日記、自測遺忘曲線。「動手做」一律是**紙上推演**（見骨架第 5 段）。
- **與其他三本興趣書完全獨立、互不引用**。記憶與系統工程的血緣（提取即重寫＝read-repair、event sourcing 重建狀態、cache 失效、最終一致性下的 source-of-truth 漂移）是本書最划算的橋，要用滿但不喧賓奪主——橋接是為了讓機制讀起來像母語，不是把記憶硬塞進分散式系統的框架裡。

### 全書六條驗收主線（finale 章逐條口試）

撰章時每一章都要想「我在替哪一條主線存證」。finale 章會逐條口試，講不出來＝沒過：

1. **編碼／儲存／提取三關卡**，以及**「提取即重寫」為何成立**——記憶經過編碼（encoding）、儲存（storage）、提取（retrieval）三道關卡，每一關都可能引入扭曲；提取不是唯讀，提取本身會把記憶帶回不穩定狀態並可能改寫它（脊椎「每次 read 觸發一次 write」的核心）。
2. **schema（基模）與誤導訊息效應（misinformation effect）如何污染記憶**——大腦用既有基模補空白、事後的措辭與提問會回灌進「原始」記憶（Bartlett、Loftus & Palmer）。
3. **假記憶可被植入的證據與限度**——完整的自傳式假事件可以被植入（lost-in-the-mall、DRM），但「可植入」不等於「一切記憶皆假」；證據強在哪、外推的爭議在哪，都要交代。
4. **系統鞏固（systems consolidation）與再鞏固（reconsolidation）的神經機制**——記憶如何從海馬迴依賴轉為皮質依賴；提取如何打開再鞏固的改寫窗口；動物現象穩、人體強療法不穩的分界。
5. **鮮明 ≠ 正確（閃光燈記憶的背叛）**——主觀生動度與自信，跟客觀準確度是脫鉤的（Neisser & Harsch、Hirst 9/11）。
6. **記憶可錯性在法庭與日常的後果與防禦**——目擊證詞、恢復記憶、記憶戰爭；個人與制度層面能做什麼、不能做什麼。

**橋接錨點清單（要用滿但不喧賓奪主，每章「從你已知的出發」優先從這裡取）**：
- **快取失效與讀取時重算** → 提取時重建、記憶不是讀檔是重算。
- **event sourcing vs snapshot** → 大腦用「事件重播」重建一段經歷，而非存一張快照；重播時用當前的 schema 補洞，所以會漂移。
- **read-repair / 讀取時順手改寫** → 再鞏固：提取（read）順手把記憶改寫（write）回去。
- **最終一致性與 source-of-truth 漂移** → 反覆回憶讓「記得的版本」逐漸偏離「真正發生的」，且自己無法察覺。
- **lossy 壓縮編碼** → 編碼階段就丟資訊、保留要旨（gist）丟細節，提取時用要旨重建細節（所以細節常是現編的）。
- **寫入放大** → 一次提取觸發的改寫，比「只是讀一下」造成的副作用大得多。

注意邊界：橋接是**直覺的腳手架**，不是**證明**。分散式系統的類比能幫讀者「感覺對了」，但記憶的真相由神經科學與心理學實證決定——當類比與實證衝突時，**以實證為準並明說類比在哪裡破掉**（例：再鞏固不是每次提取都觸發，有 boundary conditions，不像 read-repair 那麼機械）。

## 每章骨架（必須遵守，標題名稱逐字照抄——Tier-1 lint_book.py 會 grep 這些標題，順序固定）

1. **章首**：`# chNN — 章名：副標`，緊接 blockquote：
   `> **本章解決什麼問題**：…（這章在全書地圖的位置、為什麼存在、跟前後章的關係，3–5 行）`
   每個 Part 的第一章在 blockquote 後放**全書 ASCII 路線圖**（基準圖在 outline.md，CJK=2 欄對齊；照抄並把「◄你在這裡」標到本 Part），讀者偏好架構圖先行。
2. `## 從你已知的出發` — 橋接段：把本章核心錨定在讀者的後端／系統工程實戰直覺上，要**具體**（指名 cache 失效、event sourcing、read-repair、最終一致性、lossy 壓縮、寫入放大、idempotency、profiling、TD/prediction-error 中哪一個，並說清楚對應到哪），不要泛泛而談。
3. **核心 section（多個 `##`）** — 深入機制與直覺。歷史敘事服務於機制理解（為什麼這個研究在當時是突破、它解決了哪個謎），不是編年史。每個概念先給「它在解什麼問題」，禁止無動機的定義。
4. `## 直覺的陷阱` — **本書招牌段落**：這個主題最常被誤解成什麼**科普迷思／錯誤直覺**、錯誤直覺**會在哪一步把你帶溝裡**、**怎麼自我察覺**（什麼徵兆代表你其實沒懂）。用**表格或條列**。記憶這個主題迷思密度極高（記憶像錄影、情緒讓記憶更準、超憶症＝完美記憶、催眠能挖出真記憶、自信高就準、重複就記得牢…），這段是全書最有價值的段落之一。每條迷思要標明**正解是什麼**，並對齊 landscape 的 ❌/🔶 彈藥。
5. `## 紙上推演` — **動手做（純紙上推演，無 Python、無自我追蹤實驗）**。每章 2~4 題思想實驗／推演，每題標**難度（★~★★★）**與**預估時間（如 [15 分鐘]）**。題型：**預測一個研究的結果再對答案**、**找出一個壞論證的破綻**、**把一句白話宣稱翻成機制**、**設計一個假想實驗**、**口頭重講一條因果鏈**。題目後緊接 `### 推演解答` 給完整解答與討論（為什麼這樣想、常見錯路、研究實際數字）。
6. `## 自我檢核` — 5~8 題，**口頭自答**：講得出來才算過關。對齊「能不能講給另一個工程師聽」的深度，**優先出「為什麼」而非「是什麼」**。
7. `## 延伸閱讀` — **真實連結**（以 landscape 驗證過的為優先），每條一句話說明值得讀什麼、讀哪一段。沒把握的連結標「（未驗證）」。

## 深度標準

- **每章至少一個 worked example**：本書系的 worked example 是「**一條走到底的機制鏈**」或「**一個真實研究的數字逐步解讀**」，不准跳步。具體形式至少要是以下之一：
  - 把一個統合分析（meta-analysis）／實驗的**百分比逐步解讀**（例：Loftus & Palmer 碎玻璃實驗 smashed 32% vs hit 14% vs 控制 12%，一個動詞造成的差幅怎麼算、代表什麼）。
  - 走一條 **dopamine（多巴胺）預測誤差訊號**的變化（TD error 怎麼從「獎勵來了」轉成「線索預告獎勵」，橋接讀者的 RL 直覺）。
  - 走一夜 **hypnogram（睡眠分期圖）的時序**與哪一段對應哪種記憶鞏固。
  - 走一次**誤導訊息實驗的數字**（樣本、操弄、誤報率）。
  - 走一條**再鞏固的分子／時序鏈**（提取 → labile → 蛋白質合成窗口 → 重新穩定，以及干擾窗口在哪）。
  - example 帶的數字一律對照 landscape，**不四捨五入成圓整數**（lost-in-the-mall 是 n=24／約 25%，不是「一半」；Neisser/Harsch 是 106→44 人／約 2.5 年）。
- **嚴謹度誠實標示**：每個論斷明確標示自己屬於哪一級——
  - 「**這是穩固共識**」（重建論本身、測試效應、間隔效應、HM 的分離、再鞏固這個動物**現象**）；
  - 「**這是主流假說但有爭議／邊界未定**」（系統鞏固的標準模型 vs 多重痕跡理論、睡眠鞏固的具體機制）；
  - 「**這是迷思，正解是…**」（記憶像錄影、情緒讓記憶更準、HSAM＝完美記憶、催眠挖真記憶）；
  - 「**這是 unsettled debate，雙方並陳**」（再鞏固的**人體臨床療效**、壓抑／恢復記憶的記憶戰爭、déjà vu 的機制）。**重大爭議必須呈現雙方、不替讀者選邊斷言**（landscape 的 🔶/🔶🔶 條目）。
- **禁「顯然」「眾所周知」「易證」**；禁止把單一小樣本研究（Bartlett 1932、Ebbinghaus n=1）寫得比它更穩——這類研究要同時寫出**它的方法學限制**與**後續更嚴格的複製**（Gauld & Stephenson 1967、Murre & Dros 2015），這正是本書「重建 vs 儲存」對照的好素材。
- **脊椎每章復現**：每章至少一處明確把當章內容接回脊椎——「這段記憶是被讀出來的，還是被重新建構（並可能在這次回憶中被改寫）的？」或其工程版「這裡是 read，還是 read 觸發了 write？」。脊椎要能被讀者用自己的話轉述，不是口號。
- **時效性／可爭議事實標時點**：版本、數字、效應量、年份、歸屬、樣本數一律當作可能過時或被誤傳，標 `（2026-06）` 並 hedge（「約」「據報」「常被引為」「目前學界傾向」）。
- 字數：每章 **5,000~9,000 中文字**，依主題深度調整（地基章、爭議章可偏厚）。寧深勿廣，不灌水。

## 事實正確性（零信任）

神經科學／心理學科普充斥被過度簡化、被推翻、或仍有爭議的「事實」。本書的事實紀律：

- **以 `_meta/landscape-2026-06.md` 為事實基準**：撰章時引用任何實驗、數字、年份、歸屬、引文，先對照 landscape，**不要憑記憶或憑「教科書都這樣寫」**。landscape 的 ✅/⚠️/❌/🔶 標記就是寫法指令：✅ 照寫（但精確值不四捨五入）、⚠️ 寫保守版並 hedge、❌ 是「直覺的陷阱」段的彈藥（給正解）、🔶 雙方並陳不選邊。
- **每章 3–8 次 web 查證**：landscape 未涵蓋的當章特定細節（具體數字、引文、近年複製、現況）自行用 WebSearch/WebFetch 查證；查到對 landscape 的修正時，**先更新 landscape → 同步引用章 → 在 maintenance 掃描日誌記一行**；重大修正（推翻既有基準）需**兩個獨立來源**。
- **守住的高風險事實**（landscape 末節「待守住的高風險事實」逐條，撰章前必對照）：
  1. lost-in-the-mall：原始 **n=24、約 25%（6 人）**；2023 預先註冊複製 **n=123、約 35%**；別寫成圓整數或「一半」；倫理外推有爭議，雙方並陳。
  2. 再鞏固的藥：Nader 2000 用 **anisomycin（茴香黴素，大鼠）不是 propranolol**；propranolol 是後來人體研究（Brunet 2008），**臨床效果至今有爭議**（2022 有否定的 meta-analysis）。動物現象穩、人體強療法不穩，雙方並陳。
  3. 閃光燈記憶：情緒**不**讓記憶更準；Neisser/Harsch（106→44 人、約 2.5 年）與 Hirst 9/11 都證**自信與準確脫鉤**。
  4. HSAM 不是完美記憶：只在自傳式領域超群、死記不優；**HSAM 者照樣會被植入假記憶**（脊椎的實驗化身）；Jill Price 學業普通。
  5. Bartlett：核心洞見成立，但 **1932 原始實驗方法鬆散**、扭曲取決於指示語；serial ≠ repeated reproduction。
  6. HM：手術年齡寫 **27 歲**；非「完全沒有新記憶」——程序記憶保留是關鍵分離。
  7. 記憶宮殿／間隔／測試效應**是真有效的**，別為戲劇性寫成迷思。
  8. 記憶戰爭是 **unsettled debate**，雙方並陳不選邊。
  9. 「重建」**別過度延伸成「記憶全不可信」**——多數日常記憶大致準確，重建論說的是「可被污染、鮮明≠正確」。
- **引文與歸屬**：名人名言、研究歸屬、年份不得憑記憶引用；對照 landscape，landscape 沒有的查到原始來源。

## 語氣

工程師對工程師。冷靜、直接、有觀點（可以寫「我認為這是整本書最該起雞皮疙瘩的一頁」）。**不販賣神話、不喊「太神奇了」「大腦好神奇」**——記憶科學的震撼來自把機制講清楚（一個動詞改寫了你「看到」的東西；你越自信反而越可能錯；超憶症的人也會被植入假記憶），而不是堆形容詞。允許冷幽默。

每個機制出場都有白話直覺在旁邊；但也不假裝簡單——該慢的地方明說「這裡值得你停十分鐘」。涉及創傷、法庭、記憶戰爭等敏感主題時，語氣**克制、不獵奇**：承認對受害者的關切與對冤案的關切**兩者都正當**，承認科學的不確定，**不替讀者做臨床或道德裁決**。本書的可信度建立在「敢說『這我們還不知道』」，而不是把每件事都講得斬釘截鐵。
