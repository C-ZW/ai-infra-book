# ch07 — DRM：你會記得一個從沒出現的詞

> **本章解決什麼問題**：ch05 證明了「外部的措辭可以污染記憶」、ch06 證明了「整段事件可以從外部植入」——兩個都需要一個**外面來的寫入**（誤導的動詞、家人背書的假事件）。這章把張力推到最盡頭：**假記憶根本不需要任何外部誤導，大腦自己就會無中生有。** 給你一串語意相關的詞，你會用高度的確定感「記得」一個從頭到尾沒出現過的詞。這是「鮮明＝正確」這個直覺第一次被正面擊穿，也是脊椎最乾淨的一次示範——這次的「read」連污染源都沒有，扭曲完全是重建管線內生的。

這是 Part II「重建與污染」的收尾章。Part II 的弧線是一條升級階梯：ch04 大腦**自發**用 schema 補洞 → ch05 外部措辭**污染**細節 → ch06 外部背書**植入**整個事件 → ch07 **連外部都不要**，純內生的假記憶。走完這一階，你就準備好進 Part III 看這一切的神經底層了。

## 從你已知的出發

你的服務前面掛了一層 cache，而你早就學會一件事：**cache 不會只裝你明確要的那一筆。** 為了壓低 p99 延遲，它會做 prefetch（預取）——你讀 `user:1234` 的時候，cache 順手把同一個 batch、同一個 hash slot、語意相鄰的那幾筆也一起載進來，賭你下一個請求大概率會要它們。CPU 也在做同一件事，那叫推測執行（speculative execution）：分支還沒判完，它已經先把「最可能那條路」的指令和資料載進管線了。

這套機制有一個你很熟的副作用。**被預取進來的東西，和你真正請求過的東西，在 cache 裡長得一模一樣。** 一筆資料躺在 L2 裡，上面沒有寫「這是你 1 毫秒前明確 SELECT 的」還是「這是我替你猜著預取的」。如果你後來去問「我剛剛到底讀過哪些 key」，你只能看「現在 cache 裡有什麼」——而那裡面混著一堆你從沒主動要過、只是被自動載入的鄰居。要分辨「主動讀的」和「被預取的」，你得有一份獨立的 access log；cache 本身分不出來。

把這個機制原封不動搬到記憶上，你就得到了本章的全部內容。

當你看一串詞——`床、休息、清醒、疲倦、做夢、打盹、毯子、睡意、打鼾……`——你的語意網路不會只活化這幾個你「明確讀到」的節點。活化會像 prefetch 一樣**擴散**到語意相鄰的節點，而所有這些詞共同指向的那個中心節點——`睡覺`——會被間接活化到**非常強**，甚至比清單中段某些真正出現過的詞還強。然後問題來了：等你事後回憶，大腦去「讀 cache」，發現 `睡覺` 這個節點是熱的、是活化的、帶著鮮明的語意內容——它分不出這個活化是「你真的看到這個詞」造成的，還是「它被你自己的語意網路預取進來」造成的。

沒有 access log，就沒有 provenance。於是大腦做了你的 cache 會做的同一件事：**把預取進來的，當成了實際讀到的。** 你「記得」看過 `睡覺`，而且確定感很高。

這就是 DRM 典範。它之所以是全書最該起雞皮疙瘩的幾頁之一，是因為前兩章的假記憶你還能怪外面——是那個動詞、是那個多嘴的家人。這一章沒有外面。**污染源就是你自己的重建機制。** 這裡的「read 觸發 write」，連一個外部寫入者都不需要：提取這個動作本身，把「腦中被活化的」寫成了「真正輸入過的」。

## DRM：實驗長什麼樣，結果為什麼嚇人

先把這個典範的歷史與機制立穩，再走數字。

### 一個被遺忘 36 年的觀察，被重新發現

故事的 D 是 **詹姆斯·迪斯（James Deese）**。1959 年，他做了一個當時沒掀起多少波瀾的觀察：如果你給人一串全部跟某個詞強烈相關、但那個詞本身不在清單裡的詞，人在自由回憶時會「想起」那個沒出現的詞。這個發現在文獻裡躺了三十多年。

直到 1995 年，**亨利·羅迪格（Henry "Roddy" Roediger III）與凱薩琳·麥克德莫特（Kathleen McDermott）** 把迪斯的老觀察撿起來、做成一個乾淨、可大量複製的實驗典範，發表〈Creating False Memories: Remembering Words Not Presented in Lists〉（1995，*JEP: LMC*）。因為建基於 Deese 1959，這個典範的全名就把三個人的名字都收進去：**DRM（Deese–Roediger–McDermott）典範**（2026-06）。

為什麼一個 1959 年沒人理的觀察，到 1995 年變成記憶科學最常用的工具之一？因為時機。到 1990 年代，蘿芙托斯（Loftus）的誤導訊息研究（ch05）和商場走失（ch06）已經把「記憶可以被植入」推到風口浪尖，但批評者總有一句話可以反駁：**「那都是因為有人在外面誤導／暗示啊。把外部的污染源拿掉，記憶還是可靠的。」** DRM 的殺傷力就在於它把這句辯護徹底拆了——**它的清單裡沒有任何誤導訊息，沒有人暗示你看過 `睡覺`，但你照樣記得。** 假記憶不需要外部寫入者，這是它存在的全部意義。

### 實驗骨架

DRM 的操作簡單到你可以對著另一個工程師三句話講完：

```text
研究階段                          測驗階段
──────────                        ──────────
逐一呈現一串 15 個詞，            事後做「自由回憶」或「再認」：
全部語意指向同一個               ・自由回憶：請把剛剛看過的詞寫下來
未出現的「關鍵誘餌詞」     →      ・再認：給一批詞，逐一判斷「看過 / 沒看過」
（critical lure）                 關鍵測量：受試者把「關鍵誘餌詞」
例：床/休息/疲倦/做夢…             報告成「看過」的比率有多高？
全指向 → 睡覺（未出現）
```

羅迪格與麥克德莫特 1995 用了 **24 串清單、每串 15 個詞**（2026-06）。每串都圍著一個關鍵誘餌詞設計：清單裡每個詞都是那個誘餌詞的強語意聯想詞，但誘餌詞本身**從不出現**。

關鍵的測量不是「你記得多少真的詞」，而是——**你有多常、多確定地『記得』那個從沒出現的誘餌詞。**

### 數字：守住「依清單而異」，別咬死一個百分比

這是 landscape 明確要求 hedge 的地方，照規矩寫。誤記率**依清單、依測驗方式（自由回憶 vs 再認）而變動很大**，所以不存在一個「DRM 誤記率＝X%」的單一答案。我查證後（2026-06）給你的是**範圍與量級**：

| 測量方式 | 對關鍵誘餌詞的「假陽性」量級 | 備註 |
|---|---|---|
| 自由回憶（free recall） | 約 **40–55%**（依清單、版本） | Roediger & McDermott 1995 實驗一常被引為約 40% |
| 再認（recognition） | **可達 80% 以上**，逼近對真實清單詞的「命中率」 | 即：你對「沒出現的誘餌詞」說「看過」的機率，幾乎跟對「真出現的詞」一樣高 |

讓你抓住量級的一句話（這是最穩的寫法）：**對某些清單，超過半數的人在自由回憶時會「想起」那個沒出現的詞；改用再認測驗，誤判率可以飆到八成以上——逼近他們對真實看過的詞的記得率。**（2026-06，依清單而異）

但真正令人不安的不是這個數字本身，是下一段。

### 不只是「猜對了」：他們**主觀上記得**

你可能想：八成？也許他們只是覺得 `睡覺` 這個詞「應該有吧」，半猜半蒙地填了。如果只是這樣，這就不是假記憶，只是合理推測——就像你看到一堆訂單欄位、推測「應該有個 `total` 欄位」，那是推理，不是記憶。

DRM 的研究者早就堵死了這條路，用的是記憶研究的一個經典工具：**「記得 / 知道」判斷（remember / know judgment）**。再認時不只問「看過嗎」，還追問一句：

- 選 **「記得（remember）」**＝你能**重新體驗到**看到它的那一刻：它出現在清單的哪個位置、當時的聲音、你那時腦中閃過什麼。
- 選 **「知道（know）」**＝你**覺得它出現過、但回想不出具體的看到經驗**——一種光禿禿的熟悉感。

如果 DRM 的假記憶只是「合理推測」，受試者應該大量選「知道」（我覺得有，但說不上來）。**結果完全相反。** 我查證的數字（2026-06）：在羅迪格與麥克德莫特的實驗二，被誤認的關鍵誘餌詞裡，**約 72% 被判為「記得」**——也就是說，大多數人不只說「看過」，還**主觀上重新體驗到了一個從未發生的看見**。有受試者甚至能「回憶」出那個詞是用什麼聲音唸的、出現在清單第幾個。

這就是這一章該讓你停下來的地方。**這份假記憶帶著真記憶的全套主觀特徵——鮮明、具體、附帶細節、高度確定。** 你沒辦法靠「它感覺起來夠不夠真」來判斷它真不真，因為它感覺起來就是真的。鮮明，從來不是正確的證據。這條我們到 ch10 閃光燈記憶會再用一次、放到一個真實的歷史事件上放大。

## Worked example：走一遍一串清單的活化鏈

光說「語意活化」太抽象。我們拿一串真的清單，一步一步看 `睡覺` 是怎麼被活化到比真實詞還強的——這是本章的 worked example，一條走到底、不跳步的機制鏈。

拿 `睡覺（sleep）` 這串經典清單。15 個詞大致是這樣（次序與確切詞會依版本微調，這裡用通行的代表版）：

```text
床    休息    清醒    疲倦    做夢
打盹  毯子    睡意    打鼾    午睡
平靜  打哈欠  昏昏欲睡  眠     安詳
                                  ← 關鍵誘餌詞「睡覺」全程未出現
```

逐步看發生了什麼：

```text
步驟 1：你逐一讀這 15 個詞                  ← 每個詞活化它自己的語意節點
步驟 2：每個節點的活化向鄰居擴散            ← 「床」活化「睡覺」一點、「休息」也活化「睡覺」一點…
步驟 3：「睡覺」收到 15 道匯流的活化        ← 它是全清單的共同中心，被「灌」得最滿
步驟 4：「睡覺」的活化強度，可能超過        ← 清單中段某個真實詞（如「打哈欠」）只被讀過一次、
        清單中段某個你真的讀過的詞             還可能被前後詞干擾，活化反而比「睡覺」弱
步驟 5：事後提取，大腦掃描「哪些節點是熱的」 ← 「睡覺」很熱、很鮮明
步驟 6：來源監控（ch13 詳述）失敗：           ← 大腦用「這麼鮮明，多半是真看過的」這條啟發式，
        把「自己活化的」誤判成「外部輸入的」     把內生活化誤標成「看過」
步驟 7：你「記得」看過「睡覺」，且選「記得」  ← 假記憶生成，附帶主觀的看見體驗
```

關鍵在步驟 3 到 4。`睡覺` 沒出現在輸入裡，但它收到了**全部 15 個詞**各自貢獻的一點活化，是清單上唯一被 15 條線同時指向的節點。而一個真實出現、但排在清單中段的詞（比如 `打哈欠`），只被你讀過**一次**，還會受到前後詞的干擾（這是 ch03 講過的逆向／順向干擾）。結果就出現了那個荒謬但站得住的局面：**一個你從沒看過的詞，在你腦中的活化強度，超過了一個你真的看過的詞。** 等大腦事後去讀「哪些節點亮著」，它根本沒有理由懷疑 `睡覺`——它亮得比真詞還亮。

學界給這條鏈一個名字：**活化—監控架構（activation–monitoring framework）**（Roediger 等人，2026-06 查證）。兩個階段：**活化**（編碼時，擴散活化自動點亮了沒出現的誘餌詞）＋**監控**（提取時，你得判斷這個活化的來源，而這個監控會失敗）。注意這個雙階段的結構——它正好對應你 cache 的兩個問題：prefetch 把鄰居載進來（活化），以及事後分不出哪些是主動讀的（監控失敗）。活化幾乎是自動、擋不住的；能不能不被騙，全看監控這一關。而監控，靠的是一條會出錯的啟發式（「夠鮮明就是真的」），這也是為什麼**事先警告你「等下會有沒出現的詞混進來」幫助有限**——我查證的結果（2026-06）：警告若在**讀清單之前**給，能稍微降低假記憶；若等到**回憶之前**才警告，幾乎沒用。活化已經發生了，你攔不住一個已經點亮的節點。

把這條鏈接回脊椎，問那個全書反覆問的問題：**你「記得看過睡覺」這件事，是 read，還是 read 觸發了 write？** 答案是後者，而且是最純粹的後者——這次連外部寫入者都沒有。提取這個動作，把你語意網路內生的活化，**寫成**了一段「我看過這個詞」的記憶。沒有壞掉的硬碟、沒有誤導的提問、沒有多嘴的家人。只有重建機制照它的設計正常運作——而它的正常運作，就會生出假記憶。

## 為什麼 DRM 比 ch05、ch06 更傷：三個遞進的點

DRM 不只是「又一個假記憶實驗」。它在三件事上把前兩章往前推了一大步，值得各自講清楚。

**第一，它把「外部污染」這條退路堵死了。** ch05 的誤導動詞、ch06 的家人背書，都是外面寫進來的。一個還想相信記憶的人可以說：「那是因為被人誤導了，沒人誤導我的時候，我的記憶是乾淨的。」DRM 沒有給這句話任何空間。實驗者沒有暗示你看過 `睡覺`，清單裡也沒有 `睡覺`——污染完全是你自己的語意網路內生的。**假記憶是重建機制的內建產物，不是外力造成的故障。** 這是把脊椎從「記憶可被污染」升級成「記憶在沒人碰它的時候也會自己造假」。

**第二，它把「鮮明＝正確」這個直覺正面打穿。** 那 72% 的「記得」判斷不是統計細節，是這一章的核心彈藥。你的內省——「我清楚記得，連細節都記得，所以一定是真的」——在 DRM 裡被當場證偽。一段帶著完整看見體驗的記憶，可以指向一件從沒發生的事。你**沒有**一個內在的真假指示燈。這條直接餵給 ch10：當奈瑟（Neisser）的受試者對挑戰者號當天「鮮明」卻錯誤的記憶極度自信時，背後是同一個機制——鮮明度是「重建得很流暢」的副產物，不是「讀到了高保真原檔」的證明。

**第三，它便宜、乾淨、可大量複製——這在記憶科學裡是稀缺品。** 商場走失（ch06）一次只能植入幾個人，要家人配合、要好幾週、有倫理重量。DRM 在實驗室裡幾分鐘就能在一整班學生身上穩定重現。這讓它成為研究假記憶機制的「果蠅」——你想測什麼變數（分心、警告、睡眠、年齡、藥物）對假記憶的影響，DRM 是最趁手的工具。後面 ch14 會用到一個爆炸性的結果：連超憶症（HSAM）那些「記得每一天」的人，在 DRM 清單上**照樣**誤記關鍵誘餌詞——而能做出那個比較，正因為 DRM 穩定到可以拿任何人來測。

## 模糊痕跡理論：為什麼時間越久、假記憶反而越穩

到這裡你可能有一個合理的反問：**既然假記憶這麼容易生成，那它應該也很脆弱、很快就忘掉吧？** 真記憶都會隨時間衰減（ch03 的遺忘曲線），假記憶總該衰減得更快？

答案出乎意料：**很多情況下，假記憶反而比真記憶更耐久。** 這個反直覺的結果，由一個專門的理論解釋——**模糊痕跡理論（Fuzzy-Trace Theory，布雷納德 Charles Brainerd 與蕾娜 Valerie Reyna 提出）**（2026-06 查證）。

這個理論其實就是 ch02 講過的 **要旨（gist）／逐字（verbatim）** 區分的正式版。它主張記憶在編碼時是**雙軌**的，平行存下兩種痕跡：

- **逐字痕跡（verbatim trace）**：具體的表面細節——你看到的**確切**是哪個詞、什麼字形、排在第幾個、什麼聲音。
- **要旨痕跡（gist trace）**：意義、主題、語意要點——「這串詞都跟睡眠有關」。

兩種痕跡的命運不同，這是關鍵：**逐字痕跡衰減得快，要旨痕跡衰減得慢。** 把它接回你的工程直覺——逐字痕跡像高頻細節（JPEG 一壓就掉的東西），要旨痕跡像輪廓（壓很多次都還在）。

現在看這對 DRM 意味著什麼。**抵抗假記憶的，是逐字痕跡**——只有當你能調出「我確切看過的是 `床、休息、疲倦`，裡面**沒有** `睡覺`」這份逐字細節，你才能否決那個誘餌詞。模糊痕跡理論管這個叫**回想否決（recollection rejection）**：靠逐字記憶把誘餌駁回。**製造假記憶的，是要旨痕跡**——「這串都跟睡眠有關」這個要旨，跟誘餌詞 `睡覺` 完美吻合。

於是時間一拉長：

```text
剛看完              ←  逐字痕跡還在 → 能「回想否決」→ 假記憶較低
                       （「我記得我看到的詞，沒有睡覺」）
過一段時間          ←  逐字痕跡先衰減，要旨痕跡還在 → 否決失效
                       → 只剩「跟睡眠有關」這個要旨 → 誘餌詞 vs 要旨完美吻合
                       → 假記憶反而上升、且比真記憶更穩
```

這是脊椎一個特別漂亮的兌現：**時間不只是讓記憶變淡，它還改變了記憶的成分比例**——先丟掉能保護你的逐字細節，留下會騙你的要旨。一段久遠的回憶之所以「乾淨俐落、主題分明」，正是因為防偽用的細節已經掉光，剩下的是大腦用要旨重建出來的版本。**越久遠、越「清晰」的記憶，反而越可能是要旨主導的重建品。** 這條我們在 ch10（閃光燈，多年後的「鮮明」記憶）和 ch12（法庭上歷時數月的證詞）都會再撞見。

### 一個會嚇到你的推論：成人比兒童更容易產生假記憶

模糊痕跡理論最反直覺、也是 landscape 特別要我查證的一個推論：**在 DRM 這類典範裡，成人的假記憶率反而高於兒童。** 這叫 **發展逆轉（developmental reversal）**（2026-06 查證）。

你的直覺八成跟所有人一樣：大人記性比小孩好、更可靠，怎麼會更容易記錯？但用模糊痕跡理論一推就通。假記憶是**要旨主導**的；要產生「`睡覺` 跟這串詞是一夥的」這個誤認，你得先有一個夠成熟的語意網路，能自動把 `床、休息、疲倦` 抽象成「睡眠」這個主題。**小孩的語意基模還沒長好**——他比較像逐字地記下一個個具體的詞，沒那麼會「抓主題」，於是那道把誘餌詞活化起來的擴散活化反而比較弱。成人的語意網路發達、抓要旨抓得又快又自動——這在絕大多數場合是優勢，但在 DRM 裡，它正是讓你「記得」一個沒出現的詞的元兇。

這個效應有多穩？我查證到的數字（2026-06）：自 2002 年以來，這個「假記憶從兒童到成人不降反升」的發展逆轉，**在 55 個 DRM 實驗中有 53 個複製成功**（Brainerd 等人，2011 的回顧）。這不是一個搖搖晃晃的小發現，是一個結結實實的型態。

把它收進脊椎，這句話值得你抄下來：**你的語意網路越成熟，你越會抓要旨——而抓要旨，正是重建造假的引擎。** 假記憶不是大腦壞了，是大腦**太會歸納**了。一個更會把零碎輸入抽象成意義的系統，就是一個更會用意義去填補、改寫細節的系統。這跟你做系統設計時的取捨一模一樣：一個會 prefetch、會推測、會用 schema 補欄位的系統跑得更快、更「聰明」，代價就是它更容易把猜的當成真的。**記憶選了速度與意義，犧牲了保真——這不是 bug，這是設計取向。**

## 直覺的陷阱

DRM 這個主題迷思密度極高，因為它直接撞上每個人最頑固的內省直覺：「我記得就是我記得」。逐一拆解。

| 迷思／錯誤直覺 | 錯在哪一步 | 正解（怎麼自我察覺） |
|---|---|---|
| **「假記憶都是被別人誤導出來的，沒人騙我就不會有」** | 把假記憶的成因鎖死在外部寫入。DRM 整個存在的意義就是證偽這句——清單裡沒有任何誤導，誘餌詞純粹是你自己的語意網路活化出來的 | ✅ 正解：假記憶是重建機制的**內生**產物。徵兆——你發現自己堅信某事「沒人提過、我自己想起來的」，反而要更小心：自發≠可靠 |
| **「我『記得』那個細節，連聲音/位置都記得，所以一定是真的」** | 把鮮明度當成正確性的代理。DRM 裡約 72% 的假記憶被判為「記得」（帶完整看見體驗），鮮明卻指向沒發生的事 | ✅ 正解：你**沒有**內在的真假指示燈。鮮明是「重建得流暢」的副產物，不是「讀到原檔」的證明（ch10 放大） |
| **「假記憶很脆弱，過一陣子就忘了」** | 假設假記憶跟真記憶一樣衰減、甚至更快。其實假記憶是**要旨主導**的，而要旨衰減慢——逐字細節先掉，保護你的「回想否決」失效，假記憶反而更耐久 | ✅ 正解：時間久了、記憶「越清晰俐落」反而越可疑（要旨重建品）。徵兆——一段久遠記憶細節異常乾淨工整，留個問號 |
| **「大人記性比小孩可靠，所以小孩才會記錯」** | 把「可靠」當成單一維度。在 DRM 裡發展逆轉——成人假記憶率**高於**兒童（53/55 實驗複製），因為成人語意基模更發達、更會抓要旨 | ✅ 正解：成熟的歸納能力是雙面刃。越會抓主題＝越會用主題造假。別把「我是大人我可靠」當免疫證明 |
| **「事先警告我『有陷阱詞』，我就不會中招了」** | 高估了監控階段的攔截力。活化在編碼時就自動發生、擋不住；警告只有在**讀清單之前**給才稍有用，等到回憶前才警告幾乎無效 | ✅ 正解：你攔不住一個已經被點亮的節點。知道有陷阱，跟感覺得出哪個記憶是假的，是兩回事 |
| **「DRM 的誤記率就是 X%，記住這個數字」** | 想要一個乾淨的單一百分比。誤記率**依清單、依測驗方式**變動極大（自由回憶約 40–55%、再認可達 80%+） | ✅ 正解：記**量級與範圍**，不記單一數字。任何把 DRM 寫成「誤記率＝某個確切百分比」的科普，都簡化過頭了 |

這張表裡最該記住的一條，是第二條與第三條的合體：**鮮明不可信，而且久遠的鮮明更不可信。** 這正是 DRM 把你交給 Part III 的那把鑰匙。

## 紙上推演

每題標難度與時間，做完再看解答。

### 推演一：自己設計一串 6 詞的 DRM 清單 ★ [15 分鐘]

選一個你心裡覺得「語意上有一票鄰居」的詞當**關鍵誘餌詞**（例如 `醫院`、`憤怒`、`甜`），然後寫出 6 個強烈指向它、但**都不是它本身**的詞。寫完後問自己：哪幾個鄰居「指向力道」最強？如果把這串唸給一個沒看過原詞的人聽再叫他回憶，你預測他誤記出你那個誘餌詞的機率，大概落在什麼量級？為什麼是這個量級而非另一個？

#### 推演解答

關鍵不在你選了哪 6 個詞，而在你**怎麼判斷哪串會更容易騙人**。真正決定誤記率高低的，是這 6 個詞**對誘餌詞的聯想強度有多集中**——它們是不是「全部、強烈、單一地」指向同一個中心。

以 `甜` 為例，一串強的清單可能是 `糖、酸、苦、好、嚐、巧克力`。注意 `酸、苦` 也很強——它們不是指向 `甜` 的同義，而是 `甜` 的反義／共現詞，但在語意網路裡反義詞彼此高度連結，照樣會把 `甜` 活化起來。一串弱的清單可能是 `蛋糕、笑、生日、禮物、夏天、朋友`——這些詞各自有自己一票鄰居（`生日` 還連到 `蠟燭、派對`），活化被**分散**到太多方向，匯流到 `甜` 的那一份就稀了。

量級的判斷：照本章的範圍與 hedge，自由回憶下「半數上下」是個合理的預期，再認會更高；但**真實數字依清單而異**，你光憑直覺定不出精確百分比——這正是 landscape 要你別咬死單一數字的原因。你能可靠預測的是**相對強弱**（哪串比哪串更會騙人），不是絕對值。能做到這層，你就抓到 DRM 設計的核心了：**假記憶的強度，是被你設計進清單的「聯想匯流度」決定的。**

### 推演二：把一句日常宣稱翻成機制鏈 ★★ [20 分鐘]

把這句話——**「我明明清楚記得他在那場會議上說過『這個方案一定會失敗』」**——翻成一條「這句話可能是內生假記憶」的機制鏈。要求：用上「要旨／逐字」「擴散活化」「來源監控」三個概念，並標出這條鏈在哪一步最可能出錯。順便回答：你要怎麼分辨他「真的說過這句逐字台詞」還是「你用要旨重建出了這句台詞」？

#### 推演解答

機制鏈（一步步走，不跳）：

```text
① 會議當下：你編碼的是「要旨」          ← 他反對方案、語氣悲觀、態度強硬
   （ch02：編碼有損，保要旨丟逐字）         確切的逐字台詞多半沒被完整編碼
② 他真實說的話（如「我對這個方案沒信心」）  ← 在語意網路裡，跟「一定會失敗」高度相鄰
   會擴散活化到鄰近的、更強烈的表述          這就是 DRM 的擴散活化，只是發生在句子層級
③ 過一段時間：逐字痕跡先衰減              ← 你不再記得他「確切」的措辭
   （模糊痕跡：逐字衰減快、要旨衰減慢）       只剩「他很悲觀、強硬反對」這個要旨
④ 提取時：大腦用要旨重建一句具體台詞      ← 從「強硬悲觀反對」這個要旨，
   （ch02：細節是提取當下現編的）            生成一句符合要旨的、更戲劇化的逐字句子
⑤ 來源監控失敗（最可能出錯的一步）        ← 這句重建出來的話很鮮明、很「像他會說的」，
   把「我重建的」誤標成「我聽到的」          大腦用「夠鮮明＝真聽過」判它為真，附上看見體驗
```

最可能出錯的是**第 5 步**——來源監控（ch13 給正式框架）。因為前面四步（編碼丟逐字、活化鄰近強表述、要旨主導重建）都是記憶系統的**正常運作**，沒有一步是故障；唯一決定你會不會被騙的，是你能不能正確判斷「這句鮮明的台詞，來源是我的耳朵還是我的重建器」。而這個判斷靠的是會出錯的啟發式。

怎麼分辨？誠實的答案是：**單憑你自己的內省，分不出來。** 你沒有逐字痕跡可以回想否決（它早衰減了），你只有一句鮮明的重建品和一個「感覺很真」的判斷——而 DRM 已經證明這個判斷不可靠。真正能分辨的只有**外部 provenance**：會議錄影、逐字記錄、當場的筆記。這正是脊椎在日常的應用——記憶沒有 audit log，要查證一段「言之鑿鑿」的回憶，你需要的是系統外的不可變記錄，不是更努力地回想。

### 推演三：DRM 和 ch05 的誤導訊息差在哪一個關鍵點 ★★ [15 分鐘]

ch05 的碎玻璃實驗和本章的 DRM，都造出了「記得一個沒發生的東西」的假記憶。把它們並排，找出**最關鍵的那一個差異**，並說明為什麼這個差異讓 DRM 在「打擊記憶可靠性」這件事上比誤導訊息更狠。

#### 推演解答

| | ch05 誤導訊息（碎玻璃） | ch07 DRM（睡覺） |
|---|---|---|
| 假記憶的**寫入源** | **外部**——實驗者用 `smashed` 這個動詞把「碎玻璃」暗示進來 | **內生**——沒人提過 `睡覺`，是你自己的語意網路活化出來的 |
| 退路 | 「是被人誤導的，沒人騙我就不會錯」這句話**還能講** | 這句話**講不了**——污染源就是你自己 |
| 脊椎版本 | read 夾帶了一個**外部**的值，寫回 source of truth | read **自己**生成了一個值，寫回 source of truth |

最關鍵的差異是**寫入源**：外部 vs 內生。為什麼這讓 DRM 更狠？因為它拆掉了最後一條心理退路。面對誤導訊息，一個還想信任記憶的人可以說「那是因為環境有污染源，我只要待在乾淨的環境、不被誘導，我的記憶就可靠」。DRM 證明這個「乾淨環境」不存在——**只要你有一個正常運作的語意網路，污染源就內建在你腦子裡，不需要任何外部輸入。** 誤導訊息說「你的記憶可以被別人改」；DRM 說「你的記憶會自己改自己」。後者沒有任何防禦姿態能擋——你沒辦法搬離自己的大腦。

值得補一句的細節：誤導訊息效應裡，到底是原記憶被**覆寫**了，還是原記憶**還在、只是來源混淆讓你提取錯**，本身是個老辯論（McCloskey & Zaragoza 1985 的「共存說」挑戰 Loftus 的「覆寫說」）——這個我們留到 ch13 誠實處理，這裡只點出：脊椎的「write」未必是字面的覆寫，也可能是「新痕跡共存＋來源推錯」。DRM 反而是偏向後者的乾淨例子：`睡覺` 這個誘餌的活化，和真實清單詞的痕跡是**共存**的，你只是分不出哪個是哪個。

## 自我檢核

口頭自答，講得出來才算過。優先答「為什麼」。

1. DRM 典範的清單裡**有沒有**誤導訊息？沒有的話，那個「記得看過的」誘餌詞是哪來的？為什麼這比 ch05、ch06 的假記憶更傷？
2. 為什麼一個**從沒出現**的詞，在你腦中的活化，可以**強過**一個你真的看過的詞？走一遍那條活化鏈。
3. 「記得 / 知道」判斷是用來分辨什麼的？DRM 的假記憶大量落在哪一邊（約多少比例），這為什麼是這一章最重要的數字？
4. 為什麼說假記憶常常比真記憶**更耐久**？用「逐字／要旨」「回想否決」解釋，並說出時間在這裡扮演什麼角色。
5. 「發展逆轉」是什麼？為什麼**成人**反而比兒童更容易在 DRM 裡記錯？這對你「大人記性比較可靠」的直覺意味著什麼？
6. 事先警告受試者「會有陷阱詞」，為什麼幫助有限？什麼時機給警告才稍微有用，為什麼？
7. 為什麼 landscape 不准我們把 DRM 寫成「誤記率＝某個確切百分比」？影響這個比率的兩個主要變數是什麼？
8. 把 DRM 接回脊椎：當你「記得看過睡覺」時，這是 read 還是 read 觸發了 write？這次的 write 跟 ch05 的 write 差在哪？
