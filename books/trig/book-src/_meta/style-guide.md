## 這本書是什麼、不是什麼

- **是**：一本「欣賞與直覺」的書。核心主張只有一句：**三角函數不是三角形的學問，是旋轉與週期的語言**。三角形是它的搖籃，單位圓才是它的家；sin 與 cos 是一個點繞圓時投在牆上的影子。讀完要能用自己的話對另一個工程師講清楚：為什麼「量三角形」的工具會變成「描述一切會轉、會週期重複的東西」的通用語——從旋轉、複數、波，一路到傅立葉的門口。
- **不是**：
  - 不是解題訓練——不刷「解三角形」題、不背恆等式表、不出考試操練題。恆等式一律**從來源推**，不從記憶抄（這是全書的招牌姿態）。
  - 不是嚴格分析教科書——收斂、極限的 ε-δ 嚴格層交給姊妹書《馴服無限》或延伸閱讀；本書講到「直覺夠穩、能自我複核」為止，並誠實標示哪裡沒證。
  - 不是轉職／工作用書——**純興趣，和工作無關**（2026-06-13 訪談定案）。不服務任何 AI Infra 敘事。但工程橋接要讓它讀起來像母語，不是外語：用讀者熟悉的東西（螢幕旋轉、遊戲 sprite 轉向、cosine 相似度、訊號週期、atan2）當**直覺錨點**，不是當履歷素材。
- **驗收標準（全書六條主線，ch15 總收官逐條口試）**：
  1. 三角函數真正在描述什麼（旋轉與週期），三角形為什麼只是搖籃；
  2. 單位圓為什麼是「家」、弧度為什麼是「自然」的角度單位；
  3. 為什麼**恆等式都有來源**（和角公式 = 旋轉相疊），不必背；
  4. 複數乘法 = 旋轉加縮放，Euler 公式 e^{iθ}=cosθ+i·sinθ 從旋轉這一側看是什麼意思；
  5. 一個波的解剖（振幅、頻率、相位）與「sin′=cos 因為旋轉的速度還是旋轉」；
  6. 為什麼「任何週期訊號 = 一堆旋轉的圓疊起來」（傅立葉的門口），以及這在你的技術棧裡的藏身處。

## 圖片機制（本書與姊妹書最大的不同——必須嚴格遵守）

本書**每章至少一張**用程式產生的圖（旋轉、單位圓、相量、Lissajous、單位根 n 邊形、方波疊加…ASCII 畫不出來的東西）。機制如下，違反會讓建置壞掉或破壞單檔離線：

1. **每張圖由一段 Python 腳本產生**，存在 `book-src/figures/`，檔名 `chNN-slug-figname.py`（英文、kebab-case）。腳本用 `numpy` + `matplotlib`，**英文註解**，**≤ 40 行**，**確定性輸出**（用到隨機必須 `np.random.seed(0)`）。
2. 腳本把圖存到 `book-src/figures/out/chNN-slug-figname.svg`（線圖優先 SVG：清晰、檔小；需要點陣或填色漸層時才用 `.png`，`dpi=150`）。**輸出路徑必須由 `__file__` 推導（與 cwd 無關），用以下固定樣板**（複製這個骨架，不要自創存檔路徑）：
   ```python
   # ch0X figure: <one-line English description>
   from pathlib import Path
   import numpy as np
   import matplotlib
   matplotlib.use("Agg")          # headless; no display needed
   import matplotlib.pyplot as plt

   OUT = Path(__file__).resolve().parent / "out" / "ch0X-slug-figname.svg"
   OUT.parent.mkdir(parents=True, exist_ok=True)

   fig, ax = plt.subplots(figsize=(5, 5))
   # ... draw here; ALL text labels in English/math notation only ...
   ax.set_aspect("equal")         # for circle/rotation figures, keep it round
   fig.savefig(OUT, bbox_inches="tight")
   print("wrote", OUT)            # build_figures.py reads this
   ```
3. **圖內所有文字（座標軸標籤、圖例、標註）一律英文／數學記號**（`θ`、`sin x`、`Re`、`Im`、`x`、`t`）。原因：matplotlib 預設字型對中文是豆腐框 □□□，且這符合「程式與標籤一律英文」的慣例。**不要**在圖裡放中文。
4. **markdown 裡用標準圖片語法引用相對路徑**：`![繁中說明文字（caption）](figures/out/chNN-slug-figname.svg)`。路徑相對於 `book-src/`（.md 檔所在目錄）。建置時 `build_reader.py` 會把本地圖檔讀進來、base64 內嵌進輸出 HTML，所以 **source 乾淨可審、輸出仍是單檔離線**。caption（繁中）寫在 alt 文字裡，對讀者說明這張圖在看什麼、看點在哪。
5. **這段腳本同時就是該章的 Python 小實驗**（動手做）：在 `## 紙上推演` 的 `### 動手生圖` 小節裡，把腳本完整列出（```python 區塊），附「預期輸出」與「改哪個參數會看到什麼」的引導，讓讀者能跑、能改、能重生那張圖。不要另外再編一個無關的實驗——圖就是實驗。
6. 結構性 ASCII 圖（全書路線圖、單位圓示意的純文字版若有）仍要過 `check_diagrams.py`（CJK=2 欄寬）；用程式生的圖不受 check_diagrams 管，但**markdown 引用的相對路徑必須真的有對應檔案**（建置時缺檔會 warn——視為錯誤）。
7. 章節 agent 負責**寫腳本**並在 markdown 放好引用；**不需要自己跑出圖**（協調者會在 P2 之後統一跑 `build_figures.py` 產生所有 out/ 圖檔再打包）。但 agent 必須在心裡（或用算式）確認腳本畫出來的東西在數學上正確。
